= Fügen Sie ein Volume hinzu
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Sie können ein Volume erstellen und es einer vorhandenen Storage-VM hinzufügen, die für den NFS- oder SMB-Dienst konfiguriert ist.

image:workflow_admin_add_a_volume.gif["Schritte zum Hinzufügen eines Volumes"]

.Bevor Sie beginnen
* Im Cluster sollte eine Storage-VM vorhanden sein, die für den NFS- oder SMB-Dienst konfiguriert ist.
* Ab ONTAP 9.13.1 ist es möglich, Kapazitätsanalysen und Aktivitätsverfolgung standardmäßig für neue Volumes zu aktivieren. Im System Manager lassen sich die Standardeinstellungen auf Cluster- oder Storage-VM-Ebene verwalten. Weitere Informationen siehe  https://docs.netapp.com/us-en/ontap/task_nas_file_system_analytics_enable.html["Dateisystemanalyse aktivieren"].


.Schritte
. Zu *Speicher > Volumes* wechseln.
. Wählen Sie image:icon_add.gif["Symbol hinzufügen"].
. Einen Namen und eine Größe für das Volume angeben.
. Einer der folgenden Schritte kann durchgeführt werden:
+
[cols="35,65"]
|===


| Diese Schaltfläche auswählen... | Um diese Aktion durchzuführen... 


| *Speichern* | Das Volume wird mit den Systemvorgaben erstellt und hinzugefügt. Es sind keine weiteren Schritte erforderlich. 


| *Weitere Optionen* | Fahren Sie mit <<step5>> fort, um die Spezifikationen für das Volume festzulegen. 
|===
. [[step5,Schritt 5]] Der Name und die Größe des Volumes werden angezeigt, sofern Sie diese zuvor angegeben haben. Andernfalls sind Name und Größe einzugeben.
. Eine Storage-VM aus der Dropdown-Liste auswählen.
+
Es werden nur Speicher-VMs aufgelistet, die mit dem NFS-Protokoll konfiguriert sind. Wenn nur eine Speicher-VM mit dem NFS-Protokoll verfügbar ist, wird das Feld *Storage VM* nicht angezeigt.

. Um einen Cache für das Remote-Volume hinzuzufügen, *Cache für Remote-Volume hinzufügen* auswählen und die folgenden Werte angeben:
+
** Einen Cluster auswählen.
** Eine Storage-VM auswählen.
** Wählen Sie das Volume aus, das als Cache-Volume dienen soll.


. Im Abschnitt *Speicherung und Optimierung* sind die folgenden Werte anzugeben:
+
.. Die Kapazität des Volumens wird bereits angezeigt, Sie können sie jedoch ändern.
.. Im Feld *Performance Service Level* wird ein Servicelevel ausgewählt:
+
[cols="60,40"]
|===


| Wenn diese Servicestufe ausgewählt wird... | Dies geschieht... 


 a| 
Ein bestehendes Servicelevel, wie „Extreme“, „Performance“ oder „Value“.

Es werden nur die Servicelevel angezeigt, die für die Systemplattform (AFF, FAS oder andere) gültig sind.
| Eine lokale Tier oder mehrere Tiers werden automatisch ausgewählt. Weiter zu <<step9>>. 


| Benutzerdefiniert | Weiter zu <<step8c>> zur Definition eines neuen Servicelevels. 
|===
.. [[step8c, Schritt 8c]] Ab ONTAP 9.9.1 kann im System Manager manuell die lokale Ebene ausgewählt werden, auf der das Volume platziert werden soll, das erstellt wird (wenn die Serviceebene „Benutzerdefiniert“ ausgewählt wurde).
+

NOTE: Diese Option ist nicht verfügbar, wenn *Als Cache für ein Remote-Volume hinzufügen* oder *Volume-Daten im Cluster verteilen* ausgewählt wird (siehe unten).

+
|===


| Wenn diese Entscheidung getroffen wird... | Sie führen folgende Schritte aus... 


| *Manuelle Platzierung* | Die manuelle Platzierung ist aktiviert. Die Auswahl *Volume-Daten im Cluster verteilen* ist deaktiviert (siehe unten). Weiter zu <<step8d>> , um den Vorgang abzuschließen. 


| Keine Auswahl | Die manuelle Platzierung ist nicht aktiviert. Die lokale Ebene wird automatisch ausgewählt. Weiter zu <<step9>>. 
|===
.. [[step8d,Schritt 8d]] Eine lokale Stufe aus dem Dropdown-Menü auswählen.
.. Eine QoS-Richtlinie auswählen.
+
„Vorhanden“ ermöglicht die Auswahl aus einer Liste bestehender Richtlinien, während „Neu“ die Eingabe der Spezifikationen einer neuen Richtlinie erlaubt.



. [[step9,Schritt 9]] Im Abschnitt *Optimierungsoptionen* wird festgelegt, ob die Volumendaten über den Cluster verteilt werden sollen:
+
[cols="40,60"]
|===


| Wenn diese Entscheidung getroffen wird... | Dies geschieht... 


| *Volumendaten im Cluster verteilen* | Das hinzugefügte Volumen wird zu einem FlexGroup volume. Diese Option ist nicht verfügbar, wenn zuvor *Manual placement* ausgewählt wurde. 


| Keine Auswahl | Das Volume, das Sie hinzufügen, wird standardmäßig zu einem FlexVol Volume. 
|===
. Im Abschnitt *Zugriffsberechtigungen* sind die Zugriffsberechtigungen für die Protokolle anzugeben, für die das Volume konfiguriert ist.
+
Ab ONTAP 9.11.1 ist das neue Volume nicht standardmäßig freigegeben. Die Standardzugriffsberechtigungen können festgelegt werden, indem sichergestellt wird, dass die folgenden Kontrollkästchen aktiviert sind:

+
** *Export via NGS*: Erstellt das Volume mit der `default` Exportrichtlinie, die Benutzern vollen Zugriff auf die Daten gewährt.
** *Freigabe über SMB/CIFS*: Erstellt eine Freigabe mit einem automatisch generierten Namen, den Sie bearbeiten können. Der Zugriff wird gewährt an `Everyone`. Außerdem kann die Berechtigungsstufe angegeben werden.


. Im Abschnitt *Schutz* sind die Schutzmaßnahmen für das Volume anzugeben.
+
** Ab ONTAP 9.12.1 kann *Snapshot aktivieren (lokal)* ausgewählt und eine Snapshot-Richtlinie statt der Standardeinstellung gewählt werden.
** Wenn *SnapMirror (Lokal oder Remote) aktivieren* ausgewählt wird, werden die Datensicherungsstrategie und die Einstellungen für den Ziel-Cluster aus den Dropdown-Listen angegeben.


. *Speichern* auswählen.
+
Das Volume wird erstellt und dem Cluster sowie der Storage-VM hinzugefügt.

+

NOTE: Sie können die Spezifikationen dieses Volumes auch in einem Ansible Playbook speichern. Weitere Informationen finden Sie unter link:task_use_ansible_playbooks_add_edit_volumes_luns.html["Ansible Playbooks können verwendet werden, um Volumes oder LUNs hinzuzufügen oder zu bearbeiten."].
