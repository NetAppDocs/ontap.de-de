= Das Arbeitsblatt zur Volumenkonfiguration ausfüllen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Arbeitsblatt bietet eine einfache Möglichkeit, die Werte zu erfassen, die Sie beim Erstellen von Volumes für SQL Server und Hyper-V über SMB Konfigurationen benötigen.

Für jeden Band müssen Sie folgende Informationen angeben:

* Name der Storage Virtual Machine (SVM)
+
Der SVM Name ist für alle Volumes gleich.

* Volumenname
* Aggregatname
+
Volumes können auf Aggregaten erstellt werden, die sich auf einem beliebigen Knoten im Cluster befinden.

* Größe
* Verbindungspfad


Beim Erstellen von Volumes zur Speicherung von Anwendungsserverdaten ist Folgendes zu beachten:

* Wenn das Stammvolume nicht den NTFS-Sicherheitsstil aufweist, muss beim Erstellen des Volumes der Sicherheitsstil als NTFS angegeben werden.
+
Standardmäßig erben Volumes den Sicherheitsstil des SVM Root-Volumes.

* Volumes sollten mit der standardmäßigen Speicherplatzzusage konfiguriert werden.
* Optional kann die Einstellung für das automatische Größenanpassung Speicherplatzmanagement konfiguriert werden.
* Sie sollten die Option, die den Speicherplatzbedarf für Snapshots bestimmt, auf  `0` setzen.
* Die auf das Volume angewendete Snapshot-Richtlinie muss deaktiviert sein.
+
Wenn die SVM-Snapshotrichtlinie deaktiviert ist, müssen Sie keine Snapshotrichtlinie für die Volumes angeben. Die Volumes übernehmen die Snapshotrichtlinie der SVM. Wenn die Snapshotrichtlinie der SVM nicht deaktiviert ist und für die Erstellung von Snapshots konfiguriert ist, muss auf Volume-Ebene eine Snapshotrichtlinie angegeben werden, und diese Richtlinie muss deaktiviert sein. Die Erstellung und das Löschen von Snapshots werden durch Backups mit aktiviertem Shadow Copy Service und durch SQL Server-Backups gesteuert.

* Es ist nicht möglich, Load-Sharing-Mirrors für die Volumes zu konfigurieren.


Die Verbindungspfade, auf denen Freigaben erstellt werden sollen, die von den Anwendungsservern verwendet werden, sollten so gewählt werden, dass sich unterhalb des Freigabe-Einstiegspunkts keine verbundenen Volumes befinden.

Wenn Sie beispielsweise virtuelle Maschinendateien auf vier Volumes mit den Namen „vol1“, „vol2“, „vol3“ und „vol4“ speichern möchten, kann der im Beispiel gezeigte Namespace erstellt werden. Anschließend können Freigaben für die Anwendungsserver unter den folgenden Pfaden erstellt werden:  `/data1/vol1`, `/data1/vol2`, `/data2/vol3` und  `/data2/vol4`.

[listing]
----

                     Junction                       Junction
Vserver Volume       Active   Junction Path         Path Source
------- ------------ -------- -------------------   -----------
vs1     data1         true     /data1               RW_volume
vs1     vol1          true     /data1/vol1          RW_volume
vs1     vol2          true     /data1/vol2          RW_volume
vs1     data2         true     /data2               RW_volume
vs1     vol3          true     /data2/vol3          RW_volume
vs1     vol4          true     /data2/vol4          RW_volume
----
