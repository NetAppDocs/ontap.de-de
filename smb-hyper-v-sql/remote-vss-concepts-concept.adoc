= Remote VSS-Konzepte
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bestimmte Konzepte sind erforderlich, um zu verstehen, wie Remote VSS (Volume Shadow Copy Service) von Sicherungsdiensten mit Hyper-V over SMB Konfigurationen verwendet wird.

* *VSS (Volume Shadow Copy Service)*
+
Eine Microsoft-Technologie, die Sicherungskopien oder Momentaufnahmen von Daten auf einem bestimmten Volume zu einem bestimmten Zeitpunkt erstellt. VSS koordiniert zwischen Datenservern, Sicherungsanwendungen und Speichermanagementsoftware, um die Erstellung und Verwaltung konsistenter Backups zu ermöglichen.

* *Remote VSS (Remote Volume Shadow Copy Service)*
+
Eine Microsoft-Technologie, die zum Erstellen freigabebasierter Backup-Kopien von Daten verwendet wird, die sich in einem zeitpunktgenauen, datenkonsistenten Zustand befinden, wobei auf die Daten über SMB-3.0-Freigaben zugegriffen wird. Auch bekannt als _Volume Shadow Copy Service_.

* *Schattenkopie*
+
Ein Duplikat der in der Freigabe enthaltenen Daten zu einem genau definierten Zeitpunkt. Schattenkopien werden verwendet, um konsistente zeitpunktgenaue Backups der Daten zu erstellen, sodass das System oder Anwendungen die Daten auf den Original-Volumes weiterhin aktualisieren können.

* *Schattenkopiesatz*
+
Eine Sammlung von einer oder mehreren Schattenkopien, wobei jede Schattenkopie einer Freigabe entspricht. Die Schattenkopien innerhalb eines Schattenkopiesatzes repräsentieren alle Freigaben, die im selben Vorgang gesichert werden müssen. Der VSS-Client auf der VSS-fähigen Anwendung identifiziert, welche Schattenkopien in den Satz aufgenommen werden.

* *Automatische Wiederherstellung des Schattenkopie-Sets*
+
Der Teil des Sicherungsprozesses für Remote-VSS-fähige Sicherungsanwendungen, in dem das Replikatverzeichnis mit den Schattenkopien zeitpunktgenau konsistent gemacht wird. Zu Beginn der Sicherung veranlasst der VSS-Client auf der Anwendung, dass die Anwendung Software-Checkpoints der für die Sicherung vorgesehenen Daten (im Fall von Hyper-V die Dateien der virtuellen Maschine) erstellt. Der VSS-Client ermöglicht anschließend den Anwendungen die Fortsetzung. Nachdem der Schattenkopiesatz erstellt wurde, macht Remote VSS den Schattenkopiesatz beschreibbar und stellt die beschreibbare Kopie den Anwendungen zur Verfügung. Die Anwendung bereitet den Schattenkopiesatz für die Sicherung vor, indem sie eine automatische Wiederherstellung mithilfe des zuvor erstellten Software-Checkpoints durchführt. Die automatische Wiederherstellung bringt die Schattenkopien in einen konsistenten Zustand, indem die seit der Erstellung des Checkpoints vorgenommenen Änderungen an Dateien und Verzeichnissen rückgängig gemacht werden. Die automatische Wiederherstellung ist ein optionaler Schritt für VSS-fähige Sicherungen.

* *Schattenkopie-ID*
+
Eine GUID, die eine Schattenkopie eindeutig identifiziert.

* *Schattenkopiesatz-ID*
+
Eine GUID, die eine Sammlung von Schattenkopie-IDs auf demselben Server eindeutig identifiziert.

* *SnapManager for Hyper-V*
+
Die Software automatisiert und vereinfacht Sicherungs- und Wiederherstellungsvorgänge für Microsoft Windows Server 2012 Hyper-V. SnapManager for Hyper-V verwendet Remote VSS mit automatischer Wiederherstellung, um Hyper-V-Dateien über SMB-Freigaben zu sichern.
