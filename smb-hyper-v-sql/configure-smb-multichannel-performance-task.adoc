= SMB Multichannel für optimale Leistung und Redundanz konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.4 kann SMB Multichannel so konfiguriert werden, dass mehrere Verbindungen zwischen ONTAP und Clients in einer einzigen SMB-Sitzung bereitgestellt werden. Dadurch verbessern sich Durchsatz und Fehlertoleranz für Hyper-V und SQL Server over SMB Konfigurationen.

.Bevor Sie beginnen
Die SMB Multichannel-Funktionalität steht nur zur Verfügung, wenn Clients SMB 3.0 oder eine höhere Version aushandeln. SMB 3.0 und höher ist auf dem ONTAP SMB-Server standardmäßig aktiviert.

.Über diese Aufgabe
SMB-Clients erkennen und nutzen automatisch mehrere Netzwerkverbindungen, wenn eine geeignete Konfiguration im ONTAP Cluster vorhanden ist.

Die Anzahl der gleichzeitigen Verbindungen in einer SMB-Sitzung hängt von den eingesetzten Netzwerkkarten (NICs) ab:

* *1G-NICs auf Client- und ONTAP-Cluster*
+
Der Client stellt pro Netzwerkkarte eine Verbindung her und bindet die Sitzung an alle Verbindungen.

* *10G Netzwerkkarten und Netzwerkkarten mit größerer Kapazität auf Client- und ONTAP Cluster*
+
Der Client stellt bis zu vier Verbindungen pro Netzwerkkarte her und bindet die Sitzung an alle Verbindungen. Der Client kann Verbindungen über mehrere 10G Netzwerkkarten und Netzwerkkarten mit größerer Kapazität herstellen.



Sie können außerdem die folgenden Parameter ändern (erweiterte Berechtigung):

* `-max-connections-per-session`
+
Die maximal zulässige Anzahl von Verbindungen pro Multichannel-Sitzung. Der Standardwert beträgt 32 Verbindungen.

+
Wenn mehr Verbindungen als die Standardeinstellung aktiviert werden sollen, sind entsprechende Anpassungen an der Clientkonfiguration erforderlich, die ebenfalls standardmäßig 32 Verbindungen vorsieht.

* `-max-lifs-per-session`
+
Die maximale Anzahl der pro Multichannel-Sitzung angekündigten Netzwerkschnittstellen. Der Standardwert beträgt 256 Netzwerkschnittstellen.



.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[listing]
----
set -privilege advanced
----
. SMB Multichannel auf dem SMB-Server aktivieren:
+
[listing]
----
vserver cifs options modify -vserver <vserver_name> -is-multichannel-enabled true
----
. Es sollte überprüft werden, ob ONTAP SMB Multichannel-Sitzungen meldet:
+
[listing]
----
vserver cifs session show
----
. Zurück zur Administrator-Privilegstufe:
+
[listing]
----
set -privilege admin
----
