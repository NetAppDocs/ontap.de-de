= Was das Witness-Protokoll zur Verbesserung des transparenten Failovers leistet
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Witness-Protokoll bietet erweiterte Client-Failover-Funktionen für SMB 3.0 Continuous Available Shares (CA Shares). Witness ermöglicht ein schnelleres Failover, da die LIF-Failover-Wiederherstellungsphase umgangen wird. Anwendungsserver werden benachrichtigt, wenn ein Knoten nicht verfügbar ist, ohne dass auf das Timeout der SMB 3.0-Verbindung gewartet werden muss.

Der Failover erfolgt nahtlos, wobei die auf dem Client laufenden Anwendungen nicht bemerken, dass ein Failover stattgefunden hat. Wenn Witness nicht verfügbar ist, erfolgen Failover-Vorgänge weiterhin erfolgreich, jedoch ist ein Failover ohne Witness weniger effizient.

Ein verbessertes Failover durch Zeugen ist möglich, wenn die folgenden Voraussetzungen erfüllt sind:

* Es kann nur mit SMB 3.0-fähigen CIFS-Servern verwendet werden, auf denen SMB 3.0 aktiviert ist.
* Die Freigaben müssen SMB 3.0 mit der Eigenschaft „Kontinuierliche Verfügbarkeit“ für die Freigabe verwenden.
* Der SFO-Partner des Knotens, mit dem die Anwendungsserver verbunden sind, muss mindestens eine betriebsbereite Daten-LIF haben, die der Storage Virtual Machine (SVM) zugewiesen ist, die Daten für die Anwendungsserver hostet.
+
[NOTE]
====
Das Witness-Protokoll funktioniert zwischen SFO-Paaren. Da LIFs auf jeden beliebigen Knoten innerhalb des Clusters migriert werden können, muss unter Umständen jeder Knoten als Witness für seinen SFO-Partner fungieren. Das Witness-Protokoll kann kein schnelles Failover von SMB-Verbindungen auf einem bestimmten Knoten gewährleisten, wenn die SVM, die die Daten für die Anwendungsserver hostet, keine aktive Daten-LIF auf dem Partnerknoten besitzt. Daher muss jeder Knoten im Cluster mindestens eine Daten-LIF für jede SVM besitzen, die eine dieser Konfigurationen hostet.

====
* Die Anwendungsserver müssen sich mit dem CIFS-Server verbinden, indem sie den im DNS gespeicherten CIFS-Servernamen verwenden, anstatt einzelne LIF-IP-Adressen zu nutzen.
