= Überprüfen, ob die erforderlichen CIFS-Server-Optionen konfiguriert sind
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es ist zu überprüfen, ob die erforderlichen CIFS-Serveroptionen aktiviert und gemäß den Anforderungen für unterbrechungsfreien Betrieb von Hyper-V und SQL Server über SMB konfiguriert sind.

.Über diese Aufgabe
* SMB 2.x und SMB 3.0 müssen aktiviert sein.
* ODX Copy Offload muss aktiviert sein, um die leistungssteigernde Copy Offload-Funktion nutzen zu können.
* Die VSS-Schattenkopie-Dienste müssen aktiviert sein, wenn die Hyper-V over SMB Lösung Remote VSS-fähige Sicherungsdienste verwendet (nur Hyper-V).


.Schritte
. Es ist zu überprüfen, ob die erforderlichen CIFS-Serveroptionen auf der Storage Virtual Machine (SVM) aktiviert sind:
+
.. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`*set -privilege advanced*`

.. Führen Sie den folgenden Befehl aus:
+
`*vserver cifs options show -vserver _vserver_name_*`

+
Die folgenden Optionen sollten auf  `true` gesetzt werden:

+
*** `-smb2-enabled`
*** `-smb3-enabled`
*** `-copy-offload-enabled`
*** `-shadowcopy-enabled` (nur Hyper-V)




. Falls eine der Optionen nicht auf  `true` gesetzt ist, ist Folgendes zu tun:
+
.. Setzen Sie sie auf  `true` mit dem  `vserver cifs options modify` Befehl.
.. Es sollte überprüft werden, ob die Optionen auf  `true` gesetzt sind, indem der  `vserver cifs options show`-Befehl verwendet wird.


. Zurück zur Administrator-Privilegstufe:
+
`*set -privilege admin*`



.Beispiel
Die folgenden Befehle überprüfen, ob die erforderlichen Optionen für die Hyper-V over SMB Konfiguration auf SVM vs1 aktiviert sind. Im Beispiel muss ODX Copy Offload aktiviert sein, um die Optionsanforderungen zu erfüllen.

[listing]
----
cluster1::> set -privilege advanced
Warning: These advanced commands are potentially dangerous; use them
only when directed to do so by technical support personnel.
Do you wish to continue? (y or n): y

cluster1::*> vserver cifs options show -vserver vs1 -fields smb2-enabled,smb3-enabled,copy-offload-enabled,shadowcopy-enabled
vserver smb2-enabled smb3-enabled copy-offload-enabled shadowcopy-enabled
------- ------------ ------------ -------------------- ------------------
vs1     true         true         false                true

cluster-1::*> vserver cifs options modify -vserver vs1 -copy-offload-enabled true

cluster-1::*> vserver cifs options show -vserver vs1 -fields copy-offload-enabled
vserver  copy-offload-enabled
-------- --------------------
vs1      true

cluster1::*> set -privilege admin
----
