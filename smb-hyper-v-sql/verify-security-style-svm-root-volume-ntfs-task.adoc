= Es sollte sichergestellt sein, dass der Sicherheitsstil des SVM Root-Volumes auf NTFS gesetzt ist.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um unterbrechungsfreie Vorgänge für Hyper-V und SQL Server über SMB sicherzustellen, müssen Volumes mit dem NTFS-Sicherheitsstil erstellt werden. Da der Sicherheitsstil des Stammvolumes standardmäßig auf Volumes angewendet wird, die auf der Storage Virtual Machine (SVM) erstellt werden, sollte der Sicherheitsstil des Stammvolumes auf NTFS gesetzt werden.

.Über diese Aufgabe
* Sie können den Sicherheitsstil des Root-Volumes beim Erstellen der SVM festlegen.
* Wenn die SVM nicht mit dem Root-Volume im NTFS-Sicherheitsstil erstellt wurde, kann der Sicherheitsstil später mit dem  `volume modify` Befehl geändert werden.


.Schritte
. Den aktuellen Sicherheitsstil des SVM-Root-Volumes bestimmen:
+
`*volume show -vserver _vserver_name_ -fields vserver,volume,security-style*`

. Wenn das Stammvolume kein NTFS-Sicherheitsstil-Volume ist, den Sicherheitsstil auf NTFS ändern:
+
`*volume modify -vserver _vserver_name_ -volume root_volume_name -security-style ntfs*`

. Es sollte überprüft werden, ob das SVM-Root-Volume auf den NTFS-Sicherheitsstil eingestellt ist:
+
`*volume show -vserver _vserver_name_ -fields vserver,volume,security-style*`



.Beispiel
Die folgenden Befehle überprüfen, ob der Sicherheitsstil des Root-Volumes auf SVM vs1 NTFS ist:

[listing]
----
cluster1::> volume show -vserver vs1 -fields vserver,volume,security-style
vserver  volume     security-style
-------- ---------- --------------
vs1      vs1_root   unix

cluster1::> volume modify -vserver vs1 -volume vs1_root -security-style ntfs

cluster1::> volume show -vserver vs1 -fields vserver,volume,security-style
vserver  volume     security-style
-------- ---------- --------------
vs1      vs1_root   ntfs
----
