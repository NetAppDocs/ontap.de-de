= Empfehlungen für SQL Server und Hyper-V Konfigurationen über SMB
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um sicherzustellen, dass Ihre SQL Server und Hyper-V over SMB Konfigurationen robust und betriebsbereit sind, ist es wichtig, mit den empfohlenen Best Practices für die Konfiguration der Lösungen vertraut zu sein.



[[general-recommendations]]
== Allgemeine Empfehlungen

* Anwendungsserverdateien werden von allgemeinen Benutzerdaten getrennt.
+
Wenn möglich, eine gesamte Storage Virtual Machine (SVM) und deren Speicher für die Daten des Anwendungsservers vorsehen.

* Für optimale Leistung sollte die SMB-Signierung auf SVMs, die zur Speicherung der Daten des Anwendungsservers verwendet werden, nicht aktiviert werden.
* Für optimale Leistung und verbesserte Fehlertoleranz sollte SMB Multichannel aktiviert werden, um mehrere Verbindungen zwischen ONTAP und Clients in einer einzigen SMB-Sitzung bereitzustellen.
* Erstellen Sie keine permanent verfügbaren Freigaben auf anderen Freigaben als denen, die in der Hyper-V oder SQL Server over SMB Konfiguration verwendet werden.
* Die Änderungsbenachrichtigung für Freigaben, die für die kontinuierliche Verfügbarkeit verwendet werden, ist zu deaktivieren.
* Eine Volumenverschiebung sollte nicht gleichzeitig mit einer Aggregatverschiebung (ARL) durchgeführt werden, da ARL Phasen hat, in denen einige Operationen angehalten werden.
* Für Hyper-V over SMB-Lösungen sollten beim Erstellen von Cluster-VMs iSCSI-Laufwerke im Gastsystem verwendet werden. Gemeinsam genutzte  `.VHDX` Dateien werden für Hyper-V over SMB in ONTAP SMB-Freigaben nicht unterstützt.
