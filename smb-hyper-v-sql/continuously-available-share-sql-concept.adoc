= Anforderungen und Überlegungen zur kontinuierlichen Verfügbarkeit von Freigaben für SQL Server über SMB
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei der Konfiguration von permanent verfügbaren Freigaben für SQL Server über SMB-Konfigurationen, die unterbrechungsfreie Vorgänge unterstützen, sind bestimmte Anforderungen und Überlegungen zu berücksichtigen.



[[share-requirements]]
== Freigabeanforderungen

* Volumes, die zur Speicherung von Dateien virtueller Maschinen verwendet werden, müssen als NTFS Security-Style-Volumes erstellt werden.
+
Um unterbrechungsfreie Operationen von Anwendungsservern über permanent verfügbare SMB-Verbindungen bereitzustellen, muss das Volume, das die Freigabe enthält, ein NTFS-Volume sein. Es muss außerdem immer ein NTFS-Volume gewesen sein. Ein Volume mit gemischtem Sicherheitsstil oder UNIX-Sicherheitsstil kann nicht in ein NTFS-Sicherheitsstil-Volume geändert und direkt für unterbrechungsfreie Operationen über SMB-Freigaben verwendet werden. Wenn ein Volume mit gemischtem Sicherheitsstil in ein NTFS-Sicherheitsstil-Volume geändert wird und für unterbrechungsfreie Operationen über SMB-Freigaben genutzt werden soll, ist es erforderlich, manuell eine Zugriffskontrollliste (ACL) am Anfang des Volumes zu platzieren und diese ACL auf alle enthaltenen Dateien und Ordner zu übertragen. Andernfalls können Migrationen virtueller Maschinen oder Datenbankdateiexporte und -importe, bei denen Dateien auf ein anderes Volume verschoben werden, fehlschlagen, wenn entweder das Quell- oder das Zielvolume ursprünglich als Volume mit gemischtem oder UNIX-Sicherheitsstil erstellt und später in den NTFS-Sicherheitsstil geändert wurde.

* Die von den Anwendungsservern verwendeten Shares müssen mit der Eigenschaft „kontinuierlich verfügbar“ konfiguriert sein.
+
Anwendungsserver, die sich mit permanent verfügbaren Freigaben verbinden, erhalten persistente Handles, die es ihnen ermöglichen, sich nach störenden Ereignissen wie Übernahme, Rückgabe und Aggregatverschiebung unterbrechungsfrei wieder mit SMB-Freigaben zu verbinden und Dateisperren zurückzuerlangen.

* Obwohl das Volume, das die Datenbankdateien enthält, Verbindungen enthalten kann, überschreitet SQL Server beim Erstellen der Datenbankverzeichnisstruktur keine Verbindungen.
* Für erfolgreiche SnapCenter Plug-in for Microsoft SQL Server Vorgänge muss ausreichend Speicherplatz auf dem Volume vorhanden sein.
+
Das Volume, auf dem die SQL Server Datenbankdateien gespeichert sind, muss groß genug sein, um die Datenbank Verzeichnisstruktur und alle darin enthaltenen Dateien innerhalb der Freigabe aufzunehmen.

* Die folgenden Freigabeeigenschaften dürfen für kontinuierlich verfügbare Freigaben, die von den Anwendungsservern verwendet werden, nicht festgelegt werden:
+
** Home-Verzeichnis
** Attribut-Caching
** BranchCache






[[share-considerations]]
== Überlegungen zur Freigabe

* Quoten werden auf kontinuierlich verfügbaren Freigaben unterstützt.
* Die folgende Funktionalität wird für SQL Server über SMB Konfigurationen nicht unterstützt:
+
** Prüfung
** FPolicy


* Die Virenprüfung wird bei SMB-Freigaben mit der  `continuously-availability` Freigabeeigenschaft nicht durchgeführt.
