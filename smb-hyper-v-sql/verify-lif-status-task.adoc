= LIF-Status überprüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Selbst wenn Sie Storage Virtual Machines (SVMs) mit Hyper-V und SQL Server über SMB-Konfigurationen so einrichten, dass auf jedem Knoten eines Clusters LIFs vorhanden sind, kann es im täglichen Betrieb vorkommen, dass einige LIFs auf Ports eines anderen Knotens verschoben werden. Der LIF-Status muss überprüft und gegebenenfalls Korrekturmaßnahmen ergriffen werden.

.Über diese Aufgabe
Um einen nahtlosen, unterbrechungsfreien Betrieb zu ermöglichen, muss jeder Knoten in einem Cluster mindestens eine LIF für die SVM besitzen, und alle LIFs müssen einem Home-Port zugeordnet sein. Wenn einige der konfigurierten LIFs derzeit nicht ihrem Home-Port zugeordnet sind, sollten etwaige Portprobleme behoben und die LIFs anschließend wieder ihrem Home-Port zugeordnet werden.

.Schritte
. Informationen zu den konfigurierten LIFs für die SVM:
+
`*network interface show -vserver _vserver_name_*`

+
In diesem Beispiel befindet sich "`lif1`" nicht am Home-Port.

+
`network interface show -vserver vs1`

+
[listing]
----

            Logical    Status     Network          Current    Current Is
Vserver     Interface  Admin/Oper Address/Mask     Node       Port    Home
----------- ---------- ---------- ---------------- ---------- ------- ----
vs1
            lif1         up/up    10.0.0.128/24    node2      e0d     false
            lif2         up/up    10.0.0.129/24    node2      e0d     true
----
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

. Falls einige der LIFs nicht auf ihren Home-Ports sind, sind die folgenden Schritte auszuführen:
+
.. Für jede LIF ist zu bestimmen, welcher Heimatport der LIF zugeordnet ist:
+
`*network interface show -vserver _vserver_name_ -lif _lif_name_ -fields home-node,home-port*`

+
`*network interface show -vserver vs1 -lif lif1 -fields home-node,home-port*`

+
[listing]
----

vserver lif  home-node  home-port
------- ---- ---------- ---------
vs1     lif1 node1      e0d
----
.. Für jede LIF feststellen, ob der Home-Port der LIF aktiv ist:
+
`*network port show -node _node_name_ -port _port_ -fields port,link*`

+
`network port show -node node1 -port e0d -fields port,link`

+
[listing]
----

node       port link
---------- ---- ----
node1      e0d  up
----
+
In diesem Beispiel sollte "`lif1`" zurück auf seinen Heimatport migriert werden, `node1:e0d`.

+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].



. Falls sich eine der Home-Port-Netzwerkschnittstellen, denen die LIFs zugeordnet werden sollen, nicht im  `up` Zustand befindet, sollte das Problem behoben werden, damit diese Schnittstellen aktiv sind. Weitere Informationen zu  `up` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/up.html["ONTAP-Befehlsreferenz"^].
. Falls erforderlich können die LIFs auf ihre ursprünglichen Ports zurückgesetzt werden:
+
`*network interface revert -vserver _vserver_name_ -lif _lif_name_*`

+
`*network interface revert -vserver vs1 -lif lif1*`

+
Weitere Informationen zu  `network interface revert` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-revert.html["ONTAP-Befehlsreferenz"^].

. Es sollte sichergestellt sein, dass jeder Knoten im Cluster über eine aktive LIF für die SVM verfügt:
+
`*network interface show -vserver _vserver_name_*`

+
`*network interface show -vserver vs1*`

+
[listing]
----

            Logical    Status     Network          Current    Current Is
Vserver     Interface  Admin/Oper Address/Mask     Node       Port    Home
----------- ---------- ---------- ---------------- ---------- ------- ----
vs1
            lif1         up/up    10.0.0.128/24    node1      e0d     true
            lif2         up/up    10.0.0.129/24    node2      e0d     true
----
