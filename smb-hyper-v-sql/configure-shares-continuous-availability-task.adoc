= Vorhandene Freigaben für kontinuierliche Verfügbarkeit konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bestehende Freigaben können so geändert werden, dass sie dauerhaft verfügbar sind und von den Hyper-V und SQL Server-Anwendungsservern für den unterbrechungsfreien Zugriff auf Hyper-V-VM- und Konfigurationsdateien sowie SQL Server-Datenbankdateien genutzt werden.

.Über diese Aufgabe
Sie können eine bestehende Freigabe nicht als ständig verfügbare Freigabe für unterbrechungsfreie Operationen mit Anwendungsservern über SMB verwenden, wenn die Freigabe die folgenden Eigenschaften aufweist:

* Wenn die  `homedirectory` Freigabeeigenschaft für diese Freigabe festgelegt ist
* Wenn die Freigabe aktivierte Symlinks oder Widelinks enthält
* Wenn die Freigabe verbundene Volumes unterhalb des Stammverzeichnisses der Freigabe enthält


Es ist zu überprüfen, ob die folgenden beiden Freigabeparameter korrekt eingestellt sind:

* Der  `-offline-files` Parameter ist entweder auf  `manual` (Standardwert) oder  `none` gesetzt.
* Symlinks müssen deaktiviert sein.


Folgende Freigabeeigenschaften müssen konfiguriert werden:

* `continuously-available`
* `oplocks`


Die folgenden Freigabeeigenschaften dürfen nicht festgelegt werden. Falls sie in der Liste der aktuellen Freigabeeigenschaften vorhanden sind, müssen sie aus der kontinuierlich verfügbaren Freigabe entfernt werden:

* `attributecache`
* `branchcache`


.Schritte
. Die aktuellen Freigabeparametereinstellungen und die aktuelle Liste der konfigurierten Freigabeeigenschaften werden angezeigt:
+
[source, cli]
----
vserver cifs share show -vserver <vserver_name> -share-name <share_name>
----
. Gegebenenfalls können die Freigabeparameter angepasst werden, um symbolische Links zu deaktivieren und Offline-Dateien mithilfe des  `vserver cifs share modify` Befehls auf manuell zu setzen.
+
** Sie können symbolische Links deaktivieren, indem der Wert des  `-symlink` Parameters auf  `""` gesetzt wird.
** Der  `-offline-files`Parameter kann auf die richtige Einstellung gesetzt werden, indem  `manual`angegeben wird.


. Die  `continuously-available`Share-Eigenschaft und, falls erforderlich, die  `oplocks`Share-Eigenschaft hinzufügen:
+
[source, cli]
----
vserver cifs share properties add -vserver <vserver_name> -share-name <share_name> -share-properties continuously-available[,oplock]
----
+
Falls die  `oplocks` share-Eigenschaft noch nicht festgelegt ist, muss sie zusammen mit der  `continuously-available` share-Eigenschaft hinzugefügt werden.

. Entfernen Sie alle Freigabeeigenschaften, die für kontinuierlich verfügbare Freigaben nicht unterstützt werden:
+
[source, cli]
----
vserver cifs share properties remove -vserver <vserver_name> -share-name <share_name> -share-properties properties[,...]
----
+
Sie können eine oder mehrere Freigabeeigenschaften entfernen, indem die Freigabeeigenschaften als durch Kommas getrennte Liste angegeben werden.

. Es sollte überprüft werden, ob die  `-symlink` und  `-offline-files` Parameter korrekt eingestellt sind:
+
[source, cli]
----
vserver cifs share show -vserver <vserver_name> -share-name <share_name> -fields symlink-properties,offline-files
----
. Es sollte überprüft werden, ob die Liste der konfigurierten Freigabeeigenschaften korrekt ist:
+
[source, cli]
----
vserver cifs share properties show -vserver <vserver_name> -share-name <share_name>
----


.Beispiele
Das folgende Beispiel zeigt, wie eine vorhandene Freigabe mit dem Namen „share1“ auf der Storage Virtual Machine (SVM) „vs1“ für NDOs mit einem Anwendungsserver über SMB konfiguriert wird:

* Symlinks werden auf der Freigabe deaktiviert, indem der  `-symlink` Parameter auf  `""` gesetzt wird.
* Der  `-offline-file`Parameter wird geändert und auf  `manual`gesetzt.
* Die  `continuously-available`Share-Eigenschaft wird dem Share hinzugefügt.
* Die  `oplocks`Freigabeeigenschaft ist bereits in der Liste der Freigabeeigenschaften enthalten, daher muss sie nicht hinzugefügt werden.
* Die  `attributecache`Share-Eigenschaft wird vom Share entfernt.
* Die  `browsable`Freigabeeigenschaft ist optional für eine kontinuierlich verfügbare Freigabe, die für NDOs mit Anwendungsservern über SMB verwendet wird, und bleibt als eine der Freigabeeigenschaften erhalten.


[listing]
----
cluster1::> vserver cifs share show -vserver vs1 -share-name share1

                      Vserver: vs1
                        Share: share1
     CIFS Server NetBIOS Name: vs1
                         Path: /data
             Share Properties: oplocks
                               browsable
                               attributecache
           Symlink Properties: enable
      File Mode Creation Mask: -
 Directory Mode Creation Mask: -

                    Share ACL: Everyone / Full Control

                  Volume Name: data

cluster1::> vserver cifs share properties add -vserver vs1 -share-name share1 -share-properties continuously-available

cluster1::> vserver cifs share properties remove -vserver vs1 -share-name share1 -share-properties attributecache
