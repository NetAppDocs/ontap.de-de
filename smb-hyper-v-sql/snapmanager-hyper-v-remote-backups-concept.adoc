= Wie SnapManager for Hyper-V Remote-VSS-basierte Backups für Hyper-V über SMB verwaltet
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können SnapManager for Hyper-V zur Verwaltung von Remote-VSS-basierten Backup-Services verwenden. Die Nutzung des von SnapManager for Hyper-V verwalteten Backup-Services bietet Vorteile bei der Erstellung platzsparender Backup-Sätze.

Optimierungen an von SnapManager for Hyper-V verwalteten Backups umfassen Folgendes:

* SnapManager for Hyper-V gibt die Liste der virtuellen Maschinendateien in den SMB-Freigaben an, die der Schattenkopiedienst kopieren muss.
+
Durch die Bereitstellung einer gezielten Liste von Dateien der virtuellen Maschine muss der Schattenkopiedienst keine Schattenkopien aller Dateien in der Freigabe erstellen.

* Die Storage Virtual Machine (SVM) behält die Snapshots, damit SnapManager for Hyper-V sie für Wiederherstellungen verwenden kann.
+
Es gibt keine Backup-Phase. Das Backup ist der platzsparende Snapshot.



SnapManager for Hyper-V bietet Sicherungs- und Wiederherstellungsfunktionen für HyperV über SMB mithilfe des folgenden Prozesses:

. Vorbereitung der Schattenkopie-Operation
+
Die SnapManager for Hyper-V Anwendung verwendet den VSS-Client, um den Schattenkopiesatz einzurichten. Der VSS-Client sammelt Informationen darüber, welche Freigaben in den Schattenkopiesatz aufgenommen werden, und stellt diese Informationen ONTAP zur Verfügung. Ein Satz kann eine oder mehrere Schattenkopien enthalten, und eine Schattenkopie entspricht jeweils einer Freigabe.

. Erstellen des Schattenkopiesatzes (wenn automatische Wiederherstellung verwendet wird)
+
Für jede im Schattenkopiesatz enthaltene Freigabe erstellt ONTAP eine Schattenkopie und macht diese beschreibbar.

. Freigeben des Schattenkopiesatzes
+
Nachdem ONTAP die Schattenkopien erstellt hat, werden diese SnapManager for Hyper-V zur Verfügung gestellt, damit die VSS-Writer der Anwendung eine automatische Wiederherstellung durchführen können.

. Automatische Wiederherstellung des Schattenkopie-Satzes
+
Während der Erstellung des Schattenkopiesatzes gibt es einen Zeitraum, in dem aktive Änderungen an den im Backup-Satz enthaltenen Dateien vorgenommen werden. Die VSS-Writer der Anwendung müssen die Schattenkopien aktualisieren, um sicherzustellen, dass sie sich vor der Sicherung in einem vollständig konsistenten Zustand befinden.

+
[NOTE]
====
Die Art und Weise, wie die automatische Wiederherstellung erfolgt, ist applikationsspezifisch. Remote VSS ist in dieser Phase nicht beteiligt.

====
. Vervollständigung und Bereinigung des Schattenkopiesatzes
+
Der VSS-Client benachrichtigt ONTAP, nachdem die automatische Wiederherstellung abgeschlossen ist. Der Schattenkopie-Satz wird schreibgeschützt und ist anschließend für das Backup bereit. Bei der Verwendung von SnapManager for Hyper-V für das Backup werden die Dateien in einem Snapshot zum Backup; daher wird für die Backup-Phase für jedes Volume, das Freigaben im Backup-Satz enthält, ein Snapshot erstellt. Nach Abschluss des Backups wird der Schattenkopie-Satz vom CIFS-Server entfernt.
