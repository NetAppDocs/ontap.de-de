= Kontinuierlich verfügbare SMB-Shares erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem Sie Ihre Datenvolumes erstellt haben, können Sie die kontinuierlich verfügbaren Freigaben erstellen, die die Anwendungsserver für den Zugriff auf Hyper-V-VM- und Konfigurationsdateien sowie SQL Server Datenbankdateien verwenden. Das Arbeitsblatt zur Freigabekonfiguration sollte beim Erstellen der SMB-Freigaben verwendet werden.

.Schritte
. Informationen zu den vorhandenen Datenvolumes und deren Verbindungspfaden werden angezeigt:
+
`*volume show -vserver _vserver_name_ -junction*`

. Erstellen einer permanent verfügbaren SMB-Freigabe:
+
`*vserver cifs share create -vserver _vserver_name_ -share-name _share_name_ -path _path_ -share-properties oplocks,continuously-available -symlink "" [-comment text]*`

+
** Optional kann ein Kommentar zur Freigabekonfiguration hinzugefügt werden.
** Standardmäßig ist die Eigenschaft „Offline-Dateien“ für die Freigabe konfiguriert und auf  `manual` eingestellt.
** ONTAP erstellt die Freigabe mit der Windows-Standardfreigabeberechtigung  `Everyone`/ `Full Control`.


. Der vorherige Schritt ist für alle Freigaben im Freigabekonfigurationsarbeitsblatt zu wiederholen.
. Die Korrektheit der Konfiguration lässt sich mit dem  `vserver cifs share show` Befehl überprüfen.
. NTFS-Dateiberechtigungen auf den kontinuierlich verfügbaren Freigaben werden konfiguriert, indem ein Laufwerk zu jeder Freigabe zugeordnet und die Dateiberechtigungen über das Fenster *Windows Properties* festgelegt werden.


.Beispiel
Die folgenden Befehle erstellen eine kontinuierlich verfügbare Freigabe namens "`data2`" auf der Storage Virtual Machine (SVM, ehemals Vserver) vs1. Symlinks werden deaktiviert, indem der `-symlink` Parameter auf `""` gesetzt wird:

[listing]
----
cluster1::> volume show -vserver vs1 -junction
                      Junction                  Junction
Vserver   Volume      Active   Junction Path    Path Source
--------- ----------- -------- ---------------- ------------
vs1       data        true     /data            RW_volume
vs1       data1       true     /data/data1      RW_volume
vs1       data2       true     /data/data2      RW_volume
vs1       vs1_root    -        /                -

cluster1::> vserver cifs share create -vserver vs1 -share-name data2 -path /data/data2 -share-properties oplocks,continuously-available -symlink ""

cluster1::> vserver cifs share show -vserver vs1 -share-name data2

                      Vserver: vs1
