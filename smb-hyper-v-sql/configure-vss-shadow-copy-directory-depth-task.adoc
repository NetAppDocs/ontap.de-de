= Konfiguration der Verzeichnistiefe für VSS-Schattenkopien (für Hyper-V über SMB-Freigaben)
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Optional kann die maximale Verzeichnistiefe innerhalb von SMB-Freigaben konfiguriert werden, auf denen Schattenkopien erstellt werden sollen. Dieser Parameter ist nützlich, wenn die maximale Ebene von Unterverzeichnissen, auf denen ONTAP Schattenkopien erstellen soll, manuell gesteuert werden soll.

.Bevor Sie beginnen
Die VSS-Schattenkopiefunktion muss aktiviert sein.

.Über diese Aufgabe
Standardmäßig werden Schattenkopien für maximal fünf Unterverzeichnisse erstellt. Wenn der Wert auf  `0` gesetzt ist, erstellt ONTAP Schattenkopien für alle Unterverzeichnisse.

[NOTE]
====
Obwohl Sie festlegen können, dass die Verzeichnistiefe des Schattenkopie-Satzes mehr als fünf Unterverzeichnisse oder alle Unterverzeichnisse umfassen soll, gibt es eine Anforderung von Microsoft, dass die Erstellung des Schattenkopie-Satzes innerhalb von 60 Sekunden abgeschlossen sein muss. Die Erstellung des Schattenkopie-Satzes schlägt fehl, wenn sie nicht innerhalb dieses Zeitraums abgeschlossen werden kann. Die von Ihnen gewählte Verzeichnistiefe darf nicht dazu führen, dass die Erstellungszeit das Zeitlimit überschreitet.

====
.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`*set -privilege advanced*`

. Die Tiefe des VSS-Schattenkopieverzeichnisses auf das gewünschte Niveau festlegen:
+
`*vserver cifs options modify -vserver _vserver_name_ -shadowcopy-dir-depth integer*`

+
`*vserver cifs options modify -vserver vs1 -shadowcopy-dir-depth 6*`

. Zurück zur Administrator-Privilegstufe:
+
`*set -privilege admin*`
