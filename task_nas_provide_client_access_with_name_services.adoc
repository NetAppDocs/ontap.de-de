= Clientzugriff mit Namensdiensten über ONTAP System Manager bereitstellen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
ONTAP kann Host-, Benutzer-, Gruppen- oder Netzgruppeninformationen mithilfe von LDAP oder NIS abrufen, um NAS-Clients zu authentifizieren.

Dieses Verfahren erstellt oder ändert LDAP- oder NIS-Konfigurationen auf einer vorhandenen Storage-VM, die für link:task_nas_enable_linux_nfs.html["NFS"] oder link:task_nas_enable_windows_smb.html["SMB"] aktiviert ist.

Für LDAP-Konfigurationen sollten die in Ihrer Umgebung erforderlichen LDAP-Konfigurationsdetails vorliegen und ein Standard ONTAP LDAP-Schema verwendet werden.

.Schritte
. Den erforderlichen Dienst konfigurieren: *Storage > Storage VMs* auswählen.
. Wählen Sie die Storage-VM aus, klicken Sie auf *Einstellungen* und dann auf image:icon_gear.gif["Aktionssymbol"] für LDAP oder NIS.
. Alle Änderungen im Namensdienstschalter einbeziehen: image:icon_pencil.gif["Bearbeiten-Symbol"] unter Namensdienstschalter auswählen.
