= Einstellungen für automatisch generierte ARP Snapshots anpassen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.11.1 kann über die CLI die Aufbewahrungseinstellungen für Autonomous Ransomware Protection (ARP) Snapshots gesteuert werden, die automatisch als Reaktion auf vermutete Ransomware-Angriffe generiert werden.

.Bevor Sie beginnen
Die Optionen für ARP-Snapshots können nur auf einer link:../system-admin/types-svms-concept.html["Node SVM"] und nicht auf anderen SVM-Typen geändert werden.

.Schritte
. Alle aktuellen ARP Snapshot-Einstellungen:
+
[source, cli]
----
options -option-name arw*
----
. Ausgewählte aktuelle ARP-Snapshot-Einstellungen:
+
[source, cli]
----
options -option-name <arw_setting_name>
----
. ARP Snapshot-Einstellungen anpassen:
+
[source, cli]
----
options -option-name <arw_setting_name> -option-value <arw_setting_value>
----
+
Folgende Einstellungen können geändert werden:

+

NOTE: Einige der beschriebenen Befehle sind ab ONTAP 9.17.1 veraltet. Die in ONTAP 9.17.1 eingeführten Befehle unterstützen sowohl NAS- als auch SAN-Umgebungen.

+
[cols="1,3,1"]
|===
| Einstellung | Beschreibung | Unterstützte Versionen 


| `arw.snap.max.count`  a| 
Legt die maximale Anzahl von ARP-Snapshots fest, die zu einem bestimmten Zeitpunkt in einem Volume vorhanden sein können. Ältere Kopien werden gelöscht, damit die Gesamtzahl der ARP-Snapshots innerhalb dieses festgelegten Grenzwerts bleibt.
 a| 
ONTAP 9.11.1 und höher



| `arw.snap.create.interval.hours`  a| 
Gibt das Intervall _in Stunden_ zwischen ARP-Snapshots an. Ein neuer ARP-Snapshot wird erstellt, wenn ein Angriff auf Basis der Datenentropie vermutet wird und der zuletzt erstellte ARP-Snapshot älter als das angegebene Intervall ist.
 a| 
ONTAP 9.11.1 und höher



| `arw.snap.normal.retain.interval.hours`  a| 
Gibt die Dauer _in Stunden_ an, für die ein ARP-Snapshot aufbewahrt wird. Wenn ein ARP-Snapshot den Aufbewahrungsschwellenwert erreicht, wird er gelöscht.
 a| 
** ONTAP 9.11.1 bis ONTAP 9.16.1
** In ONTAP 9.17.1 und späteren Versionen als veraltet markiert




| `arw.snap.max.retain.interval.days`  a| 
Gibt die maximale Dauer _in Tagen_ an, für die ein ARP Snapshot aufbewahrt werden kann. Jeder ARP Snapshot, der älter als diese Dauer ist, wird gelöscht, wenn kein Angriff auf dem Volume gemeldet wird.


NOTE: Das maximale Aufbewahrungsintervall für ARP Snapshots wird bei einer erkannten mittleren Bedrohung ignoriert. Der als Reaktion auf die Bedrohung erstellte ARP Snapshot wird so lange aufbewahrt, bis auf die Bedrohung reagiert wurde. Wenn eine Bedrohung als Fehlalarm markiert wird, löscht ONTAP die ARP Snapshots für das Volume.
 a| 
** ONTAP 9.11.1 bis ONTAP 9.16.1
** In ONTAP 9.17.1 und späteren Versionen als veraltet markiert




| `arw.snap.create.interval.hours.post.max.count`  a| 
Gibt das Intervall _in Stunden_ zwischen ARP-Snapshots an, wenn das Volume bereits die maximale Anzahl an ARP-Snapshots enthält. Wenn die maximale Anzahl erreicht ist, wird ein ARP-Snapshot gelöscht, um Platz für eine neue Kopie zu schaffen. Mit dieser Option kann die Erstellungsgeschwindigkeit neuer ARP-Snapshots reduziert werden, um die ältere Kopie zu behalten. Wenn das Volume bereits die maximale Anzahl an ARP-Snapshots enthält, wird für die Erstellung des nächsten ARP-Snapshots das in dieser Option angegebene Intervall verwendet, anstelle von `arw.snap.create.interval.hours`.
 a| 
** ONTAP 9.11.1 bis 9.16.1
** In ONTAP 9.17.1 und späteren Versionen als veraltet markiert




| `arw.snap.low.encryption.retain.duration.hours`  a| 
Legt die Aufbewahrungsdauer _in Stunden_ für ARP Snapshots fest, die während Perioden geringer Verschlüsselungsaktivität erstellt werden.
 a| 
** ONTAP 9.17.1 und höher




| `arw.snap.new.extns.interval.hours`  a| 
Gibt das Intervall _in Stunden_ zwischen den ARP-Snapshots an, die beim Erkennen einer neuen Dateierweiterung erstellt werden. Ein neuer ARP-Snapshot wird erstellt, wenn eine neue Dateierweiterung erkannt wird; der zuvor beim Erkennen einer neuen Dateierweiterung erstellte Snapshot ist älter als dieses angegebene Intervall. Bei einer Arbeitslast, die häufig neue Dateierweiterungen erzeugt, hilft dieses Intervall, die Häufigkeit der ARP-Snapshots zu steuern. Diese Option besteht unabhängig von  `arw.snap.create.interval.hours`, die das Intervall für datenentropiebasierte ARP-Snapshots angibt.
 a| 
** ONTAP 9.11.1 bis ONTAP 9.16.1
** In ONTAP 9.17.1 und späteren Versionen als veraltet markiert




| `arw.snap.retain.hours.after.clear.suspect.false.alert`  a| 
Gibt das Intervall (in Stunden) an, in dem ein ARP-Snapshot vorsorglich aufbewahrt wird, nachdem ein Angriffsvorfall vom Administrator als Fehlalarm markiert wurde. Nach Ablauf dieser vorsorglichen Aufbewahrungsfrist kann der Snapshot gemäß der durch die Optionen `arw.snap.normal.retain.interval.hours` und `arw.snap.max.retain.interval.days` definierten Standardaufbewahrungsdauer gelöscht werden.
 a| 
** ONTAP 9.16.1 und höher




| `arw.snap.retain.hours.after.clear.suspect.real.attack`  a| 
Gibt das Intervall _in Stunden_ an, in dem ein ARP-Snapshot vorsorglich aufbewahrt wird, nachdem ein Angriffsereignis vom Administrator als tatsächlicher Angriff markiert wurde. Nach Ablauf dieser vorsorglichen Aufbewahrungsfrist kann der Snapshot gemäß der durch die Optionen `arw.snap.normal.retain.interval.hours` und `arw.snap.max.retain.interval.days` definierten Standardaufbewahrungsdauer gelöscht werden.
 a| 
** ONTAP 9.16.1 und höher




| `arw.snap.surge.interval.days`  a| 
Legt das Intervall _in Tagen_ zwischen ARP-Snapshots fest, die als Reaktion auf E/A-Spitzen erstellt werden. ONTAP erstellt eine ARP-Snapshot-Surge-Kopie, wenn ein Anstieg des E/A-Verkehrs auftritt und der zuletzt erstellte ARP-Snapshot älter als dieses angegebene Intervall ist. Diese Option legt außerdem die Aufbewahrungsdauer _in Tagen_ für einen ARP-Surge-Snapshot fest.
 a| 
ONTAP 9.11.1 und höher



| `arw.high.encryption.alert.enabled`  a| 
Aktiviert Warnmeldungen bei hohem Verschlüsselungsgrad. Wenn diese Option auf  `on` (Standardeinstellung) gesetzt ist, sendet ONTAP eine Warnmeldung, wenn der Prozentsatz der Verschlüsselung den in  `arw.high.encryption.percentage.threshold` angegebenen Schwellenwert überschreitet.
 a| 
ONTAP 9.17.1 und höher



| `arw.high.encryption.percentage.threshold`  a| 
Legt den maximalen Prozentsatz der Verschlüsselung für ein Volume fest. Wenn der Prozentsatz der Verschlüsselung diesen Schwellenwert überschreitet, behandelt ONTAP die Erhöhung als Angriff und erstellt einen ARP-Snapshot.  `arw.high.encryption.alert.enabled` muss auf  `on` gesetzt werden, damit diese Option wirksam wird.
 a| 
ONTAP 9.17.1 und höher



| `arw.snap.high.encryption.retain.duration.hours`  a| 
Gibt das Aufbewahrungsintervall _in Stunden_ für Snapshots an, die während eines Ereignisses mit hohem Verschlüsselungsschwellenwert erstellt wurden.
 a| 
ONTAP 9.17.1 und höher

|===
. Wenn Sie ARP in einer SAN-Umgebung verwenden, können auch die folgenden Einstellungen für den Auswertungszeitraum angepasst werden:
+
[cols="1,3,1"]
|===
| Einstellung | Beschreibung | Unterstützte Versionen 


| `arw.block_device.auto.learn.threshold.min_value`  a| 
Legt den minimalen Verschlüsselungsschwellenwert in Prozent während der automatischen Lernphase der Evaluierung für Blockgeräte fest.
 a| 
ONTAP 9.17.1 und höher



| `arw.block_device.auto.learn.threshold.max_value`  a| 
Legt den maximalen Verschlüsselungsschwellenwert in Prozent während der automatischen Lernphase der Evaluierung für Blockgeräte fest.
 a| 
ONTAP 9.17.1 und höher



| `arw.block_device.evaluation.phase.min_hours` | Gibt das Mindestintervall _in Stunden_ an, das die Auswertungsphase durchlaufen muss, bevor der Verschlüsselungsschwellenwert festgelegt wird.  a| 
ONTAP 9.17.1 und höher



| `arw.block_device.evaluation.phase.max_hours`  a| 
Gibt das maximale Intervall _in Stunden_ an, das die Auswertungsphase durchlaufen muss, bevor der Verschlüsselungsschwellenwert festgelegt wird.
 a| 
ONTAP 9.17.1 und höher



| `arw.block_device.evaluation.phase.min_data_ingest_size_GB` | Gibt die Mindestmenge an Daten _in GB_ an, die während der Auswertungsphase erfasst werden muss, bevor der Verschlüsselungsschwellenwert festgelegt wird.  a| 
ONTAP 9.17.1 und höher



| `arw.block_device.evaluation.phase.alert.enabled`  a| 
Gibt an, ob Warnmeldungen für die Auswertungsphase von ARP auf Blockgeräten aktiviert sind. Der Standardwert ist  `True`.
 a| 
ONTAP 9.17.1 und höher



| `arw.block_device.evaluation.phase.alert.threshold`  a| 
Legt den Schwellenwert in Prozent während der Auswertungsphase von ARP auf Blockgeräten fest. Wenn der Prozentsatz der Verschlüsselung diesen Schwellenwert überschreitet, wird eine Warnung ausgelöst.
 a| 
ONTAP 9.17.1 und höher

|===
