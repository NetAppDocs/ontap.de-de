= ONTAP Autonomous Ransomware Protection pausieren, um Workload-Ereignisse von der Analyse auszuschließen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn Sie mit ungewöhnlichen Arbeitslastereignissen rechnen, kann die Analyse des Autonomous Ransomware Protection (ARP) jederzeit vorübergehend ausgesetzt und wieder aufgenommen werden.

Ab ONTAP 9.13.1 kann die Multi-Admin-Verifizierung (MAV) aktiviert werden, sodass zwei oder mehr authentifizierte Benutzeradministratoren erforderlich sind, um das Pausieren des ARP zu ermöglichen.

link:../multi-admin-verify/enable-disable-task.html["Weitere Informationen zu MAV"].

.Über diese Aufgabe
Während einer ARP-Pause protokolliert ONTAP keine Ereignisse oder Aktionen für neue Schreibvorgänge, jedoch werden Analysen für frühere Protokolle im Hintergrund fortgesetzt.


NOTE: Die ARP disable-Funktion sollte nicht verwendet werden, um die Analyse zu pausieren. Dadurch wird ARP auf dem Volume deaktiviert und alle vorhandenen Informationen zum erlernten Workload-Verhalten gehen verloren. Dies würde einen Neustart des Lernzeitraums erfordern.

.Schritte
Sie können System Manager oder die ONTAP CLI verwenden, um ARP anzuhalten.

[role="tabbed-block"]
====
.System Manager
--
. *Storage > Volumes* auswählen und dann das Volume auswählen, bei dem ARP angehalten werden soll.
. Auf der Registerkarte *Sicherheit* in der Volumes-Übersicht im Feld *Anti-ransomware* die Option *Pause anti-ransomware* auswählen.
+

NOTE: Ab ONTAP 9.13.1 werden Sie bei Verwendung von MAV zum Schutz von ARP-Einstellungen vor dem Anhalten des Vorgangs aufgefordert, die Zustimmung eines oder mehrerer zusätzlicher Administratoren einzuholen. link:../multi-admin-verify/request-operation-task.html["Die Zustimmung aller Administratoren muss eingeholt werden."] der MAV-Genehmigungsgruppe oder der Vorgang schlägt fehl.

. Zur Wiederaufnahme der Überwachung *Anti-Ransomware fortsetzen* auswählen.


--
.CLI
--
. ARP auf einem Volume pausieren:
+
[source, cli]
----
security anti-ransomware volume pause -vserver <svm_name> -volume <vol_name>
----
. Um die Verarbeitung fortzusetzen, den  `resume` Befehl verwenden:
+
[source, cli]
----
security anti-ransomware volume resume -vserver <svm_name> -volume <vol_name>
----
+
Weitere Informationen zu  `security anti-ransomware volume` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=security+anti-ransomware+volume+["ONTAP-Befehlsreferenz"^].

. Wenn Sie MAV (verfügbar mit ARP ab ONTAP 9.13.1) zum Schutz der ARP-Einstellungen verwenden, fordert der Vorgang „Anhalten“ dazu auf, die Zustimmung eines oder mehrerer zusätzlicher Administratoren einzuholen. Die Zustimmung muss von allen Administratoren der MAV-Genehmigungsgruppe vorliegen, sonst schlägt der Vorgang fehl.
+
Wenn Sie MAV verwenden und für einen erwarteten Pausenvorgang zusätzliche Genehmigungen erforderlich sind, führt jeder Genehmiger einer MAV-Gruppe Folgendes aus:

+
.. Anfrage anzeigen:
+
[source, cli]
----
security multi-admin-verify request show
----
.. Anfrage genehmigen:
+
[source, cli]
----
security multi-admin-verify request approve -index[<number returned from show request>]
----
+
Die Antwort des letzten Gruppengenehmigers zeigt an, dass das Volumen geändert wurde und der Status von ARP pausiert ist.

+
Wenn Sie MAV verwenden und ein MAV-Gruppengenehmiger sind, kann eine Anfrage zum Anhalten eines Vorgangs abgelehnt werden:

+
[source, cli]
----
security multi-admin-verify request veto -index[<number returned from show request>]
----
+
Weitere Informationen zu  `security multi-admin-verify request` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=security+multi-admin-verify+request["ONTAP-Befehlsreferenz"^].
