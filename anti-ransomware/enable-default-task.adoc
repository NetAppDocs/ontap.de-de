= ONTAP Autonomous Ransomware Protection standardmäßig in neuen Volumes aktivieren
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.10.1 können Sie Storage-VMs (SVMs) so konfigurieren, dass neue Volumes standardmäßig mit Autonomous Ransomware Protection (ARP) aktiviert sind. Sie können diese Einstellung mit dem System Manager oder der ONTAP CLI ändern.

Ab ONTAP 9.18.1 ist ARP auf allen neuen Volumes auf Clusterebene für link:use-cases-restrictions-concept.html#supported-configurations["Unterstützte Systeme"] nach einer 12-stündigen Übergangsfrist nach einem Cluster-Upgrade oder einer Neuinstallation standardmäßig aktiviert. Wenn Sie die automatische Standardaktivierung von ARP auf Clusterebene deaktivieren, können Sie ARP weiterhin standardmäßig manuell auf allen neuen Volumes auf SVM-Ebene aktivieren.

Für ONTAP 9.17.1 und früher ist die Konfiguration auf SVM-Ebene die einzige Möglichkeit, ARP standardmäßig auf neuen Volumes zu aktivieren.

.Über diese Aufgabe
Standardmäßig werden neue Volumes mit deaktivierter ARP-Funktionalität erstellt. Die ARP-Funktionalität muss aktiviert und so eingestellt werden, dass sie für neue Volumes, die in der SVM erstellt werden, standardmäßig aktiviert ist.

Vorhandene Volumes ohne aktiviertes ARP ändern ihren ARP-Aktivierungsstatus nicht automatisch, wenn die Standardeinstellung für die SVM geändert wird. Die in diesem Verfahren beschriebenen Änderungen der SVM-Einstellungen betreffen nur neue Volumes. Informationen dazu, wie link:enable-task.html["ARP für bestehende Volumes aktivieren"].

[[arp-enablement-default-behavior-table]]
Nach der Aktivierung von ARP kann ARP je nach Umgebung und ONTAP Version in eine Übergangsphase eintreten:

[cols="25,25,50"]
|===
| Volumentyp | ONTAP Version | Verhalten nach der Aktivierung 


.2+| NAS FlexGroup | ONTAP 9.18.1 und höher | ARP/KI ist sofort ohne Lernphase aktiv 


| ONTAP 9.13.1 bis 9.17.1 | ARP startet im Lernmodus für 30 Tage 


.2+| NAS FlexVol | ONTAP 9.16.1 und höher | ARP/KI ist sofort ohne Lernphase aktiv 


| ONTAP 9.10.1 bis 9.15.1 | ARP startet im Lernmodus für 30 Tage 


| SAN-Volumes | ONTAP 9.17.1 und höher | ARP/AI wird sofort aktiv und leitet eine Evaluierungsphase ein, um einen geeigneten Alarmschwellenwert festzulegen, bevor von einem anfänglich konservativen Schwellenwert umgeschaltet wird. 
|===
.Bevor Sie beginnen
Vor der Aktivierung von ARP muss sichergestellt sein, dass Ihre Umgebung Folgendes aufweist:

.NAS-spezifische Anforderungen
* Eine Storage-VM (SVM) mit aktiviertem NFS- oder SMB-Protokoll (oder beiden).
* Ein aktives link:../concepts/namespaces-junction-points-concept.html["Verbindungspfad"] für das Volume.


.SAN-spezifische Anforderungen
* Eine Storage VM (SVM) mit aktiviertem iSCSI, FC oder NVMe Protokoll.


.Allgemeine Anforderungen
* Die link:index.html#licenses-and-enablement["korrekte Lizenz"] für Ihre ONTAP Version.
* (Empfohlen) Multi-Admin-Verifizierung (MAV) aktiviert (ONTAP 9.13.1+). Siehe link:../multi-admin-verify/enable-disable-task.html["Multi-Admin-Verifizierung aktivieren"].


.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um ARP standardmäßig auf neuen Volumes zu aktivieren.

[role="tabbed-block"]
====
.System Manager
--
. *Storage* oder *Cluster* (je nach Umgebung) auswählen, *Storage VMs* auswählen und die Storage VM auswählen, die die Volumes enthält, die mit ARP geschützt werden sollen.
. Navigieren Sie zum Tab *Einstellungen*. Unter *Sicherheit* die Kachel *Anti-ransomware* suchen und dann image:icon_pencil.gif["Bearbeiten-Symbol"] auswählen.
. Aktivieren Sie das Kontrollkästchen, um Anti-Ransomware (ARP) zu aktivieren. Aktivieren Sie das zusätzliche Kontrollkästchen, um ARP auf allen geeigneten Volumes in der Storage-VM zu aktivieren.
. Für ONTAP Versionen mit einer empfohlenen Lernperiode kann *Automatisch vom Lern- in den Aktivmodus wechseln nach ausreichendem Lernvorgang* ausgewählt werden. Dadurch kann ARP das optimale Lernintervall bestimmen und den Wechsel in den Aktivmodus automatisieren.


--
.CLI
--
.Eine bestehende SVM so anpassen, dass ARP in neuen Volumes standardmäßig aktiviert ist
Wählen Sie  `dry-run`, wenn Ihre Version von ARP ein <<arp-enablement-default-behavior-table,Lernperiode>> erfordert. Andernfalls wählen Sie  `enabled`.

[source, cli]
----
vserver modify -vserver <svm_name> -anti-ransomware-default-volume-state <dry-run|enabled>
----
.Eine neue SVM mit standardmäßig aktiviertem ARP für neue Volumes erstellen
Wählen Sie  `dry-run`, wenn Ihre Version von ARP ein <<arp-enablement-default-behavior-table,Lernperiode>> erfordert. Andernfalls wählen Sie  `enabled`.

[source, cli]
----
vserver create -vserver <svm_name> -anti-ransomware-default-volume-state <dry-run|enabled>
----
.Vorhandene SVM anpassen, um den automatischen Übergang vom Lern- in den aktiven Modus zu deaktivieren
Wenn ein Upgrade auf ONTAP 9.13.1 bis ONTAP 9.15.1 durchgeführt wurde und der Standardstatus  `dry-run` (Lernmodus) ist, ist adaptives Lernen aktiviert, sodass der Wechsel in den  `enabled` Status (aktiver Modus) automatisch erfolgt. Es besteht die Möglichkeit, diese automatische Umschaltung zu deaktivieren, sodass die Steuerung des Wechsels vom Lern- in den aktiven Modus für alle zugehörigen Volumes manuell erfolgen kann.

[source, cli]
----
vserver modify <svm_name> -anti-ransomware-auto-switch-from-learning-to-enabled false
----
.Der ARP-Status wird überprüft
[source, cli]
----
security anti-ransomware volume show
----
--
