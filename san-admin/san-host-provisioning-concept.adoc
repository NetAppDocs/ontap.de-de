= SAN-Bereitstellung mit iSCSI
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In SAN-Umgebungen sind Speichersysteme Ziele, die Speicherzielgeräte besitzen. Für iSCSI und FC werden die Speicherzielgeräte als LUNs (logische Einheiten) bezeichnet. Für Non-Volatile Memory Express (NVMe) über Fibre Channel werden die Speicherzielgeräte als Namespaces bezeichnet.

Der Speicher wird konfiguriert, indem LUNs für iSCSI und FC oder Namespaces für NVMe erstellt werden. Auf die LUNs oder Namespaces greifen Hosts über Internet Small Computer Systems Interface (iSCSI) oder Fibre Channel (FC) Protokollnetzwerke zu.

Zur Verbindung mit iSCSI-Netzwerken können Hosts Standard-Ethernet-Netzwerkadapter (NICs), TCP-Offload-Engine-Karten (TOE) mit Software-Initiatoren, konvergierte Netzwerkadapter (CNAs) oder dedizierte iSCSI-Hostbusadapter (HBAs) verwenden.

Für die Verbindung mit FC-Netzwerken sind FC-HBAs oder CNAs auf den Hosts erforderlich.

Unterstützte FC-Protokolle umfassen:

* FC
* FCoE
* NVMe




[[iscsi-target-node-network-connections-and-names]]
== iSCSI-Zielknoten-Netzwerkverbindungen und Namen

iSCSI-Zielknoten können auf verschiedene Arten mit dem Netzwerk verbunden werden:

* Über Ethernet-Schnittstellen mit in ONTAP integrierter Software.
* Über mehrere Systemschnittstellen, wobei eine Schnittstelle für iSCSI verwendet wird, die auch Datenverkehr für andere Protokolle wie SMB und NFS übertragen kann.
* Verwendung eines Unified Zieladapter (UTA) oder eines konvergierten Netzwerkadapters (CNA).


Jeder iSCSI-Knoten muss einen Node-Name haben.

Die beiden Formate bzw. Typbezeichnungen für iSCSI-Node-Namen sind _iqn_ und _eui_. Das SVM iSCSI-Ziel verwendet immer die iqn-Typbezeichnung. Der Initiator kann entweder die iqn-Typbezeichnung oder die eui-Typbezeichnung verwenden.



[[storage-system-node-name]]
== Node-Name des Storage-Systems

Jede SVM, die iSCSI ausführt, hat einen Node-Name, der auf einem umgekehrten Domänennamen und einer eindeutigen Codierungsnummer basiert.

Der Node-Name wird im folgenden Format angezeigt:

iqn.1992-08.com.netapp:sn._unique-encoding-number_

Das folgende Beispiel zeigt den standardmäßigen Node-Name für ein Speichersystem mit einer eindeutigen Codierungsnummer:

[listing]
----
iqn.1992-08.com.netapp:sn.812921059e6c11e097b3123478563412:vs.6
----


[[tcp-port-for-iscsi]]
== TCP-Port für iSCSI

Das iSCSI-Protokoll ist in ONTAP so konfiguriert, dass es den TCP-Port 3260 verwendet.

ONTAP unterstützt keine Änderung der Portnummer für iSCSI. Portnummer 3260 ist als Teil der iSCSI-Spezifikation registriert und kann von keiner anderen Anwendung oder keinem anderen Dienst verwendet werden.
