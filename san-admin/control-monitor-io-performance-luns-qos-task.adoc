= Die E/A-Leistung zu LUNs kann mithilfe von Storage QoS gesteuert und überwacht werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Ein-/Ausgabeleistung (E/A) von LUNs steuern, indem Sie LUNs Storage QoS-Richtliniengruppen zuweisen. Die Steuerung der E/A-Leistung kann dazu dienen, sicherzustellen, dass Workloads bestimmte Leistungsziele erreichen, oder um einen Workload zu drosseln, der andere Workloads beeinträchtigt.

.Über diese Aufgabe
Richtliniengruppen erzwingen eine maximale Durchsatzbegrenzung (z. B. 100 MB/s). Sie können eine Richtliniengruppe erstellen, ohne eine maximale Durchsatzbegrenzung anzugeben, wodurch Sie die Leistung überwachen können, bevor Sie die Arbeitslast steuern.

Sie können auch Storage Virtual Machines (SVMs) mit FlexVol Volumes und LUNs Richtliniengruppen zuweisen.

Beachten Sie die folgenden Anforderungen für die Zuordnung einer LUN zu einer Richtliniengruppe:

* Die LUN muss sich in der SVM befinden, zu der die Richtliniengruppe gehört.
+
Die SVM wird beim Erstellen der Richtliniengruppe angegeben.

* Wenn eine LUN einer Richtliniengruppe zugewiesen wird, kann das die LUN enthaltende Volume oder die SVM nicht einer Richtliniengruppe zugewiesen werden.


Weitere Informationen zur Verwendung von Storage QoS sind unter link:../system-admin/index.html["Systemadministrationsreferenz"] zu finden.

.Schritte
. Mit dem  `qos policy-group create` Befehl lässt sich eine Richtliniengruppe erstellen.
+
Weitere Informationen zu  `qos policy-group create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/qos-policy-group-create.html["ONTAP-Befehlsreferenz"^].

. Der  `lun create` Befehl oder der  `lun modify` Befehl mit dem  `-qos-policy-group` Parameter weist eine LUN einer Richtliniengruppe zu.
+
Weitere Informationen zu  `lun` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=lun["ONTAP-Befehlsreferenz"^].

. Die  `qos statistics` Befehle ermöglichen die Anzeige von Leistungsdaten.
. Gegebenenfalls kann der  `qos policy-group modify` Befehl verwendet werden, um die maximale Durchsatzbegrenzung der Policy-Gruppe anzupassen.
+
Weitere Informationen zu  `qos policy-group modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/qos-policy-group-modify.html["ONTAP-Befehlsreferenz"^].
