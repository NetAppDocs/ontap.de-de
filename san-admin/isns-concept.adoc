= Internet Storage Name Service (iSNS) in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Internet Storage Name Service (iSNS) ist ein Protokoll, das die automatische Erkennung und Verwaltung von iSCSI-Geräten in einem TCP/IP-Speichernetzwerk ermöglicht. Ein iSNS-Server verwaltet Informationen über aktive iSCSI-Geräte im Netzwerk, einschließlich ihrer IP-Adressen, iSCSI-Node-Namen (IQNs) und Portalgruppen.

Sie können einen iSNS-Server von einem Drittanbieter beziehen. Wenn Sie in Ihrem Netzwerk einen iSNS-Server konfiguriert und für die Verwendung durch Initiator und Ziel aktiviert haben, kann die Management-LIF einer Storage Virtual Machine (SVM) verwendet werden, um alle iSCSI-LIFs dieser SVM auf dem iSNS-Server zu registrieren. Nach Abschluss der Registrierung kann der iSCSI-Initiator den iSNS-Server abfragen, um alle LIFs für diese SVM zu ermitteln.

Wenn Sie sich für die Nutzung eines iSNS-Service entscheiden, muss sichergestellt sein, dass Ihre Storage Virtual Machines (SVMs) ordnungsgemäß bei einem Internet Storage Name Service (iSNS) Server registriert sind.

Falls kein iSNS-Server im Netzwerk vorhanden ist, muss jedes Ziel manuell so konfiguriert werden, dass es für den Host sichtbar ist.



[[what-an-isns-server-does]]
== Was ein iSNS-Server tut

Ein iSNS Server verwendet das Internet Storage Name Service (iSNS) Protokoll, um Informationen über aktive iSCSI-Geräte im Netzwerk zu verwalten, einschließlich ihrer IP-Adressen, iSCSI-Node-Namen (IQNs) und Portalgruppen.

Das iSNS-Protokoll ermöglicht die automatische Erkennung und Verwaltung von iSCSI-Geräten in einem IP-Speichernetzwerk. Ein iSCSI-Initiator kann den iSNS-Server abfragen, um iSCSI-Zielgeräte zu entdecken.

NetApp liefert oder vertreibt keine iSNS-Server. Sie können diese Server von einem von NetApp unterstützten Anbieter beziehen.



[[how-svms-interact-with-an-isns-server]]
== Wie SVMs mit einem iSNS-Server interagieren

Der iSNS-Server kommuniziert mit jeder Storage Virtual Machine (SVM) über die SVM-Management-LIF. Die Management-LIF registriert alle iSCSI-Ziel Node-Name, Alias und Portalinformationen beim iSNS-Service für eine bestimmte SVM.

Im folgenden Beispiel verwendet die SVM „`VS1`“ die SVM Management LIF „`VS1_mgmt_lif`“, um sich beim iSNS Server zu registrieren. Während der iSNS-Registrierung sendet eine SVM alle iSCSI LIFs über die SVM Management LIF an den iSNS Server. Nach Abschluss der iSNS-Registrierung verfügt der iSNS Server über eine Liste aller LIFs, die iSCSI in „`VS1`“ bereitstellen. Wenn ein Cluster mehrere SVMs enthält, muss sich jede SVM einzeln beim iSNS Server registrieren, um den iSNS Service zu nutzen.

image:bsag_c-mode_iSNS_register.png["Beispiel 1 für die Interaktion zwischen SVM und iSNS Server"]

Im nächsten Beispiel kann Host A, nachdem der iSNS-Server die Registrierung beim Ziel abgeschlossen hat, alle LIFs für „VS1“ über den iSNS-Server ermitteln, wie in Schritt 1 angegeben. Nachdem Host A die LIFs für „VS1“ ermittelt hat, kann Host A eine Verbindung zu jeder beliebigen LIF in „VS1“ herstellen, wie in Schritt 2 gezeigt. Host A kennt keine der LIFs in „VS2“, bis die Management-LIF „VS2_mgmt_LIF“ für „VS2“ sich beim iSNS-Server registriert hat.

image:bsag_c-mode_iSNS_connect.png["Beispiel 2 für die Interaktion zwischen SVM und iSNS Server"]

Wenn Sie jedoch die Schnittstellenzugriffslisten definieren, kann der Host nur die in der Schnittstellenzugriffsliste definierten LIFs verwenden, um auf das Ziel zuzugreifen.

Nach der Erstkonfiguration von iSNS aktualisiert ONTAP den iSNS-Server automatisch, wenn sich die SVM-Konfigurationseinstellungen ändern.

Zwischen dem Zeitpunkt, zu dem Sie die Konfigurationsänderungen vornehmen, und dem Versand der Aktualisierung durch ONTAP an den iSNS-Server kann eine Verzögerung von einigen Minuten auftreten. Eine sofortige Aktualisierung der iSNS-Informationen auf dem iSNS-Server kann erzwungen werden: `vserver iscsi isns update`. Weitere Informationen zu  `vserver iscsi isns update` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-iscsi-isns-update.html["ONTAP-Befehlsreferenz"^].



[[commands-for-managing-isns]]
== Befehle zur Verwaltung von iSNS

ONTAP stellt Befehle zur Verwaltung Ihres iSNS-Dienstes bereit.

|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Konfiguration eines iSNS-Dienstes
 a| 
`vserver iscsi isns create`



 a| 
Einen iSNS-Dienst starten
 a| 
`vserver iscsi isns start`



 a| 
Einen iSNS Dienst modifizieren
 a| 
`vserver iscsi isns modify`



 a| 
iSNS-Dienstkonfiguration anzeigen
 a| 
`vserver iscsi isns show`



 a| 
Eine Aktualisierung der registrierten iSNS-Informationen erzwingen
 a| 
`vserver iscsi isns update`



 a| 
Einen iSNS-Dienst beenden
 a| 
`vserver iscsi isns stop`



 a| 
Einen iSNS Dienst entfernen
 a| 
`vserver iscsi isns delete`



 a| 
Die Manpage für einen Befehl anzeigen
 a| 
`man _command name_`

|===
Weitere Informationen zu  `vserver iscsi isns` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+iscsi+isns["ONTAP-Befehlsreferenz"^].
