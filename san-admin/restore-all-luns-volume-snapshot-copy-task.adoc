= Alle LUNs in einem Volume aus einem ONTAP Snapshot wiederherstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit dem  `volume snapshot restore`-Befehl lassen sich alle LUNs in einem angegebenen Volume aus einem Snapshot wiederherstellen.

.Schritte
. Vom Host aus ist jeglicher Hostzugriff auf die LUNs zu beenden.
+
Die Verwendung von SnapRestore ohne Unterbrechung des gesamten Hostzugriffs auf die LUNs im Volume kann zu Datenbeschädigung und Systemfehlern führen.

. Die LUNs auf diesem Host werden ausgehängt, sodass der Host nicht mehr auf die LUNs zugreifen kann.
. LUNs aufheben:
+
[source, cli]
----
lun mapping delete -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -igroup <igroup_name>
----
. Bestimmen Sie den Snapshot, auf den das Volume wiederhergestellt werden soll:
+
[source, cli]
----
volume snapshot show -vserver <SVM_name> -volume <volume_name>

----
. Die Berechtigungseinstellung auf „Erweitert“ ändern:
+
[source, cli]
----
set -privilege advanced
----
. Daten wiederherstellen:
+
[source, cli]
----
volume snapshot restore -vserver <SVM_name> -volume <volume_name> -snapshot <snapshot_name>
----
. Folgen Sie den Anweisungen auf dem Bildschirm.
. LUNs neu zuordnen:
+
[source, cli]
----
lun mapping create -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -igroup <igroup_name>
----
. Es sollte überprüft werden, ob die LUNs online sind:
+
[source, cli]
----
lun show -vserver <SVM_name> -path <lun_path> -fields state
----
. Falls Ihre LUNs nicht online sind, können sie online geschaltet werden:
+
[source, cli]
----
lun modify -vserver <SVM_name> -path <lun_path> -state online
----
. Die Berechtigungseinstellung auf Administrator ändern:
+
[source, cli]
----
set -privilege admin
----
. Vom Host aus die LUNs erneut einbinden.
. Vom Host aus wird der Zugriff auf Ihre LUNs neu gestartet.
