= Konfigurationseinstellungen für speicherplatzreservierte Dateien oder LUNs mit semi-dicker Volume-Bereitstellung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Diese FlexVol volume- und Datei- oder LUN-Konfigurationskombination erfordert weniger Speicherplatz, der im Voraus zugewiesen werden muss, als die vollständig bereitgestellte Kombination, schränkt jedoch die Effizienztechnologien ein, die für das Volume verwendet werden können. Überschreibungen werden bei dieser Konfigurationskombination nach dem Best-Effort-Prinzip erfüllt.

Folgende Einstellungen sind erforderlich, um eine speicherplatzreservierte LUN in einem Volume mithilfe von Semi-Thick Provisioning zu konfigurieren:

[cols="2*"]
|===
| Volume-Einstellung | Wert 


 a| 
Garantie
 a| 
Volumen



 a| 
Teilreserve
 a| 
0



 a| 
Snapshot-Reserve
 a| 
0



 a| 
Snapshot Autolöschung
 a| 
Aktiviert, mit der Verpflichtungsstufe zerstören, einer zerstören-Liste, die alle Objekte umfasst, dem auf Volume eingestellten Auslöser und allen FlexClone LUNs und FlexClone Dateien, die für die automatische Löschung aktiviert sind.



 a| 
Automatisches Wachstum
 a| 
Optional; falls aktiviert, muss der freie Speicherplatz des Aggregats aktiv überwacht werden.

|===

 a|

 a|

|===


[[technology-restrictions]]
== Technologische Beschränkungen

Die folgenden Technologien zur Steigerung der Speichereffizienz können für diese Konfigurationskombination nicht verwendet werden:

* Komprimierung
* Deduplizierung
* ODX und FlexClone Copy Offload
* FlexClone LUNs und FlexClone Dateien, die nicht zur automatischen Löschung markiert sind (aktive Klone)
* FlexClone Unterdateien
* ODX/Copy Offload




[[additional-considerations]]
== Weitere Überlegungen

Bei der Verwendung dieser Konfigurationskombination sind folgende Punkte zu berücksichtigen:

* Wenn auf dem Volume, das diese LUN unterstützt, der Speicherplatz knapp wird, werden die Schutzdaten (FlexClone LUNs und Dateien, Snapshots) zerstört.
* Schreibvorgänge können aufgrund eines Zeitlimits fehlschlagen, wenn der freie Speicherplatz auf dem Volume erschöpft ist.


Die Komprimierung ist für AFF Plattformen standardmäßig aktiviert. Die Komprimierung muss explizit für jedes Volume deaktiviert werden, für das Semi-Thick Provisioning auf einer AFF Plattform verwendet werden soll.
