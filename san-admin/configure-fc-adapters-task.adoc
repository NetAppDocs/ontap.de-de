= FC Adapter konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Onboard FC Adapter und einige FC-Erweiterungsadapterkarten können individuell als Initiator- oder Target-Ports konfiguriert werden. Andere FC-Erweiterungsadapter sind im Werk als Initiator- oder Target-Ports konfiguriert und können nicht geändert werden. Zusätzliche FC-Ports sind auch über unterstützte UTA2-Karten verfügbar, die mit FC SFP+ Adaptern konfiguriert sind.

Initiator-Ports können verwendet werden, um direkt mit Backend-Disk-Shelves und gegebenenfalls mit fremden Speichersystemen verbunden zu werden. Target-Ports können nur zur Verbindung mit FC-Switches verwendet werden.

Die Anzahl der Onboard-Ports und der für FC konfigurierten CNA/UTA2-Ports variiert je nach Controller-Modell. Auch die unterstützten Zielerweiterungsadapter sind modellabhängig. Siehe link:https://hwu.netapp.com["NetApp Hardware Universe"^] für eine vollständige Liste der Onboard-FC-Ports und der unterstützten Zielerweiterungsadapter für Ihr Controller-Modell.



[[configure-fc-adapters-for-initiator-mode]]
== FC-Adapter für den Initiatormodus konfigurieren

Der Initiatormodus wird verwendet, um die Ports mit Bandlaufwerken, Bandbibliotheken oder Storage von Drittanbietern link:https://docs.netapp.com/us-en/ontap-fli/get-started/learn-about-san-data-migration.html["Foreign LUN Import (FLI)"^] zu verbinden.

.Bevor Sie beginnen
* Die LIFs auf dem Adapter müssen aus allen Portsets entfernt werden, zu denen sie gehören.
* Alle LIFs aller Storage Virtual Machines (SVMs), die den zu modifizierenden physischen Port verwenden, müssen migriert oder zerstört werden, bevor die Persönlichkeit des physischen Ports von Target auf Initiator geändert wird.


[NOTE]
====
NVMe/FC unterstützt den Initiator-Modus nicht.

====
.Schritte
. Alle LIFs vom Adapter entfernen:
+
[source, cli]
----
network interface delete -vserver _SVM_name_ -lif _lif_name_,_lif_name_
----
. Adapter offline schalten:
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -status-admin down
----
+
Wenn der Adapter nicht offline geht, kann das Kabel auch vom entsprechenden Adapterport am System entfernt werden.

. Den Adapter von Ziel auf Initiator ändern:
+
[source, cli]
----
system hardware unified-connect modify -t initiator _adapter_port_
----
. Der Knoten, auf dem der geänderte Adapter gehostet wird, muss neu gestartet werden.
. Es sollte sichergestellt werden, dass die FC-Ports für Ihre Konfiguration im richtigen Zustand konfiguriert sind:
+
[source, cli]
----
system hardware unified-connect show
----
. Den Adapter wieder online bringen:
+
[source, cli]
----
node run -node _node_name_ storage enable adapter _adapter_port_
----




[[configure-fc-adapters-for-target-mode]]
== FC-Adapter für den Zielmodus konfigurieren

Der Zielmodus wird verwendet, um die Ports mit FC-Initiatoren zu verbinden.

Die Konfiguration von FC-Adaptern für das FC-Protokoll und das FC-NVMe-Protokoll erfolgt nach denselben Schritten. Allerdings unterstützen nur bestimmte FC-Adapter FC-NVMe. Siehe link:https://hwu.netapp.com["NetApp Hardware Universe"^] für eine Liste der Adapter, die das FC-NVMe-Protokoll unterstützen.

.Schritte
. Den Adapter offline schalten:
+
[source, cli]
----
node run -node _node_name_ storage disable adapter _adapter_name_
----
+
Wenn der Adapter nicht offline geht, kann das Kabel auch vom entsprechenden Adapterport am System entfernt werden.

. Den Adapter von Initiator auf Ziel ändern:
+
[source, cli]
----
system node hardware unified-connect modify -t target -node _node_name_ adapter _adapter_name_
----
. Der Knoten, auf dem der geänderte Adapter gehostet wird, muss neu gestartet werden.
. Es sollte sichergestellt werden, dass der Zielport die korrekte Konfiguration aufweist:
+
[source, cli]
----
network fcp adapter show -node _node_name_
----
. Adapter online schalten:
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -state up
----




[[configure-fc-adapters-for-direct-attachment-to-fc-and-fc-nvme-hosts]]
== FC-Adapter für den direkten Anschluss an FC- und FC-NVMe-Hosts konfigurieren

Ab ONTAP 9.19.1 ist kein FC-Switch erforderlich, um Verbindungen zwischen FC- und FC-NVMe-Hosts und ONTAP Storage herzustellen. Hosts können direkt an FC-Adapterports von AFF, ASA, ASA r2 und FAS Systemen angeschlossen werden. Die Direktanschlussfähigkeit kann besonders an Remote-Standorten und in hybriden Umgebungen wertvoll sein, in denen Einfachheit entscheidend ist. Sie müssen den FC-Port offline nehmen und seine Topologie ändern, bevor er direkt an einen Host angeschlossen werden kann.

Für FC-Ports, die in Direktanschlusskonfigurationen verwendet werden, gelten folgende Einschränkungen:

* Der Port kann nur einer Storage-VM (SVM) zugewiesen werden.
* Nach der Konfiguration des Ports für den direkten Anschluss kann diesem nur eine Netzwerkschnittstelle (LIF) zugewiesen werden.
* Der Port kann entweder das FC-Protokoll oder das FC-NVMe-Protokoll unterstützen, aber nicht beide.


.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein.

[role="tabbed-block"]
====
.System Manager
--
. Im System Manager wählen Sie *Netzwerk > FC ports*.
. Wählen Sie image:icon_kabob.gif["Auswahlsymbol"] neben dem Port, den Sie für die direkte Verbindung konfigurieren möchten, und wählen Sie dann *Topologie bearbeiten*.
. Unter *Topologie* wählen Sie *Direkt* und dann *Ich möchte die Topologie ändern und den Port offline nehmen*.
. Wählen Sie *Speichern* aus.
. Verbinden Sie das Kabel für den Zielport direkt mit dem Host-Initiator.
. Wenn Sie ein AFF, ASA oder FAS-System haben, befolgen Sie diese Schritte, um eine LIF zu erstellen: Wenn Sie ein ASA r2-System haben, folgen Sie link:https://docs.netapp.com/us-en/asa-r2/administer/manage-client-vm-access.html#create-a-lif-network-interface["dieses System Manager-Verfahren"^], um eine LIF zu erstellen.
+
.. Wählen Sie *Netzwerk > Schnittstellen* und anschließend image:icon_add.gif["Symbol hinzufügen"].
.. Wählen Sie das FC- oder NVMe/FC-Protokoll.
.. Benennen Sie die LIF oder übernehmen Sie den von ONTAP generierten Namen.
.. Wählen Sie den Home-Port aus.
.. *Speichern* auswählen.
+
Der Port wird automatisch online geschaltet, sobald das LIF erstellt wird.





.Was kommt als Nächstes?
Im System Manager wählen Sie *Netzwerk > FC ports* und überprüfen Sie anschließend, ob die *Topologie* als *Direct* angezeigt wird.

--
.CLI
--
. Vergewissern Sie sich, dass der Port, den Sie für die direkte Verbindung verwenden möchten, die direkte Topologie unterstützt:
+
[source, cli]
----
network fcp adapter show -fields topologies-supported
----
. Sehen Sie sich den administrativen und betrieblichen Status des für den Adapterport verwendeten LIF an:
+
[source, cli]
----
network interface show -vserver <SVM_name>
----
. Nehmen Sie die für den Adapterport verwendete LIF offline:
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif * -home-port <adapter_port> -status-admin down
----
. Entfernen Sie alle LIFs vom Adapterport:
+
[source, cli]
----
network interface delete -vserver <SVM_name> -lif * -home-port <adapter_port>
----
. Den Adapter offline schalten:
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -status-admin down
----
. Ändern Sie die Adapterport-Topologie auf Direkt:
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -topology direct
----
. Verbinden Sie das Kabel für den Zielport direkt mit dem Host-Initiator.
. Erstellen einer einzelnen LIF-Schnittstelle:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -data protocol fcp|fcp-nvme -home-node <node_name> -home-port <port_name>
----
+
Der Port wird nach der Erstellung der LIF automatisch aktiviert.

. Überprüfen Sie, ob der Port für die direkte Verbindung konfiguriert ist:
+
[source, cli]
----
network fcp interface show
----
+
Die Topologie sollte als direkt angezeigt werden.



--
====

[[configure-fc-adapter-speed]]
== FC-Adaptergeschwindigkeit konfigurieren

Sie sollten die Zielportgeschwindigkeit Ihres Adapters an die Geschwindigkeit des Geräts anpassen, mit dem er verbunden ist, anstatt die automatische Aushandlung zu verwenden. Ein Port, der auf automatische Aushandlung eingestellt ist, kann nach einer Übernahme/Rückgabe oder einer anderen Unterbrechung länger für die Wiederherstellung der Verbindung benötigen.

.Über diese Aufgabe
Da diese Aufgabe alle Storage Virtual Machines (SVMs) und alle LIFs in einem Cluster umfasst, müssen die `-home-port` und `-home-lif` Parameter verwendet werden, um den Umfang dieser Operation einzuschränken. Wenn diese Parameter nicht verwendet werden, gilt die Operation für alle LIFs im Cluster, was möglicherweise nicht erwünscht ist.

.Bevor Sie beginnen
Alle LIFs, die diesen Adapter als ihren Home-Port verwenden, müssen offline sein.

.Schritte
. Alle LIFs dieses Adapters offline schalten:
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin down
----
. Den Adapter offline schalten:
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state down
----
+
Wenn der Adapter nicht offline geht, kann das Kabel auch vom entsprechenden Adapterport am System entfernt werden.

. Die maximale Geschwindigkeit für den Portadapter bestimmen:
+
[source, cli]
----
fcp adapter show -instance
----
+
Die Adaptergeschwindigkeit kann nicht über die maximale Geschwindigkeit hinaus verändert werden.

. Adaptergeschwindigkeit ändern:
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -speed 16
----
. Den Adapter online schalten:
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state up
----
. Alle LIFs auf dem Adapter online schalten:
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin up
----


.Verwandte Informationen
* Informationen dazu link:limit-lun-access-portsets-igroups-concept.html["Möglichkeiten zur Einschränkung des LUN-Zugriffs mit Portsätzen und Initiatorgruppen"].
* Erfahren Sie, wie link:selective-lun-map-concept.html#determine-whether-slm-is-enabled-on-a-lun-map["Selektives LUN Mapping (SLM)"] die Pfade eingeschränkt werden, die für den Zugriff auf die LUNs eines HA-Paares verwendet werden.
