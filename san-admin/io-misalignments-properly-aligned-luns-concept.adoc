= Überblick über E/A-Fehlausrichtungen bei korrekt ausgerichteten LUNs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP meldet möglicherweise E/A-Fehlausrichtungen bei korrekt ausgerichteten LUNs. Im Allgemeinen können diese Fehlausrichtungswarnungen ignoriert werden, sofern Sie sicher sind, dass Ihre LUN korrekt bereitgestellt ist und Ihre Partitionstabelle korrekt ist.

LUNs und Festplatten stellen beide Speicher als Blöcke bereit. Da die Blockgröße von Festplatten auf dem Host 512 Byte beträgt, präsentieren LUNs dem Host Blöcke dieser Größe, verwenden jedoch tatsächlich größere 4-KB-Blöcke zur Datenspeicherung. Der vom Host verwendete 512-Byte-Datenblock wird als logischer Block bezeichnet. Der von der LUN zur Datenspeicherung verwendete 4-KB-Datenblock wird als physischer Block bezeichnet. Das bedeutet, dass sich in jedem 4-KB-physikalischen Block acht logische 512-Byte-Blöcke befinden.

image:bsag-cmode-lbpb.gif["Logische Blöcke in physischen Blöcken auf LUNs"]

Das Host-Betriebssystem kann einen Lese- oder Schreib-E/A-Vorgang an jedem logischen Block starten. E/A-Vorgänge gelten nur als ausgerichtet, wenn sie am ersten logischen Block im physischen Block beginnen. Beginnt ein E/A-Vorgang an einem logischen Block, der nicht auch der Anfang eines physischen Blocks ist, gilt der E/A-Vorgang als nicht ausgerichtet. ONTAP erkennt die Fehlausrichtung automatisch und meldet sie auf der LUN. Das Vorhandensein nicht ausgerichteter E/A-Vorgänge bedeutet jedoch nicht zwangsläufig, dass die LUN ebenfalls nicht ausgerichtet ist. Es ist möglich, dass nicht ausgerichtete E/A-Vorgänge auch auf korrekt ausgerichteten LUNs gemeldet werden.

Falls weitere Untersuchungen erforderlich sind, siehe die link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/How_to_identify_unaligned_IO_on_LUNs["NetApp Knowledge Base: Wie können nicht ausgerichtete IOs auf LUNs identifiziert werden?"^]

[[achieve-io-alignment-using-lun-os-types]]
== E/A-Ausrichtung mithilfe von LUN-OS-Typen erzielen

Für ONTAP 9.7 oder früher sollte der empfohlene ONTAP LUN `ostype` Wert verwendet werden, der am besten zu Ihrem Betriebssystem passt, um eine I/O-Ausrichtung mit Ihrem Betriebssystem-Partitionierungsschema zu erzielen.

Das vom Host-Betriebssystem verwendete Partitionierungsschema trägt maßgeblich zu E/A-Fehlausrichtungen bei. Einige ONTAP LUN `ostype`-Werte verwenden einen speziellen Offset, das sogenannte „Präfix“, damit das standardmäßige Partitionierungsschema des Host-Betriebssystems ausgerichtet werden kann.


NOTE: Unter bestimmten Umständen ist möglicherweise eine benutzerdefinierte Partitionstabelle erforderlich, um eine I/O-Ausrichtung zu erreichen. Bei  `ostype`-Werten mit einem "`prefix`"-Wert größer als  `0` kann eine benutzerdefinierte Partition jedoch zu einer nicht ausgerichteten I/O führen.

Weitere Informationen zu LUNs, die in ONTAP 9.7 oder früher bereitgestellt wurden, finden sich unter link:https://kb.netapp.com/onprem/ontap/da/SAN/How_to_identify_unaligned_IO_on_LUNs["NetApp Knowledge Base: Wie nicht ausgerichtete E/A auf LUNs identifiziert werden können"^].


NOTE: Standardmäßig haben neue LUNs, die in ONTAP 9.8 oder höher bereitgestellt werden, für alle LUN-OS-Typen eine Präfix- und Suffixgröße von Null. Die I/O sollte standardmäßig mit dem unterstützten Host-OS abgestimmt sein.



[[special-io-alignment-considerations-for-linux]]
== Besondere Überlegungen zur I/O-Ausrichtung für Linux

Linux-Distributionen bieten vielfältige Möglichkeiten zur Verwendung von LUNs, beispielsweise als Rohgeräte für Datenbanken, verschiedene Volume-Manager und Dateisysteme. Es ist nicht erforderlich, Partitionen auf einer LUN zu erstellen, wenn sie als Rohgerät oder als physisches Volume in einem logischen Volume verwendet wird.

Für RHEL 5 und frühere Versionen sowie SLES 10 und frühere Versionen gilt: Wenn die LUN ohne Volume-Manager verwendet wird, sollte die LUN so partitioniert werden, dass sie eine Partition hat, die bei einem ausgerichteten Offset beginnt, also einem Sektor, der ein gerades Vielfaches von acht logischen Blöcken ist.

[[esx-boot-luns-report-as-misaligned]]
== ESX Boot-LUNs werden als falsch ausgerichtet gemeldet

Als ESX-Boot-LUNs verwendete LUNs werden von ONTAP typischerweise als nicht ausgerichtet gemeldet. ESX erstellt mehrere Partitionen auf der Boot-LUN, was eine Ausrichtung sehr schwierig macht. Nicht ausgerichtete ESX-Boot-LUNs stellen in der Regel kein Leistungsproblem dar, da die Gesamtmenge der nicht ausgerichteten E/A gering ist. Sofern die LUN mit der VMware `ostype` korrekt bereitgestellt wurde, sind keine Maßnahmen erforderlich.
