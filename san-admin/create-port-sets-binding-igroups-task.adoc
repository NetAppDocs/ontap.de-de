= Ein Portset erstellen und an eine Initiatorgruppe binden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Zusätzlich zur Verwendung von link:selective-lun-map-concept.html["Selective LUN Map (SLM)"] kann ein Portset erstellt und das Portset an eine Initiatorgruppe gebunden werden, um weiter einzuschränken, welche LIFs von einem Initiator für den Zugriff auf eine LUN verwendet werden können.

Wenn kein Portset an eine Initiatorgruppe gebunden ist, können alle Initiatoren in der Initiatorgruppe auf zugeordnete LUNs über alle LIFs auf dem Knoten, dem die LUN gehört, sowie auf dem HA-Partner des besitzenden Knotens zugreifen.

.Bevor Sie beginnen
Sie benötigen mindestens eine LIF und eine Initiatorgruppe.

Sofern Sie keine Schnittstellengruppen verwenden, werden für iSCSI und FC zwei LIFs zur Redundanz empfohlen. Für Schnittstellengruppen wird nur ein LIF empfohlen.

.Über diese Aufgabe
Die Verwendung von Portsets mit SLM ist vorteilhaft, wenn auf einem Knoten mehr als zwei LIFs vorhanden sind und ein bestimmter Initiator auf eine Teilmenge der LIFs beschränkt werden soll. Ohne Portsets sind alle Ziele auf dem Knoten für alle Initiatoren mit Zugriff auf die LUN über den Knoten, dem die LUN gehört, und den HA-Partner des Eigentümerknotens zugänglich.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.10.1 ist es möglich, mit System Manager Portsets zu erstellen und sie an Initiatorgruppen zu binden.

Wenn ein Portset erstellt und an eine Initiatorgruppe in einer ONTAP Version vor 9.10.1 gebunden werden soll, ist das ONTAP CLI Verfahren zu verwenden.

Ab ONTAP 9.12.1 ist, falls noch kein Portset vorhanden ist, das erste Portset mithilfe der ONTAP CLI-Prozedur zu erstellen.

. Im System Manager auf *Netzwerk > Übersicht > Portsets* klicken und anschließend auf *Hinzufügen*.
. Geben Sie die Informationen für das neue Portset ein und wählen Sie *Hinzufügen* aus.
. *Hosts > SAN Initiator Groups* auswählen.
. Zum Binden des Portsets an eine neue Initiatorgruppe auf *Hinzufügen* klicken.
+
Um das Portset an eine bestehende Initiatorgruppe zu binden, wählen Sie die Initiatorgruppe aus, klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"], und dann auf *Initiatorgruppe bearbeiten*.



.Verwandte Informationen
link:manage-san-initiators-task.html["Initiatoren und Initiatorgruppen anzeigen und verwalten"]

--
.CLI
--
. Einen Portsatz erstellen, der die entsprechenden LIFs enthält:
+
`portset create -vserver _vserver_name_ -portset _portset_name_ -protocol _protocol_ -port-name _port_name_`

+
Wenn Sie FC verwenden, geben Sie den  `protocol` Parameter als  `fcp` an. Wenn Sie iSCSI verwenden, geben Sie den  `protocol` Parameter als  `iscsi` an.

. Die Initiatorgruppe an den Portsatz binden:
+
`lun igroup bind -vserver _vserver_name_ -igroup _igroup_name_ -portset _portset_name_`

+
Weitere Informationen zu  `lun igroup bind` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/lun-igroup-bind.html["ONTAP-Befehlsreferenz"^].

. Überprüfen, ob die Port-Sets und LIFs korrekt sind:
+
`portset show -vserver _vserver_name_`

+
[listing]
----
Vserver   Portset   Protocol Port Names    Igroups
--------- --------- -------- ------------- --------
vs3       portset0  iscsi    lif0,lif1     igroup1
----
