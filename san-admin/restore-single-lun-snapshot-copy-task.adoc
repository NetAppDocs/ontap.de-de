= Wiederherstellung einer einzelnen LUN aus einem ONTAP Snapshot
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine einzelne LUN aus einem Snapshot wiederherstellen, ohne das gesamte Volume, das die einzelne LUN enthält, wiederherzustellen. Die LUN kann am selben Ort oder an einem neuen Pfad im Volume wiederhergestellt werden. Der Vorgang stellt nur die einzelne LUN wieder her, ohne andere Dateien oder LUNs im Volume zu beeinträchtigen. Auch Dateien mit Streams können wiederhergestellt werden.

.Bevor Sie beginnen
* Sie benötigen ausreichend Speicherplatz auf Ihrem Volume, um den Wiederherstellungsvorgang abzuschließen:
+
** Wenn Sie eine speicherplatzreservierte LUN wiederherstellen, bei der die fraktionelle Reserve 0 % beträgt, wird der einmalige Speicherplatzbedarf in der Größe der wiederhergestellten LUN benötigt.
** Wenn Sie eine speicherplatzreservierte LUN wiederherstellen, bei der die Teilreserve 100 % beträgt, benötigen Sie die doppelte Größe der wiederherzustellenden LUN.
** Wenn Sie eine nicht speicherplatzreservierte LUN wiederherstellen, wird nur der tatsächlich für die wiederhergestellte LUN verwendete Speicherplatz benötigt.


* Ein Snapshot der Ziel-LUN muss erstellt worden sein.
+
Schlägt die Wiederherstellung fehl, kann die Ziel-LUN abgeschnitten werden. In solchen Fällen lässt sich der Snapshot verwenden, um Datenverlust zu verhindern.

* Es muss ein Snapshot der Quell-LUN erstellt worden sein.
+
In seltenen Fällen kann die Wiederherstellung der LUN fehlschlagen, wodurch die Quell-LUN unbrauchbar wird. Tritt dies auf, kann mithilfe des Snapshots die LUN in den Zustand unmittelbar vor dem Wiederherstellungsversuch zurückgesetzt werden.

* Die Ziel-LUN und die Quell-LUN müssen denselben Betriebssystemtyp aufweisen.
+
Wenn Ihre Ziel-LUN einen anderen Betriebssystemtyp als Ihre Quell-LUN aufweist, kann Ihr Host nach dem Wiederherstellungsvorgang den Datenzugriff auf die Ziel-LUN verlieren.



.Schritte
. Vom Host aus sollte jeglicher Hostzugriff auf die LUN beendet werden.
. Hängen Sie die LUN auf dem Host aus, sodass der Host nicht mehr auf die LUN zugreifen kann.
. LUN-Zuordnung aufheben:
+
[source, cli]
----
lun mapping delete -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -igroup <igroup_name>
----
. Bestimmen Sie den Snapshot, auf den Ihre LUN wiederhergestellt werden soll:
+
[source, cli]
----
volume snapshot show -vserver <SVM_name> -volume <volume_name>
----
. Vor der Wiederherstellung der LUN einen Snapshot der LUN erstellen:
+
[source, cli]
----
volume snapshot create -vserver <SVM_name> -volume <volume_name> -snapshot <snapshot_name>
----
. Die angegebene LUN in einem Volume wiederherstellen:
+
[source, cli]
----
volume snapshot restore-file -vserver <SVM_name> -volume <volume_name> -snapshot <snapshot_name> -path <lun_path>
----
. Folgen Sie den Anweisungen auf dem Bildschirm.
. Falls erforderlich kann die LUN online geschaltet werden:
+
[source, cli]
----
lun modify -vserver <SVM_name> -path <lun_path> -state online
----
. Falls erforderlich kann die LUN neu zugeordnet werden:
+
[souce]
----
lun mapping create -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -igroup <igroup_name>
----
. Vom Host aus die LUN erneut einbinden.
. Vom Host aus wird der Zugriff auf die LUN neu gestartet.
