= SAN-Bereitstellung mit FC
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es ist wichtig, die grundlegenden Konzepte zu kennen, die zum Verständnis der Implementierung eines FC SAN durch ONTAP erforderlich sind.



[[how-fc-target-nodes-connect-to-the-network]]
== Wie FC-Zielknoten mit dem Netzwerk verbunden sind

Speichersysteme und Hosts verfügen über Adapter, sodass sie mit Kabeln an FC-Switches angeschlossen werden können.

Wenn ein Node mit dem FC SAN verbunden ist, registriert jede SVM den World Wide Port Name (WWPN) ihrer logischen Schnittstelle (LIF) beim Fabric Name Service des Switches. Der WWNN der SVM und der WWPN jeder logischen Schnittstelle (LIF) werden automatisch von ONTAP zugewiesen.

[NOTE]
====
Direkte Verbindungen von Hosts mit FC zu Nodes werden nicht unterstützt, NPIV ist erforderlich und dies erfordert die Verwendung eines Switches. Bei iSCSI-Sitzungen funktioniert die Kommunikation sowohl mit netzwerkgerouteten Verbindungen als auch mit Direktverbindungen. Beide Methoden werden jedoch von ONTAP unterstützt.

====


[[how-fc-nodes-are-identified]]
== Wie FC-Nodes identifiziert werden

Jede mit FC konfigurierte SVM ist durch einen weltweiten Node-Name (WWNN) identifiziert.



[[how-wwpns-are-used]]
== Wie WWPNs verwendet werden

WWPNs identifizieren jede LIF in einer SVM, die für FC konfiguriert ist. Diese LIFs nutzen die physischen FC-Ports in jedem Node im Cluster, wobei es sich um FC-Zieladapter, UTA oder UTA2 handeln kann, die in den Nodes als FC oder FCoE konfiguriert sind.

* Erstellung einer Initiatorgruppe
+
Die WWPNs der HBAs des Hosts werden verwendet, um eine Initiatorgruppe (igroup) zu erstellen. Eine Initiatorgruppe wird verwendet, um den Hostzugriff auf bestimmte LUNs zu steuern. Eine Initiatorgruppe kann erstellt werden, indem eine Sammlung von WWPNs von Initiatoren in einem FC-Netzwerk angegeben wird. Wenn eine LUN auf einem Speichersystem einer Initiatorgruppe zugeordnet wird, erhalten alle Initiatoren dieser Gruppe Zugriff auf diese LUN. Befindet sich die WWPN eines Hosts nicht in einer Initiatorgruppe, die einer LUN zugeordnet ist, hat dieser Host keinen Zugriff auf die LUN. Das bedeutet, dass die LUNs auf diesem Host nicht als Datenträger angezeigt werden.

+
Sie können auch Portsets erstellen, um eine LUN nur auf bestimmten Zielports sichtbar zu machen. Ein Portset besteht aus einer Gruppe von FC-Zielports. Eine Initiatorgruppe kann an ein Portset gebunden werden. Jeder Host in der Initiatorgruppe kann auf die LUNs nur zugreifen, indem er eine Verbindung zu den Zielports im Portset herstellt.

* Eindeutige Identifizierung von FC LIFs
+
WWPNs identifizieren eindeutig jede FC logische Schnittstelle. Das Host-Betriebssystem verwendet die Kombination aus WWNN und WWPN, um SVMs und FC LIFs zu identifizieren. Einige Betriebssysteme erfordern persistentes Binden, damit die LUN auf dem Host unter derselben Ziel-ID erscheint.





[[how-worldwide-name-assignments-work]]
== Wie die weltweite Namensvergabe funktioniert

Weltweite Namen werden in ONTAP sequenziell erstellt. Aufgrund der Art und Weise, wie ONTAP sie zuweist, kann es jedoch so erscheinen, als ob sie in einer nicht-sequenziellen Reihenfolge vergeben werden.

Jeder Adapter verfügt über eine vorkonfigurierte WWPN und WWNN, aber ONTAP verwendet diese vorkonfigurierten Werte nicht. Stattdessen weist ONTAP eigene WWPNs oder WWNNs zu, basierend auf den MAC-Adressen der integrierten Ethernet-Ports.

Die weltweiten Namen erscheinen bei ihrer Vergabe möglicherweise nicht fortlaufend, aus folgenden Gründen:

* Weltweit werden Namen für alle Nodes und Storage Virtual Machines (SVMs) im Cluster vergeben.
* Weltweit freigegebene Namen werden wiederverwendet und dem Pool verfügbarer Namen hinzugefügt.
