= Empfohlene Konfiguration zum Anschluss eines Host-Backup-Systems an ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SAN-Systeme lassen sich über einen separaten Backup-Host auf Band sichern, wodurch Leistungseinbußen auf dem Anwendungshost vermieden werden.

Für Backupzwecke ist es unerlässlich, SAN- und NAS-Daten getrennt zu speichern. Die folgende Abbildung zeigt die empfohlene physische Konfiguration für ein Host-Backup-System auf dem primären Storage-System. Volumes müssen ausschließlich als SAN konfiguriert werden. LUNs können auf ein einzelnes Volume beschränkt oder auf mehrere Volumes oder Storage-Systeme verteilt werden.

image:drw-tapebackupsan-scrn-en.gif["Tape-Backup für SAN"]

Volumes auf einem Host können aus einer einzelnen LUN bestehen, die vom Storage-System gemappt wird, oder aus mehreren LUNs mithilfe eines Volume-Managers wie VxVM auf HP-UX-Systemen.
