= Einem NVMe-Namensraum einem Subsystem zuordnen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Durch die Zuordnung eines NVMe-Namensraums zu einem Subsystem wird der Datenzugriff von Ihrem Host aus ermöglicht. Ein NVMe-Namensraum kann einem Subsystem entweder bei der Bereitstellung des Speichers oder nach der Bereitstellung zugeordnet werden.

Ab ONTAP 9.17.1 kann bei Verwendung einer SnapMirror Active Sync Konfiguration eine SVM als proximaler vServer zu einem Host hinzugefügt werden, während der Host zu einem NVMe Subsystem hinzugefügt wird. Aktiv-optimierte Pfade für einen Namespace in einem NVMe Subsystem werden nur von der als proximaler vServer konfigurierten SVM an einen Host veröffentlicht.

Ab ONTAP 9.14.1 kann die Ressourcenzuweisung für bestimmte Hosts priorisiert werden. Standardmäßig erhält ein Host beim Hinzufügen zum NVMe Subsystem die Priorität Normal. Die Standardpriorität kann über die ONTAP command line interface (CLI) manuell von Normal auf Hoch geändert werden. Hosts mit hoher Priorität werden größere I/O-Warteschlangenzahlen und Warteschlangentiefen zugewiesen.


NOTE: Wenn einem Host, der in ONTAP 9.13.1 oder früher zu einem Subsystem hinzugefügt wurde, eine hohe Priorität zugewiesen werden soll, ist dies möglich xref:../nvme/change-host-priority-nvme-task.html[die Host-Priorität ändern].

.Bevor Sie beginnen
Ihr Namespace und Ihr Subsystem sollten bereits erstellt sein. Falls ein Namespace und ein Subsystem erstellt werden müssen, siehe link:create-nvme-namespace-subsystem-task.html["NVMe Storage bereitstellen"].



[[map-an-nvme-namespace]]
== Einem NVMe-Namensraum eine Zuordnung zuweisen

.Schritte
. Den NQN vom Host abrufen.
. Den Host-NQN zum Subsystem hinzufügen:
+
[source, cli]
----
vserver nvme subsystem host add -vserver <SVM_name> -subsystem <subsystem_name> -host-nqn <Host_NQN_:subsystem._subsystem_name>
----
+
Wenn die Standardpriorität des Hosts von „Normal“ auf „Hoch“ geändert werden soll, steht die  `-priority high` Option zur Verfügung. Diese Option ist ab ONTAP 9.14.1 verfügbar. Weitere Informationen zu  `vserver nvme subsystem host add` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-nvme-subsystem-host-add.html["ONTAP-Befehlsreferenz"^].

+
Wenn eine SVM als `proximal-vserver` zu einem Host hinzugefügt werden soll, während der Host zu einem NVMe-Subsystem in einer SnapMirror active sync Konfiguration hinzugefügt wird, kann die  `-proximal-vservers` Option verwendet werden. Diese Option ist ab ONTAP 9.17.1 verfügbar. Es kann die Quell- oder Ziel-SVM oder beide hinzugefügt werden. Die SVM, in der dieser Befehl ausgeführt wird, ist die Standard-SVM.

. Den Namensraum dem Subsystem zuordnen:
+
[source, cli]
----
vserver nvme subsystem map add -vserver <SVM_name> -subsystem <subsystem_name> -path <path>
----
+
Ein Namensraum kann nur einem einzigen Subsystem zugeordnet werden. Weitere Informationen zu  `vserver nvme subsystem map add` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-nvme-subsystem-map-add.html["ONTAP-Befehlsreferenz"^].

. Es sollte überprüft werden, ob der Namensraum dem Subsystem zugeordnet ist:
+
[source, cli]
----
vserver nvme namespace show -vserver <SVM_name> -instance
----
+
Das Subsystem sollte als  `Attached subsystem` aufgeführt sein. Weitere Informationen zu  `vserver nvme namespace show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-nvme-namespace-show.html["ONTAP-Befehlsreferenz"^].
