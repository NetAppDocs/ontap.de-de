= Eine SVM für FC konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um eine Storage Virtual Machine (SVM) für FC zu konfigurieren, müssen LIFs für die SVM erstellt und diesen LIFs das FC-Protokoll zugewiesen werden.

.Über diese Aufgabe
NetApp unterstützt mindestens ein FC-LIF pro Knoten für jede SVM, die Daten mit dem FC-Protokoll bereitstellt. Es sind zwei LIFs pro Knoten und zwei Fabrics zu verwenden, wobei jedem Knoten ein LIF zugeordnet ist. Dadurch entsteht Redundanz auf der Knotenebene und im Fabric.

.Bevor Sie beginnen
* Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.
* Sie benötigen eine FC-Lizenz (link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["im Lieferumfang von ONTAP One enthalten"]) und diese muss aktiviert sein. Ist die FC-Lizenz nicht aktiviert, werden die LIFs und SVMs zwar als online angezeigt, aber der Betriebsstatus ist  `down`. Der FC-Dienst muss aktiviert sein, damit Ihre LIFs und SVMs betriebsbereit sind. Für alle FC LIFs in der SVM ist Single Initiator Zoning zu verwenden, um die Initiatoren zu hosten.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Eine Storage-VM für iSCSI mit ONTAP System Manager (9.7 und höher) konfigurieren.

[cols="2"]
|===
| FC auf einer neuen Storage-VM konfigurieren | FC auf einer vorhandenen Storage-VM konfigurieren 


 a| 
. Im System Manager wählen Sie *Storage > Storage VMs* und anschließend *Add*.
. Geben Sie einen Namen für die Storage-VM ein.
. *FC* für das *Zugriffsprotokoll* auswählen.
. Wählen Sie *FC aktivieren*.
+
Die FC-Ports werden automatisch zugewiesen.

. *Speichern* auswählen.

 a| 
. Im System Manager unter *Storage > Storage-VMs* auswählen.
. Wählen Sie die Storage-VM aus, die Sie konfigurieren möchten.
. Wählen Sie auf der Registerkarte *Einstellungen* und anschließend image:icon_gear.gif["Aktionssymbol"] neben dem FC-Protokoll aus.
. Wählen Sie *FC aktivieren* und geben Sie die IP-Adresse und Subnetzmaske für die Netzwerkschnittstelle ein. + Die FC-Ports werden automatisch zugewiesen.
. *Speichern* auswählen.


|===
--
.CLI
--
. FC-Dienst auf der SVM aktivieren:
+
[source, cli]
----
vserver fcp create -vserver <vserver_name> -status-admin up
----
. Zwei LIFs für die SVMs auf jedem Knoten, der FC bereitstellt, erstellen.
+
** Für ONTAP 9.6 und höher:
+
[source, cli]
----
network interface create -vserver <vserver_name> -lif <lif_name> -data-protocol fcp -service-policy default-data-fcp -home-node <node_name> -home-port <port_name> -address <ip_address> -netmask <netmask> -status-admin up
----
** Für ONTAP 9.5 und frühere Versionen:
+
[source, cli]
----
network interface create -vserver <vserver_name>_ -lif <lif_name> -role data -data-protocol fcp -home-node <node_name> -home-port <port_name>
----


. Es wird überprüft, ob Ihre LIFs erstellt wurden und ob ihr Betriebsstatus  `online` ist:
+
[source, cli]
----
network interface show -vserver <vserver_name> <lif_name>
----
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].
