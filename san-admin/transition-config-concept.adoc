= Fähigkeiten und Einschränkungen von migrierten LUNs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In einer SAN-Umgebung ist während der Transition eines 7-Mode-Volumes zu ONTAP eine Dienstunterbrechung erforderlich. Zum Abschluss der Transition ist das Herunterfahren der Hosts notwendig. Nach der Transition müssen die Host-Konfigurationen aktualisiert werden, bevor die Datenbereitstellung in ONTAP beginnen kann.

Ein Wartungsfenster ist einzuplanen, in dem Sie Ihre Hosts herunterfahren und den Übergang abschließen können.

LUNs, die von Data ONTAP im 7-Mode-Betrieb auf ONTAP migriert wurden, verfügen über bestimmte Fähigkeiten und Einschränkungen, die die Verwaltung der LUNs beeinflussen.

Mit migrierten LUNs sind folgende Aktionen möglich:

* Die LUN kann mit dem  `lun show` Befehl angezeigt werden.
* Die Liste der aus dem 7-Mode-Volume migrierten LUNs kann mit dem  `transition 7-mode show` Befehl angezeigt werden.
* Wiederherstellung eines Volumes aus einem 7-Mode-Snapshot
+
Durch die Wiederherstellung des Volumes werden alle im Snapshot erfassten LUNs in den vorherigen Zustand zurückversetzt.

* Eine einzelne LUN aus einem 7-Mode Snapshot mithilfe des  `snapshot restore-file` Befehls wiederherstellen
* Einen Klon einer LUN in einem 7-Mode-Snapshot erstellen
* Einen Blockbereich einer LUN wiederherstellen, der in einem 7-Mode Snapshot erfasst wurde
* Einen FlexClone des Volumes mithilfe eines 7-Mode-Snapshots erstellen


Folgendes ist bei migrierten LUNs nicht möglich:

* Auf Snapshot-gesicherte LUN-Klone, die im Volume erfasst wurden, kann zugegriffen werden
