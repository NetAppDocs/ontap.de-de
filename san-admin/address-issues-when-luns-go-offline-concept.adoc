= Möglichkeiten zur Behebung von Problemen, wenn LUNs offline gehen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn kein Speicherplatz mehr für Schreibvorgänge verfügbar ist, werden LUNs offline geschaltet, um die Datenintegrität zu gewährleisten. LUNs können aus verschiedenen Gründen keinen Speicherplatz mehr haben und offline gehen, und es gibt mehrere Möglichkeiten, dieses Problem zu beheben.

[cols="2*"]
|===
| Wenn die... | Sie können... 


 a| 
Das Aggregat ist voll
 a| 
* Weitere Festplatten hinzufügen.
* Mit dem  `volume modify` Befehl lässt sich ein Volume mit verfügbarem Speicherplatz verkleinern.
* Wenn Volumes mit Speicherplatzzusage vorhanden sind, die über verfügbaren Speicherplatz verfügen, kann die Speicherplatzzusage des Volumes auf `none`mit dem `volume modify`Befehl geändert werden.




 a| 
Das Volume ist voll, aber im umschließenden Aggregat ist noch Platz.
 a| 
* Für Speicherplatzzusage-Volumes kann der  `volume modify` Befehl verwendet werden, um die Größe des Volumes zu erhöhen.
* Für dünn bereitgestellte Volumes kann der  `volume modify` Befehl verwendet werden, um die maximale Größe Ihres Volumes zu erhöhen.
+
Falls die automatische Volumenvergrößerung nicht aktiviert ist,  `volume modify -autogrow-mode`kann sie aktiviert werden.

* Snapshots können manuell mit dem  `volume snapshot delete` Befehl gelöscht werden, oder der  `volume snapshot autodelete modify` Befehl wird verwendet, um Snapshots automatisch zu löschen.
