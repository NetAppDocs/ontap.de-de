= Informationen zu All-Flash SAN Array Konfigurationen

:toclevels: 1
:allow-uri-read: 
:toclevels: 1

[role="lead"]
Die NetApp All-Flash SAN Arrays (ASAs) sind ab ONTAP 9.7 verfügbar. ASAs sind reine All-Flash SAN-Lösungen, die auf bewährten AFF NetApp Plattformen basieren.

Zu den ASA Plattformen gehören die folgenden:

* ASA A150
* ASA A250
* ASA A400
* ASA A800
* ASA A900
* ASA C250
* ASA C400
* ASA C800



NOTE: Ab ONTAP 9.16.0 ist eine vereinfachte ONTAP Erfahrung, die speziell für reine SAN-Kunden entwickelt wurde, auf ASA r2 Systemen (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30 oder ASA A20) verfügbar. Bei einem ASA r2 System finden Sie Informationen im link:https://docs.netapp.com/us-en/asa-r2/index.html["ASA r2 Systemdokumentation"^].

ASA Plattformen nutzen symmetrisches Aktiv/Aktiv für Multipathing. Alle Pfade sind aktiv/optimiert, sodass der Host im Falle eines Storage-Failovers nicht auf den ALUA-Übergang der Failover-Pfade warten muss, um die I/O wieder aufzunehmen. Dies verkürzt die Zeit bis zum Failover.



[[set-up-an-asa]]
== Eine ASA einrichten

All-Flash SAN Arrays (ASAs) folgen dem gleichen Einrichtungsverfahren wie Nicht-ASA Systeme.

Der System Manager führt durch die notwendigen Schritte zur Initialisierung Ihres Clusters, zur Erstellung einer lokalen Ebene, zur Konfiguration von Protokollen und zur Bereitstellung von Speicher für Ihre ASA.

xref:../software_setup/concept_decide_whether_to_use_ontap_cli.html[Einstieg in die ONTAP Cluster-Einrichtung].



[[asa-host-settings-and-utilities]]
== ASA Host-Einstellungen und Dienstprogramme

Die Host-Einstellungen für die Einrichtung von All-Flash SAN Arrays (ASA) sind die gleichen wie für alle anderen SAN-Hosts.

Sie können die link:https://mysupport.netapp.com/NOW/cgi-bin/software["NetApp Host Utilities Software"^] für Ihre spezifischen Hosts von der Support-Website herunterladen.



[[ways-to-identify-an-asa-system]]
== Möglichkeiten zur Identifizierung eines ASA Systems

Ein ASA System lässt sich mithilfe des System Manager oder über die ONTAP Befehlszeile (CLI) identifizieren.

* *Im System Manager Dashboard*: *Cluster > Übersicht* auswählen und anschließend den Systemknoten auswählen.
+
Die *PERSÖNLICHKEIT* wird als *All-Flash SAN Array* angezeigt.

* *Über die Befehlszeile*: Den  `san config show` Befehl eingeben.
+
Der Wert "All-Flash SAN Array" wird für ASA Systeme als true zurückgegeben.

+
Weitere Informationen zu  `san config show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/san-config-show.html["ONTAP-Befehlsreferenz"^].
