= Wie die iSCSI-Authentifizierung funktioniert
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In der Anfangsphase einer iSCSI-Sitzung sendet der Initiator eine Anmeldeanfrage an das Speichersystem, um eine iSCSI-Sitzung zu beginnen. Das Speichersystem genehmigt oder verweigert anschließend die Anmeldeanfrage oder stellt fest, dass keine Anmeldung erforderlich ist.

iSCSI-Authentifizierungsverfahren sind:

* Challenge Handshake Authentication Protocol (CHAP): Der Initiator meldet sich mit einem CHAP-Benutzernamen und Passwort an.
+
Sie können ein CHAP-Passwort festlegen oder ein hexadezimales geheimes Passwort generieren. Es gibt zwei Arten von CHAP-Benutzernamen und Passwörtern:

+
** Eingehend, das Speichersystem authentifiziert den Initiator.
+
Eingangseinstellungen sind erforderlich, wenn Sie die CHAP-Authentifizierung verwenden.

** Ausgehend, dies ist eine optionale Einstellung, um dem Initiator die Authentifizierung des Speichersystems zu ermöglichen.
+
Ausgehende Einstellungen können nur verwendet werden, wenn im Speichersystem ein eingehender Benutzername und ein Passwort definiert sind.



* deny--Dem Initiator wird der Zugriff auf das Speichersystem verweigert.
* keine, das Speichersystem erfordert keine Authentifizierung für den Initiator.


Sie können die Liste der Initiatoren und deren Authentifizierungsverfahren definieren. Es kann auch ein Standard-Authentifizierungsverfahren definiert werden, das für Initiatoren gilt, die nicht in dieser Liste enthalten sind.
