= Wissenswertes vor dem Erstellen einer LUN
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Vor der Einrichtung Ihrer LUNs auf Ihrem Cluster sollten diese LUN-Richtlinien überprüft werden.



[[why-actual-lun-sizes-slightly-vary]]
== Warum die tatsächlichen LUN Größen leicht variieren

Im Hinblick auf die Größe Ihrer LUNs ist Folgendes zu berücksichtigen.

* Beim Erstellen einer LUN kann die tatsächliche Größe der LUN je nach OS-Typ der LUN geringfügig variieren. Der LUN OS-Typ kann nach der Erstellung der LUN nicht mehr geändert werden.
* Wenn Sie eine LUN mit der maximalen LUN-Größe erstellen, ist zu beachten, dass die tatsächliche Größe der LUN etwas geringer sein kann. ONTAP rundet das Limit geringfügig nach unten ab.
* Die Metadaten jeder LUN benötigen ca. 64 KB Speicherplatz im übergeordneten Aggregat. Wenn eine LUN erstellt wird, muss sichergestellt sein, dass das übergeordnete Aggregat über ausreichend Speicherplatz für die Metadaten der LUN verfügt. Wenn das Aggregat nicht genügend Speicherplatz für die Metadaten der LUN enthält, können einige Hosts möglicherweise nicht auf die LUN zugreifen.




[[guidelines-for-assigning-lun-ids]]
== Richtlinien für die Vergabe von LUN-IDs

Die Standard-LUN-ID beginnt üblicherweise mit 0 und wird für jede weitere zugeordnete LUN in Schritten von 1 vergeben. Der Host ordnet die LUN-ID dem Speicherort und Pfadnamen der LUN zu. Der Bereich gültiger LUN-ID-Nummern hängt vom Host ab. Detaillierte Informationen enthält die Dokumentation, die mit Ihren Host Utilities bereitgestellt wird.



[[guidelines-for-mapping-luns-to-igroups]]
== Richtlinien für die Zuordnung von LUNs zu Initiatorgruppen

* Eine LUN kann nur einmal einer Initiatorgruppe zugeordnet werden.
* Als bewährte Vorgehensweise wird empfohlen, eine LUN nur einem bestimmten Initiator über die Initiatorgruppe zuzuordnen.
* Sie können einen einzelnen Initiator mehreren Initiatorgruppen hinzufügen, aber der Initiator kann nur einer LUN zugeordnet werden.
* Sie können nicht dieselbe LUN-ID für zwei LUNs verwenden, die derselben Initiatorgruppe zugeordnet sind.
* Für Initiatorgruppen und Port-Sets sollte derselbe Protokolltyp verwendet werden.
