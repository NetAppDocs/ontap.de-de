= Informationen zu ONTAP Datenschutzmethoden für SAN-Umgebungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Ihre Daten schützen, indem Sie Kopien davon erstellen, sodass sie im Falle versehentlichen Löschens, Anwendungsabstürzen, Datenbeschädigung oder Katastrophen für eine Wiederherstellung zur Verfügung stehen. Abhängig von Ihren Anforderungen an Datenschutz und Backup bietet ONTAP verschiedene Methoden, mit denen Sie Ihre Daten schützen können.



[[snapmirror-active-sync]]
== SnapMirror Active Sync

Ab der allgemeinen Verfügbarkeit in ONTAP 9.9.1 wird ein Zero Recovery Time Objective (Zero RTO) oder ein Transparent Application Failover (TAF) bereitgestellt, um ein automatisches Failover von geschäftskritischen Anwendungen in SAN-Umgebungen zu ermöglichen. SnapMirror active sync erfordert die Installation von ONTAP Mediator 1.2 in einer Konfiguration mit entweder zwei AFF Clustern oder zwei All-Flash SAN Array (ASA) Clustern.

link:../snapmirror-active-sync/index.html["SnapMirror Active Sync"]



[[snapshot]]
== Schnappschuss

Ermöglicht das manuelle oder automatische Erstellen, Planen und Verwalten mehrerer Backups Ihrer LUNs. Snapshots benötigen nur minimalen zusätzlichen Volume-Speicherplatz und verursachen keine Performance-Einbußen. Wenn Ihre LUN-Daten versehentlich geändert oder gelöscht wurden, lässt sich diese Daten einfach und schnell aus einem der letzten Snapshots wiederherstellen.



[[flexclone-luns-flexclone-license-required]]
== FlexClone LUNs (FlexClone Lizenz erforderlich)

Stellt zeitpunktgenau beschreibbare Kopien einer anderen LUN in einem aktiven Volume oder in einem Snapshot bereit. Ein Klon und sein übergeordnetes Volume können unabhängig voneinander geändert werden, ohne sich gegenseitig zu beeinflussen.



[[snaprestore-license-required]]
== SnapRestore (Lizenz erforderlich)

Ermöglicht eine schnelle, platzsparende und bedarfsgesteuerte Datenwiederherstellung aus Snapshots eines gesamten Volumes. Mit SnapRestore kann eine LUN in einen früheren, gesicherten Zustand zurückversetzt werden, ohne dass das Storage-System neu gestartet werden muss.



[[data-protection-mirror-copies-snapmirror-license-required]]
== Datenschutz-Spiegelkopien (SnapMirror Lizenz erforderlich)

Ermöglicht asynchrone Notfallwiederherstellung, indem periodisch Snapshots der Daten auf dem Volume erstellt, diese Snapshots über ein lokales oder Weitverkehrsnetzwerk auf ein Partner-Volume, üblicherweise in einem anderen Cluster, kopiert und dort aufbewahrt werden. Die Spiegelkopie auf dem Partner-Volume bietet schnelle Verfügbarkeit und Wiederherstellung der Daten ab dem Zeitpunkt des letzten Snapshots, falls die Daten auf dem Quell-Volume beschädigt oder verloren sind.



[[snapvault-backups-snapmirror-license-required]]
== SnapVault Backups (SnapMirror Lizenz erforderlich)

Bietet platzsparende und langfristige Aufbewahrung von Backups. SnapVault Beziehungen ermöglichen die Sicherung ausgewählter Snapshots von Volumes auf einem Zielvolume und die Aufbewahrung der Backups.

Wenn Bandsicherungen und Archivierungsvorgänge durchgeführt werden, ist dies mit den Daten möglich, die bereits auf dem SnapVault sekundären Volume gesichert wurden.



[[native-tape-backup-and-recovery]]
== Native Tape-Backup und Recovery

Die Unterstützung für die meisten vorhandenen Bandlaufwerke ist in ONTAP enthalten, ebenso wie eine Methode, mit der Bandhersteller dynamisch die Unterstützung für neue Geräte hinzufügen können. ONTAP unterstützt außerdem das Remote Magnetic Tape (RMT) Protokoll, wodurch Backup und Recovery auf jedem geeigneten System möglich ist.
