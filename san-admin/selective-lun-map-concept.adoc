= Überblick über die Selective LUN Map
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Selective LUN Map (SLM) reduziert die Anzahl der Pfade vom Host zur LUN. Mit SLM ist die LUN bei der Erstellung einer neuen LUN-Zuordnung nur über Pfade auf dem Knoten, dem die LUN gehört, und seinem HA-Partner erreichbar.

SLM ermöglicht die Verwaltung einer einzelnen Initiatorgruppe pro Host und unterstützt außerdem unterbrechungsfreie LUN-Verschiebungsvorgänge, die keine Portset-Manipulation oder LUN-Neuzuordnung erfordern.

link:create-port-sets-binding-igroups-task.html["Port-Sets"]Kann in Verbindung mit SLM verwendet werden, um den Zugriff bestimmter Targets auf bestimmte Initiatoren weiter einzuschränken. Bei Verwendung von SLM mit Portsets sind LUNs auf der Menge der LIFs im Portset auf dem Node, dem die LUN gehört, und auf dem HA-Partner dieses Nodes zugänglich.

SLM ist standardmäßig auf allen neuen LUN Maps aktiviert.



[[determine-whether-slm-is-enabled-on-a-lun-map]]
== Ermitteln, ob SLM auf einer LUN-Zuordnung aktiviert ist

Wenn Ihre Umgebung eine Kombination aus LUNs enthält, die in einer ONTAP 9-Version erstellt wurden, und LUNs, die aus früheren Versionen migriert wurden, muss möglicherweise festgestellt werden, ob Selective LUN Map (SLM) für eine bestimmte LUN aktiviert ist.

Die in der Ausgabe des  `lun mapping show -fields reporting-nodes, node` Befehls angezeigten Informationen können zur Feststellung verwendet werden, ob SLM auf Ihrer LUN-Zuordnung aktiviert ist. Wenn SLM nicht aktiviert ist, wird in den Zellen unter der Spalte „`reporting-nodes`“ der Befehlsausgabe ein „-“ angezeigt. Wenn SLM aktiviert ist, wird die Liste der unter der Spalte „`nodes`“ angezeigten Knoten auch in der Spalte „`reporting-nodes`“ angezeigt.

Weitere Informationen zu  `lun mapping show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/lun-mapping-show.html["ONTAP-Befehlsreferenz"^].



[[modify-the-slm-reporting-nodes-list]]
== Die SLM-Berichtsknotenliste ändern

Wenn Sie eine LUN oder ein Volume mit LUNs innerhalb desselben Clusters auf ein anderes Hochverfügbarkeitspaar (HA) verschieben, sollten Sie die Reporting-Nodes-Liste der Selective LUN Map (SLM) vor dem Verschieben anpassen, um sicherzustellen, dass aktive, optimierte LUN-Pfade erhalten bleiben.

.Schritte
. Der Zielknoten und sein Partnerknoten werden der Reporting-Nodes-Liste des Aggregats oder Volumes hinzugefügt:
+
[source, cli]
----
lun mapping add-reporting-nodes -vserver <vserver_name> -path <lun_path> -igroup <igroup_name> [-destination-aggregate <aggregate_name>|-destination-volume <volume_name>]
----
+
Wenn Sie eine einheitliche Namenskonvention haben, können Sie mehrere LUN-Zuordnungen gleichzeitig ändern, indem Sie  `igroup_prefix*` anstelle von  `igroup_name` verwenden.

. Der Host wird erneut gescannt, damit die neu hinzugefügten Pfade erkannt werden.
. Falls Ihr Betriebssystem dies erfordert, sind die neuen Pfade der Multipath Network I/O (MPIO) Konfiguration hinzuzufügen.
. Den Befehl für den benötigten Verschiebevorgang ausführen und abwarten, bis der Vorgang abgeschlossen ist.
. Es sollte überprüft werden, ob die E/A über den Active/Optimized-Pfad erfolgt:
+
[source, cli]
----
lun mapping show -fields reporting-nodes
----
. Den vorherigen LUN-Besitzer und seinen Partnerknoten aus der Reporting-Nodes-Liste entfernen:
+
[source, cli]
----
lun mapping remove-reporting-nodes -vserver <vserver_name> -path <lun_path> -igroup <igroup_name> -remote-nodes
----
. Es sollte überprüft werden, ob die LUN aus der bestehenden LUN-Zuordnung entfernt wurde:
+
[source, cli]
----
lun mapping show -fields reporting-nodes
----
. Alle veralteten Geräteeinträge für das Host-Betriebssystem entfernen.
. Gegebenenfalls sind die Multipathing-Konfigurationsdateien anzupassen.
. Den Host erneut scannen, um die Entfernung alter Pfade zu überprüfen. + In der Host-Dokumentation sind spezifische Schritte zum erneuten Scannen der Hosts zu finden.
