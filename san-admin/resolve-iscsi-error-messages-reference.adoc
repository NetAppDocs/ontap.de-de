= Behebung von iSCSI-Fehlermeldungen auf dem Speichersystem
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es gibt eine Reihe häufiger iSCSI-bezogener Fehlermeldungen, die mit dem  `event log show` Befehl angezeigt werden können. Es ist erforderlich zu wissen, was diese Meldungen bedeuten und welche Möglichkeiten zur Behebung der darin identifizierten Probleme bestehen.

Die folgende Tabelle enthält die häufigsten Fehlermeldungen und Anweisungen zu deren Behebung:

[cols="3*"]
|===
| Nachricht | Erläuterung | Was zu tun ist 


 a| 
`ISCSI: network interface identifier disabled for use; incoming connection discarded`
 a| 
Der iSCSI-Service ist auf der Schnittstelle nicht aktiviert.
 a| 
Der  `iscsi interface enable` Befehl kann verwendet werden, um den iSCSI-Service auf der Schnittstelle zu aktivieren. Beispiel:

`iscsi interface enable -vserver vs1 -lif lif1`



 a| 
`ISCSI: Authentication failed for initiator nodename`
 a| 
CHAP ist für den angegebenen Initiator nicht korrekt konfiguriert.
 a| 
Die CHAP-Einstellungen sind zu überprüfen; es ist nicht möglich, denselben Benutzernamen und dasselbe Passwort für eingehende und ausgehende Einstellungen im Speichersystem zu verwenden:

* Die eingehenden Anmeldeinformationen auf dem Speichersystem müssen mit den ausgehenden Anmeldeinformationen auf dem Initiator übereinstimmen.
* Die ausgehenden Anmeldeinformationen auf dem Speichersystem müssen mit den eingehenden Anmeldeinformationen auf dem Initiator übereinstimmen.


|===
Weitere Informationen zu  `event log show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/event-log-show.html["ONTAP-Befehlsreferenz"^].
