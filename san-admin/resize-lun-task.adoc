= Ändern der Größe einer LUN in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Größe einer LUN vergrößern oder verkleinern.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/manage-data/modify-storage-units.html["diese Schritte"^] zu befolgen, um die Größe einer Speichereinheit zu erhöhen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

[NOTE]
====
Solaris LUNs können nicht in ihrer Größe verändert werden.

====


[[increase-the-size-of-a-lun]]
== Die Größe einer LUN vergrößern

Die Größe, auf die Sie Ihre LUN erweitern können, variiert je nach Ihrer Version von ONTAP.

|===


| ONTAP Version | Maximale LUN Größe 


| ONTAP 9.12.1P2 und höher  a| 
128 TB für AFF, FAS und ASA Plattformen



| ONTAP 9.8 und höher  a| 
* 128 TB für All-Flash SAN Array (ASA) Plattformen
* 16 TB für Nicht-ASA Plattformen




| ONTAP 9.5, 9.6, 9.7 | 16 TB 


| ONTAP 9.4 oder früher | Das Zehnfache der ursprünglichen LUN-Größe, jedoch nicht mehr als 16 TB, was die maximale LUN-Größe ist. Beispielsweise kann eine LUN mit 100 GB nur auf 1.000 GB vergrößert werden. Die tatsächliche maximale Größe der LUN beträgt möglicherweise nicht exakt 16 TB. ONTAP rundet das Limit geringfügig nach unten ab. 
|===
Sie müssen die LUN nicht offline nehmen, um ihre Größe zu erhöhen. Nachdem Sie die Größe erhöht haben, muss die LUN jedoch auf dem Host erneut gescannt werden, damit der Host die Größenänderung erkennt.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Die Größe einer LUN mit ONTAP System Manager (9.7 und höher) vergrößern.

. Im System Manager auf *Speicher > LUNs* klicken.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Bearbeiten*.
. Unter *Speicher und Optimierung* die Größe der LUN erhöhen und *Speichern* auswählen.


--
.CLI
--
Die Größe einer LUN lässt sich mit der ONTAP CLI erhöhen.

. Die Größe der LUN erhöhen:
+
[source, cli]
----
lun resize -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -size <lun_size>
----
+
Weitere Informationen zu  `lun resize` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli//lun-resize.html#description["ONTAP-Befehlsreferenz"^].

. Die erhöhte LUN-Größe kann überprüft werden:
+
[source, cli]
----
lun show -vserver <SVM_name>
----
+
[NOTE]
====
ONTAP Operationen runden die tatsächliche Maximalgröße der LUN ab, sodass sie etwas kleiner als der erwartete Wert ist. Die tatsächliche LUN-Größe kann zudem je nach Betriebssystemtyp der LUN leicht variieren. Der exakte Wert der angepassten Größe lässt sich durch Ausführen der folgenden Befehle im erweiterten Modus ermitteln:

`set -unit B`

`lun show -fields max-resize-size -volume _volume_name_ -lun _lun_name_`

====
+
Weitere Informationen zu  `lun show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/lun-show.html["ONTAP-Befehlsreferenz"^].

. Die LUN auf dem Host wird erneut gescannt.
. In der Hostdokumentation finden sich Hinweise darauf, wie die Größe der neu erstellten LUN im Host-Dateisystem sichtbar wird.


--
====


[[decrease-the-size-of-a-lun]]
== Die Größe einer LUN verringern

Bevor die Größe einer LUN verringert wird, muss der Host die Blöcke mit den LUN-Daten in den Bereich der kleineren LUN-Größe migrieren. Ein Tool wie SnapCenter sollte verwendet werden, um sicherzustellen, dass die LUN korrekt verkleinert wird, ohne dass Blöcke mit LUN-Daten abgeschnitten werden. Eine manuelle Verringerung der LUN-Größe wird nicht empfohlen.

Nachdem Sie die Größe Ihrer LUN verringert haben, benachrichtigt ONTAP den Initiator automatisch darüber, dass die LUN-Größe verringert wurde. Unter Umständen sind jedoch zusätzliche Schritte auf Ihrem Host erforderlich, damit der Host die neue LUN-Größe erkennt. In der Dokumentation Ihres Hosts finden sich spezifische Informationen zur Verkleinerung der Host-Dateistruktur.
