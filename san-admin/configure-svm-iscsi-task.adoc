= Eine SVM für iSCSI konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um eine Storage Virtual Machine (SVM) für iSCSI zu konfigurieren, müssen LIFs für die SVM erstellt und diesen LIFs das iSCSI-Protokoll zugewiesen werden.

.Über diese Aufgabe
* Sie benötigen mindestens eine iSCSI-LIF pro Knoten für jede SVM, die Daten mit dem iSCSI-Protokoll bereitstellt. Für Redundanz empfiehlt es sich, mindestens zwei LIFs pro Knoten zu erstellen.
* Ab ONTAP 9.19.1 unterstützen AFF-Systeme Aktiv/Aktiv-Multipathing mit automatischem LIF-Failover auf SVM-Ebene in SAN-Umgebungen. Sie müssen Aktiv/Aktiv-Multipathing auf AFF-Systemen aktivieren, wenn Sie die SVM erstmals erstellen.


.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Eine Storage-VM für iSCSI mit ONTAP System Manager (9.7 und höher) konfigurieren.

[cols="2"]
|===
| iSCSI auf einer neuen Storage-VM konfigurieren | iSCSI auf einer vorhandenen Storage-VM konfigurieren 


 a| 
. Im System Manager wählen Sie *Storage > Storage VMs* und anschließend *Add*.
. Geben Sie einen Namen für die Storage-VM ein.
. *iSCSI* für das *Zugriffsprotokoll* auswählen.
. Wählen Sie *iSCSI aktivieren* und geben Sie die IP-Adresse und Subnetzmaske für die logische Schnittstelle ein.
+
Jeder Knoten sollte über mindestens zwei Netzwerkschnittstellen verfügen.

. Wenn Sie möchten, dass Netzwerkpfade im Falle eines Knotenausfalls automatisch auf den HA-Partner umgeschaltet werden, gehen Sie wie folgt vor:
+
Diese Option ist in ONTAP 9.19.1 und späteren Versionen nur auf AFF- und ASA-Systemen verfügbar.

+
** Wählen Sie für AFF-Systeme *Symmetrisches Aktiv/Aktiv-Multipathing* und anschließend *Symmetrisches Aktiv/Aktiv-Multipathing aktivieren*.
** Wählen Sie für ASA-Systeme *Symmetrisches Aktiv/Aktiv-Multipathing* und anschließend *Aktivieren für SCSI- und NVMe-Protokolle*.


. *Speichern* auswählen.

 a| 
. Im System Manager unter *Storage > Storage-VMs* auswählen.
. Wählen Sie die Storage-VM aus, die Sie konfigurieren möchten.
. Wählen Sie auf der Registerkarte *Einstellungen* aus und dann image:icon_gear.gif["Aktionssymbol"] neben dem iSCSI-Protokoll.
. Wählen Sie *iSCSI aktivieren* und geben Sie die IP-Adresse und Subnetzmaske für die logische Schnittstelle ein. + Jeder Knoten sollte mindestens zwei logische Schnittstellen besitzen.
. *Speichern* auswählen.


|===
--
.CLI
--
Eine Storage-VM für iSCSI mit der ONTAP CLI konfigurieren.

. Die SVMs werden so konfiguriert, dass sie iSCSI-Datenverkehr empfangen:
+
`vserver iscsi create -vserver _vserver_name_ -target-alias _vserver_name_`

. Eine LIF für die SVMs auf jedem Knoten erstellen, die für iSCSI verwendet wird:
+
** Für ONTAP 9.6 und höher:
+
`network interface create -vserver _vserver_name_ -lif _lif_name_ -data-protocol iscsi -service-policy default-data-iscsi -home-node _node_name_ -home-port _port_name_ -address _ip_address_ -netmask _netmask_`

** Für ONTAP 9.5 und frühere Versionen:
+
`network interface create -vserver _vserver_name_ -lif _lif_name_ -role data -data-protocol iscsi -home-node _node_name_ -home-port _port_name_ -address _ip_address_ -netmask _netmask_`



. Überprüfung, ob die LIFs korrekt eingerichtet wurden:
+
`network interface show -vserver _vserver_name_`

+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

. Es sollte sichergestellt werden, dass iSCSI betriebsbereit ist und der Ziel-IQN für diese SVM vorhanden ist:
+
`vserver iscsi show –vserver _vserver_name_`

. Vom Host aus werden iSCSI-Sitzungen zu den LIFs erstellt.
