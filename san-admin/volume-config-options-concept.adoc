= SAN Volume Konfigurationsoptionen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen verschiedene Optionen für das Volume festlegen, das Ihre LUN enthält. Die Art, wie die Volume-Optionen festgelegt werden, bestimmt den für LUNs im Volume verfügbaren Speicherplatz.



[[autogrow]]
== Automatisches Wachstum

Sie können Autogrow aktivieren oder deaktivieren. Wenn Sie Autogrow aktivieren, ermöglicht Autogrow ONTAP, die Größe des Volumes automatisch bis zu einer von Ihnen festgelegten Maximalgröße zu erhöhen. Im übergeordneten Aggregat muss ausreichend Speicherplatz vorhanden sein, um das automatische Wachstum des Volumes zu unterstützen. Daher ist bei aktiviertem Autogrow eine Überwachung des freien Speicherplatzes im übergeordneten Aggregat erforderlich, und bei Bedarf muss zusätzlicher Speicherplatz hinzugefügt werden.

Die automatische Vergrößerung kann nicht zur Erstellung von Snapshots ausgelöst werden. Wenn versucht wird, einen Snapshot zu erstellen und nicht genügend Speicherplatz auf dem Volume vorhanden ist, schlägt die Snapshot-Erstellung fehl, selbst wenn die automatische Vergrößerung aktiviert ist.

Wenn die automatische Vergrößerung deaktiviert ist, bleibt die Größe Ihres Volumes gleich.



[[autoshrink]]
== Autoshrink

Sie können Autoshrink aktivieren oder deaktivieren. Wenn Autoshrink aktiviert ist, ermöglicht es ONTAP, die Gesamtgröße eines Volumes automatisch zu verringern, wenn der belegte Speicherplatz im Volume einen vordefinierten Schwellenwert unterschreitet. Dies erhöht die Speichereffizienz, indem Volumes ungenutzten freien Speicherplatz automatisch freigeben.



[[snapshot-autodelete]]
== Snapshot Autolöschung

Snapshot Autodelete löscht Snapshots automatisch, wenn einer der folgenden Fälle eintritt:

* Das Volume ist fast voll.
* Der Speicherplatz für Momentaufnahmen ist nahezu voll.
* Der Überschreibungsreservebereich ist voll.


Sie können die automatische Snapshot-Löschung so konfigurieren, dass Snapshots entweder vom ältesten zum neuesten oder vom neuesten zum ältesten gelöscht werden. Die automatische Snapshot-Löschung entfernt keine Snapshots, die mit Snapshots in geklonten Volumes oder LUNs verknüpft sind.

Wenn Ihr Volume zusätzlichen Speicherplatz benötigt und sowohl Autogrow als auch Snapshot Autodelete aktiviert sind, versucht ONTAP standardmäßig, den benötigten Speicherplatz zunächst durch Autogrow bereitzustellen. Wird durch Autogrow nicht genügend Speicherplatz bereitgestellt, wird anschließend Snapshot Autodelete ausgelöst.



[[snapshot-reserve]]
== Snapshot-Reserve

Die Snapshot-Reserve definiert den Speicherplatz im Volume, der für Snapshots reserviert ist. Der für die Snapshot-Reserve zugewiesene Speicherplatz kann nicht für andere Zwecke verwendet werden. Wenn der gesamte für die Snapshot-Reserve zugewiesene Speicherplatz belegt ist, belegen Snapshots zusätzlichen Speicherplatz auf dem Volume.
