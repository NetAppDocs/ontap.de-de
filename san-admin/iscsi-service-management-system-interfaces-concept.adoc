= iSCSI-Service-Management
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Verfügbarkeit des iSCSI-Service auf den iSCSI-Logischen Schnittstellen der Storage Virtual Machine (SVM) kann mit den  `vserver iscsi interface enable`- oder  `vserver iscsi interface disable`-Befehlen verwaltet werden.

Standardmäßig ist der iSCSI-Service auf allen iSCSI-Logischen Schnittstellen aktiviert.



[[how-iscsi-is-implemented-on-the-host]]
== Wie iSCSI auf dem Host implementiert ist

iSCSI kann auf dem Host mit Hardware oder Software implementiert werden.

iSCSI kann auf eine der folgenden Arten implementiert werden:

* Verwendung von Initiator-Software, die die Standard-Ethernet-Schnittstellen des Hosts verwendet.
* Über einen iSCSI Hostbusadapter (HBA): Ein iSCSI HBA erscheint dem Host-Betriebssystem als SCSI-Festplattenadapter mit lokalen Festplatten.
* Verwendung eines TCP Offload Engine (TOE) Adapters, der die TCP/IP-Verarbeitung auslagert.
+
Die Verarbeitung des iSCSI-Protokolls erfolgt weiterhin durch die Host-Software.
