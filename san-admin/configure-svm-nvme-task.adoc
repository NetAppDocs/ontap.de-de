= Eine Storage-VM für NVMe konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie das NVMe-Protokoll auf einem Knoten verwenden möchten, muss Ihre SVM speziell für NVMe konfiguriert werden.

.Bevor Sie beginnen
Ihre FC- oder Ethernet-Adapter müssen NVMe unterstützen. Unterstützte Adapter sind in der  https://hwu.netapp.com["NetApp Hardware Universe"^] aufgeführt.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Eine Storage-VM für NVMe mit ONTAP System Manager (9.7 und höher) konfigurieren.

[cols="2"]
|===
| NVMe auf einer neuen Storage-VM konfigurieren | NVMe auf einer vorhandenen Storage-VM konfigurieren 


 a| 
. Im System Manager auf *Storage > Storage VMs* klicken und dann auf *Add*.
. Geben Sie einen Namen für die Storage-VM ein.
. *NVMe* für das *Zugriffsprotokoll* auswählen.
. Wählen Sie *NVMe/FC aktivieren* oder *NVMe/TCP aktivieren*.
. Wenn Sie NVMe/TCP aktivieren und Netzwerkpfade im Falle eines Knotenausfalls automatisch auf den HA-Partner umgeschaltet werden sollen, gehen Sie wie folgt vor:
+
Diese Option ist in ONTAP 9.19.1 und späteren Versionen nur auf AFF- und ASA-Systemen verfügbar.

+
** Wählen Sie für AFF-Systeme *Symmetrisches Aktiv/Aktiv-Multipathing* und anschließend *Symmetrisches Aktiv/Aktiv-Multipathing aktivieren*.
** Wählen Sie für ASA-Systeme *Symmetrisches Aktiv/Aktiv-Multipathing* und anschließend *Aktivieren für SCSI- und NVMe-Protokolle*.


. *Speichern* auswählen.

 a| 
. Im System Manager auf *Storage > Storage-VMs* klicken.
. Auf die Storage-VM, die konfiguriert werden soll, klicken.
. Auf die Registerkarte *Einstellungen* wechseln und anschließend image:icon_gear.gif["Aktionssymbol"] neben dem NVMe-Protokoll auswählen.
. *NVMe/FC aktivieren* oder *NVMe/TCP aktivieren* und *Speichern* auswählen.


|===
--
.CLI
--
Eine Storage-VM für NVMe mit der ONTAP CLI konfigurieren.

. Wenn keine vorhandene SVM verwendet werden soll, kann eine neue erstellt werden:
+
[source, cli]
----
vserver create -vserver <SVM_name>
----
+
.. Es sollte überprüft werden, ob die SVM erstellt wurde:
+
[source, cli]
----
vserver show
----


. Es sollte überprüft werden, ob in Ihrem Cluster NVMe- oder TCP-fähige Adapter installiert sind:
+
Für NVMe:

+
[source, cli]
----
network fcp adapter show -data-protocols-supported fc-nvme
----
+
Für TCP:

+
[source, cli]
----
network port show
----
+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

. Wenn Sie ONTAP 9.7 oder eine frühere Version verwenden, alle Protokolle aus der SVM entfernen:
+
[source, cli]
----
vserver remove-protocols -vserver <SVM_name> -protocols iscsi,fcp,nfs,cifs,ndmp
----
+
Ab ONTAP 9.8 ist es nicht erforderlich, andere Protokolle zu entfernen, wenn NVMe hinzugefügt wird.

. Das NVMe-Protokoll zur SVM hinzufügen:
+
[source, cli]
----
vserver add-protocols -vserver <SVM_name> -protocols nvme
----
. Wenn Sie ONTAP 9.7 oder eine ältere Version verwenden, überprüfen Sie, ob NVMe das einzige auf der SVM zulässige Protokoll ist:
+
[source, cli]
----
vserver show -vserver <SVM_name> -fields allowed-protocols
----
+
Nur das NVMe-Protokoll sollte in der  `allowed protocols`Spalte angezeigt werden.

. NVMe-Dienst erstellen:
+
[source, cli]
----
vserver nvme create -vserver <SVM_name>
----
. Es sollte überprüft werden, ob der NVMe-Dienst erstellt wurde:
+
[source, cli]
----
vserver nvme show -vserver <SVM_name>
----
+
Der `Administrative Status` der SVM sollte als `up` aufgeführt sein. Weitere Informationen zu  `up` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/up.html["ONTAP-Befehlsreferenz"^].

. Eine NVMe/FC LIF erstellen:
+
** Für ONTAP 9.9.1 oder älter, FC:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -role data -data-protocol fc-nvme -home-node <home_node> -home-port <home_port>
----
** Für ONTAP 9.10.1 oder höher, FC:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -service-policy <default-data-nvme-tcp | default-data-nvme-fc> -data-protocol <fc-nvme> -home-node <home_node> -home-port <home_port> -status-admin up -failover-policy disabled -firewall-policy data -auto-revert false -failover-group <failover_group> -is-dns-update-enabled false
----
** Für ONTAP 9.10.1 oder höher, TCP:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -address <ip address> -netmask <netmask_value> -service-policy <default-data-nvme-tcp> -data-protocol <nvme-tcp> -home-node <home_node> -home-port <home_port> -status-admin up -failover-policy disabled -firewall-policy data -auto-revert false -failover-group <failover_group> -is-dns-update-enabled false
----


. Eine NVMe/FC-LIF auf dem HA-Partnerknoten erstellen:
+
** Für ONTAP 9.9.1 oder älter, FC:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -role data -data-protocol fc-nvme -home-node <home_node> -home-port <home_port>
----
** Für ONTAP 9.10.1 oder höher, FC:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -service-policy <default-data-nvme-fc> -data-protocol <fc-nvme> -home-node <home_node> -home-port <home_port> -status-admin up -failover-policy disabled -firewall-policy data -auto-revert false -failover-group <failover_group> -is-dns-update-enabled false
----
** Für ONTAP 9.10.1 oder höher, TCP:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -address <ip address> -netmask <netmask_value> -service-policy <default-data-nvme-tcp> -data-protocol <nvme-tcp> -home-node <home_node> -home-port <home_port> -status-admin up -failover-policy disabled -firewall-policy data -auto-revert false -failover-group <failover_group> -is-dns-update-enabled false
----


. Überprüfen, ob die NVMe/FC LIFs erstellt wurden:
+
[source, cli]
----
network interface show -vserver <SVM_name>
----
. Volume auf demselben Knoten wie die LIF erstellen:
+
[source, cli]
----
vol create -vserver <SVM_name> -volume <vol_name> -aggregate <aggregate_name> -size <volume_size>
----
+
Wenn eine Warnmeldung bezüglich der automatischen Effizienzrichtlinie angezeigt wird, kann diese getrost ignoriert werden.
