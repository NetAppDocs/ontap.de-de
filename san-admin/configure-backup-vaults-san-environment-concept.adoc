= Informationen zu ONTAP SnapVault Backups in einer SAN-Umgebung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapVault Konfiguration und Verwendung in einer SAN-Umgebung ist der Konfiguration und Verwendung in einer NAS-Umgebung sehr ähnlich, aber die Wiederherstellung von LUNs in einer SAN-Umgebung erfordert einige spezielle Verfahren.

SnapVault Backups enthalten eine Reihe schreibgeschützter Kopien eines Quellvolumes. In einer SAN-Umgebung werden immer ganze Volumes auf das SnapVault sekundäre Volume gesichert, nicht einzelne LUNs.

Das Verfahren zum Erstellen und Initialisieren der SnapVault Beziehung zwischen einem primären Volume mit LUNs und einem sekundären Volume, das als SnapVault Backup dient, ist identisch mit dem Verfahren, das bei FlexVol Volumes für Dateiprotokolle verwendet wird. Dieses Verfahren wird ausführlich in link:../data-protection/index.html["Datensicherung"] beschrieben.

Es ist wichtig sicherzustellen, dass sich die zu sichernden LUNs in einem konsistenten Zustand befinden, bevor die Snapshots erstellt und auf das SnapVault sekundäre Volume kopiert werden. Die Automatisierung der Snapshot-Erstellung mit SnapCenter gewährleistet, dass die gesicherten LUNs vollständig und von der ursprünglichen Anwendung nutzbar sind.

Es gibt drei grundlegende Möglichkeiten zur Wiederherstellung von LUNs aus einem SnapVault sekundäres Volume:

* Sie können eine LUN direkt vom SnapVault sekundären Volume zuordnen und einen Host mit der LUN verbinden, um auf den Inhalt der LUN zuzugreifen.
+
Die LUN ist schreibgeschützt und kann nur vom aktuellsten Snapshot im SnapVault Backup zugeordnet werden. Persistente Reservierungen und andere LUN-Metadaten gehen verloren. Falls gewünscht, lässt sich mithilfe eines Kopierprogramms auf dem Host der Inhalt der LUN zurück auf die ursprüngliche LUN kopieren, sofern diese noch zugänglich ist.

+
Die LUN hat eine andere Seriennummer als die Quell-LUN.

* Jeder Snapshot im SnapVault sekundären Volume kann auf ein neues Lese-/Schreib-Volume geklont werden.
+
Anschließend kann jede beliebige LUN im Volume zugeordnet und ein Host mit der LUN verbunden werden, um auf den Inhalt der LUN zuzugreifen. Falls gewünscht, lässt sich mithilfe eines Kopierprogramms auf dem Host der Inhalt der LUN zurück auf die ursprüngliche LUN kopieren, sofern diese noch zugänglich ist.

* Das gesamte Volume, das die LUN enthält, kann aus einem beliebigen Snapshot im SnapVault sekundären Volume wiederhergestellt werden.
+
Durch die Wiederherstellung des gesamten Volumes werden alle LUNs und alle Dateien im Volume ersetzt. Alle seit der Erstellung des Snapshots neu erstellten LUNs gehen verloren.

+
Die LUNs behalten ihre Zuordnung, Seriennummern, UUIDs und persistenten Reservierungen.
