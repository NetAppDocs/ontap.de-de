= Fehlerbehebung bei iSCSI-LUNs, die auf dem Host nicht sichtbar sind
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die iSCSI-LUNs werden dem Host als lokale Datenträger angezeigt. Wenn die Speichersystem-LUNs auf dem Host nicht als Datenträger verfügbar sind, empfiehlt sich eine Überprüfung der Konfigurationseinstellungen.

[cols="1, 3"]
|===
| Konfigurationseinstellung | Was zu tun ist 


 a| 
Verkabelung
 a| 
Es sollte überprüft werden, ob die Kabel zwischen Host und Speichersystem ordnungsgemäß angeschlossen sind.



 a| 
Netzwerkverbindungen
 a| 
Es sollte überprüft werden, ob eine TCP/IP-Verbindung zwischen Host und Storage-System besteht.

* Von der Storage-System-Konsole aus die Host-Schnittstellen anpingen, die für iSCSI verwendet werden:
+
`ping –node _node_name_ -destination _host_ip_address_for_iSCSI_`

* Von der Host-Befehlszeile aus die Storage-System-Schnittstellen anpingen, die für iSCSI verwendet werden:
+
`ping –node _node_name_ -destination _host_ip_address_for_iSCSI_`





 a| 
Systemvoraussetzungen
 a| 
Überprüfung, ob die Komponenten Ihrer Konfiguration qualifiziert sind. Zudem sollte sichergestellt sein, dass das korrekte Service Pack des Host-Betriebssystems (OS), die richtige Initiatorversion, die ONTAP Version sowie weitere Systemanforderungen vorliegen. Die Interoperability Matrix enthält die aktuellsten Systemanforderungen.



 a| 
Jumbo Frames
 a| 
Wenn in der Konfiguration Jumbo-Frames verwendet werden, sollte überprüft werden, ob Jumbo-Frames auf allen Geräten im Netzwerkpfad aktiviert sind: der Host-Ethernet-NIC, dem Storage-System und allen Switches.



 a| 
iSCSI-Service-Status
 a| 
Es sollte sichergestellt sein, dass der iSCSI-Service auf dem Storage-System lizenziert und gestartet ist.



 a| 
Initiator-Login
 a| 
Es sollte überprüft werden, ob der Initiator am Storage-System angemeldet ist. Wenn die  `iscsi initiator show` Befehlsausgabe anzeigt, dass keine Initiatoren angemeldet sind, sollte die Initiatorkonfiguration auf dem Host überprüft werden. Zudem sollte überprüft werden, ob das Storage-System als Ziel des Initiators konfiguriert ist.



 a| 
iSCSI-Node-Namen (IQNs)
 a| 
Stellen Sie sicher, dass Sie in der Initiatorgruppe-Konfiguration die korrekten Node-Namen der Initiatoren verwenden. Auf dem Host können die Initiator-Tools und -Befehle genutzt werden, um den Node-Name des Initiators anzuzeigen. Die in der Initiatorgruppe und auf dem Host konfigurierten Node-Namen der Initiatoren müssen übereinstimmen.



 a| 
LUN-Zuordnungen
 a| 
Es sollte überprüft werden, ob die LUNs einer Initiatorgruppe zugeordnet sind. Auf der Storage-System-Konsole kann einer der folgenden Befehle verwendet werden:

* `lun mapping show`zeigt alle LUNs und die Initiatorgruppen, denen sie zugeordnet sind, an.
* `lun mapping show -igroup`zeigt die LUNs, die einer bestimmten Initiatorgruppe zugeordnet sind.




 a| 
iSCSI LIFs ermöglichen
 a| 
Es sollte sichergestellt sein, dass die logischen iSCSI-Schnittstellen aktiviert sind.

|===
