= VMware VAAI Performance für ESX Hosts verbessern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt bestimmte VMware vStorage APIs für Array Integration (VAAI) Features, wenn auf dem ESX Host ESX 4.1 oder höher ausgeführt wird. Diese Features helfen dabei, Operationen vom ESX Host auf das Speichersystem auszulagern und den Netzwerkdurchsatz zu erhöhen. Die Features werden vom ESX Host in der passenden Umgebung automatisch aktiviert.

Die VAAI Funktion unterstützt die folgenden SCSI-Befehle:

* `EXTENDED_COPY`
+
Diese Funktion ermöglicht es dem Host, die Übertragung von Daten zwischen den LUNs oder innerhalb einer LUN zu initiieren, ohne dass der Host selbst am Datentransfer beteiligt ist. Dies führt zu einer Einsparung von ESX-CPU-Zyklen und einer Erhöhung des Netzwerkdurchsatzes. Die erweiterte Kopierfunktion, auch als „copy offload“ bekannt, wird in Szenarien wie dem Klonen einer virtuellen Maschine verwendet. Wenn die Funktion vom ESX Host aufgerufen wird, kopiert die copy offload Funktion die Daten innerhalb des Speichersystems, anstatt sie über das Host-Netzwerk zu übertragen. Copy offload überträgt Daten auf folgende Weise:

+
** Innerhalb einer LUN
** Zwischen LUNs innerhalb eines Volumens
** Zwischen LUNs auf verschiedenen Volumes innerhalb einer Storage Virtual Machine (SVM)
** Zwischen LUNs auf verschiedenen SVMs innerhalb eines Clusters Wenn diese Funktion nicht aufgerufen werden kann, verwendet der ESX Host automatisch die Standardbefehle READ und WRITE für den Kopiervorgang.


* `WRITE_SAME`
+
Diese Funktion lagert die Arbeit des Schreibens eines sich wiederholenden Musters, wie z. B. lauter Nullen, an ein Speicherarray aus. Der ESX Host verwendet diese Funktion bei Vorgängen wie dem Auffüllen einer Datei mit Nullen.

* `COMPARE_AND_WRITE`
+
Diese Funktion umgeht bestimmte Beschränkungen der Dateizugriffskonkurrenz, wodurch Vorgänge wie das Hochfahren virtueller Maschinen beschleunigt werden.





[[requirements-for-using-the-vaai-environment]]
== Anforderungen für die Nutzung der VAAI-Umgebung

Die VAAI-Funktionen sind Teil des ESX-Betriebssystems und werden vom ESX Host automatisch verwendet, wenn die richtige Umgebung eingerichtet ist.

Die Umgebungsanforderungen lauten wie folgt:

* Der ESX Host muss ESX 4.1 oder eine neuere Version ausführen.
* Das NetApp Speichersystem, das den VMware-Datenspeicher hostet, muss unter ONTAP betrieben werden.
* (Nur Copy Offload) Quelle und Ziel des VMware-Kopiervorgangs müssen sich auf demselben Speichersystem innerhalb desselben Clusters befinden.
+
[NOTE]
====
Die Copy Offload-Funktion unterstützt derzeit kein Kopieren von Daten zwischen VMware-Datenspeichern, die auf unterschiedlichen Speichersystemen gehostet werden.

====




[[determine-if-vaai-features-are-supported-by-esx]]
== Es kann festgestellt werden, ob die VAAI-Funktionen von ESX unterstützt werden.

Um zu überprüfen, ob das ESX-Betriebssystem die VAAI-Funktionen unterstützt, kann der vSphere Client geprüft oder auf anderem Wege auf den Host zugegriffen werden. ONTAP unterstützt standardmäßig die SCSI-Befehle.

Sie können die erweiterten Einstellungen Ihres ESX-Hosts überprüfen, um festzustellen, ob VAAI-Funktionen aktiviert sind. Die Tabelle zeigt, welche SCSI-Befehle den ESX-Steuerungsnamen entsprechen.
