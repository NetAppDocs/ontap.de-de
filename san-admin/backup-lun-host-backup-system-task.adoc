= Ein Host-Backup-System kann zum Schutz einer LUN auf dem ONTAP-Speichersystem eingesetzt werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine geklonte LUN aus einem Snapshot kann als Quelldaten für das Host-Backup-System genutzt werden.

.Bevor Sie beginnen
Es muss eine Produktions-LUN vorhanden sein, die einer Initiatorgruppe zugeordnet ist, die den WWPN oder Node-Name des Anwendungsservers enthält. Die LUN muss außerdem formatiert und für den Host zugänglich sein.

.Schritte
. Speichern Sie den Inhalt der Puffer des Host-Dateisystems auf der Festplatte.
+
Sie können den vom Host-Betriebssystem bereitgestellten Befehl verwenden. Sie können diesen Schritt auch als Teil Ihres SAN-Backup-Vorverarbeitungsskripts ausführen.

. Ein Snapshot der Produktions-LUN wird erstellt.
+
[source, cli]
----
volume snapshot create -vserver <SVM_name> -volume <volume_name> -snapshot <snapshot> -comment <comment> -foreground false
----
. Einen Klon der Produktions-LUN erstellen.
+
[source, cli]
----
volume file clone create -vserver <svm_name> -volume <volume> -source-path <path> -snapshot-name <snapshot> -destination-path <destination_path>
----
. Eine Initiatorgruppe erstellen, die den WWPN des Backup-Servers enthält.
+
[source, cli]
----
lun igroup create -vserver <SVM_name> -igroup <igroup> -protocol <protocol> -ostype <os_type> -initiator <initiator>
----
. Ordnen Sie den in Schritt 3 erstellten LUN-Klon dem Backup-Host zu.
+
[source, cli]
----
lun mapping create -vserver <SVM_name> -volume <volume_name> -lun <lun_name> -igroup <igroup>
----
+
Sie können diesen Schritt optional in das Nachbearbeitungsskript Ihrer SAN Backup-Anwendung integrieren.

. Vom Host aus die neue LUN erkennen und das Dateisystem für den Host verfügbar machen.
+
Sie können diesen Schritt optional in das Nachbearbeitungsskript Ihrer SAN Backup-Anwendung integrieren.

. Die Daten im LUN-Klon werden vom Backup-Host mithilfe der SAN-Backup-Anwendung auf Band gesichert.
. Den LUN-Klon offline schalten.
+
[source, cli]
----
lun modify -vserver <SVM_name> -path <path> -state offline
----
. Den LUN-Klon entfernen.
+
[source, cli]
----
lun delete -vserver <SVM_name> -volume <volume> -lun <lun_name>
----
. Den Snapshot entfernen.
+
[source, cli]
----
volume snapshot delete -vserver <SVM_name> -volume <volume> -snapshot <snapshot>
----
