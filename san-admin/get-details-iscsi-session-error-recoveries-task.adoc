= Weitere Details zu iSCSI-Sitzungsfehlerbehebungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Durch Erhöhen des Fehlerwiederherstellungslevels für iSCSI-Sitzungen werden detailliertere Informationen zu iSCSI-Fehlerbehebungen bereitgestellt. Die Verwendung eines höheren Fehlerwiederherstellungslevels kann zu einer geringfügigen Reduzierung der iSCSI-Sitzungsleistung führen.

.Über diese Aufgabe
ONTAP ist standardmäßig so konfiguriert, dass für iSCSI-Sitzungen die Fehlerbehebungsstufe 0 verwendet wird. Wenn ein Initiator verwendet wird, der für Fehlerbehebungsstufe 1 oder 2 qualifiziert ist, besteht die Möglichkeit, die Fehlerbehebungsstufe zu erhöhen. Die geänderte Fehlerbehebungsstufe für die Sitzung betrifft nur neu erstellte Sitzungen und hat keine Auswirkungen auf bestehende Sitzungen.

Ab ONTAP 9.4 wird die `max-error-recovery-level` Option in den `iscsi show` und `iscsi modify` Befehlen nicht mehr unterstützt.

.Schritte
. Erweiterter Modus:
+
`set -privilege advanced`

. Die aktuelle Einstellung lässt sich mit dem  `iscsi show` Befehl überprüfen.
+
`iscsi show -vserver vs3 -fields max-error-recovery-level`

+
[listing]
----
vserver max-error-recovery-level
------- ------------------------
vs3     0
----
. Der Fehlerbehandlungsgrad kann mit dem  `iscsi modify` Befehl geändert werden.
+
`iscsi modify -vserver vs3 -max-error-recovery-level 2`
