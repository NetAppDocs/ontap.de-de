= Verschieben einer LUN
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine LUN innerhalb einer Storage Virtual Machine (SVM) zwischen Volumes verschieben, jedoch nicht zwischen SVMs. LUNs, die innerhalb einer SVM zwischen Volumes verschoben werden, werden sofort und ohne Verbindungsverlust übertragen.

.Bevor Sie beginnen
Wenn Ihre LUN Selective LUN Map (SLM) verwendet, sollte link:modify-slm-reporting-nodes-task.html["die SLM Reporting-Nodes-Liste ändern"] der Zielknoten und sein HA-Partner einbezogen werden, bevor die LUN verschoben wird.

.Über diese Aufgabe
Funktionen zur Speichereffizienz wie Deduplizierung, Komprimierung und Verdichtung bleiben bei einer LUN-Verschiebung nicht erhalten. Sie müssen nach Abschluss der LUN-Verschiebung erneut angewendet werden.

Der Datenschutz durch Snapshots erfolgt auf Volume-Ebene. Daher unterliegt eine LUN nach dem Verschieben dem Datenschutzschema des Zielvolumes. Wenn für das Zielvolume keine Snapshots eingerichtet sind, werden keine Snapshots der LUN erstellt. Außerdem verbleiben alle Snapshots der LUN im ursprünglichen Volume, bis diese Snapshots gelöscht werden.

Sie können eine LUN nicht auf die folgenden Volumes verschieben:

* Ein SnapMirror Zielvolumen
* Das SVM Root-Volume


Die folgenden LUN-Typen können nicht verschoben werden:

* Eine LUN, die aus einer Datei erstellt wurde
* Eine LUN im NVFail Zustand
* Eine LUN, die sich in einer Lastverteilungsbeziehung befindet
* Eine Protokoll-Endpunkt-Klasse LUN


Wenn die Knoten in einem Cluster unterschiedliche ONTAP Versionen verwenden, kann eine LUN nur zwischen Volumes auf verschiedenen Knoten verschoben werden, wenn die Quelle eine neuere Version als das Ziel verwendet. Wenn beispielsweise der Knoten des Quell-Volumes ONTAP 9.15.1 und der Knoten des Ziel-Volumes ONTAP 9.16.1 verwendet, ist eine Verschiebung der LUN nicht möglich. LUNs können zwischen Volumes auf Knoten verschoben werden, die dieselbe ONTAP Version verwenden.

[NOTE]
====
Bei Solaris os_type LUNs mit einer Größe von 1 TB oder mehr kann es während des Verschiebens der LUN auf dem Host zu einer Zeitüberschreitung kommen. Bei diesem LUN-Typ sollte die LUN vor dem Starten des Verschiebevorgangs ausgehängt werden.

====
.Schritte
[role="tabbed-block"]
====
.System Manager
--
Verschieben einer LUN mit ONTAP System Manager (9.7 und höher).

Ab ONTAP 9.10.1 kann mit dem System Manager ein neues Volume erstellt werden, wenn eine einzelne LUN verschoben wird. In ONTAP 9.8 und 9.9.1 muss das Volume, auf das die LUN verschoben wird, bereits existieren, bevor der LUN-Verschiebungsvorgang beginnt.

Schritte

. Im System Manager auf *Speicher > LUNs* klicken.
. Klicken Sie mit der rechten Maustaste auf die LUN, die Sie verschieben möchten, klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *LUN verschieben* aus.
+
In ONTAP 9.10.1 kann ausgewählt werden, ob die LUN auf *ein vorhandenes Volume* oder auf ein *neues Volume* verschoben wird.

+
Wenn Sie die Option zum Erstellen eines neuen Volumes auswählen, geben Sie die Volume-Spezifikationen an.

. Auf *Verschieben* klicken.


--
.CLI
--
Eine LUN mit der ONTAP CLI verschieben.

. Die LUN verschieben:
+
[source, cli]
----
lun move start
----
+
Für einen sehr kurzen Zeitraum ist die LUN sowohl auf dem Quell- als auch auf dem Zielvolume sichtbar. Dies ist zu erwarten und wird nach Abschluss des Umzugs behoben.

. Den Status des Umzugs nachverfolgen und den erfolgreichen Abschluss überprüfen:
+
[source, cli]
----
lun move show
----
