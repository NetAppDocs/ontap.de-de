= Informationen zu NVMe Copy Offload
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NVMe Copy Offload ermöglicht einem NVMe-Host, Kopiervorgänge von seiner CPU auf die CPU des ONTAP Storage Controllers auszulagern. Der Host kann Daten von einem NVMe-Namespace in einen anderen kopieren, während seine CPU-Ressourcen für Anwendungsworkloads reserviert bleiben.

Angenommen, Sie müssen beispielsweise Ihre Storage-Workloads neu ausbalancieren, um die Leistungsverteilung zu verbessern. Dazu ist es erforderlich, zehn virtuelle Maschinen (VMs) mit 45 NVMe-Namespaces und einer durchschnittlichen Größe von jeweils 500 GB zu migrieren. Das bedeutet, dass etwa 22,5 TB an Daten kopiert werden müssen. Anstatt die eigene CPU für die Datenmigration zu verwenden, kann der Host NVMe Copy Offload nutzen, um die CPU-Ressourcen für Anwendungs-Workloads während des Kopiervorgangs nicht zu reduzieren.



[[nvme-copy-offload-support-and-limitations]]
== NVMe Copy Offload Unterstützung und Einschränkungen

NVMe Copy Offload wird ab ONTAP 9.18.1 unterstützt. ONTAP kann NVMe Copy Offload nicht initiieren, es muss vom Host unterstützt und initiiert werden.

Die folgenden Einschränkungen gelten für NVMe Copy-Offload-Operationen mit ONTAP:

* Die maximal unterstützte Größe für Kopiervorgänge beträgt 16MB.
* Daten können nur zwischen NVMe-Namensräumen innerhalb desselben Subsystems migriert werden.
* Daten können nur zwischen Knoten innerhalb desselben HA-Paares migriert werden.
