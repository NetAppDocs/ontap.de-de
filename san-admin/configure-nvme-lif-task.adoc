= Konfiguration eines NVMe LIF in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei der Konfiguration von NVMe LIFs müssen bestimmte Anforderungen erfüllt sein.

.Bevor Sie beginnen
NVMe muss vom FC-Adapter unterstützt werden, auf dem die LIF erstellt wird. Unterstützte Adapter sind in  https://hwu.netapp.com["Hardware Universe"^] aufgeführt.

.Über diese Aufgabe
Ab ONTAP 9.12.1 und höher können zwei NVMe-LIFs pro Knoten auf maximal 12 Knoten konfiguriert werden. In ONTAP 9.11.1 und früher können zwei NVMe-LIFs pro Knoten auf maximal zwei Knoten konfiguriert werden.

Beim Erstellen einer NVMe-LIF gelten die folgenden Regeln:

* NVMe kann das einzige Datenprotokoll auf Daten-LIFs sein.
* Für jede SVM, die SAN unterstützt, sollte eine Management-LIF konfiguriert werden.
* Für ONTAP 9.5 und höher ist eine NVMe LIF auf dem Knoten, der den Namespace enthält, und auf dem HA-Partner des Knotens zu konfigurieren.
* Nur für ONTAP 9.4:
+
** NVMe LIFs und Namespaces müssen auf demselben Knoten gehostet werden.
** Pro SVM kann nur ein NVMe Daten-LIF konfiguriert werden.




.Schritte
. LIF erstellen:
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <LIF_name> -role <LIF_role> -data-protocol {fc-nvme|nvme-tcp} -home-node <home_node> -home-port <home_port>
----
+

NOTE: NVME/TCP ist ab ONTAP 9.10.1 und höher verfügbar.

. Es sollte überprüft werden, ob die LIF erstellt wurde:
+
[source, cli]
----
network interface show -vserver <SVM_name>
----


Nach der Erstellung warten NVMe/TCP LIFs auf Port 8009 auf die Erkennung.
