= Die SVM bei einem iSNS-Server registrieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der  `vserver iscsi isns` Befehl kann verwendet werden, um die Storage Virtual Machine (SVM) so zu konfigurieren, dass sie sich bei einem iSNS-Server registriert.

.Über diese Aufgabe
Der  `vserver iscsi isns create` Befehl konfiguriert die SVM so, dass sie sich beim iSNS-Server registriert. Die SVM stellt keine Befehle zur Verfügung, mit denen sich der iSNS-Server konfigurieren oder verwalten lässt. Für die Verwaltung des iSNS-Servers stehen die Server-Administrationstools oder die vom Anbieter für den iSNS-Server bereitgestellte Schnittstelle zur Verfügung.

.Schritte
. Auf Ihrem iSNS-Server muss sichergestellt sein, dass Ihr iSNS-Service aktiv und betriebsbereit ist.
. Die SVM-Management-LIF auf einem Datenport erstellen:
+
`network interface create -vserver _SVM_name_ -lif _lif_name_ -role data -data-protocol none -home-node _home_node_name_ -home-port _home_port_ -address _IP_address_ -netmask _network_mask_`

+
Weitere Informationen zu  `network interface create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-create.html["ONTAP-Befehlsreferenz"^].

. Ein iSCSI-Service auf Ihrer SVM ist zu erstellen, falls noch keiner vorhanden ist:
+
`vserver iscsi create -vserver _SVM_name_`

. Es sollte überprüft werden, ob der iSCSI-Service erfolgreich erstellt wurde:
+
`iscsi show -vserver _SVM_name_`

. Es sollte überprüft werden, ob für die SVM eine Standardroute existiert:
+
`network route show -vserver _SVM_name_`

. Falls für die SVM keine Standardroute existiert, kann eine Standardroute erstellt werden:
+
`network route create -vserver _SVM_name_ -destination _destination_ -gateway _gateway_`

+
Weitere Informationen zu  `network route create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-route-create.html["ONTAP-Befehlsreferenz"^].

. Die SVM wird für die Registrierung beim iSNS-Service konfiguriert:
+
`vserver iscsi isns create -vserver _SVM_name_ -address _IP_address_`

+
Es werden sowohl IPv4- als auch IPv6-Adressfamilien unterstützt. Die Adressfamilie des iSNS-Servers muss mit der des SVM-Management-LIF übereinstimmen.

+
Beispielsweise ist es nicht möglich, eine SVM Management-LIF mit einer IPv4-Adresse mit einem iSNS-Server mit einer IPv6-Adresse zu verbinden.

. Es sollte überprüft werden, ob der iSNS-Service läuft:
+
`vserver iscsi isns show -vserver _SVM_name_`

. Falls der iSNS-Service nicht läuft, sollte er gestartet werden:
+
`vserver iscsi isns start -vserver _SVM_name_`
