= ONTAP SnapMirror Freigabe schlägt fehl, wenn der Snapshot nicht gelöscht werden kann.
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die folgenden Informationen sind relevant, wenn der SnapMirror Freigabevorgang fehlschlägt, weil der Snapshot nicht gelöscht werden kann.

.Problem:
Der SnapMirror Freigabevorgang schlägt fehl, wenn der Snapshot nicht gelöscht werden kann.

.Lösung:
Der Snapshot enthält ein temporäres Tag. Mit dem  `snapshot delete` Befehl und der  `-ignore-owners` Option lässt sich der temporäre Snapshot entfernen.
`snapshot delete -volume <volume_name> -snapshot <snapshot_name> -ignore-owners true -force true`

Den  `snapmirror release`Befehl erneut versuchen.

.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-release.html["SnapMirror Release"^]
