= ONTAP Cloud Mediator ist erreichbar, reagiert aber langsam
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die folgenden Informationen sind relevant, wenn der ONTAP Cloud Mediator mit einer Fehlermeldung ausfällt, die besagt, dass die Ping-Latenz höher als die empfohlene Latenz ist.

.Problem:
System Manager: Der Cloud Mediator Service ist erreichbar, reagiert aber langsam.

CLI: Der  `mediator add` Befehl schlägt mit dem Fehler fehl:
`Error: command failed: The ping latency of the BlueXP cloud server is <x> ms which is higher than twice the recommended latency of 200 ms.`

.Ursache:
Die Cluster befinden sich möglicherweise nicht in unmittelbarer Nähe der NetApp Console Cloud oder es bestehen Engpässe im Netzwerkpfad.

.Lösung:
* Den geografischen Standort und die Nähe zur NetApp Console Cloud (US East) prüfen.
* Netzwerkpfad optimieren oder Engpässe adressieren.
* Die Round-Trip-Time (RTT) kann mit Netzwerktools gemessen und die Latenz auf die empfohlenen Grenzwerte reduziert werden.
* Ein HTTP-Proxy kann zur Leistungsverbesserung beitragen.


Siehe link:cloud-mediator-config-task.html["Den ONTAP Cloud Mediator und die Cluster für SnapMirror Active Sync konfigurieren"].
