= Verbindung zwischen Standort B und ONTAP Mediator unterbrochen und Standort A ausgefallen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Zur Überprüfung der Verbindung des ONTAP Mediator oder des ONTAP Cloud Mediator kann der  `snapmirror mediator show` Befehl verwendet werden. Wenn der Verbindungsstatus „Nicht erreichbar“ ist und Site B Site A nicht erreichen kann, erscheint eine ähnliche Ausgabe wie unten gezeigt. In der Lösung sind die Schritte zur Wiederherstellung der Verbindung aufgeführt.

.Beispiel:
Mit der ONTAP Cloud Mediator-Ausgabe des Befehls `snapmirror mediator show`:

....
cluster::> snapmirror mediator show
Mediator Address Peer Cluster    Connection Status Quorum Status Type
---------------- ---------------- ----------------- ------------- -------
0.0.0.0          C1_cluster       unreachable       true          cloud
....
Verwendung der ONTAP Mediator-Ausgabe mit dem Befehl `snapmirror mediator show`:

....
cluster::> snapmirror mediator show
Mediator Address Peer Cluster     Connection Status Quorum Status
---------------- ---------------- ----------------- -------------
10.237.86.17     C1_cluster       unreachable       true
SnapMirror consistency group relationship status is out of sync.

C2_cluster::> snapmirror show -expand

----------- ---- ------------ ------- -------------- --------- ------- --------

.Lösung
Ein Failover wird erzwungen, um die E/A von Standort B zu ermöglichen, und anschließend wird eine Zero-RTO-Beziehung von Standort B zu Standort A hergestellt. Die folgenden Schritte beschreiben das Vorgehen für ein erzwungenes Failover an Standort B.

. Alle LUNs, die zur Konsistenzgruppe von Standort B gehören, ausbinden. Dies schlägt fehl, daher muss zunächst die Initiatorgruppe geändert werden, um die Replikations-Peer-SVM zu entfernen, und anschließend die LUN-Zuordnung gelöscht werden.


Beispiel:

----
C1_cluster::> lun mapping show

vs0        /vol/cg1_lun/lun_2                        igroup1       1  mixed

. Die SnapMirror Konsistenzgruppenbeziehung wird mit der Option „Erzwingen“ gelöscht.
. Den SnapMirror break-Befehl (`snapmirror break -destination_path _svm_:_volume_` auf den Konsistenzgruppe-Teil-Volumes eingeben, um Volumes von DP auf RW umzuwandeln und I/O von Site B zu ermöglichen.
+
Sie müssen den SnapMirror break-Befehl für jede Beziehung in der Konsistenzgruppe ausführen. Wenn die Konsistenzgruppe drei Volumes enthält, wird der Befehl für jedes Volume ausgeführt.

. Die Knoten von Standort A werden gestartet, um eine Zero-RTO-Beziehung von Standort B zu Standort A zu erstellen.
. Die Konsistenzgruppe mit der Option "relationship-info-only" auf Site A freigeben, um den gemeinsamen Snapshot beizubehalten und die zugehörigen LUNs der Konsistenzgruppe aufzuheben.
. Die Volumes am Standort A werden von RW zu DP konvertiert, indem auf Volume-Ebene eine Beziehung mit entweder einer Sync-Policy oder einer Async-Policy eingerichtet wird.
. Der  `snapmirror resync`Befehl wird verwendet, um die Beziehungen zu synchronisieren.
. Die SnapMirror-Beziehungen mit Sync-Richtlinie auf Standort A löschen.
. Die SnapMirror-Beziehungen mit der Sync-Richtlinie auf Site B unter Verwendung von relationship-info-only true aufheben.
. Eine Konsistenzgruppenbeziehung zwischen Standort B und Standort A erstellen.
. Von dem Quell-Cluster aus die Konsistenzgruppe erneut synchronisieren. Überprüfen, ob der Status der Konsistenzgruppe synchron ist.
. Die Host-LUN-I/O-Pfade werden erneut gescannt, sodass alle Pfade zu den LUNs wiederhergestellt werden.
