= ONTAP Cluster für SnapMirror Active Sync konfigurieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
SnapMirror active sync verwendet gekoppelte Cluster, um Ihre Daten im Falle eines Failover-Szenarios zu schützen. Vor der Konfiguration von ONTAP Mediator oder ONTAP Cloud Mediator für SnapMirror active sync muss zunächst sichergestellt werden, dass der Cluster korrekt konfiguriert ist.

.Bevor Sie beginnen
Bevor Sie ONTAP Mediator oder ONTAP Cloud Mediator konfigurieren, sollten Sie Folgendes überprüfen:

. Zwischen den Clustern besteht eine Cluster Peer-Beziehung.
+

NOTE: Der Standard-IPspace ist für SnapMirror active sync für Cluster-Peer-Beziehungen erforderlich. Ein benutzerdefinierter IPspace wird nicht unterstützt.

+
link:../peering/create-cluster-relationship-93-later-task.html["Erstellung einer Cluster Peer-Beziehung"]

. Die SVMs werden auf jedem Cluster erstellt.
+
link:../smb-config/create-svms-data-access-task.html["Erstellen einer SVM"]

. Eine Peer-Beziehung besteht zwischen den SVMs in jedem Cluster.
+
link:../peering/create-intercluster-svm-peer-relationship-93-later-task.html["Erstellen einer SVM Peer-Beziehung"]

. Die Volumes existieren für Ihre LUNs.
+
link:../smb-config/create-volume-task.html["Erstellen eines Volumens"]

. Auf jedem Knoten beider Cluster wird mindestens ein SAN LIF (entweder FC oder iSCSI, je nach Anwendbarkeit) für Ihre SAN-Konfiguration erstellt, oder auf jedem Knoten im primären Cluster wird mindestens ein NAS LIF (entweder NFS oder SMB, je nach Anwendbarkeit) für die Replikation in den sekundären Cluster erstellt.
+
link:../san-admin/manage-lifs-all-san-protocols-concept.html["Überlegungen zu LIFs in einer Cluster-SAN-Umgebung"]

+
link:../networking/create_a_lif.html["Erstellung eines LIF"]

+
link:../nfs-config/index.html["Informationen zur NFS Konfiguration"]

+
link:../smb-config/index.html["Informationen zur SMB Konfiguration"]

. Nur SAN: Die erforderlichen LUNs werden erstellt und einer Initiatorgruppe zugeordnet, die dazu dient, LUNs dem Initiator auf dem Anwendungshost zuzuordnen.
+
link:../san-admin/provision-storage.html["LUNs erstellen und Initiatorgruppen zuordnen"]

. Der Anwendungshost wird erneut gescannt, damit neue LUNs oder Volumes erkannt werden.
