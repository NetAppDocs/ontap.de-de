= Geplantes Failover von ONTAP Clustern in einer SnapMirror Active Sync-Beziehung durchführen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Bei einem geplanten Failover von ONTAP Clustern in einer SnapMirror active sync Beziehung werden die Rollen des primären und sekundären Clusters getauscht, sodass der sekundäre Cluster die Aufgaben des primären Clusters übernimmt. Während eines Failovers verarbeitet der normalerweise sekundäre Cluster Ein- und Ausgabeanforderungen lokal, ohne den Clientbetrieb zu unterbrechen.

Ein geplantes Failover kann durchgeführt werden, um die Funktionsfähigkeit der Disaster-Recovery-Konfiguration zu testen oder Wartungsarbeiten am primären Cluster vorzunehmen.

.Über diese Aufgabe
Ein geplanter Failover wird vom Administrator des sekundären Clusters initiiert. Der Vorgang erfordert das Tauschen der primären und sekundären Rollen, sodass der sekundäre Cluster die Aufgaben des primären Clusters übernimmt. Der neue primäre Cluster kann dann die Verarbeitung von Ein- und Ausgabeanfragen lokal aufnehmen, ohne den Clientbetrieb zu unterbrechen.

.Bevor Sie beginnen
* Die SnapMirror active sync-Beziehung muss synchron sein.
* Ein geplanter Failover kann nicht eingeleitet werden, wenn ein unterbrechungsfreier Vorgang ausgeführt wird. Unterbrechungsfreie Vorgänge umfassen Volume-Verschiebungen, Aggregat-Relokationen und Storage Failover.
* Der ONTAP Mediator muss konfiguriert, verbunden und im Quorum sein.


.Schritte
Ein geplantes Failover kann mithilfe der ONTAP CLI oder des System Manager durchgeführt werden.

[role="tabbed-block"]
====
.System Manager
--

IMPORTANT: Von ONTAP 9.8 bis ONTAP 9.14.1 wird SnapMirror active sync als SnapMirror Business Continuity (SM-BC) bezeichnet.

. Im System Manager **Schutz > Übersicht > Beziehungen** auswählen.
. Identifizieren Sie die SnapMirror Active Sync-Beziehung, für die ein Failover durchgeführt werden soll. Neben dem Namen der Beziehung das  `...` auswählen und anschließend **Failover** wählen.
. Zum Überwachen des Status des Failovers kann der  `snapmirror failover show` Befehl in der ONTAP CLI genutzt werden.


--
.CLI
--
. Vom Ziel-Cluster aus den Failover-Vorgang einleiten:
+
`snapmirror failover start -destination-path  <path>`

+
Beispiel für das Failover einer SAN-Konfiguration:

+
destination::>snapmirror failover start -destination-path vs1_dst:/cg/cg_dst

+
Beispiel für das Failover einer NAS-Konfiguration:

+
destination::>snapmirror failover start -destination-path vs_b:

. Vom Ziel aus lässt sich der Fortschritt des Failovers überwachen:
+
`snapmirror failover show`

. Nach Abschluss des Failover-Vorgangs kann der Status der SnapMirror synchronen Schutzbeziehung vom Ziel-Cluster aus überwacht werden:
+
`snapmirror show`
