= Geplantes ONTAP Cluster Failover fehlgeschlagen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die folgenden Informationen sind relevant, falls die geplante Failover-Operation fehlschlägt.

.Problem:
Nach Ausführung des  `snapmirror failover start` Befehls wird in der Ausgabe des  `snapmirror failover show` Befehls eine Meldung angezeigt, die darauf hinweist, dass ein unterbrechungsfreier Vorgang im Gange ist.

....
Cluster1::> snapmirror failover show
Source Destination Error
Path Path Type Status start-time end-time Reason
-------- ----------- -------- --------- ---------- ---------- ----------
vs1:/cg/cg vs0:/cg/cg planned failed 10/1/2020 10/1/2020 SnapMirror Failover cannot start because a volume move is running. Retry the command once volume move has finished.
                                                          08:35:04 08:35:04
....
.Ursache:
Ein geplanter Failover kann nicht gestartet werden, wenn ein unterbrechungsfreier Vorgang im Gange ist, einschließlich Volume Move, Aggregat-Relocation und Storage Failover.

.Lösung:
Warten Sie, bis der unterbrechungsfreie Vorgang abgeschlossen ist, und versuchen Sie den Failover-Vorgang erneut.

.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-failover-show.html["snapmirror failover show"^]
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-failover-start.html["SnapMirror Failover Start"^]
