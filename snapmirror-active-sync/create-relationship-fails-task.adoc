= Fehler beim Erstellen einer ONTAP SnapMirror Beziehung und beim Initialisieren der Konsistenzgruppe
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die folgenden Informationen sind relevant, wenn die Erstellung einer SnapMirror Beziehung und die Initialisierung einer Konsistenzgruppe fehlschlagen.

.Problem:
Erstellung der SnapMirror Beziehung und Initialisierung der Konsistenzgruppe schlägt fehl.

.Lösung:
Stellen Sie sicher, dass Sie die maximale Anzahl an Konsistenzgruppen pro Cluster nicht überschritten haben. Die Grenzwerte für Konsistenzgruppen in SnapMirror active sync sind plattformunabhängig und unterscheiden sich je nach Version von ONTAP. Siehe link:limits-reference.html["Objektgrenzen"] für Hinweise, die speziell für Ihre ONTAP Version gelten.

.Fehler:
Wenn die Konsistenzgruppe beim Initialisieren hängen bleibt, kann der Status der Konsistenzgruppen-Initialisierungen mit der ONTAP REST API, dem System Manager oder dem Befehl  `sn show -expand` überprüft werden.


IMPORTANT: Von ONTAP 9.8 bis ONTAP 9.14.1 wird SnapMirror active sync als SnapMirror Business Continuity (SM-BC) bezeichnet.

.Lösung:
Wenn die Initialisierung von Konsistenzgruppen fehlschlägt, sollte die SnapMirror active sync Beziehung entfernt, die Konsistenzgruppe gelöscht und anschließend die Beziehung neu erstellt und initialisiert werden. Dieser Ablauf unterscheidet sich je nach verwendeter ONTAP-Version.
