= Der Referenz-Snapshot für die Volumenverschiebung wird als der neueste für die ONTAP SnapMirror Beziehung angezeigt.
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die folgenden Informationen gelten, wenn der Referenz-Snapshot der Volume-Verschiebung nach einer Volume-Verschiebung als der neueste für die SnapMirror Beziehung angezeigt wird.

.Problem:
Nach der Durchführung einer Volume-Verschiebung auf einem Konsistenzgruppe-Volume kann der Referenz-Snapshot der Volume-Verschiebung fälschlicherweise als der neueste für die SnapMirror Beziehung angezeigt werden.

Der neueste Snapshot kann mit folgendem Befehl angezeigt werden:

`snapmirror show -fields newest-snapshot status -expand`

.Lösung:
Manuell eine `snapmirror resync`durchführen oder auf die nächste automatische Resynchronisierung nach Abschluss des Volume-Verschiebungsvorgangs warten.

.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-resync.html["SnapMirror Resynchronisierung"^]
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-show.html["snapmirror show"^]
