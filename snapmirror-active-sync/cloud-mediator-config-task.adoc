= Der ONTAP Cloud Mediator für SnapMirror Aktiv-Sync wird konfiguriert
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.17.1 kann mit ONTAP Cloud Mediator die Geschäftskontinuität gewährleistet werden, indem der Zustand jedes Clusters überwacht wird. ONTAP Cloud Mediator ist ein Cloud-basierter Dienst. Bei der Verwendung von ONTAP Cloud Mediator mit SnapMirror active sync ist zunächst zu bestätigen, dass die NetApp Console-Dienste und Clientinformationen konfiguriert sind und ein korrektes Cluster-Peering besteht.

Wie beim ONTAP Mediator bietet auch ONTAP Cloud Mediator einen persistenten und geschützten Speicher für Hochverfügbarkeits-(HA)-Metadaten, die von den ONTAP Clustern in einer SnapMirror Aktiv/Aktiv-Synchronisierungsbeziehung verwendet werden. ONTAP Cloud Mediator bietet eine synchrone Funktion zur Abfrage des Knotenzustands zur Unterstützung der Quorumbestimmung und dient als Ping-Proxyserver zur Erkennung der Controller-Lebendigkeit.


IMPORTANT: Wenn Sie SnapMirror active sync und ONTAP Mediator oder ONTAP Cloud Mediator mit ONTAP 9.17.1 verwenden, sollten Sie die *Bekannten Probleme und Einschränkungen* in der link:https://library.netapp.com/ecm/ecm_download_file/ECMLP2492508["ONTAP Versionshinweise"^] überprüfen, um wichtige Informationen zu diesen Konfigurationen zu erhalten.

.Bevor Sie beginnen
Bevor Sie ONTAP Cloud Mediator konfigurieren, sollten die folgenden Informationen bestätigt werden:

* Der Cluster ist konfiguriert.
+
link:cluster-config-task.html["ONTAP Cluster für SnapMirror Active Sync konfigurieren"]

* Sie haben Ihre NetApp Console-Organisations-ID aus der NetApp Console kopiert und ein Console-Mitgliedsservicekonto erstellt, das bei der Konfiguration von ONTAP Cloud Mediator verwendet wird. Beim Erstellen des Servicekontos muss die Organisation auf das Abonnement festgelegt werden, in dem der ONTAP Cloud Mediator konfiguriert wurde. Die Kategorie muss auf *Application* und der Rollentyp auf *ONTAP Mediator Setup Role* gesetzt werden. Die Client-ID und das Client Secret müssen beim Erstellen der Rolle gespeichert werden.
+
link:https://docs.netapp.com/us-en/console-setup-admin/task-iam-manage-members-permissions.html#add-members["Mitglieder der NetApp Console und Servicekonten hinzufügen"^]



.Schritte
ONTAP Cloud Mediator kann über den System Manager oder die ONTAP CLI hinzugefügt werden.

[role="tabbed-block"]
====
.System Manager
--
. Zu *Schutz > Übersicht > Mediator* navigieren und *Hinzufügen* auswählen.
. Im Fenster *Mediator hinzufügen* ist als Mediatortyp *Cloud* auszuwählen und die folgenden Informationen sind einzugeben:
+
** NetApp Console Organisations-ID
** NetApp Console Client-ID
** NetApp Console-Client-Geheimnis


. Den Cluster-Peer auswählen.
. Falls ein HTTP-Proxy verwendet wird und dieser noch nicht konfiguriert ist, sind die HTTP-Proxy-Informationen für den lokalen und den Remote-Host einzugeben.
+
Es wird empfohlen, für jeden Cluster-Peer einen anderen Proxyserver zu verwenden.

. Optional: Falls ein Root-CA-Zertifikat in ONTAP installiert werden muss, insbesondere bei Verwendung eines Proxyserver, das Zertifikat in das dafür vorgesehene Textfeld einfügen.
. Wählen Sie *Add*.
. Navigieren Sie zu *Schutz > Übersicht* und prüfen Sie den Status der Beziehung zwischen den SnapMirror Aktiv/Aktiv-Clustern und ONTAP Cloud Mediator.


--
.CLI
--
. ONTAP Cloud Mediator konfigurieren:
`snapmirror mediator add -peer-cluster <peerClusterName> -type cloud -bluexp-org-id <NetApp Console Organization ID> -service-account-client-id <Service Account Client ID> -use-http-proxy-local <true|false> -use-http-proxy-remote <true|false>`
. ONTAP Cloud Mediator Status prüfen:
`snapmirror mediator show`
+
Beispiel:

+
[listing]
----
C1_cluster::> snapmirror mediator show
Mediator Address Peer Cluster     Connection Status Quorum Status Type
---------------- ---------------- ----------------- ------------- -------
0.0.0.0          C2_cluster       connected         true          cloud
----
