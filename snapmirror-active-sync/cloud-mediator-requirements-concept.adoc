= Vorbereitung der Konfiguration des ONTAP Cloud Mediators
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Bevor Sie link:cloud-mediator-config-task.html["ONTAP Cloud Mediator konfigurieren"] beginnen, muss sichergestellt sein, dass die Voraussetzungen erfüllt sind.



[[firewall-requirements]]
== Firewall-Anforderungen

Die Firewall-Einstellungen auf dem Domänencontroller müssen HTTPS-Datenverkehr  `api.bluexp.netapp.com` von beiden Clustern zulassen.



[[proxy-server-requirements]]
== Anforderungen an den Proxyserver

Wenn Sie Proxyserver für SnapMirror Active Sync verwenden, ist sicherzustellen, dass die Proxyserver erstellt wurden und folgende Proxyserver-Informationen vorliegen:

* HTTPS Proxy-IP
* Port
* Benutzername
* Passwort




[[latency]]
== Latenz

Die empfohlene Ping-Latenz zwischen dem NetApp Console Cloud-Server und SnapMirror Aktiv/Synchron-Cluster-Peers beträgt weniger als 200 ms.

.Proxyserver auf installierte Zertifikate prüfen
Wenn Sie einen Proxy verwenden, um eine Verbindung zum ONTAP Cloud Mediator Service in der NetApp Console herzustellen, ist sicherzustellen, dass die Root-CA-Zertifikate des Proxyservers in ONTAP installiert sind:

Beispiel:

[listing]
----
C1_cluster% openssl s_client -showcerts -proxy <ip:port> -connect api.bluexp.netapp.com:443 -servername api.bluexp.netapp.com |egrep "s:|i:"
----
.Das CA-Zertifikat herunterladen:
Bei Bedarf können die Root-CA-Zertifikate von der Website der Zertifizierungsstelle heruntergeladen und auf den Clustern installiert werden.

Beispiel:

[listing]
----
C1_cluster::> security certificate install -type server-ca -vserver C1_cluster

C2_cluster::> security certificate install -type server-ca -vserver C2_cluster
----
