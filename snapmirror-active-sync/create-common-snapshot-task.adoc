= Einen gemeinsamen Snapshot zwischen ONTAP Konsistenzgruppen oder NAS SVMs erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Zusätzlich zu den regelmäßig geplanten Snapshot-Operationen kann eine gemeinsame link:../concepts/snapshot-copies-concept.html["Schnappschuss"] zwischen den Volumes in der primären SnapMirror Konsistenzgruppe und den Volumes in der sekundären SnapMirror Konsistenzgruppe in einer SAN-Konfiguration oder auf SVM-Ebene bei NAS-Konfigurationen manuell erstellt werden.

.Über diese Aufgabe
Das geplante Snapshot-Erstellungsintervall beträgt 12 Stunden für die Richtlinie Automated Failover und 3 Stunden für die Richtlinie Automated Failover Duplex.

.Bevor Sie beginnen
* Die SnapMirror Gruppenbeziehung muss synchron sein.


.Schritte
Sie können einen gemeinsamen Snapshot mit System Manager oder der ONTAP CLI erstellen.

[role="tabbed-block"]
====
.System Manager
--
. Im Ziel-Cluster zu *Schutz > Replikation* navigieren.
. Die Beziehungsquelle suchen, image:icon_kabob.gif["Symbol für Menüoptionen"] auswählen und *Aktualisieren* wählen.
. *Gemäß Richtlinie* oder *Snapshot auswählen* kann gewählt werden, um einen vorhandenen Snapshot auszuwählen.
. Wählen Sie *Aktualisieren*.


--
.CLI
--
. Vom Zielstandort aus einen gemeinsamen Snapshot erstellen:
+
`snapmirror update -destination-path <destination path>`

+
Beispiel für eine SAN Konfiguration:

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
Beispiel für eine NAS Konfiguration:

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. Der Fortschritt des Updates kann überwacht werden:
+
`destination::>snapmirror show -fields newest-snapshot`
