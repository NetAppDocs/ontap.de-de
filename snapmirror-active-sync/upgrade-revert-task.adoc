= Upgrade und Rücksetzung mit ONTAP SnapMirror active sync
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
SnapMirror active sync wird ab ONTAP 9.9.1 unterstützt. Das Aktualisieren und Zurücksetzen Ihres ONTAP Clusters oder Ihrer Controller hat Auswirkungen auf Ihre SnapMirror active sync Beziehungen, abhängig von der ONTAP Version, auf die Sie aktualisieren oder zurücksetzen.



[[refresh-a-cluster]]
== Cluster aktualisieren

Ab ONTAP 9.16.1 unterstützt SnapMirror Active Sync Vier-Knoten-Cluster in symmetrischen Aktiv/Aktiv-Konfigurationen. Mit dem Vier-Knoten-Cluster ist ein Upgrade von Controllern und Speicher möglich.

.Bevor Sie beginnen
* Lesen Sie die link:protect-task.html#configure-symmetric-activeactive-protection["Anforderungen für Vier-Knoten-Cluster"].
* Während des Technologieaktualisierung-Prozesses können asymmetrische Konfigurationen erstellt werden; nach Abschluss der Aktualisierung sollte jedoch zu einer symmetrischen Konfiguration zurückgekehrt werden.
* Diese Anweisungen gelten für eine bestehende Vier-Knoten-Konfiguration mit 50 oder weniger Konsistenzgruppen und 400 oder weniger Volume-Endpunkten.


.Schritte
. link:../volumes/move-volume-task.html["Alle SnapMirror active sync Volumes auf ein einziges Hochverfügbarkeitspaar (HA) verschieben"].
. link:../system-admin/remove-nodes-cluster-concept.html["Nicht verwendete Knoten aus dem Cluster entfernen"].
. link:../system-admin/add-nodes-cluster-concept.html["Die neuen Knoten werden dem Cluster hinzugefügt."].
. link:../volumes/move-volume-task.html["Alle Volumes verschieben"] in die neuen Knoten.
. link:../system-admin/remove-nodes-cluster-concept.html["Nicht verwendete Knoten aus dem Cluster entfernen"]dann werden sie link:../system-admin/add-nodes-cluster-concept.html["mit den neuen Knoten"] ersetzt.




[[upgrade-ontap-with-snapmirror-active-sync]]
== ONTAP mit SnapMirror Active Sync aktualisieren

Um SnapMirror active sync zu verwenden, müssen auf allen Knoten der Quell- und Ziel-Cluster ONTAP 9.9.1 oder höher ausgeführt werden.

Bei der Aktualisierung von ONTAP mit Aktiv/Aktiv-SnapMirror active sync-Beziehungen empfiehlt sich die Verwendung von xref:../upgrade/automated-upgrade-task.html[automatisiertes unterbrechungsfreies Upgrade (ANDU)]. Durch die Nutzung von ANDU bleibt gewährleistet, dass Ihre Aktiv/Aktiv-SnapMirror active sync-Beziehungen während des Upgrade-Prozesses synchron und fehlerfrei sind.

Für SnapMirror Aktiv/Aktiv-Synchronisierungsbereitstellungen sind keine Konfigurationsschritte zur Vorbereitung auf ONTAP Upgrades erforderlich. Es wird jedoch empfohlen, vor und nach dem Upgrade Folgendes zu überprüfen:

* SnapMirror Active Sync-Beziehungen sind synchron.
* Im Ereignisprotokoll sind keine Fehler im Zusammenhang mit SnapMirror verzeichnet.
* Der Mediator ist online und in beiden Clustern funktionsfähig.
* Alle Hosts können alle Pfade ordnungsgemäß sehen, um LUNs zu schützen.



NOTE: Wenn Cluster von ONTAP 9.9.1 oder 9.9.1 auf ONTAP 9.10.1 und höher aktualisiert werden, erstellt ONTAP neue xref:../consistency-groups/index.html[Konsistenzgruppen] auf sowohl Quell- als auch Ziel-Clustern für SnapMirror active sync Beziehungen, die mit System Manager konfiguriert werden können.


NOTE: Die  `snapmirror quiesce` und  `snampirror resume` Befehle werden mit SnapMirror active sync nicht unterstützt.



[[revert-to-ontap-9-9-1-from-ontap-9-10-1]]
== Zurück zu ONTAP 9.9.1 von ONTAP 9.10.1

Um Beziehungen von 9.10.1 auf 9.9.1 zurückzusetzen, müssen SnapMirror active sync-Beziehungen gelöscht werden, gefolgt von der 9.10.1 Konsistenzgruppeninstanz. Konsistenzgruppen mit einer aktiven SnapMirror active sync-Beziehung können nicht gelöscht werden. Alle FlexVol Volumes, die auf 9.10.1 aktualisiert wurden und zuvor in 9.9.1 oder früher mit einem anderen Smart Container oder einer Enterprise App verknüpft waren, sind nach dem Zurücksetzen nicht mehr verknüpft. Das Löschen von Konsistenzgruppen löscht weder die zugehörigen Volumes noch die Volume-Granularitäts-Snapshots. Weitere Informationen zu dieser Aufgabe in ONTAP 9.10.1 und höher enthält link:../consistency-groups/delete-task.html["Eine Konsistenzgruppe löschen"].



[[revert-from-ontap-9-9-1]]
== Zurücksetzen von ONTAP 9.9.1


NOTE: SnapMirror active sync wird bei gemischten ONTAP Clustern, die Versionen vor ONTAP 9.9.1 enthalten, nicht unterstützt.

Beim Zurücksetzen von ONTAP 9.9.1 auf eine frühere Version von ONTAP ist Folgendes zu beachten:

* Wenn der Cluster ein SnapMirror Active Sync-Ziel hostet, ist ein Downgrade auf ONTAP 9.8 oder früher erst zulässig, wenn die Beziehung getrennt und gelöscht wurde.
* Wenn der Cluster eine SnapMirror Aktiv-Sync-Quelle hostet, ist ein Zurücksetzen auf ONTAP 9.8 oder früher erst zulässig, nachdem die Beziehung freigegeben wurde.
* Alle vom Benutzer erstellten benutzerdefinierten SnapMirror Active Sync-Richtlinien müssen gelöscht werden, bevor auf ONTAP 9.8 oder eine frühere Version zurückgegriffen wird.


Um diese Anforderungen zu erfüllen, siehe link:remove-configuration-task.html["Entfernen einer SnapMirror Active Sync Konfiguration"].

.Schritte
. Bestätigen Sie Ihre Bereitschaft zum Zurücksetzen, indem Sie den folgenden Befehl von einem der Cluster in der SnapMirror active sync-Beziehung eingeben:
+
`cluster::> system node revert-to -version 9.7 -check-only`

+
