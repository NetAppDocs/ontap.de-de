= Informationen zur Clusterverwaltung mit ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
System Manager ist eine auf HTML5 basierende grafische Managementoberfläche, die die Verwaltung von Speichersystemen und Speicherobjekten (wie Festplatten, Volumes und Speicherebenen) über einen Webbrowser sowie die Ausführung gängiger Verwaltungsaufgaben im Zusammenhang mit Speichersystemen ermöglicht.

[NOTE]
====
* System Manager ist in der ONTAP-Software als Webdienst enthalten, standardmäßig aktiviert und über einen Browser zugänglich.
* Der Name von System Manager hat sich ab ONTAP 9.6 geändert. In ONTAP 9.5 und früher wurde er OnCommand System Manager genannt. Ab ONTAP 9.6 und später heißt er System Manager.
* Wenn der klassische System Manager verwendet wird (nur in ONTAP 9.7 und früher verfügbar), siehe  https://docs.netapp.com/us-en/ontap-system-manager-classic/index.html["System Manager Classic (ONTAP 9.0 bis 9.7)"^]


====
Mithilfe des System Manager Dashboards werden auf einen Blick Informationen zu wichtigen Warnungen und Benachrichtigungen, zur Effizienz und Kapazität von Speicherebenen und Volumes, zu den in einem Cluster verfügbaren Knoten, zum Status der Knoten in einem HA-Paar, zu den aktivsten Anwendungen und Objekten sowie zu den Leistungskennzahlen eines Clusters oder eines Knotens angezeigt.

Mit dem System Manager in ONTAP 9.7 und späteren Versionen lassen sich viele gängige Aufgaben wie die folgenden durchführen:

* Ein Cluster wird erstellt, ein Netzwerk konfiguriert und Supportdetails für den Cluster eingerichtet.
* Speicherobjekte wie Festplatten, lokale Ebenen, Volumes, Qtrees und Kontingente können konfiguriert und verwaltet werden.
* Protokolle wie SMB und NFS konfigurieren und die Dateifreigabe bereitstellen.
* Protokolle wie FC, FCoE, NVMe und iSCSI für den Blockzugriff konfigurieren.
* Netzwerkkomponenten wie Subnetze, Broadcast-Domänen, Daten- und Managementoberflächen sowie Schnittstellengruppen erstellen und konfigurieren.
* Spiegelungs- und Vault-Beziehungen einrichten und verwalten.
* Cluster-Management, Speicherknoten-Management und Management von Storage Virtual Machines (Storage-VMs) werden durchgeführt.
* Speicher-VMs erstellen und konfigurieren, Speicherobjekte verwalten, die mit Speicher-VMs verknüpft sind, sowie Speicher-VM-Dienste verwalten.
* Hochverfügbarkeitskonfigurationen (HA) in einem Cluster überwachen und verwalten.
* Serviceprozessoren können so konfiguriert werden, dass sie sich unabhängig vom Zustand des Knotens remote anmelden, den Knoten verwalten, überwachen und administrieren.




[[system-manager-terminology]]
== System Manager-Terminologie

Der System Manager verwendet für einige wichtige ONTAP-Funktionen eine andere Terminologie als die CLI.

* *Lokale Ebene*: Eine Gruppe physischer Solid-State-Laufwerke oder Festplatten, auf denen Sie Ihre Daten speichern. Diese sind möglicherweise als Aggregate bekannt. Tatsächlich wird in der ONTAP-CLI weiterhin der Begriff _Aggregate_ verwendet, um eine lokale Ebene darzustellen.
* *Cloud-Tier*: Speicher in der Cloud, der von ONTAP verwendet wird, wenn ein Teil Ihrer Daten aus verschiedenen Gründen außerhalb des eigenen Standorts gespeichert werden soll. Wer an den Cloud-Anteil einer FabricPool denkt, hat dies bereits erkannt. Und bei Verwendung eines StorageGRID Systems muss sich die Cloud möglicherweise gar nicht außerhalb des eigenen Standorts befinden. (Ein cloudähnliches Erlebnis am eigenen Standort wird als _private Cloud_ bezeichnet.)
* *Storage-VM*: Eine virtuelle Maschine, die innerhalb von ONTAP Speicher- und Datendienste für Ihre Clients bereitstellt. Dies ist möglicherweise als _SVM_ oder _vserver_ bekannt.
* *Netzwerkschnittstelle*: Adresse und Eigenschaften, die einem physischen Netzwerkport zugewiesen sind. Dies ist möglicherweise als _logische Schnittstelle (LIF)_ bekannt.
* *Pause*: Eine Aktion, die den Betrieb anhält. Vor ONTAP 9.8 wurde in anderen Versionen von System Manager möglicherweise _quiesce_ genannt.
