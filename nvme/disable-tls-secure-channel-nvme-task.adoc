= TLS-Sicherheitskanal für NVMe/TCP deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.16.1 kann ein TLS-Sicherheitskanal für NVMe/TCP-Verbindungen konfiguriert werden. Wenn ein TLS-Sicherheitskanal für NVMe/TCP-Verbindungen konfiguriert wurde, besteht jederzeit die Möglichkeit, ihn zu deaktivieren.

.Schritte
. Entfernen Sie den Host aus dem Subsystem, um den sicheren TLS-Kanal zu deaktivieren:
+
[source, cli]
----
vserver nvme subsystem host remove -vserver <svm_name> -subsystem <subsystem> -host-nqn <host_nqn>
----
. Es sollte überprüft werden, ob der sichere TLS-Kanal vom Host entfernt wurde:
+
[source, cli]
----
vserver nvme subsystem host show
----
. Den Host ohne sicheren TLS-Kanal wieder zum Subsystem hinzufügen:
+
[source, cli]
----
vserver nvme subsystem host add vserver <svm_name> -subsystem <subsystem> -host-nqn <host_nqn>
----
