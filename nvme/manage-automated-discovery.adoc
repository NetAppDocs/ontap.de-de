= Automatische Host-Erkennung von NVMe/TCP-Controllern in ONTAP verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.14.1 ist die Host-Erkennung von Controllern über das NVMe/TCP-Protokoll in IP-basierten Fabrics standardmäßig automatisiert.



[[enable-automated-host-discovery-of-nvmetcp-controllers]]
== Automatische Host-Erkennung von NVMe/TCP Controllern ermöglichen

Falls die automatische Host-Erkennung zuvor deaktiviert wurde, sich die Anforderungen jedoch geändert haben, kann sie wieder aktiviert werden.

.Schritte
. Erweiterten Berechtigungsmodus aktivieren:
+
[source, cli]
----
set -privilege advanced
----
. Automatische Erkennung:
+
[source, cli]
----
vserver nvme modify -vserver <vserver_name> -mdns-service-discovery-enabled true
----
. Es wird überprüft, ob die automatische Erkennung von NVMe/TCP Controllern aktiviert ist.
+
[source, cli]
----
vserver nvme show -fields mdns-service-discovery-enabled
----




[[disable-automated-host-discovery-of-nvmetcp-controllers]]
== Automatische Erkennung von NVMe/TCP-Controllern auf Hosts deaktivieren

Wenn es nicht erforderlich ist, dass NVMe/TCP Controller von Ihrem Host automatisch erkannt werden und unerwünschter Multicast-Verkehr in Ihrem Netzwerk festgestellt wird, empfiehlt sich das Deaktivieren dieser Funktionalität.

.Schritte
. Erweiterten Berechtigungsmodus aktivieren:
+
[source, cli]
----
set -privilege advanced
----
. Automatische Erkennung deaktivieren:
+
[source, cli]
----
vserver nvme modify -vserver <vserver_name> -mdns-service-discovery-enabled false
----
. Es wird überprüft, ob die automatische Erkennung von NVMe/TCP Controllern deaktiviert ist.
+
[source, cli]
----
vserver nvme show -fields mdns-service-discovery-enabled
----
