= Lokale UNIX-Benutzerlisten auf ONTAP NFS SVMs laden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Alternativ zur manuellen Erstellung einzelner lokaler UNIX-Benutzer in SVMs lässt sich die Aufgabe vereinfachen, indem eine Liste lokaler UNIX-Benutzer aus einem Uniform Resource Identifier (URI (`vserver services name-service unix-user load-from-uri`) in SVMs geladen wird.

.Schritte
. Eine Datei mit der Liste der lokalen UNIX-Benutzer, die geladen werden sollen, erstellen.
+
Die Datei muss Benutzerinformationen im UNIX `/etc/passwd`-Format enthalten:

+
`_user_name: password: user_ID: group_ID: full_name_`

+
Der Befehl verwirft den Wert des `_password_` Feldes und die Werte der Felder nach dem `_full_name_` Feld(`_home_directory_` und `_shell_`).

+
Die maximal unterstützte Dateigröße beträgt 2,5 MB.

. Es ist sicherzustellen, dass die Liste keine doppelten Einträge enthält.
+
Enthält die Liste doppelte Einträge, schlägt das Laden der Liste mit einer Fehlermeldung fehl.

. Kopieren Sie die Datei auf einen Server.
+
Der Server muss vom Speichersystem über HTTP, HTTPS, FTP oder FTPS erreichbar sein.

. Ermittelt wird, welcher URI für die Datei gilt.
+
Der URI ist die Adresse, die Sie dem Speichersystem angeben, um den Speicherort der Datei zu kennzeichnen.

. Die Datei mit der Liste der lokalen UNIX-Benutzer wird aus der URI in die SVMs geladen:
+
`vserver services name-service unix-user load-from-uri -vserver _vserver_name_ -uri {ftp|http|ftps|https}://_uri_ -overwrite {true|false}`

+
`-overwrite` {`true` `false`} gibt an, ob Einträge überschrieben werden. Der Standardwert ist `false`.



.Beispiel
Der folgende Befehl lädt eine Liste lokaler UNIX-Benutzer aus der URI `+ftp://ftp.example.com/passwd+` in die SVM mit dem Namen vs1. Vorhandene Benutzer auf der SVM werden durch die Informationen aus der URI nicht überschrieben.

[listing]
----
node::> vserver services name-service unix-user load-from-uri -vserver vs1
-uri ftp://ftp.example.com/passwd -overwrite false
----
