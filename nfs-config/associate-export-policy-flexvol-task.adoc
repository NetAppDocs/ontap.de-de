= Einer Volume wird eine ONTAP NFS-Exportregel zugewiesen.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Jedes im SVM enthaltene Volume muss mit einer Exportrichtlinie verknüpft sein, die Exportregeln für Clients enthält, damit auf Daten im Volume zugegriffen werden kann.

.Über diese Aufgabe
Sie können einer Volume beim Erstellen oder jederzeit danach eine Exportregel zuweisen. Sie können einer Volume nur eine Exportregel zuweisen, wobei eine Exportregel mehreren Volumes zugeordnet werden kann.

.Schritte
. Wurde beim Erstellen des Volumes keine Exportregel angegeben, ist dem Volume eine Exportregel zuzuweisen:
+
`volume modify -vserver _vserver_name_ -volume _volume_name_ -policy _export_policy_name_`

. Es wird überprüft, ob die Richtlinie dem Volume zugewiesen wurde:
+
`volume show -volume _volume_name_ -fields policy`



.Beispiel
Die folgenden Befehle weisen dem Volume vol1 auf der SVM vs1 die Exportregel nfs_policy zu und überprüfen die Zuweisung:

[listing]
----
cluster::> volume modify -v1server vs1 -volume vol1 -policy nfs_policy

cluster::>volume show -volume vol -fields policy
vserver volume       policy
------- ------------ ----------------
vs1     vol1         nfs_policy
----
