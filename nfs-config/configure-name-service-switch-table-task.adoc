= Die ONTAP NFS Name-Service-Switch-Tabelle konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Namensdienst-Switchtabelle muss korrekt konfiguriert sein, damit ONTAP lokale oder externe Namensdienste konsultieren kann, um Host-, Benutzer-, Gruppen-, Netzgruppen- oder Namenszuordnungsinformationen abzurufen.

.Bevor Sie beginnen
Es muss festgelegt sein, welche Namensdienste für Host-, Benutzer-, Gruppen-, Netzgruppen- oder Namenszuordnung in Ihrer Umgebung verwendet werden sollen.

Wenn die Verwendung von Netgroups vorgesehen ist, müssen alle in Netgroups angegebenen IPv6-Adressen gemäß RFC 5952 gekürzt und komprimiert werden.

.Über diese Aufgabe
Geben Sie keine Informationsquellen an, die nicht verwendet werden. Wenn NIS beispielsweise in Ihrer Umgebung nicht verwendet wird, geben Sie die  `-sources nis` Option nicht an.

.Schritte
. Die erforderlichen Einträge zur Name-Service-Switch-Tabelle hinzufügen:
+
`vserver services name-service ns-switch create -vserver _vserver_name_ -database _database_name_ -sources _source_names_`

. Es sollte sichergestellt werden, dass die Namensdienst-Switch-Tabelle die erwarteten Einträge in der gewünschten Reihenfolge enthält:
+
`vserver services name-service ns-switch show -vserver _vserver_name_`

+
Wenn Sie Korrekturen vornehmen möchten, müssen Sie die  `vserver services name-service ns-switch modify` oder  `vserver services name-service ns-switch delete` Befehle verwenden.



.Beispiel
Das folgende Beispiel erstellt einen neuen Eintrag in der Name-Service-Switch-Tabelle für die SVM vs1, sodass die lokale Netgroup-Datei und ein externer NIS-Server in dieser Reihenfolge zur Suche nach Netgroup-Informationen verwendet werden:

[listing]
----
cluster::> vserver services name-service ns-switch create -vserver vs1 -database netgroup -sources files,nis
----
.Nachdem Sie fertig sind
* Die angegebenen Namensdienste müssen für die SVM so konfiguriert werden, dass der Datenzugriff bereitgestellt wird.
* Wenn Sie einen Namendienst für die SVM löschen, müssen Sie ihn auch aus der Namendienst-Switchtabelle entfernen.
+
Der Clientzugriff auf das Speichersystem funktioniert möglicherweise nicht wie erwartet, wenn der Namensdienst nicht aus der Namensdienst-Umschalttabelle gelöscht wird.
