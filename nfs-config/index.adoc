= Informationen zur NFS-Konfiguration mit der ONTAP CLI
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit ONTAP 9 CLI-Befehlen lässt sich der NFS-Client-Zugriff auf Dateien konfigurieren, die sich in einem neuen Volume oder Qtree in einer neuen oder bestehenden Storage Virtual Machine (SVM) befinden.

Diese Vorgehensweisen sind zu verwenden, wenn der Zugriff auf ein Volume oder einen Qtree wie folgt konfiguriert werden soll:

* Es besteht der Wunsch, eine beliebige Version von NFS zu nutzen, die derzeit von ONTAP unterstützt wird: NFSv3, NFSv4, NFSv4.1, NFSv4.2 oder NFSv4.1 mit pNFS.
* Die Befehlszeilenschnittstelle (CLI) soll verwendet werden, nicht der System Manager oder ein automatisiertes Skripttool.
+
Informationen zur Konfiguration des NAS-Multiprotokollzugriffs mit System Manager finden sich unter link:../task_nas_provision_nfs_and_smb.html["NAS-Speicher für Windows und Linux mit NFS und SMB bereitstellen"].

* Sie möchten bewährte Verfahren anwenden, nicht jede verfügbare Option untersuchen.
+
Weitere Informationen zur Befehlssyntax finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].

* UNIX-Dateiberechtigungen werden verwendet, um das neue Volume zu sichern.
* Sie verfügen über Cluster-Administratorrechte, nicht über SVM-Administratorrechte.


Für Einzelheiten zum Funktionsumfang des ONTAP NFS-Protokolls siehe link:../nfs-admin/index.html["Informationen zu ONTAP Dateizugriff für das NFS-Protokoll"].
