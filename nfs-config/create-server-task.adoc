= ONTAP NFS-Server erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem überprüft wurde, dass NFS auf Ihrem Cluster lizenziert ist, kann mit dem  `vserver nfs create` Befehl ein NFS-Server auf der SVM erstellt und die unterstützten NFS-Versionen angegeben werden.

.Über diese Aufgabe
Die SVM kann so konfiguriert werden, dass eine oder mehrere Versionen von NFS unterstützt werden. Falls NFSv4 oder höher unterstützt wird:

* Der Domänenname für die NFSv4-Benutzer-ID-Zuordnung muss auf dem NFSv4-Server und den Ziel-Clients identisch sein.
+
Es muss nicht unbedingt mit einem LDAP- oder NIS-Domänennamen übereinstimmen, solange der NFSv4-Server und die Clients denselben Namen verwenden.

* Zielclients müssen die NFSv4-Einstellung für numerische IDs unterstützen.
* Aus Sicherheitsgründen empfiehlt sich in NFSv4-Bereitstellungen die Verwendung von LDAP für Namensdienste.


.Bevor Sie beginnen
Die SVM muss so konfiguriert sein, dass das NFS-Protokoll zulässig ist.

.Schritte
. Es sollte überprüft werden, ob NFS auf Ihrem Cluster lizenziert ist:
+
`system license show -package nfs`

+
Falls dies nicht der Fall ist, kontaktieren Sie Ihren Vertriebsmitarbeiter.

. Einen NFS-Server erstellen:
+
`vserver nfs create -vserver _vserver_name_ -v3 {enabled|disabled} -v4.0 {enabled|disabled} -v4-id-domain _nfsv4_id_domain_ -v4-numeric-ids {enabled|disabled} -v4.1 {enabled|disabled} -v4.1-pnfs {enabled|disabled}`

+
Sie können beliebige Kombinationen von NFS-Versionen aktivieren. Wenn pNFS unterstützt werden soll, müssen sowohl  `-v4.1` als auch  `-v4.1-pnfs` Optionen aktiviert werden.

+
Wenn Sie Version 4 oder höher aktivieren, sollte außerdem sichergestellt sein, dass die folgenden Optionen korrekt eingestellt sind:

+
** `-v4-id-domain`
+
Dieser optionale Parameter legt den Domänenanteil der Zeichenfolgenform von Benutzer- und Gruppennamen gemäß dem NFSv4-Protokoll fest. Standardmäßig verwendet ONTAP die NIS-Domäne, sofern eine konfiguriert ist; andernfalls wird die DNS-Domäne verwendet. Es ist ein Wert anzugeben, der mit dem von den Zielclients verwendeten Domänennamen übereinstimmt.

** `-v4-numeric-ids`
+
Dieser optionale Parameter legt fest, ob die Unterstützung für numerische Zeichenfolgenbezeichner in NFSv4-Besitzerattributen aktiviert ist. Die Standardeinstellung ist aktiviert, jedoch sollte überprüft werden, ob die Zielclients dies unterstützen.



+
Zusätzliche NFS-Funktionen lassen sich später mit dem  `vserver nfs modify` Befehl aktivieren.

. Es sollte überprüft werden, ob NFS ausgeführt wird:
+
`vserver nfs status -vserver _vserver_name_`

. Es sollte überprüft werden, ob NFS wie gewünscht konfiguriert ist:
+
`vserver nfs show -vserver _vserver_name_`



.Beispiele
Der folgende Befehl erstellt einen NFS-Server auf der SVM namens vs1 mit aktivierten NFSv3 und NFSv4.0:

[listing]
----
vs1::> vserver nfs create -vserver vs1 -v3 enabled -v4.0 enabled -v4-id-domain my_domain.com
----
Die folgenden Befehle überprüfen den Status und die Konfigurationswerte des neuen NFS-Servers mit dem Namen vs1:

[listing]
----
vs1::> vserver nfs status -vserver vs1
The NFS server is running on Vserver "vs1".

vs1::> vserver nfs show -vserver vs1

                            Vserver: vs1
                 General NFS Access: true
                             NFS v3: enabled
                           NFS v4.0: enabled
                       UDP Protocol: enabled
                       TCP Protocol: enabled
               Default Windows User: -
                NFSv4.0 ACL Support: disabled
    NFSv4.0 Read Delegation Support: disabled
   NFSv4.0 Write Delegation Support: disabled
            NFSv4 ID Mapping Domain: my_domain.com
...
----
