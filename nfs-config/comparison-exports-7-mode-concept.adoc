= Informationen zu den Vergleichen der NFS-Exporte zwischen 7-Mode und ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Exporte in ONTAP werden anders definiert und verwendet als in 7-Mode-Umgebungen.

|===


| Unterschiedsbereiche | 7-Mode | ONTAP 


 a| 
Wie Exporte definiert werden
 a| 
Exporte werden in der  `/etc/exports` Datei definiert.
 a| 
Exporte werden durch die Erstellung einer Exportregel innerhalb einer SVM definiert. Eine SVM kann mehr als eine Exportregel enthalten.



 a| 
Exportumfang
 a| 
* Exporte gelten für einen bestimmten Dateipfad oder qtree.
* Für jeden Dateipfad oder Qtree muss ein separater Eintrag in `/etc/exports` erstellt werden.
* Exporte bleiben nur dann persistent, wenn sie in der  `/etc/exports` Datei definiert sind.

 a| 
* Exportrichtlinien gelten für ein gesamtes Volume, einschließlich aller Dateipfade und Qtrees, die im Volume enthalten sind.
* Export-Richtlinien können bei Bedarf auf mehr als ein Volume angewendet werden.
* Alle Exportrichtlinien bleiben auch nach Systemneustarts erhalten.




 a| 
Fencing (Festlegung unterschiedlicher Zugriffsrechte für bestimmte Clients auf dieselben Ressourcen)
 a| 
Um bestimmten Clients unterschiedliche Zugriffsrechte auf eine einzelne exportierte Ressource bereitzustellen, ist es erforderlich, jeden Client und dessen zulässigen Zugriff in der  `/etc/exports` Datei aufzuführen.
 a| 
Exportrichtlinien bestehen aus einer Reihe einzelner Exportregeln. Jede Exportregel definiert spezifische Zugriffsrechte für eine Ressource und listet die Clients auf, die über diese Berechtigungen verfügen. Um unterschiedlichen Zugriff für bestimmte Clients zu ermöglichen, ist für jede spezifische Gruppe von Zugriffsrechten eine Exportregel zu erstellen, die Clients mit diesen Berechtigungen sind aufzulisten und anschließend sind die Regeln der Exportrichtlinie hinzuzufügen.



 a| 
Namensaliasierung
 a| 
Wenn ein Export definiert wird, kann der Name des Exports vom Namen des Dateipfads abweichen. Der  `-actual` Parameter sollte beim Definieren eines solchen Exports in der  `/etc/exports` Datei verwendet werden.
 a| 
Sie können den Namen des exportierten Volumes vom tatsächlichen Volume-Namen abweichen lassen. Dazu muss das Volume mit einem benutzerdefinierten Verbindungspfadnamen innerhalb des SVM-Namensraums eingebunden werden.


NOTE: Standardmäßig werden Volumes mit ihrem Volume-Namen eingebunden. Um den Verbindungspfadnamen eines Volumes anzupassen, ist es erforderlich, das Volume auszuhängen, umzubenennen und anschließend wieder einzubinden.

|===
