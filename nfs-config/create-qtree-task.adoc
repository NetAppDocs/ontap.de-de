= Einen ONTAP NFS Qtree erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen Qtree erstellen, um Ihre Daten zu speichern, und seine Eigenschaften mithilfe des  `volume qtree create` Befehls angeben.

.Bevor Sie beginnen
* Die SVM und das Volume, das den neuen Qtree enthalten soll, müssen bereits existieren.
* Der SVM-Sicherheitsstil muss UNIX sein, und NFS muss eingerichtet und funktionsfähig sein.


.Schritte
. Den Qtree erstellen:
+
`volume qtree create -vserver _vserver_name_ { -volume _volume_name_ -qtree _qtree_name_ | -qtree-path _qtree path_ } -security-style unix [-policy _export_policy_name_]`

+
Sie können das Volume und den Qtree als separate Argumente angeben oder das Qtree-Pfadargument im Format  `/vol/_volume_name/_qtree_name_` angeben.

+
Standardmäßig übernehmen Qtrees die Exportregeln ihres übergeordneten Volumes, können aber auch so konfiguriert werden, dass sie eigene verwenden. Wenn eine vorhandene Exportregel verwendet werden soll, kann diese beim Erstellen des Qtrees angegeben werden. Eine Exportregel kann auch später mit dem  `volume qtree modify` Befehl hinzugefügt werden.

. Es sollte überprüft werden, ob der Qtree mit dem gewünschten Verbindungspfad erstellt wurde:
+
`volume qtree show -vserver _vserver_name_ { -volume _volume_name_ -qtree _qtree_name_ | -qtree-path _qtree path_ }`



.Beispiel
Das folgende Beispiel erstellt einen Qtree namens qt01 auf SVM vs1.example.com, der einen Verbindungspfad  `/vol/data1` hat:

[listing]
----
cluster1::> volume qtree create -vserver vs1.example.com -qtree-path /vol/data1/qt01 -security-style unix
[Job 1642] Job succeeded: Successful

cluster1::> volume qtree show -vserver vs1.example.com -qtree-path /vol/data1/qt01

                      Vserver Name: vs1.example.com
                       Volume Name: data1
                        Qtree Name: qt01
  Actual (Non-Junction) Qtree Path: /vol/data1/qt01
                    Security Style: unix
                       Oplock Mode: enable
                  Unix Permissions: ---rwxr-xr-x
                          Qtree Id: 2
                      Qtree Status: normal
                     Export Policy: default
        Is Export Policy Inherited: true
----
