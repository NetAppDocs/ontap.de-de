= Wo zusätzliche Informationen zu ONTAP NFS zu finden sind
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem Sie den NFS-Client-Zugriff erfolgreich getestet haben, können Sie zusätzliche NFS-Konfigurationen vornehmen oder SAN-Zugriff hinzufügen. Wenn der Protokollzugriff abgeschlossen ist, sollte das Root-Volume der Storage Virtual Machine (SVM) geschützt werden.



[[nfs-configuration]]
== NFS-Konfiguration

Sie können den NFS-Zugriff mithilfe der folgenden Informationen und technischen Berichte weiter konfigurieren:

* link:../nfs-admin/index.html["NFS Management"]
+
Beschreibt die Konfiguration und Verwaltung von Dateizugriff mit NFS.

* https://www.netapp.com/pdf.html?item=/media/10720-tr-4067.pdf["NetApp Technischer Bericht 4067: NFS Best Practices-Leitfaden und Implementierungsleitfaden"^]
+
Dient als Betriebsanleitung für NFSv3 und NFSv4 und bietet einen Überblick über das ONTAP Betriebssystem mit Schwerpunkt auf NFSv4.

* https://www.netapp.com/pdf.html?item=/media/19371-tr-4073.pdf["NetApp Technical Report 4073: Sichere einheitliche Authentifizierung"^]
+
Erläutert, wie ONTAP für die Verwendung mit UNIX-basierten Kerberos Version 5 (krb5)-Servern für die NFS-Authentifizierung und Windows Server Active Directory (AD) als KDC und Lightweight Directory Access Protocol (LDAP) Identitätsanbieter konfiguriert wird.

* https://www.netapp.com/pdf.html?item=/media/16398-tr-3580.pdf["NetApp Technical Report 3580: NFSv4-Erweiterungen und Best Practices-Leitfaden Data ONTAP Implementierung"^]
+
Beschreibt die Best Practices, die bei der Implementierung von NFSv4-Komponenten auf AIX-, Linux- oder Solaris-Clients, die an Systeme mit ONTAP angeschlossen sind, beachtet werden sollten.





[[networking-configuration]]
== Netzwerkkonfiguration

Netzwerkfunktionen und Namensdienste lassen sich mit den folgenden Informationen und technischen Berichten weiter konfigurieren:

* link:../nfs-admin/index.html["NFS Management"]
+
Beschreibt, wie ONTAP Netzwerke konfiguriert und verwaltet werden.

* https://www.netapp.com/pdf.html?item=/media/16885-tr-4182.pdf["NetApp Technical Report 4182: Überlegungen zum Ethernet-Speicherdesign und bewährte Verfahren für Clustered Data ONTAP Konfigurationen"^]
+
Beschreibt die Implementierung von ONTAP Netzwerkkonfigurationen und bietet gängige Netzwerkbereitstellungsszenarien sowie Empfehlungen für bewährte Verfahren.

* https://www.netapp.com/pdf.html?item=/media/16328-tr-4668.pdf["NetApp Technischer Bericht 4668: Name Services Best Practices-Leitfaden"^]
+
Erläutert, wie LDAP, NIS, DNS und die lokale Dateikonfiguration für Authentifizierungszwecke konfiguriert werden.





[[san-protocol-configuration]]
== SAN Protokollkonfiguration

Wenn Sie den SAN-Zugriff für die neue SVM bereitstellen oder ändern möchten, können Sie die FC- oder iSCSI-Konfigurationsinformationen verwenden, die für mehrere Host-Betriebssysteme verfügbar sind.



[[root-volume-protection]]
== Root Volume-Schutz

Nach der Konfiguration der Protokolle auf der SVM sollte sichergestellt werden, dass das Root-Volume geschützt ist:

* link:../data-protection/index.html["Datenschutz"]
+
Beschreibt, wie ein Load-Sharing-Mirror zum Schutz des SVM-Root-Volumes erstellt wird, was eine NetApp Best Practice für NAS-fähige SVMs ist. Beschreibt außerdem, wie eine schnelle Wiederherstellung nach Volume-Ausfällen oder -Verlusten durch das Hochstufen des SVM-Root-Volumes von einem Load-Sharing-Mirror erfolgen kann.
