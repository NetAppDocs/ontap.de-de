= Einer Qtree eine ONTAP NFS-Exportregel zuweisen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Anstatt ein gesamtes Volume zu exportieren, kann auch ein bestimmter qtree auf einem Volume exportiert werden, um ihn Clients direkt zugänglich zu machen. Ein qtree kann exportiert werden, indem ihm eine Exportregel zugewiesen wird. Die Exportregel kann entweder beim Erstellen eines neuen qtrees oder durch Ändern eines bestehenden qtrees zugewiesen werden.

.Bevor Sie beginnen
Die Exportrichtlinie muss vorhanden sein.

.Über diese Aufgabe
Standardmäßig erben Qtrees die übergeordnete Exportregel des enthaltenden Volumes, sofern bei der Erstellung nichts anderes angegeben wurde.

Sie können einer qtree beim Erstellen oder jederzeit nach dem Erstellen eine Exportregel zuordnen. Einer qtree kann nur eine Exportregel zugeordnet werden, wobei eine Regel mit vielen qtrees verknüpft sein kann.

.Schritte
. Wurde beim Erstellen des Qtree keine Exportregel angegeben, kann dem Qtree eine Exportregel zugewiesen werden:
+
`volume qtree modify -vserver _vserver_name_ -qtree-path /vol/_volume_name/qtree_name_ -export-policy _export_policy_name_`

. Es wird überprüft, ob die Richtlinie dem Qtree zugewiesen wurde:
+
`volume qtree show -qtree _qtree_name_ -fields export-policy`



.Beispiel
Die folgenden Befehle weisen dem Qtree qt1 auf der SVM vs1 die Exportregel nfs_policy zu und überprüfen die Zuweisung:

[listing]
----
cluster::> volume modify -v1server vs1 -qtree-path /vol/vol1/qt1 -policy nfs_policy

cluster::>volume qtree show -volume vol1 -fields export-policy
vserver volume qtree export-policy
------- ------ ----- -------------
vs1     data1  qt01  nfs_policy
----
