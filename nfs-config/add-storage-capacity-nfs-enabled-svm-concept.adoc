= Informationen zum Hinzufügen von Speicherkapazität zu einer ONTAP NFS-fähigen SVM
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um die Speicherkapazität einer NFS-fähigen SVM zu erweitern, muss ein Volume oder ein Qtree als Speichercontainer erstellt und eine Exportregel für diesen Container erstellt oder geändert werden. Anschließend lässt sich der NFS-Client-Zugriff vom Cluster aus überprüfen und der Zugriff von Clientsystemen testen.

.Bevor Sie beginnen
* NFS muss auf der SVM vollständig eingerichtet sein.
* Die Standardexportrichtlinie des SVM-Root-Volumes muss eine Regel enthalten, die den Zugriff für alle Clients erlaubt.
* Alle Aktualisierungen Ihrer Namensdienstkonfiguration müssen vollständig abgeschlossen sein.
* Jegliche Ergänzungen oder Änderungen an einer Kerberos-Konfiguration müssen vollständig sein.
