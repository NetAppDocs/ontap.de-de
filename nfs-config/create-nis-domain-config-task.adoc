= NIS-Domänenkonfigurationen für ONTAP NFS SVMs erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn in Ihrer Umgebung ein Network Information Service (NIS) für Namensdienste verwendet wird, muss eine NIS-Domänenkonfiguration für die SVM mithilfe des  `vserver services name-service nis-domain create` Befehls erstellt werden.

.Bevor Sie beginnen
Alle konfigurierten NIS-Server müssen verfügbar und erreichbar sein, bevor die NIS-Domäne auf der SVM konfiguriert wird.

Wenn Sie NIS für Verzeichnissuchen verwenden möchten, dürfen die Zuordnungen auf Ihren NIS-Servern maximal 1.024 Zeichen pro Eintrag enthalten. Ein NIS-Server, der diese Beschränkung nicht einhält, sollte nicht angegeben werden. Andernfalls kann der Clientzugriff, der von NIS-Einträgen abhängt, fehlschlagen.

.Über diese Aufgabe
Wenn Ihre NIS-Datenbank eine  `netgroup.byhost` Zuordnung enthält, kann ONTAP diese für schnellere Suchvorgänge nutzen. Die  `netgroup.byhost`- und  `netgroup`-Zuordnungen im Verzeichnis müssen jederzeit synchron gehalten werden, um Zugriffsprobleme für Clients zu vermeiden. Ab ONTAP 9.7 können NIS- `netgroup.byhost`-Einträge mithilfe der  `vserver services name-service nis-domain netgroup-database` Befehle zwischengespeichert werden.

Die Verwendung von NIS zur Hostnamensauflösung wird nicht unterstützt.

.Schritte
. Erstellen einer NIS-Domänenkonfiguration:
+
`vserver services name-service nis-domain create -vserver vs1 -domain <domain_name> -nis-servers <IP_addresses>`

+
Sie können bis zu 10 NIS Server angeben.

+
[NOTE]
====
Das `-nis-servers` Feld ersetzt das `-servers` Feld. Das `-nis-servers` Feld kann verwendet werden, um entweder einen Hostnamen oder eine IP-Adresse für den NIS-Server anzugeben.

====
. Überprüfen, ob die Domain erstellt wurde:
+
`vserver services name-service nis-domain show`



.Beispiel
Der folgende Befehl erstellt eine NIS-Domänenkonfiguration für eine NIS-Domäne namens `nisdomain` auf der SVM mit dem Namen `vs1` mit einem NIS-Server unter der IP-Adresse `192.0.2.180`:

[listing]
----
vs1::> vserver services name-service nis-domain create -vserver vs1 -domain nisdomain -nis-servers 192.0.2.180
----
