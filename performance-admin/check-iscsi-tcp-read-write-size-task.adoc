= ONTAP iSCSI TCP lesen/schreiben-Größe prüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei iSCSI kann die TCP lesen/schreiben-Größe überprüft werden, um festzustellen, ob die Größeneinstellung ein Leistungsproblem verursacht. Ist die Größe die Ursache eines Problems, kann sie korrigiert werden.

.Bevor Sie beginnen
Für diese Aufgabe sind Befehle auf dem erweiterten Berechtigungslevel erforderlich.

.Schritte
. Wechsel zum erweiterten Berechtigungslevel:
+
`set -privilege advanced`

. Die Einstellung für die TCP-Fenstergröße prüfen:
+
`vserver iscsi show -vserv,er _vserver_name_ -instance`

. Die Einstellung für die TCP-Fenstergröße ändern:
+
`vserver iscsi modify -vserver _vserver_name_ -tcp-window-size _integer_`

. Zurück zu als Administrator angemeldet:
+
`set -privilege admin`



.Beispiel
Das folgende Beispiel ändert die TCP-Fenstergröße von `SVM1`auf 131.400 Bytes:

[listing]
----
cluster1::*> vserver iscsi modify -vserver vs1 -tcp-window-size 131400
----
