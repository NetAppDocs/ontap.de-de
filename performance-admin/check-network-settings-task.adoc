= Die ONTAP MTU Netzwerkeinstellung auf dem Speichersystem prüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Netzwerkeinstellungen des Speichersystems ändern, falls diese von denen des Clients oder anderer Netzwerkendpunkte abweichen. Während die MTU-Einstellung des Management-Netzwerks auf 1500 festgelegt ist, sollte die MTU-Größe des Datennetzwerks 9000 betragen.



[[about-this-task]]
== Über diese Aufgabe

Alle Ports innerhalb einer Broadcast-Domäne haben dieselbe MTU-Größe, mit Ausnahme des Ports e0M, der den Management-Datenverkehr verarbeitet. Ist der Port Teil einer Broadcast-Domäne, kann mit dem  `broadcast-domain modify` Befehl die MTU für alle Ports innerhalb der geänderten Broadcast-Domäne angepasst werden.

Beachten Sie, dass zwischengeschaltete Netzwerkgeräte wie Netzwerkkarten und Datenswitches auf höhere MTU-Größen als Netzwerkendpunkte eingestellt werden können. Weitere Informationen finden sich unter link:../performance-admin/check-network-settings-data-switches-task.html["Die Netzwerkeinstellungen der Datenswitches überprüfen."].

.Schritte
. Die MTU-Port-Einstellung des Speichersystems überprüfen:
+
`network port show -instance`

+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

. Die MTU der Broadcast-Domäne, die von den Ports verwendet wird, ändern:
+
`network port broadcast-domain modify -ipspace _ipspace_ -broadcast-domain _broadcast_domain_ -mtu _new _mtu_`



.Beispiel
Das folgende Beispiel ändert die MTU-Port-Einstellung auf 9000:

[listing]
----
network port broadcast-domain modify -ipspace Cluster -broadcast-domain Cluster -mtu 9000
----
