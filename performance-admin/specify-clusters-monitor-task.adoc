= Die ONTAP Cluster, die im Active IQ Unified Manager überwacht werden sollen, angeben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Cluster muss zu einem Active IQ Unified Manager Server hinzugefügt werden, damit der Cluster überwacht, der Clustererkennungsstatus angezeigt und die Leistung überwacht werden kann.

.Bevor Sie beginnen
* Sie benötigen folgende Informationen:
+
** Hostname oder Cluster-Management-IP-Adresse
+
Der Hostname ist der vollqualifizierte Domainname (FQDN) oder der Kurzname, den Unified Manager für die Verbindung zum Cluster verwendet. Dieser Hostname muss zur Cluster-Management-IP-Adresse aufgelöst werden.

+
Die Clusterverwaltungs-IP-Adresse muss die Clusterverwaltungs-LIF der administrativen Storage Virtual Machine (SVM) sein. Wenn eine Knotenverwaltungs-LIF verwendet wird, schlägt der Vorgang fehl.

** ONTAP Administrator-Benutzername und Passwort
** Protokolltyp (HTTP oder HTTPS), der auf dem Cluster konfiguriert werden kann, und die Portnummer des Clusters


* Die Rolle Application Administrator oder Storage Administrator ist erforderlich.
* Der ONTAP Administrator muss die Administratorrollen ONTAPI und SSH besitzen.
* Der Unified Manager FQDN muss ONTAP anpingen können.
+
Dies lässt sich mit dem ONTAP Befehl  `ping -node _node_name_ -destination _Unified_Manager_FQDN_` überprüfen.



.Über diese Aufgabe
Für eine MetroCluster Konfiguration müssen sowohl die lokalen als auch die Remote-Cluster hinzugefügt werden, und die Cluster müssen korrekt konfiguriert sein.

.Schritte
. *Konfiguration* > *Cluster Data Sources* auswählen.
. Auf der Seite „Cluster“ auf *Hinzufügen* klicken.
. Im Dialogfeld *Cluster hinzufügen* sind die erforderlichen Werte anzugeben, wie der Hostname oder die IP-Adresse (IPv4 oder IPv6) des Clusters, der Benutzername, das Kennwort, das Protokoll für die Kommunikation und die Portnummer.
+
Standardmäßig ist das HTTPS-Protokoll ausgewählt.

+
Sie können die Cluster-Management-IP-Adresse von IPv6 auf IPv4 oder von IPv4 auf IPv6 ändern. Die neue IP-Adresse wird nach Abschluss des nächsten Überwachungszyklus im Cluster-Grid und auf der Cluster-Konfigurationsseite angezeigt.

. Klicken Sie auf *Add*.
. Wenn HTTPS ausgewählt ist, sind die folgenden Schritte auszuführen:
+
.. Im Dialogfeld *Host autorisieren* auf *Zertifikat anzeigen* klicken, um die Zertifikatsinformationen zum Cluster anzuzeigen.
.. Klicken Sie auf *Ja*.
+
Unified Manager prüft das Zertifikat nur beim erstmaligen Hinzufügen des Clusters, prüft es jedoch nicht bei jedem API-Aufruf an ONTAP.

+
Wenn das Zertifikat abgelaufen ist, kann der Cluster nicht hinzugefügt werden. Das SSL-Zertifikat muss erneuert werden, danach kann der Cluster hinzugefügt werden.



. *Optional*: Clustererkennungsstatus anzeigen:
+
.. Der Clustererkennungsstatus ist auf der Seite *Cluster Setup* ersichtlich.


+
Der Cluster wird nach dem standardmäßigen Überwachungsintervall von etwa 15 Minuten zur Unified Manager Datenbank hinzugefügt.
