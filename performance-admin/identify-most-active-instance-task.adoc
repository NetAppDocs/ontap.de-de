= Clients oder Dateien mit hohem Datenverkehr in ONTAP identifizieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit der ONTAP Active Objects Technologie lassen sich Clients oder Dateien identifizieren, die für einen unverhältnismäßig hohen Cluster-Datenverkehr verantwortlich sind. Nachdem diese „Top“-Clients oder Dateien identifiziert wurden, besteht die Möglichkeit, die Cluster-Workloads neu zu verteilen oder weitere Schritte zur Problemlösung zu ergreifen.

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritte
. Die wichtigsten Clients, die auf den Cluster zugreifen:
+
`statistics top client show -node _node_name_ -sort-key _sort_column_ -interval _seconds_between_updates_ -iterations _iterations_ -max _number_of_instances_`

+
Der folgende Befehl zeigt die Top-Clients an, die auf  `cluster1` zugreifen:

+
[listing]
----
cluster1::> statistics top client show

cluster1 : 3/23/2016 17:59:10

Weitere Informationen zu  `statistics top client show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/statistics-top-client-show.html["ONTAP-Befehlsreferenz"^].

. Die am häufigsten auf dem Cluster aufgerufenen Dateien:
+
`statistics top file show -node _node_name_ -sort-key _sort_column_ -interval _seconds_between_updates_ -iterations _iterations_ -max _number_of_instances_`

+
