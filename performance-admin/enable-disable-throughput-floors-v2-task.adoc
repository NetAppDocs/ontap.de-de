= ONTAP Durchsatzgrenzen v2 aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Durchsatzbegrenzungen v2 auf AFF aktivieren oder deaktivieren. Standardmäßig ist sie aktiviert. Bei aktivierten Durchsatzbegrenzungen v2 können diese bei starker Auslastung der Controller eingehalten werden, was jedoch zu höheren Latenzzeiten bei anderen Workloads führen kann. Durchsatzbegrenzungen v2 gelten sowohl für QoS als auch für Adaptive QoS.

.Schritte
. Wechsel zum erweiterten Berechtigungslevel:
+
`set -privilege advanced`

. Einer der folgenden Befehle kann verwendet werden:
+
|===
| Wenn Sie möchten... | Dazu wird folgender Befehl verwendet: 


 a| 
Floors v2 deaktivieren
 a| 
`qos settings throughput-floors-v2 -enable false`



 a| 
Floors v2 aktivieren
 a| 
`qos settings throughput-floors-v2 -enable true`

|===
+
[NOTE]
====
Um Durchsatzbegrenzungen v2 in einem MetroCluster Cluster zu deaktivieren, ist der folgende Befehl auszuführen:

`qos settings throughput-floors-v2 -enable false`

Befehl sowohl auf dem Quell- als auch auf dem Ziel-Cluster.

====
+
[listing]
----
cluster1::*> qos settings throughput-floors-v2 -enable false
----
+
Weitere Informationen zu  `qos settings throughput-floors-v2` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/qos-settings-throughput-floors-v2.html["ONTAP-Befehlsreferenz"^].
