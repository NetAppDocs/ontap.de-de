= Durchsatz mit ONTAP adaptiven QoS-Richtliniengruppen festlegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mithilfe einer adaptiven QoS-Richtliniengruppe kann eine Durchsatzober- oder -untergrenze automatisch an die Volume-Größe angepasst werden, sodass das Verhältnis von IOPS zu TB/GBs erhalten bleibt, wenn sich die Größe des Volumes ändert. Das stellt einen entscheidenden Vorteil dar, wenn Hunderte oder Tausende von Workloads in einer großen Umgebung verwaltet werden.

.Bevor Sie beginnen
* Um eine Richtliniengruppe zu erstellen, ist die Rolle als Cluster-Administrator erforderlich.
* Das Speicherobjekt, das Sie der Richtliniengruppe hinzufügen möchten, muss Mitglied einer nicht-adaptiven Richtliniengruppe sein
+
Ein Speicherobjekt kann Mitglied einer adaptiven Richtliniengruppe oder einer nicht-adaptiven Richtliniengruppe sein, aber nicht beidem.

* Die SVM des Speicherobjekts und die Richtlinie müssen gleich sein.
* Das Speicherobjekt muss online sein.


.Über diese Aufgabe
* Adaptive QoS-Richtliniengruppen sind immer nicht gemeinsam genutzt: Die definierte Durchsatzobergrenze oder -untergrenze gilt für jede einzelne Mitglieds-Workload.
* Adaptive und nicht-adaptive QoS-Richtliniengruppen werden standardmäßig automatisch aktiviert, um den Durchsatz einer Arbeitslast für eine Dauer von 1 Sekunde um 50 % zu erhöhen, um kurzfristigen Anforderungen nach höherer Leistung zu entsprechen. Ab ONTAP 9.19.1 können Sie diese Standardeinstellungen ändern. In ONTAP 9.18.1 und früher können diese Einstellungen nicht geändert werden.
+
Nutzen Sie die folgenden Optionen, um die Standardeinstellungen zu ändern:

+
[cols="3"]
|===
| Option | Beschreibung | Standardeinstellung 


 a| 
`burst-percent`
| Steigerung des Durchsatzes in Prozent | 50% 


 a| 
`burst duration`
| Dauer der Durchsatzsteigerung in Sekunden | 1 Sekunde 


 a| 
`burst-iops`
| Maximale Anzahl von IOPs oberhalb der Durchsatzgrenze für die angegebene Burst-Dauer | 0 
|===
* Das Verhältnis von Durchsatzgrenzen zur Speicherobjektgröße wird durch das Zusammenspiel der folgenden Felder bestimmt:
+
** `expected-iops` ist die minimale erwartete IOPS-Zahl pro zugewiesenem TB/GB.
** `peak-iops` ist die maximal mögliche IOPS-Zahl pro zugewiesenem oder verwendetem TB/GB.
** `expected-iops-allocation`Gibt an, ob für expected-iops der zugewiesene Speicherplatz (Standard) oder der verwendete Speicherplatz verwendet wird.
** `peak-iops-allocation`Gibt an, ob der zugewiesene Speicherplatz oder der belegte Speicherplatz (Standard) für  `peak-iops` verwendet wird.
**  `absolute-min-iops` ist die absolute Mindestanzahl an IOPS. Dieses Feld kann mit sehr kleinen Speicherobjekten verwendet werden. Es überschreibt sowohl  `peak-iops` als auch  `expected-iops`, wenn  `absolute-min-iops` größer als der berechnete  `expected-iops` ist.
+
Wenn beispielsweise  `expected-iops` auf 1.000 IOPS/TB gesetzt wird und die Volume-Größe weniger als 1 GB beträgt, ist der berechnete  `expected-iops` ein Bruchteil eines IOP. Der berechnete  `peak-iops` ist ein noch kleinerer Bruchteil. Dies lässt sich vermeiden, wenn  `absolute-min-iops` auf einen realistischen Wert gesetzt wird.

** `block-size` Gibt die Blockgröße der Anwendungs-I/O an. Der Standardwert ist 32K. Gültige Werte sind 8K, 16K, 32K, 64K, ANY. ANY bedeutet, dass die Blockgröße nicht erzwungen wird.




.Schritte
. Erstellen einer adaptiven QoS-Richtliniengruppe:
+
[source, cli]
----
qos adaptive-policy-group create -policy group <policy_group> -vserver <SVM> -expected-iops <number_of_iops>/TB|GB -peak-iops <number_of_iops>/TB|GB -expected-iops-allocation-space|used-space -peak-iops-allocation allocated-space|used-space -absolute-min-iops <number_of_iops> -block-size 8K|16K|32K|64K|ANY
----
+
Weitere Informationen zu  `qos adaptive-policy-group create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/qos-adaptive-policy-group-create.html["ONTAP-Befehlsreferenz"^].

+
[NOTE]
====
`-expected-iops-allocation` und `-block-size` sind in ONTAP 9.5 und höher verfügbar. Diese Optionen werden in ONTAP 9.4 und früher nicht unterstützt.

====
+
Der folgende Befehl erstellt eine adaptive QoS-Richtliniengruppe `adpg-app1` mit `-expected-iops` auf 300 IOPS/TB gesetzt, `-peak-iops` auf 1.000 IOPS/TB gesetzt, `-peak-iops-allocation` auf `used-space` gesetzt und `-absolute-min-iops` auf 50 IOPS gesetzt:

+
[listing]
----
cluster1::> qos adaptive-policy-group create -policy group adpg-app1 -vserver vs2 -expected-iops 300iops/tb -peak-iops 1000iops/TB -peak-iops-allocation used-space -absolute-min-iops 50iops
----
. Eine adaptive QoS-Richtliniengruppe auf ein Volume anwenden:
+
[source, cli]
----
volume create -vserver SVM -volume <volume_name> -aggregate <aggregate> -size <number_of_ TB|GB> -qos-adaptive-policy-group<_policy_group>
----
+
Weitere Informationen zu  `volume create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-create.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl wendet die adaptive QoS-Richtliniengruppe `adpg-app1` auf das Volume `app1` an:

+
[listing]
----
cluster1::> volume create -vserver vs1 -volume app1 -aggregate aggr1 -size 2TB -qos-adaptive-policy-group adpg-app1
----
+
Die folgenden Befehle wenden die standardmäßige adaptive QoS-Richtliniengruppe `extreme` auf das neue Volume `app4` und auf das vorhandene Volume `app5` an. Die für die Richtliniengruppe definierte Durchsatzobergrenze gilt für Volumes `app4` und `app5` jeweils einzeln:

+
[listing]
----
cluster1::> volume create -vserver vs4 -volume app4 -aggregate aggr4 -size 2TB -qos-adaptive-policy-group extreme
----
+
[listing]
----
cluster1::> volume modify -vserver vs5 -volume app5 -qos-adaptive-policy-group extreme
----
