= ONTAP NFS TCP maximale Übertragungsgröße prüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für NFS kann überprüft werden, ob die maximale TCP-Übertragungsgröße für Lese- und Schreibvorgänge ein Leistungsproblem verursacht. Wenn angenommen wird, dass die Größe die Leistung verlangsamt, kann sie erhöht werden.

.Bevor Sie beginnen
* Zum Ausführen dieser Aufgabe sind Cluster-Administratorrechte erforderlich.
* Für diese Aufgabe müssen Sie Befehle auf dem erweiterten Berechtigungsniveau verwenden.


.Schritte
. Wechsel zur erweiterten Berechtigungsstufe:
+
`set -privilege advanced`

. Die maximale TCP-Übertragungsgröße prüfen:
+
`vserver nfs show -vserver _vserver_name_ -instance`

. Wenn die maximale TCP-Übertragungsgröße zu klein ist, sollte die Größe erhöht werden:
+
`vserver nfs modify -vserver _vserver_name_ -tcp-max-xfer-size _integer_`

. Zurückkehren zur als Administrator angemeldeten Ebene:
+
`set -privilege admin`



.Beispiel
Das folgende Beispiel ändert die maximale TCP-Übertragungsgröße von `SVM1` auf 1048576:

[listing]
----
cluster1::*> vserver nfs modify -vserver SVM1 -tcp-max-xfer-size 1048576
----
