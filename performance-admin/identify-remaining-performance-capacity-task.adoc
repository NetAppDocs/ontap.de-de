= Verbleibende Leistungskapazität in ONTAP identifizieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Leistungskapazität, auch „Spielraum“ genannt, misst, wie viel Arbeit auf einem Knoten oder einem Aggregat ausgeführt werden kann, bevor die Leistung der Workloads auf der Ressource durch Latenz beeinträchtigt wird. Die Kenntnis der verfügbaren Leistungskapazität im Cluster unterstützt die Bereitstellung und den Ausgleich von Workloads.

.Bevor Sie beginnen
Für diese Aufgabe sind Befehle auf dem erweiterten Berechtigungslevel erforderlich.

.Über diese Aufgabe
Sie können die folgenden Werte für die  `-object` Option verwenden, um Headroom Statistiken zu erfassen und anzuzeigen:

* Für CPUs, `resource_headroom_cpu`.
* Für Aggregate  `resource_headroom_aggr`.


Diese Aufgabe kann auch mit System Manager und Active IQ Unified Manager abgeschlossen werden.

.Schritte
. Wechsel zum erweiterten Berechtigungslevel:
+
`set -privilege advanced`

. Echtzeit-Headroom-Statistiken erfassen:
+
`statistics start –object resource_headroom_cpu|aggr`

+
Weitere Informationen zu  `statistics start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/statistics-start.html["ONTAP-Befehlsreferenz"^].

. Anzeige von Echtzeit-Headroom-Statistikinformationen:
+
`statistics show -object resource_headroom_cpu|aggr`

. Zurück zu als Administrator angemeldet:
+
`set -privilege admin`



.Beispiel
Das folgende Beispiel zeigt die durchschnittlichen stündlichen Headroom Statistiken für Clusterknoten.

Die verfügbare Leistungskapazität eines Knotens lässt sich berechnen, indem der  `current_utilization`-Zähler vom  `optimal_point_utilization`-Zähler subtrahiert wird. In diesem Beispiel beträgt die Auslastungskapazität für  `CPU_sti2520-213` -14 % (72 % - 86 %), was darauf hindeutet, dass die CPU in der vergangenen Stunde im Durchschnitt überbeansprucht wurde.
