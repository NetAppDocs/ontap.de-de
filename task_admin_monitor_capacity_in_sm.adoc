= Cluster-, Tier- und SVM-Kapazität im ONTAP System Manager überwachen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Mit System Manager kann überwacht werden, wie viel Speicherkapazität genutzt wurde und wie viel noch für einen Cluster, eine lokale Tier oder eine Storage-VM verfügbar ist.

Mit jeder Version von ONTAP stellt System Manager umfassendere Informationen zur Kapazitätsüberwachung bereit:

* Ab ONTAP 9.13.1 kann eine maximale Kapazitätsgrenze für eine Storage-VM aktiviert und ein Schwellenwert festgelegt werden, der Warnmeldungen auslöst, wenn der genutzte Speicher einen bestimmten Prozentsatz der maximalen Kapazität erreicht.
* Ab ONTAP 9.12.1 zeigt System Manager die für eine lokale Ebene zugesicherte Kapazität an.
* Ab ONTAP 9.10.1 ermöglicht System Manager die Anzeige historischer Daten zur Clusterkapazität sowie Prognosen dazu, wie viel Kapazität in Zukunft genutzt oder verfügbar sein wird. Außerdem kann die Kapazität lokaler Tiers und Volumes überwacht werden.



NOTE: Die Anzeige der genutzten Kapazität variiert je nach ONTAP Version. Weitere Informationen in link:./concepts/capacity-measurements-in-sm-concept.html["Kapazitätsmessungen im System Manager"].



[[view-the-capacity-of-a-cluster]]
== Kapazität eines Clusters anzeigen

Kapazitätsmessungen für einen Cluster sind im System Manager auf dem Dashboard einsehbar.

.Bevor Sie beginnen
Um Daten zur Kapazität in der Cloud anzuzeigen, ist ein Konto bei Digital Advisor erforderlich und eine Verbindung muss bestehen.

.Schritte
. Im System Manager auf *Dashboard* klicken.
. Im Abschnitt *Kapazität* sind die folgenden Informationen verfügbar:
+
--
** Gesamtkapazität des genutzten Speicherplatzes im Cluster
** Gesamte verfügbare Kapazität des Clusters
** Prozentsätze der genutzten und verfügbaren Kapazität.
** Verhältnis der Datenreduktion.
** In der Cloud genutzte Kapazität.
** Historie der Kapazitätsauslastung.
** Prognose der Kapazitätsauslastung


--
+

NOTE: Im System Manager berücksichtigen die Kapazitätsdarstellungen nicht die Kapazitäten der Root-Speicherebene (Aggregate).

. Klicken Sie auf das Diagramm, um weitere Details zur Kapazität des Clusters anzuzeigen.
+
Die Kapazitätsmessungen werden in zwei Balkendiagrammen dargestellt:

+
--
** Das obere Diagramm zeigt die physische Kapazität: die Größe des genutzten, reservierten und verfügbaren physischen Speicherplatzes.
** Das untere Diagramm zeigt die logische Kapazität: die Größe der Clientdaten, Snapshots und Klone sowie den gesamten logisch belegten Speicherplatz.


--
+
Unterhalb der Balkendiagramme befinden sich Kennzahlen zur Datenreduktion:

+
--
** Datenreduktionsverhältnis nur für die Kundendaten (Snapshots und Klone sind nicht enthalten).
** Gesamtdatenreduktionsverhältnis.


--


Weitere Informationen finden sich unter link:./concepts/capacity-measurements-in-sm-concept.html["Kapazitätsmessungen im System Manager"].



[[view-the-capacity-of-a-local-tier]]
== Kapazität einer lokalen Ebene anzeigen

Sie können Details zur Kapazität lokaler Tiers einsehen. Ab ONTAP 9.12.1 enthält die Ansicht *Kapazität* auch die zugesagte Kapazität für einen lokalen Tier, wodurch sich feststellen lässt, ob Kapazität zum lokalen Tier hinzugefügt werden muss, um die zugesagte Kapazität aufzunehmen und einen Mangel an freiem Speicherplatz zu vermeiden.

.Schritte
. *Speicher > Ebenen* auswählen.
. Wählen Sie den Namen der lokalen Tier aus.
. Auf der Seite *Übersicht* wird im Abschnitt *Kapazität* die Kapazität in einem Balkendiagramm mit drei Messwerten dargestellt:
+
** Verwendete und reservierte Kapazität
** Verfügbare Kapazität
** Zugesicherte Kapazität (beginnend mit ONTAP 9.12.1)


. Durch Klicken auf das Diagramm werden Details zur Kapazität der lokalen Ebene angezeigt.
+
Die Kapazitätsmessungen werden in zwei Balkendiagrammen dargestellt:

+
--
** Das obere Balkendiagramm zeigt die physische Kapazität: die Größe des genutzten, reservierten und verfügbaren physischen Speicherplatzes.
** Das untere Balkendiagramm zeigt die logische Kapazität: die Größe der Clientdaten, Snapshots und Klone sowie die Gesamtsumme des logisch belegten Speicherplatzes.


--
+
Unterhalb der Balkendiagramme befinden sich die Messverhältnisse zur Datenreduktion:

+
--
** Datenreduktionsverhältnis nur für die Kundendaten (Snapshots und Klone sind nicht enthalten).
** Gesamtdatenreduktionsverhältnis.


--


Weitere Informationen finden sich unter link:./concepts/capacity-measurements-in-sm-concept.html["Kapazitätsmessungen im System Manager"].

.Optionale Aktionen
* Wenn die zugesicherte Kapazität größer ist als die Kapazität der lokalen Ebene, kann es sinnvoll sein, die Kapazität der lokalen Ebene zu erhöhen, bevor der freie Speicherplatz erschöpft ist. Siehe link:./disks-aggregates/add-disks-local-tier-aggr-task.html["Kapazität zu einer lokalen Ebene hinzufügen (Festplatten zu einem Aggregat hinzufügen)"].
* Sie können auch den Speicherplatz anzeigen, den bestimmte Volumes in der lokalen Ebene verwenden, indem die Registerkarte *Volumes* ausgewählt wird.




[[view-the-capacity-of-the-volumes-in-a-storage-vm]]
== Kapazität der Volumes in einer Storage-VM anzeigen

Sie können einsehen, wie viel Speicherplatz von den Volumes einer Storage-VM belegt ist und wie viel Kapazität noch verfügbar ist. Die Gesamtmenge des belegten und verfügbaren Speicherplatzes wird als „Kapazität über alle Volumes“ bezeichnet.

.Schritte
. Wählen Sie *Storage* > *Storage VMs*.
. Klicken Sie auf den Namen der Storage-VM.
. Zum Abschnitt *Kapazität* scrollen, der ein Balkendiagramm mit den folgenden Messwerten zeigt:
+
--
** *Physisch genutzt*: Summe des physisch genutzten Speicherplatzes über alle Volumes in dieser Storage-VM.
** *Verfügbar*: Summe der verfügbaren Kapazität aller Volumes in dieser Storage-VM.
** *Logisch verwendet*: Summe des logisch verwendeten Speicherplatzes über alle Volumes in dieser Storage-VM.


--


Weitere Details zu den Messungen sind unter link:./concepts/capacity-measurements-in-sm-concept.html["Kapazitätsmessungen im System Manager"] zu finden.



[[view-the-maximum-capacity-limit-of-a-storage-vm]]
[[view-max-cap-limit-svm]]
== Maximale Kapazitätsgrenze einer Storage-VM anzeigen

Ab ONTAP 9.13.1 ist die maximale Kapazitätsgrenze einer Storage-VM sichtbar.

.Bevor Sie beginnen
Sie müssen link:manage-max-cap-limit-svm-in-sm-task.html["Das maximale Kapazitätslimit einer Storage-VM aktivieren"] ausführen, bevor Sie es ansehen können.

.Schritte
. Wählen Sie *Storage* > *Storage VMs*.
+
Die maximalen Kapazitätsmesswerte lassen sich auf zwei Arten anzeigen:

+
--
** In der Zeile für die Storage-VM zeigt die Spalte *Maximale Kapazität* ein Balkendiagramm, das die belegte Kapazität, die verfügbare Kapazität und die maximale Kapazität darstellt.
** Klicken Sie auf den Namen der Storage-VM. Auf der Registerkarte *Übersicht* kann in der linken Spalte die maximale Kapazität, die zugewiesene Kapazität und der Kapazitätsalarmschwellenwert eingesehen werden.


--
