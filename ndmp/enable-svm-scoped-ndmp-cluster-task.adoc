= SVM-bezogenes NDMP auf dem ONTAP Cluster aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn der DMA die Cluster Aware Backup (CAB) Erweiterung unterstützt, ist das Sichern aller Volumes, die auf verschiedenen Knoten in einem Cluster gehostet werden, möglich, indem SVM-scoped NDMP aktiviert wird, der NDMP-Service auf dem Cluster (admin SVM) aktiviert wird und LIFs für die Daten- und Steuerverbindung konfiguriert werden.

.Bevor Sie beginnen
Die CAB-Erweiterung muss vom DMA unterstützt werden.

.Über diese Aufgabe
Durch das Deaktivieren des knotenbezogenen NDMP-Modus wird der SVM-bezogene NDMP-Modus auf dem Cluster aktiviert.

.Schritte
. SVM-bezogener NDMP-Modus:
+
[source, cli]
----
cluster1::> system services ndmp node-scope-mode off
----
+
Der SVM-basierte NDMP-Modus ist aktiviert.

. NDMP-Service auf der Administrator-SVM aktivieren:
+
[source, cli]
----
cluster1::> vserver services ndmp on -vserver cluster1
----
+
Der Authentifizierungstyp ist standardmäßig auf  `challenge` gesetzt und die Authentifizierung im Klartext ist deaktiviert.

+
[NOTE]
====
Für eine sichere Kommunikation sollte die Authentifizierung im Klartext deaktiviert bleiben.

====
. Es sollte überprüft werden, ob der NDMP-Service aktiviert ist:
+
[source, cli]
----
cluster1::> vserver services ndmp show
----
+
[listing]
----
Vserver       Enabled   Authentication type
------------- --------- -------------------
cluster1      true      challenge
vs1           false     challenge
----
