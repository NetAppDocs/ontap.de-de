= ONTAP NDMP-Konfigurationen vorbereiten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Tape-Backup-Zugriff über das Network Data Management Protocol (NDMP) konfiguriert wird, muss überprüft werden, ob die geplante Konfiguration unterstützt wird, ob die Tape-Laufwerke auf jedem Knoten als qualifizierte Laufwerke aufgeführt sind, ob alle Knoten über Intercluster-LIFs verfügen und ob die Backup-Applikation die Cluster Aware Backup (CAB) Erweiterung unterstützt.

.Schritte
. In der Kompatibilitätsmatrix Ihres Anbieters von Applikationen finden sich Informationen zur ONTAP-Unterstützung (NetApp qualifiziert keine Backup-Anwendungen von Drittanbietern mit ONTAP oder NDMP).
+
Es sollte überprüft werden, ob die folgenden NetApp-Komponenten kompatibel sind:

+
--
** Die auf dem Cluster ausgeführte Version von ONTAP 9.
** Anbieter und Version der Backup-Anwendung: zum Beispiel Veritas NetBackup 8.2 oder CommVault.
** Die Bandgerätedetails, wie z. B. Hersteller, Modell und Schnittstelle der Bandlaufwerke: zum Beispiel IBM Ultrium 8 oder HPe StoreEver Ultrium 30750 LTO-8.
** Die Plattformen der Knoten im Cluster: zum Beispiel FAS8700 oder A400.


--
+

NOTE: Kompatibilitätsmatrizen für ältere ONTAP-Versionen für Backup-Anwendungen sind in der  https://mysupport.netapp.com/matrix["NetApp Interoperabilitätsmatrix-Tool"^] zu finden.

. Es sollte sichergestellt sein, dass Ihre Tape-Laufwerke als qualifizierte Laufwerke in der integrierten Tape-Konfigurationsdatei jedes Node aufgeführt sind:
+
.. In der Befehlszeilenschnittstelle lässt sich die integrierte Tape-Konfigurationsdatei mit dem  `storage tape show-supported-status` Befehl anzeigen.
+
....
cluster1::> storage tape show-supported-status

Node: cluster1-1
                                Is
Tape Drives                     Supported  Support Status
------------------------------  ---------  -------------------------------
Certance Ultrium 2              true       Dynamically Qualified
Certance Ultrium 3              true       Dynamically Qualified
Digital DLT2000                 true       Qualified
....
.. Vergleichen Sie Ihre Tape-Laufwerke mit der Liste der qualifizierten Laufwerke in der Ausgabe.
+
[NOTE]
====
Die Bezeichnungen der Tape-Geräte in der Ausgabe können geringfügig von den Bezeichnungen auf dem Geräteetikett oder in der Interoperabilitätsmatrix abweichen. Beispielsweise kann Digital DLT2000 auch als DLT2k bezeichnet werden. Diese geringfügigen Namensunterschiede können ignoriert werden.

====
.. Wird ein Gerät in der Ausgabe nicht als qualifiziert aufgeführt, obwohl es gemäß der Interoperabilitätsmatrix qualifiziert ist, kann eine aktualisierte Konfigurationsdatei für das Gerät entsprechend den Anweisungen auf der NetApp Support Site heruntergeladen und installiert werden.
+
http://mysupport.netapp.com/NOW/download/tools/tape_config["NetApp Downloads: Konfigurationsdateien für Tape-Geräte"^]

+
Ein qualifiziertes Gerät ist möglicherweise nicht in der integrierten Tape-Konfigurationsdatei aufgeführt, wenn das Tape-Gerät erst nach der Auslieferung des Knotens qualifiziert wurde.



. Es sollte sichergestellt werden, dass jeder Knoten im Cluster über eine Intercluster-LIF verfügt:
+
.. Zeigen Sie die Intercluster-LIFs auf den Knoten mit dem  `network interface show -service-policy default-intercluster`-Befehl an.
+
[listing]
----
cluster1::> network interface show -service-policy default-intercluster

            Logical    Status     Network            Current       Current Is
Vserver     Interface  Admin/Oper Address/Mask       Node          Port    Home
----------- ---------- ---------- ------------------ ------------- ------- ----
cluster1    IC1        up/up      192.0.2.65/24      cluster1-1    e0a     true
----
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

.. Falls auf keinem Knoten ein Intercluster-LIF existiert, kann ein Intercluster-LIF mit dem  `network interface create` Befehl erstellt werden.
+
[listing]
----
cluster1::> network interface create -vserver cluster1 -lif IC2 -service-policy default-intercluster
-home-node cluster1-2 -home-port e0b -address 192.0.2.68 -netmask 255.255.255.0
-status-admin up -failover-policy local-only -firewall-policy intercluster

cluster1::> network interface show -service-policy default-intercluster

            Logical    Status     Network            Current       Current Is
Vserver     Interface  Admin/Oper Address/Mask       Node          Port    Home
----------- ---------- ---------- ------------------ ------------- ------- ----
cluster1    IC1        up/up      192.0.2.65/24      cluster1-1    e0a     true
cluster1    IC2        up/up      192.0.2.68/24      cluster1-2    e0b     true
----
+
Weitere Informationen zu  `network interface create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-create.html["ONTAP-Befehlsreferenz"^].

+
link:../networking/networking_reference.html["Netzwerkmanagement"]



. Anhand der mit der Backup-Anwendung gelieferten Dokumentation lässt sich feststellen, ob die Backup-Anwendung Cluster Aware Backup (CAB) unterstützt.
+
Die Unterstützung von CAB ist ein Schlüsselfaktor bei der Bestimmung der Art der Datensicherung, die durchgeführt werden kann.
