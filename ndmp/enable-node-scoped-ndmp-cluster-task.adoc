= Knotenbezogenes NDMP auf dem ONTAP Cluster aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Volumes sichern, die auf einem einzelnen Knoten gehostet werden, indem Sie knotenbezogenes NDMP aktivieren, den NDMP-Service aktivieren und eine LIF für die Daten- und Steuerverbindung konfigurieren. Dies ist für alle Knoten des Clusters möglich.


NOTE: Node-scoped NDMP ist in ONTAP 9 veraltet.

.Über diese Aufgabe
Bei Verwendung von NDMP im Node-Scope-Modus muss die Authentifizierung pro Node konfiguriert werden. Weitere Informationen finden sich unter link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Protection_and_Security/NDMP/How_to_configure_NDMP_authentication_in_the_%E2%80%98node-scope%E2%80%99_mode["Der Knowledge Base Artikel „How to configure NDMP authentication in the ‘node-scope’ mode“"^].

.Schritte
. Node-spezifischen NDMP-Modus aktivieren:
+
[source, cli]
----
cluster1::> system services ndmp node-scope-mode on
----
+
NDMP Node-Scope-Mode ist aktiviert.

. NDMP-Service auf allen Knoten im Cluster aktivieren:
+
Die Verwendung des Platzhalters "`*`" aktiviert den NDMP-Service auf allen Knoten gleichzeitig.

+
Sie müssen ein Passwort für die Authentifizierung der NDMP-Verbindung durch die Backup-Anwendung angeben.

+
[source, cli]
----
cluster1::> system services ndmp on -node *
----
+
[listing]
----
Please enter password:
Confirm password:
2 entries were modified.
----
. Die  `-clear-text` Option für die sichere Kommunikation des NDMP-Passworts deaktivieren:
+
Die Wildcard-*`" disables the `-clear-text` Option wird auf allen Knoten gleichzeitig verwendet.

+
[source, cli]
----
cluster1::> system services ndmp modify -node * -clear-text false
----
. Es sollte überprüft werden, dass der NDMP-Service aktiviert und die  `-clear-text` Option deaktiviert ist:
+
[source, cli]
----
cluster1::> system services ndmp show
----
+
[listing]
----
Node                  Enabled   Clear text  User Id
--------------------- --------- ----------- ---------
cluster1-1            true      false        root
cluster1-2            true      false        root
2 entries were displayed.
----
