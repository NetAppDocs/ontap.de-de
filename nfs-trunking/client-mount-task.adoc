= ONTAP Volumes oder Datenfreigaben für NFS-Trunking einbinden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Linux- und VMware Clients, die Trunking unterstützen, können Volumes oder Datenfreigaben von einem ONTAP NFSv4.1 Server einbinden, der für Trunking aktiviert ist.

Informationen dazu link:index.html#supported-clients["unterstützte Clients"].

[role="tabbed-block"]
====
.Anforderungen an Linux-Clients
--
Wenn Sie ONTAP 9.16.1 oder höher und Red Hat Enterprise Linux Version 8.7 oder höher (für RHEL 8) oder 9.2 oder höher (für RHEL 9) als Linux-Client verwenden, ist nur ein Mountpunkt für die Trunking-Gruppe erforderlich. Die exportierten Volumes werden mit diesem Befehl unter Verwendung der  `trunkdiscovery` Option gemountet:

[source, cli]
----
mount <lif_ip>:<volume_name> </mount_path> -o trunkdiscovery,vers=4.1
----
Andernfalls ist für jede Verbindung in der Trunking-Gruppe ein separater Mountpunkt erforderlich. Das exportierte Volume wird mit Befehlen ähnlich den folgenden gemountet, wobei die  `max_connect` Option verwendet wird:

[source, cli]
----
mount <lif1_ip>:<volume_name> </mount_path1> -o vers=4.1,max_connect=16
----
[source, cli]
----
mount <lif2_ip>:<volume_name> </mount_path2> -o vers=4.1,max_connect=16
----
Der Versionswert (`vers` sollte  `4.1` oder höher sein.

Der  `max_connect`Wert entspricht der Anzahl der Verbindungen in der Trunking-Gruppe.

--
.VMware Client-Anforderungen
--
Es wird eine Mount-Anweisung benötigt, die für jede Verbindung in der Trunking-Gruppe eine IP-Adresse enthält.

Den exportierten Datenspeicher mit einem Befehl ähnlich dem folgenden einbinden:

`#esxcli storage nfs41 -H lif1_ip, lif2_ip -s /mnt/sh are1 -v nfs41share`

Die  `-H`Werte entsprechen den Verbindungen in der Bündelgruppe.
