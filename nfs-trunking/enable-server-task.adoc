= Trunking auf einem ONTAP NFS-Server aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Trunking muss auf NFS-Servern explizit aktiviert werden. NFSv4.1 ist bei der Erstellung von NFS-Servern standardmäßig aktiviert.

Nach der Aktivierung des Trunkings ist zu überprüfen, ob die folgenden Dienste wie benötigt konfiguriert sind.

* link:../nfs-config/configure-dns-host-name-resolution-task.html["DNS"]
* link:../nfs-config/using-ldap-concept.html["LDAP"]
* link:../nfs-config/kerberos-nfs-strong-security-concept.html["Kerberos"]


.Schritte
. Trunking aktivieren und sicherstellen, dass NFSv4.1 aktiviert ist:
+
`vserver nfs create -vserver _svm_name_ -v4.1 enabled -v4.1-trunking enabled`

. Es sollte überprüft werden, ob NFS ausgeführt wird:
`vserver nfs status -vserver _svm_name_`
. Es sollte überprüft werden, ob NFS wie gewünscht konfiguriert ist:
+
`vserver nfs show -vserver _svm_name_`

+
Weitere Informationen zu link:../nfs-config/create-server-task.html["NFS-Serverkonfiguration."] .. Wenn Windows-Clients von dieser SVM bedient werden, sollten die Freigaben verschoben und anschließend der Server gelöscht werden.
`vserver cifs show -vserver _svm_name_`

+
+
`vserver cifs delete -vserver _svm_name_`
