= Single-Path-Exporte für ONTAP NFS-Trunking anpassen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen bestehenden NFSv4.1 Export mit einem einzelnen Pfad (nicht trunked) für die Verwendung von Trunking anpassen. Trunking-fähige Clients können von der verbesserten Leistung profitieren, sobald Trunking auf dem Server aktiviert ist, sofern die Voraussetzungen auf Server- und Clientseite erfüllt sind.

Durch die Anpassung eines Single-Path-Exports für Trunking können exportierte Datensätze in ihren bestehenden Volumes und SVMs beibehalten werden. Hierfür ist es erforderlich, Trunking auf dem NFS-Server zu aktivieren, die Netzwerk- und Exportkonfiguration zu aktualisieren und die exportierte Freigabe auf den Clients erneut einzubinden.

Das Aktivieren von Trunking führt zu einem Neustart des Servers. VMware-Clients müssen anschließend die exportierten Datenspeicher neu einbinden; Linux-Clients müssen exportierte Volumes mit der  `max_connect` Option neu einbinden.
