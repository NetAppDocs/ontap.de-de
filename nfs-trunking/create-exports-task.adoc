= Erstellen einer ONTAP Volume Exportrichtlinie
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um Clients Zugriff auf Datenfreigaben zu ermöglichen, ist die Erstellung eines oder mehrerer Volumes erforderlich, und das Volume muss Exportregeln mit mindestens einer Regel besitzen.

Exportanforderungen des Clients:

* Linux-Clients benötigen für jede Trunking-Verbindung (das heißt, für jede LIF) einen separaten Mount und einen separaten Mountpunkt.
* VMware Clients benötigen nur einen einzigen Mountpunkt für ein exportiertes Volume, wobei mehrere LIFs angegeben werden.
+
VMware Clients benötigen Root-Zugriff in der Exportregel.



ONTAP-Exportanforderungen:

* Die Standardexportrichtlinie des SVM-Root-Volumes muss eine Regel enthalten, die allen Clients den offenen Zugriff über NFS ermöglicht.
+
Erfahren Sie mehr überlink:../nfs-config/open-export-policy-svm-root-volume-task.html["NFS-Client-Zugriff auf der ONTAP SVM."]

* Sie müssen dieselbe Regel auch der Standardrichtlinie hinzufügen, wenn Sie eine benutzerdefinierte Exportrichtlinie erstellt haben.
+
Erfahren Sie mehr überlink:../nfs-admin/default-export-policy-svms-concept.html["Standardexportrichtlinien für ONTAP NFS SVMs."]



.Schritte
. Eine Exportrichtlinie erstellen:
+
`vserver export-policy create -vserver _svm_name_ -policyname _policy_name_`

+
Der Name der Richtlinie kann bis zu 256 Zeichen lang sein.

. Es wird überprüft, ob die Exportrichtlinie erstellt wurde:
+
`vserver export-policy show -policyname _policy_name_`

+
.Beispiel
Die folgenden Befehle erstellen und überprüfen die Erstellung einer Exportrichtlinie mit dem Namen exp1 auf der SVM mit dem Namen vs1:

+
`vs1::> vserver export-policy create -vserver vs1 -policyname exp1`

. Eine Exportregel erstellen und zu einer bestehenden Exportrichtlinie hinzufügen:
+
`vserver export-policy rule create -vserver _svm_name_ -policyname _policy_name_ -ruleindex _integer_ -protocol nfs4 -clientmatch { _text | "text,text,…"_ } -rorule _security_type_ -rwrule _security_type_ -superuser _security_type_ -anon _user_ID_`

+
Der  `-clientmatch`Parameter sollte die trunking-fähigen Linux oder VMware Clients identifizieren, die den Export einbinden.

+
Erfahren Sie mehr überlink:../nfs-config/add-rule-export-policy-task.html["Exportregeln erstellen."]

. Das Volume mit einer Verbindung erstellen:
+
`volume create -vserver _svm_name_ -volume _volume_name_ -aggregate _aggregate_name_-size {integer[KB|MB|GB|TB|PB]} -security-style unix -user _user_name_or_number_ -group _group_name_or_number_ -junction-path _junction_path_ -policy _export_policy_name_`

+
Erfahren Sie mehr überlink:../nfs-config/create-volume-task.html["Volumen werden erstellt."]

. Es sollte überprüft werden, ob das Volume mit dem gewünschten Verbindungspunkt erstellt wurde:
+
`volume show -vserver _svm_name_ -volume _volume_name_ -junction-path`
