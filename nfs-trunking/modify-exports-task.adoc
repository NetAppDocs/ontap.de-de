= ONTAP Volume-Exportrichtlinien ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Damit Clients Trunking für bestehende Datenfreigaben nutzen können, kann es erforderlich sein, Exportrichtlinien und -regeln sowie die Volumes, denen sie zugeordnet sind, zu ändern. Für Linux Clients und VMware Datastores bestehen unterschiedliche Exportanforderungen.

Exportanforderungen des Clients:

* Linux-Clients benötigen für jede Trunking-Verbindung (das heißt, für jede LIF) einen separaten Mount und einen separaten Mountpunkt.
+
Wenn Sie auf ONTAP 9.14.1 aktualisieren und bereits ein Volume exportiert haben, kann dieses Volume weiterhin in einer Trunking-Gruppe verwendet werden.

* VMware Clients benötigen nur einen einzigen Mountpunkt für ein exportiertes Volume, wobei mehrere LIFs angegeben werden.
+
VMware Clients benötigen Root-Zugriff in der Exportregel.



.Schritte
. Es sollte überprüft werden, ob eine bestehende Exportrichtlinie vorhanden ist:
+
`vserver export-policy show`

. Es sollte überprüft werden, ob die bestehenden Exportrichtlinienregeln für die Trunking Konfiguration geeignet sind:
+
`vserver export-policy rule show -policyname _policy_name_`

+
Insbesondere ist zu überprüfen, ob der  `-clientmatch` Parameter die trunking-fähigen Linux- oder VMware-Clients korrekt identifiziert, die den Export einbinden.

+
Falls Anpassungen erforderlich sind, kann die Regel mit dem  `vserver export-policy rule modify` Befehl geändert oder eine neue Regel erstellt werden:

+
`vserver export-policy rule create -vserver _svm_name_ -policyname _policy_name_ -ruleindex integer -protocol nfs4 -clientmatch { text | "text,text,…" } -rorule _security_type_ -rwrule _security_type_ -superuser _security_type_ -anon _user_ID_`

+
Erfahren Sie mehr überlink:../nfs-config/add-rule-export-policy-task.html["Exportregeln erstellen."]

. Es wird überprüft, ob die vorhandenen exportierten Volumes online sind:
+
`volume show -vserver _svm_name_`
