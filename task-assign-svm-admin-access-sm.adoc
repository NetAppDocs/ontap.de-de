= Weisen Sie den SVM-Administratorzugriff mit ONTAP System Manager zu.
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Ab ONTAP 9.19.1 können ONTAP-Clusteradministratoren demjenigen, der die SVM verwaltet (SVM-Administrator), Zugriff auf den System Manager gewähren.

Dies ist besonders nützlich in einer mandantenbasierten ONTAP-Umgebung, in der der Cluster-Administrator für jeden Mandanten ein SVM-Dashboard bereitstellen kann.

Cluster-Administratoren können ihren SVM-Administratoren auch erlauben, Volumes zu erstellen.



== Schritt 1: Bereiten Sie die Management-Netzwerkschnittstelle für das SVM-Dashboard vor

Bevor Sie das SVM-Dashboard Ihrem SVM-Administrator zuweisen, stellen Sie sicher, dass die SVM über eine Management-Netzwerkschnittstelle verfügt. Wenn die SVM keine Netzwerkschnittstelle hat, können Sie eine erstellen.

.Schritte
. Im System Manager wählen Sie *Netzwerkschnittstellen > Übersicht* und anschließend *Hinzufügen*, um eine Netzwerkschnittstelle zu erstellen.
. Wählen Sie *Storage VM management* als Netzwerkschnittstellentyp aus.
. Geben Sie die *IP-Adresse* und die *Subnetzmaske* für die SVM-Management-Netzwerkschnittstelle an.




== Schritt 2: Überprüfen Sie die Konfiguration des vsadmin-Benutzerkontos

Wann immer eine SVM erstellt wird, erstellt ONTAP automatisch ein zugehöriges vsadmin-Benutzerkonto. Bevor Sie das SVM-Dashboard dem Administrator zuweisen, stellen Sie sicher, dass das vsadmin-Benutzerkonto ein Passwort hat und entsperrt ist.

.Schritte
. Öffnen Sie das System Manager-Dashboard als Clusteradministrator.
. Wählen Sie *Cluster > Speicher-VMs*.
. Wählen Sie die SVM aus, wählen Sie *Einstellungen* und anschließend *Sicherheit > Benutzer und Rollen*.
. Wählen Sie neben dem Benutzer *vsadmin* image:icon_kabob.gif["Symbol für Menüoptionen"] aus und, falls erforderlich, Folgendes:
+
.. *Passwort ändern*, um das vsadmin-Passwort zu aktualisieren.
.. *Entsperren* Sie es, um sicherzustellen, dass es nutzbar ist.






== Schritt 3: SVM-Administratoren das Hinzufügen von Volumes ermöglichen (optional)

Als ONTAP Clusteradministrator können Sie Ihren SVM-Administratoren die Berechtigung erteilen, lokale Volumes in ihren SVMs hinzuzufügen.

.Schritte
. Öffnen Sie das System Manager-Dashboard und melden Sie sich als Cluster-Administrator an.
. Wählen Sie unter *Cluster* *Storage VMs* aus.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] für die passende SVM aus.
. Wählen Sie *Bearbeiten*.
. Unter *Ressourcenzuweisung* wählen Sie das Kästchen *Volume-Erstellung auf bevorzugte lokale Tiers beschränken* aus.
. Standardmäßig werden alle lokalen Ebenen einbezogen, wenn Sie das Kästchen aktivieren. Sie können bei Bedarf bestimmte Ebenen entfernen. Wenn Sie alle lokalen Ebenen entfernen oder das Kästchen leer lassen, können SVM-Administratoren keine lokalen Volumes bereitstellen.
. Wählen Sie *Speichern*.




== Schritt 4: SVM-Administratorzugriff zuweisen

Um jedem SVM-Administrator Zugriff auf das SVM-Dashboard zu gewähren, geben Sie die SVM-Management-IP-Adresse und das vsadmin-Passwort an jeden SVM-Administrator weiter.

.Schritte
. Im System Manager wählen Sie *Netzwerkschnittstellen > Übersicht* und wählen dann die Netzwerkschnittstelle *Management*.
. Wählen Sie die IP-Adresse aus und teilen Sie sie zusammen mit dem Benutzernamen und dem Passwort *vsadmin* dem SVM-Administrator mit.
