= ONTAP Befehle zur Verwaltung von SMB-Namenszuordnungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für die Verwaltung von Namenszuordnungen gibt es spezielle ONTAP Befehle.

[cols="35,65"]
|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Eine Namenszuordnung erstellen
 a| 
`vserver name-mapping create`



 a| 
Eine Namenszuordnung an einer bestimmten Position einfügen
 a| 
`vserver name-mapping insert`



 a| 
Anzeigenamenzuordnungen anzeigen
 a| 
`vserver name-mapping show`



 a| 
Die Position zweier Namenszuordnungen wird getauscht. HINWEIS: Ein Tausch ist nicht zulässig, wenn die Namenszuordnung mit einem ip-qualifier-Eintrag konfiguriert ist.
 a| 
`vserver name-mapping swap`



 a| 
Eine Namenszuordnung ändern
 a| 
`vserver name-mapping modify`



 a| 
Eine Namenszuordnung löschen
 a| 
`vserver name-mapping delete`



 a| 
Die korrekte Namenszuordnung validieren
 a| 
`vserver security file-directory show-effective-permissions -vserver vs1 -win-user-name user1 -path / -share-name sh1`

|===
Weitere Informationen zu  `vserver name-mapping` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+name-mapping["ONTAP-Befehlsreferenz"^].
