= Vertrauenswürdige ONTAP SMB-Domänen in bevorzugten Listen hinzufügen, entfernen oder ersetzen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können vertrauenswürdige Domänen zur Liste der bevorzugten vertrauenswürdigen Domänen für den SMB-Server hinzufügen oder daraus entfernen oder die aktuelle Liste bearbeiten. Wenn eine Liste bevorzugter vertrauenswürdiger Domänen konfiguriert ist, wird diese Liste anstelle der ermittelten bidirektionalen vertrauenswürdigen Domänen bei multidomänenübergreifenden Namenszuordnungs-Suchen verwendet.

.Über diese Aufgabe
* Wenn Sie einer bestehenden Liste vertrauenswürdige Domänen hinzufügen, wird die neue Liste mit der bestehenden Liste zusammengeführt, wobei die neuen Einträge am Ende eingefügt werden. Die vertrauenswürdigen Domänen werden in der Reihenfolge durchsucht, in der sie in der Liste der vertrauenswürdigen Domänen erscheinen.
* Wenn Sie vertrauenswürdige Domänen aus der bestehenden Liste entfernen und keine Liste angeben, wird die gesamte Liste der vertrauenswürdigen Domänen für die angegebene Storage Virtual Machine (SVM) entfernt.
* Wenn die bestehende Liste der vertrauenswürdigen Domänen geändert wird, überschreibt die neue Liste die bestehende Liste.


[NOTE]
====
Sie sollten in der Liste der bevorzugten vertrauenswürdigen Domänen nur bidirektional vertrauenswürdige Domänen eintragen. Obwohl Sie ausgehende oder eingehende Vertrauensstellungen in die bevorzugte Domänenliste eintragen können, werden diese bei der Durchführung von multidomain Namenszuordnungs-Suchen nicht verwendet. ONTAP überspringt den Eintrag für die unidirektionale Domäne und fährt mit der nächsten bidirektional vertrauenswürdigen Domäne in der Liste fort.

====
.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn Sie Folgendes mit der Liste der bevorzugten vertrauenswürdigen Domänen durchführen möchten... | Verwenden Sie den Befehl... 


 a| 
Vertrauenswürdige Domänen zur Liste hinzufügen
 a| 
`+vserver cifs domain name-mapping-search add -vserver _vserver_name_ -trusted-domains FQDN, ...+`



 a| 
Vertrauenswürdige Domänen aus der Liste entfernen
 a| 
`+vserver cifs domain name-mapping-search remove -vserver _vserver_name_ [-trusted-domains FQDN, ...]+`



 a| 
Die bestehende Liste bearbeiten
 a| 
`+vserver cifs domain name-mapping-search modify -vserver _vserver_name_ -trusted-domains FQDN, ...+`

|===


.Beispiele
Der folgende Befehl fügt zwei vertrauenswürdige Domänen (cifs1.example.com und cifs2.example.com) zur Liste der bevorzugten vertrauenswürdigen Domänen hinzu, die von SVM vs1 verwendet wird:

[listing]
----
cluster1::> vserver cifs domain name-mapping-search add -vserver vs1 -trusted-domains cifs1.example.com, cifs2.example.com
----
Der folgende Befehl entfernt zwei vertrauenswürdige Domänen aus der von SVM vs1 verwendeten Liste:

[listing]
----
cluster1::> vserver cifs domain name-mapping-search remove -vserver vs1 -trusted-domains cifs1.example.com, cifs2.example.com
----
Der folgende Befehl ändert die von SVM vs1 verwendete Liste vertrauenswürdiger Domänen. Die neue Liste ersetzt die ursprüngliche Liste:

[listing]
----
cluster1::> vserver cifs domain name-mapping-search modify -vserver vs1 -trusted-domains cifs3.example.com
----
