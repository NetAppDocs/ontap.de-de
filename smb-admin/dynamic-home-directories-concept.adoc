= Informationen zur Aktivierung dynamischer Home Directorys auf ONTAP SMB-Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP Home Directorys ermöglichen die Konfiguration einer SMB-Freigabe, die je nach Benutzer, der sich verbindet, und einer Reihe von Variablen unterschiedlichen Verzeichnissen zugeordnet wird. Anstatt für jeden Benutzer separate Freigaben zu erstellen, kann eine einzige Freigabe mit wenigen Home Directory-Parametern konfiguriert werden, um die Beziehung eines Benutzers zwischen einem Einstiegspunkt (der Freigabe) und dem Home Directory (einem Verzeichnis auf der SVM) zu definieren.

Ein als Gastbenutzer angemeldeter Benutzer besitzt kein Home Directory und kann nicht auf die Home Directories anderer Benutzer zugreifen. Es gibt vier Variablen, die bestimmen, wie ein Benutzer einem Verzeichnis zugeordnet wird:

* *Freigabename*
+
Dies ist der Name der Freigabe, die Sie erstellen, mit der sich der Benutzer verbindet. Für diese Freigabe muss die Home Directory-Eigenschaft festgelegt werden.

+
Der Freigabename kann die folgenden dynamischen Namen verwenden:

+
** `%w` (der Windows-Benutzername des Benutzers)
** `%d` (der Windows-Domänenname des Benutzers)
**  `%u` (der zugeordnete UNIX-Benutzername des Benutzers) Um den Freigabenamen in allen Home-Verzeichnissen eindeutig zu machen, muss der Freigabename entweder die /`%w` oder die  `%u` Variable enthalten. Der Freigabename kann sowohl die  `%d` als auch die /`%w` Variable enthalten (zum Beispiel  `%d`/`%w`), oder der Freigabename kann einen statischen und einen variablen Teil enthalten (zum Beispiel home_/`%w`).


* *Freigabepfad*
+
Dies ist der relative Pfad, der durch die Freigabe definiert und daher einem der Freigabenamen zugeordnet ist, der an jeden Suchpfad angehängt wird, um den vollständigen Home Directory-Pfad des Benutzers vom Stammverzeichnis der SVM aus zu generieren. Er kann statisch sein (zum Beispiel, `home`), dynamisch sein (zum Beispiel, `%w`), oder eine Kombination aus beidem (zum Beispiel, `eng/%w`).

* *Suchpfade*
+
Dies ist die Menge absoluter Pfade vom Stammverzeichnis der SVM, die Sie angeben und die die ONTAP-Suche nach Home Directorys steuern. Es können ein oder mehrere Suchpfade mit dem  `vserver cifs home-directory search-path add` Befehl angegeben werden. Wenn mehrere Suchpfade angegeben werden, versucht ONTAP diese in der angegebenen Reihenfolge, bis ein gültiger Pfad gefunden wird. Weitere Informationen zu  `vserver cifs home-directory search-path add` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-cifs-home-directory-search-path-add.html["ONTAP-Befehlsreferenz"^].

* *Verzeichnis*
+
Dies ist das Home Directory des Benutzers, das für den Benutzer erstellt wird. Der Verzeichnisname ist üblicherweise der Name des Benutzers. Das Home Directory muss in einem der Verzeichnisse erstellt werden, die durch die Suchpfade definiert sind.



Als Beispiel kann folgendes Setup betrachtet werden:

* Benutzer: John Smith
* Benutzerdomäne: acme
* Benutzername: jsmith
* SVM-Name: vs1
* Home-Verzeichnis-Freigabename #1: home_ `%w` - Freigabepfad: `%w`
* Home-Verzeichnis-Freigabename #2: `%w` - Freigabepfad: `%d/%w`
* Suchpfad Nr. 1: `/vol0home/home`
* Suchpfad Nr. 2: `/vol1home/home`
* Suchpfad Nr. 3: `/vol2home/home`
* Home-Verzeichnis: `/vol1home/home/jsmith`


Szenario 1: Der Benutzer stellt eine Verbindung zu  `\\vs1\home_jsmith` her. Dies entspricht dem Namen der ersten Home-Verzeichnisfreigabe und generiert den relativen Pfad  `jsmith`. ONTAP sucht nun nach einem Verzeichnis mit dem Namen  `jsmith`, indem jeder Suchpfad der Reihe nach überprüft wird:

* `/vol0home/home/jsmith`Existiert nicht, es wird mit Suchpfad Nr. 2 fortgefahren.
* `/vol1home/home/jsmith`existiert; daher wird Suchpfad Nr. 3 nicht überprüft; der Benutzer ist nun mit seinem Home-Verzeichnis verbunden.


Szenario 2: Der Benutzer stellt eine Verbindung zu  `\\vs1\jsmith` her. Dies entspricht dem Namen der zweiten Home-Verzeichnisfreigabe und generiert den relativen Pfad  `acme/jsmith`. ONTAP sucht nun nach einem Verzeichnis mit dem Namen  `acme/jsmith`, indem jeder Suchpfad der Reihe nach überprüft wird:

* `/vol0home/home/acme/jsmith`Existiert nicht, es wird mit Suchpfad Nr. 2 fortgefahren.
* `/vol1home/home/acme/jsmith`Existiert nicht, es wird mit Suchpfad Nr. 3 fortgefahren.
* `/vol2home/home/acme/jsmith` existiert nicht; das Home-Verzeichnis existiert nicht; daher schlägt die Verbindung fehl.
