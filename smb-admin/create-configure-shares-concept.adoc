= Informationen zum Erstellen und Konfigurieren von ONTAP SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Benutzer und Anwendungen über SMB auf Daten auf dem CIFS-Server zugreifen können, müssen SMB-Shares erstellt und konfiguriert werden, was einen benannten Zugriffspunkt in einem Volume darstellt. Shares können angepasst werden, indem Share-Parameter und Share-Eigenschaften angegeben werden. Eine vorhandene Share kann jederzeit geändert werden.

Wenn eine SMB-Freigabe erstellt wird, erstellt ONTAP standardmäßig eine ACL für die Freigabe mit Vollzugriffsberechtigungen für Everyone.

SMB-Freigaben sind an den CIFS-Server auf der Storage Virtual Machine (SVM) gebunden. SMB-Freigaben werden gelöscht, wenn entweder die SVM gelöscht wird oder der mit ihr verbundene CIFS-Server von der SVM gelöscht wird. Wird der CIFS-Server auf der SVM neu erstellt, müssen die SMB-Freigaben ebenfalls neu erstellt werden.
