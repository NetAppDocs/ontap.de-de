= Wie sich Signaturrichtlinien auf die Kommunikation mit ONTAP SMB-Servern auswirken
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Zusätzlich zu den SMB-Signatur-Sicherheitseinstellungen des CIFS-Servers steuern zwei SMB-Signaturrichtlinien auf Windows-Clients die digitale Signatur der Kommunikation zwischen Clients und dem CIFS-Server. Es kann die Einstellung konfiguriert werden, die den Geschäftsanforderungen entspricht.

Die SMB-Clientrichtlinien werden über die lokalen Sicherheitsrichtlinieneinstellungen von Windows gesteuert, die mithilfe der Microsoft Management Console (MMC) oder Active Directory GPOs konfiguriert werden. Weitere Informationen zur SMB-Clientsignierung und zu Sicherheitsproblemen enthält die Microsoft Windows-Dokumentation.

Hier sind Beschreibungen der beiden SMB-Signaturrichtlinien auf Microsoft Clients:

* `Microsoft network client: Digitally sign communications (if server agrees)`
+
Diese Einstellung steuert, ob die SMB-Signaturfunktion des Clients aktiviert ist. Sie ist standardmäßig aktiviert. Wenn diese Einstellung auf dem Client deaktiviert ist, hängt die Kommunikation des Clients mit dem CIFS-Server von der SMB-Signatureinstellung auf dem CIFS-Server ab.

* `Microsoft network client: Digitally sign communications (always)`
+
Diese Einstellung steuert, ob der Client für die Kommunikation mit einem Server eine SMB-Signatur benötigt. Sie ist standardmäßig deaktiviert. Wenn diese Einstellung auf dem Client deaktiviert ist, richtet sich das Verhalten bei der SMB-Signatur nach der Richtlinieneinstellung für  `Microsoft network client: Digitally sign communications (if server agrees)` und der Einstellung auf dem CIFS-Server.

+
[NOTE]
====
Wenn Ihre Umgebung Windows-Clients umfasst, die so konfiguriert sind, dass sie SMB-Signierung erfordern, muss die SMB-Signierung auf dem CIFS-Server aktiviert werden. Andernfalls kann der CIFS-Server diesen Systemen keine Daten bereitstellen.

====

Microsoft empfiehlt nicht mehr,  `Digitally sign communications (if client agrees)` oder  `Digitally sign communications (if server agrees)` Gruppenrichtlinieneinstellungen zu verwenden. Microsoft empfiehlt außerdem nicht mehr, die  `EnableSecuritySignature` Registrierungseinstellungen zu verwenden. Diese Optionen betreffen nur das SMB 1 Verhalten und können durch die  `Digitally sign communications (always)` Gruppenrichtlinieneinstellung oder die  `RequireSecuritySignature` Registrierungseinstellung ersetzt werden.
