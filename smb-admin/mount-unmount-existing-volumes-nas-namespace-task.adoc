= Vorhandene ONTAP SMB-Volumes im NAS-Namespace einbinden oder aushängen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Volume muss im NAS-Namespace eingebunden sein, bevor Sie den NAS-Client-Zugriff auf die in den Storage Virtual Machine (SVM)-Volumes enthaltenen Daten konfigurieren können. Ein Volume kann an einem Verbindungspunkt eingebunden werden, wenn es derzeit nicht eingebunden ist. Volumes können auch ausgehängt werden.

.Über diese Aufgabe
Wenn ein Volume ausgehängt und offline genommen wird, sind alle Daten innerhalb der Verbindung, einschließlich der Daten in Volumes mit Verbindungen innerhalb des Namespace des ausgehängten Volumes, für NAS-Clients nicht zugänglich.

[NOTE]
====
Um den Dateizugriff eines NAS-Clients auf ein Volume zu beenden, reicht es nicht aus, das Volume einfach auszuhängen. Das Volume muss offline genommen werden oder es sind andere Maßnahmen erforderlich, um sicherzustellen, dass Client-seitig Dateihandle-Caches ungültig werden. Weitere Informationen enthält der folgende Knowledge Base Artikel: https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/NFSv3_clients_still_have_access_to_a_volume_after_being_removed_from_the_namespace_in_ONTAP["NFSv3-Clients haben weiterhin Zugriff auf ein Volume, nachdem sie aus dem Namespace in ONTAP entfernt wurden"]

====
Wenn ein Volume ausgehängt und offline genommen wird, gehen die darin enthaltenen Daten nicht verloren. Zudem bleiben bestehende Volume-Exportrichtlinien und SMB-Shares, die auf dem Volume oder in Verzeichnissen und Verbindungspunkten innerhalb des ausgehängten Volumes erstellt wurden, erhalten. Wird das ausgehängte Volume wieder eingebunden, können NAS-Clients mit den bestehenden Exportrichtlinien und SMB-Shares auf die darin enthaltenen Daten zugreifen.

.Schritte
. Die gewünschte Aktion ausführen:
+
|===
| Wenn Sie möchten... | Die Befehle werden eingegeben... 


 a| 
Ein Volume einbinden
 a| 
`volume mount -vserver _svm_name_ -volume _volume_name_ -junction-path _junction_path_`



 a| 
Aushängen eines Volumes
 a| 
`volume unmount -vserver _svm_name_ -volume _volume_name_`

`volume offline -vserver _svm_name_ -volume _volume_name_`

|===
. Es sollte sichergestellt werden, dass sich das Volume im gewünschten Mount-Status befindet:
+
`volume show -vserver _svm_name_ -volume _volume_name_ -fields state,junction-path,junction-active`



.Beispiele
Das folgende Beispiel bindet ein Volume namens "`sales`", das sich auf SVM "`vs1`" befindet, an den Verbindungspunkt "`/sales`" ein:

[listing]
----
cluster1::> volume mount -vserver vs1 -volume sales -junction-path /sales

cluster1::> volume show -vserver vs1 state,junction-path,junction-active

vserver   volume     state     junction-path   junction-active
--------- ---------- -------- --------------- ----------------
vs1       data       online    /data           true
vs1       home4      online    /eng/home       true
vs1       sales      online    /sales          true
----
Das folgende Beispiel hängt ein Volume namens "`data`" auf SVM "`vs1`" aus und nimmt es offline:

[listing]
----
cluster1::> volume unmount -vserver vs1 -volume data
cluster1::> volume offline -vserver vs1 -volume data

cluster1::> volume show -vserver vs1 -fields state,junction-path,junction-active

vserver   volume     state     junction-path   junction-active
--------- ---------- --------- --------------- ---------------
vs1       data       offline   -               -
vs1       home4      online    /eng/home       true
vs1       sales      online    /sales          true
----
