= Windows-Sicherungsanwendungen und Unix-ähnliche Symbol-Links auf ONTAP SMB-Servern
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn eine unter Windows laufende Backup-Anwendung auf einen Symbol-Link (Symlink) im Unix-Stil stößt, wird diesem gefolgt und die Daten werden gesichert. Ab ONTAP 9.15.1 besteht die Möglichkeit, anstelle der Daten die Symlinks selbst zu sichern. Diese Funktion wird mit ONTAP FlexGroup Volumes und FlexVols vollständig unterstützt.



[[overview]]
== Überblick

Bevor Sie ändern, wie ONTAP Symbol-Links während einer Windows-Sicherung verarbeitet, sollten Ihnen die Vorteile, die wichtigsten Konzepte und die Konfigurationsoptionen bekannt sein.



[[benefits]]
=== Vorteile

Wenn diese Funktion deaktiviert oder nicht verfügbar ist, wird jeder Symbol-Link durchlaufen und die Daten, auf die er verweist, werden gesichert. Dadurch können mitunter unnötige Daten gesichert werden und in bestimmten Fällen kann die Anwendung in einer Schleife enden. Die Sicherung der Symbol-Links selbst vermeidet diese Probleme. Da die Symbol-Link-Dateien im Vergleich zu den Daten in den meisten Fällen sehr klein sind, ist die Sicherung zudem schneller abgeschlossen. Die Gesamtleistung des Clusters kann sich aufgrund der reduzierten IO-Operationen ebenfalls verbessern.



[[windows-server-environment]]
=== Windows Server Umgebung

Diese Funktion wird von Sicherungsanwendungen unter Windows unterstützt. Die relevanten technischen Aspekte der Umgebung sollten vor der Verwendung bekannt sein.

.Erweiterte Attribute
Windows unterstützt erweiterte Attribute (EA), die zusammen zusätzliche Metadaten bilden, die optional mit den Dateien verknüpft sind. Diese Attribute werden von verschiedenen Anwendungen verwendet, wie zum Beispiel vom Windows Subsystem for Linux, wie unter  https://learn.microsoft.com/en-us/windows/wsl/file-permissions["Dateiberechtigungen für WSL"^] beschrieben. Anwendungen können beim Lesen von Daten aus ONTAP erweiterte Attribute für jede Datei anfordern.

Die symbolischen Verknüpfungen werden in den erweiterten Attributen zurückgegeben, wenn die Funktion aktiviert ist. Daher muss eine Sicherungsanwendung standardmäßige Unterstützung für erweiterte Attribute bieten, die zum Speichern der Metadaten verwendet wird. Einige Windows-Dienstprogramme unterstützen und bewahren die erweiterten Attribute. Wenn die Sicherungssoftware jedoch das Sichern und Wiederherstellen der erweiterten Attribute nicht unterstützt, werden die Metadaten, die mit jeder Datei verknüpft sind, nicht erhalten und die symbolischen Verknüpfungen werden nicht korrekt verarbeitet.

.Windows Konfiguration
Backup-Anwendungen, die auf einem Microsoft Windows-Server ausgeführt werden, können eine spezielle Berechtigung erhalten, die es ihnen ermöglicht, die normale Dateisicherheit zu umgehen. Dies erfolgt typischerweise durch das Hinzufügen der Anwendungen zur Gruppe Backup Operators. Die Anwendungen können dann Dateien nach Bedarf sichern und wiederherstellen sowie weitere zugehörige Systemoperationen ausführen. Es gibt subtile Änderungen am von den Backup-Anwendungen verwendeten SMB-Protokoll, die von ONTAP beim Lesen und Schreiben der Daten erkannt werden können.



[[requirements]]
=== Anforderungen

Die Symlink Backup-Funktion hat mehrere Anforderungen, darunter:

* Auf Ihrem Cluster läuft ONTAP 9.15.1 oder höher.
* Eine Windows-Sicherungsanwendung, der spezielle Sicherungsberechtigungen zugewiesen wurden.
* Die Backup-Anwendung muss außerdem erweiterte Attribute unterstützen und diese während der Backup-Vorgänge anfordern.
* Die ONTAP Symlink Backup-Funktion ist für die entsprechende Daten-SVM aktiviert.




[[configuration-options]]
=== Konfigurationsoptionen

Zusätzlich zur ONTAP CLI kann diese Funktion auch über die ONTAP REST API verwaltet werden. Siehe  https://docs.netapp.com/us-en/ontap-automation/whats-new.html["Neues zur ONTAP REST API und Automatisierung"^] für weitere Informationen. Die Konfiguration, die festlegt, wie ONTAP die Unix-Style-Symbol-Links verarbeitet, muss für jede SVM separat vorgenommen werden.



[[enable-the-symlink-backup-feature-in-ontap]]
== Die Symlink-Backup-Funktion in ONTAP aktivieren

Mit ONTAP 9.15.1 wurde eine Konfigurationsoption für einen bestehenden CLI-Befehl eingeführt. Mit dieser Option lässt sich die Verarbeitung von Unix-ähnlichen Symlinks aktivieren oder deaktivieren.

.Bevor Sie beginnen
Die grundlegenden <<Anforderungen>> sollten überprüft werden. Zusätzlich:

* Die Möglichkeit besteht, die CLI-Berechtigungen auf die fortgeschrittene Stufe anzuheben.
* Bestimmen Sie die Daten-SVM, die Sie modifizieren möchten. Die SVM `vs1` wird im Beispielbefehl verwendet.


.Schritte
. Die erweiterte Berechtigungsstufe wird festgelegt.
+
[source, cli]
----
set privilege advanced
----
. Symlink-Dateisicherung aktivieren.
+
[source, cli]
----
vserver cifs options modify -vserver vs1 -is-backup-symlink-enabled true
----
