= Konfigurationen von BranchCache auf ONTAP SMB-Freigaben anpassen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Konfiguration des BranchCache-Dienstes auf SVMs anpassen, einschließlich der Änderung des Pfads zum Hash-Speicherverzeichnis, der maximalen Größe des Hash-Speicherverzeichnisses, des Betriebsmodus und der unterstützten BranchCache-Versionen. Auch die Größe des Volumes, das den Hash-Speicher enthält, kann erhöht werden.

.Schritte
. Die entsprechende Maßnahme ausführen:
+
|===
| Wenn Sie möchten... | Geben Sie Folgendes ein... 


 a| 
Die Größe des Hash-Speicherverzeichnisses anpassen
 a| 
`vserver cifs branchcache modify -vserver vserver_name -hash-store-max-size {integer[KB|MB|GB|TB|PB]}`



 a| 
Die Größe des Volumes, das den Hash-Speicher enthält, erhöhen
 a| 
`volume size -vserver vserver_name -volume volume_name -new-size new_size[k|m|g|t]`Wenn das Volume, das den Hash-Speicher enthält, voll ist, kann möglicherweise die Größe des Volumes erhöht werden. Die neue Volume-Größe kann als Zahl gefolgt von einer Einheit angegeben werden.

Erfahren Sie mehr überlink:../volumes/commands-manage-flexvol-volumes-reference.html["Verwaltung von FlexVol-Volumes"]



 a| 
Den Pfad des Hash-Speicherverzeichnisses ändern
 a| 
`vserver cifs branchcache modify -vserver vserver_name -hash-store-path path -flush-hashes {true|false}`Wenn die SVM eine SVM-Disaster-Recovery-Quelle ist, darf sich der Hash-Pfad nicht auf dem Root-Volume befinden. Dies liegt daran, dass das Root Volume nicht auf das Notfallwiederherstellungsziel repliziert wird.

Der BranchCache Hash-Pfad kann Leerzeichen und beliebige gültige Dateinamenzeichen enthalten.

Wenn Sie den Hash-Pfad ändern, ist  `-flush-hashes` ein erforderlicher Parameter, der angibt, ob ONTAP die Hashes aus dem ursprünglichen Hash-Speicherort löschen soll. Für den  `-flush-hashes` Parameter können die folgenden Werte festgelegt werden:

** Wenn  `true` angegeben wird, löscht ONTAP die Hashes am ursprünglichen Speicherort und erstellt neue Hashes am neuen Speicherort, sobald neue Anfragen von BranchCache-fähigen Clients eingehen.
** Wenn  `false` angegeben wird, werden die Hashes nicht gelöscht.
+
In diesem Fall kann die vorhandenen Hashes später wiederverwendet werden, indem der Pfad zum Hash-Speicher wieder auf den ursprünglichen Speicherort geändert wird.





 a| 
Den Betriebsmodus ändern
 a| 
`vserver cifs branchcache modify -vserver vserver_name -operating-mode {per-share|all-shares|disable}`

Beim Ändern des Betriebsmodus ist Folgendes zu beachten:

** ONTAP signalisiert BranchCache-Unterstützung für eine Freigabe, wenn die SMB-Sitzung eingerichtet wird.
** Clients, die bereits Sitzungen aufgebaut haben, wenn BranchCache aktiviert wird, müssen die Verbindung trennen und erneut herstellen, um zwischengespeicherte Inhalte für diese Freigabe nutzen zu können.




 a| 
Die Unterstützung der BranchCache Version ändern
 a| 
`vserver cifs branchcache modify -vserver vserver_name -versions {v1-enable|v2-enable|enable-all}`

|===
. Die Konfigurationsänderungen lassen sich mit dem  `vserver cifs branchcache show` Befehl überprüfen.
