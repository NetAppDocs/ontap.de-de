= Richtlinien für die Bereitstellung von Offline ONTAP SMB-Dateien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es gibt einige wichtige Richtlinien, die beim Bereitstellen von Offline-Dateien auf Home-Verzeichnisfreigaben zu beachten sind, bei denen die  `showsnapshot` Freigabeeigenschaft auf Home-Verzeichnissen gesetzt ist.

Wenn die  `showsnapshot`Freigabeeigenschaft für eine Home-Verzeichnisfreigabe festgelegt ist, für die Offline-Dateien konfiguriert sind, speichern Windows-Clients alle Snapshots im  `~snapshot`Ordner im Home-Verzeichnis des Benutzers zwischen.

Windows-Clients speichern alle Snapshots im Home-Verzeichnis zwischen, wenn eine oder mehrere der folgenden Bedingungen zutreffen:

* Der Benutzer stellt das Home-Verzeichnis vom Client aus offline zur Verfügung.
+
Der Inhalt des  `~snapshot` Ordners im Home-Verzeichnis wird einbezogen und offline verfügbar gemacht.

* Der Benutzer konfiguriert die Ordnerumleitung, um einen Ordner wie  `My Documents` in das Stammverzeichnis eines Home-Verzeichnisses umzuleiten, das sich auf der CIFS-Server-Freigabe befindet.
+
Manche Windows-Clients stellen den umgeleiteten Ordner automatisch offline zur Verfügung. Wenn der Ordner in das Stammverzeichnis des Benutzerverzeichnisses umgeleitet wird, ist der  `~snapshot` Ordner im zwischengespeicherten Offline-Inhalt enthalten.



[NOTE]
====
Offline-Dateibereitstellungen, bei denen der  `~snapshot` Ordner in den Offline-Dateien enthalten ist, sollten vermieden werden. Die Snapshots im  `~snapshot` Ordner enthalten alle Daten des Volumes zu dem Zeitpunkt, an dem ONTAP den Snapshot erstellt hat. Daher verbraucht das Erstellen einer Offline-Kopie des  `~snapshot` Ordners erheblichen lokalen Speicherplatz auf dem Client, beansprucht Bandbreite während der Offline-Dateisynchronisierung und erhöht die Zeit, die für die Synchronisierung der Offline-Dateien benötigt wird.
