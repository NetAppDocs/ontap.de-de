= ONTAP SMB-Freigaben mit der Einstellung „force-group share“ erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine SMB-Freigabe mit der Einstellung „force-group share“ kann erstellt werden, wenn SMB-Benutzer, die auf Daten auf Volumes oder Qtrees mit UNIX-Dateisicherheit zugreifen, von ONTAP als Mitglieder derselben UNIX-Gruppe betrachtet werden sollen.

.Schritt
. Die SMB-Freigabe wird erstellt: `vserver cifs share create -vserver _vserver_name_ -share-name _share_name_ -path path -force-group-for-create _UNIX_group_name_`
+
Wenn der UNC-Pfad (`\\servername\sharename\filepath`) der Freigabe mehr als 256 Zeichen enthält (ohne das anfängliche „``\\``“ im UNC-Pfad), ist die Registerkarte *Sicherheit* im Windows-Eigenschaftenfenster nicht verfügbar. Dies ist ein Problem des Windows-Clients und nicht von ONTAP. Um dieses Problem zu vermeiden, sollten keine Freigaben mit UNC-Pfaden erstellt werden, die mehr als 256 Zeichen enthalten.

+
Wenn die Erzwingungsgruppe nach der Erstellung der Freigabe entfernt werden soll, kann die Freigabe jederzeit bearbeitet und eine leere Zeichenfolge ("") als Wert für den  `-force-group-for-create` Parameter angegeben werden. Wenn die Erzwingungsgruppe durch Bearbeiten der Freigabe entfernt wird, behalten alle bestehenden Verbindungen zu dieser Freigabe die zuvor festgelegte Erzwingungsgruppe als primäre GID bei.



.Beispiel
Der folgende Befehl erstellt eine Freigabe „`webpages`“, die im Web im  `/corp/companyinfo` Verzeichnis zugänglich ist, in dem alle Dateien, die SMB-Benutzer erstellen, der Gruppe webgroup1 zugewiesen werden:

`vserver cifs share create -vserver vs1 -share-name webpages -path /corp/companyinfo -force-group-for-create webgroup1`
