= Informationen zu den Auswirkungen der ONTAP SMB-Signatur auf die Leistung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn SMB-Sitzungen SMB-Signatur verwenden, kommt es zu Leistungseinbußen bei der gesamten SMB-Kommunikation mit Windows-Clients, was sowohl die Clients als auch den Server betrifft (also die Knoten im Cluster, auf denen die SVM mit dem SMB-Server ausgeführt wird).

Die Auswirkungen auf die Leistung zeigen sich in einer erhöhten CPU-Auslastung sowohl auf den Clients als auch auf dem Server, obwohl sich das Netzwerkverkehr nicht ändert.

Das Ausmaß der Leistungseinbußen hängt von der Version von ONTAP 9 ab, die Sie verwenden. Ab ONTAP 9.7 kann ein neuer Verschlüsselungs-Offload-Algorithmus eine bessere Performance bei signiertem SMB-Datenverkehr ermöglichen. SMB-Signatur-Offload ist standardmäßig aktiviert, wenn die SMB-Signierung aktiviert ist.

Für eine verbesserte SMB-Signaturleistung ist die AES-NI-Offload-Funktion erforderlich. Im Hardware Universe (HWU) kann überprüft werden, ob für Ihre Plattform AES-NI-Offload unterstützt wird.

Weitere Leistungsverbesserungen sind ebenfalls möglich, wenn Sie SMB Version 3.11 verwenden können, die den wesentlich schnelleren GCM Algorithmus unterstützt.

Je nach Netzwerk, ONTAP 9 Version, SMB Version und SVM-Implementierung kann der Einfluss der SMB-Signierung auf die Leistung stark variieren; dies lässt sich nur durch Tests in Ihrer Netzwerkumgebung überprüfen.

Die meisten Windows-Clients verhandeln standardmäßig die SMB-Signierung, sofern diese auf dem Server aktiviert ist. Wenn für einige Ihrer Windows-Clients SMB-Schutz erforderlich ist und die SMB-Signierung Leistungsprobleme verursacht, kann die SMB-Signierung auf allen Windows-Clients deaktiviert werden, die keinen Schutz vor Replay-Angriffen benötigen. Weitere Informationen zum Deaktivieren der SMB-Signierung auf Windows-Clients enthält die Microsoft Windows-Dokumentation.
