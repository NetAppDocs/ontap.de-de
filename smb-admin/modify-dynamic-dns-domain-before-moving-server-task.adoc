= Die dynamische DNS-Domäne vor dem Verschieben von ONTAP SMB-Servern anpassen.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn der in Active Directory integrierte DNS-Server die DNS-Einträge des SMB-Servers dynamisch im DNS registrieren soll, wenn der SMB-Server in eine andere Domäne verschoben wird, muss dynamisches DNS (DDNS) auf der Storage Virtual Machine (SVM) vor dem Verschieben des SMB-Servers geändert werden.

.Bevor Sie beginnen
Die DNS-Namensdienste auf der SVM müssen so angepasst werden, dass sie die DNS-Domäne verwenden, die die Service-Location-Einträge für die neue Domäne enthält, in der das SMB-Server-Computerkonto enthalten sein wird. Wenn sicheres DDNS verwendet wird, sind Active Directory-integrierte DNS-Namensserver erforderlich.

.Über diese Aufgabe
Obwohl DDNS (sofern auf der SVM konfiguriert) die DNS-Einträge für Daten-LIFs automatisch zur neuen Domäne hinzufügt, werden die DNS-Einträge der ursprünglichen Domäne nicht automatisch vom ursprünglichen DNS-Server gelöscht. Diese müssen manuell gelöscht werden.

Um die DDNS-Änderungen vor dem Verschieben des SMB-Servers abzuschließen, siehe das folgende Thema:

link:../networking/configure_dynamic_dns_services.html["Dynamische DNS-Dienste konfigurieren"]
