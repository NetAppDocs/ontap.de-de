= Verfügbare ONTAP SMB Serveroptionen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es ist hilfreich, die verfügbaren Optionen zu kennen, wenn die Anpassung des SMB-Servers in Betracht gezogen wird. Obwohl einige Optionen für die allgemeine Verwendung auf dem SMB-Server vorgesehen sind, werden mehrere zur Aktivierung und Konfiguration spezifischer SMB-Funktionen verwendet. SMB-Serveroptionen werden mit der  `vserver cifs options modify` Option gesteuert.

Die folgende Liste enthält die SMB-Serveroptionen, die auf Administratorebene verfügbar sind:

* *Konfigurieren des Zeitüberschreitungswerts für die SMB-Sitzung*
+
Durch die Konfiguration dieser Option können Sie die Anzahl der Sekunden Leerlaufzeit festlegen, nach der eine SMB-Sitzung getrennt wird. Eine inaktive Sitzung ist eine Sitzung, in der ein Benutzer keine Dateien oder Verzeichnisse auf dem Client geöffnet hat. Der Standardwert beträgt 900 Sekunden.

* *Konfigurieren des Standard-UNIX-Benutzers*
+
Durch die Konfiguration dieser Option kann der Standard-UNIX-Benutzer festgelegt werden, den der SMB-Server verwendet. ONTAP erstellt automatisch einen Standardbenutzer namens „`pcuser`“ (mit der UID 65534), erstellt eine Gruppe namens „`pcuser`“ (mit der GID 65534) und fügt den Standardbenutzer der Gruppe „`pcuser`“ hinzu. Beim Erstellen eines SMB-Servers wird „`pcuser`“ von ONTAP automatisch als Standard-UNIX-Benutzer konfiguriert.

* *Konfiguration des Gast-UNIX-Benutzers*
+
Durch die Konfiguration dieser Option kann der Name eines UNIX-Benutzers festgelegt werden, dem Benutzer zugeordnet werden, die sich aus nicht vertrauenswürdigen Domänen anmelden. Dadurch wird es einem Benutzer aus einer nicht vertrauenswürdigen Domäne ermöglicht, eine Verbindung zum SMB-Server herzustellen. Standardmäßig ist diese Option nicht konfiguriert (es gibt keinen Standardwert), daher ist es standardmäßig nicht zulässig, dass Benutzer aus nicht vertrauenswürdigen Domänen eine Verbindung zum SMB-Server herstellen.

* *Aktivieren oder Deaktivieren der Ausführung der Leseberechtigung für Modusbits*
+
Durch Aktivieren oder Deaktivieren dieser Option kann festgelegt werden, ob SMB-Clients ausführbare Dateien mit UNIX-Modusbits ausführen dürfen, auf die sie Lesezugriff haben, selbst wenn das UNIX-Ausführungsbit nicht gesetzt ist. Diese Option ist standardmäßig deaktiviert.

* *Aktivieren oder Deaktivieren der Möglichkeit, schreibgeschützte Dateien von NFS-Clients zu löschen*
+
Durch Aktivieren oder Deaktivieren dieser Option wird festgelegt, ob NFS-Clients das Löschen von Dateien oder Ordnern mit gesetztem Schreibschutz-Attribut erlauben. Die NTFS-Löschsemantik erlaubt das Löschen einer Datei oder eines Ordners mit gesetztem Schreibschutz-Attribut nicht. Die UNIX-Löschsemantik ignoriert das Schreibschutz-Bit und verwendet stattdessen die Berechtigungen des übergeordneten Verzeichnisses, um zu bestimmen, ob eine Datei oder ein Ordner gelöscht werden kann. Die Standardeinstellung ist  `disabled`, was der NTFS-Löschsemantik entspricht.

* *Konfiguration der Windows Internet Name Service Serveradressen*
+
Durch die Konfiguration dieser Option können Sie eine Liste von Windows Internet Name Service (WINS) Serveradressen als kommagetrennte Liste angeben. Sie müssen IPv4-Adressen angeben. IPv6-Adressen werden nicht unterstützt. Es gibt keinen Standardwert.



Die folgende Liste enthält die SMB-Serveroptionen, die auf der erweiterten Berechtigungsstufe verfügbar sind:

* *Erteilung von UNIX-Gruppenberechtigungen an CIFS-Server-Benutzer*
+
Die Konfiguration dieser Option legt fest, ob einem CIFS-Benutzer, der nicht der Eigentümer der Datei ist, die Gruppenberechtigung erteilt werden kann. Wenn der CIFS-Benutzer nicht der Eigentümer der UNIX-Sicherheitsdatei ist und dieser Parameter auf `true` gesetzt ist, wird die Gruppenberechtigung für die Datei erteilt. Wenn der CIFS-Benutzer nicht der Eigentümer der UNIX-Sicherheitsdatei ist und dieser Parameter auf `false` gesetzt ist, gelten die normalen UNIX-Regeln für die Vergabe der Dateiberechtigung. Dieser Parameter ist für UNIX-Sicherheitsdateien mit der Berechtigung `mode bits` anwendbar und nicht für Dateien mit dem NTFS- oder NFSv4-Sicherheitsmodus. Die Standardeinstellung ist `false`.

* *Aktivieren oder Deaktivieren von SMB 1.0*
+
SMB 1.0 ist standardmäßig auf einer SVM deaktiviert, für die in ONTAP 9.3 ein SMB-Server erstellt wird.

+
[NOTE]
====
Ab ONTAP 9.3 ist SMB 1.0 für neu erstellte SMB-Server in ONTAP 9.3 standardmäßig deaktiviert. Eine Migration auf eine neuere SMB-Version wird so bald wie möglich empfohlen, um sich auf Verbesserungen bei Sicherheit und Compliance vorzubereiten. Für Details steht Ihr NetApp Ansprechpartner zur Verfügung.

====
* *Aktivieren oder Deaktivieren von SMB 2.x*
+
SMB 2.0 ist die Mindestversion von SMB, die LIF-Failover unterstützt. Wenn SMB 2.x deaktiviert wird, deaktiviert ONTAP auch SMB 3.x automatisch.

+
SMB 2.0 wird nur auf SVMs unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren von SMB 3.0*
+
SMB 3.0 ist die minimale SMB-Version, die dauerhaft verfügbare Freigaben unterstützt. Windows Server 2012 und Windows 8 sind die minimalen Windows-Versionen, die SMB 3.0 unterstützen.

+
SMB 3.0 wird nur auf SVMs unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren von SMB 3.1*
+
Windows 10 ist die einzige Windows Version, die SMB 3.1 unterstützt.

+
SMB 3.1 wird nur auf SVMs unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren des ODX Copy Offloads*
+
ODX Copy Offload wird von Windows-Clients, die diese Funktion unterstützen, automatisch genutzt. Diese Option ist standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren des Direktkopiermechanismus für ODX Copy Offload*
+
Der Direktkopiermechanismus verbessert die Leistung des Copy-Offload-Vorgangs, wenn Windows-Clients versuchen, die Quelldatei einer Kopie in einem Modus zu öffnen, der verhindert, dass die Datei während des Kopiervorgangs geändert wird. Standardmäßig ist der Direktkopiermechanismus aktiviert.

* *Automatische Knoten-Weiterleitungen aktivieren oder deaktivieren*
+
Bei der automatischen Knotenweiterleitung verweist der SMB-Server Clients automatisch auf eine Daten-LIF, die lokal auf dem Knoten liegt, der die über die angeforderte Freigabe abgerufenen Daten hostet.

* *Aktivieren oder Deaktivieren von Exportrichtlinien für SMB*
+
Diese Option ist standardmäßig deaktiviert.

* *Aktivieren oder Deaktivieren der Verwendung von Verbindungspunkten als Reparse Points*
+
Wenn diese Option aktiviert ist, stellt der SMB-Server Verbindungspunkte SMB-Clients als Reparse-Punkte zur Verfügung. Diese Option ist nur für SMB 2.x oder SMB 3.0 Verbindungen gültig. Diese Option ist standardmäßig aktiviert.

+
Diese Option wird nur auf SVMs unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Konfigurieren der maximalen Anzahl gleichzeitiger Operationen pro TCP-Verbindung*
+
Der Standardwert ist 255.

* *Aktivieren oder Deaktivieren der lokalen Windows-Benutzer- und Gruppenfunktionalität*
+
Diese Option ist standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren der lokalen Windows-Authentifizierung von Benutzern*
+
Diese Option ist standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren der VSS-Schattenkopiefunktionalität*
+
ONTAP nutzt die Schattenkopie-Funktionalität, um Remote-Backups von Daten durchzuführen, die mit der Hyper-V over SMB Lösung gespeichert sind.

+
Diese Option wird nur auf SVMs und nur für Hyper-V over SMB Konfigurationen unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Konfiguration der Verzeichnistiefe des Schattenkopie-Verzeichnisses*
+
Durch die Konfiguration dieser Option kann die maximale Verzeichnistiefe festgelegt werden, für die beim Verwenden der Schattenkopiefunktion Schattenkopien erstellt werden.

+
Diese Option wird nur auf SVMs und nur für Hyper-V over SMB Konfigurationen unterstützt. Die Option ist auf SVMs standardmäßig aktiviert.

* *Aktivieren oder Deaktivieren der multidomain Suchfunktionen für die Namenszuordnung*
+
Wenn diese Option aktiviert ist und ein UNIX-Benutzer mithilfe eines Platzhalters (*) im Domänenteil des Windows-Benutzernamens einem Windows-Domänenbenutzer zugeordnet wird (z. B. *\joe), sucht ONTAP nach dem angegebenen Benutzer in allen Domänen mit bidirektionalen Vertrauensstellungen zur Stammdomäne. Die Stammdomäne ist die Domäne, die das Computerkonto des SMB-Servers enthält.

+
Alternativ zur Suche in allen bidirektional vertrauenswürdigen Domänen kann eine Liste bevorzugter vertrauenswürdiger Domänen konfiguriert werden. Wenn diese Option aktiviert ist und eine bevorzugte Liste konfiguriert wurde, wird die bevorzugte Liste für multidomain Namenszuordnungs-Suchen verwendet.

+
Standardmäßig sind Suchen nach multidomain Namenszuordnungen aktiviert.

* *Konfigurieren der Sektorgröße des Dateisystems*
+
Durch die Konfiguration dieser Option können Sie die Sektorgröße des Dateisystems in Bytes festlegen, die ONTAP an SMB-Clients meldet. Es gibt zwei gültige Werte:  `4096` und  `512`. Der Standardwert ist `4096`. Dieser Wert muss möglicherweise auf  `512` gesetzt werden, wenn die Windows-Anwendung nur eine Sektorgröße von 512 Byte unterstützt.

* *Dynamische Zugriffskontrolle aktivieren oder deaktivieren*
+
Durch Aktivieren dieser Option ist es möglich, Objekte auf dem SMB-Server mithilfe der Dynamic Access Control (DAC) zu schützen, einschließlich der Verwendung von Auditing zur Vorbereitung zentraler Zugriffsrichtlinien und der Verwendung von Group Policy Objects zur Implementierung zentraler Zugriffsrichtlinien. Die Option ist standardmäßig deaktiviert.

+
Diese Option wird nur auf SVMs unterstützt.

* *Zugriffsbeschränkungen für nicht authentifizierte Sitzungen (anonyme Sitzungen einschränken)*
+
Durch die Einstellung dieser Option werden die Zugriffsbeschränkungen für nicht authentifizierte Sitzungen festgelegt. Die Beschränkungen gelten für anonyme Benutzer. Standardmäßig gibt es keine Zugriffsbeschränkungen für anonyme Benutzer.

* *Aktivieren oder Deaktivieren der Darstellung von NTFS-ACLs auf Volumes mit effektiver UNIX-Sicherheit (UNIX-Sicherheitsstil-Volumes oder gemischte Sicherheitsstil-Volumes mit effektiver UNIX-Sicherheit)*
+
Das Aktivieren oder Deaktivieren dieser Option bestimmt, wie die Dateisicherheit von Dateien und Ordnern mit UNIX-Sicherheit SMB-Clients präsentiert wird. Wenn aktiviert, präsentiert ONTAP Dateien und Ordner in Volumes mit UNIX-Sicherheit für SMB-Clients als NTFS-Dateisicherheit mit NTFS-ACLs. Wenn deaktiviert, präsentiert ONTAP Volumes mit UNIX-Sicherheit als FAT-Volumes ohne Dateisicherheit. Standardmäßig werden Volumes als NTFS-Dateisicherheit mit NTFS-ACLs präsentiert.

* *Aktivieren oder Deaktivieren der SMB Fake Open Funktion*
+
Durch Aktivierung dieser Funktion wird die Leistung von SMB 2.x und SMB 3.0 verbessert, indem die Art und Weise optimiert wird, wie ONTAP beim Abfragen von Attributinformationen für Dateien und Verzeichnisse Öffnungs- und Schließanforderungen stellt. Die SMB Fake Open Funktionalität ist standardmäßig aktiviert. Diese Option ist nur für Verbindungen mit SMB 2.x oder höher nützlich.

* *Aktivieren oder Deaktivieren der UNIX-Erweiterungen*
+
Durch Aktivieren dieser Option werden UNIX-Erweiterungen auf einem SMB-Server aktiviert. UNIX-Erweiterungen ermöglichen die Anzeige von POSIX/UNIX-Sicherheit über das SMB-Protokoll. Standardmäßig ist diese Option deaktiviert.

+
Wenn Sie in Ihrer Umgebung UNIX-basierte SMB-Clients wie Mac OSX Clients haben, empfiehlt sich die Aktivierung von UNIX-Erweiterungen. Die Aktivierung von UNIX-Erweiterungen ermöglicht es dem SMB-Server, POSIX/UNIX-Sicherheitsinformationen über SMB an den UNIX-basierten Client zu übertragen, der diese Informationen dann in POSIX/UNIX-Sicherheit übersetzt.

* *Aktivieren oder Deaktivieren der Unterstützung für Kurznamensuchen*
+
Durch Aktivieren dieser Option kann der SMB-Server auch nach Kurznamen suchen. Eine Suchanfrage mit aktivierter Option versucht, sowohl 8.3-Dateinamen als auch lange Dateinamen zu finden. Der Standardwert für diesen Parameter ist  `false`.

* *Aktivieren oder Deaktivieren der Unterstützung für die automatische Ankündigung von DFS-Funktionen*
+
Durch Aktivieren oder Deaktivieren dieser Option wird festgelegt, ob SMB-Server DFS-Funktionen automatisch gegenüber SMB 2.x- und SMB 3.0-Clients bekanntgeben, die eine Verbindung zu Freigaben herstellen. ONTAP verwendet DFS-Verweise bei der Implementierung von Symbol-Links für den SMB-Zugriff. Ist die Option aktiviert, gibt der SMB-Server die DFS-Funktionen immer bekannt, unabhängig davon, ob der Zugriff auf Symbol-Links aktiviert ist. Ist die Option deaktiviert, gibt der SMB-Server die DFS-Funktionen nur dann bekannt, wenn Clients eine Verbindung zu Freigaben herstellen, bei denen der Zugriff auf Symbol-Links aktiviert ist.

* *Konfiguration der maximalen Anzahl von SMB-Credits*
+
Ab ONTAP 9.4 ermöglicht die Konfiguration der  `-max-credits` Option, die Anzahl der Credits zu begrenzen, die auf einer SMB-Verbindung vergeben werden, wenn Clients und Server SMB Version 2 oder höher verwenden. Der Standardwert ist 128.

* *Aktivieren oder Deaktivieren der Unterstützung für SMB Multichannel*
+
Durch Aktivierung der  `-is-multichannel-enabled` Option in ONTAP 9.4 und späteren Versionen kann der SMB-Server mehrere Verbindungen für eine einzelne SMB-Sitzung herstellen, wenn geeignete NICs im Cluster und auf den Clients bereitgestellt sind. Dadurch werden Durchsatz und Fehlertoleranz verbessert. Der Standardwert für diesen Parameter ist  `false`.

+
Wenn SMB Multichannel aktiviert ist, können auch die folgenden Parameter angegeben werden:

+
** Die maximal zulässige Anzahl von Verbindungen pro Multichannel-Sitzung. Der Standardwert für diesen Parameter ist 32.
** Die maximale Anzahl der pro Multichannel-Sitzung angekündigten Netzwerkschnittstellen. Der Standardwert für diesen Parameter ist 256.
