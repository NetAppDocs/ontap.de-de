= Unterstützung für Offline-Dateien auf ONTAP SMB-Freigaben mithilfe der Computer Management MMC konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie Benutzern erlauben möchten, Dateien lokal für die Offline-Nutzung zwischenzuspeichern, kann die Unterstützung für Offline-Dateien mithilfe der Computer Management MMC (Microsoft Management Console) konfiguriert werden.

.Schritte
. Um die MMC auf Ihrem Windows Server zu öffnen, im Windows Explorer mit der rechten Maustaste auf das Symbol für den lokalen Computer klicken und dann *Verwalten* auswählen.
. Im linken Bereich ist *Computerverwaltung* auszuwählen.
. *Aktion* > *Mit einem anderen Computer verbinden* auswählen.
+
Das Dialogfeld „Computer auswählen“ wird angezeigt.

. Den Namen des CIFS-Server eingeben oder auf *Durchsuchen* klicken, um den CIFS-Server zu suchen.
+
Wenn der Name des CIFS-Servers mit dem Hostnamen der Storage Virtual Machine (SVM) übereinstimmt, ist der SVM-Name einzugeben. Wenn der Name des CIFS-Servers vom SVM-Hostnamen abweicht, ist der Name des CIFS-Servers einzugeben.

. Klicken Sie auf *OK*.
. In der Konsolenstruktur *System Tools* > *Freigegebene Ordner* auswählen.
. *Teilen* auswählen.
. Im Ergebnisbereich mit der rechten Maustaste auf die Freigabe klicken.
. *Eigenschaften* auswählen.
+
Die Eigenschaften der von Ihnen ausgewählten Freigabe werden angezeigt.

. Im Tab *Allgemein* auf *Offline-Einstellungen* klicken.
+
Das Dialogfeld „Offline-Einstellungen“ wird angezeigt.

. Die Optionen für die Offline-Verfügbarkeit werden entsprechend konfiguriert.
. Klicken Sie auf *OK*.
