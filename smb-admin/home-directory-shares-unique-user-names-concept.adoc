= Informationen zu den einzigartigen ONTAP SMB-Benutzernamenanforderungen für Home-Verzeichnis-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Achten Sie darauf, beim Erstellen von Home-Verzeichnisfreigaben mithilfe der  `%w` (Windows-Benutzername) oder  `%u` (UNIX-Benutzername) Variablen zur dynamischen Generierung von Freigaben eindeutige Benutzernamen zu vergeben. Der Freigabename wird Ihrem Benutzernamen zugeordnet.

Zwei Probleme können auftreten, wenn der Name einer statischen Freigabe und der Benutzername identisch sind:

* Wenn der Benutzer die Freigaben in einem Cluster mit dem  `net view` Befehl auflistet, werden zwei Freigaben mit dem gleichen Benutzernamen angezeigt.
* Wenn sich der Benutzer mit diesem Freigabenamen verbindet, wird er immer mit der statischen Freigabe verbunden und kann nicht auf die Home-Verzeichnis-Freigabe mit demselben Namen zugreifen.


Beispielsweise gibt es eine Freigabe mit dem Namen „administrator“ und Sie haben einen Windows-Benutzernamen „administrator“. Wenn eine Home-Verzeichnis-Freigabe erstellt und eine Verbindung zu dieser Freigabe hergestellt wird, erfolgt die Verbindung mit der statischen Freigabe „administrator“, nicht mit der Home-Verzeichnis-Freigabe „administrator“.

Das Problem mit doppelten Freigabenamen kann durch einen der folgenden Schritte behoben werden:

* Die statische Freigabe wird umbenannt, sodass sie nicht mehr mit der Freigabe des Home-Verzeichnisses des Benutzers in Konflikt steht.
* Dem Benutzer einen neuen Benutzernamen zuweisen, damit dieser nicht mehr mit dem statischen Freigabenamen in Konflikt steht.
* Erstellen einer CIFS-Home-Verzeichnisfreigabe mit einem statischen Namen wie "`home`", anstatt den  `%w` Parameter zu verwenden, um Konflikte mit den Freigabenamen zu vermeiden.
