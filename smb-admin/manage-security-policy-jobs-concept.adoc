= Informationen zur Verwaltung von ONTAP SMB-Sicherheitsrichtlinienjobs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ein Sicherheitsrichtlinienauftrag existiert, kann unter bestimmten Umständen diese Sicherheitsrichtlinie oder die der Richtlinie zugewiesenen Aufgaben nicht geändert werden. Es ist wichtig zu verstehen, unter welchen Bedingungen Sicherheitsrichtlinien geändert werden können oder nicht, damit etwaige Änderungsversuche erfolgreich sind. Zu den Änderungen an der Richtlinie zählen das Hinzufügen, Entfernen oder Ändern von Aufgaben, die der Richtlinie zugewiesen sind, sowie das Löschen oder Ändern der Richtlinie.

Sie können eine Sicherheitsrichtlinie oder eine dieser Richtlinie zugewiesene Aufgabe nicht ändern, wenn ein Auftrag für diese Richtlinie existiert und sich dieser Auftrag in einem der folgenden Zustände befindet:

* Der Auftrag wird ausgeführt oder ist in Bearbeitung.
* Der Auftrag ist pausiert.
* Der Auftrag wird fortgesetzt und befindet sich im Status „Wird ausgeführt“.
* Wenn der Job auf das Failover zu einem anderen Knoten wartet.


Unter den folgenden Umständen ist es möglich, eine Sicherheitsrichtlinie oder eine dieser Richtlinie zugewiesene Aufgabe erfolgreich zu ändern, sofern ein Auftrag für eine Sicherheitsrichtlinie existiert:

* Der Policy Job wurde gestoppt.
* Der Policy-Job wurde erfolgreich abgeschlossen.
