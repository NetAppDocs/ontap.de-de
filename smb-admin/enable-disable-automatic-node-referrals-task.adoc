= Automatische Knotenweiterleitungen für ONTAP SMB aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die automatische Knotenweiterleitung für SMB kann aktiviert werden, um die Zugriffsleistung von SMB-Clients zu erhöhen. Die automatische Knotenweiterleitung kann deaktiviert werden, wenn keine Weiterleitungen von ONTAP an SMB-Clients erfolgen sollen.

.Bevor Sie beginnen
Auf der Storage Virtual Machine (SVM) muss ein CIFS-Server konfiguriert und ausgeführt werden.

.Über diese Aufgabe
Die automatische Knotenweiterleitungsfunktion von SMB ist standardmäßig deaktiviert. Diese Funktion kann bei Bedarf für jede SVM aktiviert oder deaktiviert werden.

Diese Option ist auf der Stufe „Erweitertes Berechtigungsniveau“ verfügbar.

.Schritte
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: `set -privilege advanced`
. SMB automatische Knotenweiterleitungen nach Bedarf aktivieren oder deaktivieren:
+
|===
| Wenn Sie möchten, dass SMB automatische Knotenverweise... | Führen Sie den folgenden Befehl aus... 


 a| 
Aktiviert
 a| 
`vserver cifs options modify -vserver vserver_name -is-referral-enabled true`



 a| 
Deaktiviert
 a| 
`vserver cifs options modify -vserver vserver_name -is-referral-enabled false`

|===
+
Die Optionseinstellung wird für neue SMB-Sitzungen wirksam. Clients mit bestehender Verbindung können die Knotenweiterleitung nur verwenden, wenn ihre bestehende Cache-Zeitüberschreitung abgelaufen ist.

. Zur Administratorberechtigungsstufe wechseln: `set -privilege admin`
