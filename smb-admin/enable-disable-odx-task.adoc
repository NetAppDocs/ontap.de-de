= ODX auf ONTAP SMB-Servern aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können ODX auf Storage Virtual Machines (SVMs) aktivieren oder deaktivieren. Standardmäßig ist die Unterstützung für ODX Copy Offload aktiviert, wenn auch SMB 3.0 aktiviert ist.

.Bevor Sie beginnen
SMB 3.0 muss aktiviert sein.

.Über diese Aufgabe
Wenn Sie SMB 3.0 deaktivieren, deaktiviert ONTAP auch SMB ODX. Wenn Sie SMB 3.0 wieder aktivieren, muss SMB ODX manuell wieder aktiviert werden.

.Schritte
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: `set -privilege advanced`
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn ODX Copy Offload gewünscht wird... | Geben Sie den Befehl ein... 


 a| 
Aktiviert
 a| 
`vserver cifs options modify -vserver vserver_name -copy-offload-enabled true`



 a| 
Deaktiviert
 a| 
`vserver cifs options modify -vserver vserver_name -copy-offload-enabled false`

|===
. Zurück zur Administratorrechteebene: `set -privilege admin`


.Beispiel
Das folgende Beispiel aktiviert ODX Copy Offload auf SVM vs1:

[listing]
----
cluster1::> set -privilege advanced
Warning: These advanced commands are potentially dangerous; use them
only when directed to do so by technical support personnel.
Do you wish to continue? (y or n): y

cluster1::*> vserver cifs options modify -vserver vs1 -copy-offload-enabled true

cluster1::*> set -privilege admin
----
