= Informationen zu den Anforderungen an die Groß-/Kleinschreibung von ONTAP SMB-Verzeichnissen beim Erstellen von Freigaben in einer Multiprotokollumgebung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie Freigaben in einer SVM erstellen, in der das 8.3-Namensschema verwendet wird, um zwischen Verzeichnisnamen zu unterscheiden, bei denen es nur Unterschiede in der Groß-/Kleinschreibung gibt, muss der 8.3-Name im Freigabepfad verwendet werden, damit der Client mit dem gewünschten Verzeichnispfad verbunden wird.

Im folgenden Beispiel wurden auf einem Linux-Client zwei Verzeichnisse mit den Namen „testdir“ und „TESTDIR“ erstellt. Der Verbindungspfad des Volumes, das die Verzeichnisse enthält, ist  `/home`. Die erste Ausgabe stammt von einem Linux-Client, die zweite Ausgabe von einem SMB-Client.

[listing]
----
 ls -l
drwxrwxr-x 2 user1 group1   4096 Apr 17 11:23 testdir
drwxrwxr-x 2 user1 group1   4096 Apr 17 11:24 TESTDIR
----
[listing]
----
dir

 Directory of Z:\

04/17/2015  11:23 AM    <DIR>          testdir
04/17/2015  11:24 AM    <DIR>          TESTDI~1
----
Wenn Sie eine Freigabe für das zweite Verzeichnis erstellen, muss im Freigabepfad der 8.3-Name verwendet werden. In diesem Beispiel ist der Freigabepfad zum ersten Verzeichnis  `/home/testdir` und der Freigabepfad zum zweiten Verzeichnis ist  `/home/TESTDI~1`.
