= Informationen zur Unterstützung von IPv6 mit ONTAP SMB-Zugriff und CIFS-Services
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie IPv6 auf Ihrem CIFS-Server verwenden möchten, ist es wichtig zu wissen, wie ONTAP IPv6 für den SMB-Zugriff und die Netzwerkkommunikation für CIFS-Dienste unterstützt.



[[windows-client-and-server-support]]
== Windows Client- und Serverunterstützung

ONTAP bietet Unterstützung für Windows Server und Clients, die IPv6 unterstützen. Im Folgenden wird die IPv6-Unterstützung für Microsoft Windows Clients und Server beschrieben:

* Windows 7, Windows 8, Windows Server 2008, Windows Server 2012 und spätere Versionen unterstützen IPv6 sowohl für SMB-Dateifreigabe als auch für Active Directory-Dienste, einschließlich DNS, LDAP, CLDAP und Kerberos-Dienste.
+
Sind IPv6-Adressen konfiguriert, verwenden Windows 7 und Windows Server 2008 sowie spätere Versionen IPv6 standardmäßig für Active Directory Services. Sowohl NTLM- als auch Kerberos-Authentifizierung über IPv6-Verbindungen wird unterstützt.

+
Alle von ONTAP unterstützten Windows-Clients können über IPv6-Adressen eine Verbindung zu SMB-Freigaben herstellen.



Die aktuellsten Informationen darüber, welche Windows-Clients ONTAP unterstützt, sind unter link:https://mysupport.netapp.com/matrix["Interoperabilitätsmatrix"^] zu finden.

[NOTE]
====
NT-Domänen werden für IPv6 nicht unterstützt.

====


[[additional-cifs-services-support]]
== Zusätzliche CIFS-Services-Unterstützung

Zusätzlich zur IPv6-Unterstützung für SMB-Dateifreigaben und Active Directory-Dienste bietet ONTAP IPv6-Unterstützung für Folgendes:

* Clientseitige Dienste, einschließlich Offline-Ordner, Roaming-Profile, Ordnerumleitung und Previous Versions
* Serverseitige Dienste, einschließlich dynamischer Home-Verzeichnisse (Home Directory-Funktion), Symlinks und Widelinks, BranchCache, ODX Copy Offload, automatische Node-Referrals und Previous Versions
* Dateizugriffsverwaltungsdienste, einschließlich der Verwendung von lokalen Windows-Benutzern und -Gruppen für Zugriffskontrolle und Rechteverwaltung, der Festlegung von Dateiberechtigungen und Überwachungsrichtlinien mit der CLI, Sicherheitsverfolgung, Verwaltung von Dateisperren und Überwachung der SMB-Aktivität
* NAS Multiprotokoll-Auditierung
* FPolicy
* Kontinuierlich verfügbare Freigaben, Witness-Protokoll und Remote VSS (verwendet mit Hyper-V über SMB Konfigurationen)




[[name-service-and-authentication-service-support]]
== Unterstützung für Namensdienste und Authentifizierungsdienste

Die Kommunikation mit den folgenden Namensdiensten wird mit IPv6 unterstützt:

* Domänencontroller
* DNS-Server
* LDAP-Server
* KDC-Server
* NIS-Server
