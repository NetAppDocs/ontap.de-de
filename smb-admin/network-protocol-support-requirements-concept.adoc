= Informationen zu den Anforderungen an die Unterstützung des ONTAP SMB Netzwerkprotokolls
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es ist wichtig, die Anforderungen an das Netzwerkprotokoll für die Implementierung von ONTAP BranchCache zu kennen.

Die ONTAP BranchCache Funktion kann über IPv4- und IPv6-Netzwerke mit SMB 2.1 oder höher implementiert werden.

Alle CIFS-Server und Rechner in Zweigstellen, die an der BranchCache-Implementierung beteiligt sind, müssen das SMB 2.1 oder eine spätere Protokollversion aktiviert haben. SMB 2.1 verfügt über Protokollerweiterungen, die es einem Client ermöglichen, an einer BranchCache-Umgebung teilzunehmen. Dies ist die minimale SMB-Protokollversion, die BranchCache unterstützt. SMB 2.1 unterstützt BranchCache Version 1.

Wenn BranchCache Version 2 verwendet werden soll, ist SMB 3.0 die minimal unterstützte Version. Alle CIFS-Server und Rechner in Zweigstellen, die an einer BranchCache 2-Implementierung beteiligt sind, müssen SMB 3.0 oder höher aktiviert haben.

Wenn Sie über Außenstellen verfügen, in denen einige der Clients nur SMB 2.1 und andere der Clients SMB 3.0 unterstützen, kann auf dem CIFS-Server eine BranchCache Konfiguration implementiert werden, die Caching-Unterstützung sowohl für BranchCache 1 als auch für BranchCache 2 bietet.

[NOTE]
====
Auch wenn die Microsoft BranchCache Funktion sowohl die Verwendung der HTTP/HTTPS- als auch der SMB-Protokolle als Dateizugriffsprotokolle unterstützt, unterstützt ONTAP BranchCache nur die Verwendung von SMB.
