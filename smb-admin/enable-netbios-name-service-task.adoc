= Den ONTAP SMB NetBios-Namensdienst aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9 ist der NetBios-Namensdienst (NBNS, manchmal auch Windows Internet Name Service oder WINS genannt) standardmäßig deaktiviert. Früher sendeten CIFS-Server Storage Virtual Machines (SVMs) Namensregistrierungs-Broadcasts, unabhängig davon, ob WINS im Netzwerk aktiviert war. Um solche Broadcasts auf Konfigurationen zu beschränken, in denen NBNS erforderlich ist, muss NBNS für neue CIFS-Server explizit aktiviert werden.

.Bevor Sie beginnen
* Wenn Sie bereits NBNS nutzen und auf ONTAP 9 aktualisieren, ist es nicht notwendig, diese Aufgabe durchzuführen. NBNS funktioniert weiterhin wie zuvor.
* NBNS ist über UDP (Port 137) aktiviert.
* NBNS über IPv6 wird nicht unterstützt.


.Schritte
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt.
+
[listing]
----
set -privilege advanced
----
. NBNS auf einem CIFS-Server aktivieren.
+
[listing]
----
vserver cifs options  modify -vserver <vserver name> -is-nbns-enabled true
----
. Zurück zur Administrator-Berechtigungsstufe.
+
[listing]
----
set -privilege admin
----
