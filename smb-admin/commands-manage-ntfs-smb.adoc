= ONTAP Befehle zur Verwaltung von NTFS- und SMB-Sicherheitsrichtlinien und Zugriffskontrolleinträgen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit ONTAP-Befehlen können Sie NTFS-Sicherheitsdeskriptoren und Zugriffssteuerungseinträge (ACEs) in DACLs und SACLs erstellen, ändern, löschen und Informationen dazu anzeigen. Sie können auch ONTAP-Befehle verwenden, um SMB-Sicherheitsrichtlinien, Richtlinienaufgaben und Richtlinienaufträge zu verwalten.



[[ontap-commands-for-managing-ntfs-security-descriptors-on-smb-servers]]
== ONTAP Befehle zur Verwaltung von NTFS-Sicherheitsdeskriptoren auf SMB-Servern

Für die Verwaltung von Sicherheitsdeskriptoren stehen spezielle ONTAP Befehle zur Verfügung. Sie können Sicherheitsdeskriptoren erstellen, ändern, löschen und Informationen darüber anzeigen.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
NTFS-Sicherheitsdeskriptoren erstellen
 a| 
`vserver security file-directory ntfs create`



 a| 
Vorhandene NTFS-Sicherheitsdeskriptoren ändern
 a| 
`vserver security file-directory ntfs modify`



 a| 
Informationen zu vorhandenen NTFS-Sicherheitsdeskriptoren anzeigen
 a| 
`vserver security file-directory ntfs show`



 a| 
NTFS-Sicherheitsdeskriptoren löschen
 a| 
`vserver security file-directory ntfs delete`

|===
Weitere Informationen zu  `vserver security file-directory ntfs` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs["ONTAP-Befehlsreferenz"^].



[[ontap-commands-for-managing-ntfs-dacl-access-control-entries-on-smb-servers]]
== ONTAP Befehle zur Verwaltung von NTFS DACL-Zugriffskontrolleinträgen auf SMB-Servern

Für die Verwaltung von DACL-Zugriffskontrolleinträgen (ACEs) stehen spezielle ONTAP Befehle zur Verfügung. Es ist möglich, ACEs jederzeit zu NTFS DACLs hinzuzufügen. Auch die Verwaltung bestehender NTFS DACLs durch Ändern, Löschen und Anzeigen von Informationen zu ACEs in DACLs ist möglich.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
ACEs erstellen und zu NTFS-DACLs hinzufügen
 a| 
`vserver security file-directory ntfs dacl add`



 a| 
Vorhandene ACEs in NTFS-DACLs ändern
 a| 
`vserver security file-directory ntfs dacl modify`



 a| 
Informationen zu vorhandenen ACEs in NTFS-DACLs
 a| 
`vserver security file-directory ntfs dacl show`



 a| 
Vorhandene ACEs aus NTFS-DACLs entfernen
 a| 
`vserver security file-directory ntfs dacl remove`

|===
Weitere Informationen zu  `vserver security file-directory ntfs dacl` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs+dacl["ONTAP-Befehlsreferenz"^].

 a| 
Sicherheitsrichtlinien löschen
 a| 
`vserver security file-directory policy delete`

|===

[[ontap-commands-for-managing-smb-security-policy-jobs]]
== ONTAP Befehle zur Verwaltung von SMB Security Policy Jobs

Es gibt ONTAP Befehle zum Anhalten, Fortsetzen, Stoppen und Anzeigen von Informationen über Sicherheitsrichtlinienaufträge.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Sicherheitsrichtlinien-Jobs pausieren
 a| 
`vserver security file-directory job pause ‑vserver vserver_name -id integer`



 a| 
Sicherheitsrichtlinien-Jobs fortsetzen
 a| 
`vserver security file-directory job resume ‑vserver vserver_name -id integer`



 a| 
Informationen zu Sicherheitsrichtlinien-Jobs anzeigen
 a| 
`vserver security file-directory job show ‑vserver vserver_name`Die Job-ID eines Jobs lässt sich mit diesem Befehl ermitteln.



 a| 
Sicherheitsrichtlinien-Jobs stoppen
 a| 
`vserver security file-directory job stop ‑vserver vserver_name -id integer`

|===
Weitere Informationen zu  `vserver security file-directory job` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+job["ONTAP-Befehlsreferenz"^].
