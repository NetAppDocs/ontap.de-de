= ONTAP Befehle zur Konfiguration der Offline-SMB-Dateiunterstützung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Unterstützung für Offline-Dateien kann mithilfe der ONTAP CLI konfiguriert werden, indem eine der vier Offline-Dateieinstellungen beim Erstellen von SMB-Freigaben oder jederzeit durch Ändern bestehender SMB-Freigaben angegeben wird. Die manuelle Unterstützung für Offline-Dateien ist die Standardeinstellung.

.Über diese Aufgabe
Bei der Konfiguration der Offline-Dateiunterstützung können Sie eine der folgenden vier Offline-Dateieinstellungen auswählen:

|===
| Einstellung | Beschreibung 


 a| 
`none`
 a| 
Verhindert, dass Windows-Clients Dateien auf dieser Freigabe zwischenspeichern.



 a| 
`manual`
 a| 
Ermöglicht Benutzern auf Windows-Clients, Dateien manuell zur Zwischenspeicherung auszuwählen.



 a| 
`documents`
 a| 
Ermöglicht Windows-Clients, Benutzerdokumente zwischenzuspeichern, die der Benutzer für den Offline-Zugriff verwendet.



 a| 
`programs`
 a| 
Ermöglicht Windows-Clients, Programme, die der Benutzer verwendet, für den Offline-Zugriff zwischenzuspeichern. Clients können die zwischengespeicherten Programmdateien im Offline-Modus verwenden, selbst wenn die Freigabe verfügbar ist.

|===
Sie können nur eine Offline-Dateieinstellung auswählen. Wenn eine Offline-Dateieinstellung für eine bestehende SMB-Freigabe geändert wird, ersetzt die neue Einstellung die ursprüngliche. Andere bestehende SMB-Freigabe-Konfigurationseinstellungen und Freigabeeigenschaften werden nicht entfernt oder ersetzt. Sie bleiben wirksam, bis sie explizit entfernt oder geändert werden.

.Schritte
. Die entsprechende Maßnahme ausführen:
+
|===
| Wenn Sie Offline-Dateien auf ... konfigurieren möchten | Geben Sie den Befehl ein... 


 a| 
Eine neue SMB-Freigabe
 a| 
`vserver cifs share create -vserver vserver_name -share-name share_name -path path -offline-files {none|manual|documents|programs}`



 a| 
Eine bestehende SMB-Freigabe
 a| 
`vserver cifs share modify -vserver vserver_name -share-name share_name -offline-files {none|manual|documents|programs}`

|===
. Es sollte überprüft werden, ob die SMB-Freigabekonfiguration korrekt ist: `vserver cifs share show -vserver vserver_name -share-name share_name -instance`


.Beispiel
Der folgende Befehl erstellt eine SMB-Freigabe mit dem Namen "`data1`", wobei Offline-Dateien auf  `documents` eingestellt sind:

[listing]
----
cluster1::> vserver cifs share create -vserver vs1 -share-name data1 -path /data1 -comment "Offline files" -offline-files documents

cluster1::> vserver cifs share show -vserver vs1 -share-name data1 -instance

                          Vserver: vs1
                            Share: data1
         CIFS Server NetBIOS Name: VS1
                             Path: /data1
                 Share Properties: oplocks
                                   browsable
                                   changenotify
               Symlink Properties: enable
          File Mode Creation Mask: -
     Directory Mode Creation Mask: -
                    Share Comment: Offline files
                        Share ACL: Everyone / Full Control
    File Attribute Cache Lifetime: -
                      Volume Name: -
                    Offline Files: documents
    Vscan File-Operations Profile: standard
Maximum Tree Connections on Share: 4294967295
       UNIX Group for File Create: -
----
Der folgende Befehl ändert eine vorhandene SMB-Freigabe mit dem Namen "`data1`", indem die Einstellung für Offline-Dateien auf  `manual` geändert und Werte für die Datei- und Verzeichnismodus-Erstellungsmaske hinzugefügt werden:

[listing]
----
cluster1::> vserver cifs share modify -vserver vs1 -share-name data1 -offline-files manual -file-umask 644 -dir-umask 777

cluster1::> vserver cifs share show -vserver vs1 -share-name data1 -instance

                          Vserver: vs1
                            Share: data1
         CIFS Server NetBIOS Name: VS1
                             Path: /data1
                 Share Properties: oplocks
                                   browsable
                                   changenotify
               Symlink Properties: enable
          File Mode Creation Mask: 644
     Directory Mode Creation Mask: 777
                    Share Comment: Offline files
                        Share ACL: Everyone / Full Control
    File Attribute Cache Lifetime: -
                      Volume Name: -
                    Offline Files: manual
    Vscan File-Operations Profile: standard
Maximum Tree Connections on Share: 4294967295
       UNIX Group for File Create: -
----
