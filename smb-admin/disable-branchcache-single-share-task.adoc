= Deaktivieren Sie BranchCache auf SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie keine BranchCache-Caching-Dienste für bestimmte SMB-Freigaben bereitstellen möchten, aber möglicherweise später Caching-Dienste auf diesen Freigaben anbieten wollen, kann BranchCache für jede Freigabe einzeln deaktiviert werden. Wenn BranchCache so konfiguriert ist, dass Caching für alle Freigaben angeboten wird, Sie jedoch alle Caching-Dienste vorübergehend deaktivieren möchten, kann die BranchCache-Konfiguration so geändert werden, dass das automatische Caching auf allen Freigaben gestoppt wird.

Wenn BranchCache auf einer SMB-Freigabe nach der ersten Aktivierung anschließend deaktiviert wird, stellt ONTAP das Senden von Metadaten an den anfragenden Client ein. Ein Client, der Daten benötigt, ruft diese direkt vom Content-Server (CIFS-Server auf der Storage Virtual Machine (SVM)) ab.



[[disable-branchcache-on-a-single-ontap-smb-share]]
== Deaktivieren von BranchCache auf einer einzelnen ONTAP SMB-Freigabe

Wenn für bestimmte Freigaben, die zuvor zwischengespeicherte Inhalte angeboten haben, keine Caching-Dienste mehr angeboten werden sollen, kann BranchCache auf einer bestehenden SMB-Freigabe deaktiviert werden.

.Schritt
. Führen Sie den folgenden Befehl aus:
+
[source, cli]
----
vserver cifs share properties remove -vserver <SVM_name> -share-name <share_name> -share-properties branchcache
----
+
Die BranchCache Share-Eigenschaft wird entfernt. Andere angewendete Share-Eigenschaften bleiben bestehen.



.Beispiel
Der folgende Befehl deaktiviert BranchCache auf einer bestehenden SMB-Freigabe mit dem Namen "`data2`":

[listing]
----
cluster1::> vserver cifs share show -vserver vs1 -share-name data2

                      Vserver: vs1
                        Share: data2
     CIFS Server NetBIOS Name: VS1
                         Path: /data2
             Share Properties: oplocks
                               browsable
                               changenotify
                               attributecache
                               branchcache
           Symlink Properties: -
      File Mode Creation Mask: -
 Directory Mode Creation Mask: -
                Share Comment: -
                    Share ACL: Everyone / Full Control
File Attribute Cache Lifetime: 10s
                  Volume Name: -
                Offline Files: manual
Vscan File-Operations Profile: standard

cluster1::> vserver cifs share properties remove -vserver vs1 -share-name data2 -share-properties branchcache

cluster1::> vserver cifs share show -vserver vs1 -share-name data2

                      Vserver: vs1
                        Share: data2
     CIFS Server NetBIOS Name: VS1
                         Path: /data2
             Share Properties: oplocks
                               browsable
                               changenotify
                               attributecache
           Symlink Properties: -
      File Mode Creation Mask: -
 Directory Mode Creation Mask: -
                Share Comment: -
                    Share ACL: Everyone / Full Control
File Attribute Cache Lifetime: 10s
                  Volume Name: -
                Offline Files: manual
Vscan File-Operations Profile: standard
----


[[stop-automatic-caching-on-all-ontap-smb-shares]]
== Automatisches Caching für alle ONTAP SMB-Freigaben deaktivieren

Wenn Ihre BranchCache Konfiguration das Caching auf allen SMB-Freigaben auf jeder Storage Virtual Machine (SVM) automatisch aktiviert, kann die BranchCache Konfiguration so angepasst werden, dass das automatische Caching von Inhalten für alle SMB-Freigaben beendet wird.

.Über diese Aufgabe
Um das automatische Caching auf allen SMB-Freigaben zu beenden, wird der BranchCache Betriebsmodus auf Caching pro Freigabe geändert.

.Schritte
. Konfigurieren Sie BranchCache, um die automatische Zwischenspeicherung auf allen SMB-Freigaben zu stoppen:
+
[source, cli]
----
vserver cifs branchcache modify -vserver <SVM_name> -operating-mode per-share
----
. Überprüfen Sie, ob die BranchCache-Konfiguration korrekt ist:
+
[source, cli]
----
vserver cifs branchcache show -vserver <SVM_name>
----


.Beispiel
Der folgende Befehl ändert die BranchCache-Konfiguration auf der Storage Virtual Machine (SVM, früher bekannt als Vserver) vs1, sodass das automatische Caching auf allen SMB-Freigaben gestoppt wird:

[listing]
----
cluster1::> vserver cifs branchcache modify -vserver vs1 -operating-mode per-share

cluster1::> vserver cifs branchcache show -vserver vs1

                                 Vserver: vs1
          Supported BranchCache Versions: enable_all
                      Path to Hash Store: /hash_data
          Maximum Size of the Hash Store: 20GB
Encryption Key Used to Secure the Hashes: -
        CIFS BranchCache Operating Modes: per_share
----
