= ONTAP SMB-Sitzungsinformationen anzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Informationen zu bestehenden SMB-Sitzungen anzeigen, darunter die SMB-Verbindungs- und Sitzungs-ID sowie die IP-Adresse der Workstation, die die Sitzung nutzt. Sie können Informationen zur Protokollversion der Sitzung und zum kontinuierlich verfügbaren Schutzniveau anzeigen, was dabei hilft zu erkennen, ob die Sitzung unterbrechungsfreie Vorgänge unterstützt.

.Über diese Aufgabe
Sie können Informationen zu allen Sitzungen Ihrer SVM in zusammengefasster Form anzeigen. In vielen Fällen ist die Ausgabemenge jedoch sehr groß. Welche Informationen in der Ausgabe angezeigt werden, lässt sich durch optionale Parameter anpassen:

* Der optionale  `-fields`Parameter ermöglicht die Anzeige von Ausgaben zu den ausgewählten Feldern.
+
Sie können  `-fields ?` eingeben, um festzustellen, welche Felder Sie verwenden können.

* Sie können den  `-instance` Parameter verwenden, um detaillierte Informationen über bestehende SMB-Sitzungen anzuzeigen.
* Sie können den  `-fields` Parameter oder den  `-instance` Parameter entweder allein oder in Kombination mit anderen optionalen Parametern verwenden.


.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn Sie SMB-Sitzungsinformationen anzeigen möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Für alle Sitzungen auf der SVM in zusammengefasster Form
 a| 
`vserver cifs session show -vserver vserver_name`



 a| 
Bei einer bestimmten Verbindungs-ID
 a| 
`vserver cifs session show -vserver vserver_name -connection-id integer`



 a| 
Von einer bestimmten Workstation IP-Adresse
 a| 
`vserver cifs session show -vserver vserver_name -address workstation_IP_address`



 a| 
An einer bestimmten LIF-IP-Adresse
 a| 
`vserver cifs session show -vserver vserver_name -lif-address LIF_IP_address`



 a| 
Auf einem bestimmten Knoten
 a| 
`vserver cifs session show -vserver vserver_name -node {node_name|local}`



 a| 
Von einem bestimmten Windows-Benutzer
 a| 
`vserver cifs session show -vserver vserver_name -windows-user domain_name\\user_name`



 a| 
Mit einem festgelegten Authentifizierungsmechanismus
 a| 
`vserver cifs session show -vserver vserver_name -auth-mechanism {NTLMv1|NTLMv2|Kerberos|Anonymous}`



 a| 
Mit einer bestimmten Protokollversion
 a| 
`vserver cifs session show -vserver vserver_name -protocol-version {SMB1|SMB2|SMB2_1|SMB3|SMB3_1}`

[NOTE]
====
Der permanent verfügbare Schutz und SMB Multichannel sind nur für SMB 3.0 und höhere Sitzungen verfügbar. Um deren Status für alle qualifizierenden Sitzungen anzuzeigen, sollte dieser Parameter mit dem Wert  `SMB3` oder höher angegeben werden.

====


 a| 
Mit einem festgelegten Niveau an kontinuierlich verfügbarem Schutz
 a| 
`vserver cifs session show -vserver vserver_name -continuously-available {No|Yes|Partial}`

[NOTE]
====
Wenn der Status „Dauerhaft verfügbar“  `Partial` ist, bedeutet dies, dass die Sitzung mindestens eine geöffnete, dauerhaft verfügbare Datei enthält, aber auch einige Dateien, die nicht mit dauerhaft verfügbarer Schutz geöffnet sind. Der  `vserver cifs sessions file show` Befehl kann verwendet werden, um zu bestimmen, welche Dateien in der bestehenden Sitzung nicht mit dauerhaft verfügbarer Schutz geöffnet sind.

====


 a| 
Mit einem festgelegten SMB-Signing-Sitzungsstatus
 a| 
`vserver cifs session show -vserver vserver_name -is-session-signed {true|false}`

|===


.Beispiele
Der folgende Befehl zeigt Sitzungsinformationen für die Sitzungen auf SVM vs1 an, die von einer Workstation mit der IP-Adresse 10.1.1.1 aufgebaut wurden:

[listing]
----
cluster1::> vserver cifs session show -address 10.1.1.1
Node:    node1
Vserver: vs1
Connection Session                                    Open         Idle
ID          ID      Workstation      Windows User    Files         Time
----------  ------- ---------------- ------------- ------- ------------
3151272279,
3151272280,
3151272281  1       10.1.1.1         DOMAIN\joe          2          23s
----
Der folgende Befehl zeigt detaillierte Sitzungsinformationen für Sitzungen mit kontinuierlich verfügbarem Schutz auf SVM vs1 an. Die Verbindung wurde mit dem Domänenkonto hergestellt.

[listing]
----
cluster1::> vserver cifs session show -instance -continuously-available Yes

                        Node: node1
                     Vserver: vs1
                  Session ID: 1
               Connection ID: 3151274158
Incoming Data LIF IP Address: 10.2.1.1
      Workstation IP address: 10.1.1.2
    Authentication Mechanism: Kerberos
                Windows User: DOMAIN\SERVER1$
                   UNIX User: pcuser
                 Open Shares: 1
                  Open Files: 1
                  Open Other: 0
              Connected Time: 10m 43s
                   Idle Time: 1m 19s
            Protocol Version: SMB3
      Continuously Available: Yes
           Is Session Signed: false
       User Authenticated as: domain-user
                NetBIOS Name: -
       SMB Encryption Status: Unencrypted
----
Der folgende Befehl zeigt Sitzungsinformationen zu einer Sitzung an, die SMB 3.0 und SMB Multichannel auf SVM vs1 verwendet. Im Beispiel hat sich der Benutzer von einem SMB 3.0-fähigen Client über die LIF-IP-Adresse mit dieser Freigabe verbunden; daher wurde als Authentifizierungsmechanismus standardmäßig NTLMv2 verwendet. Die Verbindung muss über Kerberos-Authentifizierung hergestellt werden, um einen dauerhaft verfügbaren Schutz zu gewährleisten.
