= Aktivieren Sie BranchCache auf bestehenden ONTAP SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können BranchCache auf einer bestehenden SMB-Freigabe aktivieren, indem Sie die  `branchcache` Freigabeeigenschaft zur bestehenden Liste der Freigabeeigenschaften hinzufügen.

.Über diese Aufgabe
* Wenn BranchCache auf der SMB-Freigabe aktiviert ist, muss die Offline-Dateikonfiguration der Freigabe auf manuelles Caching eingestellt sein.
+
Wenn die Einstellung für Offline-Dateien der vorhandenen Freigabe nicht auf manuelles Caching gesetzt ist, muss dies durch Ändern der Freigabe konfiguriert werden.

* Sie können die  `branchcache`Eigenschaft für eine Freigabe festlegen, selbst wenn BranchCache auf der Storage Virtual Machine (SVM) nicht konfiguriert und aktiviert ist.
+
Wenn Sie jedoch möchten, dass die Freigabe zwischengespeicherte Inhalte anbietet, müssen Sie BranchCache auf der SVM konfigurieren und aktivieren.

* Wenn die  `branchcache`Freigabeeigenschaft zur Freigabe hinzugefügt wird, bleiben die bestehenden Freigabeeinstellungen und Freigabeeigenschaften erhalten.
+
Die BranchCache Share-Eigenschaft wird der bestehenden Liste der Share-Eigenschaften hinzugefügt. Weitere Informationen zu  `vserver cifs share properties add` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-cifs-share-properties-add.html["ONTAP-Befehlsreferenz"^].



.Schritte
. Gegebenenfalls die Einstellung für die Offline-Dateifreigabe für manuelle Zwischenspeicherung konfigurieren:
+
.. Welche Einstellung für die Offline-Dateifreigabe vorliegt, lässt sich mit dem  `vserver cifs share show` Befehl feststellen.
.. Wenn die Einstellung für die Offline-Dateifreigabe nicht auf manuell gesetzt ist, sollte sie auf den erforderlichen Wert geändert werden: `vserver cifs share modify -vserver vserver_name -share-name share_name -offline-files manual`


. BranchCache auf einer bestehenden SMB-Freigabe aktivieren: `vserver cifs share properties add -vserver vserver_name -share-name share_name -share-properties branchcache`
. Es sollte überprüft werden, ob die BranchCache-Freigabeeigenschaft für die SMB-Freigabe festgelegt ist: `vserver cifs share show -vserver vserver_name -share-name share_name`


.Beispiel
Der folgende Befehl aktiviert BranchCache auf einer bestehenden SMB-Freigabe mit dem Namen "`data2`" und dem Pfad  `/data2` auf SVM vs1:

[listing]
----
cluster1::> vserver cifs share show -vserver vs1 -share-name data2

                      Vserver: vs1
                        Share: data2
     CIFS Server NetBIOS Name: VS1
                         Path: /data2
             Share Properties: oplocks
                               browsable
                               changenotify
                               showsnapshot
           Symlink Properties: -
      File Mode Creation Mask: -
 Directory Mode Creation Mask: -
                Share Comment: -
                    Share ACL: Everyone / Full Control
File Attribute Cache Lifetime: 10s
                  Volume Name: -
                Offline Files: manual
Vscan File-Operations Profile: standard

cluster1::> vserver cifs share properties add  -vserver vs1 -share-name data2 -share-properties branchcache

cluster1::> vserver cifs share show -vserver vs1 -share-name data2

                      Vserver: vs1
                        Share: data2
     CIFS Server NetBIOS Name: VS1
                         Path: /data2
             Share Properties: oplocks
                               browsable
                               showsnapshot
                               changenotify
                               branchcache
           Symlink Properties: -
      File Mode Creation Mask: -
 Directory Mode Creation Mask: -
                Share Comment: -
                    Share ACL: Everyone / Full Control
File Attribute Cache Lifetime: 10s
                  Volume Name: -
                Offline Files: manual
Vscan File-Operations Profile: standard
----
