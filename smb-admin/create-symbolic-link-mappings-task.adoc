= Symbol-Link-Zuordnungen für ONTAP SMB-Freigaben erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Zuordnungen von UNIX-Symbol-Links für SMB-Freigaben erstellen. Es ist möglich, entweder einen relativen Symbol-Link zu erstellen, der auf die Datei oder den Ordner relativ zu seinem übergeordneten Ordner verweist, oder einen absoluten Symbol-Link, der auf die Datei oder den Ordner über einen absoluten Pfad verweist.

.Über diese Aufgabe
Widelinks sind von Mac OS X Clients aus nicht zugänglich, wenn Sie SMB 2.x verwenden. Wenn ein Benutzer versucht, über Widelinks von einem Mac OS X Client eine Verbindung zu einer Freigabe herzustellen, schlägt der Versuch fehl. Widelinks können jedoch mit Mac OS X Clients verwendet werden, wenn Sie SMB 1 verwenden.

.Schritte
. Symbol-Link-Zuordnungen für SMB-Freigaben erstellen: `vserver cifs symlink create -vserver virtual_server_name -unix-path path -share-name share_name -cifs-path path [-cifs-server server_name] [-locality {local|free|widelink}] [-home-directory {true|false}]`
+
`-vserver` `virtual_server_name`Gibt den Namen der Storage Virtual Machine (SVM) an.

+
`-unix-path` `path` gibt den UNIX-Pfad an. Der UNIX-Pfad muss mit einem Schrägstrich (`/` beginnen und mit einem Schrägstrich (`/` enden.

+
`-share-name` `share_name` Gibt den Namen der zuzuordnenden SMB-Freigabe an.

+
`-cifs-path` `path` gibt den CIFS-Pfad an. Der CIFS-Pfad muss mit einem Schrägstrich (`/` beginnen und mit einem Schrägstrich (`/` enden.

+
`-cifs-server` `server_name`Gibt den Namen des CIFS-Servers an. Der Name des CIFS-Servers kann als DNS-Name (zum Beispiel mynetwork.cifs.server.com), IP-Adresse oder NetBIOS-Name angegeben werden. Der NetBIOS-Name kann mit dem  `vserver cifs show` Befehl ermittelt werden. Wenn dieser optionale Parameter nicht angegeben wird, ist der Standardwert der NetBIOS-Name des lokalen CIFS-Servers.

+
`-locality`  `local`|`free`|`widelink`} gibt an, ob ein lokaler Link, ein freier Link oder ein breiter Symbol-Link erstellt wird. Ein lokaler Symbol-Link verweist auf die lokale SMB-Freigabe. Ein freier Symbol-Link kann auf eine beliebige Stelle auf dem lokalen SMB-Server verweisen. Ein breiter Symbol-Link verweist auf eine beliebige SMB-Freigabe im Netzwerk. Wird dieser optionale Parameter nicht angegeben, ist der Standardwert  `local`.

+
`-home-directory`  `true` `false`} gibt an, ob die Zielfreigabe ein Home-Verzeichnis ist. Auch wenn dieser Parameter optional ist, muss dieser Parameter auf  `true` gesetzt werden, wenn die Zielfreigabe als Home-Verzeichnis konfiguriert ist. Der Standardwert ist `false`.



.Beispiel
Der folgende Befehl erstellt eine Symbol-Link-Zuordnung auf der SVM mit dem Namen vs1. Sie enthält den UNIX-Pfad `/src/`, den SMB-Freigabenamen "`SOURCE`", den CIFS-Pfad `/mycompany/source/` und die CIFS-Server-IP-Adresse 123.123.123.123 und ist ein Widelink.

[listing]
----
cluster1::> vserver cifs symlink create -vserver vs1 -unix-path /src/ -share-name SOURCE -cifs-path "/mycompany/source/" -cifs-server 123.123.123.123 -locality widelink
----
