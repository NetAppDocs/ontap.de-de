= Den standardmäßigen ONTAP SMB-Benutzer konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen Standardbenutzer konfigurieren, der verwendet wird, wenn alle anderen Zuordnungsversuche für einen Benutzer fehlschlagen oder wenn Sie keine einzelnen Benutzer zwischen UNIX und Windows zuordnen möchten. Alternativ sollte kein Standardbenutzer konfiguriert werden, wenn die Authentifizierung nicht zugeordneter Benutzer fehlschlagen soll.

.Über diese Aufgabe
Für die CIFS Authentifizierung kann anstelle der Zuordnung jedes Windows-Benutzers zu einem einzelnen UNIX-Benutzer ein Standard-UNIX-Benutzer angegeben werden.

Für die NFS Authentifizierung kann anstelle der Zuordnung jedes UNIX-Benutzers zu einem einzelnen Windows-Benutzer ein Standard-Windows-Benutzer angegeben werden.

.Schritte
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn Sie möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Den Standard-UNIX-Benutzer konfigurieren
 a| 
`vserver cifs options modify -default-unix-user _user_name_`



 a| 
Den Standard-Windows-Benutzer konfigurieren
 a| 
`vserver nfs modify -default-win-user _user_name_`

|===
