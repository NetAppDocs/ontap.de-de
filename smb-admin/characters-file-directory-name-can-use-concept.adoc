= Informationen zu gültigen Zeichen für ONTAP SMB Datei- oder Verzeichnisnamen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie von Clients mit unterschiedlichen Betriebssystemen auf eine Datei oder ein Verzeichnis zugreifen, empfiehlt sich die Verwendung von Zeichen, die in beiden Betriebssystemen gültig sind.

Wenn Sie beispielsweise unter UNIX eine Datei oder ein Verzeichnis erstellen, verwenden Sie keinen Doppelpunkt (:) im Namen, da der Doppelpunkt in MS-DOS-Datei- oder Verzeichnisnamen nicht zulässig ist. Da die Beschränkungen für zulässige Zeichen je nach Betriebssystem variieren, enthält die Dokumentation Ihres Client-Betriebssystems weitere Informationen zu verbotenen Zeichen.
