= Die BranchCache Konfiguration auf ONTAP SMB-Freigaben löschen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie zuvor BranchCache konfiguriert haben, aber nicht möchten, dass die Storage Virtual Machine (SVM) weiterhin zwischengespeicherte Inhalte bereitstellt, kann die BranchCache-Konfiguration auf dem CIFS-Server gelöscht werden. Es muss beachtet werden, was beim Löschen der Konfiguration geschieht.

Wenn die Konfiguration gelöscht wird, entfernt ONTAP die Konfigurationsinformationen für diese SVM aus dem Cluster und stoppt den BranchCache-Dienst. Es kann ausgewählt werden, ob ONTAP den Hash-Speicher auf der SVM löschen soll.

Das Löschen der BranchCache-Konfiguration unterbricht nicht den Zugriff durch BranchCache-fähige Clients. Wenn BranchCache-fähige Clients danach Metadateninformationen zu bestehenden SMB-Verbindungen für bereits im Cache gespeicherte Inhalte anfordern, antwortet ONTAP mit einem von Microsoft definierten Fehler, was den Client veranlasst, eine zweite Anfrage zu senden, mit der der tatsächliche Inhalt angefordert wird. Als Antwort auf die Anfrage nach Inhalten sendet der CIFS-Server den tatsächlichen Inhalt, der auf der SVM gespeichert ist

Nach dem Löschen der BranchCache-Konfiguration geben die SMB-Freigaben keine BranchCache-Funktionen mehr bekannt. Um auf Inhalte zuzugreifen, die zuvor nicht zwischengespeichert wurden, stellen Clients über neue SMB-Verbindungen normale SMB-Leseanforderungen.

Der Befehl, den Sie zum Löschen des BranchCache-Dienstes auf Ihrer Storage Virtual Machine (SVM) verwenden, unterscheidet sich je nachdem, ob vorhandene Hashes gelöscht oder beibehalten werden sollen.

.Schritt
. Führen Sie den entsprechenden Befehl aus:
+
|===
| Wenn Sie möchten... | Geben Sie anschließend Folgendes ein... 


 a| 
Die BranchCache Konfiguration und die vorhandenen Hashes löschen
 a| 
`vserver cifs branchcache delete -vserver _vserver_name_ -flush-hashes true`



 a| 
Die BranchCache Konfiguration löschen, aber vorhandene Hashes beibehalten
 a| 
`vserver cifs branchcache delete -vserver _vserver_name_ -flush-hashes false`

|===


.Beispiel
Das folgende Beispiel löscht die BranchCache Konfiguration auf SVM vs1 und löscht alle vorhandenen Hashes:

[listing]
----
cluster1::> vserver cifs branchcache delete -vserver vs1 -flush-hashes true
----
