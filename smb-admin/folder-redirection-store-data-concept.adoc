= Informationen zur Verwendung der Ordnerumleitung zur Speicherung von Daten auf ONTAP SMB-Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt die Microsoft Ordnerumleitung, mit der Benutzer oder Administratoren den Pfad eines lokalen Ordners auf einen Speicherort auf dem CIFS-Server umleiten können. Es erscheint so, als ob umgeleitete Ordner auf dem lokalen Windows-Client gespeichert wären, obwohl die Daten auf einer SMB-Freigabe gespeichert sind.

Die Ordnerumleitung ist hauptsächlich für Organisationen vorgesehen, die bereits Home-Verzeichnisse eingerichtet haben und die Kompatibilität mit ihrer bestehenden Home-Verzeichnisumgebung beibehalten möchten.

* `Documents`, `Desktop` und `Start Menu` sind Beispiele für Ordner, die umgeleitet werden können.
* Benutzer können Ordner von ihrem Windows-Client aus umleiten.
* Administratoren können die Ordnerumleitung zentral konfigurieren und verwalten, indem sie GPOs in Active Directory konfigurieren.
* Wenn Administratoren Roaming-Profile konfiguriert haben, ermöglicht die Ordnerumleitung Administratoren, Benutzerdaten von Profildaten zu trennen.
* Administratoren können Ordnerumleitung und Offline-Dateien zusammen verwenden, um die Datenspeicherung für lokale Ordner auf den CIFS-Server umzuleiten, während Benutzern ermöglicht wird, die Inhalte lokal zwischenzuspeichern.
