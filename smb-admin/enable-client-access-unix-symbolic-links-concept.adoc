= Informationen zur Bereitstellung von ONTAP SMB-Clientzugriff auf UNIX Symbol-Links
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Symbol-Link ist eine Datei, die in einer UNIX-Umgebung erstellt wird und einen Verweis auf eine andere Datei oder ein anderes Verzeichnis enthält. Wenn ein Client auf einen Symbol-Link zugreift, wird der Client zum Ziel der Datei oder des Verzeichnisses weitergeleitet, auf das sich der Symbol-Link bezieht. ONTAP unterstützt relative und absolute Symbol-Links, einschließlich Widelinks (absolute Links mit Zielen außerhalb des lokalen Dateisystems).

ONTAP bietet SMB-Clients die Möglichkeit, den auf der SVM konfigurierten UNIX Symbol-Links zu folgen. Diese Funktion ist optional und kann für jede Freigabe einzeln mit der  `-symlink-properties` Option des  `vserver cifs share create` Befehls mit einer der folgenden Einstellungen konfiguriert werden:

* Aktiviert mit lesen/schreiben Zugriff
* Aktiviert mit schreibgeschütztem Zugriff
* Deaktiviert durch Ausblenden von Symbol-Links für SMB-Clients
* Deaktiviert, kein Zugriff auf Symbol-Links von SMB-Clients


Wenn Sie Symbol-Links für eine Freigabe aktivieren, funktionieren relative Symbol-Links ohne weitere Konfiguration.

Wenn Sie Symbol-Links für eine Freigabe aktivieren, funktionieren absolute Symbol-Links nicht sofort. Zunächst muss eine Zuordnung zwischen dem UNIX-Pfad des Symbol-Link und dem Ziel-SMB-Pfad erstellt werden. Beim Erstellen von Zuordnungen absoluter Symbol-Links kann angegeben werden, ob es sich um einen lokalen Link oder einen Widelink handelt; Widelinks können Links zu Dateisystemen auf anderen Speichersystemen oder zu Dateisystemen sein, die in separaten SVMs auf demselben ONTAP System gehostet werden. Beim Erstellen eines Widelinks müssen die Informationen enthalten sein, denen der Client folgen kann, das heißt, es wird ein Reparse-Point erstellt, damit der Client den Verbindungsordner erkennt. Wird ein absoluter Symbol-Link zu einer Datei oder einem Verzeichnis außerhalb der lokalen Freigabe erstellt, die Lokalität jedoch auf lokal gesetzt, verweigert ONTAP den Zugriff auf das Ziel.

[NOTE]
====
Wenn ein Client versucht, einen lokalen Symbol-Link (absolut oder relativ) zu löschen, wird nur der Symbol-Link gelöscht, nicht die Zieldatei oder das Zielverzeichnis. Wenn ein Client jedoch versucht, einen Widelink zu löschen, kann die tatsächliche Zieldatei oder das tatsächliche Zielverzeichnis gelöscht werden, auf das der Widelink verweist. ONTAP hat darauf keinen Einfluss, da der Client die Zieldatei oder das Zielverzeichnis außerhalb der SVM explizit öffnen und löschen kann.

====
* *Reparse Points und ONTAP Dateisystemdienste*
+
Ein _Reparse-Point_ ist ein NTFS-Dateisystemobjekt, das optional zusammen mit einer Datei auf Volumes gespeichert werden kann. Reparse-Points bieten SMB-Clients die Möglichkeit, erweiterte Dateisystemdienste zu erhalten, wenn sie mit Volumes im NTFS-Stil arbeiten. Reparse-Points bestehen aus Standard-Tags, die den Typ des Reparse-Points identifizieren, und dem Inhalt des Reparse-Points, der von SMB-Clients zur weiteren Verarbeitung durch den Client abgerufen werden kann. Von den Objekttypen, die für erweiterte Dateisystemfunktionen verfügbar sind, implementiert ONTAP die Unterstützung für NTFS-Symbol-Links und Verzeichnisverbindungspunkte mithilfe von Reparse-Point-Tags. SMB-Clients, die den Inhalt eines Reparse-Points nicht verstehen können, ignorieren ihn einfach und stellen den erweiterten Dateisystemdienst, den der Reparse-Point ermöglichen könnte, nicht bereit.

* *Verbindungspunkte für Verzeichnisse und ONTAP Unterstützung für Symbol-Links*
+
Verzeichnis-Verbindungspunkte sind Stellen innerhalb der Verzeichnisstruktur eines Dateisystems, die auf alternative Speicherorte von Dateien verweisen können, entweder auf einem anderen Pfad (Symbol-Links) oder auf einem separaten Speichermedium (Widelinks). ONTAP SMB-Server stellen Windows-Clients Verzeichnis-Verbindungspunkte als Reparse Points zur Verfügung, wodurch fähige Clients beim Durchqueren eines Verzeichnis-Verbindungspunkts die Inhalte des Reparse Points von ONTAP abrufen können. Dadurch ist es möglich, zu verschiedenen Pfaden oder Speichermedien zu navigieren und eine Verbindung herzustellen, als wären sie Teil desselben Dateisystems.

* *Aktivieren der Widelink-Unterstützung mithilfe von Reparse-Point-Optionen*
+
Die `-is-use-junctions-as-reparse-points-enabled` Option ist in ONTAP 9 standardmäßig aktiviert. Die Option zur Aktivierung der Information ist für jede Protokollversion konfigurierbar, da nicht alle SMB-Clients Widelinks unterstützen. Dadurch können Administratoren sowohl unterstützte als auch nicht unterstützte SMB-Clients berücksichtigen. Die Option `-widelink-as-reparse-point-versions` muss für jedes Clientprotokoll aktiviert werden, das über Widelinks auf die Freigabe zugreift; standardmäßig ist dies SMB1.
