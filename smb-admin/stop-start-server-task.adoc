= ONTAP SMB-Server stoppen oder starten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den CIFS-Server auf einer SVM stoppen, was nützlich ist, wenn Aufgaben ausgeführt werden, während Benutzer nicht über SMB-Freigaben auf Daten zugreifen. Der SMB-Zugriff kann durch Starten des CIFS-Servers wiederhergestellt werden. Durch das Stoppen des CIFS-Servers können auch die auf der Storage Virtual Machine (SVM) zulässigen Protokolle geändert werden.

.Schritte
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn Sie möchten... | Geben Sie den Befehl ein... 


 a| 
Den CIFS-Server anhalten
 a| 
`vserver cifs stop -vserver _vserver_name_ [-foreground {true|false}]`



 a| 
CIFS-Server starten
 a| 
`vserver cifs start -vserver _vserver_name_ [-foreground {true|false}]`

|===
+
`-foreground` Gibt an, ob der Befehl im Vordergrund oder im Hintergrund ausgeführt wird. Wird dieser Parameter nicht eingegeben, ist er auf  `true` gesetzt und der Befehl wird im Vordergrund ausgeführt.

. Es kann mithilfe des  `vserver cifs show` Befehls überprüft werden, ob der administrative Status des CIFS-Servers korrekt ist.


.Beispiel
Die folgenden Befehle starten den CIFS-Server auf SVM vs1:

[listing]
----
cluster1::> vserver cifs start -vserver vs1

cluster1::> vserver cifs show -vserver vs1

                                          Vserver: vs1
                         CIFS Server NetBIOS Name: VS1
                    NetBIOS Domain/Workgroup Name: DOMAIN
                      Fully Qualified Domain Name: DOMAIN.LOCAL
Default Site Used by LIFs Without Site Membership:
                             Authentication Style: domain
                CIFS Server Administrative Status: up
----
