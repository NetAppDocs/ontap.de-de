= Konfiguration der UNIX-Unterstützung für Symbol-Links auf ONTAP SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Unterstützung für UNIX Symbol-Links auf SMB-Freigaben konfigurieren, indem Sie beim Erstellen von SMB-Freigaben oder jederzeit durch Ändern bestehender SMB-Freigaben eine Symbol-Link Freigabeeigenschaft festlegen. Die Unterstützung für UNIX Symbol-Links ist standardmäßig aktiviert. Die Unterstützung für UNIX Symbol-Links kann auf einer Freigabe auch deaktiviert werden.

.Über diese Aufgabe
Bei der Konfiguration der UNIX Symbol-Link-Unterstützung für SMB-Freigaben steht eine der folgenden Einstellungen zur Auswahl:

|===
| Einstellung | Beschreibung 


 a| 
`enable` (VERALTET*)
 a| 
Gibt an, dass Symbol-Links für Lese- und Schreibzugriffe aktiviert sind.



 a| 
`read_only` (VERALTET*)
 a| 
Legt fest, dass Symbol-Links für den Nur-Lese-Zugriff aktiviert sind. Diese Einstellung gilt nicht für Widelinks. Widelink-Zugriff ist immer Lese- und Schreibzugriff.



 a| 
`hide` (VERALTET*)
 a| 
Legt fest, dass SMB-Clients keine Symbol-Links sehen können.



 a| 
`no-strict-security`
 a| 
Legt fest, dass Clients Symbol-Links außerhalb der Freigabegrenzen folgen.



 a| 
`symlinks`
 a| 
Gibt an, dass Symbol-Links lokal für Lese- und Schreibzugriffe aktiviert sind. Die DFS-Ankündigungen werden nicht generiert, selbst wenn die CIFS-Option `is-advertise-dfs-enabled` auf `true` gesetzt ist. Dies ist die Standardeinstellung.



 a| 
`symlinks-and-widelinks`
 a| 
Gibt an, dass sowohl lokale Symbol-Links als auch Widelinks für Lese- und Schreibzugriffe verwendet werden. Die DFS-Ankündigungen werden sowohl für lokale Symbol-Links als auch für Widelinks generiert, selbst wenn die CIFS-Option  `is-advertise-dfs-enabled` auf  `false` gesetzt ist.



 a| 
`disable`
 a| 
Gibt an, dass Symbol-Links und Widelinks deaktiviert sind. Die DFS-Ankündigungen werden nicht generiert, selbst wenn die CIFS-Option `is-advertise-dfs-enabled` auf `true` gesetzt ist.



 a| 
`""` (null, nicht gesetzt)
 a| 
Deaktiviert Symbol-Links auf der Freigabe.



 a| 
`-` (nicht festgelegt)
 a| 
Deaktiviert Symbol-Links auf der Freigabe.

|===
[NOTE]
====
*Die Parameter _enable_, _hide_ und _read-only_ sind veraltet und werden möglicherweise in einer zukünftigen Version von ONTAP entfernt.

====
.Schritte
. Symbol-Link-Unterstützung konfigurieren oder deaktivieren:
+
|===
| Wenn dies der Fall ist... | Eingeben... 


 a| 
Eine neue SMB-Freigabe
 a| 
`+vserver cifs share create -vserver vserver_name -share-name share_name -path path -symlink-properties {enable|hide|read-only|""|-|symlinks|symlinks-and-widelinks|disable},...]+`



 a| 
Eine bestehende SMB-Freigabe
 a| 
`+vserver cifs share modify -vserver vserver_name -share-name share_name -symlink-properties {enable|hide|read-only|""|-|symlinks|symlinks-and-widelinks|disable},...]+`

|===
. Es sollte überprüft werden, ob die SMB-Freigabekonfiguration korrekt ist: `vserver cifs share show -vserver vserver_name -share-name share_name -instance`


.Beispiel
Der folgende Befehl erstellt eine SMB-Freigabe mit dem Namen "`data1`", wobei die UNIX-Symbol-Link-Konfiguration auf `enable`gesetzt ist:

[listing]
----
cluster1::> vserver cifs share create -vserver vs1 -share-name data1 -path /data1 -symlink-properties enable

cluster1::> vserver cifs share show -vserver vs1 -share-name data1 -instance
