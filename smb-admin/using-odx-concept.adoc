= Richtlinien für die Verwendung von ODX auf ONTAP SMB Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie ODX für Copy-Offload verwenden können, müssen Sie die Richtlinien kennen. Beispielsweise müssen Sie wissen, auf welchen Volume-Typen Sie ODX verwenden können, und die Überlegungen zu ODX innerhalb und zwischen Clustern verstehen.



[[volume-guidelines]]
== Volumenrichtlinien

* ODX kann für Copy Offload bei folgenden Volume-Konfigurationen nicht verwendet werden:
+
** Die Größe des Quellvolumes beträgt weniger als 1,25 GB
+
Die Volume-Größe muss mindestens 1,25 GB betragen, um ODX zu verwenden.

** Schreibgeschützte Volumes
+
ODX wird nicht für Dateien und Ordner verwendet, die sich auf Load-Sharing-Mirrors oder in SnapMirror oder SnapVault Zielvolumes befinden.

** Wenn das Quellvolume nicht dedupliziert ist


* ODX-Kopien werden nur für Intra-Cluster-Kopien unterstützt.
+
Mit ODX ist das Kopieren von Dateien oder Ordnern auf ein Volume in einem anderen Cluster nicht möglich.





[[other-guidelines]]
== Weitere Richtlinien

* In SMB-Umgebungen müssen die Dateien mindestens 256 KB groß sein, damit ODX für Copy-Offload verwendet werden kann.
+
Kleinere Dateien werden mittels eines herkömmlichen Kopiervorgangs übertragen.

* ODX Copy Offload nutzt Deduplizierung als Teil des Kopiervorgangs.
+
Wenn keine Deduplizierung auf SVM-Volumes beim Kopieren oder Verschieben von Daten erfolgen soll, ist ODX Copy Offload auf diesem SVM zu deaktivieren.

* Die Anwendung, die den Datentransfer durchführt, muss so geschrieben sein, dass sie ODX unterstützt.
+
Zu den Anwendungsoperationen, die ODX unterstützen, gehören die folgenden:

+
** Hyper-V Managementvorgänge, wie das Erstellen und Konvertieren virtueller Festplatten (VHDs), das Verwalten von Snapshots und das Kopieren von Dateien zwischen virtuellen Maschinen
** Windows Explorer Operationen
** Windows PowerShell Kopierbefehle
** Windows-Eingabeaufforderung Kopierbefehle
+
Robocopy an der Windows-Eingabeaufforderung unterstützt ODX.

+
[NOTE]
====
Die Anwendungen müssen auf Windows Servern oder Clients ausgeführt werden, die ODX unterstützen.

====
+
Weitere Informationen zu unterstützten ODX-Anwendungen auf Windows Servern und Clients bietet die Microsoft TechNet Library.
