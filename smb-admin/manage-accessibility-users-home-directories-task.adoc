= Zugänglichkeit zu ONTAP SMB-Benutzer-Home-Verzeichnissen verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Standardmäßig kann das Home-Verzeichnis eines Benutzers nur von diesem Benutzer selbst aufgerufen werden. Bei Freigaben, deren dynamischer Name mit einer Tilde ({tilde}) beginnt, lässt sich der Zugriff auf die Home-Verzeichnisse von Benutzern für Windows-Administratoren oder für beliebige andere Benutzer (öffentlicher Zugriff) aktivieren oder deaktivieren.

.Bevor Sie beginnen
Die Home-Verzeichnisfreigaben auf der Storage Virtual Machine (SVM) müssen mit dynamischen Freigabenamen konfiguriert werden, denen eine Tilde ({tilde}) vorangestellt ist. Die folgenden Fälle veranschaulichen die Anforderungen an die Freigabebenennung:

|===
| Home-Verzeichnis-Freigabename | Beispiel für einen Befehl zum Herstellen einer Verbindung zur Freigabe 


 a| 
{tilde}%d{tilde}%w
 a| 
`net use * {backslash}{backslash}IPaddress{backslash}{tilde}domain{tilde}user/u:credentials`



 a| 
{tilde}%w
 a| 
`net use * {backslash}{backslash}IPaddress{backslash}{tilde}user/u:credentials`



 a| 
{tilde}abc{tilde}%w
 a| 
`net use * {backslash}{backslash}IPaddress{backslash}abc{tilde}user/u:credentials`

|===
.Schritt
. Die entsprechende Maßnahme ausführen:
+
|===
| Wenn der Zugriff auf die Home-Verzeichnisse der Benutzer aktiviert oder deaktiviert werden soll... | Geben Sie Folgendes ein... 


| Windows Administratoren | `vserver cifs home-directory modify -vserver _vserver_name_ -is-home-dirs-access-for-admin-enabled {true{vbar}false}` Der Standardwert ist  `true`. 


| Jeder Benutzer (öffentlicher Zugriff)  a| 
.. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: +
`set -privilege advanced`
.. Zugriff aktivieren oder deaktivieren: `vserver cifs home-directory modify -vserver _vserver_name_ -is-home-dirs-access-for-public-enabled {true|false}` + Der Standardwert ist `false`.
.. Zurück zur Administratorrechteebene: +
`set -privilege admin`


|===


Das folgende Beispiel ermöglicht den öffentlichen Zugriff auf die Home-Verzeichnisse der Benutzer: +
`set -privilege advanced` +
`vserver cifs home-directory modify -vserver vs1 -is-home-dirs-access-for-public-enabled true` +
`set -privilege admin`
