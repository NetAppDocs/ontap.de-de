= Informationen zur ONTAP SMB-Signaturkonfiguration für mehrere Daten-LIFS
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie die erforderliche SMB-Signierung auf dem SMB-Server aktivieren oder deaktivieren, sollten die Richtlinien für mehrere Daten-LIFS-Konfigurationen für eine SVM bekannt sein.

Bei der Konfiguration eines SMB-Servers können mehrere Daten-LIFs eingerichtet sein. In diesem Fall enthält der DNS-Server mehrere  `A` Einträge für den CIFS-Server, die alle denselben Hostnamen des SMB-Servers verwenden, aber jeweils eine eindeutige IP-Adresse besitzen. Beispielsweise könnte ein SMB-Server mit zwei konfigurierten Daten-LIFs die folgenden DNS  `A` Einträge aufweisen:

[listing]
----
10.1.1.128 A VS1.IEPUB.LOCAL VS1
10.1.1.129 A VS1.IEPUB.LOCAL VS1
----
Normalerweise werden bei einer Änderung der erforderlichen SMB-Signatur-Einstellung nur neue Verbindungen von Clients von der Änderung der SMB-Signatur-Einstellung betroffen. Es gibt jedoch eine Ausnahme von diesem Verhalten. Es gibt einen Fall, in dem ein Client bereits eine Verbindung zu einer Freigabe hat und nach der Änderung der Einstellung eine neue Verbindung zur selben Freigabe herstellt, während die ursprüngliche Verbindung bestehen bleibt. In diesem Fall übernehmen sowohl die neue als auch die bestehende SMB-Verbindung die neuen SMB-Signatur-Anforderungen.

Das folgende Beispiel ist zu berücksichtigen:

. Client1 stellt eine Verbindung zu einer Freigabe ohne erforderliche SMB-Signierung über den Pfad  `O:\` her.
. Der Speicheradministrator ändert die SMB-Server Konfiguration, sodass die SMB-Signierung erforderlich ist.
. Client1 stellt eine Verbindung zur selben Freigabe mit der erforderlichen SMB-Signatur unter Verwendung des Pfads  `S:\` (wobei die Verbindung unter Verwendung des Pfads  `O:\` aufrechterhalten wird).
. Das Ergebnis ist, dass die SMB-Signatur beim Zugriff auf Daten sowohl über das  `O:\` als auch das  `S:\` Laufwerk verwendet wird.
