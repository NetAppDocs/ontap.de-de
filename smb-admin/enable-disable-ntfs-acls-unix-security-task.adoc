= Die Darstellung von NTFS-ACLs für ONTAP SMB-Clients bei UNIX-Sicherheitsstil-Daten konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Anzeige von NTFS-ACLs für SMB-Clients bei Daten im UNIX-Sicherheitsstil (Volumes im UNIX-Sicherheitsstil und gemischte Volumes im Sicherheitsstil mit effektiver UNIX-Sicherheit) kann aktiviert oder deaktiviert werden.

.Über diese Aufgabe
Wenn diese Option aktiviert ist, präsentiert ONTAP Dateien und Ordner auf Volumes mit effektivem UNIX-Sicherheitsstil für SMB-Clients als mit NTFS-ACLs versehen. Wenn diese Option deaktiviert ist, werden die Volumes SMB-Clients als FAT-Volumes präsentiert. Standardmäßig werden NTFS-ACLs für SMB-Clients präsentiert.

.Schritte
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: `set -privilege advanced`
. Konfiguration der UNIX NTFS ACL-Optionseinstellung: `vserver cifs options modify -vserver _vserver_name_ -is-unix-nt-acl-enabled {true|false}`
. Es wird überprüft, ob die Option auf den gewünschten Wert eingestellt ist: `vserver cifs options show -vserver _vserver_name_`
. Zurück zur Administratorrechteebene: `set -privilege admin`
