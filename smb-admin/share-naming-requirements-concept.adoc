= Informationen zu den Anforderungen an die Benennung von ONTAP SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die ONTAP Anforderungen an die Benennung von Freigaben sind bei der Erstellung von SMB-Freigaben auf dem SMB-Server zu berücksichtigen.

Die Namenskonventionen für Freigaben unter ONTAP sind die gleichen wie unter Windows und umfassen die folgenden Anforderungen:

* Der Name jeder Freigabe muss für den SMB-Server eindeutig sein.
* Bei Share-Namen wird nicht zwischen Groß- und Kleinschreibung unterschieden.
* Die maximale Länge des Freigabenamens beträgt 80 Zeichen.
* Unicode-Freigabenamen werden unterstützt.
* Freigabenamen, die mit dem $-Zeichen enden, sind versteckte Freigaben.
* Bei ONTAP 9.7 und früheren Versionen werden die administrativen Freigaben admin$, ipc$ und c$ automatisch auf jedem CIFS-Server erstellt und sind reservierte Freigabenamen. Ab ONTAP 9.8 wird die Freigabe admin$ nicht mehr automatisch erstellt.
* Der Freigabename ONTAP_ADMIN$ kann beim Erstellen einer Freigabe nicht verwendet werden.
* Freigabenamen, die Leerzeichen enthalten, werden unterstützt:
+
** Ein Leerzeichen darf weder als erstes noch als letztes Zeichen in einem Freigabenamen verwendet werden.
** Freigabenamen, die ein Leerzeichen enthalten, müssen in Anführungszeichen gesetzt werden.
+
[NOTE]
====
Einfache Anführungszeichen gelten als Bestandteil des Freigabenamens und können nicht anstelle von Anführungszeichen verwendet werden.

====


* Folgende Sonderzeichen werden bei der Benennung von SMB-Freigaben unterstützt:
+
[listing]
----
! @ # $ % & ' _ - . ~ ( ) { }
----
* Die folgenden Sonderzeichen sind bei der Benennung von SMB-Freigaben nicht zulässig:
+
[listing]
----
** [ ] " / \ : ; | < > , ? * =
----
