= Den Metadatencache für SMB-Freigaben konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Metadaten-Caching ermöglicht das Zwischenspeichern von Dateiattributen auf SMB 1.0 Clients, um einen schnelleren Zugriff auf Datei- und Ordnerattribute zu bieten. Das Attribut-Caching kann für jede Freigabe einzeln aktiviert oder deaktiviert werden. Die Gültigkeitsdauer (Time-to-Live) der zwischengespeicherten Einträge kann ebenfalls konfiguriert werden, wenn Metadaten-Caching aktiviert ist. Eine Konfiguration des Metadaten-Cachings ist nicht erforderlich, wenn Clients über SMB 2.x oder SMB 3.0 auf Freigaben zugreifen.

Wenn diese Funktion aktiviert ist, speichert der SMB-Metadatencache Pfad- und Dateiattributdaten für eine begrenzte Zeit. Dies kann die SMB-Leistung für SMB 1.0 Clients mit gängigen Arbeitslasten verbessern.

Bei bestimmten Aufgaben erzeugt SMB ein erhebliches Datenaufkommen, das mehrere identische Abfragen für Pfad- und Dateimetadaten enthalten kann. Die Anzahl redundanter Abfragen lässt sich reduzieren und die Leistung für SMB 1.0 Clients verbessern, wenn SMB-Metadaten-Caching verwendet wird, um die Informationen stattdessen aus dem Cache abzurufen.

[NOTE]
====
Es ist zwar unwahrscheinlich, aber möglich, dass der Metadaten-Cache veraltete Informationen an SMB 1.0-Clients ausliefert. Falls dieses Risiko in Ihrer Umgebung nicht akzeptabel ist, sollte diese Funktion nicht aktiviert werden.

====


[[enable-the-ontap-smb-metadata-cache]]
== Der ONTAP SMB-Metadaten-Cache kann aktiviert werden.

Die SMB-Leistung für SMB 1.0-Clients lässt sich durch Aktivierung des SMB-Metadaten-Caches verbessern. Standardmäßig ist die SMB-Metadaten-Zwischenspeicherung deaktiviert.

.Schritt
. Aktivieren Sie das SMB-Metadaten-Caching für eine neue oder bestehende Freigabe:
+
** Aktivieren Sie das SMB-Metadaten-Caching beim Erstellen einer neuen Freigabe:
+
[source, cli]
----
vserver cifs share create -vserver <SVM_name> -share-name <share_name> -path <path> -share-properties attributecache
----
** Aktivieren Sie das SMB-Metadaten-Caching für eine vorhandene Freigabe:
+
[source, cli]
----
vserver cifs share properties add -vserver <SVM_name> -share-name <share_name> -share-properties attributecache
----






[[configure-the-lifetime-of-ontap-smb-metadata-cache-entries]]
== Die Lebensdauer von ONTAP SMB Metadaten-Cache-Einträgen konfigurieren

Sie können die Lebensdauer von SMB-Metadaten-Cache-Einträgen konfigurieren, um die Leistung des SMB-Metadaten-Caches in Ihrer Umgebung zu optimieren. Der Standardwert beträgt 10 Sekunden.

.Bevor Sie beginnen
Sie müssen die SMB-Metadaten-Cache-Funktion aktiviert haben. Wenn die SMB-Metadaten-Cache-Funktion nicht aktiviert ist, wird die SMB-Cache-TTL-Einstellung nicht verwendet.

.Schritt
. Konfigurieren Sie die Lebensdauer von SMB-Metadaten-Cache-Einträgen auf einer neuen oder vorhandenen Freigabe:
+
** Konfigurieren Sie die Lebensdauer von SMB-Metadaten-Cache-Einträgen beim Erstellen einer neuen Freigabe:
+
[source, cli]
----
vserver cifs share create -vserver <SVM_name> -share-name <share_name> -path <path> -attribute-cache-ttl [integerh][integerm][integers]
----
** Konfigurieren der Lebensdauer von SMB-Metadaten-Cache-Einträgen auf einer vorhandenen Freigabe:
+
[source, cli]
----
vserver cifs share modify -vserver <SVM_name> -share-name <share_name> -attribute-cache-ttl [integerh][integerm][integers]
----




Beim Erstellen oder Ändern von Freigaben können Sie zusätzliche Konfigurationsoptionen und Eigenschaften festlegen. Weitere Informationen zu  `vserver cifs share` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+cifs+share["ONTAP-Befehlsreferenz"^].
