= ONTAP Passwortkomplexität für lokale SMB-Benutzer konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die erforderliche Passwortkomplexität bietet erhöhte Sicherheit für lokale SMB-Benutzer auf Ihren Storage Virtual Machines (SVMs). Die Funktion für erforderliche Passwortkomplexität ist standardmäßig aktiviert. Sie kann jederzeit deaktiviert und wieder aktiviert werden.

.Bevor Sie beginnen
Lokale Benutzer, lokale Gruppen und die lokale Benutzerauthentifizierung müssen auf dem CIFS-Server aktiviert sein.

[NOTE]
.Über diese Aufgabe
====
Der  `vserver cifs security modify` Befehl sollte für einen CIFS-Server im Arbeitsgruppenmodus nicht verwendet werden, da einige Optionen ungültig sind.

====
.Schritte
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn eine erforderliche Passwortkomplexität für lokale SMB-Benutzer gelten soll... | Geben Sie den Befehl ein... 


 a| 
Aktiviert
 a| 
`vserver cifs security modify -vserver _vserver_name_ -is-password-complexity-required true`



 a| 
Deaktiviert
 a| 
`vserver cifs security modify -vserver _vserver_name_ -is-password-complexity-required false`

|===
. Die Sicherheitseinstellung für die erforderliche Passwortkomplexität überprüfen: `vserver cifs security show -vserver _vserver_name_`


.Beispiel
Das folgende Beispiel zeigt, dass die erforderliche Kennwortkomplexität für lokale SMB-Benutzer für SVM vs1 aktiviert ist:

[listing]
----
cluster1::> vserver cifs security modify -vserver vs1 -is-password-complexity-required true

cluster1::> vserver cifs security show -vserver vs1 -fields is-password-complexity-required
vserver is-password-complexity-required
------- -------------------------------
vs1     true
----
