= Unterstützung für ONTAP SMB automatische Knotenweiterleitungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie die automatischen Knotenverweise für SMB aktivieren, sollten Sie sich darüber im Klaren sein, dass bestimmte ONTAP Funktionalitäten keine Verweise unterstützen.

* Die folgenden Volume-Typen unterstützen keine automatischen Knotenverweise für SMB:
+
** Schreibgeschützte Mitglieder eines Load-Sharing-Spiegels
** Zielvolume eines Datensicherungsspiegels


* Knotenverweise werden bei einer LIF-Verschiebung nicht mitverschoben.
+
Wenn ein Client eine referenzierte Verbindung über eine SMB 2.x oder SMB 3.0 Verbindung verwendet und ein Daten-LIF unterbrechungsfrei verschoben wird, verwendet der Client weiterhin dieselbe referenzierte Verbindung, auch wenn das LIF nicht mehr lokal zu den Daten ist.

* Knotenverweise werden nicht zusammen mit einer Volumenverschiebung verschoben.
+
Wenn ein Client eine referenzierte Verbindung über eine beliebige SMB-Verbindung verwendet und eine Volume-Verschiebung erfolgt, verwendet der Client weiterhin dieselbe referenzierte Verbindung, auch wenn sich das Volume nicht mehr auf demselben Knoten wie die Daten-LIF befindet.
