= Informationen zur Verwendung von Offline-Dateien, um das Zwischenspeichern von ONTAP SMB-Dateien für die Offline-Nutzung zu ermöglichen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt die Microsoft-Funktion „Offline-Dateien“ oder _Client-seitiges Caching_, mit der Dateien auf dem lokalen Host für die Offline-Nutzung zwischengespeichert werden können. Benutzer können die Offline-Dateien-Funktionalität verwenden, um auch bei unterbrochener Netzwerkverbindung weiter an Dateien zu arbeiten.

Sie können festlegen, ob Windows-Benutzerdokumente und -Programme automatisch auf einer Freigabe zwischengespeichert werden oder ob die Dateien manuell zum Zwischenspeichern ausgewählt werden müssen. Die manuelle Zwischenspeicherung ist für neue Freigaben standardmäßig aktiviert. Die offline verfügbaren Dateien werden mit der lokalen Festplatte des Windows-Clients synchronisiert. Die Synchronisierung erfolgt, wenn die Netzwerkverbindung zu einer bestimmten Speichersystemfreigabe wiederhergestellt wird.

Da Offline-Dateien und -Ordner die gleichen Zugriffsrechte behalten wie die Version der auf dem CIFS-Server gespeicherten Dateien und Ordner, muss der Benutzer über ausreichende Berechtigungen für die auf dem CIFS-Server gespeicherten Dateien und Ordner verfügen, um Aktionen an den Offline-Dateien und -Ordnern ausführen zu können.

Wenn ein Benutzer und eine andere Person im Netzwerk dieselbe Datei ändern, kann der Benutzer die lokale Version der Datei im Netzwerk speichern, die andere Version behalten oder beide speichern. Wenn der Benutzer beide Versionen behält, wird eine neue Datei mit den Änderungen des lokalen Benutzers lokal gespeichert und die zwischengespeicherte Datei mit den Änderungen aus der Version der Datei überschrieben, die auf dem CIFS-Server gespeichert ist.

Offline-Dateien können für jede Freigabe einzeln über die Freigabekonfigurationseinstellungen konfiguriert werden. Beim Erstellen oder Ändern von Freigaben steht eine von vier Offline-Ordnerkonfigurationen zur Auswahl:

* Kein Caching
+
Deaktiviert das Client-seitige Caching für die Freigabe. Dateien und Ordner werden nicht automatisch lokal auf Clients zwischengespeichert und Benutzer können das Zwischenspeichern von Dateien oder Ordnern auf dem lokalen System nicht auswählen.

* Manuelle Zwischenspeicherung
+
Ermöglicht die manuelle Auswahl von Dateien, die im freigegebenen Verzeichnis zwischengespeichert werden sollen. Dies ist die Standardeinstellung. Standardmäßig werden keine Dateien oder Ordner auf dem lokalen Client zwischengespeichert. Benutzer können auswählen, welche Dateien und Ordner sie lokal für die Offline-Nutzung zwischenspeichern möchten.

* Automatisches Dokumenten-Caching
+
Ermöglicht das automatische Zwischenspeichern von Benutzerdokumenten auf der Freigabe. Nur Dateien und Ordner, auf die zugegriffen wird, werden lokal zwischengespeichert.

* Automatisches Programm-Caching
+
Ermöglicht das automatische Zwischenspeichern von Programmen und Benutzerdokumenten auf der Freigabe. Nur Dateien, Ordner und Programme, auf die zugegriffen wird, werden lokal zwischengespeichert. Zusätzlich gestattet diese Einstellung dem Client, lokal zwischengespeicherte ausführbare Dateien auch bei bestehender Netzwerkverbindung auszuführen.
