= Es wird erläutert, wie ONTAP SMB-Server IPv6 verwenden, um eine Verbindung zu externen Servern herzustellen.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um eine Konfiguration zu erstellen, die Ihren Anforderungen entspricht, ist es erforderlich, zu wissen, wie CIFS-Server IPv6 verwenden, wenn sie Verbindungen zu externen Servern herstellen.

* Auswahl der Quelladresse
+
Wenn eine Verbindung zu einem externen Server hergestellt wird, muss die ausgewählte Quelladresse vom gleichen Typ wie die Zieladresse sein. Wenn beispielsweise eine Verbindung zu einer IPv6-Adresse hergestellt wird, muss die Storage Virtual Machine (SVM), auf der der CIFS-Server gehostet wird, über eine Daten-LIF oder eine Management-LIF mit einer IPv6-Adresse verfügen, die als Quelladresse verwendet werden kann. Entsprechend muss die SVM, wenn eine Verbindung zu einer IPv4-Adresse hergestellt wird, über eine Daten-LIF oder eine Management-LIF mit einer IPv4-Adresse verfügen, die als Quelladresse verwendet werden kann.

* Bei Servern, die dynamisch über DNS ermittelt werden, erfolgt die Serverermittlung wie folgt:
+
** Wenn IPv6 auf dem Cluster deaktiviert ist, werden nur IPv4-Serveradressen ermittelt.
** Wenn IPv6 im Cluster aktiviert ist, werden sowohl IPv4- als auch IPv6-Serveradressen ermittelt. Je nach Eignung des Servers, zu dem die Adresse gehört, und der Verfügbarkeit von IPv6- oder IPv4-Daten- oder Management-LIFs kann entweder der eine oder der andere Typ verwendet werden. Die dynamische Serverermittlung wird zur Erkennung von Domain Controllern und deren zugehörigen Diensten wie LSA, NETLOGON, Kerberos und LDAP verwendet.


* DNS Server-Konnektivität
+
Ob die SVM beim Verbinden mit einem DNS-Server IPv6 verwendet, hängt von der Konfiguration der DNS-Namensdienste ab. Wenn die DNS-Dienste so konfiguriert sind, dass sie IPv6-Adressen verwenden, werden Verbindungen über IPv6 hergestellt. Falls gewünscht, kann die Konfiguration der DNS-Namensdienste IPv4-Adressen verwenden, sodass Verbindungen zu DNS-Servern weiterhin über IPv4-Adressen erfolgen. Bei der Konfiguration der DNS-Namensdienste können Kombinationen aus IPv4- und IPv6-Adressen angegeben werden.

* LDAP Server-Konnektivität
+
Ob die SVM IPv6 für die Verbindung zu einem LDAP-Server verwendet, hängt von der LDAP-Client-Konfiguration ab. Wenn der LDAP-Client so konfiguriert ist, dass IPv6-Adressen verwendet werden, erfolgen die Verbindungen über IPv6. Falls gewünscht, kann die LDAP-Client-Konfiguration IPv4-Adressen verwenden, sodass Verbindungen zu LDAP-Servern weiterhin über IPv4-Adressen erfolgen. Bei der Konfiguration der LDAP-Client-Konfiguration können Kombinationen aus IPv4- und IPv6-Adressen angegeben werden.

+
[NOTE]
====
Die LDAP-Client-Konfiguration wird verwendet, wenn LDAP für UNIX-Benutzer-, Gruppen- und Netzgruppennamendienste konfiguriert wird.

====
* NIS Server-Konnektivität
+
Ob die SVM beim Verbinden mit einem NIS-Server IPv6 verwendet, hängt von der Konfiguration der NIS-Namensdienste ab. Wenn die NIS-Dienste so konfiguriert sind, dass sie IPv6-Adressen verwenden, werden Verbindungen über IPv6 hergestellt. Falls gewünscht, kann die Konfiguration der NIS-Namensdienste IPv4-Adressen verwenden, sodass Verbindungen zu NIS-Servern weiterhin über IPv4-Adressen erfolgen. Bei der Konfiguration der NIS-Namensdienste können Kombinationen aus IPv4- und IPv6-Adressen angegeben werden.

+
[NOTE]
====
NIS-Namensdienste werden zum Speichern und Verwalten von UNIX-Benutzer-, Gruppen-, Netzgruppen- und Hostnamensobjekten verwendet.

====
