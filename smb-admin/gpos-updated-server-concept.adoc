= Informationen zum Aktualisieren von Gruppenrichtlinienobjekten (GPOs) auf ONTAP SMB-Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Standardmäßig ruft ONTAP alle 90 Minuten Änderungen an Gruppenrichtlinienobjekten (GPOs) ab und wendet diese an. Sicherheitseinstellungen werden alle 16 Stunden aktualisiert. Wenn GPOs aktualisiert werden sollen, um neue GPO-Richtlinieneinstellungen anzuwenden, bevor ONTAP sie automatisch aktualisiert, kann eine manuelle Aktualisierung auf einem CIFS-Server mit einem ONTAP-Befehl ausgelöst werden.

* Standardmäßig werden alle Gruppenrichtlinienobjekte (GPOs) alle 90 Minuten überprüft und bei Bedarf aktualisiert.
+
Dieses Intervall ist konfigurierbar und kann über die  `Refresh interval` und  `Random offset` Gruppenrichtlinieneinstellungen festgelegt werden.

+
ONTAP fragt Active Directory nach Änderungen an Gruppenrichtlinienobjekten (GPOs) ab. Sind die in Active Directory gespeicherten GPO-Versionsnummern höher als die auf dem CIFS-Server, ruft ONTAP die neuen GPOs ab und wendet sie an. Stimmen die Versionsnummern überein, werden die GPOs auf dem CIFS-Server nicht aktualisiert.

* Die Gruppenrichtlinienobjekte (GPOs) für die Sicherheitseinstellungen werden alle 16 Stunden aktualisiert.
+
ONTAP ruft die Gruppenrichtlinienobjekte (GPOs) für Sicherheitseinstellungen alle 16 Stunden ab und wendet sie an, unabhängig davon, ob sich diese GPOs geändert haben oder nicht.

+
[NOTE]
====
Der Standardwert von 16 Stunden kann in der aktuellen ONTAP Version nicht geändert werden. Es handelt sich um eine Standardeinstellung des Windows Clients.

====
* Alle Gruppenrichtlinienobjekte (GPOs) können manuell mit einem ONTAP Befehl aktualisiert werden.
+
Dieser Befehl simuliert den Windows `gpupdate.exe``/force`-Befehl.
