= Anforderungen für die Verwendung von ODX auf ONTAP SMB-Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie ODX für Copy Offloads mit Ihrer Storage Virtual Machine (SVM) verwenden können, sind bestimmte Voraussetzungen zu berücksichtigen.



[[ontap-version-requirements]]
== ONTAP Versionsanforderungen

ONTAP Releases unterstützen ODX für Copy-Offloads.



[[smb-version-requirements]]
== SMB Versionsanforderungen

* ONTAP unterstützt ODX mit SMB 3.0 und höher.
* SMB 3.0 muss auf dem CIFS-Server aktiviert sein, bevor ODX aktiviert werden kann:
+
** Durch die Aktivierung von ODX wird auch SMB 3.0 aktiviert, falls es nicht bereits aktiviert ist.
** Durch das Deaktivieren von SMB 3.0 wird auch ODX deaktiviert.






[[windows-server-and-client-requirements]]
== Anforderungen an Windows Server und Client

Bevor Sie ODX für Copy-Offloads verwenden können, muss der Windows Client diese Funktion unterstützen.

Die link:https://mysupport.netapp.com/matrix["NetApp Interoperabilitätsmatrix"^]enthält die neuesten Informationen zu unterstützten Windows-Clients.



[[volume-requirements]]
== Volumenanforderungen

* Die Quellvolumes müssen mindestens 1,25 GB groß sein.
* Bei Verwendung komprimierter Volumes muss der Komprimierungstyp adaptiv sein und es wird nur die Komprimierungsgruppengröße 8K unterstützt.
+
Sekundärer Kompressionstyp wird nicht unterstützt.
