= Informationen zur Auswahl des ONTAP SMB Hash-Speicherorts
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei der Konfiguration von BranchCache wählen Sie aus, wo die Hashwerte gespeichert werden und wie groß der Hashspeicher sein soll. Das Verständnis der Richtlinien bei der Auswahl von Speicherort und Größe des Hashspeichers kann bei der Planung der BranchCache-Konfiguration auf einer CIFS-fähigen SVM unterstützen.

* Der Hash-Speicher sollte auf einem Volume platziert werden, auf dem atime-Aktualisierungen zulässig sind.
+
Die Zugriffszeit einer Hash-Datei wird verwendet, um häufig genutzte Dateien im Hash-Speicher zu halten. Wenn atime-Aktualisierungen deaktiviert sind, wird die Erstellungszeit zu diesem Zweck verwendet. Es ist vorzuziehen, atime zur Nachverfolgung häufig genutzter Dateien zu verwenden.

* Hashwerte können nicht auf schreibgeschützten Dateisystemen wie SnapMirror Destinationen und SnapLock Volumes gespeichert werden.
* Wenn die maximale Größe des Hash-Speichers erreicht ist, werden ältere Hashes entfernt, um Platz für neue Hashes zu schaffen.
+
Sie können die maximale Größe des Hash-Speichers erhöhen, um die Anzahl der aus dem Cache gelöschten Hashes zu reduzieren.

* Wenn das Volume, auf dem Sie Hashes speichern, nicht verfügbar oder voll ist oder wenn ein Problem mit der Kommunikation innerhalb des Clusters besteht, sodass der BranchCache-Dienst keine Hash-Informationen abrufen kann, sind BranchCache-Dienste nicht verfügbar.
+
Das Volume ist möglicherweise nicht verfügbar, weil es offline ist oder weil der Speicheradministrator einen neuen Speicherort für den Hash Store angegeben hat.

+
Dies führt nicht zu Problemen beim Dateizugriff. Falls der Zugriff auf den Hash-Speicher behindert wird, gibt ONTAP einen von Microsoft definierten Fehler an den Client zurück, woraufhin der Client die Datei über die normale SMB-Leseanforderung anfordert.
