= Oplocks beim Erstellen von ONTAP SMB-Freigaben aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Oplocks ermöglichen es Clients, Dateien zu sperren und Inhalte lokal zwischenzuspeichern, was die Leistung bei Dateivorgängen verbessern kann. Oplocks sind für SMB-Freigaben auf Storage Virtual Machines (SVMs) aktiviert. In manchen Fällen kann es sinnvoll sein, Oplocks zu deaktivieren. Oplocks können für jede Freigabe einzeln aktiviert oder deaktiviert werden.

.Über diese Aufgabe
Wenn Oplocks auf dem Volume, das eine Freigabe enthält, aktiviert sind, die Oplock-Freigabeeigenschaft für diese Freigabe jedoch deaktiviert ist, sind Oplocks für diese Freigabe deaktiviert. Die Deaktivierung von Oplocks für eine Freigabe hat Vorrang vor der Oplock-Einstellung des Volumes. Durch die Deaktivierung von Oplocks für die Freigabe werden sowohl opportunistische als auch Lease-Oplocks deaktiviert.

Neben der Angabe der Oplock-Freigabeeigenschaft können auch weitere Freigabeeigenschaften mithilfe einer durch Kommas getrennten Liste angegeben werden. Es können auch andere Freigabeparameter angegeben werden.

.Schritte
. Die entsprechende Maßnahme ausführen:
+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Oplocks bei der Freigabeerstellung aktivieren
 a| 
Geben Sie den folgenden Befehl ein: `+vserver cifs share create -vserver _vserver_name_ -share-name share_name -path path_to_share -share-properties [oplocks,...]+`

[NOTE]
====
Wenn die Freigabe nur die Standardeigenschaften  `oplocks`,  `browsable` und  `changenotify` aktiviert haben soll, muss der  `-share-properties` Parameter beim Erstellen einer SMB-Freigabe nicht angegeben werden. Wenn eine andere Kombination von Freigabeeigenschaften als die Standardwerte gewünscht ist, muss der  `-share-properties` Parameter mit der Liste der zu verwendenden Freigabeeigenschaften für diese Freigabe angegeben werden.

====


 a| 
Oplocks während der Freigabeerstellung deaktivieren
 a| 
Geben Sie den folgenden Befehl ein: `+vserver cifs share create -vserver _vserver_name_ -share-name _share_name_ -path _path_to_share_ -share-properties [other_share_property,...]+`

[NOTE]
====
Wenn Oplocks deaktiviert werden, muss beim Erstellen der Freigabe eine Liste von Freigabeeigenschaften angegeben werden, die  `oplocks` Eigenschaft sollte jedoch nicht angegeben werden.

====
|===
