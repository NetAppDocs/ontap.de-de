= Zusätzliche ONTAP SMB Home-Verzeichniskonfigurationen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können zusätzliche Home-Verzeichniskonfigurationen mithilfe der  `%w`,  `%d` und  `%u` Variablen erstellen, wodurch die Home-Verzeichniskonfiguration an Ihre Bedürfnisse angepasst werden kann.

Sie können verschiedene Home-Verzeichnis-Konfigurationen erstellen, indem Sie Variablen und statische Zeichenketten in den Freigabenamen und Suchpfaden kombinieren. Die folgende Tabelle enthält einige Beispiele, die die Erstellung unterschiedlicher Home-Verzeichnis-Konfigurationen veranschaulichen:

|===
| Pfade, die erstellt werden, wenn  `/vol1/user` Home-Verzeichnisse enthalten sind... | Share-Befehl... 


 a| 
Einen Freigabepfad erstellen `\\vs1\~win_username`, der den Benutzer zu `/vol1/user/win_username` führt
 a| 
`vserver cifs share create -share-name ~%w -path %w -share-properties oplocks,browsable,changenotify,homedirectory`



 a| 
Einen Freigabepfad erstellen `\\vs1\win_username`, der den Benutzer zu `/vol1/user/domain/win_username` führt
 a| 
`vserver cifs share create -share-name %w -path %d/%w -share-properties oplocks,browsable,changenotify,homedirectory`



 a| 
Einen Freigabepfad erstellen `\\vs1\win_username`, der den Benutzer zu `/vol1/user/unix_username` führt
 a| 
`vserver cifs share create -share-name %w -path %u -share-properties oplocks,browsable,changenotify,homedirectory`



 a| 
Einen Freigabepfad erstellen `\\vs1\unix_username`, der den Benutzer zu `/vol1/user/unix_username` führt
 a| 
`vserver cifs share create -share-name %u -path %u -share-properties oplocks,browsable,changenotify,homedirectory`

|===
