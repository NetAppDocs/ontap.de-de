= ONTAP SMB-Anforderungen für die Verwendung von Microsoft Previous Versions
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie „Vorherige Versionen“ mit Ihrem CIFS-Server verwenden können, müssen Sie wissen, welche Versionen von ONTAP und SMB sowie welche Windows-Clients dies unterstützen. Sie müssen außerdem die Anforderung an die Snapshot-Einstellung kennen.



[[ontap-version-requirements]]
== ONTAP Versionsanforderungen

Unterstützt frühere Versionen.



[[smb-protocol-version-requirements]]
== Anforderungen an die SMB-Protokollversion

Für Storage Virtual Machines (SVM) unterstützt ONTAP Previous Versions auf allen Versionen von SMB.



[[windows-client-requirements]]
== Windows Client Anforderungen

Bevor ein Benutzer mit der Funktion „Vorherige Versionen“ auf Daten in Snapshots zugreifen kann, muss der Windows Client diese Funktion unterstützen.

Die aktuellsten Informationen darüber, welche Windows Clients Previous Versions unterstützen, sind in der Interoperabilitätsmatrix zu finden.

https://mysupport.netapp.com/matrix["NetApp Interoperabilitätsmatrix-Tool"^]



[[requirements-for-snapshot-settings]]
== Anforderungen an Snapshot-Einstellungen

Um mit Hilfe von „Vorherige Versionen“ auf Daten in Snapshots zugreifen zu können, muss dem Volume, das die Daten enthält, eine aktivierte Snapshot-Richtlinie zugeordnet sein, Clients müssen auf die Snapshot-Daten zugreifen können und Snapshots müssen vorhanden sein.
