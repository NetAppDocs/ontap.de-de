= Freigabeeigenschaften zu bestehenden ONTAP SMB-Freigaben hinzufügen oder entfernen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine bestehende SMB-Freigabe anpassen, indem Sie Freigabeeigenschaften hinzufügen oder entfernen.

Die verfügbaren Freigabeeigenschaften sind wie folgt:

[cols="2,4"]
|===
| Freigabeeigenschaften | Beschreibung 


 a| 
`oplocks`
 a| 
Diese Eigenschaft legt fest, dass die Freigabe opportunistische Sperren verwendet, auch bekannt als Client-seitiges Caching.



 a| 
`browsable`
 a| 
Diese Eigenschaft ermöglicht Windows-Clients das Durchsuchen der Freigabe.



 a| 
`showsnapshot`
 a| 
Diese Eigenschaft legt fest, dass Snapshots von Clients angezeigt und durchsucht werden können.



 a| 
`changenotify`
 a| 
Diese Eigenschaft gibt an, dass die Freigabe Change Notify-Anforderungen unterstützt. Für Freigaben auf einer SVM ist dies eine standardmäßige Anfangseigenschaft.



 a| 
`attributecache`
 a| 
Diese Eigenschaft aktiviert das Caching von Dateiattributen auf der SMB-Freigabe, um einen schnelleren Zugriff auf Attribute zu ermöglichen. Standardmäßig ist das Attribut-Caching deaktiviert. Diese Eigenschaft sollte nur aktiviert werden, wenn Clients über SMB 1.0 auf Freigaben zugreifen. Diese Freigabeeigenschaft ist nicht anwendbar, wenn Clients über SMB 2.x oder SMB 3.0 auf Freigaben zugreifen.



 a| 
`continuously-available`
 a| 
Diese Eigenschaft ermöglicht es SMB-Clients, die sie unterstützen, Dateien dauerhaft zu öffnen. Dateien, die auf diese Weise geöffnet werden, sind vor Störungen wie Failover und Giveback geschützt.



 a| 
`branchcache`
 a| 
Diese Eigenschaft legt fest, dass die Freigabe es Clients ermöglicht, BranchCache-Hashwerte für die Dateien innerhalb dieser Freigabe anzufordern. Diese Option ist nur relevant, wenn „per-share“ als Betriebsmodus in der CIFS BranchCache-Konfiguration angegeben wird.



 a| 
`access-based-enumeration`
 a| 
Diese Eigenschaft legt fest, dass _Access Based Enumeration_ (ABE) für diese Freigabe aktiviert ist. ABE-gefilterte freigegebene Ordner sind für einen Benutzer basierend auf dessen individuellen Zugriffsrechten sichtbar, wodurch die Anzeige von Ordnern oder anderen freigegebenen Ressourcen verhindert wird, auf die der Benutzer keine Zugriffsrechte hat.



 a| 
`namespace-caching`
 a| 
Diese Eigenschaft legt fest, dass SMB-Clients, die sich mit dieser Freigabe verbinden, die von den CIFS-Servern zurückgegebenen Ergebnisse der Verzeichnisauflistung zwischenspeichern können, was die Leistung verbessern kann. Standardmäßig speichern SMB 1 Clients keine Ergebnisse der Verzeichnisauflistung. Da SMB 2 und SMB 3 Clients die Ergebnisse der Verzeichnisauflistung standardmäßig zwischenspeichern, bietet die Angabe dieser Freigabeeigenschaft nur SMB 1 Clientverbindungen Leistungsvorteile.



 a| 
`encrypt-data`
 a| 
Diese Eigenschaft legt fest, dass beim Zugriff auf diese Freigabe SMB-Verschlüsselung verwendet werden muss. SMB-Clients, die keine Verschlüsselung beim Zugriff auf SMB-Daten unterstützen, haben keinen Zugriff auf diese Freigabe.

|===


[[add-share-properties-on-existing-ontap-smb-shares]]
== Fügen Sie Freigabeeigenschaften zu bestehenden ONTAP SMB-Shares hinzu

Sie können eine bestehende SMB-Freigabe anpassen, indem Sie Freigabeeigenschaften hinzufügen. Dies kann hilfreich sein, wenn Sie die Freigabekonfiguration an geänderte Anforderungen in Ihrer Umgebung anpassen möchten.

.Bevor Sie beginnen
Die Freigabe, deren Eigenschaften Sie ändern möchten, muss vorhanden sein.

.Über diese Aufgabe
* Es besteht die Möglichkeit, eine oder mehrere Freigabeeigenschaften mithilfe einer durch Kommas getrennten Liste hinzuzufügen.
* Alle zuvor festgelegten Share-Eigenschaften bleiben bestehen.
+
Neu hinzugefügte Eigenschaften werden der bestehenden Liste der Share-Eigenschaften hinzugefügt.

* Wenn für bereits auf die Freigabe angewendete Freigabeeigenschaften ein neuer Wert angegeben wird, ersetzt der neu angegebene Wert den ursprünglichen Wert.
* Sie können Freigabeeigenschaften mit dem  `vserver cifs share properties add`Befehl nicht entfernen.
+
Mit dem  `vserver cifs share properties remove` Befehl lassen sich Freigabeeigenschaften entfernen.



.Schritte
. Freigabeeigenschaften hinzufügen:
+
[source, cli]
----
vserver cifs share properties add -vserver <SVM_name> -share-name <share_name> -share-properties <properties>,...
----
. Überprüfen Sie die Einstellungen der Freigabeeigenschaften:
+
[source, cli]
----
vserver cifs share show -vserver vserver_name -share-name share_name
----


.Beispiel
Der folgende Befehl fügt die  `showsnapshot`Share-Eigenschaft zu einer Freigabe mit dem Namen "`share1`" auf SVM vs1 hinzu:

[listing]
----
cluster1::> vserver cifs share properties add -vserver vs1 -share-name share1 -share-properties showsnapshot

cluster1::> vserver cifs share show -vserver vs1
Vserver    Share   Path      Properties   Comment    ACL
---------  ------  --------  ----------   --------   -----------
vs1        share1  /share1   oplocks      -          Everyone / Full Control
                             browsable
                             changenotify
                             showsnapshot
----

== Entfernen Sie die Freigabeeigenschaften bestehender ONTAP SMB-Freigaben

Sie können eine bestehende SMB-Freigabe anpassen, indem Sie Freigabeeigenschaften entfernen. Dies kann hilfreich sein, wenn Sie die Freigabekonfiguration an geänderte Anforderungen in Ihrer Umgebung anpassen möchten.

.Über diese Aufgabe
* Sie können eine oder mehrere Freigabeeigenschaften mithilfe einer durch Kommas getrennten Liste entfernen.
* Alle zuvor festgelegten, aber nicht entfernten Freigabeeigenschaften bleiben weiterhin wirksam.


.Schritte
. Eigenschaften der Freigabe entfernen:
+
[source, cli]
----
vserver cifs share properties remove -vserver <SVM_name> -share-name <share_name> -share-properties <properties>,...
----
. Überprüfen Sie die Einstellungen der Freigabeeigenschaften:
+
[source, cli]
----
vserver cifs share show -vserver <SVM_name> -share-name <share_name>
----


.Beispiel
Der folgende Befehl entfernt die  `browsable` Freigabeeigenschaft von einer Freigabe mit dem Namen "`share2`" auf SVM vs1:

[listing]
----
cluster1::> vserver cifs share properties remove -vserver vs1 -share-name share2 -share-properties browsable

cluster1::> vserver cifs share show -vserver vs1
Vserver    Share   Path      Properties   Comment    ACL
---------  ------  --------  ----------   --------   -----------
vs1        share2  /share2   oplocks      -          Everyone / Full Control
                             changenotify
----
