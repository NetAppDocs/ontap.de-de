= Informationen zu Leistungsverbesserungen von Microsoft Remote Copy auf ONTAP SMB-Servern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Microsoft Offloaded Data Transfer (ODX), auch bekannt als Copy Offload, ermöglicht direkte Datenübertragungen innerhalb oder zwischen kompatiblen Speichermedien, ohne dass die Daten über den Host-Computer übertragen werden.

ONTAP unterstützt ODX sowohl für das SMB- als auch für das SAN-Protokoll. Die Quelle kann entweder ein CIFS-Server oder eine LUN sein, und das Ziel kann ebenfalls ein CIFS-Server oder eine LUN sein.

Bei Dateiübertragungen ohne ODX werden die Daten von der Quelle gelesen und über das Netzwerk zum Client-Computer übertragen. Der Client-Computer überträgt die Daten dann über das Netzwerk zurück zum Ziel. Zusammengefasst liest der Client-Computer die Daten von der Quelle und schreibt sie zum Ziel. Bei ODX-Dateiübertragungen werden die Daten direkt von der Quelle zum Ziel kopiert.

Da ODX-Offloaded-Kopien direkt zwischen Quell- und Zielspeicher durchgeführt werden, ergeben sich erhebliche Leistungsvorteile. Zu den realisierten Leistungsvorteilen zählen eine schnellere Kopierzeit zwischen Quelle und Ziel, eine geringere Ressourcenauslastung (CPU, Speicher) auf dem Client sowie eine reduzierte Auslastung der Netzwerk-I/O-Bandbreite.

In SMB-Umgebungen ist diese Funktionalität nur verfügbar, wenn sowohl der Client als auch der Speicherserver SMB 3.0 und die ODX-Funktion unterstützen. In SAN-Umgebungen ist diese Funktionalität nur verfügbar, wenn sowohl der Client als auch der Speicherserver die ODX-Funktion unterstützen. Clientcomputer, die ODX unterstützen und bei denen ODX aktiviert ist, nutzen beim Verschieben oder Kopieren von Dateien automatisch und transparent die ausgelagerte Dateiübertragung. ODX wird unabhängig davon verwendet, ob Dateien per Drag-and-Drop im Windows Explorer verschoben werden, Befehlszeilenbefehle zum Kopieren genutzt werden oder ob eine Clientanwendung Dateikopieranforderungen initiiert.
