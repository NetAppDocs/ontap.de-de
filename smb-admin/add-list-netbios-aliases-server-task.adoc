= NetBIOS-Aliaslisten zu ONTAP SMB-Servern hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn SMB-Clients sich über einen Alias mit dem SMB-Server verbinden sollen, kann eine Liste von NetBIOS-Aliasen erstellt oder NetBIOS-Aliase zu einer bestehenden Liste von NetBIOS-Aliasen hinzugefügt werden.

.Über diese Aufgabe
* Der NetBIOS-Aliasname darf maximal 15 Zeichen lang sein.
* Auf dem SMB-Server können bis zu 200 NetBIOS-Aliase konfiguriert werden.
* Folgende Zeichen sind nicht zulässig:
+
@ # * ( ) = + [ ] | ; : " , < > \ / ?



.Schritte
. Die NetBIOS-Aliase hinzufügen: +
`vserver cifs add-netbios-aliases -vserver _vserver_name_ -netbios-aliases _NetBIOS_alias_,...`
+
`vserver cifs add-netbios-aliases -vserver vs1 -netbios-aliases alias_1,alias_2,alias_3`

+
** Sie können einen oder mehrere NetBIOS-Aliase mithilfe einer durch Kommas getrennten Liste angeben.
** Die angegebenen NetBIOS-Aliase werden der bestehenden Liste hinzugefügt.
** Eine neue Liste von NetBIOS-Aliasen wird erstellt, wenn die Liste derzeit leer ist.


. Es sollte überprüft werden, ob die NetBIOS-Aliase korrekt hinzugefügt wurden: `vserver cifs show -vserver vserver_name -display-netbios-aliases`
+
`vserver cifs show -vserver vs1 -display-netbios-aliases`

+
[listing]
----
Vserver: vs1

         Server Name: CIFS_SERVER
         NetBIOS Aliases: ALIAS_1, ALIAS_2, ALIAS_3
----
