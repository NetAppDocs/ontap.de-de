= DAC für ONTAP SMB-Server aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Option, mit der Sie Dynamic Access Control (DAC) zum Schutz von Objekten auf Ihrem CIFS-Server verwenden können, ist standardmäßig deaktiviert. Die Option muss aktiviert werden, wenn Dynamic Access Control auf dem CIFS-Server verwendet werden soll. Falls später entschieden wird, Dynamic Access Control nicht mehr zum Schutz von auf dem CIFS-Server gespeicherten Objekten zu verwenden, kann die Option deaktiviert werden.

Informationen darüber, wie die dynamische Zugriffssteuerung in Active Directory konfiguriert wird, sind in der Microsoft TechNet Library zu finden.

http://technet.microsoft.com/library/hh831717.aspx["Microsoft TechNet: Übersicht über Szenarien zur dynamischen Zugriffskontrolle"^]

.Über diese Aufgabe
Sobald die dynamische Zugriffskontrolle aktiviert ist, kann das Dateisystem ACLs mit Einträgen zur dynamischen Zugriffskontrolle enthalten. Wenn die dynamische Zugriffskontrolle deaktiviert ist, werden die aktuellen Einträge zur dynamischen Zugriffskontrolle ignoriert und neue werden nicht zugelassen.

Diese Option ist nur auf der Stufe „Erweitert“ verfügbar.

.Schritt
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: `set -privilege advanced`
. Führen Sie eine der folgenden Aktionen aus:
+
|===


| Wenn Sie möchten, dass die Dynamic Access Control... | Geben Sie den Befehl ein... 


 a| 
Aktiviert
 a| 
`vserver cifs options modify -vserver _vserver_name_ -is-dac-enabled true`



 a| 
Deaktiviert
 a| 
`vserver cifs options modify -vserver _vserver_name_ -is-dac-enabled false`

|===
. Zurück zur Administratorrechteebene: `set -privilege admin`
