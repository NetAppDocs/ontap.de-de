= Aktivieren oder Deaktivieren von ONTAP SMB Multidomain-Namenszuordnungssuchen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei der Suche nach domänenübergreifenden Namenszuordnungen kann im Domänenteil eines Windows-Namens beim Konfigurieren der Zuordnung von UNIX-Benutzernamen zu Windows-Benutzernamen ein Platzhalterzeichen (*) verwendet werden. Durch die Verwendung eines Platzhalterzeichens (*) im Domänenteil des Namens kann ONTAP alle Domänen durchsuchen, die eine bidirektionale Vertrauensstellung mit der Domäne haben, die das Computerkonto des CIFS-Servers enthält.

.Über diese Aufgabe
Als Alternative zur Suche in allen bidirektional vertrauenswürdigen Domänen kann eine Liste bevorzugter vertrauenswürdiger Domänen konfiguriert werden. Wenn eine Liste bevorzugter vertrauenswürdiger Domänen konfiguriert ist, verwendet ONTAP diese Liste anstelle der ermittelten bidirektional vertrauenswürdigen Domänen, um multidomain Namenszuordnungs-Suchen durchzuführen.

* Die Suche nach domänenübergreifenden Namenszuordnungen ist standardmäßig aktiviert.
* Diese Option ist auf der Stufe „Erweitertes Berechtigungsniveau“ verfügbar.


.Schritte
. Die Berechtigungsstufe wird auf „Erweitert“ gesetzt: `set -privilege advanced`
. Führen Sie eine der folgenden Aktionen aus:
+
|===
| Wenn Sie möchten, dass die Suche nach Multidomain-Namenszuordnungen durchgeführt wird... | Geben Sie den Befehl ein... 


 a| 
Aktiviert
 a| 
`vserver cifs options modify -vserver _vserver_name_ -is-trusted-domain-enum-search-enabled true`



 a| 
Deaktiviert
 a| 
`vserver cifs options modify -vserver _vserver_name_ -is-trusted-domain-enum-search-enabled false`

|===
. Zurück zur Administratorrechteebene: `set -privilege admin`
