= ONTAP SMB Snapshot-Konfigurationen erstellen, um den Zugriff auf frühere Versionen zu ermöglichen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Funktion „Vorherige Versionen“ ist immer verfügbar, sofern der Clientzugriff auf Snapshots aktiviert ist und Snapshots vorhanden sind. Wenn Ihre Snapshot-Konfiguration diese Anforderungen nicht erfüllt, kann eine Snapshot-Konfiguration erstellt werden, die diese Anforderungen erfüllt.

.Schritte
. Wenn dem Volume, das die Freigabe enthält, für die Sie den Zugriff auf frühere Versionen zulassen möchten, keine Snapshot-Richtlinie zugeordnet ist, muss dem Volume eine Snapshot-Richtlinie zugeordnet und diese mit dem  `volume modify` Befehl aktiviert werden.
+
Weitere Informationen zu  `volume modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-modify.html["ONTAP-Befehlsreferenz"^].

. Der Zugriff auf die Snapshots wird ermöglicht, indem der  `volume modify` Befehl verwendet wird, um die  `-snap-dir` Option auf  `true` zu setzen.
+
Weitere Informationen zu  `volume modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-modify.html["ONTAP-Befehlsreferenz"^].

. Es sollte überprüft werden, ob die Snapshot-Richtlinien aktiviert sind und der Zugriff auf die Snapshot-Verzeichnisse aktiviert ist, indem die  `volume show`- und  `volume snapshot policy show`-Befehle verwendet werden.
+
Weitere Informationen zu  `volume show` und  `volume snapshot policy show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=volume+show["ONTAP-Befehlsreferenz"^].

+
Weitere Informationen zum Konfigurieren und Verwalten von Snapshot-Richtlinien und Snapshot-Zeitplänen sind unter link:../data-protection/index.html["Datensicherung"] zu finden.
