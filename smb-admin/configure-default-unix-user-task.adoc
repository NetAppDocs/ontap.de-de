= Standardmäßigen ONTAP SMB UNIX Benutzer konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den Standard-UNIX-Benutzer konfigurieren, der verwendet wird, wenn alle anderen Zuordnungsversuche für einen Benutzer fehlschlagen oder wenn keine einzelnen Benutzer zwischen UNIX und Windows zugeordnet werden sollen. Alternativ sollte der Standard-UNIX-Benutzer nicht konfiguriert werden, wenn die Authentifizierung nicht zugeordneter Benutzer fehlschlagen soll.

.Über diese Aufgabe
Standardmäßig lautet der Name des Standard-UNIX-Benutzers „`pcuser`“, was bedeutet, dass die Benutzerzuordnung zum Standard-UNIX-Benutzer standardmäßig aktiviert ist. Ein anderer Name kann als Standard-UNIX-Benutzer angegeben werden. Der angegebene Name muss in den für die Storage Virtual Machine (SVM) konfigurierten Namensdienstdatenbanken vorhanden sein. Wenn diese Option auf eine leere Zeichenkette gesetzt ist, kann niemand als UNIX-Standardbenutzer auf den CIFS-Server zugreifen. Das bedeutet, dass jeder Benutzer ein Konto in der Passwortdatenbank haben muss, bevor auf den CIFS-Server zugegriffen werden kann.

Damit ein Benutzer eine Verbindung zum CIFS-Server mit dem Standard-UNIX-Benutzerkonto herstellen kann, müssen die folgenden Voraussetzungen erfüllt sein:

* Der Benutzer ist authentifiziert.
* Der Benutzer befindet sich in der lokalen Windows-Benutzerdatenbank des CIFS-Servers, in der Heimatdomäne des CIFS-Servers oder in einer vertrauenswürdigen Domäne (wenn die Suche nach multidomain Namenszuordnungen auf dem CIFS-Server aktiviert ist).
* Der Benutzername ist nicht explizit einer leeren Zeichenkette zugeordnet.


.Schritte
. Den Standard-UNIX-Benutzer konfigurieren:
+
|===
| Wenn Sie möchten ... | Geben Sie ... ein 


 a| 
Der Standard-UNIX-Benutzer "`pcuser`" wird verwendet.
 a| 
`vserver cifs options modify -default-unix-user pcuser`



 a| 
Ein anderes UNIX-Benutzerkonto als Standardbenutzer verwenden
 a| 
`vserver cifs options modify -default-unix-user _user_name_`



 a| 
Standard-UNIX-Benutzer deaktivieren
 a| 
`vserver cifs options modify -default-unix-user ""`

|===
+
`vserver cifs options modify -default-unix-user pcuser`

. Es sollte überprüft werden, ob der Standard-UNIX-Benutzer korrekt konfiguriert ist: `vserver cifs options show -vserver _vserver_name_`
+
Im folgenden Beispiel sind sowohl der Standard-UNIX-Benutzer als auch der Gast-UNIX-Benutzer auf SVM vs1 so konfiguriert, dass sie den UNIX-Benutzer "`pcuser`" verwenden:

+
`vserver cifs options show -vserver vs1`

+
[listing]
----

Vserver: vs1

  Client Session Timeout : 900
  Default Unix Group     : -
  Default Unix User      : pcuser
  Guest Unix User        : pcuser
  Read Grants Exec       : disabled
  Read Only Delete       : disabled
  WINS Servers           : -
----
