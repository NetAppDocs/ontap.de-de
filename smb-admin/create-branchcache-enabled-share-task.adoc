= Aktivieren Sie BranchCache für neue SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können BranchCache auf einer SMB-Freigabe aktivieren, wenn Sie die Freigabe erstellen, indem Sie die  `branchcache` Freigabeeigenschaft festlegen.

.Über diese Aufgabe
* Wenn BranchCache auf der SMB-Freigabe aktiviert ist, muss die Offline-Dateikonfiguration der Freigabe auf manuelles Caching eingestellt sein.
+
Dies ist die Standardeinstellung beim Erstellen einer Freigabe.

* Sie können beim Erstellen der BranchCache-aktivierten Freigabe auch zusätzliche optionale Freigabeparameter angeben.
* Sie können die  `branchcache`Eigenschaft für eine Freigabe festlegen, selbst wenn BranchCache auf der Storage Virtual Machine (SVM) nicht konfiguriert und aktiviert ist.
+
Wenn Sie jedoch möchten, dass die Freigabe zwischengespeicherte Inhalte anbietet, müssen Sie BranchCache auf der SVM konfigurieren und aktivieren.

* Da beim Verwenden des  `-share-properties` Parameters keine Standard-Freigabeeigenschaften auf die Freigabe angewendet werden, müssen alle anderen Freigabeeigenschaften, die zusätzlich zur  `branchcache` Freigabeeigenschaft auf die Freigabe angewendet werden sollen, durch eine durch Kommas getrennte Liste angegeben werden.
* Weitere Informationen zu  `vserver cifs share create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-cifs-share-create.html["ONTAP-Befehlsreferenz"^].


.Schritt
. Erstellen Sie eine BranchCache-fähige SMB-Freigabe:
+
[source, cli]
----
vserver cifs share create -vserver <SVM_name> -share-name <share_name> -path <path> -share-properties branchcache[,...]
----
. Überprüfen Sie, ob die BranchCache-Freigabeeigenschaft für die SMB-Freigabe festgelegt ist:
+
[source, cli]
----
vserver cifs share show
----


.Beispiel
Der folgende Befehl erstellt eine BranchCache-fähige SMB-Freigabe mit dem Namen „data“ und einem Pfad von `/data` auf SVM vs1. Standardmäßig ist die Einstellung für Offline-Dateien auf `manual` gesetzt:

[listing]
----
cluster1::> vserver cifs share create -vserver vs1 -share-name data -path /data -share-properties branchcache,oplocks,browsable,changenotify

cluster1::> vserver cifs share show -vserver vs1 -share-name data
