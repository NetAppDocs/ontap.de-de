= ONTAP SMB Home-Verzeichnis-Suchpfade hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie ONTAP SMB Home-Verzeichnisse verwenden möchten, muss mindestens ein Suchpfad für das Home-Verzeichnis hinzugefügt werden.

.Über diese Aufgabe
Ein Suchpfad für das Home-Verzeichnis kann mit dem  `vserver cifs home-directory search-path add` Befehl hinzugefügt werden.

Der  `vserver cifs home-directory search-path add` Befehl prüft den im  `-path` angegebenen Pfad während der Befehlsausführung. Wenn der angegebene Pfad nicht existiert, erzeugt der Befehl eine Meldung mit der Aufforderung, ob fortgefahren werden soll. Es kann  `y` oder  `n` ausgewählt werden. Wenn  `y` das Fortfahren gewählt wird, erstellt ONTAP den Suchpfad. Die Verzeichnisstruktur muss jedoch erstellt werden, bevor der Suchpfad in der Home Directory Konfiguration verwendet werden kann. Wenn das Fortfahren nicht gewählt wird, schlägt der Befehl fehl, der Suchpfad wird nicht erstellt. Anschließend kann die Verzeichnisstruktur für den Pfad erstellt und der  `vserver cifs home-directory search-path add` Befehl erneut ausgeführt werden.

.Schritte
. Fügen Sie einen Suchpfad für das Home-Verzeichnis hinzu: `vserver cifs home-directory search-path add -vserver vserver -path path`
. Es wird überprüft, ob der Suchpfad erfolgreich mit dem  `vserver cifs home-directory search-path show` Befehl hinzugefügt wurde.


.Beispiel
Das folgende Beispiel fügt den Pfad  `/home1` zur Home-Verzeichnis-Konfiguration auf SVM vs1 zur Home-Verzeichnis-Konfiguration hinzu.

[listing]
----
cluster::> vserver cifs home-directory search-path add -vserver vs1 -path /home1

vs1::> vserver cifs home-directory search-path show
Vserver     Position Path
----------- -------- -----------------
vs1         1        /home1
----
Im folgenden Beispiel wird versucht, den Pfad `/home2` zur Home-Verzeichniskonfiguration auf SVM vs1 hinzuzufügen. Der Pfad existiert nicht. Es wird entschieden, nicht fortzufahren.

[listing]
----
cluster::> vserver cifs home-directory search-path add -vserver vs1 -path /home2
Warning: The specified path "/home2" does not exist in the namespace
         belonging to Vserver "vs1".
Do you want to continue? {y|n}: n
----
