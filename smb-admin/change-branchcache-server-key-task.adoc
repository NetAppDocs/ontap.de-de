= Den ONTAP SMB BranchCache Server-Schlüssel ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den BranchCache Server-Schlüssel ändern, indem Sie die BranchCache Konfiguration auf der Storage Virtual Machine (SVM) modifizieren und einen anderen Server-Schlüssel angeben.

.Über diese Aufgabe
Sie können den Serverschlüssel auf einen bestimmten Wert setzen, sodass, falls mehrere Server BranchCache-Daten für dieselben Dateien bereitstellen, Clients Hashes von jedem Server mit demselben Serverschlüssel verwenden können.

Wenn Sie den Serverschlüssel ändern, müssen Sie auch den Hash-Cache leeren. Nach dem Leeren der Hashes erstellt ONTAP neue Hashes, sobald neue Anfragen von BranchCache-aktivierten Clients gestellt werden.

.Schritte
. Der Serverschlüssel kann mit folgendem Befehl geändert werden: `vserver cifs branchcache modify -vserver vserver_name -server-key text -flush-hashes true`
+
Bei der Konfiguration eines neuen Serverschlüssels müssen auch  `-flush-hashes` angegeben und der Wert auf  `true` gesetzt werden.

. Es kann überprüft werden, ob die BranchCache Konfiguration korrekt ist, indem der  `vserver cifs branchcache show` Befehl verwendet wird.


.Beispiel
Das folgende Beispiel legt einen neuen Serverschlüssel fest, der Leerzeichen enthält, und leert den Hash-Cache auf SVM vs1:

[listing]
----
cluster1::> vserver cifs branchcache modify -vserver vs1 -server-key "new vserver secret" -flush-hashes true

cluster1::> vserver cifs branchcache show -vserver vs1

                                 Vserver: vs1
          Supported BranchCache Versions: enable_all
                      Path to Hash Store: /hash_data
          Maximum Size of the Hash Store: 20GB
Encryption Key Used to Secure the Hashes: -
        CIFS BranchCache Operating Modes: per_share
----
