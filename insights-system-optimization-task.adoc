= Das System lässt sich mit den Erkenntnissen des ONTAP System Manager optimieren.
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Mit System Manager lassen sich Einblicke anzeigen, die bei der Optimierung des Systems unterstützen.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) link:https://docs.netapp.com/us-en/asa-r2/monitor/view-insights.html["diese Schritte"^] stehen Einblicke zur Verfügung, die bei der Optimierung des Systems unterstützen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

Ab ONTAP 9.11.1 sind im System Manager Einblicke verfügbar, die bei der Optimierung der Kapazität, der Sicherheitskonformität und der Konfiguration Ihres Systems unterstützen.

[IMPORTANT]
====
*Das Blockieren von Erweiterungen kann zu unerwarteten Ergebnissen führen.* Ab ONTAP 9.11.1 ist es möglich, native FPolicy für Storage-VMs mithilfe von System Manager zu aktivieren. Es kann eine System Manager Insight-Meldung angezeigt werden, in der empfohlen wird, link:insights-configure-native-fpolicy-task.html["nativ FPolicy konfigurieren"] für eine Storage-VM zu verwenden.

Im FPolicy Native Mode können bestimmte Dateierweiterungen zugelassen oder nicht zugelassen werden. System Manager empfiehlt über 3000 nicht zugelassene Dateierweiterungen, die in der Vergangenheit bei Ransomware-Angriffen verwendet wurden. Einige dieser Erweiterungen könnten von legitimen Dateien in Ihrer Umgebung verwendet werden, und deren Blockierung könnte zu unerwarteten Problemen führen.

Daher wird dringend empfohlen, die Liste der Erweiterungen an die Bedürfnisse Ihrer Umgebung anzupassen. Siehe  https://kb.netapp.com/onprem/ontap/da/NAS/How_to_remove_a_file_extension_from_a_native_FPolicy_configuration_created_by_System_Manager_using_System_Manager_to_recreate_the_policy["Entfernen einer Dateierweiterung aus einer nativ FPolicy Konfiguration, die mit System Manager erstellt wurde, durch Neuerstellung der Richtlinie mit System Manager"^].

Weitere Informationen zu nativ FPolicy finden Sie unter link:./nas-audit/fpolicy-config-types-concept.html["Fpolicy Konfigurationstypen"].

====
Basierend auf bewährten Verfahren werden diese Erkenntnisse auf einer Seite dargestellt, von der aus Sie sofort Maßnahmen zur Optimierung Ihres Systems einleiten können. Weitere Informationen finden sich unter link:./insights-system-optimization-task.html["System Manager Einblicke"].



[[view-optimization-insights]]
== Optimierungs-Einblicke anzeigen

.Schritte
. Im System Manager befindet sich *Insights* in der linken Navigationsspalte.
+
Die Seite *Insights* zeigt Gruppen von Insights an. Jede Gruppe von Insights kann ein oder mehrere Insights enthalten. Die folgenden Gruppen werden angezeigt:

+
** Benötigt Ihre Aufmerksamkeit
** Risiken beheben
** Speicher optimieren


. (Optional) Die angezeigten Erkenntnisse lassen sich filtern, indem diese Schaltflächen in der oberen rechten Ecke der Seite ausgewählt werden:
+
** image:icon-security-filter.gif["Sicherheitssymbol"]Zeigt die sicherheitsrelevanten Erkenntnisse an.
** image:icon-capacity-filter.gif["Symbol für Kapazitätsinformationen"]Zeigt die kapazitätsbezogenen Erkenntnisse an.
** image:icon-config-filter.gif["Symbol für Konfigurationsinformationen"]Zeigt die konfigurationsbezogenen Erkenntnisse an.
** image:icon-all-filter.png["Alle Einblicke Symbol"]Zeigt alle Insights an.






[[respond-to-insights-to-optimize-your-system]]
== Auf Erkenntnisse reagieren und das System optimieren

Im System Manager können Sie auf Erkenntnisse reagieren, indem Sie diese entweder verwerfen, verschiedene Möglichkeiten zur Behebung der Probleme erkunden oder den Prozess zur Behebung der Probleme einleiten.

.Schritte
. Im System Manager befindet sich *Insights* in der linken Navigationsspalte.
. Fahren Sie mit dem Mauszeiger über eine Insight, um die Schaltflächen für die folgenden Aktionen anzuzeigen:
+
** *Verwerfen*: Die Erkenntnis aus der Ansicht entfernen. Um die Erkenntnis wiederherzustellen, siehe <<customize-settings-insights>>.
** *Erkunden*: Verschiedene Möglichkeiten zur Behebung des im Insight genannten Problems werden aufgezeigt. Diese Schaltfläche erscheint nur, wenn es mehr als eine Methode zur Behebung gibt.
** *Lösung*: Der Prozess zur Behebung des im Insight genannten Problems wird eingeleitet. Es erfolgt eine Abfrage, ob die zur Anwendung der Lösung erforderliche Maßnahme durchgeführt werden soll.





NOTE: Einige dieser Aktionen können von anderen Seiten im System Manager aus initiiert werden, aber die Seite *Insights* hilft dabei, die täglichen Aufgaben zu optimieren, da diese Aktionen von dieser einen Seite aus initiiert werden können.

== Die Einstellungen für Insights anpassen

Sie können anpassen, über welche Erkenntnisse Sie im System Manager benachrichtigt werden.

.Schritte
. Im System Manager befindet sich *Insights* in der linken Navigationsspalte.
. In der oberen rechten Ecke der Seite auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und dann *Einstellungen* auswählen.
. Auf der Seite *Einstellungen* muss sichergestellt sein, dass in den Kästchen neben den Insights, über die Sie benachrichtigt werden möchten, ein Häkchen gesetzt ist. Wenn ein Insight zuvor verworfen wurde, kann es wiederhergestellt werden, indem sichergestellt wird, dass in seinem Kästchen ein Häkchen gesetzt ist.
. Auf *Speichern* klicken.




[[export-the-insights-as-a-pdf-file]]
== Die Erkenntnisse als PDF-Datei exportieren

Alle relevanten Erkenntnisse können als PDF-Datei exportiert werden.

.Schritte
. Im System Manager befindet sich *Insights* in der linken Navigationsspalte.
. In der oberen rechten Ecke der Seite auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und dann *Export* auswählen.
