= Umstellung von Onboard-Schlüsselverwaltung auf externe Schlüsselverwaltung in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie von der internen Schlüsselverwaltung auf die externe Schlüsselverwaltung umstellen möchten, müssen Sie die Konfiguration der internen Schlüsselverwaltung löschen, bevor Sie die externe Schlüsselverwaltung aktivieren können.

.Bevor Sie beginnen
* Für hardwarebasierte Verschlüsselung müssen die Datenschlüssel aller FIPS-Laufwerke oder Self-Encrypting Drives auf den Standardwert zurückgesetzt werden.
+
link:return-seds-unprotected-mode-task.html["Zurückstellen eines FIPS-Laufwerks oder SED in den ungeschützten Modus"]

* Bei softwarebasierter Verschlüsselung müssen alle Volumes entschlüsselt werden.
+
link:unencrypt-volume-data-task.html["Entschlüsselung der Volumendaten"]

* Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.


.Schritt
. Die Onboard-Schlüsselverwaltungskonfiguration für einen Cluster löschen:
+
[cols="35,65"]
|===


| Für diese ONTAP Version... | Verwenden Sie diesen Befehl... 


 a| 
ONTAP 9.6 und höher
 a| 
`security key-manager onboard disable -vserver SVM`



 a| 
ONTAP 9.5 und früher
 a| 
`security key-manager delete-key-database`

|===
+
Weitere Informationen zu  `security key-manager onboard disable` und  `security key-manager delete-key-database` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=security+key-manager["ONTAP-Befehlsreferenz"^].
