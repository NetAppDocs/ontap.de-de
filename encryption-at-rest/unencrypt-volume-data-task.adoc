= Volumendaten in ONTAP entschlüsseln
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der  `volume move start` Befehl kann verwendet werden, um Volumendaten zu verschieben und zu entschlüsseln.

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritte
. Ein vorhandenes verschlüsseltes Volume verschieben und die Daten auf dem Volume entschlüsseln:
+
`volume move start -vserver _SVM_name_ -volume volume_name -destination-aggregate _aggregate_name_ -encrypt-destination false`

+
Weitere Informationen zu  `volume move start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-move-start.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl verschiebt ein vorhandenes Volume mit dem Namen `vol1` in das Zielaggregat `aggr3` und entschlüsselt die Daten auf dem Volume:

+
[listing]
----
cluster1::> volume move start -vserver vs1 -volume vol1 -destination-aggregate aggr3 -encrypt-destination false
----
+
Das System löscht den Verschlüsselungsschlüssel für das Volume. Die Daten auf dem Volume sind unverschlüsselt.

. Es sollte überprüft werden, ob die Verschlüsselung des Volumes deaktiviert ist:
+
`volume show -encryption`

+
Weitere Informationen zu  `volume show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-show.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl zeigt an, ob Volumes auf `cluster1`verschlüsselt sind:

+
[listing]
----
cluster1::> volume show -encryption

Vserver  Volume   Aggregate   State    Encryption State
-------  ------   ---------   -----    ----------------
vs1      vol1     aggr1       online   none
----
