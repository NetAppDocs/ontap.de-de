= ONTAP FlexCache Volumes vorbelegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können ein FlexCache volume vorab befüllen, um die Zeit für den Zugriff auf zwischengespeicherte Daten zu verkürzen.

.Bevor Sie beginnen
* Sie müssen Cluster-Administrator auf der erweiterten Berechtigungsebene sein.
* Die Pfade, die Sie für die Vorbefüllung angeben, müssen existieren oder die Vorbefüllungsoperation schlägt fehl.


.Über diese Aufgabe
* Prepopulate liest nur Dateien und durchsucht Verzeichnisse
* Das  `-isRecursion`Flag gilt für die gesamte Liste der an prepopulate übergebenen Verzeichnisse


.Schritte
. Ein FlexCache Volume vorab befüllen:
+
`volume flexcache prepopulate -cache-vserver vserver_name -cache-volume -path-list _path_list_ -isRecursion true|false`

+
** Der  `-path-list` Parameter gibt den relativen Verzeichnispfad an, den Sie ausgehend vom Ursprungs-Stammverzeichnis vorausfüllen möchten. Wenn das Ursprungs-Stammverzeichnis beispielsweise /origin heißt und die Verzeichnisse /origin/dir1 und /origin/dir2 enthält, kann die Pfadliste wie folgt angegeben werden: `-path-list dir1, dir2` oder  `-path-list /dir1, /dir2`.
** Der Standardwert des  `-isRecursion`Parameters ist True.
+
Dieses Beispiel legt einen einzelnen Verzeichnispfad vorab fest:

+
[listing]
----
cluster1::*> flexcache prepopulate start -cache-vserver vs2 -cache-volume fg_cachevol_1 -path-list /dir1
  (volume flexcache prepopulate start)
[JobId 207]: FlexCache prepopulate job queued.
----
+
Dieses Beispiel füllt Dateien aus mehreren Verzeichnissen vorab aus:

+
[listing]
----
cluster1::*> flexcache prepopulate start -cache-vserver vs2 -cache-volume fg_cachevol_1 -path-list /dir1,/dir2,/dir3,/dir4
  (volume flexcache prepopulate start)
[JobId 208]: FlexCache prepopulate job queued.
----
+
Dieses Beispiel füllt eine einzelne Datei vorab aus:

+
[listing]
----
cluster1::*> flexcache prepopulate start -cache-vserver vs2 -cache-volume fg_cachevol_1 -path-list /dir1/file1.txt
  (volume flexcache prepopulate start)
[JobId 209]: FlexCache prepopulate job queued.
----
+
Dieses Beispiel füllt alle Dateien aus dem Ursprung vorab aus:

+
[listing]
----
cluster1::*> flexcache prepopulate start -cache-vserver vs2 -cache-volume fg_cachevol_1 -path-list / -isRecursion true
  (volume flexcache prepopulate start)
[JobId 210]: FlexCache prepopulate job queued.
----
+
Dieses Beispiel enthält einen ungültigen Pfad für die Vorbelegung:

+
[listing]
----
cluster1::*> flexcache prepopulate start -cache-volume vol_cache2_vs3_c2_vol_origin1_vs1_c1 -cache-vserver vs3_c2 -path-list /dir1, dir5, dir6
  (volume flexcache prepopulate start)

Error: command failed: Path(s) "dir5, dir6" does not exist in origin volume
       "vol_origin1_vs1_c1" in Vserver "vs1_c1".
----


. Anzahl der gelesenen Dateien anzeigen:
+
`job show -id job_ID -ins`
