= Informationen zur Überprüfung von ONTAP FlexCache Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.7 ist es möglich, NFS-Dateizugriffsereignisse in FlexCache Beziehungen mithilfe der nativen ONTAP-Auditing- und Dateirichtlinienverwaltung mit FPolicy zu protokollieren.

Ab ONTAP 9.14.1 wird FPolicy für FlexCache Volumes mit NFS oder SMB unterstützt. Zuvor wurde FPolicy für FlexCache Volumes mit SMB nicht unterstützt.

Natives Auditing und FPolicy werden mit denselben CLI-Befehlen konfiguriert und verwaltet wie FlexVol Volumes. Bei FlexCache Volumes gibt es jedoch einige Unterschiede im Verhalten.

* *Nativprüfung*
+
** Ein FlexCache Volume kann nicht als Ziel für Revisionsprotokolle verwendet werden.
** Wenn Sie Lese- und Schreibvorgänge auf FlexCache Volumes überwachen möchten, muss die Überwachung sowohl auf der Cache-SVM als auch auf der Origin-SVM konfiguriert werden.
+
Dies liegt daran, dass Dateisystemoperationen dort protokolliert werden, wo sie verarbeitet werden. Das heißt, Lesevorgänge werden auf der Cache-SVM und Schreibvorgänge auf der Origin-SVM protokolliert.

** Um den Ursprung von Schreibvorgängen nachzuverfolgen, werden die SVM-UUID und die MSID im Revisionsprotokoll angehängt, um das FlexCache Volume zu identifizieren, von dem der Schreibvorgang ausging.


* *FPolicy*
+
** Obwohl Schreibvorgänge auf einem FlexCache Volume auf dem Ursprungsvolume ausgeführt werden, überwachen FPolicy Konfigurationen die Schreibvorgänge auf dem Cache Volume. Dies unterscheidet sich von nativer Überwachung, bei der die Schreibvorgänge auf dem Ursprungsvolume überwacht werden.
** ONTAP erfordert zwar nicht dieselbe FPolicy-Konfiguration auf Cache- und Ursprungs-SVMs, es wird jedoch empfohlen, zwei ähnliche Konfigurationen bereitzustellen. Dies kann durch das Erstellen einer neuen FPolicy-Richtlinie für den Cache erfolgen, die wie die des Ursprungs-SVM konfiguriert ist, deren Geltungsbereich jedoch auf den Cache-SVM beschränkt ist.
** Die Größe von Erweiterungen in einer FPolicy-Konfiguration ist auf 20KB (20480 Byte) begrenzt. Wenn die Größe der in einer FPolicy-Konfiguration auf einem FlexCache Volume verwendeten Erweiterungen 20KB überschreitet, wird die EMS-Meldung `nblade.fpolicy.extn.failed` ausgelöst.
