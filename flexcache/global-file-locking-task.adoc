= Globale Dateisperrung auf ONTAP FlexCache Volumes aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.10.1 können Sie eine globale Dateisperrung auf FlexCache Volumes anwenden, um Lesezugriffe auf alle zugehörigen zwischengespeicherten Dateien zu verhindern.

.Bevor Sie beginnen
* Die globale Dateisperrung erfordert, dass auf den Clustern, die den Ursprung und alle zugehörigen Caches enthalten, ONTAP 9.9.1 oder höher ausgeführt wird. Die globale Dateisperrung kann auf neuen oder bestehenden FlexCache volumes aktiviert werden. Der Befehl kann auf einem Volume ausgeführt werden und gilt für alle zugehörigen FlexCache volumes.
* Sie müssen sich auf der erweiterten Berechtigungsebene befinden, um die globale Dateisperrung zu aktivieren.
* Wenn Sie auf eine Version von ONTAP vor 9.9.1 zurückkehren, muss die globale Dateisperrung zuerst auf dem Ursprung und den zugehörigen Caches deaktiviert werden. Zum Deaktivieren wird vom Ursprungsvolume folgender Befehl ausgeführt: `volume flexcache prepare-to-downgrade -disable-feature-set 9.10.0`
* Der Prozess zur Aktivierung der globalen Dateisperrung hängt davon ab, ob der Ursprung über vorhandene Caches verfügt:
+
** <<enable-gfl-new>>
** <<enable-gfl-existing>>






[[enable-global-file-locking-on-new-flexcache-volumes]]
[[enable-gfl-new]]
== Globale Dateisperrung auf neuen FlexCache Volumes aktivieren

.Schritte
. Das FlexCache Volume wird mit  `-is-global-file-locking` auf „true“ erstellt:
+
`volume flexcache create volume _volume_name_ -is-global-file-locking-enabled true`

+

NOTE: Der Standardwert von `-is-global-file-locking` ist "`false`". Wenn nachfolgende `volume flexcache create` Befehle auf einem Volume ausgeführt werden, muss  `-is-global-file-locking enabled` auf "`true`" gesetzt werden.





[[enable-global-file-locking-on-existing-flexcache-volumes]]
[[enable-gfl-existing]]
== Globale Dateisperrung auf vorhandenen FlexCache Volumes aktivieren

.Schritte
. Die globale Dateisperrung muss vom Ursprungsvolume aus festgelegt werden.
. Der Ursprung darf keine anderen bestehenden Beziehungen haben (zum Beispiel SnapMirror). Alle bestehenden Beziehungen müssen getrennt werden. Alle Caches und Volumes müssen zum Zeitpunkt der Befehlsausführung verbunden sein. Der Verbindungsstatus kann mit folgendem Befehl überprüft werden:
+
`volume flexcache connection-status show`

+
Der Status aller aufgeführten Datenträger sollte wie folgt angezeigt werden `connected.`. Weitere Informationen finden sich unter link:view-connection-status-origin-task.html["Status einer FlexCache Beziehung anzeigen"] oder link:synchronize-properties-origin-volume-task.html["Eigenschaften eines FlexCache Volumens von einem Ursprung synchronisieren"].

. Globale Dateisperrung für die Caches aktivieren:
+
`volume flexcache origin config show/modify -volume _volume_name_ -is-global-file-locking-enabled true`
