= ONTAP Vscan On-Demand-Aufgaben werden sofort ausgeführt
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine Aufgabe auf Abruf sofort ausführen, unabhängig davon, ob ihr ein Zeitplan zugewiesen wurde oder nicht.

.Bevor Sie beginnen
Die Scanfunktion muss auf dem SVM aktiviert sein.

.Schritt
. Eine Ad-hoc-Aufgabe sofort ausführen:
+
`vserver vscan on-demand-task run -vserver _data_SVM_ -task-name _task_name_`

+
Der folgende Befehl führt eine On-Access-Aufgabe mit dem Namen  `Task1` auf der  `vs1` SVM aus:

+
[listing]
----
cluster1::> vserver vscan on-demand-task run -vserver vs1 -task-name Task1
[Job 161]: Vscan On-Demand job is queued. Use the "job show -id 161" command to view the status.
----
+
Weitere Informationen zu  `vserver vscan on-demand-task run` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-on-demand-task-run.html["ONTAP-Befehlsreferenz"^].

+

NOTE: Sie können den  `job show` Befehl verwenden, um den Status des Auftrags anzuzeigen. Sie können die  `job pause` und  `job resume` Befehle verwenden, um den Auftrag anzuhalten und neu zu starten, oder den  `job stop` Befehl, um den Auftrag zu beenden. Weitere Informationen zu  `job` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=job["ONTAP-Befehlsreferenz"^].
