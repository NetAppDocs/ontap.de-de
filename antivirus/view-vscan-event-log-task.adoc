= Vscan-Ereignisprotokollinformationen mit ONTAP anzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der  `vserver vscan show-events` Befehl kann verwendet werden, um Ereignisprotokollinformationen zu infizierten Dateien, Aktualisierungen von Vscan Servern und Ähnlichem anzuzeigen. Ereignisinformationen können für den Cluster oder für bestimmte Nodes, SVMs oder Vscan Server angezeigt werden.

.Bevor Sie beginnen
Für die Anzeige des Vscan-Ereignisprotokolls sind erweiterte Berechtigungen erforderlich.

.Schritte
. Wechsel zum erweiterten Berechtigungslevel:
+
`set -privilege advanced`

+
Weitere Informationen zu  `set` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/set.html["ONTAP-Befehlsreferenz"^].

. Vscan-Ereignisprotokollinformationen:
+
`vserver vscan show-events`

+
Weitere Informationen zu  `vserver vscan show-events` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-show-events.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl zeigt Ereignisprotokollinformationen für den Cluster  `cluster1` an:

+
[listing]
----
cluster1::*> vserver vscan show-events

Vserver     Node            Server          Event Type        Event Time
----------- --------------- --------------- ----------------- -----------------
vs1         Cluster-01      192.168.1.1     file-infected     9/5/2014 11:37:38
vs1         Cluster-01      192.168.1.1     scanner-updated   9/5/2014 11:37:08
vs1         Cluster-01      192.168.1.1     scanner-connected 9/5/2014 11:34:55
3 entries were displayed.
----
