= ONTAP Vscan Scannerpools in MetroCluster Konfigurationen erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen auf jedem Cluster in einer MetroCluster Konfiguration primäre und sekundäre Scannerpools erstellen, die den primären und sekundären SVMs auf dem Cluster entsprechen.

.Bevor Sie beginnen
* SVMs und Vscan Server müssen sich in derselben Domäne oder in vertrauenswürdigen Domänen befinden.
* Für Scannerpools, die für eine einzelne SVM definiert sind, muss ONTAP Antivirus Connector mit dem SVM-Management-LIF oder dem SVM-Daten-LIF konfiguriert sein.
* Für Scannerpools, die für alle SVMs in einem Cluster definiert sind, muss ONTAP Antivirus Connector mit dem Cluster-Management-LIF konfiguriert sein.
* Die Liste der privilegierten Benutzer muss das Domänenbenutzerkonto enthalten, das der Vscan Server für die Verbindung zur SVM verwendet.
* Sobald der Scannerpool konfiguriert ist, lässt sich der Verbindungsstatus zu den Servern überprüfen.


.Über diese Aufgabe
MetroCluster Konfigurationen schützen Daten durch die Implementierung zweier physisch getrennter, gespiegelter Cluster. Jeder Cluster repliziert synchron die Daten und die SVM-Konfiguration des anderen. Eine primäre SVM im lokalen Cluster stellt Daten bereit, wenn der Cluster online ist. Eine sekundäre SVM im lokalen Cluster stellt Daten bereit, wenn der entfernte Cluster offline ist.

Das bedeutet, dass Sie in einer MetroCluster Konfiguration auf jedem Cluster primäre und sekundäre Scannerpools erstellen müssen, der sekundäre Pool wird aktiv, wenn der Cluster beginnt, Daten vom sekundären SVM bereitzustellen. Für Disaster Recovery (DR) ist die Konfiguration ähnlich wie bei MetroCluster.

Diese Abbildung zeigt eine typische MetroCluster/DR Konfiguration.

image:metrocluster-av-config.png["MetroCluster/DR Konfiguration"]

.Schritte
. Einen Scannerpool erstellen:
+
`vserver vscan scanner-pool create -vserver <data_SVM|cluster_admin_SVM> -scanner-pool <scanner_pool> -hostnames <Vscan_server_hostnames> -privileged-users <privileged_users>`

+
** Geben Sie eine Daten-SVM für einen Pool an, der für eine einzelne SVM definiert ist, und eine Cluster-Admin-SVM für einen Pool, der für alle SVMs in einem Cluster definiert ist.
** Für jeden Vscan Server Hostnamen ist eine IP-Adresse oder ein FQDN anzugeben.
** Die Domäne und den Benutzernamen für jeden privilegierten Benutzer angeben.


+
[NOTE]
====
Alle Scanner-Pools sind aus dem Cluster zu erstellen, das die primäre SVM enthält.

====
+
Weitere Informationen zu  `vserver vscan scanner-pool create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-create.html["ONTAP-Befehlsreferenz"^].

+
Die folgenden Befehle erstellen primäre und sekundäre Scannerpools auf jedem Cluster in einer MetroCluster Konfiguration:

+
[listing]
----
site_A::> vserver vscan scanner-pool create -vserver cifssvm1 -
scanner-pool pool1_for_siteA -hostnames scan1 -privileged-users cifs
\u1,cifs\u2

site_B::> vserver vscan scanner-pool create -vserver cifssvm1 -
scanner-pool pool1_for_siteB -hostnames scan1 -privileged-users cifs
\u1,cifs\u2

site_A::> vserver vscan scanner-pool create -vserver cifssvm1 -
scanner-pool pool2_for_siteA -hostnames scan2 -privileged-users cifs
\u1,cifs\u2

site_B::> vserver vscan scanner-pool create -vserver cifssvm1 -
scanner-pool pool2_for_siteB -hostnames scan2 -privileged-users cifs
\u1,cifs\u2
----
. Es sollte überprüft werden, ob die Scannerpools erstellt wurden:
+
`vserver vscan scanner-pool show -vserver <data_SVM|cluster_admin_SVM> -scanner-pool <scanner_pool>`

+
Der folgende Befehl zeigt die Details für den Scannerpool an `pool1_for_siteA`:

+
[listing]
----
site_A::> vserver vscan scanner-pool show -vserver cifssvm1 -scanner-pool pool1_for_siteA

                                    Vserver: cifssvm1
                               Scanner Pool: pool1_for_siteA
                             Applied Policy: idle
                             Current Status: off
         Cluster on Which Policy Is Applied: -
                  Scanner Pool Config Owner: vserver
       List of IPs of Allowed Vscan Servers:
List of Host Names of Allowed Vscan Servers: scan1
                   List of Privileged Users: cifs\u1,cifs\u2
----
+
Sie können auch den  `vserver vscan scanner-pool show`Befehl verwenden, um alle Scanner-Pools auf einer SVM anzuzeigen. Weitere Informationen zu  `vserver vscan scanner-pool show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-show.html["ONTAP-Befehlsreferenz"^].
