= ONTAP Vscan On-Access-Richtlinien aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine On-Access-Richtlinie definiert den Umfang eines On-Access-Scans. Eine On-Access-Richtlinie muss auf einer SVM aktiviert werden, bevor deren Dateien gescannt werden können.

Wenn Sie eine On-Access-Richtlinie für alle SVMs in einem Cluster erstellt haben, muss die Richtlinie auf jeder SVM einzeln aktiviert werden. Auf einer SVM kann jeweils nur eine On-Access-Richtlinie aktiviert werden.

.Schritte
. Eine On-Access-Richtlinie aktivieren:
+
`vserver vscan on-access-policy enable -vserver data_SVM -policy-name policy_name`

+
Der folgende Befehl aktiviert eine On-Access-Richtlinie namens  `Policy1` auf der  `vs1` SVM:

+
[listing]
----
cluster1::> vserver vscan on-access-policy enable -vserver vs1 -policy-name Policy1
----
. Es sollte überprüft werden, ob die On-Access-Richtlinie aktiviert ist:
+
`vserver vscan on-access-policy show -instance _data_SVM_ -policy-name _policy_name_`

+
Weitere Informationen zu  `vserver vscan on-access-policy show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-on-access-policy-show.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl zeigt die Details der  `Policy1`On-Access-Richtlinie an:

+
[listing]
----
cluster1::> vserver vscan on-access-policy show -instance vs1 -policy-name Policy1

                            Policy: Policy1
                     Policy Status: on
               Policy Config Owner: vserver
