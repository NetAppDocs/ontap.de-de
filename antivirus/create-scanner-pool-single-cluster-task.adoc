= Einen ONTAP Vscan Scannerpool auf einem einzelnen Cluster erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Scanner Pool definiert die Vscan Server und privilegierten Benutzer, die eine Verbindung zu SVMs herstellen können.

.Bevor Sie beginnen
* SVMs und Vscan Server müssen sich in derselben Domäne oder in vertrauenswürdigen Domänen befinden.
* Ein ONTAP Antivirus Connector wird mit der Cluster Management-LIF konfiguriert.
* Die Liste der privilegierten Benutzer muss die Domäne und den Benutzernamen enthalten, die der Vscan Server für die Verbindung zur SVM verwendet.
* Sobald der Scannerpool konfiguriert ist, lässt sich der Verbindungsstatus zu den Servern überprüfen.


.Schritte
. Einen Scannerpool erstellen:
+
`vserver vscan scanner-pool create -vserver cluster_admin_SVM -scanner-pool scanner_pool -hostnames Vscan_server_hostnames -privileged-users privileged_users`

+
** Eine Cluster-Admin-SVM angeben.
** Für jeden Vscan Server Hostnamen ist eine IP-Adresse oder ein FQDN anzugeben.
** Die Domäne und den Benutzernamen für jeden privilegierten Benutzer angeben.


+
Weitere Informationen zu  `vserver vscan scanner-pool create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-create.html["ONTAP-Befehlsreferenz"^].

. Es wird überprüft, ob der Scannerpool erstellt wurde:
+
`vserver vscan scanner-pool show -vserver cluster_admin_SVM -scanner-pool _scanner_pool_`

+
Der folgende Befehl zeigt die Details für den  `SP`scanner-pool an:

+
[listing]
----
cluster1::> vserver vscan scanner-pool show -vserver cluster_admin_SVM -scanner-pool SP

                                    Vserver: cluster_admin_SVM
                               Scanner Pool: SP
                             Applied Policy: idle
                             Current Status: off
         Cluster on Which Policy Is Applied: -
                  Scanner Pool Config Owner: cluster
       List of IPs of Allowed Vscan Servers: 1.1.1.1, 10.72.204.27
List of Host Names of Allowed Vscan Servers: 1.1.1.1, vmwin204-27.fsct.nb
                   List of Privileged Users: cifs\u1, cifs\u2
----
+
Sie können auch den  `vserver vscan scanner-pool show` Befehl verwenden, um alle Scannerpools im Cluster anzuzeigen. Weitere Informationen zu  `vserver vscan scanner-pool show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-show.html["ONTAP-Befehlsreferenz"^].
