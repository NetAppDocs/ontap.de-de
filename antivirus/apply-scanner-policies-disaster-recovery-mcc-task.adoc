= Scannerrichtlinien in MetroCluster ONTAP Vscan Konfigurationen anwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Scanner Richtlinie legt fest, ob ein Scannerpool aktiv ist. Einer scanner-pool-Policy muss auf die primären und sekundären scanner-pools auf jedem Cluster in einer MetroCluster Konfiguration angewendet werden.

.Über diese Aufgabe
* Auf einen scanner-pool kann nur eine Scannerrichtlinie angewendet werden.
* Wenn Sie einen scanner-pool für alle SVMs in einem Cluster erstellt haben, muss auf jeder SVM individuell eine scanner-policy angewendet werden.
* Für die Notfallwiederherstellung und MetroCluster Konfigurationen ist es erforderlich, eine Scannerrichtlinie auf jeden scanner-pool im lokalen Cluster und im Remote Cluster anzuwenden.
* In der Richtlinie, die Sie für den lokalen Cluster erstellen, muss der lokale Cluster im  `cluster` Parameter angegeben werden. In der Richtlinie, die Sie für den Remote Cluster erstellen, muss der Remote Cluster im  `cluster` Parameter angegeben werden. Der Remote Cluster kann dann im Katastrophenfall die Virenprüfung übernehmen.


.Schritte
. Scannerrichtlinie anwenden:
+
`vserver vscan scanner-pool apply-policy -vserver _data_SVM_ -scanner-pool _scanner_pool_ -scanner-policy primary|secondary|idle -cluster _cluster_to_apply_policy_on_`

+
Weitere Informationen zu  `vserver vscan scanner-pool apply-policy` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-apply-policy.html["ONTAP-Befehlsreferenz"^].

+
Eine Scannerrichtlinie kann einen der folgenden Werte haben:

+
** `Primary`gibt an, dass der scanner-pool aktiv ist.
** `Secondary`legt fest, dass der scanner-pool nur dann aktiv ist, wenn keiner der Vscan-Server im primären scanner-pool verbunden ist.
** `Idle`gibt an, dass der scanner-pool inaktiv ist.
+
[NOTE]
====
Alle Scannerrichtlinien müssen aus dem Cluster angewendet werden, das die primäre SVM enthält.

====
+
Die folgenden Befehle wenden Scannerrichtlinien auf die primären und sekundären scanner-pool auf jedem Cluster in einer MetroCluster Konfiguration an:

+
[listing]
----
cluster1::>vserver vscan scanner-pool apply-policy -vserver cifssvm1
-scanner-pool pool1_for_site1 -scanner-policy primary -cluster cluster1

cluster1::>vserver vscan scanner-pool apply-policy -vserver cifssvm1
-scanner-pool pool2_for_site1 -scanner-policy secondary -cluster cluster1

cluster1::>vserver vscan scanner-pool apply-policy -vserver cifssvm1
-scanner-pool pool2_for_site2 -scanner-policy primary -cluster cluster2

cluster1::>vserver vscan scanner-pool apply-policy -vserver cifssvm1
-scanner-pool pool1_for_site2 -scanner-policy secondary -cluster cluster2
----


. Es sollte überprüft werden, ob der scanner-pool aktiv ist:
+
`vserver vscan scanner-pool show -vserver data_SVM|cluster_admin_SVM -scanner-pool scanner_pool`

+
Weitere Informationen zu  `vserver vscan scanner-pool show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-show.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl zeigt die Details für den Scannerpool an `pool1`:

+
[listing]
----
cluster1::> vserver vscan scanner-pool show -vserver cifssvm1 -scanner-pool pool1_for_site1

                                    Vserver: cifssvm1
                               Scanner Pool: pool1_for_site1
                             Applied Policy: primary
                             Current Status: on
         Cluster on Which Policy Is Applied: cluster1
                  Scanner Pool Config Owner: vserver
       List of IPs of Allowed Vscan Servers:
List of Host Names of Allowed Vscan Servers: scan1
                   List of Privileged Users: cifs\u1,cifs\u2
----
+
Der  `vserver vscan scanner-pool show-active` Befehl kann verwendet werden, um die aktiven scanner-pool auf einer SVM anzuzeigen. Weitere Informationen zu  `vserver vscan scanner-pool show-active` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-show-active.html["ONTAP-Befehlsreferenz"^].
