= Eine Scannerrichtlinie auf einem einzelnen Cluster mit ONTAP Vscan anwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Scanner Richtlinie legt fest, ob ein Scannerpool aktiv ist. Ein scanner-pool muss aktiviert werden, bevor die darin definierten Vscan Server eine Verbindung zu einer SVM herstellen können.

.Über diese Aufgabe
* Auf einen scanner-pool kann nur eine Scannerrichtlinie angewendet werden.
* Wenn Sie einen scanner-pool für alle SVMs in einem Cluster erstellt haben, muss auf jeder SVM individuell eine scanner-policy angewendet werden.


.Schritte
. Scannerrichtlinie anwenden:
+
`vserver vscan scanner-pool apply-policy -vserver data_SVM -scanner-pool scanner_pool -scanner-policy primary|secondary|idle -cluster cluster_to_apply_policy_on`

+
Eine Scannerrichtlinie kann einen der folgenden Werte haben:

+
** `Primary`gibt an, dass der scanner-pool aktiv ist.
** `Secondary`legt fest, dass der scanner-pool nur dann aktiv ist, wenn keiner der Vscan-Server im primären scanner-pool verbunden ist.
** `Idle`gibt an, dass der scanner-pool inaktiv ist.
+
Das folgende Beispiel zeigt, dass der Scanner-Pool mit dem Namen  `SP` auf der  `vs1` SVM aktiv ist:

+
[listing]
----
cluster1::> vserver vscan scanner-pool apply-policy -vserver vs1 -scanner-pool SP  -scanner-policy primary
----


. Es sollte überprüft werden, ob der scanner-pool aktiv ist:
+
`vserver vscan scanner-pool show -vserver data_SVM|cluster_admin_SVM -scanner-pool scanner_pool`

+
Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl zeigt die Details für den  `SP`scanner-pool an:

+
[listing]
----
cluster1::> vserver vscan scanner-pool show -vserver vs1 -scanner-pool SP

                                    Vserver: vs1
                               Scanner Pool: SP
                             Applied Policy: primary
                             Current Status: on
         Cluster on Which Policy Is Applied: cluster1
                  Scanner Pool Config Owner: vserver
       List of IPs of Allowed Vscan Servers: 1.1.1.1, 10.72.204.27
List of Host Names of Allowed Vscan Servers: 1.1.1.1, vmwin204-27.fsct.nb
                   List of Privileged Users: cifs\u1, cifs\u2
----
+
Der  `vserver vscan scanner-pool show-active` Befehl kann verwendet werden, um die aktiven scanner-pool auf einer SVM anzuzeigen. Weitere Informationen zu  `vserver vscan scanner-pool show-active` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-scanner-pool-show-active.html["ONTAP-Befehlsreferenz"^].
