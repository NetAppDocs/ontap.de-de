= Status der von ONTAP Vscan gescannten Dateien zurücksetzen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Gelegentlich kann es erforderlich sein, den Scanstatus erfolgreich gescannter Dateien auf einer SVM zurückzusetzen, indem der  `vserver vscan reset` Befehl verwendet wird, um die zwischengespeicherten Informationen für die Dateien zu verwerfen. Dieser Befehl kann beispielsweise eingesetzt werden, um die Virenscan-Verarbeitung nach einer fehlerhaften Konfiguration neu zu starten. Weitere Informationen zu  `vserver vscan reset` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-reset.html["ONTAP-Befehlsreferenz"^].

.Über diese Aufgabe
Nachdem Sie den  `vserver vscan reset` Befehl ausgeführt haben, werden alle infrage kommenden Dateien beim nächsten Zugriff gescannt.

[NOTE]
====
Dieser Befehl kann die Leistung negativ beeinflussen, abhängig von der Anzahl und Größe der erneut zu scannenden Dateien.

====
.Bevor Sie beginnen
Für diese Aufgabe sind erweiterte Berechtigungen erforderlich.

.Schritte
. Wechsel zum erweiterten Berechtigungslevel:
+
`set -privilege advanced`

+
Weitere Informationen zu  `set -privilege advanced` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/set.html["ONTAP-Befehlsreferenz"^].

. Status der gescannten Dateien zurücksetzen:
+
`vserver vscan reset -vserver data_SVM`

+
Der folgende Befehl setzt den Status der gescannten Dateien auf der `vs1` SVM zurück:

+
[listing]
----
cluster1::> vserver vscan reset -vserver vs1
----
