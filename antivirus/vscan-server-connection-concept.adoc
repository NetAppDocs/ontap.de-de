= Mögliche ONTAP Vscan Verbindungsprobleme im Zusammenhang mit der Option „Scan obligatorisch“
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die  `vserver vscan connection-status show` Befehle verwenden, um Informationen zu Vscan Server-Verbindungen anzuzeigen, die bei der Fehlerbehebung von Verbindungsproblemen hilfreich sein können.

Standardmäßig verweigert die  `scan-mandatory` Option für das Scannen bei Zugriff den Dateizugriff, wenn keine Verbindung zu einem Vscan-Server für den Scanvorgang verfügbar ist. Obwohl diese Option wichtige Sicherheitsfunktionen bietet, kann sie in einigen Situationen zu Problemen führen.

* Bevor der Clientzugriff aktiviert wird, muss sichergestellt sein, dass auf jedem Knoten mit einer LIF mindestens ein Vscan Server mit einer SVM verbunden ist. Wenn nach der Aktivierung des Clientzugriffs Server mit SVMs verbunden werden müssen, muss die  `scan-mandatory` Option auf der SVM deaktiviert werden, damit der Dateizugriff nicht verweigert wird, weil keine Vscan Serververbindung verfügbar ist. Die Option kann wieder aktiviert werden, nachdem der Server verbunden wurde.
* Wenn eine Ziel-LIF alle Vscan-Serververbindungen für eine SVM hostet, geht die Verbindung zwischen Server und SVM verloren, wenn die LIF migriert wird. Damit der Dateizugriff nicht verweigert wird, weil eine Vscan-Serververbindung nicht verfügbar ist, sollte die  `scan-mandatory` Option vor der Migration der LIF deaktiviert werden. Nach der Migration der LIF kann die Option wieder aktiviert werden.


Jeder SVM sollten mindestens zwei Vscan Server zugewiesen sein. Es gilt als Best Practice, die Vscan Server über ein anderes Netzwerk als das für den Clientzugriff verwendete mit dem Speichersystem zu verbinden.

Weitere Informationen zu  `vserver vscan connection-status show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-vscan-connection-status-show.html["ONTAP-Befehlsreferenz"^].
