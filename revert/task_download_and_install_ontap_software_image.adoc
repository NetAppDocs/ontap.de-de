= Das ONTAP Software-Image herunterladen und installieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie Ihre aktuelle ONTAP Software zurücksetzen, muss die Zielsoftwareversion von der NetApp Support-Website heruntergeladen und anschließend installiert werden.



[[download-the-ontap-software-image]]
== Das ONTAP Software-Image herunterladen

Software-Images sind spezifisch für Plattformmodelle. Das korrekte Image für Ihren Cluster muss bezogen werden. Software-Images, Firmware-Versionsinformationen und die neueste Firmware für Ihr Plattformmodell sind auf der NetApp Support Site verfügbar. Software-Images enthalten die neueste Version der System-Firmware, die zum Zeitpunkt der Veröffentlichung einer bestimmten Version von ONTAP verfügbar war.


NOTE: Wenn Sie ein System mit NetApp Volume Encryption von ONTAP 9.5 oder höher zurücksetzen, muss das ONTAP Software-Image für nicht eingeschränkte Länder heruntergeladen werden, das NetApp Volume Encryption enthält. Wird das ONTAP Software-Image für eingeschränkte Länder verwendet, um ein System mit NetApp Volume Encryption zurückzusetzen, führt dies zu einem Systemabsturz und der Zugriff auf die Volumes geht verloren.

.Schritte
. Die gewünschte ONTAP Software befindet sich im link:http://mysupport.netapp.com/NOW/cgi-bin/software["Software-Downloads"^] Bereich der NetApp Support Site.
. Das Software-Image (zum Beispiel  `97_q_image.tgz`) von der NetApp Support Site kopieren
+
Sie können das Bild in das Verzeichnis auf dem HTTP-Server oder FTP-Server kopieren, von dem das Bild bereitgestellt wird, oder in einen lokalen Ordner.





[[install-the-ontap-software-image]]
== Das ONTAP Software-Image installieren

Nachdem das gewünschte ONTAP Software-Image von der NetApp Support-Website heruntergeladen wurde, erfolgt die Installation auf den Cluster-Nodes.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
+
Die erweiterte Eingabeaufforderung (`*>`) erscheint.

. Geben Sie  `y` ein, um fortzufahren, wenn Sie dazu aufgefordert werden.
. Das Software-Image installieren:
+
** Für Standardkonfigurationen oder eine Zwei-Node MetroCluster Konfiguration ist folgender Befehl einzugeben:
+
[source, cli]
----
system node image update -node * -package <http://example.com/downloads/image.tgz> -replace-package true -replace {image1|image2} -background true -setdefault true
----
+
Dieser Befehl lädt das Software-Image herunter und installiert es gleichzeitig auf allen Knoten. Um das Image auf jedem Knoten einzeln herunterzuladen und zu installieren, darf der  `-background` Parameter nicht angegeben werden. Dieser Befehl verwendet außerdem eine erweiterte Abfrage, um das Ziel-Software-Image, das als alternatives Image installiert ist, als Standard-Image für den Knoten festzulegen.

** Für eine Vier-Node- oder Acht-Node-MetroCluster Konfiguration ist auf beiden Clustern folgender Befehl einzugeben:
+
[source, cli]
----
system node image update -node * -package <http://example.com/downloads/image.tgz> -replace-package true -replace {image1|image2} -background true -setdefault false
----
+
Dieser Befehl lädt das Software-Image herunter und installiert es gleichzeitig auf allen Knoten. Um das Image auf jedem Knoten einzeln herunterzuladen und zu installieren, darf der  `-background` Parameter nicht angegeben werden. Dieser Befehl verwendet außerdem eine erweiterte Abfrage, um das Ziel-Software-Image zu ändern, das als alternatives Image auf jedem Knoten installiert wird.



. Geben Sie  `y` ein, um fortzufahren, wenn Sie dazu aufgefordert werden.
. Es sollte sichergestellt sein, dass das Software-Image auf jedem Node heruntergeladen und installiert ist:
+
[source, cli]
----
system node image show-update-progress -node *
----
+
Dieser Befehl zeigt den aktuellen Status des Software-Image-Downloads und der Installation an. Dieser Befehl sollte weiterhin ausgeführt werden, bis alle Knoten einen *Ausführungsstatus* von „Beendet“ und einen *Exit-Status* von „Erfolgreich“ melden.

+
Der Befehl zum Aktualisieren des Systemknoten-Images kann fehlschlagen und Fehler- oder Warnmeldungen anzeigen. Nach Behebung aller Fehler oder Warnungen ist eine erneute Ausführung des Befehls möglich.

+
Dieses Beispiel zeigt einen Zwei-Node-Cluster, in dem das Software-Image erfolgreich auf beiden Knoten heruntergeladen und installiert wurde:

+
[listing]
----
cluster1::*> system node image show-update-progress -node *
There is no update/install in progress
Status of most recent operation:
        Run Status:     Exited
        Exit Status:    Success
        Phase:          Run Script
        Exit Message:   After a clean shutdown, image2 will be set as the default boot image on node0.
There is no update/install in progress
Status of most recent operation:
        Run Status:     Exited
        Exit Status:    Success
        Phase:          Run Script
        Exit Message:   After a clean shutdown, image2 will be set as the default boot image on node1.
2 entries were acted on.
----
