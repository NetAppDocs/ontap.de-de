= Snapshots sollten vorbereitet werden, bevor ein ONTAP Cluster zurückgesetzt wird.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor ein ONTAP Cluster von einer beliebigen Version von ONTAP 9 zurückgesetzt wird, müssen alle Snapshot-Richtlinien deaktiviert und alle Snapshots gelöscht werden, die nach dem Upgrade auf die aktuelle Version erstellt wurden.

Wenn Sie in einer SnapMirror-Umgebung einen Reset durchführen, müssen zuvor die folgenden Spiegelbeziehungen gelöscht worden sein:

* Alle lastverteilenden Spiegelbeziehungen
* Alle in ONTAP 8.3.x erstellten Datenschutzspiegelbeziehungen
* Alle Datenschutz-Spiegelbeziehungen, falls der Cluster in ONTAP 8.3.x neu erstellt wurde.


.Schritte
. Snapshot-Richtlinien für alle Daten-SVMs deaktivieren:
+
[source, cli]
----
volume snapshot policy modify -vserver * -enabled false
----
. Snapshot-Richtlinien für die Aggregate jedes Knotens deaktivieren:
+
.. Die Aggregate des Knotens identifizieren:
+
[source, cli]
----
run -node <nodename> -command aggr status
----
.. Die Snapshot-Richtlinie für jedes Aggregat wird deaktiviert:
+
[source, cli]
----
run -node <nodename> -command aggr options aggr_name nosnap on
----
.. Wiederholen Sie diesen Schritt für jeden verbleibenden Node.


. Snapshot-Richtlinien für das Root-Volume jedes Knotens deaktivieren:
+
.. Das Root-Volume des Node identifizieren:
+
[source, cli]
----
run -node <node_name> -command vol status
----
+
Das Root-Volume wird durch das Wort „root“ in der Spalte *Optionen* der  `vol status` Befehlsausgabe identifiziert.

+
[listing]
----
vs1::> run -node node1 vol status

         Volume State           Status            Options
           vol0 online          raid_dp, flex     root, nvfail=on
                                64-bit
----
.. Die Snapshot-Richtlinie auf dem Root-Volume wird deaktiviert:
+
[source, cli]
----
run -node <node_name> vol options root_volume_name nosnap on
----
.. Wiederholen Sie diesen Schritt für jeden verbleibenden Node.


. Alle Snapshots, die nach dem Upgrade auf die aktuelle Version erstellt wurden, löschen:
+
.. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
.. Snapshots deaktivieren:
+
[source, cli]
----
snapshot policy modify -vserver * -enabled false
----
.. Die neueren Snapshots des Knotens löschen:
+
[source, cli]
----
volume snapshot prepare-for-revert -node <node_name>
----
+
Dieser Befehl löscht die neueren Snapshots auf jedem Datenvolume, Root-Aggregat und Root-Volume.

+
Falls Snapshots nicht gelöscht werden können, schlägt der Befehl fehl und Sie werden über die erforderlichen Aktionen informiert, die Sie vor dem Löschen der Snapshots durchführen müssen. Die erforderlichen Aktionen müssen abgeschlossen und anschließend der  `volume snapshot prepare-for-revert` Befehl erneut ausgeführt werden, bevor mit dem nächsten Schritt fortgefahren wird.

+
[listing]
----
cluster1::*> volume snapshot prepare-for-revert -node node1

Warning: This command will delete all snapshots that have the format used by the current version of ONTAP. It will fail if any snapshot policies are enabled, or
         if any snapshots have an owner. Continue? {y|n}: y
----
.. Es sollte überprüft werden, ob die Snapshots gelöscht wurden:
+
[source, cli]
----
volume snapshot show -node nodename
----
.. Falls noch neuere Snapshots vorhanden sind, diese zwangsweise löschen:
+
[source, cli]
----
volume snapshot delete {-fs-version 9.0 -node nodename -is-constituent true} -ignore-owners -force
----
.. Diese Schritte sind für jeden verbleibenden Knoten zu wiederholen.
.. Zurück zur Administrator-Privilegstufe:
+
[source, cli]
----
set -privilege admin
----





NOTE: Diese Schritte sind auf beiden Clustern in der MetroCluster Konfiguration durchzuführen.
