= Ändern Sie die in ONTAP 9.19.1 oder höher erstellten QoS-Richtliniengruppen, bevor Sie von ONTAP 9.19.1 zurückkehren.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Adaptive und nicht-adaptive QoS-Policy-Gruppen werden standardmäßig automatisch aktiviert, um das Durchsatzlimit einer Arbeitslast für eine Dauer von 1 Sekunde vorübergehend um 50 % zu erhöhen, um kurzfristigen Anforderungen an eine höhere Performance zu begegnen. Ab ONTAP 9.19.1 können Sie diese Standardeinstellungen ändern. In ONTAP 9.18.1 und früher können diese Einstellungen nicht geändert werden. Wenn Sie die Standardeinstellungen einer in ONTAP 9.19.1 oder später erstellten QoS-Policy-Gruppe geändert haben, müssen Sie diese zurücksetzen, bevor Sie von ONTAP 9.19.1 auf ONTAP 9.18.1 zurückkehren können.

.Schritte
. Prüfen Sie, ob QoS-Richtliniengruppen für die automatische Erhöhung des Durchsatzlimits aktiviert sind:
+
[source, cli]
----
qos policy-group show – instance
----
. Wenn Richtliniengruppen mit  `burst-percent`,  `burst-iops` und  `burst-duration` konfiguriert sind, setzen Sie diese Optionen auf 0.
+
[source, cli]
----
qos policy-group modify -policy-group <policy-group> -burst-percent 0 -burst-iops 0 -burst-duration 0
----
