= Autocommit-Zeiträume für SnapLock Volumes vor dem Zurücksetzen von ONTAP festlegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor ein ONTAP Cluster von einer beliebigen Version von ONTAP 9 zurückgesetzt wird, muss der Wert des Autocommit-Zeitraums für SnapLock Volumes in Stunden und nicht in Tagen festgelegt sein. Es empfiehlt sich, den Autocommit-Wert für die SnapLock Volumes zu überprüfen und ihn gegebenenfalls von Tagen auf Stunden zu ändern.

.Schritte
. Es wird überprüft, ob im Cluster SnapLock Volumes vorhanden sind, die nicht unterstützte Autocommit-Zeiträume haben:
+
[source, cli]
----
volume snaplock show -autocommit-period *days
----
. Nicht unterstützte Autocommit-Zeiträume auf Stunden ändern
+
[source, cli]
----
volume snaplock modify -vserver <vserver_name> -volume <volume_name> -autocommit-period value hours
----
