= Vorbereitende Aufgaben, die für Ihre ONTAP Version erforderlich sind
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Je nach Ihrer ONTAP Version sind möglicherweise zusätzliche vorbereitende Aufgaben erforderlich, bevor der Wiederherstellungsprozess gestartet werden kann.

[cols="2*"]
|===
| Wenn Sie von ... zurückkehren | Vor Beginn des Wiederherstellungsprozesses ist Folgendes zu beachten... 


| Jede ONTAP 9 Version  a| 
* link:terminate-smb-sessions.html["SMB-Sitzungen beenden, die nicht durchgehend verfügbar sind"].
* link:concept_reversion_requirements_for_snapmirror_and_snapvault_relationships.html["Überprüfung der Rückführungsanforderungen für SnapMirror und SnapVault Beziehungen"].
* link:task_reverting_systems_with_deduplicated_volumes.html["Es sollte sichergestellt werden, dass die deduplizierten Volumes über ausreichend freien Speicherplatz verfügen."].
* link:task_preparing_snapshot_copies_before_reverting.html["Snapshots vorbereiten"].
* link:task_setting_autocommit_periods_for_snaplock_volumes_before_reverting.html["Legen Sie den Autocommit-Zeitraum für SnapLock Volumes auf Stunden fest"].
* Wenn Sie eine MetroCluster Konfiguration haben, link:task_disable_asuo.html["Automatische ungeplante Umschaltung deaktivieren"].
* link:arp-warning-clear-revert.html["Auf Warnungen des autonomen Ransomware-Schutzes bezüglich ungewöhnlicher Aktivität reagieren"].




| ONTAP 9.19.1  a| 
* Wenn Sie SVMs mit Aktiv/Aktiv Multipathing-Konfigurationen haben, ist Folgendes zu beachten link:delete-active-active-svms.html["Die SVMs löschen"].
* Wenn Sie die Standardeinstellungen einer in ONTAP 9.19.1 oder höher erstellten QoS-Richtliniengruppe geändert haben, link:modify-qos-policy-groups.html["zurücksetzen"].
* Wenn Sie die Volumes auf ONTAP 9.19.1 aktualisiert und einem Benutzer einen zweiten S3-Zugriffsschlüssel hinzugefügt haben, muss link:second-s3-access-key.html["Diese Schlüssel entfernen"].




| ONTAP 9.18.1  a| 
* Wenn die automatische Aktivierung für ARP im Rahmen eines ONTAP 9.18.1-Upgrades eingestellt wurde, müssen Sie link:anti-ransomware-disable-auto-enable.html["deaktivieren"].




| ONTAP 9.17.1  a| 
* Wenn Sie die ONTAP ARP-Funktion für SAN aktiviert haben, link:anti-ransomware-disable-san.html["deaktivieren"].




| ONTAP 9.16.1  a| 
* Wenn Sie TLS für NVMe/TCP-Verbindungen konfiguriert haben, link:task-disable-tls-nvme-host.html["Die TLS-Konfiguration auf den NVMe-Hosts deaktivieren"].
* Wenn Sie die erweiterte Qtree-Leistungsüberwachung aktiviert haben, link:disable-extended-qtree-performance-monitoring.html["deaktivieren"].
* Wenn Sie CORS verwenden, um auf Ihre ONTAP s3 Buckets zuzugreifen, link:remove-cors-configuration.html["Die CORS-Konfiguration entfernen"].




| ONTAP 9.14.1 | Wenn Sie Trunking für Clientverbindungen aktiviert haben, link:remove-nfs-trunking-task.html["Trunking auf allen NFSv4.1 Servern deaktivieren"]. 


| ONTAP 9.12.1  a| 
* Wenn Sie den S3-Clientzugriff für NAS-Daten konfiguriert haben, link:remove-nas-bucket-task.html["Die S3 NAS-Bucket-Konfiguration wird entfernt."]
* Wenn Sie das NVMe-Protokoll verwenden und die In-Band Authentifizierung konfiguriert haben, link:disable-in-band-authentication.html["In-Band Authentifizierung deaktivieren"].
* Wenn Sie eine MetroCluster Konfiguration haben, link:task-disable-ipsec.html["IPsec deaktivieren"].




| ONTAP 9.11.1 | Wenn Sie den autonomen Ransomware-Schutz (ARP) konfiguriert haben, link:anti-ransomware-license-task.html["Die ARP-Lizenzierung prüfen."]. 


| ONTAP 9.6 | Wenn Sie SnapMirror synchron Beziehungen haben, link:concept_consideration_for_reverting_systems_with_snapmirror_synchronous_relationships.html["Die Beziehungen für das Zurücksetzen vorbereiten"]. 
|===
