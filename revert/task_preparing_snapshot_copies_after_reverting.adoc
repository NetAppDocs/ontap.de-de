= Snapshot-Richtlinien nach einem ONTAP Revert aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach der Rückkehr zu einer früheren Version von ONTAP müssen Snapshot-Richtlinien aktiviert werden, damit Snapshots wieder erstellt werden.

Sie reaktivieren die Snapshot-Zeitpläne, die Sie deaktiviert hatten, bevor Sie auf eine frühere Version von ONTAP zurückgekehrt sind.

.Schritte
. Snapshot-Richtlinien für alle Daten-SVMs aktivieren:
+
[source, cli]
----
volume snapshot policy modify -vserver * -enabled true
----
+
[source, cli]
----
snapshot policy modify pg-rpo-hourly -enable true
----
. Für jeden Node wird die Snapshot-Richtlinie des Root-Volumes aktiviert:
+
[source, cli]
----
run -node <node_name> vol options <volume_name> nosnap off
----
