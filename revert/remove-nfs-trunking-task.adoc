= NFSv4.1 Session Trunking vor dem Zurücksetzen von ONTAP 9.14.1 deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Trunking für Clientverbindungen aktiviert wurde, muss Trunking auf allen NFSv4.1-Servern deaktiviert werden, bevor ein Wechsel von ONTAP 9.14.1 erfolgt.

Wenn Sie den  `revert-to` Befehl eingeben, erscheint eine Warnmeldung, in der empfohlen wird, Trunking zu deaktivieren, bevor Sie fortfahren.

Nach dem Downgrade auf ONTAP 9.13.1 nutzen Clients mit trunked Verbindungen eine einzelne Verbindung. Ihr Datendurchsatz wird beeinträchtigt, es kommt jedoch zu keiner Unterbrechung. Das Rückkehrverhalten entspricht der Änderung der NFSv4.1 trunking Option für die SVM von aktiviert auf deaktiviert.

.Schritte
. Trunking auf dem NFSv4.1 Server deaktivieren:
+
[source, cli]
----
vserver nfs modify -vserver _svm_name_ -v4.1-trunking disabled
----
. Es sollte überprüft werden, ob NFS wie gewünscht konfiguriert ist:
+
[source, cli]
----
vserver nfs show -vserver _svm_name_
----
