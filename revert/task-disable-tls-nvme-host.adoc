= TLS auf NVMe-Hosts vor dem Zurücksetzen von ONTAP 9.16.1 deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn auf einem NVMe-Host ein sicherer TLS-Kanal für NVMe/TCP-Verbindungen konfiguriert ist, muss dieser deaktiviert werden, bevor der Cluster von ONTAP 9.16.1 zurückgesetzt wird.

.Schritte
. Die TLS-Sicherheitskanalkonfiguration wird vom Host entfernt:
+
[source, cli]
----
vserver nvme subsystem host unconfigure-tls-for-revert -vserver <svm_name> -subsystem <subsystem> -host-nqn <host_nqn>
----
+
Dieser Befehl entfernt den Host aus dem Subsystem und erstellt den Host anschließend ohne die TLS-Konfiguration im Subsystem neu.

. Es sollte überprüft werden, ob der sichere TLS-Kanal vom Host entfernt wurde:
+
[source, cli]
----
vserver nvme subsystem host show
----
