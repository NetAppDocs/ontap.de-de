= Vor dem Zurückkehren von ONTAP 9.11.1 sollte die Lizenzierung von Autonomous Ransomware Protection überprüft werden.
:allow-uri-read: 


[role="lead"]
Wenn Sie Autonomous Ransomware Protection (ARP) konfiguriert haben und von ONTAP 9.11.1 auf ONTAP 9.10.1 zurückkehren, kann es zu Warnmeldungen und einer eingeschränkten ARP-Funktionalität kommen.

In ONTAP 9.11.1 wurde die Anti-ransomware-Lizenz anstelle der Multi-Tenant Key Management (MTKM) Lizenz eingeführt. Wenn Ihr System über die Anti_ransomware-Lizenz, aber nicht über die MT_EK_MGMT-Lizenz verfügt, erscheint beim Zurücksetzen eine Warnung, dass ARP auf neuen Volumes nach dem Zurücksetzen nicht aktiviert werden kann.

Die Volumes mit bestehendem Schutz funktionieren nach dem Zurücksetzen weiterhin normal, und der ARP-Status kann über die ONTAP CLI angezeigt werden. System Manager kann den ARP-Status ohne die MTKM-Lizenz nicht anzeigen.

Wenn ARP nach der Rückkehr zu ONTAP 9.10.1 weiterhin funktionieren soll, muss die MTKM-Lizenz vor der Rückkehr installiert sein.link:../anti-ransomware/index.html["Informationen zur ARP-Lizenzierung."]
