= Die erweiterte Qtree-Leistungsüberwachung sollte deaktiviert werden, bevor von ONTAP 9.16.1 zurückgekehrt wird.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.16.1 kann über die ONTAP REST API auf die erweiterten Qtree-Überwachungsfunktionen zugegriffen werden, die Latenzmetriken und historische Statistiken umfassen. Wenn die erweiterte Qtree-Überwachung auf einem Qtree aktiviert ist, muss  `ext_performance_monitoring.enabled` vor dem Zurücksetzen von Version 9.16.1 auf false gesetzt werden.

Sie arbeiten mit der `ext_performance_monitoring.enabled` Einstellung, indem Sie den `/api/storage/qtrees` Endpoint verwenden. Verwenden Sie GET, um den aktuellen Wert abzurufen, POST, um ihn beim Erstellen eines neuen qtree festzulegen, und PATCH, um ihn bei einem bestehenden qtree zu ändern.

Weitere Informationen zu link:../volumes/qtrees-partition-your-volumes-concept.html#upgrading-and-reverting["Wiederherstellung von Clustern mit erweiterter Qtree-Leistungsüberwachung"].
