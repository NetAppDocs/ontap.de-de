= Freier Speicherplatz für deduplizierte Volumes sollte vor dem Zurücksetzen von ONTAP überprüft werden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor ein ONTAP Cluster von einer beliebigen Version von ONTAP 9 zurückgesetzt wird, muss sichergestellt sein, dass die Volumes über ausreichend freien Speicherplatz für den Rücksetzungsvorgang verfügen.

Das Volumen muss ausreichend Platz bieten, um die durch die Inline-Erkennung von Nullblöcken erzielten Einsparungen aufzunehmen. Siehe link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/How_to_see_space_savings_from_deduplication%2C_compression%2C_and_compaction_in_ONTAP_9["NetApp Knowledge Base: Wie sich Speicherplatzeinsparungen durch Deduplizierung, Komprimierung und Kompaktierung in ONTAP 9 anzeigen lassen"^].

Wenn sowohl Deduplizierung als auch Datenkomprimierung auf einem Volume aktiviert wurden, dessen Zustand zurückgesetzt werden soll, muss die Datenkomprimierung vor der Deduplizierung zurückgesetzt werden.

.Schritte
. Den Fortschritt der Effizienzmaßnahmen, die auf den Volumes ausgeführt werden, anzeigen:
+
[source, cli]
----
volume efficiency show -fields vserver,volume,progress
----
. Alle aktiven und in der Warteschlange befindlichen Deduplizierungsvorgänge anhalten:
+
[source, cli]
----
volume efficiency stop -vserver <svm_name> -volume <volume_name> -all
----
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Die Effizienzmetadaten eines Volumes auf die Zielversion von ONTAP herabstufen:
+
[source, cli]
----
volume efficiency revert-to -vserver <svm_name> -volume <volume_name> -version <version>
----
+
Das folgende Beispiel setzt die Effizienz-Metadaten auf dem Volume VolA auf ONTAP 9.x zurück.

+
[listing]
----
volume efficiency revert-to -vserver vs1 -volume VolA -version 9.x
----
+

NOTE: Der Befehl „volume efficiency revert-to“ setzt die Volumes zurück, die sich auf dem Knoten befinden, auf dem dieser Befehl ausgeführt wird. Mit diesem Befehl werden keine Volumes über Knoten hinweg zurückgesetzt.

. Der Fortschritt der Herabstufung kann überwacht werden:
+
[source, cli]
----
volume efficiency show -vserver <svm_name> -op-status Downgrading
----
. Falls die Rücksetzung fehlschlägt, kann die Instanz angezeigt werden, um den Grund für das Scheitern der Rücksetzung zu ermitteln.
+
[source, cli]
----
volume efficiency show -vserver <svm_name> -volume <volume_name> - instance
----
. Nach Abschluss des Rückgängigmachungsvorgangs wird zur Administratorberechtigungsstufe zurückgekehrt:
+
[source, cli]
----
set -privilege admin
----
