= Benutzerkonten überprüfen, die nach der Rückkehr zu ONTAP 9.8 auf den Service Processor zugreifen können.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In ONTAP 9.9.1 und höher wird der `-role` Parameter für Benutzerkonten in `admin` geändert. Wenn Benutzerkonten in ONTAP 9.8 oder früher erstellt, auf ONTAP 9.9.1 oder höher aktualisiert und anschließend wieder auf ONTAP 9.8 zurückgesetzt wurden, wird der `-role` Parameter auf seinen ursprünglichen Wert zurückgesetzt. Es sollte überprüft werden, ob die geänderten Werte zulässig sind.

Während des Zurücksetzens wird, falls die Rolle für einen SP-Benutzer gelöscht wurde, die EMS-Meldung "rbac.spuser.role.notfound" protokolliert.

Weitere Informationen finden sich unter link:../system-admin/accounts-access-sp-concept.html["Konten, die auf den SP zugreifen können"].
