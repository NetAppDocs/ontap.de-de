= Supportfälle mit ONTAP System Manager anzeigen und einreichen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Ab ONTAP 9.9.1 ist es möglich, Supportfälle aus Active IQ Digital Advisor (auch bekannt als Digital Advisor), die mit dem Cluster verknüpft sind, anzuzeigen. Es besteht außerdem die Möglichkeit, Clusterdetails zu kopieren, die für das Einreichen eines neuen Supportfalls auf der NetApp Support Site benötigt werden. Ab ONTAP 9.10.1 kann die Telemetrieprotokollierung aktiviert werden, wodurch das Supportpersonal bei der Problembehebung unterstützt wird.


NOTE: Um Benachrichtigungen über Firmware-Updates zu erhalten, müssen Sie bei Active IQ Unified Manager registriert sein. Weitere Informationen unter link:https://netapp.com/support-and-training/documentation/active-iq-unified-manager["Active IQ Unified Manager Dokumentationsressourcen"^].

.Schritte
. Im System Manager *Support* auswählen.
+
Eine Liste der offenen Supportfälle, die mit diesem Cluster verknüpft sind, wird angezeigt.

. Klicken Sie auf die folgenden Links, um die Verfahren auszuführen:
+
** *Fallnummer*: Details zum Fall.
** *Zur NetApp Support-Website*: Auf der Seite *My AutoSupport* der NetApp Support-Website können Wissensdatenbankartikel angezeigt oder ein neuer Supportfall eingereicht werden.
** *Meine Fälle anzeigen*: Zur Seite *Meine Fälle* auf der NetApp Support-Website navigieren.
** *Clusterdetails anzeigen*: Informationen, die für die Einreichung eines neuen Falls benötigt werden, können hier angezeigt und kopiert werden.

== Telemetrie-Protokollierung aktivieren

Ab ONTAP 9.10.1 kann die Telemetrieprotokollierung mit System Manager aktiviert werden. Wenn die Telemetrieprotokollierung erlaubt ist, erhalten die vom System Manager protokollierten Meldungen eine spezifische Telemetrie-Kennung, die den genauen Prozess angibt, der die Meldung ausgelöst hat. Alle Meldungen, die sich auf diesen Prozess beziehen, haben dieselbe Kennung, die aus dem Namen des operativen Workflows und einer Nummer besteht (zum Beispiel „add-volume-1941290“).

Bei Leistungsproblemen kann die Telemetrieprotokollierung aktiviert werden, wodurch Support-Personal den jeweiligen Prozess, für den eine Meldung ausgegeben wurde, leichter identifizieren kann. Wenn Telemetrie-Kennungen zu den Meldungen hinzugefügt werden, vergrößert sich die Protokolldatei nur geringfügig.

.Schritte
. Im System Manager *Cluster > Einstellungen* auswählen.
. Im Abschnitt *UI Settings* das Kästchen für *Allow telemetry logging* aktivieren.
