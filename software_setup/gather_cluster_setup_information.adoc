= Informationen zum Einrichten eines ONTAP Clusters sammeln
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie mit der Cluster-Einrichtung beginnen, sollten Sie die Informationen zusammentragen, die Sie für den Abschluss der Cluster-Einrichtung benötigen, wie beispielsweise den Port und die IP-Adresse der Cluster-Managementoberfläche. Zu Beginn empfiehlt es sich, alle relevanten Informationen in den Arbeitsblättern zur Cluster-Einrichtung zu erfassen. Das Arbeitsblatt zur Cluster-Einrichtung ermöglicht es, die Werte zu dokumentieren, die während des Cluster-Einrichtungsprozesses benötigt werden. Wenn ein Standardwert vorgegeben ist, kann dieser verwendet oder ein eigener Wert eingetragen werden.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Wenn Sie ein ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) haben, link:https://docs.netapp.com/us-en/asa-r2/install-setup/initialize-ontap-cluster.html["diese Schritte"^] um ein ONTAP Cluster einzurichten. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

[[admin-storage-virtual-machine-svm]]
== Admin Storage Virtual Machine (SVM)

[cols="2*"]
|===
| Informationsarten | Ihre Werte 


 a| 
Cluster-Administrator-Passwort

Das Passwort für das Administratorkonto, das der Cluster benötigt, bevor dem Cluster-Administrator Zugriff auf die Konsole oder über ein sicheres Protokoll gewährt wird.


NOTE: Aus Sicherheitsgründen wird davon abgeraten, Passwörter in diesem Arbeitsblatt zu notieren.

Die Standardregeln für Passwörter lauten wie folgt:

* Ein Passwort muss mindestens acht Zeichen lang sein.
* Ein Passwort muss mindestens einen Buchstaben und eine Zahl enthalten.

 a| 



 a| 
Cluster-Managementoberfläche Port

Der physische Port, der mit dem Datennetzwerk verbunden ist und dem Cluster-Administrator die Verwaltung des Clusters ermöglicht.
 a| 



 a| 
IP-Adresse der Cluster-Managementoberfläche

Eine eindeutige IPv4- oder IPv6-Adresse für die Cluster Managementoberfläche. Der Cluster-Administrator verwendet diese Adresse, um auf die Admin-SVM zuzugreifen und das Cluster zu verwalten. In der Regel sollte sich diese Adresse im Datennetzwerk befinden.

Diese IP-Adresse ist von dem Administrator zu erhalten, der in Ihrer Organisation für die Vergabe von IP-Adressen zuständig ist.

Beispiel: 192.0.2.66
 a| 



 a| 
Cluster-Managementoberfläche Subnetzmaske (IPv4)

Die Subnetzmaske, die den Bereich gültiger IPv4-Adressen im Cluster-Managementnetzwerk definiert.

Beispiel: 255.255.255.0
 a| 



 a| 
Netzmaske der Managementoberfläche des Clusters (IPv6)

Wenn die Cluster Managementoberfläche eine IPv6-Adresse verwendet, stellt dieser Wert die Präfixlänge dar, die den Bereich gültiger IPv6-Adressen im Cluster Managementnetzwerk definiert.

Beispiel: 64
 a| 



 a| 
Standardgateway der Cluster-Managementoberfläche

Die IP-Adresse des Routers im Cluster-Management-Netzwerk.
 a| 



 a| 
DNS-Domänenname

Der Name der DNS-Domäne Ihres Netzwerks.

Der Domainname muss aus alphanumerischen Zeichen bestehen. Zum Eingeben mehrerer DNS-Domainnamen ist jeder Name durch ein Komma oder ein Leerzeichen zu trennen.
 a| 



 a| 
Nameserver-IP-Adressen

Die IP-Adressen der DNS-Nameserver. Jede Adresse ist durch ein Komma oder ein Leerzeichen zu trennen.
 a| 

|===


[[node-information-for-each-node-in-the-cluster]]
== Knoteninformationen (für jeden Knoten im Cluster)

[cols="2*"]
|===
| Informationsarten | Ihre Werte 


 a| 
Physischer Standort des Controllers (optional)

Eine Beschreibung des physischen Standorts des Controllers. Eine Beschreibung, die angibt, wo sich dieser Node im Cluster befindet (zum Beispiel „Lab 5, Reihe 7, Rack B“), sollte verwendet werden.
 a| 



 a| 
Knoten-Managementoberflächenport

Der physische Port, der mit dem Managementnetzwerk des Nodes verbunden ist und dem Clusteradministrator die Verwaltung des Nodes ermöglicht.
 a| 



 a| 
IP-Adresse der Managementoberfläche des Node

Eine eindeutige IPv4- oder IPv6-Adresse für die Managementoberfläche des Knotens im Managementnetzwerk. Wenn der Port der Knoten-Managementoberfläche als Datenport definiert wurde, sollte diese IP-Adresse eine eindeutige IP-Adresse im Datennetzwerk sein.

Diese IP-Adresse ist von dem Administrator zu erhalten, der in Ihrer Organisation für die Vergabe von IP-Adressen zuständig ist.

Beispiel: 192.0.2.66
 a| 



 a| 
Netzwerkmaske der Managementoberfläche des Node (IPv4)

Die Subnetzmaske, die den Bereich gültiger IP-Adressen im Managementnetzwerk des Node definiert.

Wenn Sie den Managementoberflächenport als Datenport definiert haben, sollte die Netzmaske die Subnetzmaske für das Datennetzwerk sein.

Beispiel: 255.255.255.0
 a| 



 a| 
Netzmaske der Managementoberfläche des Knotens (IPv6)

Wenn die Managementoberfläche eine IPv6-Adresse verwendet, stellt dieser Wert die Präfixlänge dar, die den Bereich gültiger IPv6-Adressen im Knotenverwaltungsnetzwerk definiert.

Beispiel: 64
 a| 



 a| 
Standardgateway der Node Managementoberfläche

Die IP-Adresse des Routers im Knoten-Managementnetzwerk.
 a|
