= Protokolle auf Ihrem ONTAP Cluster konfigurieren
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Abhängig von den auf Ihrem Cluster aktivierten Lizenzen können die gewünschten Protokolle auf Ihrem Cluster aktiviert werden. Anschließend werden Netzwerkschnittstellen erstellt, über die auf den Speicher zugegriffen werden kann.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/install-setup/initialize-ontap-cluster.html["diese Schritte"^] zu folgen, um mithilfe des System Manager ein ONTAP Cluster einzurichten. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

.Schritt
. *Dashboard* auswählen und anschließend auf *Protokolle konfigurieren* klicken.
+
** Zur Aktivierung von NAS-Protokollen *NFS* oder *SMB* auswählen.
** Zur Aktivierung von SAN-Protokollen *iSCSI* oder *FC* auswählen.
** Zur Aktivierung von NVMe-Protokollen *NVMe* auswählen.
