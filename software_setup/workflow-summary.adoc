= ONTAP Cluster-Setup-Workflow
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um Ihren Cluster einzurichten, sollten Sie die Informationen sammeln, die Sie für die Einrichtung benötigen, einen Cluster erstellen und Knoten hinzufügen, Management-LIFs von IPv4 auf IPv6 konvertieren, Ihren Cluster mit Active IQ Config Advisor überprüfen und die Systemzeit im gesamten Cluster synchronisieren.

[NOTE]
====
Dieses Verfahren gilt für FAS, AFF und ASA Systeme.

* Wenn Sie ein AFX-System besitzen, folgen Sie link:https://docs.netapp.com/us-en/ontap-afx/install-setup/cluster-setup.html["diese Schritte"^] zum Einrichten eines ONTAP Clusters.
* Wenn Sie ein ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) haben, link:https://docs.netapp.com/us-en/asa-r2/install-setup/initialize-ontap-cluster.html["diese Schritte"^] um ein ONTAP Cluster einzurichten. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.


====
.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["Eins"]link:gather_cluster_setup_information.html["Informationen sammeln"]
[role="quick-margin-para"]
Vor Beginn der Cluster-Einrichtung sollten die für den Abschluss des Einrichtungsprozesses benötigten Informationen bereitgestellt werden.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["Zwei"]link:setup-cluster.html["Ein ONTAP Cluster erstellen und Knoten beitreten"]
[role="quick-margin-para"]
NetApp empfiehlt die Verwendung von System Manager zum Einrichten neuer Cluster. System Manager bietet einen einfachen und unkomplizierten Workflow für die Cluster-Einrichtung, einschließlich der Zuweisung einer Node-Management-IP-Adresse und der Initialisierung des Clusters.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["Drei"]link:convert-ipv4-to-ipv6-task.html["Management-LIFs von IPv4 auf IPv6 umstellen"]
[role="quick-margin-para"]
Ab ONTAP 9.13.1 ist es möglich, Management-LIFs auf AFF A800 und FAS 8700 Plattformen während der Ersteinrichtung des Clusters über die ONTAP Befehlszeilenschnittstelle (CLI) IPv6-Adressen zuzuweisen. Für ONTAP Versionen vor 9.13.1 oder für 9.13.1 und höher auf anderen Plattformen müssen zunächst IPv4-Adressen den Management-LIFs zugewiesen werden, und nach Abschluss der Clustereinrichtung erfolgt die Umstellung auf IPv6-Adressen.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["Vier"]link:task_check_cluster_with_config_advisor.html["Ihr Cluster kann mit Active IQ Config Advisor überprüft werden"]
[role="quick-margin-para"]
Nachdem Sie alle Knoten zu Ihrem neuen Cluster hinzugefügt haben, empfiehlt es sich, Active IQ Config Advisor auszuführen, um Ihre Konfiguration zu validieren und auf häufige Konfigurationsfehler zu überprüfen.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["Fünf"]link:task_synchronize_the_system_time_across_the_cluster.html["Die Systemzeit im gesamten Cluster wird synchronisiert."]
[role="quick-margin-para"]
Die Systemzeit im gesamten Cluster wird synchronisiert, sodass jeder Knoten im Cluster die gleiche Zeit hat und CIFS- sowie Kerberos-Fehler vermieden werden.
