= Schnellstart für ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Um mit ONTAP zu beginnen, ist es erforderlich, Ihren Cluster einzurichten, Ihr Netzwerk zu konfigurieren, die automatische Netzwerkkonfiguration von BMC zu aktivieren, eine lokale Ebene zu erstellen, Ihre Protokolle zu konfigurieren und Ihren Speicher bereitzustellen.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-1.png["Eins"] Richten Sie Ihren Cluster ein
[role="quick-margin-para"]
NetApp empfiehlt die Verwendung von System Manager, um link:software_setup/setup-cluster.html["Neue ONTAP Cluster einrichten"]. System Manager bietet einen einfachen und unkomplizierten Workflow für die Cluster-Einrichtung, einschließlich der Zuweisung einer Node-Management-IP-Adresse und der Initialisierung des Clusters.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-2.png["Zwei"]Netzwerk konfigurieren
[role="quick-margin-para"]
Das Netzwerk wird konfiguriert, indem link:networking/add_broadcast_domain.html["Broadcast-Domains"], link:networking/create_a_subnet.html["Subnetze"] und optional link:networking/create_ipspaces.html["IPspaces"] erstellt werden.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-3.png["Drei"]Automatische BMC-Netzwerkkonfiguration aktivieren
[role="quick-margin-para"]
link:system-admin/enable-sp-bmc-automatic-network-config-task.html["Die automatische BMC Netzwerkkonfiguration aktivieren"], sodass Sie den BMC der einzelnen Knoten nicht manuell IP-Adressen zuweisen müssen.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-4.png["Vier"]Lokale Ebene erstellen
[role="quick-margin-para"]
link:disks-aggregates/create-aggregates-auto-provision-task.html["Lokale Ebenen erstellen"] aus den verfügbaren Festplatten oder SSDs in Ihren Knoten. System Manager berechnet automatisch die optimale Tier-Konfiguration basierend auf Ihrer Hardware.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-5.png["Fünf"]Protokolle konfigurieren
[role="quick-margin-para"]
link:software_setup/configure-protocols.html["NAS-, SAN- oder NVMe-Protokolle aktivieren"] auf Ihrem Cluster.

.image:https://raw.githubusercontent.com/NetAppDocs/common/main/media/number-6.png["Sechs"]Speicher bereitstellen
[role="quick-margin-para"]
Speicherplatz für link:concept_nas_provision_overview.html["NAS"] oder link:san-admin/provision-storage.html["SAN"] Konfigurationen bereitstellen.

.Was kommt als Nächstes?
* link:insights-system-optimization-task.html["Erkenntnisse zur Optimierung Ihres Systems erhalten"].
* link:authentication/index.html["ONTAP Administrator-Authentifizierung und rollenbasierte Zugriffssteuerung (RBAC) einrichten"] zur Definition der Fähigkeiten von Administratoren.
* link:error-messages//configure-ems-notifications-sm-task.html["ONTAP für das Senden wichtiger Benachrichtigungen des Event Management Systems (EMS) konfigurieren"]damit Sie umgehend über Systemprobleme informiert werden, die Ihre Aufmerksamkeit erfordern.
* ONTAP kann verwendet werden, um zu link:peering/index.html["Daten für die Notfallwiederherstellung sichern"] und zu link:ransomware-solutions/ransomware-overview.html["Schutz Ihrer Daten vor Ransomware-Angriffen"].
* link:system-admin/autosupport-active-iq-digital-advisor-concept.html["Informationen zu Digital Advisor und AutoSupport"].
* Mit dem System Manager kann link:task_cp_dashboard_tour.html["den Zustand Ihres Clusters überwachen"] verwendet werden.
