= Löschen einer ONTAP Konsistenzgruppe
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie feststellen, dass Sie eine Konsistenzgruppe nicht mehr benötigen, können Sie sie löschen.

.Über diese Aufgabe
* Das Löschen einer Konsistenzgruppe löscht die Instanz der Konsistenzgruppe und hat keine Auswirkungen auf die zugehörigen Volumes oder LUNs. Das Löschen einer Konsistenzgruppe führt nicht zur Löschung der auf den einzelnen Volumes vorhandenen Snapshots, diese sind jedoch nicht mehr als Konsistenzgruppen-Snapshots zugänglich. Die Snapshots können jedoch weiterhin als gewöhnliche, granulare Volume-Snapshots verwaltet werden.
* ONTAP löscht eine Konsistenzgruppe automatisch, wenn alle Volumes in der Konsistenzgruppe gelöscht wurden.
* Das Löschen einer übergeordneten Konsistenzgruppe führt zum Löschen aller zugehörigen untergeordneten Konsistenzgruppen.
* Wenn eine Version von ONTAP zwischen 9.10.1 und 9.12.0 verwendet wird, können Volumes nur dann aus einer Konsistenzgruppe entfernt werden, wenn das Volume selbst gelöscht wird. In diesem Fall wird das Volume automatisch aus der Konsistenzgruppe entfernt. Ab ONTAP 9.12.1 ist es möglich, Volumes aus einer Konsistenzgruppe zu entfernen, ohne die Konsistenzgruppe zu löschen. Weitere Informationen zu diesem Vorgang finden sich unter xref:modify-task.html[Eine Konsistenzgruppe ändern].


.Schritte
[role="tabbed-block"]
====
.System Manager
--
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die Konsistenzgruppe aus, die Sie löschen möchten.
. Wählen Sie neben dem Namen der Konsistenzgruppe image:../media/icon_kabob.gif["Symbol für Menüoptionen"] dann *Löschen* aus.


--
.CLI
--
Ab ONTAP 9.14.1 ist das Löschen einer Konsistenzgruppe über die CLI möglich.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine Konsistenzgruppe löschen
. Die Konsistenzgruppe löschen:
+
`consistency-group delete -vserver _svm_name_ -consistency-group _consistency_group_name_`
