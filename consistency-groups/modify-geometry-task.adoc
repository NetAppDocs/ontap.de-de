= ONTAP Konsistenzgruppen-Geometrie ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.13.1 kann die Geometrie einer Konsistenzgruppe geändert werden. Das Ändern der Geometrie einer Konsistenzgruppe ermöglicht eine Anpassung der Konfiguration von untergeordneten oder übergeordneten Konsistenzgruppen, ohne laufende E/A-Operationen zu unterbrechen.

Das Ändern der Geometrie einer Konsistenzgruppe wirkt sich auf bestehende Snapshots der Konsistenzgruppe aus. Einzelheiten finden sich bei der jeweiligen Geometrieänderung, die durchgeführt werden soll.


NOTE: Die Geometrie einer Konsistenzgruppe, die mit einer Remote Datensicherungsstrategie konfiguriert ist, kann nicht geändert werden. Zuerst muss die Schutzbeziehung aufgehoben, dann die Geometrie geändert und anschließend der Remote-Schutz wiederhergestellt werden.



[[add-a-new-child-consistency-group]]
== Eine neue untergeordnete Konsistenzgruppe hinzufügen

Ab ONTAP 9.13.1 ist es möglich, einer bestehenden übergeordneten Konsistenzgruppe eine neue untergeordnete Konsistenzgruppe hinzuzufügen.

.Über diese Aufgabe
* Eine übergeordnete Konsistenzgruppe kann maximal fünf untergeordnete Konsistenzgruppen enthalten. Siehe xref:limits.html[Konsistenzgruppe Grenzen] für weitere Einschränkungen.
* Sie können einer einzelnen Konsistenzgruppe keine untergeordnete Konsistenzgruppe hinzufügen. Zuerst muss die Konsistenzgruppe <<promoten>> werden, danach kann eine untergeordnete Konsistenzgruppe hinzugefügt werden.
* Vorhandene Snapshots der Konsistenzgruppe, die vor der Erweiterungsoperation erstellt wurden, werden als partiell betrachtet. Jede Wiederherstellung, die auf einem solchen Snapshot basiert, gibt die Konsistenzgruppe zum Zeitpunkt des Snapshots wieder.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.13.1 ist es möglich, diese Operation mit System Manager durchzuführen.

.Eine neue untergeordnete Konsistenzgruppe hinzufügen
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die übergeordnete Konsistenzgruppe aus, zu der Sie eine untergeordnete Konsistenzgruppe hinzufügen möchten.
. Neben dem Namen der übergeordneten Konsistenzgruppe **Mehr** und dann **Neue untergeordnete Konsistenzgruppe** auswählen.
. Ein Name für Ihre Konsistenzgruppe wird eingegeben.
. Wählen Sie aus, ob neue oder bereits vorhandene Volumes hinzugefügt werden sollen.
+
.. Wenn Sie bereits vorhandene Volumes hinzufügen, **Vorhandene Volumes** auswählen und anschließend die gewünschten Volumes aus dem Dropdown-Menü wählen.
.. Wenn Sie neue Volumes hinzufügen, wählen Sie **Neue Volumes** aus und geben Sie anschließend die Anzahl der Volumes und deren Größe an.


. Wählen Sie **Hinzufügen**.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, mithilfe der ONTAP CLI eine untergeordnete Konsistenzgruppe hinzuzufügen.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine untergeordnete Konsistenzgruppe mit neuen Volumes hinzufügen
. Die neue Konsistenzgruppe wird erstellt. Werte für den Namen der Konsistenzgruppe, das Volume-Präfix, die Volume-Anzahl, die Volume-Größe, den Storage-Service und den Namen der Export-Policy werden angegeben:
+
`consistency-group create -vserver _SVM_name_ -consistency-group _consistency_group_ -parent-consistency-group _parent_consistency_group_ -volume-prefix _prefix_ -volume-count _number_ -size _size_ -storage-service _service_ -export-policy _policy_name_`



.Eine untergeordnete Konsistenzgruppe mit vorhandenen Volumes hinzufügen
. Die neue Konsistenzgruppe wird erstellt. Der  `volumes` Parameter akzeptiert eine durch Kommas getrennte Liste von Volume-Namen.
+
`consistency-group create -vserver _SVM_name_ -consistency-group _new_consistency_group_ -parent-consistency-group _parent_consistency_group_ -volumes _volume_`



--

[[detach]]
== Eine untergeordnete Konsistenzgruppe trennen

Ab ONTAP 9.13.1 ist es möglich, eine untergeordnete Konsistenzgruppe von ihrer übergeordneten Gruppe zu entfernen, wodurch sie in eine eigenständige Konsistenzgruppe umgewandelt wird.

.Über diese Aufgabe
* Das Trennen einer untergeordneten Konsistenzgruppe führt dazu, dass die Snapshots der übergeordneten Konsistenzgruppe ungültig und nicht mehr zugänglich werden. Volume-granulare Snapshots bleiben gültig.
* Vorhandene zeitpunktgenaue Kopien der einzelnen Konsistenzgruppe bleiben gültig.
* Dieser Vorgang schlägt fehl, wenn bereits eine einzelne Konsistenzgruppe mit demselben Namen wie die untergeordnete Konsistenzgruppe, die Sie trennen möchten, existiert. In diesem Fall muss die Konsistenzgruppe beim Trennen umbenannt werden.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.13.1 ist es möglich, diese Operation mit System Manager durchzuführen.

.Eine untergeordnete Konsistenzgruppe trennen
. *Speicher > Konsistenzgruppen* auswählen.
. Die übergeordnete Konsistenzgruppe auswählen, die das untergeordnete Element enthält, das abgetrennt werden soll.
. Wählen Sie neben der untergeordneten Konsistenzgruppe, die Sie abtrennen möchten, **Mehr** und dann **Vom übergeordneten Element abtrennen**.
. Optional kann die Konsistenzgruppe umbenannt und ein Anwendungstyp ausgewählt werden.
. **Trennen** auswählen.


--
.CLI
--
Ab ONTAP 9.14.1 ist das Trennen einer untergeordneten Konsistenzgruppe mithilfe der ONTAP CLI möglich.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine untergeordnete Konsistenzgruppe trennen
. Die Konsistenzgruppe wird getrennt. Optional kann die getrennte Konsistenzgruppe mit dem  `-new-name` Parameter umbenannt werden.
+
`consistency-group detach -vserver _SVM_name_ -consistency-group _child_consistency_group_ -parent-consistency-group _parent_consistency_group_ [-new-name _new_name_]`



--

[[move-an-existing-single-consistency-group-under-a-parent-consistency-group]]
== Eine bestehende einzelne Konsistenzgruppe unter eine übergeordnete Konsistenzgruppe verschieben

Ab ONTAP 9.13.1 ist es möglich, eine bestehende einzelne Konsistenzgruppe in eine untergeordnete Konsistenzgruppe umzuwandeln. Die Konsistenzgruppe kann entweder unter eine bestehende übergeordnete Konsistenzgruppe verschoben oder während des Verschiebevorgangs eine neue übergeordnete Konsistenzgruppe erstellt werden.

.Über diese Aufgabe
* Die übergeordnete Konsistenzgruppe darf höchstens vier untergeordnete Konsistenzgruppen haben. Eine übergeordnete Konsistenzgruppe kann maximal fünf untergeordnete Konsistenzgruppen enthalten. Siehe xref:limits.html[Konsistenzgruppe Grenzen] für weitere Einschränkungen.
* Vor diesem Vorgang erstellte Snapshots der übergeordneten Konsistenzgruppe gelten als partiell. Jede Wiederherstellung, die auf einem dieser Snapshots basiert, spiegelt die Konsistenzgruppe zum Zeitpunkt des Snapshots wider.
* Vorhandene Snapshots der einzelnen Konsistenzgruppe bleiben gültig.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.13.1 ist es möglich, diese Operation mit System Manager durchzuführen.

.Eine bestehende einzelne Konsistenzgruppe unter eine übergeordnete Konsistenzgruppe verschieben
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die Konsistenzgruppe aus, die Sie konvertieren möchten.
. **Mehr** und dann **Unter andere Konsistenzgruppe** auswählen.
. Optional kann ein neuer Name für die Konsistenzgruppe eingegeben und ein Komponententyp ausgewählt werden. Standardmäßig ist der Komponententyp Sonstige.
. Wählen Sie aus, ob Sie zu einer bestehenden übergeordneten Konsistenzgruppe migrieren oder eine neue übergeordnete Konsistenzgruppe erstellen möchten:
+
.. Um zu einer bestehenden übergeordneten Konsistenzgruppe zu migrieren, **Bestehende Konsistenzgruppe** auswählen und anschließend die Konsistenzgruppe aus dem Dropdown-Menü wählen.
.. Zum Erstellen einer neuen übergeordneten Konsistenzgruppe **Neue Konsistenzgruppe** auswählen und anschließend einen Namen für die neue Konsistenzgruppe angeben.


. **Verschieben** auswählen.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, eine einzelne Konsistenzgruppe mithilfe der ONTAP CLI unter eine übergeordnete Konsistenzgruppe zu verschieben.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine Konsistenzgruppe unter eine neue übergeordnete Konsistenzgruppe verschieben
. Die neue übergeordnete Konsistenzgruppe wird erstellt. Der  `-consistency-groups` Parameter migriert alle vorhandenen Konsistenzgruppen zur neuen übergeordneten Konsistenzgruppe.
+
`consistency-group attach -vserver _svm_name_ -consistency-group _parent_consistency_group_ -consistency-groups _child_consistency_group_`



.Eine Konsistenzgruppe unter eine bestehende Konsistenzgruppe verschieben
. Die Konsistenzgruppe verschieben:
+
`consistency-group add -vserver _SVM_name_ -consistency-group _consistency_group_ -parent-consistency-group _parent_consistency_group_`



--

== Eine untergeordnete Konsistenzgruppe hochstufen

Ab ONTAP 9.13.1 ist es möglich, eine einzelne Konsistenzgruppe zu einer übergeordneten Konsistenzgruppe zu machen. Wird die einzelne Konsistenzgruppe zu einer übergeordneten Konsistenzgruppe gemacht, entsteht gleichzeitig eine neue untergeordnete Konsistenzgruppe, die alle Volumes der ursprünglichen einzelnen Konsistenzgruppe übernimmt.

.Über diese Aufgabe
* Wenn eine untergeordnete Konsistenzgruppe in eine übergeordnete Konsistenzgruppe umgewandelt werden soll, muss zunächst <<detach>> die untergeordnete Konsistenzgruppe erstellt und anschließend dieses Verfahren angewendet werden.
* Vorhandene Snapshots der Konsistenzgruppe bleiben auch nach der Heraufstufung der Konsistenzgruppe gültig.


[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.13.1 ist es möglich, diese Operation mit System Manager durchzuführen.

.Eine untergeordnete Konsistenzgruppe hochstufen
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die Konsistenzgruppe aus, die Sie promoten möchten.
. **Mehr** und dann **Zu übergeordneter Konsistenzgruppe hochstufen** auswählen.
. Einen **Namen** eingeben und einen **Komponententyp** für die untergeordnete Konsistenzgruppe auswählen.
. **Befördern** auswählen.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, eine einzelne Konsistenzgruppe mithilfe der ONTAP CLI unter eine übergeordnete Konsistenzgruppe zu verschieben.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine untergeordnete Konsistenzgruppe hochstufen
. Die Konsistenzgruppe wird hochgestuft. Dieser Befehl erstellt eine übergeordnete und eine untergeordnete Konsistenzgruppe.
+
`consistency-group promote -vserver _SVM_name_ -consistency-group _existing_consistency_group_ -new-name _new_child_consistency_group_`



--

[[demote-a-parent-to-a-single-consistency-group]]
== Ein übergeordnetes Element zu einer einzelnen Konsistenzgruppe herabstufen

Ab ONTAP 9.13.1 ist es möglich, eine übergeordnete Konsistenzgruppe in eine einzelne Konsistenzgruppe herabzustufen. Durch das Herabstufen der übergeordneten Konsistenzgruppe wird die Hierarchie der Konsistenzgruppe aufgehoben und alle zugehörigen untergeordneten Konsistenzgruppen werden entfernt. Alle Volumes in der Konsistenzgruppe verbleiben unter der neuen, einzelnen Konsistenzgruppe.

.Über diese Aufgabe
* Vorhandene Snapshots der _übergeordneten_ Konsistenzgruppe bleiben nach der Herabstufung zu einer einzelnen Konsistenz gültig. Vorhandene Snapshots aller zugehörigen _untergeordneten_ Konsistenzgruppen dieser übergeordneten Gruppe werden bei der Herabstufung ungültig. Die einzelnen Volume-Snapshots innerhalb der untergeordneten Konsistenzgruppe bleiben weiterhin als volume-granulare Snapshots zugänglich.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.13.1 ist es möglich, diese Operation mit System Manager durchzuführen.

.Eine Konsistenzgruppe herabstufen
. *Speicher > Konsistenzgruppen* auswählen.
. Die übergeordnete Konsistenzgruppe auswählen, die herabgestuft werden soll.
. **Mehr** und dann **Zu einzelner Konsistenzgruppe herabstufen** auswählen.
. Eine Warnung weist darauf hin, dass alle zugehörigen untergeordneten Konsistenzgruppen gelöscht werden und deren Volumes unter die neue einzelne Konsistenzgruppe verschoben werden. Mit **Herabstufen** wird bestätigt, dass die Auswirkungen verstanden wurden.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, eine Konsistenzgruppe mithilfe der ONTAP CLI herabzustufen.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine Konsistenzgruppe herabstufen
. Die Konsistenzgruppe herabstufen. Mit dem optionalen  `-new-name` Parameter kann die Konsistenzgruppe umbenannt werden.
+
`consistency-group demote -vserver _SVM_name_ -consistency-group _parent_consistency_group_ [-new-name _new_consistency_group_name_]`
