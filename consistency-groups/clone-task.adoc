= Klonen einer ONTAP Konsistenzgruppe
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.12.1 ist das Klonen einer Konsistenzgruppe möglich, um eine Kopie der Konsistenzgruppe und ihres Inhalts zu erstellen. Das Klonen einer Konsistenzgruppe erstellt eine Kopie der Konsistenzgruppen-Konfiguration, ihrer Metadaten wie Anwendungstyp sowie aller Volumes und deren Inhalte wie Dateien, Verzeichnisse, LUNs oder NVMe-Namespaces.

.Über diese Aufgabe
Beim Klonen einer Konsistenzgruppe kann diese mit ihrer aktuellen Konfiguration geklont werden, wobei der Inhalt des Volumes entweder wie er ist übernommen wird oder auf einem vorhandenen Snapshot der Konsistenzgruppe basiert.

Das Klonen einer Konsistenzgruppe wird nur für die gesamte Konsistenzgruppe unterstützt. Es ist nicht möglich, eine einzelne untergeordnete Konsistenzgruppe in einer hierarchischen Beziehung zu klonen; nur die vollständige Konfiguration der Konsistenzgruppe kann geklont werden.

Beim Klonen einer Konsistenzgruppe werden die folgenden Komponenten nicht geklont:

* iGroups
* LUN-Zuordnungen
* NVMe Subsysteme
* NVMe Namespace Subsystem-Zuordnungen


.Bevor Sie beginnen
* Beim Klonen einer Konsistenzgruppe erstellt ONTAP keine SMB-Freigaben für die geklonten Volumes, wenn kein Freigabename angegeben ist. * Geklonte Konsistenzgruppen werden nicht eingebunden, wenn kein Verbindungspfad angegeben ist.
* Wenn versucht wird, eine Konsistenzgruppe auf Basis eines Snapshots zu klonen, der die aktuellen Bestandteile der Konsistenzgruppe nicht widerspiegelt, schlägt der Vorgang fehl.
* Nach dem Klonen einer Konsistenzgruppe ist die entsprechende Zuordnungsoperation erforderlich.
+
Weitere Informationen sind unter xref:../task_san_map_igroups_to_multiple_luns.html[Igroups mehreren LUNs zuordnen] oder xref:../san-admin/map-nvme-namespace-subsystem-task.html[Einem NVMe-Namensraum einem Subsystem zuordnen] verfügbar.

* Das Klonen einer Konsistenzgruppe wird für eine Konsistenzgruppe in einer SnapMirror active sync-Beziehung oder mit zugehörigen DP Volumes nicht unterstützt.


[role="tabbed-block"]
====
.System Manager
--
.Schritte
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die Konsistenzgruppe aus, die Sie aus dem Menü *Konsistenzgruppe* klonen möchten.
. Oben rechts auf der Übersichtsseite für die Konsistenzgruppe die Option *Klonen* auswählen.
. Geben Sie einen Namen für die neue, geklonte Konsistenzgruppe ein oder übernehmen Sie den Standardnamen.
+
.. Wählen Sie aus, ob Sie link:../concepts/thin-provisioning-concept.html["*Thin Provisioning*"] aktivieren möchten.
.. *Split Clone* sollte ausgewählt werden, wenn die Konsistenzgruppe von ihrer Quelle getrennt und zusätzlicher Speicherplatz für die geklonte Konsistenzgruppe bereitgestellt werden soll.


. Um die Konsistenzgruppe in ihrem aktuellen Zustand zu klonen, *Neuen Snapshot hinzufügen* auswählen.
+
Um die Konsistenzgruppe anhand eines Snapshots zu klonen, die Option *Vorhandenen Snapshot verwenden* auswählen. Bei Auswahl dieser Option öffnet sich ein neues Untermenü. Den Snapshot auswählen, der als Grundlage für den Klonvorgang verwendet werden soll.

. Wählen Sie *Clone* aus.
. Zum Menü *Konsistenzgruppe* zurückkehren, um zu bestätigen, dass die Konsistenzgruppe geklont wurde.


--
.CLI
--
Ab ONTAP 9.14.1 ist das Klonen einer Konsistenzgruppe mithilfe der CLI mit Cluster-Administratorrechten möglich.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Eine Konsistenzgruppe klonen
. Der  `consistency-group clone create`Befehl klont die Konsistenzgruppe in ihrem aktuellen zeitpunktgenauen Status. Um den Klonvorgang auf einem Snapshot zu basieren, ist der  `-source-snapshot`Parameter einzuschließen.
+
`consistency-group clone create -vserver _svm_name_ -consistency-group _clone_name_ -source-consistency-group _consistency_group_name_ [-source-snapshot _snapshot_name_]`

+
Weitere Informationen zu  `consistency-group clone create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=consistency-group+clone+create["ONTAP-Befehlsreferenz"^].
