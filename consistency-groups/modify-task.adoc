= Mitgliedsvolumes in einer ONTAP Konsistenzgruppe ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.12.1 ist es möglich, eine Konsistenzgruppe zu ändern, indem Volumes entfernt oder hinzugefügt werden (Erweiterung der Konsistenzgruppe). Ab ONTAP 9.13.1 ist es möglich, Volumes zwischen untergeordneten Konsistenzgruppen zu verschieben, sofern diese eine gemeinsame übergeordnete Gruppe haben.



[[add-volumes-to-a-consistency-group]]
== Volumes zu einer Konsistenzgruppe hinzufügen

Ab ONTAP 9.12.1 ist es möglich, Volumes unterbrechungsfrei zu einer Konsistenzgruppe hinzuzufügen.

.Über diese Aufgabe
* Sie können keine Volumes hinzufügen, die mit einer anderen Konsistenzgruppe verknüpft sind.
* Konsistenzgruppen unterstützen NAS, SAN und NVMe-Protokolle.
* Sie können bis zu 16 Bände gleichzeitig zu einer Konsistenzgruppe hinzufügen, sofern die Anpassungen innerhalb des Gesamtbereichs xref:limits.html[Konsistenzgruppe Grenzen] liegen.
* Ab ONTAP 9.13.1 ist es möglich, Volumes unterbrechungsfrei zu einer Konsistenzgruppe mit einer aktiven SnapMirror active sync- oder SnapMirror asynchronen Datensicherungsstrategie hinzuzufügen.
* Wenn Volumes zu einer durch SnapMirror active sync geschützten Konsistenzgruppe hinzugefügt werden, ändert sich der Status der SnapMirror active sync-Beziehung zu „Wird erweitert“, bis Spiegelung und Schutz für das neue Volume konfiguriert sind. Falls auf dem primären Cluster ein Ausfall auftritt, bevor dieser Vorgang abgeschlossen ist, wird die Konsistenzgruppe im Rahmen des Failover-Vorgangs auf ihre ursprüngliche Zusammensetzung zurückgesetzt.
* In ONTAP 9.12.1 und früher können Volumes nicht zu einer Konsistenzgruppe in einer SnapMirror active sync-Beziehung hinzugefügt werden. Zuerst muss die SnapMirror active sync-Beziehung gelöscht, die Konsistenzgruppe geändert und anschließend der Schutz mit SnapMirror active sync wiederhergestellt werden.
* Ab ONTAP 9.12.1 unterstützt die ONTAP REST API das Hinzufügen neuer oder vorhandener Volumes zu einer Konsistenzgruppe. Weitere Informationen zur ONTAP REST API sind unter link:https://docs.netapp.com/us-en/ontap-automation/reference/api_reference.html#access-a-copy-of-the-ontap-rest-api-reference-documentation["ONTAP REST API Referenzdokumentation"^] verfügbar.
+
Ab ONTAP 9.13.1 ist diese Funktionalität im System Manager verfügbar.

* Beim Erweitern einer Konsistenzgruppe werden Snapshots der Konsistenzgruppe, die vor der Änderung erstellt wurden, als partiell betrachtet. Jede Wiederherstellung, die auf einem solchen Snapshot basiert, gibt die Konsistenzgruppe zum Zeitpunkt des Snapshots wieder.
* Wenn Sie ONTAP 9.10.1 bis 9.11.1 verwenden, kann eine Konsistenzgruppe nicht geändert werden. Um die Konfiguration einer Konsistenzgruppe in ONTAP 9.10.1 oder 9.11.1 zu ändern, muss die Konsistenzgruppe gelöscht und anschließend eine neue Konsistenzgruppe mit den gewünschten Volumes erstellt werden.
* Ab ONTAP 9.14.1 können volume-granulare Snapshots zum Ziel-Cluster repliziert werden, wenn SnapMirror asynchron verwendet wird. Beim Erweitern einer Konsistenzgruppe mit SnapMirror asynchron werden volume-granulare Snapshots erst nach der Erweiterung der Konsistenzgruppe repliziert, wenn die SnapMirror Richtlinie MirrorAll oder MirrorAndVault ist. Es werden nur volume-granulare Snapshots repliziert, die neuer als der Basis-Snapshot der Konsistenzgruppe sind.
* Wenn Volumes zu einer Konsistenzgruppe in einer SVM-Disaster-Recovery-Beziehung hinzugefügt werden (unterstützt ab ONTAP 9.14.1), muss die SVM-Disaster-Recovery-Beziehung nach der Erweiterung der Konsistenzgruppe vom Ziel-Cluster aus aktualisiert werden. Weitere Informationen finden sich unter xref:../data-protection/update-replication-relationship-manual-task.html[Eine Replikationsbeziehung manuell aktualisieren].
* Wenn NVMe mit ONTAP 9.17.1 verwendet wird, kann eine Konsistenzgruppe nicht geändert werden.


.Schritte
[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.1 ist diese Operation mit System Manager möglich.

. *Speicher > Konsistenzgruppen* auswählen.
. Die Konsistenzgruppe, die geändert werden soll, auswählen.
. Wenn Sie eine einzelne Konsistenzgruppe bearbeiten, wählen Sie oben im Menü *Volumes* die Option *More* und dann *Expand*, um ein Volume hinzuzufügen.
+
Wenn eine untergeordnete Konsistenzgruppe bearbeitet wird, ist die übergeordnete Konsistenzgruppe zu identifizieren, die bearbeitet werden soll. Die Schaltfläche *>* zeigt die untergeordneten Konsistenzgruppen an, anschließend image:../media/icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen der untergeordneten Konsistenzgruppe auswählen, die bearbeitet werden soll. Im daraufhin angezeigten Menü die Option *Erweitern* auswählen.

. Bis zu 16 Volumes können zur Konsistenzgruppe hinzugefügt werden.
. *Speichern* auswählen. Nach Abschluss des Vorgangs sind die neu hinzugefügten Volumes im Menü *Volumes* der Konsistenzgruppe sichtbar.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, Volumes mithilfe der ONTAP CLI zu einer Konsistenzgruppe hinzuzufügen.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Vorhandene Volumes hinzufügen
. Führen Sie den folgenden Befehl aus. Der  `-volumes`Parameter akzeptiert eine durch Kommas getrennte Liste von Volumes.
+

NOTE: Den  `-parent-consistency-group` Parameter nur einbeziehen, wenn die Konsistenzgruppe in einer hierarchischen Beziehung steht.

+
`consistency-group volume add -vserver _svm_name_ -consistency-group _consistency_group_name_ -parent-consistency-group _parent_consistency_group_ -volume _volumes_`



.Neue Volumes hinzufügen
Die Vorgehensweise zum Hinzufügen neuer Volumes hängt vom verwendeten Protokoll ab.


NOTE: Der  `-parent-consistency-group` Parameter ist nur einzubeziehen, wenn die Konsistenzgruppe in einer hierarchischen Beziehung steht.

* Neue Volumes können hinzugefügt werden, ohne sie zu exportieren:
+
`consistency-group volume create -vserver _SVM_name_ -consistency-group _child_consistency_group_ -parent-consistency-group existingParentCg -volume _volume_name_ -size _size_`

* Neue NFS-Volumes hinzufügen:
+
`consistency-group volume create -vserver _SVM_name_ -consistency-group _consistency-group-name_ -volume _volume-prefix_ -volume-count _number_ -size _size_ -export-policy _policy_name_`

* Neue SAN-Volumes hinzufügen:
+
`consistency-group volume create -vserver _SVM_name_ -consistency-group _consistency-group-name_ -lun _lun_name_ -size _size_ -lun-count _number_ -igroup _igroup_name_`

* Neue NVMe-Namensräume hinzufügen:
+
`consistency-group volume create -vserver _SVM_name_ -consistency-group _consistency_group_name_ -namespace _namespace_name_ -volume-count _number_ -namespace-count _number_ -size _size_ -subsystem _subsystem_name_`



--

[[remove-volumes-from-a-consistency-group]]
== Volumes aus einer Konsistenzgruppe entfernen

Volumes, die aus einer Konsistenzgruppe entfernt werden, werden nicht gelöscht. Sie bleiben im Cluster aktiv.

.Über diese Aufgabe
* Sie können keine Volumes aus einer Konsistenzgruppe in einer SnapMirror active sync- oder SVM-Disaster-Recovery-Beziehung entfernen. Zunächst muss die SnapMirror active sync-Beziehung gelöscht werden, um die Konsistenzgruppe zu ändern, und anschließend die Beziehung wiederhergestellt werden.
* Wenn eine Konsistenzgruppe nach dem Löschvorgang keine Volumes mehr enthält, wird die Konsistenzgruppe gelöscht.
* Wenn ein Volume aus einer Konsistenzgruppe entfernt wird, bleiben vorhandene Snapshots der Konsistenzgruppe erhalten, gelten jedoch als ungültig. Die vorhandenen Snapshots können nicht zur Wiederherstellung des Inhalts der Konsistenzgruppe verwendet werden. Volume-spezifische Snapshots bleiben gültig.
* Wenn ein Volume aus dem Cluster gelöscht wird, wird es automatisch aus der Konsistenzgruppe entfernt.
* Um die Konfiguration einer Konsistenzgruppe in ONTAP 9.10.1 oder 9.11.1 zu ändern, muss die Konsistenzgruppe gelöscht und anschließend eine neue Konsistenzgruppe mit den gewünschten Mitgliedsvolumes erstellt werden.
* Das Löschen eines Volumes aus dem Cluster entfernt es automatisch auch aus der Konsistenzgruppe.


[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.1 ist diese Operation mit System Manager möglich.

.Schritte
. *Speicher > Konsistenzgruppen* auswählen.
. Die einzelne oder untergeordnete Konsistenzgruppe, die geändert werden soll, auswählen.
. Im Menü *Volumes* die Kontrollkästchen neben den einzelnen Volumes auswählen, die aus der Konsistenzgruppe entfernt werden sollen.
. *Volumes aus der Konsistenzgruppe entfernen* auswählen.
. Bestätigen Sie, dass Sie verstehen, dass durch das Entfernen der Volumes alle Snapshots der Konsistenzgruppe ungültig werden und wählen Sie *Entfernen*.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, Volumes aus einer Konsistenzgruppe mithilfe der CLI zu entfernen.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Schritt
. Die Volumes werden entfernt. Der  `-volumes`Parameter akzeptiert eine durch Kommas getrennte Liste von Volumes.
+
Den  `-parent-consistency-group` Parameter nur einbeziehen, wenn die Konsistenzgruppe in einer hierarchischen Beziehung steht.

+
`consistency-group volume remove -vserver _SVM_name_ -consistency-group _consistency_group_name_ -parent-consistency-group _parent_consistency_group_name_ -volume _volumes_`



--

[[move-volumes-between-consistency-groups]]
== Volumes zwischen Konsistenzgruppen verschieben

Ab ONTAP 9.13.1 ist es möglich, Volumes zwischen untergeordneten Konsistenzgruppen zu verschieben, die ein gemeinsames übergeordnetes Element haben.

.Über diese Aufgabe
* Volumes können nur zwischen Konsistenzgruppen verschoben werden, die unter derselben übergeordneten Konsistenzgruppe verschachtelt sind.
* Vorhandene Konsistenzgruppen-Snapshots werden ungültig und sind nicht mehr als Konsistenzgruppen-Snapshots zugänglich. Einzelne Volume-Snapshots bleiben gültig.
* Momentaufnahmen der übergeordneten Konsistenzgruppe bleiben gültig.
* Wenn alle Volumes aus einer untergeordneten Konsistenzgruppe verschoben werden, wird diese Konsistenzgruppe gelöscht.
* Änderungen an einer Konsistenzgruppe müssen xref:limits.html[Konsistenzgruppe Grenzen] entsprechen.


[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.1 ist diese Operation mit System Manager möglich.

.Schritte
. *Speicher > Konsistenzgruppen* auswählen.
. Wählen Sie die übergeordnete Konsistenzgruppe aus, die die Volumes enthält, die Sie verschieben möchten. Suchen Sie die untergeordnete Konsistenzgruppe und erweitern Sie anschließend das Menü **Volumes**. Wählen Sie die Volumes aus, die Sie verschieben möchten.
. **Verschieben** auswählen.
. Wählen Sie aus, ob die Volumes in eine neue Konsistenzgruppe oder in eine bestehende Gruppe verschoben werden sollen.
+
.. Um zu einer bestehenden Konsistenzgruppe zu wechseln, **Bestehende untergeordnete Konsistenzgruppe** auswählen und dann den Namen der Konsistenzgruppe aus dem Dropdown-Menü wählen.
.. Zum Wechsel in eine neue Konsistenzgruppe die Option **Neue untergeordnete Konsistenzgruppe** auswählen. Einen Namen für die neue untergeordnete Konsistenzgruppe eingeben und einen Komponententyp auswählen.


. **Verschieben** auswählen.


--
.CLI
--
Ab ONTAP 9.14.1 ist es möglich, Volumes mithilfe der ONTAP CLI zwischen Konsistenzgruppen zu verschieben.

.Bevor Sie beginnen
* Sie müssen sich auf der Administrator-Ebene befinden, um diese Aufgabe auszuführen.
* Ab ONTAP 9.15.1 kann jeder Benutzer mit Administratorrechten diese Aufgabe ausführen. In ONTAP 9.14.1 ist es erforderlich, Cluster- oder SVM-Administrator zu sein, um diese Aufgabe auszuführen.


.Volumes in eine neue untergeordnete Konsistenzgruppe verschieben
. Der folgende Befehl erstellt eine neue untergeordnete Konsistenzgruppe, die die angegebenen Volumes enthält.
+
Beim Erstellen der neuen Konsistenzgruppe können neue Snapshot-, QoS- und Tiering-Richtlinien festgelegt werden.

+
`consistency-group volume reassign -vserver _SVM_name_ -consistency-group _source_child_consistency_group_ -parent-consistency-group _parent_consistency_group_ -volume _volumes_ -new-consistency-group _consistency_group_name_ [-snapshot-policy _policy_ -qos-policy _policy_ -tiering-policy _policy_]`



.Volumes in eine bestehende untergeordnete Konsistenzgruppe verschieben
. Weisen Sie die Volumes neu zu. Der  `-volumes` Parameter akzeptiert eine durch Kommas getrennte Liste von Volume-Namen.
+
`consistency-group volume reassign -vserver _SVM_name_ -consistency-group _source_child_consistency_group_ -parent-consistency-group _parent_consistency_group_ -volume _volumes_ -to-consistency-group _target_consistency_group_`
