= Amazon S3 als ONTAP FabricPool Cloud-Tier einrichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Amazon S3 kann als Cloud-Tier für FabricPool eingerichtet werden. Bei Verwendung von ONTAP 9.5 oder höher kann Amazon Commercial Cloud Services (C2S) für FabricPool eingerichtet werden.

.Überlegungen zur Verwendung von Amazon S3 mit FabricPool
* A link:install-license-aws-azure-ibm-task.html["FabricPool Lizenz"] ist erforderlich, wenn von lokalen Clustern zu Amazon S3 migriert wird.
* Es wird empfohlen, dass der LIF, den ONTAP zur Verbindung mit dem Amazon S3 Objektserver verwendet, an einem 10 Gbit/s Port angeschlossen ist.
* Auf AFF und FAS Systemen sowie ONTAP Select unterstützt FabricPool die folgenden Amazon S3-Speicherklassen:
+
** Amazon S3 Standard
** Amazon S3 Standard - Seltener Zugriff (Standard - IA)
** Amazon S3 One Zone - Seltener Zugriff (One Zone - IA)
** Amazon S3 Intelligent-Tiering
** Amazon Commercial Cloud Services
** Ab ONTAP 9.11.1 wird Amazon S3 Glacier Instant Retrieval unterstützt (FabricPool unterstützt Glacier Flexible Retrieval oder Glacier Deep Archive nicht)
+
https://aws.amazon.com/s3/storage-classes/["Amazon Web Services Dokumentation: Amazon S3 Storage Classes"^]



+
Für Informationen zu nicht aufgeführten Speicherklassen steht Ihr Vertriebsmitarbeiter zur Verfügung.

* Auf Cloud Volumes ONTAP unterstützt FabricPool das Tiering von General Purpose SSD (gp2) und Throughput Optimized HDD (st1) Volumes von Amazon Elastic Block Store (EBS).


.Schritte
. Die Amazon S3-Konfigurationsinformationen werden mit dem  `storage aggregate object-store config create` Befehl und dem  `-provider-type` `AWS_S3` Parameter angegeben.
+
** Sie verwenden den  `-auth-type` `CAP`Parameter, um Anmeldeinformationen für den C2S-Zugriff zu erhalten.
+
Wenn Sie den  `-auth-type` `CAP` Parameter verwenden, müssen Sie den  `-cap-url` Parameter verwenden, um die vollständige URL anzugeben, mit der temporäre Anmeldeinformationen für den C2S-Zugriff angefordert werden.

** Der  `storage aggregate object-store config create` Befehl schlägt fehl, wenn ONTAP mit den angegebenen Informationen nicht auf Amazon S3 zugreifen kann.
** Der  `-access-key` Parameter wird verwendet, um den Zugriffsschlüssel für die Autorisierung von Anfragen an den Amazon S3 Objektspeicher anzugeben.
** Der  `-secret-password`-Parameter wird verwendet, um das Passwort (den geheimen Zugriffsschlüssel) für die Authentifizierung von Anfragen an den Amazon S3 Objektspeicher anzugeben.
** Wenn das Amazon S3-Passwort geändert wird, ist das entsprechende Passwort, das in ONTAP gespeichert ist, umgehend zu aktualisieren.
+
Dadurch kann ONTAP ohne Unterbrechung auf die Daten in Amazon S3 zugreifen.

+
[listing]
----
cluster1::> storage aggregate object-store config create
-object-store-name my_aws_store -provider-type AWS_S3
-server s3.amazonaws.com -container-name my-aws-bucket
-access-key DXJRXHPXHYXA9X31X3JX
----
+
[listing]
----
cluster1::> storage aggregate object-store config create -object-store-name my_c2s_store -provider-type AWS_S3 -auth-type CAP -cap-url https://123.45.67.89/api/v1/credentials?agency=XYZ&mission=TESTACCT&role=S3FULLACCESS -server my-c2s-s3server-fqdn -container my-c2s-s3-bucket
----


. Die Amazon S3-Konfigurationsinformationen lassen sich mit dem  `storage aggregate object-store config show` Befehl anzeigen und überprüfen.
+
Der  `storage aggregate object-store config modify`Befehl ermöglicht die Änderung der Amazon S3-Konfigurationsinformationen für FabricPool.
