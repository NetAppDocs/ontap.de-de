= Die Tiering-Richtlinie und die minimale Kühlzeit eines ONTAP-Volumes anpassen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Tiering-Richtlinie eines Volumes ändern, um festzulegen, ob Daten in die Cloud-Ebene verschoben werden, wenn sie inaktiv werden (_cold_). Für ein Volume mit der  `snapshot-only` oder  `auto` Tiering-Richtlinie kann außerdem die minimale Abkühlphase festgelegt werden, die Benutzerdaten inaktiv bleiben müssen, bevor sie in die Cloud-Ebene verschoben werden.

.Bevor Sie beginnen
Das Ändern eines Volumens auf die  `auto` Tiering-Richtlinie oder das Anpassen der minimalen Tiering-Kühlzeit erfordert ONTAP 9.4 oder höher.

.Über diese Aufgabe
Durch die Änderung der Tiering-Richtlinie eines Volumes ändert sich lediglich das nachfolgende Tiering-Verhalten dieses Volumes. Dadurch werden Daten nicht rückwirkend in die Cloud-Tier verschoben.

Eine Änderung der Tiering-Richtlinie kann beeinflussen, wie lange es dauert, bis Daten als „kalt“ gelten und in die Cloud-Tier verschoben werden.

link:tiering-policies-concept.html#what-happens-when-you-modify-a-volume-tiering-policy["Was geschieht, wenn eine Volume-Tiering-Richtlinie geändert wird?"]


NOTE: Bei einer SVM DR-Beziehung müssen Quell- und Ziel-Volumes keine FabricPool Aggregate verwenden, sie müssen jedoch dieselbe Tiering-Richtlinie nutzen.

.Schritte
. Die Tiering-Richtlinie für ein vorhandenes Volume kann mit dem  `volume modify` Befehl und dem  `-tiering-policy` Parameter geändert werden:
+
Eine der folgenden Tiering-Richtlinien kann angegeben werden:

+
** `snapshot-only` (Standard)
** `auto`
** `all`
** `none`
+
link:tiering-policies-concept.html["FabricPool Volume-Tiering-Richtlinien"]



. Wenn das Volume die  `snapshot-only` oder  `auto` Tiering-Richtlinie verwendet und die minimale Abkühlperiode für das Tiering geändert werden soll, kann der  `volume modify` Befehl mit dem  `-tiering-minimum-cooling-days` optionalen Parameter auf der erweiterten Berechtigungsstufe genutzt werden.
+
Sie können einen Wert zwischen 2 und 183 für die Mindestanzahl an Kühltagen für das Tiering angeben. Wenn eine Version von ONTAP vor 9.8 verwendet wird, kann ein Wert zwischen 2 und 63 für die Mindestanzahl an Kühltagen für das Tiering angegeben werden.



.Beispiel für die Änderung der Tiering-Richtlinie und der Mindestkühlzeit für das Tiering eines Volumens
Das folgende Beispiel ändert die Tiering-Richtlinie des Volumes "`myvol`" in der SVM "`vs1`" zu `auto` und die minimale Kühlperiode für das Tiering auf 45 Tage:

[listing]
----
cluster1::> volume modify -vserver vs1 -volume myvol
-tiering-policy auto -tiering-minimum-cooling-days 45
----
