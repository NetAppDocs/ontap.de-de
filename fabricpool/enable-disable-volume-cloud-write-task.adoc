= ONTAP Volumes in FabricPool für das direkte Schreiben in die Cloud aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.14.1 kann das direkte Schreiben in die Cloud auf einem neuen oder bestehenden Volume in einer FabricPool aktiviert oder deaktiviert werden, sodass NFS-Clients Daten direkt in die Cloud schreiben können, ohne auf Tiering-Scans warten zu müssen. SMB-Clients schreiben weiterhin auf die Performance-Ebene in einem Volume mit aktiviertem Cloud-Schreibmodus. Der Cloud-Schreibmodus ist standardmäßig deaktiviert.

Die Möglichkeit, direkt in die Cloud zu schreiben, ist beispielsweise bei Migrationen hilfreich, bei denen große Datenmengen in einen Cluster übertragen werden, die der Cluster auf der lokalen Ebene nicht verarbeiten kann. Ohne Cloud-Schreibmodus werden während einer Migration kleinere Datenmengen übertragen, dann gestuft, dann erneut übertragen und wieder gestuft, bis die Migration abgeschlossen ist. Mit Cloud-Schreibmodus ist diese Art der Verwaltung nicht mehr erforderlich, da die Daten nie auf die lokale Ebene übertragen werden.

.Bevor Sie beginnen
* Sie sind als Cluster- oder SVM-Administrator tätig.
* Sie müssen sich auf der erweiterten Berechtigungsstufe befinden.
* Das Volume muss ein Lese-/Schreib-Volume sein.
* Das Volumen muss die ALL Tiering-Richtlinie aufweisen.




[[enable-writing-directly-to-the-cloud-during-volume-creation]]
== Direktes Schreiben in die Cloud während der Volume-Erstellung ermöglichen

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Ein Volume wird erstellt und der Cloud-Schreibmodus aktiviert:
+
[source, cli]
----
volume create -vserver <svm name> -volume <volume name> -is-cloud-write-enabled <true|false> -aggregate <local tier name>
----
+
Im folgenden Beispiel wird ein Volume mit dem Namen vol1 erstellt, bei dem Cloud Write auf der FabricPool lokalen Ebene (aggr1) aktiviert ist:

+
[listing]
----
volume create -vserver vs1 -volume vol1 -is-cloud-write-enabled true -aggregate aggr1
----




[[enable-writing-directly-to-the-cloud-on-an-existing-volume]]
== Direktes Schreiben in die Cloud auf einem vorhandenen Volume aktivieren

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Ein Volume kann so geändert werden, dass der Cloud-Schreibmodus aktiviert wird:
+
[source, cli]
----
volume modify -vserver <svm name> -volume <volume name> -is-cloud-write-enabled true
----
+
Das folgende Beispiel ändert das Volume mit dem Namen vol1, um Cloud-Schreibvorgänge zu ermöglichen:

+
[listing]
----
volume modify -vserver vs1 -volume vol1 -is-cloud-write-enabled true
----




[[disable-writing-directly-to-the-cloud-on-a-volume]]
== Das direkte Schreiben in die Cloud auf einem Volume deaktivieren

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Cloud-Schreibmodus für ein Volume deaktivieren:
+
[source, cli]
----
volume modify -vserver <svm name> -volume <volume name> -is-cloud-write-enabled false
----
+
Das folgende Beispiel deaktiviert den Cloud-Schreibmodus für das Volume mit dem Namen vol1:

+
[listing]
----
volume modify -vserver vs1 -volume vol1 -is-cloud-write-enabled false
----
