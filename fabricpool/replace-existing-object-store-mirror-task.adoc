= Einen vorhandenen Objektspeicher durch einen ONTAP FabricPool Spiegel ersetzen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die FabricPool Spiegeltechnologie verwenden, um einen Objektspeicher durch einen anderen zu ersetzen. Der neue Objektspeicher muss nicht denselben Cloud-Anbieter wie der ursprüngliche Objektspeicher verwenden.

.Über diese Aufgabe
Sie können den ursprünglichen Objektspeicher durch einen Objektspeicher ersetzen, der einen anderen Cloud-Anbieter verwendet. Beispielsweise könnte Ihr ursprünglicher Objektspeicher AWS als Cloud-Anbieter nutzen, er kann jedoch durch einen Objektspeicher ersetzt werden, der Azure als Cloud-Anbieter verwendet, und umgekehrt. Der neue Objektspeicher muss jedoch die gleiche Objektgröße wie der ursprüngliche beibehalten.

.Schritte
. Einen FabricPool-Spiegel erstellen, indem ein neuer Objektspeicher zu einem bestehenden FabricPool mit dem  `storage aggregate object-store mirror` Befehl hinzugefügt wird.
+
[listing]
----
cluster1::> storage aggregate object-store mirror -aggregate aggr1 -object-store-name my-AZURE-store
----
. Der Status der Spiegel-Resynchronisierung kann mit dem  `storage aggregate object-store show-resync-status` Befehl überwacht werden.
+
[listing]
----
cluster1::> storage aggregate object-store show-resync-status -aggregate aggr1
----
+
[listing]
----
                                                  Complete
      Aggregate    Primary          Mirror        Percentage
      ---------    -----------      ----------    ----------
      aggr1        my-AWS-store     my-AZURE-store     40%
----
. Mit dem  `storage aggregate object-store> show -fields mirror-type,is-mirror-degraded`Befehl kann überprüft werden, ob der Spiegel synchronisiert ist.
+
[listing]
----
cluster1::> storage aggregate object-store show -fields mirror-type,is-mirror-degraded
----
+
[listing]
----
aggregate      object-store-name mirror-type     is-mirror-degraded
-------------- ----------------- ------------- ------------------
aggr1          my-AWS-store        primary           -
               my-AZURE-store      mirror          false
----
. Den primären Objektspeicher mit dem gespiegelten Objektspeicher mithilfe des  `storage aggregate object-store modify` Befehls austauschen.
+
[listing]
----
cluster1::> storage aggregate object-store modify -aggregate aggr1 -object-store-name my-AZURE-store -mirror-type primary
----
. Details zum FabricPool Spiegel werden mit dem  `storage aggregate object-store show -fields mirror-type,is-mirror-degraded` Befehl angezeigt.
+
Dieses Beispiel zeigt die Informationen über den FabricPool Spiegel an, einschließlich der Information, ob der Spiegel defekt ist (nicht synchronisiert).

+
[listing]
----
cluster1::> storage aggregate object-store show -fields mirror-type, is-mirror-degraded
----
+
[listing]
----
aggregate      object-store-name mirror-type     is-mirror-degraded
-------------- ----------------- ------------- ------------------
aggr1          my-AZURE-store      primary           -
               my-AWS-store        mirror          false
----
. Den FabricPool Spiegel mit dem  `storage aggregate object-store unmirror` Befehl entfernen.
+
[listing]
----
cluster1::> storage aggregate object-store unmirror -aggregate aggr1
----
. Überprüfen Sie, ob sich das FabricPool wieder in einer einzelnen Objektspeicherkonfiguration befindet, indem Sie den  `storage aggregate object-store show -fields mirror-type,is-mirror-degraded` Befehl verwenden.
+
[listing]
----
cluster1::> storage aggregate object-store show -fields mirror-type,is-mirror-degraded
----
+
[listing]
----
aggregate      object-store-name mirror-type     is-mirror-degraded
-------------- ----------------- ------------- ------------------
aggr1          my-AZURE-store      primary           -
----
