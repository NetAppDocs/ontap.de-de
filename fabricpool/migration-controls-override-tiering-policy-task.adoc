= Die Standard-FabricPool-Tiering-Richtlinie eines ONTAP-Volumes ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Standard-Tiering-Richtlinie eines Volumes für die Steuerung des Abrufs von Benutzerdaten vom Cloud-Tier zum Performance-Tier kann mit der in ONTAP 9.8 eingeführten  `-cloud-retrieval-policy` Option geändert werden.

.Bevor Sie beginnen
* Die Änderung eines Volumes mit der  `-cloud-retrieval-policy` Option erfordert ONTAP 9.8 oder höher.
* Für diese Operation ist die erweiterte Berechtigungsebene erforderlich.
* Das Verhalten von Tiering-Richtlinien mit  `-cloud-retrieval-policy` sollte verstanden werden.
+
link:tiering-policies-concept.html#how-tiering-policies-work-with-cloud-migration["Wie Tiering-Richtlinien bei der Cloud-Migration funktionieren"]



.Schritt
. Das Verhalten der Tiering-Richtlinie für ein vorhandenes Volume kann mit dem  `volume modify` Befehl und der  `-cloud-retrieval-policy` Option angepasst werden:
+
[listing]
----
 volume create -volume <volume_name> -vserver <vserver_name> - tiering-policy <policy_name> -cloud-retrieval-policy
----
+
[listing]
----
vol modify -volume fp_volume4 -vserver vs0 -cloud-retrieval-policy promote
----
