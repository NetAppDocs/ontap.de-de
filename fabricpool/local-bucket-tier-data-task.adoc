= Tier-Daten in einen lokalen ONTAP S3 Bucket
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.8 ist das Tiering von Daten in lokalen Objektspeicher mithilfe von ONTAP S3 möglich.

Das Tiering von Daten in einen lokalen Bucket bietet eine einfache Alternative zum Verschieben von Daten in eine andere lokale Ebene. Dieses Verfahren verwendet entweder einen vorhandenen Bucket im lokalen Cluster, oder ONTAP kann automatisch eine neue Storage-VM und einen neuen Bucket erstellen.

Beachten Sie, dass der primäre lokale Bucket nach dem Anhängen nicht mehr getrennt werden kann.

.Bevor Sie beginnen
* Für diesen Workflow, der einen neuen S3-Server und einen neuen Bucket erstellt oder bestehende verwendet, ist eine S3-Lizenz erforderlich. Diese Lizenz ist in link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["ONTAP One"] enthalten. Für diesen Workflow ist keine FabricPool Lizenz erforderlich.
* link:../s3-config/enable-ontap-s3-access-local-fabricpool-task.html["ONTAP S3-Zugriff für lokales FabricPool Tiering aktivieren"].


.Schritte
. Daten auf einen lokalen Bucket verschieben: Auf *Storage > Tiers* klicken, im Bereich *SSD* ein lokales Tier auswählen, auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und *Tier zu lokalem Bucket* auswählen.
. Im Abschnitt *Primäre Ebene* entweder *Vorhanden* oder *Neu* auswählen.
. Auf *Speichern* klicken.
