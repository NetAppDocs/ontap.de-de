= ONTAP Volumes in FabricPool für aggressives Vorlesen aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.14.1 kann der aggressive Lesevorlaufmodus auf Volumes in FabricPools aktiviert und deaktiviert werden. In ONTAP 9.13.1 wurde der aggressive Lesevorlaufmodus nur auf Cloud-Plattformen eingeführt. Ab ONTAP 9.14.1 ist der aggressive Lesevorlaufmodus auf allen Plattformen verfügbar, die FabricPool unterstützt, einschließlich lokaler Plattformen. Die Funktion ist standardmäßig deaktiviert.

Wenn aggressives Vorlesen deaktiviert ist, liest FabricPool nur die Dateiblöcke, die eine Clientanwendung benötigt; es ist nicht erforderlich, die gesamte Datei zu lesen. Dies kann zu reduziertem Netzwerkverkehr führen, insbesondere bei großen Dateien im GB- und TB-Bereich. Das _Aktivieren_ von aggressivem Vorlesen auf einem Volume schaltet diese Funktionalität ab, und FabricPool liest die gesamte Datei vorab sequenziell aus dem Objektspeicher, wodurch der GET-Durchsatz erhöht und die Latenzzeit für Client-Lesezugriffe auf die Datei verringert wird. Standardmäßig bleiben gestaffelte Daten beim sequenziellen Lesen kalt und werden nicht auf die lokale Ebene geschrieben.

Aggressives Vorlesen tauscht Netzwerkeffizienz gegen eine gesteigerte Leistung von gestuften Daten.

.Über diese Aufgabe
Der `aggressive-readahead-mode` Befehl hat zwei Optionen:

* `none`: Das Read-Ahead ist deaktiviert.
* `file_prefetch`: Das System liest die gesamte Datei vor der Clientanwendung in den Speicher ein.


.Bevor Sie beginnen
* Sie sind als Cluster- oder SVM-Administrator tätig.
* Sie müssen sich auf der erweiterten Berechtigungsstufe befinden.




[[enable-aggressive-read-ahead-mode-during-volume-creation]]
== Aggressiver Lesevorlaufmodus während der Volume-Erstellung aktivieren

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Ein Volume wird erstellt und der aggressive Read-Ahead-Modus aktiviert:
+
[source, cli]
----
volume create -volume <volume name>  -aggressive-readahead-mode <none|file_prefetch>
----
+
Das folgende Beispiel erstellt ein Volume namens vol1 mit aktiviertem aggressivem Vorlesen über die Option file_prefetch:

+
[listing]
----
volume create -volume vol1 -aggressive-readahead-mode file_prefetch
----




[[disable-aggressive-read-ahead-mode]]
== Aggressiver Read-Ahead-Modus deaktivieren

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Aggressiven Read-Ahead-Modus deaktivieren:
+
[source, cli]
----
volume modify -volume <volume name>  -aggressive-readahead-mode none
----
+
Das folgende Beispiel ändert ein Volume mit dem Namen vol1, sodass der aggressive Lesevorlaufmodus deaktiviert wird:

+
[listing]
----
volume modify -volume vol1 -aggressive-readahead-mode none
----




[[view-aggressive-read-ahead-mode-on-a-volume]]
== Aggressiver Lesevorlaufmodus auf einem Volume anzeigen

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Den aggressiven Vorlesemodus anzeigen:
+
[source, cli]
----
volume show -fields aggressive-readahead-mode
----
