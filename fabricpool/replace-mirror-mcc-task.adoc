= Einen FabricPool Spiegel in einer ONTAP MetroCluster Konfiguration ersetzen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn einer der Objektspeicher in einem FabricPool Spiegel zerstört wird oder in einer MetroCluster Konfiguration dauerhaft nicht mehr verfügbar ist, kann der Objektspeicher zum Spiegel gemacht werden, falls er nicht bereits der Spiegel ist, der beschädigte Objektspeicher aus dem FabricPool Spiegel entfernt und anschließend ein neuer Objektspeicher Spiegel zum FabricPool hinzugefügt werden.

.Schritte
. Falls der beschädigte Objektspeicher noch nicht der Spiegel ist, wird der Objektspeicher mit dem  `storage aggregate object-store modify` Befehl zum Spiegel gemacht.
+
[listing]
----
storage aggregate object-store modify -aggregate -aggregate fp_aggr1_A01 -name mcc1_ostore1 -mirror-type mirror
----
. Die Objektspeicherspiegelung wird mithilfe des Befehls  `storage aggregate object-store unmirror` aus dem FabricPool entfernt.
+
[listing]
----
storage aggregate object-store unmirror -aggregate <aggregate name> -name mcc1_ostore1
----
. Sie können das Tiering auf dem primären Datenspeicher fortsetzen, nachdem Sie den gespiegelten Datenspeicher entfernt haben, indem Sie  `storage aggregate object-store modify` mit der  `-force-tiering-on-metrocluster true` Option verwenden.
+
Das Fehlen eines Spiegels beeinträchtigt die Replikationsanforderungen einer MetroCluster Konfiguration.

+
[listing]
----
storage aggregate object-store modify -aggregate <aggregate name> -name mcc1_ostore1 -force-tiering-on-metrocluster true
----
. Ein Ersatz-Objektspeicher wird mithilfe des  `storage aggregate object-store config create` Befehls erstellt.
+
[listing]
----
storage aggregate object-store config create -object-store-name mcc1_ostore3 -cluster clusterA -provider-type SGWS -server <SGWS-server-1> -container-name <SGWS-bucket-1> -access-key <key> -secret-password <password> -encrypt <true|false> -provider <provider-type> -is-ssl-enabled <true|false> ipspace <IPSpace>
----
. Den Objektspeicher-Mirror zur FabricPool Mirror mithilfe des  `storage aggregate object-store mirror` Befehls hinzufügen.
+
[listing]
----
storage aggregate object-store mirror -aggregate aggr1 -name mcc1_ostore3-mc
----
. Die Objektspeicherinformationen werden mit dem  `storage aggregate object-store show` Befehl angezeigt.
+
[listing]
----
storage aggregate object-store show -fields mirror-type,is-mirror-degraded
----
+
[listing]
----
aggregate      object-store-name mirror-type     is-mirror-degraded
-------------- ----------------- ------------- ------------------
aggr1          mcc1_ostore1-mc    primary           -
               mcc1_ostore3-mc    mirror          true
----
. Der Status der Spiegel-Resynchronisierung kann mit dem  `storage aggregate object-store show-resync-status` Befehl überwacht werden.
+
[listing]
----
storage aggregate object-store show-resync-status -aggregate aggr1
----
+
[listing]
----
                                                  Complete
      Aggregate    Primary        Mirror          Percentage
      ---------    -----------    ----------      ----------
      aggr1        mcc1_ostore1-mc mcc1_ostore3-mc   40%
----
