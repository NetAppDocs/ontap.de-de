= Informationen zum Data Tiering mit ONTAP FabricPool
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit FabricPool kann Daten automatisch je nach Zugriffshäufigkeit in verschiedene Ebenen verschoben werden.

FabricPool ist eine Hybrid-Speicherlösung, die auf AFF Systemen ein All-Flash-Aggregat (ausschließlich SSDs) nutzt und auf FAS Systemen entweder ein All-Flash-Aggregat (ausschließlich SSDs) oder ein HDD-Aggregat als Leistungsebene sowie einen Objektspeicher als Cloud-Ebene einsetzt. Die Nutzung eines FabricPool trägt dazu bei, die Speicherkosten zu senken, ohne Leistung, Effizienz oder Schutz zu beeinträchtigen.

Die Cloud-Ebene kann sich auf NetApp StorageGRID oder ONTAP S3 (ab ONTAP 9.8) oder bei einem der folgenden Dienstanbieter befinden:

* Alibaba Cloud
* Amazon S3
* Amazon Commercial Cloud Services
* Google Cloud
* IBM Cloud
* Microsoft Azure Blob Storage


[NOTE]
====
Ab ONTAP 9.7 können zusätzliche Objektspeicheranbieter, die generische S3-APIs unterstützen, durch Auswahl des S3_Compatible Objektspeicheranbieters verwendet werden.
