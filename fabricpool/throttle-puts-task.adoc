= Schwellenwerte für die ONTAP FabricPool Put-Rate pro Knoten festlegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Als Storage-Administrator kann die PUT-Drosselung verwendet werden, um einen oberen Schwellenwert für die maximale PUT-Rate pro Node festzulegen.

PUT-Drosselung ist sinnvoll, wenn Netzwerkressourcen oder der Objektspeicher-Endpunkt ressourcenbeschränkt sind. Obwohl selten, können Ressourcenbeschränkungen bei leistungsschwachen Objektspeichern oder während der ersten Tage der FabricPool Nutzung auftreten, wenn TB oder PB an kalten Daten ausgelagert werden.

Die PUT-Drosselung erfolgt pro Knoten. Das minimale PUT-Drosselungs-Limit beträgt 8 MB/s. Wenn das PUT-Drosselungs-Limit auf einen Wert unter 8 MB/s gesetzt wird, beträgt der Durchsatz an diesem Knoten 8 MB/s. Mehrere Knoten, die gleichzeitig Tiering durchführen, können mehr Bandbreite verbrauchen und möglicherweise eine Netzwerkverbindung mit extrem begrenzter Kapazität auslasten.

[NOTE]
====
FabricPool PUT-Operationen konkurrieren nicht mit anderen Anwendungen um Ressourcen. FabricPool PUT-Operationen werden von Clientanwendungen und anderen ONTAP Workloads, wie SnapMirror, automatisch niedriger priorisiert („nachrangig behandelt“). PUT-Drosselung mithilfe von  `put-rate-limit` kann hilfreich sein, um den Netzwerkverkehr im Zusammenhang mit FabricPool Tiering zu reduzieren, steht jedoch in keinem Zusammenhang mit gleichzeitigem ONTAP-Verkehr.

====
.Bevor Sie beginnen
Es ist eine erweiterte Berechtigungsstufe erforderlich.

.Schritte
. Drosselung von FabricPool PUT-Operationen mithilfe der ONTAP CLI:
+
[source, cli]
----
storage aggregate object-store put-rate-limit modify -node <name> -default <true|false> -put-rate-bytes-limit <integer>[KB|MB|GB|TB|PB]
----
