= StorageGRID als ONTAP FabricPool Cloud-Tier einrichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
StorageGRID kann als Cloud-Tier für FabricPool eingerichtet werden. Beim Tiering von Daten, auf die über SAN-Protokolle zugegriffen wird, empfiehlt NetApp aufgrund von Konnektivitätsaspekten die Nutzung privater Clouds wie StorageGRID.

.Überlegungen zur Verwendung von StorageGRID mit FabricPool
* Ein CA-Zertifikat für StorageGRID ist erforderlich, es sei denn, die Zertifikatsprüfung wird explizit deaktiviert.
* Die StorageGRID Objektversionierung sollte für den Objektspeicher-Bucket nicht aktiviert werden.
* Eine FabricPool Lizenz ist nicht erforderlich.
* Wenn ein StorageGRID Knoten in einer virtuellen Maschine mit Speicherzuweisung aus einem NetApp AFF System bereitgestellt wird, ist zu bestätigen, dass für das Volume keine FabricPool Tiering-Richtlinie aktiviert ist.
+
Das Deaktivieren des FabricPool Tierings für Volumes, die mit StorageGRID Knoten verwendet werden, vereinfacht die Fehlersuche und den Speicherbetrieb.

+
[NOTE]
====
Es sollte niemals FabricPool verwendet werden, um Daten, die mit StorageGRID in Zusammenhang stehen, wieder auf StorageGRID selbst zu verschieben. Das Tiering von StorageGRID-Daten zurück auf StorageGRID erhöht die Komplexität bei der Fehlerbehebung und im Betrieb.

====


.Über diese Aufgabe
Lastausgleich ist für StorageGRID in ONTAP 9.8 und höher aktiviert. Wenn der Hostname des Servers zu mehr als einer IP-Adresse aufgelöst wird, stellt ONTAP Client-Verbindungen mit allen zurückgegebenen IP-Adressen her (bis zu maximal 16 IP-Adressen). Die IP-Adressen werden beim Verbindungsaufbau nach dem Round-Robin-Verfahren ausgewählt.

.Schritte
StorageGRID kann als Cloud-Tier für FabricPool mit ONTAP System Manager oder der ONTAP CLI eingerichtet werden.

[role="tabbed-block"]
====
.System Manager
--
. Wählen Sie *Speicher > Ebenen > Cloud-Ebene hinzufügen* und dann StorageGRID als Objektspeicher-Anbieter aus.
. Die angeforderten Informationen sind auszufüllen.
. Zum Erstellen eines Cloud-Mirrors auf *Als FabricPool Mirror hinzufügen* klicken.


Ein FabricPool Mirror bietet eine Methode, einen Datenspeicher nahtlos zu ersetzen, und trägt dazu bei, dass Ihre Daten im Katastrophenfall verfügbar sind.

--
.CLI
--
. Die StorageGRID Konfigurationsinformationen werden mit dem  `storage aggregate object-store config create` Befehl und dem  `-provider-type` `SGWS` Parameter angegeben.
+
** Der  `storage aggregate object-store config create`Befehl schlägt fehl, wenn ONTAP mit den angegebenen Informationen nicht auf StorageGRID zugreifen kann.
** Der  `-access-key`Parameter wird verwendet, um den Zugriffsschlüssel für die Autorisierung von Anfragen an den StorageGRID Objektspeicher anzugeben.
** Der  `-secret-password` Parameter wird verwendet, um das Passwort (Secret Access Key) für die Authentifizierung von Anfragen an den StorageGRID Objektspeicher anzugeben.
** Wenn das StorageGRID Passwort geändert wird, sollte das entsprechende in ONTAP gespeicherte Passwort umgehend aktualisiert werden.
+
Dadurch kann ONTAP ohne Unterbrechung auf die Daten in StorageGRID zugreifen.

** Das Setzen des  `-is-certificate-validation-enabled`-Parameters auf  `false`deaktiviert die Zertifikatsprüfung für StorageGRID. Die Verwendung signierter Zertifikate (`-is-certificate-validation-enabled true`von einer Drittanbieter-Zertifizierungsstelle gilt als empfohlene Best Practice.


+
[listing]
----
cluster1::> storage aggregate object-store config create
-object-store-name mySGWS -provider-type SGWS -server mySGWSserver
-container-name mySGWScontainer -access-key mySGWSkey
-secret-password mySGWSpass
----
. Die StorageGRID-Konfigurationsinformationen lassen sich mit dem  `storage aggregate object-store config show` Befehl anzeigen und überprüfen.
+
Der  `storage aggregate object-store config modify`Befehl ermöglicht die Änderung der StorageGRID-Konfigurationsinformationen für FabricPool.
