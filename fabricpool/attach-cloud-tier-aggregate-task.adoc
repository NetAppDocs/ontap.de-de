= Die ONTAP Cloud-Tier mit einer lokalen Tier verknüpfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem ein Objektspeicher als Cloud-Ebene eingerichtet wurde, wird die zu verwendende lokale Ebene angegeben, indem sie an FabricPool angehängt wird. In ONTAP 9.5 und höher können auch lokale Ebenen angehängt werden, die qualifizierte FlexGroup Volume-Bestandteile enthalten.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Über diese Aufgabe
Das Verbinden einer Cloud-Ebene mit einer lokalen Ebene ist ein dauerhafter Vorgang. Eine Cloud-Ebene kann nach dem Verbinden nicht mehr von einer lokalen Ebene getrennt werden. Es besteht jedoch die Möglichkeit, link:../fabricpool/create-mirror-task.html["FabricPool Spiegel"] eine lokale Ebene mit einer anderen Cloud-Ebene zu verbinden.

.Bevor Sie beginnen
Wenn die ONTAP CLI verwendet wird, um eine lokale Ebene für FabricPool einzurichten, muss die lokale Ebene bereits existieren.

[NOTE]
====
Wenn der System Manager zur Einrichtung einer lokalen Ebene für FabricPool verwendet wird, kann die lokale Ebene erstellt und gleichzeitig für FabricPool eingerichtet werden.

====
.Schritte
Eine lokale Ebene kann mit ONTAP System Manager oder der ONTAP CLI an einen FabricPool Objektspeicher angebunden werden.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Speicher > Ebenen*, wählen Sie eine Cloud-Ebene aus und klicken Sie dann auf image:icon_kabob.gif["Symbol für Menüoptionen"].
. *Lokale Ebenen anhängen* auswählen.
. Unter *Als primär hinzufügen* wird überprüft, ob die Volumes zum Anhängen geeignet sind.
. Gegebenenfalls *Volumes in Thin Provisioning konvertieren* auswählen.
. Auf *Speichern* klicken.


--
.CLI
--
.Einen Objektspeicher mit der CLI an ein Aggregat anbinden:
. *Optional*: Um anzuzeigen, wie viele Daten in einem Volume inaktiv sind, die Schritte in link:determine-data-inactive-reporting-task.html["Ermittlung des Anteils inaktiver Daten in einem Volume mithilfe der Berichterstattung über inaktive Daten"] befolgen.
+
Zu sehen, wie viele Daten in einem Volume inaktiv sind, kann bei der Entscheidung helfen, welches Aggregat für FabricPool verwendet wird.

. Den Objektspeicher mit dem  `storage aggregate object-store attach` Befehl an ein Aggregat anbinden.
+
Wenn das Aggregat noch nie mit FabricPool verwendet wurde und bereits vorhandene Volumes enthält, wird den Volumes die Standard `snapshot-only` Tiering-Richtlinie zugewiesen.

+
[listing]
----
cluster1::> storage aggregate object-store attach -aggregate myaggr
-object-store-name Amazon01B1
----
+
Sie können die  `allow-flexgroup` `true` Option verwenden, um Aggregate anzuhängen, die FlexGroup-Volume-Bestandteile enthalten.

. Die Objektspeicherinformationen werden angezeigt und mit dem  `storage aggregate object-store show` Befehl lässt sich überprüfen, ob der angeschlossene Objektspeicher verfügbar ist.
+
[listing]
----
cluster1::> storage aggregate object-store show

Aggregate    Object Store Name    Availability State
---------    -----------------    ------------------
myaggr       Amazon01B1           available
----
