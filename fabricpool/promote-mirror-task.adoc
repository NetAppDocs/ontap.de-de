= Einen ONTAP FabricPool Spiegel promoten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Objektspeicher Spiegel kann durch Hochstufen als primärer Objektspeicher zugewiesen werden. Wenn der Objektspeicher Spiegel zum primären Objektspeicher wird, wird der ursprüngliche primäre Objektspeicher automatisch zum Spiegel.

.Bevor Sie beginnen
* Der FabricPool Spiegel muss synchron sein
* Der Objektspeicher muss betriebsbereit sein


.Über diese Aufgabe
Sie können den ursprünglichen Objektspeicher durch einen Objektspeicher eines anderen Cloud-Anbieters ersetzen. Beispielsweise könnte Ihr ursprünglicher Spiegel ein AWS Objektspeicher sein, der durch einen Azure Objektspeicher ersetzt werden kann.

.Schritte
. Es kann überprüft werden, ob der FabricPool-Spiegel synchron ist, indem der  `storage aggregate object-store show-resync-status` Befehl verwendet wird. Wenn der FabricPool-Spiegel synchron ist, werden keine Einträge angezeigt. Ist der Spiegel nicht synchron, sollte auf den Abschluss der Resynchronisierung gewartet werden.
+
[listing]
----
aggregate1::> storage aggregate object-store show-resync-status -aggregate aggr1
----
+
[listing]
----
                                               Complete
      Aggregate    Primary       Mirror        Percentage
      ---------    -----------   ----------    ----------
      aggr1        my-store-1     my-store-2     40%
----
. Einen Objektspeicher-Mirror mit dem  `storage aggregate object-store modify -aggregate` Befehl aktivieren.
+
[listing]
----
cluster1::> storage aggregate object-store modify -aggregate aggr1 -name my-store-2 -mirror-type primary
----
