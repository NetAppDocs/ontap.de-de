= ONTAP FabricPool Volumes mit benutzerdefinierten, vom Benutzer erstellten Tags verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.8 unterstützt FabricPool die Objektkennzeichnung mit benutzerdefinierten, vom Benutzer erstellten Tags, sodass Objekte zur einfacheren Verwaltung klassifiziert und sortiert werden können. Wenn Sie über die Admin-Berechtigungsstufe verfügen, können neue Objekt-Tags erstellt sowie bestehende Tags geändert, gelöscht und angezeigt werden.



[[assign-a-new-tag-during-volume-creation]]
== Während der Volume-Erstellung ein neues Tag zuweisen

Sie können ein neues Objekt-Tag erstellen, wenn Sie einem oder mehreren neuen Objekten, die aus einem von Ihnen erstellten neuen Volume ausgelagert werden, Tags zuweisen möchten. Tags können zur Klassifizierung und Sortierung von Tiering-Objekten verwendet werden, um die Datenverwaltung zu erleichtern. Ab ONTAP 9.8 kann der System Manager zur Erstellung von Objekt-Tags verwendet werden.

.Über diese Aufgabe
Tags können nur auf FabricPool Volumes vergeben werden, die an StorageGRID angebunden sind. Diese Tags bleiben beim Verschieben eines Volumes erhalten.

* Pro Volume sind maximal vier Tags zulässig.
* In der CLI muss jedes Objekt-Tag ein Schlüssel-Wert-Paar sein, das durch ein Gleichheitszeichen getrennt ist.
* In der CLI müssen mehrere Tags durch ein Komma getrennt werden.
* Jeder Tag-Wert kann maximal 127 Zeichen enthalten.
* Jeder Tag-Schlüssel muss entweder mit einem alphabetischen Zeichen oder einem Unterstrich beginnen.
+
Schlüssel dürfen nur alphanumerische Zeichen und Unterstriche enthalten, und die maximal zulässige Zeichenanzahl beträgt 127.



Sie können Objekt-Tags mit ONTAP System Manager oder ONTAP CLI zuweisen.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. Zu *Storage > Tiers* navigieren.
. Eine Speicherebene mit Volumes, die getaggt werden sollen, finden.
. Die Registerkarte *Volumes* auswählen.
. Suchen Sie das Volume, das Sie taggen möchten, und wählen Sie in der Spalte *Objekt-Tags* *Klicken zum Eingeben von Tags* aus.
. Geben Sie einen Schlüssel und einen Wert ein.
. Auf *Anwenden* klicken.


--
.CLI
--
. Der  `volume create` Befehl mit der  `-tiering-object-tags` Option ermöglicht das Erstellen eines neuen Volumes mit den angegebenen Tags. Mehrere Tags können als durch Kommas getrennte Paare angegeben werden:
+
[listing]
----
volume create [ -vserver <vserver name> ] -volume <volume_name> -tiering-object-tags <key1=value1> [,<key2=value2>,<key3=value3>,<key4=value4> ]
----
+
Im folgenden Beispiel wird ein Volume mit dem Namen fp_volume1 und drei Objekttags erstellt.

+
[listing]
----
vol create -volume fp_volume1 -vserver vs0 -tiering-object-tags project=fabricpool,type=abc,content=data
----


--
====


[[modify-an-existing-tag]]
== Ein vorhandenes Tag ändern

Sie können den Namen eines Tags ändern, Tags bei bestehenden Objekten im Objektspeicher ersetzen oder neuen Objekten, die Sie später hinzufügen möchten, einen anderen Tag hinzufügen.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. Zu *Storage > Tiers* navigieren.
. Eine Speicherebene mit Volumes, die die zu ändernden Tags enthalten, kann gefunden werden.
. Die Registerkarte *Volumes* auswählen.
. Das Volume mit den Tags, die geändert werden sollen, in der Spalte *Objekt-Tags* suchen und auf den Tag-Namen klicken.
. Das Tag ändern.
. Auf *Anwenden* klicken.


--
.CLI
--
. Der  `volume modify`Befehl mit der  `-tiering-object-tags`Option dient dazu, ein vorhandenes Tag zu ändern.
+
[listing]
----
volume modify [ -vserver <vserver name> ] -volume <volume_name> -tiering-object-tags <key1=value1> [ ,<key2=value2>,
<key3=value3>,<key4=value4> ]
----
+
Das folgende Beispiel ändert den Namen des vorhandenen Tags  `type=abc` zu  `type=xyz`.

+
[listing]
----
vol modify -volume fp_volume1 -vserver vs0 -tiering-object-tags project=fabricpool,type=xyz,content=data
----


--
====


[[delete-a-tag]]
== Tag löschen

Sie können Objekt-Tags löschen, wenn Sie diese nicht mehr für ein Volume oder für Objekte im Objektspeicher festlegen möchten.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. Zu *Storage > Tiers* navigieren.
. Eine Speicherebene mit Volumes, die Tags enthalten, die gelöscht werden sollen, ermitteln.
. Die Registerkarte *Volumes* auswählen.
. Suchen Sie das Volume mit den Tags, die Sie löschen möchten, und klicken Sie in der Spalte *Objekt-Tags* auf den Tagnamen.
. Zum Löschen des Tags auf das Papierkorbsymbol klicken.
. Auf *Anwenden* klicken.


--
.CLI
--
. Der  `volume modify`Befehl mit der  `-tiering-object-tags`Option, gefolgt von einem leeren Wert (`""`), löscht ein vorhandenes Tag.
+
Das folgende Beispiel löscht die vorhandenen Tags auf fp_volume1.

+
[listing]
----
vol modify -volume fp_volume1 -vserver vs0 -tiering-object-tags ""
----


--
====


[[view-existing-tags-on-a-volume]]
== Vorhandene Tags eines Volumes anzeigen

Sie können die vorhandenen Tags eines Volumes anzeigen, um zu sehen, welche Tags verfügbar sind, bevor neue Tags zur Liste hinzugefügt werden.

.Schritte
. Der  `volume show`Befehl kann mit der  `tiering-object-tags`Option verwendet werden, um vorhandene Tags auf einem Volume anzuzeigen.
+
[listing]
----
volume show [ -vserver <vserver name> ] -volume <volume_name> -fields tiering-object-tags
----




[[check-object-tagging-status-on-fabricpool-volumes]]
== Objekt-Tagging-Status auf FabricPool Volumes prüfen

Sie können überprüfen, ob die Verschlagwortung auf einem oder mehreren FabricPool Volumes abgeschlossen ist.

.Schritte
. Der  `vol show`Befehl mit der  `-fields needs-object-retagging`Option zeigt an, ob die Verschlagwortung läuft, abgeschlossen ist oder nicht festgelegt wurde.
+
[listing]
----
vol show -fields needs-object-retagging  [ -instance | -volume <volume name>]
----
+
Einer der folgenden Werte wird angezeigt:

+
** `true`: Der Objekt-Tagging-Scanner wurde für dieses Volume noch nicht ausgeführt oder muss erneut ausgeführt werden.
** `false`: Der Objekt-Tagging-Scanner hat das Tagging für dieses Volume abgeschlossen.
** `+<->+`: Der Objekt-Tagging-Scanner ist für dieses Volume nicht anwendbar. Dies tritt bei Volumes auf, die sich nicht auf FabricPools befinden.
