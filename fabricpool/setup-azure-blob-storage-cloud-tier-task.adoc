= Azure Blob Storage als ONTAP FabricPool Cloud-Tier einrichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie ONTAP 9.4 oder höher verwenden, kann Azure Blob Storage als Cloud-Ebene für FabricPool eingerichtet werden.

.Überlegungen zur Verwendung von Microsoft Azure Blob Storage mit FabricPool
* A link:install-license-aws-azure-ibm-task.html["FabricPool Lizenz"] ist erforderlich, wenn das Tiering von lokalen ONTAP Clustern auf Azure Blob Storage erfolgt.
* Für die Nutzung von Azure Blob Storage über Cloud Volumes ONTAP oder Azure NetApp Files ist keine Lizenz erforderlich.
* Es wird empfohlen, dass der LIF, den ONTAP zur Verbindung mit dem Azure Blob Storage Objektspeicher verwendet, an einem 10 Gbit/s Port angeschlossen ist.
* FabricPool unterstützt derzeit Azure Stack nicht, bei dem es sich um lokale Azure-Dienste handelt.
* Auf Kontoebene in Microsoft Azure Blob Storage unterstützt FabricPool nur die Speicherebenen Hot und Cool.
+
FabricPool unterstützt kein Tiering auf Blob-Ebene. Ebenso wird Tiering auf die Archivspeicher-Ebene von Azure nicht unterstützt.



.Über diese Aufgabe
FabricPool unterstützt derzeit Azure Stack nicht, bei dem es sich um lokale Azure-Dienste handelt.

.Schritte
. Die Konfigurationsinformationen für Azure Blob Storage werden mit dem  `storage aggregate object-store config create` Befehl und dem  `-provider-type` `Azure_Cloud` Parameter angegeben.
+
** Der  `storage aggregate object-store config create` Befehl schlägt fehl, wenn ONTAP mit den angegebenen Informationen nicht auf Azure Blob Storage zugreifen kann.
** Sie verwenden den  `-azure-account` Parameter, um das Azure Blob Storage Konto anzugeben.
** Der  `-azure-private-key`Parameter wird verwendet, um den Zugriffsschlüssel für die Authentifizierung von Anfragen an Azure Blob Storage anzugeben.
** Wenn das Kennwort für Azure Blob Storage geändert wird, ist das entsprechende in ONTAP gespeicherte Kennwort umgehend zu aktualisieren.
+
Dadurch kann ONTAP ohne Unterbrechung auf die Daten in Azure Blob Storage zugreifen.



+
[listing]
----
cluster1::> storage aggregate object-store config create
-object-store-name MyAzure -provider-type Azure_Cloud
-server blob.core.windows.net -container-name myAzureContainer
-azure-account myAzureAcct -azure-private-key myAzureKey
----
. Die Konfigurationsinformationen von Azure Blob Storage lassen sich mit dem  `storage aggregate object-store config show` Befehl anzeigen und überprüfen.
+
Der  `storage aggregate object-store config modify`Befehl ermöglicht die Änderung der Azure Blob Storage-Konfigurationsinformationen für FabricPool.
