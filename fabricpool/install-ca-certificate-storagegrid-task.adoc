= Installation eines CA-Zertifikats auf einem ONTAP Cluster für StorageGRID
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Verwendung von CA-Zertifikaten schafft eine vertrauenswürdige Beziehung zwischen Client-Anwendungen und StorageGRID.

Sofern Sie nicht vorhaben, die Zertifikatsprüfung für StorageGRID zu deaktivieren, müssen Sie ein StorageGRID CA-Zertifikat auf dem Cluster installieren, damit ONTAP sich bei StorageGRID als Objektspeicher für FabricPool authentifizieren kann.

Obwohl StorageGRID selbstsignierte Zertifikate generieren kann, gilt die Verwendung signierter Zertifikate einer Drittanbieter-Zertifizierungsstelle als empfohlene Best Practice.

.Über diese Aufgabe
Obwohl die Installation und Verwendung von Zertifikaten der Zertifizierungsstelle (CA) als empfohlene Best Practice gilt, ist ab ONTAP 9.4 die Installation von CA-Zertifikaten für StorageGRID nicht erforderlich.

.Schritte
. Wenden Sie sich an Ihren StorageGRID Administrator, um die  https://docs.netapp.com/us-en/storagegrid-118/admin/configuring-storagegrid-certificates-for-fabricpool.html["CA-Zertifikat des StorageGRID Systems"^] zu erhalten.
. Der  `security certificate install`-Befehl mit dem  `-type` `server-ca`-Parameter dient zur Installation des StorageGRID CA-Zertifikats auf dem Cluster.
+
Der vollqualifizierte Domainname (FQDN), den Sie eingeben, muss mit dem benutzerdefinierten allgemeinen Namen auf dem StorageGRID CA-Zertifikat übereinstimmen.





[[update-an-expired-certificate]]
== Ein abgelaufenes Zertifikat aktualisieren

Um ein abgelaufenes Zertifikat zu aktualisieren, empfiehlt es sich, das neue Serverzertifikat von einer vertrauenswürdigen Zertifizierungsstelle (CA) generieren zu lassen. Zusätzlich sollte sichergestellt werden, dass das Zertifikat sowohl auf dem StorageGRID Server als auch auf dem ONTAP Cluster gleichzeitig aktualisiert wird, um Ausfallzeiten auf ein Minimum zu beschränken.

.Verwandte Informationen
