= ONTAP Daten in die Leistungsebene hochstufen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.8 können Sie als Cluster-Administrator mit erweiterten Berechtigungen Daten proaktiv von der Cloud-Ebene in die Performance-Ebene verschieben, indem Sie eine Kombination aus der  `tiering-policy` und der  `cloud-retrieval-policy` Einstellung verwenden.

.Über diese Aufgabe
Dies kann sinnvoll sein, wenn Sie FabricPool auf einem Volume nicht mehr verwenden möchten oder wenn Sie eine  `snapshot-only` Tiering-Richtlinie haben und wiederhergestellte Snapshot-Daten zurück in die Performance-Ebene verschieben möchten.



[[promote-all-data-from-a-fabricpool-volume-to-the-performance-tier]]
== Alle Daten von einem FabricPool Volume auf die Leistungsebene verschieben

Sie können proaktiv alle Daten eines FabricPool Volumes in der Cloud-Ebene abrufen und sie in die Performance-Ebene verschieben.

.Schritte
. Mit dem  `volume modify`Befehl wird  `tiering-policy`auf  `none` und  `cloud-retrieval-policy` auf  `promote` gesetzt.
+
[listing]
----
volume modify -vserver <vserver-name> -volume <volume-name> -tiering-policy none -cloud-retrieval-policy promote
----




[[promote-file-system-data-to-the-performance-tier]]
== Dateisystemdaten in die Performance-Ebene verschieben

Sie können proaktiv aktive Dateisystemdaten aus einem wiederhergestellten Snapshot in der Cloud-Ebene abrufen und in die Performance-Ebene hochstufen.

.Schritte
. Mit dem  `volume modify`Befehl wird  `tiering-policy`auf  `snapshot-only` und  `cloud-retrieval-policy` auf  `promote` gesetzt.
+
[listing]
----
volume modify -vserver <vserver-name> -volume <volume-name> -tiering-policy snapshot-only cloud-retrieval-policy promote
----




[[check-the-status-of-a-performance-tier-promotion]]
== Status einer Leistungsstufenbeförderung prüfen

Sie können den Status der Performance-Tier-Promotion überprüfen, um festzustellen, wann der Vorgang abgeschlossen ist.

.Schritte
. Den  `object-store`Befehl volume mit der  `tiering`Option verwenden, um den Status der Leistungsstufenaufwertung zu prüfen.
+
[listing]
----
volume object-store tiering show [ -instance | -fields <fieldname>, ... ] [ -vserver <vserver name> ] *Vserver
[[-volume] <volume name>] *Volume [ -node <nodename> ] *Node Name [ -vol-dsid <integer> ] *Volume DSID
[ -aggregate <aggregate name> ] *Aggregate Name
----
+
[listing]
----

[[trigger-scheduled-migration-and-tiering]]
== Geplante Migration und Tiering auslösen

Ab ONTAP 9.8 kann eine Tiering-Scan-Anforderung jederzeit ausgelöst werden, wenn nicht auf den standardmäßigen Tiering-Scan gewartet werden soll.

.Schritte
. Mit dem  `volume object-store`Befehl und der  `trigger`Option kann Migration und Tiering angefordert werden.
+
[listing]
----
volume object-store tiering trigger [ -vserver <vserver name> ] *VServer Name [-volume] <volume name> *Volume Name
----
