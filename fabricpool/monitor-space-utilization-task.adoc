= Speichernutzung einer FabricPool aktivierten ONTAP lokalen Tier überwachen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen wissen, wie viele Daten in den Performance- und Cloud-Tiers für FabricPool gespeichert sind. Diese Information hilft dabei festzustellen, ob die Tiering-Richtlinie eines Volumes geändert, das FabricPool lizenzierte Nutzungslimit erhöht oder der Speicherplatz des Cloud-Tiers erweitert werden sollte.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Über diese Aufgabe
Ab ONTAP 9.18.1 ändert der `storage aggregate show-space` Befehl die Art und Weise, wie die logische referenzierte Kapazität und die logische nicht referenzierte Kapazität gemeldet werden. Die logische referenzierte Kapazität meldet referenzierte Blöcke in allen Objekten und nicht referenzierte Blöcke in fragmentierten Objekten. Die logische nicht referenzierte Kapazität meldet nur nicht referenzierte Blöcke in Objekten, die den Füllschwellenwert überschritten haben und für die Objektlöschung und Defragmentierung in Frage kommen.

Wenn Sie beispielsweise den standardmäßigen Schwellenwert für die aggregierte Auslastung von 40 % für ONTAP S3 und StorageGRID verwenden, müssen 60 % der Blöcke in einem Objekt nicht referenziert sein, bevor die Blöcke als nicht referenzierte Kapazität gemeldet werden.

In Versionen vor ONTAP 9.18.1 meldet die logische referenzierte Kapazität referenzierte Blöcke in allen Objekten (sowohl vollständigen als auch fragmentierten Objekten). Die logische nicht referenzierte Kapazität meldet nicht referenzierte Blöcke in allen Objekten.

.Schritte
. Die Speichernutzung für FabricPool aktivierte lokale Ebenen lässt sich mit einem der folgenden Befehle anzeigen:
+
|===


| Wenn Sie anzeigen möchten... | Verwenden Sie dann diesen Befehl: 


 a| 
Die genutzte Größe der Cloud-Tier in einer lokalen Tier
 a| 
`storage aggregate show` mit dem  `-instance` Parameter



 a| 
Einzelheiten zur Speichernutzung innerhalb einer lokalen Ebene, einschließlich der referenzierten Kapazität des Objektspeichers
 a| 
`storage aggregate show-space` mit dem  `-instance` Parameter



 a| 
Speichernutzung der Objektspeicher, die den lokalen Ebenen zugeordnet sind, einschließlich des genutzten Lizenzspeicherplatzes
 a| 
`storage aggregate object-store show-space`



 a| 
Eine Liste der Volumes in einer lokalen Ebene sowie die Footprints ihrer Daten und Metadaten
 a| 
`volume show-footprint`

|===
+
Zusätzlich zur Verwendung von CLI-Befehlen kann Active IQ Unified Manager (ehemals OnCommand Unified Manager) zusammen mit FabricPool Advisor, das auf ONTAP 9.4 und späteren Clustern unterstützt wird, oder System Manager zur Überwachung der Speichernutzung eingesetzt werden.

. Je nach Bedarf ist eine der folgenden Maßnahmen zu ergreifen:
+
|===


| Wenn Sie möchten... | Dann... 


 a| 
Die Tiering-Richtlinie eines Volumens ändern
 a| 
Die Vorgehensweise in link:modify-tiering-policy-cooling-period-task.html["Verwaltung des Speicher-Tierings durch Ändern der Tiering-Richtlinie eines Volumes oder der minimalen Abkühlzeit für Tiering"] ist zu beachten.



 a| 
Das Limit für die lizenzierte Nutzung von FabricPool erhöhen
 a| 
Ihr NetApp Vertriebsmitarbeiter oder Vertriebspartner steht als Ansprechpartner zur Verfügung.

https://mysupport.netapp.com/site/global/dashboard["NetApp Support"^]



 a| 
Den Speicherplatz der Cloud-Tier erhöhen
 a| 
Wenden Sie sich an den Anbieter des Objektspeichers, den Sie für die Cloud-Tier verwenden.

|===
