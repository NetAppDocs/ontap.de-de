= Die Latenz- und Durchsatzleistung der ONTAP Cloud-Tier wird getestet.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor ein Objektspeicher an eine lokale Ebene angebunden wird, lässt sich die Latenz und die Durchsatzleistung des Objektspeichers mithilfe des Object Store Profilers testen.

[NOTE]
====
Die Ergebnisse des Objektspeicherprofilers sind eine Messung der Konnektivität zwischen ONTAP und dem Cloud-Tier Objektspeicher mithilfe von 4MB PUTs und zufälligen lesen/schreiben Bytebereichs-GETs im Bereich von 4MB bis 256KB. (Nur interne ONTAP Features, wie SnapMirror, können GETs größer als 32KB nutzen.)

Da konkurrierende Arbeitslasten oder das individuelle Verhalten von Clientanwendungen nicht berücksichtigt werden, sind die Ergebnisse des Objektspeicherprofilers kein perfekter Indikator für die Tiering-Performance.

====
.Bevor Sie beginnen
* Die Cloud-Ebene muss zu ONTAP hinzugefügt werden, bevor sie mit dem Objektspeicher-Profiling verwendet werden kann.
* Sie müssen sich im erweiterten Berechtigungsmodus der ONTAP CLI befinden.


.Schritte
. Objektspeicher-Profiling starten:
+
`storage aggregate object-store profiler start -object-store-name <name> -node <name>`

. Ergebnisse anzeigen:
+
`storage aggregate object-store profiler show`
