= ONTAP S3-Überwachungskonfigurationen anzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach Abschluss der Revisionsprotokollkonfiguration kann überprüft werden, ob das Revisionsprotokoll ordnungsgemäß konfiguriert und aktiviert ist. Es lassen sich außerdem Informationen zu allen Objektspeicher-Revisionsprotokollkonfigurationen im Cluster anzeigen.

.Über diese Aufgabe
Informationen zu Bucket- und SVM-Audit-Konfigurationen lassen sich anzeigen.

* Buckets: Der  `vserver object-store-server audit event-selector show`Befehl kann verwendet werden
+
Ohne Angabe von Parametern zeigt der Befehl die folgenden Informationen über Buckets in allen SVMs im Cluster mit Objektspeicher-Audit-Konfigurationen an:

+
** SVM-Name
** Bucket Name
** Zugriffs- und Berechtigungswerte


* SVMs: Mit dem  `vserver object-store-server audit show` Befehl
+
Ohne Angabe von Parametern zeigt der Befehl die folgenden Informationen zu allen SVMs im Cluster mit Objektspeicher-Revisionsprotokollkonfigurationen an:

+
** SVM-Name
** Revisionsstatus
** Zielverzeichnis




Sie können den  `-fields` Parameter angeben, um festzulegen, welche Revisionsprotokoll-Konfigurationsinformationen angezeigt werden.

.Schritte
Informationen zu S3-Audit-Konfigurationen:

[cols="2,4"]
|===
| Wenn Sie die Konfiguration für ... ändern möchten | Eingeben... 


| Buckets | `vserver object-store-server audit event-selector show [-vserver _svm_name_] [_parameters_]` 


| SVMs  a| 
`vserver object-store-server audit show [-vserver _svm_name_] [_parameters_]`

|===
.Beispiele
Das folgende Beispiel zeigt Informationen für einen einzelnen Bucket:

[listing]
----
cluster1::> vserver object-store-server audit event-selector show -vserver vs1 -bucket test-bucket
     Vserver     Bucket     Access          Permission
     ----------- ---------- --------------- ----------
     vs1         bucket1    read-only       allow-only
----
Das folgende Beispiel zeigt Informationen für alle Buckets auf einer SVM:

[listing]
----
cluster1::> vserver object-store-server audit event-selector show -vserver vs1

     Vserver           :vs1
     Bucket            :test-bucket
     Access            :all
     Permission        :all
----
Das folgende Beispiel zeigt den Namen, den Revisionsprotokollstatus, die Ereignistypen, das Protokollformat und das Zielverzeichnis für alle SVMs.

[listing]
----
cluster1::> vserver object-store-server audit show

Vserver     State  Event Types Log Format Target Directory
----------- ------ ----------- ---------- --------------------
vs1         false  data        json       /audit_log
----
Das folgende Beispiel zeigt die SVM-Namen und Details zum Revisionsprotokoll für alle SVMs.

[listing]
----
cluster1::> vserver object-store-server audit show -log-save-details

                  Rotation                           Rotation
      Vserver     File Size Rotation Schedule        Limit
      ----------- --------- ------------------------ --------
      vs1         100MB     -                        0
----
Das folgende Beispiel zeigt in Listenform alle Revisionsprotokoll-Konfigurationsinformationen zu allen SVMs.

[listing]
----
cluster1::> vserver object-store-server audit show -instance

                             Vserver: vs1
                      Auditing state: true
                Log Destination Path: /audit_log
       Categories of Events to Audit: data
                          Log Format: json
                 Log File Size Limit: 100MB
        Log Rotation Schedule: Month: -
  Log Rotation Schedule: Day of Week: -
          Log Rotation Schedule: Day: -
         Log Rotation Schedule: Hour: -
       Log Rotation Schedule: Minute: -
                  Rotation Schedules: -
            Log Files Rotation Limit: 0
                  Log Retention Time: 0s
----
