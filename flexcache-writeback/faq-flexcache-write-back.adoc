= Häufig gestellte Fragen zu ONTAP FlexCache Write-Back
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Diese FAQ kann hilfreich sein, wenn eine schnelle Antwort auf eine Frage gesucht wird.

.Ich möchte Write-Back verwenden. Welche Version von ONTAP muss ausgeführt werden?
Sowohl der Cache als auch der Ursprung müssen ONTAP 9.15.1 oder höher ausführen. Es wird *_dringend_* empfohlen, die neueste P-Release zu verwenden. Die Entwicklung verbessert kontinuierlich die Leistung und Funktionalität von Write-Back-fähigen Caches.

.Können Clients, die auf den Ursprungsserver zugreifen, Auswirkungen auf Clients haben, die auf den Write-Back-fähigen Cache zugreifen?
Ja. Der Ursprungsserver hat das gleiche Recht auf die Daten wie jeder Cache. Wenn eine Operation an einer Datei ausgeführt wird, die das Entfernen der Datei aus dem Cache oder das Aufheben einer Sperre/Datendelegierung erfordert, kann der Client am Cache beim Zugriff auf die Datei eine Verzögerung feststellen.

.Kann QoS auf Write-Back-fähige FlexCaches angewendet werden?
Ja. Jedem Cache und dem Ursprung können unabhängige QoS-Richtlinien zugewiesen werden. Dies hat keine direkten Auswirkungen auf durch Schreibvorgänge im Write-Back-Modus initiierten Intercluster-Verkehr. Indirekt kann der Intercluster-Schreibverkehr im Write-Back-Modus durch eine QoS-Beschränkung des Frontend-Verkehrs am Cache mit aktiviertem Write-Back verlangsamt werden.

.Wird Multi-Protokoll-NAS auf Write-Back-aktivierten FlexCaches unterstützt?
Ja. Multiprotokoll wird bei Write-Back-aktivierten FlexCaches vollständig unterstützt. Derzeit werden NFSv4.2 und S3 von FlexCache im Write-Around- oder Write-Back-Betriebsmodus nicht unterstützt.

.Werden SMB-Alternate Data Streams bei Write-Back-aktivierten FlexCaches unterstützt?
SMB-Alternative Datenströme (ADS) werden unterstützt, jedoch nicht durch Write-Back beschleunigt. Der Schreibvorgang in den ADS wird an den Ursprung weitergeleitet, was die Latenz des WAN zur Folge hat. Der Schreibvorgang entfernt außerdem die Hauptdatei, zu der der ADS gehört, aus dem Cache.

.Kann ein Cache nach seiner Erstellung zwischen Write-Around- und Write-Back-Modus wechseln?
Ja. Es genügt, das  `is-writeback-enabled` Flag im link:../flexcache-writeback/flexcache-writeback-enable-task.html [`flexcache modify` Befehl umzuschalten.

.Gibt es Bandbreitenüberlegungen, die für die Intercluster-Verbindung zwischen dem/den Cache(s) und dem Origin zu berücksichtigen sind?
Ja. FlexCache Write-Back ist stark abhängig von der Intercluster-Verbindung zwischen dem/den Cache(s) und dem Ursprung. Geringe Bandbreite und/oder verlustbehaftete Netzwerke können die Performance erheblich negativ beeinflussen. Es gibt keine spezifische Bandbreitenanforderung, da diese stark von Ihrer Workload abhängt.
