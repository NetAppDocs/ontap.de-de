= ONTAP FlexCache Write-Back aktivieren und verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.15.1 kann der FlexCache Write-Back-Modus auf FlexCache Volumes aktiviert werden, um eine bessere Performance für Edge-Computing-Umgebungen und Caches mit schreibintensiven Workloads bereitzustellen. Es kann außerdem festgestellt werden, ob Write-Back auf einem FlexCache Volume aktiviert ist oder Write-Back bei Bedarf auf dem Volume deaktiviert werden kann.

Wenn Write-Back auf dem Cache-Volume aktiviert ist, werden Schreibanforderungen an den lokalen Cache statt an das Ursprungsvolume gesendet.

.Bevor Sie beginnen
Sie müssen sich im erweiterten Berechtigungsmodus befinden.



[[create-a-new-flexcache-volume-with-write-back-enabled]]
== Ein neues FlexCache Volume mit aktivierter Rückschreibfunktion erstellen

.Schritte
Ein neues FlexCache Volume mit aktiviertem Write-Back kann mithilfe von ONTAP System Manager oder der ONTAP CLI erstellt werden.

[role="tabbed-block"]
====
.System Manager
--
. Wenn sich das FlexCache Volume in einem anderen Cluster als das Ursprungsvolume befindet, eine Cluster Peer-Beziehung erstellen:
+
.. Im lokalen Cluster auf *Schutz > Übersicht* klicken.
.. *Intercluster Settings* erweitern, auf *Add Network Interfaces* klicken und dem Cluster Intercluster-Schnittstellen hinzufügen.
+
Dies wird auf dem Remote-Cluster wiederholt.

.. Im Remote-Cluster auf *Schutz > Übersicht* klicken. Im Abschnitt Cluster Peer auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und *Passphrase generieren* auswählen.
.. Die generierte Passphrase wird kopiert und im lokalen Cluster eingefügt.
.. Im lokalen Cluster unter Cluster Peers auf *Peer Clusters* klicken und das lokale und das Remote-Cluster peeren.


. Wenn sich das FlexCache Volume in einem anderen Cluster als das Ursprungsvolume befindet, ist eine SVM Peer-Beziehung zu erstellen:
+
Unter *Storage VM Peers* image:icon_kabob.gif["Symbol für Menüoptionen"]und dann auf *Peer Storage VMs* klicken, um die Storage VMs zu peeren.

+
Wenn sich das FlexCache Volume im selben Cluster befindet, kann mit dem System Manager keine SVM Peer-Beziehung erstellt werden.

. *Speicher > Volumes* auswählen.
. Wählen Sie *Add*.
. *Weitere Optionen* auswählen und dann *Als Cache für ein Remote-Volume hinzufügen* wählen.
. *FlexCache Write-Back* aktivieren auswählen.


--
.CLI
--
. Wenn sich das zu erstellende FlexCache Volume in einem anderen Cluster befindet, ist eine Cluster Peer-Beziehung zu erstellen:
+
.. Im Ziel-Cluster eine Peer-Beziehung mit dem Quell-Cluster für den Datenschutz erstellen:
+
[source, cli]
----
cluster peer create -generate-passphrase -offer-expiration MM/DD/YYYY HH:MM:SS|1...7days|1...168hours -peer-addrs <peer_LIF_IPs> -initial-allowed-vserver-peers <svm_name>,..|* -ipspace <ipspace_name>
----
+
Ab ONTAP 9.6 ist die TLS-Verschlüsselung beim Erstellen einer Cluster Peer-Beziehung standardmäßig aktiviert. Die TLS-Verschlüsselung wird für die Intercluster-Verbindung zwischen dem Ursprung und FlexCache Volumes unterstützt. Die TLS-Verschlüsselung kann bei Bedarf für die Cluster Peer-Beziehung auch deaktiviert werden.

+
[listing]
----
cluster02::> cluster peer create -generate-passphrase -offer-expiration 2days -initial-allowed-vserver-peers *

                     Passphrase: UCa+6lRVICXeL/gq1WrK7ShR
                Expiration Time: 6/7/2017 08:16:10 EST
  Initial Allowed Vserver Peers: *
            Intercluster LIF IP: 192.140.112.101
              Peer Cluster Name: Clus_7ShR (temporary generated)

Warning: make a note of the passphrase - it cannot be displayed again.
----
.. Auf dem Quell-Cluster wird der Quell-Cluster gegenüber dem Ziel-Cluster authentifiziert:
+
[source, cli]
----
cluster peer create -peer-addrs <peer_LIF_IPs> -ipspace <ipspace>
----
+
[listing]
----
cluster01::> cluster peer create -peer-addrs 192.140.112.101,192.140.112.102

Notice: Use a generated passphrase or choose a passphrase of 8 or more characters.
        To ensure the authenticity of the peering relationship, use a phrase or sequence of characters that would be hard to guess.

Enter the passphrase:
Confirm the passphrase:

Clusters cluster02 and cluster01 are peered.
----


. Befindet sich das FlexCache Volume in einer anderen SVM als das Ursprungsvolume, wird eine SVM Peer-Beziehung mit  `flexcache` als Anwendung erstellt:
+
.. Befindet sich die SVM in einem anderen Cluster, ist eine SVM-Berechtigung für die Peering-SVMs zu erstellen:
+
[source, cli]
----
vserver peer permission create -peer-cluster <cluster_name> -vserver <svm-name> -applications flexcache
----
+
Das folgende Beispiel veranschaulicht, wie eine SVM Peer-Berechtigung erstellt wird, die für alle lokalen SVMs gilt:

+
[listing]
----
cluster1::> vserver peer permission create -peer-cluster cluster2 -vserver "*" -applications flexcache

Warning: This Vserver peer permission applies to all local Vservers. After that no explict
"vserver peer accept" command required for Vserver peer relationship creation request
from peer cluster "cluster2" with any of the local Vservers. Do you want to continue? {y|n}: y
----
.. Erstellen der SVM Peer-Beziehung:
+
[source, cli]
----
vserver peer create -vserver <local_SVM> -peer-vserver <remote_SVM> -peer-cluster <cluster_name> -applications flexcache
----


. Ein FlexCache Volume mit aktiviertem Write-Back erstellen:
+
[source, cli]
----
volume flexcache create -vserver <cache_vserver_name> -volume <cache_flexgroup_name> -aggr-list <list_of_aggregates> -origin-volume <origin flexgroup> -origin-vserver <origin_vserver name> -junction-path <junction_path> -is-writeback-enabled true
----


--

[[enable-flexcache-write-back-on-an-existing-flexcache-volume]]
== Aktivierung des FlexCache Write-Back auf einem vorhandenen FlexCache Volume

Die FlexCache Write-Back-Funktion kann auf einem vorhandenen FlexCache Volume mit ONTAP System Manager oder der ONTAP CLI aktiviert werden.

[role="tabbed-block"]
====
.System Manager
--
. *Speicher > Volumes* auswählen und ein vorhandenes FlexCache volume auswählen.
. Auf der Übersichtsseite des Volumes in der oberen rechten Ecke auf *Bearbeiten* klicken.
. Im Fenster *Volume bearbeiten* die Option *FlexCache Write-Back aktivieren* auswählen.


--
.CLI
--
. Write-Back auf einem vorhandenen FlexCache Volume aktivieren:
+
[source, cli]
----
volume flexcache config modify -volume <cache_flexgroup_name> -is-writeback-enabled true
----


--
====


[[check-if-flexcache-write-back-is-enabled]]
== Es wird geprüft, ob FlexCache Write-Back aktiviert ist.

.Schritte
Mit dem System Manager oder der ONTAP CLI lässt sich feststellen, ob das FlexCache Write-Back aktiviert ist.

[role="tabbed-block"]
====
.System Manager
--
. *Speicher > Volumes* auswählen und ein Volume auswählen.
. In der Volume-*Übersicht* die *FlexCache details* suchen und prüfen, ob FlexCache Write-Back beim FlexCache Volume auf *Enabled* gesetzt ist.


--
.CLI
--
. Es wird geprüft, ob FlexCache Write-Back aktiviert ist:
+
[source, cli]
----
volume flexcache config show -volume <cache_flexgroup_name> -fields is-writeback-enabled
----


--
====


[[disable-write-back-on-a-flexcache-volume]]
== Write-Back auf einem FlexCache Volume deaktivieren

Bevor Sie ein FlexCache Volume löschen können, muss das FlexCache Write-Back deaktiviert werden.

.Schritte
Mit dem System Manager oder der ONTAP CLI lässt sich das FlexCache Write-Back deaktivieren.

[role="tabbed-block"]
====
.System Manager
--
. *Speicher > Volumes* auswählen und ein vorhandenes FlexCache volume auswählen, bei dem FlexCache Write-Back aktiviert ist.
. Auf der Übersichtsseite des Volumes in der oberen rechten Ecke auf *Bearbeiten* klicken.
. Im Fenster *Volume bearbeiten* die Auswahl von *FlexCache Write-Back aktivieren* aufheben.


--
.CLI
--
. Write-Back deaktivieren:
+
[source, cli]
----
volume flexcache config modify -volume <cache_vol_name> -is-writeback-enabled false
----


--
====
