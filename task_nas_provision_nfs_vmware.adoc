= NFS-Speicher für VMware-Datenspeicher mit ONTAP System Manager bereitstellen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Vor der Verwendung von Virtual Storage Console for VMware vSphere (VSC) zur Bereitstellung von NFS-Volumes auf einem ONTAP basierten Speichersystem für ESXi Hosts sollte NFS mit System Manager for ONTAP 9.7 oder höher aktiviert werden.

Nachdem im System Manager ein link:task_nas_enable_linux_nfs.html["NFS-fähige Storage-VM"] erstellt wurde, erfolgt die Bereitstellung von NFS-Volumes und die Verwaltung von Datenspeichern mithilfe von VSC.

Ab VSC 7.0 ist VSC Teil des https://docs.netapp.com/us-en/ontap-tools-vmware-vsphere/index.html["ONTAP Tools für die VMware vSphere Virtual Appliance"^], das VSC, vStorage APIs for Storage Awareness (VASA) Provider und Storage Replication Adapter (SRA) für VMware vSphere Funktionen umfasst.

Überprüfung der  https://imt.netapp.com/matrix/["NetApp Interoperabilitätsmatrix"^]bestätigt die Kompatibilität zwischen den aktuellen ONTAP und VSC-Versionen.

Informationen zum Einrichten des NFS-Zugriffs für ESXi-Hosts auf Datenspeicher mithilfe von System Manager Classic (für ONTAP 9.7 und frühere Versionen) sind unter https://docs.netapp.com/us-en/ontap-system-manager-classic/nfs-config-esxi/index.html["Überblick über die NFS-Konfiguration für ESXi mit VSC"^]

Weitere Informationen finden Sie unter  https://docs.netapp.com/us-en/netapp-solutions/virtualization/vsphere_ontap_ontap_for_vsphere.html["TR-4597: VMware vSphere für ONTAP"^] und in der Dokumentation zu Ihrer VSC-Version.
