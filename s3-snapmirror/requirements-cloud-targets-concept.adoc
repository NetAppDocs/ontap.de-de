= Anforderungen für ONTAP SnapMirror S3 Cloud-Zielbeziehungen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Stellen Sie sicher, dass Ihre Quell- und Zielumgebungen die Anforderungen für SnapMirror S3-Backup-Schutz in Cloud-Zielen erfüllen.

Sie benötigen gültige Zugangsdaten beim Objektspeicheranbieter, um auf den Daten-Bucket zugreifen zu können.

Intercluster LIFs und ein IPspace sollten auf dem Cluster konfiguriert werden, bevor der Cluster eine Verbindung zu einem Cloud Objektspeicher herstellen kann. Intercluster LIFs sollten auf jedem Knoten erstellt werden, um Daten nahtlos vom lokalen Speicher in den Cloud Objektspeicher zu übertragen.

Für StorageGRID Ziele sind folgende Informationen erforderlich:

* Servername, angegeben als vollqualifizierter Domänenname (FQDN) oder IP-Adresse
* Bucket Name; der Bucket muss bereits existieren
* Zugangsschlüssel
* geheimer Schlüssel


Zusätzlich muss das CA-Zertifikat, mit dem das StorageGRID Serverzertifikat signiert wurde, auf der Admin-Storage-VM des ONTAP S3-Clusters mithilfe des  `security certificate install` Befehls installiert werden. Weitere Informationen finden sich unter link:../fabricpool/install-ca-certificate-storagegrid-task.html["Installation eines CA-Zertifikats"], wenn StorageGRID verwendet wird.

Für AWS S3-Ziele benötigen Sie die folgenden Informationen:

* Servername, angegeben als vollqualifizierter Domänenname (FQDN) oder IP-Adresse
* Bucket Name; der Bucket muss bereits existieren
* Zugangsschlüssel
* geheimer Schlüssel


Der DNS-Server für die Admin-Speicher-VM des ONTAP-Clusters muss in der Lage sein, FQDNs (falls verwendet) in IP-Adressen aufzulösen.
