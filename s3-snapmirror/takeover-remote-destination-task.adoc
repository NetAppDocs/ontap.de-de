= Übernahme vom Ziel ONTAP S3-Bucket auf dem Remote-Cluster
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn die Daten in einem Quell-Bucket nicht mehr verfügbar sind, kann die SnapMirror Beziehung unterbrochen werden, sodass der Ziel-Bucket beschreibbar ist und Daten bereitgestellt werden können.

.Über diese Aufgabe
Bei einer Übernahmeoperation wird der Quell-Bucket in einen schreibgeschützten Bucket umgewandelt und der ursprüngliche Ziel-Bucket in einen Lese-/Schreib-Bucket, wodurch die SnapMirror S3-Beziehung umgekehrt wird.

Wenn der deaktivierte Quell-Bucket wieder verfügbar ist, resynchronisiert SnapMirror S3 die Inhalte der beiden Buckets automatisch. Eine explizite Resynchronisierung der Beziehung ist nicht erforderlich, wie sie bei Volume SnapMirror Deployments notwendig ist.

Die Übernahme muss vom Remote-Cluster aus initiiert werden.

Obwohl SnapMirror S3 Objekte vom Quell-Bucket in einen Ziel-Bucket repliziert, werden Benutzer, Gruppen und Richtlinien nicht vom Quell-Objektspeicher in den Ziel-Objektspeicher repliziert.

Benutzer, Gruppenrichtlinien, Berechtigungen und ähnliche Komponenten müssen im Ziel-Objektspeicher konfiguriert werden, damit Clients während eines Failover-Ereignisses auf den Ziel-Bucket zugreifen können.

Quell- und Zielbenutzer können dieselben Zugriffs- und geheimen Schlüssel verwenden, sofern die Quellschlüssel bei der Erstellung des Benutzers im Ziel-Cluster manuell angegeben werden. Beispiel:

[listing]
----
vserver object-store-server user create -vserver svm1 -user user1 -access-key "20-characters" -secret-key "40-characters"
----
[role="tabbed-block"]
====
.System Manager
--
Failover vom nicht verfügbaren Bucket und Beginn der Datenbereitstellung:

. *Schutz > Beziehungen* auswählen und dann *SnapMirror S3* wählen.
. Klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"], wählen Sie *Failover* und dann *Failover*.


--
.CLI
--
. Einen Failover-Vorgang für den Ziel-Bucket initiieren:
`snapmirror failover start -destination-path _svm_name_:/bucket/_bucket_name_`
. Der Status des Failover-Vorgangs kann überprüft werden:
`snapmirror show -fields status`
+
.Beispiel
[listing]
----
dest_cluster::> snapmirror failover start -destination-path dest_svm1:/bucket/test-bucket-mirror
----
