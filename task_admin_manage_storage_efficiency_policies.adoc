= Speichereffizienzrichtlinien mit ONTAP System Manager verwalten
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Ab ONTAP 9.8 ist es mit dem System Manager möglich, Effizienzrichtlinien für Storage-VMs auf FAS Systemen zu aktivieren, zu deaktivieren, hinzuzufügen, zu bearbeiten oder zu löschen.


NOTE: Diese Funktion ist auf AFF Systemen nicht verfügbar.

.Schritte
. *Storage > Storage-VMs* auswählen
. Die Storage-VM auswählen, für die Effizienzrichtlinien verwaltet werden sollen.
. Auf der Registerkarte *Einstellungen* image:icon_arrow.gif["Pfeilsymbol"] im Abschnitt *Effizienzrichtlinie* wird die entsprechende Option ausgewählt. Die Effizienzrichtlinien für diese Storage-VM werden angezeigt.


Folgende Aufgaben sind möglich:

* Eine Effizienzrichtlinie kann durch Klicken auf den Umschalter in der Spalte „Status“ aktiviert oder deaktiviert werden.
* Eine Effizienzrichtlinie kann durch Klicken auf *Add+* hinzugefügt werden.
* Eine Effizienzrichtlinie kann bearbeitet werden, indem auf image:icon_kabob.gif["Symbol für Menüoptionen"] rechts neben dem Namen der Richtlinie *Bearbeiten* ausgewählt wird.
* Eine Effizienzrichtlinie kann gelöscht werden, indem auf image:icon_kabob.gif["Symbol für Menüoptionen"] rechts neben dem Richtliniennamen *Löschen* ausgewählt wird.




[[efficiency-policies-list]]
== Liste der Effizienzrichtlinien

* *Auto*
+
Legt fest, dass die Deduplizierung kontinuierlich im Hintergrund durchgeführt wird. Diese Richtlinie wird für alle neu erstellten Volumes und für alle aktualisierten Volumes festgelegt, die nicht manuell für die Hintergrunddeduplizierung konfiguriert wurden. Wenn die Richtlinie auf „`default`“ oder eine andere Richtlinie geändert wird, wird die „`auto`“-Richtlinie deaktiviert.

+
Wenn ein Volume von einem Nicht-AFF-System auf ein AFF-System verschoben wird, ist die „auto“-Richtlinie auf dem Zielknoten standardmäßig aktiviert. Wenn ein Volume von einem AFF-Knoten auf einen Nicht-AFF-Knoten verschoben wird, wird die „auto“-Richtlinie auf dem Zielknoten standardmäßig durch die „inline-only“-Richtlinie ersetzt.

* *Richtlinie*
+
Gibt den Namen einer Effizienzrichtlinie an.

* *Status*
+
Gibt den Status einer Effizienzrichtlinie an. Der Status kann einer der folgenden sein:

+
** Aktiviert
+
Legt fest, dass die Effizienzrichtlinie einer Deduplizierungsoperation zugewiesen werden kann.

** Deaktiviert
+
Gibt an, dass die Effizienzrichtlinie deaktiviert ist. Die Richtlinie kann über das Status-Drop-down-Menü aktiviert und später einem Deduplizierungsvorgang zugewiesen werden.



* *Ausgeführt von*
+
Legt fest, ob die Effizienzrichtlinie basierend auf einem Zeitplan oder auf einem Schwellenwert (Änderungsprotokollschwellenwert) ausgeführt wird.

* *QoS Richtlinie*
+
Legt den QoS-Typ für die Effizienzrichtlinie fest. Der QoS-Typ kann einer der folgenden sein:

+
** Hintergrund
+
Gibt an, dass die QoS-Richtlinie im Hintergrund ausgeführt wird, was potenzielle Leistungseinbußen bei Clientvorgängen reduziert.

** Best Effort
+
Gibt an, dass die QoS-Richtlinie auf Best-Effort-Basis ausgeführt wird, was eine maximale Auslastung der Systemressourcen ermöglicht.



* *Maximale Laufzeit*
+
Legt die maximale Laufzeitdauer einer Effizienzrichtlinie fest. Wenn dieser Wert nicht angegeben wird, wird die Effizienzrichtlinie bis zum Abschluss des Vorgangs ausgeführt.





[[details-area]]
== Detailbereich

Der Bereich unterhalb der Effizienzrichtlinienliste zeigt zusätzliche Informationen zur ausgewählten Effizienzrichtlinie an, darunter den Namen des Zeitplans und die Zeitplandetails für eine zeitplanbasierte Richtlinie sowie den Schwellenwert für eine schwellenwertbasierte Richtlinie.
