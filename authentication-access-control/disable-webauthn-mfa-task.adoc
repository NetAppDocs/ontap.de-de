= Deaktivieren der WebAuthn MFA für ONTAP System Manager Benutzer
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Als ONTAP Administrator kann die WebAuthn MFA für einen Benutzer oder eine Gruppe deaktiviert werden, indem der Benutzer oder die Gruppe mit System Manager oder der ONTAP CLI bearbeitet wird.



[[disable-webauthn-mfa-for-an-existing-user-or-group]]
== Deaktivieren Sie WebAuthn MFA für einen bestehenden Benutzer oder eine bestehende Gruppe

Sie können die WebAuthn MFA für einen bestehenden Benutzer oder eine Gruppe jederzeit deaktivieren.


NOTE: Wenn registrierte Anmeldeinformationen deaktiviert werden, bleiben diese erhalten. Wenn die Anmeldeinformationen später wieder aktiviert werden, werden dieselben Anmeldeinformationen verwendet, sodass der Benutzer sich beim Anmelden nicht erneut registrieren muss.

[role="tabbed-block"]
====
.System Manager
--
. *Cluster > Einstellungen* auswählen.
. Das Pfeilsymbol neben *Benutzer und Rollen* auswählen.
. In der Liste der Benutzer und Gruppen den Benutzer oder die Gruppe auswählen, die bearbeitet werden soll.
. In der Spalte *MFA für HTTP* für diesen Benutzer *Deaktiviert* auswählen.
. *Speichern* auswählen.


--
.CLI
--
. Einen bestehenden Benutzer oder eine Gruppe ändern, um die WebAuthn MFA für diesen Benutzer oder diese Gruppe zu deaktivieren.
+
Im folgenden Beispiel ist WebAuthn MFA deaktiviert, indem „none“ als zweites Authentifizierungsverfahren ausgewählt wird.

+
[source, console]
----
security login modify -user-or-group-name <user_or_group_name> \
                     -authentication-method domain \
                     -second-authentication-method none \
                     -application http \
                     -role admin
----
+
Weitere Informationen zu  `security login modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-modify.html["ONTAP-Befehlsreferenz"^].
