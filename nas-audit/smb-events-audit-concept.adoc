= Informationen zu SMB-Ereignissen, die ONTAP prüfen kann, um die Ergebnisse zu interpretieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP kann bestimmte SMB-Ereignisse protokollieren, darunter bestimmte Datei- und Ordnerzugriffsereignisse, bestimmte An- und Abmeldevorgänge sowie Ereignisse der zentralen Zugriffsrichtlinienbereitstellung. Zu wissen, welche Zugriffsereignisse protokolliert werden können, ist hilfreich bei der Interpretation der Ergebnisse aus den Ereignisprotokollen.

[[additional-information-about-event-4656]]
== Weitere Informationen zu Ereignis 4656

Das  `HandleID` Tag im Audit  `XML` Ereignis enthält den Handle des Objekts (Datei oder Verzeichnis), auf das zugegriffen wurde. Das  `HandleID` Tag für das EVTX 4656 Ereignis enthält unterschiedliche Informationen, abhängig davon, ob das Öffnungsereignis das Erstellen eines neuen Objekts oder das Öffnen eines vorhandenen Objekts betrifft:

* Wenn das Öffnungsereignis eine Öffnungsanforderung zum Erstellen eines neuen Objekts (Datei oder Verzeichnis) ist, zeigt das  `HandleID` Tag im Audit-XML-Ereignis ein leeres  `HandleID` an (zum Beispiel:  `<Data Name="HandleID">00000000000000;00;00000000;00000000</Data>`).
+
Das  `HandleID` ist leer, da die OPEN-Anfrage (zum Erstellen eines neuen Objekts) protokolliert wird, bevor das Objekt tatsächlich erstellt wird und bevor ein Handle existiert. Nachfolgende protokollierte Ereignisse für dasselbe Objekt enthalten das korrekte Objekthandle im  `HandleID` Tag.

* Wenn es sich bei dem Öffnungsereignis um eine Anfrage zum Öffnen eines bestehenden Objekts handelt, enthält das Audit-Ereignis den zugewiesenen Handle dieses Objekts im  `HandleID`-Tag (zum Beispiel:  `<Data Name="HandleID">00000000000401;00;000000ea;00123ed4</Data>`).
