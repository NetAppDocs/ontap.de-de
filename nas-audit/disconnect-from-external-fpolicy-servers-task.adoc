= Verbindung zu externen FPolicy Servern in ONTAP trennen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Möglicherweise muss die Verbindung zu einem externen FPolicy Server manuell getrennt werden. Dies kann sinnvoll sein, wenn der FPolicy Server Probleme bei der Verarbeitung von Benachrichtigungsanfragen hat oder wenn Wartungsarbeiten am FPolicy Server erforderlich sind.

.Schritte
. Die Verbindung zum externen FPolicy Server wird mit dem  `vserver fpolicy engine-disconnect` Befehl getrennt.
+
Weitere Informationen zu  `vserver fpolicy engine-disconnect` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-fpolicy-engine-disconnect.html["ONTAP-Befehlsreferenz"^].

. Es kann mit dem  `vserver fpolicy show-engine` Befehl überprüft werden, ob die Verbindung zum externen FPolicy Server getrennt ist.
+
Weitere Informationen zu  `vserver fpolicy show-engine` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-fpolicy-show-engine.html["ONTAP-Befehlsreferenz"^].
