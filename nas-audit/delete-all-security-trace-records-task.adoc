= Alle Sicherheitsverfolgungsdatensätze auf ONTAP SVMs löschen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn keine der vorhandenen Security-Trace-Datensätze beibehalten werden soll, ist das Löschen aller Datensätze auf einem Knoten mit einem einzigen Befehl möglich.

.Schritt
. Alle Sicherheitsverfolgungsdatensätze löschen:
+
`vserver security trace trace-result delete -node node_name -vserver vserver_name *`

+
** `-node` `node_name` ist der Name des Cluster-Knotens, auf dem das persistente Tracing-Ereignis aufgetreten ist, das gelöscht werden soll.
** `-vserver` `vserver_name` ist der Name der Storage Virtual Machine (SVM), auf der das persistente Tracing-Ereignis aufgetreten ist, das gelöscht werden soll.
