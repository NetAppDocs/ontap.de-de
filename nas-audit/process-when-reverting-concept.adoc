= Die Auswirkungen der Rücksetzung eines geprüften ONTAP Clusters
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie planen, den Cluster zurückzusetzen, sollten Sie sich über den Rücksetzungsprozess informieren, den ONTAP befolgt, wenn im Cluster Storage Virtual Machines (SVMs) mit aktivierter Überwachung vorhanden sind. Bestimmte Maßnahmen sind vor der Rücksetzung erforderlich.



[[reverting-to-a-version-of-ontap-that-does-not-support-the-auditing-of-smb-logon-and-logoff-events-and-central-access-policy-staging-events]]
== Die Rückkehr zu einer Version von ONTAP, die die Überwachung von SMB-An- und Abmeldeereignissen sowie von Ereignissen der zentralen Zugriffsrichtlinienbereitstellung nicht unterstützt

Die Unterstützung für die Überwachung von SMB-An- und Abmeldeereignissen sowie von Ereignissen der zentralen Zugriffsrichtlinienbereitstellung beginnt mit clustered Data ONTAP 8.3. Wenn auf eine Version von ONTAP zurückgesetzt wird, die diese Ereignistypen nicht unterstützt, und Überwachungskonfigurationen vorhanden sind, die diese Ereignistypen überwachen, muss die Überwachungskonfiguration für die entsprechenden SVMs mit aktivierter Überwachung vor dem Zurücksetzen geändert werden. Die Konfiguration muss so angepasst werden, dass nur Dateivorgänge überwacht werden.
