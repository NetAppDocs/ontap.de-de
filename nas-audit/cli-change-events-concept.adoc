= Informationen zu ONTAP CLI-Änderungsereignissen, die protokolliert werden können
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP kann bestimmte CLI-Änderungsereignisse protokollieren, darunter bestimmte SMB-Freigabeereignisse, bestimmte Überwachungsrichtlinienereignisse, bestimmte lokale Sicherheitsgruppenereignisse, lokale Nutzergruppe-Ereignisse und Autorisierungsrichtlinienereignisse. Das Verständnis darüber, welche Änderungsereignisse protokolliert werden können, ist hilfreich bei der Interpretation der Ergebnisse aus den Ereignisprotokollen.

Sie können CLI-Änderungsereignisse der Storage Virtual Machine (SVM) verwalten, indem Sie die Audit-Protokolle manuell rotieren, die Auditierung aktivieren oder deaktivieren, Informationen zu Auditierungsänderungsereignissen anzeigen, Auditierungsänderungsereignisse ändern und Auditierungsänderungsereignisse löschen.

Wenn als Administrator ein Befehl ausgeführt wird, der die Konfiguration im Zusammenhang mit SMB-Share, lokaler Nutzergruppe, lokaler Sicherheitsgruppe, Authorization-Policy und Audit-Policy Ereignissen ändert, wird ein Datensatz generiert und das entsprechende Ereignis protokolliert:

|===


| Prüfungskategorie | Veranstaltungen | Ereignis-IDs | Führen Sie diesen Befehl aus...

 a|

 a| 
`vserver cifs share modify` `vserver cifs share create|modify|delete` `vserver cifs share add|remove`

[5144] Netzwerkfreigabe gelöscht
 a| 
`vserver cifs share delete`
