= Rollen der Clusterkomponenten in der ONTAP FPolicy Implementierung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Cluster, die darin enthaltenen Storage Virtual Machines (SVMs) und die Daten-LIFs spielen alle eine Rolle bei einer FPolicy Implementierung.

* *Cluster*
+
Der Cluster enthält das FPolicy Management Framework und pflegt und verwaltet Informationen über alle FPolicy Konfigurationen im Cluster.

* *SVM*
+
Eine FPolicy Konfiguration wird auf SVM-Ebene definiert. Der Geltungsbereich der Konfiguration ist die SVM, und sie wirkt sich ausschließlich auf SVM-Ressourcen aus. Eine SVM Konfiguration kann keine Dateizugriffsanfragen überwachen und Benachrichtigungen für Dateizugriffe versenden, die für Daten gestellt werden, die sich auf einer anderen SVM befinden.

+
FPolicy Konfigurationen können auf der Admin-SVM definiert werden. Nachdem die Konfigurationen auf der Admin-SVM definiert wurden, sind sie in allen SVMs sichtbar und verwendbar.

* *Daten-LIFs*
+
Die Verbindungen zu den FPolicy Servern werden über Daten-LIFs hergestellt, die zur SVM mit der FPolicy Konfiguration gehören. Die für diese Verbindungen verwendeten Daten-LIFs können auf die gleiche Weise übergeben werden wie Daten-LIFs, die für den normalen Clientzugriff verwendet werden.
