= Informationen zum Status der ONTAP FPolicy Richtlinie
:allow-uri-read: 


[role="lead"]
Sie können Informationen über den Status von FPolicy Richtlinien anzeigen, um festzustellen, ob eine Richtlinie aktiviert ist, welche externe Engine für deren Verwendung konfiguriert ist, welche Sequenznummer die Richtlinie hat und welcher Storage Virtual Machine (SVM) die FPolicy Richtlinie zugeordnet ist.

.Über diese Aufgabe
Wenn Sie keine Parameter angeben, zeigt der Befehl folgende Informationen an:

* SVM-Name
* Richtlinienname
* Policy-Sequenznummer
* Richtlinienstatus


Zusätzlich zur Anzeige von Informationen über den Richtlinienstatus für auf dem Cluster oder eine bestimmte SVM konfigurierte FPolicy Richtlinien können Befehlsparameter verwendet werden, um die Ausgabe des Befehls nach anderen Kriterien zu filtern.

Sie können den  `-instance` Parameter angeben, um detaillierte Informationen zu den aufgelisteten Richtlinien anzuzeigen. Alternativ kann der  `-fields` Parameter verwendet werden, um nur die angegebenen Felder in der Befehlsausgabe anzuzeigen, oder  `-fields ?` um zu bestimmen, welche Felder verwendet werden können.

.Schritt
. Gefilterte Informationen zum FPolicy Richtlinienstatus werden mit dem entsprechenden Befehl angezeigt:
+
[cols="35,65"]
|===


| Wenn Statusinformationen zu Richtlinien angezeigt werden sollen... | Geben Sie den Befehl ein... 


 a| 
Auf dem Cluster
 a| 
`vserver fpolicy show`



 a| 
Die den angegebenen Status haben
 a| 
`vserver fpolicy show -status {on|off}`



 a| 
Auf einer bestimmten SVM
 a| 
`vserver fpolicy show -vserver vserver_name`



 a| 
Mit dem angegebenen Richtliniennamen
 a| 
`vserver fpolicy show -policy-name policy_name`



 a| 
Die die angegebene externe Engine verwenden
 a| 
`vserver fpolicy show -engine engine_name`

|===


.Beispiel
Das folgende Beispiel zeigt die Informationen zu den FPolicy Richtlinien im Cluster an:

[listing]
----

cluster1::> vserver fpolicy show
                                        Sequence
Vserver             Policy Name           Number  Status      Engine
------------------- ------------------- --------  ---------   ---------
FPolicy             cserver_policy      -         off         eng1
vs1.example.com     v1p1                -         off         eng2
vs1.example.com     v1p2                -         off         native
vs1.example.com     v1p3                -         off         native
vs1.example.com     cserver_policy      -         off         eng1
vs2.example.com     v1p1                3         on          native
vs2.example.com     v1p2                1         on          eng3
vs2.example.com     cserver_policy      2         on          eng1
----
