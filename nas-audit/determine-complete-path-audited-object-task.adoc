= Den vollständigen Pfad zum ONTAP Revisionsobjekt bestimmen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der im  `<ObjectName>`-Tag eines Revisionsprotokolls ausgegebene Objektpfad enthält den Namen des Volumes (in Klammern) und den relativen Pfad vom Stammverzeichnis des enthaltenen Volumes. Um den vollständigen Pfad des geprüften Objekts einschließlich des Verbindungspfads zu bestimmen, sind bestimmte Schritte erforderlich.

.Schritte
. Der Datenträgername und der relative Pfad zum überwachten Objekt lassen sich anhand des  `<ObjectName>`-Tags im Überwachungsereignis bestimmen.
+
In diesem Beispiel lautet der Datenträgername "`data1`" und der relative Pfad zur Datei ist `/dir1/file.txt`:

+
`<Data Name="ObjectName">` `(data1);/dir1/file.txt` `</Data>`

. Anhand des im vorherigen Schritt ermittelten Volume-Namens lässt sich ermitteln, welcher Verbindungspfad für das Volume mit dem überwachten Objekt besteht:
+
In diesem Beispiel lautet der Volumenname "`data1`" und der Verbindungspfad für das Volumen, das das überwachte Objekt enthält, ist  `/data/data1`:

+
`volume show -junction -volume data1`

+
[listing]
----

                                Junction                    Junction
Vserver   Volume       Language Active    Junction Path     Path Source
--------- ------------ -------- --------  ----------------- -----------
vs1       data1        en_US.UTF-8
                                true      /data/data1       RW_volume
----
. Der vollständige Pfad zum überwachten Objekt ergibt sich, indem der im  `<ObjectName>`-Tag gefundene relative Pfad an den Verbindungspfad für das Volume angehängt wird.
+
In diesem Beispiel ist der Verbindungspfad für das Volumen:

+
`/data/data1/dir1/file.text`
