= Anforderung für ONTAP FPolicy Bereichskonfigurationen, wenn die FPolicy Richtlinie die native Engine verwendet
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn die FPolicy-Richtlinie so konfiguriert wird, dass sie die native Engine verwendet, besteht eine spezifische Anforderung an die Definition des für die Richtlinie konfigurierten FPolicy-Bereichs.

Der FPolicy-Bereich definiert die Grenzen, innerhalb derer die FPolicy-Richtlinie gilt, beispielsweise ob die FPolicy für bestimmte Volumes oder Freigaben gilt. Es gibt eine Reihe von Parametern, die den Anwendungsbereich, auf den die FPolicy-Richtlinie angewendet wird, weiter einschränken. Einer dieser Parameter,  `-is-file-extension-check-on-directories-enabled`, legt fest, ob Dateierweiterungen in Verzeichnissen geprüft werden. Der Standardwert ist  `false`, was bedeutet, dass Dateierweiterungen in Verzeichnissen nicht geprüft werden.

Wenn eine FPolicy-Richtlinie, die die native Engine verwendet, für eine Freigabe oder ein Volume aktiviert ist und der  `-is-file-extension-check-on-directories-enabled` Parameter auf  `false` für den Geltungsbereich der Richtlinie gesetzt ist, wird der Verzeichniszugriff verweigert. Mit dieser Konfiguration, da die Dateierweiterungen für Verzeichnisse nicht geprüft werden, wird jede Verzeichnisoperation verweigert, die in den Geltungsbereich der Richtlinie fällt.

Um sicherzustellen, dass der Verzeichniszugriff bei Verwendung der nativen Engine erfolgreich ist, muss  `-is-file-extension-check-on-directories-enabled parameter` auf  `true` gesetzt werden, wenn der Bereich erstellt wird.

Wenn dieser Parameter auf  `true` gesetzt ist, erfolgen Erweiterungsprüfungen bei Verzeichnisoperationen, und die Entscheidung, ob der Zugriff erlaubt oder verweigert wird, basiert auf den in der FPolicy Scope-Konfiguration ein- oder ausgeschlossenen Erweiterungen.
