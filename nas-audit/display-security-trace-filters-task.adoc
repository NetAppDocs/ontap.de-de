= Informationen zu Sicherheitsverfolgungsfiltern in ONTAP SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Zeigen Sie die Details des Security Trace Filters auf Ihrer Storage Virtual Machine (SVM) an, um die Zugriffsvorgänge zu identifizieren, die jeder Filter nachverfolgt.

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.6 können Sie den System Manager verwenden, um den Dateizugriff auf der SVM nachzuverfolgen.

.Schritte
. Öffnen Sie das System Manager dashboard.
. Wählen Sie unter *Cluster* *Storage VMs* aus.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] für die passende SVM aus.
. Wählen Sie *Trace file access*.
. Stellen Sie das Protokoll für den Dateizugriff auf *SMB/CIFS* oder *NFS* ein.
. Geben Sie den *User name* ein.
. Geben Sie die *Client IP address* ein.
. Optional können Sie eine oder mehrere der folgenden Optionen auswählen:
+
.. Wählen Sie *Trace particular path*, um die Datei oder den Dateipfad anzugeben, der verfolgt werden soll.
.. Wählen Sie *Nur Zugriff verweigerte Einträge anzeigen*, um die Ablaufverfolgung auf Zugriff verweigerte Ereignisse zu beschränken.


. Wählen Sie *Start tracing* aus.


--
.CLI
--
Verwenden Sie die CLI, um den Dateizugriff auf der SVM zu verfolgen.

.Schritt
. Informationen zu den Einträgen des Sicherheitsverfolgungsfilters lassen sich mit dem  `vserver security trace filter show` Befehl anzeigen.
+
Weitere Informationen zu  `vserver security trace filter show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-security-trace-filter-show.html["ONTAP-Befehlsreferenz"^].



.Beispiele
Der folgende Befehl zeigt Informationen zu allen Security Trace-Filtern auf SVM vs1 an:

[listing]
----
cluster1::> vserver security trace filter show -vserver vs1
Vserver  Index   Client-IP            Path            Trace-Allow  Windows-Name
-------- -----  -----------  ----------------------   -----------  -------------
vs1        1    -            /dir1/dir2/file.txt          yes      -
vs1        2    -            /dir3/dir4/                  no       mydomain\joe
----
