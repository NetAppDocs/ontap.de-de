= ONTAP Befehle zum Ändern von Audit-Konfigurationen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn eine Revisions-Einstellung geändert werden soll, kann die aktuelle Konfiguration jederzeit angepasst werden, einschließlich der Änderung des Protokollpfad-Ziels und des Protokollformats, der Kategorien der zu prüfenden Ereignisse, der automatischen Speicherung von Protokolldateien sowie der Angabe der maximalen Anzahl an zu speichernden Protokolldateien.

[cols=""30"]
|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Pfad des Protokollziels ändern
 a| 
`vserver audit modify` mit dem  `-destination` Parameter



 a| 
Die Kategorie der zu prüfenden Ereignisse anpassen
 a| 
`vserver audit modify` mit dem  `-events` Parameter


NOTE: Um Ereignisse der zentralen Zugriffsrichtlinienbereitstellung zu prüfen, muss die Dynamic Access Control (DAC) SMB-Serveroption auf der Storage Virtual Machine (SVM) aktiviert sein.



 a| 
Das Protokollformat anpassen
 a| 
`vserver audit modify` mit dem  `-format` Parameter



 a| 
Automatisches Speichern basierend auf der Größe der internen Protokolldatei aktivieren
 a| 
`vserver audit modify` mit dem  `-rotate-size` Parameter



 a| 
Automatisches Speichern basierend auf einem Zeitintervall aktivieren
 a| 
`vserver audit modify` mit den  `-rotate-schedule-month`, `-rotate-schedule-dayofweek`, `-rotate-schedule-day`, `-rotate-schedule-hour` und  `-rotate-schedule-minute` Parametern



 a| 
Festlegen der maximalen Anzahl gespeicherter Protokolldateien
 a| 
`vserver audit modify` mit dem  `-rotate-limit` Parameter

|===
