= Voraussetzungen für ONTAP Prüfung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie die Überwachung auf Ihrer Storage Virtual Machine (SVM) konfigurieren und aktivieren, sind bestimmte Anforderungen und Überlegungen zu berücksichtigen.

* Das kombinierte Limit für NFS- und S3-Audit-aktivierte SVMs hängt von Ihrer ONTAP-Version ab:
+
|===


| ONTAP Version | Maximal 


| 9.8 und früher | 50 


| 9.9.1 und später | 400 
|===
* Die Prüfung ist nicht an SMB- oder NFS-Lizenzierung gebunden.
+
Sie können die Überwachung auch dann konfigurieren und aktivieren, wenn keine SMB- und NFS-Lizenzen auf dem Cluster installiert sind.

* NFS Auditing unterstützt Security-ACEs (Typ U).
* Für die NFS-Überwachung gibt es keine Zuordnung zwischen Modusbits und Überwachungs-ACEs.
+
Bei der Umwandlung von ACLs in Modusbits werden Audit-ACEs übersprungen. Bei der Umwandlung von Modusbits in ACLs werden keine Audit-ACEs generiert.

* Das in der Überwachungskonfiguration angegebene Verzeichnis muss existieren.
+
Wenn die Konfiguration nicht existiert, schlägt der Befehl zum Erstellen der Auditing-Konfiguration fehl.

* Das in der Audit-Konfiguration angegebene Verzeichnis muss die folgenden Anforderungen erfüllen:
+
** Das Verzeichnis darf keine symbolischen Links enthalten.
+
Wenn das in der Auditing-Konfiguration angegebene Verzeichnis symbolische Links enthält, schlägt der Befehl zum Erstellen der Auditing-Konfiguration fehl.

** Das Verzeichnis ist mit einem absoluten Pfad anzugeben.
+
Es sollte kein relativer Pfad angegeben werden, zum Beispiel  `/vs1/../`.



* Auditing ist davon abhängig, dass in den Staging-Volumes verfügbarer Speicherplatz vorhanden ist.
+
Es ist wichtig, sich darüber im Klaren zu sein und einen Plan zu haben, um sicherzustellen, dass in Aggregaten, die geprüfte Volumes enthalten, ausreichend Platz für die Staging-Volumes vorhanden ist.

* Die Überwachung ist davon abhängig, dass im Volume, das das Verzeichnis enthält, in dem die konvertierten Ereignisprotokolle gespeichert werden, ausreichend Speicherplatz verfügbar ist.
+
Es ist erforderlich, sich darüber im Klaren zu sein und einen Plan zu haben, um sicherzustellen, dass in den für die Speicherung von Ereignisprotokollen verwendeten Volumes ausreichend Speicherplatz vorhanden ist. Die Anzahl der im Auditing-Verzeichnis aufzubewahrenden Ereignisprotokolle kann beim Erstellen einer Auditing-Konfiguration mit dem  `-rotate-limit` Parameter festgelegt werden, was dazu beiträgt, dass im Volume genügend Speicherplatz für die Ereignisprotokolle zur Verfügung steht.

* Obwohl die zentrale Bereitstellung von Zugriffsrichtlinien in der Überwachungskonfiguration aktiviert werden kann, ohne Dynamic Access Control auf dem SMB-Server zu aktivieren, muss Dynamic Access Control aktiviert sein, um zentrale Bereitstellungsereignisse für Zugriffsrichtlinien zu generieren.
+
Die dynamische Zugriffskontrolle ist nicht standardmäßig aktiviert.

== Überlegungen zum Aggregatspeicherplatz bei der Aktivierung der Überwachung

Wenn eine Überwachungskonfiguration erstellt wird und die Überwachung auf mindestens einer Storage Virtual Machine (SVM) im Cluster aktiviert ist, erstellt das Überwachungssubsystem Staging-Volumes auf allen vorhandenen Aggregaten und auf allen neuen Aggregaten, die erstellt werden. Bestimmte Überlegungen zum Aggregatspeicherplatz sind zu beachten, wenn die Überwachung im Cluster aktiviert wird.

Die Erstellung des Staging-Volumes kann aufgrund von nicht verfügbarem Speicherplatz in einem Aggregat fehlschlagen. Dies kann vorkommen, wenn eine Auditing-Konfiguration erstellt wird und die vorhandenen Aggregate nicht über genügend Speicherplatz für das Staging-Volume verfügen.

Es ist sicherzustellen, dass auf den vorhandenen Aggregaten genügend Speicherplatz für die Staging-Volumes vorhanden ist, bevor die Auditierung auf einer SVM aktiviert wird.
