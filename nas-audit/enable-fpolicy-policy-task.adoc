= ONTAP FPolicy Richtlinien aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem die Konfiguration einer FPolicy Richtlinie abgeschlossen ist, wird die FPolicy Richtlinie aktiviert. Durch das Aktivieren der Richtlinie werden deren Priorität festgelegt und die Überwachung des Dateizugriffs für die Richtlinie gestartet.

.Bevor Sie beginnen
Die FPolicy Richtlinie muss mit einer zugehörigen externen Engine vorhanden sein (falls die Richtlinie für die Verwendung externer FPolicy Server konfiguriert ist) und mindestens ein zugehöriges FPolicy Ereignis aufweisen. Der Gültigkeitsbereich der FPolicy Richtlinie muss vorhanden sein und der FPolicy Richtlinie zugewiesen werden.

.Über diese Aufgabe
Die Priorität wird verwendet, wenn mehrere Richtlinien auf der Storage Virtual Machine (SVM) aktiviert sind und mehr als eine Richtlinie dasselbe Dateizugriff-Ereignis abonniert hat. Richtlinien, die die native Engine-Konfiguration verwenden, haben eine höhere Priorität als Richtlinien für jede andere Engine, unabhängig von der ihnen bei der Aktivierung der Richtlinie zugewiesenen Sequenznummer.

[NOTE]
====
Auf der Admin-SVM kann keine Richtlinie aktiviert werden.

====
.Schritte
. Die FPolicy Richtlinie wird mit dem  `vserver fpolicy enable`Konfigurationsbefehl aktiviert.
+
`vserver fpolicy enable -vserver vs1.example.com -policy-name policy1 -sequence-number 1`

. Es kann mit dem  `vserver fpolicy show` Befehl überprüft werden, ob die FPolicy Richtlinie aktiviert ist.
+
`vserver fpolicy show -vserver vs1.example.com`

+
[listing]
----

                                   Sequence
Vserver          Policy Name         Number  Status   Engine
---------------  ----------------- --------  -------- ---------
vs1.example.com  policy1                  1  on       engine1
----
