= Informationen zu Verbindungen mit externen ONTAP FPolicy Servern anzeigen
:allow-uri-read: 


[role="lead"]
Sie können Statusinformationen zu Verbindungen zu externen FPolicy Servern (FPolicy Servern) für den Cluster oder eine bestimmte Storage Virtual Machine (SVM) anzeigen. Diese Informationen können dabei helfen zu bestimmen, welche FPolicy Server verbunden sind.

.Über diese Aufgabe
Wenn Sie keine Parameter angeben, zeigt der Befehl folgende Informationen an:

* SVM-Name
* Knotenname
* FPolicy Richtlinienname
* IP-Adresse des FPolicy Servers
* FPolicy Serverstatus
* FPolicy Servertyp


Zusätzlich zur Anzeige von Informationen über FPolicy-Verbindungen im Cluster oder einer bestimmten SVM lassen sich Befehlsparameter verwenden, um die Ausgabe des Befehls nach anderen Kriterien zu filtern.

Sie können den  `-instance` Parameter angeben, um detaillierte Informationen zu den aufgelisteten Richtlinien anzuzeigen. Alternativ kann der  `-fields` Parameter verwendet werden, um nur die angegebenen Felder in der Befehlsausgabe anzuzeigen. Sie können  `?` nach dem  `-fields` Parameter eingeben, um herauszufinden, welche Felder verwendet werden können.

.Schritt
. Gefilterte Informationen zum Verbindungsstatus zwischen dem Knoten und dem FPolicy Server werden mithilfe des entsprechenden Befehls angezeigt:
+
[cols="35,65"]
|===


| Wenn Verbindungsstatusinformationen zu FPolicy Servern angezeigt werden sollen... | Eingeben... 


 a| 
Die Sie angeben
 a| 
`vserver fpolicy show-engine -server IP_address`



 a| 
Für eine bestimmte SVM
 a| 
`vserver fpolicy show-engine -vserver vserver_name`



 a| 
Die mit einer bestimmten Richtlinie verknüpft sind
 a| 
`vserver fpolicy show-engine -policy-name policy_name`



 a| 
Mit dem von Ihnen angegebenen Serverstatus
 a| 
`vserver fpolicy show-engine -server-status status`

Der Serverstatus kann einer der folgenden sein:

** `connected`
** `disconnected`
** `connecting`
** `disconnecting`




 a| 
Mit dem angegebenen Typ
 a| 
`vserver fpolicy show-engine -server-type type`

Der FPolicy Servertyp kann einer der folgenden sein:

** `primary`
** `secondary`




 a| 
Die getrennt wurden mit dem angegebenen Grund
 a| 
`vserver fpolicy show-engine -disconnect-reason text`

Eine Verbindungsunterbrechung kann verschiedene Ursachen haben. Die folgenden Gründe für eine Verbindungsunterbrechung sind häufig:

** `Disconnect command received from CLI.`
** `Error encountered while parsing notification response from FPolicy server.`
** `FPolicy Handshake failed.`
** `SSL handshake failed.`
** `TCP Connection to FPolicy server failed.`
** `The screen response message received from the FPolicy server is not valid.`


|===


.Beispiel
Dieses Beispiel zeigt Informationen über externe Engine-Verbindungen zu FPolicy-Servern auf SVM vs1.example.com:

[listing]
----
cluster1::> vserver fpolicy show-engine -vserver vs1.example.com
FPolicy                                              Server-       Server-
Vserver         Policy    Node         Server        status        type
--------------- --------- ------------ ------------- ------------- ---------
vs1.example.com policy1   node1        10.1.1.2      connected     primary
vs1.example.com policy1   node1        10.1.1.3      disconnected  primary
vs1.example.com policy1   node2        10.1.1.2      connected     primary
vs1.example.com policy1   node2        10.1.1.3      disconnected  primary
----
Dieses Beispiel zeigt nur Informationen über verbundene FPolicy Server an:

[listing]
----
cluster1::> vserver fpolicy show-engine -fields server -server-status connected
node       vserver         policy-name server
---------- --------------- ----------- -------
node1      vs1.example.com policy1     10.1.1.2
node2      vs1.example.com policy1     10.1.1.2
----
