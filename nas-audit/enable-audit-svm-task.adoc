= Die Überwachung auf ONTAP SVMs wird nach dem Einrichten der Überwachungskonfiguration aktiviert.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem die Überwachungskonfiguration abgeschlossen ist, muss die Überwachung auf der Storage Virtual Machine (SVM) aktiviert werden.

.Bevor Sie beginnen
Die SVM Audit-Konfiguration muss bereits vorhanden sein.

.Über diese Aufgabe
Wenn eine SVM Disaster Recovery ID Discard-Konfiguration zum ersten Mal gestartet wird (nach Abschluss der SnapMirror-Initialisierung) und die SVM über eine Auditing-Konfiguration verfügt, deaktiviert ONTAP die Auditing-Konfiguration automatisch. Auditing ist auf der schreibgeschützten SVM deaktiviert, um ein Überlaufen der Staging-Volumes zu verhindern. Auditing kann erst wieder aktiviert werden, nachdem die SnapMirror Beziehung aufgehoben und die SVM schreibbar ist.

.Schritte
. Audit auf der SVM aktivieren:
+
`vserver audit enable -vserver vserver_name`

+
`vserver audit enable -vserver vs1`
