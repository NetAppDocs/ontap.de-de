= Informationen zum Status der ONTAP FPolicy Passthrough-Read-Verbindung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Informationen zum Status der Passthrough-Read-Verbindung zu externen FPolicy Servern (FPolicy Server) für den Cluster oder eine bestimmte Storage Virtual Machine (SVM) anzeigen. Diese Informationen können dabei helfen zu erkennen, welche FPolicy Server über Passthrough-Read-Datenverbindungen verfügen und bei welchen FPolicy Servern die Passthrough-Read-Verbindung getrennt ist.

.Über diese Aufgabe
Wenn Sie keinen Parameter angeben, zeigt der Befehl die folgenden Informationen an:

* SVM-Name
* FPolicy Richtlinienname
* Knotenname
* IP-Adresse des FPolicy Servers
* FPolicy Passthrough-Read Verbindungsstatus


Zusätzlich zur Anzeige von Informationen über FPolicy-Verbindungen im Cluster oder einer bestimmten SVM lassen sich Befehlsparameter verwenden, um die Ausgabe des Befehls nach anderen Kriterien zu filtern.

Sie können den  `-instance` Parameter angeben, um detaillierte Informationen zu den aufgelisteten Richtlinien anzuzeigen. Alternativ kann der  `-fields` Parameter verwendet werden, um nur die angegebenen Felder in der Befehlsausgabe anzuzeigen. Sie können  `?` nach dem  `-fields` Parameter eingeben, um herauszufinden, welche Felder verwendet werden können.

.Schritt
. Gefilterte Informationen zum Verbindungsstatus zwischen dem Knoten und dem FPolicy Server werden mithilfe des entsprechenden Befehls angezeigt:
+
[cols="35,65"]
|===


| Wenn Verbindungsstatusinformationen über ... angezeigt werden sollen | Geben Sie den Befehl ein... 


 a| 
FPolicy Passthrough-Read-Verbindungsstatus für den Cluster
 a| 
`vserver fpolicy show-passthrough-read-connection`



 a| 
FPolicy Passthrough-Read-Verbindungsstatus für eine bestimmte SVM
 a| 
`vserver fpolicy show-passthrough-read-connection -vserver vserver_name`



 a| 
FPolicy Passthrough-Read-Verbindungsstatus für eine bestimmte Richtlinie
 a| 
`vserver fpolicy show-passthrough-read-connection -policy-name policy_name`



 a| 
Detaillierter Status der FPolicy Passthrough-Read-Verbindung für eine bestimmte Richtlinie
 a| 
`vserver fpolicy show-passthrough-read-connection -policy-name policy_name -instance`



 a| 
FPolicy Passthrough-Read-Verbindungsstatus für den angegebenen Status
 a| 
`vserver fpolicy show-passthrough-read-connection -policy-name policy_name -server-status status` Der Serverstatus kann einer der folgenden sein:

** `connected`
** `disconnected`


|===


.Beispiel
Der folgende Befehl zeigt Informationen über Passthrough-Leseverbindungen von allen FPolicy Servern im Cluster an:

[listing]
----
cluster1::> vserver fpolicy show-passthrough-read-connection
                                            FPolicy           Server
Vserver          Policy Name   Node         Server            Status
---------------  ------------- ------------ ----------------- --------------
vs2.example.com  pol_cifs_2    FPolicy-01   2.2.2.2           disconnected
vs1.example.com  pol_cifs_1    FPolicy-01   1.1.1.1           connected
----
Der folgende Befehl zeigt detaillierte Informationen über Passthrough-Leseverbindungen von FPolicy-Servern an, die in der Richtlinie "`pol_cifs_1`" konfiguriert sind:

[listing]
----
cluster1::> vserver fpolicy show-passthrough-read-connection -policy-name pol_cifs_1 -instance

                                             Node: FPolicy-01
                                          Vserver: vs1.example.com
                                           Policy: pol_cifs_1
                                           Server: 1.1.1.1
                Session ID of the Control Channel: 8cef052e-2502-11e3-88d4-123478563412
                                    Server Status: connected
      Time Passthrough Read Channel was Connected: 9/24/2013 10:17:45
   Time Passthrough Read Channel was Disconnected: -
Reason for Passthrough Read Channel Disconnection: none
----
