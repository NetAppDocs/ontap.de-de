= Informationen zu ONTAP FPolicy Diensten in SVM-Namensräumen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP stellt einen einheitlichen Storage Virtual Machine (SVM) Namespace bereit. Volumes im gesamten Cluster werden durch Verbindungen zu einem einzigen logischen Dateisystem zusammengeführt. Der FPolicy Server ist mit der Namespace-Topologie vertraut und stellt FPolicy-Dienste im gesamten Namespace bereit.

Der Namensraum ist spezifisch für die SVM und innerhalb dieser enthalten, daher ist er nur im Kontext der SVM sichtbar. Namensräume weisen folgende Eigenschaften auf:

* In jedem SVM existiert ein einziger Namespace, dessen Wurzel das Root-Volume ist, das im Namespace als Schrägstrich (/) dargestellt wird.
* Alle anderen Volumen haben Verbindungspunkte unterhalb des Root-Verzeichnisses (/).
* Volumenverbindungen sind für Clients transparent.
* Ein einzelner NFS Export ermöglicht den Zugriff auf den gesamten Namensraum, alternativ können Exportrichtlinien bestimmte Volumes exportieren.
* SMB-Freigaben können auf dem Volume, auf Qtrees innerhalb des Volumes oder auf einem beliebigen Verzeichnis innerhalb des Namespace erstellt werden.
* Die Namensraumarchitektur ist flexibel.
+
Beispiele für typische Namensraumarchitekturen sind wie folgt:

+
** Ein Namensraum mit einem einzigen Zweig vom Stammverzeichnis
** Ein Namensraum mit mehreren Zweigen vom Stammverzeichnis
** Ein Namespace mit mehreren unverzweigten Volumes vom Stammverzeichnis
