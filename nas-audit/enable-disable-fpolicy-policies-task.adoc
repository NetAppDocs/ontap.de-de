= ONTAP FPolicy Richtlinien aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
FPolicy-Richtlinien können nach Abschluss der Konfiguration aktiviert werden. Durch das Aktivieren der Richtlinie werden deren Priorität festgelegt und die Überwachung des Dateizugriffs für die Richtlinie gestartet. FPolicy-Richtlinien können deaktiviert werden, wenn die Dateizugriffsüberwachung für diese Richtlinie beendet werden soll.

.Bevor Sie beginnen
Bevor FPolicy Richtlinien aktiviert werden, muss die FPolicy Konfiguration abgeschlossen sein.

.Über diese Aufgabe
* Die Priorität wird verwendet, wenn mehrere Richtlinien auf der Storage Virtual Machine (SVM) aktiviert sind und mehr als eine Richtlinie dasselbe Dateizugriff-Ereignis abonniert hat.
* Richtlinien, die die native Engine-Konfiguration verwenden, haben eine höhere Priorität als Richtlinien für jede andere Engine, unabhängig von der ihnen bei der Aktivierung der Richtlinie zugewiesenen Sequenznummer.
* Wenn die Priorität einer FPolicy Richtlinie geändert werden soll, ist es erforderlich, die Richtlinie zu deaktivieren und anschließend mit der neuen Sequenznummer wieder zu aktivieren.


.Schritt
. Die entsprechende Maßnahme ausführen:
+
[cols="35,65"]
|===


| Wenn Sie möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Aktivieren einer FPolicy Richtlinie
 a| 
`vserver fpolicy enable -vserver <svm_name> -policy-name <policy_name> [-sequence-number <integer>]`



 a| 
Eine FPolicy Richtlinie deaktivieren
 a| 
`vserver fpolicy disable -vserver <svm_name> -policy-name <policy_name>`

|===
