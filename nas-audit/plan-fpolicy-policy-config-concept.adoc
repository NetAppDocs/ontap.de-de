= Informationen zu den Richtlinienkonfigurationen von ONTAP FPolicy
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie die FPolicy Richtlinie konfigurieren, ist es wichtig zu verstehen, welche Parameter beim Erstellen der Richtlinie erforderlich sind und aus welchen Gründen bestimmte optionale Parameter konfiguriert werden können. Diese Informationen unterstützen dabei, die passenden Werte für jeden Parameter festzulegen.

Beim Erstellen einer FPolicy Richtlinie wird die Richtlinie mit den folgenden Elementen verknüpft:

* Die Storage Virtual Machine (SVM)
* Ein oder mehrere FPolicy Ereignisse
* Eine externe FPolicy Engine


Sie können außerdem verschiedene optionale Richtlinieneinstellungen konfigurieren.



[[what-the-fpolicy-policy-configuration-contains]]
== Was die FPolicy Richtlinienkonfiguration enthält

Die folgende Liste der verfügbaren erforderlichen und optionalen Parameter der FPolicy Richtlinie unterstützt bei der Planung Ihrer Konfiguration:

[cols="40,20,20,20"]
|===


| Art der Information | Option | Erforderlich | Standard 


 a| 
_SVM Name_

Gibt den Namen der SVM an, auf der eine FPolicy Richtlinie erstellt werden soll.
 a| 
`-vserver` `vserver_name`
 a| 
Ja
 a| 
Keine



 a| 
_Richtlinienname_

Gibt den Namen der FPolicy Richtlinie an.

Der Name darf maximal 256 Zeichen lang sein.

[NOTE]
====
Der Name sollte bis zu 200 Zeichen lang sein, wenn die Richtlinie in einer MetroCluster oder SVM-Notfallwiederherstellungskonfiguration konfiguriert wird.

====
Der Name kann jede beliebige Kombination der folgenden ASCII-Zeichen enthalten:

* `a`durch `z`
* `A`durch `Z`
* `0`durch `9`
* "`_`", "`-`", and "`.`"

 a| 
`-policy-name` `policy_name`
 a| 
Ja
 a| 
Keine



 a| 
_Ereignisnamen_

Gibt eine durch Kommas getrennte Liste von Ereignissen an, die der FPolicy Richtlinie zugeordnet werden.

* Sie können einer Richtlinie mehrere Ereignisse zuordnen.
* Ein Ereignis ist spezifisch für ein Protokoll.
* Sie können eine einzige Richtlinie verwenden, um Dateizugriffsereignisse für mehr als ein Protokoll zu überwachen, indem Sie für jedes Protokoll, das die Richtlinie überwachen soll, ein Ereignis erstellen und dann die Ereignisse der Richtlinie zuordnen.
* Die Ereignisse müssen bereits existieren.

 a| 
`-events` `event_name`, ...
 a| 
Ja
 a| 
Keine



 a| 
_Persistenter Speicher_

Ab ONTAP 9.14.1 gibt dieser Parameter den persistenten Speicher an, in dem Dateizugriffsereignisse für asynchron nicht obligatorische Richtlinien in der SVM erfasst werden.
 a| 
`-persistent-store` `persistent_store_name`
 a| 
Nein
 a| 
Keine



 a| 
_Externer Engine Name_

Gibt den Namen der externen Engine an, die der FPolicy Richtlinie zugeordnet werden soll.

* Eine externe Engine enthält Informationen, die der Knoten benötigt, um Benachrichtigungen an einen FPolicy Server zu senden.
* Sie können FPolicy so konfigurieren, dass die ONTAP native externe Engine für einfache Dateiblockierung verwendet wird oder eine externe Engine genutzt wird, die für die Verwendung externer FPolicy Server (FPolicy Server) konfiguriert ist, um eine anspruchsvollere Dateiblockierung und Dateiverwaltung zu ermöglichen.
* Wenn Sie die native externe Engine verwenden möchten, können Sie entweder keinen Wert für diesen Parameter angeben oder  `native` als Wert angeben.
* Wenn Sie FPolicy Server verwenden möchten, muss die Konfiguration für die externe Engine bereits vorhanden sein.

 a| 
`-engine` `engine_name`
 a| 
Ja (es sei denn, die Richtlinie verwendet die interne ONTAP native Engine)
 a| 
`native`



 a| 
Ist eine obligatorische Überprüfung erforderlich?

Legt fest, ob eine obligatorische Dateizugriff-Überprüfung erforderlich ist.

* Die Einstellung für die obligatorische Überprüfung legt fest, welche Maßnahmen bei einem Dateizugriff ergriffen werden, wenn alle primären und sekundären Server ausgefallen sind oder innerhalb eines bestimmten Zeitüberschreitungszeitraums keine Antwort von den FPolicy-Servern eingeht.
* Wenn auf  `true` gesetzt, werden Dateizugriffsereignisse verweigert.
* Wenn auf  `false` gesetzt, sind Dateizugriffsereignisse zulässig.

 a| 
`-is-mandatory` {`true`|`false`}
 a| 
Nein
 a| 
`true`



 a| 
_Privilegierten Zugriff erlauben_

Legt fest, ob der FPolicy Server über eine privilegierte Datenverbindung privilegierten Zugriff auf die überwachten Dateien und Ordner erhält.

Sofern konfiguriert, können FPolicy Server über die privilegierte Datenverbindung auf Dateien vom Stamm der SVM mit den überwachten Daten zugreifen.

Für den privilegierten Datenzugriff muss SMB auf dem Cluster lizenziert sein und alle Daten-LIFs, die zur Verbindung mit den FPolicy-Servern verwendet werden, müssen so konfiguriert sein, dass  `cifs` als eines der zulässigen Protokolle vorhanden ist.

Wenn die Richtlinie so konfiguriert werden soll, dass privilegierter Zugriff zugelassen wird, muss auch der Benutzername für das Konto angegeben werden, das der FPolicy Server für privilegierten Zugriff verwendet.
 a| 
`-allow-privileged-access` {`yes`|`no`}
 a| 
Nein (es sei denn, passthrough-read ist aktiviert)
 a| 
`no`



 a| 
_Privilegierter Benutzername_

Gibt den Benutzernamen des Kontos an, das die FPolicy Server für den privilegierten Datenzugriff verwenden.

* Der Wert für diesen Parameter sollte das Format "`Domäne\Benutzername`" verwenden.
* Wenn  `-allow-privileged-access` auf  `no` gesetzt ist, wird jeder für diesen Parameter festgelegte Wert ignoriert.

 a| 
`-privileged-user-name` `user_name`
 a| 
Nein (es sei denn, privilegierter Zugriff ist aktiviert)
 a| 
Keine



 a| 
_Passthrough-Read zulassen_

Legt fest, ob die FPolicy Server Passthrough-Lesedienste für Dateien bereitstellen können, die von den FPolicy Servern auf einen Sekundärspeicher archiviert wurden (Offline-Dateien):

* Passthrough-Read ist eine Methode, um Daten aus Offline-Dateien zu lesen, ohne die Daten auf den primären Speicher wiederherzustellen.
+
Passthrough-read reduziert die Antwortzeiten, da Dateien nicht vor der Beantwortung einer Leseanfrage in den primären Speicher zurückgeholt werden müssen. Zudem optimiert Passthrough-read die Speichereffizienz, da kein primärer Speicherplatz mit Dateien belegt werden muss, die ausschließlich zum Erfüllen von Leseanfragen zurückgeholt werden.

* Wenn diese Option aktiviert ist, stellen die FPolicy Server die Daten für die Datei über einen separaten privilegierten Datenkanal bereit, der speziell für Passthrough-Reads geöffnet wird.
* Wenn Sie passthrough-read konfigurieren möchten, muss die Richtlinie ebenfalls so konfiguriert sein, dass privilegierter Zugriff erlaubt ist.

 a| 
`-is-passthrough-read-enabled` {`true`|`false`}
 a| 
Nein
 a| 
`false`

|===
