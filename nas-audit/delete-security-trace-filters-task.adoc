= Sicherheits-Tracefilter auf ONTAP SVMs löschen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ein Sicherheitsverfolgungsfiltereintrag nicht mehr benötigt wird, kann er gelöscht werden. Da maximal 10 Sicherheitsverfolgungsfilter pro Storage Virtual Machine (SVM) möglich sind, ermöglicht das Löschen nicht benötigter Filter das Erstellen neuer Filter, wenn das Maximum erreicht ist.

.Über diese Aufgabe
Um den zu löschenden Security Trace-Filter eindeutig zu identifizieren, sind die folgenden Angaben erforderlich:

* Der Name der SVM, auf die der Trace-Filter angewendet wird
* Die Filterindexnummer des Trace-Filters


.Schritte
. Die Filterindexnummer des Security Trace Filtereintrags, der gelöscht werden soll, identifizieren:
+
`vserver security trace filter show -vserver vserver_name`

+
`vserver security trace filter show -vserver vs1`

+
[listing]
----

Vserver  Index   Client-IP            Path            Trace-Allow  Windows-Name
-------- -----  -----------  ----------------------   -----------  -------------
vs1        1    -            /dir1/dir2/file.txt          yes      -
vs1        2    -            /dir3/dir4/                  no       mydomain\joe
----
. Mit den Informationen zur Filterindexnummer aus dem vorherigen Schritt den Filtereintrag löschen:
+
`vserver security trace filter delete -vserver vserver_name -index index_number`

+
`vserver security trace filter delete -vserver vs1 -index 1`

. Es wird überprüft, ob der Eintrag des Sicherheitsverfolgungsfilters gelöscht wurde:
+
`vserver security trace filter show -vserver vserver_name`

+
`vserver security trace filter show -vserver vs1`

+
[listing]
----

Vserver  Index   Client-IP            Path            Trace-Allow   Windows-Name
-------- -----  -----------  ----------------------   -----------   -------------
vs1        2    -            /dir3/dir4/                  no        mydomain\joe
----
