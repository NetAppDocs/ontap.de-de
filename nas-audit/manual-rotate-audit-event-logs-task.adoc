= Die Audit-Ereignisprotokolle manuell rotieren, um bestimmte ONTAP SVM-Ereignisprotokolle anzuzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie die Audit-Ereignisprotokolle anzeigen können, müssen diese in ein benutzerlesbares Format konvertiert werden. Wenn die Ereignisprotokolle einer bestimmten Storage Virtual Machine (SVM) vor der automatischen Protokollrotation durch ONTAP angezeigt werden sollen, ist eine manuelle Rotation der Audit-Ereignisprotokolle auf dieser SVM möglich.

.Schritt
. Die Audit-Ereignisprotokolle werden mit dem  `vserver audit rotate-log` Befehl rotiert.
+
`vserver audit rotate-log -vserver vs1`

+
Das Audit-Ereignisprotokoll wird im SVM-Audit-Ereignisprotokollverzeichnis im durch die Audit-Konfiguration festgelegten Format gespeichert (`XML` oder  `EVTX` und kann mit der entsprechenden Anwendung angezeigt werden.
