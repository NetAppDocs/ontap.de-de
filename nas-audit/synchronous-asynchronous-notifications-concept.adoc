= ONTAP FPolicy synchron und asynchron Benachrichtigungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
FPolicy sendet Benachrichtigungen über die FPolicy Schnittstelle an externe FPolicy Server. Die Benachrichtigungen werden entweder im synchronen oder asynchronen Modus gesendet. Der Benachrichtigungsmodus bestimmt, was ONTAP nach dem Senden der Benachrichtigungen an die FPolicy Server tut.

* *Asynchrone Benachrichtigungen*
+
Bei asynchronen Benachrichtigungen wartet der Knoten nicht auf eine Antwort vom FPolicy Server, was den Gesamtdurchsatz des Systems erhöht. Diese Art von Benachrichtigung ist für Anwendungen geeignet, bei denen der FPolicy Server nicht verlangt, dass aufgrund der Benachrichtigungsbewertung eine Aktion durchgeführt wird. Beispielsweise werden asynchrone Benachrichtigungen verwendet, wenn der Administrator einer Storage Virtual Machine (SVM) den Dateizugriff überwachen und Revisionsdateien protokollieren möchte.

+
Wenn ein FPolicy-Server, der im asynchronen Modus arbeitet, einen Netzausfall erlebt, werden die während des Ausfalls generierten FPolicy-Benachrichtigungen auf dem Speicherknoten gespeichert. Wenn der FPolicy-Server wieder online ist, wird er über die gespeicherten Benachrichtigungen informiert und kann diese vom Speicherknoten abrufen. Die Dauer, für die die Benachrichtigungen während eines Ausfalls gespeichert werden können, ist bis zu 10 Minuten konfigurierbar.

+
Ab ONTAP 9.14.1 ermöglicht FPolicy die Einrichtung eines persistenten Speichers zur Erfassung von Dateizugriffsereignissen für asynchrone, nicht obligatorische Richtlinien in der SVM. Persistente Speicher können die Client-E/A-Verarbeitung von der FPolicy-Benachrichtigungsverarbeitung entkoppeln, um die Client-Latenz zu reduzieren. Synchrone (obligatorische oder nicht obligatorische) und asynchron obligatorische Konfigurationen werden nicht unterstützt.

* *Synchrone Benachrichtigungen*
+
Im synchronen Modus muss der FPolicy Server jede Benachrichtigung bestätigen, bevor der Clientvorgang fortgesetzt werden darf. Diese Art von Benachrichtigung wird verwendet, wenn eine Aktion aufgrund der Auswertung der Benachrichtigung erforderlich ist. Beispielsweise werden synchrone Benachrichtigungen verwendet, wenn der SVM Administrator Anfragen anhand von Kriterien, die auf dem externen FPolicy Server festgelegt sind, zulassen oder ablehnen möchte.





[[synchronous-and-asynchronous-applications]]
== Synchrone und asynchrone Anwendungen

Es gibt viele Einsatzmöglichkeiten für FPolicy Anwendungen, sowohl asynchron als auch synchron.

Asynchrone Anwendungen sind solche, bei denen der externe FPolicy Server den Dateizugriff oder Verzeichniszugriff nicht verändert oder Daten auf der Storage Virtual Machine (SVM) modifiziert. Beispiel:

* Dateizugriff und Protokollierung
* Ressourcenmanagement


Synchrone Anwendungen sind solche, bei denen der Dateizugriff geändert oder Daten vom externen FPolicy Server modifiziert werden. Beispiel:

* Quotenmanagement
* Dateizugriffsblockierung
* Dateiarchivierung und hierarchische Speicherverwaltung
* Verschlüsselungs- und Entschlüsselungsdienste
* Komprimierungs- und Dekomprimierungsdienste
