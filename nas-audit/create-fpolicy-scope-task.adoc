= ONTAP FPolicy-Bereiche erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach dem Erstellen der FPolicy Richtlinie ist ein FPolicy Bereich zu erstellen. Beim Erstellen des Bereichs wird dieser mit einer FPolicy Richtlinie verknüpft. Ein Bereich definiert die Grenzen, auf die die FPolicy Richtlinie angewendet wird. Bereiche können Dateien basierend auf Freigaben, Exportrichtlinien, Volumes und Dateierweiterungen ein- oder ausschließen.

.Bevor Sie beginnen
Das FPolicy Scope-Arbeitsblatt muss ausgefüllt werden. Die FPolicy Richtlinie muss mit einer zugehörigen externen Engine vorhanden sein (falls die Richtlinie für die Verwendung externer FPolicy Server konfiguriert ist) und mindestens ein zugehöriges FPolicy Ereignis aufweisen.

.Schritte
. Der FPolicy-Bereich wird mithilfe des  `vserver fpolicy policy scope create` Befehls erstellt.
+
`vserver fpolicy policy scope create -vserver vs1.example.com -policy-name policy1 -volumes-to-include datavol1,datavol2`

. Die FPolicy Bereichskonfiguration kann mit dem  `vserver fpolicy policy scope show` Befehl überprüft werden.
+
`vserver fpolicy policy scope show -vserver vs1.example.com -instance`

+
[listing]
----

                   Vserver: vs1.example.com
                    Policy: policy1
         Shares to Include: -
         Shares to Exclude: -
        Volumes to Include: datavol1, datavol2
        Volumes to Exclude: -
Export Policies to Include: -
Export Policies to Exclude: -
File Extensions to Include: -
File Extensions to Exclude: -
----
