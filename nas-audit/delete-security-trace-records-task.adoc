= Sicherheitsverfolgungsdatensätze auf ONTAP SVMs löschen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem ein Filter-Trace-Datensatz zur Überprüfung der Dateizugriffssicherheit oder zur Fehlerbehebung bei Zugriffsproblemen von SMB- oder NFS-Clients verwendet wurde, kann der Security-Trace-Datensatz aus dem Security-Trace-Protokoll gelöscht werden.

.Über diese Aufgabe
Bevor Sie einen Sicherheitsverfolgungsdatensatz löschen können, muss die Sequenznummer des Datensatzes bekannt sein.

[NOTE]
====
Jede Storage Virtual Machine (SVM) kann maximal 128 Trace-Datensätze speichern. Wenn die maximale Anzahl auf der SVM erreicht ist, werden die ältesten Trace-Datensätze automatisch gelöscht, sobald neue hinzugefügt werden. Wenn das manuelle Löschen von Trace-Datensätzen auf dieser SVM vermieden werden soll, kann ONTAP die ältesten Trace-Ergebnisse nach Erreichen des Maximums automatisch löschen, um Platz für neue Ergebnisse zu schaffen.

====
.Schritte
. Die Sequenznummer des Datensatzes, der gelöscht werden soll, identifizieren:
+
`vserver security trace trace-result show -vserver vserver_name -instance`

. Den Sicherheitsprotokolleintrag löschen:
+
`vserver security trace trace-result delete -node node_name -vserver vserver_name -seqnum integer`

+
`vserver security trace trace-result delete -vserver vs1 -node node1 -seqnum 999`

+
** `-node` `node_name` ist der Name des Cluster-Knotens, auf dem das persistente Tracing-Ereignis aufgetreten ist, das gelöscht werden soll.
+
Dies ist ein Pflichtparameter.

** `-vserver` `vserver_name` ist der Name der SVM, auf der das persistente Tracing-Ereignis aufgetreten ist, das gelöscht werden soll.
+
Dies ist ein Pflichtparameter.

** `-seqnum` `integer` ist die Sequenznummer des Protokollereignisses, das gelöscht werden soll.
+
Dies ist ein Pflichtparameter.
