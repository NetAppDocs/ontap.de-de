= NAS-Speicher für Linux-Server mit NFS und ONTAP System Manager bereitstellen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Speicher-VMs können erstellt oder geändert werden, um NFS-Server für die Bereitstellung von Daten an Linux-Clients zu aktivieren.

Eine neue oder bestehende Storage-VM kann für das NFS-Protokoll mit diesem Verfahren aktiviert werden.

image:workflow_nas_enable_linux_nfs.png["Workflow-Zusammenfassung: 1 NFS aktivieren 2 Exportrichtlinie für das Root-Volume öffnen 3 DNS konfigurieren 4 Namensdienste konfigurieren 5 Verschlüsselung konfigurieren, falls erforderlich"]

.Bevor Sie beginnen
Stellen Sie sicher, dass die Konfigurationsdetails für alle in Ihrer Umgebung erforderlichen Netzwerk-, Authentifizierungs- oder Sicherheitsdienste notiert wurden.

.Schritte
. NFS auf einer Storage-VM aktivieren.
+
** Für neue Speicher-VMs: Wählen Sie *Speicher > Speicher-VMs*, wählen Sie *Hinzufügen*, geben Sie einen Speicher-VM-Namen ein und wählen Sie auf der Registerkarte *SMB/CIFS, NFS, S3* die Option *NFS aktivieren*.
+
... Die Standardsprache bestätigen.
... Netzwerkschnittstellen hinzufügen.
... Informationen des Administratorkontos der Storage-VM aktualisieren (optional).


** Für vorhandene Speicher-VMs: Wählen Sie *Cluster > Speicher-VMs*, wählen Sie die Speicher-VM aus, wählen Sie *Einstellungen* und dann image:icon_gear.gif["Aktionssymbol"] oder image:icon-edit-pencil-blue-outline.png["Bearbeiten-Symbol"] unter *NFS* und wählen Sie *NFS aktivieren*.
+
... Aktivieren oder deaktivieren Sie nach Bedarf bestimmte NFS-Protokollversionen und -Optionen.
+

NOTE: Die FPolicy-Überwachung von NFSv4.2-Dateizugriffsereignissen wird nicht unterstützt. Wenn Sie die FPolicy-Überwachung konfigurieren, müssen Sie NFSv4.2 auf der SVM deaktivieren. Weitere Informationen finden Sie unter link:./nfs-admin/enable-disable-nfs-task.html["NFS für ONTAP SVMs aktivieren oder deaktivieren"].





. Die Exportrichtlinie des Root-Volumes der Storage-VM wird geöffnet:
+
.. Wählen Sie *Storage > Volumes*, wählen Sie das Stammvolume der Storage-VM (das standardmäßig _volume-name_ _root ist), und wählen Sie dann die unter *Export Policy* angezeigte Richtlinie aus.
.. Wählen Sie *Add*, um eine Regel hinzuzufügen.
+
*** Kundenspezifikation = `0.0.0.0/0`
*** Zugriffsprotokolle = NFS
*** Zugriffsdetails = UNIX Read-Only




. Konfigurieren Sie DNS für die Hostnamensauflösung: Wählen Sie *Speicher > Speicher-VMs*, wählen Sie die Speicher-VM aus, wählen Sie *Einstellungen* und dann image:icon_gear.gif["Aktionssymbol"] unter *DNS* aus.
. Namensdienste werden nach Bedarf konfiguriert.
+
.. Wählen Sie *Speicher > Speicher-VMs*, wählen Sie die Speicher-VM aus, wählen Sie *Einstellungen* und dann image:icon_gear.gif["Aktionssymbol"]LDAP oder NIS.
.. Wählen Sie image:icon_pencil.gif["Bearbeiten-Symbol"] im Namensdienste-Umschalter aus, um alle Änderungen zu übernehmen.


. Kerberos bei Bedarf konfigurieren:
+
.. Wählen Sie *Storage > Storage VMs*, wählen Sie die Storage-VM aus und wählen Sie dann *Settings*.
.. Wählen Sie image:icon_arrow.gif["Pfeilsymbol"] in der Kerberos-Kachel und dann *Hinzufügen* aus.
