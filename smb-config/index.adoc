= Informationen zur SMB-Konfiguration mit der ONTAP CLI
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit ONTAP 9 CLI-Befehlen lässt sich der SMB-Clientzugriff auf Dateien konfigurieren, die sich in einem neuen Volume oder qtree in einer neuen oder bestehenden SVM befinden.

[NOTE]
====
_SMB_ (Server Message Block) bezeichnet moderne Dialekte des Common Internet File System (CIFS)-Protokolls. _CIFS_ erscheint weiterhin in der ONTAP-Befehlszeilenschnittstelle (CLI) und in OnCommand Management-Tools.

====
Diese Vorgehensweisen gelten, wenn der SMB-Zugriff auf ein Volume oder einen Qtree wie folgt konfiguriert werden soll:

* Sie möchten SMB Version 2 oder höher verwenden.
* Es sollen nur SMB-Clients und keine NFS-Clients bedient werden (keine Multiprotokoll-Konfiguration).
* NTFS-Dateiberechtigungen werden verwendet, um das neue Volume zu sichern.
* Sie verfügen über Cluster-Administratorrechte, nicht über SVM-Administratorrechte.
+
Für die Erstellung von SVMs und LIFs sind Cluster-Administratorrechte erforderlich. Für andere SMB-Konfigurationsaufgaben sind SVM-Administratorrechte ausreichend.

* Sie möchten die Befehlszeilenschnittstelle (CLI) verwenden, nicht System Manager oder ein automatisiertes Skripting-Tool.
+
Informationen zur Konfiguration des NAS-Multiprotokollzugriffs mit System Manager finden sich unter link:../task_nas_provision_nfs_and_smb.html["NAS-Speicher für Windows und Linux mit NFS und SMB bereitstellen"].

* Sie möchten bewährte Verfahren anwenden, nicht jede verfügbare Option untersuchen.
+
Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].



Für Einzelheiten zum Funktionsumfang des ONTAP SMB-Protokolls siehe link:../smb-admin/index.html["SMB-Referenzübersicht"].
