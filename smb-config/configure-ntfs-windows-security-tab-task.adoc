= NTFS-Dateiberechtigungen in ONTAP SMB-Freigaben konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um Benutzern oder Gruppen, die Zugriff auf eine Freigabe haben, den Dateizugriff zu ermöglichen, müssen die NTFS-Dateiberechtigungen für Dateien und Verzeichnisse in dieser Freigabe von einem Windows-Client aus konfiguriert werden.

.Bevor Sie beginnen
Der Administrator, der diese Aufgabe ausführt, muss über ausreichende NTFS-Berechtigungen verfügen, um die Berechtigungen für die ausgewählten Objekte zu ändern.

.Über diese Aufgabe
link:../smb-admin/index.html["SMB Management"] und Ihre Windows-Dokumentation enthalten Informationen darüber, wie Standard- und erweiterte NTFS-Berechtigungen festgelegt werden.

.Schritte
. Melden Sie sich bei einem Windows-Client als Administrator an.
. Im Menü *Extras* des Windows Explorers *Netzlaufwerk verbinden* auswählen.
. Das Feld *Netzlaufwerk verbinden* ausfüllen:
+
.. Einen Laufwerksbuchstaben auswählen.
.. Geben Sie im Feld *Ordner* den Namen des SMB-Servers ein, der die Freigabe enthält, die die Daten enthält, für die Berechtigungen angewendet werden sollen, sowie den Namen der Freigabe.
+
Wenn Ihr SMB-Servername SMB_SERVER01 ist und Ihre Freigabe den Namen "`SHARE1`" hat, würden Sie  `\\SMB_SERVER01\SHARE1` eingeben.

+
[NOTE]
====
Sie können die IP-Adresse der Datenschnittstelle für den SMB-Server anstelle des SMB-Servernamens angeben.

====
.. Klicken Sie auf *Finish*.


+
Das von Ihnen ausgewählte Laufwerk ist eingebunden und einsatzbereit, wobei das Windows Explorer-Fenster die in der Freigabe enthaltenen Dateien und Ordner anzeigt.

. Wählen Sie die Datei oder das Verzeichnis aus, für das NTFS-Dateiberechtigungen festgelegt werden sollen.
. Klicken Sie mit der rechten Maustaste auf die Datei oder das Verzeichnis und wählen Sie dann *Eigenschaften*.
. Die Registerkarte *Sicherheit* auswählen.
+
Die Registerkarte „Sicherheit“ zeigt die Liste der Benutzer und Gruppen an, für die NTFS-Berechtigungen festgelegt sind. Das Feld „Berechtigungen für <Object>“ zeigt eine Liste der geltenden Zulassungs- und Verweigerungsberechtigungen für den ausgewählten Benutzer oder die ausgewählte Gruppe an.

. Klicken Sie auf *Bearbeiten*.
+
Das Feld <Object> „Berechtigungen“ wird geöffnet.

. Führen Sie die gewünschten Aktionen aus:
+
|===
| Wenn Sie möchten.... | Führen Sie folgende Schritte aus... 


 a| 
Standardmäßige NTFS-Berechtigungen für einen neuen Benutzer oder eine neue Gruppe
 a| 
.. Klicken Sie auf *Add*.
+
Das Fenster „Benutzer, Computer, Dienstkonten oder Gruppen auswählen“ wird geöffnet.

.. Im Feld *Geben Sie die auszuwählenden Objektnamen ein* den Namen des Benutzers oder der Gruppe eingeben, für die NTFS-Berechtigungen hinzugefügt werden sollen.
.. Klicken Sie auf *OK*.




 a| 
Standard-NTFS-Berechtigungen für einen Benutzer oder eine Gruppe ändern oder entfernen
 a| 
Im Feld *Gruppen- oder Benutzernamen* den Benutzer oder die Gruppe auswählen, die geändert oder entfernt werden soll.

|===
. Führen Sie die gewünschten Aktionen aus:
+
|===
| Wenn Sie möchten... | Folgendes ist durchzuführen 


 a| 
Standardmäßige NTFS-Berechtigungen für einen neuen oder vorhandenen Benutzer oder eine Gruppe festlegen
 a| 
Wählen Sie im Feld *Berechtigungen für <Object>* die Kästchen *Zulassen* oder *Verweigern* für die Art des Zugriffs aus, die Sie für den ausgewählten Benutzer oder die ausgewählte Gruppe erlauben oder nicht erlauben möchten.



 a| 
Einen Benutzer oder eine Gruppe entfernen
 a| 
Klicken Sie auf *Entfernen*.

|===
+
[NOTE]
====
Wenn einige oder alle Standardberechtigungsfelder nicht auswählbar sind, liegt das daran, dass die Berechtigungen vom übergeordneten Objekt geerbt werden. Das Feld *Spezielle Berechtigungen* ist nicht auswählbar. Wenn es ausgewählt ist, bedeutet dies, dass für den ausgewählten Benutzer oder die ausgewählte Gruppe eine oder mehrere der granularen erweiterten Rechte festgelegt wurden.

====
. Nachdem das Hinzufügen, Entfernen oder Bearbeiten der NTFS-Berechtigungen für dieses Objekt abgeschlossen ist, auf *OK* klicken.
