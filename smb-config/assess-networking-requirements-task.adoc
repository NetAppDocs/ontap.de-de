= ONTAP SMB-Netzwerkanforderungen bewerten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor SMB-Speicher für Clients bereitgestellt wird, muss überprüft werden, ob die Netzwerkkonfiguration korrekt ist und die Anforderungen für die SMB-Bereitstellung erfüllt werden.

.Bevor Sie beginnen
Die folgenden Cluster Netzwerkobjekte müssen konfiguriert werden:

* Physische und logische Ports
* Broadcast-Domains
* Subnetze (falls erforderlich)
* IPspaces (falls erforderlich, zusätzlich zum Standard IPspace)
* Failover-Gruppen (falls erforderlich, zusätzlich zur standardmäßigen Failover-Gruppe für jede Broadcast-Domäne)
* Externe Firewalls


.Schritte
. Verfügbare physische und virtuelle Ports werden angezeigt: `network port show`
+
** Wenn möglich, ist der Port mit der höchsten Geschwindigkeit für das Datennetzwerk zu verwenden.
** Alle Komponenten im Datennetzwerk müssen für eine optimale Leistung die gleiche MTU-Einstellung haben.


+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

. Wenn die Zuweisung der IP-Adresse und des Netzwerkmaskenwerts für eine LIF über einen Subnetznamen erfolgen soll, ist zu prüfen, ob das Subnetz existiert und ausreichend Adressen verfügbar sind: `network subnet show`
+
Weitere Informationen zu  `network subnet show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-show.html["ONTAP-Befehlsreferenz"^].

+
Subnetze enthalten einen Pool von IP-Adressen, die zum selben Layer-3-Subnetz gehören. Subnetze werden mit dem  `network subnet create` Befehl erstellt.

+
Weitere Informationen zu  `network subnet create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-create.html["ONTAP-Befehlsreferenz"^].

. Verfügbare IPspaces anzeigen: `network ipspace show`
+
Sie können den Standard-IPspace oder einen benutzerdefinierten IPspace verwenden.

+
Weitere Informationen zu  `network ipspace show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-ipspace-show.html["ONTAP-Befehlsreferenz"^].

. Wenn Sie IPv6-Adressen verwenden möchten, vergewissern Sie sich, dass IPv6 auf dem Cluster aktiviert ist: `network options ipv6 show`
+
Bei Bedarf kann IPv6 mit dem  `network options ipv6 modify` Befehl aktiviert werden.

+
Weitere Informationen zu  `network options ipv6 show` und  `network options ipv6 modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+options+ipv6["ONTAP-Befehlsreferenz"^].
