= SMB-Server auf der ONTAP SVM mit angegebenen Arbeitsgruppen erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit dem  `vserver cifs create`Befehl lässt sich ein SMB-Server auf der SVM erstellen und die Arbeitsgruppe angeben, zu der er gehört.

.Bevor Sie beginnen
Die von Ihnen zur Datenbereitstellung verwendeten SVM und LIFs müssen so konfiguriert sein, dass das SMB-Protokoll zugelassen ist. Die LIFs müssen in der Lage sein, eine Verbindung zu den auf der SVM konfigurierten DNS-Servern herzustellen.

.Über diese Aufgabe
SMB-Server im Arbeitsgruppenmodus unterstützen die folgenden SMB-Funktionen nicht:

* SMB3 Witness-Protokoll
* SMB3 CA Shares
* SQL über SMB
* Ordnerumleitung
* Roaming Profile
* Gruppenrichtlinienobjekt (GPO)
* Volume Snapshot Service (VSS)


Weitere Informationen zu  `vserver cifs` und optionalen Konfigurationsparametern sowie Namensanforderungen finden Sie in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+cifs["ONTAP-Befehlsreferenz"^].

.Schritte
. Es sollte überprüft werden, ob SMB auf Ihrem Cluster lizenziert ist: `system license show -package cifs`
+
Die SMB-Lizenz ist mit link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["ONTAP One"] enthalten. Wenn ONTAP One nicht vorhanden ist und die Lizenz nicht installiert ist, sollte der Vertriebsmitarbeiter kontaktiert werden.

+
Eine CIFS-Lizenz ist nicht erforderlich, wenn der SMB-Server nur zur Authentifizierung verwendet wird.

. Den SMB-Server in einer Arbeitsgruppe erstellen: `vserver cifs create -vserver vserver_name -cifs-server cifs_server_name -workgroup workgroup_name [-comment text]`
+
Der folgende Befehl erstellt den SMB-Server "`smb_server01`" in der Arbeitsgruppe "`workgroup01`":

+
[listing]
----
cluster1::> vserver cifs create -vserver vs1.example.com -cifs-server SMB_SERVER01 -workgroup workgroup01
----
. Die SMB-Serverkonfiguration kann mit dem  `vserver cifs show` Befehl überprüft werden.
+
Im folgenden Beispiel zeigt die Befehlsausgabe, dass ein SMB-Server mit dem Namen "`smb_server01`" auf der SVM vs1.example.com in der Arbeitsgruppe "`workgroup01`" erstellt wurde:

+
[listing]
----
cluster1::> vserver cifs show -vserver vs0

                                          Vserver: vs1.example.com
                         CIFS Server NetBIOS Name: SMB_SERVER01
                    NetBIOS Domain/Workgroup Name: workgroup01
                      Fully Qualified Domain Name: -
                              Organizational Unit: -
Default Site Used by LIFs Without Site Membership: -
                                   Workgroup Name: workgroup01
                             Authentication Style: workgroup
                CIFS Server Administrative Status: up
                          CIFS Server Description:
                          List of NetBIOS Aliases: -
----


.Nachdem Sie fertig sind
Für einen CIFS-Server in einer Arbeitsgruppe sind lokale Benutzer und optional lokale Gruppen auf der SVM zu erstellen.
