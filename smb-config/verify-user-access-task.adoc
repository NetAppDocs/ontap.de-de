= Überprüfung des ONTAP SMB-Benutzerzugriffs auf die Freigabe
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es empfiehlt sich zu überprüfen, ob die von Ihnen konfigurierten Benutzer auf die SMB-Freigabe und die darin enthaltenen Dateien zugreifen können.

.Schritte
. Melden Sie sich auf einem Windows-Client als einer der Benutzer an, die nun Zugriff auf die Freigabe haben.
. Im Menü *Extras* des Windows Explorers *Netzlaufwerk verbinden* auswählen.
. Das Feld *Netzlaufwerk verbinden* ausfüllen:
+
.. Einen Laufwerksbuchstaben auswählen.
.. Im Feld *Ordner* ist der Freigabename einzugeben, der den Benutzern zur Verfügung gestellt wird.
+
Wenn Ihr SMB-Servername SMB_SERVER01 ist und Ihre Freigabe den Namen "`SHARE1`" hat, würden Sie  `\\SMB_SERVER01\share1` eingeben.

.. Klicken Sie auf *Finish*.


+
Das von Ihnen ausgewählte Laufwerk ist eingebunden und einsatzbereit, wobei das Windows Explorer-Fenster die in der Freigabe enthaltenen Dateien und Ordner anzeigt.

. Eine Testdatei wird erstellt, ihre Existenz überprüft, Text hineingeschrieben und anschließend die Testdatei entfernt.
