= Anforderungen und Überlegungen beim Erstellen von ONTAP SMB-Freigaben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Vor dem Erstellen einer SMB-Freigabe sind die Anforderungen an Freigabepfade und Freigabeeigenschaften, insbesondere für Home-Verzeichnisse, zu berücksichtigen.

Das Erstellen einer SMB-Freigabe erfordert die Angabe einer Verzeichnispfadstruktur (unter Verwendung der  `-path` Option im  `vserver cifs share create` Befehl), auf die Clients zugreifen. Der Verzeichnispfad entspricht dem Verbindungspfad für ein Volume oder einen Qtree, den Sie im SVM-Namespace erstellt haben. Der Verzeichnispfad und der entsprechende Verbindungspfad müssen vor dem Erstellen der Freigabe vorhanden sein.

Für Freigabepfade gelten folgende Anforderungen:

* Ein Verzeichnispfadname darf maximal 255 Zeichen lang sein.
* Enthält der Pfadname ein Leerzeichen, muss die gesamte Zeichenkette in Anführungszeichen gesetzt werden (zum Beispiel  `"/new volume/mount here"`).
* Wenn der UNC-Pfad (`\\servername\sharename\filepath`) der Freigabe mehr als 256 Zeichen enthält (ohne das anfängliche "`\\`" im UNC-Pfad), ist die Registerkarte *Sicherheit* im Windows-Eigenschaften-Dialogfeld nicht verfügbar.
+
Dies ist ein Problem des Windows-Clients und nicht von ONTAP. Um dieses Problem zu vermeiden, sollten keine Freigaben mit UNC-Pfaden erstellt werden, die mehr als 256 Zeichen enthalten.



Die Standardeinstellungen für die Freigabeeigenschaften können geändert werden:

* Die standardmäßigen Anfangseigenschaften für alle Shares sind `oplocks`, `browsable`, `changenotify` und `show-previous-versions`.
* Die Angabe von Freigabeeigenschaften beim Erstellen einer Freigabe ist optional.
+
Wenn Sie jedoch beim Erstellen der Freigabe Eigenschaften festlegen, werden die Standardwerte nicht verwendet. Wenn Sie den  `-share-properties` Parameter beim Erstellen einer Freigabe verwenden, müssen Sie alle gewünschten Freigabeeigenschaften als durch Kommas getrennte Liste angeben.

* Um eine Home-Verzeichnisfreigabe festzulegen, wird die  `homedirectory` Eigenschaft verwendet.
+
Diese Funktion ermöglicht die Konfiguration einer Freigabe, die je nach Benutzer, der sich verbindet, und einer Reihe von Variablen unterschiedlichen Verzeichnissen zugeordnet wird. Anstatt separate Freigaben für jeden Benutzer erstellen zu müssen, kann eine einzelne Freigabe mit einigen Home-Verzeichnis-Parametern konfiguriert werden, um die Beziehung eines Benutzers zwischen einem Einstiegspunkt (der Freigabe) und seinem Home-Verzeichnis (einem Verzeichnis auf der SVM) zu definieren.

+
[NOTE]
====
Diese Eigenschaft kann nach dem Erstellen der Freigabe nicht mehr hinzugefügt oder entfernt werden.

====


Für die Home-Verzeichnisfreigaben gelten folgende Anforderungen:

* Bevor Sie SMB-Home-Verzeichnisse erstellen, muss mindestens ein Suchpfad für das Home-Verzeichnis mithilfe des  `vserver cifs home-directory search-path add` Befehls hinzugefügt werden.
* Home-Verzeichnisfreigaben, die durch den Wert von  `homedirectory` im  `-share-properties`-Parameter angegeben sind, müssen die  `%w` (Windows-Benutzername) dynamische Variable im Freigabenamen enthalten.
+
Der Freigabename kann zusätzlich die  `%d` (Domänenname) dynamische Variable (zum Beispiel  `%d/%w`) oder einen statischen Anteil im Freigabenamen (zum Beispiel  `home1_%w`) enthalten.

* Wird die Freigabe von Administratoren oder Benutzern verwendet, um eine Verbindung zu den Home-Verzeichnissen anderer Benutzer herzustellen (unter Verwendung von Optionen zum  `vserver cifs home-directory modify` Befehl), muss dem dynamischen Freigabenamensmuster eine Tilde (`~` vorangestellt werden.


Weitere Informationen zu  `vserver cifs share` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+cifs+share["ONTAP-Befehlsreferenz"^].
