= ONTAP SMB-Qtrees erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen Qtree erstellen, um Ihre Daten zu speichern, und seine Eigenschaften mithilfe des  `volume qtree create` Befehls angeben.

.Bevor Sie beginnen
* Die SVM und das Volume, das den neuen Qtree enthalten soll, müssen bereits existieren.
* Der SVM-Sicherheitsstil muss NTFS sein, und SMB muss eingerichtet und funktionsfähig sein.


.Schritte
. Den Qtree erstellen: `volume qtree create -vserver vserver_name { -volume volume_name -qtree qtree_name | -qtree-path qtree path } -security-style ntfs`
+
Sie können das Volume und den Qtree als separate Argumente angeben oder das Qtree-Pfadargument im Format  `/vol/volume_name/_qtree_name` angeben.

. Es sollte überprüft werden, ob der Qtree mit dem gewünschten Verbindungspfad erstellt wurde: `volume qtree show -vserver vserver_name { -volume volume_name -qtree qtree_name | -qtree-path qtree path }`


.Beispiel
Das folgende Beispiel erstellt einen Qtree namens qt01 auf SVM vs1.example.com, der einen Verbindungspfad  `/vol/data1` hat:

[listing]
----
cluster1::> volume qtree create -vserver vs1.example.com -qtree-path /vol/data1/qt01 -security-style ntfs
[Job 1642] Job succeeded: Successful

cluster1::> volume qtree show -vserver vs1.example.com -qtree-path /vol/data1/qt01

                      Vserver Name: vs1.example.com
                       Volume Name: data1
                        Qtree Name: qt01
  Actual (Non-Junction) Qtree Path: /vol/data1/qt01
                    Security Style: ntfs
                       Oplock Mode: enable
                  Unix Permissions: ---rwxr-xr-x
                          Qtree Id: 2
                      Qtree Status: normal
                     Export Policy: default
        Is Export Policy Inherited: true
----
