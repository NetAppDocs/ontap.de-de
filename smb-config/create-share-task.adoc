= ONTAP SMB-Freigaben erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen eine SMB-Freigabe erstellen, bevor Sie Daten von einem SMB-Server mit SMB-Clients teilen können. Beim Erstellen einer Freigabe können Sie deren Eigenschaften festlegen, beispielsweise die Freigabe als Home-Verzeichnis bestimmen. Die Freigabe kann auch durch die Konfiguration optionaler Einstellungen angepasst werden.

.Bevor Sie beginnen
Der Verzeichnispfad für das Volume oder den Qtree muss im SVM-Namensraum vorhanden sein, bevor die Freigabe erstellt wird.

.Über diese Aufgabe
Beim Erstellen einer Freigabe ist die Standard-Share-ACL (Standard-Share-Berechtigungen)  `Everyone / Full Control`. Nach dem Testen des Zugriffs auf die Freigabe sollte die Standard-Share-ACL entfernt und durch eine sicherere Alternative ersetzt werden.

.Schritte
. Erstellen Sie bei Bedarf die Verzeichnispfadstruktur für die Freigabe.
+
Der  `vserver cifs share create`Befehl prüft den im  `-path`angegebenen Pfad während der Erstellung der Freigabe. Existiert der angegebene Pfad nicht, schlägt der Befehl fehl.

. Eine SMB-Freigabe wird erstellt, die dem angegebenen SVM zugeordnet ist: `+vserver cifs share create -vserver vserver_name -share-name share_name -path path [-share-properties share_properties,...] [other_attributes] [-comment text]+`
. Es sollte überprüft werden, ob die Freigabe erstellt wurde:``vserver cifs share show -share-name share_name``


.Beispiele
Der folgende Befehl erstellt eine SMB-Freigabe mit dem Namen "`SHARE1`" auf SVM `vs1.example.com`. Ihr Verzeichnispfad ist `/users`, und sie wird mit Standardeigenschaften erstellt.

[listing]
----
cluster1::> vserver cifs share create -vserver vs1.example.com -share-name SHARE1 -path /users

cluster1::> vserver cifs share show -share-name SHARE1

Vserver          Share    Path     Properties Comment  ACL
---------------  -------- -------- ---------- -------- -----------
vs1.example.com  SHARE1   /users   oplocks    -        Everyone / Full Control
                                   browsable
                                   changenotify
                                   show-previous-versions
----
