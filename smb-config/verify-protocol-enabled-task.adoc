= Es sollte überprüft werden, ob das SMB-Protokoll auf dem ONTAP SVM aktiviert ist.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie SMB auf SVMs konfigurieren und verwenden können, muss überprüft werden, ob das Protokoll aktiviert ist.

.Über diese Aufgabe
Dies geschieht üblicherweise während der SVM-Einrichtung, aber wenn das Protokoll während der Einrichtung nicht aktiviert wurde, kann es später mit dem  `vserver add-protocols` Befehl aktiviert werden.

[NOTE]
====
Sobald ein LIF erstellt wurde, kann kein Protokoll mehr hinzugefügt oder entfernt werden.

====
Protokolle auf SVMs lassen sich auch mit dem  `vserver remove-protocols` Befehl deaktivieren.

.Schritte
. Es kann überprüft werden, welche Protokolle für die SVM aktuell aktiviert und deaktiviert sind: `vserver show -vserver vserver_name -protocols`
+
Der  `vserver show-protocols`Befehl kann ebenfalls verwendet werden, um die aktuell auf allen SVMs im Cluster aktivierten Protokolle anzuzeigen.

. Gegebenenfalls kann ein Protokoll aktiviert oder deaktiviert werden:
+
** Um das SMB-Protokoll zu aktivieren: `vserver add-protocols -vserver vserver_name -protocols cifs`
** Ein Protokoll kann wie folgt deaktiviert werden: `+vserver remove-protocols -vserver vserver_name -protocols protocol_name[,protocol_name,...]+`


. Bestätigen Sie, dass die aktivierten und deaktivierten Protokolle korrekt aktualisiert wurden: `vserver show -vserver vserver_name -protocols`


.Beispiel
Der folgende Befehl zeigt, welche Protokolle auf der SVM mit dem Namen vs1 aktuell aktiviert und deaktiviert (zugelassen und nicht zugelassen) sind:

[listing]
----
vs1::> vserver show -vserver vs1.example.com -protocols
Vserver           Allowed Protocols         Disallowed Protocols
-----------       ----------------------    -----------------------
vs1.example.com   cifs                       nfs, fcp, iscsi, ndmp
----
Der folgende Befehl ermöglicht den Zugriff über SMB, indem  `cifs` zur Liste der aktivierten Protokolle auf der SVM mit dem Namen vs1 hinzugefügt wird:

[listing]
----
vs1::> vserver add-protocols -vserver vs1.example.com -protocols cifs
----
