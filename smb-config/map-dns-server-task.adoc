= ONTAP SMB-Server auf dem DNS-Server zuordnen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der DNS-Server Ihrer Site muss einen Eintrag enthalten, der den SMB-Servernamen und alle NetBIOS-Aliase auf die IP-Adresse des Daten-LIF verweist, sodass Windows-Benutzer ein Laufwerk dem SMB-Servernamen zuordnen können.

.Bevor Sie beginnen
Sie benötigen Administratorzugriff auf den DNS-Server Ihrer Website. Falls Sie keinen Administratorzugriff besitzen, ist der DNS-Administrator zu bitten, diese Aufgabe zu übernehmen.

.Über diese Aufgabe
Wenn Sie NetBIOS-Aliase für den SMB-Servernamen verwenden, ist es eine bewährte Methode, für jeden Alias einen DNS-Server-Eintragspunkt zu erstellen.

.Schritte
. Melden Sie sich beim DNS Server an.
. Vorwärts- (A - Adressdatensatz) und Rückwärts-Lookup-Einträge (PTR - Zeigerdatensatz) werden erstellt, um den SMB-Servernamen der IP-Adresse des Daten-LIF zuzuordnen.
. Wenn Sie NetBIOS-Aliase verwenden, ist ein Alias-Canonical-Name-Lookup-Eintrag (CNAME-Ressourceneintrag) zu erstellen, um jeden Alias der IP-Adresse der Daten-LIF des SMB-Servers zuzuordnen.


.Ergebnisse
Nachdem die Zuordnung im Netzwerk verbreitet wurde, können Windows-Benutzer ein Laufwerk dem SMB-Servernamen oder seinen NetBIOS-Aliasnamen zuordnen.
