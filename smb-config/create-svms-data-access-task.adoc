= ONTAP SVMs erstellen, um SMB-Datenzugriff bereitzustellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn sich in einem Cluster noch keine SVM befindet, die SMB-Clients den Datenzugriff bereitstellt, ist eine SVM zu erstellen.

.Bevor Sie beginnen
* Ab ONTAP 9.13.1 kann eine maximale Kapazität für eine Storage-VM festgelegt werden. Es besteht außerdem die Möglichkeit, Warnmeldungen zu konfigurieren, wenn die SVM einen Schwellenwert für die Kapazität erreicht. Weitere Informationen finden sich unter xref:../volumes/manage-svm-capacity.html[SVM Kapazität verwalten].


.Schritte
. Eine SVM erstellen: `vserver create -vserver _svm_name_ -rootvolume _root_volume_name_ -aggregate _aggregate_name_ -rootvolume-security-style ntfs -language C.UTF-8 -ipspace _ipspace_name_`
+
** Die NTFS-Einstellung für die  `-rootvolume-security-style` Option verwenden.
** Die Standardoption C.UTF-8 `-language` kann verwendet werden.
** Die  `ipspace` Einstellung ist optional.


. Die Konfiguration und der Status der neu erstellten SVM können überprüft werden: `vserver show -vserver _vserver_name_`
+
Das `Allowed Protocols` Feld muss CIFS enthalten. Diese Liste kann später bearbeitet werden.

+
Das  `Vserver Operational State`Feld muss den  `running`Status anzeigen. Wenn es den  `initializing`Status anzeigt, bedeutet dies, dass ein Zwischenvorgang wie die Erstellung des Root-Volumes fehlgeschlagen ist und die SVM gelöscht und neu erstellt werden muss.



.Beispiele
Der folgende Befehl erstellt eine SVM für den Datenzugriff im IPspace `ipspaceA`:

[listing]
----
cluster1::> vserver create -vserver vs1.example.com -rootvolume root_vs1 -aggregate aggr1
-rootvolume-security-style ntfs -language C.UTF-8 -ipspace ipspaceA

[Job 2059] Job succeeded:
Vserver creation completed
----
Der folgende Befehl zeigt, dass eine SVM mit einem Root-Volume von 1 GB erstellt wurde, das automatisch gestartet wurde und sich im `running` Status befindet. Das Root-Volume verfügt über eine Standard-Export-Richtlinie, die keine Regeln enthält, sodass das Root-Volume bei der Erstellung nicht exportiert wird.

NOTE: Ab ONTAP 9.13.1 kann eine adaptive QoS-Richtliniengruppenvorlage festgelegt werden, die eine Durchsatzuntergrenze und eine Obergrenze für Volumes in der SVM anwendet. Diese Richtlinie kann erst nach der Erstellung der SVM angewendet werden. Weitere Informationen zu diesem Vorgang finden sich unter xref:../performance-admin/adaptive-policy-template-task.html[Eine adaptive Richtliniengruppenvorlage festlegen].
