= Subnetze aus dem ONTAP Netzwerk löschen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn ein Subnetz nicht mehr benötigt wird und die diesem Subnetz zugewiesenen IP-Adressen freigegeben werden sollen, kann es gelöscht werden.

Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Ab ONTAP 9.12.0 kann mit dem System Manager ein Subnetz gelöscht werden*

.Schritte
. *Netzwerk > Übersicht > Subnetze* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Löschen* neben dem Subnetz, das entfernt werden soll.
. Speichern Sie Ihre Änderungen.


--
.CLI
--
*Die CLI kann zum Löschen eines Subnetzes verwendet werden*

.Über diese Aufgabe
Sie erhalten eine Fehlermeldung, wenn ein Service-Prozessor oder Netzwerkschnittstellen derzeit IP-Adressen aus den angegebenen Bereichen verwenden. Wenn die Schnittstellen die IP-Adressen auch nach dem Löschen des Subnetzes weiterhin verwenden sollen, kann die Option -force-update-lif-associations auf true gesetzt werden, um die Zuordnung des Subnetzes zu den LIFs aufzuheben.

.Schritt
Ein Subnetz löschen:

`network subnet delete -subnet-name subnet_name [-ipspace ipspace_name] [-force-update-lif- associations true]`

Der folgende Befehl löscht das Subnetz sub1 im IPspace ipspace1:

`network subnet delete -subnet-name sub1 -ipspace ipspace1`

Weitere Informationen zu  `network subnet delete` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-delete.html["ONTAP-Befehlsreferenz"^].
