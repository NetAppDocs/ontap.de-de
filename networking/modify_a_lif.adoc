= ONTAP LIFs modifizieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können eine LIF modifizieren, indem Sie Attribute wie Home-Node oder aktuellen Node, administrativen Status, IP-Adresse, Netzmaske, Failover-Policy, Firewall-Policy und Service-Policy ändern. Es ist auch möglich, die Address Family einer LIF von IPv4 auf IPv6 zu ändern.

.Über diese Aufgabe
* Wenn der administrative Status eines LIF auf „down“ geändert wird, bleiben alle ausstehenden NFSv4-Sperren so lange bestehen, bis der administrative Status des LIF wieder auf „up“ gesetzt wird.
+
Um Sperrkonflikte zu vermeiden, die auftreten können, wenn andere LIFs versuchen, auf die gesperrten Dateien zuzugreifen, müssen die NFSv4-Clients auf eine andere LIF verschoben werden, bevor der administrative Status auf „down“ gesetzt wird.

* Die von einem FC LIF verwendeten Datenprotokolle können nicht geändert werden. Es besteht jedoch die Möglichkeit, die einer Dienstrichtlinie zugewiesenen Dienste zu ändern oder die einer IP LIF zugewiesene Dienstrichtlinie zu ändern.
+
Um die von einem FC-LIF verwendeten Datenprotokolle zu ändern, muss der LIF gelöscht und neu erstellt werden. Bei Änderungen der Dienstrichtlinien eines IP-LIF kommt es während der Aktualisierung zu einer kurzen Ausfallzeit.

* Sie können weder den Heimatknoten noch den aktuellen Knoten eines node-scope Management-LIF ändern.
* Wenn ein Subnetz verwendet wird, um die IP-Adresse und den Netzwerkmaskenwert für ein LIF zu ändern, wird eine IP-Adresse aus dem angegebenen Subnetz zugewiesen; wenn die vorherige IP-Adresse des LIF aus einem anderen Subnetz stammt, wird die IP-Adresse an dieses Subnetz zurückgegeben.
* Um die Adressfamilie eines LIF von IPv4 auf IPv6 zu ändern, muss die Doppelpunktnotation für die IPv6-Adresse verwendet und ein neuer Wert für den  `-netmask-length` Parameter hinzugefügt werden.
* Die automatisch konfigurierten Link-Local-IPv6-Adressen können nicht geändert werden.
* Eine Änderung an einer logischen Schnittstelle, die dazu führt, dass die logische Schnittstelle kein gültiges Failover-Ziel mehr hat, führt zu einer Warnmeldung.
+
Wenn ein LIF, das kein gültiges Failover-Ziel hat, versucht, ein Failover durchzuführen, kann es zu einem Ausfall kommen.

* Ab ONTAP 9.5 ist es möglich, die einer LIF zugeordnete Servicerichtlinie zu ändern.
+
In ONTAP 9.5 werden Service-Richtlinien nur für Intercluster- und BGP-Peer-Dienste unterstützt. In ONTAP 9.6 ist das Erstellen von Service-Richtlinien für verschiedene Daten- und Verwaltungsdienste möglich.

* Ab ONTAP 9.11.1 ist das automatische iSCSI LIF Failover auf All-Flash SAN Array (ASA) Plattformen verfügbar.
+
Für bereits vorhandene iSCSI LIFs, das heißt LIFs, die vor dem Upgrade auf 9.11.1 oder höher erstellt wurden, lässt sich die Failover-Richtlinie auf link:../san-admin/asa-iscsi-lif-fo-task.html["Automatisches iSCSI LIF Failover aktivieren"] ändern.

* ONTAP verwendet das Network Time Protocol (NTP) zur Zeitsynchronisierung im Cluster. Nach einer Änderung der LIF-IP-Adressen kann es erforderlich sein, die NTP-Konfiguration zu aktualisieren, um Synchronisierungsfehler zu vermeiden. Weitere Informationen enthält die link:https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-Issues/CONTAP-500629["NetApp Knowledge Base: NTP-Synchronisierung schlägt nach LIF-IP-Änderung fehl"^].


Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Ab ONTAP 9.12.0 ist es möglich, den System Manager zum Bearbeiten einer Netzwerkschnittstelle zu verwenden.*

.Schritte
. *Netzwerk > Übersicht > Netzwerkschnittstellen* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Bearbeiten* neben der Netzwerkschnittstelle, die geändert werden soll.
. Ändern Sie eine oder mehrere der Netzwerkschnittstelleneinstellungen. Weitere Details finden sich unter link:create_a_lif.html["Eine LIF erstellen"].
. Speichern Sie Ihre Änderungen.


--
.CLI
--
*Die CLI kann verwendet werden, um eine LIF zu ändern*

.Schritte
. Die Attribute einer LIF können mit dem  `network interface modify` Befehl geändert werden.
+
Das folgende Beispiel zeigt, wie die IP-Adresse und die Netzwerkmaske von LIF datalif2 mithilfe einer IP-Adresse und des Netzwerkmaskenwerts aus dem Subnetz client1_sub geändert werden:

+
....
network interface modify -vserver vs1 -lif datalif2 -subnet-name client1_sub
....
+
Das folgende Beispiel zeigt, wie die Service-Richtlinie eines LIF geändert wird.

+
....
network interface modify -vserver siteA -lif node1_inter1 -service-policy example
....
+
Weitere Informationen zu  `network interface modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-modify.html["ONTAP-Befehlsreferenz"^].

. Es wird überprüft, ob die IP-Adressen erreichbar sind.
+
|===


| Wenn Sie ... verwenden | Dann verwenden Sie... 


 a| 
IPv4-Adressen
 a| 
`network ping`



 a| 
IPv6-Adressen
 a| 
`network ping6`

|===
+
Weitere Informationen zu  `network ping` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-ping.html["ONTAP-Befehlsreferenz"^].
