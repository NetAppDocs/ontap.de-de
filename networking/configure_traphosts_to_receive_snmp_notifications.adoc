= Traphosts für SNMP im ONTAP Netzwerk konfigurieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können den Traphost (SNMP-Manager) so konfigurieren, dass er Benachrichtigungen (SNMP-Trap-PDUs) empfängt, wenn SNMP-Traps im Cluster generiert werden. Es kann entweder der Hostname oder die IP-Adresse (IPv4 oder IPv6) des SNMP-Traphost angegeben werden.

.Bevor Sie beginnen
* SNMP und SNMP Traps müssen auf dem Cluster aktiviert sein.



NOTE: SNMP und SNMP Traps sind standardmäßig aktiviert.

* DNS muss auf dem Cluster konfiguriert sein, damit die Traphost-Namen aufgelöst werden können.
* IPv6 muss auf dem Cluster aktiviert sein, um SNMP-Traphosts mit IPv6-Adressen zu konfigurieren.
* Sie müssen die Authentifizierung eines vordefinierten benutzerbasierten Sicherheitsmodells (USM) und die Datenschutzanmeldeinformationen bei der Erstellung von Traphosts angegeben haben.


.Schritt
Einen SNMP Traphost hinzufügen:

....
system snmp traphost add
....

NOTE: Traps können nur gesendet werden, wenn mindestens eine SNMP-Managementstation als Traphost angegeben ist.

Der folgende Befehl fügt einen neuen SNMPv3 Traphost namens yyy.example.com mit einem bekannten USM-Benutzer hinzu:

....
system snmp traphost add -peer-address yyy.example.com -usm-username MyUsmUser
....
Der folgende Befehl fügt einen Traphost unter Verwendung der IPv6-Adresse des Hosts hinzu:

....
system snmp traphost add -peer-address 2001:0db8:1:1:209:6bff:feae:6d67
....
