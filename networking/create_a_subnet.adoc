= Subnetze für das ONTAP Netzwerk erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können ein Subnetz erstellen, um bestimmte Blöcke von IPv4- oder IPv6-Adressen zuzuweisen, die später beim Erstellen von LIFs für die SVM verwendet werden.

Dadurch können Sie LIFs einfacher erstellen, indem Sie einen Subnetznamen angeben, anstatt für jede LIF IP-Adresse und Netzmaske angeben zu müssen.

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

Die Broadcast-Domäne und der IPspace, in dem das Subnetz hinzugefügt werden soll, müssen bereits existieren.

.Über diese Aufgabe
* Alle Subnetznamen müssen innerhalb eines IPspace eindeutig sein.
* Beim Hinzufügen von IP-Adressbereichen zu einem Subnetz muss sichergestellt werden, dass es keine sich überschneidenden IP-Adressen im Netzwerk gibt, damit unterschiedliche Subnetze oder Hosts nicht versuchen, dieselbe IP-Adresse zu verwenden.
* Wenn Sie beim Definieren eines Subnetzes ein Gateway angeben, wird beim Erstellen einer LIF mit diesem Subnetz automatisch eine Standardroute zu diesem Gateway zum SVM hinzugefügt. Wenn Sie keine Subnetze verwenden oder beim Definieren eines Subnetzes kein Gateway angeben, müssen Sie mit dem  `route create` Befehl eine Route manuell zum SVM hinzufügen.
* NetApp empfiehlt, Subnetzobjekte für alle LIFs auf Daten-SVMs zu erstellen. Dies ist besonders wichtig für MetroCluster Konfigurationen, da das Subnetzobjekt ONTAP ermöglicht, Failover-Ziele auf dem Ziel-Cluster zu bestimmen, da jedem Subnetzobjekt eine zugehörige Broadcast-Domäne zugeordnet ist.


.Schritte
Sie können ein Subnetz mit ONTAP System Manager oder der ONTAP CLI erstellen.

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.0 ist es möglich, mit System Manager ein Subnetz zu erstellen.

.Schritte
. *Netzwerk > Übersicht > Subnetze* auswählen.
. Klicken Sie auf image:icon_add.gif["Symbol hinzufügen"], um ein Subnetz zu erstellen.
. Das Subnetz benennen.
. Die Subnetz-IP-Adresse angeben.
. Die Subnetzmaske festlegen.
. Definieren Sie den Bereich der IP-Adressen, aus denen das Subnetz besteht.
. Falls sinnvoll kann ein Gateway angegeben werden.
. Wählen Sie die Broadcast-Domäne aus, zu der das Subnetz gehört.
. Speichern Sie Ihre Änderungen.
+
.. Wird die eingegebene IP-Adresse oder der IP-Adressbereich bereits von einer Schnittstelle verwendet, wird folgende Meldung angezeigt:
`An IP address in this range is already in use by a LIF. Associate the LIF with this subnet?`
.. Wenn auf *OK* geklickt wird, wird die bestehende LIF dem Subnetz zugeordnet.




--
.CLI
--
Mit der CLI ein Subnetz erstellen.

....
network subnet create -subnet-name subnet_name -broadcast-domain <broadcast_domain_name> [- ipspace <ipspace_name>] -subnet <subnet_address> [-gateway <gateway_address>] [-ip-ranges <ip_address_list>] [-force-update-lif-associations <true>]
....
* `subnet_name` ist der Name des Layer-3-Subnetzes, das Sie erstellen möchten.
+
Der Name kann ein Text-String wie "Mgmt" oder ein bestimmter Subnetz-IP-Wert wie 192.0.2.0/24 sein.

* `broadcast_domain_name` ist der Name der Broadcast-Domäne, in der sich das Subnetz befinden wird.
* `ipspace_name` ist der Name des IPspace, zu dem die Broadcast-Domäne gehört.
+
Der „Default“ IPspace wird verwendet, sofern Sie für diese Option keinen Wert angeben.

* `subnet_address` ist die IP-Adresse und Maske des Subnetzes; zum Beispiel 192.0.2.0/24.
* `gateway_address` ist das Gateway für die Standardroute des Subnetzes, zum Beispiel 192.0.2.1.
* `ip_address_list` ist die Liste bzw. der Bereich von IP-Adressen, die dem Subnetz zugewiesen werden.
+
Bei den IP-Adressen kann es sich um einzelne Adressen, einen Bereich von IP-Adressen oder eine Kombination in einer durch Kommas getrennten Liste handeln.

* Der Wert `true` kann für die `-force-update-lif-associations` Option festgelegt werden.
+
Dieser Befehl schlägt fehl, wenn ein Dienstprozessor oder eine Netzwerkschnittstelle derzeit die IP-Adressen im angegebenen Bereich verwendet. Wenn dieser Wert auf „true“ gesetzt wird, werden alle manuell adressierten Schnittstellen dem aktuellen Subnetz zugeordnet und der Befehl kann erfolgreich ausgeführt werden.



Der folgende Befehl erstellt das Subnetz sub1 in der Broadcast-Domäne Default-1 im Default-IPspace. Es werden eine IPv4-Subnetz-IP-Adresse und -Maske, das Gateway sowie ein Bereich von IP-Adressen hinzugefügt:

....
network subnet create -subnet-name sub1 -broadcast-domain Default-1 -subnet 192.0.2.0/24 - gateway 192.0.2.1 -ip-ranges 192.0.2.1-192.0.2.100, 192.0.2.122
....
Der folgende Befehl erstellt das Subnetz sub2 in der Broadcast-Domäne Default im IPspace „Default“. Es wird ein Bereich von IPv6-Adressen hinzugefügt:

....
network subnet create -subnet-name sub2 -broadcast-domain Default -subnet 3FFE::/64 - gateway 3FFE::1 -ip-ranges "3FFE::10-3FFE::20"
....
Weitere Informationen zu  `network subnet create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-create.html["ONTAP-Befehlsreferenz"^].

.Nachdem Sie fertig sind
Sie können SVMs und Schnittstellen einem IPspace zuweisen, indem Sie die Adressen im Subnetz verwenden.

Wenn der Name eines bestehenden Subnetzes geändert werden soll, ist der  `network subnet rename` Befehl zu verwenden.

Weitere Informationen zu  `network subnet rename` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-rename.html["ONTAP-Befehlsreferenz"^].
