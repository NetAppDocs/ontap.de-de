= Einen ONTAP LIF zu einer Load-Balancing-Zone hinzufügen oder daraus entfernen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können einer virtuellen Maschine (SVM) eine LIF zur DNS-Lastverteilungszone hinzufügen oder daraus entfernen. Es ist auch möglich, alle LIFs gleichzeitig aus einer Lastverteilungszone zu entfernen.

.Bevor Sie beginnen
* Alle LIFs in einer Load Balancing-Zone sollten zur selben SVM gehören.
* Ein LIF kann nur Teil einer einzigen DNS Load-Balancing-Zone sein.
* Für jedes Subnetz muss eine Failover-Gruppe eingerichtet worden sein, falls die LIFs zu verschiedenen Subnetzen gehören.


.Über diese Aufgabe
Ein LIF, der sich im administrativen Status „down“ befindet, wird vorübergehend aus der DNS Load Balancing Zone entfernt. Wenn das LIF wieder den administrativen Status „up“ erreicht, wird es automatisch der DNS Load Balancing Zone hinzugefügt.

.Schritt
Einen LIF zu einer Lastverteilungszone hinzufügen oder einen LIF daraus entfernen:

[cols="30,70"]
|===


| Wenn Sie möchten... | Eingeben... 


 a| 
Eine LIF hinzufügen
 a| 
`network interface modify -vserver _vserver_name_ -lif _lif_name_ -dns-zone _zone_name_`Beispiel:
`network interface modify -vserver vs1 -lif data1 -dns-zone cifs.company.com`



 a| 
Einen einzelnen LIF entfernen
 a| 
`network interface modify -vserver _vserver_name_ -lif _lif_name_ -dns-zone none` Beispiel:  `network interface modify -vserver vs1 -lif data1 -dns-zone none`



 a| 
Alle LIFs entfernen
 a| 
`network interface modify -vserver _vserver_name_ -lif * -dns-zone none`Beispiel:
`network interface modify -vserver vs0 -lif * -dns-zone none` Es ist möglich, eine SVM aus einer Load-Balancing-Zone zu entfernen, indem alle LIFs in der SVM aus dieser Zone entfernt werden.

|===
