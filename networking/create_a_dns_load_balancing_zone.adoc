= DNS-Lastverteilungszonen für das ONTAP Netzwerk erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können eine DNS-Lastverteilungszone erstellen, um die dynamische Auswahl eines LIF basierend auf der Auslastung, das heißt der Anzahl der an ein LIF angebundenen Clients, zu ermöglichen. Eine Lastverteilungszone kann beim Erstellen eines Daten LIF erstellt werden.

.Bevor Sie beginnen
Der DNS-Forwarder auf dem standortweiten DNS-Server muss so konfiguriert sein, dass alle Anfragen für die Load Balancing Zone an die konfigurierten LIFs weitergeleitet werden.

Der link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/How_to_set_up_DNS_load_balancing_in_clustered_Data_ONTAP["NetApp Knowledge Base: Einrichten des DNS-Lastenausgleichs in Clustered Data ONTAP"^]enthält weitere Informationen zur Konfiguration des DNS-Lastausgleichs mittels bedingter Weiterleitung.

.Über diese Aufgabe
* Jede Daten-LIF kann auf DNS-Anfragen für einen DNS Load Balancing-Zonennamen reagieren.
* Eine DNS Load Balancing Zone muss im Cluster einen eindeutigen Namen haben, und der Zonenname muss die folgenden Anforderungen erfüllen:
+
** Es sollte 256 Zeichen nicht überschreiten.
** Es sollte mindestens einen Punkt enthalten.
** Das erste und das letzte Zeichen dürfen kein Punkt oder ein anderes Sonderzeichen sein.
** Zwischen den Zeichen dürfen keine Leerzeichen enthalten sein.
** Jede Bezeichnung im DNS-Namen darf 63 Zeichen nicht überschreiten.
+
Eine Bezeichnung ist der Text, der vor oder nach dem Punkt steht. Beispielsweise hat die DNS-Zone storage.company.com drei Bezeichnungen.





.Schritt
Mit dem  `network interface create` Befehl und der  `dns-zone` Option lässt sich eine DNS Load Balancing Zone erstellen. Weitere Informationen zu  `network interface create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-create.html["ONTAP-Befehlsreferenz"^].

Falls die Lastverteilungszone bereits existiert, wird die LIF hinzugefügt.

Das folgende Beispiel veranschaulicht, wie eine DNS Load-Balancing-Zone mit dem Namen storage.company.com beim Erstellen des LIF erstellt wird `lif1`:

....
network interface create -vserver vs0 -lif lif1 -home-node node1
-home-port e0c -address 192.0.2.129 -netmask 255.255.255.128 -dns-zone storage.company.com
....
