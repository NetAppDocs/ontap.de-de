= ONTAP Clusterverbindungen anzeigen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können alle aktiven Verbindungen im Cluster oder die Anzahl der aktiven Verbindungen auf dem Knoten nach Client, logischer Schnittstelle, Protokoll oder Dienst anzeigen. Sie können außerdem alle lauschenden Verbindungen im Cluster anzeigen.



[[display-active-connections-by-client-cluster-administrators-only]]
== Aktive Verbindungen nach Client anzeigen (nur für Cluster-Administratoren)

Sie können die aktiven Verbindungen pro Client anzeigen, um zu überprüfen, welchen Knoten ein bestimmter Client verwendet, und um mögliche Ungleichgewichte zwischen der Anzahl der Clients pro Knoten zu erkennen.

.Über diese Aufgabe
Die Anzahl der aktiven Verbindungen pro Client ist in folgenden Szenarien nützlich:

* Einen ausgelasteten oder überlasteten Knoten erkennen.
* Ermittlung der Ursache für den langsamen Zugriff eines bestimmten Clients auf ein Volume.
+
Sie können Details zum Knoten, auf den der Client zugreift, anzeigen und diesen mit dem Knoten vergleichen, auf dem sich das Volume befindet. Wenn der Zugriff auf das Volume das Durchlaufen des Cluster-Netzwerks erfordert, kann es bei Clients aufgrund des Fernzugriffs auf das Volume auf einem überlasteten Remote-Knoten zu Leistungseinbußen kommen.

* Überprüfung, ob alle Knoten gleichermaßen für den Datenzugriff verwendet werden.
* Ermitteln von Clients, die eine unerwartet hohe Anzahl von Verbindungen aufweisen.
* Überprüfung, ob bestimmte Clients eine Verbindung zu einem Node haben.


.Schritt
Die Anzahl der aktiven Verbindungen pro Client auf einem Knoten kann mit dem  `network connections active show-clients` Befehl angezeigt werden.

Weitere Informationen zu  `network connections active show-clients` finden sich in der link:http://docs.netapp.com/us-en/ontap-cli/network-connections-active-show-clients.html["ONTAP-Befehlsreferenz"^].

....
network connections active show-clients
Node    Vserver Name    Client IP Address     Count
------  --------------  -----------------     ------
node0   vs0             192.0.2.253                1
        vs0             192.0.2.252                2
        Cluster         192.10.2.124               5
node1   vs0             192.0.2.250                1
        vs0             192.0.2.252                3
        Cluster         192.10.2.123               4
node2   vs1             customer.example.com       1
        vs1             192.0.2.245                3
        Cluster         192.10.2.122               4
node3   vs1             customer.example.org       1
        vs1             customer.example.net       3
        Cluster         192.10.2.121               4
....


[[display-active-connections-by-protocol-cluster-administrators-only]]
== Aktive Verbindungen nach Protokoll (nur für Cluster-Administratoren) anzeigen

Sie können die Anzahl der aktiven Verbindungen nach Protokoll (TCP oder UDP) auf einem Node anzeigen, um die Nutzung der Protokolle innerhalb des Clusters zu vergleichen.

.Über diese Aufgabe
Die Anzahl der aktiven Verbindungen nach Protokoll ist in folgenden Szenarien nützlich:

* Ermittlung der UDP-Clients, die ihre Verbindung verlieren.
+
Wenn ein Knoten sich seiner Verbindungsgrenze nähert, werden UDP-Clients als erste getrennt.

* Es wird überprüft, dass keine anderen Protokolle verwendet werden.


.Schritt
Die Anzahl der aktiven Verbindungen nach Protokoll auf einem Node kann mit dem  `network connections active show-protocols` Befehl angezeigt werden.

Weitere Informationen zu  `network connections active show-protocols` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-connections-active-show-protocols.html["ONTAP-Befehlsreferenz"^].

....
network connections active show-protocols
Node     Vserver Name  Protocol   Count
-------  ------------  ---------  ------
node0
         vs0           UDP            19
         Cluster       TCP            11
node1
         vs0           UDP            17
         Cluster       TCP             8
node2
         vs1           UDP            14
         Cluster       TCP            10
node3
         vs1           UDP            18
         Cluster       TCP             4
....


[[display-active-connections-by-service-cluster-administrators-only]]
== Aktive Verbindungen nach Dienst (nur für Clusteradministratoren) anzeigen

Sie können die Anzahl der aktiven Verbindungen nach Diensttyp (z. B. NFS, SMB, Mount usw.) für jeden Node in einem Cluster anzeigen. Dies ist nützlich, um die Nutzung der Dienste innerhalb des Clusters zu vergleichen, was dabei hilft, die primäre Arbeitslast eines Node zu bestimmen.

.Über diese Aufgabe
Die Anzahl der aktiven Verbindungen pro Dienst ist in folgenden Szenarien nützlich:

* Überprüfung, ob alle Knoten für die entsprechenden Dienste genutzt werden und ob der Lastausgleich für diesen Dienst funktioniert.
* Es wird überprüft, dass keine anderen Dienste verwendet werden. Eine Anzeige der Anzahl der aktiven Verbindungen pro Dienst auf einem Node erfolgt mit dem  `network connections active show-services` Befehl.


Weitere Informationen zu  `network connections active show-services` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-connections-active-show-services.html["ONTAP-Befehlsreferenz"^].

           vs0             cifs_srv        3

           Cluster         ctlopcp        60

[[display-active-connections-by-lif-on-a-node-and-svm]]
== Aktive Verbindungen nach LIF auf einem Node und einer SVM anzeigen

Sie können die Anzahl der aktiven Verbindungen für jede LIF, nach Node und Storage Virtual Machine (SVM), anzeigen, um Verbindungsungleichgewichte zwischen den LIFs innerhalb des Clusters zu sehen.

.Über diese Aufgabe
Die Anzahl der aktiven Verbindungen nach LIF ist in folgenden Szenarien nützlich:

* Ermittlung einer überlasteten LIF durch Vergleich der Anzahl der Verbindungen auf jeder LIF.
* Überprüfung, ob der DNS-Lastausgleich für alle Daten-LIFs funktioniert.
* Vergleich der Anzahl der Verbindungen zu den verschiedenen SVMs, um die SVMs zu identifizieren, die am häufigsten verwendet werden.


.Schritt
Eine Anzeige der Anzahl aktiver Verbindungen für jedes LIF nach SVM und Node erfolgt mit dem  `network connections active show-lifs` Befehl.

Weitere Informationen zu  `network connections active show-lifs` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-connections-active-show-lifs.html["ONTAP-Befehlsreferenz"^].

....
network connections active show-lifs
Node      Vserver Name  Interface Name  Count
--------  ------------  --------------- ------
node0
          vs0           datalif1             3
          Cluster       node0_clus_1         6
          Cluster       node0_clus_2         5
node1
          vs0           datalif2             3
          Cluster       node1_clus_1         3
          Cluster       node1_clus_2         5
node2
          vs1           datalif2             1
          Cluster       node2_clus_1         5
          Cluster       node2_clus_2         3
node3
          vs1           datalif1             1
          Cluster       node3_clus_1         2
          Cluster       node3_clus_2         2
....


[[display-active-connections-in-a-cluster]]
== Aktive Verbindungen in einem Cluster anzeigen

Informationen über die aktiven Verbindungen in einem Cluster können angezeigt werden, um die LIF, den Port, den Remote-Host, den Dienst, die Storage Virtual Machines (SVMs) und das von den einzelnen Verbindungen verwendete Protokoll zu sehen.

.Über diese Aufgabe
Die Anzeige der aktiven Verbindungen in einem Cluster ist in folgenden Szenarien nützlich:

* Überprüfung, ob die einzelnen Clients das richtige Protokoll und den richtigen Dienst auf dem richtigen Node verwenden.
* Wenn ein Client Probleme beim Zugriff auf Daten mit einer bestimmten Kombination aus Node, Protokoll und Service hat, kann mit diesem Befehl ein ähnlicher Client zum Vergleich der Konfiguration oder der Paketverfolgung gefunden werden.


.Schritt
Die aktiven Verbindungen in einem Cluster werden mit dem  `network connections active show` Befehl angezeigt.

Weitere Informationen zu  `network connections active show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-connections-active-show.html["ONTAP-Befehlsreferenz"^].

Der folgende Befehl zeigt die aktiven Verbindungen auf dem Knoten node1 an:

....
network connections active show -node node1

-------  ------------------  ------------------  ----------------

Cluster  node1_clus_1:13387  192.0.2.253:7700    TCP/ctlopcp
Cluster  node1_clus_1:8340   192.0.2.252:7700    TCP/ctlopcp
Cluster  node1_clus_1:42766  192.0.2.252:7700    TCP/ctlopcp
Cluster  node1_clus_1:36119  192.0.2.250:7700    TCP/ctlopcp
vs1      data1:111           host1.aa.com:10741  UDP/port-map

vs1      data1:111           host1.aa.com:12017  UDP/port-map
vs3      data2:111           host1.aa.com:12017  UDP/port-map
....
Der folgende Befehl zeigt die aktiven Verbindungen auf SVM vs1:

....
network connections active show -vserver vs1

....


[[display-listening-connections-in-a-cluster]]
== Abhörverbindungen in einem Cluster anzeigen

Sie können Informationen über die lauschenden Verbindungen in einem Cluster anzeigen, um die LIFs und Ports zu sehen, die Verbindungen für ein bestimmtes Protokoll und einen bestimmten Dienst akzeptieren.

.Über diese Aufgabe
Das Anzeigen der lauschenden Verbindungen in einem Cluster ist in folgenden Szenarien nützlich:

* Überprüfung, ob das gewünschte Protokoll oder der gewünschte Dienst auf einem LIF lauscht, falls Clientverbindungen zu diesem LIF wiederholt fehlschlagen.
* Es wird überprüft, ob an jedem Cluster-LIF ein UDP/rclopcp-Listener geöffnet ist, falls der Remote-Datenzugriff auf ein Volume auf einem Node über eine LIF auf einem anderen Node fehlschlägt.
* Überprüfung, ob an jedem Cluster-LIF ein UDP/rclopcp-Listener geöffnet ist, falls SnapMirror-Übertragungen zwischen zwei Nodes im selben Cluster fehlschlagen.
* Überprüfung, ob an jeder Intercluster-LIF ein TCP/ctlopcp-Listener geöffnet ist, wenn SnapMirror-Übertragungen zwischen zwei Nodes in verschiedenen Clustern fehlschlagen.


.Schritt
Die Listening-Verbindungen pro Node werden mit dem  `network connections listening show`-Befehl angezeigt.

....
network connections listening show
Vserver Name     Interface Name:Local Port        Protocol/Service
---------------- -------------------------------  ----------------

vs1              data1:4046                       UDP/sm
vs1              data1:4045                       TCP/nlm-v4
vs1              data1:4045                       UDP/nlm-v4
vs1              data1:2049                       TCP/nfs
vs1              data1:2049                       UDP/nfs
vs1              data1:635                        TCP/mount
vs1              data1:635                        UDP/mount
Cluster          node0_clus_2:7700                TCP/ctlopcp
....
Weitere Informationen zu  `network connections listening show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-connections-listening-show.html["ONTAP-Befehlsreferenz"^].
