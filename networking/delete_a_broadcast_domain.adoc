= ONTAP Broadcast-Domänen löschen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn eine Broadcast-Domäne nicht mehr benötigt wird, kann sie gelöscht werden. Dabei werden die mit dieser Broadcast-Domäne verknüpften Ports in den IPspace „Default“ verschoben.

.Bevor Sie beginnen
Es dürfen keine Subnetze, Netzwerkschnittstellen oder SVMs mit der Broadcast-Domäne verknüpft sein, die gelöscht werden soll.

.Über diese Aufgabe
* Die vom System erstellte Broadcast-Domäne „Cluster“ kann nicht gelöscht werden.
* Alle mit der Broadcast-Domäne verknüpften Failover-Gruppen werden entfernt, wenn die Broadcast-Domäne gelöscht wird.


Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Ab ONTAP 9.12.0 ist es möglich, mit System Manager eine Broadcast-Domäne zu löschen*

Die Option zum Löschen wird nicht angezeigt, wenn die Broadcast-Domäne Ports enthält oder einem Subnetz zugeordnet ist.

.Schritte
. *Netzwerk > Übersicht > Broadcast domain* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Löschen* neben der Broadcast-Domäne, die entfernt werden soll.


--
.CLI
--
*Die CLI kann zum Löschen einer Broadcast-Domäne verwendet werden*

.Schritt
Eine Broadcast-Domäne löschen:

`network port broadcast-domain delete -broadcast-domain _broadcast_domain_name_ [-ipspace _ipspace_name_]`

Der folgende Befehl löscht die Broadcast-Domäne Default-1 im IPspace ipspace1:

`network port broadcast-domain delete -broadcast-domain _Default-1_ -ipspace _ipspace1_`

.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/network-port-broadcast-domain-delete.html["Netzwerkport Broadcast-Domäne löschen"^]
