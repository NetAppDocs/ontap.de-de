= Informationen zu ONTAP Broadcast-Domänen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Broadcast-Domänen dienen dazu, Netzwerkports zu gruppieren, die zum selben Layer-2-Netzwerk gehören. Die Ports in der Gruppe können dann von einer Storage Virtual Machine (SVM) für Daten- oder Verwaltungsdatenverkehr verwendet werden.


NOTE: Die Verwaltung von Broadcast-Domänen unterscheidet sich in ONTAP 9.7 und früheren Versionen. Wenn Broadcast-Domänen in einem Netzwerk mit ONTAP 9.7 oder einer früheren Version verwaltet werden müssen, siehe link:https://docs.netapp.com/us-en/ontap-system-manager-classic/networking-bd/configure_broadcast_domains_97_overview.html["Überblick über die Broadcast-Domäne (ONTAP 9.7 und früher)"^].

Eine Broadcast-Domäne befindet sich in einem IPspace. Während der Clusterinitialisierung erstellt das System zwei Standard-Broadcast-Domänen:

* Die Broadcast-Domäne „Default“ enthält Ports, die sich im „Default“ IPspace befinden.
+
Diese Ports werden primär zur Datenübertragung genutzt. Cluster-Management- und Node-Management-Ports befinden sich ebenfalls in dieser Broadcast-Domäne.

* Die Broadcast-Domäne "Cluster" enthält Ports, die sich im IPspace "Cluster" befinden.
+
Diese Ports werden für die Clusterkommunikation verwendet und umfassen alle Cluster-Ports von allen Knoten im Cluster.

+
Das System erstellt bei Bedarf zusätzliche Broadcast-Domänen im Default-IPspace. Die Broadcast-Domäne „Default“ enthält den Home-Port des Management-LIF sowie alle weiteren Ports, die auf Layer 2 Erreichbarkeit zu diesem Port haben. Zusätzliche Broadcast-Domänen werden mit „Default-1“, „Default-2“ und so weiter benannt.





[[example-of-using-broadcast-domains]]
== Beispiel für die Verwendung von Broadcast-Domänen

Eine Broadcast-Domäne ist eine Gruppe von Netzwerkports im selben IPspace, die auch über Layer-2-Erreichbarkeit zueinander verfügen und typischerweise Ports von vielen Knoten im Cluster umfasst.

Die Abbildung zeigt die Ports, die drei Broadcast-Domänen in einem Cluster mit vier Knoten zugewiesen sind:

* Die Broadcast-Domäne „Cluster“ wird während der Cluster-Initialisierung automatisch erstellt und enthält die Ports a und b von jedem Node im Cluster.
* Die Broadcast-Domäne „Default“ wird ebenfalls automatisch während der Clusterinitialisierung erstellt und enthält die Ports c und d von jedem Knoten im Cluster.
* Das System erstellt während der Clusterinitialisierung automatisch zusätzliche Broadcast-Domänen basierend auf der Layer-2-Netzwerkerreichbarkeit. Diese zusätzlichen Broadcast-Domänen werden Default-1, Default-2 und so weiter genannt.


image:Broadcast_Domains.png["Broadcast Domain-Bild"]

Eine Failover-Gruppe mit demselben Namen und denselben Netzwerkports wie jede der Broadcast-Domänen wird automatisch erstellt. Diese Failover-Gruppe wird vom System automatisch verwaltet, das heißt, wenn Ports zur Broadcast-Domäne hinzugefügt oder daraus entfernt werden, werden sie auch automatisch zu dieser Failover-Gruppe hinzugefügt oder daraus entfernt.
