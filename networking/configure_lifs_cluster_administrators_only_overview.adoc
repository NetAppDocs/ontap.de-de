= Informationen zur LIF Konfiguration für einen ONTAP Cluster
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Eine LIF (logische Schnittstelle) stellt einen Netzwerkzugriffspunkt zu einem Knoten im Cluster dar. LIFs können auf Ports konfiguriert werden, über die der Cluster Kommunikation über das Netzwerk sendet und empfängt.

Ein Clusteradministrator kann LIFs erstellen, anzeigen, ändern, migrieren, zurücksetzen oder löschen. Ein SVM-Administrator kann nur die mit der SVM verknüpften LIFs anzeigen.

Eine LIF ist eine IP-Adresse oder ein WWPN mit zugehörigen Eigenschaften, wie einer Dienstrichtlinie, einem Heimatport, einem Heimatknoten, einer Liste von Ports für das Failover und einer Firewall-Richtlinie. LIFs können auf Ports konfiguriert werden, über die der Cluster Kommunikation über das Netzwerk sendet und empfängt.


NOTE: Ab ONTAP 9.10.1 sind Firewall-Richtlinien veraltet und vollständig durch LIF-Dienstrichtlinien ersetzt. Weitere Informationen finden sich unter link:../networking/configure_firewall_policies_for_lifs.html["Firewall-Richtlinien für LIFs konfigurieren"].

LIFs können auf den folgenden Ports gehostet werden:

* Physische Ports, die nicht Teil von Schnittstellengruppen sind
* Schnittstellengruppen
* VLANs
* Physische Ports oder Schnittstellengruppen, die VLANs hosten
* Virtuelle IP-Ports (VIP)
+
Ab ONTAP 9.5 werden VIP LIFs unterstützt und auf VIP Ports gehostet.



Bei der Konfiguration von SAN-Protokollen wie FC auf einer logischen Schnittstelle (LIF) wird diese einem WWPN zugeordnet.

== LIF-Failover und Giveback

Ein LIF-Failover tritt auf, wenn ein LIF von seinem Heimatknoten oder Port zu seinem HA-Partnerknoten oder Port wechselt. Ein LIF-Failover kann automatisch von ONTAP oder manuell von einem Clusteradministrator bei bestimmten Ereignissen ausgelöst werden, wie etwa bei einer ausgefallenen physischen Ethernet-Verbindung oder wenn ein Knoten das Quorum der replizierten Datenbank (RDB) verlässt. Wenn ein LIF-Failover auftritt, setzt ONTAP den normalen Betrieb auf dem Partnerknoten fort, bis der Grund für das Failover behoben ist. Wenn der Heimatknoten oder Port wieder funktionsfähig ist, wird das LIF vom HA-Partner zurück zu seinem Heimatknoten oder Port verschoben. Diese Rückverlagerung wird als Giveback bezeichnet.

Für LIF-Failover und Giveback müssen die Ports jedes Knotens zur selben Broadcast-Domäne gehören. Um zu überprüfen, ob die relevanten Ports auf jedem Knoten zur selben Broadcast-Domäne gehören, siehe Folgendes:

* ONTAP 9.8 und höher: link:../networking/repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"]
* ONTAP 9.7 und älter: link:https://docs.netapp.com/us-en/ontap-system-manager-classic/networking-bd/add_or_remove_ports_from_a_broadcast_domain97.html["Ports zu einer Broadcast-Domäne hinzufügen oder daraus entfernen"^]


Für LIFs mit aktiviertem LIF-Failover (entweder automatisch oder manuell) gilt Folgendes:

* Für LIFs, die eine Datendienstrichtlinie verwenden, lassen sich die Einschränkungen der Failover-Richtlinie prüfen:
+
** ONTAP 9.6 und höher: link:lifs_and_service_policies96.html["LIFs und Servicerichtlinien in ONTAP 9.6 und später"]
** ONTAP 9.5 und älter: link:https://docs.netapp.com/us-en/ontap-system-manager-classic/networking/lif_roles95.html["LIF Rollen in ONTAP 9.5 und früheren Versionen"^]


* Die automatische Wiederherstellung von LIFs erfolgt, wenn die automatische Wiederherstellung auf  `true` gesetzt ist und der Heimatport des LIFs fehlerfrei ist und den LIF hosten kann.
* Bei einer geplanten oder ungeplanten Knotenübernahme schlägt das LIF auf dem übernommenen Knoten auf den HA-Partner um. Der Port, auf den das LIF umschlägt, wird vom VIF Manager bestimmt.
* Nach Abschluss des Failover-Vorgangs arbeitet das LIF normal.
* Wenn ein Giveback eingeleitet wird, kehrt der LIF zu seinem Heimatknoten und -port zurück, sofern Auto-Revert auf  `true` eingestellt ist.
* Wenn eine Ethernet-Verbindung an einem Port, der eine oder mehrere LIFs hostet, ausfällt, migriert der VIF Manager die LIFs vom ausgefallenen Port zu einem anderen Port in derselben Broadcast-Domäne. Der neue Port kann sich auf demselben Node oder dessen HA-Partner befinden. Nachdem die Verbindung wiederhergestellt ist und wenn die automatische Rücksetzung auf  `true` gesetzt ist, verschiebt der VIF Manager die LIFs zurück auf ihren Home-Node und Home-Port.
* Wenn ein Knoten das Quorum der replizierten Datenbank (RDB) verlässt, migriert der VIF Manager die LIFs vom aus dem Quorum ausgeschiedenen Knoten zu seinem HA-Partner. Nachdem der Knoten wieder im Quorum ist und wenn auto-revert auf  `true` gesetzt ist, setzt der VIF Manager die LIFs wieder auf ihren Home-Knoten und Home-Port zurück.
