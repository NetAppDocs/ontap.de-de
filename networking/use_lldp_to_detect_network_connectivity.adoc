= LLDP kann zur Erkennung der ONTAP-Netzwerkverbindung verwendet werden.
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die Verwendung von LLDP zur Erkennung der Netzwerkverbindung umfasst die Überprüfung der Bereitstellungsaspekte, das Aktivieren von LLDP auf allen Ports, das Anzeigen benachbarter Geräte und das Anpassen der LLDP-Konfigurationswerte nach Bedarf.

LLDP muss außerdem auf allen Ethernet Switches und Routern aktiviert sein, bevor Informationen über benachbarte Geräte angezeigt werden können.

ONTAP meldet derzeit die folgenden Typ-Längen-Wert-Strukturen (TLVs):

* Chassis-ID
* Port-ID
* Time-To-Live (TTL)
* Systemname
+
Der Systemname TLV wird auf CNA-Geräten nicht gesendet.



Bestimmte konvergente Netzwerkadapter (CNAs), wie der X1143 Adapter und die UTA2 Onboard-Ports, enthalten Offload-Unterstützung für LLDP:

* LLDP Offload wird für Data Center Bridging (DCB) verwendet.
* Die angezeigten Informationen können zwischen dem Cluster und dem Ethernet Switch voneinander abweichen.
+
Die vom Ethernet-Switch angezeigten Chassis-ID- und Port-ID-Daten können für CNA- und Nicht-CNA-Ports unterschiedlich sein.



Beispiel:

* Für Nicht-CNA-Ports:
+
** Die Chassis-ID ist eine feste MAC-Adresse eines der Ports auf dem Node.
** Die Port-ID ist der Portname des jeweiligen Ports auf dem Node.


* Für CNA-Ports:
+
** Chassis-ID und Port-ID sind die MAC-Adressen der jeweiligen Ports am Node.




Die vom Cluster angezeigten Daten sind jedoch für diese Porttypen konsistent.


NOTE: Die LLDP-Spezifikation definiert den Zugriff auf die gesammelten Informationen über eine SNMP-MIB. ONTAP unterstützt die LLDP-MIB jedoch derzeit nicht.



[[enable-or-disable-lldp]]
== LLDP aktivieren oder deaktivieren

Um LLDP-kompatible Nachbargeräte zu erkennen und ihnen Werbung zu senden, muss LLDP auf jedem Knoten des Clusters aktiviert sein. Ab ONTAP 9.7 ist LLDP standardmäßig auf allen Ports eines Knotens aktiviert.

.Über diese Aufgabe
Für ONTAP 9.10.1 und frühere Versionen steuert die  `lldp.enable` Option, ob LLDP an den Ports eines Knotens aktiviert oder deaktiviert ist:

* `on` aktiviert LLDP auf allen Ports.
* `off` deaktiviert LLDP auf allen Ports.


Für ONTAP 9.11.1 und höher steuert die  `lldp.enable` Option, ob LLDP an den Nicht-Cluster- und Nicht-Speicher-Ports eines Knotens aktiviert oder deaktiviert ist:

* `on`LLDP wird auf allen Nicht-Cluster- und Nicht-Speicher-Ports aktiviert.
* `off`deaktiviert LLDP auf allen Nicht-Cluster- und Nicht-Storage-Ports.


.Schritte
. Die aktuelle LLDP-Einstellung für einen Knoten oder für alle Knoten in einem Cluster anzeigen:
+
** Einzelner Knoten: `run -node node_name options lldp.enable`
** Alle Knoten: Optionen `lldp.enable`


. LLDP auf allen Ports eines Knotens oder auf allen Ports aller Knoten in einem Cluster aktivieren oder deaktivieren:
+
[cols="30,70"]
|===


| LLDP kann auf ... aktiviert oder deaktiviert werden. | Eingeben... 


 a| 
Ein Knoten
 a| 
`run -node node_name options lldp.enable {on|off}`



 a| 
Alle Knoten in einem Cluster
 a| 
`options lldp.enable {on|off}`

|===
+
** Einzelner Knoten:
+
....
run -node node_name options lldp.enable {on|off}
....
** Alle Knoten:
+
....
options lldp.enable {on|off}
....






[[view-lldp-neighbor-information]]
== LLDP-Nachbarinformationen anzeigen

Sie können Informationen über die benachbarten Geräte anzeigen, die mit jedem Port der Knoten Ihres Clusters verbunden sind, vorausgesetzt, der Port ist mit einem LLDP-kompatiblen Gerät verbunden. Der Befehl `network device-discovery show` dient zur Anzeige der Nachbarinformationen.

.Schritt
. Informationen über alle LLDP-kompatiblen Geräte, die mit den Ports eines Knotens im Cluster verbunden sind, werden angezeigt:
+
....
network device-discovery show -node node -protocol lldp
....
+
Der folgende Befehl zeigt die Nachbarn an, die mit den Ports des Knotens cluster-1_01 verbunden sind. Die Ausgabe listet die LLDP-fähigen Geräte auf, die mit jedem Port des angegebenen Knotens verbunden sind. Wenn die  `-protocol` Option weggelassen wird, werden auch CDP-fähige Geräte aufgelistet.

+
....
network device-discovery show -node cluster-1_01 -protocol lldp
Node/       Local  Discovered
Protocol    Port   Device                    Interface         Platform
----------- ------ ------------------------- ----------------  ----------------
cluster-1_01/lldp
            e2a    0013.c31e.5c60            GigabitEthernet1/36
            e2b    0013.c31e.5c60            GigabitEthernet1/35
            e2c    0013.c31e.5c60            GigabitEthernet1/34
            e2d    0013.c31e.5c60            GigabitEthernet1/33
....




[[adjust-the-interval-for-transmitting-lldp-advertisements]]
== Das Intervall für die Übertragung von LLDP-Ankündigungen anpassen

LLDP-Ankündigungen werden in regelmäßigen Abständen an LLDP-Nachbarn gesendet. Das Intervall für das Senden von LLDP-Ankündigungen kann je nach Netzwerkverkehr und Änderungen in der Netzwerktopologie erhöht oder verringert werden.

.Über diese Aufgabe
Das von IEEE empfohlene Standardintervall beträgt 30 Sekunden, aber Sie können einen Wert zwischen 5 Sekunden und 300 Sekunden eingeben.

.Schritte
. Das aktuelle LLDP-Werbezeitintervall für einen Knoten oder für alle Knoten in einem Cluster anzeigen:
+
** Einzelner Knoten:
+
....
run -node <node_name> options lldp.xmit.interval
....
** Alle Knoten:
+
....
options lldp.xmit.interval
....


. Das Intervall für das Senden von LLDP-Ankündigungen für alle Ports eines Knotens oder für alle Ports aller Knoten in einem Cluster anpassen:
+
** Einzelner Knoten:
+
....
run -node <node_name> options lldp.xmit.interval <interval>
....
** Alle Knoten:
+
....
options lldp.xmit.interval <interval>
....






[[adjust-the-time-to-live-value-for-lldp-advertisements]]
== Den Time-to-Live-Wert für LLDP-Anzeigen anpassen

Time-To-Live (TTL) ist der Zeitraum, für den LLDP-Ankündigungen im Cache benachbarter LLDP-kompatibler Geräte gespeichert werden. TTL wird in jedem LLDP-Paket angekündigt und bei jedem Empfang eines LLDP-Pakets durch einen Knoten aktualisiert. TTL kann in ausgehenden LLDP-Frames geändert werden.

.Über diese Aufgabe
* TTL ist ein berechneter Wert, das Produkt aus dem Sendeintervall (`lldp.xmit.interval` und dem Haltemultiplikator (`lldp.xmit.hold` plus eins.
* Der Standardwert für den Haltemultiplikator beträgt 4, es können jedoch Werte zwischen 1 und 100 eingegeben werden.
* Der Standard-TTL-Wert beträgt daher 121 Sekunden, wie von IEEE empfohlen, aber durch Anpassen der Werte für das Sendeintervall und den Haltemultiplikator kann ein Wert für ausgehende Frames von 6 Sekunden bis 30001 Sekunden angegeben werden.
* Wird eine IP-Adresse vor Ablauf der TTL entfernt, werden die LLDP-Informationen bis zum Ablauf der TTL zwischengespeichert.


.Schritte
. Den aktuellen Hold-Multiplikatorwert für einen Knoten oder für alle Knoten in einem Cluster anzeigen:
+
** Einzelner Knoten:
+
....
run -node <node_name> options lldp.xmit.hold
....
** Alle Knoten:
+
....
options lldp.xmit.hold
....


. Den Wert des Haltemultiplikators an allen Ports eines Knotens oder an allen Ports aller Knoten in einem Cluster anpassen:
+
** Einzelner Knoten:
+
....
run -node <node_name> options lldp.xmit.hold <hold_value>
....
** Alle Knoten:
+
....
options lldp.xmit.hold <hold_value>
....






[[view-or-clear-lldp-statistics]]
== LLDP-Statistiken anzeigen oder löschen

Sie können die LLDP-Statistiken für die Cluster- und Nicht-Cluster-Ports auf jedem Knoten einsehen, um potenzielle Netzwerkverbindungsprobleme zu erkennen. Die LLDP-Statistiken sind seit ihrer letzten Löschung kumulativ.

.Über diese Aufgabe
Bei ONTAP 9.10.1 und älteren Versionen werden aufgrund der stets aktivierten LLDP-Funktion für Cluster-Ports LLDP-Statistiken für den Datenverkehr auf diesen Ports immer angezeigt. LLDP muss auf Nicht-Cluster-Ports aktiviert sein, damit Statistiken für diese Ports angezeigt werden.

Für ONTAP 9.11.1 und höher ist LLDP für Cluster- und Speicherports immer aktiviert, daher werden LLDP-Statistiken für den Datenverkehr auf diesen Ports stets angezeigt. LLDP muss auf Nicht-Cluster- und Nicht-Speicherports aktiviert sein, damit Statistiken für diese Ports angezeigt werden.

.Schritt
Aktuelle LLDP-Statistiken für alle Ports eines Knotens anzeigen oder löschen:

[cols="40,60"]
|===


| Wenn Sie möchten... | Eingeben... 


 a| 
LLDP-Statistiken anzeigen
 a| 
`run -node node_name lldp stats`



 a| 
LLDP-Statistiken löschen
 a| 
`run -node node_name lldp stats -z`

|===


[[show-and-clear-statistics-example]]
=== Beispiel zum Anzeigen und Löschen von Statistiken

Der folgende Befehl zeigt die LLDP-Statistiken an, bevor sie gelöscht werden. Die Ausgabe zeigt die Gesamtzahl der Pakete, die seit dem letzten Löschen der Statistiken gesendet und empfangen wurden.

....
cluster-1::> run -node vsim1 lldp stats

RECEIVE
 Total frames:     190k  | Accepted frames:   190k | Total drops:         0
TRANSMIT
 Total frames:     5195  | Total failures:      0
OTHER
 Stored entries:      64
....
Der folgende Befehl löscht die LLDP Statistiken.

....
cluster-1::> The following command clears the LLDP statistics:
run -node vsim1 lldp stats -z
run -node node1 lldp stats

RECEIVE
 Total frames:        0  | Accepted frames:     0  | Total drops:         0
TRANSMIT
 Total frames:        0  | Total failures:      0
OTHER
 Stored entries:      64
....
Nachdem die Statistiken gelöscht wurden, beginnen sie nach dem Senden oder Empfangen der nächsten LLDP-Werbung wieder zu zählen.
