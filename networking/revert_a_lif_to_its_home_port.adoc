= Ein LIF wird nach einem ONTAP Node-Failover oder einer Portmigration auf seinen Home-Port zurückgesetzt.
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können eine LIF nach einem Failover oder einer Migration auf einen anderen Port, entweder manuell oder automatisch, auf ihren Home-Port zurücksetzen. Wenn der Home-Port einer bestimmten LIF nicht verfügbar ist, bleibt die LIF auf ihrem aktuellen Port und wird nicht zurückgesetzt.

.Über diese Aufgabe
* Wenn der Heimatport eines LIF administrativ in den aktiven Zustand versetzt wird, bevor die automatische Rücksetzoption festgelegt wird, erfolgt keine Rückkehr des LIF zum Heimatport.
* Der LIF wird nicht automatisch zurückgesetzt, es sei denn, der Wert der Option "auto-revert" ist auf true gesetzt.
* Es muss sichergestellt sein, dass die Option "auto-revert" für die LIFs aktiviert ist, damit diese auf ihre Home-Ports zurückkehren.


Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Mit dem System Manager kann eine Netzwerkschnittstelle auf ihren Home-Port zurückgesetzt werden.*

.Schritte
. *Netzwerk > Übersicht > Netzwerkschnittstellen* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Zurücksetzen* neben der Netzwerkschnittstelle, die geändert werden soll.
. *Zurücksetzen* setzt eine Netzwerkschnittstelle auf ihren ursprünglichen Port zurück.


--
.CLI
--
*Die CLI kann verwendet werden, um eine LIF auf ihren Heimatport zurückzusetzen.*

.Schritt
Einen LIF manuell oder automatisch auf seinen Home-Port zurücksetzen:

[cols="30,70"]
|===


| Wenn eine LIF auf ihren Home-Port zurückgesetzt werden soll... | Anschließend ist folgender Befehl einzugeben... 


| Manuell | `network interface revert -vserver vserver_name -lif lif_name` 


| Automatisch | `network interface modify -vserver vserver_name -lif lif_name -auto-revert true` 
|===
Weitere Informationen zu  `network interface` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+interface["ONTAP-Befehlsreferenz"^].
