= Den UTA2-Port für die Verwendung im ONTAP Netzwerk konvertieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Ihren UTA2-Port vom Converged Network Adapter (CNA) Modus in den Fibre Channel (FC) Modus umwandeln oder umgekehrt.

Die UTA2-Persönlichkeit ist vom CNA-Modus in den FC-Modus zu ändern, wenn das physische Medium geändert werden muss, das den Port mit seinem Netzwerk verbindet, oder um die Unterstützung von FC-Initiatoren und Zieladaptern zu ermöglichen.

[role="tabbed-block"]
====
.Vom CNA-Modus in den FC-Modus
--
.Schritte
. Den Adapter offline schalten:
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_name> -status-admin down
----
. Portmodus ändern:
+
[source, cli]
----
ucadmin modify -node <node_name> -adapter <adapter_name> -mode fcp
----
. Den Knoten neu starten und anschließend den Adapter online schalten:
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_name> -status-admin up
----
. Benachrichtigen Sie Ihren Administrator oder VIF-Manager, damit der Port je nach Bedarf gelöscht oder entfernt wird:
+
** Wenn der Port als Home-Port eines LIF verwendet wird, Mitglied einer Schnittstellengruppe (ifgrp) ist oder VLANs hostet, sollte ein Administrator Folgendes tun:
+
*** Die LIFs verschieben, den Port aus der ifgrp entfernen oder die VLANs löschen.
*** Der Port kann manuell gelöscht werden, indem der  `network port delete` Befehl ausgeführt wird. Falls der  `network port delete` Befehl fehlschlägt, sollte der Administrator die Fehler beheben und dann den Befehl erneut ausführen.


** Wenn der Port nicht als Home-Port eines LIF verwendet wird, kein Mitglied einer ifgrp ist und keine VLANs hostet, sollte der VIF Manager den Port beim Neustart aus seinen Datensätzen entfernen. Falls der VIF-Manager den Port nicht entfernt, muss der Administrator ihn nach dem Neustart manuell mit dem  `network port delete` Befehl entfernen.
+
Weitere Informationen zu  `network port delete` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-delete.html["ONTAP-Befehlsreferenz"^].



. Es sollte überprüft werden, ob das richtige SFP+ installiert ist:
+
[source, cli]
----
network fcp adapter show -instance -node -adapter
----
+
Für CNA sollte ein 10Gb Ethernet SFP verwendet werden. Für FC sollte entweder ein 8 Gb SFP oder ein 16 Gb SFP verwendet werden, bevor die Konfiguration am Node geändert wird.



--
.Vom FC-Modus in den CNA-Modus
--
.Schritte
. Den Adapter offline schalten:
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_name> -status-admin down
----
. Portmodus ändern:
+
[source, cli]
----
ucadmin modify -node <node_name> -adapter <adapter_name> -mode cna
----
. Den Knoten neu starten
. Es sollte überprüft werden, ob das richtige SFP+ installiert ist.
+
Für CNA sollte ein 10Gb Ethernet SFP verwendet werden.
