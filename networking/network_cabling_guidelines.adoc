= Bewährte Verfahren für ONTAP Netzwerkverkabelung
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Best Practices für die Netzwerkkabelung trennen den Datenverkehr in die folgenden Netzwerke: Cluster, Management und Daten.

Sie sollten einen Cluster so verkabeln, dass der Cluster-Datenverkehr über ein separates Netzwerk vom übrigen Datenverkehr geleitet wird. Es ist optional, aber empfehlenswert, den Netzwerkmanagement-Datenverkehr von Daten- und Intracluster-Datenverkehr zu trennen. Durch die Verwendung separater Netzwerke lassen sich eine höhere Leistung, eine einfachere Administration sowie verbesserte Sicherheit und ein optimierter Verwaltungszugriff auf die Knoten erreichen.

Das folgende Diagramm veranschaulicht die Netzwerkverkabelung eines vier-Knoten HA-Clusters, der drei separate Netzwerke umfasst:

image:Network_Cabling_Guidelines.png["Netzwerkverkabelung eines vier Node HA-Clusters"]

Beim Verkabeln von Netzwerkverbindungen sind bestimmte Richtlinien zu beachten:

* Jeder Knoten sollte mit drei verschiedenen Netzwerken verbunden sein.
+
Ein Netzwerk dient der Verwaltung, eines dem Datenzugriff und eines der Kommunikation innerhalb des Clusters. Die Verwaltungs- und Datennetzwerke können logisch getrennt werden.

* Es kann mehr als eine Datennetzwerkverbindung zu jedem Knoten bestehen, um den Datenverkehr der Clients zu optimieren.
* Ein Cluster kann ohne Datennetzwerkverbindungen erstellt werden, muss jedoch eine Cluster-Interconnect-Verbindung enthalten.
* Zu jedem Knoten sollten stets mindestens zwei Clusterverbindungen bestehen.


Weitere Informationen zur Netzwerkverkabelung finden sich in der  https://docs.netapp.com/us-en/ontap-systems/index.html["AFF und FAS Systemdokumentationszentrum"^] und der  https://hwu.netapp.com/Home/Index["Hardware Universe"^].
