= Informationen zu SNMP im ONTAP Netzwerk
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
SNMP kann so konfiguriert werden, dass SVMs in Ihrem Cluster überwacht werden, um Probleme zu vermeiden, bevor sie auftreten, und um auf Probleme zu reagieren, falls sie auftreten. Die Verwaltung von SNMP umfasst die Konfiguration von SNMP-Benutzern und die Konfiguration von SNMP-Traphost-Zielen (Management-Workstations) für alle SNMP-Ereignisse. SNMP ist auf Daten-LIFs standardmäßig deaktiviert.

Sie können im Daten-SVM schreibgeschützte SNMP-Benutzer erstellen und verwalten. Daten-LIFs müssen so konfiguriert sein, dass sie SNMP-Anfragen auf dem SVM empfangen.

SNMP-Netzwerkmanagement-Workstations oder Manager können den SVM-SNMP-Agenten nach Informationen abfragen. Der SNMP-Agent sammelt Informationen und leitet sie an die SNMP-Manager weiter. Der SNMP-Agent generiert außerdem Trap-Benachrichtigungen, sobald bestimmte Ereignisse eintreten. Der SNMP-Agent auf dem SVM verfügt über Leserechte, er kann weder für Set-Operationen noch zur Durchführung einer Korrekturmaßnahme als Reaktion auf einen Trap verwendet werden. ONTAP stellt einen SNMP-Agenten bereit, der mit den SNMP-Versionen v1, v2c und v3 kompatibel ist. SNMPv3 bietet erweiterte Sicherheit durch die Verwendung von Passphrasen und Verschlüsselung.

Weitere Informationen zur SNMP-Unterstützung in ONTAP-Systemen sind unter  https://www.netapp.com/pdf.html?item=/media/16417-tr-4220.pdf["TR-4220: SNMP-Unterstützung in Data ONTAP"^] zu finden.



[[mib-overview]]
== MIB-Übersicht

Eine MIB (Management Information Base) ist eine Textdatei, die SNMP-Objekte und Traps beschreibt.

MIBs beschreiben die Struktur der Verwaltungsdaten des Speichersystems und verwenden einen hierarchischen Namensraum mit Objektbezeichnern (OIDs). Jede OID identifiziert eine Variable, die mit SNMP ausgelesen werden kann.

Da MIBs keine Konfigurationsdateien sind und ONTAP diese Dateien nicht liest, wird die SNMP-Funktionalität durch MIBs nicht beeinträchtigt. ONTAP stellt die folgende MIB-Datei bereit:

* Ein NetApp benutzerdefinierter MIB (`netapp.mib`)


ONTAP unterstützt IPv6 (RFC 2465), TCP (RFC 4022), UDP (RFC 4113) und ICMP (RFC 2466) MIBs, die sowohl IPv4- als auch IPv6-Daten anzeigen, werden unterstützt.

ONTAP bietet außerdem eine kurze Querverweisfunktion zwischen Objektbezeichnern (OIDs) und Objektkurznamen in der  `traps.dat` Datei.


NOTE: Die neuesten Versionen der ONTAP MIBs und der `traps.dat`-Dateien sind auf der NetApp Support Site verfügbar. Die Versionen dieser Dateien auf der Support Site entsprechen jedoch nicht unbedingt den SNMP-Funktionen Ihrer ONTAP Version. Diese Dateien werden bereitgestellt, um die Evaluierung der SNMP-Funktionen in der neuesten ONTAP Version zu unterstützen.



[[snmp-traps]]
== SNMP-Traps

SNMP-Traps erfassen Systemüberwachungsinformationen, die als asynchrone Benachrichtigung vom SNMP-Agenten an den SNMP-Manager gesendet werden.

Es gibt drei Arten von SNMP-Traps: Standard, integriert und benutzerdefiniert. Benutzerdefinierte Traps werden in ONTAP nicht unterstützt.

Mithilfe eines Traps kann regelmäßig auf im MIB definierte Betriebsschwellenwerte oder Fehler geprüft werden. Wird ein Schwellenwert erreicht oder ein Fehler erkannt, sendet der SNMP-Agent eine Nachricht (Trap) an die Traphosts und benachrichtigt sie über das Ereignis.


NOTE: ONTAP unterstützt SNMPv1- und SNMPv3-Traps. ONTAP unterstützt keine SNMPv2c-Traps und INFORMs.



[[standard-snmp-traps]]
== Standard SNMP-Traps

Diese Traps sind in RFC 1215 definiert. Es gibt fünf Standard-SNMP-Traps, die von ONTAP unterstützt werden: coldStart, warmStart, linkDown, linkUp und authenticationFailure.


NOTE: Die authenticationFailure Trap ist standardmäßig deaktiviert. Der  `system snmp authtrap` Befehl muss verwendet werden, um die Trap zu aktivieren. Weitere Informationen zu  `system snmp authtrap` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-snmp-authtrap.html["ONTAP-Befehlsreferenz"^].



[[built-in-snmp-traps]]
== Integrierte SNMP-Traps

Integrierte Traps sind in ONTAP vordefiniert und werden bei einem Ereignis automatisch an die Netzwerkmanagementstationen auf der Traphost-Liste gesendet. Diese Traps, wie diskFailedShutdown, cpuTooBusy und volumeNearlyFull, sind in der benutzerdefinierten MIB definiert.

Jede integrierte Falle wird durch einen eindeutigen Fallencode identifiziert.
