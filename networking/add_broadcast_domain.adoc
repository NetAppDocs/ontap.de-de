= ONTAP Broadcast-Domänen erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Broadcast-Domänen gruppieren Netzwerkports im Cluster, die zum selben Layer-2-Netzwerk gehören. Die Ports können dann von SVMs verwendet werden.

Broadcast-Domänen werden beim Erstellen oder Beitreten eines Clusters automatisch angelegt. Ab ONTAP 9.12.0 ist es zusätzlich zu den automatisch erstellten Broadcast-Domänen möglich, eine Broadcast-Domäne im System Manager manuell hinzuzufügen.


NOTE: Die Vorgehensweise zum Erstellen von Broadcast-Domänen unterscheidet sich in ONTAP 9.7 und früheren Versionen. Wenn Broadcast-Domänen in einem Netzwerk mit ONTAP 9.7 oder einer früheren Version erstellt werden müssen, siehe link:https://docs.netapp.com/us-en/ontap-system-manager-classic/networking-bd/create_a_broadcast_domain97.html["Erstellen einer Broadcast-Domäne (ONTAP 9.7 und früher)"^].

.Bevor Sie beginnen
Die Ports, die Sie der Broadcast-Domäne hinzufügen möchten, dürfen keiner anderen Broadcast-Domäne angehören. Wenn die Ports, die Sie verwenden möchten, zu einer anderen Broadcast-Domäne gehören, aber ungenutzt sind, sollten diese Ports aus der ursprünglichen Broadcast-Domäne entfernt werden.

.Über diese Aufgabe
* Alle Broadcast-Domainnamen müssen innerhalb eines IPspace eindeutig sein.
* Bei den Ports, die einer Broadcast-Domäne hinzugefügt werden, kann es sich um physische Netzwerkports, VLANs oder Link-Aggregation-Gruppen/Schnittstellengruppen (LAGs/ifgrps) handeln.
* Wenn die Ports, die Sie verwenden möchten, zu einer anderen Broadcast-Domäne gehören, aber ungenutzt sind, sollten sie aus der bestehenden Broadcast-Domäne entfernt werden, bevor sie der neuen hinzugefügt werden.
* Die maximale Übertragungseinheit (MTU) der Ports, die einer Broadcast-Domäne hinzugefügt werden, wird auf den in der Broadcast-Domäne festgelegten MTU-Wert aktualisiert.
* Der MTU-Wert muss mit dem Wert aller Geräte übereinstimmen, die mit diesem Layer-2-Netzwerk verbunden sind, mit Ausnahme des e0M Ports, der den Management-Datenverkehr verarbeitet.
* Wenn Sie keinen IPspace-Namen angeben, wird die Broadcast-Domäne im IPspace „Default“ erstellt.


Um die Systemkonfiguration zu vereinfachen, wird automatisch eine Failover-Gruppe mit demselben Namen erstellt, die dieselben Ports enthält.

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. *Netzwerk > Übersicht > Broadcast domain* auswählen.
. Klicken Sie image:icon_add.gif["Symbol hinzufügen"]
. Die Broadcast-Domäne benennen.
. Die MTU festlegen.
. Wählen Sie den IPspace aus.
. Die Broadcast-Domain speichern.


Eine Broadcast-Domain kann nach dem Hinzufügen bearbeitet oder gelöscht werden.

--
.CLI
--
Wenn ONTAP 9.8 oder höher verwendet wird, werden Broadcast-Domänen automatisch basierend auf der Layer-2-Erreichbarkeit erstellt. Weitere Informationen finden sich unter link:repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"].

Sie können auch manuell eine Broadcast-Domain erstellen.

.Schritte
. Anzeigen der Ports, die derzeit keiner Broadcast-Domäne zugewiesen sind:
+
`network port show`

+
Wenn die Anzeige groß ist, kann mit dem  `network port show -broadcast-domain` Befehl nur die nicht zugewiesenen Ports angezeigt werden.

. Eine Broadcast-Domäne erstellen:
+
`network port broadcast-domain create -broadcast-domain _broadcast_domain_name_ -mtu _mtu_value_ [-ipspace _ipspace_name_] [-ports _ports_list_]`

+
.. `broadcast_domain_name` ist der Name der Broadcast-Domäne, die Sie erstellen möchten.
.. `mtu_value` ist die MTU-Größe für IP-Pakete; 1500 und 9000 sind typische Werte.
+
Dieser Wert wird auf alle Ports angewendet, die zu dieser Broadcast-Domäne hinzugefügt werden.

.. `ipspace_name` ist der Name des IPspace, zu dem diese Broadcast-Domäne hinzugefügt wird.
+
Der "Default"-IPspace wird verwendet, sofern kein Wert für diesen Parameter angegeben wird.

.. `ports_list` ist die Liste der Ports, die der Broadcast-Domäne hinzugefügt werden.
+
Die Ports werden im Format  `node_name:port_number` hinzugefügt, zum Beispiel  `node1:e0c`.



. Es sollte überprüft werden, ob die Broadcast-Domäne wie gewünscht erstellt wurde:
+
`network port show -instance -broadcast-domain new_domain`

+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].



.Beispiel
Der folgende Befehl erstellt die Broadcast-Domäne bcast1 im Default-IPspace, setzt die MTU auf 1500 und fügt vier Ports hinzu:

`network port broadcast-domain create -broadcast-domain _bcast1_ -mtu _1500_ -ports _cluster1-01:e0e,cluster1-01:e0f,cluster1-02:e0e,cluster1-02:e0f_`

Weitere Informationen zu  `network port broadcast-domain create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-broadcast-domain-create.html["ONTAP-Befehlsreferenz"^].

.Nachdem Sie fertig sind
Der Pool von IP-Adressen, die in der Broadcast-Domäne verfügbar sind, kann durch das Erstellen eines Subnetzes definiert werden, oder SVMs und Schnittstellen können dem IPspace zu diesem Zeitpunkt zugewiesen werden. Weitere Informationen finden sich unter link:../peering/index.html["Cluster- und SVM-Peering"].

Wenn der Name einer bestehenden Broadcast-Domäne geändert werden muss, kann der  `network port broadcast-domain rename` Befehl verwendet werden.

Weitere Informationen zu  `network port broadcast-domain rename` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-broadcast-domain-rename.html["ONTAP-Befehlsreferenz"^].
