= IP-Adressen zu einem Subnetz für das ONTAP Netzwerk hinzufügen oder daraus entfernen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
IP-Adressen können beim erstmaligen Erstellen eines Subnetzes hinzugefügt werden, oder es können IP-Adressen zu einem bereits bestehenden Subnetz hinzugefügt werden. Es ist auch möglich, IP-Adressen aus einem bestehenden Subnetz zu entfernen. So werden nur die erforderlichen IP-Adressen für SVMs zugewiesen.

Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Ab ONTAP 9.12.0 kann der System Manager verwendet werden, um IP-Adressen zu einem Subnetz hinzuzufügen oder daraus zu entfernen.*

.Schritte
. *Netzwerk > Übersicht > Subnetze* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Bearbeiten* neben dem Subnetz, das geändert werden soll.
. IP-Adressen hinzufügen oder entfernen.
. Speichern Sie Ihre Änderungen.
+
.. Wird die eingegebene IP-Adresse oder der IP-Adressbereich bereits von einer Schnittstelle verwendet, wird folgende Meldung angezeigt:
`An IP address in this range is already in use by a LIF. Associate the LIF with this subnet?`
.. Wenn auf *OK* geklickt wird, wird die bestehende LIF dem Subnetz zugeordnet.




--
.CLI
--
*Die CLI ermöglicht das Hinzufügen oder Entfernen von IP-Adressen zu oder aus einem Subnetz.*

.Über diese Aufgabe
Beim Hinzufügen von IP-Adressen erhalten Sie eine Fehlermeldung, wenn ein Service-Prozessor oder Netzwerkschnittstellen die IP-Adressen im hinzuzufügenden Bereich verwenden. Wenn eine manuell adressierte Schnittstelle dem aktuellen Subnetz zugeordnet werden soll, kann die Option  `-force-update-lif-associations` auf  `true` gesetzt werden.

Beim Entfernen von IP-Adressen erhalten Sie eine Fehlermeldung, wenn ein Service-Prozessor oder Netzwerkschnittstellen die zu entfernenden IP-Adressen verwenden. Wenn die Schnittstellen die IP-Adressen nach dem Entfernen aus dem Subnetz weiterhin verwenden sollen, kann die  `-force-update-lif-associations` Option auf  `true` gesetzt werden.

.Schritt
IP-Adressen zu einem Subnetz hinzufügen oder daraus entfernen:

[cols="30,70"]
|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
IP-Adressen zu einem Subnetz hinzufügen
 a| 
Netzwerk-Subnetz Adressbereiche



 a| 
IP-Adressen aus einem Subnetz entfernen
 a| 
Netzwerk-Subnetz remove-ranges

|===
Der folgende Befehl fügt die IP-Adressen 192.0.2.82 bis 192.0.2.85 zum Subnetz sub1 hinzu:

....
network subnet add-ranges -subnet-name <sub1> -ip-ranges <192.0.2.82-192.0.2.85>
....
Der folgende Befehl entfernt die IP-Adresse 198.51.100.9 aus dem Subnetz sub3:

....
network subnet remove-ranges -subnet-name <sub3> -ip-ranges <198.51.100.9>
....
Wenn der aktuelle Bereich die Adressen 1 bis 10 und 20 bis 40 umfasst und Sie die Adressen 11 bis 19 und 41 bis 50 hinzufügen möchten (also im Prinzip die Adressen 1 bis 50 zulassen), kann der bestehende Adressbereich mit dem folgenden Befehl überlappt werden. Dieser Befehl fügt nur die neuen Adressen hinzu und hat keine Auswirkungen auf die bestehenden Adressen:

....
network subnet add-ranges -subnet-name <sub3> -ip-ranges <198.51.10.1-198.51.10.50>
....
Weitere Informationen zu  `network subnet add-ranges` und  `network subnet remove-ranges` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+subnet["ONTAP-Befehlsreferenz"^].
