= Informationen zur DNS-Lastausgleichsfunktion für das ONTAP Netzwerk
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Clients binden eine SVM ein, indem sie eine IP-Adresse (zugehörig zu einer LIF) oder einen Hostnamen (zugehörig zu mehreren IP-Adressen) angeben. Standardmäßig werden LIFs vom standortweiten DNS-Server im Round-Robin-Verfahren ausgewählt, was die Arbeitslast auf alle LIFs verteilt.

Die Round-robin-Lastausgleich kann zu einer Überlastung einiger LIFs führen, daher besteht die Möglichkeit, eine DNS-Lastausgleichszone zu verwenden, die die Hostnamensauflösung in einer SVM übernimmt. Durch die Verwendung einer DNS-Lastausgleichszone wird eine bessere Verteilung der neuen Clientverbindungen auf die verfügbaren Ressourcen sichergestellt, was zu einer verbesserten Performance des Clusters führt.

Eine DNS-Lastverteilungszone ist ein DNS-Server innerhalb des Clusters, der die Last auf allen LIFs dynamisch auswertet und einen entsprechend ausgelasteten LIF zurückgibt. In einer Lastverteilungszone weist DNS jedem LIF basierend auf der Last eine Gewichtung (Metrik) zu.

Jedem LIF wird basierend auf seiner Portlast und der CPU-Auslastung seines Heimatknotens ein Gewicht zugewiesen. LIFs an weniger ausgelasteten Ports haben eine höhere Wahrscheinlichkeit, bei einer DNS-Abfrage zurückgegeben zu werden. Gewichte können auch manuell zugewiesen werden.
