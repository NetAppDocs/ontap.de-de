= Informationen zu den Netzwerkkomponenten eines ONTAP Clusters
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Es empfiehlt sich, sich mit den Netzwerkkomponenten eines Clusters vertraut zu machen, bevor das Cluster eingerichtet wird. Die Konfiguration der physischen Netzwerkkomponenten eines Clusters in logische Komponenten bietet die Flexibilität und Mandantenfähigkeit in ONTAP.

Die verschiedenen Netzwerkkomponenten in einem Cluster sind wie folgt:

* Physische Ports
+
Netzwerkschnittstellenkarten (NICs) und Hostbusadapter (HBAs) bieten physische (Ethernet- und Fibre-Channel-) Verbindungen von jedem Knoten zu den physischen Netzwerken (Management- und Datennetzwerken).

+
Informationen zu Standortanforderungen, Switch-Informationen, Portverkabelungsinformationen und Onboard-Portverkabelung des Controllers sind im Hardware Universe unter  https://hwu.netapp.com/["hwu.netapp.com"^] zu finden.

* Logische Ports
+
Virtuelle lokale Netzwerke (VLANs) und Schnittstellengruppen bilden die logischen Ports. Schnittstellengruppen behandeln mehrere physische Ports als einen einzigen Port, während VLANs einen physischen Port in mehrere separate Ports unterteilen.

* IPspaces
+
Sie können einen IPspace verwenden, um für jede SVM in einem Cluster einen separaten IP-Adressraum zu erstellen. Dadurch können Clients in administrativ getrennten Netzwerkdomänen auf Clusterdaten zugreifen, während sie sich überschneidende IP-Adressen aus demselben IP-Adress-Subnetzbereich verwenden.

* Broadcast-Domains
+
Eine Broadcast-Domäne befindet sich in einem IPspace und enthält eine Gruppe von Netzwerkports, die potenziell von vielen Knoten im Cluster stammen und demselben Layer-2-Netzwerk angehören. Die Ports in der Gruppe werden in einer SVM für den Datentransfer verwendet.

* Subnetze
+
Innerhalb einer Broadcast-Domäne wird ein Subnetz erstellt, das einen Pool von IP-Adressen enthält, die zum selben Layer-3-Subnetz gehören. Dieser Pool von IP-Adressen vereinfacht die IP-Adresszuweisung bei der LIF-Erstellung.

* Logische Schnittstellen
+
Eine logische Schnittstelle (LIF) ist eine IP-Adresse oder ein weltweiter Portname (WWPN), die einem Port zugeordnet ist. Sie ist mit Attributen wie Failover-Gruppen, Failover-Regeln und Firewall-Regeln verknüpft. Eine LIF kommuniziert über das Netzwerk durch den Port (physisch oder logisch), an den sie aktuell gebunden ist.

+
Die verschiedenen Arten von LIFs in einem Cluster sind Daten-LIFs, Cluster-bezogene Management-LIFs, Knoten-bezogene Management-LIFs, Intercluster-LIFs und Cluster-LIFs. Die Zuständigkeit für die LIFs hängt von der SVM ab, in der sich die LIF befindet. Daten-LIFs gehören zu Daten-SVMs, Knoten-bezogene Management-LIFs, Cluster-bezogene Management-LIFs und Intercluster-LIFs gehören zu den Admin-SVMs, und Cluster-LIFs gehören zur Cluster-SVM.

* DNS-Zonen
+
Während der Erstellung des LIF kann eine DNS-Zone angegeben werden, wodurch ein Name für das LIF bereitgestellt wird, der über den DNS-Server des Clusters exportiert wird. Mehrere LIFs können denselben Namen teilen, sodass die DNS-Lastverteilungsfunktion die IP-Adressen für den Namen entsprechend der Auslastung verteilt.

+
SVMs können mehrere DNS-Zonen haben.

* Routing
+
Jede SVM ist in Bezug auf die Netzwerkkommunikation autark. Eine SVM besitzt LIFs und Routen, die jeden der konfigurierten externen Server erreichen können.
