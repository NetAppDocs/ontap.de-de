= ONTAP Befehle zur Diagnose von Netzwerkproblemen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Probleme im Netzwerk können mit Befehlen wie  `ping, traceroute, ndp,` und  `tcpdump` diagnostiziert werden. Auch Befehle wie  `ping6` und  `traceroute6` können zur Diagnose von IPv6-Problemen verwendet werden.

|===


| Wenn Sie möchten... | Geben Sie diesen Befehl ein... 


| Es wird geprüft, ob der Knoten andere Hosts in Ihrem Netzwerk erreichen kann. | `network ping` 


| Es kann überprüft werden, ob der Knoten andere Hosts in Ihrem IPv6-Netzwerk erreichen kann. | `network ping6` 


| Der Pfad, den die IPv4-Pakete zu einem Netzwerkknoten nehmen, kann nachvollzogen werden. | `network traceroute` 


| Der Pfad, den die IPv6-Pakete zu einem Netzwerkknoten nehmen, kann nachvollzogen werden. | `network traceroute6` 


| Verwaltung des Neighbor Discovery Protocol (NDP) | `network ndp` 


| Statistiken über Pakete anzeigen, die auf einer bestimmten Netzwerkschnittstelle oder auf allen Netzwerkschnittstellen empfangen und gesendet werden | `run -node _node_name_ ifstat`*Hinweis*: Dieser Befehl ist über die nodeshell verfügbar. 


| Informationen über benachbarte Geräte, die von jedem Knoten und Port im Cluster erkannt werden, einschließlich des Typs des entfernten Geräts und der Geräteplattform, werden angezeigt. | `network device-discovery show` 


| Die CDP-Nachbarn des Knotens werden angezeigt (ONTAP unterstützt nur CDPv1-Ankündigungen) | `run -node _node_name_ cdpd show-neighbors`*Hinweis*: Dieser Befehl ist über die nodeshell verfügbar. 


| Die im Netzwerk gesendeten und empfangenen Pakete werden nachverfolgt. | `network tcpdump start -node _node-name_ -port _port_name_`*Hinweis*: Dieser Befehl ist über die nodeshell verfügbar. 


| Latenz und Durchsatz zwischen Intercluster- oder Intracluster-Knoten messen | `network test -path -source-node _source_nodename local_ -destination-cluster _destination_clustername_ -destination-node _destination_nodename_ -session-type _Default_, _AsyncMirrorLocal_, _AsyncMirrorRemote_, _SyncMirrorRemote_, or _RemoteDataTransfer_`Weitere Informationen finden Sie unter link:../performance-admin/index.html["Leistungsmanagement"]. 
|===
