= ONTAP Netzwerkportattribute ändern
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können die Einstellungen für Autonegotiation, Duplex-Modus, Flusskontrolle, Geschwindigkeit und Integrität eines physischen Netzwerk-Ports anpassen.

.Bevor Sie beginnen
Der Port, den Sie ändern möchten, darf keine LIFs hosten.

.Über diese Aufgabe
* Es wird nicht empfohlen, die administrativen Einstellungen der 100 GbE, 40 GbE, 10 GbE oder 1 GbE Netzwerkschnittstellen zu ändern.
+
Die von Ihnen festgelegten Werte für Duplex-Modus und Portgeschwindigkeit werden als administrative Einstellungen bezeichnet. Je nach Netzwerkbeschränkungen können die administrativen Einstellungen von den Betriebseinstellungen (das heißt, dem tatsächlich vom Port verwendeten Duplex-Modus und der Geschwindigkeit) abweichen.

* Es wird nicht empfohlen, die administrativen Einstellungen der zugrunde liegenden physischen Ports in einer Schnittstellengruppe zu ändern.
+
Der  `-up-admin`Parameter (verfügbar auf der erweiterten Berechtigungsstufe) ändert die administrativen Einstellungen des Ports.

* Es wird nicht empfohlen, die  `-up-admin`administrative Einstellung für alle Ports auf einem Knoten oder für den Port, der die letzte betriebsbereite Cluster-LIF auf einem Knoten hostet, auf false zu setzen.
* Es wird nicht empfohlen, die MTU-Größe des Management-Ports zu ändern `e0M`.
* Die MTU-Größe eines Ports in einer Broadcast-Domäne kann nicht von dem für die Broadcast-Domäne festgelegten MTU-Wert geändert werden.
* Die MTU-Größe eines VLANs darf den Wert der MTU-Größe seines Basis-Port nicht überschreiten.


.Schritte
. Die Attribute eines Netzwerk-Ports ändern:
+
`network port modify`

. Sie können das  `-ignore-health-status` Feld auf „true“ setzen, um anzugeben, dass das System den Netzwerkport-Integritätsstatus eines bestimmten Ports ignorieren kann.
+
Der Netzwerkport-Status wird automatisch von „Beeinträchtigt“ auf „Gut“ geändert, und dieser Port kann nun für das Hosting von LIFs verwendet werden. Die Flusssteuerung der Cluster-Ports sollte auf  `none` gesetzt werden. Standardmäßig ist die Flusssteuerung auf  `full` eingestellt.



Der folgende Befehl deaktiviert die Flusssteuerung am Port e0b, indem die Flusssteuerung auf „none“ gesetzt wird:

....
network port modify -node cluster-1-01 -port e0b -flowcontrol-admin none
....
Weitere Informationen zu  `network port modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-modify.html["ONTAP-Befehlsreferenz"^].
