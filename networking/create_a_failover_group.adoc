= ONTAP Failover-Gruppen erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Eine Failover-Gruppe von Netzwerkports wird erstellt, sodass ein LIF automatisch auf einen anderen Port migrieren kann, falls ein Verbindungsfehler am aktuellen Port des LIF auftritt. Dies ermöglicht dem System, den Netzwerkverkehr auf andere verfügbare Ports im Cluster umzuleiten.

Ab ONTAP 9.19.1 erweitert die systemdefinierte Failover-Gruppenrichtlinie von ONTAP die Failover-Ziele für NAS-Daten-LIFs dynamisch, sodass potenziell alle verfügbaren Ports in der Broadcast-Domäne auf jedem verfügbaren Knoten im Cluster einbezogen werden können. Sie können die systemdefinierte Richtlinie auf jede Failover-Gruppe anwenden. Vor ONTAP 9.19.1 konnten LIFs mit dieser Richtlinie nur auf einen anderen Knoten im Cluster ausweichen; wenn dieser Knoten nicht im Quorum war, fand kein Failover statt, selbst wenn andere Knoten verfügbar waren.

.Über diese Aufgabe
Der  `network interface failover-groups create` Befehl wird verwendet, um die Gruppe zu erstellen und Ports zur Gruppe hinzuzufügen.

* Die Ports, die zu einer Failover-Gruppe hinzugefügt werden, können Netzwerkports, VLANs oder Schnittstellengruppen (ifgrps) sein.
* Alle Ports, die zur Failover-Gruppe hinzugefügt werden, müssen zur selben Broadcast-Domäne gehören.
* Ein einzelner Port kann mehreren Failover-Gruppen angehören.
* Wenn Sie LIFs in verschiedenen VLANs oder Broadcast-Domänen haben, müssen Sie Failover-Gruppen für jedes VLAN oder jede Broadcast-Domäne konfigurieren.
* Failover-Gruppen gelten in SAN iSCSI oder FC Umgebungen nicht.


.Schritt
Erstellen einer Failover-Gruppe:

`network interface failover-groups create -vserver _vserver_name_ -failover-group _failover_group_name_ -targets _ports_list_`

* `_vserver_name_` ist der Name der SVM, die die Failover-Gruppe verwenden kann.
* `_failover_group_name_` ist der Name der Failover-Gruppe, die Sie erstellen möchten.
* `_ports_list_` ist die Liste der Ports, die der Failover-Gruppe hinzugefügt werden. Ports werden im Format _node_name>:<port_number>_ hinzugefügt, zum Beispiel node1:e0c.


Der folgende Befehl erstellt die Failover-Gruppe fg3 für SVM vs3 und fügt zwei Ports hinzu:

....
network interface failover-groups create -vserver vs3 -failover-group fg3 -targets cluster1-01:e0e,cluster1-02:e0e
....
.Nachdem Sie fertig sind
* Die Failover-Gruppe sollte jetzt, da sie erstellt wurde, auf eine LIF angewendet werden.
* Die Anwendung einer Failover-Gruppe, die kein gültiges Failover-Ziel für eine LIF bereitstellt, führt zu einer Warnmeldung.
+
Wenn ein LIF, das kein gültiges Failover-Ziel hat, versucht, ein Failover durchzuführen, kann es zu einem Ausfall kommen.

* Weitere Informationen zu  `network interface failover-groups create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-failover-groups-create.html["ONTAP-Befehlsreferenz"^].
