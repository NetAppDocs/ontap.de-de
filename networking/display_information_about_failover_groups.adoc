= Informationen zur ONTAP Failover-Gruppe anzeigen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können Informationen über Failover-Gruppen einsehen, einschließlich der Liste der Nodes und Ports in jeder Failover-Gruppe, ob Failover aktiviert oder deaktiviert ist und welche Art von Failover-Policy auf jede LIF angewendet wird.

.Schritte
. Die Zielports für jede Failover-Gruppe werden mit dem  `network interface failover-groups show` Befehl angezeigt.
+
Der folgende Befehl zeigt Informationen über alle Failover-Gruppen in einem Zwei-Node-Cluster an:

+
....
network interface failover-groups show
                              Failover
Vserver        Group          Targets
-------------- -------------- ---------------------------------
Cluster
               Cluster
                              cluster1-01:e0a, cluster1-01:e0b,
                              cluster1-02:e0a, cluster1-02:e0b
vs1
               Default
                              cluster1-01:e0c, cluster1-01:e0d,
                              cluster1-01:e0e, cluster1-02:e0c,
                              cluster1-02:e0d, cluster1-02:e0e
....
+
Weitere Informationen zu  `network interface failover-groups show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-failover-groups-show.html["ONTAP-Befehlsreferenz"^].

. Die Ziel-Ports und die Broadcast-Domäne für eine bestimmte Failover-Gruppe werden mit dem  `network interface failover-groups show` Befehl angezeigt.
+
Der folgende Befehl zeigt detaillierte Informationen zur Failover-Gruppe data12 für SVM vs4 an:

+
....
network interface failover-groups show -vserver vs4 -failover-group data12

       Vserver Name: vs4
Failover Group Name: data12
   Failover Targets: cluster1-01:e0f, cluster1-01:e0g, cluster1-02:e0f,
                     cluster1-02:e0g
   Broadcast Domain: Default
....
. Die von allen LIFs verwendeten Failover-Einstellungen werden mit dem  `network interface show` Befehl angezeigt.
+
Der folgende Befehl zeigt die Failover-Richtlinie und die Failover-Gruppe, die von jedem LIF verwendet werden:

+
....
network interface show -vserver * -lif * -fields failover-group,failover-policy
vserver   lif                 failover-policy        failover-group
--------  ------------------  ---------------------  --------------
Cluster   cluster1-01_clus_1  local-only             Cluster
Cluster   cluster1-01_clus_2  local-only             Cluster
Cluster   cluster1-02_clus_1  local-only             Cluster
Cluster   cluster1-02_clus_2  local-only             Cluster
cluster1  cluster_mgmt        broadcast-domain-wide  Default
cluster1  cluster1-01_mgmt1   local-only             Default
cluster1  cluster1-02_mgmt1   local-only             Default
vs1       data1               disabled               Default
vs3       data2               system-defined         group2
....
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].
