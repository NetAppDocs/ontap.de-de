= Dynamische Routen aus den Routingtabellen für das ONTAP Netzwerk entfernen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Wenn ICMP-Umleitungen für IPv4 und IPv6 empfangen werden, werden dynamische Routen zur Routingtabelle hinzugefügt. Standardmäßig werden die dynamischen Routen nach 300 Sekunden entfernt. Soll die Beibehaltung dynamischer Routen für einen anderen Zeitraum erfolgen, kann der Zeitüberschreitungswert angepasst werden.

.Über diese Aufgabe
Sie können den Zeitüberschreitungswert zwischen 0 und 65.535 Sekunden festlegen. Wenn Sie den Wert auf 0 setzen, laufen die Routen nie ab. Das Entfernen dynamischer Routen verhindert einen Verbindungsverlust, der durch das Fortbestehen ungültiger Routen verursacht wird.

.Schritte
. Der aktuelle Zeitüberschreitungswert wird angezeigt.
+
** Für IPv4:
+
....
network tuning icmp show
....
** Für IPv6:
+
....
network tuning icmp6 show
....


. Der Zeitüberschreitungswert wird geändert.
+
** Für IPv4:
+
....
network tuning icmp modify -node node_name -redirect-timeout timeout_value
....
** Für IPv6:
+
....
network tuning icmp6 modify -node node_name -redirect-v6-timeout timeout_value
....


. Es sollte überprüft werden, ob der Zeitüberschreitungswert korrekt geändert wurde.
+
** Für IPv4:
+
....
network tuning icmp show
....
** Für IPv6:
+
....
network tuning icmp6 show
....




Weitere Informationen zu  `network tuning icmp` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+tuning+icmp["ONTAP-Befehlsreferenz"^].
