= Wiederherstellung eines falsch konfigurierten ONTAP LIF
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Ein Cluster kann nicht erstellt werden, wenn das Clusternetzwerk mit einem Switch verbunden ist, aber nicht alle im Cluster IPspace konfigurierten Ports die anderen im Cluster IPspace konfigurierten Ports erreichen können.

.Über diese Aufgabe
In einem Switched Cluster kann der  `cluster create` Befehl mit folgender Fehlermeldung fehlschlagen, wenn eine Cluster-Netzwerkschnittstelle (LIF) am falschen Port konfiguriert ist oder ein Cluster-Port mit dem falschen Netzwerk verbunden ist:

....
Not all local cluster ports have reachability to one another.
Use the "network port reachability show -detail" command for more details.
....
Weitere Informationen zu  `cluster create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-create.html["ONTAP-Befehlsreferenz"^].

Die Ergebnisse des  `network port show`Befehls könnten zeigen, dass mehrere Ports dem Cluster IPspace hinzugefügt wurden, da sie mit einem Port verbunden sind, der mit einer Cluster-LIF konfiguriert ist. Die Ergebnisse des  `network port reachability show -detail`Befehls zeigen jedoch, welche Ports keine Verbindung zueinander haben.

Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

Um die Wiederherstellung von einem Cluster-LIF durchzuführen, der auf einem Port konfiguriert ist, der für die anderen mit Cluster-LIFs konfigurierten Ports nicht erreichbar ist, sind die folgenden Schritte erforderlich:

.Schritte
. Den Home-Port des Cluster-LIF auf den korrekten Port zurücksetzen:
+
....
network port modify -home-port
....
+
Weitere Informationen zu  `network port modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-modify.html["ONTAP-Befehlsreferenz"^].

. Entfernen Sie die Ports aus der Cluster-Broadcast-Domäne, auf denen keine Cluster-LIFs konfiguriert sind:
+
....
network port broadcast-domain remove-ports
....
+
Weitere Informationen zu  `network port broadcast-domain remove-ports` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-broadcast-domain-remove-ports.html["ONTAP-Befehlsreferenz"^].

. Cluster erstellen:
+
....
cluster create
....


.Ergebnis
Nach Abschluss der Clustererstellung erkennt das System die korrekte Konfiguration und ordnet die Ports den richtigen Broadcast-Domänen zu.

.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/network-port-reachability-show.html["Netzwerkport-Erreichbarkeit anzeigen"^]
