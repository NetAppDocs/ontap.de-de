= Überwachung der Erreichbarkeit von ONTAP Netzwerk-Ports
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die Erreichbarkeitsüberwachung ist in ONTAP 9.8 und höher integriert. Mit dieser Überwachung lässt sich erkennen, wenn die physische Netzwerktopologie nicht mit der ONTAP Konfiguration übereinstimmt. In einigen Fällen kann ONTAP die Port-Erreichbarkeit wiederherstellen. In anderen Fällen sind zusätzliche Schritte erforderlich.

.Über diese Aufgabe
Mit diesen Befehlen lassen sich Netzwerkfehlkonfigurationen überprüfen, diagnostizieren und beheben, die darauf zurückzuführen sind, dass die ONTAP Konfiguration weder mit der physischen Verkabelung noch mit der Netzwerk-Switch Konfiguration übereinstimmt.

.Schritt
. Port-Erreichbarkeit anzeigen:
+
....
network port reachability show
....
+
Weitere Informationen zu  `network port reachability show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-reachability-show.html["ONTAP-Befehlsreferenz"^].

. Der folgende Entscheidungsbaum und die Tabelle geben Aufschluss darüber, ob ein nächster Schritt erforderlich ist.


image:ontap_nm_image1.png["Entscheidungsbaum zur Reparaturerreichbarkeit"]

[cols="20,80"]
|===


| Erreichbarkeitsstatus | Beschreibung 


 a| 
OK
 a| 
Der Port verfügt über Layer-2-Erreichbarkeit zu seiner zugewiesenen Broadcast-Domäne. Wenn der Erreichbarkeitsstatus „ok“ ist, aber „unerwartete Ports“ vorhanden sind, kann das Zusammenführen einer oder mehrerer Broadcast-Domänen in Betracht gezogen werden. Weitere Informationen stehen in der folgenden Zeile _Unexpected ports_.

Wenn der Erreichbarkeitsstatus „ok“ ist, aber „nicht erreichbare Ports“ vorhanden sind, kann das Aufteilen einer oder mehrerer Broadcast-Domänen in Erwägung gezogen werden. Weitere Informationen enthält die folgende Zeile _Nicht erreichbare Ports_.

Wenn der Erreichbarkeitsstatus "ok" ist und keine unerwarteten oder nicht erreichbaren Ports vorhanden sind, ist Ihre Konfiguration korrekt.



 a| 
Unerwartete Ports
 a| 
Der Port ist über Layer 2 zu seiner zugewiesenen Broadcast-Domäne erreichbar; er ist jedoch auch über Layer 2 zu mindestens einer weiteren Broadcast-Domäne erreichbar.

Die physische Konnektivität und die Switch-Konfiguration sollten überprüft werden, um festzustellen, ob sie fehlerhaft sind oder ob die dem Port zugewiesene Broadcast-Domäne mit einer oder mehreren Broadcast-Domänen zusammengeführt werden muss.

Weitere Informationen finden sich unter link:merge_broadcast_domains.html["Zusammenführen von Broadcast-Domänen"].



 a| 
Nicht erreichbare Ports
 a| 
Wenn eine einzelne Broadcast-Domäne in zwei verschiedene Erreichbarkeitsgruppen aufgeteilt wurde, kann eine Broadcast-Domäne aufgeteilt werden, um die ONTAP Konfiguration mit der physischen Netzwerktopologie zu synchronisieren.

Die Liste der nicht erreichbaren Ports definiert üblicherweise die Menge der Ports, die in eine andere Broadcast-Domäne aufgeteilt werden sollten, nachdem überprüft wurde, dass die physische und die Switch-Konfiguration korrekt sind.

Weitere Informationen finden sich unter link:split_broadcast_domains.html["Aufgeteilte Broadcast-Domänen"].



 a| 
Fehlkonfigurierte Erreichbarkeit
 a| 
Der Port hat keine Layer-2-Erreichbarkeit zu seiner zugewiesenen Broadcast-Domäne, jedoch hat der Port Layer-2-Erreichbarkeit zu einer anderen Broadcast-Domäne.

Die Port-Erreichbarkeit kann wiederhergestellt werden. Bei Ausführung des folgenden Befehls weist das System den Port der Broadcast-Domäne zu, zu der er erreichbar ist:

`network port reachability repair -node -port` Weitere Informationen finden Sie unter link:repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"].



 a| 
Unerreichbarkeit
 a| 
Der Port hat keine Layer-2-Erreichbarkeit zu einer bestehenden Broadcast-Domäne.

Die Port-Erreichbarkeit kann wiederhergestellt werden. Beim Ausführen des folgenden Befehls weist das System den Port einer neu automatisch erstellten Broadcast-Domäne im Default-IPspace zu:

`network port reachability repair -node -port` Weitere Informationen finden Sie unter link:repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"]. Weitere Informationen zu  `network port reachability repair` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-reachability-repair.html["ONTAP-Befehlsreferenz"^].



 a| 
Erreichbarkeit über mehrere Domänen hinweg
 a| 
Der Port ist über Layer 2 zu seiner zugewiesenen Broadcast-Domäne erreichbar; er ist jedoch auch über Layer 2 zu mindestens einer weiteren Broadcast-Domäne erreichbar.

Die physische Konnektivität und die Switch-Konfiguration sollten überprüft werden, um festzustellen, ob sie fehlerhaft sind oder ob die dem Port zugewiesene Broadcast-Domäne mit einer oder mehreren Broadcast-Domänen zusammengeführt werden muss.

Weitere Informationen finden Sie unter link:merge_broadcast_domains.html["Zusammenführen von Broadcast-Domänen"] oder link:repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"].



 a| 
unbekannt
 a| 
Wenn der Erreichbarkeitsstatus „unbekannt“ ist, einige Minuten abwarten und den Befehl erneut ausführen.

|===
Nach der Reparatur eines Ports müssen verschobene LIFs und VLANs überprüft und behoben werden. Wenn der Port Teil einer Schnittstellengruppe war, ist es außerdem erforderlich, nachzuvollziehen, was mit dieser Schnittstellengruppe geschehen ist. Weitere Informationen finden sich unter link:repair_port_reachability.html["Reparatur der Port-Erreichbarkeit"].
