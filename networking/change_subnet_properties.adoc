= Subnetzeigenschaften für das ONTAP Netzwerk ändern
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können die Subnetzadresse und den Maskenwert, die Gateway-Adresse oder den Bereich der IP-Adressen in einem bestehenden Subnetz ändern.

.Über diese Aufgabe
* Beim Ändern von IP-Adressen muss sichergestellt werden, dass es im Netzwerk keine sich überschneidenden IP-Adressen gibt, damit unterschiedliche Subnetze oder Hosts nicht versuchen, dieselbe IP-Adresse zu verwenden.
* Wenn die Gateway-IP-Adresse hinzugefügt oder geändert wird, wird das geänderte Gateway auf neue SVMs angewendet, wenn in diesen eine LIF mit dem Subnetz erstellt wird. Eine Standardroute zum Gateway wird für die SVM erstellt, falls die Route noch nicht existiert. Es kann erforderlich sein, der SVM manuell eine neue Route hinzuzufügen, wenn die Gateway-IP-Adresse geändert wird.


Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Ab ONTAP 9.12.0 ist es möglich, die Subnetzeigenschaften mit System Manager zu ändern.*

.Schritte
. *Netzwerk > Übersicht > Subnetze* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"]*> Bearbeiten* neben dem Subnetz, das geändert werden soll.
. Änderungen vornehmen.
. Speichern Sie Ihre Änderungen.
+
.. Wird die eingegebene IP-Adresse oder der IP-Adressbereich bereits von einer Schnittstelle verwendet, wird folgende Meldung angezeigt:
`An IP address in this range is already in use by a LIF. Associate the LIF with this subnet?`
.. Wenn auf *OK* geklickt wird, wird die bestehende LIF dem Subnetz zugeordnet.




--
.CLI
--
*CLI zur Änderung der Subnetzeigenschaften verwenden*

.Schritt
Subnetzeigenschaften ändern:

....
network subnet modify -subnet-name <subnet_name> [-ipspace <ipspace_name>] [-subnet <subnet_address>] [-gateway <gateway_address>] [-ip-ranges <ip_address_list>] [-force-update-lif-associations <true>]
....
* `subnet_name` ist der Name des Subnetzes, das Sie ändern möchten.
* `ipspace` ist der Name des IPspace, in dem sich das Subnetz befindet.
* `subnet` ist die neue Adresse und Maske des Subnetzes, falls zutreffend; zum Beispiel 192.0.2.0/24.
* `gateway` ist gegebenenfalls das neue Gateway des Subnetzes, zum Beispiel 192.0.2.1. Die Eingabe von *""* entfernt den Gateway-Eintrag.
* `ip_ranges` ist die neue Liste bzw. der neue Bereich von IP-Adressen, die dem Subnetz gegebenenfalls zugewiesen werden. Die IP-Adressen können einzelne Adressen, ein Bereich von IP-Adressen oder eine Kombination in einer durch Kommas getrennten Liste sein. Der hier angegebene Bereich ersetzt die bestehenden IP-Adressen.
* `force-update-lif-associations` ist erforderlich, wenn der IP-Adressbereich geändert wird. Der Wert kann für diese Option auf *true* gesetzt werden, wenn der Bereich der IP-Adressen geändert wird. Dieser Befehl schlägt fehl, wenn ein Service-Prozessor oder Netzwerkschnittstellen die IP-Adressen im angegebenen Bereich verwenden. Das Setzen dieses Werts auf *true* ordnet alle manuell adressierten Schnittstellen dem aktuellen Subnetz zu und ermöglicht das erfolgreiche Ausführen des Befehls.


Der folgende Befehl ändert die Gateway-IP-Adresse des Subnetzes sub3:

....
network subnet modify -subnet-name <sub3> -gateway <192.0.3.1>
....
Weitere Informationen zu  `network subnet modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-modify.html["ONTAP-Befehlsreferenz"^].
