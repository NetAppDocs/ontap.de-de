= ONTAP Routinginformationen anzeigen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können Informationen zur Routing-Konfiguration für jede SVM in Ihrem Cluster anzeigen. Dies kann bei der Diagnose von Routing-Problemen unterstützen, die mit Konnektivitätsproblemen zwischen Client-Anwendungen oder -Diensten und einer LIF auf einem Node im Cluster zusammenhängen.

.Schritte
. Mit dem  `network route show` Befehl werden Routen innerhalb einer oder mehrerer SVMs angezeigt. Das folgende Beispiel zeigt eine in der vs0 SVM konfigurierte Route:
+
....
network route show
  (network route show)
Vserver             Destination     Gateway         Metric
------------------- --------------- --------------- ------
vs0
                    0.0.0.0/0       172.17.178.1    20
....
. Mit dem  `network route show-lifs` Befehl wird die Zuordnung von Routen und LIFs innerhalb einer oder mehrerer SVMs angezeigt.
+
Das folgende Beispiel zeigt LIFs mit Routen, die der vs0 SVM zugeordnet sind:

+
....
network route show-lifs
  (network route show-lifs)

Vserver: vs0
Destination             Gateway                 Logical Interfaces
----------------------  --------------          --------------------
0.0.0.0/0               172.17.178.1            cluster_mgmt,
                                                LIF-b-01_mgmt1,
                                                LIF-b-02_mgmt1
....
+
Weitere Informationen zu  `network route show` und  `network route show-lifs` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+route+show["ONTAP-Befehlsreferenz"^].

. Der  `network route active-entry show` Befehl zeigt installierte Routen auf einem oder mehreren Knoten, SVMs, Subnetzen oder Routen mit angegebenen Zielen an.
+
Das folgende Beispiel zeigt alle installierten Routen auf einer bestimmten SVM:
