= Informationen zum SVM Routing im ONTAP Netzwerk
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die Routingtabelle einer SVM legt den Netzwerkpfad fest, den die SVM zur Kommunikation mit einem Ziel verwendet. Es ist wichtig zu verstehen, wie Routingtabellen funktionieren, damit Netzwerkprobleme bereits im Vorfeld vermieden werden können.

Die Routing-Regeln lauten wie folgt:

* ONTAP leitet den Datenverkehr über die spezifischste verfügbare Route.
* ONTAP leitet den Datenverkehr als letzte Möglichkeit über eine Standard-Gateway-Route (mit 0 Bits Netzmaske), wenn spezifischere Routen nicht verfügbar sind.


Bei Routen mit gleichem Ziel, gleicher Netzmaske und gleicher Metrik kann nicht garantiert werden, dass das System nach einem Neustart oder einem Upgrade dieselbe Route verwendet. Dies ist insbesondere ein Problem, wenn mehrere Standardrouten konfiguriert sind.

Es gilt als Best Practice, für eine SVM nur eine Standardroute zu konfigurieren. Um Unterbrechungen zu vermeiden, sollte sichergestellt werden, dass die Standardroute jede Netzwerkadresse erreichen kann, die nicht über eine spezifischere Route erreichbar ist. Weitere Informationen sind unter link:https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU134["NetApp Knowledge Base: SU134 - Der Netzwerkzugriff kann durch eine fehlerhafte Routing-Konfiguration in clustered ONTAP beeinträchtigt werden"^] verfügbar.
