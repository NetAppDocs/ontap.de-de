= ONTAP LIFs migrieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Unter Umständen müssen Sie eine LIF auf einen anderen Port desselben Knotens oder auf einen anderen Knoten innerhalb des Clusters migrieren, falls der Port fehlerhaft ist oder gewartet werden muss. Die Migration einer LIF ähnelt dem LIF Failover, jedoch ist die LIF Migration ein manueller Vorgang, während das LIF Failover die automatische Migration einer LIF als Reaktion auf einen Verbindungsfehler am aktuellen Netzwerkport der LIF darstellt.

.Bevor Sie beginnen
* Für die LIFs muss eine Failover-Gruppe konfiguriert worden sein.
* Der Zielknoten und die Ports müssen betriebsbereit sein und auf dasselbe Netzwerk wie der Quellport zugreifen können.


.Über diese Aufgabe
* BGP LIFs befinden sich auf dem Home-Port und können nicht auf einen anderen Node oder Port migriert werden.
* Die auf den Ports einer Netzwerkkarte gehosteten LIFs müssen auf andere Ports im Cluster migriert werden, bevor die Netzwerkkarte vom Knoten entfernt wird.
* Der Befehl zur Migration eines Cluster-LIF ist auf dem Knoten auszuführen, auf dem der Cluster-LIF gehostet wird.
* Eine knotenbezogene LIF, wie z. B. eine knotenbezogene Management-LIF, eine Cluster-LIF oder eine Intercluster-LIF, kann nicht auf einen Remote-Knoten migriert werden.
* Wenn eine NFSv4 LIF zwischen Knoten migriert wird, tritt eine Verzögerung von bis zu 45 Sekunden auf, bevor die LIF an einem neuen Port verfügbar ist.
+
Um dieses Problem zu umgehen, kann NFSv4.1 verwendet werden, bei dem keine Verzögerung auftritt.

* Sie können iSCSI LIFs auf All-Flash SAN Array (ASA) Plattformen migrieren, auf denen ONTAP 9.11.1 oder höher ausgeführt wird.
+
Die Migration von iSCSI LIFs ist auf Ports am Home-Node oder am HA Partner beschränkt.

* Wenn es sich bei Ihrer Plattform nicht um eine All-Flash SAN Array (ASA) Plattform handelt, die ONTAP Version 9.11.1 oder höher ausführt, ist eine Migration von iSCSI LIFs von einem Knoten auf einen anderen nicht möglich.
+
Um diese Einschränkung zu umgehen, muss eine iSCSI-LIF auf dem Zielknoten erstellt werden. Informationen dazu link:../networking/create_a_lif.html["Erstellen von iSCSI LIFs"].

* Wenn Sie eine LIF (Netzwerkschnittstelle) für NFS über RDMA migrieren möchten, muss der Zielport RoCE-fähig sein. Für die Migration einer LIF mit der CLI muss ONTAP 9.10.1 oder höher ausgeführt werden, für die Migration mit System Manager ist ONTAP 9.12.1 erforderlich. Im System Manager muss nach Auswahl des RoCE-fähigen Zielports das Kontrollkästchen neben *RoCE-Ports verwenden* aktiviert werden, damit die Migration erfolgreich abgeschlossen werden kann. Weitere Informationen zu link:../nfs-rdma/configure-lifs-task.html["Konfiguration von LIFs für NFS über RDMA"].
* VMware VAAI Copy-Offload-Vorgänge schlagen fehl, wenn die Quell- oder Ziel-LIF migriert wird. Informationen zum Copy-Offload:
+
** link:../nfs-admin/support-vmware-vstorage-over-nfs-concept.html["NFS Umgebungen"]
** link:../san-admin/storage-virtualization-vmware-copy-offload-concept.html["SAN Umgebungen"]




Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI:

[role="tabbed-block"]
====
.System Manager
--
*Mit System Manager eine Netzwerkschnittstelle migrieren*

.Schritte
. *Netzwerk > Übersicht > Netzwerkschnittstellen* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Migrieren* neben der Netzwerkschnittstelle aus, die geändert werden soll.
+

NOTE: Für eine iSCSI LIF im Dialogfeld *Schnittstelle migrieren* den Zielknoten und den Port des HA-Partners auswählen.

+
Wenn Sie die iSCSI-LIF dauerhaft migrieren möchten, muss das Kontrollkästchen aktiviert werden. Die iSCSI-LIF muss offline sein, bevor sie dauerhaft migriert wird. Zudem kann eine dauerhafte Migration einer iSCSI-LIF nach deren Durchführung nicht mehr rückgängig gemacht werden. Es gibt keine Option zum Zurücksetzen.

. Auf *Migrieren* klicken.
. Speichern Sie Ihre Änderungen.


--
.CLI
--
*Verwendung der CLI zur Migration einer LIF*

.Schritt
Je nachdem, ob eine bestimmte LIF oder alle LIFs migriert werden sollen, ist die entsprechende Aktion auszuführen:

[cols="30,70"]
|===


| Wenn Sie migrieren möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Ein spezifisches LIF
 a| 
`network interface migrate`



 a| 
Alle Daten- und Cluster-Management-LIFs auf einem Knoten
 a| 
`network interface migrate-all`



 a| 
Alle LIFs von einem Port
 a| 
`network interface migrate-all -node <node> -port <port>`

|===
Das folgende Beispiel zeigt, wie eine LIF mit dem Namen  `datalif1` auf der SVM  `vs0` auf den Port  `e0d` auf  `node0b` migriert wird:

....
network interface migrate -vserver vs0 -lif datalif1 -dest-node node0b -dest-port e0d
....
Das folgende Beispiel zeigt, wie alle Daten- und Cluster-Management-LIFs vom aktuellen (lokalen) Knoten migriert werden:

....
network interface migrate-all -node local
....
--
