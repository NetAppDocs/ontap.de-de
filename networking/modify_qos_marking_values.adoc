= ONTAP Netzwerk-QoS-Markierungswerte ändern
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können die Werte für die Dienstgütekennzeichnung (QoS) für verschiedene Protokolle und für jeden IPspace ändern.

.Bevor Sie beginnen
Alle Knoten im Cluster müssen die gleiche Version von ONTAP ausführen.

.Schritt
Die QoS-Markierungswerte können mithilfe des  `network qos-marking modify` Befehls geändert werden.

* Der  `–ipspace`Parameter gibt den IPspace an, für den der QoS-Markierungseintrag geändert werden soll.
* Der  `–protocol`Parameter gibt das Protokoll an, für das der QoS-Markierungseintrag geändert werden soll.
* Der  `–dscp` Parameter gibt den Wert des Differentiated Services Code Point (DSCP) an. Die möglichen Werte liegen zwischen 0 und 63.
* Der  `-is-enabled`Parameter wird verwendet, um die QoS-Markierung für das angegebene Protokoll im durch den  `–ipspace`Parameter bereitgestellten IPspace zu aktivieren oder zu deaktivieren.


Der folgende Befehl aktiviert die QoS-Markierung für das NFS-Protokoll im Standard-IPspace:

....
network qos-marking modify -ipspace Default -protocol NFS -is-enabled true
....
Der folgende Befehl setzt den DSCP-Wert für das NFS-Protokoll im Standard-IPspace auf 20:

....
network qos-marking modify -ipspace Default -protocol NFS -dscp 20
....
Weitere Informationen zu  `network qos-marking modify` und möglichen Werten des Protokolls finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-qos-marking-modify.html["ONTAP-Befehlsreferenz"^].
