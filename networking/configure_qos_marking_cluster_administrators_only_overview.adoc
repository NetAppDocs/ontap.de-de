= Informationen zur ONTAP Netzwerk-Dienstgüte (QoS)
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Kennzeichnung der Netzwerk Quality of Service (QoS) unterstützt die Priorisierung verschiedener Datenverkehrstypen basierend auf den Netzwerkbedingungen, um die Netzwerkressourcen effektiv zu nutzen. Der Differentiated Services Code Point (DSCP)-Wert der ausgehenden IP-Pakete kann für die unterstützten Datenverkehrstypen pro IPspace festgelegt werden.



[[dscp-marking-for-uc-compliance]]
== DSCP Kennzeichnung für UC-Konformität

Die Differentiated Services Code Point (DSCP)-Markierung kann für ausgehenden (Egress) IP-Paketverkehr eines bestimmten Protokolls mit einem Standard- oder benutzerdefinierten DSCP-Code aktiviert werden. Die DSCP-Markierung ist ein Mechanismus zur Klassifizierung und Verwaltung von Netzwerkverkehr und stellt eine Komponente der Unified Capability (UC)-Compliance dar.

Die DSCP-Markierung (auch bekannt als QoS-Markierung oder Quality-of-Service-Markierung) wird durch die Angabe eines IPspace, eines Protokolls und eines DSCP-Werts aktiviert. Die Protokolle, auf die die DSCP-Markierung angewendet werden kann, sind NFS, SMB, iSCSI, SnapMirror, NDMP, FTP, HTTP/HTTPS, SSH, Telnet und SNMP.

Wird beim Aktivieren der DSCP-Markierung für ein bestimmtes Protokoll kein DSCP-Wert angegeben, wird ein Standardwert verwendet:

* Der Standardwert für Datenprotokolle/Datenverkehr ist 0x0A (10).
* Der Standardwert für Kontrollprotokolle/Datenverkehr ist 0x30 (48).
