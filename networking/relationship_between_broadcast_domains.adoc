= Bestimmen, welche LIF Failover-Richtlinie in einem ONTAP Netzwerk verwendet wird.
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Broadcast-Domänen, Failover-Gruppen und Failover-Richtlinien arbeiten zusammen, um zu bestimmen, welcher Port die Funktion übernimmt, wenn der Node oder Port, auf dem ein LIF konfiguriert ist, ausfällt.

Eine Broadcast-Domäne listet alle Ports auf, die im selben Layer-2-Ethernet-Netzwerk erreichbar sind. Ein von einem dieser Ports gesendetes Ethernet-Broadcast-Paket wird von allen anderen Ports in der Broadcast-Domäne empfangen. Diese gemeinsame Erreichbarkeitseigenschaft einer Broadcast-Domäne ist für LIFs wichtig, da ein LIF im Falle eines Failovers auf einen anderen Port in der Broadcast-Domäne weiterhin alle lokalen und entfernten Hosts erreichen kann, die vom ursprünglichen Port aus erreichbar waren.

Failover-Gruppen definieren die Ports innerhalb einer Broadcast-Domäne, die gegenseitig LIF-Failover-Abdeckung bieten. Jede Broadcast-Domäne verfügt über eine Failover-Gruppe, die alle ihre Ports umfasst. Diese Failover-Gruppe, die alle Ports in der Broadcast-Domäne enthält, ist die standardmäßige und empfohlene Failover-Gruppe für die LIF. Es können Failover-Gruppen mit kleineren, von Ihnen definierten Teilmengen erstellt werden, beispielsweise eine Failover-Gruppe von Ports mit derselben Verbindungsgeschwindigkeit innerhalb einer Broadcast-Domäne.

Eine Failover-Richtlinie legt fest, wie ein LIF die Ports einer Failover-Gruppe verwendet, wenn ein Knoten oder Port ausfällt. Die Failover-Richtlinie kann als eine Art Filter betrachtet werden, der auf eine Failover-Gruppe angewendet wird. Die Failover-Ziele eines LIF (die Menge der Ports, auf die ein LIF umschalten kann) werden bestimmt, indem die Failover-Richtlinie des LIF auf die Failover-Gruppe in der Broadcast-Domäne angewendet wird.

Die Failover-Ziele für ein LIF lassen sich mit dem folgenden CLI-Befehl anzeigen:

....
network interface show -failover
....
NetApp empfiehlt nachdrücklich die Verwendung der Standard-Failover-Richtlinie für Ihren LIF-Typ.

Es wird festgelegt, ob die empfohlene Standard-Failover-Richtlinie verwendet oder diese je nach LIF-Typ und Umgebung angepasst wird.

[[default-failover-policies-by-lif-type]]
=== Standardmäßige Failover-Richtlinien nach LIF-Typ

[cols="25,25,50"]
|===


| LIF-Typ | Standard-Failover-Richtlinie | Beschreibung 


| BGP-LIFs | deaktiviert | LIF schlägt nicht auf einen anderen Port fehl. 


| Cluster-LIFs | nur lokal | LIF schlägt nur auf Ports auf demselben Knoten fehl. 


| Cluster Management LIF | Broadcast-Domänen-weit | LIF wechselt auf Ports in derselben Broadcast-Domäne auf jedem beliebigen Node im Cluster. 


| Intercluster-LIFs | nur lokal | LIF schlägt nur auf Ports auf demselben Knoten fehl. 


| NAS Daten-LIFs | systemdefiniert | LIF wechselt auf einen anderen Knoten, der nicht der HA-Partner ist. 


| Node Management LIFs | nur lokal | LIF schlägt nur auf Ports auf demselben Knoten fehl. 


| SAN Daten-LIFs | deaktiviert | LIF schlägt nicht auf einen anderen Port fehl. 
|===
Die Failover-Richtlinie „sfo-partner-only“ ist keine Standardrichtlinie, kann jedoch verwendet werden, wenn das LIF nur auf einen Port des Home-Knotens oder des SFO-Partners umschalten soll.
