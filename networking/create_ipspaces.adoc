= IPspaces für das ONTAP Netzwerk erstellen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
IPspaces sind separate IP-Adressräume, in denen Storage Virtual Machines (SVMs) residieren. IPspaces können erstellt werden, wenn SVMs über eigenen sicheren Speicher, eigene Administration und eigenes Routing verfügen sollen. Sie können einen IPspace verwenden, um für jede SVM in einem Cluster einen separaten IP-Adressraum zu erstellen. Dadurch können Clients in administrativ getrennten Netzwerkdomänen auf Clusterdaten zugreifen, während sie sich überschneidende IP-Adressen aus demselben IP-Adress-Subnetzbereich verwenden.

[NOTE]
====

====
.Über diese Aufgabe
Es gilt eine clusterweite Beschränkung von 512 IPspaces. Die clusterweite Beschränkung reduziert sich auf 256 IPspaces für Cluster, die Knoten mit 6 GB RAM enthalten. Im Hardware Universe kann geprüft werden, ob für Ihre Plattform zusätzliche Beschränkungen gelten.

https://hwu.netapp.com/["NetApp Hardware Universe"^]


NOTE: Ein IPspace Name kann nicht "all" sein, da "all" ein systemreservierter Name ist.

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritt
. Einen IPspace erstellen:
+
....
network ipspace create -ipspace ipspace_name
....
+
`ipspace_name` ist der Name des IPspace, den Sie erstellen möchten. Der folgende Befehl erstellt den IPspace ipspace1 auf einem Cluster:

+
....
network ipspace create -ipspace ipspace1
....
+
Weitere Informationen zu  `network ipspace create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-ipspace-create.html["ONTAP-Befehlsreferenz"^].

. IPspaces anzeigen:
+
`network ipspace show`

+
....
IPspace             Vserver List          Broadcast Domains
------------------- --------------------- ----------------------------
Cluster             Cluster               Cluster
Default             Cluster1              Default
ipspace1            ipspace1              -
....
+
Der IPspace wird erstellt, zusammen mit der System-SVM für den IPspace. Die System-SVM transportiert den Management-Datenverkehr.



.Nachdem Sie fertig sind
Wenn Sie in einem Cluster mit einer MetroCluster Konfiguration einen IPspace erstellen, müssen IPspace-Objekte manuell in die Partner-Cluster repliziert werden. SVMs, die vor der Replikation des IPspace erstellt und einem IPspace zugewiesen werden, werden nicht in die Partner-Cluster repliziert.

Broadcast-Domänen werden automatisch im „Default“ IPspace erstellt und können mit folgendem Befehl zwischen IPspaces verschoben werden:

....
network port broadcast-domain move
....
Wenn beispielsweise eine Broadcast-Domäne von „Default“ nach „ips1“ verschoben werden soll, kann folgender Befehl verwendet werden:

....
network port broadcast-domain move -ipspace Default -broadcast-domain Default -to-ipspace ips1
....
