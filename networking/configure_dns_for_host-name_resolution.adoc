= DNS für die Hostnamensauflösung im ONTAP Netzwerk konfigurieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie verwenden DNS, um auf lokale oder entfernte Quellen für Hostinformationen zuzugreifen. DNS muss so konfiguriert werden, dass auf eine oder beide dieser Quellen zugegriffen werden kann.

ONTAP muss Hostinformationen abrufen können, um Clients den korrekten Zugriff zu ermöglichen. Namensdienste müssen so konfiguriert werden, dass ONTAP auf lokale oder externe DNS-Dienste zugreifen kann, um die Hostinformationen zu erhalten.

ONTAP speichert Konfigurationsinformationen für den Namensdienst in einer Tabelle, die der  `/etc/nsswitch.conf` Datei auf UNIX-Systemen entspricht.



[[configure-an-svm-and-data-lifs-for-host-name-resolution-using-an-external-dns-server]]
== Eine SVM und Daten-LIFs für die Hostnamensauflösung mithilfe eines externen DNS-Servers konfigurieren

Sie können den  `vserver services name-service dns` Befehl verwenden, um DNS auf einer SVM zu aktivieren und sie so zu konfigurieren, dass sie DNS für die Hostnamensauflösung verwendet. Hostnamen werden mithilfe externer DNS-Server aufgelöst.

.Bevor Sie beginnen
Für die Auflösung von Hostnamen muss ein standortweiter DNS-Server verfügbar sein.

Es empfiehlt sich, mehr als einen DNS-Server zu konfigurieren, um einen Single Point of Failure zu vermeiden. Der  `vserver services name-service dns create` Befehl gibt eine Warnung aus, wenn nur ein DNS-Servername eingegeben wird.

.Über diese Aufgabe
Weitere Informationen zur Konfiguration von dynamischem DNS auf der SVM sind unter xref:configure_dynamic_dns_services.html[Dynamische DNS-Dienste konfigurieren] zu finden.

.Schritte
. DNS auf der SVM aktivieren:
+
....
vserver services name-service dns create -vserver <vserver_name> -domains <domain_name> -name-servers <ip_addresses> -state enabled
....
+
Der folgende Befehl aktiviert externe DNS-Server auf der SVM vs1:

+
....
vserver services name-service dns create -vserver vs1.example.com -domains example.com -name-servers 192.0.2.201,192.0.2.202 -state enabled
....
+

NOTE: Der  `vserver services name-service dns create` Befehl führt eine automatische Konfigurationsprüfung durch und gibt eine Fehlermeldung aus, wenn ONTAP den Nameserver nicht erreichen kann.

. Der Status der Nameserver kann mit dem  `vserver services name-service dns check` Befehl validiert werden.
+
....
vserver services name-service dns check -vserver vs1.example.com
                                    Name Server
Vserver             Name Server     Status       Status Details
-------------       --------------- ------------ --------------------------
vs1.example.com     10.0.0.50       up           Response time (msec): 2
vs1.example.com     10.0.0.51       up           Response time (msec): 2
....


Informationen zu Servicerichtlinien, die sich auf DNS beziehen, sind unter link:lifs_and_service_policies96.html["LIFs und Servicerichtlinien in ONTAP 9.6 und später"] zu finden.



[[configure-the-name-service-switch-table-for-host-name-resolution]]
== Die Name Service Switch-Tabelle für die Hostnamenauflösung konfigurieren

Die Namensdienst-Umschalttabelle muss korrekt konfiguriert sein, damit ONTAP lokale oder externe Namensdienste zur Abfrage von Hostinformationen heranziehen kann.

.Bevor Sie beginnen
Es muss festgelegt sein, welcher Namensdienst für die Hostzuordnung in Ihrer Umgebung verwendet wird.

.Schritte
. Die erforderlichen Einträge zur Name-Service-Switch-Tabelle hinzufügen:
+
....
vserver services name-service ns-switch modify -vserver <vserver_name> -database <database_name> -source <source_names>
....
. Es sollte sichergestellt werden, dass die Namensdienst-Switch-Tabelle die erwarteten Einträge in der gewünschten Reihenfolge enthält:
+
....
vserver services name-service ns-switch show -vserver <vserver_name>
....


.Beispiel
Das folgende Beispiel ändert einen Eintrag in der Namensdienst-Switch-Tabelle für SVM vs1, sodass zuerst die lokale Hosts-Datei und dann ein externer DNS-Server zur Auflösung von Hostnamen verwendet werden:

....
vserver services name-service ns-switch modify -vserver vs1 -database hosts -sources files,dns
....
