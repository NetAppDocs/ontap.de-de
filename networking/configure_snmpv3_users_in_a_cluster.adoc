= Konfigurieren von SNMPv3 Benutzern in einem ONTAP Cluster
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
SNMPv3 ist im Vergleich zu SNMPv1 und SNMPv2c ein sicheres Protokoll. Für die Verwendung von SNMPv3 ist die Konfiguration eines SNMPv3-Benutzers erforderlich, um die SNMP-Dienstprogramme vom SNMP-Manager aus auszuführen.

.Schritt
Mit dem  `security login create` Befehl wird ein SNMPv3-Benutzer erstellt.

Sie werden aufgefordert, folgende Informationen anzugeben:

* Engine ID: Standardwert und empfohlener Wert ist die lokale Engine ID
* Authentifizierungsprotokoll
* Authentifizierungspasswort
* Datenschutzprotokoll
* Passwort für Datenschutzprotokoll


.Ergebnis
Der SNMPv3-Benutzer kann sich vom SNMP-Manager aus mit Benutzername und Passwort anmelden und die SNMP-Dienstprogrammbefehle ausführen.



[[snmpv3-security-parameters]]
== SNMPv3 Sicherheitsparameter

SNMPv3 beinhaltet eine Authentifizierungsfunktion, die, wenn sie ausgewählt ist, von den Benutzern verlangt, ihren Namen, ein Authentifizierungsprotokoll, einen Authentifizierungsschlüssel und das gewünschte Sicherheitsniveau beim Aufruf eines Befehls einzugeben.

Die folgende Tabelle listet die SNMPv3 Sicherheitsparameter auf:

[cols="25,25,50"]
|===


| Parameter | Befehlszeilenoption | Beschreibung 


 a| 
engineID
 a| 
-e EngineID
 a| 
Engine ID des SNMP-Agenten. Standardwert ist die lokale Engine ID (empfohlen).



 a| 
securityName
 a| 
-u Name
 a| 
Der Benutzername darf maximal 32 Zeichen lang sein.



 a| 
authProtocol
 a| 
-a {none | MD5 | SHA | SHA-256}
 a| 
Der Authentifizierungstyp kann keine, MD5, SHA oder SHA-256 sein.



 a| 
authKey
 a| 
-EINE PASSPHRASE
 a| 
Passphrase mit mindestens acht Zeichen.



 a| 
securityLevel
 a| 
-l {authNoPriv | AuthPriv | noAuthNoPriv}
 a| 
Die Sicherheitsstufe kann Authentifizierung, keine Privatsphäre; Authentifizierung, Privatsphäre; oder keine Authentifizierung, keine Privatsphäre sein.



 a| 
privProtocol
 a| 
-x { none | des | aes128}
 a| 
Das Datenschutzprotokoll kann none, des oder aes128 sein.



 a| 
privPassword
 a| 
-X Passwort
 a| 
Passwort mit mindestens acht Zeichen.

|===


[[examples-for-different-security-levels]]
== Beispiele für verschiedene Sicherheitsstufen

Dieses Beispiel zeigt, wie ein SNMPv3-Benutzer, der mit unterschiedlichen Sicherheitsstufen erstellt wurde, die SNMP Client-seitig Befehle wie  `snmpwalk` verwenden kann, um die Clusterobjekte abzufragen.

Für eine bessere Performance empfiehlt es sich, alle Objekte in einer Tabelle abzurufen, statt nur ein einzelnes Objekt oder einige wenige Objekte aus der Tabelle.


NOTE: Sie müssen  `snmpwalk` 5.3.1 oder höher verwenden, wenn das Authentifizierungsprotokoll SHA ist.



[[security-level-authpriv]]
=== Sicherheitsstufe: authPriv

Die folgende Ausgabe zeigt die Erstellung eines SNMPv3-Benutzers mit der authPriv Sicherheitsstufe.

....
security login create -user-or-group-name snmpv3user -application snmp -authentication-method usm
Enter the authoritative entity's EngineID [local EngineID]:
Which authentication protocol do you want to choose (none, md5, sha, sha2-256) [none]: md5

Enter the authentication protocol password (minimum 8 characters long):
Enter the authentication protocol password again:
Which privacy protocol do you want to choose (none, des, aes128) [none]: des
Enter privacy protocol password (minimum 8 characters long):
Enter privacy protocol password again:
....

security login create -user-or-group-name snmpv3user -application snmp -authmethod usm

....


==== FIPS Modus

FIPS erlaubt die Auswahl von *kein* Datenschutzprotokoll für das Privacy-Protokoll nicht. Daher ist es nicht möglich, einen authNoPriv SNMPv3-Benutzer im FIPS-Modus zu konfigurieren.
