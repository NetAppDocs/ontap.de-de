= ONTAP-Netzwerkverkehr mit DNS-Lastverteilung optimieren
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können Ihren Cluster so konfigurieren, dass Clientanfragen von entsprechend ausgelasteten LIFs bedient werden. Dies führt zu einer ausgewogeneren Auslastung der LIFs und Ports, was wiederum eine bessere Performance des Clusters ermöglicht.

DNS Load Balancing hilft bei der Auswahl eines angemessen ausgelasteten Daten-LIF und beim Ausgleich des Benutzernetzwerkverkehrs über alle verfügbaren Ports (physisch, Schnittstellengruppen und VLANs).

Beim DNS-Load-Balancing werden LIFs der Load-Balancing-Zone einer SVM zugeordnet. Ein standortweiter DNS-Server ist so konfiguriert, dass er alle DNS-Anfragen weiterleitet und den am wenigsten ausgelasteten LIF basierend auf dem Netzwerkverkehr und der Verfügbarkeit der Port-Ressourcen (CPU-Auslastung, Durchsatz, offene Verbindungen usw.) zurückgibt. DNS-Load-Balancing bietet die folgenden Vorteile:

* Neue Client-Verbindungen werden über die verfügbaren Ressourcen verteilt.
* Für die Auswahl der zu verwendenden LIFs beim Aufbau eines bestimmten SVM ist kein manueller Eingriff erforderlich.
* DNS Load Balancing unterstützt NFSv3, NFSv4, NFSv4.1, SMB 2.0, SMB 2.1, SMB 3.0 und S3.
