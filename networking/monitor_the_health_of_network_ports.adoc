= Den Zustand der ONTAP Netzwerkports überwachen
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Die ONTAP Verwaltung von Netzwerkports umfasst eine automatische Zustandsüberwachung sowie eine Reihe von Zustandsmonitoren, die dabei unterstützen, Netzwerkports zu identifizieren, die möglicherweise nicht für das Hosting von LIFs geeignet sind.

.Über diese Aufgabe
Stellt ein Health-Monitor fest, dass ein Netzwerkport fehlerhaft ist, warnt er Administratoren durch eine EMS-Meldung oder markiert den Port als beeinträchtigt. ONTAP vermeidet das Hosten von LIFs auf beeinträchtigten Netzwerkports, wenn für diesen LIF gesunde alternative Failover-Ziele vorhanden sind. Ein Port kann aufgrund eines Soft-Failure-Ereignisses wie Link-Flapping (Verbindungen wechseln schnell zwischen aktiv und inaktiv) oder Netzwerkpartitionierung beeinträchtigt werden:

* Netzwerk-Ports im Cluster-IPspace werden als beeinträchtigt markiert, wenn sie Link-Flapping oder einen Verlust der Layer-2-(L2-)Erreichbarkeit zu anderen Netzwerk-Ports in der Broadcast-Domäne aufweisen.
* Netzwerkports in Nicht-Cluster-IPspaces werden als beeinträchtigt markiert, wenn sie Link-Flapping aufweisen.


Folgende Verhaltensweisen eines beeinträchtigten Ports sind zu beachten:

* Ein beeinträchtigter Port kann nicht in ein VLAN oder eine Schnittstellengruppe aufgenommen werden.
+
Wenn ein Mitgliedsport einer Schnittstellengruppe als beeinträchtigt markiert ist, die Schnittstellengruppe jedoch weiterhin als fehlerfrei gilt, können LIFs auf dieser Schnittstellengruppe gehostet werden.

* LIFs werden automatisch von beeinträchtigten Ports auf fehlerfreie Ports migriert.
* Während eines Failover-Ereignisses wird ein beeinträchtigter Port nicht als Failover-Ziel betrachtet. Wenn keine fehlerfreien Ports verfügbar sind, hosten beeinträchtigte Ports LIFs gemäß der normalen Failover-Richtlinie.
* Sie können keine LIF auf einem eingeschränkten Port erstellen, migrieren oder zurücksetzen.
+
Sie können die  `ignore-health-status` Einstellung des Netzwerkports auf  `true` ändern. Anschließend kann eine LIF auf den fehlerfreien Ports gehostet werden.



.Schritte
. Melden Sie sich im erweiterten Berechtigungsmodus an:
+
....
set -privilege advanced
....
. Es kann geprüft werden, welche Systemüberwachungsfunktionen zur Überwachung des Netzwerkportzustands aktiviert sind:
+
....
network options port-health-monitor show
....
+
Der Gesundheitszustand eines Ports wird durch den Wert der Health Monitors bestimmt.

+
Die folgenden Gesundheitsmonitore sind in ONTAP verfügbar und standardmäßig aktiviert:

+
** Link-Flapping Health Monitor: Überwacht Link-Flapping
+
Wenn ein Port innerhalb von fünf Minuten mehr als einmal einen Verbindungsabbruch aufweist, wird dieser Port als beeinträchtigt markiert.

** L2 Reachability Health Monitor: Überwacht, ob alle Ports, die in derselben Broadcast-Domäne konfiguriert sind, L2-Erreichbarkeit zueinander haben.
+
Dieser Health Monitor meldet L2-Erreichbarkeitsprobleme in allen IPspaces, allerdings werden nur die Ports im Cluster-IPspace als beeinträchtigt markiert.

** CRC Monitor: Überwacht die CRC-Statistiken der Ports
+
Dieser Gesundheitsmonitor kennzeichnet einen Port nicht als beeinträchtigt, sondern generiert eine EMS-Meldung, wenn eine sehr hohe CRC-Fehlerrate festgestellt wird.



+
Weitere Informationen zu  `network options port-health-monitor show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-options-port-health-monitor-show.html["ONTAP-Befehlsreferenz"^].

. Die gewünschten Health Monitore für einen IPspace können mit dem  `network options port-health-monitor modify` Befehl aktiviert oder deaktiviert werden.
+
Weitere Informationen zu  `network options port-health-monitor modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-options-port-health-monitor-modify.html["ONTAP-Befehlsreferenz"^].

. Detaillierter Zustand eines Ports:
+
....
network port show -health
....


Die Befehlsausgabe zeigt den Gesundheitszustand des Ports,  `ignore health status` die Einstellung und eine Liste der Gründe an, warum der Port als beeinträchtigt markiert ist.

Der Gesundheitszustand eines Ports kann  `healthy` oder  `degraded` sein.

Wenn die  `ignore health status` Einstellung  `true` ist, zeigt dies an, dass der Port-Integritätsstatus vom Administrator von  `degraded` auf  `healthy` geändert wurde.

Wenn die  `ignore health status` Einstellung  `false` ist, wird der Port-Gesundheitsstatus automatisch vom System bestimmt.

Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].
