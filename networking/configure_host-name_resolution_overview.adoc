= Informationen zur Hostnamensauflösung für das ONTAP Netzwerk
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
ONTAP muss Hostnamen in numerische IP-Adressen auflösen können, um Clients und Diensten Zugriff zu ermöglichen. Storage Virtual Machines (SVMs) müssen so konfiguriert werden, dass sie lokale oder externe Namensdienste zur Auflösung von Hostinformationen verwenden. ONTAP unterstützt die Konfiguration eines externen DNS-Servers oder die Konfiguration der lokalen Hosts-Datei für die Hostnamensauflösung.

Bei Verwendung eines externen DNS-Servers kann dynamisches DNS (DDNS) konfiguriert werden, wodurch neue oder geänderte DNS-Informationen automatisch von Ihrem Speichersystem an den DNS-Server gesendet werden. Ohne dynamische DNS-Aktualisierungen müssen DNS-Informationen (DNS-Name und IP-Adresse) manuell zu den identifizierten DNS-Servern hinzugefügt werden, wenn ein neues System online geschaltet wird oder sich bestehende DNS-Informationen ändern. Dieser Prozess ist langsam und fehleranfällig. Während der Notfallwiederherstellung kann eine manuelle Konfiguration zu langen Ausfallzeiten führen.
