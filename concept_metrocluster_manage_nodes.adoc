= Informationen zur MetroCluster IP-Standortverwaltung mit ONTAP System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
MetroCluster Konfigurationen spiegeln Daten und Konfigurationen zwischen zwei ONTAP Clustern an unterschiedlichen Standorten synchron wider. Ab ONTAP 9.8 kann der System Manager als vereinfachte Oberfläche für die Verwaltung einer MetroCluster IP Konfiguration verwendet werden.


NOTE: Mit dem System Manager können MetroCluster-Operationen nur in einer MetroCluster IP-Konfiguration durchgeführt werden. In einer MetroCluster FC-Konfiguration ist die Verwaltung der einzelnen Knoten in Ihrer MetroCluster Konfiguration weiterhin mit dem System Manager möglich, jedoch können keine MetroCluster-spezifischen Operationen durchgeführt werden.

Typischerweise werden Cluster in einer MetroCluster Konfiguration an zwei geografisch getrennten Standorten eingerichtet und konfiguriert. Anschließend wird ein Peering zwischen den Clustern eingerichtet, sodass sie synchronisiert werden und Daten gemeinsam nutzen. Die beiden Cluster im Peering-Netzwerk bieten eine bidirektionale Notfallwiederherstellung (Disaster Recovery, DR), wobei jeder Cluster sowohl Quelle als auch Backup des anderen Clusters sein kann. In MetroCluster IP Konfigurationen mit acht oder vier Knoten besteht jeder Standort aus Speicherkontrollern, die als ein oder zwei Hochverfügbarkeitspaare (High Availability, HA) konfiguriert sind.

Sie können link:https://docs.netapp.com/us-en/ontap-metrocluster/install-ip/concept_mediator_requirements.html["ONTAP Mediator installieren"^] an einem dritten Standort verwenden, um den Status der Knoten und ihrer DR-Partner zu überwachen. ONTAP Mediator kann im Katastrophenfall eine Mediator-assisted unplanned switchover (MAUSO) implementieren.

Es besteht auch die Möglichkeit, eine vereinbarte Umschaltung durchzuführen, um einen der Cluster für geplante Wartungsarbeiten außer Betrieb zu nehmen. Der Partnercluster übernimmt alle Daten-I/O-Operationen für beide Cluster, bis der Cluster, an dem die Wartung durchgeführt wurde, wieder in Betrieb genommen wird und eine Rückschaltung erfolgt.

Verfahren zum Einrichten und Verwalten einer MetroCluster IP Konfiguration mit System Manager sind im link:https://docs.netapp.com/us-en/ontap-metrocluster/index.html["MetroCluster Dokumentation"^] zu finden.
