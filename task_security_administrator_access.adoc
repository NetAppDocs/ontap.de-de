= ONTAP Administratorzugriff mit System Manager verwalten
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Die einem Administrator zugewiesene Rolle bestimmt, welche Funktionen der Administrator mit System Manager ausführen kann. System Manager stellt vordefinierte Rollen für Cluster-Administratoren und Storage-VM-Administratoren bereit. Die Rolle wird beim Erstellen des Administratorkontos zugewiesen oder kann später geändert werden.

Je nachdem, wie der Kontozugriff aktiviert wurde, kann es erforderlich sein, eine der folgenden Aktionen durchzuführen:

* Ein öffentlicher Schlüssel wird mit einem lokalen Konto verknüpft.
* Ein von einer Zertifizierungsstelle signiertes digitales Serverzertifikat installieren.
* AD-, LDAP- oder NIS-Zugriff konfigurieren.


Diese Aufgaben können vor oder nach der Aktivierung des Kontozugriffs durchgeführt werden.



[[assigning-a-role-to-an-administrator]]
== Einem Administrator eine Rolle zuweisen

Einem Administrator wird wie folgt eine Rolle zugewiesen:

.Schritte
. *Cluster > Einstellungen* auswählen.
. Wählen Sie image:icon_arrow.gif["Pfeilsymbol"] neben *Benutzer und Rollen* aus.
. Wählen Sie image:icon_add.gif["Symbol hinzufügen"] unter *Benutzer* aus.
. Geben Sie einen Benutzernamen an und wählen Sie im Dropdown-Menü für *Rolle* eine Rolle aus.
. Eine Anmeldemethode und ein Passwort für den Benutzer angeben.




[[changing-an-administrators-role]]
== Ändern der Administratorrolle

Die Rolle eines Administrators kann wie folgt geändert werden:

.Schritte
. *Cluster > Einstellungen* auswählen.
. Wählen Sie den Namen des Benutzers aus, dessen Rolle Sie ändern möchten, und klicken Sie dann auf das image:icon_kabob.gif["Symbol für Menüoptionen"] Symbol, das neben dem Benutzernamen erscheint.
. Klicken Sie auf *Bearbeiten*.
. Im Dropdown-Menü für *Rolle* eine Rolle auswählen.
