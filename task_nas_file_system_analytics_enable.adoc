= ONTAP Dateisystemanalyse aktivieren
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Um Nutzungsdaten wie Kapazitätsanalysen zu erfassen und anzuzeigen, ist es erforderlich, die Dateisystemanalyse auf einem Volume zu aktivieren.


NOTE: Ab ONTAP 9.17.1 sind Volumes, die auf neu erstellten SVMs auf ONTAP Clustern für NAS-Protokolle erstellt werden, standardmäßig mit aktivierter File System Analytics (FSA) ausgestattet. FSA wird automatisch aktiviert, sobald ein Volume erstellt wird, wodurch sofort Analysefunktionen ohne zusätzliche Konfiguration zur Verfügung stehen. Wenn File System Analytics auf neuen Volumes nicht aktiviert werden soll, muss  https://docs.netapp.com/us-en/ontap-cli/volume-analytics-off.html["FSA auf neuen Volumes deaktivieren"^] von der SVM aus erfolgen.

.Über diese Aufgabe
* Ab ONTAP 9.8 kann die File System Analytics auf einem neuen oder bestehenden Volume aktiviert werden. Bei einem Upgrade eines Systems auf ONTAP 9.8 oder höher ist sicherzustellen, dass alle Upgrade-Prozesse abgeschlossen sind, bevor File System Analytics aktiviert wird.
* Die Dauer der Aktivierung der Analysefunktionen hängt von der Größe und dem Inhalt des Volumes ab. System Manager zeigt den Fortschritt an und präsentiert die Analysedaten nach Abschluss. Für genauere Informationen zum Fortschritt des Initialisierungsscans kann der ONTAP CLI-Befehl  `volume analytics show` verwendet werden.
+
** Ab ONTAP 9.15.1 können nur vier Initialisierungsscans gleichzeitig auf einem Knoten durchgeführt werden. Es muss abgewartet werden, bis ein Scan abgeschlossen ist, bevor ein neuer Scan gestartet wird. ONTAP stellt außerdem sicher, dass ausreichend verfügbarer Speicherplatz auf dem Volume vorhanden ist, und zeigt eine Fehlermeldung an, wenn dies nicht der Fall ist. Mindestens 5 bis 8 Prozent des verfügbaren Speicherplatzes des Volumes sollten frei sein. Wenn die automatische Größenanpassung aktiviert ist, wird die verfügbare Größe auf Basis der maximalen automatischen Vergrößerung berechnet.
** Ab ONTAP 9.14.1 bietet ONTAP zusätzlich zu Benachrichtigungen über Drosselungsereignisse, die den Scanfortschritt beeinflussen, eine Fortschrittsverfolgung für den Initialisierungsscan.
** Weitere Überlegungen zum Initialisierungsscan sind unter xref:./file-system-analytics/considerations-concept.html#scan-considerations[Scan-Überlegungen] zu finden.
** Weitere Informationen zu  `volume analytics show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-analytics-show.html["ONTAP-Befehlsreferenz"^].






[[enable-file-system-analytics-on-an-existing-volume]]
== Dateisystemanalyse auf einem vorhandenen Volume aktivieren

Die Dateisystemanalyse kann mit ONTAP System Manager oder der CLI aktiviert werden.

.Schritt
[role="tabbed-block"]
====
.System Manager
--
|===
| ONTAP 9.10.1 und höher | ONTAP 9.9.1 und ONTAP 9.8 


 a| 
. *Speicher > Volumes* auswählen.
. Das gewünschte Volume auswählen. Im Menü des einzelnen Volumes *File System > Explorer* auswählen.
. *Analytics aktivieren* oder *Analytics deaktivieren* auswählen.

 a| 
. *Speicher > Volumes* auswählen.
. Wählen Sie das gewünschte Volume aus, dann *Explorer*.
. *Analytics aktivieren* oder *Analytics deaktivieren* auswählen.


|===
--
.CLI
--
.Dateisystemanalyse mit der Befehlszeile aktivieren
. Führen Sie den folgenden Befehl aus:
+
[source, cli]
----
volume analytics on -vserver <svm_name> -volume <volume_name> [-foreground {true|false}]
----
+
Standardmäßig wird der Befehl im Vordergrund ausgeführt; ONTAP zeigt den Fortschritt an und präsentiert Analysedaten nach Abschluss. Wenn präzisere Informationen benötigt werden, kann der Befehl im Hintergrund mit der  `-foreground false` Option ausgeführt und anschließend mit dem  `volume analytics show` Befehl der Initialisierungsfortschritt in der CLI angezeigt werden.

. Nach erfolgreicher Aktivierung der Dateisystemanalyse können die Analysedaten mithilfe des System Manager oder der ONTAP REST API angezeigt werden.


--
Weitere Informationen zu  `volume analytics on` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-analytics-on.html["ONTAP-Befehlsreferenz"^].

== Standardeinstellungen für File System Analytics ändern

Ab ONTAP 9.13.1 ist es möglich, die SVM- oder Cluster-Einstellungen so zu ändern, dass die Dateisystemanalyse standardmäßig auf neuen Volumes aktiviert ist.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Wenn Sie den System Manager verwenden, können Sie die Einstellungen der Storage-VM oder des Clusters so ändern, dass Kapazitätsanalysen und Activity Tracking standardmäßig bei der Volume-Erstellung aktiviert werden. Die Standardaktivierung gilt nur für Volumes, die nach der Änderung der Einstellungen erstellt werden, nicht für bereits vorhandene Volumes.

.Ändern der Einstellungen für die Dateisystemanalyse auf einem Cluster
. Im System Manager zu *Clustereinstellungen* navigieren.
. Im Bereich *Cluster settings* die Registerkarte „File System Settings“ überprüfen. Zum Ändern der Einstellungen das image:icon_gear.gif["Aktionssymbol"] Symbol auswählen.
. Im Feld *Aktivitätsverfolgung* die Namen der SVMs eingeben, für die die Aktivitätsverfolgung standardmäßig aktiviert ist. Wenn das Feld leer bleibt, bleibt die Aktivitätsverfolgung auf allen SVMs deaktiviert.
+
Deaktivieren Sie das Kontrollkästchen *Auf neuen Storage-VMs aktivieren*, um die Aktivitätsverfolgung auf neuen Storage-VMs standardmäßig zu deaktivieren.

. Im Feld *Analytics* die Namen der Storage-VMs eingeben, für die die Kapazitätsanalyse standardmäßig aktiviert sein soll. Wenn das Feld leer bleibt, ist die Kapazitätsanalyse für alle SVMs deaktiviert.
+
Das Kontrollkästchen *Auf neuen Speicher-VMs aktivieren* deaktivieren, um die Kapazitätsanalyse standardmäßig auf neuen Speicher-VMs zu deaktivieren.

. *Speichern* auswählen.


.Ändern der Einstellungen für File System Analytics auf einer SVM
. Wählen Sie die SVM aus, die Sie ändern möchten, dann *Speicher-VM-Einstellungen*.
. Auf der Karte *Dateisystemanalyse* lassen sich mithilfe der Schalter die Aktivitätsverfolgung und die Kapazitätsanalyse für alle neuen Volumes auf der Storage-VM aktivieren oder deaktivieren.


--
.CLI
--
Die Storage-VM kann so konfiguriert werden, dass die Dateisystemanalyse standardmäßig für neue Volumes mithilfe der ONTAP CLI aktiviert ist.

.Dateisystemanalyse ist standardmäßig auf einer SVM aktiviert
. Die SVM so anpassen, dass Kapazitätsanalysen und Aktivitätsverfolgung standardmäßig für alle neu erstellten Volumes aktiviert sind:
+
[source, cli]
----
vserver modify -vserver <svm_name> -auto-enable-activity-tracking true -auto-enable-analytics true
----
+
Weitere Informationen zu  `vserver modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-modify.html["ONTAP-Befehlsreferenz"^].
