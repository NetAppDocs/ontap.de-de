= Erfahren Sie mehr über explizite ONTAP-Kontingente
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit expliziten Quoten lässt sich ein Kontingent für ein bestimmtes Kontingentziel festlegen oder ein Standardkontingent für ein bestimmtes Ziel überschreiben.

Ein explizites Kontingent legt ein Limit für einen bestimmten Benutzer, eine Gruppe oder ein qtree-Kontingent fest. Ein explizites Kontingent ersetzt jedes Standardkontingent, das für dasselbe Ziel gilt.

Wenn Sie ein explizites Benutzerkontingent für einen Benutzer hinzufügen, der ein abgeleitetes Benutzerkontingent hat, muss dieselbe Benutzerzuordnungseinstellung wie beim Standardbenutzerkontingent verwendet werden. Andernfalls wird das explizite Benutzerkontingent beim Ändern der Kontingentgröße abgelehnt, da es als neues Kontingent betrachtet wird.

Explizite Quotas wirken sich nur auf Standardquotas derselben Ebene (Volume oder qtree) aus. Eine explizite Benutzerkontingent für einen qtree hat beispielsweise keine Auswirkung auf das Standard-Benutzerkontingent des Volumes, das diesen qtree enthält. Das explizite Benutzerkontingent für den qtree überschreibt jedoch das Standard-Benutzerkontingent für diesen qtree (ersetzt die darin definierten Grenzwerte).

.Beispiele für explizite Quoten
Die folgenden Kontingentregeln definieren ein Benutzerkontingent, das alle Benutzer in vol1 auf 50MB Speicherplatz beschränkt. Einem Benutzer, jsmith, werden jedoch aufgrund des expliziten Kontingents (fett hervorgehoben) 80MB Speicherplatz zugewiesen:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target "" -qtree "" -disk-limit 50m

cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target "jsmith" -qtree "" -disk-limit 80m

cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Vserver: vs0               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   ""        ""      off          50MB        -       -        -          -
user   jsmith    ""      off          80MB        -       -        -          -
----
Die folgende Quotenregel beschränkt den angegebenen Benutzer, dargestellt durch vier IDs, auf 550 MB Speicherplatz und 10.000 Dateien im Volume vol1:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target " jsmith,corp\jsmith,engineering\john smith,S-1-5-32-544" -qtree "" -disk-limit 550m -file-limit 10000

cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Vserver: vs0               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   "jsmith,corp\jsmith,engineering\john smith,S-1-5-32-544"
                 ""      off         550MB        -   10000        -          -
----
Die folgende Quotenregel beschränkt die Gruppe eng1 auf 150 MB Speicherplatz und eine unbegrenzte Anzahl von Dateien im proj1 qtree:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol2 -policy-name default -type group -target "eng1" -qtree "proj1" -disk-limit 150m

cluster1::> volume quota policy rule show -vserver vs0 -volume vol2

Vserver: vs0               Policy: default           Volume: vol2
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
group  eng1      proj1   off         150MB        -       -        -          -
----
Die folgende Quotenregel beschränkt den Qtree proj1 im Volume vol2 auf 750MB Speicherplatz und 75.000 Dateien:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol2 -policy-name default -type tree -target "proj1" -disk-limit 750m -file-limit 75000

cluster1::> volume quota policy rule show -vserver vs0 -volume vol2

Vserver: vs0               Policy: default           Volume: vol2
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
tree   proj1     ""      -           750MB        -   75000        -          -
----
