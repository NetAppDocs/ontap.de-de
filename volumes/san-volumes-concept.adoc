= Erfahren Sie mehr über die Bereitstellung von SAN-Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP bietet verschiedene grundlegende Optionen für die SAN-Volume-Bereitstellung. Jede Option verwendet eine andere Methode zur Verwaltung des Volume-Speicherplatzes und der Speicheranforderungen für die ONTAP Block-Sharing-Technologien. Es ist hilfreich, die Funktionsweise der einzelnen Bereitstellungsoptionen zu kennen, um die optimale Option für die jeweilige Umgebung auszuwählen.

[TIP]
====
Es wird nicht empfohlen, SAN-LUNs und NAS-Freigaben auf demselben FlexVol volume zu platzieren. Sie sollten stattdessen separate FlexVol volumes für Ihre SAN-LUNs und Ihre NAS-Freigaben bereitstellen. Dies vereinfacht die Verwaltung und die Replikationsbereitstellungen. Dies entspricht auch der Art und Weise, wie FlexVol volumes in Active IQ Unified Manager (ehemals OnCommand Unified Manager) unterstützt werden.

====


[[thin-provisioning-for-volumes]]
== Thin Provisioning für Volumes

Wenn ein Thin-Provisioned-Volume erstellt wird, reserviert ONTAP beim Erstellen des Volumes keinen zusätzlichen Speicherplatz. Wenn Daten auf das Volume geschrieben werden, fordert das Volume den benötigten Speicherplatz vom Aggregat an, um den Schreibvorgang auszuführen. Die Verwendung von Thin-Provisioned-Volumes ermöglicht es, das Aggregat zu überprovisionieren, was das Risiko birgt, dass das Volume nicht den benötigten Speicherplatz erhält, wenn im Aggregat kein freier Speicherplatz mehr vorhanden ist.

Ein dünn-provisioniertes FlexVol Volume wird erstellt, indem die  `-space-guarantee` Option auf  `none` gesetzt wird.



[[thick-provisioning-for-volumes]]
== Thick Provisioning für Volumes

Wenn ein Thick-Provisioning-Volume erstellt wird, reserviert ONTAP ausreichend Speicherplatz im Aggregat, um sicherzustellen, dass jeder Block im Volume jederzeit beschrieben werden kann. Bei der Konfiguration eines Volumes für die Verwendung von Thick Provisioning können alle ONTAP-Speichereffizienzfunktionen wie Komprimierung und Deduplizierung eingesetzt werden, um den höheren anfänglichen Speicherbedarf auszugleichen.

Ein dick-provisioniertes FlexVol Volume wird erstellt, indem die Option  `-space-slo` (Service Level Objective) auf  `thick` gesetzt wird.



[[semi-thick-provisioning-for-volumes]]
== Semi-thick Provisioning für Volumes

Wird ein Volume mit Semi-Thick Provisioning erstellt, reserviert ONTAP Speicherplatz im Aggregat, um die Volume-Größe zu berücksichtigen. Reicht der freie Speicherplatz auf dem Volume aufgrund von Block-Sharing-Technologien nicht mehr aus, versucht ONTAP, Schutzdatenobjekte (Snapshots sowie FlexClone Dateien und LUNs) zu löschen, um den belegten Speicherplatz freizugeben. Solange ONTAP die Schutzdatenobjekte schnell genug löschen kann, um den für Überschreibungen benötigten Speicherplatz bereitzustellen, bleiben die Schreibvorgänge erfolgreich. Dies wird als „Best Effort“-Schreibgarantie bezeichnet.

[NOTE]
====
Speichereffizienztechnologien wie Deduplizierung, Komprimierung und Kompaktierung können nicht auf einem Volume verwendet werden, das Semi-Thick Provisioning nutzt.

====
Ein halb-dick-provisioniertes FlexVol Volume wird erstellt, indem die Option  `-space-slo` (Service Level Objective) auf  `semi-thick` gesetzt wird.



[[use-with-space-reserved-files-and-luns]]
== Verwendung mit speicherplatzreservierten Dateien und LUNs

Eine speicherplatzreservierte Datei oder LUN ist eine Datei oder LUN, für die bei der Erstellung Speicherplatz zugewiesen wird. Historisch hat NetApp den Begriff „thin-provisioned LUN“ verwendet, um eine LUN zu bezeichnen, bei der die Speicherplatzreservierung deaktiviert ist (eine nicht speicherplatzreservierte LUN).

[NOTE]
====
Dateien ohne reservierten Speicherplatz werden im Allgemeinen nicht als „thin-provisioned files“ bezeichnet.

====

[[support-for-scsi-thin-provisioned-luns]]
== Unterstützung für SCSI Thin-Provisioned LUNs

ONTAP unterstützt T10 SCSI Thin-Provisioned-LUNs sowie NetApp Thin-Provisioned-LUNs. T10 SCSI Thin Provisioning ermöglicht Host-Anwendungen die Unterstützung von SCSI-Funktionen, einschließlich LUN-Speicherplatzrückgewinnung und LUN-Speicherplatzüberwachung für Blockumgebungen. T10 SCSI Thin Provisioning muss von Ihrer SCSI-Host-Software unterstützt werden.

Die ONTAP `space-allocation` Einstellung wird verwendet, um die Unterstützung für T10 Thin Provisioning auf einer LUN zu aktivieren oder zu deaktivieren. Die ONTAP `space-allocation enable` Einstellung wird verwendet, um T10 SCSI Thin Provisioning auf einer LUN zu aktivieren.

Der  `[-space-allocation {enabled|disabled}]` Befehl im link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^] enthält weitere Informationen zum Aktivieren/Deaktivieren der Unterstützung für T10 Thin Provisioning und zum Aktivieren von T10 SCSI Thin Provisioning auf einer LUN.
