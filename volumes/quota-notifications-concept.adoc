= ONTAP Kontingentbenachrichtigungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Quotenbenachrichtigungen sind Meldungen, die an das Ereignismanagementsystem (EMS) gesendet und auch als SNMP-Traps konfiguriert werden.

Benachrichtigungen werden als Reaktion auf die folgenden Ereignisse gesendet:

* Ein endgültiges Kontingentlimit ist erreicht, mit anderen Worten, es wird versucht, dieses zu überschreiten
* Ein weiches Kontingent wird überschritten
* Ein weiches Kontingent wird nicht mehr überschritten


Schwellenwerte unterscheiden sich geringfügig von anderen weichen Quoten. Schwellenwerte lösen Benachrichtigungen nur dann aus, wenn sie überschritten werden, nicht wenn sie nicht mehr überschritten werden.

Hard-quota-Benachrichtigungen sind mit dem Befehl volume quota modify konfigurierbar. Sie können vollständig deaktiviert werden, und die Häufigkeit kann geändert werden, beispielsweise um das Senden redundanter Nachrichten zu verhindern.

Soft-Quota-Benachrichtigungen sind nicht konfigurierbar, da sie wahrscheinlich keine redundanten Nachrichten erzeugen und ihr einziger Zweck die Benachrichtigung ist.

Die folgende Tabelle listet die Ereignisse auf, die Kontingente an das EMS System senden:

[cols="2*"]
|===
| Wenn dies eintritt ... | Dieses Ereignis wird an den EMS gesendet... 


 a| 
Bei der Baumquote ist ein endgültiges Limit erreicht.
 a| 
`wafl.quota.qtree.exceeded`



 a| 
Ein endgültiges Limit ist beim Benutzerkontingent für das Volume erreicht.
 a| 
`wafl.quota.user.exceeded` (für einen UNIX-Benutzer)
`wafl.quota.user.exceeded.win` (für einen Windows-Benutzer)



 a| 
In einem Benutzerkontingent auf einem Qtree wird ein endgültiges Limit erreicht.
 a| 
`wafl.quota.userQtree.exceeded` (für einen UNIX-Benutzer)
`wafl.quota.userQtree.exceeded.win` (für einen Windows-Benutzer)



 a| 
Ein endgültiges Limit wird im Gruppenkontingent auf dem Volume erreicht.
 a| 
`wafl.quota.group.exceeded`



 a| 
In einem Gruppenkontingent auf einem Qtree wird ein endgültiges Limit erreicht.
 a| 
`wafl.quota.groupQtree.exceeded`



 a| 
Ein weiches Limit, einschließlich eines Schwellenwerts, wird überschritten
 a| 
`quota.softlimit.exceeded`



 a| 
Ein weiches Limit wird nicht mehr überschritten
 a| 
`quota.softlimit.normal`

|===
Die folgende Tabelle listet die SNMP-Traps auf, die von Kontingenten generiert werden:

[cols="2*"]
|===
| Wenn dies eintritt ... | Diese SNMP-Trap wird gesendet... 


 a| 
Ein endgültiges Limit wird erreicht
 a| 
quotaExceeded



 a| 
Ein weiches Limit, einschließlich eines Schwellenwerts, wird überschritten
 a| 
quotaExceeded und softQuotaExceeded



 a| 
Ein weiches Limit wird nicht mehr überschritten
 a| 
quotaNormal und softQuotaNormal

|===
[NOTE]
====
Benachrichtigungen enthalten Qtree-ID-Nummern anstelle von Qtree-Namen. Qtree-Namen lassen sich den ID-Nummern mithilfe des  `volume qtree show -id` Befehls zuordnen.
