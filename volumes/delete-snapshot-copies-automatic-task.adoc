= Aktivieren Sie die automatische ONTAP Snapshot- und LUN-Löschung, um Speicherplatz zu verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine Richtlinie für das automatische Löschen von Snapshots und FlexClone LUNs definieren und aktivieren. Das automatische Löschen von Snapshots und FlexClone LUNs kann bei der Verwaltung der Speicherplatznutzung unterstützen.

.Über diese Aufgabe
Snapshots von Lese-/Schreib-Volumes und FlexClone LUNs von Lese-/Schreib-Parent-Volumes können automatisch gelöscht werden. Die automatische Löschung von Snapshots von schreibgeschützten Volumes, zum Beispiel SnapMirror Ziel-Volumes, ist nicht möglich.

.Schritt
. Eine Richtlinie für das automatische Löschen von Snapshots kann durch Verwendung des  `volume snapshot autodelete modify` Befehls definiert und aktiviert werden.
+
Weitere Informationen zu  `volume snapshot autodelete modify` und zur Definition einer Richtlinie, die Ihren Anforderungen entspricht, finden Sie im link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-autodelete-modify.html["ONTAP-Befehlsreferenz"^].

+
Der folgende Befehl aktiviert die automatische Löschung von Snapshots und legt den Auslöser auf  `snap_reserve` für das Volume vol3 fest, das Teil der Storage Virtual Machine (SVM) vs0.example.com ist:

+
[listing]
----
cluster1::> volume snapshot autodelete modify -vserver vs0.example.com
-volume vol3 -enabled true -trigger snap_reserve
----
+
Der folgende Befehl aktiviert die automatische Löschung von Snapshots und von FlexClone LUNs, die für die automatische Löschung für das Volume vol3 markiert sind, das Teil der Storage Virtual Machine (SVM) vs0.example.com ist:

+
[listing]
----
cluster1::> volume snapshot autodelete modify -vserver vs0.example.com
-volume vol3 -enabled true -trigger volume -commitment try -delete-order
oldest_first -destroy-list lun_clone,file_clone
----
+
[NOTE]
====
Snapshots auf Aggregatebene funktionieren anders als Snapshots auf Volume-Ebene und werden automatisch von ONTAP verwaltet. Die Option zum Löschen von Aggregate-Snapshots ist immer aktiviert und hilft bei der Verwaltung der Speicherplatznutzung.

Wenn der Trigger-Parameter für ein Aggregat auf `snap_reserve` festgelegt ist, bleiben die Snapshots so lange erhalten, bis der reservierte Speicherplatz die Schwellenwertkapazität überschreitet. Daher wird der vom Snapshot belegte Speicherplatz im Befehl selbst dann als `0` aufgeführt, wenn der Trigger-Parameter nicht auf `snap_reserve` festgelegt ist, da diese Snapshots automatisch gelöscht werden. Zudem wird der von Snapshots in einem Aggregat belegte Speicherplatz als frei betrachtet und ist im Parameter für den verfügbaren Speicherplatz des Befehls enthalten.
