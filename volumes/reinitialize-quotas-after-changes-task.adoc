= ONTAP-Kontingente nach umfangreichen Änderungen neu initialisieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach umfangreichen Änderungen an bestehenden Kontingentdefinitionen müssen die Kontingente auf allen betroffenen Volumes neu initialisiert werden. Ein Beispiel für diese Art von Änderung ist das Hinzufügen oder Löschen von Kontingenten für Ziele, für die keine Kontingente erzwungen werden.

.Über diese Aufgabe
Sie verfügen über eine Storage Virtual Machine (SVM) mit erzwungenen Quotas und möchten Änderungen vornehmen, die eine vollständige Neuinitialisierung der Quotas erfordern.

.Schritte
. Der  `vserver show`Befehl mit dem  `-instance`Parameter gibt den Namen der Richtlinie an, die der SVM aktuell zugewiesen ist.
. Die Quotenregeln können durch eine der folgenden Aktionen geändert werden:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Dann... 


 a| 
Neue Quotenregeln erstellen
 a| 
Der  `volume quota policy rule create`Befehl wird verwendet



 a| 
Die Einstellungen bestehender Quotenregeln anpassen
 a| 
Der  `volume quota policy rule modify`Befehl wird verwendet



 a| 
Vorhandene Quotenregeln löschen
 a| 
Der  `volume quota policy rule delete`Befehl wird verwendet

|===
. Mit dem  `volume quota policy rule show` Befehl lässt sich überprüfen, ob die Quotenregeln korrekt konfiguriert sind.
. Die Kontingente auf jedem Volume, auf dem Sie die Kontingente geändert haben, werden neu initialisiert, indem die Kontingente für diese Volumes deaktiviert und anschließend wieder aktiviert werden.
+
.. Mit dem  `volume quota off` Befehl auf jedem betroffenen Volume werden die Quoten auf diesem Volume deaktiviert.
.. Der  `volume quota on` Befehl wird auf jedem betroffenen Volume verwendet, um Quoten auf diesem Volume zu aktivieren.
+
[NOTE]
====
Sie müssen unbedingt etwa fünf Minuten warten, bevor Sie die Kontingente auf den betroffenen Volumes reaktivieren, da der Versuch, sie fast unmittelbar nach Ausführung des  `volume quota off` Befehls zu aktivieren, zu Fehlern führen kann.

Alternativ besteht die Möglichkeit, die Befehle zur Neuinitialisierung der Quotas für ein Volume von dem Knoten aus auszuführen, der das jeweilige Volume enthält.

====
+
Der Initialisierungsprozess kann auf eine der folgenden Arten überwacht werden:

+
*** Bei Verwendung des  `volume quota on` Befehls kann der  `-foreground` Parameter hinzugefügt werden, um den Kontingentauftrag im Vordergrund auszuführen. (Standardmäßig wird der Auftrag im Hintergrund ausgeführt.)
+
Wenn der Job im Hintergrund läuft, kann sein Fortschritt mit dem  `job show` Befehl überwacht werden.

*** Der  `volume quota show` Befehl kann zur Überwachung des Status der Quota-Initialisierung verwendet werden.




. Mit dem  `volume quota show -instance` Befehl lassen sich Initialisierungsfehler wie Quotenregeln, die nicht initialisiert werden konnten, überprüfen.
. Mit dem  `volume quota report` Befehl lässt sich ein Quotenbericht anzeigen, sodass überprüft werden kann, ob die durchgesetzten Quoten den Erwartungen entsprechen.
