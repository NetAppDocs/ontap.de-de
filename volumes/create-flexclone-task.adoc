= Erstellen Sie ein ONTAP FlexClone Volume
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es kann ein FlexClone Volume zum Zweck des Datenschutzes aus einem SnapMirror Zielvolume oder aus einem übergeordneten FlexVol Volume erstellt werden, das ein SnapVault sekundäres Volume ist. Ab ONTAP 9.7 kann ein FlexClone Volume aus einem FlexGroup Volume erstellt werden. Nach der Erstellung eines FlexClone Volumes kann das übergeordnete Volume nicht gelöscht werden, solange das FlexClone Volume existiert.

.Bevor Sie beginnen
* Die FlexClone Lizenz muss auf dem Cluster installiert sein. Diese Lizenz ist mit link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["ONTAP One"] enthalten.
* Das Volume, das Sie klonen möchten, muss online sein.



NOTE: Das Klonen eines Volumes als FlexClone Volume auf einer anderen SVM wird auf MetroCluster Konfigurationen nicht unterstützt.



[[create-a-flexclone-volume-of-a-flexvol-or-flexgroup]]
== Ein FlexClone Volume eines FlexVol oder FlexGroup erstellen

.Schritt
. Ein FlexClone Volume erstellen:
+
`volume clone create`

+

NOTE: Beim Erstellen eines Lese-/Schreib FlexClone Volumes aus dem Lese-/Schreib-Parent-Volume muss kein Basis-Snapshot angegeben werden. ONTAP erstellt einen Snapshot, wenn kein spezifischer Snapshot als Basis-Snapshot für den Klon benannt wird. Der Basis-Snapshot muss für die Erstellung eines FlexClone Volumes angegeben werden, wenn das übergeordnete Volume ein Data-Protection-Volume ist.



.Beispiel
* Der folgende Befehl erstellt ein Lese-/Schreib-FlexClone Volume vol1_clone aus dem übergeordneten Volume vol1:
+
`volume clone create -vserver vs0 -flexclone vol1_clone -type RW -parent-volume vol1`

* Der folgende Befehl erstellt ein Datensicherungs-FlexClone Volume vol_dp_clone aus dem übergeordneten Volume dp_vol unter Verwendung des Basis-Snapshots snap1:
+
`volume clone create -vserver vs1 -flexclone vol_dp_clone -type DP -parent-volume dp_vol -parent-snapshot snap1`





[[create-a-flexclone-of-any-snaplock-type]]
== Einen FlexClone eines beliebigen SnapLock-Typs erstellen

Ab ONTAP 9.13.1 kann einer von drei SnapLock-Typen, `compliance`, `enterprise`, `non-snaplock`, beim Erstellen eines FlexClone eines RW-Volumes angegeben werden. Standardmäßig wird ein FlexClone Volume mit demselben SnapLock Typ wie das übergeordnete Volume erstellt. Die Standardeinstellung kann jedoch mit der  `snaplock-type` Option während der Erstellung des FlexClone Volumes überschrieben werden.

Mit dem  `non-snaplock` Parameter und der  `snaplock-type` Option lässt sich ein Nicht-SnapLock Typ FlexClone Volume aus einem SnapLock Parent Volume erstellen, wodurch eine schnellere Methode zur Wiederherstellung von Daten bei Bedarf zur Verfügung steht.

Weitere Informationen zu link:../snaplock/index.html["SnapLock"].

.Bevor Sie beginnen
Es sind die folgenden FlexClone Volume-Beschränkungen zu beachten, wenn sie einen anderen SnapLock Typ als das übergeordnete Volume aufweisen.

* Es werden ausschließlich RW-Klone unterstützt. DP-Klone mit einem SnapLock-Typ, der sich vom übergeordneten Volume unterscheidet, werden nicht unterstützt.
* Volumes mit LUNs können nicht geklont werden, wenn die Option snaplock-type auf einen anderen Wert als 'non-snaplock' gesetzt ist, da SnapLock Volumes keine LUNs unterstützen.
* Ein Volume auf einem MetroCluster gespiegelten Aggregat kann nicht mit einem Compliance SnapLock Typ geklont werden, da SnapLock Compliance Volumes auf MetroCluster gespiegelten Aggregaten nicht unterstützt werden.
* SnapLock Compliance Volumes mit Legal-Hold können nicht mit einem anderen SnapLock Typ geklont werden. Legal-Hold wird nur auf SnapLock Compliance Volumes unterstützt.
* SVM DR unterstützt keine SnapLock Volumes. Der Versuch, einen SnapLock Klon von einem Volume in einer SVM zu erstellen, die Teil einer SVM DR Beziehung ist, schlägt fehl.
* FabricPool Best Practices empfehlen, dass Klone dieselbe Tiering-Richtlinie wie das übergeordnete Volume beibehalten. Ein SnapLock Compliance-Klon eines FabricPool-fähigen Volumes kann jedoch nicht dieselbe Tiering-Richtlinie wie das übergeordnete Volume haben. Die Tiering-Richtlinie muss auf  `none` festgelegt werden. Das Erstellen eines SnapLock Compliance-Klons von einem übergeordneten Volume mit einer anderen Tiering-Richtlinie als  `none` ist nicht möglich.


.Schritte
. Ein FlexClone Volume mit einem SnapLock Typ erstellen: `volume clone create -vserver _svm_name_ -flexclone _flexclone_name_ -type RW [ -snaplock-type {non-snaplock|compliance|enterprise} ]`
+
Beispiel:

+
[listing]
----
> volume clone create -vserver vs0 -flexclone vol1_clone -type RW -snaplock-type enterprise -parent-volume vol1
----
