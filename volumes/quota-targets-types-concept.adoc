= Erfahren Sie mehr über ONTAP Quotenziele und -arten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Jedes Kontingent hat einen spezifischen Typ. Das Kontingentziel wird vom Typ abgeleitet und gibt den Benutzer, die Gruppe oder den qtree an, auf die die Kontingentgrenzen angewendet werden.

Die folgende Tabelle listet die Quotenziele auf, die Quotenarten, mit denen jedes Quotenziel verknüpft ist, sowie die Darstellung jedes Quotenziels.

[cols="15,15,35,35"]
|===


| Quotenziel | Quotentyp | Wie das Ziel dargestellt wird | Hinweise 


 a| 
Benutzer
 a| 
Benutzerkontingent
 a| 
UNIX-Benutzername UNIX UID

Eine Datei oder ein Verzeichnis, dessen UID mit dem Benutzer übereinstimmt

Windows-Benutzername im Format vor Windows 2000

Windows SID

Eine Datei oder ein Verzeichnis mit einer ACL, die dem Benutzer-SID gehört
 a| 
Benutzerkontingente können für ein bestimmtes Volume oder einen bestimmten qtree angewendet werden.



 a| 
Gruppe
 a| 
Gruppenkontingent
 a| 
UNIX Gruppenname UNIX GID

Eine Datei oder ein Verzeichnis, dessen GID mit der Gruppe übereinstimmt
 a| 
Gruppenkontingente können für ein bestimmtes Volume oder einen bestimmten Qtree angewendet werden.


NOTE: ONTAP wendet keine Gruppenkontingente basierend auf Windows-IDs an.



 a| 
qtree
 a| 
Baumkontingent
 a| 
qtree-Name
 a| 
Die Baumquoten gelten für ein bestimmtes Volume und haben keinen Einfluss auf Qtrees in anderen Volumes.



 a| 
`""`
 a| 
* Benutzerkontingent
* Gruppenkontingent
* Baumkontingent

 a| 
Doppelte Anführungszeichen ("")
 a| 
Ein Quotenziel von "" kennzeichnet ein _Standardkontingent_. Für Standardkontingente wird der Kontingenttyp durch den Wert des Feldes type bestimmt.

|===
