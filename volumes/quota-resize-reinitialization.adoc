= Informationen zur Anpassung und Neuinitialisierung von ONTAP Kontingenten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Anpassen der Kontingentgröße ist schneller als die Neuinitialisierung, jedoch ist bei bestimmten Änderungen an Ihren Kontingenten eine vollständige Kontingentneuinitialisierung erforderlich.



[[ontap-quota-resizing]]
== ONTAP Kontingentanpassung

Kontingente lassen sich bei den folgenden Arten von Änderungen an den Kontingentregeln anpassen:

* Ändern eines bestehenden Kontingents.
+
Zum Beispiel die Änderung der Grenzen eines bestehenden Kontingents.

* Hinzufügen eines Kontingents für ein Quotenziel, für das ein Standardkontingent oder ein Standard-Tracking-Kontingent existiert.
* Löschen eines Kontingents, für das ein Standardkontingent oder ein Standard-Tracking-Kontingenteintrag angegeben ist.
* Zusammenführung separater Benutzerkontingente zu einem Mehrbenutzerkontingent.


[NOTE]
====
Nach umfangreichen Änderungen an den Quoten empfiehlt sich eine vollständige Neuinitialisierung, damit alle Änderungen wirksam werden.

====
[NOTE]
====
Wenn Sie versuchen, die Größe zu ändern und nicht alle Ihre Kontingentänderungen durch einen Schreibvorgang übernommen werden können, gibt ONTAP eine Warnung aus. Im Kontingentbericht kann festgestellt werden, ob Ihr Speichersystem die Festplattennutzung für einen bestimmten Benutzer, eine Gruppe oder einen qtree erfasst. Wenn im Kontingentbericht ein Kontingent angezeigt wird, bedeutet dies, dass das Speichersystem den Speicherplatz und die Anzahl der Dateien, die dem Kontingentziel gehören, überwacht.

====
.Beispiele für Quotenänderungen, die durch eine Größenanpassung wirksam werden können
Manche Änderungen der Quotenregeln können durch eine Größenänderung wirksam werden. Folgende Kontingente sind zu berücksichtigen:

[listing]
----

#Quota Target type             disk  files thold sdisk sfile
#------------ ----             ----  ----- ----- ----- -----
*             user@/vol/vol2    50M   15K
*             group@/vol/vol2  750M   85K
*             tree@/vol/vol2      -     -
jdoe          user@/vol/vol2/  100M   75K
kbuck         user@/vol/vol2/  100M   75K
----
Angenommen, Sie nehmen die folgenden Änderungen vor:

* Die Anzahl der Dateien für das Standardbenutzerziel erhöhen.
* Ein neues Benutzerkontingent für einen neuen Benutzer, boris, der ein höheres Speicherkontingent als das Standard-Benutzerkontingent benötigt, hinzufügen.
* Den expliziten Quota-Eintrag des kbuck-Benutzers löschen; der neue Benutzer benötigt jetzt nur noch die Standard-Quota-Limits.


Diese Änderungen führen zu den folgenden Kontingenten:

[listing]
----

#Quota Target type             disk  files thold sdisk sfile
#------------ ----             ----  ----- ----- ----- -----
*             user@/vol/vol2    50M   25K
*             group@/vol/vol2  750M   85K
*             tree@/vol/vol2      -     -
jdoe          user@/vol/vol2/  100M   75K
boris         user@/vol/vol2/  100M   75K
----
Durch die Größenänderung werden all diese Änderungen aktiviert; eine vollständige Neuinitialisierung des Kontingents ist nicht erforderlich.



[[ontap-quota-reinitialization]]
== ONTAP Quota-Reinitialisierung

Obwohl die Anpassung der Kontingente schneller ist, ist eine vollständige Neuinitialisierung der Kontingente erforderlich, wenn bestimmte kleine oder umfangreiche Änderungen an den Kontingenten vorgenommen werden.

Eine vollständige Neuinitialisierung der Quote ist in folgenden Fällen erforderlich:

* Sie erstellen ein Kontingent für ein Ziel, das zuvor kein Kontingent hatte (weder ein explizites Kontingent noch eines, das von einem Standardkontingent abgeleitet ist).
* Der Sicherheitsstil eines qtree wird von UNIX auf entweder gemischt oder NTFS geändert.
* Der Sicherheitsstil eines qtree wird von gemischt oder NTFS auf UNIX geändert.
* Benutzer werden aus einem Kontingentziel mit mehreren Benutzern entfernt oder zu einem Ziel hinzugefügt, das bereits mehrere Benutzer enthält.
* Sie nehmen umfangreiche Änderungen an Ihren Kontingenten vor.


.Beispiel für Quotenänderungen, die eine Initialisierung erfordern
Angenommen, Sie haben ein Volume, das drei qtrees enthält, und die einzigen Kontingente im Volume sind drei explizite qtree-Kontingente. Es werden die folgenden Änderungen vorgenommen:

* Einen neuen qtree hinzufügen und ein neues qtree-Kontingent dafür erstellen.
* Ein Standard-Benutzerkontingent für das Volume hinzufügen.


Beide Änderungen erfordern eine vollständige Quoteninitialisierung. Eine Größenänderung macht die Quoten nicht wirksam.
