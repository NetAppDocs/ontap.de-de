= Konfigurieren Sie ein ONTAP FlexVol Volume so, dass FlexClone Dateien und FlexClone LUNs automatisch gelöscht werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können ein Volume so konfigurieren, dass FlexClone-Dateien und FlexClone-LUNs automatisch gelöscht werden, wenn der freie Speicherplatz im Volume unter einen bestimmten Schwellenwert fällt, und dass das Löschen von Klonen automatisch gestoppt wird, wenn eine Zielmenge an freiem Speicherplatz im Volume wiederhergestellt ist. Obwohl der Schwellenwert, bei dem die automatische Löschung von Klonen beginnt, nicht festgelegt werden kann, lässt sich bestimmen, ob ein Klon zur Löschung geeignet ist, und die Zielmenge an freiem Speicherplatz für ein Volume kann festgelegt werden.

Ein Volume löscht automatisch FlexClone Dateien und FlexClone LUNs, wenn der freie Speicherplatz im Volume unter einen bestimmten Schwellenwert sinkt und wenn _beide_ der folgenden Bedingungen erfüllt sind:

* Die automatische Löschfunktion ist für das Volume aktiviert, das die FlexClone Dateien und FlexClone LUNs enthält.
+
Die automatische Löschfunktion kann für ein FlexVol Volume mithilfe des  `volume snapshot autodelete modify` Befehls aktiviert werden. Der  `-trigger` Parameter muss auf  `volume` oder  `snap_reserve` gesetzt werden, damit ein Volume FlexClone Dateien und FlexClone LUNs automatisch löscht.

* Die automatische Löschfunktion ist für die FlexClone Dateien und FlexClone LUNs aktiviert.
+
Sie können die automatische Löschung für eine FlexClone Datei oder FlexClone LUN mit dem  `file clone create` Befehl mit dem  `-autodelete` Parameter aktivieren. Dadurch ist es möglich, bestimmte FlexClone Dateien und FlexClone LUNs zu erhalten, indem die automatische Löschung für die Klone deaktiviert wird und andere Volume-Einstellungen die Klon-Einstellung nicht überschreiben.



.Bevor Sie beginnen
* Das FlexVol volume muss FlexClone Dateien und FlexClone LUNs enthalten und online sein.
* Das FlexVol volume darf kein schreibgeschütztes Volume sein.


.Schritte
. Automatische Löschung von FlexClone-Dateien und FlexClone-LUNs im FlexVol volume mithilfe des  `volume snapshot autodelete modify`-Befehls aktivieren. Weitere Informationen zu  `volume snapshot autodelete modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-autodelete-modify.html["ONTAP-Befehlsreferenz"^].
+
** Für den  `-trigger`Parameter können  `volume` oder  `snap_reserve` angegeben werden.
** Für den  `-destroy-list`Parameter muss immer  `lun_clone,file_clone` angegeben werden, unabhängig davon, ob nur ein Klontyp gelöscht werden soll. Das folgende Beispiel zeigt, wie das Volume vol1 so konfiguriert werden kann, dass die automatische Löschung von FlexClone Dateien und FlexClone LUNs zur Speicherplatzrückgewinnung ausgelöst wird, bis 25 % des Volumes aus freiem Speicherplatz bestehen:
+
[listing]
----
cluster1::> volume snapshot autodelete modify -vserver vs1 -volume vol1 -enabled true -commitment disrupt -trigger volume -target-free-space 25 -destroy-list lun_clone,file_clone

Volume modify successful on volume:vol1
----
+
[NOTE]
====
Wenn FlexVol Volumes für die automatische Löschung aktiviert werden und der Wert des  `-commitment` Parameters auf  `destroy` gesetzt wird, können alle FlexClone Dateien und FlexClone LUNs, bei denen der  `-autodelete` Parameter auf  `true` gesetzt ist, gelöscht werden, wenn der freie Speicherplatz im Volume unter den angegebenen Schwellenwert sinkt. Jedoch werden FlexClone Dateien und FlexClone LUNs, bei denen der  `-autodelete` Parameter auf  `false` gesetzt ist, nicht gelöscht.

====


. Es sollte überprüft werden, ob die automatische Löschung von FlexClone-Dateien und FlexClone-LUNs im FlexVol Volume mithilfe des  `volume snapshot autodelete show` Befehls aktiviert ist. Weitere Informationen zu  `volume snapshot autodelete show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-autodelete-show.html["ONTAP-Befehlsreferenz"^].
+
Das folgende Beispiel zeigt, dass das Volume vol1 für die automatische Löschung von FlexClone Dateien und FlexClone LUNs aktiviert ist:

+
[listing]
----
cluster1::> volume snapshot autodelete show -vserver vs1 -volume vol1

         Vserver Name: vs1
          Volume Name: vol1
          Enabled: true
           Commitment: disrupt
         Defer Delete: user_created
         Delete Order: oldest_first
  Defer Delete Prefix: (not specified)
    Target Free Space: 25%
              Trigger: volume
     *Destroy List: lun_clone,file_clone*
Is Constituent Volume: false
----
. Stellen Sie sicher, dass die automatische Löschfunktion für die FlexClone Dateien und FlexClone LUNs in dem Volume, das Sie löschen möchten, aktiviert ist, indem Sie die folgenden Schritte ausführen:
+
.. Die automatische Löschung einer bestimmten FlexClone Datei oder FlexClone LUN kann mit dem  `volume file clone autodelete` Befehl aktiviert werden. Weitere Informationen zu  `volume file clone autodelete` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-file-clone-autodelete.html["ONTAP-Befehlsreferenz"^].
+
Es besteht die Möglichkeit, eine bestimmte FlexClone-Datei oder FlexClone LUN automatisch löschen zu lassen, indem der  `volume file clone autodelete` Befehl mit dem  `-force` Parameter verwendet wird.

+
Das folgende Beispiel zeigt, dass die automatische Löschung der FlexClone LUN lun1_clone im Volume vol1 aktiviert ist:

+
[listing]
----
cluster1::> volume file clone autodelete -vserver vs1 -clone-path /vol/vol1/lun1_clone -enabled true
----
+
Die automatische Löschung kann beim Erstellen von FlexClone Dateien und FlexClone LUNs aktiviert werden.

.. Es sollte überprüft werden, ob die FlexClone-Datei oder die FlexClone LUN für die automatische Löschung aktiviert ist, indem der  `volume file clone show-autodelete` Befehl verwendet wird. Weitere Informationen zu  `volume file clone show-autodelete` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-file-clone-show-autodelete.html["ONTAP-Befehlsreferenz"^].
+
Das folgende Beispiel zeigt, dass die FlexClone LUN lun1_clone für die automatische Löschung aktiviert ist:

+
[listing]
----
cluster1::> volume file clone show-autodelete -vserver vs1 -clone-path vol/vol1/lun1_clone
Vserver Name: vs1
Clone Path: vol/vol1/lun1_clone
**Autodelete Enabled: true**
----


+
Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].
