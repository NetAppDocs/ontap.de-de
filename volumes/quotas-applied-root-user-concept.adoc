= Erfahren Sie, wie ONTAP Quotas auf den Root-Benutzer angewendet werden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Root-Benutzer (UID=0) auf UNIX-Clients unterliegt qtree-Kontingenten, jedoch nicht Benutzer- oder Gruppenkontingenten. Dies ermöglicht es dem Root-Benutzer, im Namen anderer Benutzer Aktionen durchzuführen, die andernfalls durch ein Kontingent verhindert würden.

Wenn der Root-Benutzer eine Änderung des Besitzes einer Datei oder eines Verzeichnisses oder eine andere Operation (wie den UNIX `chown`-Befehl) im Namen eines Benutzers mit geringeren Berechtigungen durchführt, prüft ONTAP die Kontingente basierend auf dem neuen Besitzer, meldet jedoch keine Fehler und bricht die Operation nicht ab, selbst wenn die festen Kontingentgrenzen des neuen Besitzers überschritten werden. Dies kann nützlich sein, wenn eine administrative Aktion, wie die Wiederherstellung verlorener Daten, zu einer vorübergehenden Überschreitung der Kontingente führt.

[NOTE]
====
Nach erfolgter Eigentumsübertragung meldet das Client-System jedoch einen Speicherplatzfehler, wenn der Benutzer versucht, mehr Speicherplatz zuzuweisen, während das Kontingent noch überschritten ist.
