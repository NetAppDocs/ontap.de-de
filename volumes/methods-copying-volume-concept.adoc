= Methoden zum Kopieren eines ONTAP Volume
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Methode zum Kopieren eines Volumes hängt davon ab, ob es auf dasselbe Aggregat oder auf ein anderes Aggregat kopiert wird und ob Snapshots des ursprünglichen Volumes beibehalten werden sollen. Das Kopieren eines Volumes erstellt eine eigenständige Kopie eines Volumes, die für Tests und andere Zwecke verwendet werden kann.

Die folgende Tabelle listet die Merkmale der Kopie und die Methoden auf, die zur Erstellung dieser Kopie verwendet wurden.

[cols="2*"]
|===
| Wenn Sie ein Volume kopieren möchten... | Dann ist die Methode, die Sie anwenden, ... 


 a| 
Innerhalb desselben Aggregats und es sollen keine Snapshots vom ursprünglichen Volume kopiert werden.
 a| 
Erstellen eines FlexClone Volumens des ursprünglichen Volumens.



 a| 
Zu einem anderen Aggregat und ohne Snapshots vom ursprünglichen Volume zu kopieren.
 a| 
Erstellen eines FlexClone Volume des ursprünglichen Volumes und anschließendes Verschieben des Volumes in ein anderes Aggregat mit dem  `volume move` Befehl.



 a| 
Zu einem anderen Aggregat verschieben und alle Snapshots des ursprünglichen Volumes beibehalten.
 a| 
Das ursprüngliche Volume wird mithilfe von SnapMirror repliziert, und anschließend wird die SnapMirror Beziehung aufgehoben, um eine Lese-/Schreibkopie des Volumes zu erstellen.

|===
