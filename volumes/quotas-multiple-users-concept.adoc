= Erfahren Sie, wie ONTAP Kontingente mit mehreren Benutzern funktionieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn mehrere Benutzer demselben Kontingentziel zugeordnet werden, gelten die im Kontingent festgelegten Grenzwerte nicht für jeden einzelnen Benutzer. Stattdessen werden die Kontingentgrenzen von allen Benutzern im Kontingentziel gemeinsam genutzt.

Anders als bei Befehlen zur Verwaltung von Objekten, wie Volumes und Qtrees, kann ein Quota-Ziel, einschließlich eines Mehrbenutzer-Quotas, nicht umbenannt werden. Das bedeutet, dass nach der Definition eines Mehrbenutzer-Quotas die Benutzer im Quota-Ziel nicht geändert werden können und Benutzer weder zu einem Ziel hinzugefügt noch aus einem Ziel entfernt werden können. Wenn ein Benutzer zu einem Mehrbenutzer-Quota hinzugefügt oder daraus entfernt werden soll, muss das Quota, das diesen Benutzer enthält, gelöscht und eine neue Quota-Regel mit der festgelegten Benutzergruppe im Ziel definiert werden.

[NOTE]
====
Wenn Sie separate Benutzerkontingente zu einem Mehrbenutzerkontingent zusammenfassen, kann die Änderung durch Anpassen der Kontingente aktiviert werden. Wenn Benutzer aus einem Kontingentziel mit mehreren Benutzern entfernt oder Benutzer zu einem Ziel hinzugefügt werden, das bereits mehrere Benutzer enthält, müssen die Kontingente neu initialisiert werden, bevor die Änderung wirksam wird.

====
.Beispiel für mehr als einen Benutzer in einer Quotenregel
Im folgenden Beispiel sind zwei Benutzer in der Kontingentliste aufgeführt. Die beiden Benutzer können zusammen bis zu 80 MB Speicherplatz nutzen. Belegt einer 75 MB, stehen dem anderen nur 5 MB zur Verfügung.

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target "jsmith,chen" -qtree "" -disk-limit 80m

cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Type   Target         Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
