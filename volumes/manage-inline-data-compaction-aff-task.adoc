= Inline-Datenkomprimierung für AFF Systeme verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Inline-Datenkomprimierung auf AFF-Systemen kann auf Volume-Ebene mit dem  `volume efficiency modify` Befehl gesteuert werden. Die Datenkomprimierung ist standardmäßig für alle Volumes auf AFF-Systemen aktiviert.

.Bevor Sie beginnen
Die Datenkomprimierung erfordert, dass die Speicherplatzzusage auf  `none` gesetzt ist. Dies ist die Standardeinstellung für AFF Systeme.

[NOTE]
====
Die standardmäßige Speicherplatzzusage für Nicht-AFF-Datensicherungsvolumes ist auf keine gesetzt.

====
.Schritte
. Zur Überprüfung der Speicherplatzzusage-Einstellung für das Volume:
+
`volume show -vserver vserver_name -volume volume_name –fields space-guarantee`

. So wird die Datenkomprimierung aktiviert:
+
`volume efficiency modify -vserver vserver_name -volume volume_name -data-compaction true`

. So wird die Datenkomprimierung deaktiviert:
+
`volume efficiency modify -vserver vserver_name -volume volume_name -data-compaction false`

. Anzeigen des Status der Datenkomprimierung:
+
`volume efficiency show -instance`



.Beispiele
`cluster1::> volume efficiency modify -vserver vs1 -volume vol1 -data-compaction true` `cluster1::> volume efficiency modify -vserver vs1 -volume vol1 -data-compaction false`
