= Legen Sie den ONTAP-Speichereffizienzmodus während der Volume-Erstellung fest
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.10.1 kann der Speichereffizienzmodus beim Erstellen eines neuen AFF Volume festgelegt werden.

.Über diese Aufgabe
Sie können den Speichereffizienzmodus auf einem neuen AFF Volume mit dem Parameter `-storage-efficiency-mode` steuern. Es stehen zwei Optionen zur Auswahl, um den Speichereffizienzmodus festzulegen:  `default` oder  `efficient`. Die Wahl des Speichereffizienzmodus hängt davon ab, ob eine höhere Performance oder eine höhere Speichereffizienz für das Volume gewünscht wird. Der Parameter `-storage-efficiency-mode` wird auf Nicht-AFF Volumes oder auf Data Protection Volumes nicht unterstützt.

Der Leistungsmodus wird standardmäßig festgelegt, wenn neue AFF Volumes mit Speichereffizienz erstellt werden.

link:enable-temperature-sensitive-efficiency-concept.html["Weitere Informationen zu temperaturabhängiger Speichereffizienz und Speichereffizienzmodi"].

.Schritte
. Ein neues Volume erstellen und den Effizienzmodus festlegen:
+
[source, cli]
----
volume create -vserver <vserver name> -volume <volume name> -aggregate <aggregate name> -size <volume size> -storage-efficiency-mode <efficient|default>
----
+
Setzen Sie  `-storage-efficiency-mode` auf  `efficient` für den Effizienzmodus oder auf  `default` für den Leistungsmodus.

+
Im folgenden Beispiel wird aff_vol1 mit Effizienzmodus erstellt.



[listing]
----
volume create -vserver vs1 -volume aff_vol1 -aggregate aff_aggr1 -storage-efficiency-mode efficient -size 10g
----
