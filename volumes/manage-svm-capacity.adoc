= ONTAP SVM-Kapazitätsgrenzen verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.13.1 kann eine maximale Kapazität für eine Storage-VM (SVM) festgelegt werden. Es besteht außerdem die Möglichkeit, Warnmeldungen zu konfigurieren, wenn die SVM einen Schwellenwert für die Kapazität erreicht.

.Über diese Aufgabe
Die Kapazität einer SVM berechnet sich als Summe der FlexVols, FlexGroup Volumes, FlexClones, FlexCache Volumes. Volumes wirken sich auf die Kapazitätsberechnung aus, auch wenn sie eingeschränkt, offline oder nach dem Löschen in der Wiederherstellungswarteschlange sind. Wenn Volumes mit automatischer Größenanpassung konfiguriert sind, wird der maximale Wert für die automatische Größenanpassung des Volumes bei der Berechnung der SVM-Größe berücksichtigt; ohne automatische Größenanpassung wird die tatsächliche Größe des Volumes berechnet.

Die folgende Tabelle zeigt, wie  `autosize-mode` Parameter die Kapazitätsberechnung beeinflussen.

|===


| `autosize-mode off` | Der Größenparameter wird für die Berechnung verwendet 


| `autosize-mode grow` | Der  `max-autosize` Parameter wird für die Berechnung verwendet 


| `autosize-mode grow-shrink` | Der  `max-autosize` Parameter wird für die Berechnung verwendet 
|===
.Bevor Sie beginnen
* Um ein SVM-Limit festzulegen, ist die Rolle als Cluster-Administrator erforderlich.
* Ab ONTAP 9.16.1 können Speichergrenzen für SVMs konfiguriert werden, die Datensicherungs-Volumes enthalten, einschließlich der folgenden Datensicherungstypen:
+
** FlexVol Volumes in asynchroner DR ohne Kaskade
** FlexVol Volumes im synchronen DR (sowohl synchron als auch strikt-synchron Richtlinien)
** link:../data-protection/restore-volume-snapvault-backup-task.html["Wiederherstellen"]


* Speicherbeschränkungen für SVMs werden für die folgenden Konfigurationen _nicht_ unterstützt:
+
** SnapMirror Vault-Beziehungen
** SnapMirror Active Sync
** FlexGroup Volumes
** Konsistenzgruppen
** SVM DR
** Kaskaden
** MetroCluster


* Ab ONTAP 9.16.1 kann beim Erstellen einer Load-Sharing-Mirror-Beziehung für die Ziel-SVM kein Speicherlimit aktiviert sein.
* Bei der Migration einer SVM darf für die Quell-SVM kein Speicherlimit aktiviert sein. Zum Abschluss der Migration ist das Speicherlimit auf der Quell-SVM zu deaktivieren und anschließend die Migration abzuschließen.
* Die SVM-Kapazität ist von xref:../volumes/quotas-concept.html[Quoten] zu unterscheiden. Quoten dürfen die maximale Größe nicht überschreiten.
* Sie können kein Speicherlimit festlegen, wenn andere Vorgänge auf der SVM ausgeführt werden. Mit dem  `job show vserver <svm_name>` Befehl lassen sich vorhandene Aufträge anzeigen. Der Befehl kann erneut ausgeführt werden, sobald alle Aufträge abgeschlossen sind. Weitere Informationen zu  `job show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/job-show.html["ONTAP-Befehlsreferenz"^].




[[capacity-impact]]
== Kapazitätsauswirkungen

Wenn die Kapazitätsgrenze erreicht ist, schlagen die folgenden Operationen fehl:

* Erstellen einer LUN, eines Namespace oder eines Volumes
* Klonen einer LUN, eines Namespace oder eines Volumes
* Ändern einer LUN, eines Namespace, oder eines Volumes
* Vergrößern der Größe einer LUN, eines Namespace oder eines Volumes
* Erweitern einer LUN, eines Namespace oder eines Volumes
* Neuhosting einer LUN, eines Namespace oder eines Volume




[[set-a-capacity-limit-on-a-new-svm]]
== Eine Kapazitätsgrenze für eine neue SVM festlegen

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. Wählen Sie *Storage* > *Storage VMs*.
. Wählen Sie image:icon_add_blue_bg.gif["Symbol hinzufügen"], um die SVM zu erstellen.
. Die SVM benennen und ein *Zugriffsprotokoll* auswählen.
. Unter *Speicher-VM-Einstellungen* die Option *Maximale Kapazitätsgrenze aktivieren* auswählen.
+
Eine maximale Kapazitätsgröße für die SVM angeben.

. *Speichern* auswählen.


--
.CLI
--
.Schritte
. Die SVM wird erstellt. Zum Festlegen eines Speicherlimits ist ein  `storage-limit` Wert anzugeben. Für die Festlegung einer Schwellenwertwarnung für das Speicherlimit ist ein Prozentwert für  `-storage-limit-threshold-alert` anzugeben.
+
[source, cli]
----
vserver create -vserver <vserver_name> -aggregate <aggregate_name> -rootvolume <root_volume_name> -rootvolume-security-style {unix|ntfs|mixed} -storage-limit <value> [GiB|TIB] -storage-limit-threshold-alert <percentage> [-ipspace <IPspace_name>] [-language <language>] [-snapshot-policy <snapshot_policy_name>] [-quota-policy <quota_policy_name>] [-comment <comment>]
----
+
Wird kein Schwellenwert angegeben, wird standardmäßig eine Warnung ausgelöst, sobald die SVM zu 90 % ausgelastet ist. Zum Deaktivieren der Schwellenwertwarnung ist der Wert Null anzugeben.

. Es wird bestätigt, dass die SVM erfolgreich erstellt wurde:
+
[source, cli]
----
vserver show -vserver <vserver_name>
----
. Wenn die Speicherbegrenzung deaktiviert werden soll, ist der  `-storage-limit` Parameter für die SVM auf Null zu setzen:
+
[source, cli]
----
vserver modify -vserver <vserver_name> -storage-limit 0
----


--
====


[[set-or-modify-a-capacity-limit-on-an-existing-svm]]
== Eine Kapazitätsgrenze für eine bestehende SVM festlegen oder ändern

Sie können für eine bestehende SVM eine Kapazitätsgrenze und einen Schwellenwertalarm festlegen oder eine Kapazitätsgrenze deaktivieren.

Sobald Sie das Kapazitätslimit festgelegt haben, kann das Limit nicht mehr auf einen Wert unterhalb der aktuell zugewiesenen Kapazität geändert werden.

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. Wählen Sie *Storage* > *Storage VMs*.
. Wählen Sie die SVM aus, die Sie bearbeiten möchten. Neben dem Namen der SVM image:icon_kabob.gif["Symbol für Menüoptionen"] dann *Bearbeiten* auswählen.
. Um eine Kapazitätsbegrenzung zu aktivieren, das Kontrollkästchen neben *Kapazitätsbegrenzung aktivieren* auswählen. Einen Wert für die *Maximale Kapazität* und einen Prozentwert für den *Warnschwellenwert* eingeben.
+
Wenn die Kapazitätsbegrenzung deaktiviert werden soll, ist das Häkchen bei *Kapazitätsbegrenzung aktivieren* zu entfernen.

. *Speichern* auswählen.


--
.CLI
--
.Schritte
. Auf dem Cluster, das die SVM hostet, den  `vserver modify` Befehl ausführen. Einen numerischen Wert für  `-storage-limit` und einen Prozentwert für  `-storage-limit-threshold-alert` angeben.
+
[source, cli]
----
vserver modify -vserver <vserver_name> -storage-limit <value> [GiB|TIB] -storage-limit-threshold-alert <percentage>
----
+
Wenn Sie keinen Schwellenwert angeben, erfolgt standardmäßig eine Warnung bei 90 % Kapazität. Zum Deaktivieren der Schwellenwertwarnung ist der Wert Null anzugeben.

. Wenn die Speicherbegrenzung deaktiviert werden soll, ist der Wert  `-storage-limit` für die SVM auf Null zu setzen:
+
[source, cli]
----
vserver modify -vserver <vserver_name> -storage-limit 0
----


--
====


[[reaching-capacity-limits]]
== Erreichen der Kapazitätsgrenzen

Wenn die maximale Kapazität oder der Alarmschwellenwert erreicht ist, können die  `vserver.storage.threshold` EMS-Meldungen konsultiert oder die *Insights*-Seite im System Manager genutzt werden, um sich über mögliche Maßnahmen zu informieren. Mögliche Lösungen sind:

* Bearbeiten der maximalen Kapazitätsgrenzen der SVM
* Leeren der Wiederherstellungswarteschlange der Volumes zur Freigabe von Speicherplatz
* Snapshot löschen, um Speicherplatz für das Volume bereitzustellen
