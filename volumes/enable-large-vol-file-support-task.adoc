= ONTAP Unterstützung für große Volumes und große Dateien aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.12.1 P2 ist es möglich, ein neues Volume zu erstellen oder ein vorhandenes Volume zu modifizieren, um die Unterstützung für eine maximale Volumegröße von 300 TB, eine maximale link:../flexgroup/definition-concept.html["FlexGroup-Volume"]-Größe von 60 PB und eine maximale Dateigröße (LUN) von 128 TB zu ermöglichen.

.Bevor Sie beginnen
* Auf dem Cluster ist ONTAP 9.12.1 P2 oder höher installiert.
* Wenn die Unterstützung für große Volumes auf dem Quell-Cluster in einer SnapMirror Beziehung aktiviert wird, muss ONTAP 9.12.1 P2 oder höher sowohl auf dem Cluster, das das Quell-Volume hostet, als auch auf dem Cluster, das das Ziel-Volume hostet, installiert sein.
* Sie sind Administrator eines Clusters oder einer SVM.
* Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].




[[create-a-new-volume]]
== Ein neues Volume erstellen

.Schritt
. Ein Volume mit aktivierter Unterstützung für große Volumes und Dateien erstellen:
+
[source, cli]
----
volume create -vserver <svm_name> -volume <volume_name> -aggregate <aggregate_name> -is-large-size-enabled true
----


.Beispiel
Das folgende Beispiel erstellt ein neues Volume mit aktivierter Unterstützung für große Volume- und Dateigrößen.

[listing]
----
volume create -vserver vs1 -volume big_vol1 -aggregate aggr1 -is-large-size-enabled true
----


[[modify-an-existing-volume]]
== Ein vorhandenes Volume ändern

.Schritt
. Ein Volume so modifizieren, dass die Unterstützung für große Volumes und Dateien aktiviert ist:
+
[source, cli]
----
volume modify -vserver <svm_name> -volume <volume_name> -is-large-size-enabled true
----
+
.Beispiel
Das folgende Beispiel modifiziert ein bestehendes Volume, sodass große Volume und Dateigrößen unterstützt werden.

+
[listing]
----
volume modify -vserver vs2 -volume data_vol -is-large-size-enabled true
----


.Was kommt als Nächstes?
Wenn Sie die Unterstützung für große Volumes und Dateien für einen NFS-Client aktiviert haben, muss der Client neu eingebunden werden, bevor die neue maximale Dateigröße wirksam wird.
