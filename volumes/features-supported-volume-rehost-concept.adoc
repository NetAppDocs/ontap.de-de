= ONTAP-Funktionen werden bei einem Volume-Rehost nicht unterstützt
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es gibt mehrere ONTAP Funktionen, die das Rehosting von Volumes nicht unterstützen. Diese Funktionen sollten bekannt sein, bevor ein Rehosting Vorgang durchgeführt wird.

Die folgenden Funktionen werden bei einem Volume Rehost nicht unterstützt:

* SVM DR
* MetroCluster Konfigurationen
+

NOTE: Das Klonen eines Volumes als FlexClone volume auf einer anderen SVM wird auf MetroCluster Konfigurationen ebenfalls nicht unterstützt.

* SnapLock Volumes
* NetApp Volume Encryption (NVE) Volumes (in Versionen von ONTAP vor 9.8)
+
In ONTAP Versionen vor 9.8 muss das Volume vor dem erneuten Hosten entschlüsselt werden. Die Volume-Verschlüsselungsschlüssel hängen von den SVM-Schlüsseln ab. Wenn ein Volume auf eine andere SVM verschoben wird und die mandantenfähige Schlüsselkonfiguration entweder auf der Quell- oder der Ziel-SVM aktiviert ist, stimmen die Volume- und SVM-Schlüssel nicht überein.

+
Ab ONTAP 9.8 ist das Rehosting eines Volumes mit NVE möglich.

* FlexGroup Volumes
* Klon-Volumes
