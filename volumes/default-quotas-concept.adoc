= Erfahren Sie mehr über ONTAP Standardkontingente
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Standardkontingente verwenden, um ein Kontingent auf alle Instanzen eines bestimmten Kontingenttyps anzuwenden. Beispielsweise wirkt sich ein Standard-Benutzerkontingent auf alle Benutzer im System für das angegebene FlexVol volume oder den angegebenen qtree aus. Zusätzlich ermöglichen Standardkontingente eine einfache Anpassung der Kontingente.

Sie können Standardkontingente verwenden, um automatisch ein Limit für eine große Anzahl von Kontingentzielen festzulegen, ohne für jedes Ziel separate Kontingente erstellen zu müssen. Wenn für die meisten Benutzer der Speicherplatz auf 10 GB begrenzt werden soll, kann stattdessen ein Standard-Benutzerkontingent von 10 GB festgelegt werden, anstatt für jeden Benutzer ein Kontingent zu erstellen. Für bestimmte Benutzer, für die ein anderes Limit gelten soll, können explizite Kontingente für diese Benutzer erstellt werden. (Explizite Kontingente (Kontingente mit einem bestimmten Ziel oder einer Liste von Zielen) setzen Standardkontingente außer Kraft.)

Darüber hinaus ermöglichen Standardkontingente die Verwendung der Größenänderung anstelle der Neuinitialisierung, wenn Sie möchten, dass Kontingentänderungen wirksam werden. Wenn beispielsweise einem Volume, das bereits über ein Benutzerkontingent verfügt, ein explizites Benutzerkontingent hinzugefügt wird, kann das neue Kontingent durch eine Größenänderung aktiviert werden.

Standardkontingente können auf alle drei Arten von Kontingentzielen (Benutzer, Gruppen und Qtrees) angewendet werden.

Standardkontingente haben nicht unbedingt festgelegte Grenzwerte; ein Standardkontingent kann ein Tracking-Kontingent sein.

Eine Quote wird durch ein Ziel angegeben, das je nach Kontext entweder eine leere Zeichenkette ("") oder ein Sternchen (*) ist:

* Wenn mit dem  `volume quota policy rule create` Befehl ein Kontingent erstellt wird und der  `-target` Parameter auf eine leere Zeichenkette ("") gesetzt wird, entsteht ein Standardkontingent.
+
Weitere Informationen zu  `volume quota policy rule create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-policy-rule-create.html["ONTAP-Befehlsreferenz"^].

* Im  `volume quota policy rule create` Befehl gibt der  `-qtree` Parameter den Namen des Qtree an, auf den die Quotenregel angewendet wird. Dieser Parameter ist für Regeln vom Typ „Baum“ nicht anwendbar. Für Regeln vom Typ „Benutzer“ oder „Gruppe“ auf Volume-Ebene sollte dieser Parameter „“ enthalten.
* In der Ausgabe des  `volume quota policy rule show` Befehls erscheint ein Standardkontingent mit einer leeren Zeichenkette ("") als Ziel.
+
Weitere Informationen zu  `volume quota policy rule show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-policy-rule-show.html["ONTAP-Befehlsreferenz"^].

* In der Ausgabe des  `volume quota report`-Befehls wird ein Standardkontingent mit einem Sternchen (*) als ID und Kontingentbezeichner angezeigt.
+
Weitere Informationen zu  `volume quota report` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-report.html["ONTAP-Befehlsreferenz"^].





[[default-user-quota-example]]
== Beispiel für ein Standard-Benutzerkontingent

Die folgende Quotenregel verwendet ein Standard-Benutzerkontingent, um für jeden Benutzer ein Limit von 50 MB für vol1 festzulegen:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target "" -qtree "" -disk-limit 50m

cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Vserver: vs0               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   ""        ""      off          50MB        -       -        -          -
----
Wenn ein Benutzer auf dem System einen Befehl eingibt, der dazu führen würde, dass die Daten dieses Benutzers mehr als 50 MB in vol1 belegen (zum Beispiel durch Schreiben in eine Datei aus einem Editor), schlägt der Befehl fehl.
