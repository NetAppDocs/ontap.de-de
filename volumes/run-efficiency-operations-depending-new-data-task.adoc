= Führen Sie einen ONTAP-Effizienzvorgang basierend auf der Menge der neu geschriebenen Daten durch
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den Zeitplan für Effizienzmaßnahmen so anpassen, dass Deduplizierung oder Datenkomprimierung ausgeführt wird, wenn die Anzahl der nach der vorherigen Effizienzmaßnahme auf das Volume geschriebenen neuen Blöcke einen festgelegten Schwellenwert überschreitet. Dies gilt unabhängig davon, ob die vorherige Effizienzmaßnahme manuell oder planmäßig durchgeführt wurde.

.Über diese Aufgabe
Wenn die  `schedule` Option auf  `auto` gesetzt ist, wird die geplante Effizienzoperation ausgeführt, wenn die Menge neuer Daten den angegebenen Prozentsatz überschreitet. Der Standard-Schwellenwert beträgt 20 Prozent. Dieser Schwellenwert ist der Prozentsatz der Gesamtzahl der bereits von der Effizienzoperation verarbeiteten Blöcke.

.Schritte
. Mit dem  `volume efficiency modify`Befehl und der  `auto@num`Option lässt sich der Schwellenwert-Prozentsatz anpassen.
+
`num` ist eine zweistellige Zahl zur Angabe des Prozentsatzes.



.Beispiel
Der folgende Befehl ändert den Schwellenwertprozentsatz für das Volumen VolA auf 30 Prozent:

`volume efficiency modify -vserver vs1 -volume -VolA -schedule auto@30`
