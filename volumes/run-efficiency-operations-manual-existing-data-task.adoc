= Eine Effizienzoptimierung kann manuell auf vorhandenen Daten durchgeführt werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Effizienzmaßnahmen manuell auf den Daten ausführen, die sich in nicht temperaturempfindlichen Storage Efficiency Volumes befinden, bevor Deduplizierung, Datenkomprimierung oder Datenverdichtung aktiviert wird. Diese Maßnahmen können mit ONTAP Versionen vor ONTAP 9.8 ausgeführt werden.

.Über diese Aufgabe
Dieser Vorgang wird mit dem  `volume efficiency start` Befehl mit dem  `-scan-old-data` Parameter durchgeführt. Die  `-compression` Option funktioniert nicht mit  `-scan-old-data` auf temperaturempfindlichen Storage-Effizienz-Volumes. Die inaktive Datenkomprimierung läuft bei temperaturempfindlichen Storage-Effizienz-Volumes in ONTAP 9.8 und höher automatisch auf bereits vorhandenen Daten.

Wenn nur die Deduplizierung für ein Volume aktiviert wird, läuft die Deduplizierung auf den Daten. Wenn Deduplizierung, Datenkomprimierung und Datenverdichtung für ein Volume aktiviert werden, erfolgt zuerst die Datenkomprimierung, gefolgt von der Deduplizierung und der Datenverdichtung.

Wenn Sie die Datenkomprimierung auf vorhandene Daten anwenden, überspringt der Datenkomprimierungsvorgang standardmäßig die durch Deduplizierung gemeinsam genutzten Datenblöcke sowie die durch Snapshots gesperrten Datenblöcke. Wenn die Datenkomprimierung auf gemeinsam genutzte Blöcke angewendet wird, ist die Optimierung deaktiviert und die Fingerabdruckinformationen werden erfasst und erneut für die gemeinsame Nutzung verwendet. Das Standardverhalten der Datenkomprimierung beim Komprimieren vorhandener Daten kann geändert werden.

Pro Knoten können maximal acht Deduplizierungs-, Datenkomprimierungs- oder Datenverdichtungsvorgänge gleichzeitig ausgeführt werden. Die übrigen Vorgänge werden in eine Warteschlange gestellt.

[NOTE]
====
Die Nachbearbeitungskomprimierung wird auf AFF Plattformen nicht ausgeführt. Eine EMS-Meldung wird generiert, um darüber zu informieren, dass dieser Vorgang übersprungen wurde.

====
Weitere Informationen zu  `volume efficiency start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-efficiency-start.html["ONTAP-Befehlsreferenz"^].

.Schritte
. Mit dem  `volume efficiency start -scan-old-data` Befehl kann die Deduplizierung, Datenkomprimierung oder Datenverdichtung manuell auf die vorhandenen Daten ausgeführt werden.
+
Mit dem folgenden Befehl ist es möglich, diese Operationen manuell auf den vorhandenen Daten im Volume VolA auszuführen:

+
`volume efficiency start -vserver vs1 -volume VolA -scan-old-data true [-compression | -dedupe | -compaction ] true`
