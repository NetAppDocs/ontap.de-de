= Automatisches Löschen einer ONTAP FlexClone Datei oder FlexClone LUN verhindern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie ein FlexVol volume so konfigurieren, dass FlexClone Dateien und FlexClone LUNs automatisch gelöscht werden, kann jeder Klon, der den von Ihnen angegebenen Kriterien entspricht, gelöscht werden. Wenn bestimmte FlexClone Dateien oder FlexClone LUNs erhalten bleiben sollen, können diese vom automatischen FlexClone Löschvorgang ausgeschlossen werden.

.Bevor Sie beginnen
Eine FlexClone Lizenz muss installiert sein. Diese Lizenz ist mit link:../system-admin/manage-licenses-concept.html#licenses-included-with-ontap-one["ONTAP One"] enthalten.

.Über diese Aufgabe
Beim Erstellen einer FlexClone Datei oder FlexClone LUN ist die automatische Löschfunktion für den Klon standardmäßig deaktiviert. FlexClone Dateien und FlexClone LUNs, bei denen die automatische Löschfunktion deaktiviert ist, bleiben erhalten, wenn ein FlexVol Volume so konfiguriert wird, dass Klone automatisch gelöscht werden, um Speicherplatz auf dem Volume freizugeben.

[NOTE]
====
Wenn Sie die  `commitment` Ebene des Volumes auf  `try` oder  `disrupt` setzen, können bestimmte FlexClone-Dateien oder FlexClone-LUNs einzeln erhalten bleiben, indem für diese Klone die automatische Löschung deaktiviert wird. Wenn Sie jedoch die  `commitment` Ebene des Volumes auf  `destroy` setzen und die zerstören-Listen  `lun_clone,file_clone` enthalten, überschreibt die Volume-Einstellung die Klon-Einstellung, sodass alle FlexClone-Dateien und FlexClone-LUNs unabhängig von der Einstellung für die automatische Löschung der Klone gelöscht werden können.

====
.Schritte
. Das automatische Löschen einer bestimmten FlexClone-Datei oder FlexClone-LUN kann durch Verwendung des  `volume file clone autodelete` Befehls verhindert werden.
+
Das folgende Beispiel zeigt, wie die automatische Löschung für FlexClone LUN lun1_clone, die in vol1 enthalten ist, deaktiviert werden kann:

+
[listing]
----
cluster1::> volume file clone autodelete -vserver vs1 -volume vol1 -clone-path lun1_clone -enable false
----
+
Eine FlexClone Datei oder FlexClone LUN mit deaktivierter automatischer Löschung kann nicht automatisch gelöscht werden, um Speicherplatz auf dem Volume freizugeben.

. Es sollte überprüft werden, dass Autodelete für die FlexClone-Datei oder FlexClone-LUN mit dem  `volume file clone show-autodelete` Befehl deaktiviert ist.
+
Das folgende Beispiel zeigt, dass autodelete für die FlexClone LUN lun1_clone auf „false“ gesetzt ist:

+
[listing]
----
cluster1::> volume file clone show-autodelete -vserver vs1 -clone-path vol/vol1/lun1_clone
															Vserver Name: vs1
															Clone Path: vol/vol1/lun1_clone
															Autodelete Enabled: false
----
