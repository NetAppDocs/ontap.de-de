= Steuern und überwachen Sie die ONTAP FlexVol Volume-I/O-Leistung mit Storage QoS
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Ein-/Ausgabeleistung (E/A) von FlexVol volumes steuern, indem Sie Volumes Storage QoS-Policy-Gruppen zuweisen. Die Steuerung der E/A-Leistung kann dazu dienen, sicherzustellen, dass Workloads bestimmte Leistungsziele erreichen, oder um einen Workload zu drosseln, der andere Workloads beeinträchtigt.

.Über diese Aufgabe
Richtliniengruppen erzwingen eine maximale Durchsatzbegrenzung (z. B. 100 MB/s). Sie können eine Richtliniengruppe erstellen, ohne eine maximale Durchsatzbegrenzung anzugeben, wodurch Sie die Leistung überwachen können, bevor Sie die Arbeitslast steuern. Sie können auch eine optionale minimale Durchsatzbegrenzung angeben.

SVMs, LUNs und Dateien können ebenfalls Richtliniengruppen zugeordnet werden.

Beachten Sie die folgenden Anforderungen für die Zuordnung eines Volumes zu einer Richtliniengruppe:

* Das Volume muss von der SVM, zu der die Policy-Gruppe gehört, enthalten sein.
+
Die SVM wird beim Erstellen der Richtliniengruppe angegeben.

* Ab ONTAP 9.18.1 können Sie QoS-Richtlinien Volumes zuweisen, die in SVMs mit QoS-Richtlinien enthalten sind. Wenn Sie verschachtelte QoS-Richtlinien verwenden, wird die restriktivste Richtlinie angewendet.
* Ab ONTAP 9.14.0 können Sie Richtlinien Qtrees zuweisen, die in Volumes enthalten sind, die QoS-Richtlinien haben.


Weitere Informationen zur Verwendung von Storage QoS sind unter link:../system-admin/index.html["Systemadministrationsreferenz"] zu finden.

.Schritte
. Mit dem  `qos policy-group create` Befehl lässt sich eine Richtliniengruppe erstellen.
. Der  `volume create` Befehl oder der  `volume modify` Befehl mit dem  `-qos-policy-group` Parameter kann verwendet werden, um ein Volume einer Richtliniengruppe zuzuordnen.
. Die  `qos statistics` Befehle ermöglichen die Anzeige von Leistungsdaten.
. Gegebenenfalls kann der  `qos policy-group modify` Befehl verwendet werden, um die maximale Durchsatzbegrenzung der Policy-Gruppe anzupassen.
