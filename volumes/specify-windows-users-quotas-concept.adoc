= Windows-Benutzer für ONTAP-Kontingente festlegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für ein Kontingent kann ein Windows-Benutzer in einem von mehreren verschiedenen Formaten angegeben werden.

Die drei verfügbaren Formate zur Angabe eines Windows-Benutzers für ein Benutzerkontingent sind:

* Der Windows Name im Format vor Windows 2000.
* Die Sicherheits-ID (SID), wie sie von Windows in Textform angezeigt wird, z. B. `S-1-5-32-544`.
* Der Name einer Datei oder eines Verzeichnisses, dessen ACL dem SID dieses Benutzers gehört.
+
[NOTE]
====
Wenn Sie einen Datei- oder Verzeichnisnamen angeben, muss eine Datei oder ein Verzeichnis ausgewählt werden, das so lange besteht, wie das Benutzerkonto auf dem System vorhanden ist.

Damit ONTAP die SID aus der ACL abrufen kann, muss die ACL gültig sein.

Wenn die Datei oder das Verzeichnis in einem UNIX-artigen Qtree existiert oder wenn das Speichersystem den UNIX-Modus für die Benutzerauthentifizierung verwendet, wendet ONTAP das Benutzerkontingent auf den Benutzer an, dessen *UID* und nicht SID mit der von Datei oder Verzeichnis übereinstimmt.

Die Angabe eines Datei- oder Verzeichnisnamens zur Identifizierung eines Benutzers für ein Kontingent führt nicht dazu, dass ONTAP ein Kontingent auf diese Datei oder dieses Verzeichnis anwendet.

====
