= ONTAP Volumeneffizienzmodus ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.10.1 werden die Speichereffizienzmodi _default_ und _efficient_ auf Volume-Ebene ausschließlich für AFF Systeme unterstützt. Diese Modi bieten die Wahl zwischen Dateikomprimierung (_default_), dem Standardmodus beim Erstellen neuer AFF Volumes, oder temperaturabhängiger Speichereffizienz (_efficient_), die temperaturabhängige Speichereffizienz (TSSE) aktiviert.


NOTE: TSSE wird nur auf Thin-Provisioning-Volumes unterstützt. link:enable-temperature-sensitive-efficiency-concept.html["Weitere Informationen zu TSSE"].

.Schritte
Diese Aufgabe kann mit ONTAP System Manager oder der ONTAP CLI durchgeführt werden.

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.10.1 kann mit System Manager eine höhere Speichereffizienz mithilfe der temperaturabhängigen Speichereffizienzfunktion aktiviert werden. Die leistungsbasierte Speichereffizienz ist standardmäßig aktiviert.

. *Speicher > Volumes* auswählen.
. Suchen Sie das Volume, für das die Speichereffizienz aktiviert oder deaktiviert werden soll, und klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"].
. *Bearbeiten > Volumes* auswählen und zu *Speichereffizienz* scrollen.
. *Höhere Speichereffizienz aktivieren* auswählen.


--
.CLI
--
Sie können den  `volume efficiency modify`Befehl verwenden, um den Speichereffizienzmodus für ein AFF Volume von  `default` auf  `efficient` zu ändern, oder einen Effizienzmodus festlegen, wenn die Volume-Effizienz noch nicht eingestellt ist.

. Den Volume-Effizienzmodus ändern:
+
[listing]
----
volume efficiency modify -vserver <vserver name> -volume <volume name> -storage-efficiency-mode <default|efficient>
----


--
====
Weitere Informationen zu  `volume efficiency modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-efficiency-modify.html["ONTAP-Befehlsreferenz"^].
