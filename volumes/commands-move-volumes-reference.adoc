= ONTAP-Befehle zum Verschieben von Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die ONTAP CLI bietet spezifische Befehle zur Verwaltung von Volume-Verschiebungen. Je nach Anforderung stehen die folgenden Befehle zur Verwaltung von Kontingentregeln und Kontingentrichtlinien zur Verfügung:

[cols="2*"]
|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Einen aktiven Volume-Verschiebungsvorgang abbrechen.
 a| 
`volume move abort`



 a| 
Status eines Volumens anzeigen, das von einem Aggregat zu einem anderen verschoben wird.
 a| 
`volume move show`



 a| 
Ein Volume wird von einem Aggregat auf ein anderes Aggregat verschoben.
 a| 
`volume move start`



 a| 
Zielaggregate für Volumenverschiebungen verwalten.
 a| 
`volume move target-aggr`



 a| 
Die Umstellung eines Umzugsauftrags auslösen.
 a| 
`volume move trigger-cutover`



 a| 
Die Dauer der Zugriffssperre für Clients kann angepasst werden, falls die Standardeinstellung nicht ausreichend ist.
 a| 
`volume move start` oder  `volume move modify` mit dem  `-cutover-window` Parameter. Der  `volume move modify` Befehl ist ein erweiterter Befehl und der  `-cutover-window` ist ein erweiterter Parameter.



 a| 
Bestimmt wird, was das System unternimmt, wenn der Volume-Verschiebungsvorgang während der Zeit, in der der Clientzugriff blockiert ist, nicht abgeschlossen werden kann.
 a| 
`volume move start` oder  `volume move modify` mit dem  `-cutover-action` Parameter. Der  `volume move modify` Befehl ist ein erweiterter Befehl und der  `-cutover-action` ist ein erweiterter Parameter.

|===
