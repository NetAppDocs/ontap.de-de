= Erfahren Sie, wie ONTAP Standardbenutzer- und Gruppenquoten abgeleitete Quoten erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Standard-Benutzerkontingente oder Gruppenkontingente erstellt werden, werden automatisch entsprechende abgeleitete Benutzerkontingente oder Gruppenkontingente für jeden Benutzer oder jede Gruppe erstellt, die Dateien auf derselben Ebene besitzen.

Abgeleitete Benutzer- und Gruppenkontingente werden auf folgende Weise erstellt:

* Ein Standard-Benutzerkontingent auf einem FlexVol volume erzeugt abgeleitete Benutzerkontingente für jeden Benutzer, der eine Datei irgendwo auf dem Volume besitzt.
* Ein Standard-Benutzerkontingent für einen Qtree erstellt abgeleitete Benutzerkontingente für jeden Benutzer, dem eine Datei im Qtree gehört.
* Ein Standard-Gruppenkontingent auf einem FlexVol volume erzeugt abgeleitete Gruppenkontingente für jede Gruppe, die irgendwo auf dem Volume eine Datei besitzt.
* Ein Standard-Gruppenkontingent auf einem Qtree erzeugt abgeleitete Gruppenkontingente für jede Gruppe, die eine Datei im Qtree besitzt.


Besitzt ein Benutzer oder eine Gruppe keine Dateien auf der Ebene eines Standard-Benutzerkontingents oder Gruppenkontingents, wird kein abgeleitetes Kontingent für den Benutzer oder die Gruppe erstellt. Wenn beispielsweise ein Standard-Benutzerkontingent für den Qtree proj1 erstellt wird und der Benutzer jsmith Dateien in einem anderen Qtree besitzt, wird kein abgeleitetes Benutzerkontingent für jsmith erstellt.

Die abgeleiteten Kontingente haben dieselben Einstellungen wie die Standardkontingente, einschließlich Beschränkungen und Benutzerzuordnung. Wenn beispielsweise ein Standard-Benutzerkontingent eine Festplattenbeschränkung von 50 MB aufweist und die Benutzerzuordnung aktiviert ist, haben alle daraus abgeleiteten Kontingente ebenfalls eine Festplattenbeschränkung von 50 MB und die Benutzerzuordnung ist aktiviert.

Für drei spezielle Benutzer und Gruppen gelten jedoch keine Beschränkungen in abgeleiteten Kontingenten. Wenn die folgenden Benutzer und Gruppen Dateien auf der Ebene eines Standardbenutzer- oder Gruppenkontingents besitzen, wird ein abgeleitetes Kontingent mit derselben Benutzerzuordnungseinstellung wie das Standardbenutzer- oder Gruppenkontingent erstellt, jedoch handelt es sich dabei nur um ein Tracking-Kontingent (ohne Beschränkungen):

* UNIX Root-Benutzer (UID 0)
* UNIX Root-Gruppe (GID 0)
* Windows-Gruppe „BUILTIN\Administrators“
+
Da Kontingente für Windows-Gruppen als Benutzerkontingente erfasst werden, ist ein abgeleitetes Kontingent für diese Gruppe ein Benutzerkontingent, das von einem Standard-Benutzerkontingent und nicht von einem Standard-Gruppenkontingent abgeleitet wird.



.Beispiel für abgeleitete Benutzerkontingente
Wenn Sie ein Volume haben, auf dem drei Benutzer – root, jsmith und bob – Dateien besitzen, und Sie ein Standard-Benutzerkontingent für das Volume erstellen, erstellt ONTAP automatisch drei abgeleitete Benutzerkontingente. Daher erscheinen nach der Neuinitialisierung der Kontingente für das Volume vier neue Kontingente im Kontingentbericht:

[listing]
----
cluster1::> volume quota report
  Vserver: vs1
                                     ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol1               user    *           0B   50MB       0       -   *
vol1               user    root        5B      -       1       -
vol1               user    jsmith     30B   50MB      10       -   *
vol1               user    bob        40B   50MB      15       -   *
4 entries were displayed.
----
Die erste neue Zeile ist das von Ihnen erstellte Standard-Benutzerkontingent, das am Sternchen (*) als ID erkennbar ist. Die anderen neuen Zeilen sind die abgeleiteten Benutzerkontingente. Die abgeleiteten Kontingente für jsmith und bob haben dasselbe Festplattenlimit von 50 MB wie das Standardkontingent. Das abgeleitete Kontingent für den Root-Benutzer ist ein Tracking-Kontingent ohne Begrenzung.
