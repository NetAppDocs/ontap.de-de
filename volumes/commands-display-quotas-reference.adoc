= ONTAP-Befehle zur Anzeige von Informationen über Kontingente
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Befehle verwenden, um einen Quotenbericht mit erzwungenen Quoten und Ressourcennutzung anzuzeigen, Informationen über den Quotenstatus und Fehler oder über Quotenrichtlinien und Quotenregeln anzuzeigen.

Die folgenden Befehle können nur auf FlexVol Volumes ausgeführt werden.

[cols="2*"]
|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu den geltenden Quoten
 a| 
`volume quota report`



 a| 
Ressourcennutzung (Festplattenspeicher und Anzahl der Dateien) von Kontingentzielen anzeigen
 a| 
`volume quota report`



 a| 
Ermitteln, welche Quotenbeschränkungen betroffen sind, wenn das Schreiben in eine Datei erlaubt ist
 a| 
`volume quota report` mit dem  `-path` Parameter



 a| 
Der Quotenstatus wird angezeigt, wie zum Beispiel `on`, `off` und `initializing`
 a| 
`volume quota show`



 a| 
Informationen zur Protokollierung von Quotenmeldungen anzeigen
 a| 
`volume quota show` mit dem  `-logmsg` Parameter



 a| 
Fehler, die bei der Initialisierung und Größenänderung des Kontingents auftreten, anzeigen
 a| 
`volume quota show` mit dem  `-instance` Parameter



 a| 
Informationen zu Quotenrichtlinien
 a| 
`volume quota policy show`



 a| 
Informationen zu Quotenregeln
 a| 
`volume quota policy rule show`



 a| 
Der Name der Quotenrichtlinie, die einer Storage Virtual Machine (SVM, früher bekannt als Vserver) zugewiesen ist, wird angezeigt.
 a| 
`vserver show` mit dem  `-instance` Parameter

|===
Weitere Informationen zu  `volume quota` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=volume+quota["ONTAP-Befehlsreferenz"^].



[[when-to-use-ontap-volume-quota-policy-rule-show-and-volume-quota-report-commands]]
== Wann verwendet man die ONTAP-Befehle `volume quota policy rule show` und `volume quota report`?

Obwohl beide Befehle Informationen über Kontingente anzeigen, zeigt der `volume quota policy rule show`Befehl schnell die konfigurierten Kontingentregeln an, während der `volume quota report`Befehl, der mehr Zeit und Ressourcen benötigt, die erzwungenen Kontingente und die Ressourcennutzung anzeigt.

Der  `volume quota policy rule show` Befehl ist für folgende Zwecke nützlich:

* Die Konfiguration der Kontingentregeln sollte vor deren Aktivierung überprüft werden.
+
Dieser Befehl zeigt alle konfigurierten Quota-Regeln an, unabhängig davon, ob die Quotas initialisiert oder in ihrer Größe geändert wurden.

* Schnelle Anzeige von Quotenregeln ohne Beeinträchtigung der Systemressourcen
+
Da dieser Befehl keine Datenträger- und Dateinutzung anzeigt, ist er nicht so ressourcenintensiv wie ein Quotenbericht.

* Die Quotenregeln in einer Quotenrichtlinie anzeigen, die der SVM nicht zugewiesen ist.


Weitere Informationen zu  `volume quota policy rule show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-policy-rule-show.html["ONTAP-Befehlsreferenz"^].

Der  `volume quota report` Befehl ist für folgende Zwecke nützlich:

* Durchgesetzte Quoten, einschließlich abgeleiteter Quoten, anzeigen
* Der von jedem aktiven Kontingent belegte Speicherplatz und die Anzahl der verwendeten Dateien, einschließlich der von abgeleiteten Kontingenten betroffenen Ziele, werden angezeigt.
+
(Bei Standardkontingenten wird die Nutzung als „0“ angezeigt, da die Nutzung dem daraus abgeleiteten Kontingent zugeordnet wird.)

* Ermitteln, welche Quotenbeschränkungen Einfluss darauf haben, ob ein Schreibvorgang in eine Datei zulässig ist
+
Fügen Sie den  `-path` Parameter zum  `volume quota report` Befehl hinzu.



[NOTE]
====
Die Erstellung des Quotenberichts ist eine ressourcenintensive Operation. Wenn sie für viele FlexVol Volumes im Cluster ausgeführt wird, kann dies lange dauern. Eine effizientere Methode besteht darin, den Quotenbericht für ein bestimmtes Volume in einer SVM anzuzeigen.

====
Weitere Informationen zu  `volume quota report` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-report.html["ONTAP-Befehlsreferenz"^].
