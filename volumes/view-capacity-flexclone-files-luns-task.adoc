= Prüfen Sie die Knotenkapazität, bevor Sie ONTAP FlexClone-Dateien und FlexClone-LUNs erstellen und löschen.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es sollte festgestellt werden, ob ein Knoten über ausreichende Kapazität verfügt, um Anforderungen zum Erstellen und Löschen von FlexClone Dateien und FlexClone LUNs zu empfangen. Dies kann durch das Anzeigen der Split-Last für den Knoten erfolgen. Wenn die maximale Split-Last erreicht ist, werden keine neuen Anforderungen akzeptiert, bis die Split-Last unter den Maximalwert fällt.

.Über diese Aufgabe
Wenn der Knoten seine maximale Split-Auslastung erreicht, wird bei Erstellungs- und Löschanfragen eine  `EBUSY` Fehlermeldung ausgegeben. Wenn die Split-Auslastung des Knotens unter den Maximalwert sinkt, akzeptiert der Knoten wieder Anfragen zum Erstellen und Löschen von FlexClone Dateien und FlexClone LUNs.

Ein Knoten kann neue Anfragen annehmen, wenn das  `Allowable Split Load` Feld Kapazität anzeigt und die Erstellungsanfrage in die verfügbare Kapazität passt.

.Schritte
. Die Kapazität eines Knotens zum Erstellen und Löschen von FlexClone-Dateien und FlexClone-LUNs lässt sich mit dem  `volume file clone split load show` Befehl anzeigen.
+
Im folgenden Beispiel wird die Lastverteilung für alle Knoten in cluster1 angezeigt. Alle Knoten im Cluster verfügen über die Kapazität, FlexClone Dateien und FlexClone LUNs zu erstellen und zu löschen, wie im Feld „Allowable Split Load“ angegeben:

+
[listing]
----
cluster1::> volume file clone split load show
Node       Max        Current    Token         Allowable
           Split Load Split Load Reserved Load Split Load
---------- ---------- ---------- ------------- ----------
node1         15.97TB         0B         100MB    15.97TB
node2         15.97TB         0B         100MB    15.97TB
2 entries were displayed.
----
