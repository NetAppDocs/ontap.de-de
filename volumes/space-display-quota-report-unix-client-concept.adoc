= Unterschied in der Speichernutzung, der von ONTAP-Quota-Berichten und UNIX-Clients angezeigt wird
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der in einem Quota-Bericht für ein FlexVol volume oder einen qtree angezeigte Wert für den belegten Speicherplatz kann von dem Wert abweichen, der von einem UNIX-Client für dasselbe volume oder denselben qtree angezeigt wird. Diese Abweichung ergibt sich aus den unterschiedlichen Methoden, die der Quota-Bericht und die UNIX-Befehle zur Berechnung der Datenblöcke im volume oder qtree verwenden.

Enthält ein Volume beispielsweise eine Datei mit leeren Datenblöcken (in die keine Daten geschrieben werden), werden diese leeren Datenblöcke im Quota-Bericht für das Volume bei der Berechnung der Speichernutzung nicht berücksichtigt. Wird das Volume jedoch auf einem UNIX-Client eingebunden und die Datei als Ausgabe des  `ls` Befehls angezeigt, werden die leeren Datenblöcke ebenfalls in die Speichernutzung einbezogen. Daher zeigt der  `ls` Befehl eine höhere Dateigröße an als die im Quota-Bericht angegebene Speichernutzung.

Ebenso können die in einem Quotenbericht angezeigten Speicherplatzbelegungswerte von den Werten abweichen, die sich aus UNIX-Befehlen wie  `df` und  `du` ergeben.



[[how-a-quota-report-accounts-for-disk-space-and-file-usage]]
== Wie ein Quotenbericht Speicherplatz und Dateinutzung berücksichtigt

Die Anzahl der verwendeten Dateien und der im Quota-Bericht für ein FlexVol volume oder einen Qtree angegebene Speicherplatz hängen von der Anzahl der verwendeten Datenblöcke ab, die jedem Inode im volume oder Qtree entsprechen.

Die Blockanzahl umfasst sowohl direkte als auch indirekte Blöcke, die für reguläre Dateien und Stream-Dateien verwendet werden. Die Blöcke, die für Verzeichnisse, Access Control Lists (ACLs), Stream-Verzeichnisse und Metadateien verwendet werden, werden im Quota-Bericht nicht berücksichtigt. Bei UNIX-Sparse-Dateien werden leere Datenblöcke nicht im Quota-Bericht berücksichtigt.

Das Quota-Subsystem berücksichtigt ausschließlich vom Benutzer steuerbare Aspekte des Dateisystems. Verzeichnisse, ACLs und Snapshot-Speicher sind Beispiele für Speicherbereiche, die von der Quota-Berechnung ausgeschlossen sind. Quotas dienen der Durchsetzung von Beschränkungen, nicht der Garantie, und sie wirken nur auf das aktive Dateisystem. Die Quota-Verwaltung berücksichtigt weder bestimmte Dateisystemstrukturen noch die Speichereffizienz (wie Komprimierung oder Deduplizierung).

Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].



[[disparity-between-ls-command-and-quota-report-for-space-usage]]
== Abweichung zwischen dem Befehl „ls“ und dem Quotenbericht zur Speicherplatznutzung

Wenn der  `ls` Befehl verwendet wird, um den Inhalt eines auf einem UNIX-Client eingebundenen FlexVol Volumes anzuzeigen, können die in der Ausgabe angezeigten Dateigrößen von der im Quota-Bericht für das Volume angezeigten Speicherplatznutzung abweichen, abhängig vom Typ der Datenblöcke für die Datei.

Die Ausgabe des  `ls` Befehls zeigt lediglich die Größe einer Datei an und enthält keine indirekten Blöcke, die von der Datei verwendet werden. Leere Blöcke der Datei werden ebenfalls in die Ausgabe des Befehls einbezogen.

Wenn eine Datei keine leeren Blöcke enthält, kann die vom  `ls` Befehl angezeigte Größe aufgrund der Berücksichtigung indirekter Blöcke im Quotenbericht geringer sein als die im Quotenbericht angegebene Festplattennutzung. Enthält die Datei hingegen leere Blöcke, kann die vom  `ls` Befehl angezeigte Größe höher sein als die im Quotenbericht angegebene Festplattennutzung.

Die Ausgabe des  `ls` Befehls zeigt lediglich die Größe einer Datei an und enthält keine indirekten Blöcke, die von der Datei verwendet werden. Leere Blöcke der Datei werden ebenfalls in die Ausgabe des Befehls einbezogen.

.Beispiel für den Unterschied zwischen der vom Befehl ls erfassten Speicherplatznutzung und einem Quota-Bericht
[%collapsible]
====
Der folgende Quotenbericht zeigt ein Limit von 10 MB für einen qtree q1:

[listing]
----

                                      ----Disk----  ----Files-----   Quota
  Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
  -------  --------  ------  -------  -----  -----  ------  ------   ---------
  vol1     q1          tree    user1   10MB  10MB        1       -   q1

...
----
Eine Datei, die sich im selben Qtree befindet, kann beim Betrachten von einem UNIX-Client mit dem  `ls` Befehl eine Größe aufweisen, die das Kontingentlimit überschreitet, wie im folgenden Beispiel gezeigt:

[listing]
----
[user1@lin-sys1 q1]$ ls -lh
-rwxr-xr-x  1 user1 nfsuser  **27M** Apr 09  2013 file1
----
====
Weitere Informationen zu  `ls` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=ls["ONTAP-Befehlsreferenz"^].



[[learn-how-the-df-command-accounts-for-file-size]]
== Erfahren Sie, wie der Befehl df die Dateigröße berücksichtigt

Die Art und Weise, wie der  `df` Befehl die Speicherplatznutzung meldet, hängt von zwei Bedingungen ab: ob die Quoten für das Volume, das den Qtree enthält, aktiviert oder deaktiviert sind, und ob die Quotennutzung innerhalb des Qtree verfolgt wird.

Wenn für das Volume, das den Qtree enthält, Quotas aktiviert sind und die Quota-Nutzung innerhalb des Qtree überwacht wird, entspricht die vom  `df` Befehl gemeldete Speicherplatznutzung dem im Quota-Bericht angegebenen Wert. In diesem Fall werden Blöcke, die von Verzeichnissen, ACLs, Stream-Verzeichnissen und Metadateien belegt sind, von der Quota-Nutzung ausgeschlossen.

Wenn für das Volume keine Quotas aktiviert sind oder für den Qtree keine Quota-Regel konfiguriert ist, umfasst die gemeldete Speicherplatznutzung die von Verzeichnissen, ACLs, Stream-Verzeichnissen und Metadateien belegten Blöcke des gesamten Volumes, einschließlich anderer Qtrees innerhalb des Volumes. In diesem Fall ist die von dem  `df` Befehl gemeldete Speicherplatznutzung höher als der erwartete Wert bei aktivierter Quota-Überwachung.

Wenn der  `df` Befehl vom Mountpunkt eines Qtrees ausgeführt wird, für den die Quota-Nutzung überwacht wird, zeigt die Befehlsausgabe dieselbe Speicherplatznutzung wie der im Quota-Bericht angegebene Wert. In den meisten Fällen, wenn die Quota-Regel des Baums ein Festplattenlimit hat, entspricht die vom  `df` Befehl gemeldete Gesamtgröße dem Festplattenlimit und der verfügbare Speicherplatz entspricht der Differenz zwischen dem Quota-Festplattenlimit und der Quota-Nutzung.

In manchen Fällen kann der vom  `df` Befehl gemeldete verfügbare Speicherplatz dem gesamten verfügbaren Speicherplatz des Volumes entsprechen. Dies kann vorkommen, wenn kein Festplattenlimit für den Qtree konfiguriert ist. Ab ONTAP 9.9.1 kann dies auch auftreten, wenn der gesamte verfügbare Speicherplatz des Volumes geringer ist als der verbleibende Kontingentspeicherplatz des Qtrees. Wenn eine dieser Bedingungen vorliegt, ist die vom  `df` Befehl gemeldete Gesamtgröße ein zusammengesetzter Wert, der dem im Qtree verwendeten Kontingent zuzüglich des im FlexVol volume verfügbaren Speicherplatzes entspricht.

[NOTE]
====
Diese Gesamtgröße entspricht weder dem Qtree-Festplattenlimit noch der konfigurierten Volume-Größe. Sie kann auch je nach Schreibaktivität in anderen Qtrees oder Ihrer Speichereffizienzaktivität im Hintergrund variieren.

====
.Beispiel für die Speichernutzung, die vom <code>df</code>-Befehl und einem Quotenbericht erfasst wird
[%collapsible]
====
Der folgende Quotenbericht zeigt ein Festplattenlimit von 1 GB für qtree alice, 2 GB für qtree bob und kein Limit für qtree project1:

[listing]
----
C1_vsim1::> quota report -vserver vs0
Vserver: vs0
                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol2     alice     tree    1
                                  502.0MB    1GB       2       -   alice
vol2     bob       tree    2
                                   1003MB    2GB       2       -   bob
vol2     project1  tree    3
                                  200.8MB      -       2       -   project1
vol2               tree    *           0B      -       0       -   *
4 entries were displayed.
----
Im folgenden Beispiel zeigt die Ausgabe des  `df` Befehls für die qtrees alice und bob denselben belegten Speicherplatz wie der Quota-Bericht und dieselbe Gesamtgröße (in 1-MB-Blöcken) wie das Festplattenlimit. Dies liegt daran, dass die Quota-Regeln für qtrees alice und bob ein definiertes Festplattenlimit haben und der verfügbare Speicherplatz (1211 MB) größer ist als der verbleibende Baumquotenspeicherplatz für qtree alice (523 MB) und qtree bob (1045 MB).

[listing]
----
linux-client1 [~]$ df -m /mnt/vol2/alice
Filesystem          1M-blocks  Used Available Use% Mounted on
172.21.76.153:/vol2      1024   502       523  50% /mnt/vol2

linux-client1 [~]$ df -m /mnt/vol2/bob
Filesystem          1M-blocks  Used Available Use% Mounted on
172.21.76.153:/vol2      2048  1004      1045  50% /mnt/vol2
----
Im folgenden Beispiel zeigt die Ausgabe des  `df` Befehls für qtree project1 denselben belegten Speicherplatz wie der Quota-Bericht an, aber die Gesamtgröße wird synthetisiert, indem der verfügbare Speicherplatz im gesamten Volume (1211 MB) zur Quota-Nutzung von qtree project1 (201 MB) addiert wird, sodass sich insgesamt 1412 MB ergeben. Dies ist darauf zurückzuführen, dass die Quota-Regel für qtree project1 kein Festplattenlimit hat.

[listing]
----
linux-client1 [~]$ df -m /mnt/vol2/project1
Filesystem          1M-blocks  Used Available Use% Mounted on
172.21.76.153:/vol2      1412   201      1211  15% /mnt/vol2
----
Das folgende Beispiel zeigt, wie die Ausgabe des  `df` Befehls für das gesamte Volume den gleichen verfügbaren Speicherplatz wie Projekt1 meldet.

[listing]
----
linux-client1 [~]$ df -m /mnt/vol2
Filesystem          1M-blocks  Used Available Use% Mounted on
172.21.76.153:/vol2      2919  1709      1211  59% /mnt/vol2
----
====

[[disparity-betwee-du-command-and-quota-report-for-space-usage]]
== Abweichung zwischen dem Befehl „du“ und dem Quotenbericht zur Speicherplatznutzung

Wenn der  `du` Befehl zur Überprüfung der Speicherplatznutzung für einen auf einem UNIX-Client eingebundenen Qtree oder ein FlexVol volume ausgeführt wird, kann der Nutzungswert höher sein als der Wert, der in einem Quota-Bericht für den Qtree oder das Volume angezeigt wird.

Die Ausgabe des  `du` Befehls enthält die kombinierte Speicherplatznutzung aller Dateien im Verzeichnisbaum, beginnend auf der Ebene des Verzeichnisses, in dem der Befehl ausgeführt wird. Da der durch den  `du` Befehl angezeigte Nutzungswert auch die Datenblöcke für Verzeichnisse umfasst, ist er höher als der Wert, der in einem Quota-Bericht angezeigt wird.

.Beispiel für den Unterschied zwischen der vom Befehl „du“ erfassten Speichernutzung und einem Quotenbericht
[%collapsible]
====
Der folgende Quotenbericht zeigt ein Limit von 10MB für einen qtree q1:

[listing]
----

                                      ----Disk----  ----Files-----   Quota
  Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
  -------  --------  ------  -------  -----  -----  ------  ------   ---------
  vol1     q1        tree    user1    10MB   10MB   1       -        q1

...
----
Im folgenden Beispiel zeigt die Festplattenspeichernutzung als Ausgabe des  `du` Befehls einen höheren Wert, der das Kontingentlimit überschreitet:

[listing]
----
[user1@lin-sys1 q1]$ du -sh
**11M**     q1
----
====
Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].
