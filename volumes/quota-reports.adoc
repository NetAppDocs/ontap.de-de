= Informationen zu ONTAP Quotenberichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit Quotenberichten lassen sich Details wie die Konfiguration von Quotenregeln und -richtlinien, durchgesetzte und konfigurierte Quoten sowie Fehler, die bei der Anpassung der Quotengröße und der Reinitialisierung aufgetreten sind, anzeigen.

Das Anzeigen von Kontingentinformationen ist in Situationen wie den folgenden hilfreich:

* Konfigurieren von Kontingenten, zum Beispiel um Kontingente festzulegen und die Konfigurationen zu überprüfen
* Reaktionen auf Benachrichtigungen, dass der Speicherplatz oder die Dateigrenzen bald erreicht werden oder bereits erreicht sind
* Reaktion auf Anfragen nach mehr Speicherplatz




[[see-what-ontap-quotas-are-in-effect-using-the-quota-report]]
== Sehen Sie nach, welche ONTAP-Quoten mithilfe des Quotenberichts in Kraft sind

Da Kontingente auf vielfältige Weise interagieren, sind mehr Kontingente aktiv als nur die von Ihnen explizit erstellten. Welche Kontingente aktiv sind, lässt sich im Kontingentbericht einsehen.

Die folgenden Beispiele zeigen Quotenberichte für verschiedene Arten von Quoten, die auf ein FlexVol volume vol1 und einen in diesem Volume enthaltenen qtree q1 angewendet wurden:

.Beispiel ohne festgelegte Benutzerkontingente für den Qtree
In diesem Beispiel gibt es einen Qtree, q1, der im Volume vol1 enthalten ist. Der Administrator hat drei Kontingente erstellt:

* Ein Standard-Kontingentlimit für die Baumstruktur von 400 MB auf vol1
* Ein standardmäßiges Benutzerkontingentlimit von 100 MB auf vol1
* Ein explizites Benutzerkontingentlimit von 200 MB für den Benutzer jsmith auf vol1


Die Quotenregeln für diese Quoten sehen ähnlich aus wie im folgenden Beispiel:

[listing]
----
cluster1::*> volume quota policy rule show -vserver vs1 -volume vol1

Vserver: vs1               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
tree   ""        ""      -           400MB        -       -        -          -
user   ""        ""      off         100MB        -       -        -          -
user   jsmith    ""      off         200MB        -       -        -          -
----
Der Quotenbericht für diese Quoten sieht in etwa wie im folgenden Beispiel aus:

[listing]
----
cluster1::> volume quota report
Vserver: vs1
                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol1     -          tree   *           0B  400MB       0       -   *
vol1     -          user   *           0B  100MB       0       -   *
vol1     -          user   jsmith    150B  200MB       7       -   jsmith
vol1     q1         tree   1           0B  400MB       6       -   q1
vol1     q1         user   *           0B  100MB       0       -
vol1     q1         user   jsmith      0B  100MB       5       -
vol1     -          user   root        0B    0MB       1       -
vol1     q1         user   root        0B    0MB       8       -
----
Die ersten drei Zeilen des Quotenberichts zeigen die drei vom Administrator festgelegten Quoten an. Da zwei dieser Quoten Standardquoten sind, erstellt ONTAP automatisch abgeleitete Quoten.

In der vierten Zeile wird das Baumkontingent angezeigt, das sich aus dem Standardbaumkontingent für jeden Qtree in vol1 ergibt (in diesem Beispiel nur q1).

In der fünften Zeile wird das standardmäßige Benutzerkontingent angezeigt, das für den qtree aufgrund des Vorhandenseins des standardmäßigen Benutzerkontingents auf dem Volume und des qtree-Kontingents erstellt wird.

Die sechste Zeile zeigt das abgeleitete Benutzerkontingent, das für jsmith auf dem qtree erstellt wird, da für den qtree ein Standardbenutzerkontingent existiert (Zeile 5) und der Benutzer jsmith Dateien auf diesem qtree besitzt. Zu beachten ist, dass das für den Benutzer jsmith im qtree q1 geltende Limit nicht durch das explizite Benutzerkontingent-Limit (200MB) bestimmt wird. Dies liegt daran, dass das explizite Benutzerkontingent-Limit für das Volume gilt und daher keine Auswirkungen auf die Limits für den qtree hat. Stattdessen wird das abgeleitete Benutzerkontingent-Limit für den qtree durch das Standardbenutzerkontingent für den qtree (100MB) bestimmt.

Die letzten beiden Zeilen zeigen weitere Benutzerkontingente, die von den Standardbenutzerkontingenten auf dem Volume und auf dem qtree abgeleitet sind. Ein abgeleitetes Benutzerkontingent wurde für den Root-Benutzer sowohl auf dem Volume als auch auf dem qtree erstellt, da der Root-Benutzer Dateien sowohl auf dem Volume als auch auf dem qtree besitzt. Da der Root-Benutzer hinsichtlich der Kontingente eine Sonderbehandlung erhält, sind seine abgeleiteten Kontingente ausschließlich Tracking-Kontingente.

.Beispiel mit für den qtree festgelegten Benutzerkontingenten
Dieses Beispiel ähnelt dem vorherigen, nur dass der Administrator zwei Kontingente auf dem qtree hinzugefügt hat.

Es gibt noch ein Volume, vol1, und einen qtree, q1. Der Administrator hat die folgenden Kontingente erstellt:

* Ein Standard-Kontingentlimit für die Baumstruktur von 400 MB auf vol1
* Ein standardmäßiges Benutzerkontingentlimit von 100 MB auf vol1
* Ein explizites Benutzerkontingentlimit von 200 MB auf vol1 für den Benutzer jsmith
* Ein standardmäßiges Benutzerkontingent von 50 MB auf qtree q1
* Ein explizites Benutzerkontingent von 75MB für den Benutzer jsmith auf qtree q1


Die Quotenregeln für diese Quoten sehen wie folgt aus:

[listing]
----
cluster1::> volume quota policy rule show -vserver vs1 -volume vol1

Vserver: vs1               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
tree   ""        ""      -           400MB        -       -        -          -
user   ""        ""      off         100MB        -       -        -          -
user   ""        q1      off          50MB        -       -        -          -
user   jsmith    ""      off         200MB        -       -        -          -
user   jsmith    q1      off          75MB        -       -        -          -
----
Der Quotenbericht für diese Quoten sieht folgendermaßen aus:

[listing]
----

cluster1::> volume quota report
Vserver: vs1
                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol1     -          tree   *           0B  400MB       0       -   *
vol1     -          user   *           0B  100MB       0       -   *
vol1     -          user   jsmith   2000B  200MB       7       -   jsmith
vol1     q1         user   *           0B   50MB       0       -   *
vol1     q1         user   jsmith      0B   75MB       5       -   jsmith
vol1     q1         tree   1           0B  400MB       6       -   q1
vol1     -          user   root        0B    0MB       2       -
vol1     q1         user   root        0B    0MB       1       -
----
Die ersten fünf Zeilen des Quotenberichts zeigen die fünf vom Administrator erstellten Quoten an. Da es sich bei einigen dieser Quoten um Standardquoten handelt, erstellt ONTAP automatisch abgeleitete Quoten.

In der sechsten Zeile wird das qtree-Kontingent angezeigt, das sich aus dem Standard-qtree-Kontingent für jeden qtree in vol1 ergibt (in diesem Beispiel nur q1).

Die letzten beiden Zeilen zeigen die Benutzerkontingente, die von den Standardbenutzerkontingenten auf dem Volume und auf dem qtree abgeleitet werden. Ein abgeleitetes Benutzerkontingent wurde für den Root-Benutzer sowohl auf dem Volume als auch auf dem qtree erstellt, da der Root-Benutzer Dateien sowohl auf dem Volume als auch auf dem qtree besitzt. Da der Root-Benutzer hinsichtlich der Kontingente eine Sonderbehandlung erhält, sind seine abgeleiteten Kontingente ausschließlich Tracking-Kontingente.

Es wurden keine weiteren Standardkontingente oder abgeleiteten Kontingente aus folgenden Gründen erstellt:

* Für den Benutzer jsmith wurde kein abgeleitetes Benutzerkontingent erstellt, obwohl der Benutzer Dateien sowohl auf dem Volume als auch im qtree besitzt, da der Benutzer bereits auf beiden Ebenen explizite Kontingente besitzt.
* Es wurden keine abgeleiteten Benutzerkontingente für andere Benutzer erstellt, da keine anderen Benutzer Dateien auf dem Volume oder im qtree besitzen.
* Das standardmäßige Benutzerkontingent auf dem Volume hat kein standardmäßiges Benutzerkontingent auf dem qtree erstellt, da der qtree bereits ein standardmäßiges Benutzerkontingent hatte.




[[use-the-ontap-quota-report-to-determine-which-quotas-limit-writes-to-a-specific-file]]
== Verwenden Sie den ONTAP-Quota-Bericht, um zu ermitteln, welche Quotas das Schreiben in eine bestimmte Datei einschränken

Mit dem Befehl „volume quota report“ und einem bestimmten Dateipfad lässt sich ermitteln, welche Kontingentgrenzen Schreibvorgänge in eine Datei beeinflussen. Dies kann dabei helfen zu verstehen, welches Kontingent einen Schreibvorgang verhindert.

.Schritt
. Der Befehl volume quota report wird mit dem Parameter -path verwendet.


.Beispiel für die Anzeige von Kontingenten, die sich auf eine bestimmte Datei auswirken
Das folgende Beispiel zeigt den Befehl und die Ausgabe zur Ermittlung der für Schreibvorgänge in die Datei file1 geltenden Kontingente, die sich im qtree q1 im FlexVol volume vol2 befinden:

[listing]
----
cluster1:> volume quota report -vserver vs0 -volume vol2 -path /vol/vol2/q1/file1
Virtual Server: vs0
                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol2     q1        tree    jsmith     1MB  100MB       2   10000   q1
vol2     q1        group   eng        1MB  700MB       2   70000
vol2               group   eng        1MB  700MB       6   70000   *
vol2               user    corp\jsmith
                                      1MB   50MB       1       -   *
vol2     q1        user    corp\jsmith
                                      1MB   50MB       1       -
5 entries were displayed.
----
