= Konfigurieren Sie ONTAP-Volumes so, dass sie automatisch wachsen und schrumpfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können FlexVol Volumes so konfigurieren, dass sie je nach aktuellem Speicherplatzbedarf automatisch wachsen und schrumpfen. Das automatische Wachsen hilft zu verhindern, dass ein Volume keinen Speicherplatz mehr hat, sofern das Aggregat mehr Speicherplatz bereitstellen kann. Die automatische Verkleinerung verhindert, dass ein Volumen größer als nötig ist, und gibt dadurch Speicherplatz im Aggregat für die Nutzung durch andere Volumes frei.

.Über diese Aufgabe
Autoshrink kann nur in Kombination mit Autogrow verwendet werden, um sich ändernden Speicherplatzanforderungen gerecht zu werden, und ist nicht allein verfügbar. Wenn Autoshrink aktiviert ist, verwaltet ONTAP das Verkleinerungsverhalten eines Volumes automatisch, um eine Endlosschleife von Autogrow- und Autoshrink-Aktionen zu verhindern.

Wenn ein Volume wächst, kann die maximale Anzahl der darauf speicherbaren Dateien automatisch erhöht werden. Beim Verkleinern eines Volumes bleibt die maximale Anzahl der darauf speicherbaren Dateien unverändert, und ein Volume kann nicht automatisch unter die Größe verkleinert werden, die seiner aktuellen maximalen Dateianzahl entspricht. Aus diesem Grund ist es möglicherweise nicht möglich, ein Volume automatisch vollständig auf seine ursprüngliche Größe zu verkleinern.

Standardmäßig beträgt die maximale Größe eines Volumes 120 % der Größe, bei der die automatische Größenanpassung aktiviert ist. Falls sichergestellt werden muss, dass das Volume größer werden kann, ist die maximale Größe für das Volume entsprechend festzulegen.

.Bevor Sie beginnen
Das FlexVol volume muss online sein.

.Schritt
. Das Volume kann so konfiguriert werden, dass seine Größe automatisch vergrößert und verkleinert wird:
+
`volume autosize -vserver _SVM_name_ -volume _volume_name_ -mode grow_shrink`

+
Der folgende Befehl aktiviert die automatische Größenänderung eines Volumes namens „test2“. Das Volume ist so konfiguriert, dass es ab einer Füllmenge von 60 % verkleinert wird. Für den Beginn der Vergrößerung und die maximale Größe werden die Standardwerte verwendet.

+
[listing]
----
cluster1::> volume autosize -vserver vs2 test2 -shrink-threshold-percent 60
vol autosize: Flexible volume "vs2:test2" autosize settings UPDATED.

Volume modify successful on volume: test2
----
