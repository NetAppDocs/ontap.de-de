= Erfahren Sie mehr über ONTAP-Baumquoten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können ein Kontingent mit einem Qtree als Ziel erstellen, um die maximale Größe des Ziel-Qtrees zu begrenzen. Diese Kontingente werden auch als _Baumkontingente_ bezeichnet.


NOTE: Sie können auch Benutzer- und Gruppenquoten für einen bestimmten Qtree erstellen. Darüber hinaus werden Quoten für ein FlexVol volume manchmal an die in diesem Volume enthaltenen Qtrees vererbt.

Wenn einem Qtree ein Benutzerkontingent zugewiesen wird, ist das Ergebnis ähnlich wie bei einer Festplattenpartition, mit dem Unterschied, dass die maximale Größe des Qtrees jederzeit durch Ändern des Benutzerkontingents angepasst werden kann. Beim Anwenden eines Baumkontingents begrenzt ONTAP den Speicherplatz und die Anzahl der Dateien im Qtree, unabhängig von deren Eigentümern. Kein Benutzer, einschließlich root und Mitglieder der Gruppe BUILTIN\Administrators, kann in den Qtree schreiben, wenn der Schreibvorgang dazu führen würde, dass das Baumkontingent überschritten wird.

Die Größe des Kontingents garantiert keine bestimmte Menge an verfügbarem Speicherplatz. Die Größe des Kontingents kann größer sein als der tatsächlich im Qtree verfügbare Speicherplatz. Mit dem  `volume quota report` Befehl lässt sich die tatsächlich im Qtree verfügbare Speicherplatzmenge ermitteln.

Weitere Informationen zu  `volume quota report` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-report.html["ONTAP-Befehlsreferenz"^].



[[learn-how-ontap-user-and-group-quotas-work-with-qtrees]]
== Erfahren Sie, wie ONTAP Benutzer- und Gruppenkontingente mit qtrees funktionieren.

Baumkontingente begrenzen die Gesamtgröße des Qtrees. Um zu verhindern, dass einzelne Benutzer oder Gruppen den gesamten Qtree belegen, wird ein Benutzerkontingent oder Gruppenkontingent für diesen Qtree festgelegt.

.Beispiel für ein Benutzerkontingent in einem Qtree
[%collapsible]
====
Angenommen, Sie haben die folgenden Quotenregeln:

[listing]
----
cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Vserver: vs0               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   ""        ""      off          50MB        -       -        -       45MB
user   jsmith    ""      off          80MB        -       -        -       75MB
----
Es fällt auf, dass ein bestimmter Benutzer, kjones, zu viel Speicherplatz in einem kritischen Qtree, proj1, belegt, der sich in vol1 befindet. Der Speicherplatz dieses Benutzers kann durch Hinzufügen der folgenden Quotenregel eingeschränkt werden:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -volume vol1 -policy-name default -type user -target "kjones" -qtree "proj1" -disk-limit 20m -threshold 15m

cluster1::> volume quota policy rule show -vserver vs0 -volume vol1

Vserver: vs0               Policy: default           Volume: vol1
                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   ""        ""      off          50MB        -       -        -       45MB
user   jsmith    ""      off          80MB        -       -        -       75MB
user   kjones    proj1   off          20MB        -       -        -       15MB
----
====


[[learn-how-default-tree-quotas-on-an-ontap-flexvol-volume-create-derived-tree-quotas]]
== Erfahren Sie, wie Standard-Baumquoten auf einem ONTAP FlexVol Volume abgeleitete Baumquoten erzeugen

Wenn ein Standard-Baumkontingent auf einem FlexVol volume erstellt wird, werden automatisch entsprechende abgeleitete Baumkontingente für jeden Qtree in diesem Volume erstellt.

Diese abgeleiteten Baumquoten haben dieselben Beschränkungen wie die Standardbaumquote. Falls keine zusätzlichen Quoten vorhanden sind, haben die Beschränkungen folgende Auswirkungen:

* Benutzer können so viel Speicherplatz in einem Qtree nutzen, wie ihnen für das gesamte Volume zugewiesen ist (vorausgesetzt, sie haben das Limit für das Volume nicht überschritten, indem sie Speicherplatz im Root oder in einem anderen Qtree genutzt haben).
* Jeder der Qtrees kann so weit wachsen, dass er das gesamte Volume einnimmt.


Das Vorhandensein eines Standard-Baumkontingents auf einem Volume wirkt sich weiterhin auf alle neuen Qtrees aus, die dem Volume hinzugefügt werden. Jedes Mal, wenn ein neuer Qtree erstellt wird, wird auch ein abgeleitetes Baumkontingent erstellt.

Wie alle abgeleiteten Quoten weisen auch abgeleitete Baumquoten die folgenden Verhaltensweisen auf:

* Werden nur erstellt, wenn für das Ziel noch kein explizites Kontingent vorhanden ist.
* Erscheinen in Quotenberichten, aber nicht, wenn Quotenregeln mit dem  `volume quota policy rule show` Befehl angezeigt werden. Weitere Informationen zu  `volume quota policy rule show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-quota-policy-rule-show.html["ONTAP-Befehlsreferenz"^].


.Beispiel für abgeleitete Baumkontingente
[%collapsible]
====
Sie verfügen über ein Volume mit drei Qtrees (proj1, proj2 und proj3), wobei die einzige Baumquote eine explizite Quote für den Qtree proj1 ist, die dessen Festplattengröße auf 10 GB begrenzt. Wenn eine Standardbaumquote für das Volume erstellt und die Quoten auf dem Volume neu initialisiert werden, enthält der Quotenbericht nun vier Baumquoten:

[listing]
----
                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
vol1     proj1     tree    1           0B   10GB       1       -   proj1
vol1               tree    *           0B   20GB       0       -   *
vol1     proj2     tree    2           0B   20GB       1       -   proj2
vol1     proj3     tree    3           0B   20GB       1       -   proj3
...
----
Die erste Zeile zeigt das ursprüngliche explizite Kontingent für den Qtree proj1. Dieses Kontingent bleibt unverändert.

Die zweite Zeile zeigt das neue Standard-Baumkontingent auf dem Volume. Das Sternchen (*) als Quota Specifier kennzeichnet es als Standardkontingent. Dieses Kontingent ist das Ergebnis der von Ihnen erstellten Kontingentregel.

Die letzten beiden Zeilen zeigen die neuen abgeleiteten Baumquoten für die Qtrees proj2 und proj3. ONTAP hat diese Quoten automatisch aufgrund der Standardbaumquote auf dem Volume erstellt. Diese abgeleiteten Baumquoten haben dasselbe 20-GB-Speicherlimit wie die Standardbaumquote auf dem Volume. Für den Qtree proj1 hat ONTAP keine abgeleitete Baumquote erstellt, da dieser bereits über eine explizite Quote verfügte.

====


[[learn-how-default-user-quotas-on-an-ontap-flexvol-volume-affect-quotas-for-qtrees]]
== Erfahren Sie, wie sich die Standardbenutzerkontingente auf einem ONTAP FlexVol volume auf die Kontingente für Qtrees auswirken.

Wenn für ein FlexVol volume ein Standard-Benutzerkontingent definiert ist, wird für jeden in diesem Volume enthaltenen Qtree, für den ein explizites oder abgeleitetes Baumkontingent existiert, automatisch ein Standard-Benutzerkontingent erstellt.

Wenn bereits ein Standard-Benutzerkontingent auf dem Qtree existiert, bleibt dieses unberührt, wenn das Standard-Benutzerkontingent auf dem Volume erstellt wird.

Die automatisch erstellten Standard-Benutzerkontingente auf den Qtrees haben die gleichen Beschränkungen wie das Standard-Benutzerkontingent, das Sie für das Volume erstellen.

Ein explizites Benutzerkontingent für einen Qtree überschreibt (ersetzt die durch das automatisch erstellte Standard-Benutzerkontingent gesetzten Grenzwerte), genauso wie es ein Standard-Benutzerkontingent auf diesem Qtree überschreibt, das von einem Administrator erstellt wurde.
