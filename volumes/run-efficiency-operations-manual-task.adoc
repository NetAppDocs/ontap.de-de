= Eine Effizienzmaßnahme manuell ausführen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Effizienzoperationen manuell auf einem Volume ausführen. Dies kann sinnvoll sein, wenn die Planung von Effizienzoperationen nicht angebracht ist.

.Bevor Sie beginnen
Je nachdem, welche Effizienzoperation Sie manuell durchführen möchten, müssen Sie die Deduplizierung oder sowohl Datenkomprimierung als auch Deduplizierung auf einem Volume aktiviert haben.

.Über diese Aufgabe
Dieser Vorgang wird mit dem  `volume efficiency start` Befehl durchgeführt. Wenn die temperaturabhängige Speichereffizienz für ein Volume aktiviert ist, wird zunächst die Deduplizierung und anschließend die Datenkomprimierung durchgeführt.

Die Deduplizierung ist ein Hintergrundprozess, der während seiner Ausführung Systemressourcen beansprucht. Wenn sich die Daten in einem Volume nicht häufig ändern, ist es besser, die Deduplizierung seltener auszuführen. Mehrere gleichzeitig auf einem Speichersystem laufende Deduplizierungsvorgänge führen zu einem höheren Verbrauch von Systemressourcen.

Pro Knoten können maximal acht Deduplizierungs- oder Datenkomprimierungsvorgänge gleichzeitig ausgeführt werden. Wenn weitere Effizienzvorgänge geplant sind, werden diese in eine Warteschlange gestellt.

Ab ONTAP 9.13.1 ist es möglich, bei aktivierter temperaturabhängiger Speichereffizienz für ein Volume die Volume-Effizienz auf vorhandene Daten anzuwenden, um von der sequenziellen Packung zu profitieren und die Speichereffizienz weiter zu verbessern.



[[run-efficiency-manually]]
== Effizienz manuell ausführen

.Schritte
. Die Effizienzoptimierung für ein Volume starten: `volume efficiency start`


.Beispiel
Mit dem folgenden Befehl lässt sich manuell entweder nur die Deduplizierung oder die Deduplizierung gefolgt von logischer Komprimierung und Containerkomprimierung auf dem Volume VolA starten.

+

[listing]
----
volume efficiency start -vserver vs1 -volume VolA
----


[[repack-existing-data]]
== Vorhandene Daten neu verpacken

Um die in ONTAP 9.13.1 eingeführte sequenzielle Datenpackung auf Volumes mit aktivierter temperaturabhängiger Speichereffizienz zu nutzen, kann vorhandene Daten neu gepackt werden. Für die Nutzung dieses Befehls ist der erweiterte Berechtigungsmodus erforderlich.

.Schritte
. Berechtigungsstufe festlegen: `set -privilege advanced`
. Vorhandene Daten neu verpacken: `volume efficiency inactive-data-compression start -vserver _vserver_name_ -volume _volume_name_ -scan-mode extended_recompression`


.Beispiel
[listing]
----
volume efficiency inactive-data-compression start -vserver vs1 -volume vol1 -scan-mode extended_recompression
----
