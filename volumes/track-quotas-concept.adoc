= ONTAP Tracking-Kontingente verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Tracking-Quote erstellt einen Bericht über die Festplatten- und Dateinutzung und schränkt die Ressourcennutzung nicht ein. Wenn Tracking-Quoten verwendet werden, ist die Änderung der Quotenwerte weniger aufwändig, da Quoten angepasst werden können, anstatt sie zu deaktivieren und wieder zu aktivieren.

Um ein Tracking-Quota zu erstellen, werden die Parameter „Disk Limit“ und „Files Limit“ weggelassen. Dadurch überwacht ONTAP die Nutzung von Festplatten und Dateien für dieses Ziel auf dieser Ebene (Volume oder qtree), ohne Einschränkungen festzulegen. Tracking-Quotas werden in der Ausgabe von `show` Befehlen und im Quota-Bericht mit einem Bindestrich („-“) für alle Limits angezeigt. ONTAP erstellt Tracking-Quotas automatisch, wenn die System Manager UI verwendet wird, um explizite Quotas (Quotas mit bestimmten Zielen) zu erstellen. Bei Verwendung der CLI werden Tracking-Quotas vom Speicheradministrator zusätzlich zu den expliziten Quotas erstellt.

Sie können auch ein _Standard-Tracking-Kontingent_ festlegen, das für alle Instanzen des Ziels gilt. Standard-Tracking-Kontingente ermöglichen das Verfolgen der Nutzung für alle Instanzen eines Kontingenttyps (zum Beispiel alle Qtrees oder alle Benutzer). Außerdem besteht die Möglichkeit, eine Größenänderung anstelle einer Neuinitialisierung vorzunehmen, wenn Kontingentänderungen wirksam werden sollen.

.Beispiele
Die Ausgabe einer Tracking-Regel zeigt die für einen Qtree, einen Benutzer und eine Gruppe geltenden Tracking-Kontingente, wie im folgenden Beispiel für eine Tracking-Regel auf Volume-Ebene:

[listing]
----
Vserver: vs0             Policy: default             Volume: fv1

                                      Soft          Soft
                       User    Disk   Disk   Files  Files
Type  Target   Qtree   Mapping Limit  Limit  Limit  Limit  Threshold
----- -------- ------- ------- ------ ------ ------ ------ ---------
tree  ""       ""      -       -      -      -      -      -
user  ""       ""      off     -      -      -      -      -
group ""       ""      -       -      -      -      -      -
----
