= Erfahren Sie, wie ONTAP Benutzer-IDs in einer gemischten Umgebung ermittelt
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Benutzer von Windows- und UNIX-Clients auf Ihren ONTAP-Speicher zugreifen, werden sowohl Windows- als auch UNIX-Sicherheitsmechanismen zur Bestimmung des Dateibesitzes verwendet. Mehrere Faktoren bestimmen, ob ONTAP bei der Anwendung von Benutzerkontingenten eine UNIX- oder eine Windows-ID verwendet.

Wenn der Sicherheitsstil des Qtrees oder des FlexVol volume, das die Datei enthält, nur NTFS oder nur UNIX ist, bestimmt der Sicherheitsstil den Typ der ID, die beim Anwenden von Benutzerkontingenten verwendet wird. Für Qtrees mit gemischtem Sicherheitsstil wird der verwendete ID-Typ dadurch bestimmt, ob die Datei über eine ACL verfügt.

Die folgende Tabelle fasst zusammen, welche Art von ID verwendet wird.

[cols="3*"]
|===
| Sicherheitsstil | ACL | Keine ACL 


 a| 
UNIX
 a| 
UNIX-ID
 a| 
UNIX-ID



 a| 
Gemischt
 a| 
Windows ID
 a| 
UNIX-ID



 a| 
NTFS
 a|

 a|

|===
