= ONTAP Effizienzbetrieb und -Status anzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können überprüfen, ob Deduplizierung oder Datenkomprimierung für ein Volume aktiviert ist. Es ist außerdem möglich, den Status, den Zustand, den Komprimierungstyp und den Fortschritt der Effizienzvorgänge für ein Volume anzuzeigen.

Es stehen zwei Aufgaben zur Verfügung. Beide verwenden den Befehl `volume efficiency show`.



[[view-efficiency-status]]
== Effizienzstatus anzeigen

.Schritte
. Status einer Effizienzmaßnahme für ein Volume anzeigen: `volume efficiency show`
+
Der folgende Befehl zeigt den Status einer Effizienzoperation auf dem Volume VolA an, dem der adaptive Komprimierungstyp zugewiesen ist:

+
`volume efficiency show -instance -vserver vs1 -volume VolA`

+
Wenn der Effizienzvorgang für Volume VolA aktiviert ist und der Vorgang im Leerlauf ist, sind die folgenden Informationen in der Systemausgabe zu sehen:

+
[listing]
----
cluster1::> volume efficiency show -vserver vs1 -volume VolA

  Vserver Name: vs1
   Volume Name: VolA
   Volume Path: /vol/VolA
         State: Enabled
        Status: Idle
      Progress: Idle for 00:03:20
----




[[determine-if-volumes-contain-sequentially-packed-data]]
== Feststellen, ob Volumes sequenziell gepackte Daten enthalten

Eine Liste der Volumes, für die sequenzielles Packen aktiviert ist, kann angezeigt werden, beispielsweise wenn auf eine ONTAP Version vor 9.13.1 zurückgegangen werden muss. Für die Nutzung dieses Befehls ist der erweiterte Berechtigungsmodus erforderlich.

.Schritte
. Berechtigungsstufe festlegen: `set -privilege advanced`
. Volumes mit aktiviertem sequenziellen Packing auflisten:
+
`volume efficiency show -extended-auto-adaptive-compression true`
