= Bereiten Sie die Neuhostung eines ONTAP Volumes von einer SVM auf eine andere SVM vor.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Volume-Rehost-Operation ermöglicht die Übertragung eines NAS- oder SAN-Volumes von einer SVM auf eine andere SVM, ohne dass eine SnapMirror Kopie erforderlich ist. Das genaue Rehost-Verfahren hängt vom verwendeten Client-Zugriffsprotokoll und vom Volume-Typ ab. Die Volume-Rehost-Operation ist eine unterbrechende Maßnahme für den Datenzugriff und die Volume-Verwaltung.

Bevor Sie ein Volume von einer SVM auf eine andere verschieben können, müssen die folgenden Bedingungen erfüllt sein:

* Das Volume muss online sein
* Das Volume-Protokoll muss SAN oder NAS sein
+
** Bei NAS-Protokoll-Volumes darf das Volume nicht Teil des Verbindungspfads sein und muss ausgehängt werden.


* Befindet sich das Volume in einer SnapMirror Beziehung, muss diese SnapMirror Beziehung gelöscht und anschließend entweder nur die Beziehungsinformationen freigegeben oder vor dem Volume-Rehosting aufgehoben werden.
+
** Die SnapMirror Beziehung kann nach dem Volume-Rehosting-Vorgang erneut synchronisiert werden


* Der vserver-Subtyp sollte für Quell- und Ziel-SVMs identisch sein
+
** Volumes können nur zwischen SVMs desselben Subtyps neu gehostet werden.


* Das Volume kann weder FlexClone noch FlexClone Parent sein
+
** FlexClones müssen vor dem erneuten Hosten des übergeordneten oder geklonten Volumes aufgeteilt werden.
