= Von ONTAP unterstützte NDMP-Erweiterungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NDMP v4 bietet einen Mechanismus zur Erstellung von NDMP v4-Protokollerweiterungen, ohne das Kernprotokoll von NDMP v4 zu verändern. Es ist wichtig, die von ONTAP unterstützten NDMP v4-Erweiterungen zu kennen.

Die folgenden NDMP v4-Erweiterungen werden von ONTAP unterstützt:

* Cluster Aware Backup (CAB)
+
[NOTE]
====
Diese Erweiterung wird nur im SVM-basierten NDMP-Modus unterstützt.

====
* Connection Address Extension (CAE) für IPv6-Unterstützung
* Erweiterungsklasse 0x2050
+
Diese Erweiterung unterstützt wiederaufnehmbare Sicherungsvorgänge und Snapshot-Verwaltungserweiterungen.

+
[NOTE]
====
Die  `NDMP_SNAP_RECOVER` Nachricht, die Teil der Snapshot Management Extensions ist, wird verwendet, um einen Wiederherstellungsvorgang einzuleiten und die wiederhergestellten Daten von einem lokalen Snapshot an einen Speicherort im lokalen Dateisystem zu übertragen. In ONTAP ermöglicht diese Nachricht die Wiederherstellung von Volumes und regulären Dateien.

Die  `NDMP_SNAP_DIR_LIST`Meldung ermöglicht das Durchsuchen der Snapshots eines Volumes. Falls während eines laufenden Durchsuchungsvorgangs eine unterbrechungsfreie Operation erfolgt, muss die Backup-Anwendung den Durchsuchungsvorgang erneut initiieren.

====
* NDMP-Erweiterung für wiederanlaufbare Backups
+
Die NDMP-Funktion „Restartable Backup Extension“ (RBE) ermöglicht es, ein Tape-Backup von einem bekannten Prüfpunkt im Datentransfer vor dem Ausfall neu zu starten.
