= Funktionsweise von Dump mit ONTAP Volume Move
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Tape-Backup- und Wiederherstellungsvorgänge sowie die Volume-Verschiebung können parallel ausgeführt werden, bis das Speichersystem die endgültige Umstellungsphase einleitet. Nach dieser Phase sind neue Tape-Backup- und Wiederherstellungsvorgänge auf dem verschobenen Volume nicht mehr zulässig. Die aktuellen Vorgänge laufen jedoch bis zum Abschluss weiter.

Die folgende Tabelle beschreibt das Verhalten von Tape-Backup- und Wiederherstellungsvorgängen nach dem Volume-Verschiebevorgang:

|===
| Wenn Tape-Backup- und Wiederherstellungsvorgänge in der ... durchgeführt werden | Dann... 


 a| 
storage virtual machine (SVM) scoped NDMP-Modus, wenn die CAB-Erweiterung von der Backup-Anwendung unterstützt wird
 a| 
Inkrementelle Tape-Backup- und Wiederherstellungsvorgänge können auf lesen/schreiben- und Nur-Lese-Volumes weiterhin durchgeführt werden, ohne dass die Sicherungsrichtlinien neu konfiguriert werden müssen.



 a| 
SVM-bezogener NDMP-Modus, wenn die CAB-Erweiterung von der Backup-Anwendung nicht unterstützt wird
 a| 
Inkrementelle Tape-Backup- und Wiederherstellungsvorgänge auf lesen/schreiben- und Nur-Lese-Volumes sind weiterhin möglich, wenn die in der Backup-Richtlinie konfigurierte LIF auf den Knoten migriert wird, der das Zielaggregat hostet. Andernfalls ist nach der Volume-Verschiebung eine Basissicherung erforderlich, bevor die inkrementelle Sicherung durchgeführt werden kann.

|===
[NOTE]
====
Wenn eine Volume-Verschiebung stattfindet und das Volume, das zu einer anderen SVM auf dem Zielknoten gehört, denselben Namen wie das verschobene Volume hat, können keine inkrementellen Sicherungsvorgänge des verschobenen Volumes durchgeführt werden.
