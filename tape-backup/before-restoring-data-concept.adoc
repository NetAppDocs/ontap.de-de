= Überlegungen vor der Wiederherstellung von Daten mit ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können gesicherte Daten entweder am ursprünglichen Speicherort oder an einem anderen Zielort wiederherstellen. Wenn gesicherte Daten an einem anderen Zielort wiederhergestellt werden, muss das Ziel für den Wiederherstellungsvorgang vorbereitet werden.

Bevor Daten entweder an ihrem ursprünglichen Pfad oder an einem anderen Zielort wiederhergestellt werden, sind die folgenden Informationen erforderlich und die folgenden Anforderungen müssen erfüllt sein:

* Die Ebene der Wiederherstellung
* Der Pfad, zu dem die Daten wiederhergestellt werden
* Der während des Backups verwendete Blocking-Faktor
* Bei einer inkrementellen Wiederherstellung müssen alle Bänder in der Sicherungskette enthalten sein.
* Ein Tape-Laufwerk, das verfügbar und mit dem Band, von dem wiederhergestellt werden soll, kompatibel ist


Bevor Daten an einem anderen Zielort wiederhergestellt werden, sind die folgenden Schritte erforderlich:

* Wenn Sie ein Volume wiederherstellen, ist die Erstellung eines neuen Volumes erforderlich.
* Wenn Sie einen Qtree oder ein Verzeichnis wiederherstellen, müssen Dateien umbenannt oder verschoben werden, die wahrscheinlich denselben Namen wie die wiederherzustellenden Dateien haben.


[NOTE]
====
In ONTAP 9 werden Qtree-Namen im Unicode-Format unterstützt. Frühere Versionen von ONTAP unterstützen dieses Format nicht. Wenn ein Qtree mit Unicode-Namen in ONTAP 9 mithilfe des  `ndmpcopy` Befehls oder durch Wiederherstellung eines Backup-Images auf Band in eine frühere Version von ONTAP kopiert wird, wird der Qtree als normales Verzeichnis und nicht als Qtree im Unicode-Format wiederhergestellt.

====
[NOTE]
====
Wenn eine wiederhergestellte Datei denselben Namen wie eine vorhandene Datei hat, wird die vorhandene Datei durch die wiederhergestellte Datei überschrieben. Die Verzeichnisse werden jedoch nicht überschrieben.

====
Um eine Datei, ein Verzeichnis oder einen Qtree während der Wiederherstellung ohne Verwendung von DAR umzubenennen, muss die Umgebungsvariable EXTRACT auf  `E` gesetzt werden.



[[required-space-on-the-destination-storage-system]]
== Benötigter Speicherplatz auf dem Zielspeichersystem

Sie benötigen auf dem Zielspeichersystem etwa 100 MB mehr Speicherplatz als die Menge der wiederherzustellenden Daten.

[NOTE]
====
Der Wiederherstellungsvorgang prüft beim Start, ob auf dem Zielvolume Speicherplatz und Inodes verfügbar sind. Das Setzen der Umgebungsvariablen FORCE auf `Y` bewirkt, dass der Wiederherstellungsvorgang die Prüfungen auf Speicherplatz und Inode-Verfügbarkeit auf dem Zielpfad überspringt. Wenn auf dem Zielvolume nicht genügend Speicherplatz oder Inodes verfügbar sind, stellt der Wiederherstellungsvorgang so viele Daten wieder her, wie es der verfügbare Speicherplatz und die Inode-Verfügbarkeit auf dem Zielvolume zulassen. Der Wiederherstellungsvorgang endet, wenn kein Speicherplatz oder keine Inodes mehr vorhanden sind.

====
