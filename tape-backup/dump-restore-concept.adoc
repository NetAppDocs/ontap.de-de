= Wie die Wiederherstellung eines Speicherabbilds mit ONTAP NDMP funktioniert
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei einer Dump-Wiederherstellung werden Dateisystemdaten mithilfe eines vordefinierten Prozesses von Band auf Festplatte geschrieben.

Der Prozess in der folgenden Tabelle zeigt, wie die Wiederherstellung des Dumps funktioniert:

[cols="1,4"]
|===
| Stufe | Aktion 


 a| 
1
 a| 
ONTAP katalogisiert die Dateien, die vom Band extrahiert werden müssen.



 a| 
2
 a| 
ONTAP erstellt Verzeichnisse und leere Dateien.



 a| 
3
 a| 
ONTAP liest eine Datei vom Band, schreibt sie auf die Festplatte und legt die Berechtigungen (einschließlich ACLs) darauf fest.



 a| 
4
 a| 
ONTAP wiederholt die Schritte 2 und 3, bis alle angegebenen Dateien vom Band kopiert wurden.

|===


[[types-of-data-that-the-dump-engine-restores]]
== Datentypen, die die Dump-Engine wiederherstellt

Im Falle eines Datenverlusts oder einer Systemstörung bietet die Dump-Engine verschiedene Methoden zur Wiederherstellung aller gesicherten Daten, von einzelnen Dateien über Dateiattribute bis hin zu ganzen Verzeichnissen. Kenntnisse über die Datentypen, die die Dump-Engine wiederherstellen kann, und wann welche Wiederherstellungsmethode anzuwenden ist, können dazu beitragen, Ausfallzeiten zu minimieren.

Daten können auf einer online zugeordneten LUN wiederhergestellt werden. Hostanwendungen können jedoch erst nach Abschluss der Wiederherstellung auf diese LUN zugreifen. Nach Abschluss der Wiederherstellung sollte der Host-Cache der LUN-Daten geleert werden, um die Konsistenz mit den wiederhergestellten Daten sicherzustellen.

Die Dump-Engine kann folgende Daten wiederherstellen:

* Inhalte von Dateien und Verzeichnissen
* UNIX-Dateiberechtigungen
* ACLs
+
Wenn eine Datei, die ausschließlich UNIX-Dateiberechtigungen besitzt, in einem NTFS-Qtree oder Volume wiederhergestellt wird, verfügt die Datei über keine Windows NT ACLs. Das Speichersystem verwendet für diese Datei nur die UNIX-Dateiberechtigungen, bis eine Windows NT ACL darauf erstellt wird.

+
[NOTE]
====
Wenn ACLs, die von Speichersystemen mit Data ONTAP 8.2 gesichert wurden, auf Speichersysteme mit Data ONTAP 8.1.x und früheren Versionen wiederhergestellt werden, deren ACE-Limit unter 1.024 liegt, wird eine Standard-ACL wiederhergestellt.

====
* Qtree-Informationen
+
Qtree-Informationen werden nur verwendet, wenn ein Qtree im Stammverzeichnis eines Volumes wiederhergestellt wird. Qtree-Informationen werden nicht verwendet, wenn ein Qtree in ein Unterverzeichnis wie  `/vs1/vol1/subdir/lowerdir` wiederhergestellt wird und dadurch seine Qtree-Eigenschaften verliert.

* Alle anderen Datei- und Verzeichnisattribute
* Windows NT Streams
* LUNs
+
** Eine LUN muss auf Volume-Ebene oder Qtree-Ebene wiederhergestellt werden, damit sie als LUN erhalten bleibt.
+
Wird sie in ein Verzeichnis wiederhergestellt, erfolgt die Wiederherstellung als Datei, da keine gültigen Metadaten enthalten sind.

** Eine 7-Mode LUN wird als LUN auf einem ONTAP Volume wiederhergestellt.


* Ein 7-Mode Volume kann auf ein ONTAP Volume wiederhergestellt werden.
* VM-ausgerichtete Dateien, die auf ein Zielvolume wiederhergestellt werden, übernehmen die VM-Ausrichtungseigenschaften des Zielvolumes.
* Das Zielvolume für einen Wiederherstellungsvorgang kann Dateien mit obligatorischen oder beratenden Sperren enthalten.
+
Während der Wiederherstellung auf ein solches Zielvolume ignoriert die Dump-Engine diese Sperren.
