= Informationen zu Inkrementketten und ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Inkrementkette ist eine Reihe inkrementeller Backups desselben Pfades. Da Sie jederzeit eine beliebige Backup-Ebene festlegen können, ist das Verständnis von Inkrementketten erforderlich, um Backups und Wiederherstellungen effektiv durchführen zu können. Es können 31 Ebenen inkrementeller Backup-Operationen durchgeführt werden.

Es gibt zwei Arten von Inkrementketten:

* Eine Kette aufeinanderfolgender inkrementeller Sicherungen, also eine Folge inkrementeller Sicherungen, die mit Level 0 beginnt und bei jeder nachfolgenden Sicherung um 1 erhöht wird.
* Eine nicht aufeinanderfolgende Inkrementkette, bei der inkrementelle Backups Ebenen überspringen oder Ebenen aufweisen, die nicht in der richtigen Reihenfolge vorliegen, wie 0, 2, 3, 1, 4 oder häufiger 0, 1, 1, 1 oder 0, 1, 2, 1, 2.


Inkrementelle Backups basieren auf dem letzten Backup der niedrigeren Ebene. Beispielsweise ergibt die Reihenfolge der Backup-Ebenen 0, 2, 3, 1, 4 zwei Inkrementketten: 0, 2, 3 und 0, 1, 4. Die folgende Tabelle erläutert die Grundlagen der inkrementellen Backups:
