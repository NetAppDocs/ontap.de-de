= Funktionsweise von Dump bei ONTAP Storage Failover und ARL-Operationen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie Sicherungs- oder Wiederherstellungsvorgänge durchführen, sollte nachvollzogen werden, wie diese Vorgänge mit Storage-Failover (Übernahme und Rückgabe) oder Aggregate Relocation (ARL) ablaufen. Die  `-override-vetoes` Option bestimmt das Verhalten der Dump-Engine während eines Storage-Failover- oder ARL-Vorgangs.

Wenn eine Dump-Sicherung oder -Wiederherstellung ausgeführt wird und die  `-override-vetoes` Option auf  `false` gesetzt ist, wird ein vom Benutzer initiierter Storage-Failover- oder ARL-Vorgang gestoppt. Ist die  `–override-vetoes` Option jedoch auf  `true` gesetzt, wird der Storage-Failover- oder ARL-Vorgang fortgesetzt und die Dump-Sicherungs- oder -Wiederherstellungsoperation abgebrochen. Wird ein Storage-Failover- oder ARL-Vorgang automatisch vom Storage-System initiiert, wird eine aktive Dump-Sicherungs- oder -Wiederherstellungsoperation immer abgebrochen. Dump-Sicherungs- und -Wiederherstellungsoperationen können auch nach Abschluss eines Storage-Failover- oder ARL-Vorgangs nicht neu gestartet werden.



[[dump-operations-when-cab-extension-is-supported]]
== Dump-Operationen, wenn die CAB-Erweiterung unterstützt wird

Wenn die Backup-Anwendung die CAB-Erweiterung unterstützt, ist es möglich, inkrementelle Dump-Backup- und Wiederherstellungsvorgänge ohne Neukonfiguration der Backup-Richtlinien nach einem Storage-Failover oder einer ARL-Operation fortzusetzen.



[[dump-operations-when-cab-extension-is-not-supported]]
== Dump-Operationen, wenn die CAB-Erweiterung nicht unterstützt wird

Unterstützt die Sicherungsanwendung die CAB-Erweiterung nicht, können inkrementelle Dump-Backup- und Wiederherstellungsvorgänge weiterhin durchgeführt werden, wenn die in der Sicherungsrichtlinie konfigurierte LIF auf den Knoten migriert wird, der das Zielaggregat hostet. Andernfalls ist nach dem Storage-Failover und dem ARL-Vorgang vor der Durchführung der inkrementellen Sicherung eine Basissicherung erforderlich.

[NOTE]
====
Für Storage-Failover-Operationen muss die in der Sicherungsrichtlinie konfigurierte LIF auf den Partnerknoten migriert werden.
