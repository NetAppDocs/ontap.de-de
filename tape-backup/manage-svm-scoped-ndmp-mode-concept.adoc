= Informationen zur Verwaltung des ONTAP SVM-basierten NDMP-Modus für FlexVol volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NDMP kann für jede SVM mithilfe der NDMP-Optionen und -Befehle verwaltet werden. Die NDMP-Optionen können mit dem  `vserver services ndmp modify` Befehl geändert werden. Im SVM-bezogenen NDMP-Modus ist die Benutzerauthentifizierung in die rollenbasierte Zugriffssteuerung integriert.

NDMP kann mit dem  `vserver modify` Befehl zur Liste der zulässigen oder unzulässigen Protokolle hinzugefügt werden. Standardmäßig befindet sich NDMP in der Liste der zulässigen Protokolle. Wenn NDMP zur Liste der unzulässigen Protokolle hinzugefügt wird, können NDMP-Sitzungen nicht hergestellt werden.

Sie können den LIF-Typ, auf dem eine NDMP-Datenverbindung hergestellt wird, mithilfe der  `-preferred-interface-role` Option steuern. Während des Aufbaus einer NDMP-Datenverbindung wählt NDMP eine IP-Adresse, die dem durch diese Option angegebenen LIF-Typ zugeordnet ist. Wenn die IP-Adressen zu keinem dieser LIF-Typen gehören, kann die NDMP-Datenverbindung nicht hergestellt werden. Weitere Informationen zu  `vserver services ndmp modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-services-ndmp-modify.html["ONTAP-Befehlsreferenz"^].

== Befehle zur Verwaltung des ONTAP SVM-basierten NDMP-Modus

Mit den  `vserver services ndmp`Befehlen lässt sich NDMP auf jeder Storage Virtual Machine (SVM, früher bekannt als Vserver) verwalten.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
NDMP-Service aktivieren
 a| 
`vserver services ndmp on`

[NOTE]
====
NDMP-Service muss auf allen Knoten in einem Cluster immer aktiviert sein. NDMP-Service kann auf einem Knoten mit dem  `system services ndmp on` Befehl aktiviert werden. Standardmäßig ist der NDMP-Service auf einem Knoten immer aktiviert.

====


 a| 
NDMP-Service deaktivieren
 a| 
`vserver services ndmp off`



 a| 
NDMP-Konfiguration anzeigen
 a| 
`vserver services ndmp show`



 a| 
NDMP-Konfiguration ändern
 a| 
`vserver services ndmp modify`



 a| 
Standardmäßige NDMP-Version
 a| 
`vserver services ndmp version`



 a| 
Alle NDMP-Sitzungen werden angezeigt
 a| 
`vserver services ndmp status`



 a| 
Detaillierte Informationen zu allen NDMP-Sitzungen
 a| 
`vserver services ndmp probe`



 a| 
Eine bestimmte NDMP-Sitzung beenden
 a| 
`vserver services ndmp kill`



 a| 
Alle NDMP-Sitzungen beenden
 a| 
`vserver services ndmp kill-all`



 a| 
Das NDMP-Passwort generieren
 a| 
`vserver services ndmp generate-password`



 a| 
NDMP-Erweiterungsstatus anzeigen
 a| 
`vserver services ndmp extensions show`

Dieser Befehl ist auf der Stufe „Erweiterte Berechtigungen“ verfügbar.



 a| 
NDMP-Erweiterungsstatus (aktivieren oder deaktivieren) ändern
 a| 
`vserver services ndmp extensions modify`

Dieser Befehl ist auf der Stufe „Erweiterte Berechtigungen“ verfügbar.



 a| 
Protokollierung für die angegebene NDMP-Sitzung starten
 a| 
`vserver services ndmp log start`

Dieser Befehl ist auf der Stufe „Erweiterte Berechtigungen“ verfügbar.



 a| 
Protokollierung für die angegebene NDMP-Sitzung beenden
 a| 
`vserver services ndmp log stop`

Dieser Befehl ist auf der Stufe „Erweiterte Berechtigungen“ verfügbar.

|===
Weitere Informationen zu  `vserver services ndmp` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+services+ndmp["ONTAP-Befehlsreferenz"^].
