= ONTAP Daten mit ndmpcopy übertragen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der  `ndmpcopy`nodeshell-Befehl überträgt Daten zwischen Storage-Systemen, die NDMP v4 unterstützen. Sowohl vollständige als auch inkrementelle Datentransfers sind möglich. Vollständige oder teilweise Volumes, Qtrees, Verzeichnisse oder einzelne Dateien können übertragen werden.

.Über diese Aufgabe
Bei ONTAP 8.x und früheren Versionen sind inkrementelle Übertragungen auf maximal zwei Ebenen beschränkt (eine vollständige und bis zu zwei inkrementelle Sicherungen).

Ab ONTAP 9.0 und späteren Versionen sind inkrementelle Übertragungen auf maximal neun Ebenen beschränkt (eine vollständige und bis zu neun inkrementelle Backups).

Der Befehl  `ndmpcopy` kann an der nodeshell Befehlszeile des Quell- und Ziel-Storage-Systems oder eines Storage-Systems ausgeführt werden, das weder das Quell- noch das Zielsystem des Datentransfers ist. Der Befehl  `ndmpcopy` kann auch auf einem einzelnen Storage-System ausgeführt werden, das sowohl das Quell- als auch das Zielsystem des Datentransfers ist.

Sie können die IPv4- oder IPv6-Adressen der Quell- und Ziel-Storage-Systeme im  `ndmpcopy` Befehl verwenden. Das Pfadformat ist  `/vserver_name/volume_name \[path\]`.



.Schritte
. NDMP-Service auf den Quell- und Ziel-Storage-Systemen aktivieren:
+
|===


| Wenn Sie den Datentransfer am Quell- oder Zielsystem durchführen... | Der folgende Befehl kann verwendet werden... 


 a| 
SVM-basierter NDMP-Modus
 a| 
`vserver services ndmp on`

[NOTE]
====
Für die NDMP-Authentifizierung in der Admin-SVM ist das Benutzerkonto  `admin` und die Benutzerrolle  `admin` oder  `backup`. In der Daten-SVM ist das Benutzerkonto  `vsadmin` und die Benutzerrolle  `vsadmin` oder  `vsadmin-backup`-Rolle.

====


 a| 
Knotenbezogener NDMP-Modus
 a| 
`system services ndmp on`

|===
. Daten innerhalb eines Speichersystems oder zwischen Speichersystemen mit dem  `ndmpcopy` Befehl an der Nodeshell übertragen:
+
`::> system node run -node <node_name> < ndmpcopy [options] source_IP:source_path destination_IP:destination_path [-mcs {inet|inet6}] [-mcd {inet|inet6}] [-md {inet|inet6}]`

+
[NOTE]
====
DNS-Namen werden in ndmpcopy nicht unterstützt. Die IP-Adresse des Quell- und des Zielsystems muss angegeben werden. Die Loopback-Adresse (127.0.0.1) wird weder als Quell-IP-Adresse noch als Ziel-IP-Adresse unterstützt.

====
+
** Der `ndmpcopy` Befehl bestimmt den Adressierungsmodus für Steuerverbindungen wie folgt:
+
*** Der Adressmodus für die Steuerverbindung entspricht der angegebenen IP-Adresse.
*** Sie können diese Regeln mit den  `-mcs`- und  `-mcd`-Optionen außer Kraft setzen.


** Wenn das Quell- oder das Zielsystem das ONTAP System ist, sollte je nach NDMP-Modus (node-scoped oder SVM-scoped) eine IP-Adresse verwendet werden, die den Zugriff auf das Ziel-Volume ermöglicht.
** `source_path` und  `destination_path` sind die absoluten Pfadnamen bis hin zur granularen Ebene von Volume, Qtree, Verzeichnis oder Datei.
** `-mcs` gibt den bevorzugten Adressierungsmodus für die Steuerverbindung zum Quell-Storage-System an.
+
`inet` gibt einen IPv4-Adressmodus an und  `inet6` gibt einen IPv6-Adressmodus an.

** `-mcd`legt den bevorzugten Adressierungsmodus für die Steuerverbindung zum Ziel-Storage-System fest.
+
`inet` gibt einen IPv4-Adressmodus an und  `inet6` gibt einen IPv6-Adressmodus an.

** `-md` gibt den bevorzugten Adressierungsmodus für Datentransfers zwischen dem Quell- und dem Ziel-Storage-System an.
+
`inet` gibt einen IPv4-Adressmodus an und  `inet6` gibt einen IPv6-Adressmodus an.

+
Wenn die Option  `-md` im Befehl  `ndmpcopy` nicht verwendet wird, wird der Adressierungsmodus für die Datenverbindung wie folgt bestimmt:

+
*** Wenn eine der für die Steuerverbindungen angegebenen Adressen eine IPv6-Adresse ist, ist der Adressmodus für die Datenverbindung IPv6.
*** Wenn beide für die Steuerverbindungen angegebenen Adressen IPv4-Adressen sind, versucht der  `ndmpcopy`Befehl zunächst, für die Datenverbindung einen IPv6-Adressmodus zu verwenden.
+
Falls das fehlschlägt, verwendet der Befehl den IPv4-Adressmodus.

+
[NOTE]
====
Eine IPv6-Adresse muss, sofern angegeben, in eckige Klammern gesetzt werden.

====
+
Dieser Beispielbefehl migriert Daten von einem Quellpfad (`source_path`) zu einem Zielpfad (`destination_path`).

+
[listing]
----
> ndmpcopy -sa admin:<ndmp_password> -da admin:<ndmp_password>
 -st md5 -dt md5 192.0.2.129:/<src_svm>/<src_vol> 192.0.2.131:/<dst_svm>/<dst_vol>
----
+
Dieser Beispielbefehl legt explizit fest, dass die Steuerverbindungen und die Datenverbindung den IPv6-Adressmodus verwenden:

+
[listing]
----
> ndmpcopy -sa admin:<ndmp_password> -da admin:<ndmp_password> -st md5 -dt md5 -mcs inet6 -mcd inet6 -md
 inet6 [2001:0db8:1:1:209:6bff:feae:6d67]:/<src_svm>/<src_vol> [2001:0ec9:1:1:200:7cgg:gfdf:7e78]:/<dst_svm>/<dst_vol>
----






Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].



[[options-for-the-ndmpcopy-command]]
== Optionen für den Befehl ndmpcopy

Sie sollten die für den  `ndmpcopy`nodeshell Befehl verfügbaren Optionen verstehen, um link:transfer-data-ndmpcopy-task.html["Daten übertragen"]erfolgreich auszuführen.

Die folgende Tabelle listet die verfügbaren Optionen auf.

|===
| Option | Beschreibung 


 a| 
`-sa` `username`:[`password`]
 a| 
Diese Option legt den Authentifizierungs-Benutzernamen und das Passwort für die Verbindung mit dem Quell-Storage-System fest. Diese Option ist obligatorisch.

Für Benutzer ohne Administratorrechte muss das systemgenerierte, NDMP-spezifische Passwort des Benutzers angegeben werden. Das systemgenerierte Passwort ist sowohl für Administratoren als auch für Benutzer ohne Administratorrechte obligatorisch.



 a| 
`-da` `username`:[`password`]
 a| 
Diese Option legt den Authentifizierungs-Benutzernamen und das Passwort für die Verbindung mit dem Ziel-Storage-System fest. Diese Option ist obligatorisch.



 a| 
`-st` {`md5`|`text`}
 a| 
Diese Option legt den Quell-Authentifizierungstyp fest, der beim Verbinden mit dem Quell-Storage-System verwendet wird. Dies ist eine obligatorische Option, daher sollte entweder die  `text`- oder die  `md5`-Option angegeben werden.



 a| 
`-dt` {`md5`|`text`}
 a| 
Mit dieser Option wird der Authentifizierungstyp für die Verbindung zum Ziel-Storage-System festgelegt.



 a| 
`-l`
 a| 
Diese Option legt die für den Datentransfer verwendete Sicherungsebene auf den angegebenen Wert von level fest. Gültige Werte sind  `0`,  `1`, bis  `9`, wobei  `0` eine vollständige Übertragung und  `1` bis  `9` eine inkrementelle Übertragung angeben. Der Standardwert ist `0`.



 a| 
`-d`
 a| 
Diese Option ermöglicht die Generierung von ndmpcopy-Debug-Protokollmeldungen. Die ndmpcopy-Debug-Protokolldateien befinden sich im `/mroot/etc/log` Stammverzeichnis. Die Dateinamen der ndmpcopy-Debug-Protokolldateien sind im `ndmpcopy.yyyymmdd` Format.



 a| 
`-f`
 a| 
Diese Option aktiviert den erzwungenen Modus. Dieser Modus ermöglicht das Überschreiben von Systemdateien im  `/etc` Verzeichnis auf dem Root des 7-Mode-Volumes.



 a| 
`-h`
 a| 
Diese Option gibt die Hilfemeldung aus.



 a| 
`-p`
 a| 
Diese Option fordert zur Eingabe des Passworts für die Quell- und Zielautorisierung auf. Dieses Passwort überschreibt das für  `-sa` und  `-da` angegebene Passwort.

[NOTE]
====
Diese Option kann nur verwendet werden, wenn der Befehl in einer interaktiven Konsole ausgeführt wird.

====


 a| 
`-exclude`
 a| 
Diese Option schließt bestimmte Dateien oder Verzeichnisse vom angegebenen Pfad für den Datentransfer aus. Der Wert kann eine durch Kommas getrennte Liste von Verzeichnis- oder Dateinamen sein, wie zum Beispiel `*.pst` oder `*.txt`. Die maximale Anzahl unterstützter Ausschlussmuster beträgt 32 und die maximale Anzahl unterstützter Zeichen beträgt 255.

|===
