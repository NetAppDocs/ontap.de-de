= Skalierbarkeitsgrenzen für ONTAP Dump-Backup und Wiederherstellungssitzungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie müssen sich der maximalen Anzahl von Dump-Backup- und Wiederherstellungssitzungen bewusst sein, die gleichzeitig auf Speichersystemen mit unterschiedlicher System-Speicherkapazität durchgeführt werden können. Diese maximale Anzahl hängt vom Systemspeicher eines Speichersystems ab.

Die in der folgenden Tabelle genannten Grenzwerte gelten für die Dump- oder Wiederherstellungs-Engine. Die in den Skalierbarkeitsgrenzwerten für NDMP-Sitzungen genannten Grenzwerte gelten für den NDMP-Server, welche höher sind als die Grenzwerte der Engine.

|===
| Systemspeicher eines Speichersystems | Gesamtzahl der Dump-Backup- und Wiederherstellungssitzungen 


 a| 
Weniger als 16 GB
 a| 
4



 a| 
Größer oder gleich 16 GB, aber kleiner als 24 GB
 a| 
16



 a| 
Größer oder gleich 24 GB
 a| 
32

|===
[NOTE]
====
Wenn der  `ndmpcopy` Befehl zum Kopieren von Daten innerhalb von Speichersystemen verwendet wird, werden zwei NDMP-Sitzungen eingerichtet, eine für das Dump-Backup und die andere für das Dump-Restore.

====
Sie können den Systemspeicher Ihres Speichersystems mithilfe des  `sysconfig -a` Befehls (verfügbar über die nodeshell) abrufen. Weitere Informationen zu  `sysconfig -a` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-node-run.html["ONTAP-Befehlsreferenz"^].
