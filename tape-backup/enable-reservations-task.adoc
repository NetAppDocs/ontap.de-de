= ONTAP Bandreservierungen aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können steuern, wie ONTAP Tape-Gerät-Reservierungen verwaltet, indem Sie die  `tape.reservations` Option verwenden. Standardmäßig ist die Tape-Reservierung deaktiviert.

.Über diese Aufgabe
Die Aktivierung der Bandreservierungsoption kann Probleme verursachen, wenn Bandlaufwerke, Medienwechsler, Bridges oder Bibliotheken nicht ordnungsgemäß funktionieren. Wenn Bandbefehle melden, dass das Gerät reserviert ist, obwohl kein anderes Speichersystem das Gerät verwendet, sollte diese Option deaktiviert werden.

.Schritte
. Um entweder den SCSI Reserve/Release-Mechanismus oder SCSI Persistent Reservations zu verwenden oder Tape-Reservierungen zu deaktivieren, ist folgender Befehl in der Clustershell einzugeben:
+
`*options -option-name tape.reservations -option-value {scsi | persistent | off}*`

+
`scsi`Wählt den SCSI Reserve/Release-Mechanismus aus.

+
`persistent` SCSI Persistent Reservations wird ausgewählt.

+
`off`deaktiviert Tape-Reservierungen.
