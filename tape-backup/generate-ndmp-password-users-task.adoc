= Ein NDMP-spezifisches Passwort für ONTAP NDMP Benutzer generieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Im Storage Virtual Machine (SVM)-bezogenen NDMP-Modus muss ein Passwort für eine bestimmte Benutzer-ID generiert werden. Das generierte Passwort basiert auf dem tatsächlichen Anmeldepasswort des NDMP-Benutzers. Wenn sich das tatsächliche Anmeldepasswort ändert, muss das NDMP-spezifische Passwort erneut generiert werden.

.Schritte
. Mit dem  `vserver services ndmp generate-password` Befehl lässt sich ein NDMP-spezifisches Passwort generieren.
+
Dieses Passwort kann in allen aktuellen oder zukünftigen NDMP-Operationen verwendet werden, die eine Passworteingabe erfordern.

+
[NOTE]
====
Im Kontext der Storage Virtual Machine (SVM, früher bekannt als Vserver) können NDMP-Passwörter nur für Benutzer generiert werden, die zu dieser SVM gehören.

====
+
Das folgende Beispiel zeigt, wie ein NDMP-spezifisches Passwort für die Benutzer-ID user1 generiert wird:

+
[listing]
----

cluster1::vserver services ndmp> generate-password -vserver vs1 -user user1

Vserver: vs1
User: user1
Password: jWZiNt57huPOoD8d
----
. Wenn Sie das Passwort für Ihr reguläres Speichersystemkonto ändern, wiederholen Sie dieses Verfahren, um Ihr neues NDMP-spezifisches Passwort zu erhalten.
