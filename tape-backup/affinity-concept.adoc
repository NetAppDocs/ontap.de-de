= Informationen zu Affinität für ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Da die Backup-Anwendung CAB-fähig ist, liefert der NDMP-Server eindeutige Standortinformationen zu Volumes und Tape-Geräten. Mithilfe dieser Affinitätsinformationen kann die Backup-Anwendung ein lokales Backup anstelle eines Three-Way-Backups durchführen, wenn ein Volume und ein Tape-Gerät die gleiche Affinität aufweisen.

Wenn die NDMP-Steuerungsverbindung auf einem Knotenverwaltungs-LIF, Clusterverwaltungs-LIF oder einem Intercluster-LIF hergestellt ist, kann die Sicherungsanwendung anhand der Affinitätsinformationen feststellen, ob sich ein Volume und ein Tape-Gerät auf demselben Knoten befinden und anschließend entweder eine lokale oder eine Drei-Wege-Sicherung oder -Wiederherstellung durchführen. Wenn die NDMP-Steuerungsverbindung auf einem Daten-LIF hergestellt ist, führt die Sicherungsanwendung immer eine Drei-Wege-Sicherung durch.



[[local-ndmp-backup-and-three-way-ndmp-backup]]
== Lokale NDMP Backup und Three-way NDMP Backup

image:local_and_three-way_backup_in_vserver_aware_ndmp_mode.png["lokale und Drei-Wege-Backup-Diagramme"]

Anhand der Affinitätsinformationen zu Volumes und Tape-Geräten führt die DMA (Backup-Anwendung) ein lokales NDMP-Backup auf dem Volume und Tape-Gerät auf Node 1 im Cluster durch. Wenn das Volume von Node 1 auf Node 2 verschoben wird, ändern sich die Affinitätsinformationen zu Volume und Tape-Gerät. Daher wird für ein nachfolgendes Backup von der DMA eine Three-Way-NDMP-Backup-Operation durchgeführt. Dadurch bleibt die Backup-Richtlinie für das Volume unabhängig davon erhalten, auf welchen Node das Volume verschoben wird.
