= Wie SMTape mit ONTAP Storage-Failover und ARL-Operationen funktioniert
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie SMTape-Backup- oder Wiederherstellungsvorgänge durchführen, sollte nachvollzogen werden, wie diese Vorgänge mit Storage-Failover (Takeover und Giveback) oder einer Aggregate Relocation (ARL) Operation zusammenwirken. Die `-override-vetoes` Option bestimmt das Verhalten der SMTape Engine während eines Storage-Failovers oder einer ARL-Operation.

Wenn ein SMTape-Backup- oder Restore-Vorgang ausgeführt wird und die `-override-vetoes` Option auf `false` gesetzt ist, wird ein vom Benutzer initiierter Storage-Failover- oder ARL-Vorgang gestoppt und der Backup- oder Restore-Vorgang abgeschlossen. Wenn die Backup-Anwendung die CAB-Erweiterung unterstützt, können inkrementelle SMTape-Backup- und Restore-Vorgänge weiterhin durchgeführt werden, ohne die Backup-Richtlinien neu konfigurieren zu müssen. Wenn jedoch die `–override-vetoes` Option auf `true` gesetzt ist, wird der Storage-Failover- oder ARL-Vorgang fortgesetzt und der SMTape-Backup- oder Restore-Vorgang abgebrochen.
