= Überlegungen bei der Konfiguration des ONTAP Multipath-Tape-Zugriffs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es können zwei Pfade vom Speichersystem konfiguriert werden, um auf die Tape-Laufwerke in einer Tape-Bibliothek zuzugreifen. Wenn ein Pfad ausfällt, kann das Speichersystem die anderen Pfade nutzen, um auf die Tape-Laufwerke zuzugreifen, ohne den ausgefallenen Pfad sofort reparieren zu müssen. So wird sichergestellt, dass Tape-Operationen neu gestartet werden können.

Bei der Konfiguration des Multipath-Tape-Zugriffs von Ihrem Speichersystem sind die folgenden Punkte zu berücksichtigen:

* In Tape-Bibliotheken, die LUN-Mapping unterstützen, muss das LUN-Mapping für den Mehrwegezugriff auf eine LUN-Gruppe auf jedem Pfad symmetrisch sein.
+
Tape-Laufwerke und Medienwechsler werden in einer Tape-Bibliothek LUN-Gruppen (einem Satz von LUNs, die denselben Initiator-Pfad-Satz gemeinsam nutzen) zugewiesen. Alle Tape-Laufwerke einer LUN-Gruppe müssen für Sicherungs- und Wiederherstellungsvorgänge auf allen Pfaden verfügbar sein.

* Es können maximal zwei Pfade vom Speichersystem zur Nutzung der Tape-Laufwerke in einer Tape-Bibliothek konfiguriert werden.
* Multipath-Tape-Zugriff unterstützt Lastverteilung. Lastverteilung ist standardmäßig deaktiviert.


Im folgenden Beispiel greift das Speichersystem über zwei Initiatorpfade, 0b und 0d, auf die LUN-Gruppe 0 zu. In beiden Pfaden hat die LUN-Gruppe dieselbe LUN-Nummer, 0, und dieselbe LUN-Anzahl, 5. Das Speichersystem greift auf die LUN-Gruppe 1 nur über einen Initiatorpfad, 3d, zu.
