= Informationen zur Verwaltung des ONTAP node-scoped NDMP-Modus für FlexVol Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NDMP kann auf Knotenebene mithilfe von NDMP-Optionen und NDMP-Befehlen verwaltet werden. Die NDMP-Optionen können mit dem  `options` Befehl geändert werden. Für den Zugriff auf ein Speichersystem zur Durchführung von Tape-Backup- und Wiederherstellungsvorgängen sind NDMP-spezifische Anmeldeinformationen erforderlich.

Weitere Informationen zu  `options` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=options["ONTAP-Befehlsreferenz"^].



[[commands-for-managing-ontap-node-scoped-ndmp-mode]]
== Befehle zur Verwaltung des ONTAP node-scoped NDMP-Modus

Die  `system services ndmp`Befehle können zur Verwaltung von NDMP auf Knotenebene verwendet werden. Einige dieser Befehle sind veraltet und werden in einer zukünftigen Hauptversion entfernt.

Die folgenden NDMP-Befehle können nur auf der erweiterten Berechtigungsstufe verwendet werden:

* `system services ndmp service terminate`
* `system services ndmp service start`
* `system services ndmp service stop`
* `system services ndmp log start`
* `system services ndmp log stop`

[[user-authentication-in-a-node-scoped-ndmp-mode]]
== Benutzerauthentifizierung im knotenbezogenen NDMP-Modus

Im knotenbezogenen NDMP-Modus müssen NDMP-spezifische Anmeldeinformationen verwendet werden, um auf ein Speichersystem zuzugreifen und Tape-Backup- sowie Wiederherstellungsvorgänge durchzuführen.

Die Standardbenutzer-ID ist „root“. Vor der Verwendung von NDMP auf einem Node muss sichergestellt werden, dass das Standard-NDMP-Passwort, das dem NDMP-Benutzer zugeordnet ist, geändert wird. Auch die Standard-NDMP-Benutzer-ID kann geändert werden.
