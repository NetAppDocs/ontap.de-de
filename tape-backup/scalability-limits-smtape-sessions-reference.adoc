= ONTAP Skalierbarkeitsgrenzen für SMTape-Backup- und Wiederherstellungssitzungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bei der Durchführung von SMTape Backup- und Wiederherstellungsvorgängen über NDMP oder CLI (Tape-Seeding) ist die maximale Anzahl von SMTape Backup- und Wiederherstellungssitzungen zu berücksichtigen, die gleichzeitig auf Speichersystemen mit unterschiedlichen System-Speicherkapazitäten durchgeführt werden können. Diese maximale Anzahl hängt vom Systemspeicher eines Speichersystems ab.

[NOTE]
====
Die Skalierbarkeitsgrenzen für SMTape-Backup- und Restore-Sitzungen unterscheiden sich von den NDMP-Sitzungsgrenzen und den Löschbefehl-Sitzungsgrenzen.

====
|===
| Systemspeicher des Speichersystems | Gesamtzahl der SMTape Sicherungs- und Wiederherstellungssitzungen 


 a| 
Weniger als 16 GB
 a| 
6



 a| 
Größer oder gleich 16 GB, aber kleiner als 24 GB
 a| 
16



 a| 
Größer oder gleich 24 GB
 a| 
32

|===
Sie können den Systemspeicher Ihres Speichersystems mithilfe des  `sysconfig -a` Befehls (verfügbar über die nodeshell) abrufen. Weitere Informationen zu  `sysconfig -a` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-node-run.html["ONTAP-Befehlsreferenz"^].
