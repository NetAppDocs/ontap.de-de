= ONTAP Dump und Restore Ereignisprotokoll-Nachrichtenformat
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für jeden Sicherungs- und Wiederherstellungsvorgang wird eine Meldung in die Sicherungsprotokolldatei geschrieben.

Das Format der Ereignisprotokollmeldung für Dump und Restore ist wie folgt:

`_type timestamp identifier event (event_info)_`

Die folgende Liste beschreibt die Felder im Ereignisprotokollnachrichtenformat:

* Jede Logmeldung beginnt mit einem der in der folgenden Tabelle beschriebenen Typindikatoren:
+
|===
| Typ | Beschreibung 


 a| 
Protokoll
 a| 
Protokollierungsereignis



 a| 
dmp
 a| 
Dump-Ereignis



 a| 
rst
 a| 
Wiederherstellungsereignis

|===
* `timestamp` zeigt Datum und Uhrzeit des Ereignisses an.
* Das `identifier` Feld für ein Speicherabbildereignis enthält den Speicherabbildpfad und die eindeutige ID des Speicherabbilds. Das `identifier` Feld für ein Wiederherstellungsereignis verwendet nur den Namen des Wiederherstellungszielpfads als eindeutigen Bezeichner. Protokollierungsbezogene Ereignismeldungen enthalten kein `identifier` Feld.




[[learn-about-ontap-logging-events]]
== Informationen zu ONTAP Protokollierungsereignissen

Das Ereignisfeld einer Nachricht, die mit einem Logeintrag beginnt, gibt den Beginn einer Protokollierung oder das Ende einer Protokollierung an.

Es enthält eines der in der folgenden Tabelle aufgeführten Ereignisse:

|===
| Ereignis | Beschreibung 


 a| 
Start_Protokollierung
 a| 
Zeigt den Beginn der Protokollierung oder dass die Protokollierung nach ihrer Deaktivierung wieder aktiviert wurde.



 a| 
Stop_Logging
 a| 
Gibt an, dass die Protokollierung deaktiviert wurde.

|===


[[learn-about-ontap-dump-events]]
== Informationen zu ONTAP Dump-Ereignissen

Das Ereignisfeld für ein Dump-Ereignis enthält einen Ereignistyp, gefolgt von ereignisspezifischen Informationen in Klammern.

Die folgende Tabelle beschreibt die Ereignisse, ihre Beschreibungen und die zugehörigen Ereignisinformationen, die bei einem Dump-Vorgang protokolliert werden können:

|===
| Ereignis | Beschreibung | Veranstaltungsinformationen 


 a| 
Start
 a| 
NDMP dump wird gestartet
 a| 
Dump-Level und Dump-Typ



 a| 
Ende
 a| 
Dumps erfolgreich abgeschlossen
 a| 
Menge der verarbeiteten Daten



 a| 
Abbrechen
 a| 
Der Vorgang wurde abgebrochen.
 a| 
Menge der verarbeiteten Daten



 a| 
Optionen
 a| 
Angegebene Optionen sind aufgelistet
 a| 
Alle Optionen und ihre zugehörigen Werte, einschließlich NDMP-Optionen



 a| 
Tape_open
 a| 
Das Band ist zum lesen/schreiben geöffnet
 a| 
Der neue Tape-Gerät Name



 a| 
Tape_close
 a| 
Das Band ist für lesen/schreiben geschlossen
 a| 
Der Tape-Gerät-Name



 a| 
Phasenänderung
 a| 
Ein Dump befindet sich in einer neuen Verarbeitungsphase.
 a| 
Der neue Phasenname



 a| 
Fehler
 a| 
Ein Dump ist auf ein unerwartetes Ereignis gestoßen.
 a| 
Fehlermeldung



 a| 
Schnappschuss
 a| 
Ein Snapshot wird erstellt oder gefunden
 a| 
Name und Zeitpunkt der Momentaufnahme



 a| 
Base_dump
 a| 
Ein Basis-Dump-Eintrag in der internen Metadatei wurde gefunden.
 a| 
Niveau und Zeitpunkt des Basis-Dumps (nur für inkrementelle Dumps)

|===
