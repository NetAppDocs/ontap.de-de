= Informationen zu gängigen ONTAP NDMP Tape-Backup-Topologien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NDMP unterstützt eine Reihe von Topologien und Konfigurationen zwischen Backup-Anwendungen und Speichersystemen oder anderen NDMP-Servern, die Daten (Dateisysteme) und Tape-Dienste bereitstellen.



[[storage-system-to-local-tape]]
== Speichersystem zu lokalem Tape

In der einfachsten Konfiguration sichert eine Backup-Anwendung Daten von einem Speichersystem auf ein an das Speichersystem angeschlossenes Tape-Subsystem. Die NDMP-Steuerverbindung besteht über die Netzwerkgrenze. Die NDMP-Datenverbindung, die innerhalb des Speichersystems zwischen den Daten- und Tape-Diensten besteht, wird als NDMP-Lokalkonfiguration bezeichnet.



[[storage-system-to-tape-attached-to-another-storage-system]]
== Speichersystem zu Tape-Anbindung an ein anderes Speichersystem

Eine Backup-Anwendung kann Daten von einem Speichersystem auch auf eine Tape-Bibliothek (einen Medienwechsler mit einem oder mehreren Tape-Laufwerken) sichern, die an ein anderes Speichersystem angeschlossen ist. In diesem Fall wird die NDMP-Datenverbindung zwischen den Daten- und Tape-Diensten durch eine TCP- oder TCP/IPv6-Netzwerkverbindung bereitgestellt. Dies wird als NDMP Three-Way Storage System-to-Storage System Konfiguration bezeichnet.



[[storage-system-to-network-attached-tape-library]]
== Speichersystem zu netzwerkgebundener Tape-Bibliothek

NDMP-fähige Tape-Bibliotheken bieten eine Variante der Drei-Wege-Konfiguration. In diesem Fall ist die Tape-Bibliothek direkt an das TCP/IP-Netzwerk angeschlossen und kommuniziert über einen internen NDMP-Server mit der Backup-Anwendung und dem Speichersystem.



[[storage-system-to-data-server-to-tape-or-data-server-to-storage-system-to-tape]]
== Speichersystem zu Datenserver zu Tape oder Datenserver zu Speichersystem zu Tape

NDMP unterstützt auch Speichersystem-zu-Datenserver- und Datenserver-zu-Speichersystem-Dreierkonfigurationen, wobei diese Varianten weniger verbreitet sind. Speichersystem-zu-Server ermöglicht, dass Speichersystemdaten auf eine an den Backup-Anwendungs-Host angeschlossene Tape-Bibliothek oder auf ein anderes Datenserversystem gesichert werden. Die Server-zu-Speichersystem-Konfiguration ermöglicht die Sicherung von Serverdaten auf eine an das Speichersystem angeschlossene Tape-Bibliothek.
