= Zu beachtende Punkte bei der Nutzung des ONTAP NDMP-Service
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Beim Starten des NDMP-Service auf Ihrem Speichersystem sind eine Reihe von Aspekten zu berücksichtigen.

* Jeder Knoten unterstützt maximal 16 gleichzeitige Backups, Wiederherstellungen oder eine Kombination aus beidem unter Verwendung angeschlossener Bandlaufwerke.
* NDMP-Dienste können auf Anfrage von NDMP-Backup-Anwendungen Dateiverlaufsdaten generieren.
+
Die Dateiversionshistorie wird von Backup-Anwendungen genutzt, um die optimierte Wiederherstellung ausgewählter Datenteilmengen aus einem Backup-Image zu ermöglichen. Die Generierung und Verarbeitung der Dateiversionshistorie kann sowohl für das Speichersystem als auch für die Backup-Anwendung zeitaufwändig und CPU-intensiv sein.

+
[NOTE]
====
SMTape unterstützt keinen Dateiverlauf.

====
+
Wenn Ihre Datensicherung für die Notfallwiederherstellung konfiguriert ist, bei der das gesamte Backup-Image wiederhergestellt wird, kann die Dateiverlaufsgenerierung deaktiviert werden, um die Backup-Zeit zu verkürzen. In der Dokumentation der Backup-Anwendung ist ersichtlich, ob die NDMP-Dateiverlaufsgenerierung deaktiviert werden kann.

* Die Firewall-Richtlinie für NDMP ist standardmäßig auf allen LIF-Typen aktiviert.
* Im knotenbezogenen NDMP-Modus ist zum Sichern eines FlexVol volume erforderlich, dass die Sicherungsanwendung verwendet wird, um eine Sicherung auf einem Knoten zu initiieren, dem das Volume gehört.
+
Ein Backup des Root-Volumes eines Knotens ist jedoch nicht möglich.

* NDMP-Backup kann von jedem LIF aus durchgeführt werden, sofern dies durch die Firewall-Richtlinien zulässig ist.
+
Wenn ein Daten-LIF verwendet wird, muss ein LIF ausgewählt werden, das nicht für Failover konfiguriert ist. Wenn ein Daten-LIF während eines NDMP-Vorgangs ein Failover durchführt, schlägt der NDMP-Vorgang fehl und muss erneut ausgeführt werden.

* Im node-scoped NDMP-Modus und im Storage Virtual Machine (SVM) scoped NDMP-Modus ohne CAB-Erweiterungsunterstützung verwendet die NDMP-Datenverbindung die gleiche LIF wie die NDMP-Steuerverbindung.
* Während der LIF-Migration werden laufende Backup- und Wiederherstellungsvorgänge unterbrochen.
+
Die Sicherungs- und Wiederherstellungsvorgänge sind nach der LIF-Migration zu initiieren.

* Der NDMP-Backup-Pfad hat das Format  `_/vserver_name/volume_name/path_name_`.
+
`_path_name_` ist optional und gibt den Pfad des Verzeichnisses, der Datei oder des Snapshots an.

* Wenn ein SnapMirror Ziel mithilfe der Dump-Engine auf Band gesichert wird, werden nur die Daten im Volume gesichert.
+
Wenn jedoch ein SnapMirror Ziel mit SMTape auf Band gesichert wird, werden auch die Metadaten gesichert. Die SnapMirror Beziehungen und die zugehörigen Metadaten werden nicht auf Band gesichert. Daher wird bei der Wiederherstellung nur die Daten auf diesem Volume wiederhergestellt, die zugehörigen SnapMirror Beziehungen werden jedoch nicht wiederhergestellt.
