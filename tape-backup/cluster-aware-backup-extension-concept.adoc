= Informationen zur Cluster Aware Backup Erweiterung für ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
CAB (Cluster Aware Backup) ist eine Erweiterung des NDMP v4-Protokolls. Diese Erweiterung ermöglicht dem NDMP-Server, eine Datenverbindung auf einem Knoten herzustellen, der ein Volume besitzt. Dies ermöglicht der Backup-Anwendung außerdem festzustellen, ob sich Volumes und Tape-Geräte auf demselben Knoten in einem Cluster befinden.

Damit der NDMP-Server den Knoten, dem ein Volume gehört, identifizieren und eine Datenverbindung zu diesem Knoten herstellen kann, muss die Backup-Anwendung die CAB-Erweiterung unterstützen. Die CAB-Erweiterung erfordert, dass die Backup-Anwendung den NDMP-Server vor dem Aufbau der Datenverbindung über das zu sichernde oder wiederherzustellende Volume informiert. Dadurch kann der NDMP-Server den Knoten, auf dem sich das Volume befindet, ermitteln und die Datenverbindung entsprechend herstellen.

Dank der von der Backup-Anwendung unterstützten CAB-Erweiterung stellt der NDMP-Server Affinitätsinformationen über Volumes und Tape-Geräte bereit. Mithilfe dieser Affinitätsinformationen kann die Backup-Anwendung ein lokales Backup anstelle eines Dreiwege-Backups durchführen, wenn sich ein Volume und ein Tape-Gerät auf demselben Node in einem Cluster befinden.
