= Informationen zum Blockierungsfaktor und ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Tape-Block umfasst 1.024 Byte Daten. Während eines Tape-Backups oder einer Wiederherstellung kann die Anzahl der Tape-Blöcke angegeben werden, die bei jedem lesen/schreiben übertragen werden. Diese Zahl wird als _Blocking Factor_ bezeichnet.

Sie können einen Blockierungsfaktor von 4 bis 256 verwenden. Wenn geplant ist, ein Backup auf einem anderen System als dem System, das das Backup erstellt hat, wiederherzustellen, muss das Wiederherstellungssystem den für das Backup verwendeten Blockierungsfaktor unterstützen. Wenn beispielsweise ein Blockierungsfaktor von 128 verwendet wird, muss das System, auf dem dieses Backup wiederhergestellt wird, einen Blockierungsfaktor von 128 unterstützen.

Bei einer NDMP-Sicherung bestimmt die MOVER_RECORD_SIZE den Blockierungsfaktor. ONTAP erlaubt einen Maximalwert von 256 KB für MOVER_RECORD_SIZE.
