= Informationen zu den Betriebsmodi von ONTAP NDMP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Tape-Backup- und Wiederherstellungsvorgänge entweder auf Knotenebene oder auf Ebene der Storage Virtual Machine (SVM) durchführen. Damit diese Vorgänge auf SVM-Ebene erfolgreich durchgeführt werden können, muss der NDMP-Service auf der SVM aktiviert sein.

Wenn Sie von Data ONTAP 8.2 auf Data ONTAP 8.3 aktualisieren, wird der in 8.2 verwendete NDMP-Betriebsmodus auch nach dem Upgrade von 8.2 auf 8.3 beibehalten.

Wenn ein neues Cluster mit Data ONTAP 8.2 oder höher installiert wird, befindet sich NDMP standardmäßig im SVM-scoped NDMP-Modus. Um Tape-Backup- und Wiederherstellungsvorgänge im node-scoped NDMP-Modus durchzuführen, muss der node-scoped NDMP-Modus explizit aktiviert werden.



[[learn-about-node-scoped-ontap-ndmp-mode]]
== Informationen zum knotenbezogenen ONTAP NDMP-Betriebsmodus

Im knotenbezogenen NDMP-Modus können Tape-Backup- und Wiederherstellungsvorgänge auf Knotenebene durchgeführt werden. Der in Data ONTAP 8.2 verwendete NDMP-Betriebsmodus bleibt auch nach dem Upgrade von 8.2 auf 8.3 erhalten.

Im knotenbezogenen NDMP-Modus können Tape-Backup- und Wiederherstellungsvorgänge auf einem Knoten durchgeführt werden, dem das Volume gehört. Für diese Vorgänge müssen NDMP-Steuerungsverbindungen auf einem LIF eingerichtet werden, das auf dem Knoten gehostet wird, dem das Volume oder die Tape-Geräte gehören.

[NOTE]
====
Dieser Modus ist veraltet und wird in einer zukünftigen Hauptversion entfernt.

====


[[learn-about-svm-scoped-ontap-ndmp-mode]]
== Informationen zum SVM-basierten ONTAP NDMP-Modus

Tape-Backup- und Wiederherstellungsvorgänge auf Ebene der Storage Virtual Machine (SVM) sind erfolgreich möglich, wenn der NDMP-Service auf der SVM aktiviert ist. Alle Volumes, die auf verschiedenen Knoten in der SVM eines Clusters gehostet werden, können gesichert und wiederhergestellt werden, sofern die Sicherungsanwendung die CAB-Erweiterung unterstützt.

Eine NDMP-Steuerungsverbindung kann auf verschiedenen LIF-Typen hergestellt werden. Im SVM-bezogenen NDMP-Modus gehören diese LIFs entweder zur Daten-SVM oder zur Admin-SVM. Die Verbindung kann nur auf einem LIF hergestellt werden, wenn der NDMP-Service auf der SVM aktiviert ist, der dieses LIF gehört.

Ein Daten-LIF gehört zum Daten-SVM und der Intercluster-LIF, der Node-Management-LIF und der Cluster-Management-LIF gehören zum Admin-SVM.

Im SVM-basierten NDMP-Modus hängt die Verfügbarkeit von Volumes und Tape-Geräten für Backup- und Restore-Vorgänge vom LIF-Typ ab, auf dem die NDMP-Steuerverbindung hergestellt wird, sowie vom Status der CAB-Erweiterung. Wenn die Backup-Anwendung die CAB-Erweiterung unterstützt und ein Volume und das Tape-Gerät dieselbe Affinität aufweisen, kann die Backup-Anwendung eine lokale Backup- oder Restore-Operation anstelle einer Drei-Wege-Backup- oder Restore-Operation ausführen.
