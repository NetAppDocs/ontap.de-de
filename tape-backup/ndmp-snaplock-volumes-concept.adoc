= Informationen zu NDMP mit ONTAP SnapLock Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Durch das Erstellen mehrerer Kopien regulierter Daten ergeben sich redundante Wiederherstellungsszenarien, und mithilfe von NDMP-Dump und -Restore ist es möglich, die Write Once, Read Many (WORM)-Eigenschaften der Quelldateien auf einem SnapLock Volume zu erhalten.

Die WORM-Attribute der Dateien auf einem SnapLock Volume bleiben beim Sichern, Wiederherstellen und Kopieren von Daten erhalten, jedoch werden WORM-Attribute nur bei der Wiederherstellung auf ein SnapLock Volume durchgesetzt. Wenn eine Sicherung von einem SnapLock Volume auf ein anderes Volume als ein SnapLock Volume wiederhergestellt wird, bleiben die WORM-Attribute zwar erhalten, werden jedoch ignoriert und von ONTAP nicht durchgesetzt.
