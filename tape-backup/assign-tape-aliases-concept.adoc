= Tape-Aliasing Übersicht
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Aliasvergabe vereinfacht die Geräteidentifizierung. Die Aliasvergabe verknüpft einen physischen Pfadnamen (PPN) oder eine Seriennummer (SN) eines Bandes oder Medienwechslers mit einem permanenten, aber veränderbaren Aliasnamen.

Die folgende Tabelle beschreibt, wie die Verwendung von Bandaliasen sicherstellt, dass ein Tape-Laufwerk (oder eine Tape-Bibliothek oder ein Medienwechsler) immer mit einem einzigen Aliasnamen verknüpft ist:

|===
| Szenario | Neuzuweisung des Alias 


 a| 
Wenn das System neu startet
 a| 
Das Tape-Laufwerk erhält automatisch seinen vorherigen Alias zurück.



 a| 
Wenn ein Tape-Gerät zu einem anderen Port wechselt
 a| 
Der Alias kann so angepasst werden, dass er auf die neue Adresse verweist.



 a| 
Wenn mehr als ein System ein bestimmtes Tape-Gerät verwendet
 a| 
Der Benutzer kann den Alias so einstellen, dass er für alle Systeme gleich ist.

|===
[NOTE]
====
Beim Upgrade von Data ONTAP 8.1.x auf Data ONTAP 8.2.x ändert die Tape-Alias-Funktion von Data ONTAP 8.2.x die vorhandenen Tape-Alias-Namen. In einem solchen Fall kann es erforderlich sein, die Tape-Alias-Namen in der Backup-Anwendung zu aktualisieren.

====
Durch die Vergabe von Bandaliasen entsteht eine Entsprechung zwischen den logischen Namen von Sicherungsgeräten (zum Beispiel st0 oder mc1) und einem Namen, der dauerhaft einem Port, einem Tape-Laufwerk oder einem Medium Changer zugewiesen ist.

[NOTE]
====
st0 und st00 sind unterschiedliche logische Namen.

====
[NOTE]
====
Logische Namen und Seriennummern werden nur für den Zugriff auf ein Gerät verwendet. Nach dem Zugriff gibt das Gerät alle Fehlermeldungen über den physischen Pfadnamen zurück.

====
Für die Aliasvergabe stehen zwei Arten von Namen zur Verfügung: physischer Pfadname und Seriennummer.



[[learn-about-physical-path-names]]
== Informationen zu physischen Wegnamen

Physische Pfadnamen (PPNs) sind die numerischen Adresssequenzen, die ONTAP Tape-Laufwerken und Tape-Bibliotheken basierend auf dem SCSI-2/3-Adapter oder Switch (genauer Standort) zuweist, mit dem sie mit dem Speichersystem verbunden sind. PPNs werden auch als elektrische Namen bezeichnet.

Die PPNs von direkt angeschlossenen Geräten verwenden folgendes Format:  `host_adapter`. `device_id_lun`

[NOTE]
====
Der LUN-Wert wird nur für Tape- und Medium-Changer-Geräte angezeigt, deren LUN-Werte nicht null sind; das heißt, wenn der LUN-Wert null ist, wird der  `lun` Teil des PPN nicht angezeigt.

====
Beispielsweise gibt PPN 8.6 an, dass die Host-Adapter-Nummer 8, die Geräte-ID 6 und die logische Einheitennummer (LUN) 0 ist.

SAS Tape-Geräte sind ebenfalls direkt angeschlossene Geräte. Beispielsweise gibt die PPN 5c.4 an, dass in einem Speichersystem der SAS HBA in Steckplatz 5 angeschlossen ist, SAS Tape an Port C des SAS HBA angeschlossen ist und die Geräte-ID 4 ist.

PPNs von Fibre Channel Switch-attached Devices verwenden das folgende Format: `switch:port_id`. `device_id_lun`

Beispielsweise zeigt der PPN MY_SWITCH:5.3L2, dass das an Port 5 eines Switches namens MY_SWITCH angeschlossene Tape-Laufwerk die Geräte-ID 3 und die LUN 2 hat.

Die LUN (logische Einheitennummer) wird vom Laufwerk bestimmt. Fibre Channel, SCSI Tape-Laufwerke und Tape-Bibliotheken sowie Festplatten verfügen über PPNs.

Die PPNs von Tape-Laufwerken und Tape-Bibliotheken ändern sich nicht, es sei denn, der Name des Switches ändert sich, das Tape-Laufwerk oder die Tape-Bibliothek wird verschoben oder das Tape-Laufwerk bzw. die Tape-Bibliothek wird neu konfiguriert. Die PPNs bleiben nach einem Neustart unverändert. Wenn zum Beispiel ein Tape-Laufwerk mit dem Namen MY_SWITCH:5.3L2 entfernt und ein neues Tape-Laufwerk mit derselben Geräte-ID und LUN an Port 5 des Switches MY_SWITCH angeschlossen wird, ist das neue Tape-Laufwerk über MY_SWITCH:5.3L2 erreichbar.



[[learn-about-serial-numbers]]
== Informationen zu Seriennummern

Eine Seriennummer (SN) ist eine eindeutige Kennung für ein Tape-Laufwerk oder einen Medienwechsler. ONTAP generiert Aliase basierend auf der SN anstelle der WWN.

Da die Seriennummer (SN) eine eindeutige Kennung für ein Tape-Laufwerk oder einen Medienwechsler ist, bleibt der Alias unabhängig von den verschiedenen Verbindungspfaden zum Tape-Laufwerk oder Medienwechsler gleich. Dies ermöglicht es Speichersystemen, dasselbe Tape-Laufwerk oder denselben Medienwechsler in einer Tape-Bibliothek-Konfiguration nachzuverfolgen.

Die SN eines Tape-Laufwerks oder eines Medium Changers ändert sich nicht, selbst wenn der Fibre Channel Switch, an den das Tape-Laufwerk oder der Medium Changer angeschlossen ist, umbenannt wird. In einer Tape-Bibliothek werden jedoch, wenn ein vorhandenes Tape-Laufwerk durch ein neues ersetzt wird, von ONTAP neue Aliase generiert, da sich die SN des Tape-Laufwerks ändert. Wenn ein vorhandenes Tape-Laufwerk in einen neuen Steckplatz in einer Tape-Bibliothek verschoben oder die LUN des Tape-Laufwerks neu zugeordnet wird, generiert ONTAP ebenfalls einen neuen Alias für dieses Tape-Laufwerk.

[NOTE]
====
Die Backup-Anwendungen sind mit den neu generierten Aliasen zu aktualisieren.

====
Die Seriennummer eines Tape-Geräts verwendet folgendes Format: `SN[xxxxxxxxxx]L[X]`

`x` ist ein alphanumerisches Zeichen und L``X`` ist die LUN des Tape-Geräts. Wenn die LUN 0 ist, wird der L``X``-Teil der Zeichenkette nicht angezeigt.

Jede Seriennummer besteht aus bis zu 32 Zeichen; das Format der Seriennummer ist nicht case-sensitiv.
