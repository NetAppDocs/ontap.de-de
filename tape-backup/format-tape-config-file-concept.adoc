= Format der ONTAP Konfigurationsdatei
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Format der Konfigurationsdatei für Tapes besteht aus Feldern wie Hersteller-ID, Produkt-ID und Details zu den Kompressionstypen für ein Tape-Laufwerk. Diese Datei enthält außerdem optionale Felder zum Aktivieren der Autoload-Funktion eines Tape-Laufwerks und zum Ändern der Befehls-Zeitüberschreitungswerte eines Tape-Laufwerks.

Die folgende Tabelle zeigt das Format der Tape-Konfigurationsdatei:

|===
| Artikel | Größe | Beschreibung 


 a| 
`vendor_id` (Zeichenkette)
 a| 
bis zu 8 Bytes
 a| 
Die vom  `SCSI Inquiry` Befehl gemeldete Vendor-ID.



 a| 
`product_id`(Zeichenkette)
 a| 
bis zu 16 Bytes
 a| 
Die Produkt-ID, wie sie vom  `SCSI Inquiry` Befehl gemeldet wird.



 a| 
`id_match_size`(Nummer)
 a| 
 a| 
Die Anzahl der Bytes der Produkt-ID, die für den Abgleich verwendet werden, um das zu identifizierende Tape-Laufwerk zu erkennen, beginnend mit dem ersten Zeichen der Produkt-ID in den Inquiry-Daten.



 a| 
`vendor_pretty` (Zeichenkette)
 a| 
bis zu 16 Bytes
 a| 
Wenn dieser Parameter vorhanden ist, wird er durch die vom Befehl angezeigte Zeichenkette  `storage tape show -device-names` angegeben; andernfalls wird INQ_VENDOR_ID angezeigt.



 a| 
`product_pretty`(Zeichenkette)
 a| 
bis zu 16 Bytes
 a| 
Wenn dieser Parameter vorhanden ist, wird er durch die vom Befehl angezeigte Zeichenkette  `storage tape show -device-names` angegeben; andernfalls wird INQ_PRODUCT_ID angezeigt.

|===
[NOTE]
====
Die  `vendor_pretty` und  `product_pretty` Felder sind optional, aber wenn eines dieser Felder einen Wert hat, muss auch das andere einen Wert haben.

====
Die folgende Tabelle erläutert die Beschreibung, den Dichtecode und den Komprimierungsalgorithmus für die verschiedenen Komprimierungstypen, wie  `l`,  `m`,  `h` und  `a`:

|===
| Artikel | Größe | Beschreibung 


 a| 
`{l | m | h | a}_description=(string)`
 a| 
bis zu 24 Bytes
 a| 
Die Zeichenkette, die für den nodeshell Befehl ausgegeben wird `sysconfig -t`, die die Eigenschaften der jeweiligen Dichteeinstellung beschreibt.



 a| 
`{l | m | h | a}_density=(hex codes)`
 a| 
 a| 
Der Dichtecode, der im SCSI-Modus-Seitenblockdeskriptor entsprechend dem gewünschten Dichtecode für l, m, h oder a festgelegt wird.



 a| 
`{l | m | h | a}_algorithm=(hex codes)`
 a| 
 a| 
Der Komprimierungsalgorithmus, der auf der SCSI Compression Mode Page entsprechend dem Dichtecode und der gewünschten Dichtecharakteristik eingestellt werden soll.

|===
Die folgende Tabelle beschreibt die optionalen Felder, die in der Tape-Konfigurationsdatei verfügbar sind:

|===
| Feld | Beschreibung 


 a| 
`autoload=(Boolean yes/no)`
 a| 
Dieses Feld ist auf  `yes` gesetzt, wenn das Tape-Laufwerk über eine automatische Ladefunktion verfügt; das heißt, nachdem die Bandkassette eingelegt wurde, ist das Tape-Laufwerk betriebsbereit, ohne dass ein  `SCSI load` (Start/Stop-Einheit) Befehl ausgeführt werden muss. Der Standardwert für dieses Feld ist  `no`.



 a| 
`cmd_timeout_0x`
 a| 
Individueller Zeitüberschreitungswert. Dieses Feld sollte nur verwendet werden, wenn ein anderer Zeitüberschreitungswert als der vom Tape-Laufwerk standardmäßig verwendete Wert angegeben werden soll. Die Beispieldatei listet die vom Tape-Laufwerk verwendeten Standardwerte für die SCSI-Befehlszeitüberschreitung auf. Der Zeitüberschreitungswert kann in Minuten (m), Sekunden (s) oder Millisekunden (ms) angegeben werden.

[NOTE]
====
Dieses Feld sollten Sie nicht ändern.

====
|===
Die Konfigurationsdatei für das Tape kann von der NetApp Support Site heruntergeladen und angezeigt werden.

.Beispiel eines Tape-Konfigurationsdateiformats
Das Format der Konfigurationsdatei für das HP LTO5 ULTRIUM Tape-Laufwerk ist wie folgt:

`vendor_id`="HP"

`product_id`="Ultrium 5-SCSI"

`id_match_size`=9

`vendor_pretty`="Hewlett-Packard"

`product_pretty`="LTO-5"

`l_description`="LTO-3(ro)/4 4/800GB"

`l_density`=0x00

`l_algorithm`=0x00

`m_description`="LTO-3(ro)/4 8/1600GB cmp"

`m_density`=0x00

`m_algorithm`=0x01

`h_description`="LTO-5 1600GB"

`h_density`=0x58

`h_algorithm`=0x00

`a_description`="LTO-5 3200GB cmp"

`a_density`=0x58

`a_algorithm`=0x01

`autoload`="yes"
