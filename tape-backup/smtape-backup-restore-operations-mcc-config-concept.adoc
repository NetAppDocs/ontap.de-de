= Wie sich SMTape-Sicherungs- und Wiederherstellungsvorgänge in ONTAP MetroCluster Konfigurationen auswirken
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie SMTape-Sicherungs- und Wiederherstellungsvorgänge in einer MetroCluster Konfiguration durchführen, muss nachvollzogen werden, wie sich ein Switchover- oder Switchback-Vorgang auf SMTape-Vorgänge auswirkt.



[[smtape-backup-or-restore-operation-followed-by-switchover]]
== SMTape Backup oder Wiederherstellungsvorgang, gefolgt von einer Umschaltung

Betrachten Sie zwei Cluster: Cluster 1 und Cluster 2. Während einer SMTape-Sicherung oder -Wiederherstellung auf Cluster 1, wenn ein Switchover von Cluster 1 zu Cluster 2 eingeleitet wird, tritt Folgendes ein:

* Ist der Wert der  `–override-vetoes` Option  `false`, wird der Umschaltvorgang abgebrochen und die Sicherungs- oder Wiederherstellungsoperation fortgesetzt.
* Ist der Wert der Option  `true`, wird der SMTape-Backup- oder Wiederherstellungsvorgang abgebrochen und der Umschaltvorgang fortgesetzt.




[[smtape-backup-or-restore-operation-followed-by-switchback]]
== SMTape Backup oder Wiederherstellungsvorgang gefolgt von Switchback

Es wird ein Switchover von Cluster 1 zu Cluster 2 durchgeführt und eine SMTape Backup- oder Restore-Operation auf Cluster 2 initiiert. Die SMTape-Operation sichert oder stellt ein Volume wieder her, das sich auf Cluster 2 befindet. Zu diesem Zeitpunkt, wenn ein Switchback von Cluster 2 zu Cluster 1 eingeleitet wird, tritt Folgendes ein:

* Wenn der Wert der  `–override-vetoes` Option  `false` ist, wird der Rückschaltvorgang abgebrochen und die Sicherungs- oder Wiederherstellungsoperation fortgesetzt.
* Ist der Wert der Option  `true`, wird der Sicherungs- oder Wiederherstellungsvorgang abgebrochen und der Rückschaltvorgang fortgesetzt.




[[smtape-backup-or-restore-operation-initiated-during-a-switchover-or-switchback]]
== SMTape-Backup- oder Wiederherstellungsvorgang während eines Switchover oder Switchback initiiert

Während eines Switchover-Prozesses von Cluster 1 zu Cluster 2 schlägt ein SMTape Backup- oder Wiederherstellungsvorgang auf Cluster 1 fehl, und der Switchover wird fortgesetzt.

Während eines Switchback-Prozesses von Cluster 2 zu Cluster 1 schlägt ein SMTape Backup- oder Wiederherstellungsvorgang, der von Cluster 2 aus initiiert wird, fehl und der Switchback wird fortgesetzt.
