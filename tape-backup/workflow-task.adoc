= ONTAP Tape-Backup und Wiederherstellungs-Workflow
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Tape-Backup- und Wiederherstellungsvorgänge sind mit einer NDMP-fähigen Sicherungsanwendung möglich.

.Über diese Aufgabe
Der Tape-Backup und Restore-Workflow bietet einen Überblick über die Aufgaben, die bei der Durchführung von Tape-Backup und Restore-Operationen anfallen. Detaillierte Informationen zur Durchführung eines Backup- und Restore-Vorgangs enthält die Dokumentation der Backup-Anwendung.

.Schritte
. Eine Tape-Bibliothek Konfiguration wird eingerichtet, indem eine NDMP-unterstützte Tape-Topologie ausgewählt wird.
. NDMP-Dienste auf dem Speichersystem aktivieren.
+
Sie können die NDMP-Dienste entweder auf Knotenebene oder auf Ebene der Storage Virtual Machine (SVM) aktivieren. Dies hängt vom NDMP-Modus ab, in dem die Tape-Backup- und Wiederherstellungsoperation durchgeführt wird.

. NDMP-Optionen dienen zur Verwaltung von NDMP auf dem Speichersystem.
+
Sie können NDMP-Optionen entweder auf Knotenebene oder auf SVM-Ebene verwenden. Dies hängt vom NDMP-Modus ab, in dem die Tape-Backup- und Wiederherstellungsoperation durchgeführt wird.

+
Sie können die NDMP-Optionen auf Knotenebene mit dem  `system services ndmp modify` Befehl und auf SVM-Ebene mit dem  `vserver services ndmp modify` Befehl ändern. Weitere Informationen zu  `system services ndmp modify` und  `vserver services ndmp modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=services+ndmp+modify["ONTAP-Befehlsreferenz"^].

. Eine Tape-Backup- oder Wiederherstellungsoperation kann mit einer NDMP-fähigen Sicherungsanwendung durchgeführt werden.
+
ONTAP unterstützt sowohl Dump als auch SMTape Engines für Tape-Backup und Wiederherstellung.

+
Weitere Informationen zur Verwendung der Backup-Anwendung (auch _Datenverwaltungsanwendungen_ oder _DMAs_ genannt) für Sicherungs- oder Wiederherstellungsvorgänge enthält die Dokumentation Ihrer Backup-Anwendung.
