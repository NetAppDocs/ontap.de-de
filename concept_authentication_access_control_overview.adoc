[role="lead"]
Die Authentifizierung und Zugriffskontrolle für ONTAP-Cluster und den Zugriff auf ONTAP-Webdienste kann verwaltet werden.

Mithilfe von System Manager oder der CLI lassen sich der Client- und Administratorzugriff auf den Cluster und den Speicher steuern und absichern.

Wenn der klassische System Manager verwendet wird (nur in ONTAP 9.7 und früher verfügbar), siehe  https://docs.netapp.com/us-en/ontap-system-manager-classic/index.html["System Manager Classic (ONTAP 9.0 bis 9.7)"^]



[[client-authentication-and-authorization]]
== Client-Authentifizierung und -Autorisierung

ONTAP authentifiziert einen Clientrechner und Benutzer, indem die Identitäten mit einer vertrauenswürdigen Quelle überprüft werden. ONTAP autorisiert einen Benutzer für den Zugriff auf eine Datei oder ein Verzeichnis, indem die Anmeldeinformationen des Benutzers mit den auf der Datei oder dem Verzeichnis konfigurierten Berechtigungen verglichen werden.



[[administrator-authentication-and-rbac]]
== Administratorauthentifizierung und RBAC

Administratoren verwenden lokale oder Remote-Anmeldekonten, um sich am Cluster und der Storage VM zu authentifizieren. Role-Based Access Control (RBAC) legt fest, auf welche Befehle ein Administrator Zugriff hat.
