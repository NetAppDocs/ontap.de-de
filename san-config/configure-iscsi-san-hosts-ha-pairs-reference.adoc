= iSCSI-Netzwerke mit ONTAP Systemen konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die iSCSI-Konfiguration sollte mit Hochverfügbarkeitspaaren (HA) eingerichtet werden, die entweder direkt an die iSCSI-SAN-Hosts angeschlossen sind oder über einen oder mehrere IP-Switches mit den Hosts verbunden werden.

link:../concepts/high-availability-pairs-concept.html["HA-Paare"]sind als Meldeknoten für die Active/Optimized- und die Active/Unoptimized-Pfade definiert, die von den Hosts für den Zugriff auf die LUNs verwendet werden. Mehrere Hosts mit unterschiedlichen Betriebssystemen wie Windows, Linux oder UNIX können gleichzeitig auf den Storage zugreifen. Für Hosts ist eine unterstützte Multipathing-Lösung erforderlich, die ALUA unterstützt und konfiguriert ist. Unterstützte Betriebssysteme und Multipathing-Lösungen können auf der link:https://mysupport.netapp.com/matrix["NetApp Interoperabilitätsmatrix-Tool"^] überprüft werden.

In einer Multi-Network-Konfiguration verbinden zwei oder mehr Switches die Hosts mit dem Speichersystem. Multi-Network-Konfigurationen werden empfohlen, da sie vollständig redundant sind. In einer Single-Network-Konfiguration verbindet ein Switch die Hosts mit dem Speichersystem. Single-Network-Konfigurationen sind nicht vollständig redundant.

[NOTE]
====
link:../system-admin/single-node-clusters.html["Single-Node Konfigurationen"] werden nicht empfohlen, da sie nicht die Redundanz bieten, die für Fehlertoleranz und unterbrechungsfreien Betrieb erforderlich ist.

[[multi-network-iscsi-configurations]]
== Multi-Network iSCSI Konfigurationen

In Multi-Netzwerk-HA-Paar-Konfigurationen verbinden zwei oder mehr Switches das HA-Paar mit einem oder mehreren Hosts. Da mehrere Switches vorhanden sind, ist diese Konfiguration vollständig redundant.

image:scrn-en-drw-iscsi-dual.png["Konfiguration eines HA-Paares mit mehreren Netzwerken"]



[[single-network-iscsi-configurations]]
== iSCSI-Konfigurationen für ein einzelnes Netzwerk

Bei HA-Paarkonfigurationen mit einem einzigen Netzwerk verbindet ein Switch das HA-Paar mit einem oder mehreren Hosts. Da nur ein einziger Switch verwendet wird, ist diese Konfiguration nicht vollständig redundant.

image:scrn-en-drw-iscsi-single.png["Single-network HA Pair Konfiguration"]



[[direct-attachment-iscsi-configuration]]
== Direct-Attachment iSCSI Konfiguration

In einer direkt angeschlossenen Konfiguration sind ein oder mehrere Hosts direkt mit den Controllern verbunden.
