= Empfohlene FC- und FCoE-Zoning-Konfigurationen für ONTAP Systeme
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Zoning-Konfiguration ist erforderlich, wenn auf Ihrem Host keine Multipathing-Lösung installiert ist, wenn vier oder mehr Hosts mit Ihrem SAN verbunden sind oder wenn Selective LUN Mapping auf den Knoten in Ihrem Cluster nicht implementiert ist.

In der empfohlenen FC- und FCoE Zoning-Konfiguration umfasst jede Zone einen Initiator-Port und einen oder mehrere Ziel-LIFs. Diese Konfiguration ermöglicht jedem Host-Initiator den Zugriff auf jeden Knoten, verhindert jedoch, dass Hosts, die auf denselben Knoten zugreifen, die Ports der jeweils anderen sehen.

Alle LIFs der Storage Virtual Machine (SVM) zur Zone mit dem Host-Initiator hinzufügen. So ist es möglich, Volumes oder LUNs zu verschieben, ohne die bestehenden Zonen zu bearbeiten oder neue Zonen zu erstellen.



[[dual-fabric-zoning-configurations]]
== Dual-fabric Zoning-Konfigurationen

Dual-fabric Zoning-Konfigurationen werden empfohlen, da sie Schutz vor Datenverlust durch den Ausfall einer einzelnen Komponente bieten. In einer Dual-fabric Konfiguration ist jeder Host-Initiator über unterschiedliche Switches mit jedem Knoten im Cluster verbunden. Fällt ein Switch aus, bleibt der Datenzugriff über den verbleibenden Switch erhalten. Auf dem Host ist Multipathing-Software erforderlich, um mehrere Pfade zu verwalten.

In der folgenden Abbildung verfügt der Host über zwei Initiatoren und verwendet Multipathing Software. Es gibt zwei Zonen. link:../san-admin/selective-lun-map-concept.html["Selektives LUN Mapping (SLM)"] ist so konfiguriert, dass alle Knoten als meldende Knoten gelten.

[NOTE]
====
Die in dieser Abbildung verwendete Namenskonvention ist lediglich eine Empfehlung für eine mögliche Namenskonvention, die für Ihre ONTAP Lösung gewählt werden kann.

====
* Zone 1: HBA 0, LIF_1, LIF_3, LIF_5 und LIF_7
* Zone 2: HBA 1, LIF_2, LIF_4, LIF_6 und LIF_8


Jeder Host-Initiator ist über einen separaten Switch angebunden. Zone 1 ist über Switch 1 zugänglich. Zone 2 ist über Switch 2 zugänglich.

Jeder Host kann auf eine LIF auf jedem Knoten zugreifen. Dadurch ist weiterhin der Zugriff des Hosts auf seine LUNs möglich, falls ein Knoten ausfällt. SVMs haben Zugriff auf alle iSCSI- und FC-LIFs auf jedem Knoten im Cluster, basierend auf Ihrer SLM-Meldeknoten-Konfiguration. SLM, Portsets oder FC-Switch-Zoning können verwendet werden, um die Anzahl der Pfade von einer SVM zum Host und von einer SVM zu einer LUN zu reduzieren.

Wenn die Konfiguration mehr Knoten umfasst, sind die LIFs für die zusätzlichen Knoten in diesen Zonen enthalten.

image:scm-en-drw-dual-fabric-zoning.png["Dual-Fabric-Zoning"]

[NOTE]
====
Das Host-Betriebssystem und die Multipathing-Software müssen die Anzahl der Pfade unterstützen, die für den Zugriff auf die LUNs auf den Knoten verwendet wird.

====


[[single-fabric-zoning]]
== Single-fabric-Zoning

In einer Single Fabric-Konfiguration wird jeder Host-Initiator über einen einzelnen Switch mit jedem Speicherknoten verbunden. Single Fabric Zoning-Konfigurationen werden nicht empfohlen, da sie keinen Schutz vor Datenverlust durch den Ausfall einer einzelnen Komponente bieten. Falls eine Single Fabric Zoning-Konfiguration gewählt wird, sollte jeder Host über zwei Initiatoren für Multipathing verfügen, um die Ausfallsicherheit der Lösung zu gewährleisten. Auf dem Host ist Multipathing-Software erforderlich, um mehrere Pfade zu verwalten.

Jeder Host-Initiator sollte mindestens eine LIF von jedem Knoten haben, auf den er zugreifen kann. Die Zonierung sollte mindestens einen Pfad vom Host-Initiator zum HA-Paar von Knoten im Cluster ermöglichen, um einen Pfad für die LUN-Konnektivität bereitzustellen. Das bedeutet, dass jeder Initiator auf dem Host in seiner Zoning-Konfiguration möglicherweise nur eine Ziel-LIF pro Knoten hat. Besteht die Anforderung für Multipathing zum selben Knoten oder zu mehreren Knoten im Cluster, hat dann jeder Knoten mehrere LIFs pro Knoten in seiner Zoning-Konfiguration. So kann der Host weiterhin auf seine LUNs zugreifen, falls ein Knoten ausfällt oder ein Volume mit der LUN auf einen anderen Knoten verschoben wird. Dazu müssen die Reporting-Knoten entsprechend eingestellt sein.

Bei Verwendung von Cisco FC- und FCoE-Switches darf eine einzelne Fabric-Zone nicht mehr als eine Ziel-LIF für denselben physischen Port enthalten. Befinden sich mehrere LIFs auf demselben Port in derselben Zone, können die LIF-Ports möglicherweise nach einem Verbindungsverlust nicht wiederhergestellt werden.

====


[[zoning-restrictions-for-cisco-fc-and-fcoe-switches]]
== Zonenbeschränkungen für Cisco FC und FCoE Switches

Bei der Verwendung von Cisco FC und FCoE Switches gelten bestimmte Einschränkungen für die Nutzung von physischen Ports und logischen Schnittstellen (LIFs) in Zonen.

.Physische Ports
* FC-NVMe und FC können denselben 32 Gb-physikalischen Port gemeinsam nutzen.
* FC-NVMe und FCoE können nicht denselben physischen Port verwenden.
* FC und FCoE können denselben physischen Port nutzen, aber ihre Protokoll-LIFs müssen in separaten Zonen sein.


.Logische Schnittstellen (LIFs)
* Eine Zone kann eine LIF von jedem Zielport im Cluster enthalten.
+
Die SLM-Konfiguration sollte so überprüft werden, dass die maximal zulässige Anzahl von Pfaden zum Host nicht überschritten wird.

* Jeder LIF an einem bestimmten Port muss sich in einer separaten Zone von anderen LIFs an diesem Port befinden
* LIFs auf verschiedenen physischen Ports können in derselben Zone sein.
