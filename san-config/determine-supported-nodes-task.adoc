= Maximal unterstützte Knoten und SAN-Hosts pro ONTAP Cluster bestimmen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Anzahl der unterstützten Knoten pro Cluster variiert je nach Ihrer ONTAP-Version, Ihren Controller-Modellen und dem Protokoll Ihrer Clusterknoten. Die maximale Anzahl von SAN-Hosts, die an einen Cluster angeschlossen werden können, variiert ebenfalls abhängig von Ihrer spezifischen Konfiguration.



[[determine-the-maximum-supported-nodes-per-cluster]]
== Maximal unterstützte Anzahl an Knoten pro Cluster bestimmen

Wenn ein Knoten im Cluster für FC, FC-NVMe, FCoE oder iSCSI konfiguriert ist, unterliegt der Cluster den SAN-Knotenlimits. Knotenlimits basierend auf den Controllern in Ihrem Cluster sind im _Hardware Universe_ aufgeführt.

.Schritte
. Gehe zu https://hwu.netapp.com["NetApp Hardware Universe"^].
. Oben links, neben *Home*, *Plattformen* auswählen und anschließend den Plattformtyp auswählen.
. Wählen Sie Ihre Version von ONTAP.
+
Es wird eine neue Spalte angezeigt, in der Sie Ihre Plattformen auswählen können.

. Die in Ihrer Lösung verwendeten Plattformen auswählen.
. Unter *Wählen Sie Ihre Spezifikationen* die Auswahl von *Alle auswählen* aufheben.
. *Maximale Anzahl Knoten pro Cluster (NAS/SAN)* auswählen.
. Auf *Ergebnisse anzeigen* klicken.


.Ergebnisse
Die maximale Anzahl an Knoten pro Cluster für Ihre ausgewählten Plattformen wird angezeigt.



[[determine-if-your-cluster-can-support-more-fc-hosts]]
== Es kann festgestellt werden, ob Ihr Cluster mehr FC Hosts unterstützen kann.

Bei FC- und FC-NVMe-Konfigurationen sollte die Anzahl der Initiator-Target-Nexus (ITNs) in Ihrem System verwendet werden, um zu bestimmen, ob weitere Hosts zu Ihrem Cluster hinzugefügt werden können.

Ein ITN repräsentiert einen Pfad vom Initiator des Hosts zum Ziel des Speichersystems. Die maximale Anzahl von ITNs pro Knoten in FC und FC-NVMe Konfigurationen beträgt 2.048. Wenn die maximale Anzahl von ITNs noch nicht erreicht ist, können dem Cluster weiterhin Hosts hinzugefügt werden.

Um die Anzahl der in Ihrem Cluster verwendeten ITNs zu ermitteln, sind für jeden Knoten im Cluster die folgenden Schritte auszuführen.

.Schritte
. Alle LIFs auf einem bestimmten Knoten identifizieren.
. Führen Sie für jede LIF auf dem Node den folgenden Befehl aus:
+
[source, cli]
----
fcp initiator show -fields wwpn, lif
----
+
Die Anzahl der am unteren Rand der Befehlsausgabe angezeigten Einträge entspricht Ihrer Anzahl an ITNs für diesen LIF.

. Die Anzahl der für jedes LIF angezeigten ITNs wird festgehalten.
. Die Anzahl der ITNs für jedes LIF auf jedem Knoten im Cluster addieren.
+
Diese Gesamtzahl stellt die Anzahl der ITNs in Ihrem Cluster dar.





[[determine-if-your-cluster-can-support-more-iscsi-hosts]]
== Es wird festgestellt, ob Ihr Cluster mehr iSCSI Hosts unterstützen kann.

Die Anzahl der Hosts, die direkt an einen Knoten oder über einen oder mehrere Switches angeschlossen werden können, hängt von der Anzahl der verfügbaren Ethernet-Ports ab. Die Anzahl der verfügbaren Ethernet-Ports wird durch das Modell des Controllers sowie die Anzahl und den Typ der im Controller installierten Adapter bestimmt. Die Anzahl der unterstützten Ethernet-Ports für Controller und Adapter ist im _Hardware Universe_ verfügbar.

Bei allen Clusterkonfigurationen mit mehreren Knoten muss die Anzahl der iSCSI-Sitzungen pro Knoten ermittelt werden, um festzustellen, ob weitere Hosts zu Ihrem Cluster hinzugefügt werden können. Solange Ihr Cluster unter der maximalen Anzahl an iSCSI-Sitzungen pro Knoten bleibt, können weiterhin Hosts zu Ihrem Cluster hinzugefügt werden. Die maximale Anzahl an iSCSI-Sitzungen pro Knoten variiert je nach den Controllertypen in Ihrem Cluster.

.Schritte
. Alle Zielportalgruppen auf dem Node identifizieren.
. Die Anzahl der iSCSI-Sitzungen für jede Zielportalgruppe auf dem Knoten prüfen:
+
[source, cli]
----
iscsi session show -tpgroup _tpgroup_
----
+
Die Anzahl der Einträge am Ende der Befehlsausgabe entspricht der Anzahl Ihrer iSCSI-Sitzungen für diese Zielportalgruppe.

. Die Anzahl der für jede Zielportalgruppe angezeigten iSCSI-Sitzungen wird erfasst.
. Die Anzahl der iSCSI-Sitzungen für jede Zielportalgruppe auf dem Knoten addieren.
+
Die Gesamtzahl stellt die Anzahl der iSCSI-Sitzungen auf Ihrem Knoten dar.
