= Anforderungen an SAN-Hosts, die mit ONTAP und Nicht-NetApp-Systemen verbunden sind
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Gemeinsam genutzte SAN-Konfigurationen sind Hosts, die sowohl an ONTAP Speichersysteme als auch an Speichersysteme anderer Hersteller angeschlossen sind. Der Zugriff auf ONTAP Speichersysteme und Speichersysteme anderer Hersteller von einem einzelnen Host aus wird unterstützt, sofern bestimmte Voraussetzungen erfüllt sind.

Für alle Host-Betriebssysteme ist es eine Best Practice, separate Adapter für die Verbindung zu den Speichersystemen der jeweiligen Hersteller zu verwenden. Die Verwendung separater Adapter verringert die Wahrscheinlichkeit von Treiber- und Einstellungskonflikten. Für Verbindungen zu einem ONTAP-Speichersystem müssen Adaptermodell, BIOS, Firmware und Treiber im NetApp Interoperability Matrix Tool als unterstützt aufgeführt sein.

Sie sollten die erforderlichen oder empfohlenen Zeitüberschreitungswerte und andere Speicherparameter für den Host festlegen. Die NetApp Software muss immer zuletzt installiert bzw. die NetApp Einstellungen zuletzt angewendet werden.

* Für AIX sind die Werte aus der AIX Host Utilities Version zu verwenden, die im Interoperability Matrix Tool für Ihre Konfiguration aufgeführt ist.
* Für ESX empfiehlt sich die Anwendung der Hosteinstellungen mithilfe der Virtual Storage Console für VMware vSphere.
* Für HP-UX sind die HP-UX Standardspeichereinstellungen zu verwenden.
* Für Linux sind die Werte aus der Linux Host Utilities Version zu verwenden, die im Interoperability Matrix Tool für Ihre Konfiguration aufgeführt ist.
* Für Solaris sind die Werte aus der Solaris Host Utilities Version zu verwenden, die im Interoperability Matrix Tool für Ihre Konfiguration aufgeführt ist.
* Für Windows sollte die Version von Windows Host Utilities installiert werden, die im Interoperability Matrix Tool für Ihre Konfiguration aufgeführt ist.
