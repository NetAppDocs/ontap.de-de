= FCoE-Fabrics mit ONTAP Systemen konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
FCoE kann mithilfe von FCoE Switches auf verschiedene Weise konfiguriert werden. Direkt angeschlossene Konfigurationen werden in FCoE nicht unterstützt.

Alle FCoE Konfigurationen sind dual-fabric, vollständig redundant und erfordern Multipathing-Software auf Host-Seite. In allen FCoE Konfigurationen können mehrere FCoE und FC Switches im Pfad zwischen Initiator und Ziel vorhanden sein, bis zur maximalen Hop-Anzahl. Um Switches miteinander zu verbinden, muss auf den Switches eine Firmware-Version ausgeführt werden, die Ethernet ISLs unterstützt. Jeder Host in einer FCoE Konfiguration kann mit einem anderen Betriebssystem konfiguriert werden.

FCoE-Konfigurationen erfordern Ethernet-Switches, die FCoE-Funktionen explizit unterstützen. FCoE-Konfigurationen werden durch dasselbe Interoperabilitäts- und Qualitätssicherungsverfahren wie FC-Switches validiert. Unterstützte Konfigurationen sind in der Interoperabilitätsmatrix aufgeführt. Zu den Parametern dieser unterstützten Konfigurationen gehören das Switch-Modell, die Anzahl der Switches, die in einem einzelnen Fabric eingesetzt werden können, und die unterstützte Switch-Firmware-Version.

Die in den Abbildungen angegebenen Portnummern der FC-Zielerweiterungsadapter sind Beispiele. Die tatsächlichen Portnummern können je nach den Erweiterungssteckplätzen, in denen die FCoE-Zielerweiterungsadapter installiert sind, variieren.



[[fcoe-initiator-to-fc-target]]
== FCoE-Initiator zu FC-Ziel

Mithilfe von FCoE-Initiatoren (CNAs) können Hosts über FCoE-Switches mit beiden Controllern in einem HA-Paar zu FC-Zielports verbunden werden. Der FCoE-Switch muss ebenfalls über FC-Ports verfügen. Der Host-FCoE-Initiator ist immer mit dem FCoE-Switch verbunden. Der FCoE-Switch kann entweder direkt mit dem FC-Ziel verbunden sein oder über FC-Switches mit dem FC-Ziel verbunden werden.

[[fcoe-mixed-with-ip-storage-protocols]]
== FCoE gemischt mit IP-Speicherprotokollen

Mithilfe von Host-FCoE-Initiatoren (CNAs) können Hosts mit beiden Controllern eines HA-Paares über FCoE-Switches mit FCoE-Zielports (auch UTAs oder UTA2s genannt) verbunden werden. FCoE-Ports können keine herkömmliche Link-Aggregation zu einem einzelnen Switch verwenden. Cisco Switches unterstützen eine spezielle Art der Link-Aggregation (Virtual Port Channel), die FCoE unterstützt. Ein Virtual Port Channel aggregiert einzelne Verbindungen zu zwei Switches. Virtual Port Channels können auch für anderen Ethernet-Datenverkehr verwendet werden. Ports, die für anderen Datenverkehr als FCoE verwendet werden, einschließlich NFS, SMB, iSCSI und anderem Ethernet-Datenverkehr, können reguläre Ethernet-Ports an den FCoE-Switches verwenden.

image:scrn_en_drw_ip_storage_protocol.png["FCoE gemischt mit IP-Speicherprotokollen"]
