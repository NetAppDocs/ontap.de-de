= Flash Pool ONTAP lokale Zwischenspeicherungsrichtlinien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Caching-Richtlinien für die Volumes in einer Flash Pool lokalen Ebene ermöglichen den Einsatz von Flash als Hochleistungscache für Ihren Arbeitsdatensatz, während kostengünstigere HDDs für weniger häufig genutzte Daten verwendet werden. Wenn Cache für zwei oder mehr Flash Pool lokale Ebenen bereitgestellt wird, empfiehlt sich die Nutzung der Flash Pool SSD-Partitionierung, um SSDs über die lokalen Ebenen im Flash Pool hinweg gemeinsam zu nutzen.

Caching-Richtlinien werden auf Volumes angewendet, die sich in Flash Pool lokalen Tiers befinden. Es ist wichtig, die Funktionsweise von Caching-Richtlinien zu verstehen, bevor Änderungen vorgenommen werden.

In den meisten Fällen ist die Standard-Caching-Richtlinie von `auto` die beste Caching-Richtlinie. Die Caching-Richtlinie sollte nur geändert werden, wenn eine andere Richtlinie eine bessere Performance für Ihre Arbeitslast bietet. Die Konfiguration einer falschen Caching-Richtlinie kann die Volume-Performance erheblich beeinträchtigen, wobei sich die Leistungseinbußen im Laufe der Zeit allmählich verstärken können.

Caching-Richtlinien kombinieren eine Lese-Caching-Richtlinie und eine Schreib-Caching-Richtlinie. Der Richtlinienname setzt sich aus den Namen der Lese-Caching-Richtlinie und der Schreib-Caching-Richtlinie zusammen, getrennt durch einen Bindestrich. Fehlt der Bindestrich im Richtliniennamen, ist die Schreib-Caching-Richtlinie  `none`, außer bei der  `auto` Richtlinie.

Lesecache-Richtlinien optimieren die zukünftige Leseleistung, indem sie zusätzlich zu den auf HDDs gespeicherten Daten eine Kopie der Daten im Cache ablegen. Bei Lesecache-Richtlinien, die bei Schreibvorgängen Daten in den Cache einfügen, fungiert der Cache als _Write-Through_-Cache.

Daten, die mithilfe der Schreibcaching-Richtlinie in den Cache eingefügt werden, befinden sich ausschließlich im Cache; es gibt keine Kopie auf den Festplatten. Flash Pool Cache ist RAID-geschützt. Das Aktivieren des Schreibcachings bewirkt, dass Daten aus Schreibvorgängen sofort für Lesezugriffe aus dem Cache zur Verfügung stehen, während das Schreiben der Daten auf die Festplatten aufgeschoben wird, bis sie aus dem Cache entfernt werden.

Wenn ein Volume von einer Flash Pool lokalen Ebene auf eine Einzellaufwerk lokale Ebene verschoben wird, geht seine Caching-Richtlinie verloren; wird es später wieder auf eine Flash Pool lokale Ebene verschoben, erhält es die Standard-Caching-Richtlinie von `auto`. Wird ein Volume zwischen zwei Flash Pool lokalen Ebenen verschoben, bleibt die Caching-Richtlinie erhalten.



[[change-a-caching-policy]]
== Eine Caching-Richtlinie ändern

Die CLI kann verwendet werden, um die Caching-Richtlinie für ein Volume zu ändern, das sich auf einer Flash Pool lokalen Ebene befindet, indem der  `-caching-policy` Parameter mit dem  `volume create` Befehl genutzt wird.

Wenn Sie ein Volume auf einer Flash Pool lokalen Ebene erstellen, wird dem Volume standardmäßig die  `auto` Caching-Richtlinie zugewiesen.
