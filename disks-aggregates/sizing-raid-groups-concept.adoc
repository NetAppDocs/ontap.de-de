= Überlegungen zur Dimensionierung von ONTAP RAID-Gruppen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Konfiguration einer optimalen RAID-Gruppengröße erfordert einen Kompromiss zwischen verschiedenen Faktoren. Es ist zu entscheiden, welche Faktoren – Geschwindigkeit des RAID-Wiederaufbaus, Schutz vor Datenverlust durch Festplattenausfall, Optimierung der I/O-Leistung und Maximierung des Datenspeicherplatzes – für das (lokale Tier) Datenaggregat, das konfiguriert wird, am wichtigsten sind.

Durch die Erstellung größerer RAID-Gruppen wird der für die Datenspeicherung verfügbare Speicherplatz bei gleicher Menge an Speicherplatz für Parität (auch als „Paritätsteuer“ bekannt) maximiert. Fällt jedoch eine Festplatte in einer größeren RAID-Gruppe aus, verlängert sich die Rekonstruktionszeit, was die Leistung über einen längeren Zeitraum beeinträchtigt. Zudem erhöht eine größere Anzahl von Festplatten in einer RAID-Gruppe die Wahrscheinlichkeit eines mehrfachen Festplattenausfalls innerhalb derselben RAID-Gruppe.



[[hdd-or-array-lun-raid-groups]]
== HDD- oder Array-LUN RAID-Gruppen

Bei der Dimensionierung von RAID-Gruppen, die aus HDDs oder Array-LUNs bestehen, sind die folgenden Richtlinien zu berücksichtigen:

* Alle RAID-Gruppen in einer lokalen Ebene (Aggregat) sollten die gleiche Anzahl an Festplatten haben.
+
Zwar können Sie auf einer lokalen Ebene bis zu 50 % weniger oder mehr Festplatten in verschiedenen RAID-Gruppen haben, dies kann jedoch in einigen Fällen zu Leistungsengpässen führen, weshalb es möglichst vermieden werden sollte.

* Die empfohlene Anzahl an Festplatten in einer RAID-Gruppe liegt zwischen 12 und 20.
+
Die Zuverlässigkeit der Performance-Festplatten kann bei Bedarf eine RAID-Gruppengröße von bis zu 28 unterstützen.

* Wenn Sie die ersten beiden Richtlinien mit mehreren RAID-Gruppen-Festplattenanzahlen erfüllen können, sollte die größere Anzahl an Festplatten gewählt werden.




[[ssd-raid-groups-in-flash-pool-local-tiers-aggregates]]
== SSD RAID-Gruppen in Flash Pool lokalen Tiers (Aggregaten)

Die Größe der SSD RAID-Gruppe kann von der Größe der HDD RAID-Gruppen in einem Flash Pool lokalen Tier (Aggregat) abweichen. In der Regel ist darauf zu achten, dass für ein Flash Pool lokales Tier nur eine SSD RAID-Gruppe vorhanden ist, um die Anzahl der für die Parität benötigten SSDs zu minimieren.



[[ssd-raid-groups-in-ssd-local-tiers-aggregates]]
== SSD RAID-Gruppen in SSD Local Tiers (Aggregaten)

Für die Dimensionierung von RAID-Gruppen, die aus SSDs bestehen, gelten folgende Richtlinien:

* Alle RAID-Gruppen in einer lokalen Ebene (Aggregat) sollten eine ähnliche Anzahl von Laufwerken aufweisen.
+
Die RAID-Gruppen müssen nicht exakt die gleiche Größe haben, jedoch sollte nach Möglichkeit vermieden werden, dass eine RAID-Gruppe weniger als die Hälfte der Größe anderer RAID-Gruppen in derselben lokalen Ebene aufweist.

* Für RAID-DP liegt der empfohlene Bereich der RAID-Gruppengröße zwischen 20 und 28.
