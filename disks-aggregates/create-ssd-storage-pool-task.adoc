= Cache zu einer lokalen ONTAP-Tier hinzufügen, indem ein SSD-Speicherpool erstellt wird.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Cache bereitstellen, indem Sie eine bestehende lokale Ebene durch Hinzufügen von Solid State Drives (SSDs) in eine lokale Flash Pool Ebene umwandeln.

Sie können SSD-Speicherpools erstellen, um SSD-Cache für zwei bis vier Flash Pool lokale Ebenen bereitzustellen. Flash Pool lokale Ebenen ermöglichen den Einsatz von Flash als Hochleistungscache für den aktiven Datensatz, während kostengünstigere HDDs für weniger häufig genutzte Daten verwendet werden.

.Über diese Aufgabe
* Beim Erstellen oder Hinzufügen von Datenträgern zu einem Speicherpool muss eine Datenträgerliste angegeben werden.
+
Speicherpools unterstützen keinen  `diskcount`Parameter.

* Die im Speicherpool verwendeten SSDs sollten die gleiche Größe haben.


[role="tabbed-block"]
====
.System Manager
--
.Mit System Manager einen SSD-Cache hinzufügen (ONTAP 9.12.1 und höher)
[%collapsible%open]
=====
Ab ONTAP 9.12.1 kann mit System Manager ein SSD-Cache hinzugefügt werden.


NOTE: Speicherpooloptionen sind auf AFF Systemen nicht verfügbar.

.Schritte
. Klicken Sie auf *Cluster > Datenträger* und dann auf *Anzeigen/Ausblenden*.
. *Typ* auswählen und überprüfen, ob im Cluster Ersatz-SSDs vorhanden sind.
. Klicken Sie auf *Speicher > Ebenen* und klicken Sie auf *Speicherpool hinzufügen*.
. Wählen Sie den Datenträgertyp aus.
. Geben Sie eine Festplattengröße ein.
. Wählen Sie die Anzahl der Festplatten aus, die dem Speicherpool hinzugefügt werden sollen.
. Die geschätzte Cache-Größe wird angezeigt.


=====
.Mit System Manager einen SSD-Cache hinzufügen (nur ONTAP 9.7)
[%collapsible%open]
=====

NOTE: Die CLI-Prozedur ist zu verwenden, wenn eine ONTAP Version genutzt wird, die neuer als ONTAP 9.7 oder älter als ONTAP 9.12.1 ist.

.Schritte
. *(Zur klassischen Version zurückkehren)* auswählen.
. Klicken Sie auf *Speicher > Aggregate & Festplatten > Aggregate*.
. Wählen Sie die lokale Ebene aus und klicken Sie dann auf *Aktionen > Cache hinzufügen*.
. Als Cache-Quelle werden „Speicherpools“ oder „dedizierte SSDs“ ausgewählt.
. *(Zur neuen Benutzeroberfläche wechseln)* auswählen.
. Klicken Sie auf *Speicher > Ebenen*, um die Größe der neuen lokalen Ebene zu überprüfen.


=====
--
.CLI
--
*Erstellung eines SSD-Speicherpools über die Befehlszeilenschnittstelle*

.Schritte
. Die Namen der verfügbaren Ersatz-SSDs bestimmen:
+
`storage aggregate show-spare-disks -disk-type SSD`

+
Die in einem Speicherpool verwendeten SSDs können einem der beiden Knoten eines HA-Paars gehören.

. Den Storage-Pool erstellen:
+
`storage pool create -storage-pool _sp_name_ -disk-list _disk1_,_disk2_,...`

. *Optional:* Der neu erstellte Speicherpool kann überprüft werden:
+
`storage pool show -storage-pool _sp_name_`



--
====
.Ergebnisse
Nachdem die SSDs dem Speicherpool hinzugefügt wurden, werden sie im Cluster nicht mehr als Ersatzfestplatten angezeigt, obwohl der vom Speicherpool bereitgestellte Speicher noch keinem Flash Pool Cache zugewiesen wurde. SSDs können einer RAID-Gruppe nicht als einzelne Laufwerke hinzugefügt werden; ihr Speicher kann nur über die Zuordnungseinheiten des Speicherpools, zu dem sie gehören, bereitgestellt werden.
