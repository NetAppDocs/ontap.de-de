= Informationen zum lokalen Tier-Management von ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den System Manager oder die ONTAP CLI verwenden, um lokale Ebenen hinzuzufügen, deren Nutzung zu verwalten und ihnen Kapazität (Festplatten) hinzuzufügen.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

Folgende Aufgaben sind möglich:

* link:add-local-tier-overview-task.html["Eine lokale Ebene hinzufügen (erstellen)"]
+
Um eine lokale Tier hinzuzufügen, wird einem bestimmten Workflow gefolgt. Die Anzahl der benötigten Festplatten oder Festplattenpartitionen für die lokale Tier wird ermittelt und es wird entschieden, welche Methode zur Erstellung der lokalen Tier verwendet wird. Lokale Tiers können automatisch hinzugefügt werden, indem ONTAP die Konfiguration zuweist, oder die Konfiguration kann manuell festgelegt werden.

* link:manage-use-local-tiers-overview-task.html["Verwaltung der Nutzung lokaler Tiers"]
+
Für bestehende lokale Speicherebenen können Sie diese umbenennen, ihre Medienkosten festlegen oder ihre Laufwerks- und RAID-Gruppeninformationen bestimmen. Die RAID-Konfiguration einer lokalen Speicherebene kann geändert und lokalen Speicherebenen können Speicher-VMs (SVMs) zugewiesen werden. Die RAID-Konfiguration einer lokalen Speicherebene kann geändert und lokalen Speicherebenen können Speicher-VMs (SVMs) zugewiesen werden. Es kann festgestellt werden, welche Volumes sich auf einer lokalen Speicherebene befinden und wie viel Speicherplatz sie auf einer lokalen Speicherebene belegen. Es kann gesteuert werden, wie viel Speicherplatz Volumes belegen dürfen. Die Zuständigkeit für eine lokale Speicherebene kann mit einem HA-Paar verschoben werden. Eine lokale Speicherebene kann auch gelöscht werden.

* link:add-capacity-local-tier-overview-task.html["Kapazität (Festplatten) zu einer lokalen Ebene hinzufügen"]
+
Mithilfe verschiedener Methoden wird ein bestimmter Workflow befolgt, um die Kapazität zu erweitern. Es können Festplatten zu einer lokalen Ebene und Laufwerke zu einem Knoten oder Shelf hinzugefügt werden. Falls erforderlich, lassen sich falsch ausgerichtete Ersatzpartitionen korrigieren.
