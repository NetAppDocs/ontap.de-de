= Umbenennen einer lokalen ONTAP-Ebene
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine lokale Ebene umbenennen. Die Vorgehensweise hängt von der verwendeten Schnittstelle ab, System Manager oder CLI.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

[role="tabbed-block"]
====
.System Manager
--
*System Manager zum Umbenennen einer lokalen Ebene verwenden*

Ab ONTAP 9.10.1 kann der Name einer lokalen Ebene geändert werden.

.Schritte
. Im System Manager auf *Speicher > Ebenen* klicken.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen der lokalen Ebene.
. *Umbenennen* auswählen.
. Geben Sie einen neuen Namen für die lokale Tier an.


--
.CLI
--
*Die CLI kann verwendet werden, um eine lokale Ebene umzubenennen*

.Schritt
. Die lokale Ebene kann mithilfe der Befehlszeile umbenannt werden:
+
`storage aggregate rename -aggregate _aggr-name_ -newname _aggr-new-name_`

+
Im folgenden Beispiel wird ein Aggregat mit dem Namen "`aggr5`" in "`sales-aggr`" umbenannt:

+
....
 > storage aggregate rename -aggregate aggr5 -newname sales-aggr
....
