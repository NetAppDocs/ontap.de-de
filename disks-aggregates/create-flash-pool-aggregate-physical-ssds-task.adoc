= Eine ONTAP Flash Pool lokale Ebene mit physischen SSDs erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine lokale Flash Pool-Ebene wird erstellt, indem die Funktion auf einer bestehenden lokalen Ebene aktiviert wird, die aus HDD RAID-Gruppen besteht, und anschließend eine oder mehrere SSD RAID-Gruppen zu dieser lokalen Ebene hinzugefügt werden. Dadurch entstehen zwei Sätze von RAID-Gruppen für diese lokale Ebene: SSD RAID-Gruppen (der SSD-Cache) und HDD RAID-Gruppen.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Über diese Aufgabe
Nachdem Sie einem lokalen Tier einen SSD-Cache hinzugefügt haben, um ein Flash Pool lokales Tier zu erstellen, kann der SSD-Cache nicht mehr entfernt werden, um das lokale Tier wieder in seine ursprüngliche Konfiguration zurückzuversetzen.

Standardmäßig entspricht der RAID-Level des SSD-Cache dem RAID-Level der HDD-RAID-Gruppen. Diese Standardeinstellung kann durch Angabe der  `raidtype` Option beim Hinzufügen der ersten SSD-RAID-Gruppen überschrieben werden.

.Bevor Sie beginnen
* Es muss eine gültige lokale Ebene identifiziert sein, die aus HDDs besteht, um sie in eine Flash Pool lokale Ebene umzuwandeln.
* Die Schreibcache-Berechtigung der mit der lokalen Ebene verbundenen Volumes muss ermittelt und alle erforderlichen Schritte zur Behebung von Berechtigungsproblemen abgeschlossen sein.
* Sie müssen die SSDs, die Sie hinzufügen werden, bestimmt haben, und diese SSDs müssen dem Knoten gehören, auf dem Sie die Flash Pool lokale Ebene erstellen.
* Sie müssen die Prüfsummentypen sowohl der hinzuzufügenden SSDs als auch der bereits im lokalen Tier befindlichen HDDs ermittelt haben.
* Sie müssen die Anzahl der hinzuzufügenden SSDs und die optimale RAID-Gruppengröße für die SSD RAID-Gruppen ermittelt haben.
+
Die Verwendung von weniger RAID-Gruppen im SSD-Cache verringert die Anzahl der benötigten Paritätsfestplatten, aber größere RAID-Gruppen erfordern RAID-DP.

* Der gewünschte RAID-Level für den SSD-Cache muss festgelegt sein.
* Sie müssen die maximale Cache-Größe für Ihr System ermittelt und festgestellt haben, dass das Hinzufügen eines SSD-Cache zu Ihrer lokalen Speicherebene nicht dazu führt, dass diese überschritten wird.
* Sie müssen sich mit den Konfigurationsanforderungen für lokale Flash Pool Ebenen vertraut gemacht haben.


.Schritte
Sie können eine Flash Pool lokale Ebene mithilfe von System Manager oder der ONTAP CLI erstellen.

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.1 kann mit System Manager eine Flash Pool lokale Tier mit physischen SSDs erstellt werden.

.Schritte
. *Speicher > Ebenen* auswählen und dann eine vorhandene lokale HDD-Speicherebene auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] dann *Add Flash Pool Cache*.
. **Dedizierte SSDs als Cache verwenden** auswählen.
. Wählen Sie einen Datenträgertyp und die Anzahl der Datenträger.
. Einen RAID-Typ auswählen.
. *Speichern* auswählen.
. Suchen Sie die Speicherebene und wählen Sie dann image:icon_kabob.gif["Symbol für Menüoptionen"] aus
. *Weitere Details* auswählen. Es wird überprüft, ob Flash Pool als *Aktiviert* angezeigt wird.


--
.CLI
--
.Schritte
. Die lokale Tier wird als berechtigt markiert, eine Flash Pool lokale Tier zu werden:
+
`storage aggregate modify -aggregate _aggr_name_ -hybrid-enabled true`

+
Falls dieser Schritt nicht erfolgreich ist, die Schreibcache-Berechtigung für die Zielebene bestimmen.

. Die SSDs werden mithilfe des  `storage aggregate add` Befehls zur lokalen Ebene hinzugefügt.
+
** Sie können die SSDs anhand ihrer ID oder mithilfe der  `diskcount` und  `disktype` Parameter angeben.
** Wenn die HDDs und die SSDs nicht den gleichen Prüfsummentyp haben oder wenn die lokale Ebene eine lokale Ebene mit gemischtem Prüfsummentyp ist, muss der  `checksumstyle` Parameter verwendet werden, um den Prüfsummentyp der Datenträger anzugeben, die der lokalen Ebene hinzugefügt werden.
** Sie können einen anderen RAID-Typ für den SSD-Cache mithilfe des  `raidtype` Parameters angeben.
** Wenn die Größe der Cache-RAID-Gruppe von der Standardgröße für den verwendeten RAID-Typ abweichen soll, sollte sie jetzt mit dem  `-cache-raid-group-size` Parameter geändert werden.
