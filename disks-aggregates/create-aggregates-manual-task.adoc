= ONTAP lokale Tiers manuell hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie keine lokale Ebene mithilfe der Best-Practice-Empfehlungen von ONTAP hinzufügen möchten, kann der Vorgang manuell durchgeführt werden.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Bevor Sie beginnen
Festplatten müssen einem Node zugeordnet sein, bevor sie in einer lokalen Ebene verwendet werden können. Wenn Ihr Cluster nicht für die automatische Zuweisung des Festplattenbesitzes konfiguriert ist, muss link:manual-assign-disks-ownership-prep-task.html["Besitzverhältnisse manuell zuweisen"] erfolgen.

Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.11.1 kann, falls nicht die vom System Manager empfohlene Konfiguration zum Erstellen einer lokalen Ebene verwendet werden soll, eine gewünschte Konfiguration angegeben werden.

.Schritte
. Im System Manager auf *Speicher > Ebenen* klicken.
. Auf der Seite *Tiers* image:icon-add-local-tier.png["Lokales Tier-Symbol hinzufügen"] einen neuen lokalen Tier erstellen:
+
Auf der Seite *Lokale Ebene hinzufügen* wird die empfohlene Anzahl lokaler Ebenen angezeigt, die auf den Knoten erstellt werden können, sowie der verfügbare nutzbare Speicherplatz.

. Wenn der System Manager die Speicherempfehlung für die lokale Ebene anzeigt, im Abschnitt *Ersatzfestplatten* auf *Zur manuellen Erstellung der lokalen Ebene wechseln* klicken.
+
Auf der Seite *Lokale Ebene hinzufügen* werden Felder angezeigt, die zur Konfiguration der lokalen Ebene verwendet werden.

. Im ersten Abschnitt der Seite *Lokale Ebene hinzufügen* sind folgende Angaben zu machen:
+
.. Geben Sie den Namen der lokalen Tier ein.
.. (Optional): Aktivieren Sie das Kästchen *Mirror this local tier*, wenn die lokale Ebene gespiegelt werden soll.
.. Wählen Sie einen Datenträgertyp aus.
.. Wählen Sie die Anzahl der Festplatten.


. Im Abschnitt *RAID Konfiguration* sind folgende Schritte auszuführen:
+
.. Den RAID-Typ auswählen.
.. Die Größe der RAID-Gruppe auswählen.
.. Klicken Sie auf RAID-Zuordnung, um anzuzeigen, wie die Festplatten in der Gruppe zugewiesen sind.


. (Optional): Falls der Onboard Key Manager installiert ist, kann er im Abschnitt *Verschlüsselung* der Seite für die Verschlüsselung konfiguriert werden. Das Kästchen *Onboard Key Manager für Verschlüsselung konfigurieren* aktivieren.
+
.. Geben Sie eine Passphrase ein.
.. Geben Sie die Passphrase zur Bestätigung erneut ein.
.. Bewahren Sie die Passphrase für die zukünftige Verwendung auf, falls das System wiederhergestellt werden muss.
.. Die Schlüssel-Datenbank für die zukünftige Verwendung sichern.


. *Speichern* erstellt die lokale Speicherebene und fügt sie der Speicherlösung hinzu.

.CLI
--
Bevor Sie lokale Ebenen manuell erstellen, sollten Sie die Festplattenkonfigurationsoptionen überprüfen und die Erstellung simulieren.

Anschließend kann der  `storage aggregate create` Befehl ausgeführt und die Ergebnisse überprüft werden.

.Bevor Sie beginnen
Sie müssen die Anzahl der Festplatten und die Anzahl der benötigten Ersatzfestplatten in der lokalen Ebene ermittelt haben.

.Über diese Aufgabe
Wenn die Root-Daten-Daten-Partitionierung aktiviert ist und Sie 24 oder weniger Solid-State-Drives (SSDs) in Ihrer Konfiguration haben, wird empfohlen, die Datenpartitionen verschiedenen Knoten zuzuweisen.

Die Vorgehensweise zum Erstellen lokaler Speicherebenen auf Systemen mit aktivierter Root-Daten-Partitionierung und Root-Daten-Daten-Partitionierung ist identisch mit der Vorgehensweise zum Erstellen lokaler Speicherebenen auf Systemen mit unpartitionierten Festplatten. Wenn die Root-Daten-Partitionierung auf Ihrem System aktiviert ist, sollte die Anzahl der Festplattenpartitionen für die  `-diskcount` Option verwendet werden. Bei der Root-Daten-Daten-Partitionierung gibt die  `-diskcount` Option die Anzahl der zu verwendenden Festplatten an.


NOTE: Beim Erstellen mehrerer lokaler Ebenen zur Verwendung mit FlexGroup Volumes sollten die lokalen Ebenen möglichst einander in ihrer Größe angenähert sein.

Weitere Informationen zu  `storage aggregate create` und zu den Optionen und Anforderungen für die Erstellung lokaler Tiers finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/storage-aggregate-create.html["ONTAP-Befehlsreferenz"^].

.Schritte
. Die Liste der Ersatzfestplattenpartitionen zeigt, ob genügend Partitionen für die Erstellung Ihrer lokalen Ebene vorhanden sind:
+
`storage aggregate show-spare-disks -original-owner _node_name_`

+
Datenpartitionen werden unter  `Local Data Usable` angezeigt. Eine Root-Partition kann nicht als Ersatzpartition verwendet werden.

. Die Erstellung der lokalen Ebene simulieren:
+
`storage aggregate create -aggregate _aggregate_name_ -node _node_name_ -raidtype raid_dp -diskcount _number_of_disks_or_partitions_ -simulate true`

. Wenn bei der simulierten Befehlsausführung Warnungen angezeigt werden, sollte der Befehl angepasst und die Simulation wiederholt werden.
. Lokale Ebene erstellen:
+
`storage aggregate create -aggregate _aggr_name_ -node _node_name_ -raidtype raid_dp -diskcount _number_of_disks_or_partitions_`

. Die lokale Ebene anzeigen, um zu überprüfen, ob sie erstellt wurde:
+
`storage aggregate show-status _aggregate_name_`
