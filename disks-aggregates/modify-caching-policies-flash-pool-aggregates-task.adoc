= Ändern der Caching-Richtlinien der ONTAP Flash Pool lokalen Ebenen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Caching-Richtlinie eines Volumes sollte nur dann geändert werden, wenn eine andere Caching-Richtlinie voraussichtlich eine bessere Leistung bietet. Die Caching-Richtlinie eines Volumes kann auf einer Flash Pool lokalen Ebene geändert werden.

.Bevor Sie beginnen
Es ist zu bestimmen, ob die Caching-Richtlinie geändert werden soll.

.Über diese Aufgabe
In den meisten Fällen ist die Standard-Caching-Richtlinie von `auto` die beste Caching-Richtlinie, die verwendet werden kann. Die Caching-Richtlinie sollte nur geändert werden, wenn eine andere Richtlinie eine bessere Performance für Ihre Arbeitslast bietet. Die Konfiguration einer falschen Caching-Richtlinie kann die Volume-Performance erheblich beeinträchtigen, wobei sich die Leistungseinbußen im Laufe der Zeit allmählich verstärken können. Beim Ändern von Caching-Richtlinien ist Vorsicht geboten. Treten Leistungsprobleme bei einem Volume auf, für das die Caching-Richtlinie geändert wurde, sollte die Caching-Richtlinie auf `auto` zurückgesetzt werden.

.Schritt
. Die CLI kann verwendet werden, um die Caching-Richtlinie des Volumes zu ändern:
+
`volume modify -volume _volume_name_ -caching-policy _policy_name_`



.Beispiel
Das folgende Beispiel ändert die Caching-Richtlinie eines Volumes mit dem Namen  `vol2` auf die Richtlinie  `none`:

`volume modify -volume vol2 -caching-policy none`
