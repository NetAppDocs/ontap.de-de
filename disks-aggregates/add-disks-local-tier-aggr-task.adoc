= Kapazität zu einer lokalen ONTAP-Ebene hinzufügen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können einer lokalen Tier Festplatten hinzufügen, damit sie den zugehörigen Volumes mehr Speicherplatz zur Verfügung stellt.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

[role="tabbed-block"]
====
.System Manager (ONTAP 9.8 und höher)
--

NOTE: Ab ONTAP 9.12.1 kann mit dem System Manager die zugesicherte Kapazität einer lokalen Tier angezeigt werden, um festzustellen, ob zusätzliche Kapazität für die lokale Tier erforderlich ist. Siehe link:../concepts/capacity-measurements-in-sm-concept.html["Kapazität im System Manager überwachen"].

.Schritte
. *Speicher > Ebenen* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen der lokalen Ebene aus, zu der Sie Kapazität hinzufügen möchten.
. *Kapazität hinzufügen* auswählen.
+

NOTE: Wenn keine zusätzlichen Festplatten verfügbar sind, die hinzugefügt werden können, wird die Option *Kapazität hinzufügen* nicht angezeigt und die Kapazität der lokalen Ebene kann nicht erhöht werden.

. Die folgenden Schritte sind abhängig von der installierten ONTAP-Version auszuführen:
+
[cols="30,70"]
|===


| Wenn diese Version von ONTAP installiert ist... | Führen Sie diese Schritte aus... 


 a| 
Ab ONTAP 9.11.1
 a| 
.. Den Datenträgertyp und die Anzahl der Datenträger auswählen.
.. Wenn Festplatten zu einer neuen RAID-Gruppe hinzugefügt werden sollen, ist das Kästchen zu aktivieren. Die RAID-Zuordnung wird angezeigt.
.. *Speichern* auswählen.




 a| 
ONTAP 9.10.1, 9.9 oder 9.8
 a| 
.. Wenn der Knoten mehrere Speicherebenen enthält, kann die Anzahl der Festplatten ausgewählt werden, die der lokalen Ebene hinzugefügt werden sollen. Andernfalls, wenn der Knoten nur eine einzige Speicherebene enthält, wird die hinzugefügte Kapazität automatisch geschätzt.
.. Wählen Sie *Add*.


|===
. (Optional) Der Vorgang benötigt einige Zeit, um abgeschlossen zu werden. Wenn der Vorgang im Hintergrund ausgeführt werden soll, *Im Hintergrund ausführen* auswählen.
. Nach Abschluss des Vorgangs ist die erhöhte Kapazität in den lokalen Tier-Informationen unter *Speicher > Tiers* sichtbar.


--
.System Manager (ONTAP 9.7 und früher)
--
.Schritte
. (Nur für ONTAP 9.7) *(Zurück zur klassischen Version)* auswählen.
. *Hardware und Diagnose > Aggregate* auswählen.
. Wählen Sie die lokale Tier aus, zu der Sie Kapazitätsdisks hinzufügen möchten, und wählen Sie dann *Aktionen > Kapazität hinzufügen*.
+

NOTE: Es empfiehlt sich, Festplatten hinzuzufügen, die die gleiche Größe wie die anderen Festplatten in der lokalen Ebene haben.

. (Nur für ONTAP 9.7) *Zur neuen Benutzeroberfläche wechseln* auswählen.
. *Storage > Tiers* auswählen, um die Größe des neuen lokalen Tiers zu überprüfen.


--
.CLI
--
.Bevor Sie beginnen
Die RAID-Gruppengröße der lokalen Speicherebene, zu der der Speicher hinzugefügt wird, muss bekannt sein.

.Über diese Aufgabe
Das Verfahren zum Hinzufügen partitionierter Festplatten zu einer lokalen Ebene ist dem Verfahren zum Hinzufügen nicht partitionierter Festplatten ähnlich.

Beim Erweitern einer lokalen Ebene sollten Sie darauf achten, ob Sie partitionierte oder nicht partitionierte Festplatten zur lokalen Ebene hinzufügen. Wenn nicht partitionierte Festplatten zu einer bestehenden lokalen Ebene hinzugefügt werden, übernimmt die neue RAID-Gruppe die Größe der vorhandenen RAID-Gruppen, was die Anzahl der benötigten Paritätsfestplatten beeinflussen kann. Wenn eine nicht partitionierte Festplatte zu einer RAID-Gruppe mit partitionierten Festplatten hinzugefügt wird, wird die neue Festplatte partitioniert, wodurch eine ungenutzte Ersatzpartition entsteht.

Beim Bereitstellen von Partitionen muss sichergestellt werden, dass der Knoten nicht ohne ein Laufwerk mit beiden Partitionen als Reserve bleibt. Ist dies doch der Fall und es kommt zu einer Controller-Störung, könnten wertvolle Informationen zum Problem (die Core-Datei) für den technischen Support nicht verfügbar sein.

.Schritte

. Das Hinzufügen des Speichers zum Aggregat simulieren:
+
[source, cli]
----
storage aggregate add-disks -aggregate <aggr_name> -diskcount <number_of_disks_or_partitions> -simulate true
----
+
Sie können das Ergebnis der Speichererweiterung sehen, ohne tatsächlich Speicher bereitzustellen. Wenn bei der simulierten Befehlsausführung Warnungen angezeigt werden, besteht die Möglichkeit, den Befehl anzupassen und die Simulation zu wiederholen.

+
[listing]
----

. Den Speicher dem Aggregat hinzufügen:
+
[source, cli]
----
storage aggregate add-disks -aggregate <aggr_name> -raidgroup new -diskcount <number_of_disks_or_partitions>
----
+
Beim Erstellen einer lokalen Flash Pool Ebene müssen Sie den  `-checksumstyle` Parameter verwenden, wenn Festplatten mit einer anderen Prüfsumme als die der lokalen Ebene hinzugefügt werden oder wenn Festplatten zu einer lokalen Ebene mit gemischter Prüfsumme hinzugefügt werden.

+
Wenn Sie einem lokalen Flash Pool Tier Festplatten hinzufügen, müssen Sie den  `-disktype` Parameter verwenden, um den Festplattentyp anzugeben.

+
Sie können den  `-disksize`Parameter verwenden, um eine Größe der hinzuzufügenden Festplatten anzugeben. Es werden nur Festplatten mit ungefähr der angegebenen Größe für die lokale Tier ausgewählt.

+
[listing]
----
cl1-s2::> storage aggregate add-disks -aggregate data_1 -raidgroup new -diskcount 5
----
