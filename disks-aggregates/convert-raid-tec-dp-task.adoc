= Umstellung von ONTAP RAID-TEC auf RAID-DP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie die Größe Ihrer lokalen Ebene reduzieren und keine dreifache Parität mehr benötigen, kann die RAID-Richtlinie von RAID-TEC auf RAID-DP umgestellt und die Anzahl der für die RAID-Parität benötigten Festplatten reduziert werden.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Bevor Sie beginnen
Die maximale RAID-TEC Gruppengröße ist größer als die maximale RAID-DP Gruppengröße. Wenn die größte RAID-TEC Gruppengröße nicht innerhalb der RAID-DP-Grenzwerte liegt, ist eine Konvertierung zu RAID-DP nicht möglich.

.Über diese Aufgabe
Um die Auswirkungen der Konvertierung zwischen RAID-Typen zu verstehen, siehe die  https://docs.netapp.com/us-en/ontap-cli/storage-aggregate-modify.html#parameters["Parameter"^] für den  `storage aggregate modify` Befehl.

.Schritte
. Es wird überprüft, ob die lokale Ebene online ist und mindestens sechs Festplatten vorhanden sind:
+
`storage aggregate show-status -aggregate _aggregate_name_`

. Konvertieren der lokalen Tier von RAID-TEC zu RAID-DP:
+
`storage aggregate modify -aggregate _aggregate_name_ -raidtype raid_dp`

. Es sollte sichergestellt werden, dass die lokale RAID-Richtlinie RAID-DP ist:
+
`storage aggregate show _aggregate_name_`
