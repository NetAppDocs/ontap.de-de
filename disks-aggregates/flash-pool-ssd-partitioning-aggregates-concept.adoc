= Flash Pool SSD-Partitionierung für ONTAP Flash Pool lokale Tiers unter Verwendung von Storage Pools
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie Cache für zwei oder mehr lokale Flash Pool Ebenen bereitstellen, sollte die Flash Pool Solid-State Drive (SSD) Partitionierung verwendet werden. Die Flash Pool SSD Partitionierung ermöglicht die gemeinsame Nutzung von SSDs durch alle lokalen Ebenen, die den Flash Pool verwenden. Dadurch werden die Kosten der Parität auf mehrere lokale Ebenen verteilt, die Flexibilität der SSD-Cache-Zuweisung erhöht und die SSD-Leistung maximiert.

Damit eine SSD in einem Flash Pool lokalen Tier verwendet werden kann, muss die SSD in einen Speicherpool aufgenommen werden. SSDs, die für Root-Data Partitioning partitioniert wurden, können in einem Speicherpool nicht verwendet werden. Nachdem die SSD in den Speicherpool aufgenommen wurde, kann die SSD nicht mehr als Standalone Laufwerk verwaltet werden und kann nur dann aus dem Speicherpool entfernt werden, wenn die mit dem Flash Pool verbundenen lokalen Tiers zerstört werden und der Speicherpool zerstört wird.

SSD-Speicherpools sind in vier gleich große Zuordnungseinheiten unterteilt. Die dem Speicherpool hinzugefügten SSDs werden in vier Partitionen aufgeteilt, und jeder der vier Zuordnungseinheiten wird eine Partition zugewiesen. Die SSDs im Speicherpool müssen demselben HA-Paar gehören. Standardmäßig werden jedem Knoten im HA-Paar zwei Zuordnungseinheiten zugewiesen. Die Zuordnungseinheiten müssen dem Knoten gehören, der die lokale Tier verwaltet, die er bedient. Wenn für lokale Tiers auf einem der Knoten mehr Flash Cache benötigt wird, kann die Standardanzahl der Zuordnungseinheiten verschoben werden, sodass die Anzahl auf einem Knoten verringert und auf dem Partnerknoten erhöht wird.

Sie verwenden Ersatz-SSDs, um einen SSD-Speicherpool hinzuzufügen. Wenn der Speicherpool Zuordnungseinheiten für Flash Pool lokale Ebenen bereitstellt, die beiden Knoten im HA-Paar gehören, können die Ersatz-SSDs einem der beiden Knoten gehören. Stellt der Speicherpool jedoch Zuordnungseinheiten nur für Flash Pool lokale Ebenen bereit, die einem der Knoten im HA-Paar gehören, müssen die SSD-Ersatzlaufwerke demselben Knoten gehören.

Die folgende Abbildung zeigt ein Beispiel für die SSD-Partitionierung eines Flash Pool. Der SSD-Speicherpool dient als Cache für zwei Flash Pool lokale Tiers:

image:shared-ssds-overview.gif["Diagramm der Flash Pool SSD-Partitionierung."]

Der Storage Pool SP1 besteht aus fünf SSDs und einer Hot Spare SSD. Zwei der Allocation Units des Storage Pools sind Flash Pool FP1 zugeordnet, und zwei sind Flash Pool FP2 zugeordnet. FP1 hat einen Cache-RAID-Typ von RAID4. Daher enthalten die FP1 zugewiesenen Allocation Units nur eine Partition, die für Parität vorgesehen ist. FP2 hat einen Cache-RAID-Typ von RAID-DP. Daher enthalten die FP2 zugewiesenen Allocation Units eine Paritätspartition und eine Doppelparitätspartition.

In diesem Beispiel werden jedem Flash Pool lokalen Tier zwei Zuordnungseinheiten zugewiesen. Falls jedoch ein Flash Pool lokales Tier einen größeren Cache benötigt, können drei der Zuordnungseinheiten diesem Flash Pool lokalen Tier und nur eine dem anderen zugewiesen werden.
