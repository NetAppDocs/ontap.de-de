= Passen Sie die Größe Ihrer ONTAP RAID-Gruppen an
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Größe Ihrer RAID-Gruppen anpassen, um sicherzustellen, dass die RAID-Gruppengrößen für die Speichermenge geeignet sind, die Sie für eine lokale Ebene einplanen.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Über diese Aufgabe
Für Standard-Lokalebenen wird die Größe der RAID-Gruppen für jede lokale Ebene separat geändert. Für Flash Pool-Lokalebenen kann die RAID-Gruppengröße für die SSD-RAID-Gruppen und die HDD-RAID-Gruppen unabhängig voneinander geändert werden.

Die folgende Liste enthält einige Fakten zur Änderung der RAID-Gruppengröße:

* Standardmäßig werden, wenn die Anzahl der Festplatten oder Array-LUNs in der zuletzt erstellten RAID-Gruppe kleiner ist als die neue RAID-Gruppengröße, Festplatten oder Array-LUNs zur zuletzt erstellten RAID-Gruppe hinzugefügt, bis diese die neue Größe erreicht.
* Alle anderen bestehenden RAID-Gruppen in dieser lokalen Ebene behalten ihre Größe, es sei denn, Sie fügen ihnen explizit Festplatten hinzu.
* Es ist niemals möglich, eine RAID-Gruppe größer werden zu lassen als die aktuelle maximale RAID-Gruppengröße für die lokale Ebene.
* Die Größe bereits erstellter RAID-Gruppen kann nicht verringert werden.
* Die neue Größe gilt für alle RAID-Gruppen in dieser lokalen Ebene (oder, im Falle einer Flash Pool lokalen Ebene, für alle RAID-Gruppen des betroffenen RAID-Gruppentyps, SSD oder HDD).


.Schritte
. Verwenden Sie den entsprechenden Befehl:
+
|===


| Wenn Sie möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Die maximale RAID-Gruppengröße für die SSD RAID-Gruppen einer Flash Pool lokalen Ebene ändern
 a| 
`storage aggregate modify -aggregate aggr_name -cache-raid-group-size size`



 a| 
Die maximale Größe anderer RAID-Gruppen ändern
 a| 
`storage aggregate modify -aggregate aggr_name -maxraidsize size`

|===


.Beispiele
Der folgende Befehl ändert die maximale RAID-Gruppengröße der lokalen Ebene n1_a4 auf 20 Festplatten oder Array-LUNs:

`storage aggregate modify -aggregate n1_a4 -maxraidsize 20`

Der folgende Befehl ändert die maximale RAID-Gruppengröße der SSD-Cache-RAID-Gruppen des Flash Pool lokalen Tiers n1_cache_a2 auf 24:

`storage aggregate modify -aggregate n1_cache_a2 -cache-raid-group-size 24`
