= Speichernutzung eines Volumes in einer lokalen ONTAP-Ebene ermitteln und steuern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können feststellen, welche FlexVol Volumes den meisten Speicherplatz in einer lokalen Ebene belegen und welche Features innerhalb des Volumes diesen Speicherplatz nutzen.

Der  `volume show-footprint` Befehl liefert Informationen über den Footprint eines Volumes bzw. dessen Speicherplatznutzung innerhalb der übergeordneten lokalen Ebene.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

Der  `volume show-footprint` Befehl zeigt Details zur Speichernutzung jedes Volumes in einer lokalen Ebene an, einschließlich Offline-Volumes. Dieser Befehl schließt die Lücke zwischen den Ausgaben der  `volume show-space` und  `aggregate show-space` Befehle. Alle Prozentangaben werden als Prozentsatz der Größe der lokalen Ebene berechnet.

Das folgende Beispiel zeigt die `volume show-footprint` Befehlsausgabe für ein Volume namens testvol:

....
cluster1::> volume show-footprint testvol

      Vserver : thevs
      Volume  : testvol

      Feature                                   Used    Used%
      --------------------------------    ----------    -----
      Volume Data Footprint                  120.6MB       4%
      Volume Guarantee                        1.88GB      71%
      Flexible Volume Metadata               11.38MB       0%
      Delayed Frees                           1.36MB       0%
      Total Footprint                         2.01GB      76%
....
Die folgende Tabelle erläutert einige der wichtigsten Zeilen der Ausgabe des  `volume show-footprint`-Befehls und welche Möglichkeiten bestehen, den Speicherplatzverbrauch dieser Funktion zu verringern:

|===


| Zeilen-/Feature-Name | Beschreibung/Inhalt der Zeile | Einige Möglichkeiten zur Verringerung 


 a| 
`Volume Data Footprint`
 a| 
Die Gesamtmenge an Speicherplatz, die im lokalen Speichermedium von den Daten eines Volumes im aktiven Dateisystem und dem von den Snapshots des Volumes belegten Speicherplatz verwendet wird. Diese Zeile enthält keinen reservierten Speicherplatz.
 a| 
* Daten werden aus dem Volume gelöscht.
* Snapshots aus dem Volume werden gelöscht.




 a| 
`Volume Guarantee`
 a| 
Der vom Volume im lokalen Speicherbereich für zukünftige Schreibvorgänge reservierte Speicherplatz. Die Menge des reservierten Speicherplatzes hängt vom Garantietyp des Volumes ab.
 a| 
Änderung der Garantieart für das Volumen auf  `none`.



 a| 
`Flexible Volume Metadata`
 a| 
Der gesamte Speicherplatz, der in der lokalen Tier von den Metadatendateien des Volumes belegt wird.
 a| 
Keine direkte Methode zur Steuerung.



 a| 
`Delayed Frees`
 a| 
Blöcke, die ONTAP aus Performancegründen verwendet und die nicht sofort freigegeben werden können. Für SnapMirror-Ziele hat diese Zeile einen Wert von  `0` und wird nicht angezeigt.
 a| 
Keine direkte Methode zur Steuerung.



 a| 
`File Operation Metadata`
 a| 
Der gesamte reservierte Speicherplatz für Metadaten von Dateivorgängen.
 a| 
Keine direkte Methode zur Steuerung.



 a| 
`Total Footprint`
 a| 
Der gesamte Speicherplatz, den das Volume in der lokalen Ebene belegt. Dies ist die Summe aller Zeilen.
 a| 
Alle Methoden, die verwendet werden, um den von einem Volumen belegten Speicherplatz zu verringern.

|===
