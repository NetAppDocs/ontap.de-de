= ONTAP lokale Tiers automatisch hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn die von ONTAP empfohlene Best-Practice-Empfehlung zum automatischen Hinzufügen einer lokalen Tier in Ihrer Umgebung angemessen ist, kann die Empfehlung übernommen werden und ONTAP fügt die lokale Tier hinzu.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

.Bevor Sie beginnen
Festplatten müssen einem Node zugeordnet sein, bevor sie in einer lokalen Ebene verwendet werden können. Wenn Ihr Cluster nicht für die automatische Zuweisung des Festplattenbesitzes konfiguriert ist, muss link:manual-assign-disks-ownership-prep-task.html["Besitzverhältnisse manuell zuweisen"] erfolgen.

Weitere Informationen zu den in diesem Verfahren beschriebenen Befehlen befinden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].

.System Manager
--
.Schritte
. Im System Manager auf *Speicher > Ebenen* klicken.
. Auf der Seite *Tiers* image:icon-add-local-tier.png["Lokales Tier-Symbol hinzufügen"] einen neuen lokalen Tier erstellen:
+
Auf der Seite *Lokale Ebene hinzufügen* wird die empfohlene Anzahl lokaler Ebenen angezeigt, die auf den Knoten erstellt werden können, sowie der verfügbare nutzbare Speicherplatz.

. *Empfohlene Details* zeigt die von System Manager empfohlene Konfiguration an.
+
Der System Manager zeigt ab ONTAP 9.8 folgende Informationen an:

+
** *Lokaler Tiername* (der lokale Tiername kann ab ONTAP 9.10.1 bearbeitet werden)
** *Node-Name*
** *Nutzbare Größe*
** *Speicherart*


+
Ab ONTAP 9.10.1 werden zusätzliche Informationen angezeigt:

+
** *Disks*: Anzeige der Anzahl, Größe und des Typs der Disks
** *Layout*: Darstellung des RAID-Gruppenlayouts, einschließlich der Angabe, welche Festplatten Parität oder Daten sind und welche Steckplätze ungenutzt sind.
** *Ersatzfestplatten*: Anzeige des Node-Name, der Anzahl und Größe der Ersatzfestplatten und des Speichertyps.

 a|

 a|

--
.CLI
--
Sie führen den  `storage aggregate auto-provision` Befehl aus, um Empfehlungen für das lokale Tier-Layout zu generieren. Anschließend können lokale Tiers erstellt werden, nachdem ONTAP Empfehlungen geprüft und genehmigt wurden.

.Über diese Aufgabe
Die mit dem  `storage aggregate auto-provision` Befehl generierte Standardzusammenfassung listet die empfohlenen lokalen Tiers auf, einschließlich Namen und nutzbarer Größe. Die Liste kann eingesehen werden, um zu bestimmen, ob die empfohlenen lokalen Tiers bei entsprechender Aufforderung erstellt werden sollen.

Eine detaillierte Zusammenfassung kann auch mit der  `-verbose` Option angezeigt werden, die die folgenden Berichte anzeigt:

* Zusammenfassung pro Node der neu zu erstellenden lokalen Tiers, der erkannten Ersatzfestplatten sowie der verbleibenden Ersatzfestplatten und Partitionen nach der Erstellung der lokalen Tiers
* Neue lokale Datenebenen, die mit der Anzahl der zu verwendenden Festplatten und Partitionen erstellt werden sollen
* RAID-Gruppenlayout, das zeigt, wie Ersatzfestplatten und Partitionen in den neuen lokalen Datenebenen verwendet werden.
* Details zu den nach der Erstellung der lokalen Tier verbleibenden Ersatzfestplatten und Partitionen


Wenn Sie mit der automatischen Bereitstellungsmethode vertraut sind und Ihre Umgebung korrekt vorbereitet ist, kann die  `-skip-confirmation` Option verwendet werden, um die empfohlene lokale Tier ohne Anzeige und Bestätigung zu erstellen. Der  `storage aggregate auto-provision` Befehl wird nicht von der CLI-Sitzung  `-confirmations` Einstellung beeinflusst.

Weitere Informationen zu  `storage aggregate auto-provision` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/storage-aggregate-auto-provision.html["ONTAP-Befehlsreferenz"^].

.Schritte
. Den  `storage aggregate auto-provision`Befehl mit den gewünschten Anzeigeoptionen ausführen.
+
** Keine Optionen: Standardzusammenfassung
** `-verbose`Option: Detaillierte Zusammenfassung anzeigen
** `-skip-confirmation`Option: Empfohlene lokale Tiers ohne Anzeige oder Bestätigung erstellen


. Einer der folgenden Schritte kann durchgeführt werden:
+
[cols="35,65"]
|===


| Wenn Sie Folgendes tun möchten… | Dann ist Folgendes zu tun… 


 a| 
Die Empfehlungen von ONTAP werden übernommen.
 a| 
Die Anzeige der empfohlenen lokalen Tiers wird überprüft, anschließend erfolgt eine Reaktion auf die Aufforderung zur Erstellung der empfohlenen lokalen Tiers.

[listing]
----

Do you want to create recommended aggregates? {y|n}: y

      show-auto-provision-progress" command to track the progress.

myA400-44556677::>

----
