= Einen ONTAP Flash Pool lokalen Tier mit SSD-Speicherpool-Zuordnungseinheiten erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine Flash Pool lokale Ebene konfigurieren, indem Sie einer bestehenden HDD lokalen Ebene eine oder mehrere Zuordnungseinheiten aus einem SSD-Speicherpool hinzufügen.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

Ab ONTAP 9.12.1 kann der neu gestaltete System Manager verwendet werden, um eine Flash Pool lokale Ebene mithilfe von Speicherpool-Zuordnungseinheiten zu erstellen.

.Bevor Sie beginnen
* Es muss eine gültige lokale Ebene identifiziert sein, die aus HDDs besteht, um sie in eine Flash Pool lokale Ebene umzuwandeln.
* Die Schreibcache-Berechtigung der mit der lokalen Ebene verbundenen Volumes muss ermittelt und alle erforderlichen Schritte zur Behebung von Berechtigungsproblemen abgeschlossen sein.
* Sie müssen einen SSD-Speicherpool erstellt haben, um den SSD-Cache für diese Flash Pool lokale Ebene bereitzustellen.
+
Jede Zuordnungseinheit aus dem Speicherpool, die verwendet werden soll, muss demselben Knoten gehören, dem auch die lokale Flash Pool Ebene gehört.

* Es muss festgelegt sein, wie viel Cache dem lokalen Speicher hinzugefügt werden soll.
+
Sie fügen dem lokalen Tier Cache durch Zuordnungseinheiten hinzu. Die Größe der Zuordnungseinheiten kann später durch das Hinzufügen von SSDs zum Storage Pool erhöht werden, sofern ausreichend Platz vorhanden ist.

* Der gewünschte RAID-Typ für den SSD-Cache muss festgelegt sein.
+
Nachdem Sie einen Cache aus SSD-Speicherpools zur lokalen Speicherebene hinzugefügt haben, kann der RAID-Typ der Cache-RAID-Gruppen nicht mehr geändert werden.

* Sie müssen die maximale Cache-Größe für Ihr System ermittelt und festgestellt haben, dass das Hinzufügen eines SSD-Cache zu Ihrer lokalen Speicherebene nicht dazu führt, dass diese überschritten wird.
+
Die Menge des Caches, die zur Gesamtgröße des Caches hinzugefügt wird, lässt sich mit dem  `storage pool show` Befehl anzeigen.

* Sie müssen sich mit den Konfigurationsanforderungen für die lokale Ebene von Flash Pool vertraut gemacht haben.


.Über diese Aufgabe
Soll der RAID-Typ des Cache von dem der HDD-RAID-Gruppen abweichen, muss der Cache-RAID-Typ beim Hinzufügen der SSD-Kapazität angegeben werden. Nach dem Hinzufügen der SSD-Kapazität zur lokalen Ebene ist eine Änderung des RAID-Typs des Cache nicht mehr möglich.

Nachdem Sie einem lokalen Tier einen SSD-Cache hinzugefügt haben, um ein Flash Pool lokales Tier zu erstellen, kann der SSD-Cache nicht mehr entfernt werden, um das lokale Tier wieder in seine ursprüngliche Konfiguration zurückzuversetzen.

[role="tabbed-block"]
====
.System Manager
--
Ab ONTAP 9.12.1 ist es möglich, mit System Manager SSDs zu einem SSD-Speicherpool hinzuzufügen.

.Schritte
. Klicken Sie auf *Storage > Tiers* und wählen Sie eine vorhandene lokale HDD Storage-Tier aus.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Add Flash Pool Cache* aus.
. *Speicherpools verwenden* auswählen.
. Wählen Sie einen Speicherpool aus.
. Eine Cache-Größe und eine RAID-Konfiguration auswählen.
. Auf *Speichern* klicken.
. Die Speicherebene erneut suchen und auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken
. *Weitere Details* auswählen und überprüfen, dass Flash Pool als *Aktiviert* angezeigt wird.


--
.CLI
--
.Schritte
. Die lokale Tier wird als berechtigt markiert, eine Flash Pool lokale Tier zu werden:
+
`storage aggregate modify -aggregate _aggr_name_ -hybrid-enabled true`

+
Falls dieser Schritt nicht erfolgreich ist, die Schreibcache-Berechtigung für die Zielebene bestimmen.

. Verfügbare SSD-Speicherpool-Zuordnungseinheiten:
+
`storage pool show-available-capacity`

. Die SSD-Kapazität wird der lokalen Ebene hinzugefügt:
+
`storage aggregate add _aggr_name_ -storage-pool _sp_name_ -allocation-units _number_of_units_`

+
Wenn der RAID-Typ des Caches sich von dem der HDD-RAID-Gruppen unterscheiden soll, muss dies bei der Eingabe dieses Befehls mit dem  `raidtype` Parameter geändert werden.

+
Sie müssen keine neue RAID-Gruppe angeben; ONTAP platziert den SSD-Cache automatisch in separate RAID-Gruppen, getrennt von den HDD-RAID-Gruppen.

+
Die RAID-Gruppengröße des Caches kann nicht festgelegt werden; sie wird durch die Anzahl der SSDs im Speicherpool bestimmt.

+
Der Cache wird der lokalen Ebene hinzugefügt und die lokale Ebene ist nun eine Flash Pool lokale Ebene. Jede der lokalen Ebene hinzugefügte Zuordnungseinheit bildet eine eigene RAID-Gruppe.

. Das Vorhandensein und die Größe des SSD-Cache bestätigen:
+
`storage aggregate show _aggregate_name_`

+
Die Größe des Caches ist unter  `Total Hybrid Cache Size` aufgeführt.
