= Festplatten und lokale ONTAP-Ebenen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
_Lokale Ebenen_, auch _Aggregate_ genannt, sind logische Container für die von einem Knoten verwalteten Festplatten. Lokale Ebenen ermöglichen die Isolierung von Workloads mit unterschiedlichen Leistungsanforderungen, das Tiering von Daten mit unterschiedlichen Zugriffsmustern oder die Trennung von Daten zu regulatorischen Zwecken.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_.

* Für geschäftskritische Anwendungen, die eine möglichst geringe Latenz und eine möglichst hohe Leistung erfordern, kann eine lokale Ebene erstellt werden, die ausschließlich aus SSDs besteht.
* Um Daten mit unterschiedlichen Zugriffsmustern zu schichten, kann eine _hybride lokale Tier_ erstellt werden, wobei Flash als Hochleistungscache für einen Arbeitsdatensatz dient und kostengünstigere HDDs oder Objektspeicher für weniger häufig abgerufene Daten genutzt werden.
+
** A link:flash-pool-aggregate-caching-policies-concept.html["_Flash Pool_"] besteht sowohl aus SSDs als auch aus HDDs.
** A link:../concepts/fabricpool-concept.html["_FabricPool_"] besteht aus einer vollständig auf SSDs basierenden lokalen Schicht mit einem angeschlossenen Objektspeicher.


* Wenn aus regulatorischen Gründen archivierte Daten von aktiven Daten getrennt werden müssen, kann eine lokale Ebene verwendet werden, die aus Kapazitäts-HDDs oder einer Kombination aus Performance- und Kapazitäts-HDDs besteht.
