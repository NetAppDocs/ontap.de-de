= Eine Aktiv/Passiv-Konfiguration auf ONTAP Knoten mit Root-Datenpartitionierung einrichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ein HA-Paar werkseitig für die Verwendung der Root-Datenpartitionierung konfiguriert ist, wird die Eigentümerschaft der Datenpartitionen zwischen beiden Knoten des Paares für die Nutzung in einer Aktiv/Aktiv-Konfiguration aufgeteilt. Soll das HA-Paar in einer Aktiv/Passiv-Konfiguration verwendet werden, ist die Partitionseigentümerschaft vor der Erstellung der lokalen Datenebene zu aktualisieren.

.Bevor Sie beginnen
* Es sollte festgelegt sein, welcher Knoten der aktive und welcher der passive Knoten ist.
* Die Speicherausfallsicherung muss auf dem HA Pair konfiguriert werden.


.Über diese Aufgabe
Diese Aufgabe wird auf zwei Knoten ausgeführt: Knoten A und Knoten B.

Dieses Verfahren ist für Knoten vorgesehen, für die kein Datenaggregat aus den partitionierten Festplatten erstellt wurde.

Informationen dazu link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/What_are_the_rules_for_Advanced_Disk_Partitioning%3F["Erweiterte Festplattenpartitionierung"^].

.Schritte
Alle Befehle werden in der Cluster-Shell eingegeben.

. Die aktuelle Eigentümerstruktur der Datenpartitionen wird angezeigt:
+
`storage aggregate show-spare-disks`

+
Die Ausgabe zeigt, dass die Hälfte der Datenpartitionen einem Knoten und die andere Hälfte dem anderen Knoten zugeordnet ist. Alle Datenpartitionen sollten Reservepartitionen sein.

. Die erweiterte Berechtigungsstufe eingeben:
+
`set advanced`

. Für jede Datenpartition, die dem Knoten gehört, der der passive Knoten sein wird, wird sie dem aktiven Knoten zugewiesen:
+
`storage disk assign -force -data true -owner _active_node_name_ -disk _disk_name_`

+
Sie müssen die Partition nicht als Teil des Datenträgernamens angeben.

+
Für jede Datenpartition, die Sie neu zuweisen müssen, wäre ein Befehl ähnlich dem folgenden Beispiel einzugeben:

+
`storage disk assign -force -data true -owner cluster1-01 -disk 1.0.3`

. Bestätigen Sie, dass alle Partitionen dem aktiven Knoten zugewiesen sind.

+
Beachten Sie, dass cluster1-02 weiterhin eine freie Root-Partition besitzt.

. Zurück zu als Administrator angemeldet:
+
`set admin`

. Erstellen Sie Ihre lokale Datenebene und lassen Sie mindestens eine Datenpartition als Reserve:
+
`storage aggregate create _new_aggr_name_ -diskcount _number_of_partitions_ -node _active_node_name_`

+
Die lokale Datenebene wird erstellt und gehört dem aktiven Knoten.
