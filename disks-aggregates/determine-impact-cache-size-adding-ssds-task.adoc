= Die Auswirkungen auf die ONTAP Cache-Größe beim Hinzufügen von SSDs zu einem SSD-Speicherpool bestimmen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn durch das Hinzufügen von SSDs zu einem Speicherpool das Cache-Limit Ihres Plattformmodells überschritten wird, weist ONTAP die neu hinzugefügte Kapazität keinem Flash Pool lokalen Tier zu. Dies kann dazu führen, dass ein Teil oder die gesamte neu hinzugefügte Kapazität nicht verfügbar ist.

.Über diese Aufgabe
Wenn Sie SSDs zu einem SSD-Speicherpool hinzufügen, dem bereits Zuordnungseinheiten für Flash Pool lokale Ebenen zugewiesen sind, erhöht sich die Cache-Größe jeder dieser lokalen Ebenen und der gesamte Cache des Systems. Sind noch keine Zuordnungseinheiten des Speicherpools zugewiesen, wirkt sich das Hinzufügen von SSDs zu diesem Speicherpool nicht auf die SSD-Cache-Größe aus, bis eine oder mehrere Zuordnungseinheiten einem Cache zugewiesen werden.

.Schritte
. Die nutzbare Größe der SSDs, die dem Speicherpool hinzugefügt werden, lässt sich wie folgt bestimmen:
+
`storage disk show _disk_name_ -fields usable-size`

. Ermitteln, wie viele Zuordnungseinheiten für den Speicherpool noch nicht zugeordnet sind:
+
`storage pool show-available-capacity _sp_name_`

+
Alle nicht zugewiesenen Zuordnungseinheiten im Speicherpool werden angezeigt.

. Die Menge des hinzuzufügenden Caches lässt sich mit der folgenden Formel berechnen:
+
( 4 – Anzahl der nicht zugewiesenen Zuordnungseinheiten) × 25% × nutzbare Größe × Anzahl der SSDs
