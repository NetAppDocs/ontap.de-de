= Zuständigkeit für eine ONTAP lokale Ebene innerhalb eines HA-Paares verlagern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Zuständigkeit für lokale Ebenen zwischen den Knoten eines HA-Paares ändern, ohne den Dienst von den lokalen Ebenen zu unterbrechen.


NOTE: Vor ONTAP 9.7 verwendet System Manager den Begriff _aggregate_, um eine _lokale Ebene_ zu beschreiben. Unabhängig von der ONTAP Version verwendet die ONTAP CLI den Begriff _aggregate_. Weitere Informationen zu lokalen Ebenen finden sich unter link:../disks-aggregates/index.html["Festplatten und lokale Ebenen"].

Beide Knoten eines HA-Paares sind physisch mit den Festplatten oder Array-LUNs des jeweils anderen Knotens verbunden. Jede Festplatte oder Array-LUN gehört einem der Knoten.

Bei einer Übernahme wechselt die Eigentümerschaft aller Festplatten oder Array-LUNs innerhalb einer lokalen Ebene vorübergehend von einem Knoten zum anderen. Lokale Ebenenverschiebungsoperationen können die Eigentümerschaft jedoch auch dauerhaft ändern (zum Beispiel bei einer Lastverteilung). Die Eigentümerschaft ändert sich ohne Datenkopierprozesse oder physische Bewegung der Festplatten oder Array-LUNs.

.Über diese Aufgabe
* Da Volume-Anzahl-Grenzwerte während der lokalen Tier-Umlagerungsvorgänge programmatisch überprüft werden, ist eine manuelle Überprüfung nicht erforderlich.
+
Wenn die Volume-Anzahl das unterstützte Limit überschreitet, schlägt die lokale Tier-Verschiebung mit einer entsprechenden Fehlermeldung fehl.

* Eine lokale Tier-Verschiebung sollte nicht eingeleitet werden, wenn auf dem Quell- oder Zielknoten Systemoperationen ausgeführt werden; ebenso sollten diese Operationen nicht während der lokalen Tier-Verschiebung gestartet werden.
+
Diese Vorgänge können Folgendes umfassen:

+
** Übernahme
** Giveback
** Herunterfahren
** Ein weiterer Vorgang zur lokalen Tier-Verlagerung
** Änderungen des Festplattenbesitzes
** Lokale Tier- oder Volume-Konfigurationsvorgänge
** Speichercontroller Austausch
** ONTAP Upgrade
** ONTAP Revertierung


* Wenn Sie über eine MetroCluster Konfiguration verfügen, sollte keine lokale Tier-Verschiebung eingeleitet werden, während Notfallwiederherstellungsmaßnahmen (_switchover_, _healing_ oder _switchback_) im Gange sind.
* Wenn Sie eine MetroCluster Konfiguration haben und eine lokale Tier-Verschiebung auf einem umgeschalteten lokalen Tier initiieren, kann der Vorgang fehlschlagen, weil die Volume-Anzahl des DR-Partners überschritten wird.
* Eine lokale Tier-Verschiebung sollte nicht auf lokalen Tiers durchgeführt werden, die beschädigt sind oder sich in der Wartung befinden.
* Vor Beginn der lokalen Tier Verschiebung sollten alle Core-Dumps auf den Quell- und Zielknoten gespeichert werden.


.Schritte
. Die lokalen Ebenen auf dem Knoten anzeigen, um zu bestätigen, welche lokalen Ebenen verschoben werden sollen, und sicherzustellen, dass sie online und in gutem Zustand sind:
+
`storage aggregate show -node _source-node_`

+
Der folgende Befehl zeigt sechs lokale Ebenen auf den vier Knoten im Cluster. Alle lokalen Ebenen sind online. Node1 und Node3 bilden ein HA-Paar und Node2 und Node4 bilden ein HA-Paar.

+
[listing]
----
cluster::> storage aggregate show
Aggregate     Size Available Used% State   #Vols  Nodes  RAID Status
--------- -------- --------- ----- ------- ------ ------ -----------
aggr_0     239.0GB   11.13GB   95% online       1 node1  raid_dp,
                                                         normal
aggr_1     239.0GB   11.13GB   95% online       1 node1  raid_dp,
                                                         normal
aggr_2     239.0GB   11.13GB   95% online       1 node2  raid_dp,
                                                         normal
aggr_3     239.0GB   11.13GB   95% online       1 node2  raid_dp,
                                                         normal
aggr_4     239.0GB   238.9GB    0% online       5 node3  raid_dp,
                                                         normal
aggr_5     239.0GB   239.0GB    0% online       4 node4  raid_dp,
                                                         normal
6 entries were displayed.
----
. Den Befehl zum Starten der lokalen Tier-Verlagerung ausgeben:
+
`storage aggregate relocation start -aggregate-list _aggregate-1_, _aggregate-2_... -node _source-node_ -destination _destination-node_`

+
Der folgende Befehl verschiebt die lokalen Tiers aggr_1 und aggr_2 von Node1 zu Node3. Node3 ist der HA-Partner von Node1. Die lokalen Tiers können nur innerhalb des HA-Paars verschoben werden.

+
[listing]
----
cluster::> storage aggregate relocation start -aggregate-list aggr_1, aggr_2 -node node1 -destination node3
Run the storage aggregate relocation show command to check relocation status.
node1::storage aggregate>
----
. Der Fortschritt der lokalen Tier-Verschiebung kann mit dem  `storage aggregate relocation show` Befehl überwacht werden:
+
`storage aggregate relocation show -node _source-node_`

+
Der folgende Befehl zeigt den Fortschritt der lokalen Ebenen, die zu Node3 verschoben werden:

+
[listing]
----
cluster::> storage aggregate relocation show -node node1
Source Aggregate   Destination    Relocation Status
------ ----------- -------------  ------------------------
node1
       aggr_1      node3          In progress, module: wafl
       aggr_2      node3          Not attempted yet
2 entries were displayed.
node1::storage aggregate>
----
+
Wenn die Verlagerung abgeschlossen ist, zeigt die Ausgabe dieses Befehls für jede lokale Ebene den Verlagerungsstatus „Done“ an.
