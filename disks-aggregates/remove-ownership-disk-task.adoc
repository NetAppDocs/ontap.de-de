= ONTAP Besitzrechte von einer Festplatte entfernen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP schreibt die Eigentümerinformationen auf die Festplatte. Vor dem Entfernen einer Ersatzfestplatte oder ihres Gehäuses aus einem Knoten empfiehlt es sich, die Eigentümerinformationen zu entfernen, damit sie ordnungsgemäß in einen anderen Knoten integriert werden kann.


NOTE: Wenn die Festplatte für die Root-Datenpartitionierung partitioniert ist und ONTAP 9.10.1 oder höher verwendet wird, ist der NetApp Technical Support für Unterstützung beim Entfernen der Besitzrechte zu kontaktieren. Weitere Informationen finden sich im link:https://kb.netapp.com/onprem/ontap/hardware/Error%3A_command_failed%3A_Failed_to_remove_the_owner_of_disk["Knowledge Base Artikel: Fehler beim Entfernen des Festplattenbesitzers"^].

.Bevor Sie beginnen
Die Festplatte, von der Sie die Besitzverhältnisse entfernen möchten, muss die folgenden Anforderungen erfüllen:

* Es muss sich um eine Ersatzfestplatte handeln.
+
Sie können die Besitzrechte an einer Festplatte, die in einer lokalen Tier verwendet wird, nicht entfernen.

* Es kann sich nicht im Wartungszentrum befinden.
* Es darf sich nicht in der Bereinigung befinden.
* Es kann nicht fehlgeschlagen sein.
+
Es ist nicht notwendig, die Besitzrechte einer ausgefallenen Festplatte zu entfernen.



.Über diese Aufgabe
Wenn die automatische Festplattenzuweisung aktiviert ist, kann ONTAP die Besitzverhältnisse automatisch neu zuweisen, bevor die Festplatte aus dem Knoten entfernt wird. Aus diesem Grund wird die automatische Besitzzuweisung deaktiviert, bis die Festplatte entfernt ist, und anschließend wieder aktiviert.

.Schritte
. Wenn die automatische Zuweisung der Festplattenbesitzrechte aktiviert ist, kann sie über die Befehlszeile deaktiviert werden:
+
`storage disk option modify -node _node_name_ -autoassign off`

. Falls erforderlich kann der vorherige Schritt für den HA-Partner des Knotens wiederholt werden.
. Die Software-Eigentümerinformationen werden von der Festplatte entfernt:
+
`storage disk removeowner _disk_name_`

+
Um Besitzinformationen von mehreren Datenträgern zu entfernen, ist eine durch Kommas getrennte Liste zu verwenden.

+
Beispiel:

+
....
storage disk removeowner sys1:0a.23,sys1:0a.24,sys1:0a.25
....
. Wenn die Festplatte für die Root-Datenpartitionierung partitioniert ist und ONTAP 9.9.1 oder eine frühere Version verwendet wird, sind die Besitzrechte der Partitionen zu entfernen:
+
--
`storage disk removeowner -disk _disk_name_ -root true`

`storage disk removeowner -disk _disk_name_ -data true`

Beide Partitionen gehören keinem Knoten mehr.

--
. Wenn Sie die automatische Zuweisung des Datenträgerbesitzes zuvor deaktiviert hatten, sollte sie nach dem Entfernen oder der Neuzuweisung des Datenträgers wieder aktiviert werden:
+
`storage disk option modify -node _node_name_ -autoassign on`

. Falls erforderlich kann der vorherige Schritt für den HA-Partner des Knotens wiederholt werden.
