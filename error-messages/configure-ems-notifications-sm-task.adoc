= ONTAP EMS-Ereignisbenachrichtigungen und Filter mit System Manager konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit System Manager lässt sich konfigurieren, wie das Event Management System (EMS) Ereignisbenachrichtigungen übermittelt, sodass Sie über Systemprobleme informiert werden, die Ihre sofortige Aufmerksamkeit erfordern.

[cols="35,65"]
|===


| ONTAP Version | Mit System Manager können Sie... 


 a| 
ONTAP 9.12.1 und höher
 a| 
Das Transport Layer Security (TLS)-Protokoll kann angegeben werden, wenn Ereignisse an entfernte Syslog-Server gesendet werden.



 a| 
ONTAP 9.10.1 und höher
 a| 
E-Mail-Adressen, Syslog-Server, Webhook-Anwendungen sowie SNMP-Traphosts konfigurieren.



 a| 
ONTAP 9.10.0 bis 9.7
 a| 
Es werden nur SNMP Traphosts konfiguriert. Andere EMS Ziele können mit der ONTAP CLI konfiguriert werden. Siehe link:index.html["EMS Konfigurationsübersicht"].

|===


[[add-an-ems-event-notification-destination]]
== Ein EMS-Ereignisbenachrichtigungsziel hinzufügen

Mit System Manager kann angegeben werden, wohin EMS-Nachrichten gesendet werden.

Ab ONTAP 9.12.1 können EMS-Ereignisse über das Transport Layer Security (TLS)-Protokoll an einen festgelegten Port eines entfernten Syslog-Servers gesendet werden. Weitere Informationen zu  `event notification destination create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/event-notification-destination-create.html["ONTAP-Befehlsreferenz"^].

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisziele* auswählen.
. Klicken Sie image:../media/icon_add.gif["Symbol hinzufügen"].
. Ein Name, ein EMS-Zieltyp und Filter sind anzugeben.
+

NOTE: Bei Bedarf kann ein neuer Filter hinzugefügt werden. *Neuen Ereignisfilter hinzufügen* auswählen.

. Je nach dem ausgewählten EMS-Zieltyp sind die folgenden Angaben erforderlich:


[cols="40,60"]
|===


| Zum Konfigurieren… | Angeben oder auswählen… 


 a| 
SNMP Traphost
 a| 
* Traphost Name




 a| 
E-Mail

(Beginnend mit 9.10.1)
 a| 
* E-Mail-Adresse des Empfängers
* Mailserver
* Von der E-Mail-Adresse




 a| 
Syslog Server

(Beginnend mit 9.10.1)
 a| 
* Hostname oder IP-Adresse des Servers
* Syslog Port (beginnend mit 9.12.1)
* Syslog Transport (ab 9.12.1)
+
Die Auswahl von *TCP Encrypted* aktiviert das Transport Layer Security (TLS) Protokoll. Wenn für *Syslog port* kein Wert angegeben wird, wird ein Standardwert basierend auf der Auswahl von *Syslog transport* verwendet.





 a| 
Webhook

(Beginnend mit 9.10.1)
 a| 
* Webhook-URL
* Client-Authentifizierung (diese Option gibt die Möglichkeit an, ein Client-Zertifikat zu spezifizieren)


|===


[[create-a-new-ems-event-notification-filter]]
== Einen neuen EMS Ereignisbenachrichtigungsfilter erstellen

Ab ONTAP 9.10.1 kann mit System Manager definiert werden, welche neuen benutzerdefinierten Filter die Regeln für die Verarbeitung von EMS-Benachrichtigungen festlegen.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisfilter* auswählen.
. Klicken Sie image:../media/icon_add.gif["Symbol hinzufügen"].
. Einen Namen angeben und auswählen, ob Regeln aus einem vorhandenen Ereignisfilter kopiert oder neue Regeln hinzugefügt werden sollen.
. Je nach Ihrer Auswahl sind die folgenden Schritte auszuführen:


[cols="40,60"]
|===


| Wenn Sie sich dafür entscheiden…. | Führen Sie anschließend diese Schritte aus… 


 a| 
*Regeln aus bestehendem Ereignisfilter kopieren*
 a| 
. Wählen Sie einen vorhandenen Ereignisfilter aus.
. Die bestehenden Regeln anpassen.
. Weitere Regeln können bei Bedarf durch Klicken auf image:../media/icon_add.gif["Symbol hinzufügen"] hinzugefügt werden.




 a| 
*Neue Regeln hinzufügen*
 a| 
Geben Sie für jede neue Regel den Typ, das Namensmuster, die Schweregrade und den SNMP-Trap-Typ an.

|===


[[edit-an-ems-event-notification-destination]]
== Ein Benachrichtigungsziel für ein EMS-Ereignis bearbeiten

Ab ONTAP 9.10.1 ist es möglich, mit System Manager die Informationen zum Ziel der Ereignisbenachrichtigung zu ändern.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisziele* auswählen.
. Neben dem Namen des Ereignisziels auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Bearbeiten* klicken.
. Die Informationen zum Ereignisziel können geändert und anschließend mit *Speichern* übernommen werden.




[[edit-an-ems-event-notification-filter]]
== Einen EMS Ereignisbenachrichtigungsfilter bearbeiten

Ab ONTAP 9.10.1 kann mit System Manager ein benutzerdefinierter Filter geändert werden, um die Behandlung von Ereignisbenachrichtigungen zu ändern.


NOTE: Systemdefinierte Filter können nicht geändert werden.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisfilter* auswählen.
. Neben dem Namen des Ereignisfilters auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Bearbeiten* klicken.
. Die Informationen zum Ereignisfilter können geändert und anschließend mit *Speichern* übernommen werden.




[[delete-an-ems-event-notification-destination]]
== Löschen eines EMS-Ereignisbenachrichtigungsziels

Ab ONTAP 9.10.1 kann mit System Manager ein EMS-Ereignisbenachrichtigungsziel gelöscht werden.


NOTE: SNMP-Ziele können nicht gelöscht werden.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisziele* auswählen.
. Neben dem Namen des Ereignisziels auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Löschen* klicken.




[[delete-an-ems-event-notification-filter]]
== Löschen eines EMS-Ereignisbenachrichtigungsfilters

Ab ONTAP 9.10.1 ist es möglich, mit System Manager benutzerdefinierte Filter zu löschen.


NOTE: Systemdefinierte Filter können nicht gelöscht werden.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *Benachrichtigungsverwaltung* auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken, dann auf *Ereignisziele anzeigen* klicken.
. Auf der Seite *Benachrichtigungsverwaltung* die Registerkarte *Ereignisfilter* auswählen.
. Neben dem Namen des Ereignisfilters auf image:../media/icon_kabob.gif["Symbol für Menüoptionen"] klicken und dann auf *Löschen* klicken.
