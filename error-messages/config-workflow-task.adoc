= ONTAP EMS Konfigurationsworkflow
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wichtige EMS-Ereignisbenachrichtigungen müssen so konfiguriert werden, dass sie entweder per E-Mail, an einen Syslog-Server, an einen SNMP-Traphost oder an eine Webhook-Anwendung weitergeleitet werden. Dadurch lassen sich Systemunterbrechungen vermeiden, da rechtzeitig Korrekturmaßnahmen ergriffen werden können.

.Über diese Aufgabe
Wenn Ihre Umgebung bereits einen Syslog-Server zum Aggregieren der protokollierten Ereignisse von anderen Systemen wie Servern und Anwendungen enthält, ist es einfacher, diesen Syslog-Server auch für wichtige Ereignisbenachrichtigungen von Speichersystemen zu verwenden.

Falls in Ihrer Umgebung noch kein Syslog-Server vorhanden ist, ist es einfacher, wichtige Ereignisbenachrichtigungen per E-Mail zu erhalten.

Wenn Sie Ereignisbenachrichtigungen bereits an einen SNMP-Traphost weiterleiten, kann es sinnvoll sein, diesen Traphost auf wichtige Ereignisse zu überwachen.

image:ems-config-workflow.png["Arbeitsablauf zur EMS-Konfiguration. Dies zeigt die Schritte zur Konfiguration wichtiger EMS-Ereignisbenachrichtigungen."]
