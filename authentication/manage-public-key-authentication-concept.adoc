= Einem ONTAP Administratorkonto einen öffentlichen Schlüssel zuordnen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für die SSH Public Key Authentifizierung muss der öffentliche Schlüssel einem Administratorkonto zugeordnet werden, bevor das Konto auf die SVM zugreifen kann. Mit dem  `security login publickey create` Befehl kann ein Schlüssel einem Administratorkonto zugeordnet werden.

.Über diese Aufgabe
Wenn ein Konto über SSH sowohl mit einem Passwort als auch mit einem öffentlichen SSH-Schlüssel authentifiziert wird, erfolgt die Authentifizierung des Kontos zuerst mit dem öffentlichen Schlüssel.

.Bevor Sie beginnen
* Der SSH-Schlüssel muss generiert worden sein.
* Sie müssen Cluster- oder SVM-Administrator sein, um diese Aufgabe ausführen zu können.


.Schritte
. Einem Administratorkonto wird ein öffentlicher Schlüssel zugeordnet:
+
`security login publickey create -vserver _SVM_name_ -username _user_name_ -index _index_ -publickey _certificate_ -comment _comment_`

+
Weitere Informationen zu  `security login publickey create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-publickey-create.html["ONTAP-Befehlsreferenz"^].

. Die Änderung kann durch Anzeigen des öffentlichen Schlüssels überprüft werden:
+
`security login publickey show -vserver _SVM_name_ -username _user_name_ -index _index_`

+
Weitere Informationen zu  `security login publickey show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-publickey-show.html["ONTAP-Befehlsreferenz"^].



.Beispiel
Der folgende Befehl verknüpft einen öffentlichen Schlüssel mit dem SVM-Administratorkonto `svmadmin1` für die SVM `engData1`. Dem öffentlichen Schlüssel wird die Indexnummer 5 zugewiesen.

[listing]
----
cluster1::> security login publickey create -vserver engData1 -username svmadmin1 -index 5 -publickey
"<key text>"
----
