= Der TOTP-Geheimschlüssel für ein ONTAP Benutzerkonto wird deaktiviert.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Geht der geheime Schlüssel für das zeitbasierte Einmalpasswort (TOTP) eines lokalen Benutzers verloren, muss der verlorene Schlüssel von einem Speicheradministrator deaktiviert werden, bevor der Benutzer einen neuen TOTP-Schlüssel erstellen kann.

.Über diese Aufgabe
Diese Aufgabe kann nur von einem Cluster-Administratorkonto aus durchgeführt werden.

.Schritt
. TOTP-Geheimschlüssel deaktivieren:
+
[source, cli]
----
security login totp modify -vserver <svm_name> -username <account_username> -enabled false
----
+
Weitere Informationen zu  `security login totp modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-totp-modify.html["ONTAP-Befehlsreferenz"^].
