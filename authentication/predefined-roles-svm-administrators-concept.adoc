= Vordefinierte Rollen für ONTAP SVM Administratoren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die vordefinierten Rollen für SVM-Administratoren sollten die meisten Ihrer Anforderungen abdecken. Bei Bedarf können benutzerdefinierte Rollen erstellt werden. Standardmäßig wird einem SVM-Administrator die vordefinierte `vsadmin` Rolle zugewiesen.

Die folgende Tabelle listet die vordefinierten Rollen für SVM-Administratoren auf:

|===


| Rollenname | Fähigkeiten 


 a| 
vsadmin
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Verwaltung von Volumes, ausgenommen Volume-Verschiebungen
* Verwaltung von Kontingenten, Qtrees, Snapshots und Dateien
* Verwaltung von LUNs
* Ausführen von SnapLock-Operationen, außer privilegiertem Löschen
* Konfiguration von Protokollen: NFS, SMB, iSCSI, FC, FCoE, NVMe/FC und NVMe/TCP
* Konfiguration von Diensten: DNS, LDAP und NIS
* Überwachungsaufträge
* Überwachung von Netzwerkverbindungen und Netzwerkschnittstellen
* Überwachung des Gesundheitszustands der SVM




 a| 
vsadmin-volume
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Verwaltung von Volumes, ausgenommen Volume-Verschiebungen
* Verwaltung von Kontingenten, Qtrees, Snapshots und Dateien
* Verwaltung von LUNs
* Konfiguration von Protokollen: NFS, SMB, iSCSI, FC, FCoE, NVMe/FC und NVMe/TCP
* Konfiguration von Diensten: DNS, LDAP und NIS
* Überwachung der Netzwerkschnittstelle
* Überwachung des Gesundheitszustands der SVM




 a| 
vsadmin-protocol
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Konfiguration von Protokollen: NFS, SMB, iSCSI, FC, FCoE, NVMe/FC und NVMe/TCP
* Konfiguration von Diensten: DNS, LDAP und NIS
* Verwaltung von LUNs
* Überwachung der Netzwerkschnittstelle
* Überwachung des Gesundheitszustands der SVM




 a| 
vsadmin-backup
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Verwaltung von NDMP-Operationen
* Ein wiederhergestelltes Volume für lesen/schreiben aktivieren
* Verwaltung von SnapMirror-Beziehungen und Snapshots
* Anzeige von Volumen und Netzwerkinformationen




 a| 
vsadmin-snaplock
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Verwaltung von Volumes, ausgenommen Volume-Verschiebungen
* Verwaltung von Kontingenten, Qtrees, Snapshots und Dateien
* Durchführen von SnapLock-Operationen, einschließlich privilegiertem Löschen
* Konfiguration von Protokollen: NFS und SMB
* Konfiguration von Diensten: DNS, LDAP und NIS
* Überwachungsaufträge
* Überwachung von Netzwerkverbindungen und Netzwerkschnittstellen




 a| 
vsadmin-readonly
 a| 
* Verwaltung der lokalen Passwort- und Schlüsselinformationen des eigenen Benutzerkontos
* Überwachung des Gesundheitszustands der SVM
* Überwachung der Netzwerkschnittstelle
* Anzeigen von Volumes und LUNs
* Anzeigen von Diensten und Protokollen


|===
