= Die einem ONTAP Administrator zugewiesene Rolle ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den  `security login modify` Befehl verwenden, um die Rolle eines Cluster- oder SVM-Administratorkontos zu ändern. Es kann eine vordefinierte oder eine benutzerdefinierte Rolle zugewiesen werden.

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritt
. Die Rolle eines Cluster- oder SVM-Administrators ändern:
+
`security login modify -vserver SVM_name -user-or-group-name user_or_group_name -application application -authmethod authentication_method -role role -comment comment`

+
link:config-worksheets-reference.html["Erstellen oder Ändern von Anmeldekonten"]

+
Der folgende Befehl ändert die Rolle des AD-Cluster-Administratorkontos `DOMAIN1\guest1` zur vordefinierten `readonly` Rolle.

+
[listing]
----
cluster1::>security login modify -vserver engCluster -user-or-group-name DOMAIN1\guest1 -application ssh -authmethod domain -role readonly
----
+
Der folgende Befehl ändert die Rolle der SVM-Administratorkonten im Active Directory Gruppe Konto `DOMAIN1\adgroup` zur benutzerdefinierten `vol_role` Rolle.

+
[listing]
----
cluster1::>security login modify -vserver engData -user-or-group-name DOMAIN1\adgroup -application ssh -authmethod domain -role vol_role
----
+
Weitere Informationen zu  `security login modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-modify.html["ONTAP-Befehlsreferenz"^].
