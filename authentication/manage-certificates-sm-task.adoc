= ONTAP Zertifikate mit System Manager verwalten
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs:

[role="lead"]
Ab ONTAP 9.10.1 kann mit System Manager die Verwaltung von vertrauenswürdigen Zertifizierungsstellen, Client-/Server-Zertifikaten und lokalen (Onboard) Zertifizierungsstellen erfolgen.

Mit System Manager können Sie die von anderen Anwendungen empfangenen Zertifikate verwalten, sodass die Kommunikation dieser Anwendungen authentifiziert werden kann. Auch die eigenen Zertifikate, die Ihr System gegenüber anderen Anwendungen identifizieren, lassen sich verwalten.



[[view-certificate-information]]
== Zertifikatsinformationen anzeigen

Mit System Manager lassen sich vertrauenswürdige Zertifizierungsstellen, Client-/Serverzertifikate und lokale Zertifizierungsstellen anzeigen, die auf dem Cluster gespeichert sind.

.Schritte
. Im System Manager *Cluster > Einstellungen* auswählen.
. Scrollen Sie zum Bereich *Sicherheit*. Im Abschnitt *Zertifikate* werden folgende Details angezeigt:
+
** Die Anzahl der gespeicherten vertrauenswürdigen Zertifizierungsstellen.
** Die Anzahl der gespeicherten Client-/Serverzertifikate.
** Die Anzahl der gespeicherten lokalen Zertifizierungsstellen.


. Wählen Sie eine beliebige Zahl aus, um Details zu einer Zertifikatskategorie anzuzeigen, oder wählen Sie image:icon_arrow.gif["Pfeilsymbol"] aus, um die Seite *Zertifikate* zu öffnen, die Informationen zu allen Kategorien enthält. Die Liste zeigt die Informationen für den gesamten Cluster an. Um Informationen nur für eine bestimmte Storage-VM anzuzeigen, sind die folgenden Schritte erforderlich:
+
.. Wählen Sie *Speicher > Speicher-VMs*.
.. Die Storage-VM auswählen.
.. Zur Registerkarte *Einstellungen* wechseln.
.. Wählen Sie eine Nummer im Abschnitt *Zertifikat* aus.




.Nächste Schritte
* Auf der Seite *Zertifikate* können Sie <<Generieren einer Certificate Signing Request>>.
* Die Zertifikatsinformationen sind in drei Registerkarten unterteilt, eine für jede Kategorie. Von jeder Registerkarte aus lassen sich die folgenden Aufgaben durchführen:


[cols="30,70"]
|===


| Auf dieser Registerkarte... | Sie können diese Verfahren durchführen... 


 a| 
*Vertrauenswürdige Zertifizierungsstellen*
 a| 
* <<install-trusted-cert>>
* <<Eine vertrauenswürdige Zertifizierungsstelle löschen>>
* <<Eine vertrauenswürdige Zertifizierungsstelle erneuern>>




 a| 
*Client/Server-Zertifikate*
 a| 
* <<install-cs-cert>>
* <<gen-cs-cert>>
* <<delete-cs-cert>>
* <<renew-cs-cert>>




 a| 
*Lokale Zertifizierungsstellen*
 a| 
* <<Eine neue lokale Zertifizierungsstelle erstellen>>
* <<Ein Zertifikat mit einer lokalen Zertifizierungsstelle signieren>>
* <<Lokale Zertifizierungsstelle löschen>>
* <<Eine lokale Zertifizierungsstelle erneuern>>


|===


[[generate-a-certificate-signing-request]]
== Generieren einer Certificate Signing Request

Sie können mit System Manager auf jeder Registerkarte der Seite *Zertifikate* eine Zertifikatsignierungsanforderung (CSR) generieren. Ein privater Schlüssel und eine zugehörige CSR werden generiert, die von einer Zertifizierungsstelle signiert werden können, um ein öffentliches Zertifikat zu erstellen.

.Schritte
. Die Seite *Zertifikate* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. *+CSR generieren* auswählen.
. Vervollständigen Sie die Angaben zum Betreffnamen:
+
.. Einen *allgemeinen Namen* eingeben.
.. Ein *Land* auswählen.
.. Eine *Organisation* eingeben.
.. Eine *Organisationseinheit* eingeben.


. Wenn Sie die Standardeinstellungen überschreiben möchten, wählen Sie *Weitere Optionen* und geben zusätzliche Informationen an.




[[install-add-a-trusted-certificate-authority]]
[[install-trusted-cert]]
== Eine vertrauenswürdige Zertifizierungsstelle installieren (hinzufügen)

Im System Manager lassen sich zusätzliche vertrauenswürdige Zertifizierungsstellen installieren.

.Schritte
. Die Registerkarte *Vertrauenswürdige Zertifizierungsstellen* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. Wählen Sie image:icon_add_blue_bg.gif["Symbol hinzufügen"].
. Im Fenster *Vertrauenswürdige Zertifizierungsstelle hinzufügen* sind die folgenden Schritte auszuführen:
+
** Einen *Namen* eingeben.
** Für den *Bereich* eine Storage-VM auswählen.
** Einen *allgemeinen Namen* eingeben.
** Einen *Typ* auswählen.
** Geben Sie die *Zertifikatsdetails* ein oder importieren Sie diese.






[[delete-a-trusted-certificate-authority]]
== Eine vertrauenswürdige Zertifizierungsstelle löschen

Mit System Manager kann eine vertrauenswürdige Zertifizierungsstelle gelöscht werden.


NOTE: Vorinstallierte vertrauenswürdige Zertifizierungsstellen mit ONTAP können nicht gelöscht werden.

.Schritte
. Die Registerkarte *Vertrauenswürdige Zertifizierungsstellen* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. Wählen Sie den Namen der vertrauenswürdigen Zertifizierungsstelle aus.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen und dann *Löschen* aus.




[[renew-a-trusted-certificate-authority]]
== Eine vertrauenswürdige Zertifizierungsstelle erneuern

Mit System Manager ist es möglich, eine vertrauenswürdige Zertifizierungsstelle zu erneuern, deren Gültigkeit abgelaufen ist oder demnächst abläuft.

.Schritte
. Die Registerkarte *Vertrauenswürdige Zertifizierungsstellen* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. Wählen Sie den Namen der vertrauenswürdigen Zertifizierungsstelle aus.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Zertifikatsnamen dann *Erneuern*.




[[install-add-a-clientserver-certificate]]
[[install-cs-cert]]
== Ein Client-/Serverzertifikat installieren (hinzufügen)

Mit System Manager können zusätzliche Client-/Serverzertifikate installiert werden.

.Schritte
. Die Registerkarte *Client/Serverzertifikate* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. Wählen Sie image:icon_add_blue_bg.gif["Symbol hinzufügen"].
. Im Fenster *Client-/Serverzertifikat hinzufügen* sind die folgenden Schritte auszuführen:
+
** Einen *Zertifikatsnamen* eingeben.
** Für den *Bereich* eine Storage-VM auswählen.
** Einen *allgemeinen Namen* eingeben.
** Einen *Typ* auswählen.
** Geben Sie die *Zertifikatsdetails* ein oder importieren Sie diese. Die Zertifikatsdetails können entweder direkt eingegeben oder aus einer Textdatei kopiert und eingefügt werden. Alternativ lässt sich der Text aus einer Zertifikatsdatei durch Klicken auf *Import* importieren.
** Geben Sie den *privaten Schlüssel* ein. Sie können den privaten Schlüssel entweder direkt eingeben oder aus einer Textdatei kopieren und einfügen oder den Text aus einer privaten Schlüsseldatei importieren, indem Sie auf *Import* klicken.






[[generate-add-a-self-signed-clientserver-certificate]]
== Ein selbstsigniertes Client-/Serverzertifikat generieren (hinzufügen)

Mit System Manager lassen sich zusätzliche selbstsignierte Client/Server-Zertifikate generieren.

.Schritte
. Die Registerkarte *Client/Serverzertifikate* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. *+Selbstsigniertes Zertifikat generieren* auswählen.
. Im Fenster *Selbstsigniertes Zertifikat generieren* Folgendes ausführen:
+
** Einen *Zertifikatsnamen* eingeben.
** Für den *Bereich* eine Storage-VM auswählen.
** Einen *allgemeinen Namen* eingeben.
** Einen *Typ* auswählen.
** Eine *Hashfunktion* auswählen.
** Wählen Sie eine *Schlüssellänge*.
** Eine *Speicher-VM* auswählen.

[[sign-a-certificate-using-a-local-certificate-authority]]
== Ein Zertifikat mit einer lokalen Zertifizierungsstelle signieren

Im System Manager kann eine lokale Zertifizierungsstelle verwendet werden, um ein Zertifikat zu signieren.

.Schritte
. Den Tab *Lokale Zertifizierungsstellen* anzeigen. Siehe <<Zertifikatsinformationen anzeigen>>.
. Den Namen der lokalen Zertifizierungsstelle auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen dann *Zertifikat signieren*.
. Das Formular *Antrag auf Unterzeichnung eines Zertifikats* ausfüllen.
+
** Sie können entweder den Inhalt der Zertifikatsignierung einfügen oder eine Zertifikatsignierungsanforderungsdatei durch Klicken auf *Import* importieren.
** Geben Sie die Anzahl der Tage an, für die das Zertifikat gültig ist.
