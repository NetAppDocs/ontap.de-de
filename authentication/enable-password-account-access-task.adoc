= Zugriff auf das ONTAP-Kontopasswort aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den  `security login create`Befehl verwenden, um Administratorkonten den Zugriff auf eine Admin- oder Daten-SVM mit einem Passwort zu ermöglichen. Nach Eingabe des Befehls erfolgt eine Aufforderung zur Eingabe des Passworts.

.Über diese Aufgabe
Wenn Sie sich nicht sicher sind, welche Zugriffskontrollrolle Sie dem Anmeldekonto zuweisen möchten, kann die Rolle später mit dem  `security login modify` Befehl hinzugefügt werden.

Weitere Informationen zu  `security login modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-modify.html["ONTAP-Befehlsreferenz"^].

.Bevor Sie beginnen
Sie müssen Cluster-Administrator sein, um diese Aufgabe ausführen zu können.

.Schritt
. Lokalen Administratorkonten den Zugriff auf eine SVM mithilfe eines Kennworts ermöglichen:
+
`security login create -vserver _SVM_name_ -user-or-group-name _user_or_group_name_ -application _application_ -authmethod _authentication_method_ -role _role_ -comment _comment_`

+
Der folgende Befehl ermöglicht dem Cluster-Administratorkonto `admin1` mit der vordefinierten `backup` Rolle den Zugriff auf die Admin-SVM``engCluster`` mit einem Passwort. Nach Eingabe des Befehls erfolgt eine Aufforderung zur Eingabe des Passworts.

+
[listing]
----
cluster1::>security login create -vserver engCluster -user-or-group-name admin1 -application ssh -authmethod password -role backup
----
+
Weitere Informationen zu  `security login create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-create.html["ONTAP-Befehlsreferenz"^].
