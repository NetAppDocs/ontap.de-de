= Informationen zur ONTAP Authentifizierung für NAS-Clients
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NFS-Clients müssen ordnungsgemäß authentifiziert sein, bevor sie auf Daten auf der SVM zugreifen können. ONTAP authentifiziert die Clients, indem die UNIX-Anmeldeinformationen mit den von Ihnen konfigurierten Namensdiensten abgeglichen werden.

Wenn sich ein NFS-Client mit der SVM verbindet, ermittelt ONTAP die UNIX-Anmeldeinformationen des Benutzers, indem verschiedene Namensdienste überprüft werden, abhängig von der Namensdienstkonfiguration der SVM. ONTAP kann Anmeldeinformationen für lokale UNIX-Konten, NIS-Domänen und LDAP-Domänen prüfen. Mindestens einer dieser Dienste muss konfiguriert sein, damit ONTAP den Benutzer erfolgreich authentifizieren kann. Es kann festgelegt werden, dass mehrere Namensdienste sowie die Reihenfolge, in der ONTAP diese durchsucht, verwendet werden.

In einer reinen NFS-Umgebung mit UNIX-Volume-Sicherheitsstilen ist diese Konfiguration ausreichend, um die Authentifizierung durchzuführen und den richtigen Dateizugriff für einen Benutzer bereitzustellen, der sich von einem NFS-Client verbindet.

Wenn gemischte, NTFS- oder einheitliche Volume-Sicherheitsstile verwendet werden, muss ONTAP einen SMB-Benutzernamen für den UNIX-Benutzer zur Authentifizierung bei einem Windows-Domänencontroller erhalten. Dies kann entweder durch die Zuordnung einzelner Benutzer mithilfe lokaler UNIX-Konten oder LDAP-Domänen oder durch die Verwendung eines Standard-SMB-Benutzers erfolgen. Es kann festgelegt werden, in welcher Reihenfolge ONTAP die Namensdienste durchsucht, oder ein Standard-SMB-Benutzer kann angegeben werden.
