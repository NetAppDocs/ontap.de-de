= Informationen zu Sperren für ONTAP NFS SVMs anzeigen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Informationen über die aktuellen Dateisperren können angezeigt werden, einschließlich der Art der gehaltenen Sperren und des Sperrstatus, Details zu Bytebereichssperren, Sharelock-Modi, Delegation Locks und Opportunistic Locks sowie darüber, ob Sperren mit dauerhaften oder persistenten Handles geöffnet sind.

.Über diese Aufgabe
Die Client-IP-Adresse kann für Sperren, die über NFSv4 oder NFSv4.1 eingerichtet wurden, nicht angezeigt werden.

Standardmäßig zeigt der Befehl Informationen zu allen Sperren an. Mit Befehlsparametern lässt sich die Anzeige von Informationen zu Sperren für eine bestimmte Storage Virtual Machine (SVM) steuern oder die Ausgabe des Befehls nach anderen Kriterien filtern.

Der  `vserver locks show` Befehl zeigt Informationen über vier Arten von Sperren an:

* Byte-Range-Sperren, die nur einen Teil einer Datei sperren.
* Freigabesperren, die geöffnete Dateien sperren.
* Opportunistische Sperren, die das Client-seitige Caching über SMB steuern.
* Delegationen, die das Client-seitige Caching über NFSv4.x steuern.


Durch die Angabe optionaler Parameter lassen sich wichtige Informationen zu jedem Sperrtyp ermitteln. Weitere Informationen zu  `vserver locks show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-locks-show.html["ONTAP-Befehlsreferenz"^].

.Schritt
. Informationen zu Sperren werden mit dem  `vserver locks show` Befehl angezeigt.


.Beispiele
Das folgende Beispiel zeigt zusammenfassende Informationen zu einer NFSv4-Sperre für eine Datei mit dem Pfad `/vol1/file1`. Der Zugriffsmodus von Sharelock ist write-deny_none, und die Sperre wurde mit Schreibdelegation gewährt:

[listing]
----
cluster1::> vserver locks show

Vserver: vs0
Volume  Object Path               LIF         Protocol  Lock Type   Client
------- ------------------------- ----------- --------- ----------- -------
vol1    /vol1/file1               lif1        nfsv4     share-level -
               Sharelock Mode: write-deny_none
                                                        delegation  -
               Delegation Type: write
----
