= Informationen zur ONTAP Unterstützung für NFSv4.1
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.8 ist die nconnect-Funktionalität standardmäßig verfügbar, wenn NFSv4.1 aktiviert ist.

Frühere NFS-Client-Implementierungen verwenden lediglich eine einzige TCP-Verbindung mit einem Mount. In ONTAP kann eine einzelne TCP-Verbindung bei steigenden IOPS zu einem Engpass werden.

nconnect verbessert die Leistung von NFS-Clients, indem mehrere TCP-Verbindungen (bis zu 16) für eine einzelne Einbindung ermöglicht werden, was dazu beiträgt, den Performance-Engpass zu überwinden, der bei einer einzelnen TCP-Verbindung mit zunehmender IOPS auftreten kann.

NFSv4.1 ist in ONTAP 9.9.1 und höher standardmäßig aktiviert. In früheren Versionen kann es durch Angabe der  `-v4.1` Option und Setzen auf  `enabled` beim Erstellen eines NFS-Servers auf der Storage Virtual Machine (SVM) aktiviert werden.

ONTAP unterstützt keine NFSv4.1-Delegierungen auf Verzeichnis- und Dateiebene.
