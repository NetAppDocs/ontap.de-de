= Informationen zu Leistungsverbesserungen bei NFSv3 und NFSv4 sowie zur TCP-Übertragungsgröße für ONTAP SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Leistung von NFSv3- und NFSv4-Clients, die über ein Netzwerk mit hoher Latenz eine Verbindung zu Speichersystemen herstellen, kann durch Ändern der maximalen TCP-Übertragungsgröße verbessert werden.

Wenn Clients über ein Netzwerk mit hoher Latenz, wie zum Beispiel ein Weitverkehrsnetz (WAN) oder ein Metropolnetz (MAN) mit einer Latenz von über 10 Millisekunden, auf Speichersysteme zugreifen, kann die Verbindungsleistung durch Anpassen der maximalen TCP-Übertragungsgröße möglicherweise verbessert werden. Clients, die in einem Netzwerk mit niedriger Latenz, wie einem lokalen Netzwerk (LAN), auf Speichersysteme zugreifen, profitieren in der Regel kaum oder gar nicht von der Änderung dieser Parameter. Überwiegt die Verbesserung des Durchsatzes die Auswirkungen auf die Latenz nicht, empfiehlt sich die Verwendung dieser Parameter nicht.

Um festzustellen, ob Ihre Speicherumgebung von einer Anpassung dieser Parameter profitieren würde, sollte zunächst eine umfassende Leistungsbewertung eines leistungsschwachen NFS-Client durchgeführt werden. Es ist zu prüfen, ob die geringe Leistung auf eine übermäßige Roundtrip-Latenz und kleine Anfragen auf dem Client zurückzuführen ist. Unter diesen Bedingungen können der Client und der Server die verfügbare Bandbreite nicht vollständig nutzen, da sie den Großteil ihrer Arbeitszyklen mit dem Warten auf die Übertragung kleiner Anfragen und Antworten verbringen.

Durch die Erhöhung der NFSv3- und NFSv4-Anforderungsgröße können Client und Server die verfügbare Bandbreite effektiver nutzen, um mehr Daten pro Zeiteinheit zu übertragen, wodurch die Gesamteffizienz der Verbindung steigt.

Beachten Sie, dass die Konfiguration zwischen Speichersystem und Client variieren kann. Das Speichersystem und der Client unterstützen eine maximale Übertragungsgröße von 1 MB für Übertragungsvorgänge. Wenn das Speichersystem jedoch für eine maximale Übertragungsgröße von 1 MB konfiguriert ist, der Client aber nur 64 KB unterstützt, ist die Übertragungsgröße beim Mounten auf 64 KB oder weniger begrenzt.

Vor einer Änderung dieser Parameter ist zu beachten, dass dies zu einem zusätzlichen Speicherverbrauch im Speichersystem für die Zeit führt, die zum Zusammenstellen und Übertragen einer großen Antwort erforderlich ist. Je mehr Verbindungen mit hoher Latenz zum Speichersystem bestehen, desto höher ist der zusätzliche Speicherverbrauch. Speichersysteme mit hoher Speicherkapazität sind von dieser Änderung möglicherweise kaum betroffen. Speichersysteme mit geringer Speicherkapazität können hingegen eine spürbare Verschlechterung der Performance erfahren.

Die erfolgreiche Nutzung dieser Parameter setzt die Fähigkeit voraus, Daten von mehreren Knoten eines Clusters abzurufen. Die systembedingte Latenz des Clusternetzwerks kann die Gesamtlatenz der Antwort erhöhen. Die Gesamtlatenz steigt tendenziell bei Verwendung dieser Parameter. Daher kann sich dies negativ auf latenzempfindliche Workloads auswirken.
