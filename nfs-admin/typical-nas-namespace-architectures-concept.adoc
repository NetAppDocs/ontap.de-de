= Informationen zu den ONTAP NAS Namespace-Architekturen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es gibt mehrere typische NAS-Namespace-Architekturen, die beim Erstellen des SVM-Namespace verwendet werden können. Die Namespace-Architektur kann entsprechend den Geschäfts- und Workflow-Anforderungen ausgewählt werden.

Die oberste Ebene des Namensraums ist stets das Root-Volume, das durch einen Schrägstrich (/) dargestellt wird. Die Namensraumarchitektur unterhalb des Root-Volumes fällt in drei grundlegende Kategorien:

* Ein einzelner verzweigter Baum mit nur einer einzigen Verbindung zur Wurzel des Namensraums
* Mehrfach verzweigte Bäume mit mehreren Verbindungspunkten zur Wurzel des Namensraums
* Mehrere Standalone Volumes, jedes mit einem separaten Verbindungspunkt zum Stamm des Namensraums




[[namespace-with-single-branched-tree]]
== Namensraum mit einem einzigen verzweigten Baum

Eine Architektur mit einem einzigen verzweigten Baum besitzt einen einzigen Einfügepunkt im Stammverzeichnis des SVM-Namensraums. Der einzelne Einfügepunkt kann entweder ein verbundenes Volume oder ein Verzeichnis unterhalb des Stammverzeichnisses sein. Alle anderen Volumes werden an Verbindungspunkten unterhalb des einzelnen Einfügepunkts (der ein Volume oder ein Verzeichnis sein kann) eingebunden.

image:namespace-architecture-with-single-branched-tree.gif["Namensraum mit einem einzigen verzweigten Baum"]

Eine typische Volume Verbindungskonfiguration mit der oben genannten Namespace-Architektur könnte beispielsweise wie folgt aussehen, wobei alle Volumes unterhalb des einzigen Einfügepunkts, einem Verzeichnis namens "`data`", verbunden sind:

[listing]
----

                     Junction                       Junction
Vserver Volume       Active   Junction Path         Path Source
------- ------------ -------- -------------------   -----------
vs1     corp1        true     /data/dir1/corp1      RW_volume
vs1     corp2        true     /data/dir1/corp2      RW_volume
vs1     data1        true     /data/data1           RW_volume
vs1     eng1         true     /data/data1/eng1      RW_volume
vs1     eng2         true     /data/data1/eng2      RW_volume
vs1     sales        true     /data/data1/sales     RW_volume
vs1     vol1         true     /data/vol1            RW_volume
vs1     vol2         true     /data/vol2            RW_volume
vs1     vol3         true     /data/vol3            RW_volume
vs1     vs1_root     -        /                     -
----


[[namespace-with-multiple-branched-trees]]
== Namensraum mit mehreren verzweigten Bäumen

Eine Architektur mit mehreren verzweigten Bäumen verfügt über mehrere Einfügepunkte zum Stamm des SVM-Namensraums. Die Einfügepunkte können entweder Verbindung Volumes oder Verzeichnisse unterhalb des Stamms sein. Alle anderen Volumes werden an Verbindungspunkten unterhalb der Einfügepunkte (die Volumes oder Verzeichnisse sein können) eingebunden.

image:namespace-architecture-with-multiple-branched-trees.png["Namensraum mit mehreren verzweigten Bäumen"]

Eine typische Volume Verbindungskonfiguration mit der oben beschriebenen Namespace-Architektur könnte beispielsweise wie folgt aussehen, wobei es drei Einfügepunkte zum Root-Volume der SVM gibt. Zwei Einfügepunkte sind Verzeichnisse mit den Namen „`data`“ und „`projects`“. Ein Einfügepunkt ist ein verbundener Volume mit dem Namen „`audit`“:

[listing]
----

                     Junction                       Junction
Vserver Volume       Active   Junction Path         Path Source
------- ------------ -------- -------------------   -----------
vs1     audit        true     /audit                RW_volume
vs1     audit_logs1  true     /audit/logs1          RW_volume
vs1     audit_logs2  true     /audit/logs2          RW_volume
vs1     audit_logs3  true     /audit/logs3          RW_volume
vs1     eng          true     /data/eng             RW_volume
vs1     mktg1        true     /data/mktg1           RW_volume
vs1     mktg2        true     /data/mktg2           RW_volume
vs1     project1     true     /projects/project1    RW_volume
vs1     project2     true     /projects/project2    RW_volume
vs1     vs1_root     -        /                     -
----


[[namespace-with-multiple-stand-alone-volumes]]
== Namensraum mit mehreren Standalone Volumes

In einer Architektur mit Standalone Volumes hat jedes Volume einen Einfügepunkt im Stammverzeichnis des SVM-Namensraums, das Volume ist jedoch nicht unter einem anderen Volume verbunden. Jedes Volume hat einen eindeutigen Pfad und ist entweder direkt unter dem Stammverzeichnis verbunden oder unter einem Verzeichnis unterhalb des Stammverzeichnisses verbunden.

image:namespace-architecture-with-multiple-standalone-volumes.gif["Namensraum mit mehreren Standalone Volumes"]

Eine typische Volume Verbindungskonfiguration mit der oben genannten Namespace-Architektur könnte beispielsweise wie folgt aussehen, wobei es fünf Einfügepunkte zum Root-Volume der SVM gibt und jeder Einfügepunkt einen Pfad zu einem Volume darstellt.

[listing]
----

                     Junction                       Junction
Vserver Volume       Active   Junction Path         Path Source
------- ------------ -------- -------------------   -----------
vs1     eng          true     /eng                  RW_volume
vs1     mktg         true     /vol/mktg             RW_volume
vs1     project1     true     /project1             RW_volume
vs1     project2     true     /project2             RW_volume
vs1     sales        true     /sales                RW_volume
vs1     vs1_root     -        /                     -
----
