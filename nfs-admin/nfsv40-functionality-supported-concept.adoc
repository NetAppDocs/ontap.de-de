= Informationen zur ONTAP Unterstützung für die NFSv4.0-Funktionalität
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt alle obligatorischen Funktionen in NFSv4.0, mit Ausnahme der Sicherheitsmechanismen SPKM3 und LIPKEY.

*EMPFEHLUNG:* NFSv4.1 oder NFSv4.2 sollten anstelle von NFSv4.0 verwendet werden. NFSv4.0 ist eine ältere Version des NFS-Protokolls und wird von der NFS-Community nicht mehr aktiv weiterentwickelt.

Folgende NFSV4-Funktionalität wird unterstützt:

* *VERBINDUNG*
+
Ermöglicht einem Client, mehrere Dateioperationen in einer einzigen Remote Procedure Call (RPC)-Anfrage anzufordern.

* *Dateidelegierung*
+
Ermöglicht dem Server, die Dateikontrolle an bestimmte Clienttypen für Lese- und Schreibzugriff zu delegieren.

* *Pseudo-FS*
+
Wird von NFSv4-Servern verwendet, um Mountpunkte auf dem Speichersystem zu ermitteln. Es gibt kein Mount-Protokoll in NFSv4.

* *Sperren*
+
Mietbasiert. In NFSv4 gibt es keine separaten Protokolle für Network Lock Manager (NLM) oder Network Status Monitor (NSM).



Weitere Informationen zum NFSv4.0-Protokoll enthält RFC 3530.
