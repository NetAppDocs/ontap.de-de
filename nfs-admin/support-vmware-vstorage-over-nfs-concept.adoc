= Informationen zur Unterstützung von VMware vStorage über ONTAP NFS
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt bestimmte VMware vStorage APIs für Array Integration (VAAI)-Funktionen in einer NFS-Umgebung.



[[supported-features]]
== Unterstützte Funktionen

Folgende Funktionen werden unterstützt:

* Copy Offload
+
Ermöglicht einem ESXi-Host, virtuelle Maschinen oder virtuelle Festplatten (VMDKs) direkt zwischen Quell- und Ziel-Datenspeicherort zu kopieren, ohne den Host einzubeziehen. Dadurch werden ESXi-Host-CPU-Zyklen und Netzwerkbandbreite eingespart. Copy Offload erhält die Speicherplatzeffizienz, wenn das Quellvolume dünn bereitgestellt ist.

* Platzreservierung
+
Gewährleistet Speicherplatz für eine VMDK-Datei, indem Speicherplatz dafür reserviert wird.





[[limitations]]
== Einschränkungen

VMware vStorage over NFS weist folgende Einschränkungen auf:

* Copy-Offload-Vorgänge können in folgenden Szenarien fehlschlagen:
+
** Während der Ausführung von wafliron auf dem Quell- oder Zielvolume, da das Volume dadurch vorübergehend offline genommen wird
** Beim Verschieben des Quell- oder Ziel-Volumes
** Beim Verschieben des Quell- oder Ziel-LIF
** Während der Durchführung von Takeover- oder Giveback-Operationen
** Während der Durchführung von Umschalt- oder Rückschaltvorgängen


* Serverseitiges Kopieren kann aufgrund von Unterschieden im Dateihandle-Format im folgenden Szenario fehlschlagen:
+
Sie versuchen, Daten von SVMs, die aktuell oder zuvor Qtrees exportiert haben, auf SVMs zu kopieren, die noch nie Qtrees exportiert haben. Um diese Einschränkung zu umgehen, ist es möglich, mindestens einen Qtree auf der Ziel SVM zu exportieren.
