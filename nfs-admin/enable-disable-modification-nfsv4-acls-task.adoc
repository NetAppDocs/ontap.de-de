= Aktivieren oder Deaktivieren der NFSv4-ACL-Änderung für ONTAP SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ONTAP einen  `chmod` Befehl für eine Datei oder ein Verzeichnis mit einer ACL empfängt, wird die ACL standardmäßig beibehalten und entsprechend der Änderung des Modusbits angepasst. Es besteht die Möglichkeit, den  `-v4-acl-preserve` Parameter zu deaktivieren, wodurch das Verhalten geändert wird, sodass die ACL stattdessen entfernt wird.

.Über diese Aufgabe
Bei Verwendung des einheitlichen Sicherheitsstils legt dieser Parameter auch fest, ob die NTFS-Dateiberechtigungen erhalten bleiben oder verworfen werden, wenn ein Client einen chmod, chgroup oder chown Befehl für eine Datei oder ein Verzeichnis sendet.

Dieser Parameter ist standardmäßig aktiviert.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Führen Sie eine der folgenden Aktionen aus:
+
[cols="35,65"]
|===


| Wenn Sie möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
Beibehaltung und Änderung bestehender NFSv4-ACLs (Standard) ermöglichen
 a| 
`vserver nfs modify -vserver vserver_name -v4-acl-preserve enabled`



 a| 
Beibehaltung deaktivieren und NFSv4-ACLs beim Ändern der Modusbits entfernen
 a| 
`vserver nfs modify -vserver vserver_name -v4-acl-preserve disabled`

|===
. Zurück zur Administrator-Privilegstufe:
+
`set -privilege admin`
