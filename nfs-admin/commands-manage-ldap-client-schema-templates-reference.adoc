= ONTAP Befehle zur Verwaltung von NFS LDAP-Client-Schemavorlagen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für die Verwaltung von LDAP-Client-Schemavorlagen gibt es spezielle ONTAP-Befehle.

[NOTE]
====
SVM-Administratoren können von Cluster-Administratoren erstellte LDAP-Client-Schemas weder ändern noch löschen.

====
[cols="35,65"]
|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Eine vorhandene LDAP-Schemavorlage kopieren
 a| 
`vserver services name-service ldap client schema copy` Verfügbar ab der Berechtigungsstufe „Erweitert“ und höher.



 a| 
LDAP-Schemavorlagen anzeigen
 a| 
`vserver services name-service ldap client schema show`



 a| 
Eine LDAP-Schema-Vorlage ändern
 a| 
`vserver services name-service ldap client schema modify` Verfügbar ab der Berechtigungsstufe „Erweitert“ und höher.



 a| 
Löschen einer LDAP-Schema-Vorlage
 a| 
`vserver services name-service ldap client schema delete` Verfügbar ab der Berechtigungsstufe „Erweitert“ und höher.

|===
Weitere Informationen zu  `vserver services name-service ldap client schema` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+services+name-service+ldap+client+schema["ONTAP-Befehlsreferenz"^].
