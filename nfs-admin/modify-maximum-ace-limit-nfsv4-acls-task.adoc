= Ändern des maximalen ACE-Limits für NFSv4-ACLs für ONTAP SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die maximale Anzahl zulässiger ACEs für jede NFSv4-ACL durch Ändern des Parameters  `-v4-acl-max-aces` anpassen. Standardmäßig ist das Limit auf 400 ACEs pro ACL festgelegt. Eine Erhöhung dieses Limits kann dazu beitragen, die erfolgreiche Migration von Daten mit ACLs, die mehr als 400 ACEs enthalten, auf Speichersysteme mit ONTAP sicherzustellen.

.Über diese Aufgabe
Eine Erhöhung dieses Grenzwerts könnte die Leistung von Clients beeinträchtigen, die auf Dateien mit NFSv4-ACLs zugreifen.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Das maximale ACE-Limit für NFSv4-ACLs ändern:
+
`vserver nfs modify -v4-acl-max-aces max_ace_limit`

+
Der gültige Bereich von

+
`max_ace_limit` ist `192` zu `1024.`

. Zurück zur Administrator-Privilegstufe:
+
`set -privilege admin`
