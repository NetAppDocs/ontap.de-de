= Export-Policy-Caches für ONTAP NAS SVMs leeren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP verwendet mehrere Exportregel-Caches, um Informationen zu Exportregeln für einen schnelleren Zugriff zu speichern. Das manuelle Leeren der Exportregel-Caches (`vserver export-policy cache flush` entfernt möglicherweise veraltete Informationen und zwingt ONTAP, aktuelle Informationen aus den entsprechenden externen Ressourcen abzurufen. Dies kann bei der Lösung verschiedener Probleme im Zusammenhang mit dem Zugriff von Clients auf NFS-Exporte helfen.

.Über diese Aufgabe
Die Informationen im Exportregel-Cache könnten aus folgenden Gründen veraltet sein:

* Eine kürzlich erfolgte Änderung der Exportregel
* Eine kürzlich erfolgte Änderung an den Hostname-Einträgen in den Nameservern
* Eine kürzlich erfolgte Änderung an den Netgroup-Einträgen in den Name Servern
* Wiederherstellung nach einem Netzwerkausfall, der das vollständige Laden von Netgroups verhinderte


.Schritte
. Falls der Namensdienstcache nicht aktiviert ist, kann eine der folgenden Aktionen im erweiterten Berechtigungsmodus durchgeführt werden:
+
[cols="2*"]
|===
| Wenn ein Flush durchgeführt werden soll... | Geben Sie den Befehl ein... 


 a| 
Alle Export-Policy-Caches (außer showmount)
 a| 
`vserver export-policy cache flush -vserver vserver_name`



 a| 
Der Exportregel-Zugriffs-Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache access` Sie können den optionalen  `-node` Parameter angeben, um den Knoten zu bestimmen, auf dem der Zugriffscache geleert werden soll.



 a| 
Der Hostname-Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache host`



 a| 
Der Netgroup-Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache netgroup`Die Verarbeitung von Netzgruppen ist ressourcenintensiv. Der Netzgruppen-Cache sollte nur geleert werden, wenn ein Clientzugriffsproblem durch eine veraltete Netzgruppe verursacht wird.



 a| 
Der Showmount Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache showmount`

|===
. Wenn der Namensdienstcache aktiviert ist, kann eine der folgenden Aktionen durchgeführt werden:
+
[cols="2*"]
|===
| Wenn ein Flush durchgeführt werden soll... | Geben Sie den Befehl ein... 


 a| 
Der Exportregel-Zugriffs-Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache access` Sie können den optionalen  `-node` Parameter angeben, um den Knoten zu bestimmen, auf dem der Zugriffscache geleert werden soll.



 a| 
Der Hostname-Cache
 a| 
`vserver services name-service cache hosts forward-lookup delete-all`



 a| 
Der Netgroup-Cache
 a| 
`vserver services name-service cache netgroups ip-to-netgroup delete-all` `vserver services name-service cache netgroups members delete-all`Die Verarbeitung von Netzgruppen ist ressourcenintensiv. Der Netzgruppen-Cache sollte nur geleert werden, wenn ein Clientzugriffsproblem durch eine veraltete Netzgruppe verursacht wird.



 a| 
Der Showmount Cache
 a| 
`vserver export-policy cache flush -vserver vserver_name -cache showmount`

|===
