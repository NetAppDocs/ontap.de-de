= Die maximale TCP-Übertragungsgröße für NFSv3 und NFSv4 bei ONTAP SVMs anpassen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die  `-tcp-max-xfer-size` Option ändern, um die maximalen Übertragungsgrößen für alle TCP-Verbindungen mit den Protokollen NFSv3 und NFSv4.x zu konfigurieren.

.Über diese Aufgabe
Sie können diese Optionen für jede Storage Virtual Machine (SVM) einzeln anpassen.

Ab ONTAP 9 sind die Optionen  `v3-tcp-max-read-size` und  `v3-tcp-max-write-size` veraltet. Stattdessen ist die Option  `-tcp-max-xfer-size` zu verwenden.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Führen Sie eine der folgenden Aktionen aus:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Geben Sie den Befehl ein... 


 a| 
Die maximale Übertragungsgröße für NFSv3 oder NFSv4 TCP ändern
 a| 
`vserver nfs modify -vserver vserver_name -tcp-max-xfer-size integer_max_xfer_size`

|===
+
[cols="3*"]
|===
| Option | Bereich | Standard 


 a| 
`-tcp-max-xfer-size`
 a| 
8192 bis 1048576 Bytes
 a| 
65536 Bytes

|===
+
[NOTE]
====
Die maximal einzugebende Übertragungsgröße muss ein Vielfaches von 4 KB (4096 Byte) sein. Nicht korrekt ausgerichtete Anfragen beeinträchtigen die Leistung.

====
. Mit dem  `vserver nfs show -fields tcp-max-xfer-size` Befehl lassen sich die Änderungen überprüfen.
. Falls Clients statische Mounts verwenden, müssen diese ausgehängt und anschließend erneut eingehängt werden, damit die neue Parametergröße wirksam wird.


.Beispiel
Der folgende Befehl legt die maximale TCP-Übertragungsgröße für NFSv3 und NFSv4.x auf 1048576 Bytes auf der SVM mit dem Namen vs1 fest:

[listing]
----
vs1::> vserver nfs modify -vserver vs1 -tcp-max-xfer-size 1048576
----
