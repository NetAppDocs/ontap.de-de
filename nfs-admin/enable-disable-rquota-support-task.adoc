= rquota-Unterstützung auf ONTAP NFS SVMs aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Remote Quota Protocol (rquota) ermöglicht es NFS-Clients, Kontingentinformationen für Benutzer von einem entfernten Rechner abzurufen. Die Unterstützung für rquota-Versionen variiert je nach Ihrer ONTAP-Version.

* rquota v1 wird in ONTAP 9 und höher unterstützt.
* rquota v2 wird in ONTAP 9.12.1 und später unterstützt.
+
Wenn Sie von rquota v1 auf rquota v2 aktualisieren, kann es zu einer unerwarteten Änderung Ihres Benutzerkontingents kommen. Diese Änderung ergibt sich aus dem Unterschied in der Berechnung des Kontingents zwischen rquota v1 und rquota v2. Weitere Informationen sind in der link:https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/Why_did_the_user_quota_limit_changed_unexpectedly["NetApp Knowledge Base: Warum hat sich das Benutzerkontingent unerwartet geändert?"^] zu finden.



.Über diese Aufgabe
Standardmäßig ist rquota deaktiviert.

.Schritt
. rquota aktivieren oder deaktivieren:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Führen Sie den folgenden Befehl aus... 


 a| 
rquota-Unterstützung für SVMs aktivieren
 a| 
[source, cli]
----
vserver nfs modify -vserver vserver_name -rquota enable
----


 a| 
rquota-Unterstützung für SVMs deaktivieren
 a| 
[source, cli]
----
vserver nfs modify -vserver vserver_name -rquota disable
----
|===
+
Weitere Informationen zu Quoten sind unter link:../volumes/index.html["Logische Speicherverwaltung"] zu finden.
