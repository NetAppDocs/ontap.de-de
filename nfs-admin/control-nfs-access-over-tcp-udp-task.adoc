= NFS-Zugriff über TCP und UDP für ONTAP SVMs steuern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den NFS-Zugriff auf Storage Virtual Machines (SVMs) über TCP und UDP aktivieren oder deaktivieren, indem Sie die  `-tcp`- bzw.  `-udp`-Parameter entsprechend ändern. So lässt sich steuern, ob NFS-Clients in Ihrer Umgebung über TCP oder UDP auf Daten zugreifen können.

.Über diese Aufgabe
Diese Parameter gelten nur für NFS. Sie haben keinen Einfluss auf Hilfsprotokolle. Wenn beispielsweise NFS über TCP deaktiviert ist, sind Mount-Vorgänge über TCP weiterhin erfolgreich. Um TCP- oder UDP-Datenverkehr vollständig zu blockieren, können Exportrichtlinienregeln verwendet werden.

[NOTE]
====
Sie müssen den SnapDiff RPC Server deaktivieren, bevor Sie TCP für NFS deaktivieren, um einen Befehlsfehler zu vermeiden. TCP kann mit dem Befehl  `vserver snapdiff-rpc-server off -vserver vserver name` deaktiviert werden.

====
.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="30,70"]
|===


| Wenn NFS-Zugriff verfügbar sein soll... | Geben Sie den Befehl ein... 


 a| 
Aktiviert über TCP
 a| 
`vserver nfs modify -vserver vserver_name -tcp enabled`



 a| 
Deaktiviert über TCP
 a| 
`vserver nfs modify -vserver vserver_name -tcp disabled`



 a| 
Aktiviert über UDP
 a| 
`vserver nfs modify -vserver vserver_name -udp enabled`



 a| 
Deaktiviert über UDP
 a| 
`vserver nfs modify -vserver vserver_name -udp disabled`

|===
