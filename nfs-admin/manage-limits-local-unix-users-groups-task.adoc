= Grenzwerte für lokale UNIX-Benutzer und -Gruppen für ONTAP NFS SVMs verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für die Verwaltung von Limits für lokale UNIX-Benutzer und -Gruppen stehen spezielle ONTAP Befehle zur Verfügung. Cluster-Administratoren können diese Befehle nutzen, um Leistungsprobleme im Cluster zu untersuchen, die vermutlich mit einer zu hohen Anzahl lokaler UNIX-Benutzer und -Gruppen zusammenhängen.

.Über diese Aufgabe
Diese Befehle stehen dem Cluster-Administrator auf der erweiterten Berechtigungsstufe zur Verfügung.

.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Verwenden Sie den Befehl... 


 a| 
Informationen zu lokalen UNIX-Benutzerlimits anzeigen
 a| 
`vserver services unix-user max-limit show`



 a| 
Informationen zu lokalen UNIX-Gruppenbeschränkungen
 a| 
`vserver services unix-group max-limit show`



 a| 
Lokale UNIX-Benutzerlimits anpassen
 a| 
`vserver services unix-user max-limit modify`



 a| 
Lokale UNIX-Gruppenlimits ändern
 a| 
`vserver services unix-group max-limit modify`

|===
+
Weitere Informationen zu  `vserver services unix` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+services+unix["ONTAP-Befehlsreferenz"^].
