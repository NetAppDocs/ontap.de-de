= NFSv4 Schreibdelegationen für ONTAP SVMs aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um Schreibdelegationen für Dateien zu aktivieren oder zu deaktivieren, kann die ``-v4.0-write-delegation``oder Option angepasst werden. Durch das Aktivieren von Schreibdelegationen für Dateien wird ein Großteil des Nachrichtenaufwands vermieden, der mit Datei- und Datensatzsperren sowie dem Öffnen und Schließen von Dateien verbunden ist.

.Über diese Aufgabe
Standardmäßig sind Schreibdelegationen deaktiviert.

Der Nachteil der Aktivierung von Schreibdelegationen besteht darin, dass der Server und seine Clients zusätzliche Aufgaben ausführen müssen, um die Delegationen nach einem Neustart oder Reboot des Servers, eines Clients oder bei einer Netzwerkpartition wiederherzustellen.

.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Dann... 


 a| 
NFSv4 Schreibdateidelegierungen aktivieren
 a| 
Geben Sie den folgenden Befehl ein: `vserver nfs modify -vserver vserver_name -v4.0-write-delegation enabled`



 a| 
NFSv4.1 Schreibdateidelegierungen aktivieren
 a| 
Geben Sie den folgenden Befehl ein: `vserver nfs modify -vserver vserver_name -v4.1-write-delegation enabled`



 a| 
Deaktivieren von NFSv4 schreiben Datei-Delegationen
 a| 
Geben Sie den folgenden Befehl ein: `vserver nfs modify -vserver vserver_name -v4.0-write-delegation disabled`



 a| 
Deaktivieren von NFSv4.1 schreiben Datei-Delegationen
 a| 
Geben Sie den folgenden Befehl ein: `vserver nfs modify -vserver vserver_name -v4.1-write-delegation disabled`

|===


.Ergebnis
Die Dateidelegierungsoptionen werden sofort nach ihrer Änderung wirksam. Ein Neustart oder das erneute Starten von NFS ist nicht erforderlich.
