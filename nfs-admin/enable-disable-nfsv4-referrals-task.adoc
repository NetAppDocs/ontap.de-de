= NFSv4-Verweise für ONTAP SVMs aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können NFSv4-Verweise auf Storage Virtual Machines (SVMs) aktivieren, indem Sie die Optionen  `-v4-fsid-change` und ``-v4.0-referrals`` oder  aktivieren. Die Aktivierung von NFSv4-Verweisen kann zu schnellerem Datenzugriff für NFSv4-Clients führen, die diese Funktion unterstützen.

.Bevor Sie beginnen
Wenn Sie NFS-Verweise aktivieren möchten, müssen Sie zuerst paralleles NFS deaktivieren. Sie können nicht beides gleichzeitig aktivieren.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Führen Sie eine der folgenden Aktionen aus:
+
[cols="35,65"]
|===


| Wenn Sie möchten... | Geben Sie den Befehl ein... 


 a| 
NFSv4 Verweise aktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4-fsid-change enabled` `vserver nfs modify -vserver vserver_name -v4.0-referrals enabled`



 a| 
NFSv4-Verweise deaktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4.0-referrals disabled`



 a| 
NFSv4.1 Verweise aktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4-fsid-change enabled` `vserver nfs modify -vserver vserver_name -v4.1-referrals enabled`



 a| 
NFSv4.1-Verweise deaktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4.1-referrals disabled`

|===
. Zurück zur Administrator-Privilegstufe:
+
`set -privilege admin`
