= Die Gültigkeitsdauer für zwischengespeicherte NFS-Benutzeranmeldeinformationen für ONTAP SVMs konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Speicherdauer, für die ONTAP Anmeldeinformationen für NFS-Benutzer im internen Cache (Time-to-Live oder TTL) speichert, konfigurieren, indem der NFS-Server der Storage Virtual Machine (SVM) angepasst wird. Dies ermöglicht die Behebung bestimmter Probleme im Zusammenhang mit hoher Auslastung der Nameserver oder Änderungen der Anmeldeinformationen, die den NFS-Benutzerzugriff beeinflussen.

.Über diese Aufgabe
Diese Parameter sind auf der Stufe „Erweitertes Berechtigungsniveau“ verfügbar.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Die gewünschte Aktion ausführen:
+
[cols="35,65"]
|===


| Wenn Sie die TTL für zwischengespeicherte Daten ändern möchten... | Verwenden Sie den Befehl... 


 a| 
Positive Anmeldedaten
 a| 
`vserver nfs modify -vserver vserver_name -cached-cred-positive-ttl time_to_live`

Die TTL wird in Millisekunden gemessen. Ab ONTAP 9.10.1 und später beträgt der Standardwert 1 Stunde (3.600.000 Millisekunden). In ONTAP 9.9.1 und früher beträgt der Standardwert 24 Stunden (86.400.000 Millisekunden). Der zulässige Bereich für diesen Wert reicht von 1 Minute (60.000 Millisekunden) bis 7 Tagen (604.800.000 Millisekunden).



 a| 
Negative Anmeldeinformationen
 a| 
`vserver nfs modify -vserver vserver_name -cached-cred-negative-ttl time_to_live`

Die TTL wird in Millisekunden gemessen. Der Standardwert beträgt 2 Stunden (7.200.000 Millisekunden). Der zulässige Bereich für diesen Wert reicht von 1 Minute (60.000 Millisekunden) bis 7 Tagen (604.800.000 Millisekunden).

|===
. Zurück zur Administrator-Privilegstufe:
+
`set -privilege admin`
