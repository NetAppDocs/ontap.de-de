= Aktivieren oder Deaktivieren von NFSv4-Lesedelegationen für ONTAP SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um die NFSv4-Dateilesedelegierungen zu aktivieren oder zu deaktivieren, kann die ``-v4.0-read-delegation``oder Option geändert werden. Durch das Aktivieren von Dateilesedelegierungen kann ein Großteil des mit dem Öffnen und Schließen von Dateien verbundenen Nachrichtenaufwands eliminiert werden.

.Über diese Aufgabe
Standardmäßig sind Lese-Dateidelegationen deaktiviert.

Der Nachteil der Aktivierung von Lesedateidelegierungen besteht darin, dass der Server und seine Clients die Delegierungen nach einem Neustart oder Reboot des Servers, eines Clients oder wenn eine Netzwerkpartition auftritt, wiederherstellen müssen.

.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="35,65"]
|===


| Wenn Sie möchten... | Dann... 


 a| 
NFSv4-Lesedelegationen für Dateien aktivieren
 a| 
Führen Sie den folgenden Befehl aus:

`vserver nfs modify -vserver vserver_name -v4.0-read-delegation enabled`



 a| 
NFSv4.1-Lesedelegationen für Dateien aktivieren
 a| 
Führen Sie den folgenden Befehl aus:

+
`vserver nfs modify -vserver vserver_name -v4.1-read-delegation enabled`



 a| 
NFSv4 Lese-Dateidelegationen deaktivieren
 a| 
Führen Sie den folgenden Befehl aus:

`vserver nfs modify -vserver vserver_name -v4.0-read-delegation disabled`



 a| 
NFSv4.1 Lese-Dateidelegationen deaktivieren
 a| 
Führen Sie den folgenden Befehl aus:

`vserver nfs modify -vserver vserver_name -v4.1-read-delegation disabled`

|===


.Ergebnis
Die Dateidelegierungsoptionen werden sofort nach ihrer Änderung wirksam. Ein Neustart oder das erneute Starten von NFS ist nicht erforderlich.
