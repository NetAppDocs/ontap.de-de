= Funktionsweise der ONTAP FPolicy-Filter für Erstlesen und Erstschreiben mit NFS
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NFS-Clients erleben eine hohe Antwortzeit bei hohem Datenverkehr von lesen/schreiben-Anfragen, wenn die FPolicy unter Verwendung eines externen FPolicy-Servers mit lesen/schreiben-Operationen als überwachte Ereignisse aktiviert ist. Für NFS-Clients verringert der Einsatz von First-Read- und First-Write-Filtern in der FPolicy die Anzahl der FPolicy-Benachrichtigungen und verbessert die Performance.

Bei NFS führt der Client Ein-/Ausgabeoperationen an einer Datei durch, indem er deren Handle abruft. Dieses Handle kann auch nach Neustarts von Server und Client gültig bleiben. Daher kann der Client das Handle zwischenspeichern und Anfragen darüber senden, ohne die Handles erneut abrufen zu müssen. In einer regulären Sitzung werden zahlreiche Lese-/Schreibanfragen an den Dateiserver gesendet. Wenn für all diese Anfragen Benachrichtigungen generiert werden, kann dies zu folgenden Problemen führen:

* Eine höhere Last aufgrund zusätzlicher Benachrichtigungsverarbeitung und längerer Reaktionszeit.
* Es werden zahlreiche Benachrichtigungen an den FPolicy Server gesendet, obwohl der Server von keiner der Benachrichtigungen betroffen ist.


Nach dem Empfang der ersten lesen/schreiben-Anfrage eines Clients für eine bestimmte Datei wird ein Cache-Eintrag erstellt und der lesen/schreiben-Zähler erhöht. Diese Anfrage wird als erster lesen/schreiben-Schreibvorgang markiert und ein FPolicy-Ereignis wird generiert. Vor der Planung und Erstellung von FPolicy-Filtern für einen NFS-Client ist es wichtig, die Grundlagen der Funktionsweise von FPolicy-Filtern zu verstehen.

* First-read: Filtert die Client-Leseanfragen für First-read.
+
Wenn dieser Filter für NFS-Ereignisse verwendet wird, bestimmen die  `-file-session-io-grouping-count`- und  `-file-session-io-grouping-duration`-Einstellungen die erste Leseanforderung, für die FPolicy verarbeitet wird.

* First-write: Filtert die Schreibanfragen des Clients nach dem Prinzip „First-write“.
+
Wenn dieser Filter für NFS-Ereignisse verwendet wird, bestimmen die  `-file-session-io-grouping-count`- und  `-file-session-io-grouping-duration`-Einstellungen die erste Schreibanforderung, die von FPolicy verarbeitet wurde.



Die folgenden Optionen werden in die NFS-Datenbank aufgenommen.

[listing]
----


file-session-io-grouping-count: Number of I/O Ops on a File to Be Clubbed and Considered as One Session
for Event Generation
file-session-io-grouping-duration: Duration for Which I/O Ops on a File to Be Clubbed and Considered as
One Session for Event Generation
----
