= pNFS für ONTAP SVMs aktivieren oder deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
pNFS verbessert die Leistung, indem NFS-Clients lesen/schreiben-Operationen direkt und parallel auf Speichermedien ausführen können, wodurch der NFS-Server als potenzieller Engpass umgangen wird. Zum Aktivieren oder Deaktivieren von pNFS (parallel NFS) kann die  `-v4.1-pnfs` Option angepasst werden.

[cols="50,50"]
|===
| Wenn die ONTAP Version... | Der pNFS-Standardwert ist... 


| 9.8 oder höher | deaktiviert 


| 9.7 oder früher | aktiviert 
|===
.Bevor Sie beginnen
Für die Nutzung von pNFS ist die Unterstützung von NFSv4.1 erforderlich.

Wenn pNFS aktiviert werden soll, müssen zunächst die NFS-Weiterleitungen deaktiviert werden. Beides kann nicht gleichzeitig aktiviert sein.

Wenn pNFS mit Kerberos auf SVMs verwendet wird, muss Kerberos auf jedem LIF auf der SVM aktiviert sein.

.Schritt
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Geben Sie den Befehl ein... 


 a| 
pNFS aktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4.1-pnfs enabled`



 a| 
pNFS deaktivieren
 a| 
`vserver nfs modify -vserver vserver_name -v4.1-pnfs disabled`

|===
