= Informationen zu ONTAP NAS Namensräumen und Verbindungspunkten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein NAS-_Namespace_ ist eine logische Gruppierung von Volumes, die an _Verbindungen_ zusammengefügt sind, um eine einzelne Dateisystemhierarchie zu erstellen. Ein Client mit ausreichenden Berechtigungen kann auf Dateien im Namespace zugreifen, ohne den Speicherort der Dateien im Storage anzugeben. Mit Verbindungen versehene Volumes können sich überall im Cluster befinden.

Anstatt jedes Volume mit einer gewünschten Datei einzubinden, binden NAS-Clients einen NFS-Export ein oder greifen auf eine SMB-Freigabe zu. Der Export oder die Freigabe repräsentiert den gesamten Namensraum oder einen Zwischenstandort innerhalb des Namensraums. Der Client greift nur auf die Volumes zu, die unterhalb seines Zugriffspunkts eingebunden sind.

Sie können dem Namespace nach Bedarf Volumes hinzufügen. Es ist möglich, Verbindungspunkte direkt unterhalb einer übergeordneten Volume-Verbindung oder in einem Verzeichnis innerhalb eines Volumes zu erstellen. Ein Pfad zu einer Volume-Verbindung für ein Volume mit dem Namen „vol3“ könnte  `/vol1/vol2/vol3`, oder  `/vol1/dir2/vol3`, oder sogar  `/dir1/dir2/vol3` sein. Der Pfad wird als _Verbindungspfad_ bezeichnet.

Jede SVM verfügt über einen eindeutigen Namensraum. Das SVM Root-Volume ist der Einstiegspunkt in die Namensraumhierarchie.

[NOTE]
====
Um sicherzustellen, dass die Daten im Falle eines Knotenausfalls oder Failovers verfügbar bleiben, empfiehlt es sich, eine _load-sharing mirror_ Kopie für das SVM Root-Volume zu erstellen.

====
image:namespace-nfs-admin.gif["Namensraum und Verbindungspunkte"]

.Beispiel
Das folgende Beispiel erstellt ein Volume namens "`home4`" auf SVM vs1, das einen Verbindungspfad hat `/eng/home`:

[listing]
----
cluster1::> volume create -vserver vs1 -volume home4 -aggregate aggr1 -size 1g -junction-path /eng/home
[Job 1642] Job succeeded: Successful
----
