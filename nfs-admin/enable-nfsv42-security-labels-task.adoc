= NFS v4.2 Sicherheitslabels aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.9.1 können NFS-Sicherheitsbezeichnungen aktiviert werden. Sie sind standardmäßig deaktiviert.

.Über diese Aufgabe
Mit NFS v4.2-Sicherheitslabels sind ONTAP NFS-Server Mandatory Access Control (MAC)-fähig, speichern und rufen von Clients gesendete sec_label-Attribute ab.

Weitere Informationen finden Sie unter https://tools.ietf.org/html/rfc7204["RFC7240"^]

[NOTE]
====
NFS v4.2-Sicherheitsbezeichnungen werden für NDMP Dump-Operationen derzeit nicht unterstützt. Wenn Sicherheitsbezeichnungen bei Dateien oder Verzeichnissen vorhanden sind, schlägt der Dump fehl.

====
.Schritte
. Die Berechtigungseinstellung auf „Erweitert“ ändern:
+
``set -privilege advanced``

+
Weitere Informationen zu  `set` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/set.html["ONTAP-Befehlsreferenz"^].

. Sicherheitsbezeichnungen aktivieren:
+
``vserver nfs modify -vserver svm_name -v4.2-seclabel enabled``

+
Weitere Informationen zu  `vserver nfs modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-nfs-modify.html["ONTAP-Befehlsreferenz"^].
