= Informationen zu ONTAP NFS-Zugriffscache-Parametern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mehrere Parameter steuern die Aktualisierungsintervalle für Einträge im Zugriffscache. Das Verständnis der Funktionsweise dieser Parameter ermöglicht eine Anpassung zur Feinabstimmung des Zugriffscaches und zur Balance zwischen Leistung und Aktualität der gespeicherten Informationen.

Der Zugriffscache speichert Einträge, die aus einer oder mehreren Exportregeln bestehen, die für Clients gelten, die versuchen, auf Volumes oder Qtrees zuzugreifen. Diese Einträge werden für eine bestimmte Zeit gespeichert, bevor sie aktualisiert werden. Die Aktualisierungszeit wird durch Zugriffscache-Parameter bestimmt und hängt vom Typ des Zugriffscache-Eintrags ab.

Sie können Zugriffs-Cache-Parameter für einzelne SVMs festlegen. Dadurch können die Parameter je nach Zugriffsanforderungen der SVMs unterschiedlich sein. Nicht aktiv genutzte Zugriffs-Cache-Einträge werden nicht aktualisiert, was unnötige und verschwenderische Kommunikation mit externen Nameservern reduziert.

[cols="30,40,30"]
|===


| Cache-Eintragstyp für den Zugriff | Beschreibung | Aktualisierungsintervall in Sekunden 


 a| 
Positive Einträge
 a| 
Auf Cache-Einträge zugreifen, die nicht zu einer Zugriffsverweigerung für Clients geführt haben.
 a| 
Minimum: 300

Maximal: 86,400

Standard: 3.600



 a| 
Negative Einträge
 a| 
Auf Cache-Einträge zugreifen, die zu einer Zugriffsverweigerung für Clients geführt haben.
 a| 
Minimum: 60

Maximal: 86,400

Standard: 3.600

|===
.Beispiel
Ein NFS-Client versucht, auf ein Volume in einem Cluster zuzugreifen. ONTAP ordnet den Client einer Exportrichtlinienregel zu und stellt fest, dass der Client basierend auf der Konfiguration der Exportrichtlinienregel Zugriff erhält. ONTAP speichert die Exportrichtlinienregel als positiven Eintrag im Zugriffscache. Standardmäßig bleibt der positive Eintrag eine Stunde (3.600 Sekunden) im Zugriffscache gespeichert, danach wird der Eintrag automatisch aktualisiert, um die Informationen aktuell zu halten.

Um ein unnötiges Füllen des Zugriffscaches zu verhindern, gibt es einen zusätzlichen Parameter, mit dem vorhandene Zugriffscache-Einträge gelöscht werden, die über einen bestimmten Zeitraum nicht zur Entscheidung über den Clientzugriff verwendet wurden. Dieser `-harvest-timeout` Parameter hat einen zulässigen Bereich von 60 bis 2.592.000 Sekunden und eine Standardeinstellung von 86.400 Sekunden.
