= ONTAP NFS Qtree-IDs für Qtree-Dateioperationen validieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP kann optional eine zusätzliche Validierung von qtree-IDs durchführen. Diese Validierung stellt sicher, dass Client-Dateioperationen eine gültige qtree-ID verwenden und dass Clients Dateien nur innerhalb desselben qtrees verschieben können. Diese Validierung kann durch Ändern des  `-validate-qtree-export` Parameters aktiviert oder deaktiviert werden. Dieser Parameter ist standardmäßig aktiviert.

.Über diese Aufgabe
Dieser Parameter ist nur dann wirksam, wenn eine Exportrichtlinie direkt einem oder mehreren Qtrees auf der Storage Virtual Machine (SVM) zugewiesen wurde.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`set -privilege advanced`

. Führen Sie eine der folgenden Aktionen aus:
+
[cols="2*"]
|===
| Wenn die Qtree-ID-Validierung gewünscht ist... | Führen Sie den folgenden Befehl aus... 


 a| 
Aktiviert
 a| 
`vserver nfs modify -vserver vserver_name -validate-qtree-export enabled`



 a| 
Deaktiviert
 a| 
`vserver nfs modify -vserver vserver_name -validate-qtree-export disabled`

|===
. Zurück zur Administrator-Privilegstufe:
+
`set -privilege admin`
