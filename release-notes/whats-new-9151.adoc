[[data-protection]]
== Datenschutz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../smb-admin/windows-backup-symlinks.html[Windows-Sicherungsanwendungen und Unix-Style-Symlinks auf Servern]
 a| 
Wenn eine Windows-Sicherungsanwendung auf einen Symbol-Link (Symlink) im Unix-Stil stößt, wird dem Link gefolgt und die eigentlichen Daten werden von ONTAP zurückgegeben und gesichert. Ab ONTAP 9.15.1 besteht auch die Möglichkeit, den Symlink selbst anstelle der Daten, auf die er verweist, zu sichern. Dies kann verschiedene Vorteile bieten, darunter eine verbesserte Leistung der Sicherungsanwendungen. Die Funktion lässt sich über die ONTAP CLI oder die ONTAP REST API aktivieren.



 a| 
xref:../snapmirror-active-sync/index.html[SnapMirror active sync unterstützt symmetrische Aktiv/Aktiv-Bereitstellungen]
 a| 
SnapMirror active sync (ehemals SnapMirror Business Continuity) unterstützt jetzt symmetrische Aktiv/Aktiv-Bereitstellungen, wodurch Lese- und Schreib-I/O-Operationen von beiden Kopien einer geschützten LUN mit bidirektionaler synchroner Replizierung möglich sind.



 a| 
Erhöhtes Limit für Volumes in einer Konsistenzgruppe bei Verwendung von SnapMirror asynchron
 a| 
Konsistenzgruppen mit SnapMirror asynchronem Schutz unterstützen jetzt bis zu 80 Volumes in der Konsistenzgruppe.



 a| 
xref:../consistency-groups/configure-task.html[Unterstützung für das Administratorberechtigungsniveau für ONTAP REST API- und ONTAP CLI-Operationen mit Konsistenzgruppen]
 a| 
ONTAP CLI und ONTAP REST API Operationen für Konsistenzgruppen werden jetzt auf der als Administrator angemeldet Ebene unterstützt.



 a| 
xref:../concepts/ontap-and-vmware.html[Persistente Reservierungen für VMware vVols mit Windows Server Failover Clustering]
 a| 
ONTAP unterstützt derzeit VMware virtual volumes (vVols) sowie persistente Reservierungen mit herkömmlichen LUNs. Ab ONTAP 9.15.1 ist es auch möglich, eine persistente Reservierung mit einem vVol zu erstellen. Die Unterstützung für diese Funktion ist in ONTAP Tools für VMware vSphere 9 implementiert. Diese wird nur in einem Windows Server Failover Cluster (WSFC) unterstützt, einer Gruppe von Windows-VMs im Cluster.

[[security]]
== Sicherheit

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../nas-audit/create-persistent-stores.html[Vereinfachte Erstellung und Konfiguration des persistenten FPolicy Store]
 a| 
Sie können den FPolicy persistenten Speicher erstellen und gleichzeitig die Volume-Erstellung und -Konfiguration automatisieren, indem Sie den  `persistent-store create` Befehl verwenden.

Der erweiterte  `persistent-store create` Befehl ermöglicht auch die Verwendung des Parameters autosize-mode, der es ermöglicht, dass das Volume in Abhängigkeit vom belegten Speicherplatz wächst oder schrumpft.



 a| 
xref:../nfs-rdma/index.html[Unterstützung für NFSv3 mit RDMA]
 a| 
NFS over RDMA Konfigurationen unterstützen jetzt NFSv3.



 a| 
xref:../nas-audit/supported-file-operation-filter-fpolicy-nfsv4-concept.html[FPolicy unterstützt das NFS 4.1 Protokoll]
 a| 
FPolicy unterstützt das NFS 4.1 Protokoll.



 a| 
xref:../nas-audit/plan-fpolicy-external-engine-config-concept.html[Protobuf Engine Formatunterstützung für FPolicy]
 a| 
Protobuf ist Googles sprachneutraler Mechanismus zur Serialisierung strukturierter Daten. Es ist kleiner, schneller und einfacher als XML, was die Leistung von FPolicy verbessert.

Sie können das externe Engine-Format protobuf verwenden. Wenn protobuf ausgewählt ist, werden die Benachrichtigungsnachrichten in binärer Form mit Google Protobuf codiert. Bevor das externe Engine-Format auf protobuf gesetzt wird, sollte sichergestellt sein, dass der FPolicy Server ebenfalls die protobuf-Deserialisierung unterstützt.



 a| 
xref:../authentication/dynamic-authorization-overview.html[Dynamische Autorisierung für SSH-Verbindungen]
 a| 
ONTAP 9.15.1 bietet das grundlegende Framework für Dynamic Authorization, das eine verbesserte Sicherheit bei der Verwaltung des ONTAP Systems ermöglicht, indem Administratorbenutzern ein Sicherheitsvertrauenswert zugewiesen werden kann und bei verdächtigen Aktivitäten zusätzliche Autorisierungsprüfungen erfolgen. Dynamic Authorization kann als Teil einer datenzentrierten Zero Trust Sicherheitsarchitektur genutzt werden.



 a| 
Unterstützung für TLS 1.3 für S3-Speicherung, FlexCache und Cluster Peering-Verschlüsselung
 a| 
TLS 1.3 wird seit ONTAP 9.11.1 für den Verwaltungszugriff unterstützt, ab ONTAP 9.15.1 jedoch auch für S3 Storage, FlexCache und Cluster Peering Verschlüsselung. Einige Anwendungen, wie FabricPool, Microsoft Azure Page Blobs Storage und SnapMirror Cloud, sind in der Version 9.15.1 weiterhin auf die Nutzung von TLS 1.2 beschränkt.



 a| 
xref:../multi-admin-verify/index.html#rule-protected-commands[Zulässiger Satz regelgeschützter Befehle für die Multi-Admin-Überprüfung erweitert]
 a| 
Administratoren können Multi-Admin-Verifizierungsregeln erstellen, um die Clusterkonfiguration, das Löschen von LUNs, die Systemkonfiguration, die Sicherheitskonfiguration für IPsec und SAML, Volume-Snapshot-Operationen, die vServer-Konfiguration und andere Befehle zu schützen.



 a| 
xref:../system-admin/requirements-autosupport-reference.html[Zustellung von AutoSupport-Nachrichten mittels SMTP mit TLS]
 a| 
Während der empfohlene Transport von AutoSupport-Nachrichten zu NetApp HTTPS ist, war auch unverschlüsseltes SMTP verfügbar. Mit ONTAP 9.15.1 besteht für Kunden nun die Möglichkeit, TLS mit SMTP zu verwenden. Das SMTPS-Protokoll stellt einen sicheren Übertragungskanal her, indem sowohl der E-Mail-Verkehr als auch die optionalen Anmeldeinformationen des E-Mail-Servers verschlüsselt werden. Es wird explizites TLS verwendet, sodass TLS nach dem Aufbau der TCP-Verbindung aktiviert wird. Werden Kopien der Nachrichten an lokale E-Mail-Adressen gesendet, wird dieselbe Konfiguration verwendet.

[[storage-efficiency]]
== Speichereffizienz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../volumes/determine-space-usage-volume-aggregate-concept.html[Änderungen bei der Berichterstattung von Volume-Speicherplatzmetriken]
 a| 
Es wurden zwei neue Zähler eingeführt, die ausschließlich die verwendeten Metadaten anzeigen. Darüber hinaus wurden mehrere bestehende Zähler angepasst, um die Metadaten auszublenden und nur die Nutzerdaten anzuzeigen. Diese Änderungen ermöglichen eine übersichtlichere Darstellung der Metriken, getrennt nach den beiden Datentypen. Kunden können diese Zähler verwenden, um präzisere Chargeback-Modelle zu erstellen, indem die Metadaten vom Gesamtbetrag abgezogen und nur die tatsächlichen Nutzerdaten berücksichtigt werden.



 a| 
xref:../concepts/builtin-storage-efficiency-concept.html[Speichereffizienz mit CPU oder dediziertem Offload-Prozessor]
 a| 
ONTAP bietet Speichereffizienz und Datenkomprimierung auf den Plattformen AFF A70, AFF A90 und AFF A1K. Je nach Plattform erfolgt die Komprimierung entweder mit der Haupt-CPU oder mit einem dedizierten Offload-Prozessor. Die Speichereffizienz wird automatisch aktiviert und erfordert keine Konfiguration.

[[storage-resource-management-enhancements]]
== Verbesserungen im Ressourcenmanagement für Speicher

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../flexcache-writeback/flexcache-writeback-enable-task.html[FlexCache Write-Back-Unterstützung]
 a| 
Wenn Write-Back auf dem Cache-Volume aktiviert ist, werden Schreibanforderungen an den lokalen Cache statt an das Ursprungsvolume gesendet, was eine bessere Leistung für Edge-Computing-Umgebungen und Caches mit schreibintensiven Workloads bietet.

 a| 
Einem FlexClone volume wird ein eigener Verschlüsselungsschlüssel zugewiesen, der unabhängig vom FlexVol volume-Verschlüsselungsschlüssel (Host) ist.
