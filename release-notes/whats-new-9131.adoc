[[storage-efficiency]]
== Speichereffizienz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


| Änderung der Berichterstattung des Primärdatenreduktionsverhältnisses im System Manager  a| 
Das im System Manager angezeigte primäre Datenreduktionsverhältnis berücksichtigt bei der Berechnung nicht mehr die durch Snapshots erzielten Speicherplatzeinsparungen. Es zeigt nur das Verhältnis zwischen verwendetem logischem und verwendetem physischem Speicherplatz an. In früheren Versionen von ONTAP waren die erheblichen Speicherplatzeinsparungen durch Snapshots im primären Datenreduktionsverhältnis enthalten. Daher wird nach einem Upgrade auf ONTAP 9.13.1 ein deutlich niedrigeres primäres Verhältnis angezeigt. Datenreduktionsraten mit Snapshots sind weiterhin in der Detailansicht **Kapazität** sichtbar.



| xref:../volumes/enable-temperature-sensitive-efficiency-concept.html[Temperaturabhängige Speichereffizienz] | Die temperaturabhängige Speichereffizienz fügt das sequenzielle Packen angrenzender physischer Blöcke hinzu, um die Speichereffizienz zu verbessern. Volumes, bei denen die temperaturabhängige Speichereffizienz aktiviert ist, erhalten das sequenzielle Packen automatisch, wenn Systeme auf ONTAP 9.13.1 aktualisiert werden. 


| Durchsetzung des logischen Raums | Die Durchsetzung des logischen Speicherplatzes wird auf SnapMirror-Zielen unterstützt. 


| xref:../volumes/manage-svm-capacity.html[Unterstützung für Kapazitätsgrenzen von Storage-VMs] | Sie können Kapazitätsgrenzen für eine Storage VM (SVM) festlegen und Warnmeldungen aktivieren, wenn sich die SVM einem prozentualen Schwellenwert nähert. 
|===

| xref:..//task_nas_file_system_analytics_enable.html#modify[Dateisystemanalyse ist standardmäßig aktiviert] | Die Dateisystemanalyse ist standardmäßig für neue Volumes aktiviert. 


| xref:../flexgroup/create-svm-disaster-recovery-relationship-task.html[SVM Disaster Recovery Fanout-Beziehungen mit FlexGroup Volumes]  a| 
Die Fanout-Beschränkung von SVM DR mit FlexGroup Volumes wurde aufgehoben. SVM DR mit FlexGroup umfasst die Unterstützung von SnapMirror Fanout-Beziehungen zu acht Standorten.



| xref:../flexgroup/manage-flexgroup-rebalance-task.html[Einzelner FlexGroup Ausgleichsvorgang] | Sie können einen einzelnen FlexGroup Rebalancing-Vorgang so planen, dass er zu einem von Ihnen festgelegten Datum und einer Uhrzeit in der Zukunft beginnt. 


| xref:../fabricpool/benefits-storage-tiers-concept.html[FabricPool Leseleistung] | FabricPool bietet eine verbesserte sequenzielle Leseleistung für Single- und Multi-Stream-Workloads bei cloudbasierten Daten und Tiering-Durchsatz. Diese Verbesserung ermöglicht es, eine höhere Rate von GET- und PUT-Anfragen an den Backend-Objektspeicher zu senden. Bei lokalen Objektspeichern sollte die Leistungsreserve des Objektspeicherdienstes berücksichtigt und festgestellt werden, ob eine Drosselung der FabricPool PUT-Anfragen erforderlich ist. 


| xref:../performance-admin/guarantee-throughput-qos-task.html[Vorlagen für adaptive QoS Richtlinien] | Adaptive QoS-Richtlinienvorlagen ermöglichen das Festlegen von Durchsatzuntergrenzen auf SVM-Ebene.
