[[data-protection]]
== Datenschutz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


| xref:../data-protection/snapmirror-synchronous-disaster-recovery-basics-concept.html[Unterstützung für größere FlexVol volumes mit SnapMirror Synchronous]  a| 
Die maximal unterstützte FlexVol Volume-Größe in SnapMirror Synchronous-Konfigurationen wurde von 100 TB auf 300 TB erhöht. Sowohl der Quell- als auch der Ziel-Cluster müssen mit _ONTAP 9.12.1P2 oder neuer_ betrieben werden.



| xref:../data-protection/snapmirror-synchronous-disaster-recovery-basics-concept.html[Unterstützung für größere Datei- und LUN-Größen in SnapMirror Synchronous] | Die maximal unterstützte Datei- und LUN-Größe in SnapMirror Synchronous Konfigurationen wurde von 16 TB auf 128 TB erhöht. Sowohl der Quell- als auch der Zielcluster müssen ONTAP 9.12.1 P2 oder höher ausführen. 


| xref:../consistency-groups/index.html[Verbesserte Unterstützung für Konsistenzgruppen]  a| 
* Es besteht die Möglichkeit, Volumes zu einer Konsistenzgruppe hinzuzufügen oder daraus zu entfernen sowie eine Konsistenzgruppe (einschließlich aus einem Snapshot) zu klonen.
* Konsistenzgruppen unterstützen die Anwendungskennzeichnung, um Datenschutz- und Verwaltungsprozesse zu optimieren.
* Die ONTAP REST API unterstützt die Konfiguration von Konsistenzgruppen mit NFS/SMB Volumes oder NVMe Namespaces.




| xref:../data-protection/snapmirror-synchronous-disaster-recovery-basics-concept.html#supported-features[SnapMirror Synchronous NDO] | SnapMirror Synchronous unterstützt unterbrechungsfreie Vorgänge (NDO) wie HA-Takeover und Giveback, Volume Move und andere wartungsbezogene Vorgänge. Diese Funktion ist nur auf AFF/ASA Plattformen verfügbar. 


| xref:../mediator/index.html[ONTAP Mediator 1.5 unterstützt SnapMirror Business Continuity] | ONTAP Mediator 1.5 steht zur Überwachung von SnapMirror active sync Beziehungen zur Verfügung. 


| xref:../snapmirror-active-sync/index.html[SnapMirror Active Sync-Kontinuitätsverbesserungen] | SnapMirror active sync unterstützt die teilweise Wiederherstellung von LUNs aus Snapshots. Zusätzlich erweitert SnapMirror active sync die QoS auf Volumes, die sich nicht in der SnapMirror Beziehung befinden. 


| xref:../data-protection/convert-snapmirror-version-flexible-task.html[Indikator für den Neuaufbau des Data Warehouse für SnapMirror asynchron] | SnapMirror asynchron bietet einen Indikator, der zeigt, wie lange der Wiederaufbau eines Data Warehouse nach einer Notfallwiederherstellungsübung dauert, indem der prozentuale Fertigstellungsgrad angezeigt wird. 


| SnapLock Option zum Festlegen einer minimalen Aufbewahrungszeit „nicht spezifiziert“ absolute Aufbewahrungszeit | SnapLock beinhaltet eine Option, eine Mindestaufbewahrungszeit festzulegen, wenn die absolute Aufbewahrungszeit auf „nicht angegeben“ eingestellt ist. 


| xref:../snaplock/snapshot-lock-concept.html[Manipulationssichere Schnappschüsse] | Sie können einen Snapshot auf einem Nicht-SnapLock Volume sperren, um Schutz vor Ransomware-Angriffen zu bieten. Das Sperren von Snapshots trägt dazu bei, dass sie nicht versehentlich oder böswillig gelöscht werden.

Weitere Informationen finden sich unter link:https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU484["SU484: NetApp weist AutoSupport-Nachrichten mit unzureichender Transportsicherheit zurück"^].

[[system-manager]]
== System Manager

Ab ONTAP 9.12.1 ist der System Manager in die NetApp Console integriert. Mit der Console können Administratoren die hybride Multicloud-Infrastruktur über eine zentrale Steuerungsebene verwalten, während das vertraute System Manager Dashboard erhalten bleibt. Beim Anmelden am System Manager wird Administratoren die Möglichkeit geboten, entweder über die NetApp Console auf die System Manager-Oberfläche zuzugreifen oder direkt auf den System Manager zuzugreifen. Weitere Informationen zu xref:../concepts/sysmgr-integration-console-concept.html[System Manager Integration mit NetApp Console].
