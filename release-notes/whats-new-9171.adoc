[[data-protection]]
== Datenschutz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../nvme/support-limitations.html#features[SnapMirror Active Sync-Unterstützung für den Hostzugriff über NVMe]
 a| 
SnapMirror active sync erweitert die Unterstützung für NVMe-Zugriff für VMware-Workloads mit NVMe/TCP- und NVMe/FC-Hostzugriff für ONTAP Cluster mit zwei Knoten. Die Unterstützung von NVMe/TCP für VMware-Workloads ist abhängig von der Behebung bestehender VMware-Fehler.



 a| 
xref:../snapmirror-active-sync/index.html[ONTAP Cloud Mediator Support mit SnapMirror Active Sync]
 a| 
ONTAP Cloud Mediator wurde in ONTAP 9.17.1 eingeführt und unterstützt SnapMirror Aktiv/Aktiv-Sync-Beziehungen. Der Cloud-basierte Mediator fungiert wie ONTAP Mediator als Quorum-Zeuge für SnapMirror Aktiv/Aktiv-Sync-Beziehungen, wodurch ein transparentes Failover sichergestellt wird und die operative Komplexität bei der Wartung und Verwaltung eines dritten Standorts reduziert wird.

|===


[[s3-object-storage]]
== S3 Objektspeicher

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../s3-multiprotocol/index.html[CopyObject-Aktionsunterstützung in ONTAP S3 NAS-Buckets]
 a| 
Die CopyObject Aktion wird innerhalb des ONTAP S3 NAS-Buckets unterstützt.



 a| 
xref:../s3-multiprotocol/create-nas-bucket-task.html[Unterstützung für die Verknüpfung eines S3 NAS-Buckets mit einem Verbindungspfad]
 a| 
Beim Erstellen eines S3 NAS-Buckets mit der ONTAP CLI kann der Bucket mit dem Volume statt mit dem Verbindungspfad verknüpft werden. Bei der Verknüpfung mit dem Volume wird der Verbindungspfad automatisch aktualisiert, wenn sich der Pfad ändert, zum Beispiel beim Aushängen oder Einhängen eines Volumes.

xref:../s3-multiprotocol/index.html#object-multipart-upload[S3 Multiprotokollunterstützung für Tagging und Metadaten]
 a| 
Tagging und Benutzer-Metadaten-Schlüssel/Wert-Paare werden von der CreateMultipartUpload-Aktion in Multiprotokoll-Umgebungen (S3 und NAS) unterstützt.

[[security]]
== Sicherheit

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../anti-ransomware/index.html[Zusätzliche Hypervisor-Unterstützung]
 a| 
Ab ONTAP 9.17.1P5 unterstützt Autonomous Ransomware Protection Hyper-V, KVM und OpenStack Hypervisoren.



 a| 
xref:../anti-ransomware/index.html[Autonome Ransomware-Schutz-Snapshot-Updates]
 a| 
ONTAP führt mehrere Aktualisierungen für ARP Snapshots ein.



 a| 
xref:../system-admin/use-hsts-task.html[HTTP Strict Transport Security (HSTS) Unterstützung]
 a| 
ONTAP unterstützt HTTP Strict Transport Security für Webdienste, wodurch die Durchsetzung einer sicheren HTTPS-Kommunikation zwischen dem Browser eines Benutzers und ONTAP ermöglicht wird.



 a| 
xref:../networking/ipsec-prepare.html[IPsec-Hardware-Offload mit Link-Aggregation-Gruppen]
 a| 
ONTAP unterstützt IPsec-Hardware-Offload für Link-Aggregation-Gruppen und erweitert damit die in 9.16.1 eingeführte Hardware-Offload-Unterstützung.



 a| 
xref:../networking/ipsec-prepare.html[IPsec-Postquanten-Pre-Shared-Key-Unterstützung]
 a| 
ONTAP unterstützt Postquanten-Pre-Shared Keys für IPsec, um vor potenziellen zukünftigen Quantencomputerangriffen zu schützen.



 a| 
xref:../encryption-at-rest/manage-keys-barbican-task.html[OpenStack Barbican-Schlüsselmanager-Unterstützung]
 a| 
ONTAP unterstützt den OpenStack Barbican Key Manager für NetApp Volume Encryption (NVE)-Schlüssel.



 a| 
xref:../authentication/configure-jit-elevation-task.html[Just in Time (JIT)-Unterstützung für die Rechteerweiterung]
 a| 
ONTAP unterstützt die JIT-Berechtigungserweiterung für rollenbasierte Zugriffssteuerung (RBAC). Benutzer können eine temporäre Erhöhung der Berechtigungen auf eine konfigurierte Rolle anfordern, wodurch der Zugriff auf privilegierte Befehle bei Bedarf ermöglicht wird. Cluster-Administratoren können festlegen, wer auf die JIT-Berechtigungserweiterung zugreifen kann sowie wann und wie lange der Zugriff erlaubt ist.



 a| 
xref:../system-admin/configure-saml-authentication-task.html[Unterstützung für Entra IdP und IdP-Gruppenunterstützung für SAML-Authentifizierung]
 a| 
ONTAP unterstützt Microsoft Entra als SAML-Identitätsanbieter. Zusätzlich kann die vom IdP bereitgestellte Gruppeninformation ONTAP Rollen zugeordnet werden.



 a| 
xref:../system-admin/audit-manage-cross-cluster-requests.html[Prüfung clusterübergreifender Anfragen]
 a| 
Sie können Überwachungsvorgänge sowohl auf dem initiierenden Quell-Cluster als auch auf dem Zielcluster (Ausführungscluster) konfigurieren und ausführen. In früheren Versionen führte nur das Cluster, das die Clientanfrage empfing, die Überwachung durch. Mit dieser Funktion protokolliert nun auch ein verbundener Cluster, der eine clusterübergreifende Anfrage bearbeitet, die Aktivität. Diese Überwachungsvorgänge können aktiviert und auf jede innerhalb von ONTAP initiierte SET- oder GET-Anfrage ausgeweitet werden.



 a| 
xref:../anti-ransomware/index.html[Unterstützung für SAN mit autonomem Ransomware-Schutz]
 a| 
ARP unterstützt SAN-Volumes mit verschlüsselungsbasierter Anomalieerkennung, führt neue Befehle für detaillierte Entropie-Statistiken ein und vereinheitlicht die Ransomware-Schutzmeldungen im System Manager, die sich zuvor auf NAS-Workload konzentrierten. Konfigurierbare Erkennungsschwellenwerte und eine deterministischere Snapshot-Aufbewahrung bieten mehr Flexibilität für unterschiedliche Workloads.

[[storage-resource-management-enhancements]]
== Verbesserungen im Ressourcenmanagement für Speicher

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
xref:../task_nas_file_system_analytics_enable.html[FSA ist standardmäßig für neue Volumes aktiviert]
 a| 
Auf neu erstellten SVMs auf ONTAP-Clustern, die für NAS-Protokolle zugewiesen sind, ist die Dateisystemanalyse (FSA) standardmäßig aktiviert. FSA wird automatisch aktiviert, sobald ein Volume erstellt wird, wodurch sofort Analysefunktionen ohne zusätzliche Konfiguration zur Verfügung stehen.



 a| 
xref:../flexgroup/fast-directory-delete-asynchronous-task.html[Verbesserte Unterstützung für die Anzeige des direkten Löschfortschritts auf FlexGroup Volumes]
 a| 
Der ONTAP CLI-Befehl `volume file async-delete show` wurde erweitert und umfasst nun auch asynchron gelöschte Aufträge, die von Clients ausgegeben wurden.
