= Neues in ONTAP 9.11.1
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Informationen zu den neuen Funktionen in ONTAP 9.11.1.

Detaillierte Informationen zu bekannten Problemen, Einschränkungen und Hinweisen zum Upgrade in den aktuellen ONTAP 9-Versionen sind in den  https://library.netapp.com/ecm/ecm_download_file/ECMLP2492508["ONTAP 9 Versionshinweise"^] zu finden. Eine Anmeldung mit Ihrem NetApp Account oder die Erstellung eines Accounts ist erforderlich, um auf die Release Notes zuzugreifen.

* Informationen zu neuen und verbesserten Funktionen https://docs.netapp.com/us-en/ontap-metrocluster/releasenotes/mcc-new-features.html["ONTAP MetroCluster Funktionen"^].
* Weitere Informationen https://docs.netapp.com/us-en/ontap-systems/whats-new.html["Neue und verbesserte Unterstützung für AFF, ASA und FAS Systeme sowie unterstützte Switches"^].
* Informationen zu Aktualisierungen der  https://docs.netapp.com/us-en/ontap-automation/whats_new.html["ONTAP REST API"^].


Ein Upgrade auf die neueste Version von ONTAP ist unter xref:../upgrade/create-upgrade-plan.html[Vorbereitung auf das Upgrade von ONTAP] beschrieben.



[[data-protection]]
== Datenschutz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


| xref:../encryption-at-rest/configure-cluster-key-server-task.html[Cluster externe Schlüsselserver] | Die Unterstützung für geclusterte externe Schlüsselverwaltungsserver wurde für NetApp Partner hinzugefügt, die eine geclusterte KMIP Serverlösung anbieten. Dadurch können primäre und sekundäre KMIP Server hinzugefügt werden, wodurch die Duplizierung von Verschlüsselungsschlüsseldaten verhindert wird. Für unterstützte Partner siehe link:https://imt.netapp.com/matrix/#welcome["Interoperabilitätsmatrix-Tool"^]. 


| xref:../task_dp_create_custom_data_protection_policies.html[SnapMirror asynchrone Richtlinie im System Manager]  a| 
Mit System Manager können vorab erstellte und benutzerdefinierte Spiegelungs- und Tresorrichtlinien hinzugefügt, Legacy-Richtlinien angezeigt und die in einer Datensicherungsstrategie definierten Übertragungszeitpläne beim Schutz von Volumes und Storage-VMs überschrieben werden. Mit System Manager können zudem die Schutzbeziehungen von Volumes und Storage-VMs bearbeitet werden.


TIP: Wenn ONTAP 9.8P12 oder eine spätere ONTAP 9.8 Patch-Version ausgeführt wird, SnapMirror mit System Manager konfiguriert wurde und ein Upgrade auf ONTAP 9.9.1 oder ONTAP 9.10.1 vorgesehen ist, sollten für das Upgrade ONTAP 9.9.1P13 oder höher sowie ONTAP 9.10.1P10 oder höher verwendet werden.



| xref:../data-protection/restore-contents-volume-snapshot-task.html[SnapMirror Cloud Wiederherstellung eines einzelnen Verzeichnisses] | Ermöglicht Clusteradministratoren auf Administrator-Ebene, eine einzelne Verzeichniswiederherstellung von einem Cloud-Endpunkt durchzuführen. Die UUID des Quellendpunkts muss angegeben werden, um den Backup-Endpunkt zu identifizieren, von dem die Wiederherstellung erfolgt. Da mehrere Backups denselben  `cloud_endpoint_name` als Ziel verwenden können, muss die mit dem Backup verknüpfte UUID für den Wiederherstellungsbefehl angegeben werden. Mit dem  `snapmirror show` Befehl kann die  `source_endpoint_uuid` abgerufen werden. 


| xref:../snapmirror-active-sync/interoperability-reference.html[Verbesserte Unterstützung für SnapMirror active sync]  a| 
* SnapMirror active sync unterstützt AIX als Host
* SnapMirror active sync unterstützt Einzeldatei SnapRestore, wodurch das Wiederherstellen einer einzelnen LUN oder normalen Datei in einer SnapMirror active sync Konfiguration möglich ist.




| xref:../data-protection/reactivate-original-source-svm-task.html[SVM Datenreplizierung schnelle Resynchronisierung] | Die SVM Datenreplizierung mit schneller Resynchronisierung bietet Speicheradministratoren die Möglichkeit, einen vollständigen Neuaufbau des Data Warehouse zu umgehen und sich nach einer Notfallwiederherstellungsübung schneller zu erholen. 


| xref:../data-protection/snapmirror-svm-replication-concept.html#support-details[SVM Datenreplizierungsunterstützung mit MetroCluster] | Die SVM-DR-Quelle wird an beiden Enden einer MetroCluster Konfiguration unterstützt. 


 a| 
xref:../consistency-groups/protect-task.html[Zwei-Phasen-Konsistenzgruppen-Snapshot-Erstellung]
| In der ONTAP REST API unterstützen Konsistenzgruppen ein zweiphasiges Snapshot-Verfahren, das eine Vorprüfung ermöglicht, bevor der Snapshot festgeschrieben wird.

[[security]]
== Sicherheit

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


| xref:../anti-ransomware/index.html[Verbesserungen des autonomen Ransomware Protection (ARP)] | Der ARP-Erkennungsalgorithmus wurde verbessert, um zusätzliche Malware-Bedrohungen zu erkennen. Außerdem wird ein neuer Lizenzschlüssel zur Aktivierung von Autonomous Ransomware Protection verwendet. Bei ONTAP-Systemen, die von ONTAP 9.10.1 aktualisiert werden, bietet der bisherige Lizenzschlüssel weiterhin die gleiche Funktionalität. 


| xref:../multi-admin-verify/index.html[Multi-Admin-Verifizierung] | Wenn die Überprüfung durch mehrere Administratoren aktiviert ist, können bestimmte Vorgänge, wie das Löschen von Volumes oder Snapshots, nur nach Genehmigung durch die entsprechenden Administratoren ausgeführt werden. Dadurch wird verhindert, dass kompromittierte, böswillige oder unerfahrene Administratoren unerwünschte Änderungen vornehmen oder Daten löschen.

SnapLock unterstützt die folgenden Vorgänge auf FlexGroup volumes nicht: SnapLock für SnapVault, ereignisbasierte Aufbewahrung und Legal Hold.

Mit System Manager können zudem die Schutzbeziehungen von Volumes und Storage-VMs bearbeitet werden.

| xref:../insights-system-optimization-task.html[Erkenntnisse aus der Systemanalyse] | Auf der Seite „Einblicke“ unterstützt der System Manager bei der Optimierung des Systems, indem zusätzliche Kapazitäts- und Sicherheitseinblicke sowie neue Einblicke in die Konfiguration von Clustern und Storage-VMs angezeigt werden. 


| Verbesserungen der Benutzerfreundlichkeit  a| 
* xref:../task_admin_add_a_volume.html[Neu erstellte Volumes sind standardmäßig nicht gemeinsam nutzbar:]Sie können die Standardzugriffsberechtigungen festlegen, wie zum Beispiel den Export über NFS oder die Freigabe über SMB/CIFS, und die Berechtigungsstufe angeben.
* xref:../san-admin/manage-san-initiators-task.html[SAN-Vereinfachung:]Beim Hinzufügen oder Bearbeiten einer Initiatorgruppe können System Manager Benutzer den Verbindungsstatus der Initiatoren in der Gruppe einsehen und sicherstellen, dass verbundene Initiatoren in die Gruppe aufgenommen werden, sodass auf LUN Daten zugegriffen werden kann.




| xref:../disks-aggregates/aggregate-creation-workflow-concept.html[Erweiterte lokale Tier (Aggregat)-Operationen]  a| 
System Manager Administratoren können die Konfiguration einer lokalen Ebene festlegen, wenn sie die Empfehlung von System Manager nicht übernehmen möchten. Außerdem können Administratoren die RAID-Konfiguration einer bestehenden lokalen Ebene bearbeiten.


NOTE: Wenn Sie ONTAP 9.8P12 oder eine spätere ONTAP 9.8 Patch-Version verwenden und SnapMirror mit System Manager konfiguriert haben und ein Upgrade auf ONTAP 9.9.1 oder ONTAP 9.10.1 planen, sollten Sie für Ihr Upgrade ONTAP 9.9.1P13 oder höher und ONTAP 9.10.1P10 oder höher als Patch-Versionen verwenden.
