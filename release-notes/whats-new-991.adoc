= Neues in ONTAP 9.9.1
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Informationen zu den neuen Funktionen in ONTAP 9.9.1.

Detaillierte Informationen zu bekannten Problemen, Einschränkungen und Hinweisen zum Upgrade in den aktuellen ONTAP 9-Versionen sind in den  https://library.netapp.com/ecm/ecm_download_file/ECMLP2492508["ONTAP 9 Versionshinweise"^] zu finden. Eine Anmeldung mit Ihrem NetApp Account oder die Erstellung eines Accounts ist erforderlich, um auf die Release Notes zuzugreifen.

* Informationen zu neuen und verbesserten Funktionen https://docs.netapp.com/us-en/ontap-metrocluster/releasenotes/mcc-new-features.html["ONTAP MetroCluster Funktionen"^].
* Weitere Informationen https://docs.netapp.com/us-en/ontap-systems/whats-new.html["Neue und verbesserte Unterstützung für AFF, ASA und FAS Systeme sowie unterstützte Switches"^].
* Informationen zu Aktualisierungen der  https://docs.netapp.com/us-en/ontap-automation/whats_new.html["ONTAP REST API"^].


Ein Upgrade auf die neueste Version von ONTAP ist unter xref:../upgrade/create-upgrade-plan.html[Vorbereitung auf das Upgrade von ONTAP] beschrieben.



[[data-protection]]
== Datenschutz

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


| link:../snaplock/index.html["Unterstützung der Speichereffizienz auf SnapLock Volumes und Aggregaten"] | Die Speichereffizienzfunktionen für SnapLock Volumes und Aggregate wurden erweitert und umfassen nun auch Datenkompaktierung, volumenübergreifende Deduplizierung, adaptive Komprimierung und TSSE (Temperaturabhängige Speichereffizienz), wodurch eine größere Speicherplatzersparnis bei WORM-Daten ermöglicht wird. 


| link:../data-protection/snapmirror-svm-replication-concept.html["Unterstützung für die Konfiguration unterschiedlicher Snapshot-Richtlinien auf SVM DR Quelle und SVM DR Ziel"] | SVM DR Konfigurationen können die Mirror-Vault-Richtlinie verwenden, um unterschiedliche Snapshot-Richtlinien auf der Quelle und dem Ziel zu konfigurieren, und die Richtlinien auf dem Ziel werden nicht durch die Richtlinien auf der Quelle überschrieben. 


| link:../data-protection/snapmirror-licensing-concept.html["System Manager Unterstützung für SnapMirror Cloud"] | SnapMirror Cloud wird jetzt im System Manager unterstützt. 


| xref:../nas-audit/enable-disable-auditing-svms-task.html[SVMs mit aktiviertem Revisionsprotokoll] | Die maximale Anzahl der in einem Cluster unterstützten SVMs mit aktivierter Revisionsprotokollierung wurde von 50 auf 400 erhöht. 


| xref:../data-protection/snapmirror-synchronous-disaster-recovery-basics-concept.html[SnapMirror Synchronous] | Die maximale Anzahl unterstützter SnapMirror Synchronous-Endpunkte pro HA-Paar hat sich von 80 auf 160 erhöht. 


| xref:../flexgroup/create-snapmirror-relationship-task.html[FlexGroup SnapMirror Topologie] | FlexGroup Volumes unterstützen zwei oder mehr Verzweigungsbeziehungen, zum Beispiel A→B, A→C. Wie FlexVol Volumes unterstützt auch die FlexGroup Verzweigung maximal 8 Verzweigungszweige und eine Kaskadierung bis zu zwei Ebenen, zum Beispiel A→B→C. 
|===

[[networking]]
== Netzwerk

[cols="30%,70%"]
|===
| Aktualisieren | Beschreibung 


 a| 
link:../high-availability/index.html["Cluster-Resilienz"]
 a| 
* Portüberwachung und -vermeidung für Zwei-Knoten-Switchless-Cluster (bisher nur in Switch-Konfigurationen verfügbar)
* Automatisches Knoten-Failover, wenn ein Knoten keine Daten über sein Clusternetzwerk bereitstellen kann
* Neue Tools zur Anzeige, welche Clusterpfade Paketverluste aufweisen




 a| 
link:../networking/configure_virtual_ip_vip_lifs.html["Virtuelle IP (VIP) LIF Erweiterung"]
 a| 
* Die autonome Systemnummer (ASN) für das Border Gateway Protocol (BGP) unterstützt eine 4-Byte nicht-negative Ganzzahl.
* Der Multi-Exit-Diskriminator (MED) ermöglicht erweiterte Routenauswahl mit Unterstützung für Pfadpriorisierung. MED ist ein optionales Attribut in der BGP-Update-Nachricht.
* VIP BGP bietet eine Standardroutenautomatisierung mithilfe von BGP Peer-Gruppierung, um die Konfiguration zu vereinfachen.
