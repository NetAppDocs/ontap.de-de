= ONTAP SVM Migration anhalten und fortsetzen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Möglicherweise möchten Sie eine SVM-Migration anhalten, bevor der Migrationsumstieg beginnt. Sie können eine SVM-Migration mit dem  `vserver migrate pause` Befehl anhalten.



[[pause-migration]]
== Migration pausieren

Sie können eine SVM-Migration pausieren, bevor der Client-Cutover beginnt, indem Sie den  `vserver migrate pause` Befehl verwenden.

Einige Konfigurationsänderungen sind eingeschränkt, wenn ein Migrationsvorgang läuft; ab ONTAP 9.12.1 kann jedoch eine Migration angehalten werden, um bestimmte eingeschränkte Konfigurationen und einige Fehlerzustände zu beheben, sodass Konfigurationsprobleme korrigiert werden können, die möglicherweise den Fehler verursacht haben. Einige der Fehlerzustände, die beim Anhalten der SVM-Migration behoben werden können, umfassen Folgendes:

* `setup-configuration-failed`
* `migrate-failed`


.Schritte
. Vom Ziel-Cluster aus die Migration anhalten:
+
[source, cli]
----
vserver migrate pause -vserver <vserver name>
----




[[resume-migrations]]
== Migrationen fortsetzen

Wenn eine pausierte SVM-Migration fortgesetzt werden soll oder wenn eine SVM-Migration fehlgeschlagen ist, kann der  `vserver migrate resume` Befehl verwendet werden.

.Schritte
. Vom Ziel-Cluster aus die SVM-Migration fortsetzen:
+
[source, cli]
----
vserver migrate resume
----
. Es sollte überprüft werden, ob die SVM-Migration wieder aufgenommen wurde, und der Fortschritt kann überwacht werden:
+
[source, cli]
----
vserver migrate show
----
