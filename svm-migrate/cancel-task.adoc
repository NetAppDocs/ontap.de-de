= Eine ONTAP SVM-Migration abbrechen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn eine SVM-Migration vor deren Abschluss abgebrochen werden muss, kann der  `vserver migrate abort` Befehl verwendet werden. Eine SVM-Migration kann nur abgebrochen werden, wenn sich der Vorgang im Status „Angehalten“ oder „Fehlgeschlagen“ befindet. Eine SVM-Migration kann nicht abgebrochen werden, wenn der Status „cutover-started“ ist oder nachdem der Cutover abgeschlossen wurde. Die  `abort` Option kann nicht verwendet werden, wenn eine SVM-Migration im Gange ist.

.Schritte
. Im Ziel-Cluster den Migrationsstatus prüfen:
+
[source, cli]
----
vserver migrate show -vserver <SVM_name>
----
. Migration abbrechen:
+
[source, cli]
----
vserver migrate abort -vserver <SVM_name>
----
. Der Fortschritt des Abbruchvorgangs kann überprüft werden:
+
[source, cli]
----
vserver migrate show
----
+
Der Migrationsstatus zeigt „migrate-aborting“ an, solange der Abbruchvorgang läuft. Nach Abschluss des Abbruchvorgangs wird im Migrationsstatus nichts angezeigt.
