= Automatische Updates aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Verwenden Sie den System Manager, um automatische Updates zu aktivieren und ONTAP zu erlauben, Firmware-, System- und Sicherheitsupdates ohne Ihr Zutun herunterzuladen und zu installieren.

Sie können die ONTAP CLI ausschließlich zur Verwaltung von SP/BMC Firmware-Updates verwenden.

* Weitere Informationen zu link:../system-admin/methods-manage-sp-bmc-firmware-updates-concept.html["Verwaltung von ONTAP SP/BMC Firmware-Updates"].
* Erfahren Sie mehr über die Unterschiede zwischen der Verwaltung automatischer Updates mit System Manager und CLI im Knowledge Base Artikel: link:https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/Why_do_GUI_and_CLI_show_different_status_for_ONTAP_automatic_updates_configuration["Warum zeigen GUI und CLI unterschiedliche Status für die ONTAP-Konfiguration automatischer Updates an?"^].


Die Verfügbarkeit automatischer Updates für die einzelnen Systemkomponenten hängt von Ihrer ONTAP Version ab.

[cols="25,75"]
|===
| ONTAP Version | Verfügbare automatische Updates 


| 9.16.1 und später  a| 
* Autonomer Ransomware-Schutz mit künstlicher Intelligenz (ARP/AI)
* ONTAP Zeitzonendatenbank
* Speicher-Firmware für Speichergeräte, Festplatten und Festplattengehäuse
* SP/BMC Firmware für Serviceprozessoren und BMC Module
* Disk Qualification Package (DQP)




| 9.13.1 und später  a| 
* ONTAP Zeitzonendatenbank
* Speicher-Firmware für Speichergeräte, Festplatten und Festplattengehäuse
* SP/BMC Firmware für Serviceprozessoren und BMC Module
* Disk Qualification Package (DQP)




| 9.10.1 und später  a| 
* Speicher-Firmware für Speichergeräte, Festplatten und Festplattengehäuse
* SP/BMC Firmware für Serviceprozessoren und BMC Module
* Disk Qualification Package (DQP)


|===
Falls für Ihre Version von ONTAP kein automatisches Update verfügbar ist oder Sie automatische Updates nicht aktiviert haben, können Firmware-, Zeitzonendatenbank- und Sicherheitsupdates manuell durchgeführt werden.

.Über diese Aufgabe
Die Aktivierung automatischer Updates erfordert, dass Sie die Bedingungen einer EULA akzeptieren und das Standard-Updateverhalten für jede Systemkomponente entsprechend Ihrer Umgebung bestätigen oder konfigurieren.

Für Cluster, die in ONTAP 9.16.1 oder früher erstellt und dann auf 9.16.1 oder später aktualisiert wurden:

* Wenn die automatische Update-Funktionalität vor dem Upgrade aktiviert ist, werden die Standardeinstellungen für Systemkomponenten-Updates nach einem Upgrade nicht geändert.
* Wenn automatische Updates vor dem Upgrade nicht aktiviert sind, werden die Standardeinstellungen für Systemkomponenten-Updates in ONTAP 9.16.1 und später angewendet.


Die Standardeinstellungen für automatische Systemkomponentenaktualisierungen sind je nach ONTAP Version entweder auf automatische Aktualisierung oder auf die Anzeige von Benachrichtigungen eingestellt. Vor Abschluss des Vorgangs sollte bestätigt werden, dass diese Einstellungen für Ihre Umgebung korrekt sind.

Dies https://www.youtube.com/watch?v=GoABILT85hQ["Video"^] ist eine kurze Übersicht über die Verwendung des automatischen Aktualisierungsprozesses.

.Bevor Sie beginnen
* Sie benötigen eine gültige Supportberechtigung. Dies lässt sich auf der Seite link:https://mysupport.netapp.com/site/["NetApp Support-Website"^]*Systemdetails* prüfen.
* Um automatische Updates zu aktivieren, muss zunächst AutoSupport mit HTTPS aktiviert werden. Wenn AutoSupport in Ihrem Cluster nicht aktiviert ist oder wenn AutoSupport in Ihrem Cluster mit einem anderen Transportprotokoll aktiviert ist, kann es im Rahmen dieses Vorgangs mit HTTPS aktiviert werden.



NOTE: AutoSupport OnDemand ist standardmäßig aktiviert und funktionsfähig, wenn es so konfiguriert ist, dass Nachrichten über das HTTPS-Transportprotokoll an den technischen Support gesendet werden.

* Ab ONTAP 9.10.1 ist bei aktivierten automatischen Updates sicherzustellen, dass eine HTTPS-Verbindung zu den folgenden zusätzlichen URLs besteht:
+
** \https://support-sg-naeast.netapp.com
** \https://support-sg-nawest.netapp.com




.Schritte
[role="tabbed-block"]
====
.System Manager – ONTAP 9.16.1 und höher
--
. Im System Manager *Cluster > Einstellungen* auswählen.
. Wählen Sie unter *Software updates* die Option *Enable*.
. Unter *Automatische Update-Einstellungen* wählen Sie die Aktion aus, die für jeden Update-Typ durchgeführt werden soll.
+
*Automatisch aktualisieren* ist standardmäßig für alle Optionen ausgewählt. Sie können die Standardauswahl ändern in *Benachrichtigungen anzeigen* oder *Aktualisierungen automatisch verwerfen*.

. Wählen Sie *Ich akzeptiere das aktuelle und alle zukünftigen Updates* und anschließend *Speichern*.


--
.System Manager – ONTAP 9.15.1 und früher
--
. Im System Manager *Cluster > Einstellungen* auswählen.
. Wählen Sie unter *Automatische Aktualisierung* image:icon_kabob.gif["Menüauswahl"] und dann *Aktivieren* aus.
+
*Automatisch aktualisieren* ist standardmäßig für alle Optionen ausgewählt. Sie können die Standardauswahl ändern in *Benachrichtigungen anzeigen* oder *Aktualisierungen automatisch verwerfen*.

. Wählen Sie *Ich akzeptiere aktuelle und alle zukünftigen Aktualisierungen* und anschließend *Speichern*.


--
.CLI
--
Sie können die ONTAP CLI nur zur Verwaltung von SP/BMC-Firmware-Updates verwenden

. Automatische Firmware-Updates aktivieren:
+
[source, cli]
----
system service-processor image modify -node <node_name> -autoupdate true
----
