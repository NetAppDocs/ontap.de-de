= Wie automatische Updates für die Installation geplant werden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Alle berechtigten Knoten innerhalb desselben Clusters werden für automatische Updates zusammengefasst. Der Zeitraum, in dem die berechtigten Knoten für automatische Updates eingeplant werden, variiert je nach Prioritätsstufe des Updates und dem Prozentsatz der Systeme in Ihrer Umgebung, die das Update benötigen.

Wenn beispielsweise 10 % oder weniger Ihrer gesamten Systeme für ein Update ohne Priorität in Frage kommen, wird das Update innerhalb einer Woche für alle berechtigten Systeme eingeplant. Wenn jedoch 76 % oder mehr Ihrer gesamten Systeme für ein Update ohne Priorität in Frage kommen, wird das Update über einen Zeitraum von acht Wochen gestaffelt auf die berechtigten Systeme verteilt. Diese gestaffelte Installation trägt dazu bei, Risiken für Ihre gesamte Umgebung zu minimieren, falls bei einem Update ein Problem auftritt, das behoben werden muss.

Der prozentuale Anteil Ihrer gesamten Systeme, die wöchentlich für automatische Updates vorgesehen sind, stellt sich wie folgt dar:

| 50-100% | 30% | 70%
