[role="lead"]
Sie können LUNs erstellen und diese einer vorhandenen Storage-VM hinzufügen, die mit dem SAN-Protokoll konfiguriert ist. LUNs können außerdem gruppiert oder umbenannt werden.



[[add-luns]]
== LUNs hinzufügen

image:workflow_admin_add_LUNs.gif["Schritte zum Hinzufügen von LUNs"]

.Bevor Sie beginnen
Eine Storage-VM, die für den SAN-Dienst konfiguriert ist, sollte im Cluster vorhanden sein.

.Schritte
. Zu *Speicher > LUNs* wechseln.
. Klicken Sie image:icon_add.gif["Symbol hinzufügen"].
. Geben Sie ein Präfix an, das am Anfang jedes LUN-Namens verwendet wird. (Wenn nur eine LUN erstellt wird, den LUN-Namen eingeben.)
. Eine Storage-VM aus der Dropdown-Liste auswählen.
+
Es werden nur Speicher-VMs aufgelistet, die für das SAN-Protokoll konfiguriert sind. Wenn nur eine Speicher-VM verfügbar ist, die für das SAN-Protokoll konfiguriert ist, wird das Feld *Storage VM* nicht angezeigt.

. Geben Sie an, wie viele LUNs erstellt werden sollen und welche Größe jede LUN haben soll.
. Das Host-Betriebssystem und das LUN-Format können aus den Dropdown-Listen ausgewählt werden.
. Die Host-Initiatoren werden eingegeben und durch Kommas getrennt.
. Führen Sie eine der folgenden Aktionen aus:
+
[cols="30,70"]
|===


| Klicken Sie auf diese Schaltfläche... | Um diese Aktion durchzuführen... 


| *Speichern* | Die LUNs werden gemäß Ihren Angaben erstellt. Für andere Spezifikationen werden die Systemvorgaben verwendet. Es sind keine weiteren Schritte erforderlich. 


| *Weitere Optionen* | Fahren Sie mit <<step9-define-add-specs>> fort, um zusätzliche Spezifikationen für die LUNs zu definieren. 
|===
. [[step9-define-add-specs,Schritt 9]] Das LUN-Präfix wird bereits angezeigt, wenn Sie es zuvor eingegeben haben, kann jedoch geändert werden. Andernfalls das Präfix eingeben.
. Eine Storage-VM aus der Dropdown-Liste auswählen.
+
Es werden nur Speicher-VMs aufgelistet, die für das SAN-Protokoll konfiguriert sind. Wenn nur eine Speicher-VM verfügbar ist, die für das SAN-Protokoll konfiguriert ist, wird das Feld *Storage VM* nicht angezeigt.

. Bestimmen Sie, wie die LUNs gruppiert werden sollen:
+
[cols="40,60"]
|===


| Wenn diese Entscheidung getroffen wird... | Dies geschieht... 


| *Gruppe mit zugehörigen LUNs* | Die LUNs werden zusammen mit verwandten LUNs auf einem vorhandenen Volume auf der Storage-VM gruppiert. 


| Keine Auswahl | Die LUNs werden auf einem Volume namens „container“ zusammengefasst. 
|===
. Im Abschnitt *Speicherung und Optimierung* sind die folgenden Werte anzugeben:
+
.. Anzahl und Kapazität der LUNs werden bereits angezeigt, wenn Sie diese zuvor eingegeben haben, können jedoch geändert werden. Andernfalls sind die Werte einzugeben.
.. Im Feld *Performance Service Level* wird ein Servicelevel ausgewählt:
+
[cols="55,45"]
|===


| Wenn diese Servicestufe ausgewählt wird... | Dies geschieht... 


 a| 
Ein bestehendes Servicelevel, wie „Extreme“, „Performance“ oder „Value“.

Es werden nur die Servicelevel angezeigt, die für die Systemplattform (AFF, FAS oder andere) gültig sind.
| Es wird automatisch eine lokale Ebene ausgewählt. Weiter zu <<step13>>. 


| Benutzerdefiniert | Weiter zu <<step12c>> zur Definition eines neuen Servicelevels. 
|===
.. [[step12c, Schritt 12c]] Ab ONTAP 9.9.1 kann im System Manager manuell die lokale Ebene ausgewählt werden, auf der die LUNs platziert werden, die erstellt werden sollen (wenn die Dienstebene „Benutzerdefiniert“ ausgewählt wurde).
+
[cols="55,45"]
|===


| Wenn diese Entscheidung getroffen wird... | Sie führen folgende Schritte aus... 


| *Manuelle Platzierung* | Die manuelle Platzierung ist aktiviert. Weiter zu <<step12d>> , um den Vorgang abzuschließen. 


| Keine Auswahl | Die manuelle Auswahl ist nicht aktiviert. Die lokale Ebene wird automatisch ausgewählt. Weiter zu <<step13>>. 
|===
.. [[step12d, Schritt 12d]]Eine lokale Tier kann im Dropdown-Menü ausgewählt werden.
.. Eine QoS-Richtlinie auswählen.
+
„Vorhanden“ ermöglicht die Auswahl aus einer Liste bestehender Richtlinien, während „Neu“ die Eingabe der Spezifikationen einer neuen Richtlinie erlaubt.



. [[step13,Schritt 13]] Im Abschnitt *Hostinformationen* werden das Host-Betriebssystem und das LUN-Format bereits angezeigt, sie können jedoch geändert werden.
. Unter *Hostzuordnung* den Typ der Initiatoren für die LUNs auswählen:
+
** *Vorhandene Initiatorgruppe*: Eine Initiatorgruppe aus der angezeigten Liste auswählen.
** *Neue Initiatorgruppe unter Verwendung vorhandener Initiatorgruppen*: Der Name der neuen Gruppe ist anzugeben, und die Gruppe oder Gruppen, die zur Erstellung der neuen Gruppe verwendet werden sollen, sind auszuwählen.
** *Host-Initiatoren*: Ein Name aus der neuen Initiatorgruppe wird angegeben, und mit *+Initiator hinzufügen* werden Initiatoren zur Gruppe hinzugefügt.


. Im Abschnitt *Schutz* sind die Schutzmaßnahmen für die LUNs anzugeben.
+
Wenn *SnapMirror (Lokal oder Remote) aktivieren* ausgewählt wird, werden die Datensicherungsstrategie und die Einstellungen für den Ziel-Cluster aus den Dropdown-Listen angegeben.

. Auf *Speichern* klicken.
+
Die LUNs werden erstellt und dem Cluster sowie der Storage-VM hinzugefügt.

+

NOTE: Sie können die Spezifikationen dieser LUNs auch in einem Ansible Playbook speichern. Weitere Informationen finden Sie unter link:task_admin_use_ansible_playbooks_add_edit_volumes_luns.html["Ansible Playbooks können verwendet werden, um Volumes oder LUNs hinzuzufügen oder zu bearbeiten."].





[[rename-a-lun]]
== Eine LUN umbenennen

Eine LUN kann auf der Übersichtsseite umbenannt werden.

.Schritte
. Im System Manager auf *LUNs* klicken.
. Klicken Sie image:icon-edit-pencil-blue-outline.png["Bearbeiten-Symbol"] neben den Namen der LUN, die Sie umbenennen möchten, und ändern Sie dann den LUN-Namen.
. Auf *Speichern* klicken.
