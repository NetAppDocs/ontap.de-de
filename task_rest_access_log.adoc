= Zugriff auf das REST API-Protokoll
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Sie können auf das Protokoll zugreifen, das eine Aufzeichnung der vom System Manager durchgeführten ONTAP REST API-Aufrufe enthält. Beim Anzeigen des Protokolls besteht außerdem die Möglichkeit, API-Aufrufe erneut auszuführen und die Ausgabe zu überprüfen.

.Schritte
. Oben auf der Seite image:icon_double_arrow.gif["Anzeigesymbol"] wird das REST-API-Protokoll angezeigt.
+
Die aktuellsten Einträge werden am unteren Ende der Seite angezeigt.

. Links auf *DASHBOARD* klicken und die neuen Einträge beobachten, die für die zum Aktualisieren der Seite ausgegebenen API-Aufrufe erstellt werden.
. Auf *SPEICHER* und dann auf *Qtrees* klicken.
+
Dies veranlasst System Manager, einen spezifischen API-Aufruf durchzuführen, um eine Liste der Qtrees abzurufen.

. Den Logeintrag finden, der den API-Aufruf beschreibt und folgendes Format hat:
+
`GET /api/storage/qtrees`

+
Sie werden zusätzliche HTTP-Abfrageparameter im Eintrag sehen, wie  `max_records`.

. Klicken Sie auf den Protokolleintrag, um den GET API-Aufruf erneut auszuführen und die unformatierte JSON-Ausgabe anzuzeigen.
+
