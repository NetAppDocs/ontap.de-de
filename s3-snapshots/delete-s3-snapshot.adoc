= ONTAP S3 Snapshots löschen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können nicht mehr benötigte S3 Snapshots löschen und so Speicherplatz in Ihren Buckets freigeben. Sie können S3 Snapshots manuell entfernen oder die den S3 Buckets zugeordneten Snapshot Richtlinien ändern, um die Anzahl der aufzubewahrenden Snapshots für einen Zeitplan zu ändern.

Snapshot-Richtlinien für S3-Buckets folgen denselben Löschregeln wie die herkömmlichen ONTAP Snapshot-Richtlinien. Weitere Informationen zum Erstellen von Snapshot-Richtlinien sind unter link:../data-protection/create-snapshot-policy-task.html["Erstellen einer Snapshot-Richtlinie"] zu finden.

.Über diese Aufgabe
* Wenn eine Objektversion (in einem versionierten Bucket) oder ein Objekt (in einem nicht versionierten Bucket) in mehreren Snapshots erfasst wird, wird das Objekt erst aus dem Dateisystem entfernt, nachdem der letzte Snapshot, der es schützt, gelöscht wurde.
* Wenn Sie einen S3-Bucket löschen müssen, der Objekte mit Snapshots enthält, stellen Sie sicher, dass Sie alle Snapshots aller Objekte in diesem Bucket gelöscht haben.
* Wenn ein Cluster auf eine ONTAP Version vor ONTAP 9.16.1 zurückgesetzt werden muss, ist sicherzustellen, dass alle S3 Snapshots für alle Buckets gelöscht wurden. Es kann außerdem erforderlich sein, den  `vserver object-store-server bucket clear-snapshot-metadata` Befehl auszuführen, um die Snapshot-Metadaten für einen S3 Bucket zu entfernen. Informationen dazu finden sich unter link:../s3-snapshots/delete-s3-snapshot.html#clear-s3-snapshots-metadata["S3-Snapshots-Metadaten löschen"].
* Beim Löschen von Snapshots in Stapeln kann eine große Anzahl von Objekten entfernt werden, die in mehreren Snapshots erfasst wurden, wodurch effektiv mehr Speicherplatz freigegeben wird, als es beim Löschen einzelner Snapshots der Fall wäre. Dadurch kann mehr Speicherplatz für Ihre Speicherobjekte zurückgewonnen werden.


.Schritte
. Um einen bestimmten S3 Snapshot zu löschen, diesen Befehl ausführen:
+
[listing]
----
vserver object-store-server bucket snapshot delete -vserver <svm_name> -bucket <bucket_name> -snapshot <snapshot_name>
----
. Um alle S3-Snapshots in einem Bucket zu entfernen, diesen Befehl ausführen:
+
[listing]
----
vserver object-store-server bucket snapshot delete -vserver <svm_name> -bucket <bucket_name> -snapshot *
----




[[clear-s3-snapshots-metadata]]
== S3-Snapshots-Metadaten löschen

Bei S3-Snapshots werden auch Snapshot-Metadaten in einem Bucket generiert. Die Snapshot-Metadaten bleiben im Bucket erhalten, selbst wenn alle Snapshots daraus entfernt werden. Das Vorhandensein von Snapshot-Metadaten blockiert die folgenden Operationen:

* Cluster auf eine ONTAP Version vor ONTAP 9.16.1 zurücksetzen
* Konfiguration von SnapMirror S3 auf dem Bucket


Vor der Durchführung dieser Operationen sollten alle Snapshot-Metadaten aus dem Bucket gelöscht werden.

.Bevor Sie beginnen
Stellen Sie sicher, dass Sie alle S3-Snapshots aus einem Bucket entfernt haben, bevor Sie mit dem Löschen der Metadaten beginnen.

.Schritte
. Um die Snapshot-Metadaten aus einem Bucket zu löschen, diesen Befehl ausführen:


[listing]
----
vserver object-store-server bucket clear-snapshot-metadata -vserver <svm_name> -bucket <bucket_name>
----
