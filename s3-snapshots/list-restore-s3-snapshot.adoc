= ONTAP S3 Snapshots anzeigen und wiederherstellen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.16.1 können Sie S3 Snapshot-Daten Ihrer Buckets von S3 Clients aus anzeigen und durchsuchen. Ab ONTAP 9.18.1 ist der S3 Snapshot Bucket nativ über die ONTAP CLI zugänglich. Zusätzlich besteht die Möglichkeit, auf einem S3 Client ein einzelnes Objekt, eine Gruppe von Objekten oder einen gesamten Bucket aus einem S3 Snapshot wiederherzustellen.

.Bevor Sie beginnen
* Alle Knoten im Cluster müssen ONTAP 9.18.1 oder höher ausführen, bevor die Bucket-Snapshot-Wiederherstellung nativ in der ONTAP CLI durchgeführt werden kann. Ab ONTAP 9.18.1 ist der S3 Browser nicht mehr erforderlich, die Operationen werden aber weiterhin unterstützt.
* Für einen bestimmten Bucket ist jeweils nur eine Snapshot-Wiederherstellung zulässig.


.Über diese Aufgabe
Ab ONTAP 9.16.1 bietet die ONTAP S3 Snapshots-Funktion grundlegende Snapshot-Funktionalität für ONTAP S3 Buckets, einschließlich manueller und geplanter Snapshot-Erstellung und -Löschung, Snapshot-Richtlinien für S3 Buckets sowie S3-Client-basiertem Durchsuchen von Snapshots.

Ab ONTAP 9.18.1 wird die native ONTAP Snapshot-Wiederherstellung unterstützt, wodurch ONTAP-Administratoren eine zeitpunktgenaue Restore-Funktionalität ohne die Verwendung eines S3-Browsers zur Verfügung steht. Im Snapshot wird nur die aktuelle Bucket-Version erfasst. Der Versionsverlauf wird nicht erfasst und wird durch die S3 Snapshot Restore-Operation nicht wiederhergestellt.



[[list-and-view-s3-snapshots]]
== S3 Snapshots auflisten und anzeigen

Sie können die Details der S3-Snapshots anzeigen, vergleichen und Fehler identifizieren. Mit der ONTAP CLI können alle auf Ihren S3-Buckets erstellten Snapshots aufgelistet werden.

.Schritte
. S3-Snapshots auflisten:
+
[listing]
----
vserver object-store-server bucket snapshot show
----
+
Sie können die Snapshot-Namen, Storage-VMs, Buckets, Erstellungszeiten und Instance-UUIDs der für alle Ihre Buckets im Cluster erstellten S3-Snapshots anzeigen.

. Sie können auch einen Bucket-Namen angeben, um die Namen, Erstellungszeiten und Instanz-UUIDs aller für diesen spezifischen Bucket erstellten S3-Snapshots anzuzeigen.
+
[listing]
----
vserver object-store-server bucket snapshot show -vserver <svm_name> -bucket <bucket_name>
----




[[browse-s3-snapshots-content]]
== S3 Snapshot-Inhalte durchsuchen

Wenn Sie in Ihrer Umgebung Fehler oder Probleme feststellen, können Sie die Inhalte der S3-Bucket-Snapshots durchsuchen, um die Fehler zu identifizieren. Sie können die S3-Snapshots auch durchsuchen, um die fehlerfreien Inhalte zu ermitteln, die wiederhergestellt werden können.

S3-Snapshots werden S3-Clients als Snapshot-Buckets präsentiert. Der Name des Snapshot-Buckets hat das Format  `<base_bucket_name>-s3snap-<snapshot_name>`. Alle Snapshot-Buckets in einer Storage-VM sind über die  `ListBuckets` S3-API-Operation sichtbar.

Der S3-Snapshot-Bucket erbt die Zugriffsrichtlinien des Basis-Buckets und unterstützt ausschließlich Leseoperationen. Lösch- und schreibbasierte Vorgänge sind untersagt. Wenn Sie Berechtigungen für den Zugriff auf den Basis-Bucket haben, können Sie auch Leseoperationen der S3-API auf dem S3-Snapshot-Bucket ausführen, wie zum Beispiel  `HeadObject`,  `GetObject`,  `GetObjectTagging`,  `ListObjects`,  `ListObjectVersions`,  `GetObjectAcl` und  `CopyObject`.


NOTE: Die  `CopyObject`Operation wird für einen S3-Snapshot-Bucket nur unterstützt, wenn es sich um einen Snapshot des Quell-Buckets handelt, nicht wenn es sich um das Speicherziel des Snapshots handelt.

Weitere Informationen zu diesen Vorgängen sind unter link:../s3-config/ontap-s3-supported-actions-reference.html["ONTAP S3 unterstützte Aktionen"] zu finden.



[[restore-a-bucket-from-s3-snapshots-using-ontap]]
== Einen Bucket aus S3-Snapshots mit ONTAP wiederherstellen

Ab ONTAP 9.18.1 kann die ONTAP CLI verwendet werden, um einen gesamten Bucket mithilfe eines ONTAP S3 Snapshots wiederherzustellen. Es kann nur die Version des Buckets wiederhergestellt werden, die zum Zeitpunkt der Erstellung des ausgewählten Snapshots existierte.

.Schritte
. Identifizieren Sie den Snapshot, den Sie zum Wiederherstellen des Buckets verwenden möchten:
+
[listing]
----
vserver object-store-server bucket snapshot show
----
. Stellen Sie den Bucket wieder her:
+
[listing]
----
vserver object-store-server bucket snapshot restore start -vserver <storage VM name> -bucket <bucket name> -snapshot <snapshot name>
----




[[restore-data-from-s3-bucket-snapshots-using-an-s3-client]]
== Daten aus S3-Bucket-Snapshots mit einem S3-Client wiederherstellen

Zusätzlich zur Wiederherstellung eines gesamten Buckets in ONTAP ist es auch möglich, ein einzelnes Objekt, eine Gruppe von Objekten oder einen gesamten Bucket aus einem S3-Snapshot mithilfe eines S3-Clients wie S3cmd oder S3 Browser wiederherzustellen.

link:index.html#snapshots-on-versioned-and-non-versioned-buckets["Weitere Informationen zu versionierten und nicht versionierten Snapshots."]

Sie können den gesamten Bucket, Objekte mit einem bestimmten Präfix oder ein einzelnes Objekt mit dem  `aws s3 cp` Befehl wiederherstellen.

.Schritte
. Einen Snapshot des Basis-S3-Buckets erstellen.
+
[source, cli]
----
vserver object-store-server bucket snapshot create -vserver <svm_name> -bucket <base_bucket_name> -snapshot <snapshot_name>
----
. Das Basis-Bucket wird mithilfe des Snapshots wiederhergestellt:
+
** Einen gesamten Bucket wiederherstellen. Den Snapshot-Bucket-Namen im Format  `<base_bucket_name>-s3snap-<snapshot_name>` verwenden.
+
[source, cli]
----
aws --endpoint http://<IP> s3 cp s3://<snapshot-bucket-name> s3://<base-bucket>  --recursive
----
** Objekte in einem Verzeichnis mit dem Präfix  `dir1` wiederherstellen:
+
[source, cli]
----
aws --endpoint http://<IP> s3 cp s3://<snapshot-bucket-name>/dir1 s3://<base_bucket_name>/dir1 --recursive
----
** Ein einzelnes Objekt mit dem Namen  `web.py` wiederherstellen:
+
[source, cli]
----
aws --endpoint http://<IP> s3 cp s3:// <snapshot-bucket-name>/web.py s3://<base_bucket_name>/web.py
----
