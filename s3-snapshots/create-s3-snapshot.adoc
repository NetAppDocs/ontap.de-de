= ONTAP S3 Snapshots erstellen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie können S3-Snapshots entweder manuell erstellen oder Snapshot-Richtlinien einrichten, die automatisch S3-Snapshots für Sie erstellen. Snapshots dienen als statische Kopien von Objekten, die für Backup und Recovery verwendet werden. Zur Bestimmung der Aufbewahrungsdauer von Snapshots können Snapshot-Richtlinien erstellt werden, die die automatische Erstellung von Snapshots in festgelegten Intervallen ermöglichen.

S3 Snapshots helfen dabei, Objektdaten in S3 Buckets mit oder ohne aktivierte Objektversionierung zu schützen.


TIP: Snapshots können besonders nützlich sein, um den Datenschutz zu gewährleisten, wenn die Objektversionierung in einem S3-Bucket nicht aktiviert ist, da sie als zeitpunktgenaue Aufzeichnungen dienen, die für Wiederherstellungsvorgänge verwendet werden können, wenn keine frühere Objektversion verfügbar ist.

.Über diese Aufgabe
* Für Snapshots gelten folgende Namenskonventionen (sowohl für manuelle als auch für automatische Snapshots):
+
** S3 Snapshot Namen können bis zu 30 Zeichen lang sein
** S3 Snapshot Namen dürfen nur aus Kleinbuchstaben, Zahlen, Punkten (.) und Bindestrichen (-) bestehen.
** S3 Snapshot Namen müssen mit einem Buchstaben oder einer Zahl enden.
** S3 Snapshot Namen dürfen keine Teilzeichenfolgen enthalten `s3snap`


* Im Kontext des S3-Protokolls ist der Bucket-Name auf 63 Zeichen beschränkt. Da ONTAP S3 Snapshots über das S3-Protokoll als Buckets präsentiert werden, gelten ähnliche Beschränkungen auch für die Namen der Snapshot-Buckets. Standardmäßig wird der ursprüngliche Bucket-Name als Basis-Bucket-Name verwendet.
* Um die Zuordnung von Snapshots zu Buckets zu vereinfachen, besteht der Name des Snapshot-Buckets aus dem Namen des Basis-Buckets und einer speziellen Zeichenfolge, `-s3snap-`, die dem Snapshot-Namen vorangestellt wird. Die Namen der Snapshot-Buckets haben das Format  `<base_bucket_name>-s3snap-<snapshot_name>`.
+
Wenn zum Beispiel der folgende Befehl ausgeführt wird, um  `snap1` auf  `bucket-a` einen Snapshot-Bucket mit dem Namen  `bucket-a-s3snap-snap1` zu erstellen, ist dieser über S3-Clients zugänglich, sofern Sie die Berechtigung zum Zugriff auf den Basis-Bucket haben.

+
[listing]
----
vserver object-store-server bucket snapshot create -bucket bucket-a -snapshot snap1
----
* Es ist nicht möglich, einen Snapshot zu erstellen, dessen Snapshot-Bucket-Name mehr als 63 Zeichen enthält.
* Der Name des automatischen Snapshots enthält den Namen des Richtlinienzeitplans und den Zeitstempel, was der Namenskonvention für herkömmliche Volume Snapshots entspricht. Beispielsweise können die Namen der geplanten Snapshots  `daily-2024-01-01-0015` und  `hourly-2024-05-22-1105` sein.




[[manually-create-s3-snapshots]]
== S3-Snapshots manuell erstellen

Sie können einen S3-Snapshot manuell mithilfe der ONTAP CLI erstellen. Das Verfahren erstellt einen Snapshot nur auf dem lokalen Cluster.

.Schritte
. Einen S3-Snapshot erstellen:
+
[listing]
----
vserver object-store-server bucket snapshot create -vserver <svm_name> -bucket <bucket_name> -snapshot <snapshot_name>
----
+
Das folgende Beispiel erstellt einen Snapshot mit dem Namen  `pre-update` auf der  `vs0` Storage-VM und dem  `website-data` Bucket:

+
[listing]
----
vserver object-store-server bucket snapshot create -vserver vs0 -bucket website-data -snapshot pre-update
----




[[assign-an-s3-snapshot-policy-to-a-bucket]]
== Einem Bucket eine S3 Snapshot-Richtlinie zuweisen

Wenn Sie Snapshot-Richtlinien auf S3-Bucket-Ebene konfigurieren, erstellt ONTAP automatisch geplante S3-Snapshots für Sie. Wie bei herkömmlichen Snapshot-Richtlinien können bis zu fünf Zeitpläne für S3-Snapshots konfiguriert werden.

Eine Snapshot-Richtlinie legt typischerweise die Zeitpläne für die Erstellung von Snapshots, die Anzahl der für jeden Zeitplan aufzubewahrenden Kopien und das Zeitplanpräfix fest. Beispielsweise kann eine Richtlinie täglich um 0:10 Uhr einen S3 Snapshot erstellen, die beiden aktuellsten Kopien aufbewahren und diese  `daily-<timestamp>` benennen.

Die standardmäßige Snapshot-Richtlinie bewahrt Folgendes auf:

* Sechs stündliche Snapshots
* Zwei tägliche Snapshots
* Zwei wöchentliche Snapshots


.Bevor Sie beginnen
* Vor der Zuweisung zum S3-Bucket muss eine Snapshot-Richtlinie erstellt worden sein.



NOTE: Für S3 Snapshots gelten dieselben Regeln wie für andere ONTAP Snapshot Richtlinien. Eine Snapshot Richtlinie, deren Aufbewahrungsfrist in einem der Snapshot Zeitpläne konfiguriert ist, kann jedoch keinem S3 Bucket zugewiesen werden.

Weitere Informationen zum Erstellen von Snapshot-Richtlinien für die automatische Generierung von Snapshots sind unter link:../data-protection/configure-custom-snapshot-policies-concept.html["Übersicht zur Konfiguration benutzerdefinierter Snapshot-Richtlinien"] zu finden.

.Schritte
. Die Snapshot-Richtlinie wird Ihrem Bucket zugewiesen:
+
[listing]
----
vserver object-store-server bucket create -vserver <svm_name> -bucket <bucket_name> -snapshot-policy <policy_name>
----
+
oder

+
[listing]
----
vserver object-store-server bucket modify -vserver <svm_name> -bucket <bucket_name> -snapshot-policy <policy_name>
----



NOTE: Falls ein Cluster auf eine ONTAP Version vor ONTAP 9.16.1 zurückgesetzt werden muss, ist sicherzustellen, dass der Wert für  `snapshot-policy` für alle Buckets auf  `none` (oder  `-`) gesetzt ist.
