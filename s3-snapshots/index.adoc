= Informationen zu ONTAP S3 Snapshots
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.16.1 kann die ONTAP Snapshot-Technologie verwendet werden, um schreibgeschützte, zeitpunktgenaue Images der ONTAP S3 Buckets zu erzeugen.

Mit der S3 Snapshots Funktion lassen sich Snapshots manuell erstellen oder automatisch über Snapshot Richtlinien generieren. S3 Snapshots werden S3 Clients als S3 Buckets präsentiert. Das Durchsuchen und Wiederherstellen der Inhalte aus den Snapshots ist über S3 Clients möglich.

In ONTAP 9.16.1 erfassen S3 Snapshots nur die aktuellen Versionen der Objekte in S3 Buckets. Die nicht aktuellen Versionen von versionierten Buckets werden in den S3 Snapshots nicht erfasst. Außerdem werden die zeitpunktgenauen Objekt-Tags in den Snapshots nicht erfasst, wenn die Objekt-Tags nach der Erstellung der Snapshots geändert werden.


TIP: S3 Snapshots sind von der Clusterzeit abhängig. Der NTP-Server sollte im Cluster konfiguriert werden, um die Zeit zu synchronisieren. Weitere Informationen finden sich unter link:../system-admin/manage-cluster-time-concept.html["Clusterzeit verwalten"].

.Quota- und Platznutzung
Quotas erfassen die Anzahl der Objekte und die logische Größe, die in einem S3-Bucket verwendet werden. Wenn S3-Snapshots erstellt werden, werden die in den S3-Snapshots erfassten Objekte auf die Anzahl der Objekte und die verwendete Größe des Buckets angerechnet, bis die Snapshots aus dem Dateisystem gelöscht werden.

.Mehrteilige Objekte
Bei mehrteiligen Objekten werden nur die finalen Objekte in Snapshots erfasst. Teilweise Uploads mehrteiliger Objekte werden nicht in Snapshots erfasst.



[[snapshots-on-versioned-and-non-versioned-buckets]]
== Snapshots von versionierten und nicht versionierten Buckets

Sie können Snapshots sowohl für versionierte als auch für nicht versionierte Buckets erstellen. Der Snapshot enthält nur die aktuellen Objektversionen zum Zeitpunkt der Snapshot-Erstellung.

.Versionierte Buckets und Snapshots
In Buckets mit aktivierter Objektversionierung speichert ein Snapshot den Inhalt der aktuellsten Objektversion, nach der der Snapshot erstellt wurde. Nicht-aktuelle Versionen im Bucket werden ausgeschlossen.

Betrachten Sie dieses Beispiel: In einem Bucket, in dem die Objektversionierung aktiviert ist, hat das Objekt `obj1` die Versionen v1, v2, v3, v4, v5. Ein Snapshot `snap1` wurde von `obj1` v3 (der zum Zeitpunkt der Aufnahme aktuellsten Version) erstellt. Beim Durchsuchen `snap1`, `obj1` erscheint das Objekt mit dem Inhalt, der bei v3 erstellt wurde. Inhalte der vorherigen Versionen werden nicht zurückgegeben.


NOTE: Die nicht aktuellen Versionen bleiben im Dateisystem erhalten, bis die Snapshots gelöscht werden.

.Nicht versionierte Buckets und Snapshots
In nicht versionierten Buckets bewahren S3-Snapshots den Inhalt der letzten Commits vor der Erstellung des Snapshots.

Betrachten Sie dieses Beispiel: In einem Bucket, in dem keine Objektversionierung verfügbar ist, wurde das Objekt `obj1`mehrmals überschrieben (zu den Zeitpunkten t1, t2, t3, t4 und t5). Ein S3-Snapshot `snap1`wurde irgendwann zwischen t3 und t4 erstellt. Beim Durchsuchen `snap1`, `obj1`erscheint das Objekt mit dem Inhalt, der zu t3 erstellt wurde.



[[object-expiration-and-snapshots]]
== Objektablauf und Snapshots

Die Funktionen ONTAP S3 Object Expiration und S3 Snapshots arbeiten unabhängig voneinander. Die ONTAP Object Expiration Funktion lässt Objektversionen gemäß den für den S3-Bucket definierten Lifecycle-Management-Regeln ablaufen. S3 Snapshots sind statische Kopien der Bucket-Objekte zu einem Zeitpunkt, an dem der Snapshot erstellt wird.

Wenn die Objektversionierung in einem Bucket aktiviert ist, bleibt beim Löschen einer bestimmten Version eines Objekts aufgrund einer für diesen Bucket definierten Ablaufregel der Inhalt der abgelaufenen Objektversion weiterhin im Dateisystem erhalten, sofern diese Version als aktuelle Version in einem oder mehreren S3-Snapshots erfasst wurde. Diese Objektversion existiert im Dateisystem erst dann nicht mehr, wenn der entsprechende Snapshot gelöscht wird.

In einem Bucket, in dem die Versionierung deaktiviert ist, bleibt ein Objekt, das aufgrund einer Ablaufregel gelöscht wird, aber noch in vorhandenen S3-Snapshots erfasst ist, im Dateisystem erhalten. Das Objekt wird dauerhaft aus dem Dateisystem entfernt, wenn die Snapshots, die es erfassen, gelöscht werden.

Informationen zum Ablauf von S3-Objekten und zum Lebenszyklusmanagement finden sich unter link:../s3-config/create-bucket-lifecycle-rule-task.html["Erstellen einer Bucket Lifecycle Management-Regel"].



[[limitations-with-s3-snapshots]]
== Einschränkungen bei S3 Snapshots

Beachten Sie die folgenden Funktionsausschlüsse und Szenarien in ONTAP 9.16.1:

* Sie können bis zu 1023 Snapshots für einen S3-Bucket generieren.
* Alle S3-Snapshots und Metadaten müssen aus allen Buckets in einem Cluster gelöscht werden, bevor der Cluster auf eine ONTAP Version vor ONTAP 9.16.1 zurückgesetzt wird.
* Wenn Sie einen S3-Bucket löschen müssen, der Objekte mit Snapshots enthält, muss sichergestellt sein, dass alle entsprechenden Snapshots aller Objekte in diesem Bucket gelöscht wurden.
* S3 Snapshots werden in diesen Konfigurationen nicht unterstützt:
+
** Zu Buckets in einer SnapMirror Beziehung
** Bei Buckets, in denen die Objektsperrung aktiviert ist
** In der NetApp Console
** Im System Manager
** In ONTAP MetroCluster Konfigurationen


* S3-Snapshots werden für Buckets, die als lokale oder Remote FabricPool Kapazitätsebene verwendet werden, nicht empfohlen.
