= Das ONTAP Softwarepaket für manuelle Upgrades installieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem Sie das ONTAP Softwarepaket für ein manuelles Upgrade heruntergeladen haben, muss es lokal installiert werden, bevor das Upgrade begonnen wird.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein und geben Sie *y* ein, wenn Sie zur Fortsetzung aufgefordert werden: `set -privilege advanced`
+
Die erweiterte Eingabeaufforderung (`*>`) erscheint.

. Das Image wird installiert.
+
[cols="2"]
|===
| Wenn Sie die folgende Konfiguration haben... | Verwenden Sie diesen Befehl... 


 a| 
** Nicht MetroCluster
** 2-Node MetroCluster

 a| 
[source, cli]
----
system node image update -node * -package <location> -replace-package true -setdefault true -background true
----
`<location>` kann je nach ONTAP Version ein Webserver oder ein lokaler Ordner sein. Weitere Informationen zu  `system node image update` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-node-image-update.html["ONTAP-Befehlsreferenz"^].

Dieser Befehl installiert das Software-Image gleichzeitig auf allen Knoten. Um das Image nacheinander auf jedem Knoten zu installieren, darf der  `-background` Parameter nicht angegeben werden.



 a| 
** 4-Node MetroCluster
** 8-Node MetroCluster Konfiguration

 a| 
[source, cli]
----
system node image update -node * -package <location> -replace-package true -background true -setdefault false
----
Dieser Befehl muss auf beiden Clustern ausgeführt werden.

Dieser Befehl verwendet eine erweiterte Abfrage, um das Zielsoftware-Image zu ändern, das als alternatives Image auf jedem Knoten installiert ist.

|===
. Geben Sie  `y` ein, um fortzufahren, wenn Sie dazu aufgefordert werden.
. Es sollte sichergestellt sein, dass das Software-Image auf jedem Knoten installiert ist.
+
[source, cli]
----
system node image show-update-progress -node *
----
+
Dieser Befehl zeigt den aktuellen Status der Software-Image-Installation an. Es wird empfohlen, diesen Befehl weiterhin auszuführen, bis alle Knoten einen *Ausführungsstatus* von *Beendet* und einen *Exit-Status* von *Erfolgreich* melden.

+
Der Befehl zum Aktualisieren des Systemknoten-Images kann fehlschlagen und Fehler- oder Warnmeldungen anzeigen. Nach Behebung aller Fehler oder Warnungen ist eine erneute Ausführung des Befehls möglich.

+
Dieses Beispiel zeigt einen Zwei-Node-Cluster, in dem das Software-Image auf beiden Knoten erfolgreich installiert wurde:

+
[listing]
----
cluster1::*> system node image show-update-progress -node *

        Exit Status:    Success
        Phase:          Run Script

----
