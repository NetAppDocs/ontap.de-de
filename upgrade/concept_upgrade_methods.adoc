= ONTAP Software-Upgrade-Methoden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein automatisiertes Upgrade Ihrer ONTAP Software kann mit System Manager durchgeführt werden. Alternativ ist ein automatisiertes oder manuelles Upgrade über die ONTAP Befehlszeilenschnittstelle (CLI) möglich. Die Methode, die für das Upgrade von ONTAP verwendet wird, hängt von Ihrer Konfiguration, Ihrer aktuellen ONTAP Version und der Anzahl der Knoten in Ihrem Cluster ab. NetApp empfiehlt, für automatisierte Upgrades System Manager zu verwenden, es sei denn, Ihre Konfiguration erfordert eine andere Vorgehensweise. Wenn beispielsweise eine vier Knoten umfassende MetroCluster Konfiguration mit ONTAP 9.3 oder höher vorliegt, sollte für ein automatisiertes Upgrade (auch als automatisiertes unterbrechungsfreies Upgrade oder ANDU bezeichnet) System Manager verwendet werden.


NOTE: Wenn ein Upgrade auf ONTAP 9.15.1 oder höher über die NetApp Console durchgeführt wird, sind die link:https://docs.netapp.com/us-en/console-software-updates/get-started/software-updates.html["Upgrade-Verfahren in der NetApp Console Dokumentation"^] zu beachten.

Ein Upgrade kann entweder mit dem Rolling-Upgrade-Prozess oder mit dem Batch-Upgrade-Prozess durchgeführt werden. Beide Verfahren sind unterbrechungsfrei.

Bei automatisierten Upgrades installiert ONTAP automatisch das Ziel-ONTAP-Image auf jedem Knoten, validiert die Cluster-Komponenten, um sicherzustellen, dass das Cluster unterbrechungsfrei aktualisiert werden kann, und führt anschließend im Hintergrund ein Batch- oder Rolling-Upgrade durch, abhängig von der Anzahl der Knoten. Bei manuellen Upgrades bestätigt der Administrator manuell, dass jeder Knoten im Cluster für das Upgrade bereit ist, und führt anschließend die Schritte zur Durchführung eines Rolling-Upgrades aus.

.ONTAP Rolling Upgrades
Für Cluster mit weniger als 8 Knoten ist der Rolling-Upgrade-Prozess der Standard. Im Rolling-Upgrade-Prozess wird ein Knoten offline genommen und aktualisiert, während sein Partner dessen Speicher übernimmt. Nach Abschluss des Node-Upgrades gibt der Partnerknoten die Kontrolle an den ursprünglichen Knoten zurück, und der Vorgang wird auf dem Partnerknoten wiederholt. Jedes weitere HA-Paar wird der Reihe nach aktualisiert, bis alle HA-Paare die Zielversion ausführen.

.ONTAP Batch-Upgrade
Der Batch-Upgrade-Prozess ist der Standard für Cluster mit 8 oder mehr Knoten. Im Batch-Upgrade-Prozess wird der Cluster in zwei Stapel unterteilt. Jeder Stapel enthält mehrere HA-Paare. Im ersten Stapel wird der erste Knoten jedes HA-Paares gleichzeitig mit dem ersten Knoten aller anderen HA-Paare im Stapel aktualisiert.

Im folgenden Beispiel befinden sich in jedem Batch zwei HA-Paare. Wenn das Batch-Upgrade beginnt, werden Knoten A und Knoten C gleichzeitig aktualisiert.

image:batch_upgrade_set_1_ieops-1607.png["Batch-Upgrade Schritt 1"]

Nach Abschluss des Upgrades der ersten Knoten jedes HA-Paares werden die Partnerknoten in Batch 1 gleichzeitig aktualisiert.

Im folgenden Beispiel werden nach der Aktualisierung von Node A und Node C gleichzeitig Node B und Node D aktualisiert.

image:batch_upgrade_set_2_ieops-1619.png["Batch-Upgrade Schritt 2"]

Anschließend wird der Vorgang für die Knoten in Batch 2 wiederholt; der erste Knoten jedes HA-Paares wird gleichzeitig mit dem ersten Knoten aller anderen HA-Paare in der Batch aktualisiert.

Im folgenden Beispiel werden Node E und Node G gleichzeitig aktualisiert.

image:batch_upgrade_set_3_ieops-1612.png["Batch-Upgrade Schritt 3"]

Nach Abschluss des Upgrades der ersten Knoten jedes HA-Paares werden die Partnerknoten in Batch 2 gleichzeitig aktualisiert.

Im folgenden Beispiel werden Node F und Node H gleichzeitig aktualisiert, um den Batch-Upgrade-Prozess abzuschließen.

image:batch_upgrade_set_4_ieops-1620.png["Batch-Upgrade Schritt 4"]

== Empfohlene ONTAP Upgrade-Methoden basierend auf der Konfiguration

Die von Ihrer Konfiguration unterstützten Upgrade-Methoden sind in der Reihenfolge der empfohlenen Verwendung aufgelistet.

[cols="4"]
|===
| Konfiguration | ONTAP Version | Anzahl der Knoten | Empfohlene Upgrade-Methode 


| Standard | 9.0 oder höher | 2 oder mehr  a| 
* xref:task_upgrade_andu_sm.html[Automatisiertes unterbrechungsfreies Upgrade mit System Manager]
* xref:task_upgrade_andu_cli.html[Automatisiertes unterbrechungsfreies Upgrade über die Befehlszeile (CLI)]




| Standard | 9.0 oder höher | Einzel | link:../system-admin/single-node-clusters.html["Automatisiertes disruptives"] 


| MetroCluster | 9.3 oder höher | 8  a| 
* xref:task_upgrade_andu_cli.html[Automatisiertes unterbrechungsfreies Upgrade über die Befehlszeile (CLI)]
* xref:task_updating_a_four_or_eight_node_mcc.html[Manuelles unterbrechungsfreies Upgrade für 4 oder 8 Node MetroCluster mit der Befehlszeile]




| MetroCluster | 9.3 oder höher | 2,4  a| 
* xref:task_upgrade_andu_sm.html[Automatisiertes unterbrechungsfreies Upgrade mit System Manager]
* xref:task_upgrade_andu_cli.html[Automatisiertes unterbrechungsfreies Upgrade über die Befehlszeile (CLI)]




| MetroCluster | 9.2 oder früher | 4, 8 | xref:task_updating_a_four_or_eight_node_mcc.html[Manuelles unterbrechungsfreies Upgrade für 4 oder 8 Node MetroCluster mit der Befehlszeile] 


| MetroCluster | 9.2 oder früher | 2 | xref:task_updating_a_two_node_metrocluster_configuration_in_ontap_9_2_and_earlier.html[Manuelles, unterbrechungsfreies Upgrade für 2-Node MetroCluster mit der Befehlszeile] 
|===
ANDU mit System Manager ist die empfohlene Upgrade-Methode für alle Patch-Upgrades, unabhängig von der Konfiguration.


NOTE: Ein xref:task_updating_an_ontap_cluster_disruptively.html[manuelles disruptives Upgrade] kann auf jeder Konfiguration durchgeführt werden. Allerdings sollte ein unterbrechendes Upgrade nur durchgeführt werden, wenn der Cluster für die Dauer des Upgrades offline genommen werden kann. In einer SAN-Umgebung ist es erforderlich, alle SAN-Clients vor einem unterbrechenden Upgrade herunterzufahren oder anzuhalten. Unterbrechende Upgrades werden mit der ONTAP CLI durchgeführt.
