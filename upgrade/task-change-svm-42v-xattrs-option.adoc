= Vor einem ONTAP Upgrade der Option v4.2-xattrs einen expliziten Wert zuweisen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn Sie einen NFSv4.2-Client haben, muss vor dem Upgrade von bestimmten Releases und Patches von ONTAP 9.12.1 und später ein expliziter Wert für die Option NFSv4.2 extended attributes angegeben werden, um NFS-Antwortfehler nach dem Upgrade zu verhindern.

Wenn der  `v4.2-xattrs` Option vor dem ONTAP Upgrade auf die betroffenen Versionen niemals explizit ein Wert zugewiesen wird, werden NFSv4.2-Clients nicht darüber informiert, dass sich die Option für die erweiterten Attribute des Servers geändert hat. Dies führt bei bestimmten  `xattrs` Aufrufen aufgrund einer Diskrepanz zwischen Client und Server zu NFS-Antwortfehlern.

.Bevor Sie beginnen
Sie müssen der Option „NFSv4.2 extended attributes“ einen expliziten Wert zuweisen, wenn Folgendes zutrifft:

* Sie verwenden NFSv4.2 mit einer SVM, die mit ONTAP 9.11.1 oder einer früheren Version erstellt wurde.
* Sie aktualisieren ONTAP von einer der folgenden betroffenen Versionen und Patches:
+
** 9.12.1RC1 bis 9.12.1P11
** 9.13.1RC1 bis 9.13.1P8
** 9.14.1RC1 bis 9.14.1P1




.Über diese Aufgabe
ONTAP 9.12.1 oder höher muss verwendet werden, um den Wert mit dem in diesem Verfahren beschriebenen Befehl festzulegen.

Wenn  `v4.2-xattrs` bereits auf  `enabled` gesetzt ist, sollte es dennoch explizit auf  `enabled` gesetzt werden, um zukünftige Störungen zu vermeiden. Wenn  `v4.2-xattrs` auf deaktiviert gesetzt wird, können NFSv4.2 Clients „invalid argument“-Antworten erhalten, bis sie erneut eingebunden werden oder die  `v4.2-xattrs` Option auf  `enabled` gesetzt wird.

.Schritte
* Weisen Sie der Option  `v4.2-xattrs` einen expliziten Wert zu:
+
[source, cli]
----
nfs modify -v4.2-xattrs <enabled/disabled> -vserver <vserver_name>
----
