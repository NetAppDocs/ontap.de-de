= Informationen zu den negativen Auswirkungen sitzungsorientierter Protokolle während ONTAP Upgrades
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Cluster und sitzungsorientierte Protokolle können in bestimmten Bereichen wie dem I/O-Service während Upgrades negative Auswirkungen auf Clients und Anwendungen haben.

Wenn Sie sitzungsorientierte Protokolle verwenden, ist Folgendes zu berücksichtigen:

* SMB
+
Wenn kontinuierlich verfügbare (CA) Freigaben mit SMBv3 bereitgestellt werden, kann die automatisierte unterbrechungsfreie Upgrade-Methode (mit System Manager oder der CLI) verwendet werden, ohne dass der Client eine Unterbrechung erfährt.

+
Wenn Sie Freigaben mit SMBv1 oder SMBv2 oder Nicht-CA-Freigaben mit SMBv3 bereitstellen, werden Clientsitzungen während der Upgrade-Übernahme und des Neustarts unterbrochen. Benutzern sollte empfohlen werden, ihre Sitzungen vor dem Upgrade zu beenden.

+
Hyper-V und SQL Server über SMB unterstützen unterbrechungsfreie Betriebsabläufe (NDOs). Wenn eine Hyper-V- oder SQL Server über SMB-Lösung konfiguriert wurde, bleiben die Anwendungsserver und die darin enthaltenen virtuellen Maschinen oder Datenbanken während des ONTAP Upgrades online und bieten kontinuierliche Verfügbarkeit.

* NFSv4.x
+
NFSv4.x Clients stellen bei Verbindungsabbrüchen, die während des Upgrades auftreten, automatisch die Verbindung mithilfe der normalen NFSv4.x-Wiederherstellungsverfahren wieder her. Anwendungen können während dieses Vorgangs vorübergehend eine Verzögerung bei der Ein-/Ausgabe erleben.

* NDMP
+
Der Status geht verloren und der Client-Benutzer muss den Vorgang wiederholen.

* Datensicherung und Wiederherstellung
+
Der Status geht verloren und der Client-Benutzer muss den Vorgang wiederholen.

+

IMPORTANT: Während oder unmittelbar vor einem Upgrade sollte keine Datensicherung oder -wiederherstellung initiiert werden. Dies könnte zu Datenverlust führen.

* Anwendungen (zum Beispiel Oracle oder Exchange)
+
Die Auswirkungen hängen von den Anwendungen ab. Bei timeout-basierten Anwendungen besteht möglicherweise die Möglichkeit, die Zeitüberschreitungseinstellung auf einen längeren Wert als die ONTAP-Neustartzeit zu setzen, um negative Auswirkungen zu minimieren.
