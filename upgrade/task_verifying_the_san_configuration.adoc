= Die SAN-Hostkonfiguration sollte vor einem ONTAP Upgrade überprüft werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Upgrade von ONTAP in einer SAN-Umgebung ändert, welche Pfade direkt sind. Vor dem Upgrade eines SAN-Clusters sollte überprüft werden, ob jeder Host mit der korrekten Anzahl direkter und indirekter Pfade konfiguriert ist und ob jeder Host mit den richtigen LIFs verbunden ist.

.Schritte
. Auf jedem Host sollte überprüft werden, ob eine ausreichende Anzahl direkter und indirekter Pfade konfiguriert ist und ob jeder Pfad aktiv ist.
+
Jeder Host muss einen Pfad zu jedem Node im Cluster haben.

. Es sollte sichergestellt sein, dass jeder Host mit einem LIF auf jedem Knoten verbunden ist.
+
Sie sollten die Liste der Initiatoren nach dem Upgrade zum Vergleich protokollieren. Bei Verwendung von ONTAP 9.11.1 oder höher empfiehlt sich System Manager zur Anzeige des Verbindungsstatus, da dieser eine deutlich übersichtlichere Darstellung als die CLI bietet.

+
[role="tabbed-block"]
====
.System Manager
--
.. Im System Manager auf *Hosts > SAN Initiatorgruppen* klicken.
+
Auf der Seite wird eine Liste der Initiatorgruppen (Initiatorgruppen) angezeigt. Wenn die Liste lang ist, können zusätzliche Seiten der Liste angezeigt werden, indem auf die Seitenzahlen in der unteren rechten Ecke der Seite geklickt wird.

+
Die Spalten zeigen verschiedene Informationen zu den Initiatorgruppen an. Ab Version 9.11.1 wird auch der Verbindungsstatus der Initiatorgruppe angezeigt. Durch das Überfahren der Statusmeldungen mit dem Mauszeiger werden Details angezeigt.



--
.CLI
--
** Liste der iSCSI-Initiatoren:
+
[source, cli]
----
iscsi initiator show -fields igroup,initiator-name,tpgroup
----
** Liste der FC-Initiatoren:
+
[source, cli]
----
fcp initiator show -fields igroup,wwpn,lif
----
