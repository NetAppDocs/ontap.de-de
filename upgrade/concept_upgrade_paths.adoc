= Unterstützte ONTAP Upgradepfade
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Version von ONTAP, auf die Sie ein Upgrade durchführen können, hängt von Ihrer Hardwareplattform und der aktuell auf den Knoten Ihres Clusters ausgeführten ONTAP-Version ab.

Zur Überprüfung, ob Ihre Hardwareplattform für die Ziel-Upgrade-Version unterstützt wird, siehe  https://hwu.netapp.com["NetApp Hardware Universe"^]. Die link:https://imt.netapp.com/matrix/#welcome["NetApp Interoperabilitätsmatrix-Tool"^] kann verwendet werden, um link:confirm-configuration.html["Unterstützung für Ihre Konfiguration bestätigen"].

.Zur Ermittlung Ihrer aktuellen ONTAP Version:
* Im System Manager auf *Cluster > Übersicht* klicken.
* Über die Befehlszeilenschnittstelle (CLI) kann der  `cluster image show` Befehl verwendet werden. + Auch der  `system node image show` Befehl auf dem erweiterten Berechtigungsniveau zeigt Details an.




[[types-of-upgrade-paths]]
== Arten von Upgrade-Pfaden

Automatisierte unterbrechungsfreie Upgrades (ANDU) werden nach Möglichkeit empfohlen. Abhängig von Ihren aktuellen und Zielversionen erfolgt der Upgrade-Pfad *direkt*, *direkt über mehrere Zwischenstationen* oder *mehrstufig*.

* *Direkt*
+
Sie können jederzeit direkt auf die nächstgelegene ONTAP-Versionsfamilie mit einem einzigen Software-Image aktualisieren. Für viele Versionen kann auch ein Software-Image installiert werden, das eine direkte Aktualisierung auf Versionen ermöglicht, die bis zu vier Versionen nach der aktuell ausgeführten Version liegen.

+
Beispielsweise kann der direkte Upgrade-Pfad von 9.12.1 auf 9.13.1 oder von 9.13.1 auf 9.17.1 verwendet werden.

+
Alle _direkten_ Upgrade-Pfade werden für link:concept_mixed_version_requirements.html["Cluster mit gemischten Versionen"] unterstützt.

* *Direkter Multi-Hop*
+
Für einige automatisierte unterbrechungsfreie Upgrades (ANDU) auf nicht benachbarte Versionen ist es erforderlich, das Software-Image einer Zwischenversion sowie das der Zielversion zu installieren. Der automatisierte Upgrade-Prozess verwendet das Zwischen-Image im Hintergrund, um das Update auf die Zielversion abzuschließen.

+
Wenn der Cluster beispielsweise unter Version 9.3 läuft und Sie auf 9.7 aktualisieren möchten, werden die ONTAP Installationspakete für 9.5 und 9.7 geladen, anschließend wird ANDU auf 9.7 initiiert. ONTAP aktualisiert den Cluster automatisch zuerst auf 9.5 und dann auf 9.7. Während des Vorgangs sind mehrere Übernahme- und Rückgabevorgänge sowie damit verbundene Neustarts zu erwarten.

* *Mehrstufig*
+
Wenn für Ihre nicht benachbarte Zielversion kein direkter oder direkter Multi-Hop-Pfad verfügbar ist, ist zunächst ein Upgrade auf eine unterstützte Zwischenversion erforderlich, gefolgt von einem Upgrade auf die Zielversion.

+
Wenn Sie beispielsweise derzeit Version 9.8 verwenden und auf 9.16.1 aktualisieren möchten, ist ein mehrstufiges Upgrade erforderlich: zuerst von 9.8 auf 9.12.1 und dann von 9.12.1 auf 9.16.1. Upgrades von früheren Versionen können drei oder mehr Stufen mit mehreren Zwischen-Upgrades erfordern.

+

NOTE: Vor Beginn von mehrstufigen Upgrades sollte sichergestellt sein, dass Ihre Zielversion auf Ihrer Hardwareplattform unterstützt wird.



Vor einem größeren Upgrade empfiehlt es sich, zunächst die neueste Patch-Version der auf Ihrem Cluster laufenden ONTAP-Version zu installieren. Dadurch wird sichergestellt, dass alle Probleme in Ihrer aktuellen ONTAP-Version vor dem Upgrade behoben sind.

Wenn auf Ihrem System beispielsweise ONTAP 9.3P9 läuft und Sie ein Upgrade auf 9.11.1 planen, sollten Sie zuerst auf die neueste 9.3 Patch-Version aktualisieren und dann dem Upgrade-Pfad von 9.3 auf 9.11.1 folgen.

Weitere Informationen https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU2["Empfohlene Mindestversionen von ONTAP auf der NetApp Support Site"^].



[[supported-upgrade-paths]]
== Unterstützte Upgrade-Pfade

Die folgenden Upgrade-Pfade werden für automatisierte und manuelle Upgrades Ihrer ONTAP Software unterstützt. Diese Upgrade-Pfade gelten für On-Premises ONTAP und ONTAP Select. Es gibt verschiedene https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/task-updating-ontap-cloud.html#supported-upgrade-paths["Unterstützte Upgrade-Pfade für Cloud Volumes ONTAP"^].


NOTE: *Für ONTAP Cluster mit gemischten Versionen*: Alle _direkten_ und _direkten Multi-Hop_-Upgrade-Pfade beinhalten ONTAP Versionen, die mit Clustern mit gemischten Versionen kompatibel sind. ONTAP Versionen, die in _mehrstufigen_ Upgrades enthalten sind, sind nicht mit Clustern mit gemischten Versionen kompatibel. Zum Beispiel ist ein Upgrade von 9.8 auf 9.12.1 ein _direktes_ Upgrade. Ein Cluster mit Knoten, auf denen 9.8 und 9.12.1 ausgeführt werden, ist ein unterstützter Cluster mit gemischten Versionen. Ein Upgrade von 9.8 auf 9.13.1 ist ein _mehrstufiges_ Upgrade. Ein Cluster mit Knoten, auf denen 9.8 und 9.13.1 ausgeführt werden, ist kein unterstützter Cluster mit gemischten Versionen.

[[data-ontap-8]]
=== Data ONTAP 8

Stellen Sie sicher, dass Ihre Plattform die Zielversion ONTAP ausführen kann, indem Sie die  https://hwu.netapp.com["NetApp Hardware Universe"^] verwenden.

*Hinweis:* Im Data ONTAP 8.3 Upgrade Guide wird fälschlicherweise angegeben, dass in einem Cluster mit vier Knoten der Knoten, der Epsilon enthält, zuletzt aktualisiert werden sollte. Dies ist ab Data ONTAP 8.2.3 für Upgrades nicht mehr erforderlich. Weitere Informationen finden sich unter  https://mysupport.netapp.com/site/bugs-online/product/ONTAP/BURT/805277["NetApp Bugs Online Bug-ID 805277"^].

Von Data ONTAP 8.3.x:: Sie können direkt auf ONTAP 9.1 aktualisieren und anschließend auf spätere Versionen aktualisieren.
Ab Data ONTAP Releases vor 8.3.x, einschließlich 8.2.x:: Sie müssen zuerst auf Data ONTAP 8.3.x aktualisieren, dann auf ONTAP 9.1, dann auf spätere Versionen aktualisieren.
