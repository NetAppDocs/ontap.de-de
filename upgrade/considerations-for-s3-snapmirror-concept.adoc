= ONTAP Lizenzierung für SnapMirror S3 Konfigurationen überprüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie ONTAP aktualisieren, falls Sie SnapMirror S3 verwenden und auf ONTAP 9.12.1 oder höher aktualisieren, sollte überprüft werden, ob die entsprechenden SnapMirror Lizenzen vorhanden sind.

Nach einem Upgrade von ONTAP können Lizenzänderungen, die zwischen ONTAP 9.11.1 und früher und ONTAP 9.12.1 und später vorgenommen wurden, dazu führen, dass SnapMirror S3-Beziehungen fehlschlagen.



[[ontap-9-11-1-and-earlier]]
== ONTAP 9.11.1 und früher

* Bei der Replikation in einen von NetApp gehosteten Ziel-Bucket (ONTAP S3 oder StorageGRID) prüft SnapMirror S3, ob die SnapMirror synchron Lizenz vorhanden ist, die vor der Einführung der link:../system-admin/manage-licenses-concept.html["ONTAP One"] Software Suite im Data Protection Bundle enthalten war.
* Bei der Replikation in einen Nicht-NetApp-Ziel-Bucket prüft SnapMirror S3, ob die SnapMirror Cloud-Lizenz im Hybrid Cloud Bundle enthalten ist, das vor der Einführung der link:../system-admin/manage-licenses-concept.html["ONTAP One"] Software Suite verfügbar war.




[[ontap-9-12-1-and-later]]
== ONTAP 9.12.1 und höher

* Bei der Replikation in einen von NetApp gehosteten Ziel-Bucket (ONTAP S3 oder StorageGRID) prüft SnapMirror S3, ob die SnapMirror S3-Lizenz vorhanden ist, die im Data Protection Bundle enthalten war, das vor der Einführung der link:../system-admin/manage-licenses-concept.html["ONTAP One"] Software Suite verfügbar war.
* Bei der Replikation in einen Nicht-NetApp-Ziel-Bucket prüft SnapMirror S3, ob die SnapMirror S3 External-Lizenz vorhanden ist, die im Hybrid Cloud Bundle enthalten ist, welches vor der Einführung der link:../system-admin/manage-licenses-concept.html["ONTAP One"] Software Suite und der link:../data-protection/install-snapmirror-cloud-license-task.html["ONTAP One Kompatibilitätspaket"] verfügbar war.




[[existing-snapmirror-s3-relationships]]
== Bestehende SnapMirror S3-Beziehungen

Bestehende SnapMirror S3-Beziehungen sollten auch nach einem Upgrade von ONTAP 9.11.1 oder früher auf ONTAP 9.12.1 oder später weiterhin funktionieren, selbst wenn der Cluster nicht über die neue Lizenzierung verfügt.

Die Erstellung neuer SnapMirror S3-Beziehungen schlägt fehl, wenn auf dem Cluster nicht die erforderliche Lizenz installiert ist.
