= Kompatibilität der ONTAP Versionen für Cluster mit gemischten Versionen prüfen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In einem ONTAP Cluster mit gemischten Versionen laufen auf den Knoten für kurze Zeit zwei verschiedene Hauptversionen von ONTAP. Ein Cluster mit Knoten, auf denen ONTAP 9.8 und 9.12.1 oder ONTAP 9.9.1 und 9.13.1 ausgeführt wird, ist beispielsweise ein Cluster mit gemischten Versionen. Cluster mit Knoten, auf denen innerhalb derselben Version unterschiedliche Patch-Level wie ONTAP 9.9.1P1 und 9.9.1P5 ausgeführt werden, sind keine Cluster mit gemischten Versionen.


NOTE: Cluster mit gemischten Versionen werden für Cloud Volumes ONTAP nicht unterstützt.

NetApp unterstützt ONTAP Cluster mit gemischten Versionen für begrenzte Zeiträume und in bestimmten Szenarien.

Im Folgenden sind die häufigsten Szenarien, in denen sich ein ONTAP Cluster in einem gemischten Versionszustand befindet:

* ONTAP Software-Upgrades in großen Clustern
+
Die Aktualisierung aller Knoten in einem großen Cluster kann mehrere Tage oder Wochen dauern. Der Cluster befindet sich in einem Zustand mit gemischten Versionen, bis alle Knoten aktualisiert sind.

* ONTAP Software-Upgrades sind erforderlich, wenn geplant ist, einem Cluster neue Knoten hinzuzufügen.
+
Sie können Ihrem Cluster neue Knoten hinzufügen, um dessen Kapazität zu erweitern, oder Sie können neue Knoten im Zuge eines vollständigen Austauschs Ihrer Controller hinzufügen. In beiden Fällen kann es erforderlich sein, einen gemischten Versionszustand zu aktivieren, um die Migration Ihrer Daten von den bestehenden Controllern auf die neuen Knoten in Ihrem neuen System zu ermöglichen.



Für einen optimalen Clusterbetrieb sollte die Zeitspanne, in der sich der Cluster in einem gemischten Versionszustand befindet, so kurz wie möglich sein. Die maximale Zeitspanne, in der ein Cluster in einem gemischten Versionszustand unterstützt werden kann, hängt von der niedrigsten ONTAP Version im Cluster ab.

[cols="2"]
|===
| Wenn die niedrigste Version von ONTAP im Cluster mit gemischten Versionen ausgeführt wird, ... | Anschließend ist es möglich, sich maximal für... im gemischten Versionszustand zu befinden. 


| ONTAP 9.8 oder höher | 90 Tage 


| ONTAP 9.7 oder früher | 7 Tage 
|===
Solange sich der Cluster in einem gemischten Versionszustand befindet, sollten keine Befehle eingegeben werden, die den Clusterbetrieb oder die Konfiguration verändern, außer jenen, die für das Upgrade oder die Datenmigration erforderlich sind. Beispielsweise sollten Aktivitäten wie (aber nicht beschränkt auf) LIF-Migration, geplante Storage-Failover-Operationen oder das Erstellen oder Löschen großer Mengen von Objekten erst nach Abschluss des Upgrades und der Datenmigration durchgeführt werden.



[[mixed-version-clusters-supported-for-ontap-software-upgrades]]
== Unterstützung für Cluster mit gemischten Versionen bei ONTAP Software-Upgrades

Sie können einen gemischten Versionszustand mit jeder ONTAP Version erreichen, die für ein direktes Upgrade von Ihrer niedrigsten aktuellen Version unterstützt wird. Wenn Sie beispielsweise ONTAP 9.11.1 verwenden, können Sie einen gemischten Versionszustand mit Knoten erreichen, auf denen ONTAP 9.15.1 ausgeführt wird. Ein gemischter Versionszustand ist nicht möglich, wenn Knoten auf ONTAP 9.11.1 und ONTAP 9.16.1 ausgeführt werden. ONTAP 9.16.1 wird für ein direktes Upgrade von ONTAP 9.11.1 nicht unterstützt.


NOTE: ONTAP Patch-(P)-Versionen haben keinen Einfluss auf die Kompatibilität bei Clustern mit gemischten Versionen. Beispielsweise gilt bei Verwendung von ONTAP 9.11.1P6 als aktuelle ONTAP Version für die Kompatibilität in Clustern mit gemischten Versionen ONTAP 9.11.1. Oder bei Verwendung von ONTAP 9.12.1 und einem Upgrade auf ONTAP 9.15.1P2 ist die Ziel-ONTAP Version für die Kompatibilität in Clustern mit gemischten Versionen ONTAP 9.15.1.

Um auf eine ONTAP Version zu aktualisieren, die für ein direktes Upgrade von Ihrer aktuellen Version nicht unterstützt wird, ist ein mehrstufiges Upgrade erforderlich. Bei einem mehrstufigen Upgrade wechseln Sie zunächst in einen gemischten Versionszustand mit der höchsten Version, die für ein direktes Upgrade von Ihrer aktuellen Version unterstützt wird. Dieses Upgrade wird abgeschlossen, danach erfolgt ein separates Upgrade auf Ihre Zielversion. Beispielsweise gilt: Wenn Ihre niedrigste aktuelle Version ONTAP 9.10.1 ist und ein Upgrade auf ONTAP 9.16.1 gewünscht ist, wechseln Sie zunächst in einen gemischten Versionszustand, um alle Ihre Knoten auf ONTAP 9.14.1 zu aktualisieren, danach erfolgt ein separates Upgrade von ONTAP 9.14.1 auf ONTAP 9.16.1. Weitere Informationen zu link:concept_upgrade_paths.html#types-of-upgrade-paths["mehrstufige Upgrades"] und link:concept_upgrade_paths.html#supported-upgrade-paths["Unterstützte Upgrade-Pfade"].

Ein Cluster mit gemischten Versionen kann nur zwei Hauptversionen von ONTAP enthalten. Beispielsweise kann ein gemischter Versions-Cluster Knoten mit ONTAP 9.13.1 und 9.15.1 oder Knoten mit ONTAP 9.13.1 und 9.16.1 enthalten. Ein gemischter Versions-Cluster mit Knoten, auf denen ONTAP 9.13.1, 9.15.1 und 9.16.1 ausgeführt werden, ist nicht zulässig.

[[adding-new-nodes-to-an-ontap-cluster]]
== Hinzufügen neuer Knoten zu einem ONTAP Cluster

Wenn Sie planen, Ihrem Cluster neue Knoten hinzuzufügen und diese Knoten eine Mindestversion von ONTAP benötigen, die neuer ist als die derzeit im Cluster ausgeführte Version, müssen auf den vorhandenen Knoten in Ihrem Cluster alle unterstützten Software-Upgrades durchgeführt werden, bevor die neuen Knoten hinzugefügt werden. Idealerweise werden alle vorhandenen Knoten auf die von den hinzuzufügenden Knoten benötigte Mindestversion von ONTAP aktualisiert. Sollte dies jedoch nicht möglich sein, weil einige Ihrer vorhandenen Knoten die neuere Version von ONTAP nicht unterstützen, ist im Rahmen des Upgrade-Prozesses für einen begrenzten Zeitraum ein gemischter Versionszustand erforderlich.

.Schritte
. link:concept_upgrade_methods.html["Upgrade"]Die Knoten, die die von Ihren neuen Controllern benötigte minimale ONTAP-Version nicht unterstützen, werden auf die höchste ONTAP-Version aktualisiert, die sie unterstützen.
+
Wenn Sie beispielsweise ein FAS8080 mit ONTAP 9.5 betreiben und eine neue C-Series Plattform mit ONTAP 9.12.1 hinzufügen, sollte Ihr FAS8080 auf ONTAP 9.8 aktualisiert werden (das ist die höchste ONTAP Version, die unterstützt wird).

. link:../system-admin/add-nodes-cluster-concept.html["Die neuen Knoten zum Cluster hinzufügen"].
. link:https://docs.netapp.com/us-en/ontap-systems-upgrade/upgrade/upgrade-create-aggregate-move-volumes.html["Daten migrieren"^]von den aus dem Cluster entfernten Knoten zu den neu hinzugefügten Knoten.
. link:../system-admin/remove-nodes-cluster-concept.html["Die nicht unterstützten Knoten aus dem Cluster entfernen"].
. link:concept_upgrade_methods.html["Upgrade"]den Cluster auf die gleiche ONTAP Version und den gleichen Patch-Level, der auf den neuen Knoten läuft, oder auf link:https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU2["neueste empfohlene Patch-Version"^]die ONTAP Version, die auf den neuen Knoten läuft.
. Es sollte sichergestellt sein, dass auf allen Knoten die gleiche ONTAP Version ausgeführt wird.
+
.. Die auf dem Cluster ausgeführte ONTAP-Version anzeigen:
+
[source, cli]
----
version
----
.. Die auf jedem Knoten des Clusters ausgeführte ONTAP Version anzeigen:
+
[source, cli]
----
version *
----


+
Wenn es einen Unterschied zwischen der in der Ausgabe für die  `version *` (Cluster) und  `version` (einzelne Nodes) gemeldeten ONTAP-Version gibt, sollten alle Nodes auf dieselbe ONTAP- und Patch-Version aktualisiert werden, indem ein link:concept_upgrade_methods.html["Cluster-Image-Update"] ausgeführt wird.
