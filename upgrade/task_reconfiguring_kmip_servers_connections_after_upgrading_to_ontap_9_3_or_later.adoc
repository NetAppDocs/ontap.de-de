= KMIP-Serververbindungen nach einem Upgrade von ONTAP 9.2 oder früher neu konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nach dem Upgrade von ONTAP 9.2 oder früher auf ONTAP 9.3 oder später ist es erforderlich, alle externen Key Management (KMIP) Server-Verbindungen neu zu konfigurieren.

.Schritte
. Konfiguration der Key Manager-Konnektivität:
+
[source, cli]
----
security key-manager setup
----
. Fügen Sie Ihre KMIP-Server hinzu:
+
[source, cli]
----
security key-manager add -address <key_management_server_ip_address>
----
. Es sollte überprüft werden, ob die KMIP Server verbunden sind:
+
[source, cli]
----
security key-manager show -status
----
. Abfrage der Schlüsselserver:
+
[source, cli]
----
security key-manager query
----
. Einen neuen Authentifizierungsschlüssel und eine neue Passphrase erstellen:
+
[source, cli]
----
security key-manager create-key -prompt-for-key true
----
+
Eine Passphrase mit mindestens 32 Zeichen festlegen.

. Den neuen Authentifizierungsschlüssel abfragen:
+
[source, cli]
----
security key-manager query
----
. Weisen Sie Ihren selbstverschlüsselnden Festplatten (SEDs) den neuen Authentifizierungsschlüssel zu:
+
[source, cli]
----
storage encryption disk modify -disk <disk_ID> -data-key-id <key_ID>
----
+

NOTE: Verwenden Sie den neuen Authentifizierungsschlüssel aus Ihrer Abfrage.

. Weisen Sie den SEDs gegebenenfalls einen FIPS-Schlüssel zu:
+
[source, cli]
----
storage encryption disk modify -disk <disk_id> -fips-key-id <fips_authentication_key_id>
----
+
Wenn Ihre Sicherheitskonfiguration die Verwendung unterschiedlicher Schlüssel für die Authentifizierung von Daten und die FIPS 140-2 Authentifizierung erfordert, sollte für jeden Zweck ein separater Schlüssel erstellt werden. Andernfalls kann derselbe Authentifizierungsschlüssel für beide verwendet werden.
