= Vor einem ONTAP Upgrade sollte sichergestellt sein, dass die netgroup-Datei auf allen Knoten vorhanden ist.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie ONTAP aktualisieren, muss, falls Netzgruppen in Storage Virtual Machines (SVMs) geladen wurden, überprüft werden, ob die Netzgruppendatei auf jedem Node vorhanden ist. Eine fehlende Netzgruppendatei auf einem Node kann dazu führen, dass das Upgrade fehlschlägt.

.Schritte
. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
[source, cli]
----
set -privilege advanced
----
. Der Netgroup-Status für jede SVM wird angezeigt:
+
[source, cli]
----
vserver services netgroup status
----
. Es sollte für jede SVM sichergestellt sein, dass jeder Node denselben Hashwert der Netgroup-Datei anzeigt:
+
[source, cli]
----
vserver services name-service netgroup status
----
+
In diesem Fall kann der nächste Schritt übersprungen werden und das Upgrade oder die Rücksetzung kann fortgesetzt werden. Andernfalls ist mit dem nächsten Schritt fortzufahren.

. Auf einem beliebigen Node des Clusters die Netgroup-Datei manuell laden:
+
[source, cli]
----
vserver services netgroup load -vserver vserver_name -source uri
----
+
Dieser Befehl lädt die Netgroup-Datei auf allen Knoten herunter. Falls auf einem Knoten bereits eine Netgroup-Datei existiert, wird diese überschrieben.
