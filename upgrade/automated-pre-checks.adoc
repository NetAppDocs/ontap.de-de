= Automatisierte ONTAP-Vorabprüfungen vor einem geplanten Upgrade ausführen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Sie müssen sich nicht im Prozess der Aktualisierung Ihrer ONTAP Software befinden, um die automatisierten ONTAP Upgrade-Vorabprüfungen auszuführen. Das unabhängige Ausführen der Vorabprüfungen vom automatisierten ONTAP Upgrade-Prozess ermöglicht es, zu sehen, welche Prüfungen an Ihrem Cluster durchgeführt werden, und liefert eine Liste aller Fehler oder Warnungen, die vor Beginn des eigentlichen Upgrades behoben werden sollten. Beispielsweise kann vorgesehen sein, die ONTAP Software während eines in zwei Wochen geplanten Wartungsfensters zu aktualisieren. Während Sie auf das geplante Datum warten, lassen sich die automatisierten Upgrade-Vorabprüfungen durchführen und notwendige Korrekturmaßnahmen im Voraus vor dem Wartungsfenster ergreifen. Dadurch werden Risiken unerwarteter Konfigurationsfehler nach Beginn des Upgrades verringert.

Wenn Sie bereit sind, mit dem ONTAP Software-Upgrade zu beginnen, ist diese Vorgehensweise nicht erforderlich. Es sollte der link:automated-upgrade-task.html["automatisierter Upgrade-Prozess"] gefolgt werden, der die Ausführung der automatisierten Upgrade-Vorabprüfungen beinhaltet.


NOTE: Für MetroCluster Konfigurationen sollten diese Schritte zuerst auf Cluster A und anschließend die gleichen Schritte auf Cluster B ausgeführt werden.

.Bevor Sie beginnen
Sie sollten link:download-software-image.html["das Ziel-ONTAP-Software-Image herunterladen"].

Um die automatisierten Vorabprüfungen für ein link:../upgrade/concept_upgrade_paths.html#types-of-upgrade-paths["direktes Multi-Hop-Upgrade"] Upgrade durchzuführen, muss lediglich das Softwarepaket für Ihre Ziel ONTAP-Version heruntergeladen werden. Die Zwischenversion von ONTAP wird erst benötigt, wenn das eigentliche Upgrade beginnt. Wenn beispielsweise automatisierte Vorabprüfungen für ein Upgrade von 9.7 auf 9.11.1 ausgeführt werden, ist das Softwarepaket für ONTAP 9.11.1 herunterzuladen. Das Softwarepaket für ONTAP 9.8.1 muss nicht heruntergeladen werden.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. Das ONTAP Zielabbild validieren:
+

NOTE: Wenn Sie eine MetroCluster Konfiguration aktualisieren, sollte Cluster A validiert und anschließend der Validierungsprozess auf Cluster B wiederholt werden.

+
.. Je nachdem, welche ONTAP Version verwendet wird, ist einer der folgenden Schritte auszuführen:
+
|===


| Wenn Sie ausführen... | Folgendes ist zu tun... 


| ONTAP 9.8 oder höher  a| 
*Cluster > Übersicht* auswählen.



| ONTAP 9.5, 9.6 und 9.7  a| 
*Konfiguration* > *Cluster* > *Aktualisieren* auswählen.



| ONTAP 9.4 oder früher  a| 
*Konfiguration* > *Cluster Update* auswählen.

|===
.. In der rechten Ecke des *Übersicht*-Bereichs auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken.
.. *ONTAP Update* auswählen.
.. Auf der Registerkarte *Cluster Update* ein neues Image hinzufügen oder ein verfügbares Image auswählen.
+
|===


| Wenn Sie möchten... | Dann... 


 a| 
Ein neues Software-Image aus einem lokalen Ordner hinzufügen.

Sie sollten bereits link:download-software-image.html["das Image heruntergeladen"] eine Verbindung zum lokalen Client hergestellt haben.
 a| 
... Unter *Verfügbare Software-Images* auf *Von lokal hinzufügen* klicken.
... Navigieren Sie zu dem Speicherort, an dem Sie das Software-Image gespeichert haben, wählen Sie das Image aus und klicken Sie anschließend auf *Open*.




 a| 
Ein neues Software-Image von einem HTTP- oder FTP-Server hinzufügen
 a| 
... *Vom Server hinzufügen* auswählen.
... Im Dialogfeld *Neues Software-Image hinzufügen* ist die URL des HTTP- oder FTP-Servers einzugeben, auf den das ONTAP Software-Image von der NetApp Support Site heruntergeladen wurde.
+
Für anonymes FTP müssen Sie die URL im Format \ftp://anonymous@ftpserver angeben.

... Klicken Sie auf *Add*.




 a| 
Wählen Sie ein verfügbares Bild
 a| 
Eines der aufgelisteten Bilder kann ausgewählt werden.

|===
.. *Validieren* ausführen, um die Validierungsprüfungen vor dem Upgrade zu starten.
+
Werden während der Validierung Fehler oder Warnungen gefunden, werden diese zusammen mit einer Liste von Korrekturmaßnahmen angezeigt. Alle Fehler müssen behoben werden, bevor mit dem Upgrade fortgefahren werden kann. Es gilt als Best Practice, auch Warnungen zu beheben.

+
Ab ONTAP 9.19.1 werden Fehler und Warnungen separat angezeigt, damit Sie Probleme, die vor dem Upgrade behoben werden müssen, leichter erkennen können.





--
.CLI
--
. Das Ziel-ONTAP Software-Image wird in das Cluster-Paket-Repository geladen:
+
[source, cli]
----
cluster image package get -url location
----
+
[listing]
----
cluster1::> cluster image package get -url http://www.example.com/software/9.15.1/image.tgz

Package download completed.
Package processing completed.
----
. Es sollte sichergestellt sein, dass das Softwarepaket im Cluster-Paket-Repository verfügbar ist:
+
[source, cli]
----
cluster image package show-repository
----
+
[listing]
----
cluster1::> cluster image package show-repository
Package Version  Package Build Time
---------------- ------------------
9.15.1              MM/DD/YYYY 10:32:15
----
. Die automatisierten Vorabprüfungen für das Upgrade werden ausgeführt:
+
[source, cli]
----
cluster image validate -version <package_version_number> -show-validation-details true
----
+
[listing]
----
cluster1::> cluster image validate -version 9.15.1 -show-validation-details true

It can take several minutes to complete validation...
Validation checks started successfully.  Run the "cluster image show-update-progress" command to check validation status.
----
. Validierungsstatus prüfen:
+
[source, cli]
----
cluster image show-update-progress
----
+

NOTE: Wenn der *Status* „in Bearbeitung“ ist, abwarten und den Befehl erneut ausführen, bis er abgeschlossen ist.
