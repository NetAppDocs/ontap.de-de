= Wann sollte ONTAP aktualisiert werden?
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie sollten Ihre ONTAP Software regelmäßig aktualisieren. Ein Upgrade von ONTAP ermöglicht die Nutzung neuer und verbesserter Funktionen sowie die Implementierung aktueller Korrekturen für bekannte Probleme.



[[major-ontap-upgrades]]
== Wichtige ONTAP Upgrades

Ein größeres ONTAP Upgrade oder eine neue Feature-Version umfasst typischerweise Folgendes:

* Neue ONTAP Funktionen
* Wichtige Infrastrukturänderungen, wie grundlegende Änderungen am NetApp WAFL Betrieb oder am RAID Betrieb
* Unterstützung für neue von NetApp entwickelte Hardwaresysteme
* Unterstützung für Ersatzhardwarekomponenten wie neuere Netzwerkkarten oder Host-Bus-Adapter


Neue ONTAP-Versionen erhalten drei Jahre lang vollen Support. NetApp empfiehlt, die neueste Version nach allgemeiner Verfügbarkeit (GA) ein Jahr lang zu nutzen und anschließend die verbleibende Zeit innerhalb des vollen Supportzeitraums für die Planung des Übergangs zu einer neueren ONTAP Version zu verwenden.



[[ontap-patch-upgrades]]
== ONTAP Patch-Upgrades

Patch-Upgrades beheben zeitnah kritische Fehler, die nicht bis zum nächsten größeren ONTAP Feature Release warten können. Nicht kritische Patch-Upgrades sollten alle 3-6 Monate eingespielt werden. Kritische Patch-Upgrades sollten so schnell wie möglich installiert werden.

Weitere Informationen zu link:https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU2["Empfohlene Mindest-Patch-Level"^] für ONTAP Releases.



[[ontap-release-dates]]
== ONTAP Veröffentlichungstermine

Ab der ONTAP 9.8-Version liefert NetApp ONTAP-Releases zweimal pro Kalenderjahr aus. Auch wenn sich die Pläne ändern können, ist beabsichtigt, neue ONTAP-Releases im zweiten und vierten Quartal jedes Kalenderjahres bereitzustellen.
