= Die LIF-Failover-Konfiguration des ONTAP-Clusters sollte vor einem Upgrade überprüft werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor ONTAP aktualisiert wird, muss überprüft werden, ob die Failover-Richtlinien und Failover-Gruppen des Clusters korrekt konfiguriert sind.

Während des Upgrade-Prozesses werden LIFs entsprechend der Upgrade-Methode migriert. Abhängig von der Upgrade-Methode wird die LIF-Failover-Richtlinie möglicherweise verwendet oder auch nicht.

Bei Clustern mit 8 oder mehr Knoten erfolgt das automatische Upgrade mit der Batch-Methode. Die Batch-Upgrade-Methode beinhaltet die Aufteilung des Clusters in mehrere Upgrade-Batches, die Aktualisierung der Knoten des ersten Batches, die Aktualisierung ihrer Hochverfügbarkeitspartner (HA-Partner) und anschließend die Wiederholung dieses Prozesses für die verbleibenden Batches. In ONTAP 9.7 und früher, wenn die Batch-Methode verwendet wird, werden LIFs auf den HA-Partner des Knotens migriert, der aktualisiert wird. In ONTAP 9.8 und später, wenn die Batch-Methode verwendet wird, werden LIFs auf die andere Batch-Gruppe migriert.

Bei Clustern mit weniger als 8 Knoten wird das automatische Upgrade mit der Rolling-Methode durchgeführt. Die Rolling-Upgrade-Methode umfasst das Initiieren eines Failover-Vorgangs auf jedem Knoten eines HA-Paares, das Aktualisieren des übergebenen Knotens, das Initiieren der Rückgabe und das anschließende Wiederholen dieses Prozesses für jedes HA-Paar im Cluster. Wenn die Rolling-Methode verwendet wird, werden LIFs entsprechend der LIF-Failover-Richtlinie auf den Failover-Zielknoten migriert.

.Schritte
. Die Failover-Richtlinie für jede Daten-LIF wird angezeigt:
+
[cols="2*"]
|===
| Wenn Ihre ONTAP Version ... | Verwenden Sie diesen Befehl 


| 9.6 oder höher  a| 
`network interface show -service-policy \*data* -failover`



| 9.5 oder früher  a| 
`network interface show -role data -failover`

|===
+
Dieses Beispiel zeigt die Standard-Failover-Konfiguration für ein Zwei-Node-Cluster mit zwei Daten-LIFs:

+
[listing]
----
cluster1::> network interface show -role data -failover
         Logical         Home                  Failover        Failover
Vserver  Interface       Node:Port             Policy          Group
-------- --------------- --------------------- --------------- ---------------
vs0
         lif0            node0:e0b             nextavail       system-defined
                         Failover Targets: node0:e0b, node0:e0c,
                                           node0:e0d, node0:e0e,
                                           node0:e0f, node1:e0b,
                                           node1:e0c, node1:e0d,
                                           node1:e0e, node1:e0f
vs1
         lif1            node1:e0b             nextavail       system-defined
                         Failover Targets: node1:e0b, node1:e0c,
                                           node1:e0d, node1:e0e,
                                           node1:e0f, node0:e0b,
                                           node0:e0c, node0:e0d,
                                           node0:e0e, node0:e0f
----
+
Das Feld *Failover Targets* zeigt eine priorisierte Liste der Failover-Ziele für jede LIF. Wenn beispielsweise 'lif0' von seinem Home-Port (e0b auf node0) ausfällt, wird zunächst versucht, auf Port e0c auf node0 umzuschalten. Wenn lif0 nicht auf e0c umschalten kann, wird anschließend versucht, auf Port e0d auf node0 umzuschalten, und so weiter.

+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

. Wenn die Failover-Richtlinie für LIFs außer SAN LIFs auf *deaktiviert* gesetzt ist, kann das Failover mit dem  `network interface modify` Befehl aktiviert werden.
+
Weitere Informationen zu  `network interface modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-modify.html["ONTAP-Befehlsreferenz"^].

. Für jede LIF ist zu prüfen, ob das Feld *Failover Targets* Datenports von einem anderen Node enthält, die während des Upgrades des Home-Nodes der LIF aktiv bleiben.
+
Mit dem  `network interface failover-groups modify` Befehl lässt sich ein Failover-Ziel zur Failover-Gruppe hinzufügen.

+
.Beispiel
[listing]
----
network interface failover-groups modify -vserver vs0 -failover-group fg1 -targets sti8-vsim-ucs572q:e0d,sti8-vsim-ucs572r:e0d
----
