= Die SVM-Routing-Konfiguration des ONTAP Clusters sollte vor einem Upgrade überprüft werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um Unterbrechungen zu vermeiden, sollte vor dem Upgrade der ONTAP Software sichergestellt werden, dass die Standardroute der SVM jede Netzwerkadresse erreichen kann, die nicht durch eine spezifischere Route erreichbar ist. Es gilt als Best Practice, eine Standardroute für eine SVM zu konfigurieren. Weitere Informationen finden sich unter link:https://kb.netapp.com/Support_Bulletins/Customer_Bulletins/SU134["SU134: Der Netzwerkzugriff kann durch eine fehlerhafte Routing-Konfiguration in ONTAP beeinträchtigt werden"^].

Die Routingtabelle einer SVM legt den Netzwerkpfad fest, den die SVM zur Kommunikation mit einem Ziel verwendet. Es ist wichtig zu verstehen, wie Routingtabellen funktionieren, damit Netzwerkprobleme bereits im Vorfeld vermieden werden können.

Die Routing-Regeln lauten wie folgt:

* ONTAP leitet den Datenverkehr über die spezifischste verfügbare Route.
* ONTAP leitet den Datenverkehr als letzte Möglichkeit über eine Standard-Gateway-Route (mit 0 Bits Netzmaske), wenn spezifischere Routen nicht verfügbar sind.


Bei Routen mit gleichem Ziel, gleicher Netzmaske und gleicher Metrik kann nicht garantiert werden, dass das System nach einem Neustart oder einem Upgrade dieselbe Route verwendet. Dies kann insbesondere dann problematisch sein, wenn mehrere Standardrouten konfiguriert wurden.
