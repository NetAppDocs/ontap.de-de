= Benutzerdefinierte ONTAP IPspaces ermöglichen die Isolierung des Replikationsverkehrs.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mithilfe benutzerdefinierter IPspaces lassen sich die Interaktionen eines Clusters mit seinen Peers trennen. Diese als _designated intercluster connectivity_ bezeichnete Konfiguration ermöglicht es Dienstanbietern, den Replikationsverkehr in mandantenfähigen Umgebungen zu isolieren.

Angenommen, Sie möchten beispielsweise den Replikationsverkehr zwischen Cluster A und Cluster B vom Replikationsverkehr zwischen Cluster A und Cluster C trennen. Zu diesem Zweck können zwei IPspaces auf Cluster A erstellt werden.

Ein IPspace enthält die Intercluster-LIFs, die Sie zur Kommunikation mit Cluster B verwenden. Der andere enthält die Intercluster-LIFs, die Sie zur Kommunikation mit Cluster C verwenden, wie in der folgenden Abbildung zu sehen ist.
