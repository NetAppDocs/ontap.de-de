= ONTAP Cluster Peer-Beziehungen erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie Ihre Daten durch Replikation auf einen Remote-Cluster zu Zwecken der Datensicherung und Notfallwiederherstellung schützen können, sollte eine Cluster Peer-Beziehung zwischen dem lokalen und dem Remote-Cluster erstellt werden.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/data-protection/snapshot-replication.html["diese Schritte"^] zu folgen, um die Snapshot-Replikation einzurichten. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

Es stehen mehrere Standard-Schutzrichtlinien zur Verfügung. Eigene Schutzrichtlinien müssen erstellt worden sein, wenn benutzerdefinierte Richtlinien verwendet werden sollen.

.Bevor Sie beginnen
Wenn Sie die ONTAP CLI verwenden, müssen auf jedem Knoten in den zu verbindenden Clustern Intercluster-LIFs mit einer der folgenden Methoden erstellt worden sein:

* link:configure-intercluster-lifs-share-data-ports-task.html["Intercluster-LIFs auf gemeinsam genutzten Datenports konfigurieren"]
* link:configure-intercluster-lifs-use-dedicated-ports-task.html["Intercluster-LIFs auf dedizierten Daten-Ports konfigurieren"]
* link:configure-intercluster-lifs-use-ports-own-networks-task.html["Intercluster-LIFs in benutzerdefinierten IPspaces konfigurieren"]


.Schritte
Diese Aufgabe kann mit ONTAP System Manager oder der ONTAP CLI ausgeführt werden.

[role="tabbed-block"]
====
.System Manager
--
. Im lokalen Cluster auf *Cluster > Einstellungen* klicken.
. Im Abschnitt *Intercluster-Einstellungen* auf *Netzwerkschnittstellen hinzufügen* klicken und die IP-Adresse sowie die Subnetzmaske eingeben, um Intercluster-Netzwerkschnittstellen für den Cluster hinzuzufügen.
+
Dieser Schritt ist auf dem Remote-Cluster zu wiederholen.

. Im Remote-Cluster auf *Cluster > Einstellungen* klicken.
. Auf image:icon_kabob.gif["Symbol für Menüoptionen"] im Abschnitt *Cluster Peers* klicken und *Passphrase generieren* auswählen.
. Die Version des Remote ONTAP Clusters auswählen.
. Die generierte Passphrase kopieren.
. Im lokalen Cluster unter *Cluster Peer* auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und *Peer-Cluster* auswählen.
. Im Fenster *Peer-Cluster* die Passphrase einfügen und auf *Cluster-Peering initiieren* klicken.


--
.CLI
--
. Auf dem Ziel-Cluster eine Peer-Beziehung mit dem Quell-Cluster erstellen:
+
[source, cli]
----
cluster peer create -generate-passphrase -offer-expiration <MM/DD/YYYY HH:MM:SS|1...7days|1...168hours> -peer-addrs <peer_LIF_IPs> -initial-allowed-vserver-peers <svm_name|*> -ipspace <ipspace>
----
+
Wenn Sie sowohl  `-generate-passphrase` als auch  `-peer-addrs` angeben, kann nur der Cluster, dessen Intercluster-LIFs in  `-peer-addrs` angegeben sind, das generierte Passwort verwenden.

+
Sie können die  `-ipspace` Option ignorieren, wenn Sie keinen benutzerdefinierten IPspace verwenden. Weitere Informationen zu  `cluster peer create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-peer-create.html["ONTAP-Befehlsreferenz"^].

+
Wenn die Peering-Beziehung in ONTAP 9.6 oder höher erstellt wird und keine Verschlüsselung der Intercluster-Peering-Kommunikation gewünscht ist, muss die  `-encryption-protocol-proposed none` Option zur Deaktivierung der Verschlüsselung verwendet werden.

+
Das folgende Beispiel erstellt eine Cluster Peer-Beziehung mit einem nicht näher spezifizierten Remote-Cluster und autorisiert vorab Peer-Beziehungen mit SVMs `vs1` und `vs2` auf dem lokalen Cluster:

+
[listing]
----
cluster02::> cluster peer create -generate-passphrase -offer-expiration 2days -initial-allowed-vserver-peers vs1,vs2

                     Passphrase: UCa+6lRVICXeL/gq1WrK7ShR
                Expiration Time: 6/7/2017 08:16:10 EST
  Initial Allowed Vserver Peers: vs1,vs2
            Intercluster LIF IP: 192.140.112.101
              Peer Cluster Name: Clus_7ShR (temporary generated)

Warning: make a note of the passphrase - it cannot be displayed again.
----
+
Das folgende Beispiel erstellt eine Cluster Peer-Beziehung mit dem Remote-Cluster an den Intercluster-LIF-IP-Adressen 192.140.112.103 und 192.140.112.104 und autorisiert vorab eine Peer-Beziehung mit einer beliebigen SVM auf dem lokalen Cluster:

+
[listing]
----
cluster02::> cluster peer create -generate-passphrase -peer-addrs 192.140.112.103,192.140.112.104 -offer-expiration 2days -initial-allowed-vserver-peers *

                     Passphrase: UCa+6lRVICXeL/gq1WrK7ShR
                Expiration Time: 6/7/2017 08:16:10 EST
  Initial Allowed Vserver Peers: vs1,vs2
            Intercluster LIF IP: 192.140.112.101,192.140.112.102
              Peer Cluster Name: Clus_7ShR (temporary generated)

Warning: make a note of the passphrase - it cannot be displayed again.
----
+
Das folgende Beispiel erstellt eine Cluster Peer-Beziehung mit einem nicht näher spezifizierten Remote-Cluster und autorisiert vorab Peer-Beziehungen mit SVMs``vs1`` und `vs2` auf dem lokalen Cluster:

+

----

Warning: make a note of the passphrase - it cannot be displayed again.
----
. Im Quell-Cluster wird der Quell-Cluster gegenüber dem Ziel-Cluster authentifiziert:
+
[source, cli]
----
cluster peer create -peer-addrs <peer_LIF_IPs> -ipspace <ipspace>
----
+
Weitere Informationen zu  `cluster peer create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-peer-create.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel authentifiziert das lokale Cluster beim Remote-Cluster an den Intercluster-LIF-IP-Adressen 192.140.112.101 und 192.140.112.102:

+
[listing]
----
cluster01::> cluster peer create -peer-addrs 192.140.112.101,192.140.112.102

Notice: Use a generated passphrase or choose a passphrase of 8 or more characters.
        To ensure the authenticity of the peering relationship, use a phrase or sequence of characters that would be hard to guess.

Enter the passphrase:
Confirm the passphrase:

Clusters cluster02 and cluster01 are peered.
----
+
Geben Sie die Passphrase für die Peer-Beziehung ein, wenn Sie dazu aufgefordert werden.

. Es sollte überprüft werden, ob die Cluster Peer-Beziehung erstellt wurde:
+
[source, cli]
----
cluster peer show -instance
----
