= ONTAP Intercluster-LIFs in benutzerdefinierten IPspaces konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Intercluster-LIFs in benutzerdefinierten IPspaces konfigurieren. Dadurch lässt sich der Replikationsverkehr in mandantenfähigen Umgebungen isolieren.

Wenn ein benutzerdefiniertes IPspace erstellt wird, erstellt das System eine systemeigene Storage Virtual Machine (SVM), die als Container für die Systemobjekte in diesem IPspace dient. Die neue SVM kann als Container für beliebige Intercluster-LIFs im neuen IPspace verwendet werden. Die neue SVM hat denselben Namen wie das benutzerdefinierte IPspace.

.Schritte
. Die Ports im Cluster werden aufgelistet:
+
`network port show`

+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

+

. Benutzerdefinierte IPspaces auf dem Cluster erstellen:
+
`network ipspace create -ipspace _ipspace_`

+
Das folgende Beispiel erstellt den benutzerdefinierten IPspace `ipspace-IC1`:

+
[listing]
----
cluster01::> network ipspace create -ipspace ipspace-IC1
----
+
Weitere Informationen zu  `network ipspace create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-ipspace-create.html["ONTAP-Befehlsreferenz"^].

. Ermitteln, welche Ports für die Intercluster-Verbindung zur Verfügung stehen:
+
`network interface show -fields home-port,curr-port`

+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

. Die verfügbaren Ports aus der Standard Broadcast-Domäne entfernen:
+
`network port broadcast-domain remove-ports -broadcast-domain Default -ports ports`

+
Ein Port kann nicht gleichzeitig in mehr als einer Broadcast-Domäne sein. Weitere Informationen zu  `network port broadcast-domain remove-ports` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-broadcast-domain-remove-ports.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel entfernt Ports `e0e` und `e0f` aus der Standard-Broadcast-Domäne:

+
[listing]
----
cluster01::> network port broadcast-domain remove-ports -broadcast-domain Default -ports
cluster01-01:e0e,cluster01-01:e0f,cluster01-02:e0e,cluster01-02:e0f
----

. Eine Broadcast-Domäne im benutzerdefinierten IPspace erstellen:
+
`network port broadcast-domain create -ipspace _ipspace_ -broadcast-domain _broadcast_domain_ -mtu _MTU_ -ports _ports_`

+
Das folgende Beispiel erstellt die Broadcast-Domäne `ipspace-IC1-bd` im IPspace `ipspace-IC1`:

+
[listing]

cluster01::> network port broadcast-domain create -ipspace ipspace-IC1 -broadcast-domain
ipspace-IC1-bd -mtu 1500 -ports cluster01-01:e0e,cluster01-01:e0f,
cluster01-02:e0e,cluster01-02:e0f
----

. Intercluster-LIFs auf der System-SVM erstellen und der Broadcast-Domäne zuweisen:
+
|===
| Option | Beschreibung 


 a| 
*In ONTAP 9.6 und höher:*
 a| 
`network interface create -vserver _system_SVM_ -lif _LIF_name_ -service-policy default-intercluster -home-node _node_ -home-port _port_ -address _port_IP_ -netmask _netmask_`



 a| 
*In ONTAP 9.5 und früher:*
 a| 
`network interface create -vserver _system_SVM_ -lif _LIF_name_ -role intercluster -home-node _node_ -home-port _port_ -address _port_IP_ -netmask _netmask_`

|===
+
Die LIF wird in der Broadcast-Domäne erstellt, der der Home-Port zugewiesen ist. Die Broadcast-Domäne verfügt über eine Standard Failover-Gruppe mit demselben Namen wie die Broadcast-Domäne. Weitere Informationen zu  `network interface create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-create.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel erstellt Intercluster-LIFs `cluster01_icl01` und `cluster01_icl02` in der Broadcast-Domäne `ipspace-IC1-bd`:

+
[listing]
----
cluster01::> network interface create -vserver ipspace-IC1 -lif cluster01_icl01 -service-
policy default-intercluster -home-node cluster01-01 -home-port e0e -address 192.168.1.201
-netmask 255.255.255.0

cluster01::> network interface create -vserver ipspace-IC1 -lif cluster01_icl02 -service-
policy default-intercluster -home-node cluster01-02 -home-port e0e -address 192.168.1.202
-netmask 255.255.255.0
----
. Es sollte überprüft werden, ob die Intercluster-LIFs erstellt wurden:
+
|===
| Option | Beschreibung 


 a| 
*In ONTAP 9.6 und höher:*
 a| 
`network interface show -service-policy default-intercluster`



 a| 
*In ONTAP 9.5 und früher:*
 a| 
`network interface show -role intercluster`

|===
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

+
[listing]
----
cluster01::> network interface show -service-policy default-intercluster
            Logical    Status     Network            Current       Current Is
Vserver     Interface  Admin/Oper Address/Mask       Node          Port    Home
----------- ---------- ---------- ------------------ ------------- ------- ----
ipspace-IC1
            cluster01_icl01
                       up/up      192.168.1.201/24   cluster01-01  e0e     true
            cluster01_icl02
                       up/up      192.168.1.202/24   cluster01-02  e0f     true
----
. Es sollte sichergestellt werden, dass die Intercluster-LIFs redundant sind:
+
|===
| Option | Beschreibung 


 a| 
*In ONTAP 9.6 und höher:*
 a| 
`network interface show -service-policy default-intercluster -failover`



 a| 
*In ONTAP 9.5 und früher:*
 a| 
`network interface show -role intercluster -failover`

|===
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

+
