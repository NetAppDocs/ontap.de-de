= ONTAP Intercluster SVM Peer-Beziehungen hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn eine SVM nach der Konfiguration einer Cluster Peer-Beziehung erstellt wird, muss die Peer-Beziehung für die SVM manuell hinzugefügt werden. Mit dem  `vserver peer create` Befehl kann eine Peer-Beziehung zwischen SVMs erstellt werden. Nachdem die Peer-Beziehung erstellt wurde, kann  `vserver peer accept` auf dem Remote-Cluster ausgeführt werden, um die Peer-Beziehung zu autorisieren.

.Bevor Sie beginnen
Die Quell- und Zielcluster müssen per Peering verbunden sein.

.Über diese Aufgabe
Es können Peer-Beziehungen zwischen SVMs im selben Cluster für die lokale Datensicherung erstellt werden. Weitere Informationen zu  `vserver peer create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-create.html["ONTAP-Befehlsreferenz"^].

Administratoren verwenden gelegentlich den  `vserver peer reject` Befehl, um eine vorgeschlagene SVM Peer-Beziehung abzulehnen. Befindet sich die Beziehung zwischen SVMs im  `rejected` Status, muss die Beziehung gelöscht werden, bevor eine neue erstellt werden kann. Weitere Informationen zu  `vserver peer reject` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-reject.html["ONTAP-Befehlsreferenz"^].

.Schritte
. Erstellen Sie auf dem Quell-Cluster für die Datensicherung eine Peer-Beziehung zu einer SVM auf dem Ziel-Cluster:
+
`vserver peer create -vserver _local_SVM_ -peer-vserver _remote_SVM_ -applications snapmirror|file-copy|lun-copy -peer-cluster _remote_cluster_`

+
Das folgende Beispiel erstellt eine Peer-Beziehung zwischen der lokalen SVM``pvs1`` und der Remote-SVM``vs1``

+
[listing]
----
cluster01::> vserver peer create -vserver pvs1 -peer-vserver vs1 -applications snapmirror -peer-cluster cluster02
----
+
Wenn die lokalen und entfernten SVMs denselben Namen haben, muss ein _lokaler Name_ verwendet werden, um die SVM Peer-Beziehung zu erstellen:

+
[listing]
----
cluster01::> vserver peer create -vserver vs1 -peer-vserver
vs1 -applications snapmirror -peer-cluster cluster01
-local-name cluster1vs1LocallyUniqueName
----
. Auf dem Datensicherungs-Quell-Cluster überprüfen, ob die Peer-Beziehung initiiert wurde:
+
`vserver peer show-all`

+
Weitere Informationen zu  `vserver peer show-all` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-show-all.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel zeigt, dass die Peer-Beziehung zwischen SVM``pvs1`` und SVM``vs1`` initiiert wurde:

+
[listing]
----
cluster01::> vserver peer show-all
            Peer        Peer                      Peering
Vserver     Vserver     State      Peer Cluster   Applications
--------    --------    ---------  -------------  ------------
pvs1        vs1         initiated   Cluster02       snapmirror
----
. Im Ziel-Cluster für den Datenschutz die ausstehende SVM Peer-Beziehung anzeigen:
+
`vserver peer show`

+
Weitere Informationen zu  `vserver peer show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-show.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel listet die ausstehenden Peer-Beziehungen für  `cluster02` auf:

+
[listing]
----
cluster02::> vserver peer show

                   Peer               Peer
Vserver            Vserver            State
-----------        -----------        ------------
vs1                pvs1               pending
----
. Auf dem Ziel-Cluster für den Datenschutz die ausstehende Peer-Beziehung autorisieren:
+
`vserver peer accept -vserver _local_SVM_ -peer-vserver _remote_SVM_`

+
Weitere Informationen zu  `vserver peer accept` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-accept.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel autorisiert die Peer-Beziehung zwischen der lokalen SVM `vs1` und der Remote-SVM `pvs1`:

+
[listing]
----
cluster02::> vserver peer accept -vserver vs1 -peer-vserver pvs1
----
. Die SVM Peer-Beziehung überprüfen:
+
`vserver peer show`

+
[listing]
----
cluster01::> vserver peer show
            Peer        Peer                           Peering        Remote
Vserver     Vserver     State        Peer Cluster      Applications   Vserver
----------- ----------- ------------ ----------------- -------------- ---------
pvs1        vs1         peered       cluster02         snapmirror     vs1
----
