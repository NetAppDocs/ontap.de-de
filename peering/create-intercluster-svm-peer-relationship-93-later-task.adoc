= Erstellen von ONTAP Intercluster SVM Peer-Beziehungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit dem  `vserver peer create` Befehl lässt sich eine Peer-Beziehung zwischen SVMs auf lokalen und entfernten Clustern erstellen.

.Bevor Sie beginnen
* Die Quell- und Zielcluster müssen per Peering verbunden sein.
* Sie benötigen "vorautorisierte" Peer-Beziehungen für die SVMs auf dem Remote-Cluster.
+
Weitere Informationen finden sich unter link:create-cluster-relationship-93-later-task.html["Erstellung einer Cluster Peer-Beziehung"].



.Über diese Aufgabe
Sie können Peer-Beziehungen für mehrere SVMs „vorautorisieren“, indem Sie die SVMs in der `-initial-allowed-vserver` Option angeben, wenn Sie eine Cluster Peer-Beziehung erstellen. Weitere Informationen finden sich unter link:create-cluster-relationship-93-later-task.html["Erstellung einer Cluster Peer-Beziehung"].

.Schritte
. Im Ziel-Cluster für den Datenschutz werden die SVMs angezeigt, die für das Peering vorautorisiert sind:
+
`vserver peer permission show`

+
[listing]
----
cluster02::> vserver peer permission show
Peer Cluster         Vserver               Applications
-------------------  --------------------  --------------------
cluster02            vs1,vs2               snapmirror
----
. Auf dem Quell-Cluster für den Datenschutz eine Peer-Beziehung zu einer vorautorisierten SVM auf dem Ziel-Cluster für den Datenschutz erstellen:
+
`vserver peer create -vserver _local_SVM_ -peer-cluster _peer_cluster_ -peer-vserver _remote_SVM_ -application _application_`

+
Weitere Informationen zu  `vserver peer create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-peer-create.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel erstellt eine Peer-Beziehung zwischen der lokalen SVM `pvs1` und der vorautorisierten Remote-SVM `vs1`:

+
[listing]
----
cluster01::> vserver peer create -vserver pvs1 -peer-cluster cluster02 -peer-vserver vs1 -application snapmirror
----
. Die SVM Peer-Beziehung überprüfen:
+
`vserver peer show`

+
[listing]
----
cluster01::> vserver peer show
            Peer        Peer                           Peering        Remote
Vserver     Vserver     State        Peer Cluster      Applications   Vserver
----------- ----------- ------------ ----------------- -------------- ---------
pvs1        vs1         peered       cluster02         snapmirror     vs1
----
