= Überlegungen zur ONTAP File System Analytics
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bestimmte Nutzungsbeschränkungen und potenzielle Auswirkungen auf die Leistung im Zusammenhang mit der Implementierung von File System Analytics sind zu beachten.



[[svm-protected-relationships]]
== SVM-geschützte Beziehungen

Wenn Sie die Dateisystemanalyse auf Volumes aktiviert haben, deren enthaltende SVM in einer Schutzbeziehung steht, werden die Analysedaten nicht auf die Ziel-SVM repliziert. Falls die Quell-SVM im Rahmen einer Wiederherstellung neu synchronisiert werden muss, ist die Analyse auf den gewünschten Volumes nach der Wiederherstellung manuell wieder zu aktivieren.



[[performance-considerations]]
== Leistungsüberlegungen

In manchen Fällen kann die Aktivierung der File System Analytics die Leistung während der ersten Metadatenerfassung beeinträchtigen. Dies tritt typischerweise auf Systemen mit maximaler Auslastung auf. Um die Aktivierung der Analytics auf solchen Systemen zu vermeiden, stehen die Leistungsüberwachungstools des ONTAP System Manager zur Verfügung.

Wenn ein deutlicher Anstieg der Latenz auftritt, siehe link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/High_or_fluctuating_latency_after_turning_on_NetApp_ONTAP_File_System_Analytics["NetApp Knowledge Base: Hohe oder schwankende Latenz nach Aktivierung von NetApp ONTAP File System Analytics"^].



[[scan-considerations]]
== Scan-Überlegungen

Wenn Sie die Kapazitätsanalyse aktivieren, führt ONTAP einen Initialisierungsscan für die Kapazitätsanalyse durch. Der Scan greift auf Metadaten aller Dateien in Volumes zu, für die die Kapazitätsanalyse aktiviert ist. Während des Scans werden keine Dateidaten gelesen. Ab ONTAP 9.14.1 lässt sich der Fortschritt des Scans über die ONTAP REST API, im **Explorer**-Tab des System Manager oder mit dem  `volume analytics show` ONTAP CLI-Befehl nachverfolgen. Bei einem Drosselungsereignis stellt ONTAP eine Benachrichtigung bereit.

Beim Aktivieren der Dateisystemanalyse für ein Volume sollte sichergestellt werden, dass mindestens 5 bis 8 Prozent des verfügbaren Speicherplatzes frei sind. Wenn die automatische Größenanpassung aktiviert ist, wird die verfügbare Größe auf Basis der maximalen automatischen Vergrößerung berechnet. Ab ONTAP 9.15.1 erscheint eine Fehlermeldung, wenn beim Aktivieren der Dateisystemanalyse für ein Volume nicht genügend Speicherplatz verfügbar ist.

Nach Abschluss des Scans wird File System Analytics kontinuierlich in Echtzeit aktualisiert, wenn sich das Dateisystem ändert.

Die für den Scan benötigte Zeit ist proportional zur Anzahl der Verzeichnisse und Dateien auf dem Volume. Da beim Scan Metadaten erfasst werden, hat die Dateigröße keinen Einfluss auf die Scanzeit.

Weitere Informationen zum Initialisierungsscan sind unter link:https://www.netapp.com/pdf.html?item=/media/20707-tr-4867.pdf["TR-4867: Leitlinien für bewährte Verfahren zur File System Analytics"^] zu finden.



[[best-practices]]
=== Best Practices

Sie sollten den Scan auf Volumes starten, die keine Aggregate gemeinsam nutzen. Es lässt sich mit dem folgenden Befehl anzeigen, welche Aggregate aktuell welche Volumes hosten:

`volume show -volume _comma-separated-list_of_volumes_ -fields aggr-list`

Während des Scans bedienen die Volumes weiterhin den Client-Verkehr. Es wird empfohlen, den Scan in Zeiten zu starten, in denen mit geringerem Client-Verkehr zu rechnen ist.

Wenn der Client-Datenverkehr zunimmt, werden Systemressourcen beansprucht und der Scan nimmt mehr Zeit in Anspruch.

Ab ONTAP 9.12.1 ist es möglich, die Datenerfassung im System Manager und mit der ONTAP CLI zu pausieren.

* Wenn Sie die ONTAP CLI verwenden:
+
** Sie können die Datenerfassung mit folgendem Befehl pausieren: `volume analytics initialization pause -vserver _svm_name_ -volume _volume_name_`
** Sobald sich der Client-Datenverkehr verlangsamt hat, kann die Datenerfassung mit folgendem Befehl fortgesetzt werden: `volume analytics initialization resume -vserver _svm_name_ -volume _volume_name_`


* Wenn Sie den System Manager verwenden, werden in der *Explorer*-Ansicht des Volumenmenüs die Schaltflächen *Datenerfassung pausieren* und *Datenerfassung fortsetzen* zur Verwaltung des Scans genutzt.
