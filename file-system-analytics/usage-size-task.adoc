= ONTAP Nutzungsanalyse mit FSA aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.12.1 kann die Nutzungsanalyse aktiviert werden, um anzuzeigen, welche Verzeichnisse innerhalb eines Volumes den meisten Speicherplatz belegen. Die Gesamtzahl der Verzeichnisse in einem Volume oder die Gesamtzahl der Dateien in einem Volume kann angezeigt werden. Die Berichterstattung ist auf die 25 Verzeichnisse beschränkt, die den meisten Speicherplatz belegen.

Die Analysen für große Verzeichnisse werden alle 15 Minuten aktualisiert. Die letzte Aktualisierung kann oben auf der Seite am Zeitstempel „Letzte Aktualisierung“ überprüft werden. Über die Schaltfläche „Herunterladen“ können Daten in eine Excel-Arbeitsmappe heruntergeladen werden. Der Download-Vorgang läuft im Hintergrund und zeigt die zuletzt gemeldeten Informationen für das ausgewählte Volume an. Wenn der Scan keine Ergebnisse liefert, sollte sichergestellt werden, dass das Volume online ist. Ereignisse wie SnapRestore führen dazu, dass File System Analytics die Liste der großen Verzeichnisse neu erstellt.

.Schritte
. *Speicher > Volumes* auswählen. Das gewünschte Volume auswählen.
. Im Menü des einzelnen Volumes die Option *Dateisystem* auswählen. Anschließend die Registerkarte *Nutzung* auswählen.
. Der *Analytics*-Schalter wird umgelegt, um die Nutzungsanalyse zu aktivieren.
. Der System Manager zeigt ein Balkendiagramm an, das die Verzeichnisse mit der größten Größe in absteigender Reihenfolge identifiziert.
+

NOTE: ONTAP zeigt möglicherweise nur unvollständige oder gar keine Daten an, während die Liste der wichtigsten Verzeichnisse erfasst wird. Der Fortschritt des Scans kann im Tab *Nutzung* angezeigt werden, der während des Scans angezeigt wird.



Um mehr Einblicke in ein bestimmtes Verzeichnis zu erhalten, ist es möglich, xref:../task_nas_file_system_analytics_view.html[ONTAP Dateisystemaktivität anzeigen].
