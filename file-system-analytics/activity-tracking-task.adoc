= ONTAP Aktivitätsverfolgung mit FSA aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.10.1 umfasst die File System Analytics eine Aktivitätsverfolgungsfunktion, mit der sich Hot Objects identifizieren und die Daten als CSV-Datei herunterladen lassen. Ab ONTAP 9.11.1 wird die Aktivitätsverfolgung auf den SVM-Bereich ausgeweitet. Ebenfalls ab ONTAP 9.11.1 verfügt System Manager über eine Zeitleiste für die Aktivitätsverfolgung, wodurch bis zu fünf Minuten an Aktivitätsverfolgungsdaten betrachtet werden können.

Die Aktivitätsverfolgung ermöglicht die Überwachung in vier Kategorien:

* Verzeichnisse
* Dateien
* Clients
* Benutzer


Die Aktivitätsverfolgung zeigt für jede überwachte Kategorie Lese-IOPS, Schreib-IOPS, Lesedurchsatz und Schreibdurchsatz an. Abfragen zur Aktivitätsverfolgung werden alle 10 bis 15 Sekunden aktualisiert und beziehen sich auf Hotspots, die im System im vorherigen Fünf-Sekunden-Intervall erkannt wurden.

Die Informationen zur Aktivitätsverfolgung sind nur annähernd, und die Genauigkeit der Daten hängt von der Verteilung des eingehenden I/O-Verkehrs ab.

Bei der Anzeige der Aktivitätsverfolgung im System Manager auf Volume-Ebene wird nur das Menü des erweiterten Volumes aktiv aktualisiert. Wenn die Ansicht von Volumes minimiert ist, werden sie erst aktualisiert, wenn die Volume-Anzeige erweitert wird. Sie können die Aktualisierungen mit der Schaltfläche *Pause Refresh* stoppen. Aktivitätsdaten können in einem CSV-Format heruntergeladen werden, das alle zeitpunktgenauen Daten anzeigt, die für das ausgewählte Volume erfasst wurden.

Mit der ab ONTAP 9.11.1 verfügbaren Zeitleistenfunktion kann die Hotspot-Aktivität auf einem Volume oder einer SVM protokolliert werden, wobei die Daten kontinuierlich etwa alle fünf Sekunden aktualisiert und die vorherigen fünf Minuten an Daten gespeichert werden. Zeitleistendaten werden nur für Felder gespeichert, die im sichtbaren Bereich der Seite liegen. Wenn eine Tracking-Kategorie ausgeblendet oder so gescrollt wird, dass die Zeitleiste nicht mehr sichtbar ist, wird die Datenerfassung für die Zeitleiste gestoppt. Standardmäßig sind Zeitleisten deaktiviert und werden automatisch deaktiviert, wenn Sie die Registerkarte „Aktivität“ verlassen.



[[enable-activity-tracking-for-a-single-volume]]
== Aktivitätsverfolgung für ein einzelnes Volume aktivieren

Die Aktivitätsverfolgung lässt sich mit ONTAP System Manager oder der CLI aktivieren.

.Über diese Aufgabe
Wenn Sie RBAC mit der ONTAP REST API oder dem System Manager verwenden, müssen Sie benutzerdefinierte Rollen erstellen, um den Zugriff auf die Aktivitätsverfolgung zu verwalten. Siehe xref:role-based-access-control-task.html[Rollenbasierte Zugriffskontrolle] für diesen Prozess.

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. *Speicher > Volumes* auswählen. Das gewünschte Volume auswählen. Im Menü des einzelnen Volumes „File System“ und anschließend die Registerkarte „Activity“ auswählen.
. Die *Aktivitätsverfolgung* muss aktiviert sein, damit einzelne Berichte zu den wichtigsten Verzeichnissen, Dateien, Clients und Benutzern angezeigt werden können.
. Um Daten ohne Aktualisierungen detaillierter zu analysieren, kann *Aktualisierung anhalten* ausgewählt werden. Die Daten können außerdem heruntergeladen werden, um einen CSV-Datensatz des Berichts zu erhalten.


--
.CLI
--
.Schritte
. Aktivitätsverfolgung aktivieren:
+
`volume activity-tracking on -vserver _svm_name_ -volume _volume_name_`

. Es kann mit folgendem Befehl überprüft werden, ob der Aktivitätsverfolgungsstatus für ein Volume aktiviert oder deaktiviert ist:
+
`volume activity-tracking show -vserver _svm_name_ -volume _volume_name_ -state`

. Nach der Aktivierung lassen sich die Daten der Aktivitätsverfolgung mit dem ONTAP System Manager oder der ONTAP REST API anzeigen.


--
====

== Aktivitätsverfolgung für mehrere Volumes aktivieren

Die Aktivitätsverfolgung kann für mehrere Volumes mit System Manager oder der CLI aktiviert werden.

.Über diese Aufgabe
Wenn Sie RBAC mit der ONTAP REST API oder dem System Manager verwenden, müssen Sie benutzerdefinierte Rollen erstellen, um den Zugriff auf die Aktivitätsverfolgung zu verwalten. Siehe xref:role-based-access-control-task.html[Rollenbasierte Zugriffskontrolle] für diesen Prozess.

[role="tabbed-block"]
====
.System Manager
--
.Für bestimmte Volumes aktivieren
. *Speicher > Volumes* auswählen. Das gewünschte Volume auswählen. Im Menü des einzelnen Volumes „File System“ und anschließend die Registerkarte „Activity“ auswählen.
. Wählen Sie die Volumes aus, für die Sie die Aktivitätsverfolgung aktivieren möchten. Oben in der Volumeliste die Schaltfläche *Weitere Optionen* auswählen. *Aktivitätsverfolgung aktivieren* auswählen.
. Um die Aktivitätsverfolgung auf SVM-Ebene anzuzeigen, die gewünschte SVM unter *Speicher > Volumes* auswählen. Zum Tab „Dateisystem“ und dann zu „Aktivität“ navigieren, dort erscheinen die Daten für die Volumes, bei denen die Aktivitätsverfolgung aktiviert ist.


.Für alle Volumes aktivieren
. *Speicher > Volumes* auswählen. Eine SVM aus dem Menü auswählen.
. Zur Registerkarte *Dateisystem* wechseln und die Registerkarte *Mehr* auswählen, um die Aktivitätsverfolgung auf allen Volumes in der SVM zu aktivieren.


--
.CLI
--
Ab ONTAP 9.13.1 kann die Aktivitätsverfolgung für mehrere Volumes mithilfe der ONTAP CLI aktiviert werden.

.Schritte
. Aktivitätsverfolgung aktivieren:
+
`volume activity-tracking on -vserver _svm_name_ -volume [*|!_volume_names_]`

+
 `*`dient zur Aktivierung der Aktivitätsverfolgung für alle Volumes auf der angegebenen Storage-VM.

+
Verwenden Sie  `!` gefolgt von Volumennamen, um die Aktivitätsverfolgung für alle Volumen auf der SVM mit Ausnahme der benannten Volumen zu aktivieren.

. Bestätigen, dass der Vorgang erfolgreich war:
+
`volume show -fields activity-tracking-state`

. Nach der Aktivierung lassen sich die Daten der Aktivitätsverfolgung mit dem ONTAP System Manager oder der ONTAP REST API anzeigen.


--
====
