= pNFS-Befehle, Statistiken und Ereignisprotokolle
:allow-uri-read: 


[role="lead"]
Diese ONTAP CLI-Befehle beziehen sich speziell auf pNFS. Mit ihnen lassen sich Konfigurationen vornehmen, Fehler beheben und Statistiken erfassen.



[[enable-nfsv4-1]]
== NFSv4.1 aktivieren

[listing]
----
nfs modify -vserver SVM -v4.1 enabled
----


[[enable-pnfs]]
== pNFS aktivieren

[listing]
----
nfs modify -vserver SVM -v4.1-pnfs enabled
----

[[view-nfs-version-information-for-connected-clients]]
== NFS-Versionsinformationen für verbundene Clients

NFS-Verbindungen lassen sich auch mit dem  `nfs connected-clients show` Befehl anzeigen. Zu beachten ist, dass die angezeigte Liste der Clients solche umfasst, die in den letzten 48 Stunden aktiven NFS-Datenverkehr hatten. Inaktive NFS-Clients (auch wenn sie noch eingebunden sind) erscheinen möglicherweise erst, wenn auf die Einbindung zugegriffen wird. Eine Filterung auf nur kürzlich genutzte Clients ist möglich, indem die  `-idle-time` Funktion angegeben wird.

Beispielsweise werden Clients mit Aktivität in den letzten 10 Minuten für die pNFS SVM angezeigt:

[listing]
----
cluster::*> nfs connected-clients show -vserver PNFS -idle-time <10m>

 Node: node-01

Vserver: PNFS Data-Ip: 10.x.x.x Local Remote Client-Ip Protocol Volume Policy Idle-Time Reqs Reqs Trunking

10.x.x.a nfs4.2 PNFS_root default 9m 10s 0 149 false 10.x.x.a nfs4.2 FG_0001 default 9m 10s 135847 0 false 10.x.x.b nfs4.2 PNFS_root default 8m 12s 0 157 false 10.x.x.b nfs4.2 FG_0001 default 8m 12s 52111 0 false
----
