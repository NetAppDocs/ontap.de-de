= ONTAP S3 Architektur mit FlexGroup Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In ONTAP ist die zugrunde liegende Architektur für einen Bucket ein link:../flexgroup/definition-concept.html["FlexGroup-Volume"], ein einzelner Namespace, der aus mehreren konstituierenden Mitgliedsvolumes besteht, aber als ein einzelnes Volume verwaltet wird.

image:fg-overview-s3-config.gif["FlexGroup Volume dargestellt als eine Gruppe von FlexVol Volumes, die als Constituents oder Mitglieder bezeichnet werden"]

Der Zugriff auf den Bucket erfolgt über autorisierte Benutzer und Client-Anwendungen.

image:s3-svm-layout.png["S3 SVM Bucket-Zugriff"]

[NOTE]
====
Wird ein Bucket ausschließlich für S3-Anwendungen verwendet, einschließlich der Verwendung als FabricPool Endpunkt, unterstützt das zugrunde liegende FlexGroup Volume nur das S3-Protokoll.

Ab ONTAP 9.12.1 kann das S3-Protokoll auch in link:../s3-multiprotocol/index.html["Multiprotokoll NAS Volumes"] aktiviert werden, die für die Verwendung von NAS-Protokollen vorkonfiguriert sind. Wenn das S3-Protokoll in Multiprotokoll-NAS-Volumes aktiviert ist, können Clientanwendungen Daten über NFS, SMB und S3 lesen und schreiben.

====


[[bucket-limits]]
== Bucket-Limits



[[minimum-capacity]]
=== Mindestkapazität

Die Mindestkapazität des Buckets wird durch die ONTAP Plattform bestimmt.

* 95GB für lokale Plattformen.
* 1,6 GB für Lab on Demand.
* 200 MB für ONTAP Select.




[[maximum-size]]
=== Maximale Größe

Die maximale Bucket-Kapazität ist auf die maximale FlexGroup-Größe von 60PB begrenzt.



[[maximum-number-of-buckets]]
=== Maximale Anzahl an Buckets

Die maximale Anzahl an Buckets beträgt 1000 pro FlexGroup Volume oder 12.000 Buckets pro Cluster (bei Verwendung von 12 FlexGroup Volumes).



[[automatic-flexgroup-sizing-with-ontap-9-14-1-and-later]]
== Automatische FlexGroup Dimensionierung mit ONTAP 9.14.1 und höher

Ab ONTAP 9.14.1 basiert die Standardgröße des FlexGroup auf der Größe der enthaltenen Buckets. Das FlexGroup-Volume wächst oder schrumpft automatisch, wenn Buckets hinzugefügt oder entfernt werden.

Wenn beispielsweise ein anfänglicher Bucket_A mit 100GB bereitgestellt wird, wird das FlexGroup Thin-Provisioned-Volume ebenfalls auf 100GB festgelegt. Wenn zwei zusätzliche Buckets erstellt werden, Bucket_B mit 300GB und Bucket_C mit 500GB, wächst das FlexGroup Volume auf 900GB.

(Bucket_A mit 100GB + Bucket_B mit 300GB + Bucket_C mit 500GB = 900GB.)

Wenn Bucket_A gelöscht wird, wird das zugrunde liegende FlexGroup Volume auf 800GB verkleinert.



[[fixed-default-flexgroup-sizes-in-ontap-9-13-1-and-earlier]]
== Feste Standardgrößen für FlexGroup in ONTAP 9.13.1 und früher

Um die Kapazität für Bucket-Erweiterungen bereitzustellen, sollte die gesamte belegte Kapazität aller Buckets auf dem FlexGroup Volume weniger als 33 % der maximalen FlexGroup Volume-Kapazität betragen, basierend auf den verfügbaren Speicheraggregaten im Cluster. Falls dies nicht erfüllt werden kann, wird der neu erstellte Bucket auf einem neuen, automatisch erstellten FlexGroup Volume bereitgestellt.

Vor ONTAP 9.14.1 ist die Größe des FlexGroup standardmäßig abhängig von der jeweiligen Umgebung festgelegt:

* 1,6PB in ONTAP
* 100TB in ONTAP Select


Wenn ein Cluster nicht über genügend Kapazität verfügt, um ein FlexGroup Volume in der Standardgröße bereitzustellen, reduziert ONTAP die Standardgröße um die Hälfte, bis es in der bestehenden Umgebung bereitgestellt werden kann.

In einer 300-TB-Umgebung wird ein FlexGroup Volume beispielsweise automatisch mit 200 TB bereitgestellt (1,6-PB-, 800-TB- und 400-TB-FlexGroup Volumes wären zu groß für die Umgebung).
