= Festlegung des Standorts für die Bereitstellung neuer ONTAP S3-Speicherkapazität
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor Sie einen neuen S3-Bucket erstellen, muss entschieden werden, ob er in einer neuen oder in einer bestehenden SVM platziert wird. Diese Entscheidung bestimmt den Workflow.

.Auswahlmöglichkeiten
* Wenn ein Bucket in einer neuen SVM oder einer SVM bereitgestellt werden soll, die nicht für S3 aktiviert ist, sind die Schritte in den folgenden Abschnitten zu befolgen.
+
link:create-svm-s3-task.html["Eine SVM für S3 erstellen"]

+
link:create-bucket-task.html["Einen Bucket für S3 erstellen"]

+
Obwohl S3 in einer SVM zusammen mit NFS und SMB koexistieren kann, kann die Erstellung einer neuen SVM in Betracht gezogen werden, wenn eine der folgenden Bedingungen zutrifft:

+
** S3 wird zum ersten Mal auf einem Cluster aktiviert.
** Sie haben bereits SVMs in einem Cluster, in dem Sie die S3-Unterstützung nicht aktivieren möchten.
** Sie haben eine oder mehrere S3-fähige SVMs in einem Cluster und möchten einen weiteren S3-Server mit anderen Leistungseigenschaften. Nach der Aktivierung von S3 auf der SVM erfolgt die Bereitstellung eines Buckets.


* Wenn der erste Bucket oder ein zusätzlicher Bucket auf einer bestehenden S3-fähigen SVM bereitgestellt werden soll, sind die Schritte im folgenden Thema zu befolgen.
+
link:create-bucket-task.html["Einen Bucket für S3 erstellen"]
