= ONTAP S3 Netzwerkanforderungen beurteilen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor S3-Speicher für Clients bereitgestellt wird, ist zu überprüfen, dass die Netzwerkkonfiguration korrekt ist, sodass die S3-Bereitstellungsanforderungen erfüllt werden.

.Bevor Sie beginnen
Die folgenden Cluster Netzwerkobjekte müssen konfiguriert werden:

* Physische und logische Ports
* Broadcast-Domains
* Subnetze (falls erforderlich)
* IPspaces (falls erforderlich, zusätzlich zum Standard IPspace)
* Failover-Gruppen (falls erforderlich, zusätzlich zur standardmäßigen Failover-Gruppe für jede Broadcast-Domäne)
* Externe Firewalls


.Über diese Aufgabe
Für Remote FabricPool Kapazitätsstufen (Cloud) und Remote-S3-Clients ist eine Daten-SVM zu verwenden und Daten-LIFs sind zu konfigurieren. Für FabricPool Cloud-Tiers sind außerdem Intercluster-LIFs zu konfigurieren, Cluster-Peering ist nicht erforderlich.

Für lokale FabricPool Kapazitätsebenen muss die System-SVM (genannt "`Cluster`") verwendet werden, aber es bestehen zwei Optionen für die LIF-Konfiguration:

* Sie können die Cluster-LIFs verwenden.
+
Bei dieser Option ist keine weitere LIF-Konfiguration erforderlich, aber der Datenverkehr auf den Cluster-LIFs wird zunehmen. Außerdem ist die lokale Tier für andere Cluster nicht zugänglich.

* Sie können Daten- und Intercluster-LIFs verwenden.
+
Diese Option erfordert zusätzliche Konfigurationen, einschließlich der Aktivierung der LIFs für das S3-Protokoll, aber die lokale Ebene ist auch als Remote FabricPool Cloud-Ebene für andere Cluster zugänglich.



.Schritte
. Die verfügbaren physischen und virtuellen Ports werden angezeigt:
+
`network port show`

+
** Wenn möglich, ist der Port mit der höchsten Geschwindigkeit für das Datennetzwerk zu verwenden.
** Alle Komponenten im Datennetzwerk müssen für eine optimale Leistung die gleiche MTU-Einstellung haben.


. Wenn die Verwendung eines Subnetznamens zur Zuweisung der IP-Adresse und des Netzwerkmaskenwerts für eine LIF vorgesehen ist, sollte überprüft werden, ob das Subnetz existiert und ausreichend Adressen verfügbar sind:
+
`network subnet show`

+
Subnetze enthalten einen Pool von IP-Adressen, die zum selben Layer-3-Subnetz gehören. Subnetze werden mit dem  `network subnet create` Befehl erstellt.

+
Weitere Informationen zu  `network subnet show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-show.html["ONTAP-Befehlsreferenz"^].

. Verfügbare IPspaces anzeigen:
+
`network ipspace show`

+
Sie können den Standard-IPspace oder einen benutzerdefinierten IPspace verwenden.

. Wenn Sie IPv6-Adressen verwenden möchten, sollte überprüft werden, ob IPv6 auf dem Cluster aktiviert ist:
+
`network options ipv6 show`

+
Bei Bedarf kann IPv6 mit dem  `network options ipv6 modify` Befehl aktiviert werden.
