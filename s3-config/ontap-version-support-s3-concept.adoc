= ONTAP Version und Plattformunterstützung für S3-Objektspeicher
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
S3 Objektspeicher ist auf allen AFF, FAS und ONTAP Select Plattformen mit ONTAP 9.8 und höher verfügbar.

Wie bei anderen Protokollen wie FC, iSCSI, NFS, NVMe_oF und SMB ist für S3 die Installation einer Lizenz erforderlich, bevor es in ONTAP verwendet werden kann. Die S3-Lizenz ist eine kostenlose Lizenz, muss jedoch auf Systemen installiert werden, die auf ONTAP 9.8 aktualisiert werden. Die S3-Lizenz kann von der link:https://mysupport.netapp.com/site/systems/master-license-keys/ontaps3["Lizenzschlüssel-Seite"^] NetApp Support-Website heruntergeladen werden.

Bei neuen ONTAP 9.8 und höheren Systemen ist die S3-Lizenz vorinstalliert.



[[cloud-volumes-ontap]]
== Cloud Volumes ONTAP

ONTAP S3 ist in Cloud Volumes ONTAP wie in lokalen Umgebungen konfiguriert und funktioniert genauso, mit einer Ausnahme:

* Beim Erstellen von Buckets in Cloud Volumes ONTAP sollte mithilfe der CLI-Prozedur sichergestellt werden, dass das zugrunde liegende FlexGroup Volume nur Aggregate von einem einzigen Knoten verwendet. Die Verwendung von Aggregaten aus mehreren Knoten wirkt sich auf die Performance aus, da sich die Knoten in geografisch getrennten Verfügbarkeitszonen befinden und anfällig für Latenzprobleme sind.


|===
| Cloud-Provider | ONTAP Version 


| Google Cloud | ONTAP 9.12.1 und höher 


| AWS | ONTAP 9.11.0 und höher 


| Azure | ONTAP 9.9.1 und höher 
|===


[[amazon-fsx-for-netapp-ontap]]
== Amazon FSx for NetApp ONTAP

S3-Objektspeicher wird auf Amazon FSx für NetApp Dienste unterstützt, die ONTAP 9.11 und höher verwenden.



[[s3-support-with-metrocluster]]
== S3-Unterstützung mit MetroCluster

Ab ONTAP 9.14.1 ist es möglich, einen S3 Objektspeicherserver auf einer SVM in einem gespiegelten Aggregat in MetroCluster IP- und FC-Konfigurationen zu aktivieren.

Ab ONTAP 9.12.1 ist es möglich, einen S3-Objektspeicherserver auf einer SVM in einem nicht gespiegelten Aggregat in einer MetroCluster IP Konfiguration zu aktivieren. Weitere Informationen zu den Einschränkungen nicht gespiegelter Aggregate in MetroCluster IP Konfigurationen sind unter link:https://docs.netapp.com/us-en/ontap-metrocluster/install-ip/considerations_unmirrored_aggrs.html["Überlegungen zu nicht gespiegelten Aggregaten"^] zu finden.

SnapMirror S3 wird in MetroCluster Konfigurationen nicht unterstützt.



[[s3-public-preview-in-ontap-9-7]]
== Öffentliche Vorschau von S3 in ONTAP 9.7

In ONTAP 9.7 wurde S3 Objektspeicher als öffentliche Vorschau eingeführt. Diese Version war nicht für Produktionsumgebungen vorgesehen und wird ab ONTAP 9.8 nicht mehr aktualisiert. Nur ONTAP 9.8 und spätere Versionen unterstützen S3 Objektspeicher in Produktionsumgebungen.

Mit der öffentlichen Vorschauversion 9.7 erstellte S3-Buckets können in ONTAP 9.8 und höher verwendet werden, jedoch nicht von Funktionserweiterungen profitieren. Für Buckets, die mit der öffentlichen Vorschauversion 9.7 erstellt wurden, wird empfohlen, deren Inhalte in 9.8-Buckets zu migrieren, um Unterstützung für Funktionen, Sicherheit und Leistungsverbesserungen zu erhalten.
