= ONTAP S3 unterstützte Aktionen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP S3-Aktionen werden von den Standard-S3-REST-APIs unterstützt, sofern nachfolgend nichts anderes angegeben ist. Weitere Informationen enthält die link:https://docs.aws.amazon.com/AmazonS3/latest/API/Type_API_Reference.html["Amazon S3 API-Referenz"^].


NOTE: Diese S3-Aktionen werden speziell bei der Verwendung von nativen S3-Buckets in ONTAP unterstützt. Einige dieser Aktionen, wie beispielsweise solche im Zusammenhang mit Versionierung, Objektsperren und anderen Funktionen, werden bei der Verwendung von link:../s3-multiprotocol/index.html["S3 NAS Buckets (S3 in Multiprotokoll NAS Volumes)"] nicht unterstützt.

Sofern für einen bestimmten Vorgang nichts anderes angegeben ist, werden ab ONTAP 9.8 die folgenden gängigen Anforderungsheader unterstützt:

* `Authorization`
* `Connection`
* `Content-Length`
* `Content-MD5`
* `Content-Type`
* `Date`
* `Expect`
* `Host`
* `x-amz-date`

[[object-operations]]
== Objektoperationen

Ab ONTAP 9.19.1 unterstützt ONTAP S3 die Durchsetzung bedingter Schreibvorgänge beim Hochladen von Objekten und bedingter Löschvorgänge beim Löschen von Objekten. Bedingte Schreib- und Löschvorgänge bieten eine Möglichkeit, das Überschreiben oder unbeabsichtigte Löschen von Objekten zu verhindern. Bedingte Operationen funktionieren nur, wenn die Anfrage mit der AWS_V4-Signaturmethode über eine TLS-Verbindung gestellt wird. Die folgenden bedingten Header und Objektoperationen werden unterstützt:

|===
| Objektoperation | Bedingter Header 


 a| 
PutObject

CopyObject

CompleteMultipartUpload
 a| 
If-None-Match

If-Match



 a| 
DeleteObject

DeleteObjects
 a| 
If-Match

|===
Ab ONTAP 9.9.1 unterstützt ONTAP S3 Objektmetadaten und Tagging.

* PutObject und CreateMultipartUpload enthalten Schlüssel-Wert-Paare mithilfe von  `x-amz-meta-<key>.`
+
Zum Beispiel:  `x-amz-meta-project: ontap_s3`.

* GetObject und HeadObject geben benutzerdefinierte Metadaten zurück.
* Im Gegensatz zu Metadaten können Tags unabhängig von Objekten gelesen werden mit:
+
** PutObjectTagging
** GetObjectTagging
** DeleteObjectTagging




Ab ONTAP 9.11.1 unterstützt ONTAP S3 die Objektversionierung und zugehörige Aktionen mit diesen ONTAP APIs:

* GetBucketVersioning
* ListBucketVersions
* PutBucketVersioning


Sofern für eine bestimmte Operation nicht anders angegeben, wird der folgende URI-Abfrageparameter unterstützt:

* `versionId` (wie für Objektoperationen ab ONTAP 9.12.1 erforderlich)

[[group-policies]]
== Gruppenrichtlinien

Diese Operationen sind nicht S3-spezifisch und werden im Allgemeinen mit Identitäts- und Verwaltungsprozessen (IAM) in Verbindung gebracht. ONTAP unterstützt diese Befehle, verwendet jedoch nicht die IAM REST-APIs.

* Richtlinie erstellen
* AttachGroup Richtlinie




[[user-management]]
== Benutzerverwaltung

Diese Operationen sind nicht spezifisch für S3 und stehen im Allgemeinen im Zusammenhang mit IAM-Prozessen.

* CreateUser
* DeleteUser
* CreateGroup
* DeleteGroup




[[s3-actions-by-release]]
== S3-Aktionen nach Release

.ONTAP 9.14.1
ONTAP 9.14.1 bietet Unterstützung für S3 Object Lock.


NOTE: Legal hold-Operationen (Sperren ohne festgelegte Aufbewahrungszeiten) sind nicht unterstützt.

* GetObjectLockConfiguration
* GetObjectRetention
* PutObjectLockConfiguration
* PutObjectRetention


.ONTAP 9.13.1
ONTAP 9.13.1 bietet Unterstützung für das Bucket Lifecycle Management.

* DeleteBucketLifecycleConfiguration
* GetBucketLifecycleConfiguration
* PutBucketLifecycleConfiguration


.ONTAP 9.12.1
ONTAP 9.12.1 bietet Unterstützung für Bucket-Richtlinien und die Möglichkeit, Objekte zu kopieren.

* DeleteBucketPolicy
* GetBucketPolicy
* PutBucketPolicy
* CopyObject
* UploadPartCopy


.ONTAP 9.11.1
ONTAP 9.11.1 bietet Unterstützung für Versionierung, vorab signierte URLs, Chunked Uploads und Unterstützung für gängige S3-Aktionen wie das Erstellen und Löschen von Buckets mit S3-APIs.

* ONTAP S3 unterstützt jetzt das Signieren von Anfragen für Chunked Uploads mit `x-amz-content-sha256:
STREAMING-AWS4-HMAC-SHA256-PAYLOAD`
* ONTAP S3 unterstützt jetzt Clientanwendungen, die vorab signierte URLs verwenden, um Objekte freizugeben oder anderen Benutzern das Hochladen von Objekten zu ermöglichen, ohne dass Benutzeranmeldeinformationen erforderlich sind.
* CreateBucket
* DeleteBucket
* GetBucketVersioning
* ListBucketVersions
* PutBucket
* PutBucketVersioning
* DeleteObjects
* ListObjectVersions



NOTE: Da die zugrundeliegende FlexGroup erst erstellt wird, wenn der erste Bucket erstellt wird, muss zunächst ein Bucket in ONTAP erstellt werden, bevor ein externer Client mit CreateBucket einen Bucket erstellen kann.

.ONTAP 9.10.1
ONTAP 9.10.1 bietet Unterstützung für SnapMirror S3 und GetBucketLocation.

* GetBucketLocation


.ONTAP 9.9.1
ONTAP 9.9.1 erweitert ONTAP S3 um Unterstützung für Objektmetadaten und Tagging.

* PutObject und CreateMultipartUpload enthalten jetzt Schlüssel-Wert-Paare mit  `x-amz-meta-<key>`. Zum Beispiel:  `x-amz-meta-project: ontap_s3`.
* GetObject und HeadObject geben nun benutzerdefinierte Metadaten zurück.


Tags können auch mit Buckets verwendet werden. Im Gegensatz zu Metadaten können Tags unabhängig von Objekten gelesen werden mit:

* PutObjectTagging
* GetObjectTagging
* DeleteObjectTagging
