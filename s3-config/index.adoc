= Informationen zur ONTAP S3 Konfiguration
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.8 kann ein ONTAP Simple Storage Service (S3) Objektspeicherserver in einem ONTAP Cluster aktiviert werden, wobei vertraute Verwaltungstools wie ONTAP System Manager verwendet werden, um leistungsstarken Objektspeicher für Entwicklung und Betrieb in ONTAP schnell bereitzustellen und die Speichereffizienz sowie Sicherheit von ONTAP zu nutzen.


NOTE: Ab Juli 2024 sind Inhalte aus technischen Berichten, die zuvor als PDFs veröffentlicht wurden, in die ONTAP Produktdokumentation integriert. Die ONTAP S3 Dokumentation enthält nun Inhalte aus _TR-4814: S3 in ONTAP best practices_.



[[s3-configuration-with-system-manager-and-the-ontap-cli]]
== S3-Konfiguration mit System Manager und der ONTAP CLI

Sie können ONTAP S3 mit System Manager und der ONTAP CLI konfigurieren und verwalten. Wenn S3 aktiviert und Buckets mit System Manager erstellt werden, wählt ONTAP bewährte Standardwerte für eine vereinfachte Konfiguration aus. Falls Konfigurationsparameter angegeben werden müssen, kann die ONTAP CLI verwendet werden. Wenn der S3-Server und die Buckets über die CLI konfiguriert werden, ist eine Verwaltung mit System Manager weiterhin möglich, oder umgekehrt.

Wenn Sie mit dem System Manager einen S3-Bucket erstellen, konfiguriert ONTAP standardmäßig die höchste verfügbare Leistungsservice-Stufe Ihres Systems. Auf einem AFF System wäre die Standardeinstellung beispielsweise *Extreme*. Leistungsservice-Stufen sind vordefinierte adaptive Quality of Service (QoS) Richtliniengruppen. Statt einer der Standardservice-Stufen kann eine benutzerdefinierte QoS-Richtliniengruppe oder keine Richtliniengruppe angegeben werden.

Vordefinierte adaptive QoS-Richtliniengruppen sind:

* *Extrem*: Wird für Anwendungen genutzt, die die geringste Latenz und die höchste Leistung erwarten.
* *Leistung*: Wird für Anwendungen mit moderaten Leistungsanforderungen und Latenz verwendet.
* *Value*: Wird für Anwendungen verwendet, bei denen Durchsatz und Kapazität wichtiger sind als Latenz.
* *Benutzerdefiniert*: Eine benutzerdefinierte QoS-Richtlinie oder keine QoS-Richtlinie angeben.


Wenn Sie *Für Tiering verwenden* auswählen, werden keine Leistungs-Servicelevel ausgewählt, und das System versucht, kostengünstige Medien mit optimaler Leistung für die gestaffelten Daten auszuwählen.

Siehe auch: link:../performance-admin/adaptive-qos-policy-groups-task.html["Adaptive QoS-Richtliniengruppen"].

ONTAP versucht, diesen Bucket auf lokalen Tiers mit den am besten geeigneten Festplatten bereitzustellen, um das gewählte Servicelevel zu erfüllen. Wenn jedoch angegeben werden muss, welche Festplatten im Bucket enthalten sein sollen, empfiehlt sich die Konfiguration des S3 Objektspeichers über die CLI, wobei die lokalen Tiers (Aggregate) spezifiziert werden. Wird der S3 Server über die CLI konfiguriert, ist eine Verwaltung weiterhin mit System Manager möglich, falls gewünscht.

Wenn die Möglichkeit bestehen soll, festzulegen, welche Aggregate für Buckets verwendet werden, ist dies nur über die CLI möglich.



[[configuring-s3-buckets-on-cloud-volumes-ontap]]
== Konfiguration von S3-Buckets auf Cloud Volumes ONTAP

Wenn Sie Buckets von Cloud Volumes ONTAP bereitstellen möchten, wird dringend empfohlen, die zugrunde liegenden Aggregate manuell auszuwählen, um sicherzustellen, dass sie nur einen Knoten verwenden. Die Verwendung von Aggregaten beider Knoten kann die Leistung beeinträchtigen, da sich die Knoten in geografisch getrennten Verfügbarkeitszonen befinden und daher anfällig für Latenzprobleme sind. Daher sollte in Cloud Volumes ONTAP-Umgebungen xref:create-bucket-task.html[S3 Buckets über die Befehlszeile konfigurieren].

Ansonsten werden S3-Server auf Cloud Volumes ONTAP genauso konfiguriert und gewartet wie in lokalen Umgebungen.
