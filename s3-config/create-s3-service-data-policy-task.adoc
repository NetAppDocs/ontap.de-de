= Die ONTAP S3 Service-Datenrichtlinie erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Dienstrichtlinien für S3-Daten- und Verwaltungsdienste erstellen. Eine S3-Daten-Dienstrichtlinie ist erforderlich, um den S3-Datenverkehr auf LIFs zu ermöglichen.

.Über diese Aufgabe
Eine S3-Datenrichtlinie für Dienste ist erforderlich, wenn Sie Daten-LIFs und Intercluster-LIFs verwenden. Sie ist nicht erforderlich, wenn Sie Cluster-LIFs für den lokalen Tiering-Anwendungsfall verwenden.

Wenn für eine LIF eine Dienstrichtlinie festgelegt wird, wird diese Richtlinie verwendet, um eine Standardrolle, eine Failover-Richtlinie und eine Datenprotokollliste für die LIF zu erstellen.

Obwohl für SVMs und LIFs mehrere Protokolle konfiguriert werden können, ist es eine Best Practice, dass S3 das einzige Protokoll bei der Bereitstellung von Objektdaten ist.

.Schritte
. Die Berechtigungseinstellung auf „Erweitert“ ändern:
+
`set -privilege advanced`

. Erstellen einer Dienstdatenrichtlinie:
+
`network interface service-policy create -vserver _svm_name_ -policy _policy_name_ -services data-core,data-s3-server`

+
Die  `data-core`- und  `data-s3-server`-Dienste sind die einzigen, die für die Aktivierung von ONTAP S3 erforderlich sind, obwohl bei Bedarf weitere Dienste eingeschlossen werden können.

+
Weitere Informationen zu  `network interface service-policy create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-service-policy-create.html["ONTAP-Befehlsreferenz"^].
