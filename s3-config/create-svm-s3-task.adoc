= Eine SVM für ONTAP S3 erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Obwohl S3 in einer SVM mit anderen Protokollen koexistieren kann, kann es sinnvoll sein, eine neue SVM zu erstellen, um den Namensraum und die Arbeitslast zu isolieren.

.Über diese Aufgabe
Wenn Sie S3 Objektspeicher ausschließlich über eine SVM bereitstellen, benötigt der S3 Server keine DNS-Konfiguration. Wenn jedoch andere Protokolle verwendet werden, kann eine DNS-Konfiguration auf der SVM sinnvoll sein.

Wenn Sie den S3-Zugriff auf eine neue Storage-VM mithilfe des System Manager konfigurieren, werden Sie aufgefordert, Zertifikats- und Netzwerkinformationen einzugeben, und die Storage-VM sowie der S3-Objektspeicherserver werden in einem einzigen Vorgang erstellt.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
Sie sollten darauf vorbereitet sein, den S3-Servernamen als vollqualifizierten Domänennamen (FQDN) einzugeben, den Clients für den S3-Zugriff verwenden. Der FQDN des S3-Servers darf nicht mit einem Bucket-Namen beginnen.

Sie sollten darauf vorbereitet sein, IP-Adressen für die Schnittstellenrolle Data einzugeben.

Wenn Sie ein von einer externen Zertifizierungsstelle signiertes Zertifikat verwenden, erfolgt während dieses Vorgangs eine Aufforderung zur Eingabe; alternativ besteht die Möglichkeit, ein systemgeneriertes Zertifikat zu verwenden.

. S3 auf einer Storage-VM aktivieren.
+
.. Eine neue Storage-VM hinzufügen: *Speicher > Storage-VMs* auswählen und dann auf *Hinzufügen* klicken.
+
Falls es sich um ein neues System ohne vorhandene Storage-VMs handelt: *Dashboard > Protokolle konfigurieren* auswählen.

+
Wenn Sie einen S3-Server zu einer bestehenden Storage-VM hinzufügen, auf *Speicher > Storage-VMs* klicken, eine Storage-VM auswählen, auf *Einstellungen* klicken und dann image:icon_gear.gif["Aktionssymbol"] unter *S3* klicken.

.. *S3 aktivieren* auswählen und anschließend den S3-Servernamen eingeben.
.. Wählen Sie den Zertifikatstyp aus.
+
Unabhängig davon, ob ein systemgeneriertes Zertifikat oder ein eigenes ausgewählt wird, ist es für den Clientzugriff erforderlich.

.. Die Netzwerkschnittstellen eingeben.


. Wenn das systemgenerierte Zertifikat ausgewählt wurde, werden die Zertifikatsinformationen angezeigt, sobald die Erstellung der neuen Storage-VM bestätigt ist. *Herunterladen* kann ausgewählt und das Zertifikat für den Clientzugriff gespeichert werden.
+
** Der geheime Schlüssel wird nicht erneut angezeigt.
** Falls Sie die Zertifikatsinformationen erneut benötigen: *Speicher > Speicher-VMs* auswählen, die Speicher-VM auswählen und auf *Einstellungen* klicken.




--
.CLI
--
. Es sollte überprüft werden, ob S3 auf Ihrem Cluster lizenziert ist:
+
[source, cli]
----
system license show -package s3
----
+
Falls dies nicht der Fall ist, kontaktieren Sie Ihren Vertriebsmitarbeiter.

. Eine SVM erstellen:
+
[source, cli]
----
vserver create -vserver <svm_name> -subtype default -rootvolume <root_volume_name> -aggregate <aggregate_name> -rootvolume-security-style unix -language C.UTF-8 -data-services <data-s3-server> -ipspace <ipspace_name>
----
+
** Die UNIX-Einstellung für die  `-rootvolume-security-style` Option verwenden.
** Die Standardoption C.UTF-8 `-language` kann verwendet werden.
** Die  `ipspace` Einstellung ist optional.


. Die Konfiguration und der Status der neu erstellten SVM können überprüft werden:
+
[source, cli]
----
vserver show -vserver <svm_name>
----
+
Das  `Vserver Operational State`Feld muss den  `running`Status anzeigen. Wenn es den  `initializing`Status anzeigt, bedeutet dies, dass ein Zwischenvorgang wie die Erstellung des Root-Volumes fehlgeschlagen ist und die SVM gelöscht und neu erstellt werden muss.



.Beispiele
Der folgende Befehl erstellt eine SVM für den Datenzugriff im IPspace ipspaceA:

[listing]
----
cluster-1::> vserver create -vserver svm1.example.com -rootvolume root_svm1 -aggregate aggr1 -rootvolume-security-style unix -language C.UTF-8 -data-services data-s3-server -ipspace ipspaceA

[Job 2059] Job succeeded:
Vserver creation completed
----
Der folgende Befehl zeigt, dass eine SVM mit einem Root-Volume von 1 GB erstellt wurde, das automatisch gestartet wurde und sich im `running` Status befindet. Das Root-Volume verfügt über eine Standard-Export-Richtlinie, die keine Regeln enthält, sodass das Root-Volume bei der Erstellung nicht exportiert wird. Standardmäßig wird das vsadmin-Benutzerkonto erstellt und befindet sich im `locked` Status. Die vsadmin-Rolle ist dem Standard-vsadmin-Benutzerkonto zugewiesen.
