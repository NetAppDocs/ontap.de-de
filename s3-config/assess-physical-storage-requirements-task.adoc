= Physische Speicheranforderungen für ONTAP S3 bewerten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Bevor S3-Speicher für Clients bereitgestellt wird, muss sichergestellt sein, dass in bestehenden Aggregaten ausreichend Speicherplatz für den neuen Objektspeicher vorhanden ist. Ist dies nicht der Fall, können Festplatten zu bestehenden Aggregaten hinzugefügt oder neue Aggregate des gewünschten Typs und Speicherorts erstellt werden.

.Über diese Aufgabe
Wenn Sie in einer S3-fähigen SVM einen S3-Bucket erstellen, wird ein FlexGroup Volume link:../s3-config/architecture.html#automatic-flexgroup-sizing-with-ontap-9-14-1-and-later["automatisch erstellt"] zur Unterstützung des Buckets erstellt. Sie können ONTAP die zugrunde liegenden Aggregate und FlexGroup Komponenten automatisch auswählen lassen (Standard) oder Sie können die zugrunde liegenden Aggregate und FlexGroup Komponenten selbst auswählen.

Wenn Sie die Aggregate und FlexGroup Komponenten festlegen – beispielsweise, wenn bestimmte Leistungsanforderungen an die zugrunde liegenden Datenträger bestehen –, sollte sichergestellt werden, dass die Aggregatkonfiguration den Best-Practice-Richtlinien für die Bereitstellung eines FlexGroup Volumes entspricht. Weitere Informationen:

* link:../flexgroup/index.html["FlexGroup Volumenmanagement"]
* https://www.netapp.com/pdf.html?item=/media/17251-tr4571a.pdf["NetApp Technical Report 4571-a: Die wichtigsten Best Practices für NetApp ONTAP FlexGroup Volumes"^]


Wenn Sie Buckets von Cloud Volumes ONTAP bereitstellen, wird dringend empfohlen, die zugrunde liegenden Aggregate manuell auszuwählen, um sicherzustellen, dass sie nur einen Knoten verwenden. Die Verwendung von Aggregaten beider Knoten kann die Leistung beeinträchtigen, da sich die Knoten in geografisch getrennten Verfügbarkeitszonen befinden und daher anfällig für Latenzprobleme sind. Informationen dazu link:create-bucket-task.html["Erstellen von Buckets für Cloud Volumes ONTAP"].

Sie können den ONTAP S3 Server verwenden, um eine lokale FabricPool Kapazitäts-Tier zu erstellen, das heißt im selben Cluster wie das Performance-Tier. Dies kann beispielsweise nützlich sein, wenn SSD-Festplatten an ein HA-Paar angeschlossen sind und _kalte_ Daten auf HDD-Festplatten in einem anderen HA-Paar ausgelagert werden sollen. In diesem Anwendungsfall sollten sich der S3 Server und der Bucket mit dem lokalen Kapazitäts-Tier daher in einem anderen HA-Paar als das Performance-Tier befinden. Lokales Tiering wird auf Ein-Knoten- und Zwei-Knoten-Clustern nicht unterstützt.

.Schritte
. Anzeigen des verfügbaren Speicherplatzes in bestehenden Aggregaten:
+
`storage aggregate show`

+
Falls ein Aggregat mit ausreichend Speicherplatz oder der erforderlichen Knotenposition vorhanden ist, sollte dessen Name für die S3-Konfiguration festgehalten werden.

. Falls keine Aggregate mit ausreichend Speicherplatz oder der erforderlichen Knotenposition vorhanden sind, können Festplatten zu einem bestehenden Aggregat mit dem  `storage aggregate add-disks` Befehl hinzugefügt werden oder ein neues Aggregat mit dem  `storage aggregate create` Befehl erstellt werden.
