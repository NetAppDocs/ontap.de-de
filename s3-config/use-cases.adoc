= Primäre Anwendungsfälle von ONTAP S3
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Dies sind die wichtigsten Anwendungsfälle für den Clientzugriff auf ONTAP S3 Services:

* Die Verwendung von FabricPool zum Tiering inaktiver Daten in einen Bucket in ONTAP ermöglicht ONTAP zu ONTAP Tiering. Das Tiering in einen Bucket innerhalb des link:enable-ontap-s3-access-local-fabricpool-task.html["lokales Cluster"] oder das Tiering in einen Bucket auf einem link:enable-ontap-s3-access-remote-fabricpool-task.html["Remote Cluster"] wird jeweils unterstützt. Das Tiering zu ONTAP S3 ermöglicht die Nutzung kostengünstigerer ONTAP Systeme für inaktive Daten und Einsparungen bei neuer Flash-Kapazität, ohne dass zusätzliche FabricPool Lizenzen oder neue Technologien verwaltet werden müssen.
* Ab ONTAP 9.12.1 kann das S3-Protokoll auch in link:../s3-multiprotocol/index.html["Multiprotokoll NAS Volumes"] aktiviert werden, die für die Verwendung von NAS-Protokollen vorkonfiguriert sind. Wenn das S3-Protokoll in Multiprotokoll-NAS-Volumes aktiviert ist, können Client-Anwendungen Daten über S3, NFS und SMB lesen und schreiben, was eine Vielzahl zusätzlicher Anwendungsfälle eröffnet. Einer der häufigsten Anwendungsfälle ist, dass NAS-Clients Daten auf ein Volume schreiben und S3-Clients dieselben Daten lesen und spezialisierte Aufgaben wie Analysen, Business Intelligence, maschinelles Lernen und optische Zeichenerkennung durchführen.



NOTE: ONTAP S3 ist geeignet, wenn S3-Funktionen auf bestehenden ONTAP Clustern ohne zusätzliche Hardware und Verwaltung aktiviert werden sollen. NetApp StorageGRID ist die führende Lösung von NetApp für Objektspeicher. StorageGRID wird für native S3-Anwendungen empfohlen, die den vollständigen Funktionsumfang von S3, erweiterte ILM-Funktionen oder Kapazitäten benötigen, die in ONTAP-basierten Systemen nicht erreichbar sind. Weitere Informationen sind in der link:https://docs.netapp.com/us-en/storagegrid-118/index.html["StorageGRID Dokumentation"^] zu finden.
