= Quoten für ONTAP FlexGroup Volumes verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
In ONTAP 9.4 und früher können Quotenregeln auf FlexGroup Volumes nur zu Berichtszwecken angewendet werden, jedoch nicht zur Durchsetzung von Kontingentlimits. Ab ONTAP 9.5 können Limits für Quotenregeln, die auf FlexGroup Volumes angewendet werden, durchgesetzt werden.

.Über diese Aufgabe
* Ab ONTAP 9.5 können harte, weiche und Schwellenwert-Kontingente für FlexGroup Volumes festgelegt werden.
+
Sie können diese Grenzwerte festlegen, um den Speicherplatz oder die Anzahl der Dateien, die ein bestimmter Benutzer, eine Gruppe oder ein bestimmter Qtree erstellen kann, oder beides zu beschränken. Warnmeldungen zum Kontingentlimit werden in den folgenden Szenarien generiert:

+
** Wenn die Nutzung ein konfiguriertes Softlimit überschreitet, gibt ONTAP eine Warnmeldung aus, aber weiterer Datenverkehr ist weiterhin erlaubt.
+
Wenn die Nutzung später wieder unter das konfigurierte Soft Limit fällt, wird eine Entwarnungsmeldung ausgegeben.

** Wenn die Nutzung einen konfigurierten Schwellenwert überschreitet, gibt ONTAP eine zweite Warnmeldung aus.
+
Es wird keine Entwarnungsmeldung ausgegeben, wenn die Nutzung später unter einen konfigurierten Schwellenwert fällt.

** Wenn die Nutzung ein konfiguriertes endgültiges Limit erreicht, verhindert ONTAP den weiteren Ressourcenverbrauch durch Abweisung des Datenverkehrs.


* In ONTAP 9.5 können Quotenregeln auf dem Ziel-FlexGroup Volume einer SnapMirror Beziehung weder erstellt noch aktiviert werden.
* Während der Quoteninitialisierung werden die Quoten nicht durchgesetzt und es erfolgen keine Benachrichtigungen über Quotenüberschreitungen nach der Quoteninitialisierung.
+
Um zu überprüfen, ob bei der Quoteninitialisierung Quoten überschritten wurden, kann der  `volume quota report` Befehl verwendet werden.

[[behavior-of-flexgroup-volumes-when-quota-limits-are-exceeded]]
== Verhalten der FlexGroup Volumes bei Überschreitung der Kontingentlimits

Ab ONTAP 9.5 werden Kontingentlimits auf FlexGroup volumes unterstützt. Es bestehen einige Unterschiede bei der Durchsetzung von Kontingentlimits auf einem FlexGroup volume im Vergleich zu einem FlexVol volume.

FlexGroup Volumes können die folgenden Verhaltensweisen aufweisen, wenn die Kontingentgrenzen überschritten werden:

* Die Speicher- und Dateinutzung in einem FlexGroup Volume kann bis zu 5 Prozent höher als das konfigurierte endgültige Limit sein, bevor das Kontingentlimit durch die Ablehnung weiteren Datenverkehrs durchgesetzt wird.
+
Um die beste Leistung zu bieten, kann ONTAP zulassen, dass der Speicherplatzverbrauch das konfigurierte endgültige Limit um eine kleine Marge überschreitet, bevor die Kontingentdurchsetzung beginnt. Dieser zusätzliche Speicherplatzverbrauch überschreitet nicht 5 Prozent der konfigurierten endgültigen Limits, 1 GB oder 65536 Dateien, je nachdem, welcher Wert niedriger ist.

* Nachdem das Kontingentlimit erreicht wurde, und ein Benutzer oder Administrator einige Dateien oder Verzeichnisse löscht, sodass die Kontingentnutzung nun unter dem Limit liegt, kann die nachfolgende kontingentverbrauchende Dateioperation mit einer Verzögerung fortgesetzt werden (die Fortsetzung kann bis zu 5 Sekunden dauern).
* Wenn der gesamte Speicherplatz und die Dateinutzung eines FlexGroup volume die konfigurierten Kontingentlimits überschreiten, kann es zu einer leichten Verzögerung bei der Protokollierung einer Ereignisprotokollmeldung kommen.
* Es kann zu Fehlermeldungen der Art „`no space`“ kommen, wenn einige Bestandteile des FlexGroup volume voll sind, die Kontingentgrenzen aber noch nicht erreicht sind.
* Operationen wie das Umbenennen einer Datei oder eines Verzeichnisses oder das Verschieben von Dateien zwischen Qtrees auf Kontingentzielen, für die endgültige Kontingentlimits konfiguriert sind, können im Vergleich zu ähnlichen Operationen auf FlexVol volumes länger dauern.




[[examples-of-quota-enforcement-for-flexgroup-volumes]]
== Beispiele für die Durchsetzung von Kontingenten bei FlexGroup Volumes

Anhand der Beispiele lässt sich nachvollziehen, wie Quoten mit Limits in ONTAP 9.5 und später konfiguriert werden.

.Beispiel 1: Durchsetzung einer Quotenregel mit Festplattenlimits
. Es empfiehlt sich, eine Quotenrichtlinienregel vom Typ  `user` mit sowohl einem erreichbaren Soft-Disk-Limit als auch einem Festplattenlimit zu erstellen.
+
[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -policy-name default -volume FG -type user -target "" -qtree "" -disk-limit 1T -soft-disk-limit 800G
----
. Die Quotenrichtlinienregel kann angezeigt werden:
+
[listing]
----
cluster1::> volume quota policy rule show -vserver vs0 -policy-name default -volume FG

Vserver: vs0               Policy: default           Volume: FG

                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   ""        ""      off           1TB    800GB       -        -          -
----
. Zur Aktivierung der neuen Quotenregel werden Quoten auf dem Volume initialisiert:
+
[listing]
----
cluster1::> volume quota on -vserver vs0 -volume FG -foreground true
[Job 49] Job succeeded: Successful
----
. Sie können die Festplattennutzung und die Dateinutzungsinformationen des FlexGroup volume mithilfe des Quotenberichts einsehen.
+
[listing]
----
cluster1::> volume quota report -vserver vs0 -volume FG
Vserver: vs0

                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
FG                 user    root      50GB      -       1       -
FG                 user    *         800GB    1TB      0       -   *
2 entries were displayed.
----


Nachdem das Festplattenlimit erreicht ist, wird das Ziel der Quotenrichtlinienregel (in diesem Fall der Benutzer) daran gehindert, weitere Daten in die Dateien zu schreiben.

.Beispiel 2: Durchsetzung einer Kontingentregel für mehrere Benutzer
. Es empfiehlt sich, eine Quotenrichtlinienregel vom Typ  `user` zu erstellen, bei der mehrere Benutzer im Kontingentziel angegeben werden (UNIX-Benutzer, SMB-Benutzer oder eine Kombination aus beiden) und bei der die Regel sowohl ein erreichbares Soft-Disk-Limit als auch ein Festplattenlimit aufweist.
+
[listing]
----
cluster1::> quota policy rule create -vserver vs0 -policy-name default -volume FG -type user -target "rdavis,ABCCORP\RobertDavis" -qtree "" -disk-limit 1TB -soft-disk-limit  800GB
----
. Die Quotenrichtlinienregel kann angezeigt werden:
+
[listing]
----
cluster1::> quota policy rule show -vserver vs0 -policy-name default -volume FG

Vserver: vs0               Policy: default           Volume: FG

                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   "rdavis,ABCCORP\RobertDavis"  "" off  1TB  800GB  -  -
----
. Zur Aktivierung der neuen Quotenregel werden Quoten auf dem Volume initialisiert:
+
[listing]
----
cluster1::> volume quota on -vserver vs0 -volume FG -foreground true
[Job 49] Job succeeded: Successful
----
. Es kann überprüft werden, ob der Quotenstatus aktiv ist:
+
[listing]
----
cluster1::> volume quota show -vserver vs0 -volume FG
              Vserver Name: vs0
               Volume Name: FG
               Quota State: on
               Scan Status: -
          Logging Messages: on
          Logging Interval: 1h
          Sub Quota Status: none
  Last Quota Error Message: -
Collection of Quota Errors: -
----
. Sie können die Festplattennutzung und die Dateinutzungsinformationen des FlexGroup volume mithilfe des Quotenberichts einsehen.
+
[listing]
----
cluster1::> quota report -vserver vs0 -volume FG
Vserver: vs0

                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
FG                 user    rdavis,ABCCORP\RobertDavis  0B  1TB  0  -   rdavis,ABCCORP\RobertDavis
----
+
Das Kontingentlimit wird unter allen im Kontingentziel aufgeführten Benutzern aufgeteilt.



Sobald das Festplattenlimit erreicht ist, sind die im Kontingentziel aufgeführten Benutzer daran gehindert, weitere Daten in die Dateien zu schreiben.

.Beispiel 3: Erzwingen des Kontingents mit aktivierter Benutzerzuordnung
. Es empfiehlt sich, eine Quotenrichtlinienregel vom Typ  `user` zu erstellen, einen UNIX-Benutzer oder einen Windows-Benutzer als Quotenziel mit  `user-mapping` auf  `on` festzulegen und die Regel sowohl mit einem erreichbaren Soft-Disk-Limit als auch mit Festplattenlimit zu erstellen.
+
Die Zuordnung zwischen UNIX- und Windows-Benutzern muss zuvor mit dem  `vserver name-mapping create` Befehl konfiguriert werden.

+
[listing]
----
cluster1::> quota policy rule create -vserver vs0 -policy-name default -volume FG -type user -target rdavis -qtree "" -disk-limit 1TB -soft-disk-limit  800GB -user-mapping on
----
. Die Quotenrichtlinienregel kann angezeigt werden:
+
[listing]
----
cluster1::> quota policy rule show -vserver vs0 -policy-name default -volume FG

Vserver: vs0               Policy: default           Volume: FG

                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
user   rdavis    ""      on           1TB    800GB       -        -          -
----
. Zur Aktivierung der neuen Quotenregel werden Quoten auf dem Volume initialisiert:
+
[listing]
----
cluster1::> volume quota on -vserver vs0 -volume FG -foreground true
[Job 49] Job succeeded: Successful
----
. Es kann überprüft werden, ob der Quotenstatus aktiv ist:
+
[listing]
----
cluster1::> volume quota show -vserver vs0 -volume FG
              Vserver Name: vs0
               Volume Name: FG
               Quota State: on
               Scan Status: -
          Logging Messages: on
          Logging Interval: 1h
          Sub Quota Status: none
  Last Quota Error Message: -
Collection of Quota Errors: -
----
. Sie können die Festplattennutzung und die Dateinutzungsinformationen des FlexGroup volume mithilfe des Quotenberichts einsehen.
+
[listing]
----
cluster1::> quota report -vserver vs0 -volume FG
Vserver: vs0

                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
FG                 user    rdavis,ABCCORP\RobertDavis  0B  1TB  0  -   rdavis
----
+
Das Kontingentlimit wird zwischen dem im Kontingentziel aufgeführten Benutzer und dem entsprechenden Windows oder UNIX-Benutzer gemeinsam genutzt.



Sobald das Festplattenlimit erreicht ist, wird sowohl dem im Kontingentziel aufgeführten Benutzer als auch dem entsprechenden Windows- oder UNIX-Benutzer das Schreiben weiterer Daten in die Dateien untersagt.

.Beispiel 4: Überprüfen der Qtree-Größe bei aktiviertem Kontingent
. Es empfiehlt sich, eine Quotenrichtlinienregel vom Typ  `tree` zu erstellen, bei der die Regel sowohl ein erreichbares Soft-Disk-Limit als auch ein Festplattenlimit hat.
+
[listing]
----
cluster1::> quota policy rule create -vserver vs0 -policy-name default -volume FG -type tree -target tree_4118314302 -qtree "" -disk-limit 48GB -soft-disk-limit 30GB
----
. Die Quotenrichtlinienregel kann angezeigt werden:
+
[listing]
----
cluster1::> quota policy rule show -vserver vs0

Vserver: vs0               Policy: default           Volume: FG

                                               Soft             Soft
                         User         Disk     Disk   Files    Files
Type   Target    Qtree   Mapping     Limit    Limit   Limit    Limit  Threshold
-----  --------  ------- -------  --------  -------  ------  -------  ---------
tree   tree_4118314302  "" -          48GB        -      20        -
----
. Zur Aktivierung der neuen Quotenregel werden Quoten auf dem Volume initialisiert:
+
[listing]
----
cluster1::> volume quota on -vserver vs0 -volume FG -foreground true
[Job 49] Job succeeded: Successful
----
+
.. Sie können die Festplattennutzung und die Dateinutzungsinformationen des FlexGroup volume mithilfe des Quotenberichts einsehen.
+
....
cluster1::> quota report -vserver vs0
Vserver: vs0
----Disk---- ----Files----- Quota
Volume Tree Type ID Used Limit Used Limit Specifier
------- -------- ------ ------- ----- ----- ------ ------ ---------
FG tree_4118314302 tree 1 30.35GB 48GB 14 20 tree_4118314302
....
+
Das Kontingentlimit wird zwischen dem im Kontingentziel aufgeführten Benutzer und dem entsprechenden Windows oder UNIX-Benutzer gemeinsam genutzt.



. Von einem NFS-Client aus kann mit dem  `df` Befehl die gesamte Speicherplatznutzung, der verfügbare Speicherplatz und der belegte Speicherplatz angezeigt werden.
+
[listing]
----
scsps0472342001# df -m /t/10.53.2.189/FG-3/tree_4118314302
Filesystem 1M-blocks Used Available Use% Mounted on
10.53.2.189/FG-3 49152 31078 18074 63% /t/10.53.2.189/FG-3
----
+
Mit endgültigem Limit wird die Speicherplatznutzung von einem NFS-Client wie folgt berechnet:

+
** Gesamter Platzbedarf = endgültiges Limit für Baum
** Freier Speicherplatz = endgültiges Limit minus Qtree-Speicherbelegung. Ohne endgültiges Limit wird die Speicherbelegung von einem NFS-Client wie folgt berechnet:
** Speicherplatznutzung = Kontingentnutzung
** Gesamtspeicherplatz = Summe aus Kontingentnutzung und physischem freien Speicherplatz im Volume


. Im SMB-Share kann im Windows Explorer der gesamte Speicherplatzverbrauch, der verfügbare Speicherplatz und der belegte Speicherplatz angezeigt werden.
+
Bei der Berechnung des Speicherplatzbedarfs einer SMB-Freigabe sind folgende Punkte zu berücksichtigen:

+
** Das Benutzerkontingent endgültiges Limit für den Benutzer und die Gruppe wird bei der Berechnung des insgesamt verfügbaren Speicherplatzes berücksichtigt.
** Der niedrigste Wert aus dem freien Speicherplatz der Baumkontingentregel, der Benutzerkontingentregel und der Gruppenkontingentregel wird als freier Speicherplatz für die SMB-Freigabe betrachtet.
** Der gesamte Speicherplatzbedarf ist für SMB variabel und hängt vom endgültigen Limit ab, das dem minimalen freien Speicherplatz innerhalb des Baums, des Benutzers und der Gruppe entspricht.






[[apply-rules-and-limits-on-the-flexgroup-volume]]
== Regeln und Beschränkungen auf dem FlexGroup Volume anwenden

.Schritte
. Quotenregeln für Ziele erstellen: `volume quota policy rule create -vserver vs0 -policy-name quota_policy_of_the_rule -volume flexgroup_vol -type {tree|user|group} -target target_for_rule -qtree qtree_name [-disk-limit hard_disk_limit_size] [-file-limit hard_limit_number_of_files] [-threshold threshold_disk_limit_size] [-soft-disk-limit soft_disk_limit_size] [-soft-file-limit soft_limit_number_of_files]`
+
** Der Quotenzieltyp kann  `user`,  `group` oder  `tree` für FlexGroup Volumes sein.
** Bei der Erstellung von Kontingentregeln für FlexGroup Volumes wird die Angabe eines Pfades als Ziel nicht unterstützt.
** Ab ONTAP 9.5 ist es möglich, Festplattenlimit, hartes Dateilimit, weiches Festplattenlimit, weiches Dateilimit und Schwellenwertlimit-Kontingente für FlexGroup Volumes anzugeben.
+
In ONTAP 9.4 und früher können beim Erstellen von Quota-Regeln für FlexGroup Volumes das Festplattenlimit, das Dateilimit, der Schwellenwert für das Festplattenlimit, das weiche Festplattenlimit oder das weiche Dateilimit nicht angegeben werden.





Das folgende Beispiel zeigt eine Standardquotenregel, die für den Benutzerzieltyp erstellt wird:

[listing]
----
cluster1::> volume quota policy rule create -vserver vs0 -policy-name quota_policy_vs0_1 -volume fg1 -type user -target "" -qtree ""
----
Das folgende Beispiel zeigt die Erstellung einer Baumquotenregel für den Qtree mit dem Namen qtree1:

[listing]
----
cluster1::> volume quota policy rule create -policy-name default -vserver vs0 -volume fg1 -type tree -target "qtree1"
----
. Die Quoten für das angegebene FlexGroup Volume werden aktiviert: `volume quota on -vserver svm_name -volume flexgroup_vol -foreground true`


[listing]
----
cluster1::> volume quota on -vserver vs0 -volume fg1 -foreground true
----
. Der Status der Quoteninitialisierung kann überwacht werden: `volume quota show -vserver svm_name`


FlexGroup Volumes könnten den  `mixed` Zustand anzeigen, was darauf hindeutet, dass sich alle zugehörigen Volumes noch nicht im gleichen Zustand befinden.

[listing]
----
cluster1::> volume quota show -vserver vs0
                                          Scan
Vserver    Volume        State            Status
---------  ------------  ---------------  ------
vs0        fg1           initializing         95%
vs0        vol1          off                   -
2 entries were displayed.
----
. Der Quotenbericht für das FlexGroup Volume mit aktiven Quoten: `volume quota report -vserver svm_name -volume flexgroup_vol`
+
Mit dem  `volume quota report` Befehl kann kein Pfad für FlexGroup Volumes angegeben werden.

+
Das folgende Beispiel zeigt das Benutzerkontingent für das FlexGroup Volume `fg1`:

+
....
cluster1::> volume quota report -vserver vs0 -volume fg1
  Vserver: vs0
                                      ----Disk----  ----Files-----   Quota
  Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
  -------  --------  ------  -------  -----  -----  ------  ------   ---------
  fg1                user    *           0B      -       0       -   *
  fg1                user    root       1GB      -       1       -   *
  2 entries were displayed.
....
+
Das folgende Beispiel zeigt das Baumkontingent für das FlexGroup Volume `fg1`:

+
[listing]
----
cluster1::> volume quota report -vserver vs0 -volume fg1
Vserver: vs0

                                    ----Disk----  ----Files-----   Quota
Volume   Tree      Type    ID        Used  Limit    Used   Limit   Specifier
-------  --------  ------  -------  -----  -----  ------  ------   ---------
fg1      qtree1  tree      1         68KB      -      18       -   qtree1
fg1              tree      *           0B      -       0       -   *
2 entries were displayed.
----


.Ergebnisse
Die Quotenregeln und -grenzen gelten für das FlexGroup Volume.

Die Nutzung kann bis zu 5 Prozent höher als das konfigurierte endgültige Limit sein, bevor ONTAP das Kontingent durchsetzt, indem weiterer Datenverkehr abgelehnt wird.
