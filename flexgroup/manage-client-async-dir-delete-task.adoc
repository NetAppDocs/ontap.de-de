= Clientrechte für das asynchrone Löschen von ONTAP-Verzeichnissen mit FlexGroups verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.11.1 können Speicheradministratoren Berechtigungen für ein Volume erteilen, damit NFS- und SMB-Clients asynchrone Löschvorgänge durchführen können. Wenn asynchrone Löschung im Cluster aktiviert ist, können Linux-Clientbenutzer den  `mv` Befehl und Windows-Clientbenutzer den  `rename` Befehl verwenden, um ein Verzeichnis auf dem angegebenen Volume zu löschen, indem es in ein verstecktes Verzeichnis verschoben wird, das standardmäßig den Namen .ontaptrashbin trägt.

Die Rechte werden pro Band gewährt.

Sie können nur Verzeichnisse verschieben. Sie können keine Dateien in das Verzeichnis .ontaptrashbin verschieben.

link:fast-directory-delete-asynchronous-task.html#delete-directories-asynchronously["Informationen zur Verwendung von ONTAP zum asynchronen Löschen von Verzeichnissen aus FlexVol und FlexGroup Volumes"].



[[enable-client-asynchronous-directory-delete]]
== Asynchrone Verzeichnislöschung auf Clientseite aktivieren

.Schritte
. Wechseln Sie über die Cluster-CLI in den erweiterten Berechtigungsmodus: `-privilege advanced`
. Das asynchrone Löschen auf Clientseite am Mountpunkt eines Volumes kann aktiviert werden, und optional kann ein alternativer Name für das Papierkorbverzeichnis angegeben werden:
+
`volume file async-delete client enable volume _volname_ vserver _vserverName_ trashbinname _name_`

+
Beispiel mit dem Standardnamen für den Papierkorb:

+
[listing]
----
cluster1::*> volume file async-delete client enable -volume v1 -vserver  vs0

Info: Async directory delete from the client has been enabled on volume "v1" in
      Vserver "vs0".
----
+
Beispiel für die Angabe eines alternativen Papierkorbnamens:

+
[listing]
----
cluster1::*> volume file async-delete client enable -volume test -trashbin .ntaptrash -vserver vs1

Success: Async directory delete from the client is enabled on volume "v1" in
      Vserver "vs0".
----
. Überprüfung, ob asynchrones Löschen auf Clientseite aktiviert ist:
+
`volume file async-delete client show`

+
Beispiel:

+
[listing]
----
cluster1::*> volume file async-delete client show

Vserver Volume      async-delete client TrashBinName
------------ ------------ ----------------------- -------------
vs1          vol1         Enabled          .ntaptrash
vs2          vol2         Disabled          -

2 entries were displayed.
----




[[disable-client-asynchronous-directory-delete]]
== Deaktivieren der asynchronen Verzeichnislöschung auf Clientseite

.Schritte
. Über die Cluster-CLI wird die asynchrone Verzeichnislöschung auf Clientebene deaktiviert:
+
`volume file async-delete client disable volume _volname_ vserver _vserverName_`

+
Beispiel:

+
[listing]
----
cluster1::*> volume file async-delete client disable -volume vol1 -vserver vs1

       Success: Asynchronous directory delete client disabled successfully on volume.
----
. Überprüfung, ob asynchrones Löschen auf dem Client deaktiviert ist:
+
`volume file async-delete client show`

+
Beispiel:

+
[listing]
----
cluster1::*> volume file async-delete client show

Vserver      Volume      async-delete client      TrashBinName
------------ ------------ ----------------------- -------------
vs1          vol1         Disabled                -
vs2          vol2         Disabled                -

2 entries were displayed.
----
