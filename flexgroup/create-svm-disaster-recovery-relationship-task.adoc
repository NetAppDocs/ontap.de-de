= Erstellen von SVM-Notfallwiederherstellungsbeziehungen für ONTAP FlexGroup Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.9.1 ist das Erstellen von SVM Disaster Recovery (SVM DR) Beziehungen mithilfe von FlexGroup Volumes möglich. Eine SVM DR Beziehung bietet Redundanz und die Möglichkeit, FlexGroup Volumes im Katastrophenfall wiederherzustellen, indem die SVM-Konfiguration und deren Daten synchronisiert und repliziert werden. Für SVM DR ist eine SnapMirror Lizenz erforderlich.

.Bevor Sie beginnen
Sie können keine FlexGroup SVM DR-Beziehung erstellen, wenn Folgendes zutrifft.

* Eine FlexClone FlexGroup Konfiguration existiert
* Das FlexGroup Volume ist Teil einer kaskadierenden Beziehung
* Das FlexGroup Volume ist Teil einer Fanout-Beziehung, und Ihr Cluster verwendet eine ONTAP Version vor ONTAP 9.12.1. (Ab ONTAP 9.13.1 werden Fanout-Beziehungen unterstützt.)


.Über diese Aufgabe
* Auf allen Knoten beider Cluster muss die gleiche ONTAP Version laufen wie auf dem Knoten, auf dem die SVM DR Unterstützung hinzugefügt wurde (ONTAP 9.9.1 oder höher).
* Die SVM DR-Beziehung zwischen den primären und sekundären Standorten sollte intakt sein und sowohl auf den primären als auch auf den sekundären SVMs ausreichend Platz bieten, um die FlexGroup Volumes zu unterstützen.
* Ab ONTAP 9.12.1 können FabricPool, FlexGroup und SVM DR gemeinsam genutzt werden. In Versionen vor ONTAP 9.12.1 funktionierten jeweils zwei dieser Funktionen zusammen, jedoch nicht alle drei gemeinsam.
* Wenn eine FlexGroup SVM DR-Beziehung erstellt wird, bei der das FlexGroup Volume Teil einer Fanout-Beziehung ist, sind die folgenden Anforderungen zu berücksichtigen:
+
** Der Quell- und Ziel-Cluster muss mit ONTAP 9.13.1 oder höher betrieben werden.
** SVM DR mit FlexGroup Volumes unterstützt SnapMirror Fanout-Beziehungen zu acht Standorten.




Informationen zum Erstellen einer SVM DR-Beziehung sind unter link:../data-protection/snapmirror-svm-replication-workflow-concept.html["SVM-Replikation von SnapMirror verwalten"] verfügbar.

.Schritte
. Eine SVM DR Beziehung erstellen oder eine bestehende Beziehung verwenden.
+
link:../data-protection/replicate-entire-svm-config-task.html["Eine vollständige SVM-Konfiguration replizieren"]

. Ein FlexGroup Volume wird am primären Standort mit der erforderlichen Anzahl an Bestandteilen erstellt.
+
link:create-task.html["Erstellen eines FlexGroup Volumens"].

+
Warten Sie, bis FlexGroup und alle seine Bestandteile erstellt sind, bevor Sie fortfahren.

. Um das FlexGroup Volume zu replizieren, wird die SVM am sekundären Standort aktualisiert: `snapmirror update -destination-path destination_svm_name: -source-path source_svm_name:`
+
Sie können auch überprüfen, ob bereits ein geplantes SnapMirror Update existiert, indem Sie  `snapmirror show -fields schedule` eingeben

. Am sekundären Standort kann überprüft werden, ob die SnapMirror Beziehung intakt ist: `snapmirror show`
