= ONTAP FlexGroup Volumes so konfigurieren, dass ihre Größe automatisch wächst und schrumpft
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.3 können FlexGroup Volumes so konfiguriert werden, dass sie sich je nach aktuellem Speicherplatzbedarf automatisch vergrößern und verkleinern.

.Bevor Sie beginnen
Das FlexGroup volume muss online sein.

.Über diese Aufgabe
Die automatische Größenanpassung von FlexGroup Volumes ist in zwei Modi möglich:

* Die Größe des Volumes im Modus der automatischen Größenanpassung erhöhen (`grow`)
+
Automatisches Wachstum hilft zu verhindern, dass ein FlexGroup Volume keinen Speicherplatz mehr hat, sofern das Aggregate zusätzlichen Speicherplatz bereitstellen kann. Die maximale Größe für das Volume kann konfiguriert werden. Die Vergrößerung wird automatisch ausgelöst, basierend auf der Menge der auf das Volume geschriebenen Daten im Verhältnis zum aktuell belegten Speicherplatz und den festgelegten Schwellenwerten.

+
Standardmäßig beträgt die maximale Größe eines Volumes 120 % der Größe, bei der die automatische Größenanpassung aktiviert ist. Falls sichergestellt werden muss, dass das Volume größer werden kann, ist die maximale Größe für das Volume entsprechend festzulegen.

* Die Größe des Volumens wird im automatischen (`grow_shrink` Modus verkleinert
+
Die automatische Verkleinerung verhindert, dass ein Volumen größer als nötig ist, und gibt dadurch Speicherplatz im Aggregat für die Nutzung durch andere Volumes frei.

+
Autoshrink kann nur in Kombination mit Autogrow verwendet werden, um sich ändernden Speicherplatzanforderungen gerecht zu werden, und ist nicht allein verfügbar. Wenn Autoshrink aktiviert ist, verwaltet ONTAP das Verkleinerungsverhalten eines Volumes automatisch, um eine Endlosschleife von Autogrow- und Autoshrink-Aktionen zu verhindern.



Wenn ein Volume wächst, kann die maximale Anzahl der darauf speicherbaren Dateien automatisch erhöht werden. Beim Verkleinern eines Volumes bleibt die maximale Anzahl der darauf speicherbaren Dateien unverändert, und ein Volume kann nicht automatisch unter die Größe verkleinert werden, die seiner aktuellen maximalen Dateianzahl entspricht. Aus diesem Grund ist es möglicherweise nicht möglich, ein Volume automatisch vollständig auf seine ursprüngliche Größe zu verkleinern.

.Schritte
. Das Volume wird so konfiguriert, dass es seine Größe automatisch vergrößert und verkleinert: `volume autosize -vserver vserver_name -volume vol_name -mode [grow | grow_shrink]`
+
Sie können außerdem die maximale Größe, die minimale Größe und Schwellenwerte für das Vergrößern oder Verkleinern des Volumens angeben.

+
Der folgende Befehl aktiviert die automatische Größenänderung für ein Volume namens  `fg1`. Das Volume ist so konfiguriert, dass es auf eine maximale Größe von 5 TB anwächst, wenn es zu 70 % gefüllt ist.

+
[listing]
----
cluster1::> volume autosize -volume fg1 -mode grow -maximum-size 5TB -grow-threshold-percent 70
vol autosize: volume "vs_src:fg1" autosize settings UPDATED.
----
