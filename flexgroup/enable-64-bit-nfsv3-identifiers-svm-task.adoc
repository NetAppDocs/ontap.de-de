= 64-Bit-NFSv3-Kennungen auf ONTAP SVMs mit FlexGroups aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um die hohe Dateianzahl von FlexGroup Volumes zu unterstützen und Datei-ID-Kollisionen zu vermeiden, sollten 64-Bit-Dateibezeichner auf der SVM aktiviert werden, auf der das FlexGroup Volume erstellt werden muss.

.Schritte
. Melden Sie sich mit erweiterten Berechtigungen an: `set -privilege advanced`
. Die SVM wird so angepasst, dass sie 64-Bit-NFSv3-FSIDs und Datei-IDs verwendet: `vserver nfs modify -vserver svm_name -v3-64bit-identifiers enabled`
+
[listing]
----
cluster1::*> vserver nfs modify -vserver vs0 -v3-64bit-identifiers enabled

Warning: You are attempting to increase the number of bits used for NFSv3
         FSIDs and File IDs from 32 to 64 on Vserver "vs0". This could
         result in older client software no longer working with the volumes
         owned by Vserver "vs0".
Do you want to continue? {y|n}: y

Warning: Based on the changes you are making to the NFS server on Vserver
         "vs0", it is highly recommended that you remount all NFSv3 clients
         connected to it after the command completes.
Do you want to continue? {y|n}: y
----


.Nachdem Sie fertig sind
Alle Clients müssen neu eingebunden werden. Dies ist erforderlich, da sich die Dateisystem-IDs ändern und die Clients beim Ausführen von NFS-Operationen möglicherweise Meldungen über veraltete Dateihandles erhalten.
