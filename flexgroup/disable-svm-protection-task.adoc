= SVM-Schutz auf ONTAP FlexGroup Volumes deaktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn das SVM DR-Flag auf `protected` einem FlexGroup Volume gesetzt ist, kann das Flag auf unprotected gesetzt werden, um SVM DR `protection` auf einem FlexGroup Volume zu deaktivieren.

.Bevor Sie beginnen
* Die SVM DR Beziehung zwischen Primär- und Sekundärsystem ist fehlerfrei.
* Der SVM DR-Schutzparameter ist auf  `protected` eingestellt.


.Schritte
. Der Schutz kann deaktiviert werden, indem der  `volume modify` Befehl verwendet wird, um den  `vserver-dr-protection` Parameter für das FlexGroup Volume auf  `unprotected` zu ändern.
+
[listing]
----
cluster2::> volume modify -vserver vs1 -volume fg_src -vserver-dr-protection unprotected
[Job 5384] Job is queued: Modify fg_src.
[Job 5384] Steps completed: 4 of 4.
cluster2::>
----
. Die SVM am sekundären Standort wird aktualisiert: `snapmirror update -destination-path destination_svm_name: -source-path Source_svm_name:`
. Es sollte überprüft werden, ob die SnapMirror Beziehung intakt ist: `snapmirror show`
. Es sollte überprüft werden, ob die FlexGroup SnapMirror Beziehung entfernt wurde: `snapmirror show -expand`


.Verwandte Informationen
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-show.html["snapmirror show"^]
* link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-update.html["SnapMirror update"^]
