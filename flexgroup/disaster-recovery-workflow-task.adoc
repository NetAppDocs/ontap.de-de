= Workflow zur Notfallwiederherstellung für ONTAP FlexGroup Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn auf dem Quell-FlexGroup-Volume eine Störung auftritt, sollte das Ziel-FlexGroup-Volume aktiviert und der Clientzugriff umgeleitet werden. Abhängig davon, ob das Quell-FlexGroup-Volume wiederhergestellt werden kann, sollte entweder das Quell-FlexGroup-Volume reaktiviert oder die SnapMirror Beziehung umgekehrt werden.

image:flexgroup-dr-activation.gif["FlexGroup Volumes Disaster-Recovery-Workflow"]

.Über diese Aufgabe
Der Clientzugriff auf das Ziel-FlexGroup-Volume ist für einen kurzen Zeitraum blockiert, wenn einige SnapMirror-Operationen wie SnapMirror-Unterbrechung und Resynchronisierung ausgeführt werden. Wenn der SnapMirror-Vorgang fehlschlägt, kann es vorkommen, dass einige der Bestandteile in diesem Zustand verbleiben und der Zugriff auf das FlexGroup-Volume verweigert wird. In solchen Fällen sollte der SnapMirror-Vorgang erneut versucht werden.
