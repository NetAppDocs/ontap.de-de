= Die Größe von ONTAP FlexGroup Volumes verringern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.6 kann die Größe eines FlexGroup Volumes auf einen Wert unterhalb der aktuellen Größe verringert werden, um den ungenutzten Speicherplatz aus dem Volume freizugeben. Wenn die Größe eines FlexGroup Volumes reduziert wird, passt ONTAP automatisch die Größe aller FlexGroup Bestandteile an.

.Schritt
. Die aktuelle FlexGroup Volume-Größe kann mit folgendem Befehl angezeigt werden: 'volume size -vserver _vserver_name_ -volume _fg_name_'
. Die Größe des FlexGroup Volumens verringern: `volume size -vserver _vserver_name_ -volume _fg_name_ _new_size_`
+
Wenn Sie die neue Größe festlegen, können Sie entweder einen niedrigeren Wert als die aktuelle Größe oder einen negativen Wert unter Verwendung des Minuszeichens (-) angeben, um den die aktuelle Größe des FlexGroup Volumens verringert wird.

+
[NOTE]
====
Wenn die automatische Verkleinerung für das Volume (`volume autosize`-Kommando aktiviert ist, wird die minimale automatische Größenanpassung auf die neue Größe des Volumes festgelegt.

====
+
Das folgende Beispiel zeigt die aktuelle Größe des FlexGroup Volumes mit dem Namen volX und passt die Größe des Volumes auf 10 TB an:

+
[listing]
----
cluster1::> volume size -vserver svm1 -volume volX
(volume size)
vol size: FlexGroup volume 'svm1:volX' has size 15TB.

cluster1::> volume size -vserver svm1 -volume volX 10TB
(volume size)
vol size: FlexGroup volume 'svm1:volX' size set to 10TB.
----
+
Das folgende Beispiel zeigt die aktuelle Größe des FlexGroup Volume mit dem Namen volX und reduziert die Größe des Volumes um 5 TB:

+
[listing]
----
cluster1::> volume size -vserver svm1 -volume volX
(volume size)
vol size: FlexGroup volume 'svm1:volX' has size 15TB.

cluster1::> volume size -vserver svm1 -volume volX -5TB
(volume size)
vol size: FlexGroup volume 'svm1:volX' size set to 10TB.
----
