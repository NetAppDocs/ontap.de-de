= ONTAP FlexGroup Volumes durch Umverteilung von Dateidaten ausgleichen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.16.1 kann die erweiterte Kapazitätsverteilung aktiviert werden, um die Datenverteilung zwischen FlexGroup Mitgliedsvolumes zu ermöglichen, wenn sehr große Dateien wachsen und Speicherplatz auf einem Mitgliedsvolume belegen.

Die erweiterte Kapazitätsverteilung erweitert die in ONTAP 9.12.1 eingeführte detaillierte Datenfunktionalität, die es ONTAP ermöglicht, link:manage-flexgroup-rebalance-task.html["FlexGroup Volumes neu verteilen"] indem Dateien auf andere Mitglieder verschoben werden. Ab ONTAP 9.16.1 werden bei Aktivierung der erweiterten Kapazitätsverteilung mit der  `-granular-data advanced` Option sowohl die grundlegenden Funktionen zur Dateiverteilung als auch die erweiterten Kapazitätsfunktionen aktiviert.

[NOTE]
====
Sowohl die Dateineuverteilung als auch die erweiterte Kapazitätsverteilung sind standardmäßig deaktiviert. Nach der Aktivierung dieser Funktionen können sie nicht mehr deaktiviert werden. Wenn die Kapazitätsverteilung deaktiviert werden muss, ist eine Wiederherstellung eines Snapshots erforderlich, der vor der Aktivierung der erweiterten Kapazitätsverteilung erstellt wurde.

====
Die erweiterte Kapazitätsverteilung wird ausgelöst, wenn neue Schreibvorgänge 10 GB oder 1 % des freien Speicherplatzes eines Volumes erreichen.



[[how-files-are-distributed]]
== Wie Dateien verteilt werden

Wenn eine Datei erstellt wird oder so groß wird, dass sie eine erweiterte Kapazitätsverteilung auslöst, wird die Datei in Streifen zwischen 1GB und 10GB auf die einzelnen FlexGroup Volumes verteilt.

Wenn die erweiterte Kapazitätsverteilung aktiviert ist, wird ONTAP bestehende große Dateien nicht nachträglich streifen. Wenn eine bestehende große Datei nach Aktivierung der erweiterten Kapazitätsverteilung weiter wächst, kann neuer Inhalt in bestehenden großen Dateien abhängig von der Dateigröße und dem verfügbaren Speicherplatz über die FlexGroup Volumes verteilt werden.

Eine Methode zur Bestimmung der Stripe-Breite bei der erweiterten Kapazitätsverteilung besteht darin, den verfügbaren freien Speicherplatz auf dem Mitgliedsvolume zu nutzen. Die erweiterte Kapazitätsverteilung erzeugt einen Datei-Stripe, der 1 % des verfügbaren freien Speicherplatzes ausmacht. Das bedeutet, dass Stripes bei größerem verfügbarem Speicherplatz zunächst breiter sein können und sie kleiner werden, wenn die FlexGroup sich füllt.



[[supported-protocols]]
== Unterstützte Protokolle

Erweiterter Kapazitätsausgleich wird mit den folgenden Protokollen unterstützt:

* NFSv3, NFSv4, NFSv4.1
* pNFS
* SMB




[[enable-advanced-capacity-balancing]]
== Erweiterter Kapazitätsausgleich aktivieren

Die erweiterte Kapazitätsverteilung ist standardmäßig deaktiviert. Die erweiterte Kapazitätsverteilung muss aktiviert werden, damit die FlexGroup Kapazität automatisch ausgeglichen wird. Zu beachten ist, dass diese Funktion nach der Aktivierung nicht mehr deaktiviert werden kann, jedoch kann ein Snapshot wiederhergestellt werden, der vor der Aktivierung der erweiterten Kapazitätsverteilung erstellt wurde.

.Bevor Sie beginnen
* Alle Knoten im Cluster müssen ONTAP 9.16.1 oder höher ausführen.
* Sie können nicht auf eine Version vor ONTAP 9.16.1 zurückgreifen, wenn die erweiterte Kapazitätsverteilung aktiviert ist. Wenn ein Zurücksetzen erforderlich ist, muss zunächst ein Snapshot wiederhergestellt werden, der vor der Aktivierung der erweiterten Kapazitätsverteilung erstellt wurde.
* Wenn NFS Copy Offload (`vserver nfs -vstorage enabled` auf einer SVM aktiviert ist, kann die erweiterte Kapazitätsverteilung auf einem FlexGroup Volume nicht aktiviert werden. Ebenso kann NFS Copy Offload nicht aktiviert werden, wenn die erweiterte Kapazitätsverteilung auf einem FlexGroup Volume in einer SVM aktiviert ist.
* Erweiterter Kapazitätsausgleich wird bei FlexCache Write-Back nicht unterstützt.
* SnapMirror Übertragungen werden bei ONTAP Versionen vor ONTAP 9.16.1 nicht unterstützt, wenn die erweiterte Kapazitätsverteilung auf Volumes in Clustern aktiviert ist, auf denen ONTAP 9.16.1 oder höher ausgeführt wird.
* Deaktivieren Sie SMB Multichannel, bevor Sie die erweiterte Kapazitätsverteilung aktivieren. Die Verwendung von SMB Multichannel mit erweiterter Kapazitätsverteilung kann zu hoher Latenz führen. Weitere Informationen finden sich unter link:https://kb.netapp.com/on-prem/ontap/da/NAS/NAS-Issues/CONTAP-400433["CONTAP-400433: Hohe Latenz beim Lesen/Schreiben bei Verwendung von FlexGroup Rebalancing/GDD über SMB Multichannel-aktivierte Clients"^].


.Über diese Aufgabe
Während der Erstellung von DP Ziel-Volumes mit einer der granularen Datenoptionen (einfach oder erweitert) wird die Einstellung im Ziel als „deaktiviert“ angezeigt, bis die SnapMirror-Übertragung abgeschlossen ist. Nach Abschluss der Übertragung wird im DP Ziel „granulare Daten“ als „aktiviert“ angezeigt.



[[enable-advanced-capacity-balancing-during-flexgroup-creation]]
=== Erweiterter Kapazitätsausgleich während der FlexGroup-Erstellung aktivieren

.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um beim Erstellen eines neuen FlexGroup Volume die erweiterte Kapazitätsverteilung zu aktivieren.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Speicher > Volumes* und klicken Sie auf image:icon_add_blue_bg.gif["Symbol hinzufügen"].
. Im Fenster *Volume hinzufügen* den Volume-Namen und die Größe eingeben. Anschließend auf *Weitere Optionen* klicken.
. Unter *Speicher und Optimierung* die Option *Volumedaten im Cluster verteilen (FlexGroup)* auswählen.
. *Erweiterte Kapazitätsverteilung* auswählen.
. Die Konfiguration des Volumes abschließen und auf *Speichern* klicken.


--
.CLI
--
. Ein Volume mit aktiviertem erweitertem Kapazitätsausgleich erstellen:
+
[source, cli]
----
volume create -vserver <svm name> -volume <volume name> -size <volume size> -auto-provision-as flexgroup -junction-path /<path> -granular-data advanced
----
+
Beispiel:

+
[listing]
----
volume create -vserver vs0 -volume newvol -size 1TB -auto-provision-as flexgroup -junction-path /newvol -granular-data advanced
----


--
====


[[enable-advanced-capacity-balancing-on-existing-flexgroup-volumes]]
=== Erweiterte Kapazitätsverteilung auf bestehenden FlexGroup Volumes aktivieren

.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um den erweiterten Kapazitätsausgleich zu aktivieren.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Storage > Volumes*, klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"], und wählen Sie *Bearbeiten > Volume*.
. Im Fenster *Volume bearbeiten* unter *Speicher und Optimierung* die Option *Erweiterte Kapazitätsverteilung* auswählen.
. Auf *Speichern* klicken.


--
.CLI
--
. Ändern eines bestehenden FlexGroup Volumes zur Aktivierung des erweiterten Kapazitätsausgleichs:
+
[source, cli]
----
volume modify -vserver <svm name> -volume <volume name> -granular-data advanced
----
+
Beispiel:

+
[listing]
----
volume modify -vserver vs0 -volume newvol  -granular-data advanced
----


--
====
