= Bestandteile aus ONTAP FlexGroup Volumes verschieben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Bestandteile eines FlexGroup Volumes von einem Aggregat auf ein anderes verschieben, um die Last auszugleichen, wenn bestimmte Bestandteile mehr Datenverkehr aufweisen. Das Verschieben von Bestandteilen trägt auch dazu bei, Speicherplatz auf einem Aggregat freizugeben, um die vorhandenen Bestandteile zu vergrößern.

.Bevor Sie beginnen
Um ein FlexGroup Volumenelement zu verschieben, das sich in einer SnapMirror Beziehung befindet, muss die SnapMirror Beziehung initialisiert worden sein.

.Über diese Aufgabe
Eine Volumenverschiebung ist nicht möglich, solange die Bestandteile des FlexGroup Volumens erweitert werden.

.Schritte
. Die FlexGroup Volumenkomponente identifizieren, die verschoben werden soll:
+
`volume show -vserver svm_name -is-constituent true`

+
[listing]
----
cluster1::> volume show -vserver vs2 -is-constituent true
Vserver   Volume       Aggregate    State      Type       Size  Available Used%
--------- ------------ ------------ ---------- ---- ---------- ---------- -----
vs2       fg1          -            online     RW        400TB    15.12TB   62%
vs2       fg1__0001    aggr1        online     RW         25TB     8.12MB   59%
vs2       fg1__0002    aggr2        online     RW         25TB     2.50TB   90%
...
----
. Ein Aggregat identifizieren, zu dem die FlexGroup Volumenkomponente verschoben werden kann:
+
`volume move target-aggr show -vserver svm_name -volume vol_constituent_name`

+
Der verfügbare Platz im ausgewählten Aggregat muss größer sein als die Größe des FlexGroup Volume-Bestandteils, den Sie verschieben.

+
[listing]
----
cluster1::> volume move target-aggr show -vserver vs2 -volume fg1_0002
Aggregate Name   Available Size   Storage Type
--------------   --------------   ------------
aggr2            467.9TB          hdd
node12a_aggr3    100.34TB         hdd
node12a_aggr2    100.36TB         hdd
node12a_aggr1    100.36TB         hdd
node12a_aggr4    100.36TB         hdd
5 entries were displayed.
----
. Es sollte überprüft werden, ob die FlexGroup Volumenkomponente in das gewünschte Aggregat verschoben werden kann:
+
`volume move start -vserver svm_name -volume vol_constituent_name -destination-aggregate aggr_name -perform-validation-only true`

+
[listing]
----
cluster1::> volume move start -vserver vs2 -volume fg1_0002 -destination-aggregate node12a_aggr3 -perform-validation-only true
Validation succeeded.
----
. Die FlexGroup Volume-Komponente verschieben:
+
`volume move start -vserver svm_name -volume vol_constituent_name -destination-aggregate aggr_name [-allow-mixed-aggr-types {true|false}]`

+
Der Volume Move Vorgang läuft als Hintergrundprozess.

+
Ab ONTAP 9.5 ist es möglich, FlexGroup-Volume-Bestandteile von einem Fabric Pool in einen Nicht-Fabric Pool oder umgekehrt zu verschieben, indem der  `-allow-mixed-aggr-types` Parameter auf  `true` gesetzt wird. Standardmäßig ist die `-allow-mixed-aggr-types` Option auf `false` eingestellt.

+
[NOTE]
====
Der  `volume move`Befehl zum Aktivieren der Verschlüsselung auf FlexGroup Volumes kann nicht verwendet werden.

====
+
[listing]
----
cluster1::> volume move start -vserver vs2 -volume fg1_002 -destination-aggregate node12a_aggr3
----
+
[NOTE]
====
Wenn der Volume Move Vorgang aufgrund eines aktiven SnapMirror Vorgangs fehlschlägt, sollte der SnapMirror Vorgang mit dem  `snapmirror abort -h` Befehl abgebrochen werden. In einigen Fällen kann auch der SnapMirror Abbruchvorgang fehlschlagen. In solchen Situationen sollte der Volume Move Vorgang abgebrochen und später erneut versucht werden. Weitere Informationen zu  `snapmirror abort` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-abort.html["ONTAP-Befehlsreferenz"^].

====
. Den Status des Volume-Verschiebungsvorgangs überprüfen:
+
`volume move show -volume vol_constituent_name`

+
Das folgende Beispiel zeigt den Zustand eines FlexGroup Teilvolumens, das die Replikationsphase abgeschlossen hat und sich in der Umstellungsphase des Volume Move-Vorgangs befindet:

+
[listing]
----
cluster1::> volume move show -volume fg1_002
Vserver   Volume     State    Move Phase  Percent-Complete Time-To-Complete
--------- ---------- -------- ----------  ---------------- ----------------
vs2       fg1_002    healthy  cutover     -                -
----
