= Überlegungen zur Erstellung von SnapVault Backup-Beziehungen und einheitlichen Datenschutzbeziehungen für ONTAP FlexGroup Volumes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es sind die Überlegungen zur Erstellung einer SnapVault Backup Beziehung und einer einheitlichen Datensicherungsbeziehung für FlexGroup Volumes zu beachten.

* Sie können eine SnapVault Backup-Beziehung und eine einheitliche Datensicherungsbeziehung resynchronisieren, indem Sie die  `-preserve` Option nutzen, mit der Snapshots auf dem Ziel-Volume erhalten bleiben, die neuer sind als der letzte gemeinsame Snapshot.
* Langzeitaufbewahrung wird bei FlexGroup Volumes nicht unterstützt.
+
Die Langzeitspeicherung ermöglicht das Erstellen von Snapshots direkt auf dem Ziel-Volume, ohne dass die Snapshots auf dem Quell-Volume gespeichert werden müssen.

* Die  `snapshot`Befehls `expiry-time`option wird für FlexGroup Volumes nicht unterstützt.
* Die Speichereffizienz kann auf dem Ziel-FlexGroup-Volume einer SnapVault Backup-Beziehung und einer einheitlichen Data Protection-Beziehung nicht konfiguriert werden.
* Snapshots einer SnapVault Backup-Beziehung und einer einheitlichen Datensicherungsbeziehung für FlexGroup Volumes können nicht umbenannt werden.
* Ein FlexGroup Volume kann nur die Quellvolume einer einzigen Sicherungsbeziehung oder Wiederherstellungsbeziehung sein.
+
Ein FlexGroup Volume kann nicht die Quelle von zwei SnapVault Beziehungen, zwei Wiederherstellungsbeziehungen oder einer SnapVault Backup-Beziehung und einer Wiederherstellungsbeziehung sein.

* Wenn ein Snapshot auf dem Quell-FlexGroup Volume gelöscht und ein Snapshot mit demselben Namen neu erstellt wird, schlägt die nächste Aktualisierungsübertragung auf das Ziel-FlexGroup Volume fehl, wenn das Zielvolume einen Snapshot mit demselben Namen enthält.
+
Dies liegt daran, dass Snapshots für FlexGroup Volumes nicht umbenannt werden können.
