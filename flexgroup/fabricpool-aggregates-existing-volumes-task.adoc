= Aggregate in FabricPool für bestehende ONTAP FlexGroup Volumes verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.5 wird FabricPool für FlexGroup Volumes unterstützt. Wenn Aggregate in FabricPool für bestehende FlexGroup Volumes verwendet werden sollen, besteht die Möglichkeit, entweder die Aggregate, auf denen sich das FlexGroup Volume befindet, in Aggregate in FabricPool zu konvertieren oder die FlexGroup Volume-Bestandteile in Aggregate in FabricPool zu migrieren.

.Bevor Sie beginnen
* Das FlexGroup Volume muss die Speicherplatzgarantie auf  `none` gesetzt haben.
* Wenn Sie die Aggregate, auf denen sich das FlexGroup Volume befindet, in Aggregate in FabricPool konvertieren möchten, müssen die Aggregate ausschließlich SSD-Festplatten verwenden.


.Über diese Aufgabe
Wenn sich ein vorhandenes FlexGroup Volume auf Nicht-SSD-Aggregaten befindet, müssen die FlexGroup Volume-Bestandteile auf Aggregate in FabricPool migriert werden.

.Auswahlmöglichkeiten
* Um die Aggregate, auf denen sich das FlexGroup Volumen befindet, in Aggregate in FabricPool zu konvertieren, sind die folgenden Schritte erforderlich:
+
.. Die Tiering-Richtlinie für das bestehende FlexGroup Volume festlegen: `volume modify -volume flexgroup_name -tiering-policy [auto|snapshot|none|backup]`
+
[listing]
----
cluster-2::> volume modify -volume fg1 -tiering-policy auto
----
.. Die Aggregate, auf denen das FlexGroup Volume liegt, werden identifiziert: `volume show -volume flexgroup_name -fields aggr-list`
+
[listing]
----
cluster-2::> volume show -volume fg1 -fields aggr-list
vserver volume aggr-list
------- ------ -----------
vs1     fg1    aggr1,aggr3
----
.. Jedem in der Aggregatliste aufgeführten Aggregat wird ein Objektspeicher zugeordnet: `storage aggregate object-store attach -aggregate aggregate name -name object-store-name -allow-flexgroup true`
+
Alle Aggregate müssen an einen Objektspeicher angehängt werden.

+
[listing]
----
cluster-2::> storage aggregate object-store attach -aggregate aggr1 -object-store-name Amazon01B1
----


* Um die FlexGroup Volumenbestandteile auf Aggregate in FabricPool zu migrieren, sind die folgenden Schritte erforderlich:
+
.. Die Tiering-Richtlinie für das bestehende FlexGroup Volume festlegen: `volume modify -volume flexgroup_name -tiering-policy [auto|snapshot|none|backup]`
+
[listing]
----
cluster-2::> volume modify -volume fg1 -tiering-policy auto
----
.. Jeder Bestandteil des FlexGroup Volumens wird in ein Aggregat in FabricPool im selben Cluster verschoben: `volume move start -volume constituent-volume -destination-aggregate FabricPool_aggregate -allow-mixed-aggr-types true`
+
Alle FlexGroup-Volumenbestandteile müssen auf Aggregate in FabricPool verschoben werden (falls die FlexGroup-Volumenbestandteile auf gemischten Aggregattypen liegen), und es ist sicherzustellen, dass alle Bestandteile gleichmäßig auf die Knoten im Cluster verteilt sind.

+
[listing]
----
cluster-2::> volume move start -volume fg1_001 -destination-aggregate FP_aggr1 -allow-mixed-aggr-types true
----
