= ONTAP FlexGroup Volumes durch Verschieben von Dateien neu ausbalancieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.12.1 ist es möglich, FlexGroup Volumes neu auszubalancieren, indem Dateien unterbrechungsfrei von einem Bestandteil in einem FlexGroup Volume zu einem anderen Bestandteil verschoben werden.

FlexGroup Rebalancing unterstützt die Umverteilung der Kapazität, wenn sich im Laufe der Zeit durch das Hinzufügen neuer Dateien und das Wachstum von Dateien Ungleichgewichte entwickeln. Nach dem manuellen Start der Rebalancing-Operation wählt ONTAP die Dateien aus und verschiebt sie automatisch und unterbrechungsfrei.

[NOTE]
====
Es ist zu beachten, dass das Rebalancing von FlexGroup die Systemleistung beeinträchtigt, wenn im Rahmen eines einzelnen oder mehrerer Rebalancing-Vorgänge eine große Anzahl von Dateien verschoben wird, da dabei mehrteilige Inodes erstellt werden. Jede im Rahmen eines Rebalancing-Vorgangs verschobene Datei hat zwei mehrteilige Inodes, die dieser Datei zugeordnet sind. Je größer der Anteil der Dateien mit mehrteiligen Inodes an der Gesamtzahl der Dateien in einer FlexGroup ist, desto größer ist die Auswirkung auf die Leistung. Bestimmte Anwendungsfälle, wie eine FlexVol zu FlexGroup Konvertierung, können zu einer erheblichen Anzahl von mehrteiligen Inodes führen.

====
Die Neuausrichtung ist nur verfügbar, wenn auf allen Knoten im Cluster ONTAP 9.12.1 oder neuere Versionen ausgeführt werden. Die Granulardatenfunktionalität muss auf jedem FlexGroup Volume aktiviert werden, auf dem die Neuausrichtung durchgeführt wird. Sobald diese Funktion aktiviert ist, ist eine Rückkehr zu ONTAP 9.11.1 oder früheren Versionen nur möglich, wenn dieses Volume gelöscht oder ein Snapshot wiederhergestellt wird, der vor der Aktivierung dieser Einstellung erstellt wurde.

Ab ONTAP 9.14.1 führt ONTAP einen Algorithmus ein, der Dateien in Volumes mit aktivierten Granulardaten unterbrechungsfrei und proaktiv ohne Benutzereingriff verschiebt. Der Algorithmus kommt in sehr spezifischen, gezielten Szenarien zum Einsatz, um Leistungsengpässe zu beheben. Szenarien, in denen dieser Algorithmus zum Einsatz kommen kann, umfassen eine sehr hohe Schreiblast auf eine bestimmte Gruppe von Dateien auf einem Knoten im Cluster oder eine kontinuierlich wachsende Datei in einem sehr stark frequentierten übergeordneten Verzeichnis.

Ab ONTAP 9.16.1 kann auch link:enable-adv-capacity-flexgroup-task.html["fortschrittlicher Kapazitätsausgleich"] aktiviert werden, um die Daten einer großen Datei zwischen FlexGroup Mitgliedsvolumes neu zu verteilen.



[[flexgroup-rebalancing-considerations]]
== FlexGroup Überlegungen zur Neuausrichtung

Sie sollten sich darüber im Klaren sein, wie die FlexGroup Neuausrichtung funktioniert und wie sie mit anderen ONTAP Funktionen interagiert.

* FlexVol zu FlexGroup Konvertierung
+
Es wird empfohlen, die automatische FlexGroup Neuverteilung nach einer FlexVol-zu-FlexGroup Konvertierung _nicht_ zu verwenden. Stattdessen können vorhandene Dateien mit dem  `volume rebalance file-move start` Befehl neu verteilt werden, der ab ONTAP 9.10.1 verfügbar ist. Dieser Vorgang ist standardmäßig unterbrechungsfrei (`-is-disruptive false`. Falls einige belegte Dateien nicht verschoben werden können, kann der Befehl im unterbrechenden Modus (`-is-disruptive true` während eines geplanten Wartungsfensters erneut ausgeführt werden. Weitere Informationen zu  `volume rebalance file-move start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-rebalance-file-move-start.html["ONTAP-Befehlsreferenz"^].

+
Die automatische FlexGroup Neuausgleichsfunktion kann die Leistung beeinträchtigen, wenn große Mengen von Dateien verschoben werden, wie etwa bei einer FlexVol zu FlexGroup Konvertierung, und bis zu 50 bis 85 % der Daten auf dem FlexVol Volume in einen neuen Bestandteil verschoben werden.

* Minimale und maximale Dateigröße
+
Die Dateiauswahl für den automatischen Lastausgleich basiert auf den gespeicherten Blöcken. Die minimale Dateigröße, die für den Lastausgleich berücksichtigt wird, beträgt standardmäßig 100 MB (kann mit dem unten gezeigten Parameter min-file-size auf bis zu 20 MB konfiguriert werden), und die maximale Dateigröße beträgt 100 GB.

* Dateien in Momentaufnahmen
+
Sie können die FlexGroup Neuverteilung so konfigurieren, dass nur Dateien berücksichtigt werden, die aktuell in keinem Snapshot vorhanden sind. Beim Start der Neuverteilung wird eine Benachrichtigung angezeigt, falls während einer Neuverteilung eine Snapshot-Operation geplant ist.

+
Snapshots sind eingeschränkt, wenn eine Datei verschoben wird und am Zielort eine Rahmung erfolgt. Eine Snapshot-Wiederherstellung ist nicht zulässig, während eine Datei-Lastverteilung im Gange ist.

+
Snapshots, die nach der  `granular-data` Option erstellt werden, können nicht auf ein System repliziert werden, auf dem ONTAP 9.11.1 oder frühere Versionen ausgeführt werden, da ONTAP 9.11.1 und frühere Versionen keine mehrteiligen Inodes unterstützen.

* SnapMirror Operationen
+
FlexGroup Neuausgleich sollte zwischen geplanten SnapMirror Vorgängen stattfinden. Ein SnapMirror Vorgang kann fehlschlagen, wenn eine Datei vor Beginn eines SnapMirror Vorgangs verschoben wird und diese Verschiebung nicht innerhalb des 24-minütigen SnapMirror Wiederholungszeitraums abgeschlossen ist. Jede neue Dateiverschiebung, die nach Beginn einer SnapMirror Übertragung startet, schlägt nicht fehl.

* Speichereffizienz bei dateibasierter Komprimierung
+
Bei der speichereffizienten dateibasierten Komprimierung wird die Datei vor dem Verschieben an das Ziel dekomprimiert, sodass die Komprimierungsersparnis verloren geht. Die Komprimierungsersparnis wird wiederhergestellt, nachdem ein manuell gestarteter Hintergrundscanner auf dem FlexGroup Volume nach dem Rebalancing ausgeführt wurde. Wenn jedoch eine Datei mit einem Snapshot auf einem beliebigen Volume verknüpft ist, wird die Datei bei der Komprimierung ignoriert.

* Deduplizierung
+
Das Verschieben deduplizierter Dateien kann die Gesamtspeicherauslastung des FlexGroup Volumes erhöhen. Beim Dateiausgleich werden nur eindeutige Blöcke an das Ziel verschoben, wodurch dieser Speicherplatz auf der Quelle freigegeben wird. Gemeinsam genutzte Blöcke verbleiben auf der Quelle und werden an das Ziel kopiert. Während dies das Ziel erreicht, die belegte Kapazität eines nahezu vollen Quellbestandteils zu reduzieren, kann es aufgrund der Kopien gemeinsam genutzter Blöcke an den neuen Zielen auch zu einer erhöhten Gesamtspeicherauslastung auf dem FlexGroup Volume führen. Dies ist auch möglich, wenn Dateien verschoben werden, die Teil eines Snapshots sind. Die Speicherplatzersparnis wird erst vollständig erkannt, wenn der Snapshot-Zeitplan erneut durchläuft und keine Kopien der Dateien mehr in Snapshots vorhanden sind.

* FlexClone Volumes
+
Wenn ein Dateiausgleich im Gange ist, wenn ein FlexClone Volume erstellt wird, wird der Ausgleich nicht auf dem FlexClone Volume durchgeführt. Ein Ausgleich auf dem FlexClone Volume sollte nach dessen Erstellung durchgeführt werden.

* Datei verschieben
+
Wird eine Datei während eines FlexGroup Lastausgleichs verschoben, wird ihre Größe im Rahmen der Quotenberechnung sowohl auf dem Quell- als auch auf dem Zielbestandteil erfasst. Nach Abschluss der Verschiebung erfolgt die Quotenberechnung wieder wie gewohnt, und die Dateigröße wird nur noch auf dem neuen Zielbestandteil erfasst.

* Autonomous Ransomware Protection
+
Ab ONTAP 9.13.1 wird der autonome Ransomware-Schutz sowohl bei störenden als auch bei nicht-störenden Rebalance-Vorgängen unterstützt.

* Objektspeicher-Volumes
+
Die Neuausrichtung der Volumenkapazität wird für Objektspeicher-Volumes wie S3 Buckets nicht unterstützt.





[[enable-flexgroup-rebalancing]]
== Aktivieren der FlexGroup Neuausbalancierung

Ab ONTAP 9.12.1 ist es möglich, die automatische unterbrechungsfreie FlexGroup Volumen-Neuverteilung zu aktivieren, um Dateien zwischen FlexGroup Komponenten neu zu verteilen.

Ab ONTAP 9.13.1 kann eine einzelne FlexGroup Rebalancing-Operation so geplant werden, dass sie zu einem zukünftigen Datum und einer zukünftigen Uhrzeit beginnt.

.Bevor Sie beginnen
Die  `granular-data` Option muss auf dem FlexGroup Volume aktiviert sein, bevor die FlexGroup Neugewichtung aktiviert werden kann. Dies kann mit einer der folgenden Methoden erfolgen:

* Wenn ein FlexGroup Volume mit dem  `volume create` Befehl erstellt wird
* Durch die Änderung eines vorhandenen FlexGroup Volumes, um die Einstellung mit dem  `volume modify` Befehl zu aktivieren
* Automatische Einstellung, wenn das FlexGroup Rebalancing mit dem  `volume rebalance` Befehl initiiert wird
+

NOTE: Wenn Sie ONTAP 9.16.1 oder höher verwenden und link:enable-adv-capacity-flexgroup-task.html["FlexGroup fortschrittlicher Kapazitätsausgleich"] mithilfe der  `granular-data advanced` Option in der ONTAP CLI oder über den System Manager aktiviert ist, ist das FlexGroup Rebalancing ebenfalls aktiviert.



.Schritte
Die Neuausrichtung von FlexGroup kann mit ONTAP System Manager oder der ONTAP CLI verwaltet werden.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Speicher > Volumes* und suchen Sie das FlexGroup Volume, das neu verteilt werden soll.
. Wählen Sie image:icon_dropdown_arrow.gif["Dropdown-Symbol"], um die Volumendetails anzuzeigen.
. Unter *FlexGroup Saldo-Status* die Option *Neu ausbalancieren* auswählen.
+

NOTE: Die Option *Rebalance* ist nur verfügbar, wenn der FlexGroup Status unausgeglichen ist.

. Im Fenster *Volume neu ausbalancieren* lassen sich die Standardeinstellungen nach Bedarf ändern.
. Um den Rebalancing-Vorgang zu planen, *Rebalance Later* auswählen und Datum sowie Uhrzeit eingeben.


--
.CLI
--
. Automatisches Rebalancing wird gestartet:
+
[source, cli]
----
volume rebalance start -vserver <SVM name> -volume <volume name>
----
+
Optional lassen sich die folgenden Optionen angeben:

+
[[-max-runtime] <time interval>] Maximale Laufzeit

+
[-max-threshold <percent>] Maximaler Ungleichgewichtsschwellenwert pro Bestandteil

+
[-min-threshold <percent>] Minimaler Ungleichgewichtsschwellenwert pro Bestandteil

+
[-max-file-moves <integer>] Maximale Anzahl gleichzeitiger Dateiverschiebungen pro Bestandteil

+
[-min-file-size {<integer>[KB|MB|GB|TB|PB]}] Minimale Dateigröße

+
[-start-time <mm/dd/yyyy-00:00:00>] Startdatum und Uhrzeit für die Neuausrichtung des Zeitplans

+
[-exclude-snapshots {true|false}] Dateien ausschließen, die in Snapshots feststecken

+
Beispiel:

+
[listing]
----
volume rebalance start -vserver vs0 -volume fg1
----


--
====


[[modify-flexgroup-rebalance-configurations]]
== Konfigurationen für das FlexGroup Rebalancing ändern

Sie können eine FlexGroup Rebalancing-Konfiguration ändern, um den Schwellenwert für das Ungleichgewicht, die Anzahl gleichzeitiger Dateiverschiebungen, die minimale Dateigröße, die maximale Laufzeit zu aktualisieren sowie Snapshots einzuschließen oder auszuschließen. Optionen zur Änderung Ihrer FlexGroup Rebalancing-Planung sind ab ONTAP 9.13.1 verfügbar.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Speicher > Volumes* und suchen Sie das FlexGroup Volume, das neu verteilt werden soll.
. Wählen Sie image:icon_dropdown_arrow.gif["Dropdown-Symbol"], um die Volumendetails anzuzeigen.
. Unter *FlexGroup Saldo-Status* die Option *Neu ausbalancieren* auswählen.
+

NOTE: Die Option *Rebalance* ist nur verfügbar, wenn der FlexGroup Status unausgeglichen ist.

. Im Fenster *Volume neu ausbalancieren* lassen sich die Standardeinstellungen nach Bedarf ändern.


--
.CLI
--
. Automatisches Rebalancing ändern:
+
[source, cli]
----
volume rebalance modify -vserver <SVM name> -volume <volume name>
----
+
Sie können eine oder mehrere der folgenden Optionen angeben:

+
[[-max-runtime] <time interval>] Maximale Laufzeit

+
[-max-threshold <percent>] Maximaler Ungleichgewichtsschwellenwert pro Bestandteil

+
[-min-threshold <percent>] Minimaler Ungleichgewichtsschwellenwert pro Bestandteil

+
[-max-file-moves <integer>] Maximale Anzahl gleichzeitiger Dateiverschiebungen pro Bestandteil

+
[-min-file-size {<integer>[KB|MB|GB|TB|PB]}] Minimale Dateigröße

+
[-start-time <mm/dd/yyyy-00:00:00>] Startdatum und Uhrzeit für die Neuausrichtung des Zeitplans

+
[-exclude-snapshots {true|false}] Dateien ausschließen, die in Snapshots feststecken



--
====


[[stop-flexgroup-rebalance]]
== Neuausrichtung von FlexGroup beenden

Nachdem die FlexGroup Neuausrichtung aktiviert oder geplant wurde, kann sie jederzeit gestoppt werden.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Speicher > Volumes* und suchen Sie das FlexGroup Volume.
. Wählen Sie image:icon_dropdown_arrow.gif["Dropdown-Symbol"], um die Volumendetails anzuzeigen.
. *Rebalancing stoppen* auswählen.


--
.CLI
--
. Das FlexGroup Rebalancing beenden:
+
[source, cli]
----
volume rebalance stop -vserver <SVM name> -volume <volume name>
----


--
====

.System Manager
--
. Navigieren Sie zu *Speicher > Volumes* und suchen Sie das FlexGroup Volume.
. Wählen Sie image:icon_dropdown_arrow.gif["Dropdown-Symbol"] aus, um die FlexGroup Details anzuzeigen.
. Der *FlexGroup Balance Status* wird im unteren Bereich des Detailfensters angezeigt.
