= ONTAP FlexGroup Volumes mit Snapshots schützen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Snapshot-Richtlinien erstellen, die die Erstellung von Snapshots automatisch verwalten, oder Sie können Snapshots für FlexGroup Volumes manuell erstellen. Ein gültiger Snapshot wird für ein FlexGroup Volume nur erstellt, nachdem ONTAP für jede Komponente des FlexGroup Volumes erfolgreich einen Snapshot erstellen konnte.

.Über diese Aufgabe
* Wenn Sie mehrere FlexGroup Volumes mit einer Snapshot-Richtlinie verknüpft haben, sollte sichergestellt werden, dass sich die FlexGroup Volumes Zeitpläne nicht überschneiden.
* Ab ONTAP 9.8 beträgt die maximale Anzahl der auf einem FlexGroup Volume unterstützten Snapshots 1023.



NOTE: Ab ONTAP 9.8 gibt der  `volume snapshot show` Befehl für FlexGroup Volumes die Snapshot-Größe anhand logischer Blöcke an, anstatt die jüngsten zugehörigen Blöcke zu berechnen. Diese neue Methode zur Größenberechnung kann dazu führen, dass die Snapshot-Größe größer erscheint als bei Berechnungen in früheren Versionen von ONTAP.

.Schritte
. Eine Snapshot-Richtlinie erstellen oder einen Snapshot manuell erstellen:
+
|===


| Wenn Sie ein... erstellen möchten | Geben Sie diesen Befehl ein... 


 a| 
Snapshot-Richtlinie
 a| 
`volume snapshot policy create`


NOTE: Die mit der Snapshot-Richtlinie eines FlexGroup Volumes verknüpften Zeitpläne müssen ein Intervall von mehr als 30 Minuten aufweisen.

Wenn ein FlexGroup Volume erstellt wird, wird die  `default` Snapshot-Richtlinie auf das FlexGroup Volume angewendet.



 a| 
Snapshot manuell
 a| 
`volume snapshot create`


NOTE: Nachdem Sie einen Snapshot für ein FlexGroup volume erstellt haben, können Sie die Attribute des Snapshots nicht mehr ändern. Um die Attribute zu ändern, ist es erforderlich, den Snapshot zu löschen und anschließend neu zu erstellen.

|===


Der Clientzugriff auf das FlexGroup Volume wird kurzzeitig unterbrochen, wenn ein Snapshot erstellt wird.

. Es wird überprüft, ob ein gültiger Snapshot für das FlexGroup Volume erstellt wurde: `volume snapshot show -volume volume_name -fields state`
+
[listing]
----
cluster1::> volume snapshot show -volume fg -fields state
vserver volume snapshot               state
------- ------ ---------------------- -----
fg_vs   fg     hourly.2016-08-23_0505 valid
----
. Die Snapshots der Bestandteile des FlexGroup Volumens anzeigen: `volume snapshot show -is-constituent true`
+
[listing]
----
cluster1::> volume snapshot show -is-constituent true
                                                                 ---Blocks---
Vserver  Volume   Snapshot                                  Size Total% Used%
-------- -------- ------------------------------------- -------- ------ -----
fg_vs    fg__0001
                  hourly.2016-08-23_0505                    72MB     0%   27%
         fg__0002
                  hourly.2016-08-23_0505                    72MB     0%   27%
         fg__0003
                  hourly.2016-08-23_0505                    72MB     0%   27%
...
         fg__0016
                  hourly.2016-08-23_0505                    72MB     0%   27%
----
