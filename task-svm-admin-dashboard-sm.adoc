= Verwalten und überwachen Sie ONTAP SVMs mit System Manager
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Ab ONTAP 9.19.1 können SVM-Administratoren ihre eigenen SVMs mithilfe eines dedizierten System Manager-Dashboards überwachen und verwalten.

.Bevor Sie beginnen
Ihr Cluster-Administrator muss Ihnen Folgendes zur Verfügung stellen:

* Management-IP-Adresse für die SVM
* Anmeldeinformationen (Benutzername und Passwort) für das SVM System Manager Dashboard (der vsadmin-Benutzer muss entsperrt sein)
* Optional Berechtigungen zum Hinzufügen von Volumes auf der SVM


.Schritte
. Geben Sie die Management-IP-Adresse der SVM in einem Browser ein.
. Geben Sie Ihren Benutzernamen und Ihr Passwort für den SVM-Administrator (vsadmin) ein.
. Wählen Sie *Sign in* aus.
+
Das ONTAP System Manager Dashboard wird geöffnet.

. Sie können folgende Informationen über die SVM einsehen:
+
[cols="1,2"]
|===
| SVM-Informationen | Beschreibung 


| Protokolle | Die auf der SVM aktivierten Protokolle. 


| Details | NIS- und LDAP-Informationen, Sprache und Aufbewahrungsfrist nach dem Löschen eines Volumes. 


| Performance | Latenz, IOPS und Durchsatzleistungsstatistiken pro Stunde, Tag, Woche, Monat oder Jahr für das ausgewählte Protokoll. 
|===
. Optional können Sie die Daten zur weiteren Analyse in einer durch Kommas getrennten Wertdatei (csv) herunterladen.
. Wählen Sie *Storage > Volumes*.
+
Die Volumes-Dashboard-Seite wird geöffnet.

+
Der Clusteradministrator kann Ihnen die Berechtigung zum Hinzufügen von Volumes zur SVM erteilen. Wird diese Berechtigung nicht erteilt, schlägt die Auswahl der Schaltfläche *Hinzufügen* fehl und es wird eine Fehlermeldung angezeigt.
