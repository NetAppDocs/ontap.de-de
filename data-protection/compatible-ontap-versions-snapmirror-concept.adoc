= Kompatible ONTAP Versionen für SnapMirror Beziehungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Quell- und Zielvolumes müssen kompatible ONTAP Versionen ausführen, bevor eine SnapMirror Daten­schutz­beziehung erstellt wird. Vor einem ONTAP Upgrade sollte überprüft werden, ob die aktuelle ONTAP Version mit der Ziel-ONTAP Version für SnapMirror Beziehungen kompatibel ist.



[[unified-replication-relationships]]
== Vereinheitlichte Replikationsbeziehungen

Für SnapMirror Beziehungen vom Typ "`XDP`", bei Verwendung von lokalen oder Cloud Volumes ONTAP Versionen:

Ab ONTAP 9.9.0:

* ONTAP 9.x.0 Releases sind ausschließlich für die Cloud verfügbar und unterstützen Cloud Volumes ONTAP Systeme. Das Sternchen (*) nach der Versionsnummer kennzeichnet eine reine Cloud-Version.
+

NOTE: ONTAP 9.16.0 bildet eine Ausnahme von der Cloud-only-Regel, da es Unterstützung für link:https://docs.netapp.com/us-en/asa-r2/learn-more/software-support-limitations.html["ASA r2-Systeme"^] bietet. Das Pluszeichen (+) nach der Release-Version zeigt sowohl eine ASA r2- als auch eine Cloud-unterstützte Version an. ASA r2-Systeme unterstützen SnapMirror-Beziehungen nur zu anderen ASA r2-Systemen.

* ONTAP 9.x.1 Releases sind allgemeine Releases und unterstützen sowohl On-Premises als auch Cloud Volumes ONTAP Systeme.



NOTE: Wenn link:../flexgroup/enable-adv-capacity-flexgroup-task.html["fortschrittlicher Kapazitätsausgleich"] auf Volumes in Clustern aktiviert ist, auf denen ONTAP 9.16.1 oder höher ausgeführt wird, werden SnapMirror-Übertragungen zu Clustern, auf denen ONTAP Versionen vor ONTAP 9.16.1 ausgeführt werden, nicht unterstützt.


NOTE: Interoperabilität ist bidirektional.

[[snapmirror-svm-disaster-recovery-relationships]]
== SnapMirror SVM-Beziehungen zur Katastrophenwiederherstellung

[NOTE]
====
* Diese Matrix gilt für die SVM Datenmobilität Migrationsfunktion ab ONTAP 9.10.1.
* Sie können SVM DR verwenden, um eine SVM zu migrieren, die die für link:../svm-migrate/index.html#prerequisites["SVM Migration (SVM Datenmobilität)"] angegebenen Einschränkungen nicht erfüllt.
* In beiden Fällen dürfen maximal 2 *neuere* Hauptversionen von ONTAP zwischen Quell- und Ziel-Cluster liegen, wobei die Zielversion die gleiche oder eine neuere ONTAP Version als die Quellversion sein muss.


====
.Für SVM-Notfallwiederherstellungsdaten und SVM-Schutz:
Die SVM-Notfallwiederherstellung wird nur zwischen Clustern unterstützt, auf denen dieselbe Version von ONTAP ausgeführt wird. *Versionsunabhängigkeit wird für die SVM-Replikation nicht unterstützt*.

.Für die SVM-Notfallwiederherstellung bei der SVM-Migration:
* Die Replikation wird in eine Richtung unterstützt, von einer früheren Version von ONTAP auf dem Quell-Cluster zur gleichen oder einer späteren Version von ONTAP auf dem Ziel-Cluster.
* Die ONTAP Version auf dem Ziel-Cluster darf höchstens zwei Hauptversionen der lokalen Version oder zwei Hauptversionen der Cloud-Version neuer sein (beginnend mit ONTAP 9.9.0), wie in der folgenden Tabelle gezeigt.
+
** Die Replikation wird für Anwendungsfälle des langfristigen Datenschutzes nicht unterstützt.




Das Sternchen (*) nach der Versionsnummer kennzeichnet eine reine Cloud-Version.

Zur Ermittlung der Unterstützung befindet sich die Quellversion in der linken Tabellenspalte, die Zielversion in der obersten Zeile (DR/Migration für gleichartige Versionen und nur Migration für neuere Versionen).


NOTE: Wenn Sie ONTAP 9.10.1 oder höher verwenden, kann die link:../svm-migrate/index.html["SVM Datenmobilität"] Funktion anstelle von SVM DR verwendet werden, um SVMs von einem Cluster auf einen anderen zu migrieren.
