= Erstellung eines ONTAP SnapMirror Replikationsauftragszeitplans
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Jobplan legt fest, wann SnapMirror die Datensicherungsbeziehung, der der Plan zugewiesen ist, automatisch aktualisiert. Es besteht die Möglichkeit, den System Manager oder den  `job schedule cron create` Befehl zu verwenden, um einen Replikationsjobplan zu erstellen. Weitere Informationen zu  `job schedule cron create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/job-schedule-cron-create.html["ONTAP-Befehlsreferenz"^].

.Über diese Aufgabe
Sie weisen einen Jobplan zu, wenn Sie eine Datenschutzbeziehung erstellen. Wenn Sie keinen Jobplan zuweisen, muss die Beziehung manuell aktualisiert werden.

.Schritte
Ein Replikationsauftragsplan kann mit System Manager oder der ONTAP CLI erstellt werden.

[role="tabbed-block"]
====
.System Manager
--
. Zu *Schutz > Übersicht* navigieren und *Lokale Richtlinieneinstellungen* erweitern.
. Im Bereich *Zeitpläne* auf image:icon_arrow.gif["Pfeilsymbol"] klicken.
. Im Fenster *Zeitpläne* auf image:icon_add.gif["Symbol hinzufügen"] klicken.
. Im Fenster *Zeitplan hinzufügen* den Namen des Zeitplans eingeben sowie den Kontext und den Zeitplantyp auswählen.
. Auf *Speichern* klicken.


--
.CLI
--
. Einen Arbeitszeitplan erstellen:
+
[source, cli]
----
job schedule cron create -name <job_name> -month <month> -dayofweek <day_of_week> -day <day_of_month> -hour <hour> -minute <minute>
----
+
Für  `-month`,  `-dayofweek` und  `-hour` kann  `all` angegeben werden, damit der Job jeden Monat, an jedem Wochentag bzw. zu jeder Stunde ausgeführt wird.

+
Ab ONTAP 9.10.1 kann der Vserver in den Jobplan einbezogen werden:

+
[listing, subs="+quotes"]
----
job schedule cron create -name <job_name> *-vserver <Vserver_name>* -month <month> -dayofweek <day_of_week> -day <day_of_month> -hour <hour> -minute <minute>
----
+

NOTE: Der minimal unterstützte Zeitplan (RPO) für FlexVol Volumes in einer Volume SnapMirror Beziehung beträgt 5 Minuten. Der minimal unterstützte Zeitplan (RPO) für FlexGroup Volumes in einer Volume SnapMirror Beziehung beträgt 30 Minuten.

+
Das folgende Beispiel erstellt einen Jobplan mit dem Namen  `my_weekly`, der samstags um 3:00 Uhr morgens ausgeführt wird:

+
[listing]
----
cluster_dst::> job schedule cron create -name my_weekly -dayofweek "Saturday" -hour 3 -minute 0
----
