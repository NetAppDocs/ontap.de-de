= Eine Regel für eine ONTAP SnapMirror Richtlinie definieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Für benutzerdefinierte Richtlinien mit dem `vault` oder `mirror-vault` Richtlinientyp muss mindestens eine Regel definiert werden, die festlegt, welche Snapshots während der Initialisierung und Aktualisierung übertragen werden. Es können auch Regeln für Standardrichtlinien mit dem `vault` oder `mirror-vault` Richtlinientyp definiert werden.

.Über diese Aufgabe
Jede Richtlinie mit dem `vault` oder `mirror-vault`-Richtlinientyp muss eine Regel enthalten, die angibt, welche Snapshots repliziert werden. Die Regel `bi-monthly` gibt beispielsweise an, dass nur Snapshots mit dem SnapMirror-Label `bi-monthly` repliziert werden. Das SnapMirror-Label wird bei der Konfiguration der Snapshot-Richtlinie auf der Quelle angegeben.

Jedem Richtlinientyp sind eine oder mehrere systemdefinierte Regeln zugeordnet. Diese Regeln werden einer Richtlinie automatisch zugewiesen, wenn deren Richtlinientyp angegeben wird. Die folgende Tabelle zeigt die systemdefinierten Regeln.

[cols="3*"]
|===


| Systemdefinierte Regel | Wird in Richtlinientypen verwendet | Ergebnis 


 a| 
sm_created
 a| 
async-mirror, mirror-vault, Sync, StrictSync
 a| 
Ein von SnapMirror erstellter Snapshot wird bei der Initialisierung und Aktualisierung übertragen.



 a| 
all_source_snapshots
 a| 
async-mirror
 a| 
Neue Snapshots auf der Quelle werden bei Initialisierung und Aktualisierung übertragen.



 a| 
täglich
 a| 
Vault, Mirror-Vault
 a| 
Neue Snapshots auf dem Quell-Cluster mit dem SnapMirror-Label `daily` werden bei Initialisierung und Aktualisierung übertragen.



 a| 
wöchentlich
 a| 
Vault, Mirror-Vault
 a| 
Neue Snapshots auf dem Quell-Cluster mit dem SnapMirror-Label `weekly` werden bei Initialisierung und Aktualisierung übertragen.



 a| 
monatlich
 a| 
mirror-vault
 a| 
Neue Snapshots auf dem Quell-Cluster mit dem SnapMirror-Label `monthly` werden bei Initialisierung und Aktualisierung übertragen.



 a| 
app_consistent
 a| 
Sync, StrictSync
 a| 
Snapshots mit dem SnapMirror-Label `app_consistent` auf der Quelle werden synchron zum Ziel repliziert. Unterstützt ab ONTAP 9.7.

|===
Mit Ausnahme des Richtlinientyps „`async-mirror`“ lassen sich bei Bedarf zusätzliche Regeln für Standard- oder benutzerdefinierte Richtlinien angeben. Beispiel:

* Für die  `MirrorAndVault`Standardrichtlinie könnte eine Regel mit dem Namen  `bi-monthly` erstellt werden, um Snapshots auf dem Quell-System mit dem  `bi-monthly` SnapMirror Label abzugleichen.
* Für eine benutzerdefinierte Richtlinie mit dem  `mirror-vault` Richtlinientyp könnte eine Regel mit dem Namen  `bi-weekly` erstellt werden, um Snapshots auf dem Quell-Cluster mit dem  `bi-weekly` SnapMirror Label abzugleichen.


.Schritt
. Eine Regel für eine Richtlinie definieren:
+
`snapmirror policy add-rule -vserver _SVM_ -policy _policy_for_rule_ -snapmirror-label _snapmirror-label_ -keep _retention_count_`

+
Das folgende Beispiel fügt eine Regel mit der Bezeichnung SnapMirror `bi-monthly` zur Standard `MirrorAndVault`richtlinie hinzu:

+
[listing]
----
cluster_dst::> snapmirror policy add-rule -vserver svm1 -policy MirrorAndVault -snapmirror-label bi-monthly -keep 6
----
+
Das folgende Beispiel fügt eine Regel mit der Bezeichnung SnapMirror `bi-weekly` zur benutzerdefinierten `my_snapvault` Richtlinie hinzu:

+
[listing]
----
cluster_dst::> snapmirror policy add-rule -vserver svm1 -policy my_snapvault -snapmirror-label bi-weekly -keep 26
----
+
Das folgende Beispiel fügt eine Regel mit der Bezeichnung SnapMirror `app_consistent` zur benutzerdefinierten `Sync` Richtlinie hinzu:

+
[listing]
----
cluster_dst::> snapmirror policy add-rule -vserver svm1 -policy Sync -snapmirror-label app_consistent -keep 1
----
+
Weitere Informationen zu  `snapmirror policy add-rule` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-policy-add-rule.html["ONTAP-Befehlsreferenz"^].

+
Anschließend können Snapshots aus dem Quell-Cluster repliziert werden, die diesem SnapMirror-Label entsprechen:

+
[listing]
----
cluster_src::> snapshot create -vserver vs1 -volume vol1 -snapshot snapshot1 -snapmirror-label app_consistent
----
