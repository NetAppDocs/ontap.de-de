= ONTAP SnapMirror Replikationsworkflow
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapMirror bietet drei Arten von Datenschutzbeziehungen: SnapMirror DR, Archiv (früher bekannt als SnapVault), und einheitliche Replizierung. Für jede Art der Beziehung kann derselbe grundlegende Arbeitsablauf verwendet werden.

Ab der allgemeinen Verfügbarkeit in ONTAP 9.9.1 link:../snapmirror-active-sync/index.html["SnapMirror Active Sync"] bietet Zero Recovery Time Objective (Zero RTO) oder Transparent Application Failover (TAF) die Möglichkeit, ein automatisches Failover geschäftskritischer Anwendungen in SAN-Umgebungen zu ermöglichen.

Für jede Art von SnapMirror Datensicherungsbeziehung ist der Arbeitsablauf derselbe: Zielvolume erstellen, Jobzeitplan erstellen, Richtlinie festlegen, Beziehung erstellen und initialisieren.

Ab ONTAP 9.3 kann mit dem  `snapmirror protect` Befehl eine Datensicherungsbeziehung in einem einzigen Schritt konfiguriert werden. Auch wenn  `snapmirror protect` verwendet wird, ist das Verständnis jedes einzelnen Schritts im Arbeitsablauf erforderlich.
