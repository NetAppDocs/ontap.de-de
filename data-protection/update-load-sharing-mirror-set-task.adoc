= Aktualisierung einer ONTAP Load-Sharing Mirror Beziehung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Load-sharing Mirror (LSM) Beziehungen werden für SVM Root Volumes automatisch aktualisiert, nachdem ein Volume in der SVM eingebunden oder ausgehängt wurde, und während  `volume create` Vorgängen, die die  `junction-path` Option enthalten. Eine LSM Beziehung kann manuell aktualisiert werden, wenn eine Aktualisierung vor dem nächsten geplanten Update gewünscht ist.

Lastverteilungs-Spiegelbeziehungen werden in den folgenden Fällen automatisch aktualisiert:

* Es ist Zeit für ein planmäßiges Update
* Ein Mount- oder Unmount-Vorgang wird auf einem Volume im SVM Root-Volume durchgeführt.
* Ein `volume create` Befehl wird ausgeführt, der die `junction-path` Option enthält
+
Weitere Informationen zu  `volume create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-create.html["ONTAP-Befehlsreferenz"^].



.Schritt
. Eine Load-Sharing Mirror Beziehung manuell aktualisieren:
+
Vor dem Ausführen dieses Befehls sind die Variablen in spitzen Klammern durch die erforderlichen Werte zu ersetzen.

+
[source, cli]
----
snapmirror update-ls-set -source-path <SVM:volume>
----
+
Das folgende Beispiel aktualisiert die Load-Sharing Mirror Beziehung für das Root-Volume `svm1_root`:

+
[listing]
----
cluster_src::> snapmirror update-ls-set -source-path svm1:svm1_root
----
+
Weitere Informationen zu  `snapmirror update` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-update.html["ONTAP-Befehlsreferenz"^].
