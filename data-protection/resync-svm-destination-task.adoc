= Die Daten auf einer ONTAP SnapMirror Ziel-SVM werden erneut synchronisiert
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
ONTAP 9.11.1 bietet die Möglichkeit, bei einer Notfallwiederherstellungsübung den kompletten Neuaufbau des Data Warehouse zu umgehen, sodass eine schnellere Rückkehr zur Produktion möglich ist.

Ab ONTAP 9.8 ist es möglich, mit System Manager die Daten und Konfigurationsdetails von der Quell-Speicher-VM zur Ziel-Speicher-VM in einer unterbrochenen Schutzbeziehung zu resynchronisieren und die Beziehung wiederherzustellen.

Die Resynchronisierung wird nur vom Ziel der ursprünglichen Beziehung aus durchgeführt. Dabei werden alle Daten in der Ziel-Speicher-VM gelöscht, die neuer sind als die Daten in der Quell-Speicher-VM.

.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um diese Aufgabe auszuführen.

[role="tabbed-block"]
====
.System Manager
--
. Im Ziel-Cluster die gewünschte Schutzbeziehung auswählen: *Schutz > Beziehungen* anklicken.
. Optional kann *Schnelle Resynchronisierung durchführen* ausgewählt werden, um einen vollständigen Neuaufbau des Data Warehouse während einer Notfallwiederherstellungsübung zu umgehen.
. Klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"] und dann auf *Neu synchronisieren*.
. Unter *Beziehungen* lässt sich der Fortschritt der Resynchronisierung überwachen, indem der *Übertragungsstatus* für die Beziehung angezeigt wird.


--
.CLI
--
. Vom Ziel-Cluster aus die Beziehung neu synchronisieren:
+
[source, cli]
----
snapmirror resync -source-path <svm>: -destination-path <svm>: -quick-resync true|false
----
