= Erstellen einer benutzerdefinierten ONTAP SnapMirror Replikationsrichtlinie
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine benutzerdefinierte Replikationsrichtlinie erstellen, wenn die Standardrichtlinie für eine Beziehung nicht geeignet ist. Beispielsweise kann es sinnvoll sein, Daten bei einer Netzwerkübertragung zu komprimieren oder die Anzahl der Versuche zu ändern, die SnapMirror zum Übertragen von Snapshots unternimmt.

Es kann beim Erstellen einer Replikationsbeziehung eine Standard- oder eine benutzerdefinierte Richtlinie verwendet werden. Für eine benutzerdefinierte Archivierungsrichtlinie (früher SnapVault) oder eine einheitliche Replikationsrichtlinie ist es erforderlich, eine oder mehrere _Regeln_ zu definieren, die festlegen, welche Snapshots während der Initialisierung und Aktualisierung übertragen werden. Es kann außerdem ein Zeitplan für die Erstellung lokaler Snapshots auf dem Ziel-Cluster definiert werden.

Der _Richtlinientyp_ der Replikationsrichtlinie bestimmt die Art der Beziehung, die sie unterstützt. Die folgende Tabelle zeigt die verfügbaren Richtlinientypen.

[cols="2*"]
|===


| Richtlinientyp | Beziehungstyp 


 a| 
async-mirror
 a| 
SnapMirror DR



 a| 
Vault
 a| 
SnapVault



 a| 
mirror-vault
 a| 
Einheitliche Replikation



 a| 
strict-sync-mirror
 a| 
SnapMirror synchron im StrictSync Modus (unterstützt ab ONTAP 9.5)



 a| 
Sync Mirror
 a| 
SnapMirror synchron im Sync Modus (unterstützt ab ONTAP 9.5)

|===
[TIP]
====
Wenn Sie eine benutzerdefinierte Replikationsrichtlinie erstellen, ist es ratsam, die Richtlinie an einer Standardrichtlinie zu orientieren.

====
.Schritte
Sie können benutzerdefinierte Datenschutzrichtlinien mit System Manager oder der ONTAP CLI erstellen. Ab ONTAP 9.11.1 ist es möglich, mit System Manager benutzerdefinierte Mirror- und Vault-Richtlinien zu erstellen sowie ältere Richtlinien anzuzeigen und auszuwählen. Diese Funktion ist auch in ONTAP 9.8P12 und späteren Patches von ONTAP 9.8 verfügbar.

Benutzerdefinierte Schutzrichtlinien sowohl auf dem Quell-Cluster als auch auf dem Ziel-Cluster erstellen.

[role="tabbed-block"]
====
.System Manager
--
. Klicken Sie auf *Schutz > Übersicht > Lokale Richtlinieneinstellungen*.
. Unter *Schutzrichtlinien* auf image:icon_arrow.gif["Pfeilsymbol"] klicken.
. Im Bereich *Schutzrichtlinien* auf image:icon_add.gif["Symbol hinzufügen"] klicken.
. Geben Sie den Namen der neuen Richtlinie ein und wählen Sie den Geltungsbereich der Richtlinie aus.
. Wählen Sie einen Richtlinientyp aus. Zum Hinzufügen einer reinen Vault- oder Mirror-Richtlinie *Asynchron* auswählen und auf *Legacy-Richtlinientyp verwenden* klicken.
. Die Pflichtfelder ausfüllen.
. Auf *Speichern* klicken.
. Diese Schritte sind auf dem anderen Cluster zu wiederholen.


--
.CLI
--
. Erstellen einer benutzerdefinierten Replikationsrichtlinie:
+
[source, cli]
----
snapmirror policy create -vserver <SVM> -policy _policy_ -type <async-mirror|vault|mirror-vault|strict-sync-mirror|sync-mirror> -comment <comment> -tries <transfer_tries> -transfer-priority <low|normal> -is-network-compression-enabled <true|false>
----
+
Ab ONTAP 9.5 kann der Zeitplan für die Erstellung eines gemeinsamen Snapshot-Zeitplans für SnapMirror synchron Beziehungen mithilfe des  `-common-snapshot-schedule` Parameters festgelegt werden. Standardmäßig beträgt der gemeinsame Snapshot-Zeitplan für SnapMirror synchron Beziehungen eine Stunde. Für den Snapshot-Zeitplan für SnapMirror synchron Beziehungen kann ein Wert zwischen 30 Minuten und zwei Stunden angegeben werden.

+
Das folgende Beispiel erstellt eine benutzerdefinierte Replikationsrichtlinie für SnapMirror DR, die die Netzwerkkomprimierung für Datenübertragungen aktiviert:

+
[listing]
----
cluster_dst::> snapmirror policy create -vserver svm1 -policy DR_compressed -type async-mirror -comment "DR with network compression enabled" -is-network-compression-enabled true
----
+
Das folgende Beispiel erstellt eine benutzerdefinierte Replikationsrichtlinie für SnapVault:

+
[listing]
----
cluster_dst::> snapmirror policy create -vserver svm1 -policy my_snapvault -type vault
----
+
Das folgende Beispiel erstellt eine benutzerdefinierte Replikationsrichtlinie für die einheitliche Replikation:

+
[listing]
----
cluster_dst::> snapmirror policy create -vserver svm1 -policy my_unified -type mirror-vault
----
+
Das folgende Beispiel erstellt eine benutzerdefinierte Replikationsrichtlinie für eine SnapMirror synchron Beziehung im StrictSync Modus:

+
[listing]
----
cluster_dst::> snapmirror policy create -vserver svm1 -policy my_strictsync -type strict-sync-mirror -common-snapshot-schedule my_sync_schedule
----
+
Weitere Informationen zu  `snapmirror policy create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-policy-create.html["ONTAP-Befehlsreferenz"^].



.Nachdem Sie fertig sind
Für die Richtlinientypen "`vault`" und "`mirror-vault`" sind Regeln zu definieren, die festlegen, welche Snapshots während der Initialisierung und Aktualisierung übertragen werden.

Mit dem  `snapmirror policy show` Befehl lässt sich überprüfen, ob die SnapMirror Richtlinie erstellt wurde.

Weitere Informationen zu  `snapmirror policy show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-policy-show.html["ONTAP-Befehlsreferenz"^].
