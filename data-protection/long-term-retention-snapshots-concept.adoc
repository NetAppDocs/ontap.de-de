= Informationen zu ONTAP Langzeitaufbewahrungs-Snapshots
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapMirror Beziehungen mit einem Richtlinientyp von entweder „vault“ oder „mirror-vault“ ermöglichen die direkte Erstellung von Snapshots auf dem sekundären Volume der SnapMirror Beziehung. Diese Snapshots werden auf dem Ziel als Backups aufbewahrt. Diese Snapshots werden häufig für die langfristige Aufbewahrung erstellt und als Snapshots zur langfristigen Aufbewahrung bezeichnet.

Ein Langzeit-Aufbewahrungs-Snapshot wird erstellt, indem in der SnapMirror Richtlinienregel ein Zeitplan für die Snapshot-Erstellung, das Snapshot-Namenspräfix, das SnapMirror Label und die Aufbewahrungsanzahl angegeben werden. Dieser Snapshot bleibt auf dem SnapMirror Ziel-Volume erhalten, unabhängig von den Aufbewahrungsregeln auf der Quelle.

Langzeitaufbewahrungs-Snapshots sind nur für FlexVol SnapMirror Konfigurationen verfügbar. Langzeitaufbewahrungs-Snapshots können für FlexGroup SnapMirror Konfigurationen nicht erstellt werden.

In einer SnapMirror Kaskadenbeziehung können Langzeit-Snapshots nur auf dem letzten Volume der Kaskade erstellt werden.
