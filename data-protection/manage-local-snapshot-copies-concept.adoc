= Informationen zur Verwaltung lokaler ONTAP Snapshots
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Snapshot ist ein schreibgeschütztes, zeitpunktgenaues Image eines Volumes. Das Image benötigt minimalen Speicherplatz und verursacht einen vernachlässigbaren Performance-Overhead, da nur die Änderungen an Dateien seit dem letzten Snapshot aufgezeichnet werden.

Mithilfe eines Snapshots kann der gesamte Inhalt eines Volumes oder einzelne Dateien oder LUNs wiederhergestellt werden. Snapshots werden im Verzeichnis  `.snapshot` auf dem Volume gespeichert.

In ONTAP 9.4 und höher kann ein FlexVol Volume bis zu 1023 Snapshots enthalten. In ONTAP 9.3 und früher kann ein Volume bis zu 255 Snapshots enthalten.

[NOTE]
====
Ab ONTAP 9.8 können FlexGroup Volumes 1023 Snapshots enthalten. Weitere Informationen finden sich unter link:../flexgroup/protect-snapshot-copies-task.html["Schutz von FlexGroup Volumes mithilfe von Snapshots"].

====
Es gibt mehrere gängige ONTAP Snapshot-Typen:

* `sched_created`: Snapshots, die automatisch von einer ONTAP-Snapshot-Richtlinie auf Basis vordefinierter Zeitpläne (zum Beispiel stündlich, täglich, wöchentlich oder monatlich) erstellt werden.
* `sm_created`: Automatische Snapshots, die im Zuge von SnapMirror-Replikationsübertragungen (Aktualisierungs- oder Initialisierungsvorgängen) erstellt werden.
* `application_created`: Snapshots, die von Integrationssoftware auf Host- oder Datenbankebene wie SnapCenter oder SnapDrive generiert werden.
* `user_created`: Snapshots, die manuell von einem Administrator entweder über die ONTAP CLI oder den System Manager ausgelöst werden.
* `sv_created`: Snapshots, die automatisch während Cluster-Operationen erstellt werden, wie zum Beispiel bei Storage Virtual Machine (SVM) DR oder bestimmten Schutz-Workflows.
* `luns_created`: Snapshots, die erstellt wurden, um den Zustand von LUN- oder NVMe-Namespace-Klonen zu sichern oder zu erfassen.
