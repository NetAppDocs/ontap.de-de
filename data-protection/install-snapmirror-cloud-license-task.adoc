= Installation einer ONTAP SnapMirror Cloud-Lizenz
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapMirror Cloud-Beziehungen können mithilfe vorqualifizierter Backup-Anwendungen von Drittanbietern orchestriert werden. Ab ONTAP 9.9.1 ist es auch möglich, die SnapMirror Cloud-Replikation mit System Manager zu orchestrieren. Sowohl SnapMirror- als auch SnapMirror Cloud-Kapazitätslizenzen sind erforderlich, wenn mit System Manager On-Premises ONTAP-Backups auf Objektspeicher orchestriert werden. Außerdem ist es notwendig, die SnapMirror Cloud-API-Lizenz anzufordern und zu installieren.

.Über diese Aufgabe
Die SnapMirror cloud- und SnapMirror S3-Lizenzen sind Clusterlizenzen, keine Node-Lizenzen, daher werden sie nicht mit dem ONTAP One Lizenzbundle geliefert. Diese Lizenzen sind im separaten ONTAP One Compatibility Bundle enthalten. Für die Aktivierung von SnapMirror cloud ist die Anforderung dieses Bundles erforderlich.

Zusätzlich erfordert die System Manager Orchestrierung von SnapMirror Cloud-Backups in Objektspeicher einen SnapMirror Cloud-API-Schlüssel. Diese API-Lizenz ist eine clusterweite Einzelinstanzlizenz, das bedeutet, sie muss nicht auf jedem Knoten im Cluster installiert werden.

.Schritte
Es ist erforderlich, das ONTAP One Kompatibilitätspaket und die SnapMirror Cloud-API-Lizenz anzufordern und herunterzuladen und diese anschließend mit System Manager zu installieren.

. Die Cluster-UUID für den Cluster, der lizenziert werden soll, ermitteln und notieren.
+
Die Cluster-UUID wird benötigt, wenn Sie Ihre Anfrage zur Bestellung des ONTAP One Compatibility bundle für Ihren Cluster einreichen.

. Das NetApp Vertriebsteam kann kontaktiert werden, um das ONTAP One Kompatibilitätspaket anzufordern.
. Die SnapMirror Cloud-API-Lizenz kann gemäß den Anweisungen auf der NetApp Support Site angefordert werden.
+
link:https://mysupport.netapp.com/site/tools/snapmirror-cloud-api-key["Lizenzschlüssel für die SnapMirror Cloud-API anfordern"^]

. Nachdem Sie die Lizenzdateien erhalten und heruntergeladen haben, kann der System Manager verwendet werden, um die ONTAP Cloud Compatibility NLF und die SnapMirror cloud API NLF in den Cluster hochzuladen:
+
.. *Cluster > Einstellungen* auswählen.
.. Im Fenster *Einstellungen* auf *Lizenzen* klicken.
.. Im Fenster *Lizenzen* auf image:icon_add.gif["Symbol hinzufügen"] klicken.
.. Im Dialogfeld *Lizenz hinzufügen* auf *Durchsuchen* klicken, um die heruntergeladene NLF auszuwählen, und anschließend auf *Hinzufügen* klicken, um die Datei in den Cluster hochzuladen.
