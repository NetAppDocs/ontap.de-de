= ONTAP SnapMirror globale Drosselung verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Globale Netzwerkdrosselung ist für alle SnapMirror- und SnapVault-Übertragungen auf Knotenebene verfügbar.

.Über diese Aufgabe
SnapMirror globale Drosselung beschränkt die Bandbreite, die für eingehende und/oder ausgehende SnapMirror und SnapVault Transfers verwendet wird. Die Beschränkung wird clusterweit auf allen Knoten im Cluster durchgesetzt.

Wenn beispielsweise die ausgehende Bandbreite auf 100 MBps begrenzt ist, beträgt die ausgehende Bandbreite auf jedem Knoten im Cluster 100 MBps. Wenn die globale Begrenzung deaktiviert ist, ist sie auf allen Knoten deaktiviert.

Obwohl Datentransferraten oft in Bit pro Sekunde (bps) angegeben werden, müssen die Drosselungswerte in Kilobyte pro Sekunde (KBps) eingegeben werden.

[NOTE]
====
In ONTAP 9.9.1 und früheren Versionen hat die Drosselung keine Auswirkung auf  `volume move` Übertragungen oder Load-Sharing Mirror Übertragungen. Ab ONTAP 9.10.0 kann eine Option zur Drosselung eines Volume Move Vorgangs angegeben werden. Weitere Details finden sich unter link:https://kb.netapp.com/on-prem/ontap/Perf/Perf-KBs/How_to_throttle_volume_move_in_ONTAP_9.10_or_later["Drosselung der Volume-Verschiebung in ONTAP 9.10 und neueren Versionen"^].

====
Die globale Drosselung funktioniert zusammen mit der Drosselungsfunktion pro Beziehung für SnapMirror und SnapVault Transfers. Die Drosselung pro Beziehung wird so lange angewendet, bis die kombinierte Bandbreite der pro-Beziehung-Transfers den Wert der globalen Drosselung überschreitet, danach wird die globale Drosselung angewendet. Ein Drosselungswert `0`bedeutet, dass die globale Drosselung deaktiviert ist.

[NOTE]
====
SnapMirror globale Drosselung hat keine Auswirkung auf SnapMirror synchron Beziehungen, wenn sie In-Sync sind. Die Drosselung wirkt sich jedoch auf SnapMirror synchron Beziehungen aus, wenn sie eine asynchrone Übertragungsphase wie eine Initialisierungsoperation oder nach einem Out Of Sync-Ereignis durchführen. Aus diesem Grund wird die Aktivierung der globalen Drosselung bei SnapMirror synchron Beziehungen nicht empfohlen.

====
.Schritte
. Globale Drosselung aktivieren:
+
`options -option-name replication.throttle.enable on|off`

+
Das folgende Beispiel zeigt, wie die globale Drosselung von SnapMirror aktiviert wird `cluster_dst`:

+
[listing]
----
cluster_dst::> options -option-name replication.throttle.enable on
----
. Die maximale Gesamtbandbreite, die von eingehenden Übertragungen auf dem Ziel-Cluster verwendet wird, angeben:
+
`options -option-name replication.throttle.incoming.max_kbs <KBps>`

+
Die empfohlene minimale Bandbreite beträgt 4 Kilobyte pro Sekunde (KBps), die maximale bis zu 2 Terabyte pro Sekunde (TBps). Der Standardwert für diese Option ist  `unlimited`, was bedeutet, dass es keine Begrenzung für die insgesamt genutzte Bandbreite gibt.

+
Das folgende Beispiel zeigt, wie die maximale gesamte Bandbreite für eingehende Übertragungen auf 100 Megabit pro Sekunde (Mbit/s) festgelegt wird:

+
[listing]
----
cluster_dst::> options -option-name replication.throttle.incoming.max_kbs 12500
----
+
[NOTE]
====
100 Megabit pro Sekunde (Mbps) = 12500 Kilobyte pro Sekunde (KBps)

====
. Die maximale Gesamtbandbreite, die von ausgehenden Übertragungen auf dem Quell-Cluster verwendet wird, angeben:
+
`options -option-name replication.throttle.outgoing.max_kbs <KBps>`

+
Die empfohlene minimale Bandbreite beträgt 4 KBps, die maximale bis zu 2 TBps. Der Standardwert für diese Option ist  `unlimited`, was bedeutet, dass es keine Begrenzung für die insgesamt genutzte Bandbreite gibt. Die Parameterwerte werden in Kilobyte pro Sekunde (KBps) angegeben.

+
Das folgende Beispiel zeigt, wie die maximale Gesamtbandbreite für ausgehende Übertragungen auf 100 Mbit/s festgelegt wird:

+
[listing]
----
cluster_src::> options -option-name replication.throttle.outgoing.max_kbs 12500
----
