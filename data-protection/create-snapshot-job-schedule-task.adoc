= Erstellen eines ONTAP Snapshot Jobplans
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Snapshot-Richtlinie erfordert mindestens einen Snapshot-Auftragsplan. Es besteht die Möglichkeit, System Manager oder den  `job schedule cron create` Befehl zu verwenden, um einen Auftragsplan zu erstellen. Weitere Informationen zu  `job schedule cron create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/job-schedule-cron-create.html["ONTAP-Befehlsreferenz"^].

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/data-protection/policies-schedules.html#create-a-new-protection-policy-schedule["diese Schritte"^] zu folgen, um einen Snapshot-Jobzeitplan zu erstellen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

Standardmäßig bildet ONTAP die Namen von Snapshots, indem an den Jobplannamen ein Zeitstempel angehängt wird.

Wenn Sie sowohl den Tag des Monats als auch den Wochentag angeben, werden die Werte unabhängig voneinander betrachtet. Beispielsweise wird ein Cron-Schedule mit der Tag-Spezifikation `Friday` und der Tag-des-Monats-Spezifikation `13`jeden Freitag und am 13. Tag jedes Monats ausgeführt, nicht nur an jedem Freitag, der auf den 13. fällt.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Schutz > Übersicht* und erweitern Sie *Lokale Richtlinieneinstellungen*.
. Im Bereich *Zeitpläne* auf image:icon_arrow.gif["Pfeilsymbol"] klicken.
. Im Fenster *Zeitpläne* auf image:icon_add.gif["Symbol hinzufügen"] klicken.
. Im Fenster *Zeitplan hinzufügen* den Namen des Zeitplans eingeben sowie den Kontext und den Zeitplantyp auswählen.
. Auf *Speichern* klicken.


--
.CLI
--
. Einen Arbeitszeitplan erstellen:
+
[source, cli]
----
job schedule cron create -name <job_name> -month <month> -dayofweek <day_of_week> -day <day_of_month> -hour <hour> -minute <minute>
----
+
Für  `-month`,  `-dayofweek` und  `-hour` kann  `all` angegeben werden, damit der Job jeden Monat, an jedem Wochentag bzw. zu jeder Stunde ausgeführt wird.

+
Ab ONTAP 9.10.1 kann der Vserver in den Jobplan einbezogen werden:

+
[listing, subs="+quotes"]
----
job schedule cron create -name <job_name> *-vserver <Vserver_name>* -month <month> -dayofweek <day_of_week> -day <day_of_month> -hour <hour> -minute <minute>
----
+
Das folgende Beispiel erstellt einen Jobplan mit dem Namen  `myweekly`, der samstags um 3:00 Uhr morgens ausgeführt wird:

+
[listing]
----
cluster1::> job schedule cron create -name myweekly -dayofweek "Saturday" -hour 3 -minute 0
----
+
Das folgende Beispiel erstellt einen Zeitplan mit dem Namen `myweeklymulti`, der mehrere Tage, Stunden und Minuten angibt:

+
[listing]
----
job schedule cron create -name myweeklymulti -dayofweek "Monday,Wednesday,Sunday" -hour 3,9,12 -minute 0,20,50
----
