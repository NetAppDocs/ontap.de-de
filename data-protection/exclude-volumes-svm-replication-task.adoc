= Ausschließen von Volumes aus einer ONTAP SnapMirror SVM DR-Beziehung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Standardmäßig werden alle RW-Datenvolumes der Quell-SVM repliziert. Wenn nicht alle Volumes der Quell-SVM geschützt werden sollen, kann die  `-vserver-dr-protection unprotected` Option des  `volume modify` Befehls verwendet werden, um Volumes von der SVM-Replikation auszuschließen.

.Schritte
. Ein Volume von der SVM-Replikation ausschließen:
+
`volume modify -vserver _SVM_ -volume _volume_ -vserver-dr-protection unprotected`

+
Weitere Informationen zu  `volume modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-modify.html["ONTAP-Befehlsreferenz"^].

+
Im folgenden Beispiel wird das Volume `volA_src` von der SVM-Replikation ausgeschlossen:

+
[listing]
----
cluster_src::> volume modify -vserver SVM1 -volume volA_src -vserver-dr-protection unprotected
----
+
Wenn Sie später ein Volume in die SVM-Replikation einbeziehen möchten, das Sie ursprünglich ausgeschlossen haben, führen Sie den folgenden Befehl aus:

+
`volume modify -vserver _SVM_ -volume _volume_ -vserver-dr-protection protected`

+
Das folgende Beispiel beinhaltet das Volumen `volA_src` in der SVM-Replikation:

+
[listing]
----
cluster_src::> volume modify -vserver SVM1 -volume volA_src -vserver-dr-protection protected
----
. Die SVM-Replikationsbeziehung wird erstellt und initialisiert, wie in link:replicate-entire-svm-config-task.html["Replikation einer gesamten SVM-Konfiguration"] beschrieben.
