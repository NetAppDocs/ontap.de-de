= Eine ONTAP SnapMirror Replikationsbeziehung manuell aktualisieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Möglicherweise ist eine manuelle Aktualisierung einer Replikationsbeziehung erforderlich, wenn eine Aktualisierung fehlschlägt, weil das Quellvolume verschoben wurde.

.Über diese Aufgabe
SnapMirror bricht alle Übertragungen von einem verschobenen Quellvolume ab, bis die Replikationsbeziehung manuell aktualisiert wird.

Ab ONTAP 9.5 werden SnapMirror synchron Beziehungen unterstützt. Obwohl Quell- und Zielvolume in diesen Beziehungen stets synchron sind, wird die Ansicht vom sekundären Cluster nur stündlich mit dem primären synchronisiert. Wenn die zeitpunktgenauen Daten am Ziel angezeigt werden sollen, ist eine manuelle Aktualisierung durch Ausführen des  `snapmirror update` Befehls erforderlich.

.Schritt
. Eine Replikationsbeziehung manuell aktualisieren:
+
`snapmirror update -source-path <SVM:volume>|<cluster://SVM/volume>, ... -destination-path <SVM:volume>|<cluster://SVM/volume>, ...`

+
[NOTE]
====
Dieser Befehl muss von der Ziel-SVM oder dem Ziel-Cluster ausgeführt werden. Der Befehl schlägt fehl, wenn auf der Quelle und dem Ziel kein gemeinsamer Snapshot vorhanden ist. Mit  `snapmirror initialize` kann die Beziehung neu initialisiert werden. Weitere Informationen zu  `snapmirror initialize` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-initialize.html["ONTAP-Befehlsreferenz"^].

====
+
Das folgende Beispiel aktualisiert die Beziehung zwischen dem Quellvolume `volA` auf `svm1` und dem Zielvolume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_src::> snapmirror update -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror update` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-update.html["ONTAP-Befehlsreferenz"^].
