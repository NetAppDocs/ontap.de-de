= Eine einzelne Datei aus einem ONTAP Snapshot wiederherstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den  `volume snapshot restore-file` Befehl verwenden, um eine einzelne Datei oder LUN aus einem Snapshot wiederherzustellen. Die Datei kann an einem anderen Speicherort im übergeordneten lesen/schreiben-Volume wiederhergestellt werden, wenn keine vorhandene Datei ersetzt werden soll.

.Über diese Aufgabe
Wenn eine vorhandene LUN wiederhergestellt wird, wird ein LUN-Klon erstellt und in Form eines Snapshots gesichert. Während des Wiederherstellungsvorgangs ist lesen und schreiben auf die LUN möglich.

Dateien mit Streams werden standardmäßig wiederhergestellt.

.Schritte
. Snapshots in einem Volume auflisten:
+
`volume snapshot show -vserver _SVM_ -volume _volume_`

+
Weitere Informationen zu  `volume snapshot show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-show.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel zeigt die Momentaufnahmen in `vol1`:

+
[listing]
----

clus1::> volume snapshot show -vserver vs1 -volume vol1

Vserver Volume Snapshot                State    Size  Total% Used%
------- ------ ---------- ----------- ------   -----  ------ -----
vs1	    vol1   hourly.2013-01-25_0005  valid   224KB     0%    0%
               daily.2013-01-25_0010   valid   92KB      0%    0%
               hourly.2013-01-25_0105  valid   228KB     0%    0%
               hourly.2013-01-25_0205  valid   236KB     0%    0%
               hourly.2013-01-25_0305  valid   244KB     0%    0%
               hourly.2013-01-25_0405  valid   244KB     0%    0%
               hourly.2013-01-25_0505  valid   244KB     0%    0%

7 entries were displayed.
----
. Eine Datei aus einem Snapshot wiederherstellen:
+
`volume snapshot restore-file -vserver _SVM_ -volume _volume_ -snapshot _snapshot_ -path _file_path_ -restore-path _destination_path_`

+
Weitere Informationen zu  `volume snapshot restore-file` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-restore-file.html["ONTAP-Befehlsreferenz"^].

+
Das folgende Beispiel stellt die Datei  `myfile.txt` wieder her:

+
[listing]
----
cluster1::> volume snapshot restore-file -vserver vs0 -volume vol1 -snapshot daily.2013-01-25_0010 -path /myfile.txt
----
