= Informationen zur Konfiguration benutzerdefinierter ONTAP Snapshot-Richtlinien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Snapshot-Richtlinie definiert, wie das System Snapshots erstellt. Die Richtlinie legt fest, wann Snapshots erstellt werden, wie viele Kopien aufbewahrt werden und wie diese benannt werden. Beispielsweise könnte ein System täglich um 0:10 Uhr einen Snapshot erstellen, die beiden aktuellsten Kopien aufbewahren und die Kopien „daily.`_timestamp_`.“ nennen.

Die Standardrichtlinie für ein Volume erstellt automatisch Snapshots nach folgendem Zeitplan, wobei die ältesten Snapshots gelöscht werden, um Platz für neuere Kopien zu schaffen:

* Maximal sechs stündliche Momentaufnahmen, die fünf Minuten nach der vollen Stunde erstellt werden.
* Maximal zwei tägliche Schnappschüsse, aufgenommen von Montag bis Samstag jeweils 10 Minuten nach Mitternacht.
* Maximal zwei wöchentliche Snapshots, die jeden Sonntag um 0:15 Uhr erstellt werden.


Sofern beim Erstellen eines Volumes keine Snapshot-Richtlinie angegeben wird, übernimmt das Volume die Snapshot-Richtlinie der zugehörigen Storage Virtual Machine (SVM).
