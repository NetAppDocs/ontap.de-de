= Eine ONTAP SnapMirror SVM-Replikationsbeziehung kann so umgewandelt werden, dass SVM-Konfigurationsinformationen einbezogen werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine SVM DR-Beziehung, die SVM-Konfigurationsinformationen während der Replikation ausschließt, mithilfe des  `-identity-preserve true` Parameters in eine Beziehung umwandeln, die die Konfigurationsinformationen repliziert.

.Bevor Sie beginnen
Wenn derzeit SnapMirror-Transfers laufen, entweder abwarten, bis diese abgeschlossen sind, oder sie abbrechen.

.Schritte
. Vom Ziel-Cluster aus die Beziehung stilllegen:
+
`snapmirror quiesce -destination-path <destination_svm>`:

. Wenn der Beziehungsstatus auf „ruhend“ gesetzt ist, löschen Sie die Beziehung vom Ziel-Cluster aus:
+
`snapmirror delete -destination-path <destination_svm>`:

. Vom Quell-Cluster die Beziehung freigeben, ohne SnapMirror Snapshot-Informationen zu löschen:
+
`snapmirror release -destination-path <destination_svm>: -relationship-info-only true`

. Vom Ziel aus die Replikationsbeziehung mit  `-identity-preserve true` erneut erstellen:
+
`snapmirror create -source-path <source_svm>: -destination-path <destination_svm>: -policy <policy> -schedule <schedule> -identity-preserve true`

. Vom Ziel-Cluster aus die SnapMirror Beziehung neu synchronisieren:
+
`snapmirror resync -destination-path <destination_svm>:`
