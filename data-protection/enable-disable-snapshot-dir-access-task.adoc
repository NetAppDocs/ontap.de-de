= Aktivieren und Deaktivieren des NFS- und SMB-Clientzugriffs auf das ONTAP Snapshot-Verzeichnis
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den Zugriff auf das Snapshot-Verzeichnis mithilfe der ONTAP CLI `-snapdir-access` Option des `volume modify` Befehls aktivieren und deaktivieren, und ab ONTAP 9.10.1 besteht die Möglichkeit, mit System Manager den Zugriff von Clientsystemen auf ein Snapshot-Verzeichnis auf einem Volume zu aktivieren oder zu deaktivieren. Durch die Aktivierung des Zugriffs wird das Snapshot-Verzeichnis für Clients sichtbar, und Windows-Clients können ein Laufwerk mit dem Snapshot-Verzeichnis verbinden, um dessen Inhalt anzuzeigen und darauf zuzugreifen. NFS- und SMB-Clients können dann eine Datei oder LUN aus einem Snapshot wiederherstellen.

Sie können den Zugriff auf das Snapshot-Verzeichnis eines Volumes aktivieren oder deaktivieren, indem die Volume-Einstellungen oder die Freigabeeinstellungen des Volumes bearbeitet werden.



[[enable-or-disable-client-access-to-snapshot-directory-by-editing-a-volume]]
== Clientzugriff auf das Snapshot-Verzeichnis durch Bearbeiten eines Volumes aktivieren oder deaktivieren

.Schritte
Sie können den Zugriff auf das Client-Snapshot-Verzeichnis mithilfe von ONTAP System Manager oder der ONTAP CLI aktivieren und deaktivieren. Das Snapshot-Verzeichnis auf einem Volume ist standardmäßig für Clients zugänglich.

[role="tabbed-block"]
====
.System Manager
--
. *Speicher > Volumes* auswählen.
. Wählen Sie das Volume aus, das das Snapshots-Verzeichnis enthält, das entweder angezeigt oder ausgeblendet werden soll.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Bearbeiten*.
. Im Abschnitt *Snapshot (Lokale) Einstellungen* wird *Snapshot-Verzeichnis für Clients anzeigen* ausgewählt oder abgewählt.
. Auf *Speichern* klicken.


--
.CLI
--
. Status des Zugriffs auf das Snapshot-Verzeichnis prüfen:
+
[source, cli]
----
volume show -vserver <SVM_name> -volume <vol_name> -fields snapdir-access
----
+
Beispiel:

+
[listing]
----

clus1::> volume show -vserver vs0 -volume vol1 -fields snapdir-access
vserver volume snapdir-access
------- ------ --------------
vs0     vol1   false
----
+
Weitere Informationen zu  `volume show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-show.html["ONTAP-Befehlsreferenz"^].

. Der Zugriff auf das Snapshot-Verzeichnis kann aktiviert oder deaktiviert werden:
+
[source, cli]
----
volume modify -vserver <SVM_name> -volume <vol_name> -snapdir-access <true|false>
----
+
Das folgende Beispiel ermöglicht den Zugriff auf das Snapshot-Verzeichnis auf vol1:

+
[listing]
----

clus1::> volume modify -vserver vs0 -volume vol1 -snapdir-access true
Volume modify successful on volume vol1 of Vserver vs0.
----
+
Weitere Informationen zu  `volume modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-modify.html["ONTAP-Befehlsreferenz"^].



--
====

== Clientzugriff auf das Snapshot-Verzeichnis kann durch Bearbeiten einer Freigabe aktiviert oder deaktiviert werden.

Das Snapshot-Verzeichnis auf einem Volume ist standardmäßig für Clients zugänglich.

.Schritte
. *Speicher > Freigaben* auswählen.
. Wählen Sie das Volume aus, das das Snapshots-Verzeichnis enthält, das entweder angezeigt oder ausgeblendet werden soll.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Bearbeiten*.
. Im Abschnitt *Freigabeeigenschaften* wird die Option *Clients den Zugriff auf das Snapshot-Verzeichnis erlauben* ausgewählt oder abgewählt.
. Auf *Speichern* klicken.
