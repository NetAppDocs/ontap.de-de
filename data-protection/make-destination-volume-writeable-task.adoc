= Das ONTAP SnapMirror Zielvolume beschreibbar machen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Zielvolume muss beschreibbar gemacht werden, bevor Daten von diesem Volume an Clients ausgeliefert werden können. Um Daten von einem SnapMirror Ziel bereitzustellen, wenn eine Quelle nicht verfügbar ist, werden die geplanten Übertragungen zum Ziel gestoppt und anschließend die SnapMirror Beziehung getrennt, um das Ziel beschreibbar zu machen.

.Über diese Aufgabe
Diese Aufgabe muss von der Ziel-SVM oder dem Ziel-Cluster ausgeführt werden.

.Schritte
Sie können System Manager oder die ONTAP CLI verwenden, um ein Zielvolume beschreibbar zu machen.

[role="tabbed-block"]
====
.System Manager
--
. Die Schutzbeziehung wird ausgewählt, indem auf *Schutz > Beziehungen* und dann auf den gewünschten Volume-Namen geklickt wird.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"].
. Geplante Übertragungen anhalten: *Pause* auswählen.
. Das Ziel beschreibbar machen: Auf *Break* klicken.
. Auf der Hauptseite *Beziehungen* lässt sich überprüfen, ob der Beziehungsstatus als "abgebrochen" angezeigt wird.


.Nächste Schritte
Sie müssen link:resynchronize-relationship-task.html["die Replikationsbeziehung rückwärts resynchronisieren"] ausführen, nachdem das Zielvolume beschreibbar gemacht wurde.

Wenn das deaktivierte Quellvolume wieder verfügbar ist, sollte die Beziehung erneut rücksynchronisiert werden, um die aktuellen Daten auf das ursprüngliche Quellvolume zu kopieren.

--
.CLI
--
. Geplante Transfers zum Ziel-Cluster anhalten:
+
[source, cli]
----
snapmirror quiesce -source-path <SVM:volume|cluster://SVM/volume> -destination-path <SVM:volume|cluster://SVM/volume>
----
+
Das folgende Beispiel stoppt geplante Übertragungen zwischen dem Quell-Volume `volA` auf `svm1` und dem Ziel-Volume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_dst::> snapmirror quiesce -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror quiesce` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-quiesce.html["ONTAP-Befehlsreferenz"^].

. Laufende Transfers zum Ziel-Cluster beenden:
+
[source, cli]
----
snapmirror abort -source-path <SVM:volume|cluster://SVM/volume> -destination-path <SVM:volume|cluster://SVM/volume>
----
+

NOTE: Dieser Schritt ist für SnapMirror synchron Beziehungen nicht erforderlich (unterstützt ab ONTAP 9.5).

+
Das folgende Beispiel stoppt laufende Übertragungen zwischen dem Quellvolume `volA` auf `svm1` und dem Zielvolume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_dst::> snapmirror abort -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror abort` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-abort.html["ONTAP-Befehlsreferenz"^].

. Die SnapMirror DR-Beziehung beenden:
+
[source, cli]
----
snapmirror break -source-path <SVM:volume|cluster://SVM/volume> -destination-path <SVM:volume|cluster://SVM/volume>
----
+
Das folgende Beispiel unterbricht die Beziehung zwischen dem Quellvolume `volA` auf `svm1` und dem Zielvolume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_dst::> snapmirror break -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror break` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-break.html["ONTAP-Befehlsreferenz"^].



.Nächste Schritte
Sie müssen link:resynchronize-relationship-task.html["die Replikationsbeziehung resynchronisieren"] ausführen, nachdem das Zielvolume beschreibbar gemacht wurde.
