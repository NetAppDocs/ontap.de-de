= Eine Datei aus einem ONTAP Snapshot auf einem NFS- oder SMB-Client wiederherstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Benutzer auf einem NFS- oder SMB-Client kann eine Datei direkt aus einem Snapshot wiederherstellen, ohne dass ein Systemadministrator des Speichersystems eingreifen muss.

Jedes Verzeichnis im Dateisystem enthält ein Unterverzeichnis mit dem Namen `.snapshot` , das für NFS- und SMB-Benutzer zugänglich ist. Das `.snapshot` Unterverzeichnis enthält Unterverzeichnisse, die den Snapshots des Volumes entsprechen:

....
$ ls .snapshot
daily.2017-05-14_0013/              hourly.2017-05-15_1106/
daily.2017-05-15_0012/              hourly.2017-05-15_1206/
hourly.2017-05-15_1006/             hourly.2017-05-15_1306/
....
Jedes Unterverzeichnis enthält die im Snapshot referenzierten Dateien. Wenn Benutzer versehentlich eine Datei löschen oder überschreiben, kann die Datei im übergeordneten Lese-/Schreibverzeichnis wiederhergestellt werden, indem die Datei aus dem Snapshot-Unterverzeichnis in das Lese-/Schreibverzeichnis kopiert wird:

....
$ ls my.txt
ls: my.txt: No such file or directory
$ ls .snapshot
daily.2017-05-14_0013/              hourly.2017-05-15_1106/
daily.2017-05-15_0012/              hourly.2017-05-15_1206/
hourly.2017-05-15_1006/             hourly.2017-05-15_1306/
$ ls .snapshot/hourly.2017-05-15_1306/my.txt
my.txt
$ cp .snapshot/hourly.2017-05-15_1306/my.txt .
$ ls my.txt
my.txt
....
