= Erstellen eines SMB-Servers für eine ONTAP SnapMirror Ziel-SVM in einer DR-Beziehung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn die Quell-SVM über eine SMB-Konfiguration verfügt und Sie  `identity-preserve` auf  `false` gesetzt haben, ist ein SMB-Server für die Ziel-SVM zu erstellen. Ein SMB-Server ist für einige SMB-Konfigurationen erforderlich, beispielsweise für Freigaben während der Initialisierung der SnapMirror Beziehung.

.Schritte
. Die Ziel-SVM wird mit dem  `vserver start` Befehl gestartet.
+
[listing]
----
destination_cluster::> vserver start -vserver dvs1
[Job 30] Job succeeded: DONE
----
+
Weitere Informationen zu  `vserver start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-start.html["ONTAP-Befehlsreferenz"^].

. Es sollte überprüft werden, ob sich die Ziel-SVM im  `running` Zustand befindet und der Subtyp  `dp-destination` ist, indem der  `vserver show` Befehl verwendet wird.
+
[listing]
----
destination_cluster::> vserver show
                                   Admin      Operational Root
Vserver  Type    Subtype           State      State       Volume     Aggregate
-------- ------- ----------       ---------- ----------- ---------- ----------
dvs1     data    dp-destination    running    running       -         -
----
+
Weitere Informationen zu  `vserver show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-show.html["ONTAP-Befehlsreferenz"^].

. Eine LIF wird mit dem  `network interface create` Befehl erstellt.
+
[listing]
----
destination_cluster::>network interface create -vserver dvs1 -lif NAS1 -role data -data-protocol cifs -home-node destination_cluster-01 -home-port a0a-101  -address 192.0.2.128 -netmask 255.255.255.128
----
+
Weitere Informationen zu  `network interface create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-create.html["ONTAP-Befehlsreferenz"^].

. Eine Route kann mithilfe des  `network route create` Befehls erstellt werden.
+
[listing]
----
destination_cluster::>network route create -vserver dvs1 -destination 0.0.0.0/0
-gateway 192.0.2.1
----
+
link:../networking/networking_reference.html["Netzwerkmanagement"]

+
Weitere Informationen zu  `network route create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-route-create.html["ONTAP-Befehlsreferenz"^].

. DNS kann mithilfe des  `vserver services dns create` Befehls konfiguriert werden.
+
[listing]
----
destination_cluster::>vserver services dns create -domains mydomain.example.com -vserver
dvs1 -name-servers 192.0.2.128 -state enabled
----
+
Weitere Informationen zu  `vserver services dns create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+services+dns+create["ONTAP-Befehlsreferenz"^].

. Der bevorzugte Domänencontroller wird mit dem  `vserver cifs domain preferred-dc add` Befehl hinzugefügt.
+
[listing]
----
destination_cluster::>vserver cifs domain preferred-dc add -vserver dvs1 -preferred-dc
192.0.2.128 -domain mydomain.example.com
----
+
Weitere Informationen zu  `vserver cifs domain preferred-dc add` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-cifs-domain-preferred-dc-add.html["ONTAP-Befehlsreferenz"^].

. Der SMB-Server wird mit dem  `vserver cifs create` Befehl erstellt.
+
[listing]
----
destination_cluster::>vserver cifs create -vserver dvs1 -domain mydomain.example.com
-cifs-server CIFS1
----
+
Weitere Informationen zu  `vserver cifs create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-cifs-create.html["ONTAP-Befehlsreferenz"^].

. Die Ziel-SVM wird mit dem  `vserver stop` Befehl angehalten.
+
[listing]
----
destination_cluster::> vserver stop -vserver dvs1
[Job 46] Job succeeded: DONE
----
+
Weitere Informationen zu  `vserver stop` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/vserver-stop.html["ONTAP-Befehlsreferenz"^].
