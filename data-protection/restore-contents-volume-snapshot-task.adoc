= Wiederherstellung des Inhalts eines Volumes aus einem ONTAP Snapshot
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Volume kann auf einen früheren Zeitpunkt zurückgesetzt werden, indem es aus einem Snapshot wiederhergestellt wird. Mit System Manager oder dem  `volume snapshot restore` Befehl lässt sich der Inhalt eines Volumes aus einem Snapshot wiederherstellen. Weitere Informationen zu  `volume snapshot restore` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-restore.html["ONTAP-Befehlsreferenz"^].

.Über diese Aufgabe
Wenn das Volume SnapMirror Beziehungen aufweist, sollten alle Spiegelkopien des Volumes unmittelbar nach der Wiederherstellung aus einem Snapshot manuell repliziert werden. Andernfalls können unbrauchbare Spiegelkopien entstehen, die gelöscht und neu erstellt werden müssen.

.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um aus einem früheren Snapshot wiederherzustellen.

[role="tabbed-block"]
====
.System Manager
--
. Auf *Speicher* klicken und ein Volume auswählen.
. Unter *Snapshot-Kopien* auf image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Snapshot, den Sie wiederherstellen möchten, klicken und *Wiederherstellen* auswählen.


--
.CLI
--
. Snapshots in einem Volume auflisten:
+
[source, cli]
----
volume snapshot show -vserver <svm> -volume <volume>
----
+
Das folgende Beispiel zeigt die Momentaufnahme in  `vol1`:

+
[listing]
----

clus1::> volume snapshot show -vserver vs1 -volume vol1

Vserver Volume Snapshot                State    Size  Total% Used%
------- ------ ---------- ----------- ------   -----  ------ -----
vs1	    vol1   hourly.2013-01-25_0005  valid   224KB     0%    0%
               daily.2013-01-25_0010   valid   92KB      0%    0%
               hourly.2013-01-25_0105  valid   228KB     0%    0%
               hourly.2013-01-25_0205  valid   236KB     0%    0%
               hourly.2013-01-25_0305  valid   244KB     0%    0%
               hourly.2013-01-25_0405  valid   244KB     0%    0%
               hourly.2013-01-25_0505  valid   244KB     0%    0%

7 entries were displayed.
----
. Den Inhalt eines Volumes aus einem Snapshot wiederherstellen:
+
[source, cli]
----
volume snapshot restore -vserver <svm> -volume <volume> -snapshot <snapshot>
----
+
Das folgende Beispiel stellt den Inhalt von  `vol1` wieder her:

+
[listing]
----
cluster1::> volume snapshot restore -vserver vs0 -volume vol1 -snapshot daily.2013-01-25_0010
----
