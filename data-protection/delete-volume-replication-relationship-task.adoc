= Löschen einer ONTAP SnapMirror Volume-Replikationsbeziehung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die  `snapmirror delete` und  `snapmirror release` Befehle verwenden, um eine Volume-Replikationsbeziehung zu löschen. Anschließend können nicht mehr benötigte Ziel-Volumes manuell gelöscht werden.

.Über diese Aufgabe
Der  `snapmirror release`Befehl löscht alle von SnapMirror erstellten Snapshots aus der Quelle. Mit der  `-relationship-info-only`Option besteht die Möglichkeit, die Snapshots zu erhalten.

.Schritte
. Die Replikationsbeziehung anhalten:
+
`snapmirror quiesce -destination-path <SVM:volume>|<cluster://SVM/volume>`

+
[listing]
----
cluster_dst::> snapmirror quiesce -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror quiesce` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-quiesce.html["ONTAP-Befehlsreferenz"^].

. (Optional) Trennen der Replikationsbeziehung, wenn das Ziel-Volume ein lesen/schreiben-Volume sein muss. Sie können diesen Schritt überspringen, wenn Sie das Ziel-Volume löschen möchten oder das Volume nicht für lesen/schreiben benötigt wird:
+
`snapmirror break -source-path <SVM:volume>|<cluster://SVM/volume>, …​ -destination-path <SVM:volume>|<cluster://SVM/volume>, …​`

+
[listing]
----
cluster_dst::> snapmirror break -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror break` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-break.html["ONTAP-Befehlsreferenz"^].

. Die Replikationsbeziehung löschen:
+
`snapmirror delete -source-path <SVM:volume>|<cluster://SVM/volume>, ... -destination-path <SVM:volume>|<cluster://SVM/volume>, ...`

+
[NOTE]
====
Dieser Befehl muss vom Ziel-Cluster oder der Ziel-SVM ausgeführt werden.

====
+
Das folgende Beispiel löscht die Beziehung zwischen dem Quellvolume `volA` auf `svm1` und dem Zielvolume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_dst::> snapmirror delete -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror delete` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-delete.html["ONTAP-Befehlsreferenz"^].

. Informationen zur Replikationsbeziehung aus der Quell-SVM freigeben:
+
`snapmirror release -source-path <SVM:volume>|<cluster://SVM/volume>, ... -destination-path <SVM:volume>|<cluster://SVM/volume>, ...`

+
[NOTE]
====
Dieser Befehl muss vom Quell-Cluster oder der Quell-SVM ausgeführt werden.

====
+
Das folgende Beispiel gibt Informationen für die angegebene Replikationsbeziehung von der Quell-SVM  `svm1` frei:

+
[listing]
----
cluster_src::> snapmirror release -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror release` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-release.html["ONTAP-Befehlsreferenz"^].
