= Informationen zu erweiterten Abfragen für ONTAP SnapMirror Beziehungsoperationen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können erweiterte Abfragen verwenden, um SnapMirror Operationen an mehreren SnapMirror Beziehungen gleichzeitig durchzuführen. Beispielsweise kann es mehrere nicht initialisierte SnapMirror Beziehungen geben, die mit einem einzigen Befehl initialisiert werden sollen.

.Über diese Aufgabe
Es besteht die Möglichkeit, erweiterte Abfragen auf die folgenden SnapMirror Operationen anzuwenden:

* Initialisierung nicht initialisierter Beziehungen
* Wiederaufnahme ruhender Beziehungen
* Resynchronisierung unterbrochener Beziehungen
* Aktualisierung inaktiver Beziehungen
* Abbruch von Beziehungsdatenübertragungen


.Schritt
. Eine SnapMirror Operation auf vielen Beziehungen durchführen:
+
`_snapmirror command_ {-state state } *`

+
Der folgende Befehl initialisiert SnapMirror Beziehungen, die sich in einem  `Uninitialized` Zustand befinden:

+
[listing]
----
vs1::> snapmirror initialize {-state Uninitialized} *
----
+
Weitere Informationen zu  `snapmirror initialize` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-initialize.html["ONTAP-Befehlsreferenz"^].
