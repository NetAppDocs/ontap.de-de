= Erstellen einer ONTAP Snapshot-Richtlinie
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Snapshot-Richtlinie legt fest, wann Snapshots erstellt werden, wie viele Kopien aufbewahrt werden und wie diese benannt werden. Beispielsweise kann ein System täglich um 0:10 Uhr einen Snapshot erstellen, die beiden aktuellsten Kopien aufbewahren und sie „daily.`_timestamp_`.“ nennen. Eine Snapshot-Richtlinie kann bis zu fünf Auftragspläne enthalten.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/data-protection/policies-schedules.html#create-a-snapshot-policy["diese Schritte"^] zu befolgen, um eine Snapshot Policy zu erstellen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

Standardmäßig bildet ONTAP die Namen von Snapshots, indem dem Namen des Jobplans ein Zeitstempel angehängt wird:

[listing]
----
daily.2017-05-14_0013/              hourly.2017-05-15_1106/
daily.2017-05-15_0012/              hourly.2017-05-15_1206/
hourly.2017-05-15_1006/             hourly.2017-05-15_1306/
----
Es kann ein Präfix anstelle des Auftragsplannamens verwendet werden, falls gewünscht.

Die  `snapmirror-label` Option ist für die SnapMirror Replikation. Weitere Informationen finden sich unter link:define-rule-policy-task.html["Definieren einer Regel für eine Richtlinie"].

.Schritte
Sie können eine Snapshot-Richtlinie mithilfe von System Manager oder der ONTAP CLI erstellen. Das Verfahren erstellt eine Snapshot-Richtlinie nur auf dem lokalen Cluster.

[role="tabbed-block"]
====
.System Manager
--
. Navigieren Sie zu *Schutz > Übersicht* und erweitern Sie *Lokale Richtlinieneinstellungen*.
. Im Bereich *Snapshot-Richtlinien* auf image:icon_arrow.gif["Pfeilsymbol"] klicken.
. Im Tab *Snapshot-Richtlinien* auf image:icon_add.gif["Symbol hinzufügen"] klicken.
. Im Fenster *Snapshot-Richtlinie hinzufügen* den Richtliniennamen eingeben und den Geltungsbereich auswählen.
. Klicken Sie image:icon_add.gif["Symbol hinzufügen"].
. Um einen Zeitplan auszuwählen, auf den aktuell angezeigten Zeitplannamen klicken, auf image:icon_dropdown_arrow.gif["Dropdown-Symbol"] klicken und einen anderen Zeitplan auswählen.
. Geben Sie die maximale Anzahl der aufzubewahrenden Snapshots ein und, falls erforderlich, die SnapMirror-Bezeichnung und die SnapLock-Aufbewahrungsdauer.
. Auf *Speichern* klicken.


--
.CLI
--
. Erstellen einer Snapshot-Richtlinie:
+
[source, cli]
----
volume snapshot policy create -vserver <SVM> -policy <policy_name> -enabled true|false -schedule1 <schedule1_name> -count1 <copies_to_retain> -prefix1 <snapshot_prefix> -snapmirror-label1 <snapshot_label> ... -schedule5 <schedule5_name> -count5 <copies_to_retain> -prefix5 <snapshot_prefix> -snapmirror-label5 <snapshot_label>
----
+
Das folgende Beispiel erstellt eine Snapshot-Richtlinie mit dem Namen  `snap_policy_daily`, die nach einem  `daily` Zeitplan ausgeführt wird. Die Richtlinie hat maximal fünf Snapshots, jeweils mit dem Namen  `daily`.`_timestamp_` und dem SnapMirror-Label  `daily`:

+
[listing]
----
cluster1::> volume snapshot policy create -vserver vs0 -policy snap_policy_daily -schedule1 daily -count1 5 -snapmirror-label1 daily
----
