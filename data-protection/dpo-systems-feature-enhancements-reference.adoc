= Funktionserweiterungen von ONTAP DPO-Systemen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ab ONTAP 9.6 erhöht sich die maximale Anzahl unterstützter FlexVol Volumes, wenn die DP_Optimized (DPO) Lizenz installiert ist. Ab ONTAP 9.4 unterstützen Systeme mit der DPO Lizenz SnapMirror Backoff, volumenübergreifende Hintergrunddeduplizierung, die Verwendung von Snapshot-Blöcken als Donoren und Kompaktierung.

Ab ONTAP 9.6 wurde die maximal unterstützte Anzahl von FlexVol Volumes auf sekundären oder Datensicherungssystemen erhöht, sodass eine Skalierung auf bis zu 2.500 FlexVol Volumes pro Knoten oder bis zu 5.000 im Failover-Modus möglich ist. Die Erhöhung der FlexVol Volumes wird mit dem link:../data-protection/snapmirror-licensing-concept.html#data-protection-optimized-license["DP_Optimized (DPO) Lizenz"] ermöglicht. Ein link:../system-admin/manage-license-task.html#view-details-about-a-license["SnapMirror Lizenz"] ist weiterhin sowohl auf dem Quell- als auch auf dem Zielknoten erforderlich.

Ab ONTAP 9.4 sind die folgenden Funktionserweiterungen für DPO-Systeme verfügbar:

* SnapMirror backoff: In DPO-Systemen erhält der Replikationsverkehr die gleiche Priorität wie die Client-Workloads.
+
SnapMirror Backoff ist auf DPO-Systemen standardmäßig deaktiviert.

* Volume Hintergrunddeduplizierung und volumeübergreifende Hintergrunddeduplizierung: Volume Hintergrunddeduplizierung und volumeübergreifende Hintergrunddeduplizierung sind in DPO Systemen aktiviert.
+
Sie können den  `storage aggregate efficiency cross-volume-dedupe start -aggregate _aggregate_name_ -scan-old-data true` Befehl ausführen, um die vorhandenen Daten zu deduplizieren. Die empfohlene Vorgehensweise ist, den Befehl außerhalb der Spitzenzeiten auszuführen, um die Auswirkungen auf die Leistung zu verringern.

+
Weitere Informationen zu  `storage aggregate efficiency cross-volume-dedupe start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/storage-aggregate-efficiency-cross-volume-dedupe-start.html["ONTAP-Befehlsreferenz"^].

* Erhöhte Einsparungen durch die Verwendung von Snapshot-Blöcken als Spender: Die Datenblöcke, die im aktiven Dateisystem nicht verfügbar sind, aber in Snapshots gespeichert sind, werden als Spender für die Volume-Deduplizierung verwendet.
+
Die neuen Daten können mit den in Snapshots gespeicherten Daten dedupliziert werden, wodurch die Snapshot-Blöcke effektiv gemeinsam genutzt werden. Der vergrößerte freie Speicherplatz führt zu weiteren Einsparungen, insbesondere wenn das Volume eine große Anzahl von Snapshots enthält.

* Compaction: Die Datenkompaktierung ist auf DPO-Volumes standardmäßig aktiviert.
