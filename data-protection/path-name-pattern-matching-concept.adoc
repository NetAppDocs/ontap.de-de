= Informationen zur Mustererkennung von Pfadnamen in ONTAP SnapMirror Befehlen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mithilfe von Mustervergleich können Sie die Quell- und Zielpfade in  `snapmirror` Befehlen angeben.

 `snapmirror`Befehle verwenden vollqualifizierte Pfadnamen im folgenden Format: `vserver:volume`. Der Pfadname kann abgekürzt werden, wenn der SVM-Name nicht eingegeben wird. Wenn dies erfolgt, nimmt der  `snapmirror` Befehl den lokalen SVM-Kontext des Benutzers an.

Unter der Annahme, dass die SVM "`vserver1`" heißt und das Volume "`vol1`" heißt, lautet der vollqualifizierte Pfadname  `vserver1:vol1`.

Sie können das Sternchen (*) in Pfaden als Platzhalter verwenden, um übereinstimmende, vollqualifizierte Pfadnamen auszuwählen. Die folgende Tabelle enthält Beispiele für die Verwendung des Platzhalters zur Auswahl eines Bereichs von Volumes.

[cols="2*"]
|===


 a| 
`*`
 a| 
Entspricht allen Pfaden.



 a| 
`vs*`
 a| 
Findet alle SVMs und Volumes, deren SVM-Namen mit  `vs` beginnen.



 a| 
`*:*src*`
 a| 
Findet alle SVMs mit Volumennamen, die den `src` Text enthalten.



 a| 
`*:vol*`
 a| 
Findet alle SVMs mit Volumennamen, die mit  `vol` beginnen.

|===
[listing]
----
vs1::> snapmirror show -destination-path *:*dest*
                                                                                Progress
Source              Destination  Mirror        Relationship  Total              Last
Path          Type  Path         State         Status        Progress   Healthy Updated
------------- ---- ------------ ------------- -------------- ---------- ------- --------
vs1:sm_src2
              DP   vs2:sm_dest1
                                Snapmirrored  Idle           -          true    -
----
Weitere Informationen zu  `snapmirror show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-show.html["ONTAP-Befehlsreferenz"^].
