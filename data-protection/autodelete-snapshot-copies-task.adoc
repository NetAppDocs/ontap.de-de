= Automatisches Löschen von ONTAP Snapshots
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können den  `volume snapshot autodelete modify` Befehl verwenden, um die automatische Löschung von Snapshots auszulösen, wenn der Snapshot-Reserve überschritten wird. Standardmäßig werden die ältesten Snapshots zuerst gelöscht. Weitere Informationen zu  `volume snapshot autodelete modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/volume-snapshot-autodelete-modify.html["ONTAP-Befehlsreferenz"^].

.Über diese Aufgabe
LUN- und Dateiklone werden gelöscht, wenn keine Snapshots mehr gelöscht werden müssen.

.Schritt
. Automatisches Löschen von Snapshots:
+
`volume snapshot autodelete modify -vserver _SVM_ -volume _volume_ -enabled true|false -trigger volume|snap_reserve`

+
Das folgende Beispiel löscht Snapshots für  `vol1` automatisch, wenn der Snapshot-Reserve erschöpft ist:

+
[listing]
----
cluster1::> volume snapshot autodelete modify -vserver vs0 -volume vol1 -enabled true -trigger snap_reserve
----
