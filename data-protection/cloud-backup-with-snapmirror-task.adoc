= Daten in der Cloud mit ONTAP SnapMirror Cloud sichern
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
_SnapMirror Cloud_ bietet ONTAP Replikation auf Objektspeicherziele. Dies ermöglicht es Ihnen, ältere Backup-to-Tape-Architekturen durch Objektspeicher für die langfristige Datenaufbewahrung und Archivierung zu ersetzen. Sie können Anbieter wie Amazon S3, Alibaba Cloud, Azure Blob, Google Cloud Storage, IBM Cloud Object Storage, NetApp StorageGRID oder ONTAP S3 als Ihren Cloud-Objektspeicher verwenden.


NOTE: SnapMirror Cloud-Replikation, die von System Manager verwaltet wird, unterstützt ausschließlich StorageGrid und ONTAP S3 als Objektspeicherziel. Andere Anbieter werden nicht unterstützt.


NOTE: Die Unterstützung für SnapMirror Cloud-Replikation zu anderen Objektspeicheranbietern mit anderen Backup und Recovery-Produkten variiert. Informationen zu den Supportrichtlinien erhalten Sie von Ihrem Drittanbieter.


NOTE: Die Nutzung der SnapMirror Cloud-APIs ist grundsätzlich auf Backup-Partner beschränkt. Kunden, die eigene API-basierte Lösungen entwickeln möchten, erhalten Informationen zu den Lizenzbedingungen über ihr NetApp Account-Team.

Ab ONTAP 9.19.1:

* SnapMirror Cloud erhöht die Anzahl der pro Cluster unterstützten S3-Objektspeicher-Buckets von 6 auf 100. Diese Begrenzung gilt für die Gesamtzahl der Objektspeicher, die über alle SnapMirror Cloud Objektspeicher-Endpunkte auf einem ONTAP Cluster konfiguriert sind. Bei der Nutzung dieser Funktion sind folgende Aspekte zu berücksichtigen:
+
** Wenn Sie mehr als 6 Buckets pro Cluster verwenden, sollten Sie die Virtual-Hosted-Style-URL-Konfiguration für Anbieter nutzen, die dies unterstützen, einschließlich AWS S3, Alibaba Cloud, StorageGrid, Google Cloud Storage, IBM Cloud Object Storage, ONTAP S3 und andere S3-kompatible Anbieter. Diese Konfiguration wird empfohlen, um eine skalierbare Adressierung des Objektspeichers zu ermöglichen und zugrunde liegende Einschränkungen zu vermeiden.
** Die Wiederherstellung bei Bedarf ist standardmäßig aktiviert.
** Die Latenz von Lesevorgängen während bedarfsgesteuerter Wiederherstellungen kann höher sein, je nachdem, wie viele gleichzeitige Wiederherstellungen im System aktiv sind.




Ab ONTAP 9.18.1:

* SnapMirror Cloud unterstützt Backup- und Recovery-Operationen für FlexGroup Volumes auf MetroCluster Konfigurationen unter Verwendung des bestehenden link:https://docs.netapp.com/us-en/ontap-automation/get-started/access_rest_api.html["ONTAP REST-APIs"^]. Diese Funktionalität ermöglicht die Erstellung von SnapMirror Cloud-Beziehungen für FlexGroup Volumes auf MetroCluster Konfigurationen, die nach einem Switchover und Switchback vom Partnerstandort aus verwaltet werden.


Vor der Nutzung der SnapMirror Cloud-Funktion empfiehlt es sich, einen SnapMirror Cloud-API-Lizenzschlüssel von der NetApp Support Site anzufordern: link:https://mysupport.netapp.com/site/tools/snapmirror-cloud-api-key["Lizenzschlüssel für die SnapMirror Cloud API anfordern"^].

Gemäß den Anweisungen ist eine kurze Beschreibung Ihres Geschäftsvorhabens bereitzustellen und der API-Schlüssel durch eine E-Mail an die angegebene Adresse anzufordern. Innerhalb von 24 Stunden sollte eine E-Mail-Antwort mit weiteren Anweisungen zum Erwerb des API-Schlüssels eingehen.


NOTE: Sie benötigen möglicherweise sowohl einen SnapMirror Cloud Lizenzschlüssel als auch einen API-Schlüssel. ONTAP One enthält zwar die SnapMirror Cloud Lizenz selbst, der Lizenzschlüssel ist jedoch erforderlich, um die SnapMirror Cloud Funktion zu aktivieren. Der Lizenzschlüssel muss gegebenenfalls separat generiert und installiert werden.



[[add-a-cloud-object-store]]
== Einen Cloud-Objektspeicher hinzufügen

Bevor Sie SnapMirror Cloud-Backups konfigurieren, müssen Sie einen StorageGRID oder ONTAP S3 Cloud-Objektspeicher hinzufügen.

.Schritte
. *Schutz > Übersicht > Cloud-Objektspeicher* auswählen.
. Klicken Sie image:icon_add.gif["Symbol hinzufügen"].




[[back-up-using-the-default-policy]]
== Sicherung mit der Standardrichtlinie

Sie können schnell ein SnapMirror Cloud-Backup für ein vorhandenes Volume mit der standardmäßigen Cloud-Datensicherungsstrategie, DailyBackup, konfigurieren.

.Schritte
. *Schutz > Übersicht* auswählen und *Volumes in der Cloud sichern* wählen.
. Wenn Sie zum ersten Mal ein Backup in der Cloud erstellen, geben Sie Ihren SnapMirror Cloud-API-Lizenzschlüssel wie angegeben in das Lizenzfeld ein.
. *Authentifizieren und fortfahren* auswählen.
. Wählen Sie ein Quellvolume aus.
. Einen Cloud Objektspeicher auswählen.
. Auf *Speichern* klicken.




[[create-a-custom-cloud-backup-policy]]
== Erstellen einer benutzerdefinierten Cloud Backup-Richtlinie

Wenn Sie nicht die Standard-Cloud-Richtlinie DailyBackup für Ihre SnapMirror Cloud-Backups verwenden möchten, kann eine eigene Richtlinie erstellt werden.

.Schritte
. *Schutz > Übersicht > Lokale Richtlinieneinstellungen* auswählen und *Schutzrichtlinien* selektieren.
. Auf *Hinzufügen* klicken und die Details der neuen Richtlinie eingeben.
. Im Abschnitt *Richtlinientyp* *In der Cloud sichern* auswählen, um anzugeben, dass eine Cloud-Richtlinie erstellt wird.
. Auf *Speichern* klicken.




[[create-a-backup-from-the-volumes-page]]
== Eine Sicherung kann auf der Seite *Volumes* erstellt werden.

Die Seite *Volumes* im System Manager kann verwendet werden, wenn Cloud-Backups für mehrere Volumes gleichzeitig ausgewählt und erstellt werden sollen oder wenn eine benutzerdefinierte Datensicherungsstrategie genutzt wird.

.Schritte
. *Speicher > Volumes* auswählen.
. Wählen Sie die Volumes aus, die Sie in der Cloud sichern möchten, und klicken Sie auf *Protect*.
. Im Fenster *Volume schützen* auf *Weitere Optionen* klicken.
. Eine Richtlinie auswählen.
+
Sie können die Standardrichtlinie DailyBackup oder eine von Ihnen erstellte benutzerdefinierte Cloud-Richtlinie auswählen.

. Einen Cloud Objektspeicher auswählen.
. Auf *Speichern* klicken.




[[restore-from-the-cloud]]
== Wiederherstellung aus der Cloud

Mit dem System Manager ist es möglich, gesicherte Daten aus dem Cloud-Speicher auf ein anderes Volume im Quell-Cluster wiederherzustellen.


NOTE: Wenn Sie ONTAP 9.16.1 oder höher verwenden und eine SnapMirror Cloud-Einzeldateiwiederherstellung auf ein FlexGroup Volume durchführen, sollten Dateien nur in ein neues Verzeichnis im FlexGroup Volume wiederhergestellt werden, und Granulare Daten muss auf  `advanced` auf dem Ziel-FlexGroup Volume gesetzt sein. Weitere Informationen zum Festlegen der  `-granular-data advanced` Option finden sich unter link:../flexgroup/enable-adv-capacity-flexgroup-task.html["ONTAP FlexGroup Volumes durch Umverteilung von Dateidaten ausgleichen"].

.Schritte
. Im Quell-Cluster einer SnapMirror-to-Cloud-Beziehung auf *Storage > Volumes* klicken.
. Wählen Sie das Volume aus, das Sie wiederherstellen möchten.
. Die Registerkarte *In der Cloud sichern* auswählen.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben das Quellvolume, das Sie wiederherstellen möchten, um das Menü anzuzeigen, und wählen Sie *Wiederherstellen*.
. Unter *Quelle* eine Storage-VM auswählen und anschließend den Namen des Volumes eingeben, auf das die Daten wiederhergestellt werden sollen.
. Unter *Ziel* den Snapshot auswählen, der wiederhergestellt werden soll.
. Auf *Speichern* klicken.




[[delete-a-snapmirror-cloud-relationship]]
== Eine SnapMirror Cloud-Beziehung löschen

Mit dem System Manager lässt sich eine Cloud-Beziehung löschen.

.Schritte
. Klicken Sie auf *Speicher > Volumes* und wählen Sie das Volume aus, das Sie löschen möchten.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben das Quellvolume und wählen Sie *Löschen*.
. *Cloud Objektspeicher-Endpunkt löschen (optional)* kann ausgewählt werden, wenn der Cloud Objektspeicher-Endpunkt gelöscht werden soll.
. Klicken Sie auf *Delete*.




[[remove-a-cloud-object-store]]
== Entfernen eines Cloud-Objektspeichers

Sie können den System Manager verwenden, um einen Cloud Objektspeicher zu entfernen, sofern dieser nicht Teil einer Cloud Backup Beziehung ist. Wenn ein Cloud Objektspeicher Teil einer Cloud Backup Beziehung ist, kann er nicht gelöscht werden.

.Schritte
. *Schutz > Übersicht > Cloud-Objektspeicher* auswählen.
. Wählen Sie den Objektspeicher, den Sie löschen möchten, klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Löschen*.
