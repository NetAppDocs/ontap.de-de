= Snapshots manuell erstellen und löschen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Snapshots manuell erstellen, wenn Sie nicht auf die Erstellung eines geplanten Snapshots warten können, und Sie können Snapshots löschen, wenn sie nicht mehr benötigt werden.

.Über diese Aufgabe
Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/data-protection/create-snapshots.html#step-2-create-a-snapshot["diese Schritte"^] zu befolgen, um einen On-Demand Snapshot zu erstellen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.



[[create-a-snapshot-manually]]
== Snapshot manuell erstellen

Sie können manuell einen Snapshot mit System Manager oder der ONTAP CLI erstellen.

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. Zu *Speicher > Volumes* navigieren und die Registerkarte *Snapshots* auswählen.
. Klicken Sie image:icon_add.gif["Symbol hinzufügen"].
. Im Fenster *Snapshot hinzufügen* lässt sich der Standard-Snapshot-Name übernehmen oder bei Bedarf bearbeiten.
. *Optional*: Eine SnapMirror Beschriftung kann hinzugefügt werden.
. Klicken Sie auf *Add*.


--
.CLI
--
. Snapshot erstellen:
+
[source, cli]
----
volume snapshot create -vserver <SVM> -volume <volume> -snapshot <snapshot_name>
----


--
====


[[delete-snapshots-manually]]
== Snapshots manuell löschen

Sie können einen Snapshot manuell über den System Manager oder die ONTAP CLI löschen.

[role="tabbed-block"]
====
.System Manager
--
.Schritte
. Navigieren Sie zu *Speicher > Volumes* und wählen Sie die Registerkarte *Snapshot copies* aus.
. Den Snapshot, den Sie löschen möchten, suchen, auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken und *Löschen* auswählen.
. Im Fenster *Snapshot löschen* wird *Snapshot löschen* ausgewählt.
. Klicken Sie auf *Delete*.


--
.CLI
--
. Mit dem  `volume snapshot show` Befehl lässt sich überprüfen, welche Snapshots gelöscht werden sollen.
+
[source, cli]
----
volume snapshot show -vserver <SVM> -volume <volume>
----
+
In diesem Beispiel zeigt der Befehl die Snapshots auf dem Volume vol3 in der SVM vs3 an.

+

. Einen Snapshot löschen:
+
[cols="2*"]
|===
| Wenn Sie möchten... | Geben Sie diesen Befehl ein... 


 a| 
Einen einzelnen Snapshot löschen
 a| 
[source, cli]
----
volume snapshot delete -vserver _svm_name_ -volume _vol_name_ -snapshot _snapshot_name_
----


 a| 
Mehrere Snapshots löschen
 a| 
[source, cli]
----
volume snapshot delete -vserver _svm_name_ -volume _vol_name_ -snapshot _snapshot_name1_[,_snapshot_name2_,...]
----


 a| 
Alle Snapshots löschen
 a| 
[source, cli]
----
volume snapshot delete -vserver _svm_name_ -volume _vol_name_ -snapshot *
----
|===
