= Kriterien für die Platzierung von Volumes auf ONTAP SnapMirror Ziel-SVMs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Beim Replizieren von Volumes von der Quell-SVM zur Ziel-SVM ist es wichtig, die Kriterien für die Auswahl der Aggregate zu kennen.

Die Auswahl der Aggregate erfolgt anhand der folgenden Kriterien:

* Volumes werden immer auf Nicht-Root-Aggregaten platziert.
* Nicht-Root-Aggregate werden basierend auf dem verfügbaren freien Speicherplatz und der Anzahl der bereits auf dem Aggregat gehosteten Volumes ausgewählt.
+
Aggregate mit mehr freiem Speicherplatz und weniger Volumes erhalten Priorität. Das Aggregat mit der höchsten Priorität wird ausgewählt.

* Quellvolumina auf FabricPool Aggregaten werden auf FabricPool Aggregaten am Ziel mit derselben Tiering-Policy platziert.
* Wenn sich ein Volume auf der Quell-SVM in einem Flash Pool Aggregat befindet, wird das Volume in einem Flash Pool Aggregat auf der Ziel-SVM platziert, sofern ein solches Aggregat existiert und über genügend freien Speicherplatz verfügt.
* Wenn die  `-space-guarantee` Option des zu replizierenden Volumes auf  `volume` gesetzt ist, werden nur Aggregate berücksichtigt, deren freier Speicherplatz größer als die Volume-Größe ist.
* Die Volume-Größe wächst während der Replikation auf dem Ziel-SVM automatisch basierend auf der Quell-Volume-Größe.
+
Wenn Sie die Größe auf der Ziel-SVM vorab reservieren möchten, müssen Sie das Volume anpassen. Die Volume-Größe wird auf der Ziel-SVM nicht automatisch basierend auf der Quell-SVM verkleinert.



Wenn ein Volume von einem Aggregat auf ein anderes verschoben werden soll, kann der  `volume move` Befehl auf der Ziel-SVM verwendet werden.
