= Eine ONTAP SnapMirror Replikationsbeziehung neu synchronisieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Eine Replikationsbeziehung muss neu synchronisiert werden, nachdem ein Zielvolume beschreibbar gemacht wurde, nachdem ein Update fehlgeschlagen ist, weil kein gemeinsamer Snapshot auf dem Quell- und Zielvolume vorhanden ist, oder wenn die Replikationsrichtlinie für die Beziehung geändert werden soll.

Ab ONTAP 9.8 kann mit System Manager eine umgekehrte Resynchronisierung durchgeführt werden, um eine bestehende Schutzbeziehung zu löschen und die Funktionen der Quell- und Zielvolumes umzukehren. Anschließend wird das Zielvolume zur Datenbereitstellung verwendet, während das Quellvolume repariert oder ersetzt, aktualisiert und die ursprüngliche Systemkonfiguration wiederhergestellt wird.

[NOTE]
====
System Manager unterstützt keine umgekehrte Resynchronisierung mit Intracluster-Beziehungen. Mit der ONTAP CLI lassen sich umgekehrte Resynchronisierungsvorgänge mit Intracluster-Beziehungen durchführen.

====
.Über diese Aufgabe
* Obwohl für die Resynchronisierung kein Basistransfer erforderlich ist, kann sie zeitaufwändig sein. Die Resynchronisierung kann außerhalb der Spitzenzeiten durchgeführt werden.
* Volumes, die Teil einer Fan-Out- oder Kaskadenkonfiguration sind, können länger für die Resynchronisierung benötigen. Es ist nicht ungewöhnlich, dass die SnapMirror Beziehung den Status „preparing“ über einen längeren Zeitraum meldet.
* Ab ONTAP 9.13.1 versucht ONTAP standardmäßig, die Schnellsynchronisierung zu nutzen, um die Synchronisierungszeit zu verkürzen. Die folgenden Bedingungen müssen erfüllt sein, damit die Schnellsynchronisierung standardmäßig verwendet wird:
+
** FlexVol Volumes enthalten keine Klone auf dem Volume
** Bei Verwendung der MirrorAllSnapshots Richtlinie
+

NOTE: Die Verwendung von  `-quick-resync` kann aufgrund der Entfernung der Speichereffizienz bei übertragenen Datenblöcken zusätzlichen Speicherplatz auf dem Resync-Ziel-Volume beanspruchen. Dieser zusätzliche Speicherplatz wird im Rahmen der Inline- oder Post-Replikations-Speichereffizienzanwendung auf dem Resync-Ziel wiederhergestellt.

+
Der `-quick-resync` Parameter ist optional. Die Schnell-Resynchronisierung kann durch Verwendung des `-quick-resync true|false` Parameters mit dem `snapmirror resync` Befehl aktiviert oder deaktiviert werden.

+
Weitere Informationen zu `-quick-resync` finden sich unter link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-resync.html["ONTAP-Befehlsreferenz"^].





.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um diese Aufgabe auszuführen. Bei Verwendung der ONTAP CLI ist das Verfahren unabhängig davon gleich, ob ein Ziel-Volume beschreibbar gemacht oder die Replikationsbeziehung aktualisiert wird.

[role="tabbed-block"]
====
.System Manager Reverse-Resynchronisierung
--
Nachdem Sie link:make-destination-volume-writeable-task.html["eine Beziehung beenden"] ein Ziel beschreibbar gemacht haben, wird die Beziehung durch Rücksynchronisierung umgekehrt:

. Im Ziel-Cluster auf *Schutz > Beziehungen* klicken.
. Fahren Sie mit dem Mauszeiger über die abgebrochene Beziehung, klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie *Reverse Resync* aus.
. Im Fenster *Umgekehrte Resynchronisierungsbeziehung* auf *Umgekehrte Resynchronisierung* klicken.
. Unter *Beziehungen* lässt sich der Fortschritt der umgekehrten Resynchronisierung überwachen, indem *Übertragungsstatus* für die Beziehung angezeigt wird.


.Nächste Schritte
Wenn die ursprüngliche Quelle wieder verfügbar ist, kann die ursprüngliche Beziehung wiederhergestellt werden, indem die umgekehrte Beziehung aufgehoben und eine weitere Reverse-Resync-Operation durchgeführt wird. Der Reverse-Resync-Prozess kopiert alle Änderungen von der Seite, die die Daten bereitstellt, auf die ursprüngliche Quelle und macht die ursprüngliche Quelle wieder lesen/schreiben-fähig.

--
.System Manager Resynchronisierung
--
. *Schutz > Beziehungen* auswählen.
. Bewegen Sie den Mauszeiger über die Beziehung, die Sie neu synchronisieren möchten, und klicken Sie auf image:icon_kabob.gif["Symbol für Menüoptionen"] und wählen Sie dann *Trennen*.
. Wenn der Beziehungsstatus „Abgebrochen“ angezeigt wird, image:icon_kabob.gif["Symbol für Menüoptionen"] und anschließend *Neu synchronisieren* ausgewählt wird.
. Unter *Beziehungen* lässt sich der Fortschritt der Resynchronisierung überwachen, indem der Beziehungsstatus überprüft wird. Der Status ändert sich zu „Gespiegelt“, wenn die Resynchronisierung abgeschlossen ist.


--
.CLI
--
. Die Quell- und Ziel-Volumes werden erneut synchronisiert:
+
[source, cli]
----
snapmirror resync -source-path <SVM:volume|cluster://SVM/volume> -destination-path <SVM:volume|cluster://SVM/volume> -type DP|XDP -policy <policy>
----
+

NOTE: Dieser Befehl muss von der Ziel-SVM oder dem Ziel-Cluster ausgeführt werden.

+
Das folgende Beispiel resynchronisiert die Beziehung zwischen dem Quellvolume `volA` auf `svm1` und dem Zielvolume `volA_dst` auf `svm_backup`:

+
[listing]
----
cluster_dst::> snapmirror resync -source-path svm1:volA -destination-path svm_backup:volA_dst
----
+
Weitere Informationen zu  `snapmirror resync` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/snapmirror-resync.html["ONTAP-Befehlsreferenz"^].
