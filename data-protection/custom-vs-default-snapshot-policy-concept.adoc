= Wann eine benutzerdefinierte ONTAP Snapshot-Richtlinie konfiguriert werden sollte
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn die Standard-Snapshot-Richtlinie für ein Volume nicht geeignet ist, kann eine benutzerdefinierte Richtlinie konfiguriert werden, die Häufigkeit, Aufbewahrungsdauer und Namen der Snapshots anpasst. Der Zeitplan wird hauptsächlich durch die Änderungsrate des aktiven Dateisystems bestimmt.

Ein stark frequentiertes Dateisystem wie eine Datenbank wird beispielsweise stündlich gesichert, während selten genutzte Dateien einmal täglich gesichert werden. Selbst bei einer Datenbank erfolgt üblicherweise ein- bis zweimal täglich eine vollständige Sicherung, während die Transaktionsprotokolle stündlich gesichert werden.

Weitere Faktoren sind die Bedeutung der Dateien für Ihr Unternehmen, Ihre Service Level Agreement (SLA), Ihr Recovery Point Objective (RPO) und Ihr Recovery Time Objective (RTO). Im Allgemeinen sollten nur so viele Snapshots aufbewahrt werden, wie unbedingt erforderlich sind.
