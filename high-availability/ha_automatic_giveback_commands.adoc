= ONTAP automatische Rückgabebefehle
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Standardmäßig gibt der übernehmende Partnerknoten den Speicher automatisch wieder zurück, wenn der offline gegangene Knoten wieder online ist, wodurch die High-Availability-Paarbeziehung wiederhergestellt wird. In den meisten Fällen ist dies das gewünschte Verhalten. Falls die automatische Rückgabe deaktiviert werden muss, zum Beispiel um die Ursache der Übernahme vor der Rückgabe zu untersuchen, ist auf die Wechselwirkung von abweichenden Einstellungen zu achten.

|===


| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Die automatische Rückgabe wird aktiviert, sodass die Rückgabe erfolgt, sobald der übernommene Knoten hochfährt, den Status „Warten auf Rückgabe“ erreicht und die Verzögerung vor automatischer Rückgabe abgelaufen ist.

Die Standardeinstellung ist „true“.
 a| 
`storage failover modify ‑node _nodename_ ‑auto‑giveback true`



 a| 
Automatische Rückgabe deaktivieren. Die Standardeinstellung ist „true“.

*Hinweis:* Wenn dieser Parameter auf „false“ gesetzt wird, wird die automatische Rückgabe nach einer Übernahme im Panikfall nicht deaktiviert; die automatische Rückgabe nach einer Übernahme im Panikfall muss durch Setzen des  `‑auto‑giveback‑after‑panic` Parameters auf „false“ deaktiviert werden.
 a| 
`storage failover modify ‑node _nodename_ ‑auto‑giveback false`



 a| 
Die automatische Rückgabe nach Übernahme im Panikfall ist zu deaktivieren (diese Einstellung ist standardmäßig aktiviert).
 a| 
`storage failover modify ‑node _nodename_ ‑auto‑giveback‑after‑panic false`



 a| 
Die automatische Rückgabe wird um eine bestimmte Anzahl von Sekunden verzögert (Standardwert ist 600). Diese Option legt die Mindestzeit fest, die ein Knoten in der Übernahme verbleibt, bevor eine automatische Rückgabe durchgeführt wird.
 a| 
`storage failover modify ‑node _nodename_ ‑delay‑seconds _seconds_`

|===


[[how-variations-of-the-storage-failover-modify-command-affect-automatic-giveback]]
== Wie sich Variationen des Befehls storage failover modify auf die automatische Rückgabe auswirken

Die Funktionsweise der automatischen Rückführung hängt davon ab, wie die Parameter des Befehls storage failover modify konfiguriert werden.

Die folgende Tabelle listet die Standardeinstellungen für die  `storage failover modify` Befehlsparameter auf, die für Übernahmeereignisse gelten, die nicht durch einen Panikzustand verursacht wurden.

|===


| Parameter | Standardeinstellung 


 a| 
`-auto-giveback _true_ | _false_`
 a| 
`_true_`



 a| 
`-delay-seconds _integer_ (seconds)`
 a| 
600



 a| 
`-onreboot _true_ | _false_`
 a| 
`_true_`

|===
Die folgende Tabelle beschreibt, wie Kombinationen der  `-onreboot` und  `-auto-giveback` Parameter die automatische Rückgabe bei Übernahmeereignissen beeinflussen, die nicht durch eine Panik verursacht wurden.

|===


| `storage failover modify` verwendete Parameter | Grund der Übernahme | Erfolgt ein automatisches Giveback? 


 a| 
`-onreboot _true_`

`-auto-giveback _true_`
| Neustartbefehl | Ja 


| Stoppbefehl oder Neustartvorgang, der vom Service Processor ausgelöst wird | Ja 


 a| 
`-onreboot _true_`

`-auto-giveback _false_`
| Neustartbefehl | Ja 


| Stoppbefehl oder Neustartvorgang, der vom Service Processor ausgelöst wird | Nein 


 a| 
`-onreboot _false_`

`-auto-giveback _true_`
| Neustartbefehl | Nicht zutreffend, in diesem Fall findet keine Übernahme statt 


| Stoppbefehl oder Neustartvorgang, der vom Service Processor ausgelöst wird | Ja 


 a| 
`-onreboot _false_`

`-auto-giveback _false_`
| Neustartbefehl | Nein 


| Stoppbefehl oder Neustartvorgang, der vom Service Processor ausgelöst wird | Nein 
|===
Der  `-auto-giveback`Parameter steuert die Rückgabe nach einem Panic und allen anderen automatischen Takeovers. Wenn der  `-onreboot`Parameter auf  `true`gesetzt ist und eine Übernahme aufgrund eines Reboots erfolgt, wird die automatische Rückgabe immer durchgeführt, unabhängig davon, ob der  `-auto-giveback`Parameter auf  `true`gesetzt ist.

Der  `-onreboot`Parameter gilt für Neustarts und Halt-Befehle, die von ONTAP ausgeführt werden. Wenn der  `-onreboot`Parameter auf false gesetzt ist, findet im Fall eines Knotenneustarts keine Übernahme statt. Daher kann keine automatische Rückgabe erfolgen, unabhängig davon, ob der  `-auto-giveback`Parameter auf true gesetzt ist. Eine Client-Unterbrechung tritt auf.

== Die Auswirkungen automatischer Rückmeldeparameterkombinationen, die in Paniksituationen gelten.

`_true_`

|===
Die folgende Tabelle beschreibt, wie sich Parameterkombinationen des  `storage failover modify` Befehls auf die automatische Rückgabe in Paniksituationen auswirken.

[cols="60,40"]
|===


| `storage failover` verwendete Parameter | Erfolgt nach einem Panic eine automatische Rückgabe? 


| `-onpanic true`
`-auto-giveback true`
`-auto-giveback-after-panic true` | Ja 


| `-onpanic true`
`-auto-giveback true`
`-auto-giveback-after-panic false` | Ja 


| `-onpanic true`
`-auto-giveback false`
`-auto-giveback-after-panic true` | Ja 


| `-onpanic true`
`-auto-giveback false`
`-auto-giveback-after-panic false` | Nein 


| `-onpanic false`Wenn  `-onpanic`auf  `false` gesetzt ist, findet keine Übernahme/Rückgabe statt, unabhängig vom Wert, der für  `-auto-giveback` oder  `-auto-giveback-after-panic` festgelegt ist. | Nein 
|===

NOTE: Eine Übernahme kann durch einen Fehler ausgelöst werden, der nicht mit einer Panik zusammenhängt. Ein _Ausfall_ tritt auf, wenn die Kommunikation zwischen einem Node und seinem Partner verloren geht, was auch als _Heartbeat-Verlust_ bezeichnet wird. Wenn ein Takeover aufgrund eines Ausfalls erfolgt, wird der Giveback durch den Parameter `-onfailure` anstelle des Parameters `-auto-giveback-after-panic parameter` gesteuert.


NOTE: Wenn ein Knoten in Panik gerät, sendet er ein Panikpaket an seinen Partnerknoten. Wird dieses Panikpaket aus irgendeinem Grund nicht vom Partnerknoten empfangen, kann die Panik fälschlicherweise als Fehler interpretiert werden. Ohne den Empfang des Panikpakets weiß der Partnerknoten lediglich, dass die Kommunikation unterbrochen wurde, und nicht, dass eine Panik aufgetreten ist. In diesem Fall behandelt der Partnerknoten den Kommunikationsverlust als Fehler anstatt als Panik, und die Rückgabe wird durch den  `-onfailure` Parameter (und nicht durch den  `-auto-giveback-after-panic parameter`) gesteuert.

Weitere Informationen zu  `storage failover modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/storage-failover-modify.html["ONTAP-Befehlsreferenz"^].
