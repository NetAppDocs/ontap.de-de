= Der Bannertext, der auf ONTAP Cluster- und SVM-Ebene angezeigt wird, kann verwaltet werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können das Banner auf Cluster- oder SVM-Ebene verwalten. Das für den Cluster konfigurierte Banner wird auch für alle SVMs verwendet, für die keine Banner-Nachricht definiert ist. Ein später für eine SVM erstelltes Banner überschreibt das Cluster-Banner für diese SVM.

.Auswahlmöglichkeiten
* Das Banner auf Clusterebene verwalten:
+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Ein Banner für alle Anmeldesitzungen anzeigen lassen
 a| 
Banner auf Clusterebene festlegen:

`security login banner modify -vserver <cluster_name> { [-message "text"] | [-uri <ftp_or_http_addr>] }`



 a| 
Das Banner für alle (Cluster- und SVM-)Logins entfernen
 a| 
Setzen Sie das Banner auf eine leere Zeichenkette (``):

`security login banner modify -vserver * -message`



 a| 
Ein von einem SVM-Administrator erstelltes Banner überschreiben
 a| 
Die SVM-Bannernachricht ändern:

`security login banner modify -vserver <svm_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`

|===
* Das Banner auf SVM-Ebene verwalten:
+
Die Angabe von  `-vserver <svm_name>` ist im SVM Kontext nicht erforderlich.

+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Das vom Clusteradministrator bereitgestellte Banner kann durch ein anderes Banner für die SVM ersetzt werden.
 a| 
Ein Banner für die SVM erstellen:

`security login banner modify -vserver <svm_name> { [-message "text"] | [-uri <ftp_or_http_addr>] }`



 a| 
Das vom Clusteradministrator bereitgestellte Banner wird unterdrückt, sodass für die SVM kein Banner angezeigt wird.
 a| 
Das SVM-Banner für die SVM auf eine leere Zeichenkette setzen:

`security login banner modify -vserver <svm_name> -message`



 a| 
Das Banner auf Cluster-Ebene wird verwendet, wenn die SVM derzeit ein Banner auf SVM-Ebene verwendet.
 a| 
Das SVM-Banner wird auf  `-` gesetzt:

`security login banner modify -vserver <svm_name> -message -`

|===
