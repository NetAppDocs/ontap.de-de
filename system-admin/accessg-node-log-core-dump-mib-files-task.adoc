= ONTAP-Knotenprotokolle, Core-Dumps und MIB-Dateien können mit einem Webbrowser abgerufen werden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Webdienst der Service Processor Infrastructure (`spi` ist standardmäßig aktiviert, um einem Webbrowser den Zugriff auf die Protokoll-, Core-Dump- und MIB-Dateien eines Knotens im Cluster zu ermöglichen. Die Dateien bleiben auch dann zugänglich, wenn der Knoten ausgefallen ist, sofern der Knoten von seinem Partner übernommen wird.

.Bevor Sie beginnen
* Der Cluster-Management-LIF muss aktiv sein.
+
Sie können die Management-LIF des Clusters oder eines Knotens verwenden, um auf den  `spi` Webdienst zuzugreifen. Es wird jedoch empfohlen, die Cluster-Management-LIF zu verwenden.

+
Der  `network interface show` Befehl zeigt den Status aller LIFs im Cluster an.

+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

* Sie müssen ein lokales Benutzerkonto verwenden, um auf den  `spi` Webdienst zuzugreifen, Domänenbenutzerkonten werden nicht unterstützt.
* Falls Ihr Benutzerkonto nicht über die  `admin` Rolle verfügt (die standardmäßig Zugriff auf den  `spi` Webdienst hat), muss Ihrer Zugriffskontrollrolle Zugriff auf den  `spi` Webdienst gewährt werden.
+
Der  `vserver services web access show` Befehl zeigt an, welche Rollen Zugriff auf welche Webdienste haben.

* Wenn Sie das  `admin` Benutzerkonto nicht verwenden (das  `http` Zugriffsmethode standardmäßig beinhaltet), muss Ihr Benutzerkonto mit der  `http` Zugriffsmethode eingerichtet sein.
+
Der  `security login show`Befehl zeigt die Zugriffs- und Anmeldemethoden der Benutzerkonten sowie deren Zugriffskontrollrollen.

+
Weitere Informationen zu  `security login show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-show.html["ONTAP-Befehlsreferenz"^].

* Wenn Sie HTTPS für sicheren Webzugriff verwenden möchten, muss SSL aktiviert und ein digitales Zertifikat installiert sein.
+
Der  `system services web show`Befehl zeigt die Konfiguration der Webprotokoll-Engine auf Cluster-Ebene an.



.Über diese Aufgabe
Der  `spi`Webdienst ist standardmäßig aktiviert, und der Dienst kann manuell deaktiviert werden (`vserver services web modify -vserver * -name spi -enabled false`.

Die  `admin`Rolle erhält standardmäßig Zugriff auf den  `spi`Webdienst, und der Zugriff kann manuell deaktiviert werden (`services web access delete -vserver _cluster_name_ -name spi -role admin`.

.Schritte
. Den Webbrowser auf die  `spi` Webdienst-URL in einem der folgenden Formate richten:
+
** `http://_cluster-mgmt-LIF_/spi/`
** `https://_cluster-mgmt-LIF_/spi/`
+
`cluster-mgmt-LIF` ist die IP-Adresse des Cluster-Management-LIF.



. Wenn der Browser dazu auffordert, Benutzerkonto und Passwort eingeben.
+
Nach der Authentifizierung Ihres Kontos zeigt der Browser Links zu den  `/mroot/etc/log/`,  `/mroot/etc/crash/` und  `/mroot/etc/mib/` Verzeichnissen jedes Knotens im Cluster an.
