= Probleme beim Zugriff auf ONTAP Webdienste beheben
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Konfigurationsfehler führen zu Problemen beim Zugriff auf Webdienste. Die Fehler lassen sich beheben, wenn LIF, Firewall-Richtlinie, Webprotokoll-Engine, Webdienste, digitale Zertifikate und die Benutzerzugriffsberechtigung korrekt konfiguriert sind.

Die folgende Tabelle unterstützt bei der Identifizierung und Behebung von Konfigurationsfehlern bei Webdiensten:

|===
| Dieses Zugriffsproblem... | Tritt aufgrund dieses Konfigurationsfehlers auf... | Um den Fehler zu beheben... 


 a| 
Ihr Webbrowser gibt eine  `unable to connect` oder  `failure to establish a connection` Fehlermeldung zurück, wenn Sie versuchen, auf einen Webdienst zuzugreifen.
 a| 
Ihre LIF ist möglicherweise nicht korrekt konfiguriert.
 a| 
Stellen Sie sicher, dass Sie den LIF anpingen können, der den Webdienst bereitstellt.

[NOTE]
====
Sie verwenden den  `network ping` Befehl, um einen LIF anzupingen.

====


 a| 
Ihre Firewall ist möglicherweise falsch konfiguriert.
 a| 
Es muss sichergestellt sein, dass eine Firewall-Richtlinie eingerichtet ist, die HTTP oder HTTPS unterstützt, und dass diese Richtlinie dem LIF zugewiesen ist, der den Webdienst bereitstellt.

[NOTE]
====
Sie verwenden die  `system services firewall policy` Befehle, um Firewall-Richtlinien zu verwalten. Sie verwenden den  `network interface modify` Befehl mit dem  `-firewall-policy` Parameter, um eine Richtlinie mit einem LIF zu verknüpfen.

====


 a| 
Ihre Webprotokoll-Engine ist möglicherweise deaktiviert.
 a| 
Stellen Sie sicher, dass die Webprotokoll-Engine aktiviert ist, damit Webdienste zugänglich sind.

[NOTE]
====
Sie verwenden die  `system services web` Befehle, um die Webprotokoll-Engine für den Cluster zu verwalten.

====


 a| 
Ihr Webbrowser gibt eine `not found` Fehlermeldung zurück, wenn Sie versuchen, auf einen Webdienst zuzugreifen.
 a| 
Der Webdienst ist möglicherweise deaktiviert.
 a| 
Stellen Sie sicher, dass jeder Webdienst, auf den Zugriff gewährt werden soll, einzeln aktiviert ist.

[NOTE]
====
Der  `vserver services web modify`Befehl wird verwendet, um einen Webdienst für den Zugriff zu aktivieren.

====


 a| 
Der Webbrowser kann sich mit dem Benutzernamen und dem Passwort nicht bei einem Webdienst anmelden.
 a| 
Der Benutzer kann nicht authentifiziert werden, das Zugriffsverfahren ist nicht korrekt oder der Benutzer ist nicht berechtigt, auf den Webdienst zuzugreifen.
 a| 
Stellen Sie sicher, dass das Benutzerkonto existiert und mit dem korrekten Zugriffsverfahren und Authentifizierungsverfahren konfiguriert ist. Außerdem muss die Rolle des Benutzers zum Zugriff auf den Webdienst berechtigt sein.

[NOTE]
====
Sie verwenden die  `security login` Befehle, um Benutzerkonten sowie deren Zugriffs- und Authentifizierungsverfahren zu verwalten. Für den Zugriff auf den ONTAP API-Webdienst ist die  `ontapi` Zugriffsmethode erforderlich. Für den Zugriff auf alle anderen Webdienste ist die  `http` Zugriffsmethode erforderlich. Sie verwenden die  `vserver services web access` Befehle, um den Zugriff einer Rolle auf einen Webdienst zu verwalten.

====


 a| 
Sie stellen über HTTPS eine Verbindung zu Ihrem Webdienst her und Ihr Webbrowser zeigt an, dass Ihre Verbindung unterbrochen ist.
 a| 
Möglicherweise ist SSL auf dem Cluster oder der Storage Virtual Machine (SVM), die den Webdienst bereitstellt, nicht aktiviert.
 a| 
Es muss sichergestellt sein, dass SSL auf dem Cluster oder der SVM aktiviert ist und dass das digitale Zertifikat gültig ist.

[NOTE]
====
Sie verwenden die  `security ssl` Befehle zur Verwaltung der SSL-Konfiguration für HTTP-Server und den  `security certificate show` Befehl zur Anzeige von Informationen zu digitalen Zertifikaten.

====


 a| 
Sie stellen eine Verbindung zu Ihrem Webdienst über HTTPS her, und Ihr Webbrowser zeigt an, dass die Verbindung nicht vertrauenswürdig ist.
 a| 
Möglicherweise verwenden Sie ein selbstsigniertes digitales Zertifikat.
 a| 
Es ist sicherzustellen, dass das dem Cluster oder der SVM zugeordnete digitale Zertifikat von einer vertrauenswürdigen Zertifizierungsstelle signiert ist.

[NOTE]
====
Sie verwenden den  `security certificate generate-csr` Befehl, um eine digitale Zertifikatsignierungsanforderung zu generieren, und den  `security certificate install` Befehl, um ein von einer Zertifizierungsstelle signiertes digitales Zertifikat zu installieren. Sie verwenden die  `security ssl` Befehle, um die SSL-Konfiguration für den Cluster oder die SVM zu verwalten, die den Webdienst bereitstellt.

====
