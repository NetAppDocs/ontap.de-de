= Zugriff auf die Systemkonsole eines ONTAP Knotens
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ein Knoten im Bootmenü oder an der Bootumgebungs-Eingabeaufforderung hängen bleibt, kann darauf nur über die Systemkonsole (auch als serielle Konsole bezeichnet) zugegriffen werden. Die Systemkonsole eines Knotens ist über eine SSH-Verbindung zum SP oder zum Cluster zugänglich.

.Über diese Aufgabe
Sowohl SP als auch ONTAP bieten Befehle, mit denen Sie auf die Systemkonsole zugreifen können. Vom SP aus ist jedoch nur die Systemkonsole des eigenen Knotens zugänglich. Vom Cluster aus kann auf die Systemkonsole jedes anderen Knotens im Cluster (außer dem lokalen Knoten) zugegriffen werden.

.Schritte
. Zugriff auf die Systemkonsole eines Knotens:
+
|===
| Wenn Sie sich in der... befinden | Geben Sie diesen Befehl ein... 


 a| 
SP CLI des Knotens
 a| 
`system console`



 a| 
ONTAP CLI
 a| 
`system node run-console`

|===
. Melden Sie sich an der Systemkonsole an, wenn Sie dazu aufgefordert werden.
. Zum Verlassen der Systemkonsole Strg+D drücken.


.Beispiele für den Zugriff auf die Systemkonsole
Das folgende Beispiel zeigt das Ergebnis der Eingabe des  `system console` Befehls an der Eingabeaufforderung „`SP node2`“. Die Systemkonsole zeigt an, dass node2 an der Bootumgebungs-Eingabeaufforderung hängt. Der  `boot_ontap` Befehl wird an der Konsole eingegeben, um den Node auf ONTAP zu booten. Anschließend wird Strg+D gedrückt, um die Konsole zu verlassen und zum SP zurückzukehren.

[listing]
----
SP node2> system console
Type Ctrl-D to exit.

LOADER>
LOADER> boot_ontap
...
*******************************
*                             *
* Press Ctrl-C for Boot Menu. *
*                             *
*******************************
...
----
(Strg+D wird gedrückt, um die Systemkonsole zu verlassen.)

[listing]
----

Connection to 123.12.123.12 closed.
SP node2>
----
Das folgende Beispiel zeigt das Ergebnis der Eingabe des  `system node run-console` Befehls aus ONTAP, um auf die Systemkonsole von node2 zuzugreifen, die an der Boot-Umgebungs-Eingabeaufforderung hängt. Der  `boot_ontap` Befehl wird an der Konsole eingegeben, um node2 in ONTAP zu starten. Anschließend wird Strg+D gedrückt, um die Konsole zu verlassen und zu ONTAP zurückzukehren.

[listing]
----
cluster1::> system node run-console -node node2
Pressing Ctrl-D will end this session and any further sessions you might open on top of this session.
Type Ctrl-D to exit.

LOADER>
LOADER> boot_ontap
...
*******************************
*                             *
* Press Ctrl-C for Boot Menu. *
*                             *
*******************************
...
----
(Strg+D wird gedrückt, um die Systemkonsole zu verlassen.)

[listing]
----

Connection to 123.12.123.12 closed.
cluster1::>
----
