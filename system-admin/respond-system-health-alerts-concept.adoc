= Informationen zur Reaktion auf Systemzustandswarnungen in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn eine Systemzustandswarnung auftritt, kann sie bestätigt werden, weitere Informationen dazu eingesehen, die zugrunde liegende Ursache behoben und ein erneutes Auftreten verhindert werden.

Wenn ein Gesundheitsmonitor eine Warnung ausgibt, ist eine Reaktion auf eine der folgenden Arten möglich:

* Informationen über die Warnung, einschließlich der betroffenen Ressource, des Schweregrads, der wahrscheinlichen Ursache, der möglichen Auswirkungen und der Korrekturmaßnahmen, sind verfügbar.
* Detaillierte Informationen zur Warnung, wie zum Beispiel der Zeitpunkt, zu dem die Warnung ausgelöst wurde, und ob sie bereits von jemand anderem bestätigt wurde, sind verfügbar.
* Gesundheitsbezogene Informationen über den Zustand der betroffenen Ressource oder des betroffenen Subsystems, wie zum Beispiel eines bestimmten Gehäuses oder einer bestimmten Festplatte, sind verfügbar.
* Die Benachrichtigung wird bestätigt, um anzuzeigen, dass jemand an dem Problem arbeitet, und um sich selbst als „Acknowledger“ zu identifizieren.
* Das Problem kann durch die in der Warnmeldung angegebenen Korrekturmaßnahmen gelöst werden, wie zum Beispiel das Beheben der Verkabelung, um ein Verbindungsproblem zu lösen.
* Die Benachrichtigung ist zu löschen, falls sie nicht automatisch vom System entfernt wurde.
* Eine Warnung kann unterdrückt werden, damit sie den Gesundheitsstatus eines Subsystems nicht beeinflusst.
+
Das Unterdrücken ist sinnvoll, wenn ein Problem verstanden wurde. Nach dem Unterdrücken einer Warnmeldung kann diese weiterhin auftreten, aber der Subsystemzustand wird als „ok-with-suppressed.“ angezeigt, wenn die unterdrückte Warnmeldung auftritt.
