= ONTAP SP/BMC-Firmware-Updates verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP enthält ein SP Firmware-Image, das als _Baseline-Image_ bezeichnet wird. Wenn eine neue Version der SP Firmware später verfügbar ist, besteht die Möglichkeit, diese herunterzuladen und die SP Firmware auf die heruntergeladene Version zu aktualisieren, ohne die ONTAP Version zu aktualisieren.

[NOTE]
====
Dieses Thema gilt sowohl für den SP als auch für den BMC.

====
ONTAP bietet die folgenden Methoden zur Verwaltung von SP Firmware-Updates:

* Die automatische Update-Funktionalität des SP ist standardmäßig aktiviert, wodurch die SP-Firmware in den folgenden Szenarien automatisch aktualisiert wird:
+
** Beim Upgrade auf eine neue Version von ONTAP
+
Der ONTAP Upgrade-Prozess umfasst automatisch das SP Firmware-Update, sofern die mit ONTAP gebündelte SP Firmware-Version neuer ist als die auf dem Knoten ausgeführte SP-Version.

+
[NOTE]
====
ONTAP erkennt ein fehlgeschlagenes automatisches SP-Update und leitet eine Korrekturmaßnahme ein, um das automatische SP-Update bis zu dreimal zu wiederholen. Wenn alle drei Wiederholungsversuche fehlschlagen, siehe link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/Health_Monitor_SPAutoUpgradeFailedMajorAlert__SP_upgrade_fails_-_AutoSupport_Message["NetApp Knowledge Base: Health Monitor SPAutoUpgradeFailedMajorAlert SP Upgrade fehlgeschlagen - AutoSupport Message"^].

====
** Wenn eine Version der SP Firmware von der NetApp Support-Website heruntergeladen wird und die heruntergeladene Version neuer ist als die Version, die aktuell auf dem SP ausgeführt wird
** Beim Downgrade oder der Rückkehr zu einer früheren Version von ONTAP
+
Die SP Firmware wird automatisch auf die neueste kompatible Version aktualisiert, die von der ONTAP Version unterstützt wird, auf die Sie zurückgesetzt oder herabgestuft wurden. Eine manuelle Aktualisierung der SP Firmware ist nicht erforderlich.



+
Sie haben die Möglichkeit, die automatische SP-Update-Funktion mithilfe des  `system service-processor image modify` Befehls zu deaktivieren. Es wird jedoch empfohlen, die Funktion aktiviert zu lassen. Das Deaktivieren der Funktion kann zu suboptimalen oder nicht qualifizierten Kombinationen zwischen dem ONTAP Image und dem SP Firmware-Image führen.

* ONTAP ermöglicht es Ihnen, ein SP Update manuell auszulösen und mithilfe des  `system service-processor image update` Befehls festzulegen, wie das Update durchgeführt werden soll.
+
Folgende Optionen können angegeben werden:

+
** Das zu verwendende SP-Firmware-Paket (`-package`)
+
Sie können die SP-Firmware auf ein heruntergeladenes Paket aktualisieren, indem Sie den Paketdateinamen angeben. Der  `system image package show`Befehl zeigt alle auf einem Knoten verfügbaren Paketdateien an (einschließlich der Dateien für das SP-Firmware-Paket).

** Ob das Basis SP Firmwarepaket für das SP Update verwendet werden soll (`-baseline`)
+
Sie können die SP Firmware auf die Basisversion aktualisieren, die mit der aktuell laufenden Version von ONTAP gebündelt ist.



+
[NOTE]
====
Wenn einige der erweiterten Aktualisierungsoptionen oder Parameter verwendet werden, können die Konfigurationseinstellungen des BMC vorübergehend gelöscht werden. Nach dem Neustart kann es bis zu 10 Minuten dauern, bis ONTAP die BMC Konfiguration wiederherstellt.

====
* ONTAP ermöglicht die Anzeige des Status des neuesten SP-Firmware-Updates, das von ONTAP mit dem  `system service-processor image update-progress show` Befehl ausgelöst wurde.


Jegliche bestehende Verbindung zum SP wird getrennt, wenn die SP Firmware aktualisiert wird. Dies ist sowohl bei einer automatisch als auch bei einer manuell ausgelösten SP Firmware-Aktualisierung der Fall.
