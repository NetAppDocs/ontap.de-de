= Die Positionsparameter der ONTAP CLI in der Befehlseingabe verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Funktionalität der Positionsparameter der ONTAP CLI nutzen, um die Effizienz der Befehlseingabe zu steigern. Es besteht die Möglichkeit, einen Befehl abzufragen, um die Positionsparameter dieses Befehls zu identifizieren.



[[what-a-positional-parameter-is]]
== Was ein Positionsparameter ist

* Ein Positionsparameter ist ein Parameter, bei dem der Parametername nicht vor der Angabe des Parameterwerts angegeben werden muss.
* Ein Positionsparameter kann im Befehlseingang mit nicht positionsbezogenen Parametern vermischt werden, solange er seine relative Reihenfolge mit anderen Positionsparametern im selben Befehl beachtet, wie in der  `*_command_name ?_*` Ausgabe angezeigt.
* Ein Positionsparameter kann ein erforderlicher oder optionaler Parameter für einen Befehl sein.
* Ein Parameter kann für einen Befehl positionell, für einen anderen jedoch nicht positionell sein.


[NOTE]
====
Die Verwendung der Positionsparameterfunktionalität in Skripten wird nicht empfohlen, insbesondere wenn die Positionsparameter für den Befehl optional sind oder optionale Parameter vor ihnen aufgeführt werden.

====


[[identify-a-positional-parameter]]
== Einen Positionsparameter identifizieren

Ein Positionsparameter kann in der  `*_command_name ?_*`-Befehlsausgabe identifiziert werden. Ein Positionsparameter ist durch eckige Klammern um den Parameternamen in einem der folgenden Formate gekennzeichnet:

* [`_-parameter_name_`] `_parameter_value_` zeigt einen erforderlichen Parameter an, der positionell ist.
* [[`_-parameter_name_`] `_parameter_value_`] zeigt einen optionalen Positionsparameter an.


Wenn der Parameter beispielsweise in der `*_command_name ?_*` Ausgabe wie folgt angezeigt wird, ist er positionsbezogen für den Befehl, in dem er vorkommt:

* `[-lif] <lif-name>`
* `[[-lif] <lif-name>]`


Wird der Parameter jedoch wie folgt angezeigt, ist er für den Befehl, in dem er vorkommt, nicht positionell:

* `-lif <lif-name>`
* `[-lif <lif-name>]`




[[examples-of-using-positional-parameters]]
== Beispiele für die Verwendung von Positionsparametern

Im folgenden Beispiel zeigt die `*_volume create ?_*` Ausgabe, dass drei Parameter für den Befehl positionell sind: `-volume`, `-aggregate` und `-size`.

[listing]
----
cluster1::> volume create ?
   -vserver <vserver name>                Vserver Name
   [-volume] <volume name>                Volume Name
   [-aggregate] <aggregate name>          Aggregate Name
  [[-size] {<integer>[KB|MB|GB|TB|PB]}]   Volume Size
  [ -state {online|restricted|offline|force-online|force-offline|mixed} ]
                                          Volume State (default: online)
  [ -type {RW|DP|DC} ]                    Volume Type (default: RW)
  [ -policy <text> ]                      Export Policy
  [ -user <user name> ]                   User ID
  ...
  [ -space-guarantee|-s {none|volume} ]   Space Guarantee Style (default: volume)
  [ -percent-snapshot-space <percent> ]   Space Reserved for Snapshot Copies
  ...
----
Im folgenden Beispiel wird der  `volume create` Befehl angegeben, ohne die Funktionalität der Positionsparameter zu nutzen:

`cluster1::> volume create -vserver svm1 -volume vol1 -aggregate *aggr1 -size 1g* -percent-snapshot-space 0`

Die folgenden Beispiele nutzen die Funktionalität der Positionsparameter, um die Effizienz der Befehlseingabe zu erhöhen. Die Positionsparameter sind im  `volume create` Befehl zwischen nichtpositionsbezogenen Parametern eingefügt, und die Werte der Positionsparameter werden ohne die Parameternamen angegeben. Die Positionsparameter werden in der gleichen Reihenfolge angegeben, wie sie in der  `*volume create ?*` Ausgabe angezeigt wird. Das bedeutet, dass der Wert für  `-volume` vor dem von  `-aggregate` angegeben wird, welcher wiederum vor dem von  `-size` angegeben wird.

`cluster1::> volume create *vol2* *aggr1* *1g* -vserver svm1 -percent-snapshot-space 0`

`cluster1::> volume create -vserver svm1 *vol3* -snapshot-policy default *aggr1* -nvfail off *1g* -space-guarantee none`
