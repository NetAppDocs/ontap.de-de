= ONTAP Jobs und Jobpläne verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Aufträge werden in eine Auftragswarteschlange gestellt und im Hintergrund ausgeführt, sobald Ressourcen verfügbar sind. Wenn ein Auftrag zu viele Clusterressourcen verbraucht, kann er gestoppt oder pausiert werden, bis die Auslastung des Clusters sinkt. Auch die Überwachung und das erneute Starten von Aufträgen ist möglich.



[[job-categories]]
== Stellenkategorien

Es gibt drei Kategorien von Jobs, die Sie verwalten können: server-affiliierte, cluster-affiliierte und private Jobs.

Ein Job kann in eine der folgenden Kategorien fallen:

* *Serverbezogene Jobs*
+
Diese Aufträge werden vom Management-Framework einem bestimmten Knoten zur Ausführung zugewiesen.

* *Cluster-zugeordnete Jobs*
+
Diese Aufträge werden vom Management-Framework in eine Warteschlange gestellt, um auf einem beliebigen Knoten im Cluster ausgeführt zu werden.

* *Private Aufträge*
+
Diese Jobs sind knotenspezifisch und nutzen weder die replizierte Datenbank (RDB) noch andere Clustermechanismen. Die Befehle zur Verwaltung privater Jobs erfordern die Berechtigungsstufe Advanced oder höher.





[[commands-for-managing-jobs]]
== Befehle zur Verwaltung von Jobs

Wenn Sie einen Befehl eingeben, der einen Job startet, informiert der Befehl Sie normalerweise darüber, dass der Job in die Warteschlange gestellt wurde, und kehrt dann zur CLI-Befehlszeile zurück. Einige Befehle zeigen jedoch stattdessen den Fortschritt des Jobs an und kehren erst zur CLI-Befehlszeile zurück, wenn der Job abgeschlossen ist. In diesen Fällen kann mit Strg+C der Job in den Hintergrund verschoben werden.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu allen Jobs
 a| 
`job show`



 a| 
Informationen zu Jobs auf Knotenbasis anzeigen
 a| 
`job show bynode`



 a| 
Informationen zu clusterbezogenen Jobs
 a| 
`job show-cluster`



 a| 
Informationen zu abgeschlossenen Aufträgen anzeigen
 a| 
`job show-completed`



 a| 
Informationen zur Auftragsverlaufshistorie anzeigen
 a| 
`job history show`

Für jeden Knoten im Cluster werden bis zu 25.000 Auftragsdatensätze gespeichert. Daher kann die Anzeige des vollständigen Auftragsverlaufs längere Zeit in Anspruch nehmen. Um potenziell lange Wartezeiten zu vermeiden, empfiehlt es sich, die Aufträge nach Knoten, Storage Virtual Machine (SVM) oder Datensatz-ID anzuzeigen.



 a| 
Die Liste der privaten Jobs anzeigen
 a| 
`job private show` (Erweiterte Berechtigungsstufe)



 a| 
Informationen zu abgeschlossenen privaten Aufträgen
 a| 
`job private show-completed` (Erweiterte Berechtigungsstufe)



 a| 
Informationen zum Initialisierungsstatus für Jobmanager
 a| 
`job initstate show` (Erweiterte Berechtigungsstufe)



 a| 
Den Fortschritt eines Auftrags überwachen
 a| 
`job watch-progress`



 a| 
Den Fortschritt eines privaten Jobs überwachen
 a| 
`job private watch-progress` (Erweiterte Berechtigungsstufe)



 a| 
Einen Job pausieren
 a| 
`job pause`



 a| 
Einen privaten Auftrag pausieren
 a| 
`job private pause` (Erweiterte Berechtigungsstufe)



 a| 
Einen pausierten Job wieder aufnehmen
 a| 
`job resume`



 a| 
Einen pausierten privaten Job wieder aufnehmen
 a| 
`job private resume` (Erweiterte Berechtigungsstufe)



 a| 
Einen Job stoppen
 a| 
`job stop`



 a| 
Einen privaten Job stoppen
 a| 
`job private stop` (Erweiterte Berechtigungsstufe)



 a| 
Einen Job löschen
 a| 
`job delete`



 a| 
Einen privaten Job löschen
 a| 
`job private delete` (Erweiterte Berechtigungsstufe)



 a| 
Einen einem Cluster zugeordneten Auftrag von einem nicht verfügbaren Knoten, dem er zugeordnet ist, trennen, sodass ein anderer Knoten die Zuständigkeit für diesen Auftrag übernehmen kann
 a| 
`job unclaim` (Erweiterte Berechtigungsstufe)

[[commands-for-managing-job-schedules]]
== Befehle zur Verwaltung von Arbeitsplänen

Viele Aufgaben – beispielsweise Volume-Snapshots – können so konfiguriert werden, dass sie nach festgelegten Zeitplänen ausgeführt werden. Zeitpläne, die zu bestimmten Zeiten ausgeführt werden, werden als _Cron-Schedules_ (ähnlich wie UNIX `cron`-Schedules) bezeichnet. Zeitpläne, die in Intervallen ausgeführt werden, werden als _Intervall-Schedules_ bezeichnet. Sie verwenden die `job schedule` Befehle, um Job-Schedules zu verwalten.

Jobpläne passen sich nicht an manuelle Änderungen von Datum und Uhrzeit des Clusters an. Diese Jobs werden basierend auf der aktuellen Clusterzeit geplant, die zum Zeitpunkt der Erstellung des Jobs oder der letzten Ausführung galt. Daher empfiehlt es sich, bei manuellen Änderungen von Datum oder Uhrzeit des Clusters die  `job show` und  `job history show` Befehle zu verwenden, um zu überprüfen, ob alle geplanten Jobs entsprechend Ihren Anforderungen in die Warteschlange gestellt und abgeschlossen wurden.

Wenn der Cluster Teil einer MetroCluster Konfiguration ist, müssen die Jobpläne auf beiden Clustern identisch sein. Daher ist bei Erstellung, Änderung oder Löschung eines Jobplans derselbe Vorgang auch auf dem Remote-Cluster durchzuführen.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu allen Zeitplänen anzeigen
 a| 
`job schedule show`



 a| 
Anzeige der Liste der Jobs nach Zeitplan
 a| 
`job schedule show-jobs`



 a| 
Informationen zu Cron-Schedules anzeigen
 a| 
`job schedule cron show`



 a| 
Informationen zu Intervallplänen
 a| 
`job schedule interval show`



 a| 
Einen Cron-Schedule erstellen
 a| 
`job schedule cron create`

Ab ONTAP 9.10.1 kann die SVM in den Jobplan aufgenommen werden.



 a| 
Intervallplan erstellen
 a| 
`job schedule interval create`

Mindestens einer der folgenden Parameter muss angegeben werden: `-days`, `-hours`, `-minutes` oder `-seconds`.



 a| 
Einen Cron-Schedule ändern
 a| 
`job schedule cron modify`



 a| 
Einen Intervallplan modifizieren
 a| 
`job schedule interval modify`



 a| 
Einen Zeitplan löschen
 a| 
`job schedule delete`



 a| 
Einen Cron-Schedule löschen
 a| 
`job schedule cron delete`



 a| 
Löschen eines Intervallplans
 a| 
`job schedule interval delete`

|===
