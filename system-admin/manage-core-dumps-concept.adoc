= Knoten-Core-Dumps für einen ONTAP Cluster verwalten (nur für Cluster-Administratoren)
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn ein Knoten abstürzt, tritt ein Core-Dump auf und das System erstellt eine Core-Dump-Datei, die der technische Support zur Fehlerbehebung verwenden kann. Es besteht die Möglichkeit, die Attribute des Core-Dumps zu konfigurieren oder anzuzeigen. Eine Core-Dump-Datei kann außerdem gespeichert, angezeigt, segmentiert, hochgeladen oder gelöscht werden.

Core Dumps können auf folgende Weise verwaltet werden:

* Konfiguration von Core Dumps und Anzeige der Konfigurationseinstellungen
* Anzeige von grundlegenden Informationen, dem Status und den Attributen von Core Dumps
+
Core-Dump-Dateien und Berichte werden im  `/mroot/etc/crash/` Verzeichnis eines Knotens gespeichert. Der Verzeichnisinhalt kann mit den  `system node coredump` Befehlen oder einem Webbrowser angezeigt werden.

* Speichern des Core-Dump-Inhalts und Hochladen der gespeicherten Datei an einen bestimmten Speicherort oder an den technischen Support
+
ONTAP verhindert das Initiieren des Speicherns einer Core-Dump-Datei während einer Übernahme, einer Aggregatverlagerung oder einer Rückgabe.

* Löschen nicht mehr benötigter Core Dump-Dateien




[[commands-for-managing-core-dumps]]
== Befehle zur Verwaltung von Core Dumps

Sie verwenden die  `system node coredump config` Befehle zur Verwaltung der Konfiguration von Core Dumps, die  `system node coredump` Befehle zur Verwaltung der Core Dump-Dateien und die  `system node coredump reports` Befehle zur Verwaltung von Application Core Reports.

Weitere Informationen zu den in diesem Thema beschriebenen Befehlen finden Sie in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Core Dumps konfigurieren
 a| 
`system node coredump config modify`



 a| 
Konfigurationseinstellungen für Core Dumps anzeigen
 a| 
`system node coredump config show`



 a| 
Grundlegende Informationen zu Core Dumps
 a| 
`system node coredump show`



 a| 
Manuelles Auslösen eines Core Dumps beim Neustart eines Knotens
 a| 
`system node reboot` mit sowohl dem  `-dump` als auch dem  `-skip-lif-migration-before-reboot` Parameter

[NOTE]
====
Der Parameter https://docs.netapp.com/us-en/ontap-cli//system-node-reboot.html#parameters[`skip-lif-migration-before-reboot`link:^] gibt an, dass die LIF-Migration vor einem Neustart übersprungen wird.

====


 a| 
Manuelles Auslösen eines Core Dump beim Herunterfahren eines Knotens
 a| 
`system node halt` mit sowohl dem  `-dump` als auch dem  `-skip-lif-migration-before-shutdown` Parameter

[NOTE]
====
Der Parameter https://docs.netapp.com/us-en/ontap-cli/system-node-halt.html#parameters[`skip-lif-migration-before-shutdown`^] gibt an, dass die LIF-Migration vor einem Herunterfahren übersprungen wird.

====


 a| 
Einen bestimmten Core Dump speichern
 a| 
`system node coredump save`



 a| 
Alle nicht gespeicherten Core-Dumps auf einem bestimmten Knoten speichern
 a| 
`system node coredump save-all`



 a| 
Eine AutoSupport-Nachricht mit einer von Ihnen angegebenen Core-Dump-Datei generieren und senden
 a| 
`system node autosupport invoke-core-upload`

[NOTE]
====
Der  `-uri`optionale Parameter gibt ein alternatives Ziel für die AutoSupport-Nachricht an.

====


 a| 
Statusinformationen zu Core Dumps anzeigen
 a| 
`system node coredump status`



 a| 
Löschen eines bestimmten Core Dumps
 a| 
`system node coredump delete`



 a| 
Alle ungespeicherten Core-Dumps oder alle gespeicherten Core-Dateien auf einem Knoten löschen
 a| 
`system node coredump delete-all`



 a| 
Anwendungs-Core-Dump-Berichte anzeigen
 a| 
`system node coredump reports show`



 a| 
Löschen eines Application Core Dump-Berichts
 a| 
`system node coredump reports delete`

|===
