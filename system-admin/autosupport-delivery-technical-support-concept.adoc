= Informationen darüber, wie ONTAP AutoSupport OnDemand Lieferanweisungen vom technischen Support erhält
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
AutoSupport OnDemand kommuniziert regelmäßig mit dem technischen Support, um Anweisungen zum Senden, erneuten Senden und Ablehnen von AutoSupport-Nachrichten sowie zum Hochladen großer Dateien auf die NetApp Support-Website zu erhalten. AutoSupport OnDemand ermöglicht das bedarfsgesteuerte Senden von AutoSupport-Nachrichten, anstatt auf die Ausführung des wöchentlichen AutoSupport-Auftrags zu warten.

AutoSupport OnDemand besteht aus folgenden Komponenten:

* AutoSupport OnDemand Client, der auf jedem Knoten ausgeführt wird
* AutoSupport OnDemand Service, der im technischen Support angesiedelt ist


Der AutoSupport OnDemand Client fragt den AutoSupport OnDemand Service regelmäßig ab, um Zustellungsanweisungen vom technischen Support zu erhalten. Beispielsweise kann der technische Support den AutoSupport OnDemand Service nutzen, um die Generierung einer neuen AutoSupport Nachricht anzufordern. Wenn der AutoSupport OnDemand Client den AutoSupport OnDemand Service abfragt, erhält der Client die Zustellungsanweisungen und sendet die neue AutoSupport Nachricht bedarfsgerecht, wie angefordert.

AutoSupport OnDemand ist standardmäßig aktiviert. Allerdings ist AutoSupport OnDemand auf einige AutoSupport-Einstellungen angewiesen, um weiterhin mit dem technischen Support zu kommunizieren. AutoSupport OnDemand kommuniziert automatisch mit dem technischen Support, wenn die folgenden Voraussetzungen erfüllt sind:

* AutoSupport ist aktiviert.
* AutoSupport ist so konfiguriert, dass Nachrichten an den technischen Support gesendet werden.
* AutoSupport ist für die Verwendung des HTTPS-Transportprotokolls konfiguriert.


Der AutoSupport OnDemand Client sendet HTTPS-Anfragen an denselben technischen Supportstandort, an den AutoSupport Nachrichten gesendet werden. Der AutoSupport OnDemand Client akzeptiert keine eingehenden Verbindungen.

[NOTE]
====
AutoSupport OnDemand verwendet das Benutzerkonto „autosupport“, um mit dem technischen Support zu kommunizieren. ONTAP verhindert, dass dieses Konto gelöscht wird.

====
Wenn AutoSupport OnDemand deaktiviert, aber AutoSupport weiterhin aktiviert bleiben soll, kann der Befehl  `system node autosupport modify -ondemand-state disable` verwendet werden. Weitere Informationen zu  `system node autosupport modify -ondemand-state disable` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-node-autosupport-modify.html#parameters["ONTAP-Befehlsreferenz"^].

Die Lieferanweisungen können Anforderungen für AutoSupport enthalten, Folgendes auszuführen:

* Neue AutoSupport-Nachrichten generieren.
+
Der technische Support kann neue AutoSupport-Nachrichten anfordern, um bei der Problemuntersuchung zu unterstützen.

* Neue AutoSupport-Meldungen generieren, die Core-Dump-Dateien oder Performance-Archivdateien auf die NetApp Support-Website hochladen.
+
Der technische Support kann zur Fehlerbehebung Core-Dump- oder Performance-Archivdateien anfordern.

* Zuvor generierte AutoSupport-Nachrichten erneut senden.
+
Diese Anfrage erfolgt automatisch, wenn eine Nachricht aufgrund eines Zustellungsfehlers nicht empfangen wurde.

* Die Zustellung von AutoSupport-Nachrichten für bestimmte Auslöseereignisse deaktivieren.
+
Der technische Support kann die Übermittlung nicht verwendeter Daten deaktivieren.
