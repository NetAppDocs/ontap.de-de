= Lizenzarten und Lizenzierungsverfahren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Das Verständnis der Lizenztypen und der lizenzierten Methode hilft Ihnen bei der Verwaltung der Lizenzen in einem Cluster.



[[license-types]]
== Lizenzarten

Ein Paket kann einen oder mehrere der folgenden Lizenztypen im Cluster installiert haben. Der  `system license show` Befehl zeigt den oder die installierten Lizenztypen für ein Paket an.

* Standardlizenz (`license`)
+
Eine Standardlizenz ist eine knotengebundene Lizenz. Sie wird für einen Knoten mit einer bestimmten Systemseriennummer (auch als _Controller-Seriennummer_ bezeichnet) ausgestellt. Eine Standardlizenz ist nur für den Knoten gültig, der die entsprechende Seriennummer besitzt.

+
Die Installation einer standardmäßigen, knotengebundenen Lizenz berechtigt einen Knoten zur Nutzung der lizenzierten Funktionalität. Damit der Cluster die lizenzierte Funktionalität nutzen kann, muss mindestens ein Knoten für die Funktionalität lizenziert sein. Es kann nicht konform sein, lizenzierte Funktionalität auf einem Knoten zu verwenden, der keine Berechtigung für die Funktionalität hat.

* Standortlizenz (`site`)
+
Eine Standortlizenz ist nicht an eine bestimmte Systemseriennummer gebunden. Wenn eine Standortlizenz installiert ist, sind alle Knoten im Cluster zur lizenzierten Funktionalität berechtigt. Der  `system license show` Befehl zeigt Standortlizenzen unter der Clusterseriennummer an.

+
Wenn ein Cluster über eine Standortlizenz verfügt und ein Knoten aus dem Cluster entfernt wird, nimmt der Knoten die Standortlizenz nicht mit und ist nicht mehr berechtigt, die lizenzierte Funktionalität zu nutzen. Wird einem Cluster mit Standortlizenz ein Knoten hinzugefügt, ist dieser Knoten automatisch zur durch die Standortlizenz gewährten Funktionalität berechtigt.

* Evaluierungslizenz (`demo`)
+
Eine Evaluierungslizenz ist eine temporäre Lizenz, die nach einem bestimmten Zeitraum abläuft (angegeben durch den  `system license show` Befehl). Damit kann bestimmte Softwarefunktionalität ausprobiert werden, ohne eine Berechtigung erwerben zu müssen. Es handelt sich um eine clusterweite Lizenz, die nicht an eine bestimmte Seriennummer eines Knotens gebunden ist.

+
Wenn Ihr Cluster über eine Evaluierungslizenz für ein Paket verfügt und Sie einen Knoten aus dem Cluster entfernen, nimmt der Knoten die Evaluierungslizenz nicht mit.





[[licensed-method]]
== Lizenziertes Verfahren

Es ist möglich, sowohl eine clusterweite Lizenz (die  `site` oder  `demo`-Typ) als auch eine knotengebundene Lizenz (der  `license`-Typ) für ein Paket zu installieren. Daher kann ein installiertes Paket mehrere Lizenztypen im Cluster haben. Für den Cluster gibt es jedoch nur eine _lizenzierte Methode_ für ein Paket. Das  `licensed method`-Feld des  `system license status show`-Befehls zeigt die Berechtigung an, die für ein Paket verwendet wird. Die Lizenzierungsmethode wird durch den Befehl wie folgt bestimmt:

* Wenn für ein Paket nur ein Lizenztyp im Cluster installiert ist, ist der installierte Lizenztyp die lizenzierte Methode.
* Wenn für ein Paket keine Lizenzen im Cluster installiert sind, ist die lizenzierte Methode  `none`.
* Wenn für ein Paket mehrere Lizenztypen im Cluster installiert sind, wird die lizenzierte Methode in der folgenden Prioritätsreihenfolge des Lizenztyps bestimmt: -`site`,  `license` und  `demo`.
+
Beispiel:

+
** Wenn Sie über eine Standortlizenz, eine Standardlizenz und eine Evaluierungslizenz für ein Paket verfügen, ist die lizenzierte Methode für das Paket im Cluster  `site`.
** Wenn Sie eine Standardlizenz und eine Evaluierungslizenz für ein Paket besitzen, ist die Lizenzierungsmethode für das Paket im Cluster  `license`.
** Wenn Sie nur eine Evaluierungslizenz für ein Paket besitzen, ist die Lizenzierungsmethode für das Paket im Cluster  `demo`.
