= Attribute eines ONTAP Knotens ändern
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Attribute eines Knotens nach Bedarf ändern. Zu den Attributen, die geändert werden können, gehören die Eigentümerinformationen des Knotens, Standortinformationen, Asset-Tag und die Berechtigung zur Teilnahme am Cluster.

.Über diese Aufgabe
Die Teilnahmeberechtigung eines Knotens am Cluster kann auf der Ebene erweiterter Berechtigungen mithilfe des  `–eligibility` Parameters des  `system node modify` oder  `cluster modify` Befehls geändert werden. Wenn die Teilnahmeberechtigung eines Knotens auf  `false` gesetzt wird, wird der Knoten im Cluster inaktiv.

[NOTE]
====
Die Knotenberechtigung kann nicht lokal geändert werden. Sie muss von einem anderen Knoten aus geändert werden. Die Knotenberechtigung kann auch nicht mit einer Cluster HA-Konfiguration geändert werden.

====
[NOTE]
====
Sie sollten die Berechtigung eines Knotens nicht auf  `false` setzen, außer in Situationen wie der Wiederherstellung der Knotenkonfiguration oder längerer Wartung des Knotens. Der SAN- und NAS-Datenzugriff auf den Knoten kann beeinträchtigt sein, wenn der Knoten nicht berechtigt ist.

====
.Schritte
. Mit dem  `system node modify` Befehl lassen sich die Attribute eines Knotens ändern.


.Beispiel für die Änderung von Knotenattributen
Der folgende Befehl ändert die Attribute des Knotens "`node1`". Der Besitzer des Knotens wird auf "`Joe Smith`" und das Asset-Tag auf "`js1234`" gesetzt:

[listing]
----
cluster1::> system node modify -node node1 -owner "Joe Smith" -assettag js1234
----
