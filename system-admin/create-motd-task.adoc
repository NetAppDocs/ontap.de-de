= Nachrichtentext für ONTAP Benutzer für den Message-of-the-Day erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es besteht die Möglichkeit, eine Nachricht des Tages (MOTD) zu erstellen, um Informationen an authentifizierte CLI-Benutzer weiterzugeben. Die MOTD wird in einer Konsolensitzung (nur für Clusterzugriff) oder einer SSH-Sitzung (für Cluster- oder SVM-Zugriff) angezeigt, nachdem ein Benutzer authentifiziert wurde, jedoch bevor die Clustershell-Eingabeaufforderung erscheint.

.Schritte
. Mit dem  `security login motd modify` Befehl lässt sich eine MOTD für den Cluster oder die SVM erstellen:
+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Eine Nachricht angeben, die eine einzelne Zeile ist
 a| 
Verwenden Sie den  `-message` „[.code]``text``“ Parameter, um den Text anzugeben.



 a| 
Zeilenumbrüche (auch bekannt als Zeilenenden) einbeziehen
 a| 
Der Befehl kann ohne den  `-message`- oder  `-uri`-Parameter verwendet werden, um den interaktiven Modus zum Bearbeiten der MOTD zu starten.



 a| 
Inhalte von einem Speicherort hochladen, um sie für die MOTD zu verwenden
 a| 
Der  `-uri`Parameter gibt den FTP- oder HTTP-Speicherort des Inhalts an.

|===
+
Die maximale Größe für eine MOTD beträgt 2.048 Bytes, einschließlich Zeilenumbrüchen.

+
`Security login motd modify`beschreibt die Escape-Sequenzen, die verwendet werden können, um die MOTD so zu konfigurieren, dass dynamisch generierte Inhalte angezeigt werden.

+
Eine mit dem  `-uri` Parameter erstellte MOTD ist statisch. Sie wird nicht automatisch aktualisiert, um nachfolgende Änderungen des Quellinhalts widerzuspiegeln.

+
Eine für den Cluster erstellte MOTD wird standardmäßig auch für alle SVM-Logins angezeigt, zusammen mit einer auf SVM-Ebene erstellten MOTD, die separat für eine bestimmte SVM erstellt werden kann. Das Setzen des  `-is-cluster-message-enabled`Parameters auf  `false`für eine SVM verhindert, dass die MOTD auf Cluster-Ebene für diese SVM angezeigt wird.

. Es kann überprüft werden, ob die MOTD erstellt wurde, indem sie mit dem  `security login motd show` Befehl angezeigt wird.
+
Wird der  `-message` Parameter mit einer leeren Zeichenkette (``) angegeben, werden MOTDs angezeigt, die nicht konfiguriert sind oder keinen Inhalt haben.

+
Weitere Informationen zu  `security login motd modify` und Parametern, mit denen das MOTD dynamisch generierte Inhalte im link:https://docs.netapp.com/us-en/ontap-cli/security-login-motd-modify.html["ONTAP-Befehlsreferenz"^] anzeigen kann.



.Beispiele für das Erstellen von MOTDs
Das folgende Beispiel verwendet den nicht-interaktiven Modus, um eine MOTD für das  `cluster1` Cluster zu erstellen:

[listing]
----
cluster1::> security login motd modify -message "Greetings!"
----
Das folgende Beispiel verwendet den interaktiven Modus, um eine MOTD für die  `svm1` SVM zu erstellen, die Escape-Sequenzen verwendet, um dynamisch generierte Inhalte anzuzeigen:

[listing]
----
cluster1::> security login motd modify -vserver svm1

Enter the message of the day for Vserver "svm1".
Max size: 2048. Enter a blank line to terminate input. Press Ctrl-C to abort.
0        1         2         3         4         5         6         7         8
12345678901234567890123456789012345678901234567890123456789012345678901234567890
Welcome to the \n SVM.  Your user ID is '\N'. Your last successful login was \L.
----
Das folgende Beispiel zeigt die erstellten MOTDs:

[listing]
----
cluster1::> security login motd show
Vserver: cluster1
Is the Cluster MOTD Displayed?: true
Message
-----------------------------------------------------------------------------
Greetings!

Vserver: svm1
Is the Cluster MOTD Displayed?: true
Message
-----------------------------------------------------------------------------
Welcome to the \n SVM.  Your user ID is '\N'. Your last successful login was \L.

2 entries were displayed.
----
Weitere Informationen zu  `security login motd show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-motd-show.html["ONTAP-Befehlsreferenz"^].
