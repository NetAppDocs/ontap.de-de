= ONTAP Befehlsverlauf anzeigen und jeden Befehl aus dem Verlauf erneut ausführen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Jede CLI-Sitzung speichert den Verlauf aller darin ausgeführten Befehle. Der Befehlsverlauf der aktuellen Sitzung kann angezeigt werden. Auch das erneute Ausführen von Befehlen ist möglich.

Um den Befehlsverlauf anzuzeigen, kann der  `history` Befehl verwendet werden.

Um einen Befehl erneut auszuführen, kann der  `redo` Befehl mit einem der folgenden Argumente verwendet werden:

* Eine Zeichenkette, die mit einem Teil eines vorherigen Befehls übereinstimmt
+
Wenn der einzige von Ihnen ausgeführte `volume` Befehl `volume show` war, kann der `redo volume` Befehl verwendet werden, um den Befehl erneut auszuführen.

* Die numerische ID eines vorherigen Befehls, wie sie vom  `history` Befehl aufgelistet wird
+
Beispielsweise kann der  `redo 4` Befehl verwendet werden, um den vierten Befehl in der Befehlsliste erneut auszugeben.

* Ein negativer Offset vom Ende der Verlaufsliste
+
Beispielsweise kann der  `redo -2`-Befehl verwendet werden, um den Befehl erneut auszuführen, der vor zwei Befehlen ausgeführt wurde.



Beispielsweise würde der folgende Befehl eingegeben, um den Befehl, der an dritter Stelle vom Ende der Befehlshistorie steht, zu wiederholen:

[listing]
----
cluster1::> redo -3
----
