= Befehle zur Verwaltung von ONTAP AutoSupport
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Mit den  `system node autosupport` Befehlen lassen sich die AutoSupport Konfiguration ändern oder anzeigen, Informationen über vorherige AutoSupport Nachrichten anzeigen sowie eine AutoSupport Nachricht senden, erneut senden oder abbrechen.



[[configure-autosupport]]
== Konfiguration von AutoSupport

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Steuern, ob AutoSupport-Nachrichten gesendet werden
 a| 
`system node autosupport modify` mit dem  `-state` Parameter



 a| 
Steuern, ob AutoSupport-Nachrichten an den technischen Support gesendet werden
 a| 
`system node autosupport modify` mit dem  `-support` Parameter



 a| 
Einrichten von AutoSupport oder Ändern der Konfiguration von AutoSupport
 a| 
`system node autosupport modify`



 a| 
Aktivieren und deaktivieren Sie AutoSupport-Nachrichten an Ihre interne Supportorganisation für einzelne Auslöseereignisse, und geben Sie zusätzliche Subsystemberichte an, die in Nachrichten aufgenommen werden, die als Reaktion auf einzelne Auslöseereignisse gesendet werden.
 a| 
`system node autosupport trigger modify`

|===


[[display-information-about-the-autosupport-configuration]]
== Informationen zur AutoSupport Konfiguration anzeigen

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Anzeigen der AutoSupport-Konfiguration
 a| 
`system node autosupport show` mit dem  `-node` Parameter



 a| 
Eine Zusammenfassung aller Adressen und URLs anzeigen, die AutoSupport-Nachrichten empfangen
 a| 
`system node autosupport destinations show`



 a| 
Anzeigen, welche AutoSupport-Meldungen bei einzelnen Auslöseereignissen an Ihre interne Supportorganisation gesendet werden
 a| 
`system node autosupport trigger show`



 a| 
Status der AutoSupport Konfiguration sowie der Zustellung an verschiedene Ziele anzeigen
 a| 
`system node autosupport check show`



 a| 
Detaillierter Status der AutoSupport Konfiguration sowie der Lieferung an verschiedene Zielorte
 a| 
`system node autosupport check show-details`

|===


[[display-information-about-past-autosupport-messages]]
== Informationen zu früheren AutoSupport-Nachrichten anzeigen

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu einer oder mehreren der 50 neuesten AutoSupport-Nachrichten anzeigen
 a| 
`system node autosupport history show`



 a| 
Informationen zu kürzlich generierten AutoSupport-Meldungen zum Hochladen von Core-Dump- oder Performance-Archivdateien auf die Website des technischen Supports oder eine bestimmte URI anzeigen
 a| 
`system node autosupport history show-upload-details`



 a| 
Die Informationen in den AutoSupport-Meldungen einschließlich Name und Größe jeder für die Meldung gesammelten Datei sowie etwaiger Fehler sind einsehbar.
 a| 
`system node autosupport manifest show`

|===


[[send-resend-or-cancel-autosupport-messages]]
== Senden, erneutes Senden oder Stornieren von AutoSupport-Nachrichten

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Eine lokal gespeicherte AutoSupport-Nachricht, identifiziert durch ihre AutoSupport-Sequenznummer, erneut übertragen


NOTE: Wenn eine AutoSupport-Nachricht erneut übertragen wird und der Support diese Nachricht bereits erhalten hat, erstellt das Supportsystem keinen doppelten Fall. Wurde die Nachricht hingegen vom Support nicht erhalten, analysiert das AutoSupport-System die Nachricht und erstellt bei Bedarf einen Fall.
 a| 
`system node autosupport history retransmit`



 a| 
Eine AutoSupport-Nachricht generieren und senden, beispielsweise zu Testzwecken
 a| 
`system node autosupport invoke`


NOTE: Verwenden Sie den  `-force` Parameter, um eine Nachricht auch dann zu senden, wenn AutoSupport deaktiviert ist. Verwenden Sie den  `-uri` Parameter, um die Nachricht an das von Ihnen angegebene Ziel statt an das konfigurierte Ziel zu senden.



 a| 
Eine AutoSupport-Nachricht abbrechen
 a| 
`system node autosupport history cancel`

|===
