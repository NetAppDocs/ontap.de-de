= Ein ONTAP Anmeldebanner erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es besteht die Möglichkeit, ein Banner zu erstellen, das eine Nachricht für Personen anzeigt, die versuchen, auf den Cluster oder die SVM zuzugreifen. Das Banner wird in einer Konsolensitzung (nur für den Clusterzugriff) oder einer SSH-Sitzung (für den Cluster- oder SVM-Zugriff) angezeigt, bevor ein Benutzer zur Authentifizierung aufgefordert wird.

.Schritte
. Mit dem  `security login banner modify` Befehl lässt sich ein Banner für den Cluster oder die SVM erstellen:
+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Eine Nachricht angeben, die eine einzelne Zeile ist
 a| 
Verwenden Sie den  `-message` „[.code]``<text>``“ Parameter, um den Text anzugeben.



 a| 
Zeilenumbrüche (auch bekannt als Zeilenenden) in die Nachricht einbeziehen
 a| 
Der Befehl kann ohne den  `-message`- oder  `-uri`-Parameter verwendet werden, um den interaktiven Modus zum Bearbeiten des Banners zu starten.



 a| 
Inhalte von einem Speicherort hochladen, um sie für das Banner zu verwenden
 a| 
Der  `-uri`Parameter gibt den FTP- oder HTTP-Speicherort des Inhalts an.

|===
+
Die maximale Größe für ein Banner beträgt 2.048 Bytes, einschließlich Zeilenumbrüchen.

+
Ein mit dem  `-uri` Parameter erstelltes Banner ist statisch. Sie wird nicht automatisch aktualisiert, um nachfolgende Änderungen des Quellinhalts widerzuspiegeln.

+
Das für den Cluster erstellte Banner wird auch für alle SVMs angezeigt, die kein vorhandenes Banner haben. Jedes nachfolgend erstellte Banner für eine SVM überschreibt das Banner auf Cluster-Ebene für diese SVM. Die Angabe des  `-message` Parameters mit einem Bindestrich in Anführungszeichen (`"-"` für die SVM setzt die SVM zurück, sodass das Banner auf Cluster-Ebene verwendet wird.

. Es kann überprüft werden, ob das Banner erstellt wurde, indem es mit dem  `security login banner show` Befehl angezeigt wird.
+
Die Angabe des  `-message` Parameters mit einer leeren Zeichenkette (`""` zeigt Banner ohne Inhalt an.

+
Durch Angabe des `-message` Parameters mit `"-"` werden alle (Admin- oder Daten-)SVMs angezeigt, für die kein Banner konfiguriert ist.



.Beispiele für die Erstellung von Bannern
Das folgende Beispiel verwendet den nicht-interaktiven Modus, um ein Banner für den Cluster "`cluster1`" zu erstellen:

[listing]
----
cluster1::> security login banner modify -message "Authorized users only!"

cluster1::>
----
Das folgende Beispiel verwendet den interaktiven Modus, um ein Banner für die  `svm1` SVM zu erstellen:

[listing]
----
cluster1::> security login banner modify -vserver svm1

Enter the message of the day for Vserver "svm1".
Max size: 2048. Enter a blank line to terminate input. Press Ctrl-C to abort.
0        1         2         3         4         5         6         7         8
12345678901234567890123456789012345678901234567890123456789012345678901234567890
The svm1 SVM is reserved for authorized users only!


cluster1::>
----
Das folgende Beispiel zeigt die erstellten Banner:

[listing]
----
cluster1::> security login banner show
Vserver: cluster1
Message
-----------------------------------------------------------------------------
Authorized users only!

Vserver: svm1
Message
-----------------------------------------------------------------------------
The svm1 SVM is reserved for authorized users only!

2 entries were displayed.

cluster1::>
----
