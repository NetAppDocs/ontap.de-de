= Einen ONTAP-Knoten über das Bootmenü verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Über das Bootmenü können Sie Konfigurationsprobleme auf einem Knoten beheben, das Administratorpasswort zurücksetzen, Festplatten initialisieren, die Knotenkonfiguration zurücksetzen und die Knotenkonfigurationsinformationen auf das Bootgerät wiederherstellen.


NOTE: Wenn ein HA-Paar  https://docs.netapp.com/us-en/ontap/encryption-at-rest/support-storage-encryption-concept.html["Verschlüsselung von SAS- oder NVMe-Laufwerken (SED, NSE, FIPS)"] verwendet wird, müssen Sie die Anweisungen im Thema  https://docs.netapp.com/us-en/ontap/encryption-at-rest/return-seds-unprotected-mode-task.html["Zurückstellen eines FIPS-Laufwerks oder SED in den ungeschützten Modus"] für alle Laufwerke innerhalb des HA-Paares befolgen, bevor Sie das System initialisieren (Boot-Optionen 4 oder 9). Andernfalls kann es in Zukunft zu Datenverlust kommen, wenn die Laufwerke wiederverwendet werden.

.Schritte
. Der Knoten kann neu gestartet werden, um über den  `system node reboot` Befehl an der Systemeingabeaufforderung auf das Boot-Menü zuzugreifen.
+
Der Knoten startet den Neustartvorgang.

. Während des Neustartvorgangs Strg+C drücken, um das Startmenü anzuzeigen, wenn dazu aufgefordert wird.
+
Der Knoten zeigt die folgenden Optionen für das Bootmenü an:

+
[listing]
----
(1) Normal Boot.
(2) Boot without /etc/rc.
(3) Change password.
(4) Clean configuration and initialize all disks.
(5) Maintenance mode boot.
(6) Update flash from backup config.
(7) Install new software first.
(8) Reboot node.
(9) Configure Advanced Drive Partitioning.
(10) Set onboard key management recovery secrets.
(11) Configure node for external key management.
Selection (1-11)?
----
+
[NOTE]
====
Boot-Menü Option (2) Booten ohne /etc/rc ist veraltet und hat keine Auswirkung auf das System.

====
. Eine der folgenden Optionen kann durch Eingabe der entsprechenden Nummer ausgewählt werden:
+
[cols="35,65"]
|===
| Zu... | Auswählen... 


 a| 
Der Knoten wird im normalen Modus weiter hochgefahren.
 a| 
1) Normaler Startvorgang



 a| 
Das Passwort des Knotens ändern, das gleichzeitig das Passwort des „admin“-Kontos ist
 a| 
3) Passwort ändern



 a| 
Die Festplatten des Knotens werden initialisiert und ein Root-Volume für den Knoten erstellt.
 a| 
4) Konfiguration bereinigen und alle Festplatten initialisieren

[NOTE]
====
Diese Menüoption löscht alle Daten auf den Festplatten des Knotens und setzt die Knotenkonfiguration auf die Werkseinstellungen zurück.

====
Wählen Sie diesen Menüpunkt nur aus, nachdem der Knoten aus einem Cluster entfernt wurde und nicht einem anderen Cluster beigetreten ist.

Bei einem Knoten mit internen oder externen Festplattengehäusen wird das Root-Volume auf den internen Festplatten initialisiert. Sind keine internen Festplattengehäuse vorhanden, wird das Root-Volume auf den externen Festplatten initialisiert.

Wenn der Knoten, den Sie initialisieren möchten, über Festplatten verfügt, die für Root-Datenpartitionierung partitioniert sind, müssen die Festplatten vor der Initialisierung des Knotens entpartitioniert werden, siehe *9) Configure Advanced Drive Partitioning* und link:../disks-aggregates/index.html["Festplatten- und Aggregatverwaltung"].



 a| 
Wartungsarbeiten an Aggregaten und Festplatten sowie detaillierte Informationen zu Aggregaten und Festplatten sind möglich.
 a| 
5) Start im Wartungsmodus

Sie beenden den Wartungsmodus mit dem  `halt` Befehl.



 a| 
Die Konfigurationsinformationen werden vom Root-Volume des Knotens auf das Boot-Gerät, wie eine PC CompactFlash Karte, wiederhergestellt.
 a| 
6) Flash aus der Backup-Konfiguration aktualisieren

ONTAP speichert einige Konfigurationsinformationen des Knotens auf dem Boot-Gerät. Beim Neustart des Knotens werden die Informationen auf dem Boot-Gerät automatisch auf das Root-Volume des Knotens gesichert. Wenn das Boot-Gerät beschädigt ist oder ersetzt werden muss, ist diese Menüoption zu verwenden, um die Konfigurationsinformationen vom Root-Volume des Knotens auf das Boot-Gerät wiederherzustellen.



 a| 
Neue Software auf dem Knoten installieren
 a| 
7) Zuerst die neue Software installieren

Falls die ONTAP Software auf dem Boot-Gerät keine Unterstützung für das Storage Array bietet, das Sie für das Root-Volume verwenden möchten, kann über diese Menüoption eine Version der Software bezogen werden, die Ihr Storage Array unterstützt, und diese auf dem Node installiert werden.

Diese Menüoption dient ausschließlich der Installation einer neueren Version der ONTAP Software auf einem Knoten, auf dem kein Root-Volume installiert ist. Diese Menüoption ist _nicht_ für das Upgrade von ONTAP zu verwenden.



 a| 
Den Knoten neu starten
 a| 
8) Neustart des Knotens



 a| 
Alle Festplatten entpartitionieren und deren Besitzerinformationen entfernen oder die Konfiguration bereinigen und das System mit ganzen oder partitionierten Festplatten initialisieren
 a| 
9) Erweiterte Laufwerkspartitionierung konfigurieren

Die Option „Erweiterte Laufwerkspartitionierung“ bietet zusätzliche Verwaltungsfunktionen für Datenträger, die für Root-Data- oder Root-Data-Data-Partitionierung konfiguriert sind. Die folgenden Optionen sind über Boot Option 9 verfügbar:

[listing]
----
(9a) Unpartition all disks and remove their ownership information.
(9b) Clean configuration and initialize system with partitioned disks.
(9c) Clean configuration and initialize system with whole disks.
(9d) Reboot the node.
(9e) Return to main boot menu.
----
|===
