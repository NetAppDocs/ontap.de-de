= ONTAP Befehle zur Verwaltung von Konfigurationssicherungszeitplänen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die  `system configuration backup settings` Befehle verwenden, um Sicherungspläne für die Konfiguration zu verwalten.

Diese Befehle sind auf der Zugangsstufe „Erweitert“ verfügbar.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Die Einstellungen für einen Konfigurationssicherungsplan ändern:

* Geben Sie eine Remote-URL (HTTP, HTTPS, FTP oder FTPS) an, unter der die Konfigurationssicherungsdateien zusätzlich zu den Standardspeicherorten im Cluster hochgeladen werden.
* Ein Benutzername, der für die Anmeldung an der Remote-URL verwendet wird, ist anzugeben.
* Die Anzahl der aufzubewahrenden Backups für jeden Konfigurations-Backup-Zeitplan festlegen

 a| 
`system configuration backup settings modify`

Wenn Sie HTTPS in der Remote-URL verwenden, kann die  `-validate-certification` Option zum Aktivieren oder Deaktivieren der digitalen Zertifikatsvalidierung verwendet werden. Die Zertifikatsvalidierung ist standardmäßig deaktiviert.

[NOTE]
====
Der Webserver, auf den Sie die Konfigurationssicherungsdatei hochladen, muss PUT-Operationen für HTTP und POST-Operationen für HTTPS aktiviert haben. Weitere Informationen enthält die Dokumentation Ihres Webservers.

====


 a| 
Das Passwort festlegen, das für die Anmeldung an der Remote-URL verwendet wird.
 a| 
`system configuration backup settings set-password`



 a| 
Die Einstellungen für den Konfigurationssicherungsplan
 a| 
`system configuration backup settings show`

[NOTE]
====
Sie legen den  `-instance` Parameter fest, um den Benutzernamen und die Anzahl der aufzubewahrenden Backups für jeden Zeitplan anzuzeigen.

====
|===
