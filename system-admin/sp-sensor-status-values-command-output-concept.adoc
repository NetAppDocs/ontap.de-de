= ONTAP SP Sensorstatuswerte in der Systemsensor-Befehlsausgabe
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Diskrete Sensoren haben keine Schwellenwerte. Ihre Messwerte, die unter der `Current` Spalte in der SP-CLI `system sensors`-Befehlsausgabe angezeigt werden, haben keine tatsächliche Bedeutung und werden daher vom SP ignoriert. Die `Status` Spalte in der `system sensors`-Befehlsausgabe zeigt die Statuswerte diskreter Sensoren im Hexadezimalformat an.

Beispiele für diskrete Sensoren sind Sensoren für Lüfterfehler, Netzteilfehler (PSU) und Systemfehler. Die genaue Liste der diskreten Sensoren hängt von der Plattform ab.

Sie können den SP CLI `system sensors get` `sensor_name` Befehl zur Unterstützung bei der Interpretation der Statuswerte der meisten diskreten Sensoren verwenden. Die folgenden Beispiele zeigen die Ergebnisse der Eingabe von `system sensors get` `sensor_name` für die diskreten Sensoren CPU0_Error und IO_Slot1_Present:

[listing]
----
SP node1> system sensors get CPU0_Error
Locating sensor record...
Sensor ID              : CPU0_Error (0x67)
 Entity ID             : 7.97
 Sensor Type (Discrete): Temperature
 States Asserted       : Digital State
                         [State Deasserted]
----
[listing]
----
SP node1> system sensors get IO_Slot1_Present
Locating sensor record...
Sensor ID              : IO_Slot1_Present (0x74)
 Entity ID             : 11.97
 Sensor Type (Discrete): Add-in Card
 States Asserted       : Availability State
                        [Device Present]
----
Obwohl der  `system sensors get` `sensor_name` Befehl die Statusinformationen für die meisten diskreten Sensoren anzeigt, liefert er keine Statusinformationen für die diskreten Sensoren System_FW_Status, System_Watchdog, PSU1_Input_Type und PSU2_Input_Type. Die folgenden Informationen dienen zur Interpretation der Statuswerte dieser Sensoren.



[[systemfwstatus]]
== System_FW_Status

Der Zustand des Sensors System_FW_Status wird in Form von `0xAABB` angezeigt. Die Informationen von `AA` und `BB` können kombiniert werden, um den Zustand des Sensors zu bestimmen.

`AA` kann einen der folgenden Werte annehmen:

|===
| Werte | Zustand des Sensors 


 a| 
01
 a| 
System Firmware-Fehler



 a| 
02
 a| 
System-Firmware hängt



 a| 
04
 a| 
Fortschritt der Systemfirmware

|===
`BB` kann einen der folgenden Werte annehmen:

|===
| Werte | Zustand des Sensors 


 a| 
00
 a| 
Die Systemsoftware wurde ordnungsgemäß heruntergefahren



 a| 
01
 a| 
Speicherinitialisierung läuft



 a| 
02
 a| 
NVMEM-Initialisierung läuft (wenn NVMEM vorhanden ist)



 a| 
04
 a| 
Wiederherstellen der Werte des Speichercontroller-Hubs (MCH) (wenn NVMEM vorhanden ist)



 a| 
05
 a| 
Benutzer hat Setup aufgerufen



 a| 
13
 a| 
Starten des Betriebssystems oder des LOADER



 a| 
1F
 a| 
BIOS wird gestartet



 a| 
20
 a| 
LOADER läuft



 a| 
21
 a| 
Der LOADER programmiert die primäre BIOS-Firmware. Das System darf nicht heruntergefahren werden.



 a| 
22
 a| 
Der LOADER programmiert die alternative BIOS-Firmware. Das System darf nicht heruntergefahren werden.



 a| 
2F
 a| 
ONTAP läuft



 a| 
60
 a| 
SP hat das System ausgeschaltet



 a| 
61
 a| 
SP hat das System eingeschaltet



 a| 
62
 a| 
SP hat das System zurückgesetzt



 a| 
63
 a| 
SP Watchdog Power Cycle



 a| 
64
 a| 
SP Watchdog Kaltreset

|===
Beispielsweise bedeutet der System_FW_Status Sensorstatus 0x042F „System-Firmware-Fortschritt (04), ONTAP läuft (2F)“.



[[systemwatchdog]]
== System_Watchdog

Der System_Watchdog Sensor kann einen der folgenden Zustände aufweisen:

* *0x0080*
+
Der Zustand dieses Sensors hat sich nicht geändert



|===
| Werte | Zustand des Sensors 


 a| 
0x0081
 a| 
Timer-Interrupt



 a| 
0x0180
 a| 
Zeitüberschreitung



 a| 
0x0280
 a| 
Hard Reset



 a| 
0x0480
 a| 
Strom abschalten



 a| 
0x0880
 a| 
Stromkreislauf

|===
Beispielsweise bedeutet der System_Watchdog Sensorstatus 0x0880, dass eine Watchdog Zeitüberschreitung auftritt und einen Systemneustart auslöst.



[[psu1inputtype-and-psu2inputtype]]
== PSU1_Input_Type und PSU2_Input_Type

Bei Gleichstromnetzteilen (DC) sind die Sensoren PSU1_Input_Type und PSU2_Input_Type nicht relevant. Bei Wechselstromnetzteilen (AC) können die Sensoren einen der folgenden Statuswerte annehmen:

|===
| Werte | Zustand des Sensors 


 a| 
0x01 xx
 a| 
220V Netzteiltyp



 a| 
0x02 xx
 a| 
110V Netzteiltyp

|===
Beispielsweise bedeutet der Sensorstatus PSU1_Input_Type 0x0280, dass der Sensor meldet, dass der Netzteiltyp 110V ist.
