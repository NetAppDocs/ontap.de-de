= Informationen zur Implementierung der ONTAP Revisionsprotokollierung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Managementaktivitäten, die im Revisionsprotokoll erfasst werden, sind in Standard-AutoSupport-Berichten enthalten, und bestimmte Protokollierungsaktivitäten sind in EMS-Meldungen enthalten. Das Revisionsprotokoll kann auch an von Ihnen festgelegte Ziele weitergeleitet werden, und Revisionsprotokolldateien lassen sich über die ONTAP CLI oder einen Webbrowser anzeigen.

Ab ONTAP 9.11.1 lässt sich der Inhalt des Revisionsprotokolls mit System Manager anzeigen.

Ab ONTAP 9.12.1 stellt ONTAP Manipulationswarnungen für Revisionsprotokolle bereit. ONTAP führt täglich einen Hintergrundprozess aus, um Manipulationen an audit.log-Dateien zu überprüfen und sendet eine EMS-Warnung, wenn veränderte oder manipulierte Protokolldateien gefunden werden.

Beginnend mit ONTAP 9.17.1 und mit ONTAP 9.16.1 P4 sowie späteren Patch-Releases der Version 9.16.1 link:audit-manage-cross-cluster-requests.html["Auch Fernverwaltungsaktivitäten, die von einem verbundenen Cluster mittels clusterübergreifender Operationen initiiert werden, können protokolliert werden."]. Diese Aktivitäten umfassen benutzergesteuerte und interne Vorgänge, die von einem anderen Cluster ausgehen.

.In ONTAP protokollierte Managementaktivitäten
ONTAP protokolliert Managementaktivitäten, die auf einem Cluster durchgeführt werden, wie zum Beispiel welche Anfrage gestellt wurde, welcher Benutzer die Anfrage ausgelöst hat, welche Zugriffsmethode der Benutzer verwendet hat und zu welchem Zeitpunkt die Anfrage gestellt wurde.

Managementaktivitäten können zu einer der folgenden Typen gehören:

* *SET-Anfragen*:
+
** Diese Anfragen beziehen sich typischerweise auf Befehle oder Operationen ohne Anzeige.
** Diese Anfragen werden beispielsweise gestellt, wenn Sie einen  `create`,  `modify` oder  `delete`-Befehl ausführen.
** SET-Anfragen werden standardmäßig protokolliert.


* *GET-Anfragen*:
+
** Diese Anfragen rufen Informationen ab und zeigen sie in der Managementoberfläche an.
** Diese Anfragen werden gestellt, wenn Sie beispielsweise einen  `show` Befehl ausführen.
** GET-Anfragen werden standardmäßig nicht protokolliert, aber es kann gesteuert werden, ob GET-Anfragen, die von der ONTAP CLI (`-cliget`), von der ONTAP API (`-ontapiget`) oder von der ONTAP REST API (`-httpget`) gesendet werden, in der Datei protokolliert werden.




.Aufzeichnung und Rotation des Revisionsprotokolls
ONTAP protokolliert Verwaltungsaktivitäten in der  `/mroot/etc/log/mlog/audit.log` Datei eines Knotens. Befehle aus den drei Shells für CLI-Befehle: der Clustershell, der Nodeshell und der nicht-interaktiven Systemshell sowie API-Befehle werden hier protokolliert. Interaktive Systemshell-Befehle werden nicht protokolliert. Revisionsprotokolle enthalten Zeitstempel, die anzeigen, ob alle Knoten in einem Cluster synchronisiert sind.

Die  `audit.log` Datei wird vom AutoSupport Tool an die angegebenen Empfänger gesendet. Der Inhalt kann auch sicher an von Ihnen festgelegte externe Ziele weitergeleitet werden, beispielsweise an einen Splunk oder einen syslog Server.

Die  `audit.log` Datei wird täglich rotiert. Die Rotation erfolgt auch, wenn sie 100 MB groß ist, und die vorherigen 48 Kopien bleiben erhalten (mit maximal insgesamt 49 Dateien). Wenn die Revisionsdatei ihre tägliche Rotation durchführt, wird keine EMS-Meldung generiert. Wenn die Revisionsdatei rotiert wird, weil die Dateigrößenbeschränkung überschritten wird, wird eine EMS-Meldung generiert.

Beim Aktivieren der GET-Überwachung empfiehlt es sich, die Protokollweiterleitung zu konfigurieren, um Datenverlust durch schnelle Protokollrotation zu vermeiden. Weitere Informationen enthält der folgende Knowledge Base Artikel: https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/Enabling_audit-log_forwarding["Aktivieren der Weiterleitung des Revisionsprotokolls"^].
