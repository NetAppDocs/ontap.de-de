= ONTAP Befehle zur Verwaltung der Webprotokoll-Engine
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie verwenden die  `system services web` Befehle zur Verwaltung der Webprotokoll-Engine. Sie verwenden die  `system services firewall policy create` und  `network interface modify` Befehle, um Webzugriffsanfragen das Passieren der Firewall zu ermöglichen.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Die Webprotokoll-Engine wird auf Clusterebene konfiguriert:

* Die Webprotokoll-Engine für den Cluster kann aktiviert oder deaktiviert werden.
* SSLv3 für den Cluster aktivieren oder deaktivieren
* FIPS 140-2-Konformität für sichere Webdienste (HTTPS) aktivieren oder deaktivieren

 a| 
`system services web modify`



 a| 
Die Konfiguration der Webprotokoll-Engine auf Clusterebene anzeigen, feststellen, ob die Webprotokolle im gesamten Cluster funktionsfähig sind, und anzeigen, ob die FIPS 140-2-Konformität aktiviert und online ist.
 a| 
`system services web show`



 a| 
Die Konfiguration der Webprotokoll-Engine auf Knotenebene sowie die Aktivität der Webdienstverarbeitung für die Knoten im Cluster aufzeigen
 a| 
`system services web node show`



 a| 
Eine Firewall-Richtlinie erstellen oder einer bestehenden Firewall-Richtlinie den HTTP- oder HTTPS-Protokolldienst hinzufügen, damit Webzugriffsanfragen durch die Firewall geleitet werden können
 a| 
`system services firewall policy create`

Das Setzen des  `-service` Parameters auf  `http` oder  `https` ermöglicht, dass Webzugriffsanfragen durch die Firewall geleitet werden.



 a| 
Einer Firewall-Richtlinie einem LIF zuordnen
 a| 
`network interface modify`

Der  `-firewall-policy`Parameter kann verwendet werden, um die Firewall-Richtlinie eines LIF zu ändern.
