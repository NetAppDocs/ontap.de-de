= Regeln zur Angabe von Werten in ONTAP CLI
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die meisten Befehle enthalten einen oder mehrere erforderliche oder optionale Parameter. Für viele Parameter muss ein Wert angegeben werden. Es gibt einige Regeln für die Angabe von Werten in der Befehlszeile.

* Ein Wert kann eine Zahl, ein boolescher Wert, eine Auswahl aus einer Liste vordefinierter Werte oder ein Text-String sein.
+
Einige Parameter akzeptieren eine durch Kommas getrennte Liste mit zwei oder mehr Werten. Durch Kommas getrennte Listen von Werten müssen nicht in Anführungszeichen (" ") stehen. Wann immer Text, ein Leerzeichen oder ein Abfragezeichen angegeben wird (sofern es sich nicht um eine Abfrage oder um Text handelt, der mit einem Kleiner-als- oder Größer-als-Zeichen beginnt), muss die Entität in Anführungszeichen gesetzt werden.

* Die CLI interpretiert ein Fragezeichen ((`?`) als Befehl zum Anzeigen von Hilfeinformationen für einen bestimmten Befehl.
* Bei einigen Texten, die Sie in der CLI eingeben, wie Befehlsnamen, Parameter und bestimmte Werte, wird nicht zwischen Groß- und Kleinschreibung unterschieden.
+
Wenn Sie beispielsweise Parameterwerte für die  `vserver cifs` Befehle eingeben, wird die Groß-/Kleinschreibung ignoriert. Die meisten Parameterwerte, wie etwa die Namen von Nodes, Storage Virtual Machines (SVMs), Aggregaten, Volumes und logischen Schnittstellen, sind jedoch case-sensitiv.

* Wenn der Wert eines Parameters gelöscht werden soll, der eine Text-String oder eine Liste erwartet, wird ein leeres Paar Anführungszeichen ("") oder ein Bindestrich ("-") angegeben.
* Das Hash-Zeichen (`#`, auch bekannt als Pfundzeichen, kennzeichnet einen Kommentar für eine Befehlszeileneingabe; falls es verwendet wird, sollte es nach dem letzten Parameter in einer Befehlszeile stehen.
+
Die CLI ignoriert den Text zwischen  `#` und dem Zeilenende.



Im folgenden Beispiel wird eine SVM mit einem Textkommentar erstellt. Anschließend wird die SVM so geändert, dass der Kommentar gelöscht wird:

[listing]
----
cluster1::> vserver create -vserver vs0 -subtype default -rootvolume root_vs0
-aggregate aggr1 -rootvolume-security-style unix -language C.UTF-8 -is-repository false -ipspace ipspaceA -comment "My SVM"
cluster1::> vserver modify -vserver vs0 -comment ""
----
Im folgenden Beispiel zeigt ein Befehlszeilenkommentar, der das  `#` Zeichen verwendet, an, was der Befehl bewirkt.

[listing]
----
cluster1::> security login create -vserver vs0 -user-or-group-name new-admin
-application ssh -authmethod password #This command creates a new user account
----
Weitere Informationen zu  `security login create` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/security-login-create.html["ONTAP-Befehlsreferenz"^].
