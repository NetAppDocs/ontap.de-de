= Knoten zu einem ONTAP Cluster hinzufügen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Nachdem ein Cluster erstellt wurde, kann es durch das Hinzufügen von Knoten erweitert werden. Es wird jeweils nur ein Knoten hinzugefügt.

.Bevor Sie beginnen
* Installieren und konfigurieren Sie Ihr neues Speichersystem, indem Sie die Hardwareanforderungen prüfen, Ihren Standort vorbereiten und die Hardwarekomponenten (Controller, Storage Shelves und Switches) installieren und verkabeln. Siehe die folgende ONTAP-Hardwaresystemdokumentation für Details:
+
** link:https://docs.netapp.com/us-en/ontap-systems/aff-aseries/index.html["AFF A-Serien-Systeme"^]
** link:https://docs.netapp.com/us-en/ontap-systems/aff-cseries/index.html["AFF C-Serien-Systeme"^]
** link:https://docs.netapp.com/us-en/ontap-systems/allsan-landing/index.html["ASA-Systeme"^]
** link:https://docs.netapp.com/us-en/ontap-systems/afx/index.html["AFX-Systeme"^]
** link:https://docs.netapp.com/us-en/ontap-systems/asa-r2/index.html["ASA R2-Systeme"^]
** link:https://docs.netapp.com/us-en/ontap-systems/fas/index.html["FAS-Systeme"^]


* Prüfen Sie die Verkabelungs- und Konfigurationsanforderungen Ihres Cluster-Switch-Modells für die Cluster-Ports Ihres neuen Speichersystems. Siehe die link:https://docs.netapp.com/us-en/ontap-systems-switches/index.html["Switch-Dokumentation für ONTAP Hardware-Systeme"^]
+
.. Wählen Sie die *Cluster Switches* oder *Shared Switches* in der Kachel *Unterstützte Switches* aus.
.. Wählen Sie die *Install*-Dokumentation für Ihr Switch-Modell aus.
.. Suchen Sie nach dem Thema *Review cabling and configuration considerations* und überprüfen Sie es für Ihr Speichersystem.


* Wenn Sie einem Cluster mit mehreren Knoten weitere Knoten hinzufügen, müssen alle vorhandenen Knoten im Cluster fehlerfrei sein (erkennbar an  `cluster show`). Weitere Informationen zu  `cluster show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-show.html["ONTAP-Befehlsreferenz"^].
* Wenn Sie einem Cluster ohne Switches mit zwei Knoten weitere Knoten hinzufügen, muss das Cluster ohne Switches mit zwei Knoten mithilfe eines von NetApp unterstützten Cluster-Switches in ein Cluster mit Switch-Anschluss umgewandelt werden.

* Wenn einem Einzelknotencluster ein zweiter Knoten hinzugefügt wird, muss der zweite Knoten installiert und das Clusternetzwerk konfiguriert sein.
* Wenn im Cluster die automatische SP-Konfiguration aktiviert ist, muss das für den SP angegebene Subnetz über verfügbare Ressourcen verfügen, damit der beitretende Knoten das angegebene Subnetz zur automatischen Konfiguration des SP verwenden kann.
* Folgende Informationen für das Node-Management-LIF des neuen Knotens müssen vorliegen:
+
** Port
** IP-Adresse
** Netzmaske
** Standardgateway




.Über diese Aufgabe
Die Anzahl der Knoten muss gerade sein, damit sie HA-Paare bilden können. Nachdem mit dem Hinzufügen eines Knotens zum Cluster begonnen wurde, muss der Vorgang abgeschlossen werden. Der Knoten muss Teil des Clusters sein, bevor ein weiterer Knoten hinzugefügt werden kann.

.Schritte
. Der Knoten, der dem Cluster hinzugefügt werden soll, wird eingeschaltet.
+
Der Knoten wird hochgefahren und der Knoteneinrichtungsassistent startet auf der Konsole.

+
[listing]
----
Welcome to node setup.

You can enter the following commands at any time:
  "help" or "?" - if you want to have a question clarified,
  "back" - if you want to change previously answered questions, and
  "exit" or "quit" - if you want to quit the setup wizard.
     Any changes you made before quitting will be saved.

To accept a default or omit a question, do not enter a value.

Enter the node management interface port [e0M]:
----
. Den Node Setup-Assistenten beenden: `exit`
+
Der Node Setup-Assistent wird beendet, und es erscheint eine Anmeldeaufforderung mit dem Hinweis, dass die Einrichtungsaufgaben nicht abgeschlossen wurden.

+
Weitere Informationen zu  `exit` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/exit.html["ONTAP-Befehlsreferenz"^].

. Melden Sie sich beim Administratorkonto mit dem  `admin`Benutzernamen an.
. Den Cluster Setup-Assistenten starten:
+
[listing]
----
::> cluster setup

Welcome to the cluster setup wizard.

You can enter the following commands at any time:
 "help" or "?" - if you want to have a question clarified,
 "back" - if you want to change previously answered questions, and
 "exit" or "quit" - if you want to quit the cluster setup wizard.
 Any changes you made before quitting will be saved.

You can return to cluster setup at any time by typing "cluster setup".
To accept a default or omit a question, do not enter a value....

Use your web browser to complete cluster setup by accessing
https://<node_mgmt_or_e0M_IP_address>

Otherwise, press Enter to complete cluster setup using the
command line interface:
----
+
[NOTE]
====
Weitere Informationen zum Einrichten eines Clusters mithilfe der Setup-GUI finden Sie in der link:../task_admin_add_nodes_to_cluster.html["Dokumentation zur Knotenverwaltung"]. Weitere Informationen zu  `cluster setup` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-setup.html["ONTAP-Befehlsreferenz"^].

====
. Drücken Sie die Eingabetaste, um diese Aufgabe über die Befehlszeilenschnittstelle (CLI) abzuschließen. Wenn Sie aufgefordert werden, einen neuen Cluster zu erstellen oder einem bestehenden beizutreten, geben Sie  `*join*` ein.
+
[listing]
----
Do you want to create a new cluster or join an existing cluster? {create, join}:
join
----
+
Wenn die auf dem neuen Knoten ausgeführte ONTAP-Version von der Version auf dem bestehenden Cluster abweicht, meldet das System einen `System checks Error: Cluster join operation cannot be performed at this time` Fehler. Dies ist das erwartete Verhalten. Zum Fortfahren wird der `cluster add-node -allow-mixed-version-join true -cluster-ips <IP address> -node-names <new_node_name>` Befehl auf einem vorhandenen Knoten im Cluster mit erweitertem Berechtigungsniveau ausgeführt.

. Folgen Sie den Anweisungen, um den Knoten einzurichten und ihn dem Cluster hinzuzufügen:
+
** Um den Standardwert für eine Eingabeaufforderung zu übernehmen, die Eingabetaste drücken.
** Zum Eingeben eines eigenen Werts für eine Eingabeaufforderung den Wert eingeben und anschließend die Eingabetaste drücken.


. Die vorhergehenden Schritte sind für jeden weiteren Knoten zu wiederholen, den Sie hinzufügen möchten.


.Nachdem Sie fertig sind
Nach dem Hinzufügen von Knoten zum Cluster sollte das Storage-Failover für jedes HA-Paar aktiviert werden.
