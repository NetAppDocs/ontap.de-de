= Zugriff auf das ONTAP SP/BMC eines Knotens von einem Administrationshost
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können sich von einem Administrationshost aus beim SP eines Knotens anmelden, um Knotenverwaltungsaufgaben remote durchzuführen.

.Bevor Sie beginnen
Folgende Bedingungen müssen erfüllt sein:

* Der Administrationshost, den Sie für den Zugriff auf den SP verwenden, muss SSHv2 unterstützen.
* Ihr Benutzerkonto muss bereits für den Zugriff auf den SP eingerichtet sein.
+
Um auf den SP zuzugreifen, muss Ihr Benutzerkonto mit dem  `-application` Parameter des  `security login create` Befehls auf  `service-processor` und dem  `-authmethod` Parameter auf  `password` erstellt worden sein.



[NOTE]
====
Diese Aufgabe gilt sowohl für den SP als auch für den BMC.

====
Wenn der SP für die Verwendung einer IPv4- oder IPv6-Adresse konfiguriert ist und fünf SSH-Anmeldeversuche von einem Host innerhalb von 10 Minuten fehlschlagen, weist der SP die SSH-Anmeldeanfragen zurück und setzt die Kommunikation mit der IP-Adresse des Hosts für 15 Minuten aus. Nach 15 Minuten wird die Kommunikation wieder aufgenommen, und ein erneuter Anmeldeversuch beim SP ist möglich.

ONTAP verhindert die Erstellung oder Verwendung von systemreservierten Namen (wie "`root`" und "`naroot`") für den Zugriff auf den Cluster oder den SP.

.Schritte
. Vom Administrationshost aus beim SP anmelden:
+
`*ssh username@SP_IP_address*`

. Wenn Sie dazu aufgefordert werden, das Passwort für  `username` einzugeben.
+
Die SP-Eingabeaufforderung erscheint, was anzeigt, dass Sie Zugriff auf die SP CLI haben.



.Beispiele für SP Zugriff von einem Administrationshost
Das folgende Beispiel zeigt, wie Sie sich beim SP mit einem Benutzerkonto  `joe` anmelden, das für den Zugriff auf den SP eingerichtet wurde.

[listing]
----
[admin_host]$ ssh joe@192.168.123.98
joe@192.168.123.98's password:
SP>
----
Die folgenden Beispiele zeigen, wie die globale IPv6-Adresse oder die vom Router angekündigte IPv6-Adresse verwendet werden kann, um sich beim SP auf einem Node anzumelden, auf dem SSH für IPv6 eingerichtet ist und der SP für IPv6 konfiguriert ist.

[listing]
----
[admin_host]$ ssh joe@fd22:8b1e:b255:202::1234
joe@fd22:8b1e:b255:202::1234's password:
SP>
----
[listing]
----
[admin_host]$ ssh joe@fd22:8b1e:b255:202:2a0:98ff:fe01:7d5b
joe@fd22:8b1e:b255:202:2a0:98ff:fe01:7d5b's password:
SP>
----
