= Zugriff auf ONTAP Webdienste verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Webdienst ist eine Anwendung, auf die Benutzer über HTTP oder HTTPS zugreifen können. Der Clusteradministrator kann die Webprotokoll-Engine einrichten, SSL konfigurieren, einen Webdienst aktivieren und Benutzern einer Rolle den Zugriff auf einen Webdienst ermöglichen.

Ab ONTAP 9.6 werden die folgenden Webdienste unterstützt:

* Service Processor-Infrastruktur (`spi`)
+
Dieser Dienst stellt die Protokoll-, Core-Dump- und MIB-Dateien eines Knotens für den HTTP- oder HTTPS-Zugriff über die Cluster-Management-LIF oder eine Knoten-Management-LIF zur Verfügung. Die Standardeinstellung ist `enabled`.

+
Bei einer Anfrage zum Zugriff auf die Protokolldateien oder Core-Dump-Dateien eines Knotens erstellt der `spi` Webdienst automatisch einen Mountpunkt von einem Knoten zum Root-Volume eines anderen Knotens, auf dem sich die Dateien befinden. Es ist nicht erforderlich, den Mountpunkt manuell zu erstellen.

* ONTAP APIs (`ontapi`)
+
Dieser Dienst ermöglicht das Ausführen von ONTAP-APIs, um administrative Funktionen mit einem Remote-Programm durchzuführen. Die Standardeinstellung ist `enabled`.

+
Dieser Dienst wird möglicherweise für einige externe Verwaltungstools benötigt. Wenn beispielsweise der System Manager verwendet wird, sollte dieser Dienst aktiviert bleiben.

* Data ONTAP Erkennung (`disco`)
+
Dieser Dienst ermöglicht es externen Verwaltungsanwendungen, den Cluster im Netzwerk zu erkennen. Die Standardeinstellung ist `enabled`.

* Support-Diagnose (`supdiag`)
+
Dieser Dienst steuert den Zugriff auf eine privilegierte Umgebung auf dem System, um die Problemanalyse und -behebung zu unterstützen. Die Standardeinstellung ist `disabled`. Dieser Dienst sollte nur auf Anweisung des technischen Supports aktiviert werden.

* System Manager (`sysmgr`)
+
Dieser Dienst steuert die Verfügbarkeit des System Manager, der in ONTAP enthalten ist. Die Standardeinstellung ist `enabled`. Dieser Dienst wird nur auf dem Cluster unterstützt.

* Firmware-Update Baseboard Management Controller (BMC) (`FW_BMC`)
+
Dieser Dienst ermöglicht Ihnen das Herunterladen von BMC-Firmware-Dateien. Die Standardeinstellung ist `enabled`.

* ONTAP Dokumentation (`docs`)
+
Dieser Dienst ermöglicht den Zugriff auf die ONTAP Dokumentation. Die Standardeinstellung ist `enabled`.

* ONTAP RESTful APIs (`docs_api`)
+
Dieser Dienst ermöglicht den Zugriff auf die ONTAP RESTful API-Dokumentation. Die Standardeinstellung ist `enabled`.

* Datei-Upload und Download (`fud`)
+
Dieser Dienst bietet das Hoch- und Herunterladen von Dateien. Die Standardeinstellung ist `enabled`.

* ONTAP Messaging (`ontapmsg`)
+
Dieser Dienst unterstützt eine Publish- und Subscribe-Schnittstelle, mit der Sie Ereignisse abonnieren können. Die Standardeinstellung ist `enabled`.

* ONTAP Portal (`portal`)
+
Dieser Dienst implementiert das Gateway in einen virtuellen Server. Die Standardeinstellung ist `enabled`.

* ONTAP RESTful-Schnittstelle (`rest`)
+
Dieser Dienst unterstützt eine RESTful-Schnittstelle, die zur Fernverwaltung aller Elemente der Clusterinfrastruktur verwendet wird. Die Standardeinstellung ist `enabled`.

* Support für Security Assertion Markup Language (SAML) Service Provider (`saml`)
+
Dieser Dienst stellt Ressourcen zur Unterstützung des SAML-Dienstanbieters bereit. Die Standardeinstellung ist `enabled`.

* SAML Service Provider (`saml-sp`)
+
Dieser Dienst bietet Dienste wie SP-Metadaten und den Assertion Consumer Service für den Service Provider an. Die Standardeinstellung ist `enabled`.



Ab ONTAP 9.7 werden die folgenden zusätzlichen Dienste unterstützt:

* Konfigurationssicherungsdateien (`backups`)
+
Dieser Dienst ermöglicht Ihnen das Herunterladen von Konfigurationssicherungsdateien. Die Standardeinstellung ist `enabled`.

* ONTAP Sicherheit (`security`)
+
Dieser Dienst unterstützt die Verwaltung von CSRF-Tokens für eine verbesserte Authentifizierung. Die Standardeinstellung ist `enabled`.
