= Zugriff auf das ONTAP SP/BMC eines Knotens von der Systemkonsole
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Auf den SP kann über die Systemkonsole (auch serielle Konsole genannt) zugegriffen werden, um Überwachungs- oder Fehlerbehebungsaufgaben durchzuführen.

.Über diese Aufgabe
Diese Aufgabe gilt sowohl für den SP als auch für den BMC.

.Schritte
. Auf die SP CLI kann über die Systemkonsole zugegriffen werden, indem an der Eingabeaufforderung Strg+G gedrückt wird.
. Melden Sie sich bei der SP CLI an, wenn Sie dazu aufgefordert werden.
+
Die SP-Eingabeaufforderung erscheint, was anzeigt, dass Sie Zugriff auf die SP CLI haben.

. Die SP CLI wird beendet und zur Systemkonsole zurückgekehrt, indem Strg+D gedrückt und anschließend die Eingabetaste betätigt wird.


.Beispiel für den Zugriff auf die SP CLI von der Systemkonsole aus
Das folgende Beispiel zeigt das Ergebnis des Drückens von Strg+G in der Systemkonsole, um auf die SP CLI zuzugreifen. Der  `help system power` Befehl wird an der SP Eingabeaufforderung eingegeben, gefolgt von Strg+D und anschließend der Eingabetaste, um zur Systemkonsole zurückzukehren.

[listing]
----
cluster1::>
----
(Drücken Sie Strg+G, um auf die SP CLI zuzugreifen.)

[listing]
----
Switching console to Service Processor
Service Processor Login:
Password:
SP>
SP> help system power
system power cycle - power the system off, then on
system power off - power the system off
system power on - power the system on
system power status - print system power status
SP>
----
(Drücken Sie Strg+D und dann die Eingabetaste, um zur Systemkonsole zurückzukehren.)
