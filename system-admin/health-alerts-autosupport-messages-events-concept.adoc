= Informationen zu den Auslösern für ONTAP AutoSupport Gesundheitswarnungen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Systemzustandswarnungen lösen AutoSupport-Meldungen und Ereignisse im Event Management System (EMS) aus, sodass der Zustand des Systems mithilfe von AutoSupport-Meldungen und dem EMS zusätzlich zur direkten Nutzung des Zustandsüberwachungssystems überwacht werden kann.

Ihr System sendet innerhalb von fünf Minuten nach einer Warnung eine AutoSupport-Meldung. Die AutoSupport-Meldung enthält alle Warnungen, die seit der vorherigen AutoSupport-Meldung generiert wurden, mit Ausnahme von Warnungen, die eine Warnung für dieselbe Ressource und wahrscheinliche Ursache innerhalb der Vorwoche duplizieren.

Einige Warnmeldungen lösen keine AutoSupport-Meldungen aus. Eine Warnmeldung löst keine AutoSupport-Meldung aus, wenn ihre Integritätsrichtlinie das Senden von AutoSupport-Meldungen deaktiviert. Beispielsweise kann eine Integritätsrichtlinie AutoSupport-Meldungen standardmäßig deaktivieren, da AutoSupport bereits beim Auftreten des Problems eine Meldung generiert. Richtlinien können so konfiguriert werden, dass sie keine AutoSupport-Meldungen auslösen, indem der  `system health policy definition modify` Befehl verwendet wird.

Sie können eine Liste aller durch Alarme ausgelösten AutoSupport-Nachrichten, die in der Vorwoche gesendet wurden, mit dem  `system health autosupport trigger history show` Befehl anzeigen.

Alarme lösen auch die Generierung von Ereignissen im EMS aus. Ein Ereignis wird jedes Mal generiert, wenn ein Alarm erstellt und jedes Mal, wenn ein Alarm gelöscht wird.
