= Automatische Netzwerkkonfiguration für ONTAP SP/BMC aktivieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Aktivierung der automatischen Netzwerkkonfiguration für den SP wird gegenüber der manuellen Konfiguration des SP Netzwerks bevorzugt. Da die automatische Netzwerkkonfiguration des SP clusterweit gilt, ist keine manuelle Verwaltung des SP Netzwerks für einzelne Knoten erforderlich.

[NOTE]
====
Diese Aufgabe gilt sowohl für den SP als auch für den BMC.

====
* Das Subnetz, das Sie für die SP automatische Netzwerkkonfiguration verwenden möchten, muss bereits im Cluster definiert sein und darf keine Ressourcenkonflikte mit der SP Netzwerkschnittstelle aufweisen.
+
Der  `network subnet show` Befehl zeigt Subnetzinformationen für den Cluster an.

+
Weitere Informationen zu  `network subnet show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-subnet-show.html["ONTAP-Befehlsreferenz"^].

+
Der Parameter, der die Subnetzzuordnung erzwingt (der  `-force-update-lif-associations` Parameter der  `network subnet` Befehle), wird nur auf Netzwerk-LIFs und nicht auf der SP Netzwerkschnittstelle unterstützt.

* Wenn IPv6-Verbindungen für den SP verwendet werden sollen, muss IPv6 für ONTAP bereits konfiguriert und aktiviert sein.
+
Der  `network options ipv6 show` Befehl zeigt den aktuellen Status der IPv6-Einstellungen für ONTAP an. Weitere Informationen zu  `network options ipv6 show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-options-ipv6-show.html["ONTAP-Befehlsreferenz"^].



.Schritte
. Geben Sie die IPv4- oder IPv6-Adressfamilie und den Namen des Subnetzes an, das der SP verwenden soll, indem Sie den  `system service-processor network auto-configuration enable` Befehl verwenden.
. Die automatische SP-Netzwerkkonfiguration kann mit dem  `system service-processor network auto-configuration show` Befehl angezeigt werden.
. Wenn anschließend die SP IPv4 oder IPv6 Netzwerkschnittstelle für alle Knoten, die sich im Quorum befinden, deaktiviert oder wieder aktiviert werden soll, ist der  `system service-processor network modify` Befehl mit den Parametern  `-address-family` [`IPv4`|`IPv6`] und  `-enable` [`true`|`false`] zu verwenden.
+
Wenn die automatische SP-Netzwerkkonfiguration aktiviert ist, kann die SP-IP-Adresse eines Knotens, der sich im Quorum befindet, nicht geändert werden. Es kann lediglich die SP-IPv4- oder SP-IPv6-Netzwerkschnittstelle aktiviert oder deaktiviert werden.

+
Befindet sich ein Knoten außerhalb des Quorums, kann die SP-Netzwerkkonfiguration des Knotens, einschließlich der SP-IP-Adresse, durch Ausführen von  `system service-processor network modify` auf dem Knoten geändert werden, wobei zu bestätigen ist, dass die automatische SP-Netzwerkkonfiguration für den Knoten überschrieben werden soll. Wenn der Knoten dem Quorum wieder beitritt, erfolgt die automatische SP-Neukonfiguration für den Knoten basierend auf dem angegebenen Subnetz.
