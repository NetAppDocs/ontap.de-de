= Einen Node mithilfe der ONTAP Node-Konfigurationssicherungsdatei wiederherstellen
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Node-Konfiguration wird mithilfe der Konfigurationssicherungsdatei wiederhergestellt, die identifiziert und dem wiederherzustellenden Node zur Verfügung gestellt wurde.

.Über diese Aufgabe
Diese Aufgabe sollte nur durchgeführt werden, um sich von einem Desaster zu erholen, das zum Verlust der lokalen Konfigurationsdateien des Knotens geführt hat.

.Schritte
. Wechsel zur erweiterten Berechtigungsstufe:
+
`set -privilege advanced`

. Wenn der Knoten fehlerfrei ist, kann auf der erweiterten Berechtigungsstufe eines anderen Knotens der  `cluster modify` Befehl mit den Parametern  `-node` und  `-eligibility` verwendet werden, um ihn als nicht berechtigt zu markieren und vom Cluster zu isolieren.
+
Wenn der Node nicht in Ordnung ist, sollte dieser Schritt übersprungen werden.

+
Dieses Beispiel ändert node2 so, dass es nicht mehr für die Teilnahme am Cluster infrage kommt, damit seine Konfiguration wiederhergestellt werden kann:

+
[listing]
----
cluster1::*> cluster modify -node node2 -eligibility false
----
+
Weitere Informationen zu  `cluster modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/cluster-modify.html["ONTAP-Befehlsreferenz"^].

. Der  `system configuration recovery node restore` Befehl auf der erweiterten Berechtigungsebene stellt die Konfiguration des Node aus einer Konfigurationssicherungsdatei wieder her.
+
Wenn der Knoten seine Identität, einschließlich seines Namens, verloren hat, sollte der  `-nodename-in-backup` Parameter verwendet werden, um den Node-Name in der Konfigurationssicherungsdatei anzugeben.

+
Dieses Beispiel stellt die Konfiguration des Knotens mithilfe einer der auf dem Knoten gespeicherten Konfigurationssicherungsdateien wieder her:

+
[listing]
----
cluster1::*> system configuration recovery node restore -backup cluster1.8hour.2011-02-22.18_15_00.7z

Warning: This command overwrites local configuration files with
         files contained in the specified backup file. Use this
         command only to recover from a disaster that resulted
         in the loss of the local configuration files.
         The node will reboot after restoring the local configuration.
Do you want to continue? {y|n}: y
----
+
Die Konfiguration wird wiederhergestellt, und der Node wird neu gestartet.

. Wenn der Node als nicht geeignet markiert wurde, kann der Node mit dem  `system configuration recovery cluster sync` Befehl als geeignet markiert und mit dem Cluster synchronisiert werden.
. In einer SAN-Umgebung kann der  `system node reboot` Befehl verwendet werden, um den Node neu zu starten und das SAN-Quorum wiederherzustellen.


.Nachdem Sie fertig sind
Wenn der Cluster zuvor neu erstellt wurde und die Node-Konfiguration mithilfe einer Konfigurationssicherungsdatei wiederhergestellt wird, die vor dieser Neuerstellung des Clusters erstellt wurde, muss der Cluster erneut erstellt werden.
