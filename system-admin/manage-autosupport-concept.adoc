= Informationen zu ONTAP AutoSupport
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
AutoSupport ist ein Mechanismus, der den Zustand Ihres Systems proaktiv überwacht und automatisch Nachrichten an NetApp technischen Support, Ihre interne Supportorganisation und einen Supportpartner sendet. Obwohl AutoSupport Nachrichten an den technischen Support standardmäßig aktiviert sind, müssen die richtigen Optionen festgelegt sein und ein gültiger Mail-Host vorhanden sein, damit Nachrichten an Ihre interne Supportorganisation gesendet werden.

Nur der Cluster-Administrator kann die AutoSupport-Verwaltung durchführen. Der Administrator der Storage Virtual Machine (SVM) hat keinen Zugriff auf AutoSupport.

AutoSupport ist standardmäßig aktiviert, wenn das Speichersystem zum ersten Mal konfiguriert wird. AutoSupport beginnt 24 Stunden nach der Aktivierung von AutoSupport damit, Nachrichten an den technischen Support zu senden. Der 24-Stunden-Zeitraum kann verkürzt werden, indem das System aktualisiert oder zurückgesetzt, die AutoSupport-Konfiguration geändert oder die Systemzeit auf einen anderen Zeitraum als 24 Stunden eingestellt wird.

[NOTE]
====
Sie können AutoSupport jederzeit deaktivieren, es sollte jedoch aktiviert bleiben. Die Aktivierung von AutoSupport kann die Problemermittlung und -behebung erheblich beschleunigen, falls ein Problem mit Ihrem Speichersystem auftritt. Standardmäßig sammelt das System AutoSupport-Informationen und speichert sie lokal, selbst wenn Sie AutoSupport deaktivieren.

====
Weitere Informationen zu AutoSupport sind auf der NetApp Support Site verfügbar.
