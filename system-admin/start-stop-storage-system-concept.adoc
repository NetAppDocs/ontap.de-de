= Einen ONTAP Knoten für Wartungs- oder Fehlerbehebungszwecke starten oder stoppen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Möglicherweise müssen Sie einen Knoten aus Wartungs- oder Fehlerbehebungsgründen starten oder stoppen. Dies ist über die ONTAP CLI, die Boot-Umgebungseingabeaufforderung oder die SP CLI möglich.

Die Verwendung des SP CLI-Befehls `system power off` oder `system power cycle` zum Ausschalten oder Neustarten eines Knotens kann zu einem unsachgemäßen Herunterfahren des Knotens (auch als _dirty shutdown_ bezeichnet) führen und ist kein Ersatz für ein ordnungsgemäßes Herunterfahren mit dem ONTAP `system node halt` Befehl.



[[reboot-a-node-at-the-system-prompt]]
== Einen Node an der Systemaufforderung neu starten

Ein Knoten kann im normalen Modus über die Systemeingabeaufforderung neu gestartet werden. Ein Knoten ist so konfiguriert, dass er vom Boot-Gerät, wie einer PC CompactFlash Karte, startet.

.Schritte
. Wenn der Cluster vier oder mehr Knoten enthält, ist zu überprüfen, dass der neu zu startende Knoten kein Epsilon hält:
+
.. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`*set -privilege advanced*`

.. Ermitteln Sie, welcher Knoten Epsilon enthält:
+
`*cluster show*`

+
Das folgende Beispiel zeigt, dass "`node1`" epsilon enthält:

+
[listing]
----
cluster1::*> cluster show
Node                 Health  Eligibility   Epsilon
-------------------- ------- ------------  ------------
node1                true    true          true
node2                true    true          false
node3                true    true          false
node4                true    true          false
4 entries were displayed.
----
.. Wenn der neu zu startende Knoten Epsilon enthält, ist Epsilon vom Knoten zu entfernen:
+
`*cluster modify -node _node_name_ -epsilon false*`

.. Epsilon einem anderen Knoten zuweisen, der aktiv bleibt:
+
`*cluster modify -node _node_name_ -epsilon true*`

.. Zurück zur Administrator-Privilegstufe:
+
`*set -privilege admin*`



. Mit dem  `system node reboot` Befehl wird der Node neu gestartet.
+
Wenn der  `-skip-lif-migration` Parameter nicht angegeben wird, versucht der Befehl, Daten- und Cluster-Management-LIFs vor dem Neustart synchron auf einen anderen Knoten zu migrieren. Wenn die LIF-Migration fehlschlägt oder eine Zeitüberschreitung auftritt, wird der Neustartvorgang abgebrochen und ONTAP zeigt eine Fehlermeldung an, die auf den Fehler bei der LIF-Migration hinweist.

+
[listing]
----
cluster1::> system node reboot -node node1 -reason "software upgrade"
----
+
Der Knoten startet den Neustartvorgang. Die ONTAP Anmeldeaufforderung erscheint und zeigt damit an, dass der Neustartvorgang abgeschlossen ist.





[[boot-ontap-at-the-boot-environment-prompt]]
== ONTAP an der Eingabeaufforderung der Startumgebung starten

Die aktuelle Version oder die Backup-Version von ONTAP kann gestartet werden, wenn Sie sich an der Boot-Umgebungs-Eingabeaufforderung eines Knotens befinden.

.Schritte
. Auf die Bootumgebungs-Eingabeaufforderung kann von der Storage-System-Konsole aus mit dem  `system node halt` Befehl zugegriffen werden.
+
Die Storage-System-Konsole zeigt die Eingabeaufforderung für die Startumgebung an.

. An der Eingabeaufforderung der Bootumgebung einen der folgenden Befehle eingeben:
+
|===
| Und obendrein ... | Eingeben... 


 a| 
Die aktuelle Version von ONTAP
 a| 
`boot_ontap`



 a| 
Das primäre ONTAP Image vom Boot-Gerät
 a| 
`boot_primary`



 a| 
Das ONTAP Backup-Image vom Boot-Gerät
 a| 
`boot_backup`

|===
+
Wenn Sie sich nicht sicher sind, welches Image Sie verwenden sollen, sollte  `boot_ontap` in der ersten Instanz verwendet werden.





[[shut-down-a-node]]
== Einen Knoten herunterfahren

Sie können einen Knoten herunterfahren, wenn er nicht mehr reagiert oder wenn das Supportpersonal Sie im Rahmen der Fehlerbehebung dazu auffordert.

.Schritte
. Wenn der Cluster vier oder mehr Knoten enthält, ist zu überprüfen, ob der Knoten, der heruntergefahren werden soll, kein Epsilon hält:
+
.. Stellen Sie die Berechtigungsstufe auf „Erweitert“ ein:
+
`*set -privilege advanced*`

.. Ermitteln Sie, welcher Knoten Epsilon enthält:
+
`*cluster show*`

+
Das folgende Beispiel zeigt, dass "`node1`" epsilon enthält:

+
[listing]
----
cluster1::*> cluster show
Node                 Health  Eligibility   Epsilon
-------------------- ------- ------------  ------------
node1                true    true          true
node2                true    true          false
node3                true    true          false
node4                true    true          false
4 entries were displayed.
----
.. Wenn der abzuschaltende Knoten Epsilon enthält, dann Epsilon aus dem Knoten entfernen:
+
`*cluster modify -node _node_name_ -epsilon false*`

.. Epsilon einem anderen Knoten zuweisen, der aktiv bleibt:
+
`*cluster modify -node _node_name_ -epsilon true*`

.. Zurück zur Administrator-Privilegstufe:
+
`*set -privilege admin*`



. Der  `system node halt` Befehl dient zum Herunterfahren des Knotens.
+
Wenn der  `-skip-lif-migration` Parameter nicht angegeben wird, versucht der Befehl, Daten- und Cluster-Management-LIFs vor dem Herunterfahren synchron auf einen anderen Knoten zu migrieren. Wenn die LIF-Migration fehlschlägt oder ein Timeout auftritt, wird der Herunterfahrvorgang abgebrochen und ONTAP zeigt eine Fehlermeldung an, die auf den Fehler bei der LIF-Migration hinweist.

+
Sie können beim Herunterfahren manuell einen Core-Dump auslösen, indem Sie den  `-dump` Parameter verwenden.

+
Das folgende Beispiel fährt den Knoten mit dem Namen "`node1`" für Hardwarewartungsarbeiten herunter:

+
[listing]
----
cluster1::> system node halt -node node1 -reason 'hardware maintenance'
----
