= In AutoSupport-Nachrichten gesendete Protokolldateien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
AutoSupport-Meldungen können mehrere wichtige Protokolldateien enthalten, die es dem technischen Supportpersonal ermöglichen, die jüngsten Systemaktivitäten zu überprüfen.

Alle Arten von AutoSupport-Meldungen können die folgenden Protokolldateien enthalten, wenn das Subsystem Protokolldateien aktiviert ist:

|===
| Protokolldatei | Datenmenge, die aus der Datei enthalten ist 


 a| 
* Protokolldateien aus dem  `/mroot/etc/log/mlog/` Verzeichnis
* Die MESSAGES-Protokolldatei

 a| 
Es werden nur neue Zeilen, die seit der letzten AutoSupport-Nachricht bis zu einem festgelegten Maximum zu den Protokollen hinzugefügt wurden, aufgenommen. Dadurch ist gewährleistet, dass AutoSupport-Nachrichten eindeutige, relevante (nicht überlappende) Daten enthalten.

(Protokolldateien von Partnern bilden die Ausnahme; für Partner sind die maximal zulässigen Daten enthalten.)



 a| 
* Protokolldateien aus dem  `/mroot/etc/log/shelflog/` Verzeichnis
* Protokolldateien aus dem  `/mroot/etc/log/acp/` Verzeichnis
* Protokolldaten des Event Management System (EMS)

 a| 
Die aktuellsten Datenzeilen bis zu einem festgelegten Maximum.

|===
Der Inhalt von AutoSupport-Nachrichten kann sich zwischen verschiedenen Versionen von ONTAP ändern.
