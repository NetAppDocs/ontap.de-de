= Befehle zur Verwaltung von Mountpunkten auf ONTAP Nodes
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der `spi` Webdienst erstellt automatisch einen Mountpunkt von einem Knoten zum Root-Volume eines anderen Knotens, sobald eine Anfrage zum Zugriff auf die Logdateien oder Core-Dateien eines Knotens eingeht. Obwohl Sie Mountpunkte nicht manuell verwalten müssen, kann dies mithilfe der `system node root-mount` Befehle erfolgen.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Einen Mountpunkt manuell von einem Knoten zum Root-Volume eines anderen Knotens erstellen
 a| 
`system node root-mount create`Von einem Knoten zu einem anderen kann nur ein einziger Mountpunkt existieren.



 a| 
Vorhandene Mountpunkte auf den Knoten im Cluster werden angezeigt, einschließlich des Zeitpunkts ihrer Erstellung und ihres aktuellen Status.
 a| 
`system node root-mount show`



 a| 
Einen Mountpunkt von einem Knoten zum Root-Volume eines anderen Knotens löschen und Verbindungen zu diesem Mountpunkt zwangsweise schließen
 a| 
`system node root-mount delete`

|===
