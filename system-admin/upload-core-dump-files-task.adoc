= ONTAP AutoSupport Core-Dump-Dateien hochladen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Beim Speichern einer Core-Dump-Datei wird eine Ereignismeldung generiert. Wenn der AutoSupport-Dienst aktiviert und so konfiguriert ist, dass er Meldungen an den NetApp Support sendet, wird eine AutoSupport-Meldung übermittelt und eine automatische Bestätigungs-E-Mail wird an Sie gesendet.

.Bevor Sie beginnen
* Sie haben AutoSupport mit den folgenden Einstellungen eingerichtet:
+
** AutoSupport ist auf dem Knoten aktiviert.
** AutoSupport ist so konfiguriert, dass Nachrichten an den technischen Support gesendet werden.
** AutoSupport ist für die Verwendung des HTTPS-Transportprotokolls konfiguriert.
+
Das SMTP-Transportprotokoll wird beim Senden von Nachrichten, die große Dateien wie Core-Dump-Dateien enthalten, nicht unterstützt.





.Über diese Aufgabe
Sie können die Core-Dump-Datei auch über den AutoSupport-Service über HTTPS mit dem  `system node autosupport invoke-core-upload` Befehl hochladen, falls dies vom NetApp Support angefordert wird.

https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/How_to_upload_an_ONTAP_9_core_file_for_analysis["Hochladen einer ONTAP 9 Core-Datei zur Analyse"^]

.Schritte
. Die Core-Dump-Dateien eines Knotens lassen sich mit dem  `system node coredump show` Befehl anzeigen.
+
Im folgenden Beispiel werden Core Dump-Dateien für den lokalen Knoten angezeigt:

+
[listing]
----
cluster1::> system node coredump show -node local
Node:Type Core Name Saved Panic Time
--------- ------------------------------------------- ----- -----------------
node:kernel
core.4073000068.2013-09-11.15_05_01.nz true 9/11/2013 15:05:01
----
. Eine AutoSupport-Nachricht generieren und eine Core-Dump-Datei mit dem  `system node autosupport invoke-core-upload` Befehl hochladen.
+
Im folgenden Beispiel wird eine AutoSupport-Nachricht generiert und an den Standardort, den technischen Support, gesendet, und die Core-Dump-Datei wird an den Standardort, die NetApp Support-Website, hochgeladen:

+
[listing]
----
cluster1::> system node autosupport invoke-core-upload -core-filename
core.4073000068.2013-09-11.15_05_01.nz -node local
----
+
Im folgenden Beispiel wird eine AutoSupport-Nachricht generiert und an den im URI angegebenen Speicherort gesendet, und die Core-Dump-Datei wird an den URI hochgeladen:

+
[listing]
----
cluster1::> system node autosupport invoke-core-upload -uri https//files.company.com -core-filename
core.4073000068.2013-09-11.15_05_01.nz -node local
----
