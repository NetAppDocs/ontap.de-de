= Eine ONTAP CLI Sitzung aufzeichnen und die aufgezeichneten Sitzungen verwalten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können eine CLI-Sitzung in einer Datei mit einem festgelegten Namen und einer festgelegten Größenbegrenzung aufzeichnen und die Datei anschließend an ein FTP- oder HTTP-Ziel hochladen. Es besteht außerdem die Möglichkeit, Dateien anzuzeigen oder zu löschen, in denen zuvor CLI-Sitzungen aufgezeichnet wurden.



[[record-a-cli-session]]
== Eine CLI-Sitzung aufzeichnen

Ein Protokoll einer CLI-Sitzung endet, wenn die Aufzeichnung gestoppt oder die CLI-Sitzung beendet wird oder wenn die Datei die festgelegte Größenbeschränkung erreicht. Die standardmäßige Dateigrößenbeschränkung beträgt 1 MB. Die maximale Dateigrößenbeschränkung beträgt 2 GB.

Das Aufzeichnen einer CLI-Sitzung ist beispielsweise dann nützlich, wenn Sie ein Problem beheben und detaillierte Informationen speichern möchten oder wenn Sie eine dauerhafte Aufzeichnung der Speicherplatznutzung zu einem bestimmten Zeitpunkt erstellen möchten.

.Schritte
. Die aktuelle CLI-Sitzung in einer Datei aufzeichnen:
+
[source, cli]
----
system script start
----
+
Weitere Informationen zu  `system script start` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-script-start.html["ONTAP-Befehlsreferenz"^].

+
ONTAP beginnt mit der Aufzeichnung Ihrer CLI-Sitzung in der angegebenen Datei.

. Fahren Sie mit Ihrer CLI-Sitzung fort.
. Nach Abschluss die Aufzeichnung der Sitzung beenden:
+
[source, cli]
----
system script stop
----
+
Weitere Informationen zu  `system script stop` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/system-script-stop.html["ONTAP-Befehlsreferenz"^].

+
ONTAP beendet die Protokollierung Ihrer CLI-Sitzung.





[[commands-for-managing-records-of-cli-sessions]]
== Befehle zur Verwaltung von Protokollen von CLI-Sitzungen

Sie verwenden die  `system script` Befehle, um Aufzeichnungen von CLI-Sitzungen zu verwalten.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Die aktuelle CLI-Sitzung in einer angegebenen Datei aufzeichnen.
 a| 
`system script start`



 a| 
Die Aufzeichnung der aktuellen CLI-Sitzung wird gestoppt
 a| 
`system script stop`



 a| 
Informationen zu Protokollen von CLI-Sitzungen
 a| 
`system script show`



 a| 
Einen Protokolleintrag einer CLI-Sitzung an ein FTP- oder HTTP-Ziel hochladen
 a| 
`system script upload`



 a| 
Einen Datensatz einer CLI-Sitzung löschen
 a| 
`system script delete`

|===
.Verwandte Informationen
link:../concepts/manual-pages.html["ONTAP-Befehlsreferenz"]



[[commands-for-managing-the-automatic-timeout-period-of-cli-sessions]]
== Befehle zur Verwaltung des automatischen Zeitüberschreitungszeitraums von CLI-Sitzungen

Der Zeitüberschreitungswert gibt an, wie lange eine CLI-Sitzung inaktiv bleibt, bevor sie automatisch beendet wird. Der CLI-Zeitüberschreitungswert gilt clusterweit. Das heißt, jeder Node in einem Cluster verwendet denselben CLI-Zeitüberschreitungswert.

Standardmäßig beträgt die automatische Zeitüberschreitung für CLI-Sitzungen 30 Minuten.

Die  `system timeout` Befehle werden verwendet, um die automatische Zeitüberschreitungsperiode von CLI-Sitzungen zu verwalten.

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Automatische Zeitüberschreitungsperiode für CLI-Sitzungen
 a| 
`system timeout show`



 a| 
Die automatische Zeitüberschreitungsperiode für CLI-Sitzungen anpassen
 a| 
`system timeout modify`

|===
.Verwandte Informationen
link:../concepts/manual-pages.html["ONTAP-Befehlsreferenz"]
