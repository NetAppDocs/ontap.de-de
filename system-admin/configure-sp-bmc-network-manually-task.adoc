= Das ONTAP SP/BMC Netzwerk manuell konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn keine automatische Netzwerkkonfiguration für den SP eingerichtet ist, muss das SP Netzwerk eines Knotens manuell konfiguriert werden, damit der SP über eine IP-Adresse erreichbar ist.

.Bevor Sie beginnen
Wenn IPv6-Verbindungen für den SP verwendet werden sollen, muss IPv6 für ONTAP bereits konfiguriert und aktiviert sein. Die  `network options ipv6` Befehle verwalten die IPv6-Einstellungen für ONTAP. Weitere Informationen zu  `network options ipv6` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=network+options+ipv6["ONTAP-Befehlsreferenz"^].

[NOTE]
====
Diese Aufgabe gilt sowohl für den SP als auch für den BMC.

====
Der SP kann so konfiguriert werden, dass er IPv4, IPv6 oder beides verwendet. Die SP IPv4 Konfiguration unterstützt statische und DHCP-Adressierung, und die SP IPv6 Konfiguration unterstützt nur statische Adressierung.

Wenn die automatische SP-Netzwerkkonfiguration eingerichtet ist, ist keine manuelle Konfiguration des SP-Netzwerks für einzelne Knoten erforderlich, und der  `system service-processor network modify` Befehl ermöglicht lediglich das Aktivieren oder Deaktivieren der SP-Netzwerkschnittstelle.

.Schritte
. Das SP Netzwerk für einen Node kann mit dem  `system service-processor network modify` Befehl konfiguriert werden.
+
** Der  `-address-family`Parameter gibt an, ob die IPv4- oder die IPv6-Konfiguration des SP geändert wird.
** Der  `-enable` Parameter aktiviert die Netzwerkschnittstelle der angegebenen IP-Adressfamilie.
** Der  `-dhcp`Parameter gibt an, ob die Netzwerkkonfiguration vom DHCP-Server oder die von Ihnen angegebene Netzwerkadresse verwendet wird.
+
DHCP kann (durch Einstellen von  `-dhcp` auf  `v4`) nur aktiviert werden, wenn IPv4 verwendet wird. DHCP kann für IPv6-Konfigurationen nicht aktiviert werden.

** Der  `-ip-address`Parameter gibt die öffentliche IP-Adresse für den SP an.
+
Eine Warnmeldung erscheint, wenn versucht wird, das SP-Netzwerk manuell mit Adressen zu konfigurieren, die einem Subnetz zugewiesen sind. Das Ignorieren der Warnung und das Fortsetzen der manuellen Adresszuweisung kann zu einer doppelten Adresszuweisung führen.

** Der  `-netmask`Parameter gibt die Subnetzmaske für den SP (bei Verwendung von IPv4) an.
** Der  `-prefix-length` Parameter gibt die Netzwerkpräfixlänge der Subnetzmaske für den SP (bei Verwendung von IPv6) an.
** Der  `-gateway`Parameter gibt die Gateway-IP-Adresse für den SP an.


. Das SP Netzwerk für die verbleibenden Knoten im Cluster wird konfiguriert, indem Schritt 1 wiederholt wird.
. Die SP-Netzwerkkonfiguration kann angezeigt und der SP Einrichtungsstatus überprüft werden, indem der  `system service-processor network show` Befehl mit den  `–instance` oder  `–field setup-status` Parametern verwendet wird.
+
Der SP Setup-Status für einen Knoten kann einer der folgenden sein:

+
** `not-setup` -- Nicht konfiguriert
** `succeeded` -- Konfiguration erfolgreich
** `in-progress` Konfiguration läuft
** `failed` -- Konfiguration fehlgeschlagen




.Beispiel für die Konfiguration des SP Netzwerks
Das folgende Beispiel konfiguriert den SP eines Knotens für die Verwendung von IPv4, aktiviert den SP und zeigt die SP Netzwerkkonfiguration an, um die Einstellungen zu überprüfen:

[listing]
----

cluster1::> system service-processor network modify -node local
-address-family IPv4 -enable true -ip-address 192.168.123.98
-netmask 255.255.255.0 -gateway 192.168.123.1

cluster1::> system service-processor network show -instance -node local
