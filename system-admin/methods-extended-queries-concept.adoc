= Erweiterte Abfragen mit den Befehlen modify und delete in der ONTAP CLI verwenden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können erweiterte Abfragen verwenden, um Objekte mit bestimmten Werten abzugleichen und Operationen an ihnen durchzuführen.

Erweiterte Abfragen werden durch Einschließen in geschweifte Klammern ({}) angegeben. Eine erweiterte Abfrage muss als erstes Argument nach dem Befehlsnamen, vor allen anderen Parametern, angegeben werden. Beispielsweise wird der folgende Befehl verwendet, um alle Volumes offline zu schalten, deren Namen die Zeichenkette  `tmp` enthalten:

[listing]
----
cluster1::> volume modify {-volume *tmp*} -state offline
----
Erweiterte Abfragen sind im Allgemeinen nur mit `modify` und `delete` Befehlen sinnvoll. Sie haben keine Bedeutung in `create` oder `show` Befehlen.

Die Kombination aus Abfragen und Änderungsoperationen ist ein nützliches Werkzeug. Bei falscher Implementierung kann sie jedoch zu Verwirrung und Fehlern führen. Beispielsweise bewirkt die Verwendung des (erweiterte Berechtigung) `system node image modify` Befehls zum Festlegen des Standard-Software-Images eines Knotens automatisch, dass das andere Software-Image nicht mehr als Standard festgelegt wird. Der Befehl im folgenden Beispiel ist im Grunde eine Nulloperation:

[listing]
----
cluster1::*> system node image modify {-isdefault true} -isdefault false
----
Dieser Befehl legt das aktuelle Standardimage als nicht standardmäßiges Image fest und setzt anschließend das neue Standardimage (das vorherige nicht standardmäßige Image) auf das nicht standardmäßige Image, sodass die ursprünglichen Standardeinstellungen erhalten bleiben. Für die korrekte Ausführung des Vorgangs kann der Befehl wie im folgenden Beispiel verwendet werden:

[listing]
----
cluster1::*> system node image modify {-iscurrent false} -isdefault true
----
