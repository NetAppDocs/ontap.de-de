= Anzeigeeinstellungen für die ONTAP CLI festlegen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Anzeigeeinstellungen für eine CLI-Sitzung mithilfe des  `set` Befehls und des  `rows` Befehls festlegen. Die von Ihnen festgelegten Einstellungen gelten nur für die aktuelle Sitzung. Sie sind nicht sitzungsübergreifend persistent.

.Über diese Aufgabe
Sie können die folgenden CLI-Anzeigeeinstellungen festlegen:

* Die Berechtigungsstufe der Befehlssitzung
* Ob Bestätigungen für potenziell unterbrechende Befehle erteilt werden
* Ob  `show`Befehle alle Felder anzeigen
* Das oder die als Feldtrennzeichen zu verwendenden Zeichen
* Die Standardeinheit bei der Angabe von Datengrößen
* Die Anzahl der Zeilen, die der Bildschirm in der aktuellen CLI-Sitzung anzeigt, bevor die Schnittstelle die Ausgabe pausiert
+
Wird die gewünschte Zeilenanzahl nicht angegeben, wird sie automatisch anhand der tatsächlichen Höhe des Terminals angepasst. Ist die tatsächliche Höhe nicht definiert, beträgt die Standardzeilenanzahl 24.

* Die standardmäßige Storage Virtual Machine (SVM) oder der Node
* Ob ein fortlaufender Befehl bei einem Fehler gestoppt werden soll


.Schritte
. Um die Anzeigeeinstellungen der Befehlszeile festzulegen, den  `set` Befehl verwenden.
+
Um die Anzahl der Zeilen festzulegen, die der Bildschirm in der aktuellen CLI-Sitzung anzeigt, kann auch der  `rows` Befehl verwendet werden.

+
Weitere Informationen zu  `set` und  `rows` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/["ONTAP-Befehlsreferenz"^].



.Beispiel für das Festlegen von Anzeigeeinstellungen in der Befehlszeile
Im folgenden Beispiel wird ein Komma als Feldtrennzeichen festgelegt,  `GB` als Standard-Datengrößeneinheit festgelegt und die Anzahl der Zeilen auf 50 gesetzt:

[listing]
----
cluster1::> set -showseparator "," -units GB
cluster1::> rows 50
----
