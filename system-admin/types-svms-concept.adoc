= Arten von SVMs in einem ONTAP Cluster
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein Cluster besteht aus vier Arten von SVMs, die bei der Verwaltung des Clusters und seiner Ressourcen sowie beim Datenzugriff für Clients und Anwendungen helfen.

Ein Cluster enthält die folgenden SVM-Typen:

* Admin-SVM
+
Der Cluster-Setup-Prozess erstellt automatisch die Admin-SVM für den Cluster. Die Admin-SVM repräsentiert den Cluster.

* Knoten-SVM
+
Eine Node SVM wird erstellt, wenn der Knoten dem Cluster beitritt, und die Node SVM repräsentiert die einzelnen Knoten des Clusters.

* System SVM (erweitert)
+
Eine System-SVM wird für die Kommunikation auf Clusterebene in einem IPspace automatisch erstellt.

* Data SVM
+
Eine Daten-SVM repräsentiert die datenbereitstellenden SVMs. Nach der Cluster-Einrichtung muss ein Cluster-Administrator Daten-SVMs erstellen und diesen SVMs Volumes hinzufügen, um den Datenzugriff aus dem Cluster zu ermöglichen.

+
Ein Cluster muss mindestens eine Daten-SVM besitzen, um Daten an seine Clients bereitzustellen.



[NOTE]
====
Sofern nicht anders angegeben, bezieht sich der Begriff SVM auf eine Data (datenservierende) SVM.

In der CLI werden SVMs als Vservers angezeigt.
