= Informationen zur Anpassung von Systemzustandswarnungen in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können steuern, welche Warnmeldungen ein Health Monitor generiert, indem Sie die System Health Policies aktivieren oder deaktivieren, die festlegen, wann Warnmeldungen ausgelöst werden. Dadurch lässt sich das Health Monitoring System an die jeweilige Umgebung anpassen.

Den Namen einer Richtlinie kann entweder durch die Anzeige detaillierter Informationen zu einer generierten Warnung oder durch die Anzeige von Richtliniendefinitionen für einen bestimmten Health Monitor, Node oder eine bestimmte Warnungs-ID ermittelt werden.

Das Deaktivieren von Gesundheitsrichtlinien unterscheidet sich vom Unterdrücken von Warnmeldungen. Wenn eine Warnmeldung unterdrückt wird, hat dies keine Auswirkungen auf den Gesundheitszustand des Subsystems, die Warnmeldung kann aber weiterhin auftreten.

Wenn eine Richtlinie deaktiviert wird, löst die in ihrem Richtlinienregelausdruck definierte Bedingung oder der definierte Zustand keine Warnung mehr aus.

.Beispiel einer Benachrichtigung, die deaktiviert werden soll
Angenommen, es tritt eine Warnung auf, die für Sie nicht nützlich ist. Sie verwenden den  `system health alert show –instance` Befehl, um die Policy-ID der Warnung zu erhalten. Sie verwenden die Policy-ID im  `system health policy definition show` Befehl, um Informationen zur Policy anzuzeigen. Nach Überprüfung des Regelausdrucks und weiterer Informationen zur Policy wird entschieden, die Policy zu deaktivieren. Sie verwenden den  `system health policy definition modify` Befehl, um die Policy zu deaktivieren.
