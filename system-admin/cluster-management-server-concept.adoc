= Informationen zum ONTAP Cluster Management Server
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Der Cluster-Management-Server, auch __admin__SVM genannt, ist eine spezialisierte Implementierung einer Storage Virtual Machine (SVM), die den Cluster als eine einzige verwaltbare Einheit darstellt. Zusätzlich zur Funktion als oberste administrative Domäne besitzt der Cluster-Management-Server Ressourcen, die logisch nicht zu einer Daten-SVM gehören.

Der Cluster-Management-Server ist im Cluster immer verfügbar. Der Cluster-Management-Server kann über die Konsole oder die Cluster-Management-LIF aufgerufen werden.

Bei Ausfall des Heimnetzwerkports schlägt die Cluster-Management-LIF automatisch auf einen anderen Node im Cluster um. Abhängig von den Konnektivitätseigenschaften des verwendeten Managementprotokolls kann der Failover wahrgenommen werden oder auch nicht. Bei Verwendung eines verbindungslosen Protokolls (zum Beispiel SNMP) oder einer eingeschränkten Verbindung (zum Beispiel HTTP) wird der Failover in der Regel nicht bemerkt. Bei einer langfristigen Verbindung (zum Beispiel SSH) ist nach dem Failover eine erneute Verbindung zum Cluster-Management-Server erforderlich.

Bei der Erstellung eines Clusters werden alle Eigenschaften der Cluster-Management-LIF konfiguriert, einschließlich ihrer IP-Adresse, Netzmaske, Gateway und Port.

Im Gegensatz zu einer Daten-SVM oder einer Knoten-SVM verfügt ein Cluster Management-Server nicht über ein Root-Volume oder hostet Benutzer-Volumes (obwohl er System-Volumes hosten kann). Außerdem kann ein Cluster Management-Server nur LIFs vom Typ Cluster Management haben.

Wenn Sie den  `vserver show` Befehl ausführen, erscheint der Cluster Management-Server in der Ausgabeliste dieses Befehls.
