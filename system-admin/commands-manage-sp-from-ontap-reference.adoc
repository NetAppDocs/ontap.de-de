= ONTAP Befehle für das Service Processor Management
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP stellt Befehle zur Verwaltung des SP bereit, einschließlich der SP Netzwerk­konfiguration, SP Firmware Image, des SSH Zugriffs auf den SP und der allgemeinen SP Administration.



[[commands-for-managing-the-sp-network-configuration]]
== Befehle zur Verwaltung der SP Netzwerkkonfiguration

|===
| Wenn Sie möchten... | Diesen ONTAP Befehl ausführen... 


 a| 
Die SP automatische Netzwerkkonfiguration ermöglicht dem SP die Nutzung der IPv4 oder IPv6 Adressfamilie des angegebenen Subnetzes.
 a| 
`system service-processor network auto-configuration enable`



 a| 
Die automatische SP Netzwerkkonfiguration für die IPv4- oder IPv6-Adressfamilie des für den SP angegebenen Subnetzes deaktivieren
 a| 
`system service-processor network auto-configuration disable`



 a| 
Anzeige der automatischen SP-Netzwerkkonfiguration
 a| 
`system service-processor network auto-configuration show`



 a| 
Das SP Netzwerk für einen Knoten kann manuell konfiguriert werden, einschließlich Folgendem:

* Die IP-Adressfamilie (IPv4 oder IPv6)
* Ob die Netzwerkschnittstelle der angegebenen IP-Adressfamilie aktiviert werden sollte
* Wenn Sie IPv4 verwenden, ob die Netzwerkkonfiguration vom DHCP-Server oder die von Ihnen angegebene Netzwerkadresse verwendet wird
* Die öffentliche IP-Adresse für den SP
* Die Netzmaske für den SP (bei Verwendung von IPv4)
* Die Netzwerkpräfixlänge der Subnetzmaske für den SP (bei Verwendung von IPv6)
* Die Gateway-IP-Adresse für den SP

 a| 
`system service-processor network modify`



 a| 
Die SP Netzwerkkonfiguration wird angezeigt, einschließlich der folgenden:

* Die konfigurierte Adressfamilie (IPv4 oder IPv6) und ob sie aktiviert ist
* Der Typ des Fernverwaltungsgeräts
* Der aktuelle SP Status und Verbindungsstatus
* Netzwerkkonfiguration, wie z. B. IP-Adresse, MAC-Adresse, Netzmaske, Präfixlänge der Subnetzmaske, vom Router zugewiesene IP-Adresse, Link-Local-IP-Adresse und Gateway-IP-Adresse
* Zeitpunkt der letzten Aktualisierung des SP
* Der Name des Subnetzes, das für die automatische SP-Konfiguration verwendet wird
* Ob die vom IPv6-Router zugewiesene IP-Adresse aktiviert ist
* SP Netzwerk-Setup-Status
* Grund für das Fehlschlagen der SP-Netzwerkeinrichtung

 a| 
`system service-processor network show`

Die vollständigen SP Netzwerkdetails werden angezeigt, wenn der  `-instance` Parameter verwendet wird.



 a| 
Die SP API-Service-Konfiguration einschließlich der folgenden Punkte anpassen:

* Ändern des vom SP API-Service verwendeten Ports
* Aktivieren oder Deaktivieren des SP API-Service

 a| 
`system service-processor api-service modify`

(Erweiterte Berechtigungsstufe)



 a| 
Anzeigen der SP API-Service-Konfiguration
 a| 
`system service-processor api-service show`

(Erweiterte Berechtigungsstufe)



 a| 
Die SSL- und SSH-Zertifikate, die vom SP API-Service für die interne Kommunikation verwendet werden, erneuern.
 a| 
* Für ONTAP 9.5 oder höher: `system service-processor api-service renew-internal-certificates`
* Für ONTAP 9.4 oder älter: `system service-processor api-service renew-certificates`
+
(Erweiterte Berechtigungsstufe)



|===


[[commands-for-managing-the-sp-firmware-image]]
== Befehle zur Verwaltung des SP Firmware-Images

|===
| Wenn Sie möchten... | Diesen ONTAP Befehl ausführen... 


 a| 
Die Details des aktuell installierten SP-Firmware-Images werden angezeigt, einschließlich der folgenden:

* Der Typ des Fernverwaltungsgeräts
* Das Image (primär oder Backup), von dem der SP gebootet wird, sein Status und die Firmware-Version
* Ob die automatische Firmware-Aktualisierung aktiviert ist und der letzte Aktualisierungsstatus

 a| 
`system service-processor image show`

Der  `-is-current`Parameter gibt das Image (primär oder Backup) an, von dem der SP aktuell gebootet ist, nicht ob die installierte Firmware-Version die aktuellste ist.



 a| 
Automatische Firmware-Aktualisierung für SP aktivieren oder deaktivieren
 a| 
`system service-processor image modify`

Standardmäßig wird die SP-Firmware automatisch mit der Aktualisierung von ONTAP oder beim manuellen Herunterladen einer neuen Version der SP-Firmware aktualisiert. Es wird nicht empfohlen, die automatische Aktualisierung zu deaktivieren, da dies zu suboptimalen oder nicht qualifizierten Kombinationen zwischen dem ONTAP Image und dem SP Firmware-Image führen kann.



 a| 
Manuelles Herunterladen eines SP-Firmware-Images auf einen Knoten
 a| 
`system node image get`

[NOTE]
====
Bevor Sie die  `system node image`Befehle ausführen, muss die Berechtigungsstufe auf „advanced (`set -privilege advanced`“ gesetzt werden, wobei bei entsprechender Aufforderung *y* einzugeben ist, um fortzufahren.

====
Das SP Firmware-Image ist mit ONTAP gebündelt. Ein manueller Download der SP Firmware ist nicht erforderlich, es sei denn, eine andere SP Firmware-Version als die mit ONTAP gebündelte soll verwendet werden.



 a| 
Der Status des neuesten von ONTAP ausgelösten SP-Firmware-Updates wird angezeigt, einschließlich der folgenden Informationen:

* Start- und Endzeitpunkt des neuesten SP Firmware-Updates
* Ob ein Update läuft und der Prozentsatz des Abschlusses

 a| 
`system service-processor image update-progress show`

|===


[[commands-for-managing-ssh-access-to-the-sp]]
== Befehle zur Verwaltung des SSH-Zugriffs auf den SP

|===
| Wenn Sie möchten... | Diesen ONTAP Befehl ausführen... 


 a| 
SP erhält nur für die angegebenen IP-Adressen Zugriff.
 a| 
`system service-processor ssh add-allowed-addresses`



 a| 
Der Zugriff der angegebenen IP-Adressen auf den SP wird blockiert
 a| 
`system service-processor ssh remove-allowed-addresses`



 a| 
Die IP-Adressen, die auf den SP zugreifen können, anzeigen
 a| 
`system service-processor ssh show`

|===


[[commands-for-general-sp-administration]]
== Befehle für die allgemeine SP Verwaltung

|===
| Wenn Sie möchten... | Diesen ONTAP Befehl ausführen... 


 a| 
Allgemeine SP-Informationen, einschließlich der folgenden, werden angezeigt:

* Der Typ des Fernverwaltungsgeräts
* Der aktuelle SP Status
* Ob das SP Netzwerk konfiguriert ist
* Netzwerkinformationen, wie die öffentliche IP-Adresse und die MAC-Adresse
* Die SP-Firmware-Version und die Intelligent Platform Management Interface (IPMI)-Version
* Ob die automatische SP-Firmware-Aktualisierung aktiviert ist

 a| 
`system service-processor show`Die Anzeige vollständiger SP-Informationen erfordert den  `-instance`Parameter.



 a| 
Der SP auf einem Knoten wird neu gestartet.
 a| 
`system service-processor reboot-sp`



 a| 
Eine AutoSupport-Nachricht generieren und senden, die die vom angegebenen Node erfassten SP-Protokolldateien enthält
 a| 
`system node autosupport invoke-splog`



 a| 
Die Zuordnungskarte der gesammelten SP-Protokolldateien im Cluster wird angezeigt, einschließlich der Sequenznummern der SP-Protokolldateien, die sich auf jedem Sammelknoten befinden.
 a| 
`system service-processor log show-allocations`

|===
