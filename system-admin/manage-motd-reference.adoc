= Die Verwaltung des Message-of-the-Day-Textes, der auf ONTAP Cluster- und SVM-Ebene angezeigt wird
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können die Nachricht des Tages (MOTD) auf Cluster- oder SVM-Ebene verwalten. Standardmäßig ist die für den Cluster konfigurierte MOTD auch für alle SVMs aktiviert. Zusätzlich kann auf SVM-Ebene für jede SVM eine MOTD konfiguriert werden. Die MOTD auf Cluster-Ebene kann vom Clusteradministrator für jede SVM aktiviert oder deaktiviert werden.

Weitere Informationen zu den link:https://docs.netapp.com/us-en/ontap-cli//security-login-motd-modify.html#parameters["Fluchtsequenzen"^]Funktionen, die zur dynamischen Generierung von Inhalten für die MOTD in der ONTAP-Befehlsreferenz verwendet werden können.

.Auswahlmöglichkeiten
* Die MOTD auf Clusterebene verwalten:
+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Eine MOTD für alle Anmeldungen erstellen, wenn noch keine MOTD vorhanden ist.
 a| 
Eine MOTD auf Clusterebene festlegen:

`security login motd modify -vserver <cluster_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`



 a| 
Die MOTD für alle Anmeldungen wird geändert, wenn keine MOTDs auf SVM-Ebene konfiguriert sind.
 a| 
Die MOTD auf Clusterebene ändern:

`security login motd modify -vserver <cluster_name> { [-message "<text>"] } | [-uri <ftp_or_http_addr>] }`



 a| 
Die MOTD für alle Anmeldungen wird entfernt, wenn keine MOTDs auf SVM-Ebene konfiguriert sind.
 a| 
Setzen der MOTD auf Clusterebene auf eine leere Zeichenkette (`""`:

`security login motd modify -vserver <cluster_name> -message ""`



 a| 
Jede SVM zeigt die MOTD auf Clusterebene an, anstatt die MOTD auf SVM-Ebene zu verwenden.
 a| 
Eine MOTD auf Clusterebene festlegen und anschließend alle MOTDs auf SVM-Ebene auf eine leere Zeichenkette setzen, wobei die MOTD auf Clusterebene aktiviert ist:

.. `security login motd modify -vserver <cluster_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`
.. `security login motd modify { -vserver !"<cluster_name>" } -message "" -is-cluster-message-enabled true`




 a| 
Eine MOTD wird nur für ausgewählte SVMs angezeigt, ohne eine MOTD auf Clusterebene zu verwenden.
 a| 
Die MOTD auf Clusterebene wird auf eine leere Zeichenkette gesetzt, anschließend werden die MOTDs auf SVM-Ebene für die ausgewählten SVMs festgelegt:

.. `security login motd modify -vserver <cluster_name> -message ""`
.. `security login motd modify -vserver <svm_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`
+
Diesen Schritt können Sie bei Bedarf für jede SVM wiederholen.





 a| 
Die gleiche SVM-Ebene-MOTD wird für alle (Daten- und Verwaltungs-)SVMs verwendet.
 a| 
Den Cluster und alle SVMs so einstellen, dass dasselbe MOTD verwendet wird:

`security login motd modify -vserver * { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`

[NOTE]
====
Im interaktiven Modus fordert die CLI dazu auf, die MOTD für den Cluster und jede SVM einzeln einzugeben. Die gleiche MOTD kann in jede Instanz eingefügt werden, wenn die entsprechende Aufforderung erscheint.

====


 a| 
Eine MOTD auf Clusterebene soll optional für alle SVMs verfügbar sein, aber nicht bei Cluster-Logins angezeigt werden.
 a| 
Eine MOTD auf Clusterebene festlegen, aber deren Anzeige für den Cluster deaktivieren:

`security login motd modify -vserver <cluster_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] } -is-cluster-message-enabled false`



 a| 
Entfernen aller MOTDs auf Cluster- und SVM-Ebene, wenn nur einige SVMs sowohl MOTDs auf Cluster- als auch auf SVM-Ebene haben
 a| 
Für den Cluster und alle SVMs wird eine leere Zeichenkette als MOTD verwendet:

`security login motd modify -vserver * -message ""`



 a| 
Die MOTD wird nur für die SVMs geändert, die eine nicht-leere Zeichenkette haben, wenn andere SVMs eine leere Zeichenkette verwenden und auf Clusterebene eine andere MOTD genutzt wird.
 a| 
Erweiterte Abfragen ermöglichen eine gezielte Änderung der MOTD:

`security login motd modify { -vserver !"<cluster_name>" -message !"" } { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`



 a| 
Alle MOTDs, die bestimmten Text enthalten (zum Beispiel „Januar“ gefolgt von „2015“), werden überall in einer ein- oder mehrzeiligen Nachricht angezeigt, selbst wenn der Text auf verschiedene Zeilen verteilt ist.
 a| 
Eine Abfrage zeigt MOTDs an:

[listing]
----
security login motd show -message *"January"*"2015"*
----


 a| 
Interaktiv eine MOTD erstellen, die mehrere und aufeinanderfolgende Zeilenumbrüche (auch bekannt als End-of-Line oder EOLs) enthält
 a| 
Im interaktiven Modus wird durch Drücken der Leertaste und anschließend der Eingabetaste eine Leerzeile erzeugt, ohne die Eingabe für die MOTD zu beenden.

|===
* Das MOTD auf SVM-Ebene verwalten:
+
Die Angabe von  `-vserver <svm_name>` ist im SVM Kontext nicht erforderlich.

+
|===
| Wenn Sie möchten... | Dann... 


 a| 
Eine andere SVM-Level-MOTD verwenden, wenn die SVM bereits eine vorhandene SVM-Level-MOTD hat
 a| 
Die MOTD auf SVM-Ebene ändern:

`security login motd modify -vserver <svm_name> { [-message "<text>"] | [-uri <ftp_or_http_addr>] }`



 a| 
Nur die Cluster-Ebene MOTD für die SVM verwenden, wenn die SVM bereits eine SVM-Ebene MOTD hat.
 a| 
Die MOTD auf SVM-Ebene wird auf eine leere Zeichenfolge gesetzt, danach kann der Clusteradministrator die MOTD auf Clusterebene für die SVM aktivieren:

.. `security login motd modify -vserver <svm_name> -message ""`
.. (Für den Cluster-Administrator) `security login motd modify -vserver <svm_name> -is-cluster-message-enabled true`




 a| 
Keine MOTD für die SVM anzeigen, wenn sowohl die MOTD auf Cluster-Ebene als auch auf SVM-Ebene derzeit für die SVM angezeigt werden.
 a| 
Die MOTD auf SVM-Ebene wird auf eine leere Zeichenkette gesetzt, anschließend deaktiviert der Cluster-Administrator die MOTD auf Cluster-Ebene für die SVM:

.. `security login motd modify -vserver <svm_name> -message ""`
.. (Für den Cluster-Administrator) `security login motd modify -vserver <svm_name> -is-cluster-message-enabled false`


|===
