= Informationen zu den verfügbaren ONTAP Cluster-Integritätsmonitoren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Es gibt mehrere Systemüberwachungsmodule, die verschiedene Teile eines Clusters überwachen. Systemüberwachungsmodule unterstützen bei der Behebung von Fehlern in ONTAP-Systemen, indem sie Ereignisse erkennen, Warnmeldungen senden und Ereignisse löschen, sobald diese behoben sind.

|===
| Name des Health Monitors (Kennung) | Subsystemname (Kennung) | Zweck 


 a| 
Ethernet Switch
 a| 
Switch (Switch-Health)
 a| 
Der ONTAP Ethernet Switch Health Monitor (CSHM) überwacht den Status von Cluster- und Speichernetzwerk-Switches und erfasst Protokolle zur Analyse. Standardmäßig fragt CSHM jeden Switch alle 5 Minuten per SNMPv2c ab, um Ressourcentabellen mit Informationen zu Supportfähigkeit, Überwachungsstatus, Temperatursensoren, CPU-Auslastung, Schnittstellenkonfigurationen und -verbindungen, Cluster Switch-Redundanz sowie Lüfter- und Netzteilbetrieb zu aktualisieren. Zusätzlich erfasst CSHM, sofern konfiguriert, stündlich Protokolle per SSH/SCP, die über AutoSupport zur weiteren Analyse gesendet werden. Auf Anfrage kann CSHM auch eine detailliertere Protokollerfassung für den technischen Support per SSH/SCP durchführen.

Weitere Einzelheiten sind unter link:https://docs.netapp.com/us-en/ontap-systems-switches/switch-cshm/config-overview.html["Switch-Zustandsüberwachung"^] zu finden.



 a| 
MetroCluster Fabric
 a| 
Switch
 a| 
Überwacht die MetroCluster Konfiguration der Back-End-Fabric-Topologie und erkennt Fehlkonfigurationen wie falsche Verkabelung und Zonierung sowie ISL-Ausfälle.



 a| 
MetroCluster Status
 a| 
Interconnect, RAID und Storage
 a| 
Überwacht FC-VI Adapter, FC-Initiator-Adapter, verbliebene Aggregate und Festplatten sowie Inter-Cluster-Ports



 a| 
Knotenkonnektivität (node-connect)
 a| 
CIFS unterbrechungsfreie Vorgänge (CIFS-NDO)
 a| 
Überwacht SMB-Verbindungen für unterbrechungsfreien Betrieb von Hyper-V-Anwendungen.



 a| 
Speicher (SAS-Anschluss)
 a| 
Überwacht Gehäuse, Festplatten und Adapter auf Knotenebene hinsichtlich geeigneter Pfade und Verbindungen.



 a| 
System
 a| 
nicht zutreffend
 a| 
Aggregiert Informationen von anderen Gesundheitsmonitoren.



 a| 
Systemkonnektivität (system-connect)
 a| 
Speicher (SAS-Anschluss)
 a| 
Überwacht auf Clusterebene die Regale hinsichtlich geeigneter Pfade zu zwei HA-Clusterknoten.

|===
