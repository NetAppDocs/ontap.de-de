= Befehle zur Überwachung des Zustands Ihres ONTAP Systems
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die  `system health` Befehle ermöglichen die Anzeige von Informationen über den Zustand der Systemressourcen, die Reaktion auf Warnmeldungen sowie die Konfiguration zukünftiger Warnmeldungen. Mit den CLI-Befehlen lassen sich detaillierte Informationen darüber anzeigen, wie die Zustandsüberwachung konfiguriert ist. Weitere Informationen zu  `system health` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=system+health["ONTAP-Befehlsreferenz"^].



[[display-the-status-of-system-health]]
== Status des Systemzustands anzeigen

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Der Gesundheitszustand des Systems wird angezeigt, der den Gesamtstatus der einzelnen Gesundheitsmonitore widerspiegelt.
 a| 
`system health status show`



 a| 
Der Gesundheitszustand der Subsysteme, für die eine Gesundheitsüberwachung verfügbar ist, wird angezeigt.
 a| 
`system health subsystem show`

|===


[[display-the-status-of-node-connectivity]]
== Status der Knotenverbindung anzeigen

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Details zur Konnektivität vom Node zum Storage Shelf werden angezeigt, einschließlich Portinformationen, HBA-Portgeschwindigkeit, I/O-Durchsatz und der Rate von I/O-Operationen pro Sekunde
 a| 
`storage shelf show -connectivity`

Mit dem  `-instance` Parameter werden detaillierte Informationen zu jedem Shelf angezeigt.



 a| 
Informationen zu Laufwerken und Array-LUNs werden angezeigt, einschließlich des nutzbaren Speicherplatzes, der Gehäuse- und Einschubnummern sowie des zugehörigen Node-Name
 a| 
`storage disk show`

Mit dem  `-instance` Parameter werden detaillierte Informationen zu jedem Laufwerk angezeigt.



 a| 
Detaillierte Informationen zu den Ports der Storage Shelves, einschließlich Porttyp, Geschwindigkeit und Status
 a| 
`storage port show`

Der  `-instance` Parameter zeigt detaillierte Informationen zu jedem Adapter an.

|===


[[monitor-cluster-and-storage-network-switches]]
== Cluster- und Speichernetzwerk-Switches überwachen

[cols="3*"]
|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... (ONTAP 9.8 und höher) | Verwenden Sie diesen Befehl... (ONTAP 9.7 und früher) 


 a| 
Die vom Cluster überwachten Switches anzeigen
 a| 
`system switch ethernet show`
 a| 
`system cluster-switch show`



 a| 
Zeigt die Switches an, die der Cluster aktuell überwacht, einschließlich der Switches, die gelöscht wurden (angezeigt in der Spalte Reason der Befehlsausgabe)

Dieser Befehl ist auf der Stufe „Erweitert“ verfügbar
 a| 
`system switch ethernet show-all`
 a| 
`system cluster-switch show-all`



 a| 
Überwachung eines nicht erkannten Switch konfigurieren
 a| 
`system switch ethernet create`
 a| 
`system cluster-switch create`



 a| 
Informationen über einen Switch, den das Cluster überwacht, können geändert werden (z. B. Gerätename, IP-Adresse, SNMP-Version und Community-Zeichenfolge)
 a| 
`system switch ethernet modify`
 a| 
`system cluster-switch modify`



 a| 
Einen Switch aus der Überwachung entfernen
 a| 
`system switch ethernet modify -disable-monitoring`
 a| 
`system cluster-switch modify -disable-monitoring`



 a| 
Einen Switch aus der Überwachung entfernen
 a| 
`system switch ethernet delete`
 a| 
`system cluster-switch delete`



 a| 
Die Switch-Konfigurationsinformationen, die in der Datenbank gespeichert sind, dauerhaft entfernen (dadurch wird die automatische Erkennung des Switches wieder aktiviert)
 a| 
`system switch ethernet delete -force`
 a| 
`system cluster-switch delete -force`



 a| 
Protokollerfassung mit einem Switch durchführen
 a| 
`system switch ethernet log`
 a| 
`system cluster-switch log`

|===
[NOTE]
====
Weitere Einzelheiten finden Sie unter link:https://docs.netapp.com/us-en/ontap-systems-switches/switch-cshm/config-overview.html["Switch-Zustandsüberwachung"^] und link:https://docs.netapp.com/us-en/ontap-systems-switches/switch-cshm/config-log-collection.html["Protokollerfassung konfigurieren"^].

====


[[respond-to-generated-alerts]]
== Auf generierte Warnmeldungen reagieren

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu generierten Warnmeldungen anzeigen, wie die Ressource und den Node, bei dem die Warnmeldung ausgelöst wurde, sowie den Schweregrad und die wahrscheinliche Ursache der Warnmeldung
 a| 
`system health alert show`



 a| 
Informationen zu jeder generierten Warnung
 a| 
`system health alert show -instance`



 a| 
Anzeigen, dass jemand an einer Benachrichtigung arbeitet
 a| 
`system health alert modify`



 a| 
Eine Warnung bestätigen
 a| 
`system health alert modify -acknowledge`



 a| 
Eine nachfolgende Warnung unterdrücken, damit diese den Gesundheitszustand eines Subsystems nicht beeinträchtigt.
 a| 
`system health alert modify -suppress`



 a| 
Eine Warnung löschen, die nicht automatisch gelöscht wurde
 a| 
`system health alert delete`



 a| 
Informationen zu den AutoSupport-Meldungen anzeigen, die durch Warnmeldungen in der letzten Woche ausgelöst wurden, beispielsweise um festzustellen, ob eine Warnmeldung eine AutoSupport-Meldung ausgelöst hat
 a| 
`system health autosupport trigger history show`

|===


[[configure-future-alerts]]
== Zukünftige Benachrichtigungen konfigurieren

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Die Richtlinie, die steuert, ob ein bestimmter Ressourcenstatus eine bestimmte Warnung auslöst, kann aktiviert oder deaktiviert werden.
 a| 
`system health policy definition modify`

|===


[[display-information-about-how-health-monitoring-is-configured]]
== Informationen darüber, wie die Gesundheitsüberwachung konfiguriert ist, anzeigen

|===
| Wenn Sie möchten... | Verwenden Sie diesen Befehl... 


 a| 
Informationen zu Gesundheitsmonitoren wie deren Nodes, Namen, Subsystemen und Status werden angezeigt
 a| 
`system health config show`

[NOTE]
====
Der  `-instance`Parameter zeigt detaillierte Informationen zu jedem Health Monitor an.

====


 a| 
Informationen zu den Warnmeldungen, die ein Gesundheitsmonitor potenziell generieren kann, werden angezeigt
 a| 
`system health alert definition show`

[NOTE]
====
Der  `-instance`Parameter zeigt detaillierte Informationen zu jeder Alarmdefinition an.

====


 a| 
Informationen zu den Richtlinien des Gesundheitsmonitors, die festlegen, wann Warnmeldungen ausgelöst werden, werden angezeigt.
 a| 
`system health policy definition show`

[NOTE]
====
Verwenden Sie den  `-instance` Parameter, um detaillierte Informationen zu jeder Richtlinie anzuzeigen. Andere Parameter ermöglichen das Filtern der Liste der Warnungen, zum Beispiel nach Richtlinienstatus (aktiviert oder deaktiviert), Health Monitor, Alert und so weiter.
