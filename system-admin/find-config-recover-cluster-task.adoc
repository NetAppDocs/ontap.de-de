= Die Sicherungsdatei der ONTAP Clusterkonfiguration zum Wiederherstellen eines Clusters finden
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Zur Wiederherstellung eines Clusters wird die Konfiguration entweder von einem Knoten im Cluster oder aus einer Clusterkonfigurationssicherungsdatei verwendet.

.Schritte
. Wählen Sie einen Konfigurationstyp aus, um den Cluster wiederherzustellen.
+
** Ein Knoten im Cluster
+
Besteht der Cluster aus mehr als einem Knoten und verfügt einer der Knoten über eine Clusterkonfiguration aus der Zeit, als sich der Cluster in der gewünschten Konfiguration befand, kann der Cluster mithilfe der auf diesem Knoten gespeicherten Konfiguration wiederhergestellt werden.

+
In den meisten Fällen ist der Knoten mit dem Replikationsring mit der aktuellsten Transaktions-ID am besten für die Wiederherstellung der Cluster-Konfiguration geeignet. Der  `cluster ring show` Befehl auf der erweiterten Berechtigungsstufe ermöglicht die Anzeige einer Liste der auf jedem Knoten im Cluster verfügbaren Replikationsringe.

** Eine Clusterkonfiguration-Sicherungsdatei
+
Wenn Sie keinen Knoten mit der korrekten Clusterkonfiguration identifizieren können oder wenn der Cluster nur aus einem einzigen Knoten besteht, kann eine Sicherungsdatei der Clusterkonfiguration zur Wiederherstellung des Clusters verwendet werden.

+
Wenn Sie den Cluster aus einer Konfigurationssicherungsdatei wiederherstellen, gehen alle seit der Sicherung vorgenommenen Konfigurationsänderungen verloren. Nach der Wiederherstellung müssen alle Abweichungen zwischen der Konfigurationssicherungsdatei und der aktuellen Konfiguration behoben werden. Siehe link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/ONTAP_Configuration_Backup_Resolution_Guide["NetApp Knowledge Base: ONTAP Konfigurationssicherungs-Resolution Guide"^] für Hinweise zur Fehlerbehebung.



. Wenn Sie sich für die Verwendung einer Clusterkonfigurations-Sicherungsdatei entschieden haben, muss die Datei dem Knoten zur Verfügung stehen, den Sie zur Wiederherstellung des Clusters verwenden möchten.
+
|===
| Wenn sich die Konfigurationssicherungsdatei befindet... | Dann... 


 a| 
Unter einer entfernten URL
 a| 
Der  `system configuration backup download`-Befehl auf der erweiterten Berechtigungsebene kann verwendet werden, um ihn auf den wiederherzustellenden Node herunterzuladen.



 a| 
Auf einem Knoten im Cluster
 a| 
.. Der  `system configuration backup show` Befehl auf der Ebene mit erweiterten Berechtigungen ermöglicht das Auffinden einer Cluster-Konfigurationssicherungsdatei, die erstellt wurde, als sich der Cluster in der gewünschten Konfiguration befand.
.. Falls sich die Sicherungsdatei der Clusterkonfiguration nicht auf dem Knoten befindet, den Sie für die Wiederherstellung des Clusters verwenden möchten, dann kann der  `system configuration backup copy` Befehl verwendet werden, um sie auf den wiederherzustellenden Knoten zu kopieren.


|===
