= Die Ausgabe des ONTAP show-Befehls kann mit dem Parameter fields eingeschränkt werden.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Wenn der  `-instance` Parameter mit einem  `show` Befehl zur Anzeige von Details verwendet wird, kann die Ausgabe sehr umfangreich sein und mehr Informationen enthalten, als benötigt werden. Der  `-fields` Parameter eines  `show` Befehls ermöglicht die Anzeige nur der Informationen, die angegeben werden.

Beispielsweise führt das Ausführen von  `volume show -instance` wahrscheinlich zu mehreren Informationsseiten. Mit  `volume show -fields` `_fieldname[,fieldname...]_` lässt sich die Ausgabe so anpassen, dass nur das angegebene Feld oder die angegebenen Felder enthalten sind (zusätzlich zu den standardmäßig angezeigten Feldern). Mit  `-fields ?` lassen sich gültige Felder für einen  `show` Befehl anzeigen.

Das folgende Beispiel zeigt den Unterschied in der Ausgabe zwischen dem  `-instance` Parameter und dem  `-fields` Parameter:

[listing]
----
cluster1::> volume show -instance

                                   Vserver Name: cluster1-1
                                    Volume Name: vol0
                                 Aggregate Name: aggr0
                                    Volume Size: 348.3GB
                             Volume Data Set ID: -
                      Volume Master Data Set ID: -
                                   Volume State: online
                                    Volume Type: RW
                                   Volume Style: flex
                                         ...
                          Space Guarantee Style: volume
                      Space Guarantee in Effect: true
                                         ...
Press <space> to page down, <return> for next line, or 'q' to quit...
...
cluster1::>

cluster1::> volume show -fields space-guarantee,space-guarantee-enabled

vserver  volume space-guarantee space-guarantee-enabled
-------- ------ --------------- -----------------------
cluster1-1 vol0   volume          true
cluster1-2 vol0   volume          true
vs1      root_vol
                  volume          true
vs2      new_vol
                  volume          true
vs2      root_vol
                  volume          true
...
cluster1::>
----
