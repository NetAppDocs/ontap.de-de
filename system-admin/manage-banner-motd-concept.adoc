= Informationen zum ONTAP Anmeldebanner und zum Text der Nachricht des Tages
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP ermöglicht die Konfiguration eines Anmeldebanners oder einer Nachricht des Tages (MOTD), um administrative Informationen an System Manager und CLI-Benutzer des Clusters oder der Storage Virtual Machine (SVM) zu kommunizieren.

In einer Konsolensitzung (nur für Clusterzugriff) oder einer SSH-Sitzung (für Cluster- oder SVM-Zugriff) wird ein Banner angezeigt, bevor ein Benutzer zur Authentifizierung wie zum Beispiel einem Passwort aufgefordert wird. Das Banner kann beispielsweise dazu verwendet werden, einer Person, die versucht, sich am System anzumelden, eine Warnmeldung wie die folgende anzuzeigen:

[listing]
----
$ ssh admin@cluster1-01

This system is for authorized users only. Your IP Address has been logged.

Password:

----
Eine MOTD (Meldung des Tages) wird in einer Konsolensitzung (nur für Clusterzugriff) oder einer SSH-Sitzung (für Cluster- oder SVM-Zugriff) angezeigt, nachdem ein Benutzer authentifiziert wurde, aber bevor die Clustershell-Eingabeaufforderung erscheint. Zum Beispiel kann die MOTD verwendet werden, um eine Willkommens- oder Informationsnachricht wie die folgende anzuzeigen, die nur authentifizierte Benutzer sehen:

[listing]
----
$ ssh admin@cluster1-01

Password:

Greetings. This system is running ONTAP 9.0.
Your user name is 'admin'. Your last login was Wed Apr 08 16:46:53 2015 from 10.72.137.28.

----
Sie können den Inhalt des Banners oder der MOTD mit dem  `security login banner modify` oder  `security login motd modify` Befehl jeweils auf folgende Weise erstellen oder ändern:

* Sie können die CLI interaktiv oder nichtinteraktiv verwenden, um den Text für das Banner oder die MOTD festzulegen.
+
Der interaktive Modus, der beim Aufruf des Befehls ohne den  `-message` oder  `-uri` Parameter gestartet wird, ermöglicht die Verwendung von Zeilenumbrüchen (auch bekannt als Zeilenenden) in der Nachricht.

+
Der nichtinteraktive Modus, der den  `-message` Parameter zur Angabe der Nachrichtenzeichenfolge verwendet, unterstützt keine Zeilenumbrüche.

* Es besteht die Möglichkeit, Inhalte von einem FTP- oder HTTP-Speicherort hochzuladen und für das Banner oder die MOTD zu nutzen.
* Die MOTD kann so konfiguriert werden, dass dynamische Inhalte angezeigt werden.
+
Beispiele für die dynamische Anzeige der MOTD, die konfiguriert werden können, sind unter anderem die folgenden:

+
** Clustername, Node-Name oder SVM-Name
** Datum und Uhrzeit des Clusters
** Name des Benutzers, der sich anmeldet
** Letzter Login des Benutzers auf einem beliebigen Node im Cluster
** Anmeldegerätname oder IP-Adresse
** Name des Betriebssystems
** Software Release Version
** Effektive Clusterversion-Zeichenfolge


+
Das Banner unterstützt keine dynamischen Inhalte. Weitere Informationen zu  `security login motd modify` und den Escape-Sequenzen, mit denen die MOTD so konfiguriert werden kann, dass dynamisch generierte Inhalte angezeigt werden, finden Sie im link:https://docs.netapp.com/us-en/ontap-cli/security-login-motd-modify.html["ONTAP-Befehlsreferenz"^].



Das Banner und die MOTD lassen sich auf Cluster- oder SVM-Ebene verwalten:

* Die folgenden Fakten gelten für das Banner:
+
** Das für den Cluster konfigurierte Banner wird auch für alle SVMs verwendet, für die keine Banner-Nachricht definiert ist.
** Für jede SVM kann ein Banner auf SVM-Ebene konfiguriert werden.
+
Wenn ein Banner auf Clusterebene konfiguriert wurde, wird es durch das Banner auf SVM-Ebene für die jeweilige SVM überschrieben.



* Für die MOTD gelten folgende Fakten:
+
** Standardmäßig ist die für den Cluster konfigurierte MOTD auch für alle SVMs aktiviert.
** Zusätzlich kann auf SVM-Ebene für jede SVM eine MOTD konfiguriert werden.
+
In diesem Fall sehen Benutzer, die sich bei der SVM anmelden, zwei MOTDs, eine auf Cluster-Ebene und die andere auf SVM-Ebene.

** Die MOTD auf Clusterebene kann vom Cluster-Administrator für jede SVM aktiviert oder deaktiviert werden.
+
Wenn der Clusteradministrator die MOTD auf Clusterebene für eine SVM deaktiviert, sieht ein Benutzer, der sich bei der SVM anmeldet, die MOTD auf Clusterebene nicht.
