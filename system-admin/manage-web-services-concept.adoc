= Übersicht zur Verwaltung von Webdiensten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen Webdienst für den Cluster oder eine Storage Virtual Machine (SVM) aktivieren oder deaktivieren, die Einstellungen für Webdienste anzeigen und steuern, ob Benutzer einer Rolle auf einen Webdienst zugreifen können.

Webdienste für den Cluster oder eine SVM lassen sich auf folgende Weise verwalten:

* Aktivieren oder Deaktivieren eines bestimmten Webdienstes
* Festlegung, ob der Zugriff auf einen Webdienst ausschließlich auf verschlüsseltes HTTP (SSL) beschränkt ist
* Anzeige der Verfügbarkeit von Webdiensten
* Benutzern einer Rolle den Zugriff auf einen Webservice erlauben oder verweigern
* Anzeigen der Rollen, die auf einen Webdienst zugreifen dürfen


Damit ein Benutzer auf einen Webdienst zugreifen kann, müssen alle folgenden Bedingungen erfüllt sein:

* Der Benutzer muss authentifiziert sein.
+
Ein Webdienst könnte beispielsweise nach einem Benutzernamen und einem Passwort fragen. Die Antwort des Benutzers muss mit einem gültigen Konto übereinstimmen.

* Dem Benutzer muss die korrekte Zugriffsmethode zugewiesen werden.
+
Die Authentifizierung ist nur für Benutzer mit der korrekten Zugriffsmethode für den jeweiligen Webdienst erfolgreich. Für den ONTAP API Webdienst  `ontapi` müssen Benutzer die  `ontapi` Zugriffsmethode haben. Für alle anderen Webdienste müssen Benutzer die  `http` Zugriffsmethode haben.

+
[NOTE]
====
Die  `security login`Befehle werden verwendet, um die Zugriffsmethoden und Authentifizierungsmethoden von Benutzern zu verwalten.

====
* Der Webdienst muss so konfiguriert sein, dass die Zugriffskontrollrolle des Benutzers zulässig ist.
+
[NOTE]
====
Sie verwenden die  `vserver services web access`Befehle, um den Zugriff einer Rolle auf einen Webdienst zu steuern.

====


Wenn eine Firewall aktiviert ist, muss die Firewall-Richtlinie für die LIF, die für Webdienste verwendet werden soll, so eingerichtet sein, dass HTTP oder HTTPS zulässig ist.

Wenn Sie HTTPS für den Zugriff auf Webdienste verwenden, muss SSL für den Cluster oder die SVM, die den Webdienst anbietet, ebenfalls aktiviert sein, und Sie müssen ein digitales Zertifikat für den Cluster oder die SVM bereitstellen.
