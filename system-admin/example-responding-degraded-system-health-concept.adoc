= Informationen zur Reaktion auf einen beeinträchtigten ONTAP Systemzustand
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Anhand eines konkreten Beispiels für eine beeinträchtigte Systemintegrität, die durch ein Shelf ohne zwei Pfade zu einem Knoten verursacht wird, wird angezeigt, was die CLI anzeigt, wenn auf eine Warnung reagiert wird.

Nach dem Start von ONTAP wird der Systemzustand überprüft und es zeigt sich, dass der Status beeinträchtigt ist:

[listing]
----

      cluster1::>system health status show
        Status
        ---------------
        degraded
----
Warnmeldungen werden angezeigt, um herauszufinden, wo das Problem liegt, und es zeigt sich, dass Shelf 2 nicht über zwei Pfade zu Node1 verfügt:

[listing]
----

      cluster1::>system health alert show
               Node: node1
           Resource: Shelf ID 2
           Severity: Major
	   Indication Time: Mon Nov 10 16:48:12 2013
     Probable Cause: Disk shelf 2 does not have two paths to controller
                     node1.
    Possible Effect: Access to disk shelf 2 via controller node1 will be
                     lost with a single hardware component failure (e.g.
                     cable, HBA, or IOM failure).
 Corrective Actions: 1. Halt controller node1 and all controllers attached to disk shelf 2.
                     2. Connect disk shelf 2 to controller node1 via two paths following the rules in the Universal SAS and ACP Cabling Guide.
                     3. Reboot the halted controllers.
                     4. Contact support personnel if the alert persists.
----
Details zur Warnung werden angezeigt, um weitere Informationen zu erhalten, einschließlich der Warnungs-ID:

[listing]
----

      cluster1::>system health alert show -monitor node-connect -alert-id DualPathToDiskShelf_Alert -instance
                  Node: node1
               Monitor: node-connect
              Alert ID: DualPathToDiskShelf_Alert
     Alerting Resource: 50:05:0c:c1:02:00:0f:02
             Subsystem: SAS-connect
       Indication Time: Mon Mar 21 10:26:38 2011
    Perceived Severity: Major
        Probable Cause: Connection_establishment_error
           Description: Disk shelf 2 does not have two paths to controller node1.
    Corrective Actions: 1. Halt controller node1 and all controllers attached to disk shelf 2.
                        2. Connect disk shelf 2 to controller node1 via two paths following the rules in the Universal SAS and ACP Cabling Guide.
                        3. Reboot the halted controllers.
                        4. Contact support personnel if the alert persists.
       Possible Effect: Access to disk shelf 2 via controller node1 will be lost with a single
 hardware component failure (e.g. cable, HBA, or IOM failure).
           Acknowledge: false
              Suppress: false
                Policy: DualPathToDiskShelf_Policy
          Acknowledger: -
            Suppressor: -
Additional Information: Shelf uuid: 50:05:0c:c1:02:00:0f:02
                        Shelf id: 2
                        Shelf Name: 4d.shelf2
                        Number of Paths: 1
                        Number of Disks: 6
                        Adapter connected to IOMA:
                        Adapter connected to IOMB: 4d
Alerting Resource Name: Shelf ID 2
----
Sie bestätigen die Benachrichtigung, um anzuzeigen, dass Sie daran arbeiten.

[listing]
----

      cluster1::>system health alert modify -node node1 -alert-id DualPathToDiskShelf_Alert -acknowledge true
----
Sie beheben die Verkabelung zwischen Shelf 2 und Node1 und starten das System anschließend neu. Anschließend wird der Systemzustand erneut überprüft und es zeigt sich, dass der Status `OK`:

[listing]
----

      cluster1::>system health status show
        Status
        ---------------
        OK
----
