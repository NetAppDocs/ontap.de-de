= ONTAP Befehle für die Remote-Knotenverwaltung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können Knotenverwaltungsaufgaben remote ausführen, indem Sie entweder auf den SP zugreifen und SP-CLI-Befehle ausführen oder auf den BMC zugreifen und BMC-CLI-Befehle ausführen. Für mehrere häufig durchgeführte Remote-Knotenverwaltungsaufgaben können auch ONTAP-Befehle von einem anderen Knoten im Cluster verwendet werden. Einige SP- und BMC-Befehle sind plattformspezifisch und möglicherweise auf Ihrer Plattform nicht verfügbar.

Bestimmte Befehlstypen können sowohl in SP als auch in BMC Befehlssätzen verwendet werden. Die Unterschiede zwischen ihnen werden beim Eingeben der Befehlszeile angezeigt.

.Beispiele
* Der `help` Befehl in SP: `SP> help`
* Der `help` Befehl in BMC: `BMC> help`


Erfahren Sie mehr über die `help` Befehlsoptionen für SP und BMC in link:online-help-at-sp-bmc-cli-task.html["Die Hilfeinformationen in der ONTAP SP/BMC CLI anzeigen"].

Erfahren Sie mehr über die entsprechende ONTAP BMC CLI in link:bmc-cli-commands-reference.html["Unterstützte CLI-Befehle für den ONTAP Baseboard Management Controller"].

Die Standby-Stromversorgung bleibt eingeschaltet, um den SP ohne Unterbrechung in Betrieb zu halten. Während des Power-Cycles tritt eine kurze Pause auf, bevor die Stromversorgung wieder eingeschaltet wird.

[NOTE]
====
Die Verwendung dieser Befehle zum Ausschalten oder Neustarten des Knotens kann zu einem unsachgemäßen Herunterfahren des Knotens (auch als _dirty shutdown_ bezeichnet) führen und ist kein Ersatz für ein ordnungsgemäßes Herunterfahren mit dem ONTAP  `system node halt` Befehl.

====

Einen Core Dump erstellen und den Knoten zurücksetzen
 a| 
`system core` [`-f`]

Die  `-f`Option erzwingt die Erstellung eines Core-Dumps und das Zurücksetzen des Knotens.
 a| 
`system core`
 a| 
`system node coredump trigger`

(Erweiterte Berechtigungsstufe)



 a| 
Diese Befehle haben dieselbe Wirkung wie das Drücken der Non-maskable Interrupt (NMI) Taste an einem Knoten, was ein unsauberes Herunterfahren des Knotens verursacht und beim Anhalten des Knotens einen Speicherauszug der Core-Dateien erzwingt. Diese Befehle sind hilfreich, wenn ONTAP auf dem Knoten hängt oder nicht auf Befehle wie  `system node shutdown` reagiert. Die generierten Core-Dump-Dateien werden in der Ausgabe des  `system node coredump show` Befehls angezeigt. Der SP bleibt betriebsbereit, solange die Stromversorgung des Knotens nicht unterbrochen wird.



 a| 
Der Knoten kann mit einem optional angegebenen BIOS-Firmware-Image (primär, Backup oder aktuell) neu gestartet werden, um Probleme wie ein beschädigtes Image des Boot-Geräts des Knotens zu beheben.

{`primary` | `backup` | `current`}

`system node reset` mit dem  `-firmware` {`primary` |  `backup` |  `current`} Parameter
