= ONTAP AutoSupport einrichten
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Sie können einen ONTAP Cluster so konfigurieren, dass AutoSupport-Nachrichten an den technischen Support von NetApp gesendet und E-Mail-Kopien an Ihre interne Supportorganisation verschickt werden. Im Rahmen dessen ist es auch möglich, die Konfiguration zu testen, bevor sie in einer Produktionsumgebung verwendet wird.

.Über diese Aufgabe
Ab ONTAP 9.5 aktivieren und konfigurieren Sie AutoSupport für alle Knoten eines Clusters gleichzeitig. Wenn ein neuer Knoten dem Cluster beitritt, übernimmt der Knoten automatisch dieselbe AutoSupport-Konfiguration. Um dies zu unterstützen, ist der Gültigkeitsbereich des CLI-Befehls `system node autosupport modify` auf Clusterebene. Die `-node` Befehlsoption bleibt aus Gründen der Abwärtskompatibilität erhalten, wird jedoch ignoriert.


NOTE: In ONTAP 9.4 und früheren Versionen ist der Befehl  `system node autosupport modify` knotenspezifisch. Wenn Ihr Cluster mit ONTAP 9.4 oder einer früheren Version ausgeführt wird, muss AutoSupport auf jedem Knoten im Cluster aktiviert und konfiguriert werden.

.Bevor Sie beginnen
Die empfohlene Transportkonfiguration für die Zustellung von AutoSupport-Nachrichten an NetApp ist HTTPS (HTTP mit TLS). Diese Option bietet die robustesten Funktionen und die beste Sicherheit.

Weitere Informationen unter link:requirements-autosupport-reference.html["Vorbereitung auf die Verwendung von AutoSupport"] vor der Konfiguration Ihres ONTAP Clusters.

.Schritte
. Sicherstellen, dass AutoSupport aktiviert ist:
+
[listing]
----
system node autosupport modify -state enable
----
. Wenn der technische Support von NetApp AutoSupport-Nachrichten erhalten soll, ist folgender Befehl zu verwenden:
+
[listing]
----
system node autosupport modify -support enable
----
+
Diese Option muss aktiviert werden, wenn AutoSupport mit AutoSupport OnDemand verwendet werden soll oder wenn große Dateien wie Core-Dump- und Performance-Archivdateien an den technischen Support oder eine bestimmte URL hochgeladen werden sollen.

+

NOTE: AutoSupport OnDemand ist standardmäßig aktiviert und funktionsfähig, wenn es so konfiguriert ist, dass Nachrichten über das HTTPS-Transportprotokoll an den technischen Support gesendet werden.

. Wenn Sie den technischen Support von NetApp für den Empfang von AutoSupport-Nachrichten aktiviert haben, geben Sie an, welches Transportprotokoll für diese Nachrichten verwendet werden soll.
+
Sie können aus den folgenden Optionen wählen:

+
|===


| Wenn Sie möchten... | Anschließend werden die folgenden Parameter des  `system node autosupport modify` Befehls festgelegt... 


 a| 
Das Standard-HTTPS-Protokoll verwenden
 a| 
.. Setzen Sie  `-transport` auf  `https`.
.. Wenn Sie einen Proxy verwenden, setzen Sie  `-proxy-url` auf die URL Ihres Proxys. Diese Konfiguration unterstützt die Kommunikation mit AutoSupport OnDemand und das Hochladen großer Dateien.




 a| 
SMTP verwenden
 a| 
Setzen Sie  `-transport` auf  `smtp`.

Diese Konfiguration unterstützt weder AutoSupport OnDemand noch das Hochladen großer Dateien.

|===
. Wenn Ihre interne Supportorganisation oder ein Supportpartner AutoSupport-Nachrichten erhalten soll, sind die folgenden Maßnahmen erforderlich:
+
.. Die Empfänger in Ihrer Organisation werden durch Festlegen der folgenden Parameter des  `system node autosupport modify` Befehls identifiziert:
+
|===


| Diesen Parameter festlegen... | Dazu... 


 a| 
`-to`
 a| 
Bis zu fünf durch Kommas getrennte individuelle E-Mail-Adressen oder Verteilerlisten in Ihrer internen Supportorganisation, die wichtige AutoSupport-Nachrichten erhalten



 a| 
`-noteto`
 a| 
Bis zu fünf durch Kommas getrennte individuelle E-Mail-Adressen oder Verteilerlisten in Ihrer internen Supportorganisation, die eine gekürzte Version wichtiger AutoSupport-Nachrichten erhalten, die für Mobiltelefone und andere mobile Geräte vorgesehen ist



 a| 
`-partner-address`
 a| 
Bis zu fünf durch Kommas getrennte individuelle E-Mail-Adressen oder Verteilerlisten in Ihrer Support-Partnerorganisation, die alle AutoSupport-Nachrichten erhalten

|===
.. Es kann überprüft werden, ob die Adressen korrekt konfiguriert sind, indem die Ziele mit dem  `system node autosupport destinations show` Befehl aufgelistet werden.


. Wenn im vorherigen Schritt die Empfängeradressen für Ihre interne Supportorganisation konfiguriert wurden oder der SMTP-Transport für Nachrichten an den technischen Support gewählt wurde, erfolgt die SMTP-Konfiguration durch Festlegen der folgenden Parameter des  `system node autosupport modify` Befehls:
+
**  `-mail-hosts` auf einen oder mehrere Mail-Hosts festlegen, durch Kommas getrennt.
+
Es kann maximal fünf festgelegt werden.

+
Sie können für jeden Mail-Host einen Portwert konfigurieren, indem Sie nach dem Mail-Host-Namen einen Doppelpunkt und die Portnummer angeben: zum Beispiel  `mymailhost.example.com:5678`, wobei 5678 der Port für den Mail-Host ist.

** Setzen Sie  `-from`auf die E-Mail-Adresse, von der die AutoSupport-Nachricht gesendet wird.


. DNS konfigurieren.
. Optional können Befehlsoptionen hinzugefügt werden, um bestimmte Einstellungen zu ändern:
+
|===


| Wenn Sie dies tun möchten ... | Anschließend werden die folgenden Parameter des  `system node autosupport modify` Befehls festgelegt... 


 a| 
Private Daten können verborgen werden, indem sensible Daten in den Nachrichten entfernt, maskiert oder codiert werden.
 a| 
Setzen Sie  `-remove-private-data` auf  `true`. Wenn von  `false` zu  `true` gewechselt wird, werden der gesamte AutoSupport-Verlauf und alle zugehörigen Dateien gelöscht.



 a| 
Das Senden von Leistungsdaten in periodischen AutoSupport-Nachrichten beenden
 a| 
Setzen Sie  `-perf` auf  `false`.

|===
. Wenn Sie SMTP zum Zustellen von AutoSupport-Nachrichten an NetApp verwenden, kann optional TLS für eine verbesserte Sicherheit aktiviert werden.
+
.. Die für den neuen Parameter verfügbaren Werte werden angezeigt:
+
[listing]
----
cluster1::> system node autosupport modify -smtp-encryption ?
----
.. TLS für die SMTP-Nachrichtenzustellung aktivieren:
+
[listing]
----
cluster1::> system node autosupport modify -smtp-encryption start_tls
----
.. Aktuelle Konfiguration anzeigen:
+
[listing]
----
cluster1::> system node autosupport show -fields smtp-encryption
----


. Die Gesamtkonfiguration lässt sich mit dem  `system node autosupport show` Befehl und dem  `-node` Parameter überprüfen.
. Die AutoSupport-Funktion kann mit dem  `system node autosupport check show` Befehl überprüft werden.
+
Falls Probleme gemeldet werden, kann mit dem  `system node autosupport check show-details` Befehl weitere Informationen angezeigt werden.

. Es wird geprüft, ob AutoSupport-Nachrichten gesendet und empfangen werden:
+
.. Den  `system node autosupport invoke`Befehl mit dem  `-type`Parameter, der auf  `test`gesetzt ist, verwenden:
+
[listing]
----
cluster1::> system node autosupport invoke -type test -node node1
----
.. Bestätigt wird, dass NetApp Ihre AutoSupport-Nachrichten erhält:
+
[listing]
----
system node autosupport history show -node local
----
+
Der Status der letzten ausgehenden AutoSupport-Nachricht sollte sich schließlich zu `sent-successful` für alle entsprechenden Protokollziele ändern.

.. Optional kann bestätigt werden, dass AutoSupport-Nachrichten an Ihre interne Supportorganisation oder an Ihren Supportpartner gesendet werden, indem die E-Mails jeder Adresse geprüft werden, die für die  `-to`,  `-noteto` oder  `-partner-address` Parameter des  `system node autosupport modify` Befehls konfiguriert wurde.
