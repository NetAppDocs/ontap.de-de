= ONTAP AutoSupport-Einstellungen verwalten
:allow-uri-read: 


[role="lead"]
Mit System Manager lassen sich die Einstellungen für Ihr AutoSupport-Konto verwalten.

Weitere Informationen zu AutoSupport Konfigurationsoptionen, einschließlich Einstellungen, die im System Manager nicht verfügbar sind, sind unter  `system-node-autosupport-modify` in der  https://docs.netapp.com/us-en/ontap-cli/system-node-autosupport-modify.html["ONTAP-Befehlsreferenz"^] zu finden.



[[view-autosupport-settings]]
== Einstellungen für AutoSupport anzeigen

Mit dem System Manager lassen sich die Einstellungen Ihres AutoSupport-Kontos anzeigen.

.Schritte
. Im System Manager auf *Cluster > Einstellungen* klicken.
+
Im Abschnitt *AutoSupport* werden die folgenden Informationen angezeigt:

+
** Status
** Transportprotokoll
** Proxyserver
** Von der E-Mail-Adresse


. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Weitere Optionen*.
+
Zusätzliche Informationen zu den AutoSupport Verbindungs- und E-Mail-Einstellungen werden angezeigt. Auch der Übertragungsverlauf der Nachrichten ist aufgeführt.





[[generate-and-send-autosupport-data]]
== Generieren und Senden von AutoSupport-Daten

Im System Manager kann die Generierung von AutoSupport-Nachrichten initiiert werden, und es besteht die Möglichkeit auszuwählen, von welchem Cluster-Node oder welchen Nodes die Daten erfasst werden.

.Schritte
. Im System Manager *Cluster > Einstellungen* auswählen.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"] und anschließend *Generieren und Senden*.
. Ein Betreff eingeben.
. Das Kästchen unter *Daten sammeln von* gibt die Knoten an, von denen die Daten gesammelt werden.




[[test-the-connection-to-autosupport]]
== Die Verbindung zu AutoSupport testen

Im System Manager kann eine Testnachricht gesendet werden, um die Verbindung zu AutoSupport zu überprüfen.

.Schritte
. Im System Manager auf *Cluster > Einstellungen* klicken.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Verbindung testen*.
. Geben Sie einen Betreff für die Nachricht ein.




[[enable-or-disable-autosupport]]
== Aktivieren oder Deaktivieren von AutoSupport

AutoSupport bietet nachweisliche Geschäftsvorteile für NetApp Kunden, darunter die proaktive Identifizierung möglicher Konfigurationsprobleme und eine beschleunigte Lösung von Supportfällen. AutoSupport ist in neuen Systemen standardmäßig aktiviert. Falls erforderlich, kann mit dem System Manager die Fähigkeit von AutoSupport, den Zustand Ihres Speichersystems zu überwachen und Benachrichtigungen zu senden, deaktiviert werden. AutoSupport kann nach der Deaktivierung wieder aktiviert werden.

.Über diese Aufgabe
Bevor Sie AutoSupport deaktivieren, sollten Sie sich darüber im Klaren sein, dass Sie damit das NetApp Call-Home-System abschalten und die folgenden Vorteile verlieren:

* *Gesundheitsüberwachung*: AutoSupport überwacht den Zustand Ihres Speichersystems und sendet Benachrichtigungen an den technischen Support und Ihre interne Supportorganisation.
* *Automatisierung*: AutoSupport automatisiert die Meldung von Supportfällen. Die meisten Supportfälle werden automatisch erstellt, bevor die Kunden das Problem bemerken.
* *Schnellere Problemlösung*: Systeme, die AutoSupport-Daten senden, haben Supportfälle, die in der Hälfte der Zeit gelöst werden im Vergleich zu Fällen von Systemen, die keine AutoSupport-Daten senden.
* *Schnellere Upgrades*: AutoSupport ermöglicht Self-Service-Workflows für Kunden, wie Versions-Upgrades, Add-ons, Verlängerungen und die Automatisierung von Firmware-Updates im System Manager.
* *Weitere Funktionen*: Bestimmte Funktionen in anderen Tools funktionieren nur, wenn AutoSupport aktiviert ist, beispielsweise einige Workflows in der NetApp Console.


.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Deaktivieren*.
. Wenn AutoSupport wieder aktiviert werden soll, im Abschnitt *AutoSupport* image:../media/icon_kabob.gif["Symbol für Menüoptionen"] auswählen und anschließend *Aktivieren* wählen.




[[suppress-the-generation-of-support-cases]]
== Unterdrücken der Generierung von Supportfällen

Ab ONTAP 9.10.1 kann mit dem System Manager eine Anfrage an AutoSupport gesendet werden, um die Generierung von Supportfällen zu unterdrücken.

.Über diese Aufgabe
Um die Generierung von Supportfällen zu unterdrücken, werden die Knoten und die Anzahl der Stunden angegeben, für die die Unterdrückung erfolgen soll.

Das Unterdrücken von Supportfällen kann besonders hilfreich sein, wenn Sie nicht möchten, dass AutoSupport während der Wartung Ihrer Systeme automatisierte Fälle erstellt.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Supportfallgenerierung unterdrücken* aus.
. Geben Sie die Anzahl der Stunden ein, in denen die Unterdrückung erfolgen soll.
. Wählen Sie die Knoten aus, für die die Unterdrückung erfolgen soll.




[[resume-the-generation-of-support-cases]]
== Die Generierung von Supportfällen fortsetzen

Ab ONTAP 9.10.1 kann mit dem System Manager die Generierung von Supportfällen durch AutoSupport wieder aufgenommen werden, falls diese unterdrückt wurde.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Resume Support Case Generation*.
. Wählen Sie die Knoten aus, für die die Generierung fortgesetzt werden soll.




[[edit-autosupport-settings]]
== Einstellungen für AutoSupport bearbeiten

Mit System Manager lassen sich die Verbindungs- und E-Mail-Einstellungen für Ihr AutoSupport-Konto anpassen.

.Schritte
. *Cluster > Einstellungen* auswählen.
. Im Abschnitt *AutoSupport*, wählen Sie image:../media/icon_kabob.gif["Symbol für Menüoptionen"], dann *Weitere Optionen*.
. Im Abschnitt *Verbindungen* oder im Abschnitt *E-Mail* image:../media/icon_edit.gif["Bearbeiten-Symbol"] können die Einstellungen für den jeweiligen Abschnitt geändert werden.
