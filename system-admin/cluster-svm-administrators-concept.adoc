= Informationen zu den Administratorrollen für ONTAP Cluster und SVM
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Cluster-Administratoren verwalten den gesamten Cluster und die darin enthaltenen Storage Virtual Machines (SVMs, früher bekannt als Vserver). SVM-Administratoren verwalten nur ihre eigenen Daten-SVMs.

Clusteradministratoren können den gesamten Cluster und seine Ressourcen verwalten. Sie können auch Daten-SVMs einrichten und die SVM-Verwaltung an SVM-Administratoren delegieren. Die spezifischen Fähigkeiten, über die Clusteradministratoren verfügen, hängen von ihren Zugriffssteuerungsrollen ab. Standardmäßig verfügt ein Clusteradministrator mit dem Kontonamen oder Rollennamen „admin“ über alle Fähigkeiten zur Verwaltung des Clusters und der SVMs.

SVM-Administratoren können nur ihre eigenen SVM-Speicher und Netzwerkressourcen wie Volumes, Protokolle, LIFs und Dienste verwalten. Die spezifischen Berechtigungen, über die SVM-Administratoren verfügen, hängen von den Zugriffskontrollrollen ab, die von Cluster-Administratoren zugewiesen werden.

[NOTE]
====
Die ONTAP Befehlszeilenschnittstelle (CLI) verwendet weiterhin den Begriff _Vserver_ in der Ausgabe, und  `vserver` als Befehls- oder Parametername wurde nicht geändert.
