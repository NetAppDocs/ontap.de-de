= Arten von ONTAP AutoSupport Nachrichten und deren Inhalt
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
AutoSupport-Meldungen enthalten Statusinformationen zu unterstützten Subsystemen. Zu wissen, was AutoSupport-Meldungen enthalten, kann bei der Interpretation oder Reaktion auf Nachrichten helfen, die per E-Mail empfangen oder auf der Digital Advisor Website angezeigt werden.

|===
| Nachrichtentyp | Art der Daten, die die Nachricht enthält 


 a| 
Ereignisgesteuert
 a| 
Dateien, die kontextsensitive Daten über das spezifische Subsystem enthalten, in dem das Ereignis aufgetreten ist



 a| 
Täglich
 a| 
Protokolldateien



 a| 
Performance
 a| 
Leistungsdaten, die während der letzten 24 Stunden erfasst wurden



 a| 
Wöchentlich
 a| 
Konfigurations- und Statusdaten



 a| 
Ausgelöst durch den  `system node autosupport invoke` Befehl
 a| 
Hängt vom im  `-type` Parameter angegebenen Wert ab:

* `test`Sendet eine vom Benutzer ausgelöste Nachricht mit einigen grundlegenden Daten.
+
Diese Nachricht löst außerdem eine automatische E-Mail-Antwort vom technischen Support an die angegebenen E-Mail-Adressen aus, wobei die  `-to` Option verwendet wird, sodass bestätigt werden kann, dass AutoSupport-Nachrichten empfangen werden.

* `performance` sendet Leistungsdaten.
* `all`sendet eine vom Benutzer ausgelöste Nachricht mit einem vollständigen Datensatz, ähnlich der wöchentlichen Nachricht, einschließlich Fehlerbehebungsdaten aus jedem Subsystem.
+
Der technische Support fordert diese Nachricht in der Regel an.





 a| 
Ausgelöst durch den  `system node autosupport invoke-core-upload` Befehl
 a| 
Core Dump-Dateien für einen Knoten



 a| 
Ausgelöst durch den  `system node autosupport invoke-performance-archive` Befehl
 a| 
Leistungsarchivdateien für einen bestimmten Zeitraum



 a| 
Ausgelöst durch AutoSupport OnDemand
 a| 
AutoSupport OnDemand kann neue oder frühere Nachrichten anfordern:

* Neue Nachrichten können, je nach Art der AutoSupport-Sammlung,  `test`,  `all` oder  `performance` sein.
* Frühere Nachrichten hängen von der Art der erneut gesendeten Nachricht ab.


AutoSupport OnDemand kann neue Nachrichten anfordern, die die folgenden Dateien auf die NetApp Support-Website unter  http://mysupport.netapp.com/["mysupport.netapp.com"^] hochladen:

* Core Dump
* Performance-Archiv


|===
