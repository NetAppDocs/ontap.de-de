= Informationen zu ONTAP Konfigurationssicherungsdateien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Konfigurationssicherungsdateien sind Archivdateien (.7z), die Informationen zu allen konfigurierbaren Optionen enthalten, die für den ordnungsgemäßen Betrieb des Clusters und der darin enthaltenen Nodes erforderlich sind.

Diese Dateien speichern die lokale Konfiguration jedes Knotens sowie die clusterweite replizierte Konfiguration. Konfigurationssicherungsdateien werden verwendet, um die Konfiguration des Clusters zu sichern und wiederherzustellen.

Es gibt zwei Arten von Konfigurationssicherungsdateien:

* *Node-Konfigurationssicherungsdatei*
+
Jeder fehlerfreie Knoten im Cluster verfügt über eine Sicherungsdatei der Knotenkonfiguration, die alle Konfigurationsinformationen und Metadaten enthält, die für den fehlerfreien Betrieb des Knotens im Cluster erforderlich sind.

* *Sicherungsdatei für die Cluster-Konfiguration*
+
Diese Dateien enthalten ein Archiv aller Node-Konfigurationssicherungsdateien im Cluster sowie die replizierten Clusterkonfigurationsinformationen (die replizierte Datenbank oder RDB-Datei). Clusterkonfigurationssicherungsdateien ermöglichen die Wiederherstellung der Konfiguration des gesamten Clusters oder eines beliebigen Node im Cluster. Die Clusterkonfigurationssicherungspläne erstellen diese Dateien automatisch und speichern sie auf mehreren Nodes im Cluster.



[NOTE]
====
Konfigurationssicherungsdateien enthalten ausschließlich Konfigurationsinformationen. Sie enthalten keine Benutzerdaten. Informationen zur Wiederherstellung von Benutzerdaten sind unter link:../data-protection/index.html["Datensicherung"] zu finden.
