= Eine ONTAP Inter-SVM oder Intra-SVM HDFA-Option bestimmen
:allow-uri-read: 


[role="lead"]
Nachdem die Dichte Ihrer HDFs ermittelt wurde, kann entschieden werden, ob auf die HDFs über NFS zugegriffen wird, und es stehen Informationen zu Inter-SVM HDFA- und Intra-SVM HDFA-Optionen zur Verfügung.


TIP: Wenn ausschließlich SMB-Clients auf die HDFs zugreifen, sollten alle HDFs in einer einzigen SVM erstellt werden. In der Windows-Clientkonfiguration ist ersichtlich, wie DFS-Ziele für den Lastausgleich verwendet werden können.



[[inter-svm-hdfa-deployment]]
== Inter-SVM HDFA Bereitstellung

Ein HDFA zwischen SVMs erfordert, dass für jede HDF im HDFA eine SVM erstellt wird. Dadurch haben alle HDFs innerhalb des HDFA denselben Verbindungspfad, was die Konfiguration auf der Clientseite erleichtert.

Im <<Figure-1,Abbildung 1>> Beispiel befindet sich jede HDF in ihrer eigenen SVM. Dies ist eine inter-SVM-HDFA-Bereitstellung. Jede HDF hat einen Verbindungspfad von /hotspots. Außerdem hat jede IP einen DNS-A-Record mit dem Hostnamen cache. Diese Konfiguration nutzt DNS-Round-Robin, um die Mounts über die verschiedenen HDFs hinweg im Lastausgleich zu verteilen.

image:flexcache-hotspot-hdfa-one-hdf-per-svm.png["Abbildung 1: 4x1x4 Inter-SVM HDFA Konfiguration"]



[[intra-svm-hdfa-deployment]]
== Intra-SVM HDFA Bereitstellung

Eine Intra-SVM erfordert, dass jede HDF einen eindeutigen Verbindungspfad besitzt, jedoch befinden sich alle HDFs in einer einzigen SVM. Diese Konfiguration ist in ONTAP einfacher, da nur eine SVM benötigt wird, erfordert jedoch eine erweiterte Konfiguration auf der Linux-Seite mit  `autofs` und der Platzierung von Daten-LIFs in ONTAP.

Im <<Figure-2,Abbildung 2>> Beispiel befindet sich jede HDF in derselben SVM. Dies ist eine intra-SVM-HDFA-Bereitstellung und erfordert eindeutige Verbindungspfade. Damit der Lastausgleich ordnungsgemäß funktioniert, ist für jede IP ein eindeutiger DNS-Name zu erstellen und die Daten-LIFs, auf die der Hostname aufgelöst wird, nur auf den Knoten zu platzieren, auf denen sich die HDF befindet. Außerdem ist  `autofs` mit mehreren Einträgen zu konfigurieren, wie in link:flexcache-hotspot-remediation-client-config.html["Linux-Client-Konfiguration"] erläutert.
