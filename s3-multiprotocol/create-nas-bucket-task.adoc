= Einen ONTAP S3 NAS Bucket erstellen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein S3 NAS Bucket ist eine Zuordnung zwischen einem S3 Bucket-Namen und einem NAS-Pfad. S3 NAS Buckets ermöglichen S3-Zugriff auf jeden Teil eines SVM-Namensraums mit vorhandenen Volumes und Verzeichnisstruktur.

.Bevor Sie beginnen
* Ein S3-Objektserver ist in einer SVM konfiguriert, die NAS-Daten enthält.
* Die NAS-Daten entsprechen dem link:nas-data-requirements-client-access-reference.html["Anforderungen für den S3-Clientzugriff"].


.Über diese Aufgabe
Sie können S3 NAS-Buckets so konfigurieren, dass beliebige Dateien und Verzeichnisse innerhalb des Stammverzeichnisses der SVM angegeben werden.

Sie können auch Bucket-Richtlinien festlegen, die den Zugriff auf NAS-Daten basierend auf einer beliebigen Kombination dieser Parameter erlauben oder verbieten:

* Dateien und Verzeichnisse
* Benutzer- und Gruppenberechtigungen
* S3 Operationen


Beispielsweise möchten Sie möglicherweise eine Bucket-Richtlinie einrichten, die einer großen Gruppe von Benutzern nur Lesezugriff auf Daten gewährt, und eine andere Bucket-Richtlinie, die es einer begrenzten Gruppe erlaubt, Operationen an einer Teilmenge dieser Daten durchzuführen.

Ab ONTAP 9.18.1 können Sie NAS-Buckets erstellen, die es Anwendungen ermöglichen, auf Daten auf FlexCache-Volumes über das S3-Protokoll zuzugreifen. Alle Knoten im Cluster müssen ONTAP 9.18.1 oder neuer ausführen. Bevor Sie mit dem S3-Protokoll auf ein FlexCache-Volume zugreifen können, müssen Sie die  `-is-s3-enabled` Option auf  `true` link:../flexcache/create-volume-task.html["auf dem FlexCache Volume"] setzen. Der Parameter ist standardmäßig auf  `false` gesetzt.

Ab ONTAP 9.17.1 ist es möglich, einen S3 NAS-Bucket direkt mit einem Volume statt mit dem Verbindungspfad zu verknüpfen. Standardmäßig ist ein S3-Bucket auf einem NAS-Volume mit einem Verbindungspfad verknüpft, was jederzeit von einem ONTAP-Administrator geändert werden kann. Diese Änderungen können potenziell den Betrieb von S3-Buckets beeinträchtigen. Ab ONTAP 9.17.1 kann die  `-is-nas-path-mutable false` Option zusammen mit dem  `vserver object-store-server bucket create` Befehl in der ONTAP CLI verwendet werden, um die Verknüpfung des S3 NAS-Buckets mit einem Volume zu ermöglichen. Standardmäßig  `-is-nas-path-mutable` ist  `true` eingestellt.

Da es sich bei S3 NAS "buckets" um Mappings und nicht um S3 buckets handelt, gelten die folgenden Eigenschaften von Standard S3 buckets nicht für S3 NAS buckets.

* *aggr-list \ aggr-list-multiplier \ storage-service-level \ volume \ size \ exclude-aggr-list \ qos-policy-group* + Es werden keine Volumes oder Qtree erstellt, wenn S3 NAS Buckets konfiguriert werden.
* *role \ is -protected \ is -protected-on-ontap \ is -protected-on-cloud* + S3 NAS Buckets werden nicht mit SnapMirror S3 geschützt oder gespiegelt, sondern nutzen stattdessen den regulären SnapMirror-Schutz auf Volume-Granularität.
* *Versionsstatus* + NAS-Volumes verfügen üblicherweise über Snapshot-Technologie zum Speichern verschiedener Versionen. Allerdings ist die Versionierung in S3 NAS-Buckets derzeit nicht verfügbar.
* *logical-used \ object-count* + Entsprechende Statistiken sind für NAS-Volumes über die Volume-Befehle verfügbar.
* *Multipart-Objekte* + Ab ONTAP 9.16.1 werden Multipart-Objekte in S3-NAS-Buckets unterstützt, wenn link:../flexgroup/enable-adv-capacity-flexgroup-task.html["fortschrittlicher Kapazitätsausgleich"] auf dem zugrunde liegenden FlexGroup Volume aktiviert ist. Erweitertes Kapazitätsbalancing kann nur auf FlexGroup Volumes aktiviert werden. Auf FlexVol Volumes kann es nicht aktiviert werden.


.Schritte
Sie können den System Manager oder die ONTAP CLI verwenden, um einen NAS Bucket zu erstellen.

[role="tabbed-block"]
====
.System Manager
--
Fügen Sie einen neuen S3 NAS-Bucket auf einer NAS-fähigen Storage-VM hinzu.

. Klicken Sie auf *Speicher > Buckets* und anschließend auf *Hinzufügen*.
. Geben Sie einen Namen für den S3 NAS-Bucket ein und wählen Sie die Storage-VM aus, geben Sie keine Größe ein, und klicken Sie dann auf *Weitere Optionen*.
. Geben Sie einen gültigen Pfadnamen ein oder klicken Sie auf „Durchsuchen“, um aus einer Liste gültiger Pfadnamen auszuwählen. + Bei Eingabe eines gültigen Pfadnamens werden Optionen, die für die S3 NAS Konfiguration nicht relevant sind, ausgeblendet.
. Wenn Sie bereits S3-Benutzer NAS-Benutzern zugeordnet und Gruppen erstellt haben, können Sie deren Berechtigungen konfigurieren und dann auf *Speichern* klicken. + Die S3-Benutzer müssen bereits NAS-Benutzern zugeordnet sein, bevor in diesem Schritt Berechtigungen konfiguriert werden.


Andernfalls auf *Speichern* klicken, um die S3 NAS-Bucket-Konfiguration abzuschließen.

--
.CLI
--
. Erstellen eines S3 NAS-Buckets in einer SVM, die NAS-Dateisysteme enthält. +
`vserver object-store-server bucket create -vserver <svm_name> -bucket <bucket_name> -type nas -nas-path <junction_path> -is-nas-path-mutable true|false [-comment <text>]`
+
Beispiel 1: Erstellen eines S3 NAS Buckets

+
`cluster1::> vserver object-store-server bucket create -bucket testbucket -type nas -path /vol1`

+
Beispiel 2: Erstellen eines S3 NAS-Buckets und Verknüpfen des Buckets mit einem Volume

+
`vserver object-store-server bucket create -vserver vs1 -bucket nasbucket1 -type nas -nas-path /pathA/dir1 -is-nas-path-mutable false`
