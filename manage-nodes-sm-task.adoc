Mit dem System Manager können Sie Knoten zu einem Cluster hinzufügen und umbenennen. Sie können außerdem Knoten neu starten, übernehmen und wieder zurückgeben.



[[add-nodes-to-a-cluster]]
== Knoten zu einem Cluster hinzufügen

Sie können die Größe und Leistungsfähigkeit Ihres Clusters durch Hinzufügen neuer Knoten erhöhen.

.Bevor Sie beginnen
Die neuen Knoten sollten bereits mit dem Cluster verkabelt sein.

.Über diese Aufgabe
Für die Arbeit mit System Manager in ONTAP 9.8 und höher oder ONTAP 9.7 existieren separate Prozesse.

[role="tabbed-block"]
====
.ONTAP 9.8 und neuere Verfahren
--
[[add-nodes-cluster-98]]
*Hinzufügen von Knoten zu einem Cluster mit System Manager (ONTAP 9.8 und höher)*

.Schritte
. *Cluster > Übersicht* auswählen.
+
Die neuen Controller werden als Knoten angezeigt, die mit dem Cluster-Netzwerk verbunden sind, aber nicht im Cluster sind.

. Wählen Sie *Add*.
+
** Die Knoten werden dem Cluster hinzugefügt.
** Speicherplatz wird implizit zugewiesen.




--
.ONTAP 9.7 Verfahren
--
[[add-nodes-cluster-97]]
*Hinzufügen von Knoten zu einem Cluster mit System Manager (ONTAP 9.7)*

.Schritte
. *(Zur klassischen Version zurückkehren)* auswählen.
. *Konfigurationen > Clustererweiterung* auswählen.
+
System Manager erkennt die neuen Knoten automatisch.

. *Zur neuen Benutzeroberfläche wechseln* auswählen.
. *Cluster > Übersicht* zeigt die neuen Nodes an.


--
====


[[shut-down-reboot-or-edit-service-processor]]
== Dienstprozessor herunterfahren, neu starten oder bearbeiten

Wenn ein Node neu gestartet oder heruntergefahren wird, führt der HA-Partner automatisch eine Übernahme durch.


NOTE: Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/administer/reboot-take-over-give-back-nodes.html["diese Schritte"^] zu befolgen, um einen Node herunterzufahren und neu zu starten. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

.Schritte
. *Cluster > Übersicht* auswählen.
. Unter *Knoten* image:icon_kabob.gif["Symbol für Menüoptionen"] auswählen.
. Den Node auswählen und dann *Herunterfahren*, *Neustart* oder *Service Processor bearbeiten* auswählen.
+
Wenn ein Node neu gestartet wurde und auf die Rückgabe wartet, ist die Option *Giveback* ebenfalls verfügbar.

+
Wenn Sie *Dienstprozessor bearbeiten* auswählen, können Sie *Manuell* wählen, um die IP-Adresse, Subnetzmaske und das Gateway einzugeben, oder Sie können *DHCP* für dynamische Hostkonfiguration wählen.





[[rename-nodes]]
== Knoten umbenennen

Ab ONTAP 9.14.1 kann ein Node-Name von der Cluster-Übersichtsseite aus umbenannt werden.


NOTE: Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/administer/rename-nodes.html["diese Schritte"^] zu folgen, um einen Node umzubenennen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

.Schritte
. *Cluster* auswählen. Die Cluster Übersichtseite wird angezeigt.
. Nach unten zum Abschnitt *Knoten* scrollen.
. Neben dem Knoten, den Sie umbenennen möchten, image:icon_kabob.gif["Symbol für Menüoptionen"] auswählen und anschließend *Umbenennen* wählen.
. Den Node-Name ändern und anschließend *Umbenennen* auswählen.
