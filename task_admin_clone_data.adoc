= Volumes und LUNs für Testzwecke mit ONTAP System Manager klonen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Sie können Volumes und LUNs klonen, um temporäre, beschreibbare Kopien für Testzwecke zu erstellen. Die Klone spiegeln den aktuellen, zeitpunktgenauen Zustand der Daten wider. Klone können auch verwendet werden, um zusätzlichen Benutzern Zugriff auf Daten zu ermöglichen, ohne ihnen Zugriff auf Produktionsdaten zu geben.

.Bevor Sie beginnen
Die FlexClone-Lizenz sollte link:system-admin/install-license-task.html["installiert"] auf dem Speichersystem vorhanden sein.



[[cloning-a-volume]]
== Klonen eines Volumens

Einen Klon eines Volumes wie folgt erstellen:

.Schritte
. *Speicher > Volumes* auswählen.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben den Namen des Volumes, das Sie klonen möchten.
. *Clone* aus der Liste auswählen.
. Geben Sie einen Namen für den Klon an und schließen Sie die übrigen Auswahlen ab.
. Auf *Klonen* klicken und überprüfen, ob der Volume-Klon in der Liste der Volumes erscheint.


Alternativ ist es möglich, ein Volume aus der *Übersicht* zu klonen, die angezeigt wird, wenn die Volume-Details angezeigt werden.



[[cloning-a-lun]]
== Klonen einer LUN

Sie können Kopien Ihrer LUNs erstellen, indem Sie die LUNs im aktiven Volume klonen. Diese FlexClone LUNs sind lesbare und beschreibbare Kopien der ursprünglichen LUNs im aktiven Volume.

Eine speicherplatzreservierte FlexClone LUN benötigt genauso viel Speicherplatz wie die speicherplatzreservierte übergeordnete LUN. Wenn die FlexClone LUN nicht speicherplatzreserviert ist, muss sichergestellt sein, dass das Volume über ausreichend Speicherplatz verfügt, um Änderungen an der FlexClone LUN aufzunehmen.


NOTE: Dieses Verfahren gilt für FAS, AFF und ASA Systeme. Bei einem ASA r2 System (ASA A1K, ASA A90, ASA A70, ASA A50, ASA A30, ASA A20 oder ASA C30) ist link:https://docs.netapp.com/us-en/asa-r2/manage-data/data-cloning.html["diese Schritte"^] zum Klonen von Daten zu folgen. ASA r2 Systeme bieten eine vereinfachte ONTAP Erfahrung speziell für ausschließlich SAN-basierte Kunden.

.Schritte
[role="tabbed-block"]
====
.System Manager
--
. *Speicher > LUNs* auswählen.
. Klicken Sie image:icon_kabob.gif["Symbol für Menüoptionen"] neben dem Namen der LUN, die Sie klonen möchten.
. *Clone* aus der Liste auswählen.
. Geben Sie einen Namen für den Klon an und schließen Sie die übrigen Auswahlen ab.
. Auf *Klonen* klicken und überprüfen, ob der LUN-Klon in der Liste der LUNs erscheint.


Alternativ ist es möglich, eine LUN aus der *Übersicht* zu klonen, die beim Anzeigen der LUN-Details angezeigt wird.

Wenn ein LUN-Klon erstellt wird, aktiviert System Manager automatisch die Löschung des Klons, sobald Speicherplatz benötigt wird.

--
.CLI
--
. Vor dem Klonen sollte überprüft werden, ob die LUNs keiner Initiatorgruppe zugeordnet oder beschrieben sind.
. Mit dem  `lun show`Befehl lässt sich überprüfen, ob die LUN existiert.
+
`lun show -vserver vs1`

+
[listing]
----
Vserver  Path              State     Mapped    Type     Size
-------- ----------------- --------- --------- -------- -------
vs1      /vol/vol1/lun1    online   unmapped  windows  47.07MB
----
. Mit dem  `volume file clone create` Befehl wird die FlexClone LUN erstellt.
+
`volume file clone create -vserver vs1 -volume vol1 -source-path lun1 -destination-path/lun1_clone`

+
Wenn die FlexClone LUN für die automatische Löschung verfügbar sein soll, schließen Sie  `-autodelete true` ein. Wenn diese FlexClone LUN in einem Volume mit Semi-Thick Provisioning erstellt wird, muss die automatische Löschung für alle FlexClone LUNs aktiviert sein.

. Mit dem  `lun show`Befehl lässt sich überprüfen, ob eine LUN erstellt wurde.
+
`lun show -vserver vs1`

+
[listing]
----

Vserver  Path                    State    Mapped   Type     Size
-------- ----------------------- -------- -------- -------- ------
vs1      /vol/volX/lun1          online   unmapped windows  47.07MB
vs1      /vol/volX/lun1_clone    online   unmapped windows  47.07MB
----
