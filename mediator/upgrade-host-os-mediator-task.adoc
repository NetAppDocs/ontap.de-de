= Aktualisieren Sie das Host-Betriebssystem und installieren oder aktualisieren Sie ONTAP Mediator neu.
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um das Host-Betriebssystem für ONTAP Mediator mit installiertem SCST auf eine neuere Version zu aktualisieren, müssen Sie ONTAP Mediator deinstallieren. Nach der Aktualisierung des Betriebssystems können Sie Ihre aktuelle Version von ONTAP Mediator neu installieren oder auf eine neuere Version aktualisieren.

.Über diese Aufgabe
* Diese Aufgabe gilt für ONTAP Mediator-Versionen mit installiertem SCST. Wenn Sie ONTAP Mediator 1.12 oder höher mit Unterstützung nur für HTTPS installiert haben, ist das SCST-Paket nicht installiert und Sie können das Host-Betriebssystem aktualisieren, ohne ONTAP Mediator zu beeinträchtigen. Anweisungen zur Aktualisierung des Betriebssystems finden Sie in der Dokumentation des Host-Betriebssystemanbieters. Nachdem Sie das Host-Betriebssystem aktualisiert haben, falls Sie Ihre Version von ONTAP Mediator aktualisieren müssen, lesen Sie link:install-mediator-pkg-task.html["Das ONTAP Mediator Installationspaket installieren"].
* Vor dem Upgrade des Host-Betriebssystems für ONTAP Mediator mit dem Tool leapp-upgrade ONTAP Mediator deinstallieren. Das Tool prüft in den registrierten Repositories auf neue RPM-Versionen.
* Der ONTAP Mediator Installer installiert eine .rpm-Datei, die das Tool leapp-upgrade in die Suche einbezieht. Da der Installer die Datei entpackt, anstatt sie aus einem registrierten Repository herunterzuladen, kann das Tool kein Upgrade finden. Das Tool leapp-upgrade ist für die Deinstallation des Pakets zu verwenden.


.Schritte
. Die Protokolldateien sichern:
+
....
[rootmediator-host ~]# tar -czf ontap_mediator_file_backup.tgz -C /opt/netapp/lib/ontap_mediator ./log ./ontap_mediator/server_config/ontap_mediator.user_config.yaml
[rootmediator-host ~]# tar -tf ontap_mediator_file_backup.tgz
./log/
./log/ontap_mediator.log
./log/scstadmin.log
./log/ontap_mediator_stdout.log
./log/ontap_mediator_requests.log
./log/install_20230419134611.log
./log/scst.log
./log/ontap_mediator_syslog.log
./ontap_mediator/server_config/ontap_mediator.user_config.yaml
[rootmediator-host ~]#
....
. Upgrade mit dem leapp-upgrade Tool durchführen:
+
....
[rootmediator-host ~]# leapp preupgrade --target 8.4
  ..<snip upgrade checks>..
  ..<fix issues found>..
[rootmediator-host ~]# leapp upgrade --target 8.4
  ..<snip upgrade>..
[rootmediator-host ~]# cat /etc/os-release | head -2
NAME="Red Hat Enterprise Linux"
VERSION="8.4 (Ootpa)"
[rootmediator-host ~]#
....
. ONTAP Mediator neu installieren:
+

NOTE: Die restlichen Schritte sollten unmittelbar nach der Neuinstallation von ONTAP Mediator erfolgen, um einen Verlust von Protokolldateien zu verhindern.

+
....
[rootmediator-host ~]# ontap-mediator-1.12.1/ontap-mediator-1.12.1

ONTAP Mediator: Self Extracting Installer

  ..<snip installation>..
[rootmediator-host ~]#
....
. ontap_mediator anhalten:
+
....
[rootmediator-host ~]# systemctl stop ontap_mediator
[rootmediator-host ~]#
....
. Die Protokolldateien ersetzen:
+
....
[rootmediator-host ~]# tar -xf ontap_mediator_file_backup.tgz -C /opt/netapp/lib/ontap_mediator
[rootmediator-host ~]#
....
. ontap_mediator starten:
+
....
[rootmediator-host ~]# systemctl start ontap_mediator
[rootmediator-host ~]#
....
. Alle ONTAP Cluster werden wieder mit dem aktualisierten ONTAP Mediator verbunden:

--
