= Lasten werden durch das Verschieben von LUNs mit ONTAP System Manager ausgeglichen
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Sie können eine LUN auf ein anderes Volume innerhalb der Storage-VM verschieben, um die Last auszugleichen, oder sie auf ein Volume mit einem höheren Performance-Servicelevel verschieben, um die Leistung zu verbessern.



[[move-restrictions]]
== Verschiebungsbeschränkungen

* Eine LUN kann nicht innerhalb desselben Volumes in einen Qtree verschoben werden.
* Eine mit der CLI aus einer Datei erstellte LUN kann nicht mit System Manager verschoben werden.
* LUNs, die online sind und Daten bereitstellen, können nicht verschoben werden.
* LUNs können nicht verschoben werden, wenn der zugewiesene Speicherplatz im Zielvolume die LUN nicht aufnehmen kann (auch wenn Autogrow für das Volume aktiviert ist).
* LUNs auf SnapLock Volumes können nicht mit System Manager verschoben werden.


image:workflow_balance_load_by_moving_luns.gif["Workflow-Zusammenfassung: 1 LUN auswählen 2 Vorhandenes Volume oder Qtree zum Verschieben auswählen 3 Verschieben auswählen"]

.Schritte
. *Speicher > LUNs* auswählen.
. Wählen Sie die LUN aus, die Sie verschieben möchten, und klicken Sie auf *Verschieben*.
. Wählen Sie ein vorhandenes Volume aus, auf das Sie die LUN verschieben möchten. Wenn das Volume Qtrees enthält, wählen Sie den Qtree aus.



NOTE: Während des Verschiebevorgangs wird die LUN sowohl auf dem Quell- als auch auf dem Zielvolume angezeigt.
