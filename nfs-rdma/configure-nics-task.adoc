= Netzwerkkarten für NFS über RDMA konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
NFS über RDMA erfordert eine NIC Konfiguration sowohl für das Client-System als auch für die Speicherplattform.



[[storage-platform-configuration]]
== Konfiguration der Speicherplattform

Informationen zur Unterstützung von Ziel-RNICs finden sich in der  https://hwu.netapp.com/["NetApp Hardware Universe"^].

Bei Verwendung einer Hochverfügbarkeitskonfiguration (HA) müssen beide Knoten dieselbe RNIC verwenden, um RDMA-Failover zu unterstützen. Die NIC muss RoCE-fähig sein.

* Ab ONTAP 9.10.1 kann eine Liste der RDMA Offload-Protokolle mit dem Befehl angezeigt werden:
+
[source, cli]
----
network port show -rdma-protocols roce
----
+
Weitere Informationen zu  `network port show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-show.html["ONTAP-Befehlsreferenz"^].

* Ab ONTAP 9.16.1 wird die Verwendung der Prioritätsflusssteuerung (PFC) empfohlen. PFC wird mit dem  `network port modify` Befehl konfiguriert:
+
[source, cli]
----
network port modify -node <nodename> -port <portname> -flowcontrol-admin pfc -pfc-queues-admin 3
----
* Vor ONTAP 9.16.1 wird die Verwendung der standardmäßigen globalen Flusssteuerung (GFC) empfohlen. Wenn die Flusssteuerungseinstellung geändert wurde, kann die GFC mit dem  `network port modify` Befehl konfiguriert werden:
+
[source, cli]
----
network port modify -node <nodename> -port <portname> -flowcontrol-admin full
----
+
Weitere Informationen zu  `network port modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-port-modify.html["ONTAP-Befehlsreferenz"^].





[[client-system-configuration]]
== Client-Systemkonfiguration

Informationen zu unterstützten Client-Betriebssystemen für NFS über RDMA sind in der  https://imt.netapp.com/matrix/["NetApp Interoperabilitätsmatrix (IMT)"^] zu finden. Für RoCE v2 unterstützte RNICs wird auf die jeweilige RNIC-Herstellerdokumentation verwiesen.

Obwohl Client und Server direkt verbunden werden können, wird die Verwendung von Switches für eine verbesserte Failover-Performance empfohlen.

Der Client, der Server, alle Switches und alle Ports an den Switches müssen mit Jumbo Frames konfiguriert werden. Die Flusssteuerungskonfiguration auf den Clients und Switches sollte mit der Flusssteuerungskonfiguration von ONTAP übereinstimmen. Ab ONTAP 9.16.1 gilt es als Best Practice, die priorisierte Flusssteuerung auf ONTAP, den Switches und den Clients zu aktivieren und zu konfigurieren. Vor ONTAP 9.16.1 wird empfohlen, die globale Flusssteuerung zu verwenden.

Nachdem diese Konfiguration bestätigt wurde, lässt sich der NFS-Export mithilfe von RDMA einbinden.

[role="tabbed-block"]
====
.System Manager
--
Es ist erforderlich, ONTAP 9.12.1 oder höher zu verwenden, um Netzwerkschnittstellen mit NFS über RDMA mithilfe von System Manager zu konfigurieren.

.Schritte
. Es kann überprüft werden, ob RDMA unterstützt wird. Unter *Network > Ethernet Ports* lässt sich in der Gruppenansicht der entsprechende Node auswählen. Beim Erweitern des Node ist im Feld *RDMA protocols* für den jeweiligen Port zu sehen: Der Wert *RoCE* zeigt an, dass RDMA unterstützt wird, ein Bindestrich (*-*) weist darauf hin, dass es nicht unterstützt wird.
. Zum Hinzufügen eines VLANs *+ VLAN* auswählen. Den entsprechenden Knoten auswählen. Im Dropdown-Menü *Port* wird bei den verfügbaren Ports der Text *RoCE Enabled* angezeigt, wenn diese RDMA unterstützen. Es wird kein Text angezeigt, wenn sie RDMA nicht unterstützen.
. Im Arbeitsablauf in xref:../task_nas_enable_linux_nfs.html[NAS-Speicher für Linux-Server mit NFS bereitstellen] wird die Konfiguration eines neuen NFS Servers beschrieben.
+
Beim Hinzufügen von Netzwerkschnittstellen besteht die Möglichkeit, *Use RoCE ports* auszuwählen. Diese Option ist für alle Netzwerkschnittstellen zu wählen, die NFS über RDMA verwenden sollen.



--
.CLI
--
. Es kann mit folgendem Befehl überprüft werden, ob der RDMA-Zugriff auf dem NFS-Server aktiviert ist:
+
`vserver nfs show-vserver <SVM_name>`

+
Standardmäßig `-rdma` sollte diese Funktion aktiviert sein. Falls sie nicht aktiviert ist, kann der RDMA-Zugriff auf dem NFS-Server aktiviert werden:

+
`vserver nfs modify -vserver <SVM_name> -rdma enabled`

. Den Client über NFSv4.0 über RDMA einbinden:
+
.. Die Eingabe für den Parameter „proto“ hängt von der Server-IP-Protokollversion ab. Handelt es sich um IPv4, verwenden Sie  `proto=rdma`. Handelt es sich um IPv6, verwenden Sie  `proto=rdma6`.
.. Der NFS-Zielport ist als  `port=20049` anstelle des Standardports 2049 anzugeben:
+
`mount -o vers=4,minorversion=0,proto=rdma,port=20049 <Server_IP_address>:/<volume_path> <mount_point>`



. *OPTIONAL*: Falls Sie den Client aushängen müssen, führen Sie den Befehl  `unmount <mount_path>` aus.
