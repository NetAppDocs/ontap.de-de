= LIFs für NFS über RDMA konfigurieren
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Um NFS über RDMA zu verwenden, müssen Ihre LIFs (Netzwerkschnittstellen) RDMA-kompatibel konfiguriert sein. Sowohl die LIF als auch ihr Failover-Paar müssen RDMA unterstützen können.



[[create-a-new-lif]]
== Eine neue LIF erstellen

[role="tabbed-block"]
====
.System Manager
--
Für die Erstellung einer Netzwerkschnittstelle für NFS über RDMA mit System Manager ist ONTAP 9.12.1 oder höher erforderlich.

.Schritte
. *Netzwerk > Übersicht > Netzwerkschnittstellen* auswählen.
. Wählen Sie image:icon_add.gif["Symbol hinzufügen"].
. Wenn Sie *NFS,SMB/CIFS,S3* auswählen, besteht die Möglichkeit, *RoCE-Ports zu verwenden*. Das Kontrollkästchen für *RoCE-Ports verwenden* kann ausgewählt werden.
. Die Storage-VM und der Home-Node werden ausgewählt. Ein **Name**, eine **IP-Adresse** und eine **Subnetzmaske** werden zugewiesen.
. Nach Eingabe der IP-Adresse und Subnetzmaske filtert System Manager die Liste der Broadcast-Domänen auf diejenigen, die über RoCE-fähige Ports verfügen. Eine Broadcast-Domäne kann ausgewählt werden. Ein Gateway kann optional hinzugefügt werden.
. *Speichern* auswählen.


--
.CLI
--
.Schritte
. Eine LIF erstellen:
+
`network interface create -vserver _SVM_name_ -lif _lif_name_ -service-policy _service_policy_name_ -home-node _node_name_ -home-port _port_name_ {-address _IP_address_ -netmask _netmask_value_ | -subnet-name _subnet_name_} -firewall-policy _policy_name_ -auto-revert {true|false} -rdma-protocols roce`

+
** Die Dienstrichtlinie muss entweder default-data-files oder eine benutzerdefinierte Richtlinie sein, die den data-nfs Netzwerkschnittstellendienst enthält.
** Der  `-rdma-protocols`Parameter akzeptiert eine Liste, die standardmäßig leer ist. Wenn  `roce`als Wert hinzugefügt wird, kann die LIF nur auf Ports konfiguriert werden, die RoCE Offload unterstützen, was sich sowohl auf die LIF-Migration als auch auf das Failover auswirkt.




--
====


[[modify-a-lif]]
== Eine LIF modifizieren

[role="tabbed-block"]
====
.System Manager
--
Für die Erstellung einer Netzwerkschnittstelle für NFS über RDMA mit System Manager ist ONTAP 9.12.1 oder höher erforderlich.

.Schritte
. *Netzwerk > Übersicht > Netzwerkschnittstellen* auswählen.
. Wählen Sie image:icon_kabob.gif["Symbol für Menüoptionen"] *> Bearbeiten* neben der Netzwerkschnittstelle, die geändert werden soll.
. *RoCE Ports verwenden* aktivieren, um NFS über RDMA zu ermöglichen, oder das Kontrollkästchen deaktivieren, um es zu deaktivieren. Wenn sich die Netzwerkschnittstelle an einem RoCE-fähigen Port befindet, erscheint neben *RoCE Ports verwenden* ein Kontrollkästchen.
. Die übrigen Einstellungen können nach Bedarf angepasst werden.
. *Speichern* bestätigt die vorgenommenen Änderungen.


--
.CLI
--
. Sie können den Status Ihrer LIFs mit dem  `network interface show` Befehl prüfen. Die Service-Richtlinie muss den data-nfs Netzwerkschnittstellendienst enthalten. Die  `-rdma-protocols` Liste sollte  `roce` enthalten. Wenn eine dieser Bedingungen nicht zutrifft, sollte die LIF angepasst werden.
+
Weitere Informationen zu  `network interface show` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-show.html["ONTAP-Befehlsreferenz"^].

. Zum Ändern der LIF den folgenden Befehl ausführen:
+
`network interface modify vserver _SVM_name_ -lif _lif_name_ -service-policy _service_policy_name_ -home-node _node_name_ -home-port _port_name_ {-address _IP_address_ -netmask _netmask_value_ | -subnet-name _subnet_name_} -firewall-policy _policy_name_ -auto-revert {true|false} -rdma-protocols roce`

+
Weitere Informationen zu  `network interface modify` finden sich in der link:https://docs.netapp.com/us-en/ontap-cli/network-interface-modify.html["ONTAP-Befehlsreferenz"^].




NOTE: Wenn eine LIF so geändert wird, dass sie ein bestimmtes Offload-Protokoll erfordert, während die LIF derzeit keinem Port zugewiesen ist, der dieses Protokoll unterstützt, tritt ein Fehler auf.

--
====


[[migrate-a-lif]]
== Ein LIF migrieren

ONTAP ermöglicht auch die Migration von Netzwerkschnittstellen (LIFs) zur Verwendung von NFS über RDMA. Bei dieser Migration muss sichergestellt werden, dass der Zielport RoCE-fähig ist. Ab ONTAP 9.12.1 kann dieses Verfahren im System Manager abgeschlossen werden. Bei der Auswahl eines Zielports für die Netzwerkschnittstelle wird im System Manager angegeben, ob die Ports RoCE-fähig sind.

Eine Migration einer LIF zu einer NFS over RDMA-Konfiguration ist nur unter folgenden Bedingungen möglich:

* Es handelt sich um eine NFS RDMA Netzwerkschnittstelle (LIF), die auf einem RoCE fähigen Port gehostet wird.
* Es handelt sich um eine NFS TCP Netzwerkschnittstelle (LIF), die auf einem RoCE fähigen Port gehostet wird.
* Es handelt sich um eine NFS TCP-Netzwerkschnittstelle (LIF), die auf einem Port ohne RoCE-Funktion gehostet wird.


Weitere Informationen zur Migration einer Netzwerkschnittstelle sind unter xref:../networking/migrate_a_lif.html[Ein LIF migrieren] zu finden.
