= Speicherplatz lässt sich durch Komprimierung, Verdichtung und Deduplizierung mit ONTAP System Manager einsparen.
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Bei Volumes auf Nicht-AFF-Clustern können Deduplizierung, Datenkomprimierung und Datenverdichtung zusammen oder unabhängig voneinander ausgeführt werden, um eine optimale Speicherplatzersparnis zu erreichen.

* Deduplizierung entfernt doppelte Datenblöcke.
* Datenkomprimierung komprimiert die Datenblöcke, um die Menge des benötigten physischen Speicherplatzes zu reduzieren.
* Datenverdichtung speichert mehr Daten auf kleinerem Raum, um die Speichereffizienz zu erhöhen.



NOTE: Diese Aufgaben werden für Volumes auf Nicht-AFF-Clustern unterstützt. Alle Inline-Speichereffizienzfunktionen, wie Inline-Deduplizierung und Inline-Komprimierung, sind auf AFF Volumes standardmäßig aktiviert.

.Schritte
. *Speicher > Volumes* auswählen.
. Neben dem Namen des Volumes, für das Speicherplatz eingespart werden soll, auf image:icon_kabob.gif["Symbol für Menüoptionen"] klicken.
. *Bearbeiten* auswählen und zu *Speichereffizienz* scrollen.
. _Optional_: Wenn Sie die Hintergrunddeduplizierung aktivieren möchten, stellen Sie sicher, dass das Kontrollkästchen aktiviert ist.
. _Optional_: Wenn die Hintergrundkomprimierung aktiviert werden soll, ist die Effizienzrichtlinie anzugeben und sicherzustellen, dass das Kontrollkästchen aktiviert ist.
. _Optional_: Wenn Sie die Inline-Komprimierung aktivieren möchten, stellen Sie sicher, dass das Kontrollkästchen aktiviert ist.
