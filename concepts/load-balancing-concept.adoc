= Lastverteilung in ONTAP
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Leistung von Workloads beginnt durch Latenz beeinträchtigt zu werden, wenn die Arbeitslast auf einem Knoten die verfügbaren Ressourcen übersteigt. Ein überlasteter Knoten kann verwaltet werden, indem die verfügbaren Ressourcen erhöht werden (z. B. durch Aufrüsten von Festplatten oder CPU) oder die Last reduziert wird (z. B. durch Verschieben von Volumes oder LUNs auf andere Knoten, falls erforderlich).

Sie können auch ONTAP _storage quality of service (QoS)_ verwenden, um zu gewährleisten, dass die Leistung geschäftskritischer Workloads nicht durch konkurrierende Workloads beeinträchtigt wird:

* Für eine konkurrierende Arbeitslast kann eine QoS-Durchsatzobergrenze festgelegt werden, um deren Auswirkungen auf die Systemressourcen zu begrenzen (QoS Max).
* Es kann ein QoS-Durchsatz-Mindestwert für eine geschäftskritische Arbeitslast festgelegt werden, sodass die Mindestdurchsatzziele unabhängig von der Nachfrage konkurrierender Arbeitslasten erfüllt werden (QoS Min).
* Sie können für dieselbe Arbeitslast eine QoS-Obergrenze und einen QoS-Untergrenze festlegen.




[[throughput-ceilings]]
== Durchsatzgrenzen

Eine Durchsatzbegrenzung schränkt den Durchsatz einer Arbeitslast auf eine maximale Anzahl von IOPS oder MB/s ein. In der folgenden Abbildung stellt die Durchsatzbegrenzung für Arbeitslast 2 sicher, dass sie die Arbeitslasten 1 und 3 nicht „benachteiligt“.

Eine Richtliniengruppe definiert die Durchsatzobergrenze für eine oder mehrere Workloads. Ein Workload repräsentiert die I/O-Operationen für ein Speicherobjekt: ein Volume, eine Datei oder eine LUN oder alle Volumes, Dateien oder LUNs in einer SVM. Die Obergrenze kann beim Erstellen der Richtliniengruppe festgelegt werden oder nach der Überwachung der Workloads.

[NOTE]
====
Der Durchsatz bei Arbeitslasten kann den festgelegten Höchstwert um bis zu 10 Prozent überschreiten, insbesondere wenn eine Arbeitslast schnellen Durchsatzänderungen unterliegt. Zur Bewältigung von Lastspitzen kann der Höchstwert um bis zu 50 Prozent überschritten werden.

====

[[throughput-floors]]
== Durchsatzuntergrenzen

Ein Durchsatzminimum garantiert, dass der Durchsatz einer Arbeitslast nicht unter eine Mindestanzahl von IOPS fällt. In der folgenden Abbildung stellen die Durchsatzminima für Arbeitslast 1 und Arbeitslast 3 sicher, dass sie die Mindestdurchsatzziele unabhängig von der Nachfrage durch Arbeitslast 2 erreichen.

[TIP]
====
Wie die Beispiele zeigen, drosselt eine Durchsatzobergrenze den Durchsatz direkt. Eine Durchsatzuntergrenze drosselt den Durchsatz indirekt, indem den Arbeitslasten Priorität eingeräumt wird, für die die Untergrenze festgelegt wurde.

====
Eine Arbeitslast repräsentiert die E/A-Operationen für ein Volume, eine LUN oder, ab ONTAP 9.3, eine Datei. Eine Richtliniengruppe, die ein Durchsatzminimum definiert, kann nicht auf eine SVM angewendet werden. Das Minimum kann beim Erstellen der Richtliniengruppe festgelegt oder nach der Überwachung der Arbeitslasten angegeben werden.

[NOTE]
====
Der Durchsatz einer Arbeitslast kann unter den festgelegten Mindestwert fallen, wenn die Leistungskapazität (Spielraum) des Knotens oder des Aggregats nicht ausreicht oder während kritischer Operationen wie  `volume move trigger-cutover`. Selbst wenn ausreichend Kapazität vorhanden ist und keine kritischen Operationen stattfinden, kann der Durchsatz einer Arbeitslast um bis zu 5 Prozent unter den festgelegten Mindestwert sinken.

====

[[adaptive-qos]]
== Adaptive QoS

Normalerweise ist der Wert der einem Speicherobjekt zugewiesenen Richtliniengruppe festgelegt. Der Wert muss manuell geändert werden, wenn sich die Größe des Speicherobjekts ändert. Eine Erhöhung des auf einem Volume belegten Speicherplatzes erfordert in der Regel eine entsprechende Erhöhung der für das Volume festgelegten Durchsatzobergrenze.

_Adaptive QoS_ skaliert den Wert der Richtliniengruppe automatisch an die Workloadgröße und hält das Verhältnis von IOPS zu TB/GB bei Änderungen der Workloadgröße konstant. Das ist ein entscheidender Vorteil, wenn Hunderte oder Tausende von Workloads in einer großen Umgebung verwaltet werden.

Adaptives QoS wird üblicherweise zur Anpassung des maximalen Durchsatzes verwendet, kann aber auch zur Verwaltung des minimalen Durchsatzes (bei steigender Arbeitslast) eingesetzt werden. Die Arbeitslastgröße wird entweder als der für das Speicherobjekt zugewiesene Speicherplatz oder als der vom Speicherobjekt belegte Speicherplatz angegeben.


NOTE: Verwendeter Speicherplatz ist für Durchsatz-Floors in ONTAP 9.5 und höher verfügbar. Für Durchsatz-Floors in ONTAP 9.4 und früher wird dies nicht unterstützt.

+ Ab ONTAP 9.13.1 kann adaptives QoS verwendet werden, um Durchsatzunter- und -obergrenzen auf SVM-Ebene festzulegen.

* Eine Richtlinie für zugewiesenen Speicherplatz hält das Verhältnis von IOPS/TB|GB entsprechend der nominalen Größe des Speicherobjekts aufrecht. Beträgt das Verhältnis 100 IOPS/GB, hat ein 150 GB Volume eine Durchsatzobergrenze von 15.000 IOPS, solange das Volume diese Größe beibehält. Wird das Volume auf 300 GB geändert, passt die adaptive QoS die Durchsatzobergrenze auf 30.000 IOPS an.
* Die Richtlinie „Belegter Speicherplatz“ (Standardeinstellung) hält das Verhältnis von IOPS/TB|GB entsprechend der tatsächlich gespeicherten Datenmenge vor Speichereffizienzen aufrecht. Bei einem Verhältnis von 100 IOPS/GB hätte ein 150 GB Volume mit 100 GB gespeicherten Daten eine Durchsatzobergrenze von 10.000 IOPS. Wenn sich die Menge des belegten Speicherplatzes ändert, passt die adaptive QoS die Durchsatzobergrenze entsprechend dem Verhältnis an.
