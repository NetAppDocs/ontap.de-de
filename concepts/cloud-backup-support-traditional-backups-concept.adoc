= Cloud Backup und Unterstützung für traditionelle Backups
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Zusätzlich zu SnapMirror und SnapVault Datensicherungsbeziehungen, die bei ONTAP 9.7 und früheren Versionen nur von Festplatte zu Festplatte möglich waren, gibt es mittlerweile mehrere Backup-Lösungen, die eine kostengünstigere Alternative für die langfristige Datenaufbewahrung bieten.

Zahlreiche Drittanbieter-Datensicherungsanwendungen bieten herkömmliche Backups für ONTAP-verwaltete Daten. Veeam, Veritas und Commvault sowie weitere Anbieter bieten integrierte Backups für ONTAP Systeme.

Ab ONTAP 9.8 bietet SnapMirror cloud asynchrone Replizierung von Snapshots von ONTAP Instanzen zu Objektspeicherendpunkten. Für die SnapMirror cloud Replizierung ist eine lizenzierte Anwendung zur Orchestrierung und Verwaltung von Workflows zum Datenschutz erforderlich. SnapMirror cloud Beziehungen werden von ONTAP Systemen zu ausgewählten lokalen und öffentlichen Cloud-Objektspeicherzielen unterstützt, einschließlich AWS S3, Google Cloud Storage Platform oder Microsoft Azure Blob Storage, was eine höhere Effizienz mit Backup-Software von Anbietern ermöglicht. Eine Liste der unterstützten zertifizierten Anwendungen und Objektspeicheranbieter ist bei Ihrem NetApp Ansprechpartner erhältlich.

Bei Interesse an Cloud-nativem Datenschutz kann die NetApp Console zur Konfiguration von SnapMirror- oder SnapVault-Beziehungen zwischen lokalen Volumes und Cloud Volumes ONTAP Instanzen in der öffentlichen Cloud verwendet werden.

Die Konsole bietet außerdem Backups von Cloud Volumes ONTAP Instanzen mithilfe eines Software as a Service (SaaS) Modells. Benutzer können ihre Cloud Volumes ONTAP Instanzen mithilfe von NetApp Backup and Recovery in S3 und S3-kompatible öffentliche Cloud-Objektspeicher sichern.
