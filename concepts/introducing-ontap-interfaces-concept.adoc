= ONTAP Benutzeroberflächen
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP Datenmanagement-Software bietet mehrere Schnittstellen, mit denen Sie Ihre ONTAP Cluster verwalten können. Diese Schnittstellenoptionen bieten unterschiedliche Zugriffsebenen und Funktionalitäten und geben Ihnen die Flexibilität, Ihre ONTAP Cluster entsprechend Ihrer Umgebung zu verwalten.

Jede dieser Schnittstellen kann zur Verwaltung Ihrer ONTAP Cluster und zur Durchführung von Datenverwaltungsoperationen genutzt werden.



[[ontap-system-manager]]
== ONTAP System Manager

ONTAP System Manager ist eine webbasierte Schnittstelle, die eine vereinfachte und intuitive Möglichkeit zur Verwaltung Ihres Clusters bietet. Es lassen sich gängige Vorgänge wie Speicherkonfiguration, Datenschutz sowie Netzwerkeinrichtung und -verwaltung administrieren. System Manager bietet zudem Risiko- und Clusterleistungsüberwachung sowie Einblicke, die dabei unterstützen, auf Clusterprobleme zu reagieren und Problemen vorzubeugen, bevor sie auftreten. link:../concept_administration_overview.html["Mehr erfahren"].

ONTAP 9.7 markierte einen wichtigen Wendepunkt für ONTAP System Manager. In dieser Version hat NetApp zwei Versionen des ONTAP System Manager bereitgestellt: eine überarbeitete, schlankere und intuitiv Version sowie die Vorgängerversion, die vor ONTAP 9.7 existierte. Nach ONTAP 9.7 wurde die überarbeitete Version als ONTAP System Manager weitergeführt, während die Vorgängerversion in System Manager Classic umbenannt wurde. System Manager Classic wurde zuletzt in ONTAP 9.7 aktualisiert. Wenn Sie System Manager Classic verwenden, ist die zugehörige Dokumentation verfügbar https://docs.netapp.com/us-en/ontap-system-manager-classic/index.html["separat"^].



[[netapp-console]]
== NetApp Console

Ab ONTAP 9.12.1 kann die NetApp Console webbasierte Schnittstelle verwendet werden, um die hybride Multicloud-Infrastruktur von einer zentralen Steuerungsebene aus zu verwalten, während das vertraute System Manager Dashboard erhalten bleibt. Die Console ermöglicht das Erstellen und Verwalten von Cloud-Speicher (zum Beispiel NetApp Backup and Recovery), die Nutzung von NetApp Datendiensten (zum Beispiel Cloud Backup) sowie die Steuerung vieler lokaler und Edge-Speichergeräte. Das Hinzufügen von lokalen ONTAP Systemen zur Console ermöglicht die Verwaltung aller Speicher- und Datenressourcen über eine einzige Schnittstelle. https://docs.netapp.com/us-en/console-family/["Mehr erfahren"^].



[[ontap-command-line-interface]]
== ONTAP Befehlszeilenschnittstelle

Die link:../system-admin/index.html["ONTAP Befehlszeilenschnittstelle (CLI)"] ist eine textbasierte Schnittstelle, die die Interaktion mit einem Cluster, Knoten, SVM und mehr mithilfe von link:../concepts/manual-pages.html["Befehle"] ermöglicht. CLI-Befehle stehen basierend auf link:../system-admin/cluster-svm-administrators-concept.html["Rollentyp"] zur Verfügung. Auf die ONTAP CLI kann über SSH oder eine Konsolenverbindung zu einem Knoten im Cluster zugegriffen werden.



[[ontap-rest-api]]
== ONTAP REST API

Ab ONTAP 9.6 steht eine RESTful-API zur Verfügung, mit der Clusteroperationen programmatisch verwaltet und automatisiert werden können. Die API ermöglicht verschiedene ONTAP administrative Aufgaben, wie das Erstellen und Verwalten von Volumes, Snapshots und Aggregaten sowie die Überwachung der Clusterleistung. Der Zugriff auf die ONTAP REST-API ist direkt möglich, zum Beispiel mit einem Tool wie curl oder mit jeder Programmiersprache, die einen REST-Client unterstützt, wie Python, PowerShell und Java. https://docs.netapp.com/us-en/ontap-automation/get-started/ontap_automation_options.html["Mehr erfahren"^]


NOTE: ONTAPI ist eine proprietäre ONTAP API, die der ONTAP REST API vorausging. Wenn Sie ONTAPI verwenden, sollte die Planung Ihrer https://docs.netapp.com/us-en/ontap-automation/migrate/ontapi_disablement.html["Migration zur ONTAP REST API"^] berücksichtigt werden.



[[netapp-toolkits-and-frameworks]]
== NetApp Toolkits und Frameworks

NetApp stellt Client-Toolkits für spezifische Entwicklungssprachen und -umgebungen bereit, die die ONTAP REST API abstrahieren und die Erstellung von Automatisierungscode erleichtern. https://docs.netapp.com/us-en/ontap-automation/get-started/ontap_automation_options.html#client-software-toolkits["Mehr erfahren"^].

Zusätzlich zu diesen Toolkits ist es möglich, mithilfe von Frameworks Automatisierungscode zu erstellen und bereitzustellen https://docs.netapp.com/us-en/ontap-automation/get-started/ontap_automation_options.html#automation-frameworks["Mehr erfahren"^].
