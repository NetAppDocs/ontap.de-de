= Volumes, qtrees, Dateien und LUNs
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP stellt Daten für Clients und Hosts aus logischen Containern bereit, die als FlexVol Volumes bezeichnet werden. Da diese Volumes nur lose mit dem sie enthaltenden Datenaggregat gekoppelt sind, bieten sie eine größere Flexibilität bei der Verwaltung von Daten als herkömmliche Volumes.

Sie können einem Aggregat mehrere FlexVol volumes zuweisen, von denen jedes für eine andere Applikation oder einen anderen Service dediziert ist. Sie können ein FlexVol volume vergrößern und verkleinern, ein FlexVol volume verschieben und effiziente Kopien eines FlexVol volume erstellen. Sie können _qtrees_ verwenden, um ein FlexVol volume in überschaubarere Einheiten zu partitionieren, und _quotas_, um die Ressourcennutzung des Volumes zu begrenzen.

Volumes enthalten Dateisysteme in einer NAS-Umgebung und LUNs in einer SAN-Umgebung. Eine LUN (Logical Unit Number) ist ein Bezeichner für ein Gerät, das als _logische Einheit_ bezeichnet wird und von einem SAN-Protokoll adressiert wird.

LUNs sind die grundlegende Speichereinheit in einer SAN-Konfiguration. Der Windows Host erkennt LUNs auf Ihrem Speichersystem als virtuelle Festplatten. LUNs können bei Bedarf unterbrechungsfrei auf andere Volumes verschoben werden.

Neben den Datenvolumina gibt es einige spezielle Volumes, die bekannt sein sollten:

* Ein _Node Root Volume_ (typischerweise "`vol0`") enthält Knotenkonfigurationsinformationen und Protokolle.
* Ein _SVM Root Volume_ dient als Einstiegspunkt in den vom SVM bereitgestellten Namespace und enthält Namespace-Verzeichnisinformationen.
* _Systemvolumes_ enthalten spezielle Metadaten wie Service-Audit-Protokolle.


Diese Volumes können nicht zur Datenspeicherung verwendet werden.

*_FlexGroup Volumes_*

In manchen Unternehmen kann ein einzelner Namensraum Petabytes an Speicherplatz benötigen, was sogar die Kapazität eines FlexVol Volumes von 100 TB weit übersteigt.

Ein _FlexGroup volume_ unterstützt bis zu 400 Milliarden Dateien mit 200 zugehörigen Mitgliedsvolumes, die zusammenarbeiten, um die Last- und Speicherplatzverteilung dynamisch und gleichmäßig auf alle Mitglieder zu verteilen.

Für ein FlexGroup Volume fallen keine Wartungs- oder Verwaltungskosten an. Das FlexGroup Volume wird erstellt und für die NAS-Clients freigegeben. ONTAP erledigt den Rest.
