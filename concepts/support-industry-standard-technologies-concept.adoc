= Unterstützung für branchenübliche Netzwerktechnologien
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP unterstützt alle gängigen branchenüblichen Netzwerktechnologien. Zu den wichtigsten Technologien gehören IPspaces, DNS-Lastverteilung und SNMP-Traps.

Broadcast-Domänen, Failover-Gruppen und Subnetze sind in xref:nas-path-failover-concept.adoc[NAS Pfad-Failover] beschrieben.



[[ipspaces]]
== IPspaces

Sie können einen _IPspace_ verwenden, um für jeden virtuellen Datenserver in einem Cluster einen separaten IP-Adressraum zu erstellen. Dadurch können Clients in administrativ getrennten Netzwerkdomänen auf Clusterdaten zugreifen, während sie sich überschneidende IP-Adressen aus demselben IP-Adress-Subnetzbereich verwenden.

Ein Dienstanbieter könnte beispielsweise unterschiedliche IPspaces für Mandanten konfigurieren, die dieselben IP-Adressen verwenden, um auf einen Cluster zuzugreifen.



[[dns-load-balancing]]
== DNS Load Balancing

Sie können _DNS Load Balancing_ verwenden, um den Netzwerkverkehr der Benutzer auf die verfügbaren Ports zu verteilen. Ein DNS-Server wählt dynamisch eine Netzwerkschnittstelle für den Datenverkehr aus, basierend auf der Anzahl der an diese Schnittstelle angeschlossenen Clients.



[[snmp-traps]]
== SNMP-Traps

Sie können _SNMP-Traps_ verwenden, um regelmäßig auf Betriebsschwellenwerte oder Ausfälle zu prüfen. SNMP-Traps erfassen Systemüberwachungsinformationen, die asynchron von einem SNMP-Agenten an einen SNMP-Manager gesendet werden.



[[fips-compliance]]
== FIPS-Konformität

ONTAP erfüllt die Anforderungen des Federal Information Processing Standards (FIPS) 140-2 für alle SSL-Verbindungen. Der SSL-FIPS-Modus kann aktiviert und deaktiviert, SSL-Protokolle global festgelegt und schwache Verschlüsselungsverfahren wie RC4 deaktiviert werden.
