= Informationen zur Speichereffizienz des dedizierten ONTAP Offload-Prozessors
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Speichereffizienz des dedizierten Offload-Prozessors führt die sequenzielle Packung angrenzender physischer Blöcke durch und nutzt einen dedizierten Offload-Prozessor für die 32k Datenkomprimierung. Mit der 32k-Komprimierung wird die Speicherplatzeinsparung auf Aggregatebene erzielt, im Gegensatz zu Plattformen mit adaptiver 8k-Komprimierung, bei denen die Speicherplatzeinsparung auf Volumenebene erfolgt. Plattformen, die die Speichereffizienz des dedizierten Offload-Prozessors verwenden, nutzen keine temperaturabhängige Speichereffizienz (TSSE), das bedeutet, die Komprimierung basiert nicht auf heißen oder kalten Daten. Dadurch ergeben sich sofortige Speicherplatzeinsparungen ohne Leistungseinbußen.

Die dedizierte Offload-Prozessor-Speichereffizienz ist auf den folgenden Plattformen und ONTAP Versionen standardmäßig aktiviert.


NOTE: AFF A20 Plattform unterstützt keine dedizierte Storage-Effizienz durch Offload-Prozessor. Für AFF A20 Plattformen werden mit SnapMirror Technologie migrierte Daten automatisch mithilfe der Haupt-CPU in 32k Inline-Komprimierung umgewandelt.

* AFF C80
* AFF C60
* AFF C30
* AFF A90
* AFF A70
* AFF A50
* AFF A30
* AFF A1K
* FAS90
* FAS70


Für die folgenden Plattformen ist die Speichereffizienz automatisch aktiviert und erfordert keine Konfiguration. Dies gilt für alle neu erstellten Thin-Provisioned Volumes und vorhandene Daten, einschließlich Volumes, die von anderen Plattformen verschoben wurden. Die Deduplizierung ist unabhängig von der Einstellung der Speicherplatzzusage aktiviert. Komprimierung und Datenverdichtung setzen voraus, dass die Speicherplatzzusage auf Keine gesetzt ist. Daten, die mithilfe von Volume Move oder der SnapMirror Technologie migriert werden, werden automatisch in 32k Inline-Komprimierung konvertiert.

* AFF C80
* AFF C60
* AFF C30
* AFF A1K
* AFF A90
* AFF A70
* AFF A50
* AFF A30


Für die folgenden Plattformen ist die Speichereffizienz standardmäßig nur für bestehende Thin-Provisioned Volumes aktiviert, bei denen die Speichereffizienz vor dem Upgrade auf ONTAP 9.15.1 oder später aktiviert war. Die Speichereffizienz kann für neu erstellte Volumes mithilfe der ONTAP CLI oder der ONTAP REST API Methode aktiviert werden. Daten, die mithilfe von Volume Move oder SnapMirror Technologie migriert werden, werden nur dann in 32k Inline-Komprimierung konvertiert, wenn die Komprimierung auf der ursprünglichen Plattform aktiviert war.

* FAS90
* FAS70


Informationen zum Upgrade eines Controllers auf eine der unterstützten Plattformen sind unter  https://docs.netapp.com/us-en/ontap-systems-upgrade/choose_controller_upgrade_procedure.html["ONTAP Hardware Upgrade Dokumentation"^] zu finden.
