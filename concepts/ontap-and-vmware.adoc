Sie können ONTAP und verwandte NetApp Produkte mit VMware vSphere integrieren. Je nach Technologieumgebung und Geschäftsanforderungen stehen verschiedene Optionen zur Verfügung.



[[selected-concepts-and-terminology]]
== Ausgewählte Konzepte und Terminologie

Zu Beginn der Nutzung von ONTAP und verwandten NetApp Produkten in einer VMware-Umgebung ist es hilfreich, zunächst mit einigen der wichtigsten Begriffe und Konzepte vertraut zu sein.

Logische Einheitennummer:: Eine LUN ist eine Nummer, die zur Identifizierung einer _logischen Einheit_ innerhalb eines Storage Area Network (SAN) verwendet wird. Diese adressierbaren Geräte sind typischerweise logische Datenträger, auf die über das Small Computer System Interface (SCSI)-Protokoll oder eine seiner abgeleiteten Varianten zugegriffen wird.
VMware vSphere virtuelles Volumen:: Ein virtuelles Volume (vVol) bietet eine Abstraktion auf Volume-Ebene für den von einer virtuellen Maschine genutzten Speicher. Es bietet mehrere Vorteile und stellt eine Alternative zur Verwendung einer herkömmlichen LUN dar.
persistente Reservierungen:: Persistente Reservierungen werden mit SCSI-3 unterstützt und stellen eine Verbesserung gegenüber den früheren SCSI-2-Reservierungen dar. Sie ermöglichen mehreren Client-Initiatoren die Kommunikation mit einem einzelnen Ziel, während andere Knoten ausgeschlossen werden. Die Reservierungen bleiben auch dann bestehen, wenn der Bus zur Fehlerbehebung zurückgesetzt wird.



NOTE: Ab ONTAP 9.15.1 kann eine persistente Reservierung für ein virtuelles Volume mithilfe von SCSI-3 erstellt werden. Diese Funktion wird nur mit ONTAP Tools für VMware vSphere 9 in Verbindung mit einem Windows Server Failover Cluster (WSFC) unterstützt.

Windows Server Failover Clustering:: Microsoft WSFC ist eine Funktion des Windows Server-Betriebssystems, die Fehlertoleranz und Hochverfügbarkeit bietet. Mehrere Serverknoten (physisch oder virtuell) werden zu einem Cluster zusammengefasst, um im Fehlerfall Resilienz zu gewährleisten. WSFC wird häufig zur Bereitstellung von Infrastrukturdiensten wie Datenbank-, Datei- und Namensraumservern verwendet.
VMware vSphere Storage APIs – Storage Awareness:: VASA ist eine Sammlung von APIs, die die Integration der Storage-Systeme mit vCenter für Management und Administration ermöglichen. Die Architektur basiert auf mehreren Komponenten, einschließlich des _VASA Provider_, der die Kommunikation zwischen VMware vSphere und den Storage-Systemen übernimmt. Mit ONTAP ist der Provider als Teil der ONTAP tools for VMware vSphere implementiert.
VMware vSphere Storage APIs – Array Integration:: VAAI ist eine Sammlung von APIs, die die Kommunikation zwischen VMware vSphere ESXi Hosts und den Speichergeräten ermöglicht. Die API umfasst eine Reihe von Basisoperationen, die von den Hosts verwendet werden, um Speicheroperationen auf das Array auszulagern. VAAI kann die Leistung bei speicherintensiven Aufgaben deutlich verbessern.
NetApp SnapCenter:: SnapCenter ist eine zentrale und skalierbare Plattform, die Datenschutz für Anwendungen, Datenbanken, Host-Dateisysteme und virtuelle Maschinen unter Verwendung von ONTAP-Speichersystemen bietet. Dabei werden die nativen ONTAP-Technologien einschließlich Snapshot, SnapRestore, FlexClone, SnapMirror und SnapVault genutzt.




[[netapp-plug-ins-and-related-technologies]]
== NetApp Plug-ins und verwandte Technologien

NetApp bietet umfassende Unterstützung für die Integration von ONTAP und verwandten Produkten mit VMware vSphere Technologien.



[[ontap-tools-for-vmware-vsphere]]
=== ONTAP Tools for VMware vSphere

ONTAP tools für VMware vSphere ist eine Sammlung von Tools zur Integration von ONTAP und vSphere. Es implementiert die Provider-Funktionalität des VASA-API-Frameworks. ONTAP tools umfasst außerdem das vCenter Plug-in, einen Storage Replication Adapter (SRA) für VMware Site Recovery Manager sowie einen REST-API-Server, der für die Entwicklung von Automatisierungsanwendungen verwendet werden kann.



[[nfs-plug-in-for-vmware-vaai]]
=== NFS Plug-In für VMware VAAI

Das NetApp NFS Plug-In für VMware VAAI ermöglicht den Zugriff auf die VAAI-Funktionen. Das Plug-In kann auf ESXi-Hosts installiert werden und ermöglicht es diesen, VAAI mit den NFS-Datenspeichern auf ONTAP zu nutzen. Es bietet mehrere Funktionen, darunter Klonen, Speicherplatzreservierungen und Snapshot-Auslagerung.



[[vmware-live-site-recovery]]
=== VMware Live Site Recovery

VMware Live Site Recovery, ehemals Site Recovery Manager (SRM), bietet eine Notfallwiederherstellungsfunktion. Live Site Recovery integriert sich mit ONTAP tools for VMware vSphere, um auf die ONTAP Datenverwaltungsfunktionen zuzugreifen und diese zu nutzen.



[[vsphere-metro-storage-cluster]]
=== vSphere Metro Storage Cluster

vSphere Metro Storage Cluster (vMSC) ist eine Technologie, die vSphere in einer _stretched cluster_-Bereitstellung ermöglicht und unterstützt. vMSC Lösungen werden mit NetApp MetroCluster und SnapMirror active sync (ehemals SMBC) unterstützt. Diese Lösungen bieten eine verbesserte Geschäftskontinuität im Falle eines Domänenausfalls. Das Resilienzmodell basiert auf Ihren spezifischen Konfigurationsentscheidungen.



[[snapcenter-plug-in-for-vmware-vsphere]]
=== SnapCenter Plug-in for VMware vSphere

Das SnapCenter Plug-in für VMware vSphere (SCV) ist eine Linux-basierte virtuelle Appliance, die zusammen mit dem SnapCenter Server oder als eigenständige Anwendung bereitgestellt werden kann. In beiden Fällen stellt SCV Backup- und Wiederherstellungsfunktionen für VMs, Datenspeicher und VMDKs bereit. Die Vorgänge sind schnell, platzsparend, absturzkonsistent und VM-konsistent.
