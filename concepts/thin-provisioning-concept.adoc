= Thin Provisioning
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP bietet neben Snapshots eine breite Palette an Technologien zur Steigerung der Speichereffizienz. Zu den wichtigsten Technologien gehören Thin Provisioning, Deduplizierung, Komprimierung sowie FlexClone Volumes, Dateien und LUNs. Wie Snapshots basieren auch diese alle auf dem Write Anywhere File Layout (WAFL) von ONTAP.

Ein _thin-provisioned_ Volume oder eine LUN ist ein Volume, für das Speicher nicht im Voraus reserviert wird. Stattdessen wird Speicher dynamisch zugewiesen, sobald er benötigt wird. Freier Speicher wird an das Speichersystem zurückgegeben, wenn Daten im Volume oder in der LUN gelöscht werden.

Angenommen, Ihre Organisation muss 5.000 Benutzern Speicherplatz für ihre Home-Verzeichnisse bereitstellen. Es wird geschätzt, dass die größten Home-Verzeichnisse 1 GB Speicherplatz belegen.

In diesem Fall könnten Sie 5 TB physischen Speicher erwerben. Für jedes Volume, das ein Home-Verzeichnis speichert, würde ausreichend Speicher reserviert, um den Bedarf der größten Nutzer zu decken.

In der Praxis wissen Sie aber auch, dass der Kapazitätsbedarf für Home-Verzeichnisse innerhalb Ihrer Community stark variiert. Auf jeden Nutzer mit hohem Speicherbedarf kommen zehn, die wenig oder gar keinen Speicherplatz benötigen.

Thin Provisioning ermöglicht es, den Bedarf großer Speichernutzer zu decken, ohne Speicherplatz kaufen zu müssen, der möglicherweise nie genutzt wird. Da Speicherplatz erst bei Verbrauch zugewiesen wird, kann ein Datenaggregat von 2 TB „überprovisioniert“ werden, indem jedem der 5.000 Volumes im Aggregat nominell eine Größe von 1 GB zugewiesen wird.

Solange Ihre Annahme zutrifft, dass das Verhältnis von leichten zu starken Benutzern 10:1 beträgt, und solange Sie aktiv den freien Speicherplatz auf dem Aggregat überwachen, können Sie sicher sein, dass Volume-Schreibvorgänge nicht aufgrund von Speicherplatzmangel fehlschlagen.
