= Deduplizierung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Deduplizierung reduziert den benötigten physischen Speicherplatz für ein Volume (oder alle Volumes in einem AFF Aggregat), indem doppelte Blöcke verworfen und durch Verweise auf einen einzigen gemeinsam genutzten Block ersetzt werden. Lesevorgänge deduplizierter Daten verursachen in der Regel keine Leistungseinbußen. Das Schreiben verursacht nur eine vernachlässigbare Belastung, außer auf überlasteten Knoten.

Da während der normalen Nutzung Daten geschrieben werden, verwendet WAFL einen Batch-Prozess, um einen Katalog von _Block-Signaturen_ zu erstellen. Nach dem Start der Deduplizierung vergleicht ONTAP die Signaturen im Katalog, um doppelte Blöcke zu identifizieren. Wenn eine Übereinstimmung vorliegt, wird ein Byte-für-Byte-Vergleich durchgeführt, um zu überprüfen, ob sich die potenziellen Blöcke seit der Erstellung des Katalogs nicht geändert haben. Nur wenn alle Bytes übereinstimmen, wird der doppelte Block verworfen und sein Speicherplatz freigegeben.
