= Root-Datenpartitionierung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Jeder Knoten muss ein Root-Aggregat für die Konfigurationsdateien des Speichersystems haben. Das Root-Aggregat hat den RAID-Typ des Datenaggregats.

System Manager unterstützt keine Root-Data oder Root-Data-Data Partitionierung.

Ein Root-Aggregat vom Typ RAID-DP besteht typischerweise aus einer Datenfestplatte und zwei Paritätsfestplatten. Das ist eine erhebliche „Paritätsteuer“, die für Systemdateien des Speichersystems anfällt, wenn das System bereits zwei Festplatten als Paritätsfestplatten für jede RAID-Gruppe im Aggregat reserviert.

_Root-Datenpartitionierung_ reduziert die Paritätsteuer, indem das Root-Aggregat auf Festplattenpartitionen aufgeteilt wird, wobei auf jeder Festplatte eine kleine Partition als Root-Partition und eine große Partition für Daten reserviert wird.

image:root-data.gif["Beispiel für die Root-Daten-Partitionierung"]

Wie die Abbildung zeigt, gilt: Je mehr Festplatten für die Speicherung des Root-Aggregats verwendet werden, desto kleiner ist die Root-Partition. Dies ist auch bei einer Form der Root-Daten-Partitionierung namens _root-data-data partitioning_ der Fall, bei der eine kleine Partition als Root-Partition und zwei größere, gleich große Partitionen für Daten erstellt werden.

image:root-data-data.gif["Beispiel für die Partitionierung von Root-Daten und Daten"]

Beide Arten der Root-Daten-Partitionierung sind Teil der ONTAP Funktion _Advanced Drive Partitioning (ADP)_. Beide werden werkseitig konfiguriert: Root-Daten-Partitionierung für FAS2xxx-, FAS9000-, FAS8200-, FAS80xx- und AFF Einstiegssysteme, Root-Daten-Daten-Partitionierung nur für AFF Systeme.

Weitere Informationen zu link:https://kb.netapp.com/Advice_and_Troubleshooting/Data_Storage_Software/ONTAP_OS/What_are_the_rules_for_Advanced_Disk_Partitioning["Erweiterte Laufwerkspartitionierung"^].



[[drives-partitioned-and-used-for-the-root-aggregate]]
== Partitionierte Laufwerke, die für das Root-Aggregat verwendet werden

Die Laufwerke, die für die Verwendung im Root-Aggregat partitioniert sind, hängen von der Systemkonfiguration ab.

Die Kenntnis der Anzahl der für das Root-Aggregat verwendeten Laufwerke hilft dabei zu bestimmen, wie viel der Laufwerkskapazität für die Root-Partition reserviert ist und wie viel für die Verwendung in einem Datenaggregat zur Verfügung steht.

Die Möglichkeit zur Partitionierung der Root-Daten wird für Einstiegsplattformen, All Flash FAS Plattformen und FAS Plattformen mit ausschließlich angeschlossenen SSDs unterstützt.

Bei Einsteigerplattformen werden nur die internen Laufwerke partitioniert.

Bei All Flash FAS Plattformen und FAS Plattformen mit ausschließlich SSDs werden alle beim Systemstart an den Controller angeschlossenen Laufwerke partitioniert, bis zu einem Limit von 24 pro Knoten. Laufwerke, die nach der Systemkonfiguration hinzugefügt werden, werden nicht partitioniert.
