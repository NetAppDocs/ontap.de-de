= RDMA-Übersicht
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die Remote Direct Memory Access (RDMA) Angebote von ONTAP unterstützen latenztkritische und mit einer hohen Bandbreite Workloads. RDMA ermöglicht das direkte Kopieren von Daten zwischen dem Speicher des Speichersystems und dem Speicher des Hostsystems, wodurch CPU-Unterbrechungen und Overhead umgangen werden.



[[nfs-over-rdma]]
== NFS über RDMA

Ab ONTAP 9.10.1 kann link:../nfs-rdma/index.html["NFS über RDMA"] so konfiguriert werden, dass die Verwendung von NVIDIA GPUDirect Storage für GPU-beschleunigte Workloads auf Hosts mit unterstützten NVIDIA GPUs möglich ist.


NOTE: RDMA wird mit dem SMB-Protokoll nicht unterstützt.



[[cluster-interconnect-rdma]]
== Cluster Interconnect RDMA

Cluster Interconnect RDMA reduziert die Latenz, verkürzt die Failover-Zeiten und beschleunigt die Kommunikation zwischen den Nodes in einem Cluster.

Ab ONTAP 9.10.1 wird Cluster Interconnect RDMA für bestimmte Hardware-Systeme in Verbindung mit X1151A Cluster NICs unterstützt.
