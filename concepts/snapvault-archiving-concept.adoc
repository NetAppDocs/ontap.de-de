= SnapVault Archivierung
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Die SnapMirror Lizenz wird sowohl zur Unterstützung von SnapVault Beziehungen für Backup als auch von SnapMirror Beziehungen für Disaster Recovery verwendet. Ab ONTAP 9.3 sind SnapVault Lizenzen veraltet, und SnapMirror Lizenzen können zur Konfiguration von Vault-, Mirror- und Mirror-and-Vault-Beziehungen verwendet werden. SnapMirror Replikation wird für ONTAP-to-ONTAP Replikation von Snapshots verwendet und unterstützt sowohl Backup- als auch Disaster-Recovery-Anwendungsfälle.

SnapVault ist eine Archivierungstechnologie, die für die Festplatten-zu-Festplatten-Snapshot-Replikation zur Einhaltung von Standards und für andere Governance-bezogene Zwecke entwickelt wurde. Im Gegensatz zu einer SnapMirror Beziehung, bei der das Ziel in der Regel nur die aktuell im Quell-Volume vorhandenen Snapshots enthält, bewahrt ein SnapVault Ziel typischerweise zeitpunktgenaue Snapshots auf, die über einen deutlich längeren Zeitraum erstellt wurden.

Beispielsweise könnten Sie monatliche Snapshots Ihrer Daten über einen Zeitraum von 20 Jahren aufbewahren, um die gesetzlichen Buchführungsvorschriften für Ihr Unternehmen einzuhalten. Da keine Anforderung besteht, Daten aus dem Vault-Speicher bereitzustellen, können auf dem Zielsystem langsamere, kostengünstigere Festplatten verwendet werden.

Wie bei SnapMirror, SnapVault führt beim ersten Aufruf einen Basistransfer durch. Es wird ein Snapshot des Quell-Volumes erstellt, anschließend werden die Kopie und die von ihr referenzierten Datenblöcke auf das Ziel-Volume übertragen. Im Gegensatz zu SnapMirror, SnapVault sind ältere Snapshots nicht im Basistransfer enthalten.

Aktualisierungen erfolgen asynchron gemäß dem von Ihnen konfigurierten Zeitplan. Die in der Richtlinie für die SnapMirror Beziehung definierten Regeln legen fest, welche neuen Snapshots in Aktualisierungen einbezogen werden und wie viele Kopien beibehalten werden. Die in der Richtlinie definierten Labels („monthly“ zum Beispiel) müssen mit einem oder mehreren Labels übereinstimmen, die in der Snapshot-Richtlinie auf der Quelle definiert sind. Andernfalls schlägt die Replikation fehl.


NOTE: SnapMirror und SnapVault teilen sich dieselbe Befehlsinfrastruktur. Die gewünschte Methode wird bei der Erstellung einer Richtlinie festgelegt. Beide Methoden erfordern peered Cluster und peered SVMs.
