= SnapMirror Cloud-Backups in Objektspeicher
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SnapMirror Cloud ist eine Backup und Recovery-Technologie für ONTAP Nutzer, die ihre Datensicherungsprozesse in die Cloud oder auf eine lokale Objektspeicherlösung verlagern möchten. Organisationen, die von herkömmlichen Backup-to-Tape-Architekturen abrücken, können Objektspeicher als alternatives Repository für die langfristige Datenaufbewahrung und Archivierung verwenden. SnapMirror Cloud bietet ONTAP zu Objektspeicher Replikation als Teil einer inkrementellen, kontinuierlichen Backup-Strategie.

SnapMirror Cloud Replikation ist eine lizenzpflichtige ONTAP Funktion. SnapMirror Cloud wurde in ONTAP 9.8 als Erweiterung der Familie von SnapMirror Replikationstechnologien eingeführt. Während SnapMirror häufig für ONTAP-zu-ONTAP Backups verwendet wird, nutzt SnapMirror Cloud dieselbe Replikations-Engine, um Snapshots für ONTAP Backups in S3-kompatible Objektspeicher zu übertragen.

Für Backup-Anwendungsfälle konzipiert, unterstützt SnapMirror Cloud sowohl Langzeitaufbewahrung als auch Archivierungs-Workflows. Wie bei SnapMirror führt das initiale SnapMirror Cloud Backup einen Basistransfer eines Volumes durch. Bei nachfolgenden Backups erstellt SnapMirror Cloud einen Snapshot des Quell-Volumes und überträgt den Snapshot mit nur den geänderten Datenblöcken an ein Objektspeicherziel.

SnapMirror Cloud-Beziehungen können zwischen ONTAP-Systemen und ausgewählten lokalen und öffentlichen Cloud-Objektspeicherzielen konfiguriert werden, einschließlich Amazon S3, Google Cloud Storage und Microsoft Azure Blob Storage. Weitere lokale Objektspeicherziele sind StorageGRID und ONTAP S3.

Zusätzlich zur Verwendung von ONTAP System Manager zur Verwaltung von SnapMirror Cloud-Konfigurationen stehen verschiedene Orchestrierungsoptionen für die Verwaltung von SnapMirror Cloud-Backups zur Verfügung:

* Mehrere Drittanbieter-Partner für Backup bieten Unterstützung für SnapMirror Cloud-Replikation. Teilnehmende Anbieter sind auf der link:https://www.netapp.com/blog/new-backup-architecture-snapdiff-v3/["NetApp Blog"^] verfügbar.
* NetApp Backup und Recovery für eine NetApp-native Lösung für ONTAP Umgebungen
* APIs zum entwickeln individueller Software für Datensicherungs-Workflows oder zur Nutzung von Automatisierungs-Tools



NOTE: SnapMirror Cloud-Replikation, die von System Manager verwaltet wird, unterstützt ausschließlich StorageGrid und ONTAP S3 als Objektspeicherziel. Andere Anbieter werden nicht unterstützt.


NOTE: Die Nutzung der SnapMirror Cloud-APIs ist grundsätzlich auf Backup-Partner beschränkt. Kunden, die eigene API-basierte Lösungen entwickeln möchten, erhalten Informationen zu den Lizenzbedingungen über ihr NetApp Account-Team.
